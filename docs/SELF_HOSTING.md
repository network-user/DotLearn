# Self-hosting

Two ways to run DotLearn on your own server:

- **Bare-metal (recommended for a small server).** No Docker. Caddy serves
  the static site with automatic HTTPS and reverse-proxies the API; the API
  runs under systemd. Lowest RAM and disk footprint.
- **Docker.** `docker compose up` for an isolated stack. See the bottom of
  this page.

The frontend is local-first and works without the backend. The API only
powers the topic-submission form (`/submit`) and the admin panel. Fuzzy
search for submissions can optionally use Elasticsearch, but the default is
an in-memory implementation (enough for dozens of submissions, zero extra
RAM/disk).

---

## Bare-metal deploy (Debian / Ubuntu)

### Prerequisites

- A Debian or Ubuntu server with root/sudo.
- A domain name with an **A/AAAA record pointing at the server's IP** (the
  certificate cannot be issued otherwise).
- Ports **80** and **443** reachable from the internet (no other web server
  occupying them).

### One command

```bash
git clone <repo-url> /opt/dotlearn
cd /opt/dotlearn
sudo bash scripts/deploy.sh
```

The script asks for three things on the first run:

1. **Domain** (e.g. `learn.example.com`)
2. **Email** for Let's Encrypt
3. **Admin password** (min 8 chars)

…then it does everything else:

- installs Node 20, pnpm (corepack) and Caddy if missing;
- creates an unprivileged `dotlearn` service user;
- writes `.env` and generates the admin secrets (password hash, TOTP secret,
  backup codes, JWT secrets); **the QR code and backup codes are printed
  once, save them**;
- builds the frontend and API;
- assembles a production-only API bundle and copies the static site out;
- installs and starts the `dotlearn-api` systemd service and Caddy;
- **reclaims disk**: removes build dependencies and prunes the pnpm store.

When it finishes:

- Site: `https://<domain>`
- Admin: `https://<domain>/<VITE_ADMIN_PATH>` (default `/admin`; change it,
  see below)

Re-running `sudo bash scripts/deploy.sh` rebuilds and restarts without
re-asking (it keeps the existing `.env` and secrets). Use it to apply code
changes, or `make update` to `git pull` first.

### Where things live

| Path                                       | What                                     |
| ------------------------------------------ | ---------------------------------------- |
| `/opt/dotlearn/api`                        | Production API bundle (run by systemd)   |
| `/var/www/dotlearn`                        | Static site (served by Caddy)            |
| `/var/lib/dotlearn/data`                   | API data (submissions, hidden-topics)    |
| `/etc/dotlearn/dotlearn.env`               | Single config + secrets file (chmod 600) |
| `/etc/systemd/system/dotlearn-api.service` | API service unit                         |
| `/etc/caddy/Caddyfile`                     | Caddy site config                        |

### Day-2 operations

From the repo directory (`make` shortcuts, or the raw commands):

```bash
make status      # systemctl status dotlearn-api caddy
make logs        # journalctl -u dotlearn-api -f
make logs-web    # journalctl -u caddy -f
make restart     # systemctl restart dotlearn-api caddy
make update      # git pull --ff-only && sudo bash scripts/deploy.sh
make backup      # snapshot /var/lib/dotlearn/data to ./backups/
make restore FILE=backups/dotlearn-data-YYYY-MM-DD-HHMMSS.tar.gz
```

## Secrets

Generated by the deploy script. To (re)generate manually:

```bash
cd apps/api
ADMIN_LOGIN=admin node scripts/gen-admin-secrets.mjs '<password>'
```

It prints the `ADMIN_*` lines for `.env` on stdout and the QR code + backup
codes on stderr (shown once).

### Rotation

- **Password / TOTP / backup codes:** rerun the generator, replace the
  `ADMIN_*` values in `.env`, `make restart`. Then open the admin panel and
  hit **"Reset all sessions"** to invalidate refresh tokens issued earlier.
- **`VITE_ADMIN_PATH`:** baked into the frontend at build time. Change it in
  `.env`, then `sudo bash scripts/deploy.sh` to rebuild. The old path returns
  404 afterwards.
- **JWT secrets:** rotating them invalidates all issued tokens; after
  `make restart` everyone must log in again.

## Key `.env` variables

| Variable                                                    | Purpose                                                                                                      |
| ----------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| `DOMAIN`                                                    | Public domain. Drives HTTPS (Let's Encrypt) and CORS.                                                        |
| `ACME_EMAIL`                                                | Contact email for the ACME/Let's Encrypt account.                                                            |
| `VITE_API_BASE`                                             | Where the frontend calls the API. Set to `https://$DOMAIN` (same origin).                                    |
| `VITE_ADMIN_PATH`                                           | Admin route on the frontend. Make it hard to guess (`/secret-admin-7f2c`).                                   |
| `WEB_ORIGIN`                                                | CORS allowlist; must equal `https://$DOMAIN`.                                                                |
| `HOST` / `PORT`                                             | API bind address (loopback `127.0.0.1`) and port (Caddy proxies it).                                         |
| `TRUSTED_PROXY_HOPS`                                        | Reverse proxies in front of the API (Caddy = `1`).                                                           |
| `DATA_DIR`                                                  | API data directory (`/var/lib/dotlearn/data`).                                                               |
| `ADMIN_LOGIN` / `ADMIN_PASSWORD_HASH` / `ADMIN_TOTP_SECRET` | The single admin account.                                                                                    |
| `ADMIN_BACKUP_CODES_HASHED`                                 | JSON array of SHA-256 hashes of the backup codes.                                                            |
| `ADMIN_JWT_SECRET` / `ADMIN_REFRESH_SECRET`                 | Signing keys for access and refresh JWTs (must differ).                                                      |
| `ES_ENABLED`                                                | `false` → in-memory search (default). `true` → Elasticsearch at `ES_NODE`.                                   |
| `PRESENCE_TTL_MS` / `PRESENCE_MAX_TRACKED`                  | Optional online-counter tuning: liveness window (default `90000` ms) and in-memory id cap (default `50000`). |

Other auth timings (`ADMIN_ACCESS_TTL`, `ADMIN_LOCKOUT_TTL`, …) are optional;
sensible defaults live in `apps/api/src/modules/auth/auth.config.ts`.

## Optional: Elasticsearch

Only needed for fuzzy submission search at larger scale. It costs ~1 GB of
RAM (JVM), so skip it on a small server. To enable:

1. Run an Elasticsearch reachable from the server.
2. Set in `.env`: `ES_ENABLED=true` and `ES_NODE=http://127.0.0.1:9200`.
3. `make restart`.

The index is rebuilt automatically from the data files on start.

## Alternative: Docker

```bash
cp .env.example .env
# Fill DOMAIN, ACME_EMAIL, admin secrets (see "Secrets" above), VITE_API_BASE,
# VITE_ADMIN_PATH, WEB_ORIGIN. Leave ES_ENABLED=false.
docker compose up --build -d
```

This brings up `web` (nginx static on `:8080`) and `api` (`:3000`). Bring
your own TLS in front (Caddy/nginx/Cloudflare), or adapt the compose to add a
proxy. Elasticsearch is behind a profile and **off by default**; start it
with `docker compose --profile search up -d` (and set `ES_ENABLED=true`).

Data persists in the named volume `dotlearn-data`. Back it up with:

```bash
docker run --rm -v dotlearn_dotlearn-data:/data -v "$(pwd)":/backup alpine \
  tar czf /backup/dotlearn-data-$(date +%F).tar.gz -C /data .
```
