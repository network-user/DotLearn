# First concept

Hook, one or two paragraphs of prose: the problem this concept solves and what the reader will be able to do at the end. No definitions, no lists, no code yet - the first paragraph receives a drop cap.

## The mental model

Explain the core idea once, properly, and anchor it with the concept's primary figure.

## Worked example

Build incrementally; one new idea per code block, at most ~10 lines of code between prose.

```sql
SELECT id, name FROM example WHERE id = 1;
```

## Try it live

Pose a concrete "change X and watch Y" challenge, then give the reader a live block.

## Edge cases

At least one honest surprise: NULL, empty input, duplicates, ordering guarantees.
