# ООП принципы с примерами для тестов

## Суть

Инкапсуляция прячет состояние за API, наследование переиспользует базовое поведение, полиморфизм подставляет реализации через общий тип, абстракция задаёт контракт без деталей. Вместе они структурируют page objects, клиенты API и фикстуры так, чтобы сценарии читались как бизнес-шаги, а не как набор полей.

## Что ответить на собесе

- Инкапсуляция: локаторы private, наружу `submitOrder()`.
- Наследование: `AuthenticatedPage` extends `BasePage`.
- Полиморфизм: `PaymentClient` real vs wiremock под одним типом.
- Абстракция: `abstract void open()` в базовой странице.
- Честно: если наследник только добавляет один метод, композиция часто чище.

## Как это работает

Инкапсуляция:

```
class CartPage {
  private final By total = By.cssSelector("[data-test=total]");

  Money readTotal(WebDriver driver) {
    String raw = driver.findElement(total).getText();
    return Money.parse(raw);
  }
}
```

Полиморфизм + абстракция:

```
interface SmsGateway { void send(String phone, String code); }

class FakeSmsGateway implements SmsGateway {
  public void send(String phone, String code) { /* store */ }
}

void verifyOtp(SmsGateway gateway) {
  gateway.send("+1", "1234");
}
```

Python mirror:

```python
class CartPage:
    def __init__(self) -> None:
        self._total_selector = "[data-test=total]"

    def read_total(self, driver) -> str:
        return driver.find_element(self._total_selector).text
```

## Подводные камни

**Анемичная модель.** PO только геттеры без поведения, ООП номинальное.

**Наследование ради reuse полей.** Shared protected locators хрупки.

**Полиморфизм без контракт-теста.** Новая реализация проходит compile, падает в CI.

**Абстрактный класс с состоянием.** База с WebDriver полем усложняет parallel.

**Нарушение инкапсуляции в тесте.** Прямой доступ к private через reflection антипаттерн.
