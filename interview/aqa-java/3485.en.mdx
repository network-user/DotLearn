# Set in Java for unique test tags

## In short

A Set stores unique elements by equals/hashCode (HashSet) or compareTo (TreeSet). Duplicate adds are ignored or follow Map-side replace policy. LinkedHashSet keeps insertion order. In asserts I compare `Set.copyOf(apiTags)` to expected without caring about order.

## How to answer in an interview

- HashSet O(1) contains for a «already ran case id» gate.
- TreeSet sorted iteration for stable reports without extra sort.
- LinkedHashSet when order matters but uniqueness too.
- EnumSet for compact feature flag bitmasks.
- Honestly: mutable objects in a Set without equals override fail dedup silently.

## How it works

```
Set<String> tags = new HashSet<>();
tags.add("smoke");
tags.add("smoke");
assert tags.size() == 1;
```

TreeSet:

```
Set<String> ordered = new TreeSet<>(List.of("beta", "alpha"));
```

Python set assert:

```python
def test_unique_tags():
    tags = {"smoke", "regression", "smoke"}
    assert len(tags) == 2
    assert tags == {"smoke", "regression"}
```

## Pitfalls

**Order assert on HashSet.** Flaky nightly runs.

**Custom object hash change.** Element lost after mutating a hashed field.

**Null in HashSet.** One null allowed; TreeSet null rules vary by JDK era.

**Set.of duplicate arg.** IllegalArgumentException at creation.

**ConcurrentHashSet.newKeySet.** Not identical java.util.Set semantics for every op.
