# throw и throws в тестовых методах

## Суть

`throw` создаёт и выбрасывает exception в точке выполнения. `throws` в сигнатуре метода объявляет checked exceptions, которые caller должен обработать или пробросить. В тестах `throw` валидируют негативные ветки, `throws` на `@Test` допустим для IO setup.

## Что ответить на собесе

- Guard: `if (token == null) throw new IllegalStateException("no token")`.
- `throws IOException` на helper читающий fixture с диска.
- `assertThrows` заменяет ручной throw/catch для expected failure.
- Re-throw: `catch (SQLException e) { throw new RuntimeException(e); }` только осознанно.
- Честно: throws на каждом тесте раздувает сигнатуру, лучше try-with-resources в `@BeforeEach`.

## Как это работает

throw:

```
void requireAdult(int age) {
  if (age < 18) {
    throw new IllegalArgumentException("too young");
  }
}
```

throws:

```
void readConfig(Path path) throws IOException {
  Files.readString(path);
}
```

JUnit:

```
assertThrows(IllegalArgumentException.class, () -> requireAdult(15));
```

Python mirror:

```python
def require_adult(age: int) -> None:
    if age < 18:
        raise ValueError("too young")


def test_throw_contract():
    import pytest
    with pytest.raises(ValueError):
        require_adult(15)
```

## Подводные камни

**throws без throw.** Пустое объявление вводит в заблуждение.

**throw checked без throws.** Compile error, нужен wrap или throws.

**throws Exception.** Слишком широко, глотает реальные баги.

**finally throw.** Маскирует assertion failure.

**Error throw.** Не для бизнес-валидации в тестовом helper.
