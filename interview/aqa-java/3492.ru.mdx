# Типы данных Java в автотестах

## Суть

Java делит типы на примитивы (8 штук, значение в стеке) и ссылочные (классы, интерфейсы, массивы, enum, хранятся в heap). Обертки (`Integer`, `Boolean`) нужны для generics и nullable полей в test data. Выбор типа влияет на сравнение, сериализацию фикстур и boxing overhead.

## Что ответить на собесе

- `int`/`long` для счётчиков и id в assert, `BigDecimal` для денег вместо `double`.
- `boolean` для флагов feature toggle, не `Boolean` без null семантики.
- `String` immutable, безопасен как ключ и expected в JSON сравнении.
- Массивы vs `List`: в параметризованных тестах предпочитаю `List` и streams.
- Честно: autoboxing null `Integer` в арифметике даёт NPE без явного guard.

## Как это работает

Примитивы и ссылки:

```
int caseId = 42;
long startedAt = System.currentTimeMillis();
boolean flaky = false;
double ratio = 0.95;

Integer nullableRetries = null;
String suiteName = "checkout-smoke";
int[] priorities = {1, 2, 3};
```

Enum для статусов:

```
enum RunState { QUEUED, RUNNING, PASSED, FAILED }
RunState state = RunState.PASSED;
```

Python mirror типов в pytest parametrize:

```python
from enum import Enum


class RunState(Enum):
    QUEUED = "queued"
    PASSED = "passed"


def test_typed_fixture(run_state: RunState = RunState.PASSED) -> None:
    assert run_state == RunState.PASSED
```

## Подводные камни

**float сравнение.** `assertEquals(0.1 + 0.2, 0.3)` падает, нужен delta или `BigDecimal`.

**char vs String.** `'A'` не равен `"A"` в assert без конвертации.

**Примитив default.** `int` default 0 маскирует неинициализацию в POJO фикстурах.

**Raw arrays covariant.** `String[]` assignment к `Object[]` → `ArrayStoreException` в runtime.

**Optional не замена null.** В test DTO лучше явный nullable wrapper или validation.
