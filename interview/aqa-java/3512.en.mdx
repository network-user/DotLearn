# forEach: terminal or intermediate?

## In short

`Stream.forEach` is a terminal operation: it runs the pipeline, consumes elements, and closes the stream. `Iterable.forEach` on a collection is not Stream API but also traverses without returning a Stream. After `stream.forEach` you cannot chain `map` or another `forEach` on the same instance.

## How to answer in an interview

- For assert logging: `rows.stream().forEach(log::debug)` as terminal.
- I minimize side effects in `forEach`; I prefer `collect` plus assert on the list.
- `forEachOrdered` when order matters in a sequential pipeline.
- Collection `forEach` does not close a stream; it is a different API.
- Honestly: `forEach` to mutate an external list is worse than `collect(toList())`.

## How it works

Stream terminal:

```
List<String> ids = List.of("a", "b");
ids.stream()
  .map(String::toUpperCase)
  .forEach(System.out::println);
```

Cannot repeat:

```
Stream<String> s = ids.stream();
s.forEach(System.out::println);
// s.map(String::trim); // IllegalStateException
```

Collection Iterable:

```
ids.forEach(id -> cache.put(id, load(id)));
```

Python mirror:

```python
def stream_for_each(values: list[str], action) -> None:
    for item in values:
        action(item.upper())


def test_terminal_consume():
    seen: list[str] = []
    stream_for_each(["a"], seen.append)
    assert seen == ["A"]
```

## Pitfalls

**parallel forEach.** Order is nondeterministic.

**Exception in forEach.** May not stop the whole pipeline with a clear aggregate message.

**Null action.** NPE before traversal starts.

**Confusion with map.** `map` is intermediate; `forEach` is terminal; nothing after terminal.

**Mutable external list.** ConcurrentModification if the source collection changes inside forEach.
