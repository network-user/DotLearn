# Comparable и Comparator при сортировке в тестах

## Суть

`Comparable<T>` задаёт natural order внутри класса через `compareTo`. `Comparator<T>` описывает внешний порядок через `compare(a,b)` без изменения DTO. `Collections.sort(list)` использует comparable; второй аргумент или `list.sort(comparator)` меняет критерий. В assert сравниваю отсортированные списки независимо от порядка insertion из API.

## Что ответить на собесе

- DTO implements Comparable по primary key для stable default sort.
- Comparator.comparing(TestRun::startedAt) для отчёта flaky timeline.
- Несколько критериев: `thenComparing`.
- Null-safe: `Comparator.nullsLast`.
- Честно: compareTo inconsistent with equals ломает TreeSet dedup в test fixture.

## Как это работает

Comparable:

```
public record CaseResult(int id, int ms) implements Comparable<CaseResult> {
  public int compareTo(CaseResult other) {
    return Integer.compare(this.id, other.id);
  }
}
```

Comparator:

```
Comparator<CaseResult> byDuration = Comparator.comparingInt(CaseResult::ms);
results.sort(byDuration);
```

Python mirror:

```python
from dataclasses import dataclass


@dataclass(order=True)
class CaseResult:
    id: int
    ms: int


def test_sort_natural_order():
    rows = [CaseResult(3, 90), CaseResult(1, 120)]
    assert sorted(rows) == [CaseResult(1, 120), CaseResult(3, 90)]
```

## Подводные камни

**Integer overflow в compareTo.** `(a - b)` unsafe, только `Integer.compare`.

**Comparator contract violate.** Not transitive → TreeSet crash cryptic.

**Reverse order mutate.** sorted() возвращает new list, sort() in place.

**Locale String compare.** CI locale меняет order имён.

**Parallel sort unstable.** equal elements swap местами.
