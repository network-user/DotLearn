# Stream и Collection: отличия и reuse

## Суть

`Collection` хранит элементы в памяти и поддерживает мутацию и повторный обход. `Stream` одноразовый ленивый pipeline над источником (collection, array, generator), не хранит данные. После terminal operation stream закрыт, повторный terminal бросает `IllegalStateException`.

## Что ответить на собесе

- Collection живёт весь тест, Stream создаю заново из collection на каждый assert pass.
- Мутации collection во время активного stream дают ConcurrentModification.
- Infinite stream только с limit/skip в тестах на pagination logic.
- Для materialized expected держу `List`, для transform использую stream.
- Честно: reuse stream невозможен by design, это не баг.

## Как это работает

```
List<Integer> ids = new ArrayList<>(List.of(1, 2, 3));
ids.add(4);

long count = ids.stream().filter(v -> v > 1).count();

Stream<Integer> once = ids.stream();
once.forEach(System.out::println);
// once.count(); IllegalStateException
```

Новый pipeline:

```
long again = ids.stream().filter(v -> v > 1).count();
```

Python mirror:

```python
def count_gt(values: list[int], bound: int) -> int:
    return len([v for v in values if v > bound])


def test_stream_like_count():
    assert count_gt([1, 2, 3], 1) == 2
```

## Подводные камни

**Закрытый stream.** Сохранение Stream в поле page object ошибка.

**parallel + ordered assert.** Порядок forEach не гарантирован.

**modify source.** filter на live DB result list во время append.

**primitive stream close.** Также одноразовые.

**Collection.removeIf vs stream filter.** removeIf мутирует, stream filter нет.
