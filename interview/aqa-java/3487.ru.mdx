# Ключевое слово throw в Java тестах

## Суть

`throw` немедленно создаёт Throwable и unwinds stack до matching catch или пробрасывает дальше. Checked exception требует throws или try/catch в сигнатуре caller. В JUnit 5 `assertThrows` ожидает throw как success path. throw отделяет error flow от return values.

## Что ответить на собесе

- Guard clause: `if (id <= 0) throw new IllegalArgumentException(...)`.
- Custom domain exception несёт context для assert message.
- Re-throw с `catch (E e) { throw e; }` или wrap `new ServiceException("ctx", e)`.
- assertThrows проверяет type и message.
- Честно: throw для control flow в happy path test helper хуже Optional.

## Как это работает

```
void requireNonBlank(String value) {
  if (value == null || value.isBlank()) {
    throw new IllegalArgumentException("blank input");
  }
}
```

JUnit:

```
assertThrows(IllegalArgumentException.class, () -> requireNonBlank(" "));
```

Python raise mirror:

```python
def require_non_blank(value: str) -> None:
    if not value or not value.strip():
        raise ValueError("blank input")


def test_throw_guard():
    import pytest
    with pytest.raises(ValueError, match="blank"):
        require_non_blank(" ")
```

## Подводные камни

**throw null.** Allowed but NPE in handler.

**Error vs Exception.** catch Exception misses OutOfMemory.

**finally return.** Suppresses throw silently nightmare.

**assertThrows lazy lambda.** Method ref must execute inside lambda.

**Checked in lambda.** Unchecked wrap or throws on test method.
