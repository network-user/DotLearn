# Ключевые особенности Java Stream API

## Суть

Stream API (Java 8+) даёт функциональный pipeline над данными: ленивые intermediate ops, eager terminal ops, optional parallel fork. Stream не хранит элементы, только описывает вычисление над source. Это снижает boilerplate в test utility, но требует понимания одноразовости stream и отличия stateless/stateful операций.

## Что ответить на собесе

- Lazy: `filter` не бежит, пока нет `collect`.
- `parallelStream()` делит spliterator, нужен immutable source.
- Stateless ops (`map`, `filter`) composable; `sorted`, `distinct` stateful и дороже по памяти.
- Primitive streams (`IntStream`) избегают boxing в perf-sensitive assert.
- Честно: parallel на CI с 2 vCPU иногда медленнее из-за overhead fork/join.

## Как это работает

Lazy chain:

```
long failed = cases.stream()
    .filter(c -> c.durationMs() > 500)
    .peek(c -> log.debug(c.name()))
    .count();
```

Parallel:

```
List<String> urls = pages.parallelStream()
    .map(Page::canonicalUrl)
    .collect(Collectors.toList());
```

Python summary table:

```python
FEATURES = {
    "lazy_intermediate": "runs only after terminal",
    "parallel": "needs thread-safe source",
    "no_storage": "pipeline describes computation",
}


def test_stream_features_doc():
    assert FEATURES["lazy_intermediate"].startswith("runs")
```

## Подводные камни

**sorted на infinite stream.** Hang или OOM в генераторе.

**peek для логики.** Side effect, не замена map.

**parallel + ThreadLocal.** Контекст теряется между worker.

**toMap duplicate key.** IllegalStateException без merge function.

**Stream closed twice.** IllegalStateException в nested helper.
