# Абстрактный класс как шаблон

## Суть

Абстрактный класс объединяет частично готовую реализацию и обязательные abstract методы, которые подкласс должен дописать. Напрямую `new` запрещён: работают только конкретные наследники. Это шаблон для семейства сущностей с общим поведением и разными деталями, типичный base test / base page в automation.

## Что ответить на собесе

- `abstract void fillForm()` в base, `submit()` уже реализован.
- Поля superclass доступны child, в отличие от чистого interface до Java 8 fields.
- Один extends, несколько implements: `class X extends Base implements RetryPolicy`.
- Если abstract метод не реализован, класс остаётся abstract.
- Честно: толстый abstract base хуже композиции helper-объектов в modern test design.

## Как это работает

Шаблон животных (учебный пример, переименован):

```
abstract class BrowserSession {
  abstract void openHome();

  void logStart() {
    System.out.println("session boot");
  }
}

class ChromeRun extends BrowserSession {
  void openHome() { /* navigate chrome */ }
}

class FirefoxRun extends BrowserSession {
  void openHome() { /* navigate firefox */ }
}
```

Python mirror:

```python
from abc import ABC, abstractmethod


class BrowserSession(ABC):
    @abstractmethod
    def open_home(self) -> None:
        raise NotImplementedError

    def log_start(self) -> None:
        print("session boot")
```

## Подводные камни

**Забытый abstract на классе.** Compile error при первом abstract методе.

**Private abstract.** Нельзя: abstract должен быть видим подклассу.

**Constructor в abstract.** Child обязан вызвать super, иначе incomplete object.

**Non-static abstract в static context.** Путаница с static abstract (нельзя).

**Полиморфный массив.** `BrowserSession[]` с разными child ок, но equals/hash не унифицированы.
