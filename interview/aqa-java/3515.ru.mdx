# == и equals() в assert

## Суть

`==` для примитивов сравнивает значения, для объектов ссылки (same instance). `equals()` сравнивает логическое равенство содержимого, если переопределён. В тестах почти всегда нужен `assertEquals` (equals), не `==`, кроме проверки singleton или enum identity.

## Что ответить на собесе

- Строки: всегда `assertEquals(expected, actual)`, не `==`.
- DTO: переопределённый equals или AssertJ `usingRecursiveComparison`.
- Примитивы: `==` ок в `assertTrue(a == b)`.
- Enum constant: `==` безопасен, equals тоже true.
- Честно: String pool literals могут совпасть по `==`, на это не полагаюсь.

## Как это работает

```
String a = new String("ok");
String b = new String("ok");
assertFalse(a == b);
assertTrue(a.equals(b));
```

Кастомный DTO:

```
record CaseId(int value) {}

assertEquals(new CaseId(1), new CaseId(1));
```

Python mirror:

```python
def same_content(a: str, b: str) -> bool:
    return a == b


def test_value_equality():
    assert same_content("ok", "ok")
```

## Подводные камни

**String intern.** `"a" == "a"` true, `new String("a")` нет.

**equals без hashCode.** HashSet/HashMap assert ломается.

**Double equals.** 0.1 + 0.2, нужен delta assert.

**Arrays.** `assertArrayEquals`, не `equals` на массивах.

**Mockito same().** Проверка identity, не equals content.
