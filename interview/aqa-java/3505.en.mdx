# ArrayList vs LinkedList: choice in AQA

## In short

`ArrayList` is array-backed: fast `get`, costly middle insert. `LinkedList` is doubly linked: O(1) insert at a known iterator, slow indexed access. For test case lists, expected rows, and REST payload fields, `ArrayList` is almost always enough.

## How to answer in an interview

- Parameterized data with random access: `ArrayList`.
- Event queue in a custom listener: `Deque`/`LinkedList`.
- I compare not only big-O but constant factors and cache locality.
- Immutable expected copy: `List.copyOf(arrayList)`.
- Honestly: a micro-benchmark in a unit test is rarely worth it; I choose by API usage.

## How it works

```
List<String> cases = new ArrayList<>();
cases.add("login");
cases.add("checkout");
assertEquals("login", cases.get(0));

Deque<String> queue = new LinkedList<>();
queue.addLast("retry-1");
queue.pollFirst();
```

Python mirror:

```python
def pick_structure(frequent_index: bool) -> str:
    return "arraylist" if frequent_index else "linkedlist"


def test_choice():
    assert pick_structure(True) == "arraylist"
```

## Pitfalls

**LinkedList.get in a loop.** Quadratic slowdown in assert loops.

**List interface only.** You lose `addFirst` without casting to Deque.

**Arrays.asList fixed size.** Cannot add; often confused with ArrayList.

**SubList structural change.** ConcurrentModification on the parent.

**parallelStream on LinkedList.** Poor locality, rare benefit.
