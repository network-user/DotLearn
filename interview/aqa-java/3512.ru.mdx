# forEach: terminal или intermediate?

## Суть

`Stream.forEach` это terminal operation: запускает pipeline, потребляет элементы, закрывает stream. `Iterable.forEach` на коллекции не Stream API, но тоже обход без возврата Stream. После `stream.forEach` нельзя добавить `map` или второй `forEach` на том же instance.

## Что ответить на собесе

- В assert логировании: `rows.stream().forEach(log::debug)` как terminal.
- Side effects в `forEach` минимизирую, предпочитаю `collect` + assert на списке.
- `forEachOrdered` если критичен порядок в sequential pipeline.
- Collection `forEach` не закрывает stream, это другой API.
- Честно: `forEach` для mutate внешнего списка хуже `collect(toList())`.

## Как это работает

Stream terminal:

```
List<String> ids = List.of("a", "b");
ids.stream()
  .map(String::toUpperCase)
  .forEach(System.out::println);
```

Повтор невозможен:

```
Stream<String> s = ids.stream();
s.forEach(System.out::println);
// s.map(String::trim); // IllegalStateException
```

Collection Iterable:

```
ids.forEach(id -> cache.put(id, load(id)));
```

Python mirror:

```python
def stream_for_each(values: list[str], action) -> None:
    for item in values:
        action(item.upper())


def test_terminal_consume():
    seen: list[str] = []
    stream_for_each(["a"], seen.append)
    assert seen == ["A"]
```

## Подводные камни

**parallel forEach.** Порядок не детерминирован.

**Exception в forEach.** Не всегда прерывает весь pipeline с понятным aggregate message.

**Null action.** NPE до старта обхода.

**Путаница с map.** `map` intermediate, `forEach` terminal, нельзя после terminal.

**Mutable external list.** ConcurrentModification если меняю source collection внутри forEach.
