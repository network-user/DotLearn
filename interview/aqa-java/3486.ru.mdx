# HashMap для lookup fixture по id

## Суть

HashMap хранит уникальные ключи с hash bucket lookup, amortized O(1) get/put. Collisions через chain или tree (Java 8+). Null key одна, null values allowed. В тестах map response id → entity для O(1) assert вместо nested loop.

## Что ответить на собесе

- `getOrDefault` для optional field.
- `computeIfAbsent` lazy fixture factory.
- load factor 0.75 resize doubles buckets.
- LinkedHashMap access-order для LRU cache simulation.
- Честно: hash DoS не topic unit test, но immutable keys good practice.

## Как это работает

```
Map<Long, UserDto> byId = new HashMap<>();
for (UserDto u : response) {
  byId.put(u.id(), u);
}
assert byId.get(42L).email().equals("qa@test");
```

Merge:

```
map.merge(key, 1, Integer::sum);
```

Python dict:

```python
def index_by_id(rows: list[dict]) -> dict[int, dict]:
    return {row["id"]: row for row in rows}


def test_hash_lookup():
    data = index_by_id([{"id": 1, "v": "a"}, {"id": 2, "v": "b"}])
    assert data[2]["v"] == "b"
```

## Подводные камни

**Key mutable after put.** Lost in wrong bucket.

**equals without hashCode.** Broken contract, get misses.

**ConcurrentHashMap null key.** Not allowed unlike HashMap.

**toMap duplicate stream.** Exception without merge function.

**Mod count during iteration.** Fail-fast entrySet iterator.
