# Сравнение в Java: нет оператора Is

## Суть

В Java нет оператора `is` как в C# (`x is Type`). Для типов используют `instanceof` (Java 16+ pattern matching: `if (obj instanceof String s)`). Для равенства значений объектов применяют `equals()`, для ссылок `==`. На собесе часто проверяют, что кандидат не путает Java с другими языками.

## Что ответить на собесе

- Проверка типа: `value instanceof Map<?, ?>`.
- Pattern matching: `if (payload instanceof ErrorBody body) assertNotNull(body.code())`.
- Равенство строк: `equals`, не `==`.
- `Objects.equals(a, b)` null-safe сравнение в assert helper.
- Честно: в тестах избегаю цепочек `getClass() ==`, предпочитаю polymorphic API.

## Как это работает

```
Object payload = Map.of("code", 422);

if (payload instanceof Map<?, ?> map) {
  assertEquals(422, map.get("code"));
}
```

Сравнение ссылок vs значений:

```
String left = new String("fail");
String right = new String("fail");
assertTrue(left.equals(right));
assertFalse(left == right);
```

Python mirror (`is` как identity):

```python
def is_same_instance(a, b) -> bool:
    return a is b


def test_identity_vs_equality():
    a = ["x"]
    b = ["x"]
    assert a == b
    assert not is_same_instance(a, b)
```

## Подводные камни

**instanceof null.** Всегда false, не NPE.

**Raw Map instanceof.** Generics стираются, проверяю структуру аккуратно.

**equals без type check.** ClassCastException внутри equals если subtype поля.

**Путаница с `is` из C#.** На собесе проговариваю Java альтернативы явно.

**Enum switch.** Не нужен instanceof, использую switch по enum.
