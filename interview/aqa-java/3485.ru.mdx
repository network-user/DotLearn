# Set в Java для уникальных test tags

## Суть

Set хранит уникальные элементы по equals/hashCode (HashSet) или compareTo (TreeSet). Дубликаты add игнорируются или replace policy в Map side. LinkedHashSet сохраняет insertion order. В assert сравниваю `Set.copyOf(apiTags)` с expected без учёта порядка.

## Что ответить на собесе

- HashSet O(1) contains для gate «already ran case id».
- TreeSet sorted iteration для stable report без extra sort.
- LinkedHashSet когда order важен но uniqueness too.
- EnumSet для bitmask feature flags compact.
- Честно: mutable object in Set без equals override dedup fails silently.

## Как это работает

```
Set<String> tags = new HashSet<>();
tags.add("smoke");
tags.add("smoke");
assert tags.size() == 1;
```

TreeSet:

```
Set<String> ordered = new TreeSet<>(List.of("beta", "alpha"));
```

Python set assert:

```python
def test_unique_tags():
    tags = {"smoke", "regression", "smoke"}
    assert len(tags) == 2
    assert tags == {"smoke", "regression"}
```

## Подводные камни

**Order assert on HashSet.** Flaky nightly.

**Custom object hash change.** Element lost after mutate field in hash.

**Null in HashSet.** One null allowed, TreeSet null first only old JDK.

**Set.of duplicate arg.** IllegalArgumentException at create.

**ConcurrentHashSet.newKeySet.** Not java.util.Set semantics identical for all ops.
