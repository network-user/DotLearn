# Наследование классов в Java для AQA

## Суть

Наследование связывает подкласс с суперклассом через `extends`: потомок получает неприватные поля и методы, может расширять API и переопределять поведение. В тестовых фреймворках и page object слоях это основа переиспользования базовых шагов и полиморфных фикстур.

## Что ответить на собесе

- Базовый класс с общими хелперами, наследники под конкретные сервисы или страницы.
- `@Override` на переопределённых методах, `super()` в конструкторе для инициализации родителя.
- Полиморфизм: переменная типа родителя ссылается на объект потомка.
- Композиция предпочтительнее глубоких иерархий в тестовом коде.
- Честно: множественное наследование классов в Java недоступно, только один `extends`.

## Как это работает

```
class Vehicle {
  protected String label;

  Vehicle(String label) {
    this.label = label;
  }

  String describe() {
    return "vehicle " + label;
  }
}

class Truck extends Vehicle {
  Truck(String label) {
    super(label);
  }

  @Override
  String describe() {
    return "truck " + label;
  }
}
```

В тесте:

```
Vehicle unit = new Truck("cargo");
assertEquals("truck cargo", unit.describe());
```

Python mirror для pytest base:

```python
class ApiClient:
    def __init__(self, base_url: str) -> None:
        self.base_url = base_url

    def ping(self) -> str:
        return f"ok {self.base_url}"


class OrdersClient(ApiClient):
    def ping(self) -> str:
        return f"orders {self.base_url}"


def test_inheritance_polymorphism():
    client: ApiClient = OrdersClient("https://api.test")
    assert client.ping() == "orders https://api.test"
```

## Подводные камни

**private не наследуется.** Потомок не видит private поля родителя напрямую.

**Конструктор родителя.** Без явного `super(...)` компилятор требует безаргументный конструктор суперкласса.

**Глубокие деревья.** Page object с пятью уровнями наследования ломается при смене UI.

**final методы.** Переопределить нельзя, тестовый stub через наследник не сработает.

**Liskov нарушения.** Потомок, ужесточающий контракт метода, ломает полиморфные тесты.
