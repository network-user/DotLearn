# What is the Stream API in Java

## In short

The Stream API is a set of interfaces (`Stream`, `IntStream`, collectors) for declarative processing of sequences from collections, arrays, or generators. It separates describing transformations from execution time and supports sequential and parallel modes. For Java backend autotests it is a compact way to build expected data sets from ORM entities.

## How to answer in an interview

- A stream does not modify the source collection.
- Pipeline: source → intermediate* → terminal.
- Parallel needs associative reduce and a thread-safe source.
- For IO-bound tests a stream does not replace an async client; it only transforms in-memory data.
- Honestly: an imperative loop is sometimes simpler for break/continue with complex branches.

## How it works

```
List<Integer> evenDoubled = numbers.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * 2)
    .collect(Collectors.toList());
```

Parallel:

```
long sum = hugeList.parallelStream()
    .mapToLong(this::scoreOf)
    .sum();
```

Conceptual schema in pytest:

```python
def stream_pipeline(numbers: list[int]) -> list[int]:
    return [n * 2 for n in numbers if n % 2 == 0]


def test_stream_definition():
    assert stream_pipeline([1, 2, 3, 4]) == [4, 8]
```

## Pitfalls

**Stream from a concurrent collection.** CME or lost elements.

**Infinite generate stream.** Terminal count without limit never returns.

**Primitive stream boxing.** `Stream<Integer>` vs `IntStream` memory cost.

**Custom spliterator bug.** Parallel split skews on uneven work.

**Closing resource streams.** `Files.lines` needs try-with-resources.
