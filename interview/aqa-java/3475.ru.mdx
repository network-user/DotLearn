# Терминальные операции Stream API

## Суть

Терминальная операция закрывает pipeline и материализует результат: коллекция (`collect`), агрегат (`reduce`, `count`, `min`, `max`), void side effect (`forEach`). Без terminal промежуточные шаги не выполняются. В тестах terminal превращает stream ответов сервиса в assert-friendly структуру.

## Что ответить на собесе

- `collect(toList())` для snapshot сравнения JSON list.
- `reduce` суммирует метрики latency по кейсам.
- `count` быстрее materialize всего списка, если нужен только размер.
- `Optional` из `min/max` проверяю через `orElseThrow` в negative test.
- Честно: `forEach` без assert внутри только для debug log, не для verification.

## Как это работает

collect:

```
List<String> names = Stream.of("api", "ui", "db")
    .map(String::toUpperCase)
    .collect(Collectors.toList());
```

reduce и count:

```
int total = Stream.of(120, 80, 200).reduce(0, Integer::sum);
long qty = Stream.of("a", "b", "c").count();
```

min:

```
OptionalInt fastest = IntStream.of(900, 450, 1200).min();
```

pytest mirror:

```python
def terminal_collect(items: list[str]) -> list[str]:
    return [s.upper() for s in items]


def test_terminal_ops():
    assert terminal_collect(["api", "ui"]) == ["API", "UI"]
    assert sum([120, 80, 200]) == 400
    assert min([900, 450, 1200]) == 450
```

## Подводные камни

**collect mutable list mutation.** Unmodifiable list from `toList()` Java 16+.

**reduce без identity.** `Optional` empty на пустом stream.

**forEach order parallel.** Non-deterministic порядок логов.

**count после limit.** Считает не больше limit, но всё равно traverse.

**joining без filter empty.** Лишние запятые в CSV assert.
