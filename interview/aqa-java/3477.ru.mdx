# Что такое Stream API в Java

## Суть

Stream API это набор интерфейсов (`Stream`, `IntStream`, collectors) для декларативной обработки последовательностей из коллекций, массивов, генераторов. Он разделяет описание трансформаций и момент выполнения, поддерживает sequential и parallel режимы. Для автотестов Java backend это компактный способ подготовить expected data set из ORM entities.

## Что ответить на собесе

- Stream не модифицирует source collection.
- Pipeline: source → intermediate* → terminal.
- Parallel требует associative reduce и thread-safe source.
- Для IO-bound тестов stream не заменяет async client, только in-memory transform.
- Честно: императивный loop иногда проще для break/continue с complex branch.

## Как это работает

```
List<Integer> evenDoubled = numbers.stream()
    .filter(n -> n % 2 == 0)
    .map(n -> n * 2)
    .collect(Collectors.toList());
```

Параллель:

```
long sum = hugeList.parallelStream()
    .mapToLong(this::scoreOf)
    .sum();
```

Концептуальная схема в pytest:

```python
def stream_pipeline(numbers: list[int]) -> list[int]:
    return [n * 2 for n in numbers if n % 2 == 0]


def test_stream_definition():
    assert stream_pipeline([1, 2, 3, 4]) == [4, 8]
```

## Подводные камни

**Stream from concurrent collection.** CME или lost elements.

**Infinite stream generate.** Terminal count без limit never returns.

**Primitive stream boxing.** `Stream<Integer>` vs `IntStream` memory.

**Custom spliterator bug.** Parallel split skew на uneven work.

**Closing resource stream.** Files.lines нужен try-with-resources.
