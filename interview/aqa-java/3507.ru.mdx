# Функциональные интерфейсы в тестовом коде

## Суть

Функциональный интерфейс имеет ровно один абстрактный метод (SAM) и может содержать `default`/`static` методы. Он принимает лямбды и method references. В тестах это `Predicate` для фильтрации ответов, `Supplier` для ленивых фикстур, `Consumer` для logging hooks.

## Что ответить на собесе

- `@FunctionalInterface` на кастомном `RetryPolicy` для compile-time guard.
- `Predicate<String>` в assert helper: `rows.stream().filter(predicate)`.
- Method reference `String::isBlank` вместо `s -> s.isBlank()`.
- Не добавляю второй abstract method без понимания ломания лямбд.
- Честно: для 1-2 строк лямбда ок, для сложной логики именованный класс читабельнее.

## Как это работает

```
@FunctionalInterface
interface WaitCondition {
  boolean ready();
}

void waitUntil(WaitCondition condition) {
  while (!condition.ready()) { /* poll */ }
}

waitUntil(() -> driver.getTitle().contains("Dashboard"));
```

Built-in:

```
Predicate<Integer> positive = value -> value > 0;
List<Integer> filtered = ids.stream().filter(positive).toList();
```

Python mirror:

```python
from typing import Callable


def wait_until(condition: Callable[[], bool]) -> None:
    while not condition():
        pass


def test_functional_condition():
    flag = {"ready": False}
    flag["ready"] = True
    wait_until(lambda: flag["ready"])
```

## Подводные камни

**Два abstract метода.** Интерфейс перестаёт быть functional, лямбда не компилируется.

**Object methods.** equals/hashCode не считаются abstract SAM.

**Serializable lambda.** В distributed test runner сериализация лямбд боль.

**this capture.** Лямбда держит ссылку на outer test instance, GC leak в long suite.

**Exception в SAM.** Checked в лямбде требует обёртки.
