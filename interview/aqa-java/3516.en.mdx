# Comparison in Java: no Is operator

## In short

Java has no `is` operator like C# (`x is Type`). For types use `instanceof` (Java 16+ pattern matching: `if (obj instanceof String s)`). For object value equality use `equals()`; for references use `==`. Interviews often check that candidates do not confuse Java with other languages.

## How to answer in an interview

- Type check: `value instanceof Map<?, ?>`.
- Pattern matching: `if (payload instanceof ErrorBody body) assertNotNull(body.code())`.
- String equality: `equals`, not `==`.
- `Objects.equals(a, b)` for null-safe comparison in assert helpers.
- Honestly: in tests I avoid `getClass() ==` chains; I prefer polymorphic APIs.

## How it works

```
Object payload = Map.of("code", 422);

if (payload instanceof Map<?, ?> map) {
  assertEquals(422, map.get("code"));
}
```

Reference vs value:

```
String left = new String("fail");
String right = new String("fail");
assertTrue(left.equals(right));
assertFalse(left == right);
```

Python mirror (`is` as identity):

```python
def is_same_instance(a, b) -> bool:
    return a is b


def test_identity_vs_equality():
    a = ["x"]
    b = ["x"]
    assert a == b
    assert not is_same_instance(a, b)
```

## Pitfalls

**instanceof null.** Always false, not NPE.

**Raw Map instanceof.** Generics erase; check structure carefully.

**equals without type check.** ClassCastException inside equals on subtype fields.

**Confusion with C# `is`.** In interviews I state Java alternatives explicitly.

**Enum switch.** No instanceof needed; switch on the enum.
