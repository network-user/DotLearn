# Перегрузка методов

## Суть

Перегрузка (overloading) это несколько методов с одним именем и разной сигнатурой (число, тип или порядок параметров) в одном классе. Возвращаемый тип alone не создаёт перегрузку. Компилятор выбирает версию на этапе compile-time по аргументам вызова. В helper-классах тестов перегрузка упрощает API: один `click` для By, String, WebElement.

## Что ответить на собесе

- `void type(CharSequence)` и `void type(String)` разные overload.
- `type(int)` и `type(long)` при `type(5)` выберет int (widening позже).
- Varargs конфликтует с array overload, нужна явность.
- Перегрузка не polymorphism runtime (это override).
- Честно: десять overload вместо builder усложняет discoverability в IDE.

## Как это работает

Retry helper:

```
class Retry {
  int delayMs(int base) { return base; }
  int delayMs(int base, int factor) { return base * factor; }
  double delayMs(double seconds) { return (int)(seconds * 1000); }
}
```

Вызов:

```
Retry r = new Retry();
r.delayMs(200);
r.delayMs(200, 3);
r.delayMs(0.5);
```

Python mirror:

```python
def delay_ms(base: int, factor: int = 1) -> int:
    return base * factor
```

## Подводные камни

**Ambiguous call.** `log(null)` между `log(String)` и `log(Throwable)` не компилируется.

**Autoboxing overload.** `print(int)` vs `print(Integer)` при null.

**Return type only.** `int parse()` и `String parse()` в одном классе ошибка.

**Bridge methods + overload.** Generics создают synthetic методы, редкие сюрпризы.

**Override vs overload.** Перегрузка в child не override parent, скрывает имя.
