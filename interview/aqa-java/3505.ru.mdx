# ArrayList против LinkedList: выбор в AQA

## Суть

`ArrayList` backed массивом: быстрый `get`, дорогая вставка в середину. `LinkedList` двусвязный: O(1) вставка при известном итераторе, медленный indexed access. Для тестовых списков кейсов, expected rows и REST payload полей почти всегда достаточно `ArrayList`.

## Что ответить на собесе

- Параметризованные данные с random access: `ArrayList`.
- Очередь событий в custom listener: `Deque`/`LinkedList`.
- Сравниваю не только big-O, но и constant factor и cache locality.
- Immutable copy expected: `List.copyOf(arrayList)`.
- Честно: микробенчмарк в unit тесте редко оправдан, выбираю по API usage.

## Как это работает

```
List<String> cases = new ArrayList<>();
cases.add("login");
cases.add("checkout");
assertEquals("login", cases.get(0));

Deque<String> queue = new LinkedList<>();
queue.addLast("retry-1");
queue.pollFirst();
```

Python mirror:

```python
def pick_structure(frequent_index: bool) -> str:
    return "arraylist" if frequent_index else "linkedlist"


def test_choice():
    assert pick_structure(True) == "arraylist"
```

## Подводные камни

**LinkedList.get в цикле.** Квадратичная деградация в assert loop.

**List interface only.** Теряю `addFirst` без cast к Deque.

**Arrays.asList fixed size.** Нельзя add, путают с ArrayList.

**SubList structural change.** ConcurrentModification на parent.

**parallelStream на LinkedList.** Плохая локальность, редкий выигрыш.
