# LinkedList в Java: когда уместен

## Суть

LinkedList это doubly-linked nodes: O(1) add/remove at head/tail with iterator, O(n) get(i). Реализует List и Deque. Подходит для queue/stack с частыми insert на краях без random access. В типичном AQA коде ArrayDeque выигрывает по cache; LinkedList редок, но полезен для LRU-like test cache prototype.

## Что ответить на собесе

- Deque operations `addFirst`, `pollLast` O(1).
- get(5000) walks nodes, avoid in assertion loop.
- ListIterator remove during iteration legal.
- Memory overhead per node vs ArrayList array slot.
- Честно: «нужен linked» без профиля часто ошибка, ArrayList default.

## Как это работает

```
Deque<String> queue = new LinkedList<>();
queue.addLast("case-a");
queue.addLast("case-b");
String next = queue.pollFirst();
```

Iterator remove:

```
ListIterator<String> it = list.listIterator();
while (it.hasNext()) {
  if (it.next().startsWith("skip")) it.remove();
}
```

Python deque:

```python
from collections import deque


def test_linked_like_deque():
    q: deque[str] = deque()
    q.append("case-a")
    q.append("case-b")
    assert q.popleft() == "case-a"
```

## Подводные камни

**Index loop get(i).** O(n^2) на sort-like pass.

**Queue interface LinkedList.** Legacy choice; ArrayDeque faster.

**serialVersionUID nodes.** Deep copy в clone fixture heavy.

**Null element allowed.** Multiple null in list, Set contrast.

**Fail-fast iterator.** Concurrent modify без iterator throws CME.
