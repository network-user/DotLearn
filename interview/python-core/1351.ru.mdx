# Пространства имён и правило LEGB: как Python находит переменную

## Суть

Пространство имён - это таблица, сопоставляющая имена объектам в памяти. В Python их четыре уровня - локальное, объемлющее, глобальное, встроенное, - а поиск имени идёт по правилу LEGB от внутреннего к внешнему.

## Что ответить на собесе

Пространство имён - это отображение «имя → объект», которое не даёт именам конфликтовать. Уровней четыре: локальное (имена внутри функции), объемлющее/enclosing (имена внешней функции для вложенной), глобальное (уровень модуля) и встроенное (built-in: `len`, `print`, исключения). Когда интерпретатор встречает имя, он ищет его по правилу LEGB - сначала в локальном, потом в объемлющем, затем в глобальном и в самом конце во встроенном; берётся первое совпадение. Это и определяет область видимости: где имя доступно и какое из одноимённых перекроет остальные.

## Как это работает

Локальное - имена внутри тела функции, живут только во время её вызова:

```python
def my_function():
    x = 10          # x в локальном пространстве
    print(x)

my_function()
```

Объемлющее - имена внешней функции, видимые из вложенной:

```python
def outer():
    y = 20

    def inner():
        print(y)    # y берётся из enclosing-пространства
    inner()

outer()
```

Глобальное - имена уровня модуля; встроенное - то, что доступно всегда (`print`, `len`). Поиск идёт по цепочке LEGB - от внутреннего к внешнему:

## Подводные камни

- LEGB - порядок чтения. Чтобы присваивать имя из внешней области, нужны `global` или `nonlocal`, иначе присваивание создаёт новую локальную переменную.
- Присваивание имени где угодно в теле функции делает его локальным на весь вызов - чтение до присваивания даст `UnboundLocalError`.
- У классов своё пространство имён, но оно не участвует в LEGB вложенных функций: методы не видят имена класса напрямую, только через `self`/имя класса.
- Затенение built-in (`list = [...]`, `id = 5`) перекрывает встроенное имя в текущей области - частый источник скрытых багов.
