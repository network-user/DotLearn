# itertools: какие строительные блоки для итераторов даёт стандартная библиотека

## Суть

`itertools` - набор ленивых функций над итераторами: они не материализуют результат в память, а отдают элементы по запросу. Это позволяет обрабатывать огромные или бесконечные потоки данных, тратя память лишь на текущий элемент.

## Что ответить на собесе

`itertools` делит инструменты на три группы, и все они возвращают ленивый итератор, а не список.

- Бесконечные генераторы: `count`, `cycle`, `repeat`.
- Укорачивают или фильтруют поток: `chain`, `compress`, `dropwhile`, `takewhile`, `groupby`, `islice`.
- Комбинаторика: `product`, `permutations`, `combinations`. Ленивость позволяет собирать конвейер и применять к данным, которые целиком в память не влезают.

Бесконечные источники нужно обязательно ограничивать снаружи (`next`, `islice` или условие выхода), иначе цикл не завершится.

```python
import itertools

counter = itertools.count(start=10, step=2)
print([next(counter) for _ in range(5)])      # [10, 12, 14, 16, 18]

cycler = itertools.cycle('AB')
print([next(cycler) for _ in range(5)])        # ['A', 'B', 'A', 'B', 'A']

print(list(itertools.repeat('Hi', 3)))         # ['Hi', 'Hi', 'Hi']
```

Фильтрация и склейка потока:

```python
import itertools

print(list(itertools.chain('ABC', 'DEF')))                 # ['A','B','C','D','E','F']
print(list(itertools.compress('ABCDEF', [1,0,1,0,1,0])))   # ['A', 'C', 'E']
print(list(itertools.dropwhile(lambda x: x < 3, [1,2,3,4,5])))  # [3, 4, 5]
print(list(itertools.takewhile(lambda x: x < 4, [1,2,3,4,5])))  # [1, 2, 3]
```

`groupby` группирует только подряд идущие одинаковые элементы:

```python
import itertools

for key, group in itertools.groupby([1, 1, 2, 2, 3, 3]):
    print(key, list(group))   # 1 [1, 1] / 2 [2, 2] / 3 [3, 3]
```

Комбинаторика - перебор вариантов без ручных вложенных циклов:

```python
import itertools

print(list(itertools.product('AB', repeat=2)))       # AA AB BA BB
print(list(itertools.permutations('ABC', 2)))        # AB AC BA BC CA CB
print(list(itertools.combinations('ABC', 2)))        # AB AC BC
```

## Подводные камни

- `count`, `cycle`, `repeat()` без `times` бесконечны: `list()` по ним зависнет. Ограничивайте через `islice`.
- `groupby` группирует соседей, а не все одинаковые значения. Чтобы сгруппировать весь массив, его сначала сортируют по тому же ключу.
- Итератор одноразовый: пройдя по нему один раз, второй раз вы получите пустоту. Если результат нужен дважды - сохраните в список или используйте `itertools.tee`.
- `permutations` учитывает порядок (`AB` != `BA`), `combinations` - нет. Путаница между ними - классическая ошибка в комбинаторных задачах.
