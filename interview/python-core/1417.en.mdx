# What is an iterator, and which iterators do you encounter in Python?

## In short

An iterator is an object that implements the `__iter__` and `__next__` protocol, yields elements one at a time, and remembers where it stopped. This lets you traverse data without loading the entire collection into memory.

## How to answer in an interview

An iterator is an object that has `__next__()`, which returns the next element, and `__iter__()`, which returns the iterator itself. When the elements run out, `__next__()` raises `StopIteration`, and the `for` loop catches it. A list on its own is iterable but not an iterator; `iter()` turns it into one. In the standard library, iterators include generators (functions with `yield`), as well as the lazy results of `map()`, `filter()`, `zip()`, `enumerate`, and objects from `itertools`.

The iteration protocol rests on two methods:

- `__iter__()` - returns the iterator itself, so the object works with `for`.
- `__next__()` - returns the next element and, on exhaustion, raises `StopIteration`.

You write your own iterator like this:

```python
class MyIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.index < len(self.data):
            result = self.data[self.index]
            self.index += 1
            return result
        raise StopIteration


for item in MyIterator([1, 2, 3]):
    print(item)
```

Built-in iterator sources:

```python
nums = iter([1, 2, 3])
print(next(nums))  # 1

squared = map(lambda x: x ** 2, [1, 2, 3])
print(list(squared))  # [1, 4, 9]

evens = filter(lambda x: x % 2 == 0, [1, 2, 3, 4])
print(list(evens))  # [2, 4]
```

A generator is the most compact way to get an iterator: `yield` implements both protocol methods by itself.

## Pitfalls

- An iterable and an iterator are different things: a list is iterable, but you cannot traverse it with `next()` until you call `iter()`.
- An iterator is single-use: once it reaches `StopIteration`, it does not restart. For another pass you need a new one.
- In Python 3, `map`, `filter`, and `zip` are lazy - if you forget to wrap them in `list()`, it is easy to end up with nothing after the very first traversal.
- In `__next__`, always raise exactly `StopIteration`, otherwise the `for` loop will loop forever or fail with a different error.
