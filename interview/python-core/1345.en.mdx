# A class as a decorator: without which method does nothing work?

## In short

For an instance of a class to be usable as a decorator, the class must define `__call__`. It is this method that makes the instance callable with parentheses, and a decorator is essentially a callable object.

## How to answer in an interview

A decorator is any callable object that takes a function and returns a function (or a callable). When a decorator is written as a class, the `__init__` constructor remembers the wrapped function, and `__call__` runs on each call of the decorated function - that is where the extra logic lives. Without `__call__` an instance of the class cannot be called with parentheses, and decoration does not work. The advantage of a class over a function decorator is that it conveniently holds state between calls.

## How it works

`@MyDecorator` over a function is equivalent to `func = MyDecorator(func)`. First `__init__` runs, saving the function. After that every call of the decorated name goes into `__call__`.

```python
class CallCounter:
    def __init__(self, func):
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Вызов №{self.count}")
        return self.func(*args, **kwargs)


@CallCounter
def greet(name):
    return f"Привет, {name}"


print(greet("Аня"))
print(greet("Боря"))
```

Since the state (`count`) lives in the instance, it persists between calls: the first call prints "Вызов №1", the second "Вызов №2". In a function decorator you would have to build a closure or an external variable for this.

## Pitfalls

- Without `__call__` the instance cannot be called - you get `TypeError: object is not callable`.
- `*args, **kwargs` in `__call__` are needed to forward any arguments of the decorated function; otherwise the decorator only works for a specific signature.
- A class decorator replaces the function with an instance, so `__name__`, `__doc__` and metadata are lost. Restoring them is harder than in a function decorator (where `functools.wraps` helps).
- A single decorator instance wraps a single function: when applied to several functions, a separate instance with its own state is created for each.
