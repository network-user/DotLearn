# Как пробросить пойманное исключение дальше, не подавляя его?

## Суть

Голый `raise` без аргументов внутри блока `except` повторно поднимает текущее исключение и отдаёт его вышестоящему обработчику, сохраняя исходный traceback.

## Что ответить на собесе

Если я поймал исключение, но не могу или не должен его полностью обработать, я выполняю локальную реакцию (например, логирование) и пишу `raise` без аргументов. Это не создаёт новое исключение, а пробрасывает то же самое выше по стеку, с нетронутым traceback. Подавлять ошибку через пустой `except` или возвращать `None` я не буду - это скрывает проблему.

## Как это работает

`raise` внутри `except` использует «текущее» активное исключение, которое интерпретатор держит в `sys.exc_info()`. Поэтому аргументы не нужны: Python и так знает, что повторно поднимать. Traceback при этом не обрезается до текущей строки, а указывает на исходное место сбоя.

```python
def process_data(data):
    try:
        result = 10 / data
    except ZeroDivisionError:
        print("Локально зафиксировали деление на ноль")
        raise

def main():
    try:
        process_data(0)
    except ZeroDivisionError:
        print("Ошибка перехвачена на верхнем уровне")

main()
```

Здесь `process_data` логирует факт ошибки, но решение, что с ней делать, принимает `main`. Управление поднимается ровно туда, где есть подходящий обработчик.

## Подводные камни

- `raise` без аргументов вне блока `except` бросит `RuntimeError: No active exception to re-raise`.
- Не путайте голый `raise` с `raise err`: второй вариант может перезаписать traceback на текущую строку. Для смены типа с сохранением причины используйте `raise NewError(...) from err`.
- Пустой `except: pass` молча проглатывает ошибку - почти всегда это баг, а не фича.
- Логировать стоит до `raise`, иначе при ранней передаче исключения локальная реакция не успеет выполниться.
