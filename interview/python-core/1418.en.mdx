# Can you apply a decorator to a class rather than a function?

## In short

Yes. A class decorator is a function that receives a class, modifies it somehow, and returns it back. The principle is the same as for a function decorator, except the input object is a class.

## How to answer in an interview

A decorator is any callable that takes another object and returns a replacement. For a class, the syntax `@deco` above `class C` is equivalent to `C = deco(C)`. Inside the decorator I can add methods or attributes to the class, wrap `__init__`, or replace the class with another. This is handy for cross-cutting functionality - logging, registration in a registry, caching, permission checks - without editing the class body. If you need to intervene deeper, in the creation of the class itself, you use metaclasses, but for most tasks a decorator is enough.

## How it works

Let's build a decorator that adds a `greet` method to every decorated class:

```python
def add_greet_method(cls):
    def greet(self):
        return f"Hello from {self.__class__.__name__}!"
    cls.greet = greet
    return cls


@add_greet_method
class MyClass:
    def __init__(self, name):
        self.name = name


obj = MyClass("Example")
print(obj.greet())  # Hello from MyClass!
```

What happens step by step:

1. `@add_greet_method` means `MyClass = add_greet_method(MyClass)`.
2. Inside, a `greet` function is declared and assigned to the class as an attribute - now it is a full-fledged method.
3. `return cls` hands back the same class, already extended.
4. Any instance of `MyClass` gets access to `greet`.

A decorator can also replace the class entirely - for example, return a wrapper that intercepts instance creation.

## Pitfalls

- A class decorator runs once, at the moment the class is defined, not on every instance creation.
- If the decorator returns not the class itself but a wrapper function, you lose `isinstance`, the name, and the metadata of the original class - be careful.
- When adding an attribute as a method (`cls.greet = greet`), remember: a regular function becomes a bound method, but a nested function without `self` cannot be attached this way.
- To control the class creation process itself (rather than tweaking it after the fact), a decorator is not enough - you need a metaclass.
