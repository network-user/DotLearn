# Как Pydantic проверяет данные по аннотациям и что именно он гарантирует

## Суть

Pydantic берёт обычные аннотации типов Python как спецификацию и при создании модели валидирует входные данные против неё: проверяет типы, ограничения значений и структуру, а где возможно — приводит данные к нужному типу. На вход годятся простые типы, коллекции, вложенные модели и пользовательские типы.

## Что ответить на собесе

Pydantic описывает схему через классы-наследники `BaseModel` с аннотированными полями и валидирует данные в конструкторе.

- При инициализации каждое поле проходит валидацию по типу и дополнительным ограничениям (`gt`, `max_length` и т.п.).
- При несоответствии поднимается `ValidationError` со всеми провалившимися полями и причинами, а не только первой ошибкой.
- По умолчанию совместимые значения приводятся (строка `"30"` в `int`); строгое поведение включается через `Strict`-типы или `model_config`.

## Как это работает

```python
from pydantic import BaseModel, ValidationError, conint

class User(BaseModel):
    id: conint(gt=0)
    name: str
    age: int

user = User(id=1, name='Alice', age=30)
print(user.age)  # 30

try:
    User(id=-1, name='Bob', age='thirty')
except ValidationError as e:
    print(e)  # перечислит провал по id (gt=0) и по age (не int)
```

Что Pydantic покрывает:

- **Простые типы**: `int`, `float`, `str`, `bool`.
- **Коллекции**: `list`, `tuple`, `set`, `dict` с типизацией элементов.
- **Вложенные модели**: поле-`BaseModel` валидируется рекурсивно.
- **Ограничения значений**: `conint`, `constr`, `Field(gt=..., max_length=...)`.

Валидация запускается автоматически в конструкторе, поэтому после успешного создания объекта данные гарантированно соответствуют схеме.

## Подводные камни

- По умолчанию `'30'` превратится в `30` — это не баг, а режим лояльного приведения. Нужна строгость — включайте `Strict`-типы.
- Произвольные классы без аннотаций Pydantic не валидирует — поля без типа просто игнорируются как часть схемы.
- `ValidationError` собирает все ошибки сразу; не рассчитывайте, что он упадёт на первой.
- В Pydantic v2 многое переименовано (`conint` остался, но рекомендуют `Annotated[int, Field(gt=0)]`); следите за версией в проекте.
