# Можно ли навесить декоратор вручную, без знака @?

## Суть

Да. `@decorator` над функцией — это лишь сахар для `func = decorator(func)`. Тот же результат получается прямым вызовом декоратора и переприсваиванием имени.

## Что ответить на собесе

Декоратор — обычная функция, которая принимает функцию и возвращает новую. Запись `@my_decorator` перед `def func` Python разворачивает в `func = my_decorator(func)` сразу после определения. Значит, можно обойтись без `@`: определить функцию, потом вызвать декоратор и присвоить результат обратно тому же имени. Это полезно, когда нужно навешивать декоратор условно, выбирать его в рантайме или применять к функции, объявленной не нами.

## Как это работает

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Перед вызовом")
        result = func(*args, **kwargs)
        print("После вызова")
        return result
    return wrapper


def say_hello():
    print("Hello!")


say_hello = my_decorator(say_hello)
say_hello()
```

Строка `say_hello = my_decorator(say_hello)` — это ровно то, что сделал бы `@my_decorator`. Имя `say_hello` теперь указывает на `wrapper`. Вызов напечатает «Перед вызовом», затем «Hello!», затем «После вызова».

Ручное применение естественно ложится на условную логику:

```python
def trace(func):
    def wrapper(*args, **kwargs):
        print("вызов:", func.__name__)
        return func(*args, **kwargs)
    return wrapper


def compute(x):
    return x * 2


DEBUG = True
if DEBUG:
    compute = trace(compute)

print(compute(21))
```

Так декоратор подключается только при `DEBUG`, чего синтаксисом `@` сделать нельзя.

## Подводные камни

- Имя слева должно совпадать с тем, что вызывается дальше: если присвоить `wrapped = my_decorator(say_hello)`, то старое `say_hello` останется недекорированным.
- `@` применяется один раз в момент определения функции; ручной способ можно вызвать когда угодно и даже несколько раз — повторное оборачивание накладывает декоратор поверх декоратора.
- Метаданные функции теряются одинаково в обоих способах. Чтобы `wrapper` сохранил `__name__` и `__doc__` оригинала, оборачивайте его в `functools.wraps(func)`.
- Декоратор с аргументами — это фабрика: вручную пишется как `func = decorator(arg)(func)`, в два вызова.
