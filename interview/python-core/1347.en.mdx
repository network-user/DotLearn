# Can you apply a decorator manually, without the @ sign?

## In short

Yes. `@decorator` over a function is just sugar for `func = decorator(func)`. The same result is obtained by calling the decorator directly and reassigning the name.

## How to answer in an interview

A decorator is an ordinary function that takes a function and returns a new one. Python expands `@my_decorator` before `def func` into `func = my_decorator(func)` right after the definition. So you can do without `@`: define the function, then call the decorator and assign the result back to the same name. This is useful when you need to apply a decorator conditionally, choose it at runtime, or apply it to a function we did not declare ourselves.

## How it works

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Перед вызовом")
        result = func(*args, **kwargs)
        print("После вызова")
        return result
    return wrapper


def say_hello():
    print("Hello!")


say_hello = my_decorator(say_hello)
say_hello()
```

The line `say_hello = my_decorator(say_hello)` is exactly what `@my_decorator` would do. The name `say_hello` now points to `wrapper`. The call prints "Перед вызовом", then "Hello!", then "После вызова".

Manual application fits naturally with conditional logic:

```python
def trace(func):
    def wrapper(*args, **kwargs):
        print("вызов:", func.__name__)
        return func(*args, **kwargs)
    return wrapper


def compute(x):
    return x * 2


DEBUG = True
if DEBUG:
    compute = trace(compute)

print(compute(21))
```

This way the decorator is hooked up only when `DEBUG` is set, which the `@` syntax cannot do.

## Pitfalls

- The name on the left must match the one called afterward: if you assign `wrapped = my_decorator(say_hello)`, the old `say_hello` stays undecorated.
- `@` is applied once at the moment the function is defined; the manual approach can be invoked whenever you like, and even several times - repeated wrapping stacks a decorator on top of a decorator.
- Function metadata is lost the same way in both approaches. For `wrapper` to keep the original's `__name__` and `__doc__`, wrap it in `functools.wraps(func)`.
- A decorator with arguments is a factory: manually it is written as `func = decorator(arg)(func)`, in two calls.
