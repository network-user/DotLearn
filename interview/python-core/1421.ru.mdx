# Что именно возвращает декоратор и чем заменяется исходная функция

## Суть

Декоратор возвращает функцию — как правило, обёртку, которая внутри вызывает оригинал и довешивает поведение. Имя исходной функции после `@decorator` начинает ссылаться именно на эту обёртку.

## Что ответить на собесе

Декоратор возвращает функцию, и именно она встаёт под имя оригинала.

- `@deco` над `def f` - сахар для `f = deco(f)`, поэтому возвращаемое значение декоратора окажется под именем `f`.
- Обычно возвращают внутреннюю `wrapper`, которая вызывает оригинал и добавляет логику до и после.
- Чтобы wrapper не «съел» аргументы и результат, пишут `def wrapper(*args, **kwargs): return func(*args, **kwargs)`, а метаданные сохраняют через `functools.wraps`.

## Как это работает

Декоратор получает исходную функцию, оборачивает её и возвращает обёртку:

```python
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("До вызова")
        result = func(*args, **kwargs)
        print("После вызова")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    return f"Привет, {name}!"

print(say_hello("Мир"))
```

Шаги:

1. `my_decorator(say_hello)` выполняется в момент объявления — возвращает `wrapper`.
2. Имя `say_hello` теперь указывает на `wrapper`, а не на исходную функцию.
3. Вызов `say_hello("Мир")` запускает `wrapper`: печатает «До вызова», вызывает оригинал, печатает «После вызова» и возвращает его результат.
4. `functools.wraps` копирует `__name__`, `__doc__` и прочее с оригинала на обёртку.

## Подводные камни

- Если wrapper не возвращает `func(...)`, декорированная функция всегда отдаёт `None` — частая ошибка.
- Wrapper без `*args, **kwargs` сломается на функциях с аргументами.
- Без `functools.wraps` у функции «потеряются» имя и докстрока: `say_hello.__name__` станет `'wrapper'`, что мешает интроспекции и отладке.
- Декоратор выполняется один раз при объявлении; путать это с выполнением на каждый вызов — типичное заблуждение.
