# Что такое итератор и какие итераторы встречаются в Python?

## Суть

Итератор — объект, реализующий протокол `__iter__` и `__next__`, который выдаёт элементы по одному и помнит, где остановился. Это позволяет обходить данные без загрузки всей коллекции в память.

## Что ответить на собесе

Итератор — это объект, у которого есть `__next__()`, отдающий следующий элемент, и `__iter__()`, возвращающий сам итератор. Когда элементы кончаются, `__next__()` бросает `StopIteration`, и цикл `for` это ловит. Сам по себе список — итерируемый, но не итератор; итератор из него делает `iter()`. В стандартной библиотеке итераторами являются генераторы (функции с `yield`), а также ленивые результаты `map()`, `filter()`, `zip()`, `enumerate` и объекты из `itertools`.

Протокол итерации держится на двух методах:

- `__iter__()` — возвращает сам итератор, чтобы объект годился для `for`.
- `__next__()` — отдаёт очередной элемент, а на исчерпании поднимает `StopIteration`.

Свой итератор пишется так:

```python
class MyIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.index < len(self.data):
            result = self.data[self.index]
            self.index += 1
            return result
        raise StopIteration


for item in MyIterator([1, 2, 3]):
    print(item)
```

Встроенные источники итераторов:

```python
nums = iter([1, 2, 3])
print(next(nums))  # 1

squared = map(lambda x: x ** 2, [1, 2, 3])
print(list(squared))  # [1, 4, 9]

evens = filter(lambda x: x % 2 == 0, [1, 2, 3, 4])
print(list(evens))  # [2, 4]
```

Генератор — самый компактный способ получить итератор: `yield` сам реализует оба метода протокола.

## Подводные камни

- Итерируемый объект (`iterable`) и итератор — разные вещи: список итерируемый, но обходить его методом `next()` нельзя, пока не сделаешь `iter()`.
- Итератор одноразовый: дойдя до `StopIteration`, он не перезапускается. Для повторного прохода нужен новый.
- `map`, `filter`, `zip` в Python 3 ленивые — забыв обернуть их в `list()`, легко получить пустоту после первого же обхода.
- В `__next__` обязательно поднимайте именно `StopIteration`, иначе цикл `for` зациклится или упадёт с другой ошибкой.
