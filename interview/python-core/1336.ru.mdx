# Итератор против генератора: где проходит граница?

## Суть

Итератор - любой объект, реализующий пару методов `__iter__()` и `__next__()`. Генератор - это уже готовый итератор, который Python собирает за вас из функции с `yield`. То есть всякий генератор является итератором, но не наоборот.

## Что ответить на собесе

Всякий генератор - это итератор, но не наоборот: генератор лишь самый удобный способ получить итератор.

- Итератор задаётся протоколом: вернуть себя через `__iter__` и выдавать элемент через `__next__`, поднимая `StopIteration` по исчерпании.
- Генератор - функция с `yield`: интерпретатор сам реализует протокол и хранит локальное состояние и точку остановки между вызовами.
- Для ленивого перебора больших или бесконечных потоков почти всегда берут генератор; руками `__next__` пишут редко, когда нужна нестандартная логика обхода.

## Как это работает

Оба подхода перебирают данные по одному элементу, не материализуя всю последовательность в памяти. Разница в том, кто отвечает за машинерию протокола.

Итератор вручную: вы сами держите состояние (индекс) и сами решаете, когда бросить `StopIteration`.

```python
class RangeIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.index >= len(self.data):
            raise StopIteration
        value = self.data[self.index]
        self.index += 1
        return value

for item in RangeIterator([1, 2, 3]):
    print(item)
```

Генератор делает ту же работу короче. Вызов функции возвращает объект-генератор, а каждый `yield` замораживает кадр выполнения и при следующем `next()` продолжает с того же места.

```python
def first_three():
    yield 1
    yield 2
    yield 3

for item in first_three():
    print(item)
```

## Подводные камни

- Генератор одноразовый: после полного прохода он исчерпан, повторный `for` не даст ничего. Чтобы пройти заново, создавайте новый объект генератора.
- `iter(obj)` и `next(obj)` - это вызовы дандер-методов; голый класс без `__next__` итератором не является, даже если у него есть `__iter__`.
- Не путайте итератор и итерируемое: список итерируем, но сам не итератор - `for` берёт у него свежий итератор через `__iter__`.
- Генератор хранит ссылки на свои локальные переменные, пока жив, - не держите тяжёлый генератор без надобности.
