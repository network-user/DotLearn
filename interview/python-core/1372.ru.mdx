# Какие встроенные типы данных есть в Python и как их сгруппировать

## Суть

Встроенные типы удобно держать в голове не списком, а группами: числа, логика, текст, коллекции и «пустота». В каждую группу входит несколько конкретных типов, и почти любые данные раскладываются по этим полкам.

## Что ответить на собесе

Встроенные типы удобно держать группами, а не списком.

- Числовые (`int`, `float`, `complex`), логический `bool`, строковый `str`, коллекции (`list`, `tuple`, `set`, `dict`) и `NoneType` с единственным значением `None`.
- Коллекции делятся на изменяемые (`list`, `set`, `dict`) и неизменяемые (`tuple`, `str`).
- `bool` - подкласс `int`, поэтому `True` равен `1`, а `False` равен `0`.

Числа покрывают три случая: целые без ограничения по разрядности, дробные с плавающей точкой и комплексные.

```python
a = 5          # int
b = 3.14       # float
c = 2 + 3j     # complex
```

Логика — это всего два значения, но технически `bool` наследуется от `int`:

<PyDemo

  code={`print(True + True)   # 2
print(isinstance(True, int))  # True`}
/>

Текст хранится в неизменяемой строке `str`, а коллекции отличаются изменяемостью и наличием порядка:

```python
fruits = ["apple", "banana"]   # list: изменяемый, упорядоченный
point = (10.0, 20.0)            # tuple: неизменяемый
unique = {1, 2, 3}             # set: уникальные элементы, без порядка
person = {"name": "Alice"}     # dict: пары ключ-значение
```

Отсутствие значения выражается через `None` типа `NoneType` — это синглтон, поэтому его сравнивают через `is`, а не `==`.

```python
result = None
print(result is None)  # True
```

## Подводные камни

- `bool` — подкласс `int`, поэтому `True == 1` и список `[True, 1]` в множестве схлопнется в один элемент.
- Ключами `dict` и элементами `set` могут быть только хешируемые объекты: `list` туда положить нельзя, а `tuple` из неизменяемых элементов — можно.
- `is` для проверки на `None` обязателен; для чисел и строк `is` использовать нельзя — это деталь кеширования, а не равенство значений.
- `float` хранит приближение: `0.1 + 0.2 != 0.3`.
