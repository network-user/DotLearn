# Можно ли поменять отдельный символ строки прямо в ней самой?

## Суть

Нельзя: строки в Python неизменяемы (immutable). Любая правка одного символа на деле создаёт новую строку, а старая остаётся нетронутой.

## Что ответить на собесе

Нельзя: строка - неизменяемый объект, присвоить `s[1] = 'a'` нельзя, это поднимет `TypeError`.

- Изменённый вариант собирают как новую строку через срезы и конкатенацию.
- Либо временно переходят к списку символов и склеивают обратно через `"".join(...)`.
- Неизменяемость даёт строкам хешируемость, безопасность при передаче и возможность интернирования.

## Как это работает

Поскольку модифицировать на месте нельзя, мы конструируем новую строку из кусков вокруг нужной позиции.

<PyDemo title="Замена символа создаёт новую строку" autoRun code={`original = "hello"

# заменить символ с индексом 1 ('e') на 'a'

Если правок много, удобнее конвертировать в список, отредактировать и склеить обратно:

```python
chars = list("hello")
chars[1] = 'a'
result = "".join(chars)   # 'hallo'
```

Так каждая операция явно показывает: исходная строка остаётся прежней, а итог - это новый объект.

## Подводные камни

- **`s[i] = x` всегда падает.** Это не опечатка в синтаксисе, а принципиальный запрет: `TypeError: 'str' object does not support item assignment`.
- **Срез не меняет оригинал.** `original[:1]` возвращает копию-подстроку, исходная переменная остаётся прежней, пока вы не присвоите ей новое значение.
- **Конкатенация в цикле дорога.** Сборка строки через `+=` в длинном цикле создаёт промежуточные объекты на каждой итерации; для массовой склейки используйте `"".join(...)`.
- **Индекс за пределами в срезе не падает.** `original[2:]` для короткой строки просто вернёт пустую строку, тогда как прямое обращение `original[10]` поднимет `IndexError`.
