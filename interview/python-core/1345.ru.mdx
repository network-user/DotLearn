# Класс в роли декоратора: без какого метода ничего не заработает?

## Суть

Чтобы экземпляр класса можно было использовать как декоратор, класс обязан определять `__call__`. Именно этот метод делает экземпляр вызываемым со скобками, а декоратор по сути и есть вызываемый объект.

## Что ответить на собесе

Декоратор - любой вызываемый объект, принимающий функцию и возвращающий функцию (или вызываемое). У класса-декоратора ключевой метод - `__call__`.

- `__init__` запоминает оборачиваемую функцию, а `__call__` исполняется при каждом вызове декорированной функции - там и живёт дополнительная логика.
- Без `__call__` экземпляр нельзя вызвать со скобками, и декорирование не сработает.
- Плюс класса перед функцией-декоратором - он удобно хранит состояние между вызовами.

## Как это работает

`@MyDecorator` над функцией эквивалентно `func = MyDecorator(func)`. Сначала срабатывает `__init__`, сохраняя функцию. Дальше каждый вызов уже декорированного имени уходит в `__call__`.

```python
class CallCounter:
    def __init__(self, func):
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"Вызов №{self.count}")
        return self.func(*args, **kwargs)


@CallCounter
def greet(name):
    return f"Привет, {name}"


print(greet("Аня"))
print(greet("Боря"))
```

Поскольку состояние (`count`) живёт в экземпляре, оно сохраняется между вызовами: первый вызов напечатает «Вызов №1», второй — «Вызов №2». В функции-декораторе для этого пришлось бы городить замыкание или внешнюю переменную.

## Подводные камни

- Без `__call__` экземпляр не вызвать — будет `TypeError: object is not callable`.
- `*args, **kwargs` в `__call__` нужны, чтобы прокидывать любые аргументы декорируемой функции, иначе декоратор работает только для конкретной сигнатуры.
- Класс-декоратор заменяет функцию экземпляром, поэтому теряются `__name__`, `__doc__` и метаданные. Восстановить их сложнее, чем в функции-декораторе (там помогает `functools.wraps`).
- Один экземпляр декоратора оборачивает одну функцию: при навешивании на несколько функций для каждой создаётся свой экземпляр со своим состоянием.
