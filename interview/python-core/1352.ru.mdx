# Как написать декоратор, который логирует имя вызываемой функции

## Суть

Декоратор - это обёртка: он принимает функцию и возвращает новую функцию с дополнительным поведением. Чтобы логировать имя перед вызовом, достаточно во внутренней обёртке вывести `func.__name__` и затем передать управление оригиналу.

## Что ответить на собесе

Декоратор получает функцию аргументом и возвращает обёртку, которая логирует имя и зовёт оригинал.

- Внутри объявляю `wrapper(*args, **kwargs)`, печатаю `func.__name__`, затем возвращаю `func(*args, **kwargs)` - так сохраняются поведение и результат.
- `@decorator` над функцией - это просто сахар для `f = decorator(f)`.
- Логирование добавляется, не трогая код самой функции.

## Как это работает

Обёртка замыкает ссылку на исходную функцию и прокидывает любые аргументы через `*args`/`**kwargs`, а имя берёт из атрибута `__name__`.

<PyDemo title="Декоратор логирует имя вызываемой функции" autoRun code={`import functools


def log_call(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Calling function: {func.__name__}")
        return func(*args, **kwargs)

    return wrapper


@log_call
def greet(name):
    return f"Hello, {name}"


print(greet("Ann"))`} />

Что здесь происходит по шагам:

1. `log_call` получает `func` и возвращает `wrapper`.
2. `wrapper` печатает имя через `func.__name__`, затем вызывает оригинал и возвращает его результат.
3. `*args, **kwargs` пропускают любые позиционные и именованные аргументы без потерь.
4. Запись `@log_call` эквивалентна `greet = log_call(greet)`.
5. `functools.wraps` копирует метаданные (`__name__`, `__doc__`) с оригинала на обёртку.

## Подводные камни

- Без `functools.wraps` обёрнутая функция начнёт «представляться» как `wrapper`: `greet.__name__` вернёт `"wrapper"`, что ломает интроспекцию и другие декораторы, читающие имя.
- Забытый `return func(...)` молча превратит результат в `None` - частая ошибка.
- Если нужен декоратор с параметрами (`@log_call(level="info")`), потребуется ещё один уровень вложенности: фабрика, возвращающая сам декоратор.
- Несколько декораторов применяются снизу вверх: ближайший к функции оборачивает её первым.
