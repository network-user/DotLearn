# Why can't you access a generator by index?

## In short

A generator does not support `gen[i]`: it does not store its elements and produces them one at a time, so you can only reach a particular one by iterating. An attempt to index it raises `TypeError: 'generator' object is not subscriptable`.

## How to answer in an interview

Indexing requires the `__getitem__` protocol, which a generator does not have - it implements only the iterator protocol (`__next__`). A generator computes values lazily and does not keep past ones in memory, so the notion of "element number N" simply does not exist for it without iteration. If I need a specific element by position, I either iterate the generator myself, or use `itertools.islice`, or, if there is little data, materialize it into a list with `list()` and then work with the list.

## How it works

A generator is created by a function with `yield` or by a generator expression. Calling the function returns a generator object:

```python
def count_up_to(limit):
    count = 1
    while count <= limit:
        yield count
        count += 1
```

Direct indexing is not possible:

```python
gen = count_up_to(5)
# gen[2]  ->  TypeError: 'generator' object is not subscriptable
```

You can get an element by position via iteration:

```python
gen = count_up_to(5)
for i, value in enumerate(gen):
    if i == 2:
        print(value)  # 3
        break
```

A more concise way is `itertools.islice`, which takes a slice of an iterator:

```python
from itertools import islice

gen = count_up_to(5)
print(next(islice(gen, 2, 3)))  # 3
```

## Pitfalls

- `islice` and manual iteration still "advance" the generator: the passed elements are lost and you cannot return to them.
- Slice syntax `gen[1:3]` does not work either - for that you need `islice(gen, 1, 3)`.
- Materializing into a list with `list()` restores indexing, but loads everything into memory and is not suitable for infinite generators.
- Negative indices (`gen[-1]`) are fundamentally impossible: the end is unknown until the generator is exhausted.
