# Как в Python устроено преобразование типов: явное против неявного?

## Суть

Преобразование типа - это получение значения нужного типа из значения другого. Бывает явным, когда тип задаёт сам программист через `int()`, `float()`, `str()`, и неявным, когда интерпретатор сам согласует типы внутри операции, например приводит `int` к `float` при сложении.

## Что ответить на собесе

Есть две формы преобразования - явная и неявная.

- Явная: сам вызываешь конструктор типа - `int("42")`, `float(7)`, `str(5)` - и получаешь новый объект нужного типа.
- Неявная: интерпретатор сам расширяет тип в смешанной арифметике, чтобы не терять точность - `5 + 2.5` даёт `float`.
- Между несовместимыми типами неявных преобразований нет: число и строку сложить нельзя, будет `TypeError`. Неявность ограничена числовой башней (`int` -> `float` -> `complex`).

## Как это работает

Явное преобразование - через встроенные конструкторы:

```python
int("42")     # 42   строка -> целое
str(42)       # "42" целое -> строка
float(42)     # 42.0 целое -> вещественное
```

Неявное - интерпретатор сам подтягивает менее «широкий» тип к более «широкому»:

```python
integer_number = 5
float_number = 2.5
result = integer_number + float_number  # 7.5, тип float
```

Типичное место явного приведения - пользовательский ввод и данные из файлов, где всё приходит строкой:

```python
raw = "10 20 30"
numbers = list(map(int, raw.split()))  # [10, 20, 30]
```

## Подводные камни

- `int()` от строки требует чистого целого без точки: `int("3.5")` упадёт с `ValueError`, нужно `int(float("3.5"))`.
- `bool` - подкласс `int`, поэтому `True + 1` даёт `2`; это источник скрытых багов при подсчётах.
- Неявного приведения строки к числу нет: `"2" + 2` - это `TypeError`, а не `4`.
- `int(3.9)` отбрасывает дробь и даёт `3` (усечение к нулю), а не округление - для округления берите `round`.
- Преобразование создаёт новый объект, исходный не меняется: имена остаются связанными со старым значением, пока вы их не переприсвоите.
