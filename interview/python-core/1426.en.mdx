# How does type conversion work in Python: explicit versus implicit?

## In short

Type conversion is obtaining a value of the desired type from a value of another type. It can be explicit, when the type is specified by the programmer through `int()`, `float()`, `str()`, and implicit, when the interpreter reconciles the types within an operation on its own — for example, promoting `int` to `float` during addition.

## How to answer in an interview

Python has two forms. Explicit — you call the type constructor yourself: `int("42")`, `float(7)`, `str(5)` — and get a new object of the desired type. Implicit — the interpreter widens the type on its own in mixed arithmetic so as not to lose precision: `5 + 2.5` yields a `float`. Important: Python does not perform implicit conversions between incompatible types — you cannot add a number and a string, you get a `TypeError`. So implicitness is mostly limited to the numeric tower (`int` -> `float` -> `complex`).

## How it works

Explicit conversion — via the built-in constructors:

```python
int("42")     # 42   string -> integer
str(42)       # "42" integer -> string
float(42)     # 42.0 integer -> float
```

Implicit — the interpreter promotes the less "wide" type to the wider one on its own:

```python
integer_number = 5
float_number = 2.5
result = integer_number + float_number  # 7.5, type float
```

A typical place for explicit conversion is user input and data from files, where everything arrives as a string:

```python
raw = "10 20 30"
numbers = list(map(int, raw.split()))  # [10, 20, 30]
```

## Pitfalls

- `int()` from a string requires a clean integer with no decimal point: `int("3.5")` fails with `ValueError`, you need `int(float("3.5"))`.
- `bool` is a subclass of `int`, so `True + 1` gives `2`; this is a source of hidden bugs in calculations.
- There is no implicit conversion from string to number: `"2" + 2` is a `TypeError`, not `4`.
- `int(3.9)` drops the fractional part and gives `3` (truncation toward zero), not rounding — for rounding use `round`.
- Conversion creates a new object; the original does not change: names stay bound to the old value until you reassign them.
