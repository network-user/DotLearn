# Why plain datetime is not enough for time zones and why pytz exists

## In short

The standard library's `datetime` can store a date and time and compute intervals, but out of the box it knows nothing about time zones. `pytz` is a third-party database of real zones that attaches the correct offset and daylight-saving-time logic to `datetime` objects.

## How to answer in an interview

On its own `datetime` creates "naive" objects with no zone attached: it computes intervals, formats and parses, but it does not care about the UTC offset or DST. To get an "aware" object you need a `tzinfo` implementation. Historically this came from `pytz`, because it ships the full IANA database and handles DST transitions correctly. In Python 3.9+ the built-in `zoneinfo` was added for this purpose, which reads the same database and is preferable in new projects.

## How it works

Without a zone, `datetime` knows nothing about the offset:

```python
from datetime import datetime

naive = datetime(2023, 10, 5, 14, 30)
print(naive)          # 2023-10-05 14:30:00
print(naive.tzinfo)   # None
```

`pytz` provides a zone object that, via `astimezone`, recomputes the time and sets the offset:

```python
from datetime import datetime, timezone
import pytz

utc_now = datetime(2023, 10, 5, 18, 30, tzinfo=timezone.utc)

ny_tz = pytz.timezone('America/New_York')
ny_time = utc_now.astimezone(ny_tz)

print(ny_time)  # 2023-10-05 14:30:00-04:00
```

Here `astimezone` converts the moment in time into the New York zone and attaches the `-04:00` offset. Knowing the zone lets you correctly show the same instant to users from different regions.

## Pitfalls

- Do not call `pytz.timezone(...)` and immediately plug it into the `datetime(..., tzinfo=tz)` constructor - with `pytz` this gives a stale offset (LMT). The correct way is `tz.localize(naive)` or `astimezone`.
- `datetime.now()` with no argument returns a naive local time; for zones use `datetime.now(tz)`.
- In Python 3.9+ prefer `zoneinfo.ZoneInfo('America/New_York')` - it does not require `localize` and works with the constructor directly.
- Naive and aware objects cannot be compared or subtracted from each other - you get a `TypeError`.
