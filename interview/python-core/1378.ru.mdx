# Декораторы: как навесить поведение на функцию, не трогая её код

## Суть

Декоратор — это функция, которая принимает другую функцию и возвращает новую, оборачивающую исходную дополнительным поведением. Запись `@decorator` над определением — синтаксический сахар для `func = decorator(func)`.

## Что ответить на собесе

Декоратор опирается на то, что функции в Python - объекты первого класса: их можно передавать и возвращать.

- Берёт функцию аргументом, строит внутри обёртку `wrapper` (что-то до и после вызова оригинала) и возвращает её вместо исходной.
- Запись `@timing` ровно эквивалентна `func = timing(func)`.
- Так выносят сквозную логику - логирование, замер времени, кеширование, проверку прав - в одно место и навешивают на любую функцию, не меняя её тело.

## Как это работает

Обёртка принимает `*args, **kwargs`, чтобы прозрачно прокидывать любые аргументы в оригинал и возвращать его результат:

```python
import functools
import time

def timing(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter() - start:.4f}s")
        return result
    return wrapper

@timing
def work(n):
    return sum(range(n))

work(1000)
```

`@functools.wraps(func)` копирует `__name__`, `__doc__` и прочие метаданные оригинала на обёртку — без него `work.__name__` стал бы `"wrapper"`.

## Подводные камни

- Без `functools.wraps` теряются `__name__` и `__doc__` функции — ломаются интроспекция, документация и иногда фреймворки, опирающиеся на имя.
- Обёртка обязана принимать `*args, **kwargs` и возвращать `return func(...)`, иначе аргументы или результат потеряются.
- Декоратор применяется один раз — в момент определения функции, а не при каждом вызове. Тяжёлую инициализацию кладут в тело декоратора, а не в `wrapper`.
- Декоратор с аргументами (`@retry(times=3)`) — это уже три уровня вложенности: функция, возвращающая декоратор, возвращающий обёртку.
