# Декораторы в Python: зачем нужны и какие задачи решают

## Суть

Декоратор - это обёртка, которая добавляет функции или методу новое поведение, не переписывая их тело. Он принимает функцию, возвращает другую функцию и тем самым вклинивает свою логику до и после вызова оригинала.

## Что ответить на собесе

Декораторы позволяют отделить вспомогательную логику от основной. Я выношу повторяющиеся вещи - логирование, кеширование, проверку прав, замер времени, валидацию входа - в отдельный декоратор и навешиваю его на любые функции через синтаксис `@`. Под капотом это обычная функция высшего порядка: она получает целевую функцию аргументом и возвращает обёртку, которая вызывает оригинал, окружая его дополнительным кодом. Выигрыш - переиспользование, чистота основного кода и единая точка для правок.

## Как это работает

Декоратор берёт функцию и возвращает новую функцию, обычно вызывающую исходную внутри себя. Запись `@deco` над `def f` эквивалентна `f = deco(f)`.

```python
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Перед вызовом")
        result = func(*args, **kwargs)
        print("После вызова")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Привет, {name}!")
    return name

say_hello("Аня")
```

Разбор:

- `func` - функция, которую оборачиваем.
- `wrapper(*args, **kwargs)` - обёртка, прозрачно пробрасывающая любые аргументы.
- `result = func(...)` - запуск оригинала; результат сохраняем и возвращаем, чтобы не потерять.
- `@functools.wraps(func)` - переносит имя, docstring и сигнатуру оригинала на обёртку.
- `@my_decorator` - то же, что `say_hello = my_decorator(say_hello)`.

## Подводные камни

- Без `functools.wraps` у обёрнутой функции подменяются `__name__` и `__doc__` - ломаются интроспекция, документация и иногда сериализация.
- Если обёртка не пробрасывает `*args, **kwargs` и не возвращает результат, вы потеряете аргументы и возвращаемое значение.
- Декоратор выполняется один раз в момент определения функции, а обёртка - при каждом вызове; тяжёлую инициализацию делайте в теле декоратора, а не в `wrapper`.
- Декоратору с параметрами нужен дополнительный уровень вложенности (фабрика декораторов), это частая путаница.
