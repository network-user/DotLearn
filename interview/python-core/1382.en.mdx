# How Pydantic validates data against annotations and what exactly it guarantees

## In short

Pydantic treats ordinary Python type annotations as a specification and, when a model is created, validates the input data against it: it checks types, value constraints and structure, and where possible coerces the data to the required type. The input can be simple types, collections, nested models and custom types.

## How to answer in an interview

Pydantic describes a schema through classes that subclass `BaseModel` with annotated fields. At initialization time it runs each field through validation against its type and any extra constraints (`gt`, `max_length`, etc.). If a value doesn't fit, a `ValidationError` is raised that lists all the fields that failed and the reasons, not just the first error. An important detail: by default Pydantic coerces compatible values (the string `"30"` into `int`), and strict behavior is enabled via `Strict` types or `model_config`.

## How it works

```python
from pydantic import BaseModel, ValidationError, conint

class User(BaseModel):
    id: conint(gt=0)
    name: str
    age: int

user = User(id=1, name='Alice', age=30)
print(user.age)  # 30

try:
    User(id=-1, name='Bob', age='thirty')
except ValidationError as e:
    print(e)  # lists the failure on id (gt=0) and on age (not an int)
```

What Pydantic covers:

- **Simple types**: `int`, `float`, `str`, `bool`.
- **Collections**: `list`, `tuple`, `set`, `dict` with typed elements.
- **Nested models**: a `BaseModel` field is validated recursively.
- **Value constraints**: `conint`, `constr`, `Field(gt=..., max_length=...)`.

Validation runs automatically in the constructor, so once the object is successfully created the data is guaranteed to match the schema.

## Pitfalls

- By default `'30'` turns into `30` — that's not a bug but the lenient coercion mode. If you need strictness, enable `Strict` types.
- Pydantic doesn't validate arbitrary classes without annotations — fields without a type are simply ignored as part of the schema.
- `ValidationError` gathers all the errors at once; don't count on it failing on the first one.
- In Pydantic v2 a lot was renamed (`conint` remains, but `Annotated[int, Field(gt=0)]` is recommended); watch the version used in the project.
