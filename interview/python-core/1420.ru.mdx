# Перебор словаря парами: зачем нужен метод items()

## Суть

Когда в цикле нужны и ключ, и значение одновременно, вызывайте `.items()` — он отдаёт представление с парами, которое распаковывается прямо в заголовке `for`.

## Что ответить на собесе

Для одновременного доступа к ключу и значению беру `dict.items()`.

- Возвращает не список, а view-объект - динамическое представление, отражающее текущее состояние словаря.
- В цикле каждая пара распаковывается в две переменные: `for key, value in d.items():`.
- Только ключи - `for k in d` (или `.keys()`); только значения - `.values()`.

## Как это работает

`.items()` отдаёт объект `dict_items` — это представление (view), а не копия. Каждый элемент представления — это кортеж `(ключ, значение)`, который Python автоматически распаковывает в две переменные заголовка цикла.

<PyDemo title="Распаковка пар словаря через items()" autoRun code={`student_grades = {
    'Alice': 85,
    'Bob': 92,
    'Charlie': 78,
}

for name, grade in student_grades.items():
print(f"{name}: {grade}")`} />

Здесь `name` на каждом шаге получает ключ, а `grade` — связанное значение. Распаковка работает потому, что `.items()` выдаёт именно пары-кортежи. Без `.items()` обычный `for x in student_grades` пройдёт только по ключам.

Поскольку это view, а не снимок, представление видит изменения словаря в реальном времени:

```python
d = {'a': 1}
view = d.items()
d['b'] = 2
print(list(view))   # [('a', 1), ('b', 2)]
```

## Подводные камни

- Нельзя менять размер словаря (добавлять или удалять ключи) во время итерации по `.items()` — Python бросит `RuntimeError: dictionary changed size during iteration`. Менять значения существующих ключей при этом допустимо.
- `.items()` — это view, а не список: индексировать `d.items()[0]` нельзя, нужен `list(d.items())`.
- Распаковка `for name, grade in ...` упадёт с `ValueError`, если перебирать сам словарь, а не его `.items()` (по ключам распаковывать в две переменные нечего).
