# Изменяемый объект как значение по умолчанию: почему это классическая ловушка

## Суть

Значение по умолчанию вычисляется один раз - при определении функции, а не при каждом вызове. Если это изменяемый объект (список, словарь), он живёт между вызовами и накапливает изменения, ломая ожидания.

## Что ответить на собесе

Дефолтные аргументы создаются единожды, в момент компиляции `def`, и хранятся в `func.__defaults__`. Когда дефолт - изменяемый объект, все вызовы без явного аргумента делят один и тот же объект, и правки в нём переживают вызов. Лечение стандартное: в качестве дефолта ставят `None`, а реальный объект создают внутри тела функции при `arg is None`. Так каждый вызов получает свежую копию.

## Как это работает

Опасная версия - список как дефолт:

```python
def append_to_list(value, my_list=[]):
    my_list.append(value)
    return my_list

print(append_to_list(1))   # [1]
print(append_to_list(2))   # [1, 2]   <- не [2]!
print(append_to_list(3))   # [1, 2, 3]
```

Список `[]` создан один раз и переиспользуется во всех вызовах без второго аргумента - изменения утекают между ними. Убедиться можно, заглянув в дефолты:

```python
print(append_to_list.__defaults__)   # ([1, 2, 3],) - тот самый общий список
```

Безопасная версия через `None`:

```python
def append_to_list(value, my_list=None):
    if my_list is None:
        my_list = []
    my_list.append(value)
    return my_list

print(append_to_list(1))   # [1]
print(append_to_list(2))   # [2]
print(append_to_list(3))   # [3]
```

Теперь свежий список рождается при каждом вызове, и состояние не протекает.

## Подводные камни

- Та же ловушка с любым изменяемым дефолтом: `dict`, `set`, экземпляр класса. Правило про `None` распространяется на все.
- Проверяйте именно `if my_list is None`, а не `if not my_list`: пустой переданный список тоже ложен, и вы случайно подмените валидный аргумент.
- Иногда общий объект используют намеренно - как примитивный кэш или счётчик. Это работает, но опирается на побочный эффект и читается плохо; лучше явный кэш.
- Неизменяемые дефолты (`int`, `str`, `tuple`, `None`) безопасны: их нельзя поменять на месте, поэтому утечки состояния не возникает.
