# How do you swap two variables in a single line?

## In short

The idiom `a, b = b, a` swaps values without a temporary variable by packing the right-hand side into a tuple and unpacking it into the left-hand side.

## How to answer in an interview

In Python the swap is one line: `a, b = b, a`. No intermediate variable is needed, because the right side first evaluates the whole tuple `(b, a)` from the current values, and only then unpacks its elements into `a` and `b`. It's idiomatic, readable, and works for any number of variables.

## How it works

An assignment in Python evaluates the right-hand side before writing anything to the left. So the order is:

1. The right side builds the tuple `(b, a)` from the current values.
2. The left side lists the target names `a` and `b`.
3. The finished tuple is unpacked: the first element goes into `a`, the second into `b`.

```python
x = 5
y = 10

x, y = y, x

print("x:", x)  # x: 10
print("y:", y)  # y: 5
```

The same trick scales to any number of variables, for example `a, b, c = c, a, b`.

## Pitfalls

- The atomicity of evaluating the right-hand side is what matters: the values are taken before the assignment, so the order of names on the left and right doesn't "clobber" the data.
- When unpacking, the number of names on the left must match the number of values on the right, otherwise you get a `ValueError`.
- In chains with indices like `a[i], a[j] = a[j], a[i]`, the indices are also evaluated by tuple rules - this is safe for swapping list elements.
- Don't confuse it with `a = b = value`: that assigns one value to both, it doesn't swap them.
