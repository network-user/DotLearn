# Какие объекты Python годятся на роль декоратора?

## Суть

Декоратором может выступить любой вызываемый объект (callable): обычная функция, метод, класс или экземпляр с `__call__`. Главное требование - принять декорируемый объект и вернуть замену, которая встанет на его место.

## Что ответить на собесе

Декоратором может быть любой callable, который получает функцию (или класс) и возвращает замену.

- Чаще всего это функция, возвращающая обёртку-замыкание.
- Так же работают классы (их экземпляр заменяет исходную функцию) и объекты с методом `__call__`.
- Синтаксис `@dec` над `def f` - сахар для `f = dec(f)`; применяют для сквозной логики (логирование, кэш, проверка прав) без переписывания функции.

## Как это работает

Базовый случай - функция, которая внутри строит и возвращает обёртку:

Строка `@my_decorator` эквивалентна `say_hello = my_decorator(say_hello)`. После этого имя `say_hello` указывает уже на `wrapper`.

Декоратором может быть и класс - тогда его экземпляр заменяет функцию, а вызовы перехватывает `__call__`:

```python
class Counter:
    def __init__(self, func):
        self.func = func
        self.calls = 0
    def __call__(self, *args, **kwargs):
        self.calls += 1
        return self.func(*args, **kwargs)

@Counter
def ping():
    return "pong"
```

## Подводные камни

- Обёртка теряет `__name__`, `__doc__` и сигнатуру оригинала - используйте `functools.wraps`, иначе интроспекция и автодокументация ломаются.
- Обёртка должна пробрасывать `*args, **kwargs`, иначе декоратор сработает только для функций без аргументов.
- Не путайте callable-объект (это сам декоратор) с результатом его вызова: класс как декоратор оборачивает функцию своим экземпляром, а не возвращаемым значением.
- Декоратор выполняется один раз в момент определения функции, а не при каждом вызове - тяжёлую инициализацию кладите в тело декоратора, а не в `wrapper`.
