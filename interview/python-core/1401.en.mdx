# What is match/case and why isn't it just a switch?

## In short

`match/case` (Python 3.10+) is structural pattern matching: a value is tested against patterns, and the branch of the first match runs. Unlike a `switch`, it can take apart the structure of data and bind its parts to variables.

## How to answer in an interview

`match/case` appeared in 3.10 and implements pattern matching, not a simple choice by equality. Beyond comparing against constants, it can unpack sequences and dicts, check classes, and bind matched parts to free names right inside the pattern. The `case _` pattern catches everything else, like `default`. The main difference from a `switch`: I'm matching a value not against a value, but against the shape of the data.

## How it works

The simplest case - comparison against constants, where `_` plays the role of "otherwise":

```python
def describe(value):
    match value:
        case 0:
            return "Zero"
        case 1:
            return "One"
        case _:
            return "Other"

print(describe(0))   # Zero
print(describe(42))  # Other
```

The power of the construct is in destructuring. Names inside a pattern aren't compared - they're bound to the matched parts:

```python
def process(data):
    match data:
        case [x, y, z]:
            return f"three: {x}, {y}, {z}"
        case {"name": name, "age": age}:
            return f"{name}, {age}"
        case _:
            return "unknown"

print(process([1, 2, 3]))                    # three: 1, 2, 3
print(process({"name": "Alice", "age": 30})) # Alice, 30
```

Here `[x, y, z]` matches only a list of exactly three elements, while `{"name": ..., "age": ...}` matches a dict that has both keys (extra keys are allowed).

## Pitfalls

- A name in a pattern without a dot is a capture (binding), not a comparison. `case RED:` doesn't check the value against the variable `RED`, it overwrites `RED` with the matched value. For value comparison, use a dot: `case Color.RED:`.
- Patterns without a guard don't cover everything - without `case _`, an unmatched value simply falls through `match` with no error. Add `case _` if you need a default branch.
- A dict pattern matches when the required keys are present, even if the dict has extras - it's not strict equality.
- `match/case` is unavailable in Python 3.9 and below; on older runtimes the code won't even parse.
