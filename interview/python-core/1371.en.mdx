# What is a closure, and how does a function remember outer variables?

## In short

A closure is a nested function that keeps access to the outer function's variables even after the outer function has finished. The function effectively "carries the needed environment with it".

## How to answer in an interview

A closure arises when an inner function references a variable from the enclosing scope and that function "lives" longer than its creator. The captured variables are stored in cells, accessible via `__closure__`. Function factories and decorators are built on closures. If inside the closure you need to assign to an outer variable rather than just read it, you need `nonlocal`, otherwise Python creates a new local variable.

## How it works

The inner function sees the outer function's variables. When it is returned outward, the link to those variables is preserved:

```python
def make_adder(x):
    def adder(y):
        return x + y
    return adder

add_five = make_adder(5)   # the closure captured x = 5
print(add_five(10))        # 15
print(add_five.__closure__[0].cell_contents)  # 5
```

## Pitfalls

- The classic trap is capturing a loop variable: lambdas in a loop capture the variable itself, not its value at that step. The fix is to bind the value through a default argument: `lambda y, x=i: x + y`.
- To assign to an outer variable inside a closure you need `nonlocal`; without it you get a new local variable.
- A closure captures the variable by reference, not by copy: if the outer variable changes before the call, the closure sees the new value.
