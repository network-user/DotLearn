# itertools: what iterator building blocks does the standard library give you

## In short

`itertools` is a set of lazy functions over iterators: they don't materialize the result in memory but yield elements on demand. This lets you process huge or infinite data streams while spending memory only on the current element.

## How to answer in an interview

`itertools` splits its tools into three groups. Infinite generators: `count`, `cycle`, `repeat`. Functions that shorten or filter an input stream: `chain`, `compress`, `dropwhile`, `takewhile`, `groupby`, `islice`. And combinatorics: `product`, `permutations`, `combinations`. The key idea is that they all return a lazy iterator rather than a list, so you can assemble them into a pipeline and apply it to data that doesn't fit in memory all at once.

Infinite sources must be bounded from the outside (`next`, `islice`, or an exit condition), otherwise the loop will never finish.

```python
import itertools

counter = itertools.count(start=10, step=2)
print([next(counter) for _ in range(5)])      # [10, 12, 14, 16, 18]

cycler = itertools.cycle('AB')
print([next(cycler) for _ in range(5)])        # ['A', 'B', 'A', 'B', 'A']

print(list(itertools.repeat('Hi', 3)))         # ['Hi', 'Hi', 'Hi']
```

Filtering and joining a stream:

```python
import itertools

print(list(itertools.chain('ABC', 'DEF')))                 # ['A','B','C','D','E','F']
print(list(itertools.compress('ABCDEF', [1,0,1,0,1,0])))   # ['A', 'C', 'E']
print(list(itertools.dropwhile(lambda x: x < 3, [1,2,3,4,5])))  # [3, 4, 5]
print(list(itertools.takewhile(lambda x: x < 4, [1,2,3,4,5])))  # [1, 2, 3]
```

`groupby` groups only consecutive equal elements:

```python
import itertools

for key, group in itertools.groupby([1, 1, 2, 2, 3, 3]):
    print(key, list(group))   # 1 [1, 1] / 2 [2, 2] / 3 [3, 3]
```

Combinatorics - enumerating options without manual nested loops:

```python
import itertools

print(list(itertools.product('AB', repeat=2)))       # AA AB BA BB
print(list(itertools.permutations('ABC', 2)))        # AB AC BA BC CA CB
print(list(itertools.combinations('ABC', 2)))        # AB AC BC
```

## Pitfalls

- `count`, `cycle`, and `repeat()` without `times` are infinite: a `list()` over them will hang. Bound them with `islice`.
- `groupby` groups neighbors, not all equal values. To group an entire array, sort it first by the same key.
- An iterator is single-use: once you've walked it, a second pass yields nothing. If you need the result twice, save it to a list or use `itertools.tee`.
- `permutations` cares about order (`AB` != `BA`), `combinations` doesn't. Confusing the two is a classic mistake in combinatorial problems.
