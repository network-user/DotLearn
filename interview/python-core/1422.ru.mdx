# Перехватить исключение, отреагировать и пробросить его наверх

## Суть

Ловите исключение в `except`, выполняете локальные действия (лог, очистка), а потом голым `raise` без аргументов отправляете то же исключение дальше с нетронутым трейсбеком.

## Что ответить на собесе

Ошибку надо отработать, но не гасить - ловлю в `try/except` и внутри пишу голый `raise`.

- Голый `raise` повторно возбуждает текущее исключение, сохраняя оригинальный traceback.
- Это не то же, что `raise e`: пустой `raise` оставляет трейс как есть, а `raise e` или `raise NewError()` могут затереть исходную точку сбоя.
- Чтобы подменить тип ошибки, но не потерять причину, применяю `raise NewError(...) from e` - это связывает исключения цепочкой.

```python
import logging

def process_data(value):
    try:
        return 10 / value
    except ZeroDivisionError:
        logging.warning("Деление на ноль в process_data")
        raise

try:
    process_data(0)
except ZeroDivisionError:
    print("Ошибка обработана уровнем выше")
```

Что происходит по шагам:

1. `10 / value` при `value == 0` поднимает `ZeroDivisionError`.
2. Внутренний `except` ловит его и делает побочное действие — пишет лог.
3. Голый `raise` повторно возбуждает то самое исключение; traceback указывает на исходную строку деления.
4. Внешний `try/except` принимает проброшенную ошибку и обрабатывает её на своём уровне.

Так локальная реакция (логирование) и финальная обработка разнесены по разным уровням, а информация об ошибке не теряется.

## Подводные камни

- `raise e` вместо голого `raise` обрезает или искажает оригинальный traceback — труднее искать причину.
- Чтобы сменить тип исключения и сохранить контекст, используйте `raise NewError("...") from e`; без `from` Python всё равно покажет «During handling of the above exception», но явная цепочка читается лучше.
- `raise` без активного исключения (вне `except`) бросит `RuntimeError: No active exception to re-raise`.
- Логику очистки ресурсов, которая должна сработать при любом исходе, лучше выносить в `finally`, а не дублировать в `except`.
