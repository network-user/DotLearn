# range is an iterable, not an iterator: what's the difference and why it matters

## In short

`range` is not an iterator but a lazy iterable sequence. It has `__iter__`, which produces a fresh iterator on demand, but `range` itself has no `__next__`. That's why the same `range` can be iterated as many times as you like.

## How to answer in an interview

There are two distinct concepts. An iterable can hand out an iterator on demand via `__iter__` — these include lists, strings, `range`. An iterator yields elements one at a time via `__next__` and is single-use: once it reaches the end it is exhausted. `range` belongs to the first category: every call to `iter(range(5))` creates a new, independent iterator, while the `range` object itself stays unchanged and reusable. On top of that it stores only the bounds and the step, so it takes constant memory regardless of length.

## How it works

```python
r = range(5)

for i in r:
    print(i)        # 0 1 2 3 4

for i in r:
    print(i)        # 0 1 2 3 4 again - the object isn't exhausted
```

To get an actual iterator, pass `range` to `iter()`:

```python
r = range(5)
it = iter(r)

print(next(it))  # 0
print(next(it))  # 1
print(iter(it) is it)  # True - an iterator returns itself
```

You can check the difference directly:

```python
r = range(5)
print(hasattr(r, '__iter__'))   # True
print(hasattr(r, '__next__'))   # False - so it's not an iterator
```

## Pitfalls

- `next(range(5))` fails with `TypeError`: `range` itself has no `__next__`, you need `iter()` first.
- An iterator is single-use: once `it` is exhausted, iterating over it again with `for` yields nothing. A `range`, on the other hand, can be looped over many times.
- `range` supports indexing, slicing, `len()` and `in` — it's a full-fledged sequence, not just a generator of numbers.
- Don't confuse it with generators: a generator is both an iterable and an iterator (and single-use), while `range` is only an iterable.
