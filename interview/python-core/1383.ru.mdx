# range — итерируемый объект, а не итератор: в чём разница и почему это важно

## Суть

`range` — это не итератор, а итерируемая ленивая последовательность. У неё есть `__iter__`, который по запросу выдаёт свежий итератор, но самого `__next__` у `range` нет. Поэтому один и тот же `range` можно перебирать сколько угодно раз.

## Что ответить на собесе

`range` - итерируемый объект (iterable), а не итератор, поэтому переиспользуем.

- Iterable отдаёт итератор через `__iter__` (списки, строки, `range`); итератор выдаёт элементы через `__next__` и одноразов.
- Каждый `iter(range(5))` создаёт новый независимый итератор, а сам `range` остаётся неизменным.
- `range` хранит только границы и шаг, поэтому занимает константную память независимо от длины.

Чтобы получить именно итератор, передайте `range` в `iter()`:

```python
r = range(5)
it = iter(r)

print(next(it))  # 0
print(next(it))  # 1
print(iter(it) is it)  # True - итератор возвращает сам себя
```

Проверить разницу можно напрямую:

```python
r = range(5)
print(hasattr(r, '__iter__'))   # True
print(hasattr(r, '__next__'))   # False - значит не итератор
```

## Подводные камни

- `next(range(5))` упадёт с `TypeError`: у самого `range` нет `__next__`, сначала нужен `iter()`.
- Итератор одноразов: исчерпав `it`, повторный `for` по нему ничего не выдаст. `range` же можно гонять в цикле многократно.
- `range` поддерживает индексацию, срезы, `len()` и `in` — это полноценная последовательность, а не просто генератор чисел.
- Не путайте с генераторами: генератор одновременно и iterable, и iterator (и одноразов), а `range` — только iterable.
