# Iterator vs generator: where is the line?

## In short

An iterator is any object that implements the pair of methods `__iter__()` and `__next__()`. A generator is a ready-made iterator that Python builds for you out of a function with `yield`. In other words, every generator is an iterator, but not the other way around.

## How to answer in an interview

An iterator is defined by a protocol: the object must be able to return itself via `__iter__` and yield the next element via `__next__`, raising `StopIteration` once exhausted. A generator is a special, most convenient way to obtain an iterator: you write a function with `yield`, and the interpreter implements the protocol for you, keeping the local state and the suspension point between calls. That is why, for lazy iteration over large or infinite streams, you almost always reach for a generator, and you rarely write `__next__` by hand - only when you need non-standard traversal logic.

## How it works

Both approaches iterate over data one element at a time, without materializing the whole sequence in memory. The difference is who is responsible for the protocol machinery.

A manual iterator: you hold the state (the index) yourself and decide yourself when to raise `StopIteration`.

```python
class RangeIterator:
    def __init__(self, data):
        self.data = data
        self.index = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.index >= len(self.data):
            raise StopIteration
        value = self.data[self.index]
        self.index += 1
        return value

for item in RangeIterator([1, 2, 3]):
    print(item)
```

A generator does the same work more concisely. Calling the function returns a generator object, and each `yield` freezes the execution frame and, on the next `next()`, resumes from the same place.

```python
def first_three():
    yield 1
    yield 2
    yield 3

for item in first_three():
    print(item)
```

## Pitfalls

- A generator is single-use: after a full pass it is exhausted, and a repeated `for` yields nothing. To go through again, create a new generator object.
- `iter(obj)` and `next(obj)` are calls to dunder methods; a bare class without `__next__` is not an iterator, even if it has `__iter__`.
- Do not confuse an iterator with an iterable: a list is iterable but is not itself an iterator - `for` obtains a fresh iterator from it via `__iter__`.
- A generator holds references to its local variables for as long as it is alive - do not keep a heavy generator around without need.
