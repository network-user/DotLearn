# What are the ways to obtain a generator in Python

## In short

You can get a lazy iterator in Python in three ways: a function with `yield`, a generator expression in parentheses, and a custom class with `__iter__`/`__next__` methods. All three yield values on demand and don't hold the entire sequence in memory.

## How to answer in an interview

Most often I write a function with `yield` - it preserves its state between calls and hands out elements one at a time. For short transformations I use a generator expression `(x for x in ...)`; it's more compact than a list and doesn't materialize the result. If the iteration logic is complex and holds a lot of state, I write an iterator class with `__iter__` and `__next__` that ends the iteration via `raise StopIteration`. The main advantage of all the options is laziness: values are computed as they are requested.

## How it works

A function with `yield` pauses at each value and resumes from the same place on the next `next()`.

```python
def count_up_to(limit):
    count = 1
    while count <= limit:
        yield count
        count += 1
```

A generator expression is the same generator, but written in a single line in parentheses:

```python
squares = (x * x for x in range(10))
```

An iterator class gives full control over the state and the end-of-iteration signal:

```python
class Countdown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current
```

## Pitfalls

- A generator is single-use: once fully traversed it is exhausted, and iterating over it again yields nothing. To go through it again, you must create it anew.
- Don't confuse `(x for x in ...)` (a generator) with `[x for x in ...]` (a list): the latter immediately allocates memory for all elements.
- In an iterator class `__iter__` must return an object with `__next__`; if you return the same object, repeated iteration continues from the current, already-shifted state.
- A `return` inside a generator function does not return a value but ends the generator, turning into `StopIteration`.
