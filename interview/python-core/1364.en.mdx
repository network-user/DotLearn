# How does filter() help you select elements by a condition?

## In short

`filter(predicate, collection)` lazily walks the collection and keeps only the elements for which the predicate returned a truthy value. The result is an iterator, not a ready-made list.

## How to answer in an interview

`filter()` takes a predicate function and an iterable, runs the predicate over each element, and returns an iterator of the elements where the result was truthy. It's a lazy object: elements are computed on demand, so I wrap it in `list()` or `tuple()` when I need a materialized result. If you pass `None` instead of a function, `filter()` drops every element that is falsy by its boolean value.

## How it works

The signature is `filter(function, iterable)`. The predicate accepts a single element and returns a value that is treated as truthy or falsy. The output is an iterator object that can be traversed once.

```python
def is_even(number):
    return number % 2 == 0

numbers = [1, 2, 3, 4, 5, 6]

even = filter(is_even, numbers)
print(list(even))  # [2, 4, 6]
```

A special case is `None` as the first argument: then the truthiness of the element itself becomes the predicate.

```python
mixed = [0, 1, "", "hi", None, [], [1]]
print(list(filter(None, mixed)))  # [1, 'hi', [1]]
```

`filter()` is often replaced by a generator expression, which reads more naturally: `(n for n in numbers if is_even(n))`.

## Pitfalls

- The iterator from `filter()` is single-use: after the first full pass it's empty, and a second `list()` returns `[]`.
- On its own, `filter()` computes nothing until you start iterating over it - that's laziness, not an instant result.
- In Python 2, `filter()` returned a list; code that relied on this breaks in Python 3 because of the switch to an iterator.
- The predicate should only check a condition and have no side effects, otherwise the laziness makes the behavior non-obvious.
