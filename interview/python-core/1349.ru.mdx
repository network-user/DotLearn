# Динамика и строгость типов: к какой модели типизации относится Python

## Суть

Python сочетает динамическую и сильную типизацию: тип привязан к значению и выясняется в рантайме, при этом язык не делает тихих приведений между несовместимыми типами.

## Что ответить на собесе

Типизация в Python динамическая и сильная.

- Динамическая: тип не объявляется заранее и определяется в рантайме по значению; одно имя может сегодня держать `int`, завтра `str`.
- Сильная: язык не складывает молча несовместимые типы - `1 + "2"` даст `TypeError`, а не «3» и не «12», преобразование всегда явное.
- Это отличает Python от слаботипизированных языков вроде JavaScript, где такие выражения «как-нибудь» приводятся.

## Как это работает

Динамика: тип принадлежит объекту, а имя - это просто ссылка, которую можно переназначить на значение другого типа.

```python
x = 10        # сейчас x ссылается на int
x = "Hello"   # теперь x ссылается на str
```

Строгость: операция над несовместимыми типами падает с ошибкой, а не угадывает намерение.

```python
result = 10 + "20"   # TypeError: unsupported operand type(s)
```

Чтобы сложить, приводим явно:

```python
result = 10 + int("20")   # 30
```

## Подводные камни

- Динамическая типизация - не отсутствие типов: каждый объект всегда имеет конкретный тип, проверить который можно через `type()` или `isinstance()`.
- Аннотации типов (`x: int`) и mypy - это статическая проверка поверх языка; сам интерпретатор их в рантайме не навязывает.
- «Сильная» и «статическая» - разные оси: C статически и относительно сильно типизирован, Python динамически и сильно, JS динамически и слабо.
- Не путайте `==` (сравнение значений) с приведением: `1 == 1.0` истинно, но это не значит, что `int` неявно стал `float` где попало.
