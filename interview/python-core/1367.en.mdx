# Which collections can you build with a comprehension?

## In short

A comprehension is a compact way to build a new collection from an iterable. There are four forms: list, set, dict, and generator; they differ only in the brackets.

## How to answer in an interview

Comprehensions exist for lists `[...]`, sets `{...}`, dictionaries `{k: v ...}`, and generators `(...)`. They share the same skeleton: an expression, a `for` loop, and an optional `if` filter. The list, set, and dict forms build the whole collection at once, while the generator form returns a lazy object that computes elements on demand and saves memory. There is no tuple comprehension: `(...)` is a generator expression, not a tuple.

## How it works

One template - four results depending on the brackets:

```python
nums = range(5)

[x ** 2 for x in nums]          # list:        [0, 1, 4, 9, 16]
{x ** 2 for x in nums}          # set:         {0, 1, 4, 9, 16}
{x: x ** 2 for x in nums}       # dict:        {0: 0, 1: 1, ...}
(x ** 2 for x in nums)          # generator:   lazy object
```

Adding a filter:

```python
even_squares = [x ** 2 for x in range(10) if x % 2 == 0]  # [0, 4, 16, 36, 64]
```

A set discards duplicates, a dict keeps the last value when keys collide, and a generator builds nothing until you start iterating over it.

## Pitfalls

- Parentheses produce a **generator**, not a tuple - it can be traversed only once, after which it is empty.
- Set and dict comprehensions use the same curly braces; the presence of `key: value` tells them apart. Empty `{}` is a dictionary, not a set.
- In the dict form, repeated keys silently overwrite each other - the last value wins.
- Long nested comprehensions quickly lose readability; sometimes a plain loop is clearer.
- The loop variable inside a comprehension does not "leak" outside (unlike Python 2): after `[x for x in ...]` the name `x` is undefined outside.
