# Decorators: adding behavior to a function without touching its code

## In short

A decorator is a function that takes another function and returns a new one that wraps the original with extra behavior. The `@decorator` notation above a definition is syntactic sugar for `func = decorator(func)`.

## How to answer in an interview

A decorator relies on the fact that functions in Python are first-class objects: they can be passed around and returned. The decorator takes a function as an argument, builds a `wrapper` inside that does something before and after calling the original, and returns that wrapper instead of the original function. The `@timing` notation is exactly equivalent to `func = timing(func)`. This is how cross-cutting logic - logging, timing, caching, permission checks - is factored into one place and applied to any function without changing its body.

## How it works

The wrapper accepts `*args, **kwargs` so it can transparently forward any arguments to the original and return its result:

```python
import functools
import time

def timing(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {time.perf_counter() - start:.4f}s")
        return result
    return wrapper

@timing
def work(n):
    return sum(range(n))

work(1000)
```

`@functools.wraps(func)` copies `__name__`, `__doc__` and other metadata from the original onto the wrapper - without it `work.__name__` would become `"wrapper"`.

To convince yourself that `@` is just an assignment, expand it by hand:

```python
def shout(func):
    def wrapper(*a, **k):
        return func(*a, **k).upper()
    return wrapper

def greet(name):
    return f"hi {name}"

greet = shout(greet)   # same as @shout
print(greet("ann"))    # HI ANN
```

## Pitfalls

- Without `functools.wraps` the function's `__name__` and `__doc__` are lost - this breaks introspection, documentation, and sometimes frameworks that rely on the name.
- The wrapper must accept `*args, **kwargs` and `return func(...)`, otherwise arguments or the result are lost.
- A decorator runs once - at the moment the function is defined, not on every call. Heavy initialization goes into the decorator body, not into `wrapper`.
- A decorator with arguments (`@retry(times=3)`) is already three levels of nesting: a function that returns a decorator that returns a wrapper.
