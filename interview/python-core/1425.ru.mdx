# В чём разница между @dec и @dec() при навешивании декоратора?

## Суть

`@dec` применяет сам `dec` как декоратор: он получает функцию и возвращает обёртку. `@dec(...)` сначала вызывает `dec`, а уже его результат становится декоратором - эта форма нужна, когда декоратору надо передать настройки.

## Что ответить на собесе

Разница в одном уровне вызова.

- При `@dec` интерпретатор делает `f = dec(f)` - значит `dec` обязан сам принимать функцию.
- При `@dec(arg)` сначала выполняется `dec(arg)`, и получается `f = dec(arg)(f)` - то есть `dec` не декоратор, а фабрика: принимает параметры и возвращает настоящий декоратор.
- Поэтому декоратор с параметрами - это три вложенных уровня: фабрика, декоратор, обёртка.

Без параметров - один уровень вложенности:

```python
def simple_decorator(func):
    def wrapper(*args, **kwargs):
        print("до")
        result = func(*args, **kwargs)
        print("после")
        return result
    return wrapper

@simple_decorator
def say_hello():
    print("Hello!")
```

Здесь `say_hello = simple_decorator(say_hello)`.

С параметрами добавляется ещё один внешний слой - фабрика:

```python
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                func(*args, **kwargs)
        return wrapper
    return decorator

@repeat(times=3)
def greet():
    print("Привет!")
```

Развёрнуто это `greet = repeat(times=3)(greet)`: сначала `repeat(times=3)` возвращает `decorator`, и уже он оборачивает `greet`.

## Подводные камни

- `@dec()` без аргументов - не то же самое, что `@dec`: первая форма вызовет `dec` без параметров и применит результат, и сработает только если `dec` написан как фабрика.
- Перепутать уровни легко: если у фабрики забыть вернуть `decorator`, получите `None` на месте функции и `TypeError` при вызове.
- Чтобы декоратор работал и с аргументами, и без них (`@dec` и `@dec(...)`), пишут гибкую версию с проверкой, передана ли функция первым позиционным аргументом.
- Не забывайте `functools.wraps` на самой внутренней `wrapper` в обоих вариантах.
