# Type dynamics and strictness: which typing model does Python belong to

## In short

Python combines dynamic and strong typing: a type is bound to a value and resolved at runtime, while the language performs no silent conversions between incompatible types.

## How to answer in an interview

Typing in Python is dynamic and strong. Dynamic, because a type is not declared in advance and is determined at runtime by the value held in a variable; the same name can hold an `int` today and a `str` tomorrow. Strong, because the language does not silently add incompatible types together: `1 + "2"` raises a `TypeError`, not "3" or "12". Conversion between types is always explicit. This sets Python apart from weakly typed languages like JavaScript, where such expressions get coerced "somehow".

## How it works

Dynamism: the type belongs to the object, and a name is just a reference that can be reassigned to a value of another type.

```python
x = 10        # now x references an int
x = "Hello"   # now x references a str
```

Strictness: an operation on incompatible types fails with an error rather than guessing the intent.

```python
result = 10 + "20"   # TypeError: unsupported operand type(s)
```

To add them, we convert explicitly:

```python
result = 10 + int("20")   # 30
```

The dynamic nature lets you write generic code without declaring argument types:

```python
def sum_numbers(numbers):
    total = 0
    for number in numbers:
        total += number
    return total

print(sum_numbers([1, 2, 3, 4, 5]))   # 15
```

## Pitfalls

- Dynamic typing is not the absence of types: every object always has a concrete type, which you can check via `type()` or `isinstance()`.
- Type annotations (`x: int`) and mypy are static checks layered on top of the language; the interpreter itself does not enforce them at runtime.
- "Strong" and "static" are different axes: C is statically and relatively strongly typed, Python is dynamically and strongly typed, JS is dynamically and weakly typed.
- Do not confuse `==` (value comparison) with coercion: `1 == 1.0` is true, but that does not mean an `int` implicitly became a `float` everywhere.
