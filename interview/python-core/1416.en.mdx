# How do you materialize a generator into a regular list?

## In short

Wrap the generator in `list()` - the constructor walks through it to the end and collects all the yielded values into a regular list.

## How to answer in an interview

A generator is a lazy iterator, so on its own it is not a list. To get a list, I pass the generator to `list()`: this function iterates over it until `StopIteration` and puts the elements into a new list. After that you get index access, slices, and all list methods. There is one downside: the whole sequence ends up in memory at once, so for infinite or very large generators you should not do this.

## How it works

`list()` accepts any iterable and consumes it entirely. Take a generator function:

```python
def simple_generator():
    for i in range(5):
        yield i
```

Unlike `return`, `yield` freezes the function's state: on the next step execution continues from the same place. Now let's materialize the result:

```python
gen = simple_generator()
result_list = list(gen)
print(result_list)  # [0, 1, 2, 3, 4]
```

The same effect comes from unpacking `[*gen]` or a generator expression inside a list comprehension: `[x for x in gen]`.

## Pitfalls

- A generator is single-use: after `list(gen)` it is exhausted, and a second pass returns an empty result.
- An infinite generator (`while True: yield ...`) will make `list()` hang and overflow memory.
- On large data, materialization kills the generator's main advantage - memory savings. If you only need a traversal or an aggregate (`sum`, `max`, `any`), work with the generator directly.
