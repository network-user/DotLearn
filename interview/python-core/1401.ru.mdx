# Что такое match/case и почему это не обычный switch?

## Суть

`match/case` (Python 3.10+) - это структурное сопоставление с образцом: значение проверяется на соответствие шаблонам, и выполняется ветка первого совпавшего. В отличие от `switch`, он умеет разбирать структуру данных и привязывать её части к переменным.

## Что ответить на собесе

`match/case` появился в 3.10 и реализует pattern matching, а не простой выбор по равенству. Помимо сравнения с константами он умеет распаковывать последовательности и словари, проверять классы и связывать совпавшие части со свободными именами прямо в шаблоне. Шаблон `case _` ловит всё остальное, как `default`. Главное отличие от `switch`: я сопоставляю не значение со значением, а значение с формой данных.

## Как это работает

Простейший случай - сравнение с константами, где `_` играет роль "иначе":

```python
def describe(value):
    match value:
        case 0:
            return "Zero"
        case 1:
            return "One"
        case _:
            return "Other"

print(describe(0))   # Zero
print(describe(42))  # Other
```

Сила конструкции - в разборе структуры. Имена внутри шаблона не сравниваются, а связываются с совпавшими частями:

```python
def process(data):
    match data:
        case [x, y, z]:
            return f"three: {x}, {y}, {z}"
        case {"name": name, "age": age}:
            return f"{name}, {age}"
        case _:
            return "unknown"

print(process([1, 2, 3]))                    # three: 1, 2, 3
print(process({"name": "Alice", "age": 30})) # Alice, 30
```

Здесь `[x, y, z]` совпадёт только со списком ровно из трёх элементов, а `{"name": ..., "age": ...}` - со словарём, где есть оба ключа (лишние ключи допускаются).

## Подводные камни

- Имя в шаблоне без точки - это захват (binding), а не сравнение. `case RED:` не сверит значение с переменной `RED`, а перезапишет `RED` совпавшим значением. Для сравнения по значению используйте точку: `case Color.RED:`.
- Шаблоны без guard покрывают не всё - без `case _` несовпавшее значение просто провалится мимо `match` без ошибки. Добавляйте `case _`, если нужна ветка по умолчанию.
- Шаблон-словарь совпадает при наличии нужных ключей, даже если в словаре есть лишние - это не строгое равенство.
- `match/case` недоступен в Python 3.9 и ниже; на старых рантаймах код вообще не распарсится.
