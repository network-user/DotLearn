# Что такое хеш и где в Python без него не обойтись?

## Суть

Хеш - это результат хеш-функции: данные любого размера превращаются в значение фиксированной длины. На хешах держатся словари и множества, проверка целостности файлов и хранение паролей.

## Что ответить на собесе

Хеш-функция отображает вход произвольной длины в значение фиксированного размера.

- Встроенная `hash()` даёт целое для неизменяемых объектов - по нему `dict` и `set` находят элемент почти за константное время.
- Для устойчивых криптографических хешей (SHA-256 и т.п.) есть модуль `hashlib`.
- Ключевое свойство: одинаковый вход всегда даёт одинаковый хеш, а равные объекты обязаны иметь равный хеш - иначе словарь сломается.

Внутри: словарь хеширует ключ, по хешу вычисляет позицию в таблице и кладёт туда значение. Поэтому ключами могут быть только хешируемые (как правило, неизменяемые) объекты.

```python
print(hash("abc") == hash("abc"))  # True: одинаковый вход - одинаковый хеш
print(hash((1, 2)))                # кортежи хешируемы
# hash([1, 2]) -> TypeError: список не хешируем
```

Для криптографии берут `hashlib`: он работает с байтами и устойчив к коллизиям.

## Подводные камни

- Хешируемость и неизменяемость связаны: списки и словари как ключи использовать нельзя, кортежи из неизменяемых элементов - можно.
- Контракт: если `a == b`, то `hash(a) == hash(b)`. Переопределяя `__eq__`, обязательно согласуйте `__hash__`, иначе объекты потеряются в `dict`/`set`.
- `hash()` встроенной функции не криптостойкий и может отличаться между запусками (рандомизация хеша строк) - для безопасности используйте `hashlib`, а не `hash()`.
- Одинаковый хеш у разных объектов (коллизия) - норма; равенство всё равно проверяется через `__eq__`.
