# A function as an argument: what does it mean for functions to be "first-class" in Python

## In short

Yes, in Python you can pass a function into another function as an ordinary value. Functions here are first-class objects, so everything you can do with any data you can do with them.

## How to answer in an interview

Functions in Python are first-class objects. You can assign them to variables, put them in lists and dicts, return them from other functions, and pass them as arguments. This is exactly what `map`, `filter`, `sorted(key=...)`, decorators, and callbacks rely on. By passing behavior as a parameter, you lift the varying part out and reuse a common skeleton instead of copying code.

## How it works

A function name without parentheses is a reference to the function object, not a call to it. That reference is what you pass:

```python
def square(x):
    return x * x

numbers = [1, 2, 3, 4, 5]

squared = map(square, numbers)
print(list(squared))   # [1, 4, 9, 16, 25]
```

`map(square, numbers)` receives the `square` function itself (without `()`) and applies it to each element. The same trick works in sorting and filtering, where the function defines the criterion:

```python
words = ['ccc', 'a', 'bb']

print(sorted(words, key=len))                  # ['a', 'bb', 'ccc']
print(list(filter(lambda w: len(w) > 1, words)))  # ['ccc', 'bb']
```

You can also write your own higher-order function that takes an action as a parameter:

```python
def apply_twice(func, value):
    return func(func(value))

print(apply_twice(lambda x: x + 3, 10))   # 16
```

## Pitfalls

- `square` is a reference to the function, `square()` is an immediate call. Pass `square()` instead of `square` and the argument receives the call's result (or an error about missing parameters).
- `map` and `filter` return a lazy iterator, not a list. You can walk it once; for printing or a second pass, wrap it in `list()`.
- When you pass an object's method (`obj.method`), you pass a bound method along with `self` - that's no longer a "bare" function.
- Lambdas are handy, but overusing them hurts readability and complicates debugging - they have no meaningful name in a traceback.
