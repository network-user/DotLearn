# Decorators in Python: why they are needed and what problems they solve

## In short

A decorator is a wrapper that adds new behavior to a function or method without rewriting their body. It takes a function, returns another function, and thereby injects its own logic before and after the original call.

## How to answer in an interview

Decorators let you separate auxiliary logic from the main logic. I move recurring concerns - logging, caching, permission checks, timing, input validation - into a separate decorator and attach it to any functions via the `@` syntax. Under the hood it's an ordinary higher-order function: it receives the target function as an argument and returns a wrapper that calls the original, surrounding it with extra code. The payoff is reuse, a clean main codebase, and a single point for changes.

## How it works

A decorator takes a function and returns a new function, usually one that calls the original inside itself. The line `@deco` above `def f` is equivalent to `f = deco(f)`.

```python
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Before the call")
        result = func(*args, **kwargs)
        print("After the call")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")
    return name

say_hello("Ann")
```

Breakdown:

- `func` - the function we are wrapping.
- `wrapper(*args, **kwargs)` - the wrapper, transparently forwarding any arguments.
- `result = func(...)` - running the original; we save the result and return it so as not to lose it.
- `@functools.wraps(func)` - transfers the original's name, docstring, and signature onto the wrapper.
- `@my_decorator` - the same as `say_hello = my_decorator(say_hello)`.

## Pitfalls

- Without `functools.wraps`, the wrapped function's `__name__` and `__doc__` get replaced - this breaks introspection, documentation, and sometimes serialization.
- If the wrapper doesn't forward `*args, **kwargs` and doesn't return the result, you'll lose the arguments and the return value.
- A decorator runs once at the moment the function is defined, while the wrapper runs on every call; do heavy initialization in the decorator's body, not in `wrapper`.
- A decorator with parameters needs an extra level of nesting (a decorator factory), which is a common source of confusion.
