# What is the difference between @dec and @dec() when applying a decorator?

## In short

`@dec` applies `dec` itself as the decorator: it receives the function and returns a wrapper. `@dec(...)` first calls `dec`, and its result becomes the decorator — this form is needed when you have to pass settings to the decorator.

## How to answer in an interview

It all comes down to one level of calling. With `@dec` the interpreter does `f = dec(f)` — so `dec` must accept the function itself. With `@dec(arg)`, `dec(arg)` runs first, giving `f = dec(arg)(f)` — meaning `dec` here is not a decorator but a decorator factory: it takes parameters and returns the actual decorator. That is why a parameterized decorator has three nested levels: the factory, the decorator, and the wrapper.

Without parameters — one level of nesting:

```python
def simple_decorator(func):
    def wrapper(*args, **kwargs):
        print("before")
        result = func(*args, **kwargs)
        print("after")
        return result
    return wrapper

@simple_decorator
def say_hello():
    print("Hello!")
```

Here `say_hello = simple_decorator(say_hello)`.

With parameters, one more outer layer is added — the factory:

```python
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                func(*args, **kwargs)
        return wrapper
    return decorator

@repeat(times=3)
def greet():
    print("Hello!")
```

Expanded, this is `greet = repeat(times=3)(greet)`: first `repeat(times=3)` returns `decorator`, and that in turn wraps `greet`.

## Pitfalls

- `@dec()` without arguments is not the same as `@dec`: the first form calls `dec` with no parameters and applies the result, and it only works if `dec` is written as a factory.
- It is easy to mix up the levels: if the factory forgets to return `decorator`, you get `None` in place of the function and a `TypeError` on the call.
- To make a decorator work both with and without arguments (`@dec` and `@dec(...)`), people write a flexible version that checks whether a function was passed as the first positional argument.
- Do not forget `functools.wraps` on the innermost `wrapper` in both variants.
