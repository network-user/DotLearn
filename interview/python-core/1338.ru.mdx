# Анонимные lambda-функции: что это и где уместны?

## Суть

`lambda` создаёт небольшую безымянную функцию из одного выражения, результат которого возвращается без явного `return`. Её стихия - короткие одноразовые callable, которые удобно передать прямо в `sorted`, `map` или `filter`.

## Что ответить на собесе

Лямбда - это функция-выражение: `lambda параметры: выражение`, без имени и без операторов, только одно вычисляемое выражение.

- Семантически не отличается от функции через `def`: тот же тип `function`, просто без привязанного имени.
- Беру там, где функция нужна на месте и один раз: ключ сортировки, преобразование в `map`, предикат в `filter`.
- Логика сложнее одного выражения или функция переиспользуется - делаю обычный `def`.

## Как это работает

Общая форма: ключевое слово, параметры, двоеточие, единственное выражение.

```python
lambda arguments: expression
```

Классический пример - ключ сортировки. Отсортируем кортежи по второму элементу:

<PyDemo title="lambda как ключ сортировки" autoRun code={`data = [(1, 2), (3, 1), (5, 0)]
sorted_data = sorted(data, key=lambda pair: pair[1])
print(sorted_data)  # [(5, 0), (3, 1), (1, 2)]`} />

Здесь `key` получает функцию, превращающую каждый кортеж в значение, по которому идёт сравнение.

Лямбды часто соседствуют с функциями высшего порядка:

- `map(fn, iterable)` - применяет `fn` к каждому элементу;
- `filter(fn, iterable)` - оставляет элементы, на которых `fn` истинна;
- `functools.reduce(fn, iterable)` - сворачивает всё к одному значению.

```python
numbers = [1, 2, 3, 4]
incremented = list(map(lambda n: n + 1, numbers))
print(incremented)  # [2, 3, 4, 5]
```

## Подводные камни

- Лямбда - только одно выражение: ни `=`, ни `if`/`for` как операторов, ни нескольких строк. Нужно больше - пишите `def`.
- Поздняя привязка замыканий: `[lambda: i for i in range(3)]` все вернут 2, потому что `i` читается в момент вызова. Лечится параметром по умолчанию `lambda i=i: i`.
- Присваивание лямбды переменной (`f = lambda x: x`) - антипаттерн по PEP 8; если функции нужно имя, объявляйте её через `def`.
- Для извлечения атрибута/элемента часто чище взять `operator.itemgetter`/`attrgetter` вместо лямбды - быстрее и читабельнее.
