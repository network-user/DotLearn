# What exactly does a decorator return, and what replaces the original function

## In short

A decorator returns a function - usually a wrapper that calls the original inside and adds behavior. After `@decorator`, the original function's name starts to reference that wrapper.

## How to answer in an interview

A decorator is a callable that takes a function and returns a function. Writing `@deco` above `def f` is syntactic sugar for `f = deco(f)`. So the decorator's return value is exactly what ends up under the name `f`. Usually you return an inner `wrapper` function that calls the original and adds logic before and after. So that the wrapper does not "eat" the arguments and the result, it is written as `def wrapper(*args, **kwargs): return func(*args, **kwargs)`, and the original's metadata is preserved via `functools.wraps`.

## How it works

The decorator receives the original function, wraps it, and returns the wrapper:

```python
import functools

def my_decorator(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("Before the call")
        result = func(*args, **kwargs)
        print("After the call")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    return f"Hello, {name}!"

print(say_hello("World"))
```

The steps:

1. `my_decorator(say_hello)` runs at definition time - it returns `wrapper`.
2. The name `say_hello` now points to `wrapper`, not to the original function.
3. Calling `say_hello("World")` runs `wrapper`: it prints "Before the call", calls the original, prints "After the call", and returns its result.
4. `functools.wraps` copies `__name__`, `__doc__`, and the rest from the original onto the wrapper.

## Pitfalls

- If the wrapper does not return `func(...)`, the decorated function always returns `None` - a common mistake.
- A wrapper without `*args, **kwargs` will break on functions that take arguments.
- Without `functools.wraps`, the function "loses" its name and docstring: `say_hello.__name__` becomes `'wrapper'`, which hinders introspection and debugging.
- A decorator runs once, at definition time; confusing this with running on every call is a typical misconception.
