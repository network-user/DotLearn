# How do you get an element's position right inside a loop over a sequence?

## In short

Wrap the sequence in `enumerate()` - on each step it yields an "index, element" pair, and a manual counter becomes unnecessary.

## How to answer in an interview

The idiomatic way is `for i, value in enumerate(seq)`. This is better than `for i in range(len(seq))` with a `seq[i]` access, because the code reads cleaner and does not depend on whether the object supports indexing. `enumerate` has a second argument, `start`, for when numbering should begin somewhere other than zero.

## How it works

`enumerate()` takes any iterable and returns a lazy iterator. On each step it yields a `(index, element)` tuple, which is unpacked into two loop variables.

```python
fruits = ['apple', 'banana', 'cherry']

for index, value in enumerate(fruits):
    print(index, value)
```

Output:

```text
0 apple
1 banana
2 cherry
```

If you need to start counting from one, pass `start`:

```python
for number, value in enumerate(fruits, start=1):
    print(number, value)
```

## Pitfalls

- `enumerate` returns an iterator, not a list: you cannot traverse it again, so wrap it in `list()` for that.
- The first element of the tuple is the index, the second is the value. Mixing up the unpacking order is a common mistake.
- The `start` argument shifts only the numbering, not the elements themselves; indexing of the original sequence does not change.
