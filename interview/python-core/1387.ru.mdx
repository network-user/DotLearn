# Как Python сам освобождает память: устройство сборщика мусора

## Суть

В Python память освобождается автоматически. Основа - подсчёт ссылок: объект уничтожается, как только на него не остаётся ссылок. Поверх этого работает отдельный сборщик циклов, который ловит объекты, замкнутые друг на друга и оттого недостижимые счётчиком.

## Что ответить на собесе

У CPython два уровня управления памятью. Первый и основной - reference counting: у каждого объекта есть счётчик ссылок, и при достижении нуля объект немедленно освобождается. Этот механизм детерминированный и быстрый, но у него слепое пятно - циклические ссылки: два объекта ссылаются друг на друга, их счётчики никогда не падают до нуля, хотя извне они недостижимы. Поэтому есть второй уровень - циклический сборщик из модуля `gc`, который периодически обходит объекты-контейнеры, находит недостижимые циклы и освобождает их. Им можно управлять вручную: запускать, отключать, менять пороги.

Подсчёт ссылок: новая ссылка увеличивает счётчик, исчезновение - уменьшает.

```python
a = []   # счётчик = 1
b = a    # счётчик = 2
del a    # счётчик = 1
del b    # счётчик = 0 -> память освобождается
```

Циклические ссылки счётчик не разрулит:

```python
class Node:
    def __init__(self):
        self.reference = None

node1 = Node()
node2 = Node()
node1.reference = node2
node2.reference = node1

del node1
del node2
# счётчики остались > 0; цикл найдёт и освободит сборщик циклов
```

Сборщик циклов время от времени обходит объекты, строит граф достижимости, выделяет недостижимые циклы и освобождает их. Запуск привязан к порогам по числу аллокаций. Управлять можно через `gc`:

```python
import gc

gc.collect()   # принудительный проход сборщика
gc.disable()   # отключить автоматическую сборку циклов
gc.enable()    # включить обратно
```

## Подводные камни

- Reference counting освобождает память сразу, а сборщик циклов - не мгновенно: между созданием мусора и его сбором проходит время.
- `__del__` на объектах в цикле раньше мешал их собрать; в современных версиях это решено, но финализаторы с побочными эффектами в циклах всё равно рискованны.
- Отключение `gc.disable()` ускоряет код без циклов, но при наличии циклов приводит к росту памяти - включайте обратно.
- Сборщик циклов обходит только контейнеры (списки, словари, объекты с `__dict__`); простые типы вроде чисел и строк собираются исключительно счётчиком ссылок.
- Подсчёт ссылок - деталь CPython. В PyPy или Jython его нет, и порядок освобождения объектов другой, поэтому не полагайтесь на немедленный вызов `__del__`.
