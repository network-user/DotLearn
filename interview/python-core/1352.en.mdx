# How to write a decorator that logs the name of the called function

## In short

A decorator is a wrapper: it takes a function and returns a new function with extra behavior. To log the name before the call, it's enough to print `func.__name__` inside the inner wrapper and then hand control over to the original.

## How to answer in an interview

A decorator is a callable that receives a function as an argument and returns another function. Inside, I declare `wrapper(*args, **kwargs)`, print `func.__name__`, and then return `func(*args, **kwargs)` so the original behavior and its result are preserved. The `@decorator` syntax above a function definition is just sugar for `f = decorator(f)`. This is how I add logging without touching the function's own code.

## How it works

The wrapper closes over a reference to the original function and forwards any arguments through `*args`/`**kwargs`, while taking the name from the `__name__` attribute.

```python
import functools


def log_call(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"Calling function: {func.__name__}")
        return func(*args, **kwargs)

    return wrapper


@log_call
def greet(name):
    return f"Hello, {name}"


print(greet("Ann"))
```

What happens here step by step:

1. `log_call` receives `func` and returns `wrapper`.
2. `wrapper` prints the name via `func.__name__`, then calls the original and returns its result.
3. `*args, **kwargs` pass any positional and keyword arguments through without loss.
4. The line `@log_call` is equivalent to `greet = log_call(greet)`.
5. `functools.wraps` copies metadata (`__name__`, `__doc__`) from the original onto the wrapper.

## Pitfalls

- Without `functools.wraps` the wrapped function starts to "introduce itself" as `wrapper`: `greet.__name__` returns `"wrapper"`, which breaks introspection and other decorators that read the name.
- A forgotten `return func(...)` silently turns the result into `None` - a common mistake.
- If you need a decorator with parameters (`@log_call(level="info")`), you'll need one more level of nesting: a factory that returns the decorator itself.
- Multiple decorators are applied bottom to top: the one closest to the function wraps it first.
