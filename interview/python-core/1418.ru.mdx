# Можно ли навешивать декоратор на класс, а не на функцию?

## Суть

Да. Декоратор класса — это функция, которая получает класс, как-то его меняет и возвращает обратно. Принцип тот же, что и у декоратора функции, только объект на входе — класс.

## Что ответить на собесе

Да: декоратор класса - функция, получающая класс и возвращающая заменяющий, а `@deco` над `class C` равно `C = deco(C)`.

- Внутри можно добавить классу методы или атрибуты, обернуть `__init__`, подменить сам класс другим.
- Удобно для сквозной функциональности - логирования, регистрации в реестре, кэширования, проверки прав - без правки тела класса.
- Чтобы вмешиваться в само создание класса, берут метаклассы, но для большинства задач хватает декоратора.

## Как это работает

Соберём декоратор, который добавляет всем декорированным классам метод `greet`:

```python
def add_greet_method(cls):
    def greet(self):
        return f"Hello from {self.__class__.__name__}!"
    cls.greet = greet
    return cls


@add_greet_method
class MyClass:
    def __init__(self, name):
        self.name = name


obj = MyClass("Example")
print(obj.greet())  # Hello from MyClass!
```

Что происходит по шагам:

1. `@add_greet_method` означает `MyClass = add_greet_method(MyClass)`.
2. Внутри объявляется функция `greet` и присваивается классу как атрибут — теперь это полноценный метод.
3. `return cls` отдаёт тот же класс, уже расширенный.
4. Любой экземпляр `MyClass` получает доступ к `greet`.

Декоратор может и подменять класс целиком — например, вернуть обёртку, перехватывающую создание экземпляров.

## Подводные камни

- Декоратор класса выполняется один раз, в момент определения класса, а не при каждом создании экземпляра.
- Если декоратор возвращает не сам класс, а функцию-обёртку, теряются `isinstance`, имя и метаданные исходного класса — будьте осторожны.
- Добавляя атрибут как метод (`cls.greet = greet`), помните: обычная функция станет связанным методом, а вот вложенную функцию без `self` так навешивать нельзя.
- Для управления самим процессом создания класса (а не его доработки постфактум) декоратора мало — нужен метакласс.
