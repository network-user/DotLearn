# Generic types: one piece of code for any type, checked by the type checker

## In short

Generics describe a function or class through a type parameter, and the concrete type is substituted at the point of use. This removes duplication and gives you static type checking without losing reusability.

## How to answer in an interview

A generic lets you write a structure or function once so that it works with any type, while the type checker (mypy, pyright) sees the concrete type at every use site. You declare the type parameter with `TypeVar('T')` and inherit the class from `Generic[T]`. When you write `Stack[int]`, the checker knows `pop()` returns an `int` and catches an error before runtime. Important: in Python itself these are annotations, not runtime generics - at runtime `Stack[int]` and `Stack[str]` behave identically, there is no type checking during execution, everything happens at the code-analysis stage.

## How it works

```python
from typing import TypeVar, Generic

T = TypeVar("T")

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def is_empty(self) -> bool:
        return not self._items

int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop())  # 2

str_stack: Stack[str] = Stack()
str_stack.push("hello")
str_stack.push("world")
print(str_stack.pop())  # world
```

`TypeVar("T")` introduces a type parameter, `Generic[T]` binds it to the class, and `Stack[int]` fixes `T = int` for a particular instance. A generic function looks simpler:

```python
from typing import TypeVar

T = TypeVar("T")

def first(items: list[T]) -> T:
    return items[0]

print(first([10, 20]))      # 10
print(first(["a", "b"]))    # a
```

In Python 3.12 this is written even more concisely through the `class Stack[T]:` and `def first[T](...)` syntax, without an explicit `TypeVar`.

## Pitfalls

- Generics in Python are a static-checking tool, not a runtime one. In executable code they do not prevent passing the wrong type; only the type checker sees the error.
- Create a `TypeVar` once per module and reuse it; you cannot define `T` on one line and expect it to be linked to another.
- Without a variable annotation (`Stack[int]`), the type checker will not infer the parameter on its own unless it is set explicitly in the constructor.
- `TypeVar("T", int, str)` (with constraints) and `bound=...` are different things: constraints allow only the listed types, while `bound` allows any subtype of the given one.
