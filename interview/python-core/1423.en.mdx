# Python exception hierarchy: built-in classes and your own subclasses

## In short

All exceptions form a tree rooted at `BaseException`; almost everything you catch in applications inherits from `Exception`. You create your own errors by subclassing `Exception` or one of its subclasses.

## How to answer in an interview

Exceptions in Python are classes arranged in a hierarchy. At the top is `BaseException`; from it branch the system-level ones (`SystemExit`, `KeyboardInterrupt`) and the general `Exception`, from which almost everything application-level inherits. Under `Exception` there are meaningful groups: `ArithmeticError` (with `ZeroDivisionError`, `OverflowError`), `LookupError` (with `IndexError`, `KeyError`), as well as `ValueError`, `TypeError`, `OSError`, `ImportError` and others. I make my own exceptions subclasses of `Exception` (or a narrower class) so they can be caught both individually and through the base type.

## How it works

The inheritance tree determines which `except` will catch an error: catching a base class also intercepts all of its descendants.

### Key built-in classes

- **BaseException** — the root of the hierarchy. You should not catch it directly: beneath it sit `SystemExit` and `KeyboardInterrupt`, which are usually not intercepted.
- **Exception** — the base for application errors; it (and its descendants) is what you catch in `try/except`.
- **ArithmeticError** — arithmetic; this includes `ZeroDivisionError`, `OverflowError`, `FloatingPointError`.
- **LookupError** — access by a nonexistent index/key; includes `IndexError` and `KeyError`.
- **ValueError** — the type is correct but the value is invalid; **TypeError** — an unsuitable type; **NameError** — no such variable.
- **OSError** — OS-level errors; in Python 3 it also replaces the former `IOError`. **ImportError** — an import failure, with the subclass `ModuleNotFoundError`.

### Your own exception classes

For domain errors you create subclasses of `Exception`:

```python
class PaymentError(Exception):
    """Payment error."""

try:
    raise PaymentError("Insufficient funds")
except PaymentError as e:
    print(e)
```

Inheriting from `Exception` is enough — you only need to override `__init__` to add extra fields.

## Pitfalls

- `except Exception` will not catch `KeyboardInterrupt` and `SystemExit` — they are under `BaseException` but outside `Exception`. And that is correct: catching them breaks program termination.
- The order of `except` matters: a more specific class must come before the base one, otherwise the base will catch everything first and the narrow block becomes unreachable.
- `IOError` in Python 3 is an alias for `OSError`; there is no separate class.
- A bare `except:` catches absolutely everything, including `BaseException`; you should not do this — it masks both system signals and errors in the handler code itself.
