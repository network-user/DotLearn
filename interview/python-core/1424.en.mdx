# Which Python objects can serve as a decorator?

## In short

Any callable object can act as a decorator: an ordinary function, a method, a class, or an instance with `__call__`. The only requirement is to accept the decorated object and return a replacement that takes its place.

## How to answer in an interview

A decorator is simply a callable that receives a function (or class) and returns something to replace it. Most often it is a function returning a closure wrapper, but classes work just the same (their instance replaces the original function), as do objects with a `__call__` method. The syntax `@dec` above `def f` is sugar for `f = dec(f)`. It is used to attach cross-cutting logic — logging, caching, permission checks — without rewriting the function itself.

## How it works

The basic case is a function that builds and returns a wrapper inside itself:

```python
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("before the call")
        result = func(*args, **kwargs)
        print("after the call")
        return result
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
```

The line `@my_decorator` is equivalent to `say_hello = my_decorator(say_hello)`. After that, the name `say_hello` points to `wrapper`.

A class can also be a decorator — then its instance replaces the function, and calls are intercepted by `__call__`:

```python
class Counter:
    def __init__(self, func):
        self.func = func
        self.calls = 0
    def __call__(self, *args, **kwargs):
        self.calls += 1
        return self.func(*args, **kwargs)

@Counter
def ping():
    return "pong"
```

## Pitfalls

- The wrapper loses the original's `__name__`, `__doc__` and signature — use `functools.wraps`, otherwise introspection and auto-documentation break.
- The wrapper must forward `*args, **kwargs`, otherwise the decorator will only work for functions without arguments.
- Do not confuse the callable object (which is the decorator itself) with the result of calling it: a class used as a decorator wraps the function with its instance, not with a returned value.
- A decorator runs once at the moment the function is defined, not on every call — put heavy initialization in the decorator body, not in `wrapper`.
