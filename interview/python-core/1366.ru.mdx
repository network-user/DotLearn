# Что отдаёт метод items() словаря и как этим пользоваться?

## Суть

`items()` возвращает объект-представление `dict_items` со всеми парами «ключ-значение». По нему удобно перебирать ключ и значение одновременно за один цикл.

## Что ответить на собесе

`items()` даёт `dict_items` - динамическое представление (view) пар словаря, а не их копию.

- По нему итерируются, распаковывая каждый элемент в `key, value`.
- Представление живое: меняется словарь - меняется и то, что видно через `items()`.
- При необходимости материализуют через `list()`, но для простого перебора это не нужно.

## Как это работает

Каждый элемент `dict_items` - кортеж `(ключ, значение)`, поэтому в цикле его можно распаковать сразу в две переменные.

<PyDemo title="Перебор пар ключ-значение через items()" autoRun code={`student_grades = {"Alice": 85, "Bob": 92, "Charlie": 78}

for name, grade in student_grades.items():

Представление отражает текущее состояние словаря:

```python
d = {"a": 1}
view = d.items()
d["b"] = 2
print(list(view))  # [('a', 1), ('b', 2)] - view увидел новый ключ
```

Помимо `items()` есть `keys()` и `values()` - они работают по тому же принципу представлений.

## Подводные камни

- `items()` - это не список, а view; индексация `d.items()[0]` не работает, сначала нужен `list()`.
- Изменять словарь (добавлять/удалять ключи) прямо во время итерации по `items()` нельзя - будет `RuntimeError: dictionary changed size during iteration`.
- View остаётся связанным со словарём: сохранив `items()` в переменную, вы получаете окно в актуальные данные, а не снимок прошлого.
- Порядок пар совпадает с порядком вставки (гарантируется с Python 3.7), но полагаться на сортировку по ключам нельзя.
