# Какие коллекции можно собрать через comprehension?

## Суть

Comprehension - компактная запись для построения новой коллекции из итерируемой. Форм четыре: списковая, множественная, словарная и генераторная; различают их только скобки.

## Что ответить на собесе

Comprehension есть для списков `[...]`, множеств `{...}`, словарей `{k: v ...}` и генераторов `(...)`.

- Внутри один и тот же костяк: выражение, `for`-перебор и необязательный фильтр `if`.
- Списковая, множественная и словарная формы строят коллекцию сразу целиком, генераторная возвращает ленивый объект и экономит память.
- Кортежного comprehension нет: `(...)` - это генераторное выражение, не кортеж.

## Как это работает

Один шаблон - четыре результата в зависимости от скобок:

```python
nums = range(5)

[x ** 2 for x in nums]          # список:     [0, 1, 4, 9, 16]
{x ** 2 for x in nums}          # множество:  {0, 1, 4, 9, 16}
{x: x ** 2 for x in nums}       # словарь:    {0: 0, 1: 1, ...}
(x ** 2 for x in nums)          # генератор:  ленивый объект
```

Добавляется фильтр:

```python
even_squares = [x ** 2 for x in range(10) if x % 2 == 0]  # [0, 4, 16, 36, 64]
```

Множество отбросит дубликаты, словарь при совпадении ключей оставит последнее значение, а генератор не построит ничего, пока его не начнут перебирать.

## Подводные камни

- Круглые скобки дают **генератор**, а не кортеж - его можно пройти лишь один раз, после чего он пуст.
- Set- и dict-comprehension используют одни и те же фигурные скобки; их различает наличие `key: value`. Пустые `{}` - это словарь, не множество.
- В словарном варианте повторяющиеся ключи молча затирают друг друга - останется последнее значение.
- Длинные вложенные comprehensions быстро теряют читаемость; иногда обычный цикл понятнее.
- Переменная цикла внутри comprehension не «утекает» наружу (в отличие от Python 2): после `[x for x in ...]` имя `x` снаружи не определено.
