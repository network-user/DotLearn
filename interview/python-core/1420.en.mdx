# Iterating a dictionary by pairs: why you need the items() method

## In short

When a loop needs both the key and the value at once, call `.items()` - it yields a view of pairs that unpack right in the `for` header.

## How to answer in an interview

For simultaneous access to a key and a value I use `dict.items()`. The method returns not a list but a view object - a dynamic representation that reflects the dictionary's current state. In the loop I unpack each pair into two variables: `for key, value in d.items():`. If I need only the keys, `for k in d` (or `.keys()`) is enough; if only the values, `.values()`.

## How it works

`.items()` returns a `dict_items` object - this is a view, not a copy. Each element of the view is a `(key, value)` tuple, which Python automatically unpacks into the two variables of the loop header.

```python
student_grades = {
    'Alice': 85,
    'Bob': 92,
    'Charlie': 78,
}

for name, grade in student_grades.items():
    print(f"{name}: {grade}")
```

Here `name` gets the key on each step, and `grade` gets the associated value. Unpacking works because `.items()` yields exactly tuple pairs. Without `.items()`, a plain `for x in student_grades` walks only over the keys.

Since this is a view, not a snapshot, the view sees dictionary changes in real time:

```python
d = {'a': 1}
view = d.items()
d['b'] = 2
print(list(view))   # [('a', 1), ('b', 2)]
```

## Pitfalls

- You cannot change the dictionary's size (add or remove keys) while iterating over `.items()` - Python will raise `RuntimeError: dictionary changed size during iteration`. Changing the values of existing keys is allowed, though.
- `.items()` is a view, not a list: you cannot index `d.items()[0]`, you need `list(d.items())`.
- Unpacking `for name, grade in ...` will fail with `ValueError` if you iterate over the dictionary itself rather than its `.items()` (there is nothing to unpack into two variables from keys alone).
