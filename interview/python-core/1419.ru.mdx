# Почему к генератору нельзя обратиться по индексу?

## Суть

Генератор не поддерживает `gen[i]`: он не хранит элементы и порождает их по одному, поэтому добраться до нужного можно только перебором. Попытка индексации даёт `TypeError: 'generator' object is not subscriptable`.

## Что ответить на собесе

Индексация требует протокола `__getitem__`, которого у генератора нет - он реализует только протокол итератора (`__next__`).

- Генератор вычисляет значения лениво и не держит прошлые в памяти, так что «элемента номер N» без перебора у него нет.
- Нужен элемент по позиции - перебираю сам или беру `itertools.islice`.
- Если данных немного - материализую в список через `list()` и работаю уже со списком.

## Как это работает

Генератор создаётся функцией с `yield` или генераторным выражением. Вызов функции возвращает объект-генератор:

```python
def count_up_to(limit):
    count = 1
    while count <= limit:
        yield count
        count += 1
```

Прямая индексация невозможна:

```python
gen = count_up_to(5)
# gen[2]  ->  TypeError: 'generator' object is not subscriptable
```

Получить элемент по позиции можно перебором:

```python
gen = count_up_to(5)
for i, value in enumerate(gen):
    if i == 2:
        print(value)  # 3
        break
```

Лаконичнее всего это делается через `itertools.islice`, которая берёт срез итератора:

```python
from itertools import islice

gen = count_up_to(5)
print(next(islice(gen, 2, 3)))  # 3
```

## Подводные камни

- `islice` и ручной перебор всё равно "проматывают" генератор: пройденные элементы теряются, повторно к ним не вернуться.
- Срез-синтаксис `gen[1:3]` тоже не работает: для этого нужен `islice(gen, 1, 3)`.
- Материализация в список через `list()` восстанавливает индексацию, но загружает всё в память и не годится для бесконечных генераторов.
- Отрицательные индексы (`gen[-1]`) невозможны принципиально: конец неизвестен, пока генератор не исчерпан.
