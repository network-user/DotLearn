# Generic-типы: один код для любых типов с проверкой типизатора

## Суть

Дженерики описывают функцию или класс через параметр типа, а конкретный тип подставляется в месте использования. Это убирает дублирование и даёт статическую проверку типов без потери универсальности.

## Что ответить на собесе

Generic позволяет написать структуру или функцию один раз так, чтобы она работала с любым типом, но при этом типизатор (mypy, pyright) видел конкретный тип в каждом применении. Параметр типа объявляют через `TypeVar('T')`, а класс наследуют от `Generic[T]`. Когда вы пишете `Stack[int]`, проверяющий знает, что `pop()` вернёт `int`, и отловит ошибку до запуска. Важно: в самом Python это аннотации, а не дженерики времени выполнения — в рантайме `Stack[int]` и `Stack[str]` ведут себя одинаково, никакой проверки типов при исполнении нет, всё работает на этапе анализа кода.

## Как это работает

```python
from typing import TypeVar, Generic

T = TypeVar("T")

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def is_empty(self) -> bool:
        return not self._items

int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop())  # 2

str_stack: Stack[str] = Stack()
str_stack.push("hello")
str_stack.push("world")
print(str_stack.pop())  # world
```

`TypeVar("T")` вводит параметр типа, `Generic[T]` связывает его с классом, а `Stack[int]` фиксирует `T = int` для конкретного экземпляра. Generic-функция выглядит проще:

```python
from typing import TypeVar

T = TypeVar("T")

def first(items: list[T]) -> T:
    return items[0]

print(first([10, 20]))      # 10
print(first(["a", "b"]))    # a
```

В Python 3.12 это записывается ещё короче через синтаксис `class Stack[T]:` и `def first[T](...)`, без явного `TypeVar`.

## Подводные камни

- Дженерики в Python — инструмент статической проверки, а не рантайма. В исполняемом коде они не запрещают передать неверный тип, ошибку видит только типизатор.
- `TypeVar` создают один раз на модуль и переиспользуют; нельзя завести `T` в одной строке и ждать связи с другим.
- Без аннотации переменной (`Stack[int]`) типизатор не выведет параметр сам, если он не задан явно в конструкторе.
- `TypeVar("T", int, str)` (с ограничениями) и `bound=...` — разные вещи: ограничения допускают только перечисленные типы, `bound` — любой подтип указанного.
