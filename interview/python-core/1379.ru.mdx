# Generic-типы: один код для любых типов с проверкой типизатора

## Суть

Дженерики описывают функцию или класс через параметр типа, а конкретный тип подставляется в месте использования. Это убирает дублирование и даёт статическую проверку типов без потери универсальности.

## Что ответить на собесе

Generic пишут один раз для любого типа, но типизатор (mypy, pyright) видит конкретный тип в каждом применении.

- Параметр типа объявляют через `TypeVar('T')`, класс наследуют от `Generic[T]`.
- При `Stack[int]` проверяющий знает, что `pop()` вернёт `int`, и отловит ошибку до запуска.
- Это только аннотации: в рантайме `Stack[int]` и `Stack[str]` ведут себя одинаково, проверки типов при исполнении нет.

## Как это работает

<PyDemo title="Generic-класс Stack[T] для любого типа" autoRun code={`from typing import TypeVar, Generic

T = TypeVar("T")

class Stack(Generic[T]):
def **init**(self) -> None:
self.\_items: list[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

    def is_empty(self) -> bool:
        return not self._items

int_stack: Stack[int] = Stack()
int_stack.push(1)
int_stack.push(2)
print(int_stack.pop()) # 2

str_stack: Stack[str] = Stack()
str_stack.push("hello")
str_stack.push("world")
print(str_stack.pop()) # world`} />

`TypeVar("T")` вводит параметр типа, `Generic[T]` связывает его с классом, а `Stack[int]` фиксирует `T = int` для конкретного экземпляра. Generic-функция выглядит проще:

```python
from typing import TypeVar

T = TypeVar("T")

def first(items: list[T]) -> T:
    return items[0]

print(first([10, 20]))      # 10
print(first(["a", "b"]))    # a
```

В Python 3.12 это записывается ещё короче через синтаксис `class Stack[T]:` и `def first[T](...)`, без явного `TypeVar`.

## Подводные камни

- Дженерики в Python представляют собой инструмент статической проверки, а не рантайма. В исполняемом коде они не запрещают передать неверный тип, ошибку видит только типизатор.
- `TypeVar` создают один раз на модуль и переиспользуют; нельзя завести `T` в одной строке и ждать связи с другим.
- Без аннотации переменной (`Stack[int]`) типизатор не выведет параметр сам, если он не задан явно в конструкторе.
- Между `TypeVar("T", int, str)` (с ограничениями) и `bound=...` есть разница: ограничения допускают только перечисленные типы, а `bound` означает любой подтип указанного.
