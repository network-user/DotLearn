# Can you change a single character of a string in place?

## In short

No: strings in Python are immutable. Any edit to a single character actually creates a new string, while the old one stays untouched.

## How to answer in an interview

A string is an immutable object, so you can't assign `s[1] = 'a'` - it raises a `TypeError`. To get a modified version, you build a new string with slices and concatenation, or temporarily switch to a list of characters. Immutability gives strings hashability, safety when passed around, and the ability to be interned.

## How it works

Since you can't modify in place, we construct a new string from the pieces around the target position.

```python
original = "hello"

# replace the character at index 1 ('e') with 'a'
new_string = original[:1] + 'a' + original[2:]

print(new_string)   # 'hallo'
print(original)     # 'hello' — the source is unchanged
```

If there are many edits, it's more convenient to convert to a list, edit it, and join it back:

```python
chars = list("hello")
chars[1] = 'a'
result = "".join(chars)   # 'hallo'
```

This way every operation makes it explicit: the original string stays as it was, and the result is a new object.

## Pitfalls

- **`s[i] = x` always fails.** It's not a syntax typo but a fundamental prohibition: `TypeError: 'str' object does not support item assignment`.
- **A slice doesn't change the original.** `original[:1]` returns a copy-substring; the source variable stays the same until you assign a new value to it.
- **Concatenation in a loop is expensive.** Building a string with `+=` in a long loop creates intermediate objects on every iteration; for bulk joining use `"".join(...)`.
- **An out-of-range index in a slice doesn't fail.** `original[2:]` on a short string just returns an empty string, whereas direct access `original[10]` raises an `IndexError`.
