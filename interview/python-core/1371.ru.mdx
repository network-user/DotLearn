# Что такое замыкание и как функция запоминает внешние переменные?

## Суть

Замыкание - это вложенная функция, которая сохраняет доступ к переменным внешней функции даже после того, как та завершилась. Функция как бы «уносит с собой» нужное окружение.

## Что ответить на собесе

Замыкание возникает, когда внутренняя функция ссылается на переменную объемлющей области и живёт дольше, чем её создатель.

- Захваченные переменные хранятся в ячейках (cell), доступных через `__closure__`.
- На замыканиях строятся фабрики функций и декораторы.
- Чтобы присваивать внешнюю переменную (а не читать), нужен `nonlocal`, иначе Python создаст новую локальную.

## Как это работает

Внутренняя функция видит переменные внешней. Когда её возвращают наружу, связь с этими переменными сохраняется:

```python
def make_adder(x):
    def adder(y):
        return x + y
    return adder

add_five = make_adder(5)   # замыкание захватило x = 5
print(add_five(10))        # 15
print(add_five.__closure__[0].cell_contents)  # 5
```

## Подводные камни

- Классическая ловушка - захват переменной цикла: лямбды в цикле захватывают саму переменную, а не её значение на шаге. Решение - связать значение через аргумент по умолчанию: `lambda y, x=i: x + y`.
- Для присваивания внешней переменной внутри замыкания нужен `nonlocal`; без него получится новая локальная переменная.
- Замыкание захватывает переменную по ссылке, а не копию: если внешняя переменная меняется до вызова, замыкание увидит новое значение.
