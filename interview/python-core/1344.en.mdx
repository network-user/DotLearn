# What turns an ordinary object into an iterator?

## In short

An object becomes an iterator when it has a pair of methods: `__iter__()` (returns the object itself) and `__next__()` (yields the next element and raises `StopIteration` when they run out).

## How to answer in an interview

An iterator is an object that implements the iteration protocol of two methods. `__iter__()` must return the iterator itself, and `__next__()` yields the next element on each call. When there is nothing more to yield, `__next__()` raises `StopIteration`, and the `for` loop terminates cleanly. It is important not to confuse an iterable (has only `__iter__`) with an iterator (has both methods): a list is iterable but is not itself an iterator - its `__iter__` creates a separate iterator.

## How it works

Under the hood, the `for` loop first calls `iter(obj)`, that is `__iter__()`, to obtain an iterator. Then it repeatedly invokes `next()` - that is `__next__()` - until it catches `StopIteration`.

```python
class CountUpTo:
    def __init__(self, limit):
        self.limit = limit
        self.current = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.limit:
            value = self.current
            self.current += 1
            return value
        raise StopIteration


for n in CountUpTo(3):
    print(n)
```

This code prints `0`, `1`, `2`. Since `__iter__` returns `self`, the iteration state is stored right in the instance. This is how you can describe potentially infinite sequences without holding them in memory in full.

## Pitfalls

- `__iter__` must return an iterator. If it returns something without `__next__`, the `for` loop fails with `TypeError`.
- Single-use: if `__iter__` returns `self`, the iterator is exhausted after the first pass. A second `for` loop over the same instance yields nothing.
- A forgotten `raise StopIteration` leads to an infinite loop.
- An iterable (`__iter__`) and an iterator (`__iter__` + `__next__`) are different things. For an object to be traversable many times, `__iter__` must return a new iterator each time.
