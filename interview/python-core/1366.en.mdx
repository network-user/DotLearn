# What does a dictionary's items() method return, and how do you use it?

## In short

`items()` returns a `dict_items` view object containing all the "key-value" pairs. It is convenient for iterating over keys and values at the same time in a single loop.

## How to answer in an interview

`items()` gives a `dict_items` - a dynamic view of the dictionary's pairs, not a copy of them. You iterate over it, unpacking each element into `key, value`. The view is live: if the dictionary changes, so does what is visible through `items()`. When needed it can be materialized with `list()`, but for simple iteration that is unnecessary.

## How it works

Each element of `dict_items` is a `(key, value)` tuple, so in a loop it can be unpacked directly into two variables.

```python
student_grades = {"Alice": 85, "Bob": 92, "Charlie": 78}

for name, grade in student_grades.items():
    print(f"{name}: {grade}")
```

The view reflects the current state of the dictionary:

```python
d = {"a": 1}
view = d.items()
d["b"] = 2
print(list(view))  # [('a', 1), ('b', 2)] - the view sees the new key
```

Besides `items()` there are `keys()` and `values()` - they work on the same view principle.

## Pitfalls

- `items()` is not a list but a view; indexing `d.items()[0]` does not work, you first need `list()`.
- You must not modify the dictionary (add/remove keys) while iterating over `items()` - it raises `RuntimeError: dictionary changed size during iteration`.
- The view stays bound to the dictionary: storing `items()` in a variable gives you a window into the current data, not a snapshot of the past.
- The order of pairs matches insertion order (guaranteed since Python 3.7), but you cannot rely on sorting by keys.
