# Как материализовать генератор в обычный список?

## Суть

Оберните генератор в `list()` — конструктор пройдётся по нему до конца и соберёт все выданные значения в обычный список.

## Что ответить на собесе

Генератор - ленивый итератор, поэтому списком сам не является; материализуют его через `list()`.

- `list()` итерируется по генератору до `StopIteration` и складывает элементы в новый список.
- После этого доступны индекс, срезы и все методы списка.
- Минус: вся последовательность разом окажется в памяти, поэтому для бесконечных или очень больших генераторов так делать не стоит.

## Как это работает

`list()` принимает любой итерируемый объект и поглощает его целиком. Возьмём функцию-генератор:

```python
def simple_generator():
    for i in range(5):
        yield i
```

`yield`, в отличие от `return`, замораживает состояние функции: при следующем шаге выполнение продолжится с того же места. Теперь материализуем результат:

```python
gen = simple_generator()
result_list = list(gen)
print(result_list)  # [0, 1, 2, 3, 4]
```

Тот же эффект даёт распаковка `[*gen]` или генераторное выражение внутри списочного: `[x for x in gen]`.

## Подводные камни

- Генератор одноразовый: после `list(gen)` он исчерпан, повторный проход вернёт пустой результат.
- Бесконечный генератор (`while True: yield ...`) приведёт `list()` к зависанию и переполнению памяти.
- На больших данных материализация убивает главное преимущество генератора — экономию памяти. Если нужен лишь обход или агрегат (`sum`, `max`, `any`), работайте с генератором напрямую.
