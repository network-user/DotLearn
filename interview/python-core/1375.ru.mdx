# Зачем классу **slots** и какой ценой он экономит память

## Суть

`__slots__` заранее фиксирует список разрешённых атрибутов экземпляра. Взамен Python не заводит для каждого объекта словарь `__dict__`, что экономит память и слегка ускоряет доступ к атрибутам.

## Что ответить на собесе

`__slots__` фиксирует список атрибутов и убирает `__dict__`, экономя память ценой динамики.

- По умолчанию каждый экземпляр хранит атрибуты в словаре `__dict__` - гибко, но затратно по памяти.
- Со списком имён в `__slots__` Python выделяет компактные слоты вместо словаря: экономия памяти при миллионах объектов и чуть быстрее чтение/запись.
- Плата - потеря динамики: новые, не объявленные атрибуты добавить уже нельзя.

`__slots__` перечисляет допустимые имена атрибутов. Попытка завести любое другое имя завершится ошибкой:

Без `__slots__` у объекта есть `__dict__`, и в него можно дописать что угодно. Со `__slots__` этого словаря нет:

```python
class Free:
    pass

f = Free()
f.anything = 42        # ок, попадает в __dict__
print(f.__dict__)      # {'anything': 42}
```

## Подводные камни

- `__slots__` не наследуется автоматически: если подкласс не объявит свой `__slots__`, у его экземпляров снова появится `__dict__`, и экономия пропадёт.
- Со слотами нельзя добавить атрибут, которого нет в списке.
- Если всё же нужны произвольные атрибуты, добавьте `'__dict__'` в `__slots__` — но тогда экономия памяти теряется.
- Часть инструментов и библиотек (некоторые сериализаторы, кеширование через `__dict__`) рассчитывают на наличие словаря и со слотами могут работать иначе.
