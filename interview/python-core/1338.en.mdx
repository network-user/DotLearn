# Anonymous lambda functions: what are they and where do they fit?

## In short

`lambda` creates a small unnamed function from a single expression, whose result is returned without an explicit `return`. Its natural habitat is short, one-off callables that are convenient to pass straight into `sorted`, `map` or `filter`.

## How to answer in an interview

A lambda is an expression function: it is written as `lambda parameters: expression`, has no name, and contains no statements, only a single expression to evaluate. Semantically it is no different from an ordinary function created with `def` - the same `function` type, just without a bound name. I reach for it where a function is needed on the spot and only once: a sort key, a simple transformation in `map`, a predicate in `filter`. If the logic is more than one expression, or the function is reused, I write a regular `def`.

## How it works

The general form: the keyword, parameters, a colon, a single expression.

```python
lambda arguments: expression
```

A classic example is a sort key. Let's sort tuples by their second element:

```python
data = [(1, 2), (3, 1), (5, 0)]
sorted_data = sorted(data, key=lambda pair: pair[1])
print(sorted_data)  # [(5, 0), (3, 1), (1, 2)]
```

Here `key` receives a function that turns each tuple into the value used for comparison.

Lambdas often sit alongside higher-order functions:

- `map(fn, iterable)` - applies `fn` to each element;
- `filter(fn, iterable)` - keeps elements for which `fn` is truthy;
- `functools.reduce(fn, iterable)` - folds everything down to a single value.

```python
numbers = [1, 2, 3, 4]
incremented = list(map(lambda n: n + 1, numbers))
print(incremented)  # [2, 3, 4, 5]
```

## Pitfalls

- A lambda is a single expression only: no `=`, no `if`/`for` as statements, no multiple lines. Need more - write a `def`.
- Late binding of closures: `[lambda: i for i in range(3)]` will all return 2, because `i` is read at call time. Cure it with a default parameter `lambda i=i: i`.
- Assigning a lambda to a variable (`f = lambda x: x`) is an anti-pattern per PEP 8; if a function needs a name, declare it with `def`.
- To extract an attribute/element it is often cleaner to take `operator.itemgetter`/`attrgetter` instead of a lambda - faster and more readable.
