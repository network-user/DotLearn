# How to build a list in one line with a list comprehension?

## In short

A list comprehension is a Python idiom that builds a new list from any iterable source in a single expression. It's essentially a collapsed `for` loop into which you can embed both a transformation of an element and selection by a condition.

## How to answer in an interview

A list comprehension lets you describe building a list declaratively: `[expression for item in source if condition]`. It reads left to right as "take each element of the source, filter it if needed, and put the applied expression into the result." Compared with a manual loop and `append`, this form is shorter, more expressive and usually a bit faster, because the loop runs at the interpreter level rather than in Python code with a method call on every iteration.

## How it works

The full form has four parts:

```python
new_list = [expression for item in iterable if condition]
```

- `expression` - what to put into the result for the current element;
- `item` - the variable that takes each value from the source in turn;
- `iterable` - what we iterate over;
- `condition` - an optional filter; without it all elements are taken.

A simple case without a condition - squares of numbers:

```python
squares = [x ** 2 for x in range(10)]
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81]
```

With a filter we keep only the even ones:

```python
evens = [x for x in range(10) if x % 2 == 0]
# [0, 2, 4, 6, 8]
```

Nested loops are listed left to right in the same order as plain `for` loops:

```python
pairs = [(x, y) for x in range(3) for y in range(3)]
# [(0, 0), (0, 1), (0, 2), (1, 0), ... (2, 2)]
```

## Pitfalls

- An overloaded comprehension with three loops and a couple of conditions reads worse than a plain loop - in that case unfold it back.
- A ternary operator and a filter are different things: `[x if x else 0 for x in data]` puts the condition into the expression, while `[x for x in data if x]` filters. The position of `if` changes the meaning.
- A comprehension materializes the whole list in memory. If it's large and you only need a single pass, use a generator expression `(... for ...)` in parentheses.
- The loop variable inside a comprehension does not leak into the surrounding scope (unlike Python 2) - don't expect to see `item` after the comprehension.
