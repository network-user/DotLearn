# What built-in data types does Python have, and how can you group them

## In short

It is convenient to keep the built-in types in your head not as a flat list but in groups: numbers, logic, text, collections, and "nothingness". Each group contains several concrete types, and almost any data fits onto these shelves.

## How to answer in an interview

Python has numeric types (`int`, `float`, `complex`), the boolean `bool`, the string `str`, collections (`list`, `tuple`, `set`, `dict`), and a separate `NoneType` with the single value `None`. Collections split into mutable (`list`, `set`, `dict`) and immutable (`tuple`, `str`). `bool` is a subclass of `int`, so `True` equals `1` and `False` equals `0`.

Numbers cover three cases: integers with no bit-width limit, floating-point fractions, and complex numbers.

```python
a = 5          # int
b = 3.14       # float
c = 2 + 3j     # complex
```

Logic is just two values, but technically `bool` inherits from `int`:

```python
print(True + True)   # 2
print(isinstance(True, int))  # True
```

Text is stored in the immutable string `str`, while collections differ in mutability and whether they keep order:

```python
fruits = ["apple", "banana"]   # list: mutable, ordered
point = (10.0, 20.0)            # tuple: immutable
unique = {1, 2, 3}             # set: unique elements, unordered
person = {"name": "Alice"}     # dict: key-value pairs
```

The absence of a value is expressed through `None` of type `NoneType` - it is a singleton, so it is compared with `is`, not `==`.

```python
result = None
print(result is None)  # True
```

## Pitfalls

- `bool` is a subclass of `int`, so `True == 1`, and the list `[True, 1]` collapses into a single element inside a set.
- Keys of a `dict` and elements of a `set` can only be hashable objects: you cannot put a `list` in there, but a `tuple` of immutable elements is fine.
- `is` is mandatory for checking against `None`; for numbers and strings you must not use `is` - that is a caching detail, not value equality.
- `float` stores an approximation: `0.1 + 0.2 != 0.3`.
