# Какими способами в Python можно получить генератор

## Суть

Ленивый итератор в Python получают тремя способами: функцией с `yield`, генераторным выражением в круглых скобках и собственным классом с методами `__iter__`/`__next__`. Все три выдают значения по требованию и не держат всю последовательность в памяти.

## Что ответить на собесе

Ленивый итератор получают тремя способами; объединяет их ленивость - значения считаются по мере запроса.

- Функция с `yield` - чаще всего: сохраняет состояние между обращениями и отдаёт элементы по одному.
- Генераторное выражение `(x for x in ...)` - для коротких преобразований: компактнее списка и не материализует результат.
- Класс-итератор с `__iter__` и `__next__` - когда логика сложная и хранит много состояния; перебор завершается через `raise StopIteration`.

## Как это работает

Функция с `yield` приостанавливается на каждом значении и продолжает с того же места при следующем `next()`.

```python
def count_up_to(limit):
    count = 1
    while count <= limit:
        yield count
        count += 1
```

Генераторное выражение - тот же генератор, но записанный одной строкой в круглых скобках:

```python
squares = (x * x for x in range(10))
```

Класс-итератор даёт полный контроль над состоянием и сигналом окончания:

```python
class Countdown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= 0:
            raise StopIteration
        self.current -= 1
        return self.current
```

## Подводные камни

- Генератор одноразовый: после полного прохода он исчерпан, повторный цикл по нему даст пустоту. Чтобы пройти заново, нужно создать его заново.
- Не путайте `(x for x in ...)` (генератор) и `[x for x in ...]` (список): второй сразу занимает память под все элементы.
- В классе-итераторе `__iter__` обязан вернуть объект с `__next__`; если вернуть сам объект, повторная итерация продолжится с текущего, уже сдвинутого состояния.
- `return` внутри генераторной функции не возвращает значение, а завершает её, превращаясь в `StopIteration`.
