# Поверхностное и глубокое копирование объектов: где проходит граница

## Суть

`copy.copy` дублирует только внешнюю оболочку объекта, а ссылки на вложенные структуры остаются общими с оригиналом. `copy.deepcopy` спускается вглубь и копирует каждый уровень, поэтому результат полностью отвязан от исходника.

## Что ответить на собесе

В модуле `copy` две функции. Поверхностная копия (`copy`) создаёт новый объект верхнего уровня, но всё, что лежит внутри, шарится по ссылке: меняешь вложенный список в копии - меняется и в оригинале. Глубокая копия (`deepcopy`) рекурсивно проходит по всей структуре и создаёт независимые дубликаты, так что две ветки больше никак не связаны. Выбираю по задаче: если хватает отвязки верхнего уровня - беру `copy`, если нужна полная независимость с вложенностью - `deepcopy`, помня про его цену по времени.

Поверхностная копия делает новый контейнер, но кладёт в него те же самые объекты-элементы:

```python
import copy

source = [1, 2, [3, 4]]
shallow = copy.copy(source)

shallow[2][0] = "changed"

print(source)   # [1, 2, ['changed', 4]]
print(shallow)  # [1, 2, ['changed', 4]]
```

Внешний список новый, но вложенный список один на двоих, поэтому правка видна с обеих сторон.

Глубокая копия рекурсивно дублирует и вложенные объекты тоже:

```python
import copy

source = [1, 2, [3, 4]]
deep = copy.deepcopy(source)

deep[2][0] = "changed"

print(source)  # [1, 2, [3, 4]]
print(deep)    # [1, 2, ['changed', 4]]
```

Теперь у каждой структуры свой вложенный список, и оригинал остаётся нетронутым.

## Подводные камни

- Присваивание `b = a` вообще не копия - это второе имя того же объекта; обе функции из `copy` тут ни при чём.
- `deepcopy` дороже по времени и памяти: на больших графах объектов это заметно.
- Для циклических ссылок `deepcopy` корректно отрабатывает за счёт внутреннего memo-словаря и не зацикливается.
- Неизменяемые элементы (числа, строки, кортежи без вложенных списков) `deepcopy` может не дублировать - им незачем, и это нормально.
- Поведение копирования настраивается через `__copy__` и `__deepcopy__` на собственных классах.
