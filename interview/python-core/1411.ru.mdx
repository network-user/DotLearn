# Как проверить, можно ли использовать объект как ключ словаря?

## Суть

Объект хешируем, если у него есть рабочий метод `__hash__` и его можно класть в ключи словаря или элементы множества. Самая прямая проверка - вызвать `hash(obj)`: вернулось число - хешируемый, прилетел `TypeError` - нет.

## Что ответить на собесе

Хешируемость - это сводимость объекта к целочисленному хешу, неизменному за время жизни объекта.

- Хешируемы обычно неизменяемые типы: числа, строки, кортежи из хешируемых элементов, `frozenset`.
- Изменяемые контейнеры (список, словарь, множество) нехешируемы, потому что их хеш мог бы меняться.
- Проверить можно вызовом `hash()` в `try/except TypeError` либо через `isinstance(obj, collections.abc.Hashable)`.

## Как это работает

Структуры вроде `dict` и `set` хранят элементы по их хешу, поэтому требуют, чтобы ключи и элементы были хешируемыми и не меняли свой хеш.

```python
def is_hashable(obj):
    try:
        hash(obj)
        return True
    except TypeError:
        return False

is_hashable("example")   # True  — строка неизменяема
is_hashable((1, 2, 3))   # True  — кортеж из чисел
is_hashable([1, 2, 3])   # False — список изменяем
is_hashable({1, 2})      # False — множество изменяемо
```

Альтернатива - проверка через абстрактный базовый класс:

```python
from collections.abc import Hashable
isinstance("example", Hashable)   # True
isinstance([1, 2, 3], Hashable)   # False
```

Связь с неизменяемостью прямая: если бы хеш менялся, объект «потерялся» бы в таблице, ведь искать его стали бы по новому хешу.

## Подводные камни

- **Хешируемость кортежа зависит от содержимого.** `(1, [2])` нехешируем, потому что внутри изменяемый список; `hash((1, [2]))` поднимет `TypeError`.
- **`__hash__ = None` отключает хеш.** Если у пользовательского класса определён `__eq__`, но не `__hash__`, экземпляры становятся нехешируемыми автоматически.
- **`isinstance(obj, Hashable)` может обмануть.** Он проверяет лишь наличие `__hash__`, но фактический вызов всё равно может упасть (как с кортежем со списком внутри). Надёжнее реально вызвать `hash()`.
- **Хешируемость не равна уникальности.** Разные объекты могут иметь одинаковый хеш (коллизия) - это нормально и разрешается внутри таблицы.
