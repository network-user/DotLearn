# Shallow and deep copying of objects: where is the line?

## In short

`copy.copy` duplicates only the outer shell of an object, while references to nested structures remain shared with the original. `copy.deepcopy` descends all the way down and copies every level, so the result is completely detached from the source.

## How to answer in an interview

The `copy` module has two functions. A shallow copy (`copy`) creates a new top-level object, but everything inside is shared by reference: change a nested list in the copy - it changes in the original too. A deep copy (`deepcopy`) recursively walks the entire structure and creates independent duplicates, so the two branches are no longer connected in any way. I choose based on the task: if detaching the top level is enough - I take `copy`; if I need full independence with nesting - `deepcopy`, keeping its time cost in mind.

A shallow copy makes a new container but puts the very same element objects into it:

```python
import copy

source = [1, 2, [3, 4]]
shallow = copy.copy(source)

shallow[2][0] = "changed"

print(source)   # [1, 2, ['changed', 4]]
print(shallow)  # [1, 2, ['changed', 4]]
```

The outer list is new, but the nested list is one for both, so the edit is visible from either side.

A deep copy recursively duplicates the nested objects too:

```python
import copy

source = [1, 2, [3, 4]]
deep = copy.deepcopy(source)

deep[2][0] = "changed"

print(source)  # [1, 2, [3, 4]]
print(deep)    # [1, 2, ['changed', 4]]
```

Now each structure has its own nested list, and the original stays untouched.

## Pitfalls

- Assignment `b = a` is not a copy at all - it is a second name for the same object; neither function from `copy` is involved here.
- `deepcopy` is more expensive in time and memory: on large object graphs this is noticeable.
- For cyclic references `deepcopy` works correctly thanks to an internal memo dictionary and does not loop forever.
- Immutable elements (numbers, strings, tuples without nested lists) may not be duplicated by `deepcopy` - they don't need to be, and that is fine.
- Copying behavior is customizable via `__copy__` and `__deepcopy__` on your own classes.
