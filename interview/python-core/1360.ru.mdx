# Чем посмотреть полный набор членов произвольного объекта во время выполнения?

## Суть

Встроенная `dir(obj)` возвращает отсортированный список имён, доступных у объекта, включая унаследованные атрибуты, методы и дандер-члены.

## Что ответить на собесе

Для интроспекции объекта я беру `dir(obj)` - она отдаёт список строк со всеми доступными именами, в том числе унаследованными от родительских классов и базовых типов. Если нужны только собственные атрибуты экземпляра, смотрю `vars(obj)` или `obj.__dict__`; если нужно одно конкретное имя - `getattr` и `hasattr`. `dir()` без аргументов выдаёт имена из текущей локальной области.

## Как это работает

`dir()` собирает имена по всей цепочке наследования (MRO), а не только у самого экземпляра, поэтому в выдачу попадают и методы базовых классов, и дандеры вроде `__init__`, `__class__`. Результат всегда отсортирован по алфавиту и состоит из строк.

```python
class MyClass:
    def __init__(self):
        self.attribute1 = "value1"
        self.attribute2 = "value2"

    def method1(self):
        pass

    def method2(self):
        pass

obj = MyClass()

names = dir(obj)
print(names)
```

В выводе окажутся не только `attribute1`, `attribute2`, `method1`, `method2`, но и весь набор унаследованных дандер-членов. Если нужны именно данные конкретного экземпляра без наследования и методов класса, точнее работает `obj.__dict__`:

```python
print(obj.__dict__)
```

## Подводные камни

- `dir()` показывает имена, но не различает, где они определены - у экземпляра, у класса или у предка. Для собственных полей экземпляра берите `__dict__` или `vars()`.
- Объект может определять `__dir__`, и тогда выдача `dir()` кастомизирована - это не всегда исчерпывающий перечень реального состояния.
- Не путайте `dir()` (имена как строки) с `getattr(obj, name)` - чтобы получить значение, имя нужно разрешить отдельно.
- `dir()` не покажет атрибуты, добавляемые динамически через `__getattr__`, если их фактически нет в `__dict__`.
