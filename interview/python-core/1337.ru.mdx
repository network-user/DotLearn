# Правило LEGB: как Python находит имя переменной?

## Суть

В Python четыре уровня видимости имён, и интерпретатор просматривает их строго по порядку LEGB: Local (локальная), Enclosing (объемлющая), Global (модульная) и Built-in (встроенная). Первое совпадение и определяет, на что ссылается имя.

## Что ответить на собесе

Python ищет имя строго по порядку LEGB и берёт первое совпадение.

- Local (локальная функции) → Enclosing (объемлющие функции) → Global (имена модуля) → Built-in (`len`, `print`, `Exception`).
- Присваивание по умолчанию создаёт локальное имя.
- Чтобы писать в глобальную или объемлющую переменную, нужны `global` и `nonlocal` соответственно.

## Как это работает

Каждый уровень - это отдельное пространство имён, и поиск идёт изнутри наружу.

Локальная область создаётся на каждый вызов функции заново:

```python
def show():
    x = 10
    print(x)

show()
```

Объемлющая область - переменные внешней функции, видимые во вложенной:

Глобальная область - имена уровня модуля, доступные всем его функциям:

```python
z = 30

def read_global():
    print(z)

read_global()
```

Встроенная область - то, что Python даёт изначально, без импортов:

```python
print(len("Hello"))
```

## Подводные камни

- Присваивание внутри функции делает имя локальным на весь её код. Если до присваивания вы читаете ту же переменную, получите `UnboundLocalError`, а не значение из глобали.
- `global x` нужен, чтобы переприсвоить модульную переменную; `nonlocal x` - чтобы изменить переменную ближайшей объемлющей функции (не глобальной).
- Мутация и присваивание - разные вещи: `lst.append(1)` для глобального списка работает без `global`, а `lst = [...]` создаст локальную переменную.
- Затенение встроенных имён (`list = [1, 2]`) ломает доступ к функции `list()` в этой области - имя из L перекрывает B.
