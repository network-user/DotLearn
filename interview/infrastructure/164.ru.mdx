# Минимальный рабочий набор команд Git и что каждая из них делает

## Суть

Git - распределённая система контроля версий: она фиксирует изменения в коде и даёт команде работать над ним совместно. Базовый набор команд покрывает создание репозитория, фиксацию правок и обмен ими с удалённым сервером.

## Что ответить на собесе

Git хранит историю изменений и позволяет работать над кодом параллельно. Из ежедневного минимума я бы назвал: `git init` создаёт репозиторий, `git clone` копирует существующий, `git add` кладёт изменения в индекс, `git commit` фиксирует их снимком, `git status` показывает состояние, `git push` отправляет коммиты на сервер, `git pull` забирает чужие изменения, `git branch` и `git checkout` управляют ветками, `git merge` сливает их, а `git log` показывает историю. Ключевая идея - трёхступенчатая модель: рабочая директория, индекс (staging) и история коммитов; `add` двигает из первого во второй, `commit` из второго в третий.

```bash
git init
```
Создаёт новый репозиторий в текущей папке - с этого начинается отслеживание изменений.

```bash
git clone <repository-url>
```
Делает локальную копию существующего репозитория.

```bash
git add <file-name>
git add .
```
Помещает изменения в индекс (staging area). Точка добавляет все изменения сразу.

```bash
git commit -m "Описание изменений"
```
Записывает содержимое индекса в историю - делает снимок состояния проекта.

```bash
git status
```
Показывает, что добавлено в индекс, а что ещё нет.

```bash
git push origin <branch-name>
git pull origin <branch-name>
```
`push` отправляет коммиты на удалённый сервер, `pull` забирает чужие изменения и сливает их в локальную копию.

```bash
git branch <new-branch-name>
git checkout <branch-name>
git merge <branch-name>
git log
```
`branch` создаёт ветку, `checkout` переключает, `merge` сливает изменения одной ветки в другую, `log` выводит историю коммитов.

## Подводные камни

- `git add` не фиксирует изменения - он лишь готовит их в индексе; без `commit` ничего не попадёт в историю.
- `git pull` - это `fetch` + `merge`: при расхождении он может создать коммит слияния или потребовать разрешить конфликт.
- `git commit -m` без `add` зафиксирует только то, что уже в индексе, а не все правки рабочей директории.
- Современный Git предлагает `git switch` и `git restore` вместо перегруженного `git checkout` - на собесе уместно про это упомянуть.
