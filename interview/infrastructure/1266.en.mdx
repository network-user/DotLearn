# The Linux file permission model: owner, group, others, and three flags

## In short

In Linux every file and directory stores three sets of permissions - for the owner, for the group, and for everyone else. Each set has exactly three flags: read (r), write (w), and execute (x). You manage them with `chmod` (permissions), `chown` (owner), and `chgrp` (group).

## How to answer in an interview

Linux permissions are arranged as a "who × what's allowed" matrix. Horizontally there are three classes of subjects: the file's owner, its group, and everyone else. For each class three bits are defined - r, w, x. In `ls -l` output this is ten characters: the first is the object type, then three triplets in a row. I change permissions with `chmod` (symbolically `u+x` or numerically `755`), the owner with `chown`, the group with `chgrp`. The numeric notation is convenient because each digit is a sum of bits: 4 (r) + 2 (w) + 1 (x).

The subject class is determined like this: if you are the owner, the owner's permissions apply; otherwise, if you are in the file's group, the group's permissions apply; otherwise, the permissions for others apply. The check goes exactly in this order and stops at the first match.

The flags mean different things for a file and a directory:

- **r** - read the file / list the names in a directory.
- **w** - modify the file / create and delete entries in a directory.
- **x** - run the file as a program / enter the directory (make it current).

The `ls -l` string looks like `-rwxr-xr--`:

- The first character is the type (`-` file, `d` directory, `l` symlink).
- Characters 2-4 (`rwx`) - owner.
- Characters 5-7 (`r-x`) - group.
- Characters 8-10 (`r--`) - others.

The numeric form sums the bits within each triplet: r=4, w=2, x=1.

```bash
chmod 755 file.txt
```

- `7` = 4+2+1 = rwx for the owner.
- `5` = 4+0+1 = r-x for the group.
- `5` = r-x for others.

The symbolic form precisely adds or removes a flag:

```bash
chmod u+x file.txt
chmod go-w file.txt
```

- `u+x` - add execute for the owner.
- `go-w` - remove write from group and others.

Changing the owner and group:

```bash
chown newowner file.txt
chown newowner:newgroup file.txt
chgrp newgroup file.txt
```

A from-scratch example:

```bash
touch example.txt
chmod 644 example.txt
```

`644` gives the owner rw-, and the group and others r--.

## Pitfalls

- **A directory needs the x bit, not just r.** Without `x` you can't enter it even if `r` is set - `ls` will show names, but accessing the file contents will fail.
- **`chmod 777` is almost always a mistake.** It means "everything to everyone", a direct security hole. Grant only the minimum necessary.
- **Numeric `chmod` overwrites all bits at once**, while symbolic (`+`/`-`) changes only the specified ones. Confusing them leads to accidentally granting extra permissions.
- **`chown` usually requires root.** An ordinary user can't "give away" their file to another without `sudo`.
- **Recursion is dangerous:** `chmod -R 755` on a tree of files and directories will slap `x` onto regular files too. Often you need separate rules for files and directories.
- **Permissions on a symlink are ignored** - the target file's permissions apply, and `chmod` follows the link by default.
