# What does the operating system store about each process?

## In short

For each process the OS keeps a control block (PCB): the PID identifier, state, priority, execution context (registers and stack), a reference to the parent, the table of open descriptors, the memory it occupies, and accumulated CPU time. The scheduler relies on these attributes when switching tasks.

## How to answer in an interview

A process is a running instance of a program, and the OS describes it with a Process Control Block structure. The minimal set interviewers expect to hear: the PID as a unique identifier; the state (running, ready, waiting, terminated); priority for the scheduler; the saved context (registers, instruction pointer, stack pointer), which lets the process be paused and resumed from the same spot; the parent's PID, defining the hierarchy; the file descriptor table; information about allocated memory (code, data, stack, heap); accumulated CPU time. It is exactly the context that makes preemptive multitasking possible: when switching, the OS saves one process's context and restores another's.

The OS stores attributes in the PCB and, on a context switch, dumps the current process's registers into its PCB and then loads the next process's registers. Some of these attributes are visible from user space, for example through `psutil`.

```python
import psutil

process = psutil.Process()

print(f"PID: {process.pid}")
print(f"Status: {process.status()}")
print(f"Priority (nice): {process.nice()}")

parent = process.parent()
print(f"Parent PID: {parent.pid if parent else 'no parent'}")

print(f"Open files: {len(process.open_files())}")
print(f"RSS memory: {process.memory_info().rss} bytes")

cpu = process.cpu_times()
print(f"CPU time: {cpu.user + cpu.system} seconds")
```

- `pid`: the process identifier;
- `status()`: the current state;
- `nice()`: the priority value (the lower the nice, the higher the priority);
- `parent()`: the parent process;
- `open_files()`: open file descriptors;
- `memory_info().rss`: resident memory;
- `cpu_times()`: CPU time spent.

## Pitfalls

- On Linux a low nice value means high priority, not low, which is a common source of confusion.
- A process's state is not just "running/not running": there is a separate zombie (terminated but not yet reaped by the parent) and the uninterruptible sleep state, in which a process cannot be killed with an ordinary signal.
- Do not confuse a process and a thread: threads of one process share memory and descriptors, but each has its own context and stack.
- `rss` shows resident physical memory and, because of shared pages, overstates real consumption when summed across processes.
- When a parent terminates, its children are adopted by init/systemd (PID 1), rather than "always being stopped"; the behavior depends on settings and signals.
