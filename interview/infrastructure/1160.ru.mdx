# Что операционная система хранит про каждый процесс?

## Суть

Про каждый процесс ОС держит блок управления (PCB): идентификатор PID, состояние, приоритет, контекст выполнения (регистры и стек), ссылку на родителя, таблицу открытых дескрипторов, занятую память и накопленное процессорное время. На эти атрибуты опирается планировщик при переключении задач.

## Что ответить на собесе

Процесс — это запущенный экземпляр программы, и ОС описывает его структурой Process Control Block. Минимальный набор, который ждут услышать:

- PID как уникальный идентификатор и состояние (running, ready, waiting, terminated), плюс приоритет для планировщика.
- Сохранённый контекст (регистры, счётчик команд, указатель стека) — он позволяет приостановить и возобновить процесс с того же места.
- PID родителя, задающий иерархию; таблица файловых дескрипторов; сведения о выделенной памяти (код, данные, стек, куча); накопленное CPU-время.
- Именно контекст делает возможной вытесняющую многозадачность: при переключении ОС сохраняет контекст одного процесса и восстанавливает контекст другого.

ОС хранит атрибуты в PCB и при переключении контекста сбрасывает регистры текущего процесса в его PCB, а затем загружает регистры следующего. Часть этих атрибутов видна из пользовательского пространства, например через `psutil`.

```python
import psutil

process = psutil.Process()

print(f"PID: {process.pid}")
print(f"Status: {process.status()}")
print(f"Priority (nice): {process.nice()}")

parent = process.parent()
print(f"Parent PID: {parent.pid if parent else 'no parent'}")

print(f"Open files: {len(process.open_files())}")
print(f"RSS memory: {process.memory_info().rss} bytes")

cpu = process.cpu_times()
print(f"CPU time: {cpu.user + cpu.system} seconds")
```

- `pid` — идентификатор процесса;
- `status()` — текущее состояние;
- `nice()` — значение приоритета (чем меньше nice, тем выше приоритет);
- `parent()` — родительский процесс;
- `open_files()` — открытые файловые дескрипторы;
- `memory_info().rss` — резидентная память;
- `cpu_times()` — потраченное процессорное время.

## Подводные камни

- В Linux низкое значение nice означает высокий приоритет, а не низкий — частая путаница.
- Состояние процесса не сводится к «работает/не работает»: есть отдельный зомби (terminated, но не reaped родителем) и состояние uninterruptible sleep, в котором процесс нельзя убить обычным сигналом.
- Не путайте процесс и поток: потоки одного процесса делят память и дескрипторы, но имеют собственный контекст и стек.
- `rss` показывает резидентную физическую память и из-за разделяемых страниц завышает реальное потребление при суммировании по процессам.
- При завершении родителя его потомков усыновляет init/systemd (PID 1), а не «всегда останавливает» — поведение зависит от настроек и сигналов.
