# The minimal working set of Git commands and what each one does

## In short

Git is a distributed version control system: it records changes in code and lets a team work on it together. The basic command set covers creating a repository, recording edits, and exchanging them with a remote server.

## How to answer in an interview

Git stores the history of changes and lets you work on code in parallel. From the daily minimum I'd name: `git init` creates a repository, `git clone` copies an existing one, `git add` stages changes, `git commit` records them as a snapshot, `git status` shows the state, `git push` sends commits to the server, `git pull` fetches others' changes, `git branch` and `git checkout` manage branches, `git merge` joins them, and `git log` shows the history. The key idea is the three-stage model: working directory, index (staging), and commit history; `add` moves from the first to the second, `commit` from the second to the third.

```bash
git init
```
Creates a new repository in the current folder - this is where change tracking begins.

```bash
git clone <repository-url>
```
Makes a local copy of an existing repository.

```bash
git add <file-name>
git add .
```
Places changes into the index (staging area). The dot adds all changes at once.

```bash
git commit -m "Change description"
```
Writes the index's contents into history - takes a snapshot of the project's state.

```bash
git status
```
Shows what's staged and what isn't yet.

```bash
git push origin <branch-name>
git pull origin <branch-name>
```
`push` sends commits to the remote server, `pull` fetches others' changes and merges them into the local copy.

```bash
git branch <new-branch-name>
git checkout <branch-name>
git merge <branch-name>
git log
```
`branch` creates a branch, `checkout` switches, `merge` merges changes from one branch into another, `log` prints the commit history.

## Pitfalls

- `git add` doesn't record changes - it only prepares them in the index; without `commit` nothing lands in history.
- `git pull` is `fetch` + `merge`: on divergence it may create a merge commit or require resolving a conflict.
- `git commit -m` without `add` records only what's already in the index, not all edits in the working directory.
- Modern Git offers `git switch` and `git restore` instead of the overloaded `git checkout` - it's worth mentioning this in an interview.
