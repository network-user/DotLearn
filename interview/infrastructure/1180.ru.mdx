# CMD против ENTRYPOINT: кто задаёт команду, а кто только аргументы?

## Суть

Обе инструкции описывают, что запустится при старте контейнера, но по-разному реагируют на аргументы из `docker run`. CMD легко затирается целиком, ENTRYPOINT остаётся неизменным ядром команды.

## Что ответить на собесе

Разница в том, как они реагируют на аргументы из `docker run`:

- CMD - это поведение по умолчанию: всё, что я передам в `docker run`, полностью заменит её.
- ENTRYPOINT задаёт фиксированную команду, а аргументы из `docker run` лишь дописываются к ней, поэтому контейнер с ENTRYPOINT ведёт себя как самостоятельный исполняемый файл.
- На практике их связывают: ENTRYPOINT - сама программа, CMD - её аргументы по умолчанию, которые можно переопределить, не трогая саму программу.

## Как это работает

CMD выполняется, только если при запуске не передали свою команду. Стоит указать что-то в `docker run` - и CMD исчезает целиком.

```dockerfile
FROM ubuntu:latest
CMD ["echo", "Hello, World!"]
```

Запуск без аргументов выведет `Hello, World!`. А `docker run <image> ls` подменит всю строку, и отработает только `ls`.

ENTRYPOINT задаёт основу, которую нельзя выкинуть простой передачей аргументов - аргументы приклеиваются к ней справа.

```dockerfile
FROM ubuntu:latest
ENTRYPOINT ["echo"]
```

Теперь `docker run <image> "Hello, World!"` выполнит именно `echo "Hello, World!"`: `echo` неизменен, а строка - его аргумент.

Связка раскрывает обе инструкции: ENTRYPOINT - команда, CMD - аргументы по умолчанию к ней.

```dockerfile
FROM ubuntu:latest
ENTRYPOINT ["echo"]
CMD ["Hello, World!"]
```

Без аргументов получится `echo "Hello, World!"`. С аргументом `docker run <image> "Goodbye, World!"` CMD подменится, и выйдет `echo "Goodbye, World!"`, тогда как сам `echo` останется на месте.

## Подводные камни

- Разница в форме записи. Exec-форма `["echo"]` запускает процесс напрямую, а shell-форма `echo $X` оборачивает его в `/bin/sh -c`. В shell-форме ENTRYPOINT внешний CMD и аргументы из `docker run` игнорируются - частая причина «почему мои аргументы не доходят».
- Без exec-формы ENTRYPOINT процесс не получает PID 1 напрямую, и сигналы (SIGTERM при остановке) могут не доходить до приложения.
- Если задан только CMD, любой аргумент в `docker run` затирает его полностью - легко случайно «потерять» команду запуска.
- Переопределить сам ENTRYPOINT можно лишь явным флагом `--entrypoint`, а не позиционными аргументами.
