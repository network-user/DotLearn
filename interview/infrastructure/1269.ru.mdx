# Базовый набор команд системного администратора Linux

## Суть

Администрирование Linux держится на нескольких группах команд: повышение прав (`sudo`), управление пользователями (`useradd`, `usermod`), права на файлы (`chmod`, `chown`), службы (`systemctl`), диск (`df`, `du`) и процессы (`ps`, `top`, `kill`).

## Что ответить на собесе

Я бы сгруппировал команды по задачам, а не зубрил списком:

- Привилегии - `sudo` для разового повышения прав.
- Пользователи - `useradd`/`usermod` (например, `usermod -aG` добавляет в группу, не затирая существующие).
- Права на файлы - `chmod` и `chown`; службы в современных системах - через `systemctl` (start/stop/status/enable).
- Диск - `df` для разделов и `du` для конкретных каталогов; процессы - `ps`/`top` для просмотра и `kill` для завершения по PID.

Главное на собесе - показать, что понимаешь, _зачем_ команда, а не только её синтаксис.

## Как это работает

**Привилегии - `sudo`:**

```bash
sudo apt update
```

Временно повышает права для одной команды; кто и что может через sudo, описано в `/etc/sudoers`.

**Пользователи - `useradd`, `usermod`:**

```bash
sudo useradd newuser
sudo usermod -aG sudo newuser
```

Первая заводит пользователя, вторая добавляет его в группу `sudo`. Флаг `-aG` важен: `-a` (append) дописывает группу, иначе `-G` перезапишет весь список групп.

**Права - `chmod`, `chown`:**

```bash
chmod 755 script.sh
chown user:group script.sh
```

`755` - владельцу rwx, остальным r-x. `chown user:group` меняет владельца и группу разом.

**Службы - `systemctl`:**

```bash
sudo systemctl start apache2
sudo systemctl status apache2
sudo systemctl enable apache2
```

`start` запускает сейчас, `status` показывает состояние, `enable` включает автозапуск при загрузке (это разные вещи).

**Диск - `df`, `du`:**

```bash
df -h
du -sh /var/log
```

`df -h` - занятость файловых систем, `du -sh` - суммарный размер каталога.

**Процессы - `ps`, `top`, `kill`:**

```bash
ps aux
top
kill 1234
```

`ps aux` - снимок всех процессов, `top` - живая картина нагрузки, `kill 1234` шлёт сигнал процессу с PID 1234.

## Подводные камни

- **`usermod -G` без `-a` стирает группы.** Забыли `-a` - и пользователь вылетел из всех прежних групп. Почти всегда нужен `usermod -aG`.
- **`start` != `enable`.** `systemctl start` поднимает службу до перезагрузки; чтобы она стартовала после ребута, нужен `enable`. Частая путаница.
- **`kill` по умолчанию шлёт SIGTERM, а не «убивает».** Процесс может его перехватить и проигнорировать. Жёсткое завершение - `kill -9` (SIGKILL), но это крайняя мера: не даёт корректно закрыть ресурсы.
- **`df` и `du` могут расходиться.** Удалённый, но открытый файл всё ещё занимает место на диске (виден в `df`, но не в `du`), пока его держит процесс.
- **`sudo` - не «режим бога навсегда».** Права действуют на конкретную команду; пайпы и редиректы выполняются уже без root (`sudo echo x > /root/f` упадёт - редирект делает шелл).
- **`useradd` vs `adduser`.** Низкоуровневый `useradd` по умолчанию не создаёт домашний каталог и не спрашивает пароль; в Debian-системах дружелюбнее обёртка `adduser`.
