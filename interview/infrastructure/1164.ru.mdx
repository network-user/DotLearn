# За что отвечают systemd и sysfs в современном Linux

## Суть

systemd — это менеджер инициализации и служб: он поднимает процессы при загрузке и управляет их жизненным циклом. sysfs — виртуальная файловая система в `/sys`, через которую ядро отдаёт информацию об устройствах и параметрах в виде обычных файлов.

## Что ответить на собесе

systemd и sysfs решают разные задачи и не конкурируют. systemd отвечает за запуск и управление сервисами: параллельный старт с учётом зависимостей, перезапуск упавших процессов, единый интерфейс `systemctl`. Он пришёл на смену System V init. sysfs — это не про сервисы, а про доступ к внутренностям ядра: через файлы в `/sys` можно прочитать или поменять параметры железа и драйверов. Грубо говоря, systemd управляет тем, что запускается, а sysfs показывает, на чём это работает.

## Как это работает

**systemd.** Всё описывается «юнитами»: сервис, точка монтирования, сокет, таймер. systemd читает зависимости между юнитами и запускает то, что можно, параллельно, ускоряя загрузку. Управление идёт через `systemctl`:

```bash
sudo systemctl start nginx      # поднять службу
sudo systemctl stop nginx       # остановить
sudo systemctl restart nginx    # перезапустить
sudo systemctl status nginx     # текущее состояние
sudo systemctl enable nginx     # автозапуск при загрузке
```

**sysfs.** Монтируется в `/sys` и раскладывает данные ядра по дереву каталогов, где каждый узел — конкретный аспект системы. Читаешь файл — получаешь значение, пишешь в файл — меняешь параметр:

```bash
# прочитать текущий режим управления частотой CPU
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor

# переключить на максимальную производительность
echo "performance" | sudo tee /sys/devices/system/cpu/cpu0/cpufreq/scaling_governor
```

## Подводные камни

- sysfs не хранит данные на диске — это интерфейс ядра в памяти, изменения через `echo` не переживают перезагрузку. Для постоянных настроек нужны конфиги (sysctl, udev-правила, systemd-юниты).
- Запись в `/sys` требует root: обычный `echo > файл` упадёт с Permission denied, поэтому используют `sudo tee`.
- Не путайте sysfs (`/sys`, про устройства) и procfs (`/proc`, про процессы) — это две разные виртуальные ФС.
- `systemctl start` поднимает службу один раз, но без `enable` она не стартует после перезагрузки.
