# Модель прав на файлы в Linux: владелец, группа, остальные и три флага

## Суть

В Linux каждый файл и каталог хранит три набора прав - для владельца, для группы и для всех остальных. В каждом наборе ровно три флага: чтение (r), запись (w) и выполнение (x). Управляют этим командами `chmod` (права), `chown` (владелец) и `chgrp` (группа).

## Что ответить на собесе

Права в Linux устроены как матрица «кто × что можно»:

- Три класса субъектов: владелец файла, его группа и все прочие; для каждого задаётся три бита - r, w, x.
- В выводе `ls -l` это десять символов: первый - тип объекта, дальше три тройки подряд.
- Меняю права через `chmod` (символьно `u+x` или числом `755`), владельца через `chown`, группу через `chgrp`.
- Числовая запись удобна тем, что каждая цифра - это сумма битов: 4 (r) + 2 (w) + 1 (x).

Класс субъекта определяется так: если ты владелец - действуют права владельца; иначе если ты в группе файла - права группы; иначе - права остальных. Проверка идёт именно в этом порядке и останавливается на первом совпадении.

Флаги для файла и каталога значат разное:

- **r** - читать файл / получать список имён в каталоге.
- **w** - менять файл / создавать и удалять записи в каталоге.
- **x** - запускать файл как программу / входить в каталог (делать его текущим).

Строка `ls -l` выглядит как `-rwxr-xr--`:

- Первый символ - тип (`-` файл, `d` каталог, `l` симлинк).
- Символы 2-4 (`rwx`) - владелец.
- Символы 5-7 (`r-x`) - группа.
- Символы 8-10 (`r--`) - остальные.

Числовая форма складывает биты в каждой тройке: r=4, w=2, x=1.

```bash
chmod 755 file.txt
```

- `7` = 4+2+1 = rwx для владельца.
- `5` = 4+0+1 = r-x для группы.
- `5` = r-x для остальных.

Символьная форма точечно добавляет или убирает флаг:

```bash
chmod u+x file.txt
chmod go-w file.txt
```

- `u+x` - добавить владельцу выполнение.
- `go-w` - снять запись у группы и остальных.

Смена владельца и группы:

```bash
chown newowner file.txt
chown newowner:newgroup file.txt
chgrp newgroup file.txt
```

Пример с нуля:

```bash
touch example.txt
chmod 644 example.txt
```

`644` даёт владельцу rw-, а группе и остальным r--.

## Подводные камни

- **Каталогу нужен бит x, а не только r.** Без `x` нельзя зайти внутрь, даже если `r` стоит - `ls` покажет имена, но обращение к содержимому файлов сорвётся.
- **`chmod 777` - почти всегда ошибка.** Это «всё всем», прямая дыра в безопасности. Выдавайте минимально необходимое.
- **Числовой `chmod` затирает все биты сразу**, а символьный (`+`/`-`) меняет только указанные. Путаница приводит к случайной выдаче лишних прав.
- **`chown` обычно требует root.** Обычный пользователь не может «подарить» свой файл другому без `sudo`.
- **Рекурсия опасна:** `chmod -R 755` на дереве с файлами и каталогами навесит `x` и на обычные файлы. Часто нужны раздельные правила для файлов и каталогов.
- **Права на симлинк игнорируются** - действуют права целевого файла, а `chmod` по умолчанию идёт по ссылке.
