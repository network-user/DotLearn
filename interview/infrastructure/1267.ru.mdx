# Как посмотреть загрузку оперативной памяти в Linux

## Суть

Быстрый ответ - `free -h`: компактная сводка по ОЗУ и swap в человекочитаемых единицах. Когда нужны точные цифры по каждому виду памяти, читают виртуальный файл `/proc/meminfo`.

## Что ответить на собесе

Для оперативного взгляда беру `free -h` - он сразу показывает total/used/free, а главное колонку `available` - реальную оценку того, сколько памяти доступно новым процессам без свопа. Если нужны детали по конкретным счётчикам (Buffers, Cached, Slab и т.п.), смотрю `/proc/meminfo` - это псевдофайл из procfs, который ядро обновляет на лету. Ключевой момент: высокое `used` не значит нехватку памяти - кэш страниц считается «занятым», но отдаётся приложениям по запросу, поэтому ориентироваться надо на `available`, а не на `free`.

## Как это работает

`free` берёт данные из того же `/proc/meminfo` и сводит их в таблицу:

```bash
free -h
```

- `-h` - человекочитаемый формат (M, G).

```
              total        used        free      shared  buff/cache   available
Mem:           7.8G        2.1G        3.5G        1.2G        2.2G        4.3G
Swap:          2.0G          0B        2.0G
```

- **total** - весь объём.
- **used** - занято процессами (без учёта возвращаемого кэша).
- **free** - совсем не используется.
- **buff/cache** - буферы и кэш страниц, отдаётся приложениям при необходимости.
- **available** - оценка того, что реально получит новый процесс. Главная колонка.

`/proc/meminfo` даёт развёрнутую картину:

```bash
cat /proc/meminfo
```

```
MemTotal:        8074320 kB
MemFree:         3654320 kB
MemAvailable:    4532120 kB
Buffers:          123456 kB
Cached:          2345678 kB
SwapTotal:       2097148 kB
SwapFree:        2097148 kB
```

- **MemTotal / MemFree** - всего и совсем свободно.
- **MemAvailable** - доступно новым программам с учётом возвращаемого кэша.
- **Buffers / Cached** - память под буферы и кэш.
- **SwapTotal / SwapFree** - объём и остаток swap.

## Подводные камни

- **`free` показывает «занято», но это вводит в заблуждение.** Большой `buff/cache` - норма: Linux заполняет свободную RAM кэшем. Смотреть нужно на `available`, а не на `free`.
- **`MemFree` != свободная память для приложений.** Низкий `MemFree` при высоком `MemAvailable` означает, что всё в порядке - кэш отдадут по запросу.
- **Единицы.** Без `-h` у `free` числа в килобайтах (или зависят от флага), а `/proc/meminfo` всегда в kB - легко ошибиться на три порядка.
- **swap в `free` не равно нехватке RAM.** Немного занятого swap нормально; тревожный сигнал - активный своп (рост swap used + падение производительности), что видно скорее по `vmstat`/`si`/`so`.
- **`/proc/meminfo` отражает всю систему, а не один процесс.** Память конкретного процесса смотрят в `/proc/<pid>/status` или через `ps`/`top` (RSS).
