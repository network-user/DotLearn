# Hard and symbolic links: how they differ at the inode level

## In short

A symlink is a separate file that stores a path to the target, like a shortcut; it breaks if the target is moved or deleted, and it can point into another filesystem. A hardlink is a second name for the same inode: the data lives as long as at least one name exists, but the link only works within a single filesystem.

## How to answer in an interview

The difference is in what the link points to. A symbolic link points to a path — it is a tiny file with the route to the target written inside. Delete the original and the symlink dangles in the air (a "broken" link). A hard link points not to a path but directly at the inode, that is, at the very structure holding the file's data. So the original name and the hardlink are equals: deleting one of them does not touch the data until the inode's link count reaches zero. A symlink can reference directories and cross filesystem boundaries; a hardlink cannot.

A symbolic link is a pointer file with a route inside:

```bash
ln -s /path/to/original/file /path/to/symlink
```

`ln -s` creates a symlink; the first argument is the target, the second is where the link itself appears. The target path is not checked for existence, so you can create a link "to nowhere".

A hard link adds another name to the same inode:

```bash
ln /path/to/original/file /path/to/hardlink
```

After this, `ls -i` shows the same inode number for both names, and in `ls -l` output the link count grows to 2.

## Pitfalls

- A hardlink cannot be created for a directory and cannot span different filesystems — they have separate inode spaces.
- Deleting a symlink's target does not raise an error immediately: the link remains, but accessing it gives "No such file or directory".
- Space is physically freed not when a file is deleted, but when the inode's hard link count drops to zero and no one holds it open.
- `cp -L` follows the symlink and copies the target's contents, while `cp -P` copies the link itself — easy to mix up the behavior.
