# How to check RAM usage in Linux

## In short

The quick answer is `free -h`: a compact summary of RAM and swap in human-readable units. When you need exact figures for each kind of memory, you read the virtual file `/proc/meminfo`.

## How to answer in an interview

For a quick look I use `free -h` - it immediately shows total/used/free, and most importantly the `available` column, which is a realistic estimate of how much memory is available to new processes without swapping. If I need details on specific counters (Buffers, Cached, Slab, etc.), I look at `/proc/meminfo` - a procfs pseudo-file that the kernel updates on the fly. The key point: high `used` does not mean a shortage of memory - the page cache counts as "used" but is handed back to applications on demand, so you should look at `available`, not `free`.

## How it works

`free` takes its data from the same `/proc/meminfo` and condenses it into a table:

```bash
free -h
```

- `-h` - human-readable format (M, G).

```
              total        used        free      shared  buff/cache   available
Mem:           7.8G        2.1G        3.5G        1.2G        2.2G        4.3G
Swap:          2.0G          0B        2.0G
```

- **total** - the entire amount.
- **used** - taken by processes (not counting the reclaimable cache).
- **free** - completely unused.
- **buff/cache** - buffers and the page cache, handed to applications when needed.
- **available** - an estimate of what a new process would actually get. The key column.

`/proc/meminfo` gives a detailed picture:

```bash
cat /proc/meminfo
```

```
MemTotal:        8074320 kB
MemFree:         3654320 kB
MemAvailable:    4532120 kB
Buffers:          123456 kB
Cached:          2345678 kB
SwapTotal:       2097148 kB
SwapFree:        2097148 kB
```

- **MemTotal / MemFree** - total and completely free.
- **MemAvailable** - available to new programs, accounting for reclaimable cache.
- **Buffers / Cached** - memory used by buffers and cache.
- **SwapTotal / SwapFree** - the size and remaining swap.

## Pitfalls

- **`free` shows "used", but that's misleading.** A large `buff/cache` is normal: Linux fills free RAM with cache. You should look at `available`, not `free`.
- **`MemFree` != memory free for applications.** A low `MemFree` together with a high `MemAvailable` means everything is fine - the cache will be released on demand.
- **Units.** Without `-h`, `free`'s numbers are in kilobytes (or depend on the flag), and `/proc/meminfo` is always in kB - it's easy to be off by three orders of magnitude.
- **Swap in `free` does not equal a RAM shortage.** A bit of used swap is normal; the alarming signal is active swapping (rising swap used + dropping performance), which is more visible via `vmstat`/`si`/`so`.
- **`/proc/meminfo` reflects the whole system, not a single process.** A specific process's memory is checked in `/proc/<pid>/status` or via `ps`/`top` (RSS).
