# How do pre-commit hooks filter code before the commit even happens?

## In short

A pre-commit hook is a script that Git runs locally before creating a commit. If a check fails, the commit isn't created, and bad code simply never lands in history.

## How to answer in an interview

Pre-commit is a Git client-side hook that fires at the moment of `git commit`, before the commit is written. It usually runs linters, formatters, fast tests, and secret scanners. A non-zero exit code from any hook cancels the commit. In practice, hooks are rarely written by hand: you install the `pre-commit` framework with a declarative `.pre-commit-config.yaml` listing ready-made checks. An important nuance: this is local protection, it can be bypassed with the `--no-verify` flag, so the same checks are duplicated in CI as a mandatory barrier.

Hooks live in `.git/hooks/`. When you run `git commit`, Git calls the executable file `pre-commit`. A return of 0 - the commit is created, any other value - it's canceled.

```bash
pip install pre-commit
```

The configuration is described declaratively. An example for `flake8` (PEP 8 checking):

```yaml
repos:
  - repo: https://github.com/pycqa/flake8
    rev: 7.1.1
    hooks:
      - id: flake8
```

After `pre-commit install`, the framework replaces `.git/hooks/pre-commit` with its own runner:

```bash
pre-commit install
```

Now every commit automatically runs `flake8` over the changed files; on violations you get a list of problems, and the commit is blocked until they're fixed.

## Pitfalls

- The hook is bypassed with `git commit --no-verify` (`-n`), so it's a convenience, not a guarantee. The hard barrier is built in CI.
- Pre-commit lives in `.git/hooks/` and isn't cloned together with the repository. To have hooks for everyone, you need the framework + `pre-commit install` (or a shared setup command).
- Heavy checks and full test runs in pre-commit are annoying: the commit starts to lag. Long checks are moved to pre-push or CI.
- By default many hooks work only on staged files, not the whole project - this is both a speed plus and a source of misses.
