# Linux: какими командами посмотреть, что сейчас выполняется в системе?

## Суть

Для статического снимка процессов берут `ps` (например `ps aux`), для живого, постоянно обновляемого монитора - `top`, а его удобную цветную версию - `htop`. Снимок отвечает на вопрос «что было в этот момент», монитор - «что происходит прямо сейчас».

## Что ответить на собесе

Если нужен разовый список всех процессов с владельцами и потреблением ресурсов - `ps aux`, часто в связке с `grep` для поиска конкретного процесса. Если нужно наблюдать за нагрузкой в динамике (кто ест CPU и память сейчас) - `top` или `htop`. Разница принципиальная: `ps` печатает один кадр и завершается, а `top`/`htop` - это интерактивные мониторы, которые обновляют картину раз в пару секунд и позволяют сразу отсортировать процессы или завершить нужный.

## Как это работает

Команда `ps` (process status) выводит снимок процессов на момент запуска:

```bash
ps aux
```

- `a` - процессы всех пользователей, а не только текущего;
- `u` - читаемый формат с владельцем, долей CPU и памяти;
- `x` - в том числе процессы без управляющего терминала (демоны).

В выводе - столбцы USER, PID, %CPU, %MEM, время старта и сама команда. Чтобы найти конкретный процесс, комбинируют с `grep`:

```bash
ps aux | grep nginx
```

Команда `top` показывает то же, но в реальном времени: сверху сводка по системе (load average, память, число процессов), снизу - список, который обновляется. Внутри: `q` - выход, `k` - послать сигнал процессу по PID, `M`/`P` - сортировка по памяти/CPU.

`htop` - это улучшенный `top`: цветной интерфейс, навигация стрелками и мышью, удобная фильтрация. Его обычно ставят отдельно:

```bash
sudo apt-get install htop   # Debian/Ubuntu
sudo dnf install htop       # Fedora/RHEL
```

## Подводные камни

- **`ps aux` - это снимок, а не монитор.** По одному вызову нельзя судить о пике нагрузки: процесс мог нагрузить CPU за миллисекунду до или после.
- **`%CPU` в `ps` - усреднённое значение** за время жизни процесса, а не мгновенное; для текущей нагрузки смотрите `top`.
- **`htop` есть не везде.** На голом сервере или в минимальном контейнере его придётся ставить, а прав может не быть - тогда остаётся `top`.
- **Поиск через grep ловит сам grep.** `ps aux | grep python` нередко показывает строку с самим процессом grep; это сбивает с толку при подсчёте.
