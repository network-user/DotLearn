# Жизненный цикл процесса в Linux: PID, состояния и планировщик

## Суть

Процессом в Linux заведует ядро: планировщик нарезает процессорное время между задачами, а каждый процесс живёт под уникальным PID и переходит между состояниями (выполняется, ждёт, остановлен, зомби). Создаётся процесс вызовом `fork()`, меняет своё тело через `exec()`, а родитель забирает результат через `wait()`.

## Что ответить на собесе

Управление процессами - базовая функция ядра. Процесс - это запущенный экземпляр программы со своим адресным пространством, кодом, данными и идентификатором PID. Ядро через планировщик распределяет CPU между процессами по выбранному алгоритму, а сами процессы переходят между состояниями: работает на ядре, ждёт ввода-вывода, приостановлен или стал зомби после завершения. Создание идёт через `fork()`, подмена образа - через `exec()`, синхронизация с потомком - через `wait()`. На уровне команд за наблюдением и управлением стоят `ps`, `top` и `kill`.

## Как это работает

Процесс - это исполняемая программа в памяти со своим состоянием. Ключевые элементы:

- **PID** - уникальный идентификатор, по которому процесс адресуют команды и сигналы.
- **Планировщик** - часть ядра, раздающая процессорное время. Алгоритмы (Round Robin, FIFO, CFS и другие) определяют, кто получит ядро следующим.
- **Состояния процесса:**
  - **Running** - процесс прямо сейчас исполняется на CPU или готов к этому.
  - **Waiting** - ждёт события, например завершения ввода-вывода.
  - **Stopped** - приостановлен сигналом и не выполняется.
  - **Zombie** - завершился, но запись в таблице процессов ещё не убрана родителем.

**Создание через fork().** Новый процесс - это копия текущего. Вызов возвращает 0 потомку и PID потомка родителю, а -1 сигналит об ошибке.

```c
pid_t pid = fork();
if (pid == 0) {
    // ветка дочернего процесса
} else if (pid > 0) {
    // ветка родительского процесса, pid = PID потомка
} else {
    // fork() не удался
}
```

**Подмена образа через exec().** Семейство `exec*()` заменяет тело текущего процесса новой программой, сохраняя PID. После удачного вызова код ниже уже не исполнится.

```c
execl("/bin/ls", "ls", NULL);
```

**Завершение и синхронизация через wait().** Родитель блокируется до завершения потомка и читает его статус выхода; без этого потомок зависает зомби.

```c
int status;
wait(&status);
```

Команды поверх этого механизма: `ps` - снимок процессов, `top` - живая динамика, `kill` - отправка сигнала (по умолчанию `SIGTERM`).

## Подводные камни

- **Зомби-процессы.** Если родитель не вызвал `wait()`, потомок остаётся в таблице как зомби. Лечится сбором статуса родителем; при гибели родителя сирота переходит к `init`/`systemd` (PID 1), который его пожинает.
- **`kill` не всегда убивает.** Это отправка сигнала, а не гарантированное завершение. `SIGTERM` (15) можно перехватить и проигнорировать, `SIGKILL` (9) - нельзя.
- **`fork()` копирует, но лениво.** Адресное пространство разделяется по copy-on-write, а не дублируется сразу, иначе порождение процессов было бы неподъёмным по памяти.
- **Приоритет через nice.** Низкий `nice` (вплоть до -20) повышает приоритет, высокий (до 19) понижает; уменьшать nice ниже нуля может только root.
