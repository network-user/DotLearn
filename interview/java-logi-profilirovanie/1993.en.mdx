# What makes up a structured log event?

## In short

A log event is a single record of something that happened at runtime: when it happened, how important it is, who emitted it, what occurred, and which context helps correlate it. A structured format (JSON, key-value) lets you filter in ELK, Loki, or Splunk and build alerts without regex on free text.

## How to answer in an interview

- I list fields: timestamp, level, logger/source, message, MDC/trace metadata.
- I mention correlation id, request id, user id, and span id for distributed tracing.
- I explain SLF4J levels: TRACE/DEBUG/INFO/WARN/ERROR.
- I show that the message should be parameterized while details live in structured fields.
- I add honestly: PII and secrets in metadata are unacceptable; mask them at the appender/layout level.

## How it works

A typical JSON event from a Java service (SLF4J + structured encoder):

```json
{
  "timestamp": "2026-07-13T11:22:01.453Z",
  "level": "ERROR",
  "logger": "com.acme.billing.PaymentGateway",
  "message": "payment capture failed",
  "traceId": "7f3c2a1b9e004d12",
  "spanId": "a91f0042bc11",
  "orderId": "ord-88421",
  "errorCode": "CARD_DECLINED",
  "durationMs": 842
}
```

`timestamp` orders events and aligns with metrics. `level` controls noise in production (often INFO+). `logger` points to the source class or module. `message` is a short human-readable fact; `orderId`, `errorCode`, and `durationMs` give machine-readable context for search and dashboards. `traceId`/`spanId` tie the log to a microservice chain. In Java, MDC (`Mapped Diagnostic Context`) carries such keys through a thread-bound map for the request lifetime.

## Pitfalls

- **Unstructured text.** `log.info("user " + id + " paid " + sum)` breaks parsing.
- **Missing timezone.** Local time without offset distorts order across regions.
- **Wrong levels.** ERROR on an expected business failure floods on-call.
- **Huge stack traces on every INFO.** Bloats the index and storage cost.
- **Secrets in metadata.** Tokens, PAN, or passwords in JSON fields.
