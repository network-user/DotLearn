# WHERE или HAVING: где в конвейере SQL отсекаются строки и группы

## Суть

`WHERE` работает раньше: он отфильтровывает отдельные строки до того, как они попадут в группировку и агрегаты. `HAVING` включается позже - после `GROUP BY` - и отбирает уже готовые группы по значениям агрегатных функций.

## Что ответить на собесе

«Логический порядок выполнения такой: `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`. Из него всё и следует. `WHERE` фильтрует сырые строки до агрегации, поэтому в нём нельзя использовать `COUNT()` или `SUM()` - их ещё не существует. `HAVING` срабатывает после группировки и как раз предназначен для условий на агрегаты, например "только отделы, где сотрудников больше десяти". Практическое правило: всё, что можно, отсеивайте в `WHERE` - так до группировки доходит меньше данных и запрос быстрее; `HAVING` оставляйте строго для условий на агрегаты».

`WHERE` отбрасывает строки до группировки. В агрегат попадут только сотрудники с зарплатой выше 50000.

```sql
SELECT department, COUNT(*)
FROM employees
WHERE salary > 50000
GROUP BY department;
```

`HAVING` фильтрует результат уже после `GROUP BY`, опираясь на агрегатные значения.

```sql
SELECT department, COUNT(*) AS cnt
FROM employees
GROUP BY department
HAVING COUNT(*) > 10;
```

Их можно сочетать: сначала `WHERE` сужает набор строк, затем группировка, затем `HAVING` отсекает группы.

```sql
SELECT department, AVG(salary) AS avg_salary
FROM employees
WHERE active = true
GROUP BY department
HAVING AVG(salary) > 70000;
```

## Подводные камни

- В `WHERE` нельзя ссылаться на агрегатные функции - на этой стадии групп ещё нет, и попытка написать `WHERE COUNT(*) > 10` даст ошибку.
- Условие на неагрегированный столбец, ошибочно перенесённое в `HAVING`, работает, но впустую: оно отфильтрует строки уже после лишней группировки. Такие условия выгоднее держать в `WHERE`.
- Алиасы из `SELECT` в стандартном SQL не видны в `WHERE` и формально в `HAVING`, потому что `SELECT` вычисляется позже; PostgreSQL допускает алиас только в `GROUP BY` и `ORDER BY`.
- `HAVING` без `GROUP BY` трактует всю выборку как одну группу - это валидно, но легко принять за ошибку, если этого не ожидать.
