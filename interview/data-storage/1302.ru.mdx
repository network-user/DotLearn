# Объединяем данные двух таблиц без оператора JOIN: какие есть пути?

## Суть

Связать строки двух таблиц можно и без `JOIN`: либо через подзапрос, который фильтрует одну таблицу по значениям из другой, либо собрав результат уже в коде приложения. Оба способа решают ту же задачу, просто переносят логику соединения в другое место.

## Что ответить на собесе

Связать строки двух таблиц можно двумя путями, и `JOIN` для этого не единственный инструмент:

- **Подзапрос**: внутренний `SELECT` возвращает набор ключей из одной таблицы, внешний запрос отбирает по ним строки другой через `IN` или `EXISTS`. Логика остаётся в базе.
- **Сборка в коде**: вытащить обе выборки по отдельности и склеить на стороне приложения, например через словарь по ключу. Удобно, когда нужна нестандартная обработка, которую тяжело выразить на SQL.
- Минус ручной сборки: две поездки в базу и риск отдать соединение из БД на откуп приложению.

## Как это работает

### Подзапрос

Подзапрос отбирает ключи из одной таблицы, а внешний запрос использует их как фильтр для другой. Возьмём `orders` и `customers` и достанем заказы клиентов из конкретного города.

```sql
SELECT order_id, order_date
FROM orders
WHERE customer_id IN (
    SELECT customer_id
    FROM customers
    WHERE city = 'New York'
);
```

Внутренний `SELECT` возвращает идентификаторы клиентов из Нью-Йорка, а внешний оставляет только те заказы, чей `customer_id` попал в этот список.

### Сборка в приложении

Если логику соединения проще описать в коде, выбираем обе таблицы отдельно и связываем строки через словарь с доступом по ключу за O(1).

```python
import sqlite3

conn = sqlite3.connect('example.db')
cursor = conn.cursor()

cursor.execute("SELECT order_id, customer_id, order_date FROM orders")
orders = cursor.fetchall()

cursor.execute("SELECT customer_id, name, city FROM customers")
customers = cursor.fetchall()

customer_dict = {cid: (name, city) for cid, name, city in customers}

combined_data = []
for order_id, customer_id, order_date in orders:
    if customer_id in customer_dict:
        name, city = customer_dict[customer_id]
        combined_data.append((order_id, customer_id, order_date, name, city))

for row in combined_data:
    print(row)

conn.close()
```

Словарь `customer_dict` превращает поиск клиента по `customer_id` в операцию за константное время, поэтому общая сложность сборки линейна по числу заказов.

## Подводные камни

- Подзапрос с `IN` по большому списку ключей может оказаться медленнее обычного `JOIN`: планировщик не всегда строит для него эффективный план, иногда `EXISTS` или сам `JOIN` лучше.
- При сборке в приложении легко получить квадратичную сложность, если искать клиента линейным перебором списка вместо словаря.
- Ручное соединение тянет в память сразу обе таблицы и делает несколько запросов - на больших объёмах это бьёт и по сети, и по памяти.
- Коррелированный подзапрос (ссылающийся на внешнюю строку) выполняется для каждой строки и легко становится узким местом.
