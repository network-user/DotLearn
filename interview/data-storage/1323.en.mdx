# How do the JOIN types differ and what result does each produce?

## In short

A JOIN glues together rows from several tables by a match condition. INNER keeps only the matched pairs; LEFT and RIGHT pull all rows from one side, filling in missing values with NULL; FULL OUTER takes everything from both tables; CROSS produces a Cartesian product with no condition at all.

## How to answer in an interview

A JOIN is needed to assemble into one result data that is spread across different tables. INNER JOIN returns only rows that have a match in both tables - it's the workhorse. LEFT JOIN keeps all rows of the left table and substitutes NULL wherever there is no match on the right; RIGHT JOIN does the same mirrored for the right table. FULL OUTER JOIN combines the results of LEFT and RIGHT: all rows of both tables remain, and unmatched slots are filled with NULL. CROSS JOIN uses no join condition at all and pairs every row with every other - it's used rarely because of the explosive growth in the number of rows.

## How it works

Logically related entities usually live in different tables (employees and departments), and a JOIN brings them together by key.

INNER JOIN - only the matched pairs:

```sql
SELECT employees.name, departments.name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;
```

LEFT JOIN (LEFT OUTER JOIN) - all employees, even those without a department (then the department fields will be NULL):

```sql
SELECT employees.name, departments.name
FROM employees
LEFT JOIN departments ON employees.department_id = departments.id;
```

RIGHT JOIN (RIGHT OUTER JOIN) - the mirror of LEFT: all departments, including empty ones:

```sql
SELECT employees.name, departments.name
FROM employees
RIGHT JOIN departments ON employees.department_id = departments.id;
```

FULL OUTER JOIN - all rows of both tables, non-matches filled with NULL:

```sql
SELECT employees.name, departments.name
FROM employees
FULL OUTER JOIN departments ON employees.department_id = departments.id;
```

CROSS JOIN - a Cartesian product, no join condition:

```sql
SELECT employees.name, departments.name
FROM employees
CROSS JOIN departments;
```

## Pitfalls

- A LEFT JOIN with a filter on the right table in `WHERE` (instead of `ON`) turns into an INNER JOIN: the condition `WHERE dept.id = 5` discards rows with NULL.
- If the join key is duplicated, the JOIN multiplies rows - the result count becomes larger than either source table.
- A RIGHT JOIN is almost always rewritten as a LEFT JOIN for readability: just swap the tables.
- Not all DBMSs support FULL OUTER JOIN (MySQL until recent versions, for example) - there it is emulated via a UNION of two LEFT/RIGHT JOINs.
- A CROSS JOIN without `WHERE` on large tables easily yields millions of rows - it's a common source of hung queries.
