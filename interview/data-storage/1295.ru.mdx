# Как посмотреть, сколько клиентов сейчас подключено к СУБД

## Суть

Текущее число подключений читают из системных представлений или статусных счётчиков самой СУБД. В PostgreSQL это строки в `pg_stat_activity`, в MySQL — переменная состояния `Threads_connected`.

## Что ответить на собесе

Метрики о соединениях смотрю через служебные представления СУБД, а не угадываю.

- **PostgreSQL**: `SELECT count(*) FROM pg_stat_activity;` — представление содержит по строке на каждый активный backend.
- **MySQL**: `SHOW STATUS LIKE 'Threads_connected';`.
- Это нужно, чтобы не упереться в `max_connections`: при исчерпании лимита новые клиенты получают отказ.
- Мониторинг подключений — ранний сигнал о пуле, утечке соединений или внезапном всплеске нагрузки.

## Как это работает

В PostgreSQL живые соединения отражает представление `pg_stat_activity` — по одной строке на каждый серверный процесс. Простой подсчёт строк даёт число подключений:

```sql
SELECT count(*) FROM pg_stat_activity;
```

Часто полезнее разрезать по состоянию или по базе, чтобы отделить реально работающие сессии от простаивающих:

```sql
SELECT state, count(*)
FROM pg_stat_activity
GROUP BY state;
```

В MySQL число открытых соединений лежит в статусной переменной `Threads_connected`, которую отдаёт `SHOW STATUS`:

```sql
SHOW STATUS LIKE 'Threads_connected';
```

`SHOW STATUS` возвращает множество счётчиков сервера, а фильтр `LIKE` оставляет только нужный. Рядом стоит держать в голове `max_connections` — потолок, выше которого новые подключения отвергаются.

## Подводные камни

- **`pg_stat_activity` включает и саму сессию мониторинга.** Запрос видит в том числе собственное соединение, так что результат на единицу больше «чужих» сессий.
- **Не путайте «подключено» и «активно».** Большинство строк в `pg_stat_activity` могут быть в состоянии `idle` — это открытый, но простаивающий пул. Для диагностики нагрузки фильтруйте по `state = 'active'`.
- **`Threads_connected` против `Threads_running`.** Первое — все открытые соединения, второе — реально выполняющие работу прямо сейчас. На собесе путаница между ними — частый промах.
- **Нужны права.** Без достаточных привилегий вы увидите только свои сессии, а не полную картину по серверу.
- **Утечка соединений маскируется под рост нагрузки.** Постоянно растущее число idle-сессий обычно означает не нагрузку, а не закрывающийся пул в приложении.
