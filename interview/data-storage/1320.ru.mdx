# Курсор базы данных: построчный доступ к результату запроса

## Суть

Курсор - это объект-указатель на результирующую выборку запроса, через который строки читаются по одной. Он позволяет обходить результат, обрабатывать каждую строку отдельно и не держать всю выборку в памяти разом.

## Что ответить на собесе

Курсор нужен там, где результат запроса нельзя или не хочется загружать целиком.

- Запрос может вернуть миллионы строк: курсор отдаёт их порциями по мере итерации, память расходуется ровно на текущий блок.
- Жизненный цикл простой: создаём курсор, выполняем запрос, читаем строки (`fetchone`/`fetchmany`/итерация), закрываем.
- В драйверах Python (DB-API) `cursor` служит ещё и точкой входа для выполнения SQL: `cursor.execute(...)`, затем чтение.
- Важно различать курсоры: серверный (например, named cursor в psycopg2) реально не тянет весь результат на клиент, а обычный буферизованный, наоборот, может.

## Как это работает

Этапы работы с курсором: создание, выполнение запроса, чтение, закрытие. Освобождение ресурсов на закрытии обязательно.

```python
import sqlite3

connection = sqlite3.connect("example.db")
cursor = connection.cursor()

cursor.execute("SELECT id, name FROM users")

for row in cursor:
    print(f"User ID: {row[0]}, Name: {row[1]}")

cursor.close()
connection.close()
```

Здесь `connection.cursor()` создаёт курсор, `cursor.execute(...)` выполняет запрос, итерация `for row in cursor` забирает строки по одной (каждая из них - кортеж), а `close()` освобождает ресурсы.

Чтобы явно контролировать чтение, используют методы выборки:

```python
cursor.execute("SELECT id, name FROM users")
first = cursor.fetchone()
batch = cursor.fetchmany(100)
rest = cursor.fetchall()
```

`fetchone()` возвращает одну строку или `None`, `fetchmany(n)` возвращает список до n строк, а `fetchall()` возвращает все оставшиеся. На больших выборках безопаснее `fetchmany` в цикле, чем `fetchall`. Гарантировать закрытие удобнее всего через контекстный менеджер: `with connection.cursor() as cursor:`.

## Подводные камни

- `fetchall()` на огромной таблице сводит на нет всю экономию памяти: он материализует весь результат. Для потоковой обработки используйте итерацию или `fetchmany`.
- Незакрытый курсор удерживает ресурсы и блокировки. Всегда закрывайте его (лучше через `with`), особенно в долгоживущих соединениях.
- В sqlite3 курсор не изолирует данные: параллельная запись в то же соединение во время итерации может привести к неожиданному результату.
- Обычный курсор многих драйверов буферизует весь ответ на клиенте. Для реальной потоковой выгрузки нужен серверный/named-курсор.
- Курсор связан со своим соединением: если закрыть соединение, курсор становится невалиден.
