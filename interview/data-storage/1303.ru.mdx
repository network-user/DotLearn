# Зачем в SQLAlchemy вызывают scalars() и что он меняет в результате?

## Суть

`scalars()` разворачивает результат запроса из строк-кортежей в плоский список одиночных значений. Когда каждая строка содержит ровно один столбец, метод избавляет от ручного доставания `row[0]`.

## Что ответить на собесе

`scalars()` разворачивает строки-кортежи в плоский список одиночных значений.

- По умолчанию `session.execute(stmt)` отдаёт `Result`, где каждая строка представляет собой кортеж (`Row`), даже если выбран один столбец.
- `result.scalars()` берёт первый элемент каждой строки: вместо `[('Alice',), ('Bob',)]` получаю `['Alice', 'Bob']`. Дальше идут `.all()`, `.first()`, `.one()` или итерация.
- Особенно важно с ORM: `select(User)` без `scalars()` даёт кортежи с объектами внутри, а со `scalars()` получаем сразу список экземпляров `User`.
- Есть шорткат `session.scalars(stmt)`.

## Как это работает

```python
from sqlalchemy import create_engine, select, MetaData, Table, Column, Integer, String
from sqlalchemy.orm import Session

engine = create_engine('sqlite:///:memory:')
metadata = MetaData()

users = Table('users', metadata,
    Column('id', Integer, primary_key=True),
    Column('name', String)
)

metadata.create_all(engine)

with engine.connect() as connection:
    connection.execute(users.insert(), [
        {'name': 'Alice'},
        {'name': 'Bob'},
        {'name': 'Charlie'}
    ])
    connection.commit()

with Session(engine) as session:
    stmt = select(users.c.name)
    result = session.execute(stmt)
    names = result.scalars().all()
    print(names)  # ['Alice', 'Bob', 'Charlie']
```

Шаги по порядку:

1. `create_engine('sqlite:///:memory:')` поднимает временную базу в памяти.
2. `Table('users', ...)` описывает структуру таблицы со столбцами `id` и `name`.
3. `connection.execute(users.insert(), [...])` вставляет строки.
4. `select(users.c.name)` формирует запрос на выборку одного столбца.
5. `session.execute(stmt)` возвращает `Result` из строк-кортежей.
6. `result.scalars().all()` берёт первый столбец каждой строки и собирает плоский список.

## Подводные камни

- `scalars()` берёт только **первый** столбец. Если в `select()` указано несколько колонок, остальные молча теряются - тогда нужен обычный `result.all()` с кортежами.
- Не путайте методы: `.scalars()` (множество значений) против `.scalar()` (одно значение из первой строки или `None`) и `.scalar_one()` (ровно одно, иначе исключение).
- Для одного объекта по первичному ключу часто проще `session.get(User, pk)`, а не запрос со `scalars().first()`.
- На старом legacy-API (`Query`) метода `scalars()` нет - он часть стиля 2.0 с `select()`.
