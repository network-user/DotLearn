# Что такое транзакция и зачем группировать операции

## Суть

Транзакцией называют набор операций, который СУБД выполняет как одно неделимое действие: либо успешны все, либо не применяется ни одна. Так база остаётся целостной даже при ошибках и параллельном доступе.

## Что ответить на собесе

Транзакция объединяет несколько изменений в единицу работы с гарантиями ACID.

- Классическим примером служит перевод денег: списание с одного счёта и зачисление на другой обязаны произойти вместе.
- Если после списания что-то падает, `ROLLBACK` отменяет всё, и согласованность не нарушается.
- Подтверждение даёт `COMMIT`, отмену - `ROLLBACK`, а границы задаёт начало транзакции.

## Как это работает

В коде транзакцию реализует блок между началом и `commit`/`rollback`. На примере SQLite и Python:

```python
import sqlite3

connection = sqlite3.connect('bank.db')
cursor = connection.cursor()

try:
    cursor.execute('UPDATE accounts SET balance = balance - 100 WHERE account_id = 1')
    cursor.execute('UPDATE accounts SET balance = balance + 100 WHERE account_id = 2')
    connection.commit()
except Exception as error:
    connection.rollback()
    print(f"Transaction failed: {error}")
finally:
    connection.close()
```

Оба `UPDATE` идут внутри одной транзакции. `commit()` фиксирует их разом; при любом исключении `rollback()` возвращает балансы в исходное состояние. Драйвер `sqlite3` сам открывает транзакцию перед первым изменяющим запросом, поэтому явный `BEGIN` не нужен: достаточно вовремя вызвать `commit` или `rollback`.

## Подводные камни

- Забытый `commit()` означает потерю изменений: при закрытии соединения незафиксированная транзакция откатывается.
- Длинная транзакция держит блокировки и мешает другим. Границы стоит делать как можно уже.
- В `sqlite3` поведение автокоммита зависит от параметра `isolation_level`; со значением `None` каждый запрос фиксируется сразу, и привычная транзакционная семантика пропадает.
- `rollback` откатывает изменения в базе, но не откатывает побочные эффекты вашего кода (отправленные письма, изменённые переменные); за это отвечает приложение.
