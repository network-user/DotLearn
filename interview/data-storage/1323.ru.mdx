# Чем различаются типы JOIN и какой результат даёт каждый?

## Суть

JOIN склеивает строки из нескольких таблиц по условию совпадения. INNER оставляет только пары с совпадением; LEFT и RIGHT тянут все строки одной стороны, добивая недостающее значениями NULL; FULL OUTER берёт всё из обеих таблиц; CROSS даёт декартово произведение без всякого условия.

## Что ответить на собесе

JOIN нужен, чтобы собрать в один результат данные, разнесённые по разным таблицам.

- **INNER JOIN** — только строки с совпадением в обеих таблицах, рабочая лошадка.
- **LEFT JOIN** — все строки левой таблицы, NULL там, где справа пары нет; **RIGHT JOIN** — то же зеркально для правой.
- **FULL OUTER JOIN** объединяет LEFT и RIGHT: остаются все строки обеих таблиц, несовпавшие места заполняются NULL.
- **CROSS JOIN** не использует условие соединения и сцепляет каждую строку с каждой — применяют редко из-за взрывного роста числа строк.

## Как это работает

Логически связанные сущности обычно живут в разных таблицах (сотрудники и отделы), и JOIN сводит их по ключу.

INNER JOIN - только совпавшие пары:

```sql
SELECT employees.name, departments.name
FROM employees
INNER JOIN departments ON employees.department_id = departments.id;
```

LEFT JOIN (LEFT OUTER JOIN) - все сотрудники, даже без отдела (тогда поля отдела будут NULL):

```sql
SELECT employees.name, departments.name
FROM employees
LEFT JOIN departments ON employees.department_id = departments.id;
```

RIGHT JOIN (RIGHT OUTER JOIN) - зеркало LEFT: все отделы, в том числе пустые:

```sql
SELECT employees.name, departments.name
FROM employees
RIGHT JOIN departments ON employees.department_id = departments.id;
```

FULL OUTER JOIN - все строки обеих таблиц, несовпадения заполняются NULL:

```sql
SELECT employees.name, departments.name
FROM employees
FULL OUTER JOIN departments ON employees.department_id = departments.id;
```

CROSS JOIN - декартово произведение, условие соединения отсутствует:

```sql
SELECT employees.name, departments.name
FROM employees
CROSS JOIN departments;
```

## Подводные камни

- LEFT JOIN с фильтром по правой таблице в `WHERE` (вместо `ON`) превращается в INNER JOIN: условие `WHERE dept.id = 5` отбросит строки с NULL.
- Если ключ соединения дублируется, JOIN размножит строки - количество результата станет больше любой из исходных таблиц.
- RIGHT JOIN почти всегда переписывают как LEFT JOIN ради читаемости: достаточно поменять таблицы местами.
- Не все СУБД поддерживают FULL OUTER JOIN (например, MySQL до недавних версий) - там его эмулируют через UNION двух LEFT/RIGHT JOIN.
- CROSS JOIN без `WHERE` на больших таблицах легко даёт миллионы строк - это частый источник зависших запросов.
