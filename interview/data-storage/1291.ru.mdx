# Управление схемой через Alembic: revision, downgrade, heads

## Суть

Alembic версионирует схему базы данных: `revision` заводит новую миграцию, `downgrade` откатывает базу на шаг назад, `heads` показывает текущие вершины дерева ревизий. Эта тройка покрывает создание, откат и диагностику изменений.

## Что ответить на собесе

Alembic - штатный инструмент миграций для SQLAlchemy. Каждая миграция хранится отдельным файлом с функциями `upgrade()` и `downgrade()` и ссылкой на родительскую ревизию, образуя направленную цепочку.

- `revision` создаёт такой файл миграции.
- `downgrade` применяет обратные изменения.
- `heads` выводит ревизии без потомков.
- Если веток несколько и появляется больше одной головы, их сводят командой `merge`.

`revision` генерирует новый файл миграции в каталоге `versions`:

```bash
alembic revision -m "add new table"
```

Флаг `-m` задаёт человекочитаемое имя. С `--autogenerate` Alembic сам сравнит модели SQLAlchemy с текущей схемой и предзаполнит изменения.

`downgrade` выполняет функцию `downgrade()` нужных миграций:

```bash
alembic downgrade -1
```

Аргумент `-1` означает откат на одну ревизию относительно текущей; можно указать и конкретный идентификатор ревизии или `base`, чтобы откатить всё.

`heads` показывает вершины, то есть ревизии, на которые никто не ссылается как на родителя:

```bash
alembic heads
```

В норме голова одна. Несколько голов означают разошедшиеся ветки после параллельной разработки.

## Подводные камни

- `revision` (без `--autogenerate`) создаёт пустую миграцию, и тело `upgrade()`/`downgrade()` придётся писать руками. Это частая причина «миграции, которые ничего не делают».
- Автогенерация видит не всё: смену типа колонки, серверные значения по умолчанию и переименования она часто пропускает или трактует как drop + add. Сгенерированное надо вычитывать.
- Несколько `heads` ломают `upgrade head` неоднозначностью; ветки нужно слить через `alembic merge`.
- `downgrade` хорош на dev-среде, но в продакшене обратная миграция нередко теряет данные (например, при удалении колонки). Откат не является бесплатной отменой.
