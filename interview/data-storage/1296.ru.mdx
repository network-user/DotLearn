# Подсчёт строк в таблице: точный COUNT и быстрая оценка

## Суть

Точное число строк даёт `SELECT COUNT(*) FROM table_name;`, но на больших таблицах он сканирует данные и тормозит. Когда хватает приблизительной цифры, её берут из статистики оптимизатора без полного перебора.

## Что ответить на собесе

Выбор зависит от того, нужна ли точная цифра.

- **Точно**: `SELECT COUNT(*) FROM table;`. На миллионах строк он проходит по всем строкам (или по индексу), а в PostgreSQL ещё и проверяет видимость каждой строки из-за MVCC, поэтому недёшев.
- **Приблизительно** (дашборд, прикидка объёма) - беру оценку из системной статистики: `reltuples` из `pg_class` в PostgreSQL или строку из `information_schema.tables` в MySQL.
- Оценка почти мгновенна, но число приблизительное и зависит от свежести статистики.

## Как это работает

Базовый, точный способ использует агрегат `COUNT`:

```sql
SELECT COUNT(*) FROM employees;
```

`COUNT(*)` считает все строки результата. Запрос точен, но его стоимость растёт с размером таблицы: движок обходит строки, а в PostgreSQL дополнительно отсеивает версии строк, невидимые текущей транзакции.

Когда достаточно оценки, её читают из статистики, которую СУБД и так собирает для планировщика. В PostgreSQL приблизительное число строк хранит поле `reltuples` системного каталога `pg_class`:

```sql
SELECT reltuples::bigint AS approx_rows
FROM pg_class
WHERE relname = 'employees';
```

В MySQL похожую оценку отдаёт `information_schema`:

```sql
SELECT table_rows
FROM information_schema.tables
WHERE table_name = 'employees';
```

Оба варианта возвращают результат мгновенно, потому что не трогают сами данные, а читают накопленную статистику.

## Подводные камни

- **`COUNT(*)` и `COUNT(col)` не являются одним и тем же.** Первый считает все строки, второй пропускает строки, где `col IS NULL`. Подмена одного другим: классическая ошибка.
- **Оценка устаревает.** `reltuples` и `table_rows` обновляются при `ANALYZE`/`VACUUM` (или автоматически), поэтому после массовой вставки или удаления цифра может заметно врать.
- **`reltuples` бывает дробным и даже -1.** В свежесозданной или ни разу не проанализированной таблице PostgreSQL значение может быть условным; не используйте его как точное.
- **Индекс не делает `COUNT(*)` бесплатным.** Он может ускорить обход за счёт index-only scan, но это всё равно проход по всем записям, а не готовый счётчик. Материализованного «числа строк» в реляционной таблице нет.
- **`WHERE` убивает быструю оценку.** Трюк со статистикой работает для всей таблицы; как только нужен `COUNT` с условием, приходится возвращаться к реальному сканированию или оценкам из плана `EXPLAIN`.
