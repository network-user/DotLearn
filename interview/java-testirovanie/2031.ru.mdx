# Что такое mutation testing?

## Суть

Mutation testing меняет bytecode production-кода (mutants: `+` на `-`, `>` на `>=`) и прогоняет тесты. Хороший test suite «убивает» mutant (test fails). Выживший mutant значит assertion слабый при formal line coverage green.

## Что ответить на собесе

- Метрика mutation score = killed / total mutants.
- Инструменты: PIT (pitest-maven-plugin) для Java.
- Дорого по CPU; гоняю nightly на critical modules, не каждый commit.
- Surviving mutant смотрю в HTML report, добавляю assert на boundary.
- Честно: 100% mutation score impractical; target 70-85% on payment core.

## Как это работает

Original:

```java
int discount(int total) {
    if (total > 100) return total - 10;
    return total;
}
```

Mutant A: `>` → `>=` ; Mutant B: `- 10` → `+ 10`.

Test only `discount(150)==140` kills B maybe, but Mutant A survives if never assert `discount(100)==100`. PIT config:

```xml
<plugin>
  <groupId>org.pitest</groupId>
  <artifactId>pitest-maven</artifactId>
  <version>1.15.8</version>
  <configuration>
    <targetClasses>
      <param>com.acme.billing.*</param>
    </targetClasses>
    <mutationThreshold>75</mutationThreshold>
  </configuration>
</plugin>
```

`mvn org.pitest:pitest-maven:mutationCoverage` генерирует report: green killed, red survived with line highlight.

## Подводные камни

- **Equivalent mutants.** `(x * 2)` vs `(x << 1)` both survive legitimately, score noise.
- **Full repo PIT on CI.** Hours runtime, devs disable plugin.
- **Tests with no assertions.** Mutants die by luck, false confidence.
- **Mock-heavy code.** Mutations in unreachable branches inflate score.
- **Chasing score over readability.** Absurd asserts only for PIT.
