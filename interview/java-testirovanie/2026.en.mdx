# Unit vs integration vs E2E: what's the difference?

## In short

Unit tests check one module in isolation (milliseconds, thousands of tests). Integration tests check module boundaries with real infrastructure (seconds). E2E runs a user scenario through UI/API end to end (minutes, few tests, critical smoke paths).

## How to answer in an interview

- Pyramid: many unit, fewer integration, even fewer E2E.
- Unit: pure Java + mocks/fakes; CI feedback under one minute per module.
- Integration: Testcontainers, Spring slice, WireMock.
- E2E: Selenium/Playwright REST journey checkout-to-email.
- Honestly: E2E is flaky and expensive; I do not duplicate unit coverage at UI level.

## How it works

Unit:

```java
@Test
void discountAppliesOnlyAboveThreshold() {
    PricingEngine engine = new PricingEngine(new BigDecimal("0.10"));
    Money total = engine.checkout(Money.of("120", "USD"));
    assertEquals(new BigDecimal("108.00"), total.amount());
}
```

Integration:

```java
@SpringBootTest(webEnvironment = RANDOM_PORT)
class CheckoutApiIT {
    @Autowired TestRestTemplate client;
    @Test void postsOrderAndReturns201() {
        ResponseEntity<OrderView> res = client.postForEntity("/orders", sampleRequest(), OrderView.class);
        assertEquals(HttpStatus.CREATED, res.getStatusCode());
    }
}
```

E2E (conceptual REST flow):

```java
@Test
void customerCanPayAndSeeReceipt() {
    String orderId = api.createOrder(token, lineItem);
    api.pay(token, orderId, validCard());
    Receipt receipt = api.fetchReceipt(token, orderId);
    assertEquals("PAID", receipt.status());
}
```

Different failure signals: unit pinpoints class, integration config/SQL, E2E broken deployment or auth flow.

## Pitfalls

- **Ice-cream cone.** Mostly manual E2E, no unit, slow release.
- **"Integration" with everything mocked.** False confidence.
- **E2E on shared staging.** Data collisions, order-dependent tests.
- **Same scenario on all three layers.** Maintenance tax.
- **No contract between services.** Integration green, prod provider breaks consumer.
