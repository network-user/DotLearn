# How do you measure code coverage with JaCoCo?

## In short

JaCoCo instruments Java bytecode and counts which instructions, branches, and lines ran during tests. Reports reveal test gaps; the metric complements but does not replace assertion quality.

## How to answer in an interview

- On-the-fly agent via `-javaagent` or Maven/Gradle `prepare-agent` plugin.
- Metrics: line, branch, method, class; HTML report at `target/site/jacoco`.
- CI threshold via `check` goal: fail build below 80% on critical modules.
- Exclude DTO/generated code or the percentage is meaningless.
- Honestly: 100% line coverage is possible with zero asserts; I watch branch and mutation testing for critical code.

## How it works

Maven plugin in `pom.xml`:

```xml
<plugin>
  <groupId>org.jacoco</groupId>
  <artifactId>jacoco-maven-plugin</artifactId>
  <version>0.8.12</version>
  <executions>
    <execution>
      <goals><goal>prepare-agent</goal></goals>
    </execution>
    <execution>
      <id>report</id>
      <phase>test</phase>
      <goals><goal>report</goal></goals>
    </execution>
    <execution>
      <id>check</id>
      <goals><goal>check</goal></goals>
      <configuration>
        <rules>
          <rule>
            <element>BUNDLE</element>
            <limits>
              <limit>
                <counter>LINE</counter>
                <value>COVEREDRATIO</value>
                <minimum>0.80</minimum>
              </limit>
            </limits>
          </rule>
        </rules>
      </configuration>
    </execution>
  </executions>
</plugin>
```

`mvn test` runs Surefire with the agent; `.exec` is written to `target/jacoco.exec`. The report highlights uncovered `if/else` branches in red. Gradle: `jacoco { toolVersion = "0.8.12" }` plus `jacocoTestReport` after `test`.

## Pitfalls

- **Chasing percentage.** Tests without asserts on exception paths.
- **Generated code in denominator.** Lombok/builders inflate "uncovered."
- **Multi-module without aggregate report.** 90% in a submodule hides 40% overall.
- **Forked tests without agent.** JVM fork without `-javaagent` yields empty exec.
- **Integration-only coverage.** Unit gaps masked by e2e; regressions hard to locate.
