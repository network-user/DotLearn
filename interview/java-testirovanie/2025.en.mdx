# What are edge cases in testing?

## In short

Edge cases are inputs and states at domain boundaries: empty collection, min/max int, leap day, zero amount, concurrent duplicate id. Off-by-one, overflow, and unhandled nulls usually break there first.

## How to answer in an interview

- Boundary Value Analysis: min, min+1, max-1, max.
- Equivalence classes: group "valid email" vs "no @" vs "unicode local part."
- For money/time use dedicated types; edge cases live in their unit tests.
- Property-based (jqwik) generates boundaries automatically.
- Honestly: not all edge cases are equal; I prioritize by risk and prod frequency.

## How it works

```java
class RangeValidator {
    static boolean inPercentRange(int value) {
        return value >= 0 && value <= 100;
    }
}

class RangeValidatorTest {
    @ParameterizedTest
    @ValueSource(ints = { -1, 0, 1, 99, 100, 101 })
    void boundaryPercents(int value) {
        boolean expected = value >= 0 && value <= 100;
        assertEquals(expected, RangeValidator.inPercentRange(value));
    }
}

class SequenceStats {
    static int max(int[] values) {
        if (values == null || values.length == 0) {
            throw new IllegalArgumentException("empty");
        }
        int peak = values[0];
        for (int v : values) {
            if (v > peak) peak = v;
        }
        return peak;
    }
}
```

Tests for `null`, `{}`, `{42}`, `{Integer.MIN_VALUE, Integer.MAX_VALUE}`. Calendar service adds: 2024-02-29 valid, 2023-02-29 invalid.

## Pitfalls

- **Happy path only.** Prod sends qty=0 and unicode NBSP in SKU.
- **Magic boundary without spec.** Code uses 100, requirements say 99.
- **Integer overflow edge.** `MAX_VALUE + 1` wraps silently.
- **Locale/timezone edge.** Instant at DST gap duplicates/skips local time.
- **Concurrent edge ignored.** Two threads, same idempotency key.
