# Что такое edge cases в тестировании?

## Суть

Edge cases это входы и состояния на границе домена: пустая коллекция, min/max int, leap day, zero amount, concurrent duplicate id. Именно там чаще ломаются off-by-one, overflow и необработанные null.

## Что ответить на собесе

- Boundary Value Analysis: min, min+1, max-1, max.
- Equivalence classes: группирую «валидный email» vs «без @» vs «unicode local part».
- Для money/time использую dedicated types, edge cases в их unit tests.
- Property-based (jqwik) генерирует границы автоматически.
- Честно: не все edge cases равны; приоритизирую по risk и частоте в prod.

## Как это работает

```java
class RangeValidator {
    static boolean inPercentRange(int value) {
        return value >= 0 && value <= 100;
    }
}

class RangeValidatorTest {
    @ParameterizedTest
    @ValueSource(ints = { -1, 0, 1, 99, 100, 101 })
    void boundaryPercents(int value) {
        boolean expected = value >= 0 && value <= 100;
        assertEquals(expected, RangeValidator.inPercentRange(value));
    }
}

class SequenceStats {
    static int max(int[] values) {
        if (values == null || values.length == 0) {
            throw new IllegalArgumentException("empty");
        }
        int peak = values[0];
        for (int v : values) {
            if (v > peak) peak = v;
        }
        return peak;
    }
}
```

Тесты на `null`, `{}`, `{42}`, `{Integer.MIN_VALUE, Integer.MAX_VALUE}`. Calendar service дополнительно: 2024-02-29 valid, 2023-02-29 invalid.

## Подводные камни

- **Только happy path.** Prod шлёт qty=0 и unicode NBSP в SKU.
- **Magic boundary без spec.** «100» лимит в коде, в требованиях 99.
- **Integer overflow edge.** `MAX_VALUE + 1` silent wrap.
- **Locale/timezone edge.** Instant at DST gap duplicated/skipped local time.
- **Concurrent edge ignored.** Two threads same idempotency key.
