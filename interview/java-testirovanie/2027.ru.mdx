# Line, branch и condition coverage: отличия

## Суть

Line coverage считает выполненные строки bytecode. Branch coverage требует пройти обе стороны каждого jump (if/else, switch, ternary). Condition coverage (MC/DC lite) проверяет независимое влияние каждого boolean subexpression. Строгость растёт: line < branch < condition.

## Что ответить на собесе

- 100% line не implies 100% branch: `if (x) foo()` без else branch red.
- JaCoCo report показывает diamond icons на непокрытых ветках.
- Сложное условие `a && b || c` нужно несколько inputs для branch/condition.
- CI gate обычно line + branch на domain packages.
- Честно: coverage метрика не равна correctness; mutation testing дополняет.

## Как это работает

```java
static String classify(int age, boolean vip) {
    if (age >= 18 && vip) {
        return "adult-vip";
    }
    if (age >= 18) {
        return "adult";
    }
    return "minor";
}
```

| Test input | Lines hit | Branches hit |
|------------|-----------|--------------|
| (20, true) | all partial | true path first if |
| (20, false) | needs 2nd if | else implicit vip |
| (16, false) | minor return | underage branch |

Один test `(20,true)` даёт high line % но не покрывает minor path. Condition coverage добавляет `(17,false)`, `(20,false)`, `(16,true)` чтобы toggling `vip` менял outcome при fixed age.

JaCoCo HTML: зелёная строка, жёлтый diamond partial branch, красный missed.

## Подводные камни

- **Гонка за line 100%.** Empty catch swallowed.
- **Ternary и switch без default.** Hidden branch.
- **Short-circuit logic.** `a && b()` второй operand never evaluated test misses.
- **Generated bytecode inflation.** Lombok builder skews line metric.
- **Condition coverage cost.** Комбinatorial explosion на 5+ flags.
