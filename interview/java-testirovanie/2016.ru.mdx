# Как измерить code coverage с JaCoCo?

## Суть

JaCoCo инструментирует bytecode Java и считает, какие инструкции, ветви и строки выполнились во время тестов. Отчёт показывает пробелы в тестах; метрика дополняет, но не заменяет качество assertions.

## Что ответить на собесе

- On-the-fly agent через `-javaagent` или Maven/Gradle plugin `prepare-agent`.
- Метрики: line, branch, method, class; HTML report в `target/site/jacoco`.
- В CI порог через `check` goal: fail build ниже 80% на critical modules.
- Exclude DTO/generated code, иначе процент бессмысленен.
- Честно: 100% line coverage возможен при нулевых assert; смотрю branch и mutation testing для критичного кода.

## Как это работает

Maven plugin в `pom.xml`:

```xml
<plugin>
  <groupId>org.jacoco</groupId>
  <artifactId>jacoco-maven-plugin</artifactId>
  <version>0.8.12</version>
  <executions>
    <execution>
      <goals><goal>prepare-agent</goal></goals>
    </execution>
    <execution>
      <id>report</id>
      <phase>test</phase>
      <goals><goal>report</goal></goals>
    </execution>
    <execution>
      <id>check</id>
      <goals><goal>check</goal></goals>
      <configuration>
        <rules>
          <rule>
            <element>BUNDLE</element>
            <limits>
              <limit>
                <counter>LINE</counter>
                <value>COVEREDRATIO</value>
                <minimum>0.80</minimum>
              </limit>
            </limits>
          </rule>
        </rules>
      </configuration>
    </execution>
  </executions>
</plugin>
```

`mvn test` запускает Surefire с agent, `.exec` файл пишется в `target/jacoco.exec`. Report подсвечивает красным непокрытые ветви `if/else`. Gradle: `jacoco { toolVersion = "0.8.12" }` + `jacocoTestReport` после `test`.

## Подводные камни

- **Гонка за процентом.** Тесты без assert на exception path.
- **Generated code в denominator.** Lombok/builder раздувают «непокрытое».
- **Multi-module без aggregate report.** Видишь 90% в submodule, 40% overall скрыто.
- **Offloaded tests без agent.** Forked JVM без `-javaagent` даёт пустой exec.
- **Coverage на integration only.** Unit gaps маскируются e2e, локализация регрессий медленная.
