# Rethrowing exceptions with throws

## In short

The `throws` clause on a method declares checked exceptions the method does not handle and passes to callers. The compiler requires the caller to catch or declare them too. Unchecked (`RuntimeException`) need not be declared, though sometimes listed for documentation. Rethrowing preserves the stack trace when thrown again without wrapping.

## How to answer in an interview

- `throws IOException` in DAO; handle at controller/service boundary.
- Multi-throws: `throws IOException, ParseException`.
- Rethrow: `catch (IOException ex) { throw ex; }` or `throw new ServiceException("msg", ex)`.
- In overrides, `throws` may only declare checked exceptions allowed by the parent (subset).
- Honestly: `throws Exception` on public API is a lazy contract; I avoid it.

## How it works

```java
public final class CsvReportLoader {

    public List<Row> load(Path csvPath) throws IOException {
        return Files.readAllLines(csvPath).stream()
            .skip(1)
            .map(Row::parse)
            .toList();
    }
}

public final class ReportService {

    private final CsvReportLoader loader = new CsvReportLoader();

    public List<Row> fetchMonthly(Path path) {
        try {
            return loader.load(path);
        } catch (IOException ex) {
            throw new ReportUnavailableException("Cannot read " + path, ex);
        }
    }
}
```

`load` propagates checked `IOException` upward. `fetchMonthly` maps it to a domain unchecked type so the REST layer need not declare IOException.

Override with narrowed throws:

```java
interface Storage {
    void save(byte[] payload) throws IOException;
}

class S3Storage implements Storage {
    @Override
    public void save(byte[] payload) throws IOException {
        client.putObject(bucket, key, payload);
    }
}
```

You cannot add `throws SQLException` in the override if the interface does not declare it.

## Pitfalls

- **Forgot throws on a private method.** Compiler requires handle inside or throws.
- **throws on interface breaks lambdas.** Checked exceptions in functional interfaces are awkward.
- **Swallow and throw new without cause.** Loses the original stack trace.
- **throws RuntimeException in signature.** Noise; does not affect the compiler.
- **Method throws, caller ignores.** Only unchecked can be skipped; checked will not compile.
