# Виды исключений в Java: checked, unchecked, errors

## Суть

Java делит `Throwable` на три практических категории: checked exceptions (компилятор контролирует обработку), unchecked exceptions (`RuntimeException`, баги логики), errors (`Error`, фатальные сбои JVM). Механизм отделяет happy path от error handling через try/catch/finally и throws. Цель: контролируемый failure вместо silent crash.

## Что ответить на собесе

- Checked: `IOException`, `SQLException`, `TimeoutException` (если не runtime).
- Unchecked: `NPE`, `IllegalStateException`, `ConcurrentModificationException`.
- Errors: `OutOfMemoryError`, `StackOverflowError`, `NoClassDefFoundError`.
- Errors в production коде не catch-у; unchecked фиксирую тестами и валидацией.
- Честно: ловить `StackOverflowError` и продолжать бессмысленно, нужен fix рекурсии.

## Как это работает

Checked file read:

```java
public String readToken(Path secrets) throws IOException {
    return Files.readString(secrets).trim();
}
```

Unchecked validation:

```java
public UUID requireUuid(String raw) {
    try {
        return UUID.fromString(raw);
    } catch (IllegalArgumentException ex) {
        throw new BadRequestException("Invalid uuid: " + raw, ex);
    }
}
```

Error scenario (не обрабатываем):

```java
void brokenRecursion() {
    brokenRecursion();
}
```

Вызов закончится `StackOverflowError`. Правильный ответ: исправить алгоритм, увеличить stack только как временную меру.

Сводка:

| Вид | Проверка компилятором | Типичный пример | Обработка |
|-----|----------------------|-----------------|-----------|
| Checked | да | IOException | try/catch или throws |
| Unchecked | нет | NullPointerException | fix + optional catch |
| Error | нет | OutOfMemoryError | не catch в app code |

## Подводные камни

- **Throwable catch all.** Маскирует Error и OOM.
- **Путаница Exception vs RuntimeException.** Не все Exception checked.
- **Custom extends Error.** Крайне редко и сбивает с толку.
- **Перехват Error в library.** Скрывает неrecoverable состояние JVM.
- **Одинаковая обработка I/O и NPE.** Разная семантика recovery.
