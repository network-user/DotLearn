# Multi-catch: multiple exceptions in one catch block

## In short

Since Java 7 one `catch` can handle multiple types with `|`, provided they are unrelated by inheritance. One handler runs for all listed exceptions. The exception variable is effectively final: reassignment is not allowed. The compiler emits bytecode with multiple catch branches merged into one block.

## How to answer in an interview

- Syntax: `catch (IOException | SQLException ex)`.
- You cannot use `IOException | Exception` because IOException is a subtype of Exception.
- Separate catch blocks still go from specific to general.
- Multi-catch removes duplication when recovery logic is identical.
- Honestly: if handling differs, separate catch blocks read better than multi-catch.

## How it works

```java
public void importRecords(Path csvPath, Connection db) {
    try {
        var lines = Files.readAllLines(csvPath);
        insertBatch(db, lines);
    } catch (IOException | SQLException ex) {
        throw new DataImportException("Import failed for " + csvPath, ex);
    }
}
```

Both checked types need try/catch or throws on the method; here we catch and wrap in a domain unchecked type.

Unchecked multi-catch:

```java
try {
    routePayment(request);
} catch (IllegalArgumentException | IllegalStateException ex) {
    metrics.increment("payment.validation.failure");
    return PaymentResult.rejected(ex.getMessage());
}
```

Invalid:

```java
// does not compile
catch (Exception | IOException ex) { }
```

Because `IOException extends Exception`.

Since Java 9 you can catch by type without a variable when ignoring:

```java
try {
    risky();
} catch (IOException _) {
    return Optional.empty();
}
```

## Pitfalls

- **Overlapping types in multi-catch.** Compile error; easy to miss in hierarchies.
- **Different recovery in one block.** `if (ex instanceof ...)` brings back boilerplate.
- **Catching Throwable | something.** Throwable swallows everything; the second type is pointless.
- **Mutating ex.** Effectively final: cannot do `ex = new IOException()`.
- **Logging without context.** Multi-catch hides which type fired unless you log the class.
