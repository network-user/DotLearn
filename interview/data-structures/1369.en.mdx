# How do you check whether a value occurs among a dictionary's values?

## In short

The `in` operator on a dictionary searches by keys. To check values, use the `dict.values()` view: `target in my_dict.values()`.

## How to answer in an interview

`value in my_dict` checks for the presence of a key, not a value - you have to keep that in mind. To search by values, use `value in my_dict.values()`. An important note on complexity: looking up a key in a dictionary is O(1) on average, because under the hood it's a hash table, whereas iterating over `values()` is linear, O(n), since it's a sequential comparison of all values.

## How it works

`values()` returns a dynamic view of the dictionary's values. The `in` operator walks over it and compares each value with the target.

```python
prices = {'apple': 1, 'banana': 2, 'orange': 3}

print(2 in prices)            # False - 2 is not among the keys
print(2 in prices.values())   # True  - 2 is among the values
print('apple' in prices)      # True  - check by key
```

If you need to check values frequently and the dictionary is large, it's worth keeping a reverse index - a set of values `set(prices.values())` - so the check becomes O(1) again.

## Pitfalls

- A common mistake is writing `value in my_dict` expecting a search by value; in fact this checks for a key.
- `values()` is a view, not a list: it reflects changes to the dictionary, but you can't "rewind" it as an iterator without a new call.
- Searching through `values()` is linear. For frequent checks, cache the values in a `set`.
- The comparison uses `==`, so `1 in {'a': True}.values()` returns `True`, because `True == 1`.
