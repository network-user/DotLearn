# frozenset: when do you need an immutable set and what does it give you?

## In short

`frozenset` is a frozen version of `set`: the same collection of unique elements, but you cannot change it after creation. This immutability makes it hashable, so a `frozenset` can go where a regular `set` is not allowed.

## How to answer in an interview

`frozenset` is needed when you want a set that is at the same time hashable. A regular `set` is mutable and therefore not hashable - it cannot be a dict key or an element of another set. `frozenset` can do that: it has no `add`/`remove`, its hash is stable, so it works as a key or as an element of a nested set. All read operations (union, intersection, membership check) work just like with a regular set.

## How it works

`frozenset` supports all operations that do not change its contents: `in`, `union`, `intersection`, iteration. Methods like `add` and `remove` are not available on it.

```python
immutable = frozenset([1, 2, 3, 4])

# immutable.add(5)
# AttributeError: 'frozenset' object has no attribute 'add'

# frozenset as a dict key - you cannot do this with a regular set
by_group = {frozenset([1, 2, 3]): "value"}
print(by_group[frozenset([1, 2, 3])])  # value

# frozenset as an element of another set
nested = {frozenset([1, 2]), frozenset([3, 4])}
```

The key logic: a dict and a set require hashable elements, and hashability implies immutability. A regular `set` can be changed, so its hash would be unstable - Python does not hash such an object. A `frozenset` has fixed contents and a stable hash, so it lives happily as a key or a nested element.

## Pitfalls

- `frozenset` is immutable only itself. If it contained mutable objects - but it cannot, since elements must also be hashable - so only immutable values end up inside.
- Operations like `union` do not change the original `frozenset`, they return a new one. Expecting an in-place mutation is pointless.
- Creating a `frozenset` costs the same as creating a set: deduplication and hashing of elements. For a hot path it is not free.
- Two `frozenset`s with the same elements are equal and have the same hash regardless of order - this is exactly what their use as keys relies on.
