# Which is faster to look up by key: a list or a dictionary?

## In short

A dictionary looks up by key in amortized O(1) thanks to a hash table, while a list does it in O(n) because it scans elements one by one. If you need fast access by key, the choice clearly favors the dictionary.

## How to answer in an interview

For lookup by key a dictionary is fundamentally faster: the key is run through a hash function, and Python jumps straight to the slot where the value sits, and that's amortized O(1). A list doesn't offer such luxury: to check whether a value is present, it goes from the start and compares elements one by one, that is O(n). So I use lists where order matters or where duplicates are needed, and dictionaries when I need to retrieve data by a unique key frequently.

## How it works

A list is an ordered dynamic array. Access by index is instant, O(1), but searching for a value (`x in my_list`) is linear: Python compares elements in turn until it finds the one it needs or reaches the end. Appending to the end is amortized O(1), while inserting in the middle is O(n) because of shifting the tail.

A dictionary stores "key-value" pairs in a hash table. The key must be unique and hashable. On lookup, the key is turned by a hash function into a number that points to a position in the table, so access barely depends on the dictionary's size.

```python
my_list = [1, 2, 3, 4, 5]
my_dict = {1: "a", 2: "b", 3: "c", 4: "d", 5: "e"}

element_in_list = 3 in my_list  # O(n): scanning elements
element_in_dict = 3 in my_dict  # O(1): check by the key's hash
```

The difference in `in` is the key here: for a list it's a search over values, for a dictionary it's a check for the presence of a key in the hash table.

## Pitfalls

- A dictionary's O(1) is amortized. With a large number of collisions, lookup can degrade to O(n).
- `x in my_dict` checks keys, not values. To search by values you need `x in my_dict.values()`, and that's O(n) again.
- Dictionary keys must be hashable. A list or another mutable object won't become a key; you'll get a `TypeError`.
- A dictionary uses more memory than a list of the same length, in exchange for lookup speed. It's a deliberate trade-off.
