# Два ключа попали в одну ячейку: как хеш-таблица выходит из положения?

## Суть

Когда разные ключи дают один индекс, это коллизия. Решают её двумя семействами методов: цепочками (chaining) - складывают всё в список внутри ячейки, и открытой адресацией (open addressing) - ищут другую свободную ячейку в самой таблице.

## Что ответить на собесе

Коллизия - это ситуация, когда хеш-функция для двух разных ключей выдаёт один и тот же индекс. Полностью избежать коллизий нельзя, потому что входов бесконечно много, а ячеек конечное число, поэтому таблица должна уметь их разрешать. Два классических подхода. Первый - метод цепочек: каждая ячейка хранит коллекцию (обычно связный список) всех элементов с этим индексом, и при коллизии мы просто дописываем туда новую пару. Второй - открытая адресация: все элементы живут прямо в массиве, и при занятой ячейке мы по определённому правилу (линейное, квадратичное пробирование или двойное хеширование) ищем следующую свободную. CPython, кстати, для своих `dict` использует именно открытую адресацию.

**Цепочки.** Каждая ячейка - это маленькая коллекция. Вставка дописывает пару в конец, поиск проходит по коллекции и сравнивает ключи.

```python
class ChainedHashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]

    def _index(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        bucket = self.table[self._index(key)]
        for pair in bucket:
            if pair[0] == key:
                bucket.remove(pair)
                break
        bucket.append((key, value))

    def get(self, key):
        for k, v in self.table[self._index(key)]:
            if k == key:
                return v
        return None
```

**Открытая адресация.** Элементы лежат в самом массиве. При коллизии линейно шагаем вперёд, пока не найдём пустую ячейку (это линейное пробирование).

```python
class OpenAddressingTable:
    def __init__(self, size):
        self.size = size
        self.table = [None] * size

    def _index(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        index = self._index(key)
        while self.table[index] is not None and self.table[index][0] != key:
            index = (index + 1) % self.size
        self.table[index] = (key, value)

    def get(self, key):
        index = self._index(key)
        while self.table[index] is not None:
            k, v = self.table[index]
            if k == key:
                return v
            index = (index + 1) % self.size
        return None
```

## Подводные камни

- В цепочках при множестве коллизий ячейка вырождается в длинный список, и поиск деградирует с O(1) до O(n). Защита - хорошая хеш-функция и контроль фактора загрузки.
- В открытой адресации удаление элемента ломает цепочку поиска: нельзя просто поставить `None`, иначе следующий за ним элемент станет «невидим». Используют надгробия (tombstones) - специальные метки удалённых ячеек.
- Открытая адресация плохо себя ведёт при высоком факторе загрузки: чем плотнее таблица, тем длиннее пробирование. Около 70-75% заполнения её обычно расширяют (rehash).
- Линейное пробирование страдает от кластеризации - занятые ячейки слипаются в длинные группы. Квадратичное пробирование и двойное хеширование разбивают такие кластеры.
