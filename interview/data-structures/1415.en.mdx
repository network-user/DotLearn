# How do you sort a list of dictionaries by the value of one of the fields?

## In short

Pass the `key` parameter to `sorted()` - a function that returns the value of the needed field for each dictionary. The classic form: `sorted(data, key=lambda d: d['field'])`.

## How to answer in an interview

Via `sorted()` with the `key` argument: `sorted(people, key=lambda x: x['age'])`. The `key` function is called once for each element and determines which value to compare by. To reverse, add `reverse=True`; to sort by several fields, return a tuple from `key`. An alternative to the lambda is `operator.itemgetter('age')`, which is slightly faster and reads cleaner. The sort is stable, so elements with an equal key keep their original order.

## How it works

```python
people = [
    {'name': 'Alice', 'age': 30},
    {'name': 'Bob', 'age': 25},
    {'name': 'Charlie', 'age': 35},
]

sorted_people = sorted(people, key=lambda x: x['age'])
print(sorted_people)
```

`sorted()` builds a new list without touching the original. The parameter `key=lambda x: x['age']` says: compare dictionaries by the value of the `'age'` field. The result is `Bob (25)`, `Alice (30)`, `Charlie (35)`.

Multiple criteria and reverse order:

```python
sorted(people, key=lambda x: x['age'], reverse=True)
sorted(people, key=lambda x: (x['age'], x['name']))
```

The second call sorts first by age, and on equal age by name, because tuples are compared element by element from left to right.

## Pitfalls

- `sorted()` returns a new list; to mutate the list in place there is `list.sort()` with the same `key`.
- If the field is not present in every dictionary, a lambda with `x['field']` will fail with a `KeyError`; `x.get('field', default)` is safer.
- You cannot compare values of different, non-comparable types (for example, a number and a string under one key) - it will be a `TypeError`.
- `reverse=True` does not break stability: with equal keys the relative order is preserved, not mirrored.
