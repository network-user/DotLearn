# Три способа выкинуть элемент из списка: где remove, где del, а где pop?

## Суть

Удалять из списка можно тремя инструментами, и выбор зависит от того, что у вас на руках. `remove()` работает по значению, `del` и `pop()` - по индексу, но `pop()` ещё и возвращает то, что вынул.

## Что ответить на собесе

В Python для удаления из списка есть три разных инструмента под три разные ситуации. Если я знаю значение и хочу убрать его первое вхождение - беру `list.remove(value)`. Если я работаю с индексом или срезом и возвращаемое значение мне не нужно - использую оператор `del`, он же умеет удалять и саму переменную из пространства имён. А если мне нужно одновременно достать элемент и убрать его из списка - тут только `list.pop(index)`, без аргумента он снимает последний элемент. Главное помнить про исключения: `remove` кидает `ValueError` при отсутствии значения, а `pop` - `IndexError` при выходе за границы.

## Как это работает

`remove()` ищет первое совпадение по значению и стирает его. Дубликаты за ним остаются.

```python
fruits = ['apple', 'banana', 'cherry', 'banana']
fruits.remove('banana')
print(fruits)  # ['apple', 'cherry', 'banana']
```

`del` - это оператор уровня языка, а не метод. Он бьёт по индексу, по срезу и даже по целой переменной.

```python
numbers = [0, 1, 2, 3, 4, 5]
del numbers[1]
print(numbers)  # [0, 2, 3, 4, 5]

del numbers[1:3]
print(numbers)  # [0, 4, 5]
```

`pop()` снимает элемент по индексу и тут же отдаёт его значение. Без аргумента берёт хвост списка - это и делает его удобным для стека.

```python
letters = ['a', 'b', 'c', 'd']
last_letter = letters.pop()
print(last_letter)  # 'd'

second_letter = letters.pop(1)
print(second_letter)  # 'b'
print(letters)  # ['a', 'c']
```

## Подводные камни

- `remove()` удаляет только **первое** вхождение. Чтобы вычистить все дубликаты, нужен цикл или list comprehension.
- Если значения нет в списке, `remove()` поднимает `ValueError`, а не возвращает что-то молча.
- `pop(i)` на пустом списке или с выходящим за границы индексом даёт `IndexError`.
- `del list[i]` не возвращает удалённое - если значение нужно, это сразу `pop()`, иначе придётся читать элемент до удаления.
- Удаление по индексу в цикле опасно: индексы сдвигаются, и можно проскочить элементы. Идите по копии или с конца.
