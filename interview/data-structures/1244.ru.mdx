# Рекурсия против цикла: в чём принципиальная разница?

## Суть

Рекурсивная функция решает задачу, вызывая саму себя на уменьшенных входных данных, пока не достигнет базового случая. Цикл решает ту же задачу повторением блока инструкций без новых вызовов. Главное отличие в стоимости: рекурсия растит стек вызовов, итерация работает в фиксированном объёме памяти.

## Что ответить на собесе

Рекурсия и итерация взаимозаменяемы по результату, но различаются по реализации и ресурсам.

- Рекурсия описывает решение через само себя и обязана иметь базовый случай, иначе уйдёт в бесконечность и упрётся в лимит стека.
- Каждый вложенный вызов занимает кадр в стеке, поэтому глубокая рекурсия в Python падает с `RecursionError` примерно на тысяче уровней.
- Итерация хранит состояние в обычных переменных и не растит стек, поэтому по памяти и константам почти всегда выгоднее.
- Рекурсию беру там, где структура задачи древовидная (обход дерева, «разделяй и властвуй»), а итерацию — для линейных проходов и когда важна производительность.

## Как это работает

Рекурсия раскладывает задачу на похожие подзадачи меньшего размера. Классика — факториал:

```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

Вызов `factorial(3)` разворачивается в `3 * factorial(2)`, затем `2 * factorial(1)` и так до базового случая `n == 0`, после чего цепочка сворачивается обратно. Каждый незавершённый вызов висит в стеке, ожидая результата вложенного.

Тот же результат итеративно — без роста стека:

```python
def factorial(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result
```

Здесь состояние накапливается в одной переменной `result`, а память не зависит от `n`.

## Подводные камни

- **Нет базового случая или он недостижим** — бесконечная рекурсия и `RecursionError`. Всегда проверяйте, что аргумент строго движется к базовому случаю.
- **Глубина стека**. CPython по умолчанию ограничивает рекурсию (`sys.getrecursionlimit()` ~ 1000). Для длинных линейных задач рекурсия неприменима без переписывания в цикл.
- **Python не делает оптимизацию хвостовых вызовов**, поэтому даже хвостовая рекурсия не превращается в цикл автоматически.
- **Накладные расходы на вызовы**. Каждый рекурсивный вызов — это создание кадра стека; на горячих участках это заметно медленнее цикла.
