# Нужен обычный список из ключей словаря - как его получить?

## Суть

`dict.keys()` отдаёт не список, а лёгкое представление (view) ключей. Если нужен именно `list`, оберните вызов в `list()`: `list(my_dict.keys())`.

## Что ответить на собесе

Метод `.keys()` возвращает объект-представление `dict_keys`, а не список. Это динамический «вид» на ключи: он не копирует данные и отражает изменения словаря в реальном времени. Перебирать его в цикле можно напрямую, но если мне нужен настоящий список - с индексацией, срезами или возможностью изменить его независимо от словаря - я явно конвертирую через `list(my_dict.keys())`. То же самое касается `.values()` и `.items()`: все три возвращают view, а не списки.

## Как это работает

```python
my_dict = {'name': 'Alice', 'age': 30, 'city': 'New York'}

keys_view = my_dict.keys()      # объект представления dict_keys
keys_list = list(keys_view)     # обычный список

print(keys_list)                # ['name', 'age', 'city']
```

Представление - это окно в словарь. Оно живое: добавили ключ в словарь - и view сразу его показывает.

```python
d = {'a': 1}
view = d.keys()
d['b'] = 2
print(list(view))   # ['a', 'b'] - view увидел новый ключ
```

`list()` же делает снимок: получившийся список застывает и от словаря больше не зависит. Порядок ключей в современном Python (3.7+) совпадает с порядком вставки.

## Подводные камни

- `dict.keys()[0]` не сработает - у view нет индексации. Сначала `list(...)`, потом индекс.
- View **живой**: если итерироваться по нему и одновременно менять словарь, поймаете `RuntimeError: dictionary changed size during iteration`. Делайте снимок через `list()` заранее.
- Конвертация в список копирует ключи в память - на огромных словарях это лишний расход, если вам нужно просто пройтись циклом (тогда хватит самого view).
- `dict_keys` ведёт себя как множество и поддерживает операции `&`, `|`, `-`. У списка такого нет - это иногда повод оставить именно view.
