# append vs extend: a single element or merging an iterable element by element

## In short

`append(x)` adds exactly one element `x` to the end of the list (even if it is a list - it goes in whole as a single nested unit). `extend(iterable)` walks through the iterable and merges each of its elements separately.

## How to answer in an interview

`append` increases the list length by one: whatever you pass becomes the last element. `extend` increases the length by the number of elements in the argument: it iterates over it and adds them one by one, effectively like `+=`. That's why `append([1,2])` gives a nested list, while `extend([1,2])` gives two new flat elements. Both methods modify the list in place and return `None`.

## How it works

`append` - a single element, no matter the type:

```python
fruits = ['apple', 'banana']
fruits.append('orange')
print(fruits)            # ['apple', 'banana', 'orange']

fruits.append([1, 2])
print(fruits)            # ['apple', 'banana', 'orange', [1, 2]]
print(len(fruits))       # 4 - the nested list is one element
```

`extend` - element-by-element merging of an iterable:

```python
fruits = ['apple', 'banana']
fruits.extend(['orange', 'grape'])
print(fruits)            # ['apple', 'banana', 'orange', 'grape']
print(len(fruits))       # 4
```

`extend` accepts any iterable object, not only a list - a string, tuple, set, or generator:

```python
chars = ['a']
chars.extend('xy')       # the string is iterated character by character
print(chars)             # ['a', 'x', 'y']
```

## Pitfalls

- Both methods work in place and return `None`. Writing `fruits = fruits.append('x')` will reset the variable to `None` - a common mistake.
- `lst.extend('abc')` breaks the string into characters `'a', 'b', 'c'` instead of adding it whole. To put a string in as a single element, use `append`.
- `lst.extend(5)` fails with `TypeError`: a number is not iterable. `append(5)` works fine.
- `a.extend(b)` is equivalent to `a += b` (for lists), but not to `a = a + b`: the latter creates a new list rather than modifying the original in place.
