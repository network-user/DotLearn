# Какие базовые алгоритмы сортировки нужно знать и чем они различаются?

## Суть

Минимальный джентльменский набор: пузырьковая, вставками, выбором (все O(n²)), а также быстрая, слиянием и пирамидальная (O(n log n)). Различаются они скоростью, расходом памяти и устойчивостью.

## Что ответить на собесе

Квадратичные сортировки просты, но на больших данных непригодны; рабочие алгоритмы дают O(n log n).

- Пузырёк, вставками, выбором пишутся в пару строк, но на больших данных непригодны из-за O(n²).
- Быстрая: быстрая на практике, in-place, но худший случай O(n²) при плохом опорном.
- Слиянием стабильна и гарантированно O(n log n), но требует O(n) памяти; пирамидальная in-place и O(n log n), но нестабильна.
- В реальном коде вызывают `sorted()` / `list.sort()` — это Timsort, гибрид слияния и вставок, стабильный и O(n log n).

## Как это работает

Квадратичные методы. Пузырёк гоняет соседние пары до полной упорядоченности, вставками наращивает отсортированный префикс, выбором ищет минимум и ставит его в начало:

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

Быстрая сортировка — «разделяй и властвуй» вокруг опорного элемента:

```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[0]
    less = [x for x in arr[1:] if x <= pivot]
    greater = [x for x in arr[1:] if x > pivot]
    return quick_sort(less) + [pivot] + quick_sort(greater)
```

Слияние делит массив пополам, сортирует половины и сливает их:

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    merged, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            merged.append(left[i]); i += 1
        else:
            merged.append(right[j]); j += 1
    merged.extend(left[i:])
    merged.extend(right[j:])
    return merged
```

Пирамидальная строит max-кучу и по одному снимает максимум с вершины, восстанавливая кучу.

## Подводные камни

- Быстрая сортировка с опорным «первый элемент» деградирует до O(n²) на уже отсортированных данных; на практике берут медиану трёх или случайный pivot.
- Стабильность важна, когда сортируют по ключу, сохраняя порядок равных: слияние и Timsort стабильны, быстрая и пирамидальная — нет.
- Слияние требует O(n) дополнительной памяти, тогда как пирамидальная и быстрая работают почти на месте.
- На практике почти всегда правильный ответ — встроенный `sorted()`; собственную сортировку пишут лишь для учебных целей или экзотических ограничений.
