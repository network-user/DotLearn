# Что годится в ключи словаря, а что нет

## Суть

Ключом `dict` может быть только хешируемый объект, а хешируемость в Python почти всегда означает неизменяемость: строки, числа, кортежи из неизменяемых элементов. Списки и словари в ключи не годятся.

## Что ответить на собесе

Ключи словаря обязаны быть хешируемыми, а хешируемость почти всегда означает неизменяемость.

- Доступ по ключу идёт через хеш-таблицу: Python берёт `hash(key)`, по нему выбирает корзину и там сравнивает ключи через `==`.
- От ключа требуются два свойства — стабильный хеш на всё время жизни и корректное сравнение.
- Неизменяемые встроенные типы (str, int, float, bool, tuple, frozenset) этому удовлетворяют.
- Изменяемые (list, dict, set) — нет, поэтому при попытке взять их ключом летит `TypeError: unhashable type`.

Хешируемость определяется методом `__hash__`. У объекта есть стабильный хеш, если `__hash__` возвращает одно и то же число всю жизнь объекта, а равные объекты дают равный хеш.

Строки и числа подходят сразу:

<PyDemo title="Строки и числа как ключи словаря" autoRun code={`prices = {"apple": 50, "pear": 70}
print(prices["apple"])  # 50

coords = {1: "one", 2.5: "two-and-half"}
print(coords[1]) # one`} />

Кортеж хешируем, только если хешируемы все его элементы — это рекурсивное свойство:

```python
points = {(1, 2): "A", (3, 4): "B"}
print(points[(1, 2)])  # A
```

Сравнение хешей напрямую упрощённо выглядит так:

```python
print(hash("apple") == hash("apple"))  # True
print(hash((1, 2)) == hash((1, 2)))    # True
```

## Подводные камни

- Кортеж со списком внутри уже не хешируем: `hash((1, [2]))` бросит `TypeError`, потому что элемент-список изменяем.
- `1`, `1.0` и `True` для словаря — один и тот же ключ, потому что они равны и имеют равный хеш. `{1: "a", True: "b", 1.0: "c"}` схлопнется в один элемент.
- Свой класс по умолчанию хешируем (по `id`), но как только вы определяете `__eq__`, Python обнуляет `__hash__`, и объект становится нехешируемым, пока вы не зададите `__hash__` явно.
- Список нельзя сделать ключом, но можно превратить в `tuple`; множество — во `frozenset`.
