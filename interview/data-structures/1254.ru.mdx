# Собираем свой list: как изнутри устроен динамический массив

## Суть

Свой `list` пишется поверх обычного массива фиксированного размера. Храним отдельно ёмкость и реальное число элементов; пока есть запас - кладём в свободную ячейку, кончился запас - выделяем массив побольше, переносим туда данные и продолжаем. Это и есть динамический массив, на котором стоит встроенный `list`.

## Что ответить на собесе

`list` в Python - это динамический массив: элементы лежат подряд, а блок переразмеряется по мере роста.

- Чтобы повторить вручную, нужны три вещи: буфер фиксированного размера, счётчик занятых ячеек (size) и заявленная ёмкость (capacity).
- При `append` проверяю, есть ли место; если нет - удваиваю ёмкость, копирую старые элементы в новый буфер и только потом вставляю.
- Удаление по индексу сдвигает хвост влево.
- Доступ по индексу - O(1), вставка в конец - амортизированно O(1), удаление из середины - O(n).

1. **Инициализация.** Заводим буфер под небольшое число элементов - стартовую ёмкость.
2. **Добавление.** Перед вставкой сравниваем size и capacity. Место есть - кладём за O(1). Места нет - сначала расширяемся.
3. **Расширение.** Создаём буфер вдвое больше, переносим элементы, переключаемся на него. Разовая операция O(n), но из-за удвоения она амортизируется до константы на вставку.
4. **Удаление по индексу.** Сдвигаем все элементы правее на одну позицию влево, затираем хвостовую ячейку и уменьшаем size - это O(n).

```python
class CustomList:
    def __init__(self):
        self.capacity = 4
        self.size = 0
        self.data = [None] * self.capacity

    def append(self, item):
        if self.size == self.capacity:
            self._resize(2 * self.capacity)
        self.data[self.size] = item
        self.size += 1

    def _resize(self, new_capacity):
        new_data = [None] * new_capacity
        for i in range(self.size):
            new_data[i] = self.data[i]
        self.data = new_data
        self.capacity = new_capacity

    def remove(self, index):
        if 0 <= index < self.size:
            for i in range(index, self.size - 1):
                self.data[i] = self.data[i + 1]
            self.size -= 1
            self.data[self.size] = None
        else:
            raise IndexError("Index out of range")

    def __getitem__(self, index):
        if 0 <= index < self.size:
            return self.data[index]
        raise IndexError("Index out of range")

    def __len__(self):
        return self.size
```

- `__init__` - буфер со стартовой ёмкостью 4.
- `append` - вставка в конец с расширением при нехватке места.
- `_resize` - новый буфер и перенос данных.
- `remove` - удаление по индексу со сдвигом хвоста.
- `__getitem__` / `__len__` - доступ по индексу и текущий размер.

## Подводные камни

- **Не путайте size и capacity.** Длина списка для пользователя - это `size`, а не размер буфера. `__len__` обязан возвращать size, иначе наружу полезут ячейки `None`.
- **Затирать хвост после `remove` важно**, иначе на последней позиции останется висячая ссылка на удалённый объект - утечка памяти и неверный результат при будущем расширении.
- **Проверяйте границы индекса** и в `remove`, и в `__getitem__`: без проверки доступ к чужой ячейке буфера вернёт мусор или `None` вместо честной ошибки.
- **Сжатие здесь не реализовано.** После множества удалений буфер останется большим; в продакшен-реализации добавляют усадку при низком заполнении.
