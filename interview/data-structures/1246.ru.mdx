# Индексация в Python: как достать элемент по позиции?

## Суть

Доступ к элементу последовательности в Python идёт через квадратные скобки: `seq[i]`. Нумерация начинается с нуля, а отрицательные индексы адресуют элементы с конца — `seq[-1]` это последний элемент.

## Что ответить на собесе

В Python к элементу последовательности обращаются по индексу через `[]`.

- Прямая нумерация начинается с 0, поэтому первый элемент это `seq[0]`, а у списка длины n последний это `seq[n-1]`.
- Отрицательные индексы удобны: `seq[-1]` всегда последний, `seq[-2]` предпоследний, и не нужно вычислять длину вручную.
- Индексация работает одинаково для списков, кортежей и строк — все они упорядоченные последовательности; для списков и кортежей доступ это O(1) (под капотом массив указателей).
- Выход за границы кидает `IndexError`, и об этом важно помнить при работе с пользовательскими данными.

## Как это работает

Индекс — это смещение от начала. Для списка из трёх элементов валидны индексы 0, 1, 2:

```python
my_list = ['apple', 'banana', 'cherry']
my_list[0]   # 'apple'
my_list[1]   # 'banana'
my_list[2]   # 'cherry'
```

Отрицательные индексы отсчитываются от конца, при этом `-1` соответствует позиции `len - 1`:

```python
my_list[-1]  # 'cherry'
my_list[-2]  # 'banana'
my_list[-3]  # 'apple'
```

То же работает для строк и кортежей, ведь это тоже последовательности:

```python
"hello"[0]    # 'h'
"hello"[-1]   # 'o'
(10, 20, 30)[1]  # 20
```

## Подводные камни

- **`IndexError` за границами**. `my_list[3]` для списка из трёх элементов упадёт; индекс должен быть в диапазоне `-len .. len-1`.
- **Индексирование строки даёт строку, а не символ**. В Python нет отдельного типа char: `"hello"[0]` это строка длины 1.
- **Не путать индекс со срезом**. `seq[1]` это элемент, а `seq[1:2]` это подпоследовательность; срез за границами не падает, а возвращает пустой результат.
- **Не у всех коллекций есть индексы**. У `set` и обычного `dict` нет позиционного доступа `[i]`; у словаря `d[k]` это доступ по ключу, а не по позиции.
