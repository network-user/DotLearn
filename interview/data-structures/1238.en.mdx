# In what ways does a hash table resolve key collisions?

## In short

Two basic families of methods: chaining (elements with the same hash are stored in a list next to the slot) and open addressing (everything is kept in the array itself, and a free slot is found by probing - linear, quadratic, or double hashing).

## How to answer in an interview

"A collision is when two different keys map to the same index. It is resolved in two ways. The first is chaining: each slot leads to a linked list (or tree), and colliding keys are appended there; simple and resilient to high occupancy. The second is open addressing: everything sits right in the array, and on a collision the next free slot is found by a deterministic scheme. Probing schemes: linear (step +1), quadratic (the step grows quadratically), and double hashing (the step is set by a second hash function). Chaining is simpler and tolerates a high load factor; open addressing is friendlier to the cache but suffers from clustering."

**Chaining.** An array slot is the head of a list. On a collision the new element is appended to the right list, and the search runs through it linearly.

```python
class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]

    def insert(self, key, value):
        index = hash(key) % self.size
        bucket = self.table[index]
        for i, (k, _) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        bucket.append((key, value))

    def get(self, key):
        index = hash(key) % self.size
        for k, v in self.table[index]:
            if k == key:
                return v
        return None
```

**Open addressing.** The data sits in the array itself; if a slot is occupied, the next one is found according to the probing scheme.

```python
class OpenAddressingHashTable:
    def __init__(self, size):
        self.size = size
        self.table = [None] * size

    def insert(self, key, value):
        index = hash(key) % self.size
        while self.table[index] is not None and self.table[index][0] != key:
            index = (index + 1) % self.size
        self.table[index] = (key, value)

    def get(self, key):
        index = hash(key) % self.size
        while self.table[index] is not None:
            if self.table[index][0] == key:
                return self.table[index][1]
            index = (index + 1) % self.size
        return None
```

## Pitfalls

- **Clustering** with linear probing: occupied slots stick together into long runs, and probing gets longer. Quadratic and double hashing soften this.
- With open addressing, **deletion is harder**: you can't simply clear a slot, otherwise the probing chain for other keys breaks. You need "tombstones".
- With chaining, a poor hash function can let one list grow to O(n) - access degrades. In CPython the buckets are not lists but part of a single open-addressing array.
- The **load factor** is critical for both methods: above a threshold the table must be grown and rehashed, otherwise the number of collisions rises.
- In the naive open addressing from the example, a reserve of free slots is mandatory, otherwise insertion will loop forever on a full array.
