# Build your own list: how a dynamic array works under the hood

## In short

Your own `list` is written on top of an ordinary fixed-size array. We keep capacity and the real number of elements separately; while there is headroom, we put an element into a free slot, and once the headroom runs out, we allocate a larger array, move the data there, and keep going. That is exactly the dynamic array on which the built-in `list` is built.

## How to answer in an interview

`list` in Python is a dynamic array: elements lie contiguously in memory, and the block resizes as it grows. To reproduce it by hand, you need three things: a fixed-size buffer, a counter of occupied slots (size), and a declared capacity (capacity). On `append` I check whether there is room; if not, I double the capacity, copy the old elements into a new buffer, and only then insert. Deletion by index shifts the tail to the left. Access by index is O(1), appending at the end is amortized O(1), and deletion from the middle is O(n).

1. **Initialization.** We set up a buffer for a small number of elements - the starting capacity.
2. **Adding.** Before inserting, we compare size and capacity. If there is room, we put it in O(1). If there is no room, we expand first.
3. **Expansion.** We create a buffer twice as large, move the elements, and switch to it. A one-off O(n) operation, but because of doubling it amortizes to a constant per insertion.
4. **Deletion by index.** We shift all elements to the right of it one position to the left, clear the tail slot, and decrease size - this is O(n).

```python
class CustomList:
    def __init__(self):
        self.capacity = 4
        self.size = 0
        self.data = [None] * self.capacity

    def append(self, item):
        if self.size == self.capacity:
            self._resize(2 * self.capacity)
        self.data[self.size] = item
        self.size += 1

    def _resize(self, new_capacity):
        new_data = [None] * new_capacity
        for i in range(self.size):
            new_data[i] = self.data[i]
        self.data = new_data
        self.capacity = new_capacity

    def remove(self, index):
        if 0 <= index < self.size:
            for i in range(index, self.size - 1):
                self.data[i] = self.data[i + 1]
            self.size -= 1
            self.data[self.size] = None
        else:
            raise IndexError("Index out of range")

    def __getitem__(self, index):
        if 0 <= index < self.size:
            return self.data[index]
        raise IndexError("Index out of range")

    def __len__(self):
        return self.size
```

- `__init__` - a buffer with a starting capacity of 4.
- `append` - appending at the end with expansion when there is not enough room.
- `_resize` - a new buffer and data transfer.
- `remove` - deletion by index with a tail shift.
- `__getitem__` / `__len__` - access by index and the current size.

## Pitfalls

- **Do not confuse size and capacity.** The list length for the user is `size`, not the buffer size. `__len__` must return size, otherwise the `None` slots will leak out.
- **Clearing the tail after `remove` is important**, otherwise a dangling reference to the deleted object will remain in the last position - a memory leak and an incorrect result on a future expansion.
- **Check index bounds** in both `remove` and `__getitem__`: without a check, accessing a foreign buffer slot will return garbage or `None` instead of an honest error.
- **Shrinking is not implemented here.** After many deletions the buffer stays large; a production implementation adds shrinking at a low load factor.
