# How do you drop duplicates from a list, and when does the original order matter?

## In short

Deduplication comes down to the choice of data structure: a set kills duplicates instantly but destroys order; a dictionary (or a manual pass) keeps order at the cost of slightly more code.

## How to answer in an interview

If order doesn't matter - `list(set(items))` is the shortest and fastest route, because membership checks in a set run in O(1). If order matters, since Python 3.7 I use `list(dict.fromkeys(items))`: a regular dict guarantees insertion order, and duplicates are cut off as identical keys. A manual loop with `if x not in result` also preserves order, but it's O(n^2) and only suitable for short lists.

## How it works

Via a set - duplicates are dropped thanks to the uniqueness of elements in a `set`:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = list(set(items))
```

Order won't be preserved: a set is unordered.

When order needs to be kept, a dictionary keyed by elements works best - it's O(n) and remembers insertion order:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = list(dict.fromkeys(items))
# unique == [1, 2, 3, 4, 5]
```

A manual pass is the clear but slow option: each `not in` check scans the accumulated list:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = []
for item in items:
    if item not in unique:
        unique.append(item)
```

Before Python 3.7, order was guaranteed only by `collections.OrderedDict.fromkeys(items)`; now a regular `dict` does the same thing.

## Pitfalls

- All three set/dict-based approaches require the elements to be hashable. A list of lists (`[[1], [2]]`) passed through `set` fails with `TypeError: unhashable type`.
- `list(set(...))` doesn't just lose order - it's unpredictable across runs for strings due to hash randomization; don't rely on it for reproducible output.
- The manual loop with `not in` has quadratic complexity; on tens of thousands of elements it slows down noticeably compared to `dict.fromkeys`.
- A set compares by equality and hash: `1`, `1.0`, and `True` collapse into a single element, because they are equal and have the same hash.
