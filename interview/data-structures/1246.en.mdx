# Indexing in Python: how do you get an element by its position?

## In short

Access to a sequence element in Python goes through square brackets: `seq[i]`. Numbering starts at zero, and negative indices address elements from the end - `seq[-1]` is the last element.

## How to answer in an interview

In Python you access a sequence element by index using `[]`. Forward numbering starts at 0, so the first element is `seq[0]`, and for a list of length n the last is `seq[n-1]`. Python's convenience is in negative indices: `seq[-1]` is always the last, `seq[-2]` the second to last, and there is no need to compute the length by hand. Indexing works the same for lists, tuples, and strings, because they are all ordered sequences. For lists and tuples, access by index is O(1): under the hood is an array of pointers, and the position is computed by arithmetic. Going out of bounds raises an `IndexError`, and that is important to remember when working with user data.

## How it works

An index is an offset from the start. For a list of three elements, indices 0, 1, 2 are valid:

```python
my_list = ['apple', 'banana', 'cherry']
my_list[0]   # 'apple'
my_list[1]   # 'banana'
my_list[2]   # 'cherry'
```

Negative indices count from the end, where `-1` corresponds to position `len - 1`:

```python
my_list[-1]  # 'cherry'
my_list[-2]  # 'banana'
my_list[-3]  # 'apple'
```

The same works for strings and tuples, since they are sequences too:

```python
"hello"[0]    # 'h'
"hello"[-1]   # 'o'
(10, 20, 30)[1]  # 20
```

## Pitfalls

- **`IndexError` out of bounds**. `my_list[3]` for a list of three elements will fail; the index must be in the range `-len .. len-1`.
- **Indexing a string yields a string, not a character**. Python has no separate char type: `"hello"[0]` is a string of length 1.
- **Do not confuse an index with a slice**. `seq[1]` is an element, while `seq[1:2]` is a subsequence; a slice out of bounds does not fail but returns an empty result.
- **Not every collection has indices**. A `set` and a plain `dict` have no positional access `[i]`; for a dictionary, `d[k]` is access by key, not by position.
