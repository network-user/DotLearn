# How to remove duplicates from a list, and when order matters

## In short

If order doesn't matter, run the list through `set`. If order matters, use `dict.fromkeys`, which removes duplicates but remembers the first appearance of each element.

## How to answer in an interview

The shortest way is `list(set(data))`, but `set` does not preserve order, so the result may get shuffled. When order needs to be preserved, I use `list(dict.fromkeys(data))`: since Python 3.7 dictionaries keep insertion order, and keys are unique by definition. Both options work in O(n) and require the elements to be hashable.

## How it works

A set stores only unique values, so duplicates are discarded at insertion time:

```python
data = [1, 2, 2, 3, 4, 4, 5]
unique = list(set(data))
print(unique)  # order not guaranteed, for example [1, 2, 3, 4, 5]
```

`dict.fromkeys` builds a dictionary where the list's elements become keys. A repeated key does not create a new entry, and the order of first appearance is preserved:

```python
data = [3, 1, 2, 1, 3]
unique = list(dict.fromkeys(data))
print(unique)  # [3, 1, 2] — order of first appearance
```

If the elements are not hashable (for example, nested lists), neither approach works, and you have to track what you've seen manually:

```python
seen = []
for item in [[1], [1], [2]]:
    if item not in seen:
        seen.append(item)
print(seen)  # [[1], [2]]
```

## Pitfalls

- `set` does not preserve order, and for the same data it may look stable, but you can't rely on that.
- Both fast approaches require hashable elements: you can't put a `list` or a `dict` inside.
- The manual approach using `if item not in seen` for lists runs in O(n^2) — on large data this is slow.
- `dict.fromkeys` assigns `None` to all keys; we only need the keys, so we wrap it in `list(...)`.
