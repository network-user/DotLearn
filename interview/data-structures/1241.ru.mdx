# Массив или связный список: чем платим за быстрый доступ и за быструю вставку?

## Суть

Массив кладёт элементы подряд в памяти и даёт мгновенный доступ по индексу, но расплачивается дорогими вставками в середину. Связный список соединяет узлы ссылками, поэтому вставка и удаление при готовой ссылке дёшевы, зато добраться до n-го элемента можно только обходом.

## Что ответить на собесе

Это компромисс между скоростью доступа и стоимостью изменений.

- Массив хранит данные в непрерывном блоке, адрес элемента вычисляется по индексу - чтение `O(1)`, но вставка/удаление в середине требует сдвига хвоста, это `O(n)`.
- Связный список - цепочка узлов; добавить или удалить узел, имея ссылку на соседа, - `O(1)`, зато доступ по индексу - `O(n)`, потому что приходится идти от головы.
- Список тратит дополнительную память на указатели, а массив укладывается компактнее и дружелюбнее к кэшу процессора.
- Кстати, `list` в Python - это динамический массив, а не связный список.

Цена за это - перестройка при изменениях в середине: чтобы вставить элемент, всё, что правее, нужно сдвинуть.

Связный список не требует непрерывного блока. Каждый узел - отдельная ячейка со ссылкой на следующего:

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        node = Node(data)
        if self.head is None:
            self.head = node
            return
        current = self.head
        while current.next is not None:
            current = current.next
        current.next = node
```

Вставка в голову здесь делается за `O(1)` - достаточно переставить ссылку. А вот поиск n-го элемента всегда идёт по цепочке от `head`.

```
массив:        [10][20][30][40]      <- индекс даёт адрес мгновенно
связный список: 10 -> 20 -> 30 -> 40  <- к 30 идём по ссылкам от головы
```

## Подводные камни

- "Связный список даёт O(1) вставку" - только если у вас уже есть ссылка на нужное место. Поиск этого места - всё те же `O(n)`.
- На практике массив часто обгоняет список даже там, где теория обещает обратное: непрерывная память дружит с кэшем процессора, а прыжки по указателям - нет.
- В Python `list` - динамический массив; для настоящего двусвязного списка с дешёвыми операциями по краям берут `collections.deque`.
- `append` в конец списка Python амортизированно `O(1)`, но изредка вызывает копирование всего массива при росте ёмкости - это уже `O(n)` на конкретной вставке.
