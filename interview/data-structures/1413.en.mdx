# What set of operations is available on mappings in Python?

## In short

A mapping (in Python this is the dictionary) is a "key → value" structure. On top of it work insertion, update, deletion, membership check, access by key, and three ways to traverse it: by keys, by values, and by pairs.

## How to answer in an interview

A dictionary supports the full set of mapping operations: writing by key (`d[k] = v`) both adds and updates at once, `del d[k]` and `d.pop(k)` delete, the `in` operator checks for the presence of a key in O(1), and the methods `keys()`, `values()`, `items()` return view objects for iteration. An important detail: all the basic operations - insertion, deletion, lookup - are O(1) on average thanks to the hash table under the hood.

## How it works

Insertion and update are one and the same assignment-by-key operation:

```python
d = {}
d['key1'] = 'value1'
d['key1'] = 'new_value'
```

If the key was absent, a record is created; if it was present, the value is overwritten.

Deletion:

```python
del d['key1']
value = d.pop('key2', None)
```

`del` will raise a `KeyError` if the key is missing, while `pop` with a default is safe and also returns the deleted value.

Membership check and traversal:

```python
exists = 'key1' in d

for key in d:
    print(key)
for value in d.values():
    print(value)
for key, value in d.items():
    print(key, value)
```

`keys()`, `values()`, and `items()` return not lists but dynamic views, which reflect changes to the dictionary and at the same time are easily turned into a list via `list()`.

## Pitfalls

- Accessing a missing key through `d[k]` raises a `KeyError`; for soft access use `d.get(k)` or `d.setdefault(k, default)`.
- View objects are "live": if you modify the dictionary while iterating over `items()`, you will get a `RuntimeError`.
- Only a hashable (immutable) object can be a key - a list will not become a key, while a tuple of immutable elements will.
- Since Python 3.7 insertion order is preserved, but you should not rely on a dictionary as an ordered structure for sorting - this is an implementation detail concerning the order of addition, not sorting.
