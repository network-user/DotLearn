# Как отсортировать список словарей по значению одного из полей?

## Суть

Передайте в `sorted()` параметр `key` - функцию, которая для каждого словаря возвращает значение нужного поля. Классика: `sorted(data, key=lambda d: d['field'])`.

## Что ответить на собесе

Через `sorted()` с аргументом `key`: `sorted(people, key=lambda x: x['age'])`.

- Функция `key` вызывается один раз для каждого элемента и определяет, по какому значению сравнивать.
- Для разворота добавляют `reverse=True`, для сортировки по нескольким полям из `key` возвращают кортеж.
- Альтернатива лямбде - `operator.itemgetter('age')`, она чуть быстрее и читается чище.
- Сортировка стабильна, поэтому элементы с равным ключом сохраняют исходный порядок.

## Как это работает

`sorted()` строит новый список, не трогая исходный. Параметр `key=lambda x: x['age']` говорит: сравнивать словари по значению поля `'age'`. Результат - `Bob (25)`, `Alice (30)`, `Charlie (35)`.

Несколько критериев и обратный порядок:

```python
sorted(people, key=lambda x: x['age'], reverse=True)
sorted(people, key=lambda x: (x['age'], x['name']))
```

Второй вызов сортирует сначала по возрасту, а при равном возрасте - по имени, потому что кортежи сравниваются поэлементно слева направо.

## Подводные камни

- `sorted()` возвращает новый список; чтобы менять список на месте, есть `list.sort()` с тем же `key`.
- Если поле есть не во всех словарях, лямбда с `x['field']` упадёт с `KeyError`; безопаснее `x.get('field', default)`.
- Нельзя сравнивать значения разных несравнимых типов (например, число и строка под одним ключом) - будет `TypeError`.
- `reverse=True` не ломает стабильность: при равных ключах относительный порядок сохраняется, а не зеркалится.
