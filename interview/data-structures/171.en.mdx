# Binary tree: how is it structured and what is it for?

## In short

A binary tree is a hierarchical structure where each node has at most two children: left and right. It's the basis for search trees and heaps, which provide fast search, insertion, and removal.

## How to answer in an interview

A binary tree is nodes, each of which has at most two children. A node without a parent is the root; nodes without children are leaves. The structure itself is a foundation: binary search trees (BST) are built on it, where values to the left of a node are smaller and to the right are larger, as well as heaps for priority queues. In a balanced BST, search, insertion, and removal run in O(log n), because at each step we cut off half the tree.

## How it works

Each node has up to two children: left and right. A node without children is a leaf, the single node without a parent is the root.

In a **binary search tree (BST)**, an invariant holds for each node: all values in the left subtree are smaller than the node's value, and in the right subtree are larger. This is what gives fast search.

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

def insert(root, value):
    if root is None:
        return Node(value)
    if value < root.value:
        root.left = insert(root.left, value)
    else:
        root.right = insert(root.right, value)
    return root

def search(root, target):
    if root is None or root.value == target:
        return root
    if target < root.value:
        return search(root.left, target)
    return search(root.right, target)
```

Operations in a BST:

- **Search:** from the root we go left if the target is smaller than the node, and right if it's larger, until a match or an empty spot.
- **Insertion:** we descend by the same rules and hang the new node in the found empty position.
- **Removal:** we handle three cases: a leaf, a node with one child, and a node with two; in the last one the node is replaced by the minimum element of the right subtree.

Binary trees are used in implementations of sets and dictionaries, in heaps for heapsort and priority queues, and also in expression trees in compilers.

## Pitfalls

- O(log n) for a BST holds only for a balanced tree. If you insert sorted data, the tree degenerates into a list, and operations become O(n). Self-balancing trees (AVL, red-black) save you from this.
- Don't confuse a plain binary tree with a search tree: the "smaller on the left, larger on the right" invariant exists only in a BST.
- A recursive traversal of a deep tree can hit Python's recursion limit; sometimes you need an iterative traversal with a stack.
- Removing a node with two children is the most common source of errors: it's important to correctly choose the replacement (the minimum of the right subtree or the maximum of the left).
