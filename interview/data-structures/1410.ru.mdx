# Какой набор операций даёт тип set для работы с уникальными элементами?

## Суть

`set` реализует классические операции над множествами: объединение, пересечение, разность, симметрическую разность плюс проверки подмножества и надмножества. Этого набора достаточно для большинства задач на уникальные коллекции.

## Что ответить на собесе

Множество поддерживает теоретико-множественные операции, каждая доступна и оператором, и одноимённым методом.

- Объединение `|` / `union`, пересечение `&` / `intersection`, разность `-` / `difference`, симметрическая разность `^` / `symmetric_difference`.
- Отношения проверяются операторами сравнения: `<=` / `issubset` и `>=` / `issuperset`.
- Поскольку элементы хешируются, все эти операции работают быстро, а сами элементы обязаны быть хешируемыми.

```python
set_a = {1, 2, 3}
set_b = {3, 4, 5}

set_a | set_b   # объединение: {1, 2, 3, 4, 5}
set_a & set_b   # пересечение: {3}
set_a - set_b   # разность: {1, 2}
set_a ^ set_b   # симметрическая разность: {1, 2, 4, 5}
```

Отношения между множествами:

```python
set_c = {1, 2}
set_c <= set_a   # подмножество: True
set_a >= set_c   # надмножество: True
```

Операторная форма (`|`, `&`, ...) требует, чтобы оба операнда были множествами, а методы (`union`, `intersection`, ...) принимают любой итерируемый аргумент - это удобнее, когда второй операнд, например, список.

## Подводные камни

- **Операторы строже методов.** `set_a | [1, 2]` поднимет `TypeError`, а `set_a.union([1, 2])` отработает - метод принимает любой iterable.
- **Разность не симметрична.** `set_a - set_b` и `set_b - set_a` дают разные результаты; не путайте с `^`, которая симметрична.
- **Множество неупорядочено.** Порядок вывода элементов не определён, полагаться на него нельзя; для упорядоченного результата сортируйте отдельно.
- **Элементы должны быть хешируемыми.** Положить список внутрь множества не выйдет; для вложенных неизменяемых наборов используйте `frozenset`.
- **`{}` - это словарь, а не множество.** Пустое множество создаётся только через `set()`.
