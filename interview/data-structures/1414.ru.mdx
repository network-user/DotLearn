# Как склеить два списка - ключи и значения - в один словарь?

## Суть

Спарьте элементы двух списков функцией `zip()` и оберните результат в `dict()`. Всё умещается в одну строку: `dict(zip(keys, values))`.

## Что ответить на собесе

Идиоматичный способ - `dict(zip(keys, values))`, читается за секунду и работает за O(n).

- `zip` идёт по обоим спискам параллельно и выдаёт кортежи `(ключ, значение)`.
- `dict` собирает из этих пар словарь.
- Нюанс, который любят спрашивать: `zip` останавливается на самом коротком списке, поэтому при разной длине хвост длинного потеряется без ошибки.

## Как это работает

```python
keys = ['name', 'age', 'city']
values = ['Alice', 30, 'New York']

result = dict(zip(keys, values))
print(result)
```

Пошагово:

1. `zip(keys, values)` создаёт ленивый итератор пар: `('name', 'Alice')`, `('age', 30)`, `('city', 'New York')`.
2. `dict(...)` проходит по этим парам и строит из них словарь.
3. На выходе: `{'name': 'Alice', 'age': 30, 'city': 'New York'}`.

Тот же результат даёт dict-comprehension, если по дороге нужно что-то преобразовать: `{k: v for k, v in zip(keys, values)}`.

## Подводные камни

- При разной длине списков `zip` молча обрежет результат по короткому. Если нужно поймать рассинхрон - используйте `zip(keys, values, strict=True)` (Python 3.10+), он бросит `ValueError`.
- Дубликаты в `keys` схлопнутся: последнее значение перезапишет предыдущие, словарь хранит ключ один раз.
- Ключи должны быть хешируемыми - список в роли ключа даст `TypeError`.
- `zip` возвращает одноразовый итератор: пройдя по нему один раз, второй раз вы получите пустоту.
