# Чем выкинуть повторы из списка и когда важен исходный порядок?

## Суть

Дедупликацию решает выбор структуры данных: множество убивает повторы мгновенно, но рушит порядок; словарь (или ручной проход) держит порядок ценой чуть большего кода.

## Что ответить на собесе

Без порядка - `list(set(items))`, с порядком - `list(dict.fromkeys(items))`.

- `list(set(items))` - самый короткий и быстрый путь, проверка членства в множестве работает за O(1).
- Когда порядок важен, начиная с Python 3.7 беру `list(dict.fromkeys(items))`: словарь гарантирует порядок вставки, дубли отсекаются как одинаковые ключи.
- Ручной цикл с `if x not in result` тоже даёт порядок, но он O(n^2) и подходит только для коротких списков.

## Как это работает

Через множество - повторы отбрасываются за счёт уникальности элементов в `set`:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = list(set(items))
```

Порядок не сохранится: множество не упорядочено.

С сохранением порядка лучше всего работает словарь по ключам - он O(n) и помнит порядок вставки:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = list(dict.fromkeys(items))
# unique == [1, 2, 3, 4, 5]
```

Ручной проход - наглядный, но медленный вариант: каждая проверка `not in` перебирает накопленный список:

```python
items = [1, 2, 2, 3, 4, 4, 5]
unique = []
for item in items:
    if item not in unique:
        unique.append(item)
```

До Python 3.7 порядок гарантировал только `collections.OrderedDict.fromkeys(items)`; сейчас обычный `dict` делает то же самое.

## Подводные камни

- Все три способа через множество/словарь требуют, чтобы элементы были хешируемыми. Список из списков (`[[1], [2]]`) через `set` упадёт с `TypeError: unhashable type`.
- `list(set(...))` не просто теряет порядок - он непредсказуем между запусками для строк из-за рандомизации хеша, не полагайтесь на него для воспроизводимого вывода.
- Ручной цикл с `not in` имеет квадратичную сложность; на десятках тысяч элементов он заметно тормозит по сравнению с `dict.fromkeys`.
- Множество сравнивает по равенству и хешу: `1`, `1.0` и `True` схлопнутся в один элемент, потому что они равны и имеют одинаковый хеш.
