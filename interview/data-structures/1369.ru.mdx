# Как проверить, что нужное значение встречается среди значений словаря?

## Суть

Оператор `in` для словаря ищет по ключам. Чтобы проверить значения, обращайтесь к представлению `dict.values()`: `target in my_dict.values()`.

## Что ответить на собесе

Оператор `in` для словаря ищет по ключам, а для поиска по значениям нужен `dict.values()`.

- `value in my_dict` проверяет наличие ключа, а не значения - это надо помнить.
- Для поиска по значениям берут `value in my_dict.values()`.
- Поиск ключа в среднем O(1): под капотом хеш-таблица.
- Перебор `values()` линейный, O(n), - это последовательное сравнение всех значений.

## Как это работает

`values()` возвращает динамическое представление (view) значений словаря. Оператор `in` проходит по нему и сравнивает каждое значение с искомым.

<PyDemo title="Поиск по ключам и по значениям" autoRun code={`prices = {'apple': 1, 'banana': 2, 'orange': 3}

print(2 in prices)            # False - 2 нет среди ключей
print(2 in prices.values())   # True  - 2 есть среди значений
print('apple' in prices)      # True  - проверка по ключу`} />

Если значения нужно проверять часто, а словарь большой, стоит держать обратный индекс - множество значений `set(prices.values())`, тогда проверка снова станет O(1).

## Подводные камни

- Частая ошибка - писать `value in my_dict`, ожидая поиск по значениям; на самом деле так проверяется ключ.
- `values()` - это view, а не список: он отражает изменения словаря, но повторно «перемотать» его как итератор нельзя без нового вызова.
- Поиск по `values()` линейный. Для частых проверок кешируйте значения в `set`.
- Сравнение идёт по `==`, поэтому `1 in {'a': True}.values()` вернёт `True`, ведь `True == 1`.
