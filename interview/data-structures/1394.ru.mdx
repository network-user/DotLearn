# frozenset: когда нужно неизменяемое множество и что это даёт?

## Суть

`frozenset` - замороженная версия `set`: тот же набор уникальных элементов, но изменить его после создания нельзя. Эта неизменяемость делает его хешируемым, поэтому `frozenset` можно класть туда, куда обычный `set` не пускают.

## Что ответить на собесе

`frozenset` нужен, когда требуется множество, которое одновременно хешируемо.

- Обычный `set` изменяем и потому нехешируем - его нельзя сделать ключом словаря или элементом другого множества.
- `frozenset` это умеет: у него нет `add`/`remove`, его хеш стабилен, значит, он подходит как ключ или элемент вложенного множества.
- Все операции чтения (объединение, пересечение, проверка наличия) работают как у обычного множества.

## Как это работает

`frozenset` поддерживает все операции, которые не меняют содержимое: `in`, `union`, `intersection`, итерацию. Методов вроде `add` и `remove` у него нет.

```python
immutable = frozenset([1, 2, 3, 4])

# immutable.add(5)
# AttributeError: 'frozenset' object has no attribute 'add'

# frozenset как ключ словаря - так нельзя с обычным set
by_group = {frozenset([1, 2, 3]): "value"}
print(by_group[frozenset([1, 2, 3])])  # value

# frozenset как элемент другого множества
nested = {frozenset([1, 2]), frozenset([3, 4])}
```

Ключевая логика: словарь и множество требуют хешируемых элементов, а хешируемость предполагает неизменяемость. Обычный `set` менять можно, значит, его хеш был бы нестабилен - Python такой объект не хеширует. У `frozenset` содержимое зафиксировано, хеш стабилен, и он спокойно живёт ключом или вложенным элементом.

## Подводные камни

- `frozenset` неизменяем только сам по себе. Если внутри лежат изменяемые объекты - а их там и не может быть, элементы тоже должны быть хешируемыми, - так что внутрь попадают только неизменяемые значения.
- Операции вроде `union` не меняют исходный `frozenset`, а возвращают новый. Ждать мутации на месте бессмысленно.
- Создание `frozenset` стоит как создание множества: дедупликация и хеширование элементов. Для горячего пути это не бесплатно.
- Два `frozenset` с одинаковыми элементами равны и имеют одинаковый хеш независимо от порядка - на это и опирается использование их как ключей.
