# Which basic sorting algorithms should you know, and how do they differ?

## In short

The minimum decent set: bubble, insertion, and selection sort (all O(n²)), plus quicksort, merge sort, and heapsort (O(n log n)). They differ in speed, memory consumption, and stability.

## How to answer in an interview

The simple quadratic sorts (bubble, insertion, selection) take a couple of lines to write, but they're unusable on large data because of O(n²). The efficient O(n log n) ones: quicksort (fast in practice, in-place, but its worst case is O(n²) with a bad pivot), merge sort (stable, guaranteed O(n log n), but requires O(n) extra memory), and heapsort (in-place, guaranteed O(n log n), unstable). In real code you call `sorted()` / `list.sort()`: that's Timsort, a hybrid of merge and insertion sort, stable and O(n log n).

## How it works

The quadratic methods. Bubble sort runs adjacent pairs until fully ordered, insertion sort grows a sorted prefix, selection sort finds the minimum and puts it at the front:

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

Quicksort is "divide and conquer" around a pivot element:

```python
def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[0]
    less = [x for x in arr[1:] if x <= pivot]
    greater = [x for x in arr[1:] if x > pivot]
    return quick_sort(less) + [pivot] + quick_sort(greater)
```

Merge sort splits the array in half, sorts the halves, and merges them:

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    merged, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            merged.append(left[i]); i += 1
        else:
            merged.append(right[j]); j += 1
    merged.extend(left[i:])
    merged.extend(right[j:])
    return merged
```

Heapsort builds a max-heap and removes the maximum from the top one at a time, restoring the heap each time.

## Pitfalls

- Quicksort with "first element" as the pivot degrades to O(n²) on already-sorted data; in practice people take the median of three or a random pivot.
- Stability matters when you sort by a key while preserving the order of equal items: merge sort and Timsort are stable, quicksort and heapsort are not.
- Merge sort requires O(n) extra memory, whereas heapsort and quicksort work almost in place.
- In practice the right answer is almost always the built-in `sorted()`; you write your own sort only for learning purposes or exotic constraints.
