# Хеш-таблица изнутри: как ключ превращается в адрес

## Суть

Хеш-таблица — это массив, в котором позицию для каждой пары ключ-значение подсказывает хеш-функция. За счёт прямого вычисления индекса вставка, поиск и удаление идут за амортизированное O(1).

## Что ответить на собесе

Хеш-таблица хранит пары ключ-значение поверх обычного массива, а позицию подсказывает хеш-функция.

- Хеш-функция переводит ключ в число, остаток от деления на размер таблицы даёт индекс бакета.
- Когда два ключа попадают в один бакет — это коллизия; её разрешают цепочками (в бакете список пар) или открытой адресацией (ищут следующую свободную ячейку).
- Средняя сложность операций O(1), но в худшем случае при плохой хеш-функции вырождается в O(n).

## Как это работает

Три кирпича: хеш-функция, массив бакетов и стратегия разрешения коллизий. Ниже — учебная реализация с цепочками: каждый бакет это список, куда складываются пары при совпадении индексов.

```python
class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]

    def _index(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        bucket = self.table[self._index(key)]
        for i, (k, _) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        bucket.append((key, value))

    def search(self, key):
        for k, v in self.table[self._index(key)]:
            if k == key:
                return v
        return None
```

При вставке мы не просто добавляем пару, а сначала проверяем, нет ли уже такого ключа — иначе один ключ хранился бы дважды. Поиск пробегает только по нужному бакету, а не по всей таблице.

## Подводные камни

- Ключ обязан быть хешируемым и неизменяемым по хешу: список ключом быть не может, кортеж — может. Изменяемый объект как ключ ломает связь «значение → бакет».
- Плохое распределение хешей сводит все пары в один бакет, и O(1) превращается в O(n) — поэтому важны рост таблицы и рехеширование при заполнении.
- `hash()` в CPython рандомизирован между запусками для строк (защита от hash-flooding), поэтому порядок обхода и конкретные индексы воспроизводить нельзя.
