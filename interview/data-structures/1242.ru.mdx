# Во что обходится перебор всех элементов списка по времени?

## Суть

Полный обход списка из n элементов стоит O(n): каждый элемент посещается ровно один раз, поэтому время растёт линейно с длиной списка.

## Что ответить на собесе

Обход списка - это `O(n)`, где n - число элементов: цикл проходит по всем элементам по одному разу.

- Удвоили длину списка - примерно удвоилось и время.
- Важно не путать сложность обхода со сложностью того, что делается внутри цикла: операция за `O(k)` на итерации даёт общую `O(n*k)`.
- Например, `if x in other_list` внутри цикла превращает безобидный обход в `O(n*m)`.
- Сам по себе проход по списку, без вложенной тяжёлой работы, - это честные `O(n)`.

## Как это работает

Линейная сложность означает, что время пропорционально размеру входа: список из 10 элементов обойдётся примерно в 10 раз дольше, чем из одного. В Python обход обычно делают циклом `for`:

```python
my_list = [1, 2, 3, 4, 5]

for element in my_list:
    print(element)
```

Каждый элемент посещается один раз, лишних проходов нет. Тот же O(n) у типичных задач свёртки - например, суммирования:

## Подводные камни

- Вложенный цикл по тому же списку - это уже `O(n^2)`, а не `O(n)`. Классическая ловушка - проверка дубликатов перебором всех пар.
- Оператор `in` для списка сам по себе `O(n)`. Внутри цикла он даёт `O(n*m)`; если нужны частые проверки принадлежности, берите `set` с `O(1)`.
- `len(list)` в Python - это `O(1)`, длина хранится готовой, так что вызывать его в условии цикла не страшно.
- Срез или копирование списка (`list[:]`, `list.copy()`) - тоже `O(n)`: незаметная копия внутри цикла легко раздувает сложность.
