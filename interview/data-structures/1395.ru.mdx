# Хешируемый объект своими руками: контракт __hash__ и __eq__

## Суть

Чтобы экземпляр класса можно было класть в `set` или брать ключом `dict`, классу нужны два согласованных метода: `__hash__()` возвращает стабильное целое, а `__eq__()` задаёт равенство. И сам объект стоит делать неизменяемым - меняющийся хеш ломает хеш-таблицы.

## Что ответить на собесе

Хешируемость держится на паре `__hash__` и `__eq__`, связанных контрактом: если два объекта равны по `__eq__`, их хеши обязаны совпадать. Обратное не требуется - разные объекты могут иметь одинаковый хеш, это нормальная коллизия. Хеш должен быть постоянным на всё время жизни объекта, поэтому поля, которые в нём участвуют, не должны меняться. Удобный приём - считать хеш от кортежа тех же полей, что сравниваются в `__eq__`. И железное правило: переопределил `__eq__` - переопредели и `__hash__`.

## Как это работает

Возьмём точку на плоскости. В `__eq__` сравниваем координаты, в `__hash__` хешируем кортеж этих же координат.

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if isinstance(other, Point):
            return self.x == other.x and self.y == other.y
        return NotImplemented

    def __hash__(self):
        return hash((self.x, self.y))
```

Теперь точки с одинаковыми координатами равны и дают один хеш, поэтому корректно работают как ключи и элементы:

```python
seen = {Point(1, 2), Point(1, 2)}
print(len(seen))  # 1: дубликат схлопнулся
```

Кортеж в `__hash__` удобен тем, что сам считает хеш от своих элементов и гарантирует: одинаковые координаты дадут одинаковый результат. Те же поля идут и в сравнение, и в хеш - так контракт соблюдается автоматически.

## Подводные камни

- Главный инвариант: `a == b` обязано влечь `hash(a) == hash(b)`. Нарушите - объект "потеряется" в словаре: положили по одному хешу, ищете по другому.
- Если переопределить только `__eq__`, Python обнулит `__hash__` (сделает экземпляры нехешируемыми). Всегда задавайте оба.
- Хеш должен быть стабилен. Считаете его от изменяемого поля, меняете поле - и объект больше не находится в множестве. Поэтому хешируемые объекты делают неизменяемыми по этим полям.
- В `__eq__` лучше вернуть `NotImplemented` для чужого типа, а не `False`: это даёт Python шанс попробовать симметричное сравнение со второй стороны.
