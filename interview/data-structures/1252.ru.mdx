# Почему динамический массив растёт сам и чем он отличается от обычного

## Суть

Динамический массив - это коллекция в непрерывной памяти, которая сама меняет вместимость. Кончилось место - он берёт блок побольше и переезжает туда; элементов стало мало - может отдать лишнюю память обратно. Поэтому работать с ним можно, не зная итоговый размер заранее.

## Что ответить на собесе

Динамический массив хранит элементы подряд, как статический, но умеет переразмеряться на ходу.

- Когда выделенная ёмкость заполнена, он аллоцирует новый блок (как правило, в два раза больше), копирует туда старые элементы и продолжает работу.
- За счёт стратегии удвоения ёмкости вставка в конец остаётся амортизированно O(1): редкие дорогие копирования размазываются по множеству дешёвых добавлений.
- В Python такой структурой и является `list`.

## Как это работает

Ключевое отличие от статического массива - разделение двух величин: фактического числа элементов (size) и выделенной ёмкости (capacity). Capacity всегда не меньше size, а разница - это запас под будущие вставки.

1. **Старт.** Массив создаётся с небольшой начальной ёмкостью.
2. **Вставка.** Есть свободная ячейка - элемент кладётся в неё за O(1). Свободного места нет - срабатывает расширение.
3. **Расширение.** Выделяется новый блок (обычно ×2), все элементы переносятся туда, старый блок освобождается. Это разовая операция O(n), но происходит она редко.
4. **Сжатие.** При массовом удалении реализация может перейти на блок поменьше, чтобы не держать впустую память.

Из-за геометрического роста ёмкости n добавлений в сумме делают порядка n операций копирования, а не n², отсюда и амортизированная константа.

```python
dynamic_array = []

dynamic_array.append(1)
dynamic_array.append(2)
dynamic_array.append(3)

dynamic_array.pop()

print(dynamic_array)  # [1, 2]
```

## Подводные камни

- **Вставка в начало или середину - это O(n)**, а не O(1): все элементы правее сдвигаются. Амортизированная константа касается только добавления в конец.
- **Расширение не бесплатно**: отдельный `append`, попавший на границу ёмкости, выполнится за O(n). Если важна предсказуемость по времени (реалтайм), это стоит учитывать.
- **Рост капасити стоит памяти**: после удвоения до половины блока может пустовать, так что под капотом память расходуется с запасом.
- **`pop()` с конца - O(1)**, а `pop(0)` - O(n) из-за сдвига; для очереди берите `collections.deque`, а не список.
