# What does bubble sort cost in time and memory?

## In short

In time, bubble sort is quadratic: O(n^2) in the average and worst cases, O(n) on an already sorted array with early exit. In memory it is O(1), because it sorts in place.

## How to answer in an interview

"Bubble sort is a stable in-place sort. Time: average and worst case O(n^2), since two nested passes give on the order of n*n comparisons. The best case O(n) is achievable if you add a flag and break out of the loop when a pass makes no swaps - then a sorted input is processed in a single pass. Memory is O(1): apart from a couple of temporary variables, nothing is needed. In practice it isn't used because of the quadratic asymptotics, but it's handy as a teaching example."

## How it works

The algorithm repeatedly walks the array, compares neighbors, and swaps them when the order is violated. On each pass the largest of the remaining elements "bubbles up" to the end, so the region being sorted shrinks.

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr
```

The `swapped` flag is the key touch: if a full inner pass makes no swap, the array is already sorted and the outer loop breaks. This is exactly what gives O(n) in the best case.

## Pitfalls

- Without the `swapped` flag there is no O(n) best case - the algorithm always runs all n passes and stays O(n^2).
- The inner loop bound is `n - i - 1`: on the i-th pass the last i elements are already in place, so there's no need to touch them again.
- Bubble sort is **stable** (equal elements keep their original order), because it swaps only on a strict `>`. Replacing it with `>=` breaks stability.
- O(1) memory is a common trap: people confuse it with time complexity. These are different axes of evaluation.
- For real data, use merge sort or quicksort; bubble sort loses to them by orders of magnitude already at thousands of elements.
