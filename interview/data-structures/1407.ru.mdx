# Хеширование без воды: что это и зачем оно везде?

## Суть

Хеширование - это сворачивание данных любой длины в значение фиксированного размера через хеш-функцию. Один и тот же вход всегда даёт один и тот же хеш, а малейшее изменение входа кардинально меняет результат.

## Что ответить на собесе

Хеш-функция принимает на вход данные произвольной длины и возвращает строку или число фиксированной длины - хеш. Ключевых свойств у неё четыре: детерминированность (одинаковый вход даёт одинаковый выход), скорость вычисления, лавинный эффект (крошечное изменение входа сильно меняет хеш) и стойкость к коллизиям (трудно подобрать два входа с одним хешем). На практике хеширование решает три класса задач: быстрый поиск через хеш-таблицы - на этом держатся словари и множества Python; контроль целостности данных - сравнили хеш скачанного файла с эталонным и убедились, что он не побился; и хранение паролей - в базе лежат не сами пароли, а их хеши, чтобы утечка не раскрывала исходники.

## Как это работает

Хеш-функция отображает огромное (потенциально бесконечное) множество входов в конечное множество выходов фиксированной длины. Из-за этого коллизии неизбежны, но хорошая функция делает их редкими и непредсказуемыми.

```python
import hashlib

data = "Hello, World!"

hash_object = hashlib.sha256()
hash_object.update(data.encode('utf-8'))
hash_hex = hash_object.hexdigest()

print(hash_hex)
```

- `hashlib.sha256()` создаёт объект криптографической функции с хешем длиной 256 бит.
- `update(...)` принимает именно байты, поэтому строку прогоняем через `encode('utf-8')`.
- `hexdigest()` отдаёт хеш в виде шестнадцатеричной строки.

Лавинный эффект легко увидеть: меняем один символ - хеш меняется до неузнаваемости.

```python
import hashlib

print(hashlib.md5(b"cat").hexdigest())  # d077f244def8a70e5ea758bd8352fcd8
print(hashlib.md5(b"car").hexdigest())  # 0c5c8b4e7e6f... совсем другой
```

## Подводные камни

- Хеш - это не шифрование. Шифрование обратимо при наличии ключа, а криптографический хеш восстановить обратно нельзя по построению.
- Для паролей нельзя брать «голый» SHA-256: он слишком быстрый и уязвим к перебору. Нужны медленные функции с солью - bcrypt, scrypt, Argon2.
- MD5 и SHA-1 криптографически сломаны - для подписей и паролей их использовать нельзя, только для неответственных контрольных сумм.
- Встроенный `hash()` в Python для строк рандомизируется между запусками процесса (защита от hash-flooding), поэтому его нельзя сохранять на диск как стабильный идентификатор.
- Объект может быть ключом словаря, только если он хешируемый: изменяемые типы вроде `list` и `dict` хешировать нельзя.
