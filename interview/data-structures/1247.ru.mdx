# Стоимость вставки в список Python: O(1) или O(n)?

## Суть

Цена вставки зависит от позиции. `append()` в конец — амортизированные O(1), потому что список это динамический массив. `insert(i, x)` в середину или начало — O(n): все элементы правее точки вставки сдвигаются на одну позицию.

## Что ответить на собесе

Список в Python это динамический массив указателей в непрерывном блоке памяти. Поэтому доступ по индексу O(1), а вот вставка зависит от места. `append()` обычно просто пишет элемент в свободный хвост за O(1); изредка места не хватает, и Python выделяет блок побольше и копирует туда всё за O(n), но эти дорогие ресайзы происходят настолько редко, что в среднем (амортизированно) append остаётся O(1). А `insert(i, x)` в произвольную позицию вынужден сдвинуть вправо все элементы после индекса i, чтобы освободить ячейку, — это O(n). Вставка в начало `insert(0, x)` — худший случай, сдвигаются все элементы. Если нужны частые вставки в начало, я беру `collections.deque` с O(1) на обоих концах.

## Как это работает

Список переаллоцируется с запасом: ёмкость растёт не по одному, а кратно, поэтому амортизированная стоимость `append` остаётся константной.

```python
my_list = [1, 2, 3, 4, 5]

my_list.append(6)        # O(1) амортизированно: пишем в хвост
my_list.insert(2, 'a')   # O(n): сдвигаем элементы с индекса 2 вправо
# -> [1, 2, 'a', 3, 4, 5, 6]
```

При `insert(2, 'a')` элементы `3, 4, 5, 6` физически переезжают на одну ячейку правее, чтобы освободить место под `'a'`. Чем ближе к началу вставка, тем больше элементов двигается.

## Подводные камни

- **`append` это O(1), но не строго**. Отдельные вызовы при ресайзе стоят O(n); корректная формулировка — «амортизированные O(1)».
- **`insert(0, x)` — антипаттерн в цикле**. n вставок в начало дают O(n²); для очереди берите `deque.appendleft` с O(1).
- **`pop(0)` тоже O(n)** по той же причине — сдвиг всех элементов влево, тогда как `pop()` с конца это O(1).
- **Память при ресайзе**. Список держит запасную ёмкость, поэтому занимает больше, чем сумма элементов; это плата за быстрый append.
