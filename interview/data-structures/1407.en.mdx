# Hashing, no fluff: what is it and why is it everywhere?

## In short

Hashing is the collapsing of data of any length into a fixed-size value via a hash function. The same input always produces the same hash, while the slightest change to the input drastically changes the result.

## How to answer in an interview

A hash function takes data of arbitrary length as input and returns a string or number of fixed length - the hash. It has four key properties: determinism (the same input gives the same output), speed of computation, the avalanche effect (a tiny change to the input greatly changes the hash), and collision resistance (it's hard to find two inputs with the same hash). In practice, hashing solves three classes of problems: fast lookup via hash tables - this is what Python dictionaries and sets are built on; data integrity checking - you compare the hash of a downloaded file with the reference one and make sure it wasn't corrupted; and password storage - the database holds not the passwords themselves but their hashes, so that a leak doesn't reveal the originals.

## How it works

A hash function maps a huge (potentially infinite) set of inputs onto a finite set of fixed-length outputs. Because of this, collisions are inevitable, but a good function makes them rare and unpredictable.

```python
import hashlib

data = "Hello, World!"

hash_object = hashlib.sha256()
hash_object.update(data.encode('utf-8'))
hash_hex = hash_object.hexdigest()

print(hash_hex)
```

- `hashlib.sha256()` creates a cryptographic function object with a 256-bit hash.
- `update(...)` takes bytes specifically, so we run the string through `encode('utf-8')`.
- `hexdigest()` returns the hash as a hexadecimal string.

The avalanche effect is easy to see: change one character and the hash changes beyond recognition.

```python
import hashlib

print(hashlib.md5(b"cat").hexdigest())  # d077f244def8a70e5ea758bd8352fcd8
print(hashlib.md5(b"car").hexdigest())  # 0c5c8b4e7e6f... completely different
```

## Pitfalls

- A hash is not encryption. Encryption is reversible given the key, whereas a cryptographic hash cannot be reversed by design.
- For passwords you can't use a "bare" SHA-256: it is too fast and vulnerable to brute force. You need slow functions with a salt - bcrypt, scrypt, Argon2.
- MD5 and SHA-1 are cryptographically broken - they must not be used for signatures or passwords, only for non-critical checksums.
- Python's built-in `hash()` for strings is randomized between process runs (protection against hash flooding), so it must not be saved to disk as a stable identifier.
- An object can be a dictionary key only if it is hashable: mutable types like `list` and `dict` cannot be hashed.
