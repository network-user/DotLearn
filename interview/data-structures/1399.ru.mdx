# append против extend: один элемент или поэлементное вливание итерируемого

## Суть

`append(x)` добавляет в конец списка ровно один элемент `x` (даже если это список - он войдёт целиком как одна вложенная единица). `extend(iterable)` проходит по итерируемому и вливает каждый его элемент по отдельности.

## Что ответить на собесе

`append` добавляет один элемент, `extend` вливает итерируемое поэлементно.

- `append` увеличивает длину на единицу: что передали, то и стало последним элементом.
- `extend` увеличивает длину на число элементов аргумента: итерируется по нему и докладывает их по одному, фактически как `+=`.
- Поэтому `append([1,2])` даёт вложенный список, а `extend([1,2])` - два новых плоских элемента.
- Оба метода меняют список на месте и возвращают `None`.

## Как это работает

`append` - один элемент, неважно какого типа:

<PyDemo title="append добавляет ровно один элемент" autoRun code={`fruits = ['apple', 'banana']
fruits.append('orange')
print(fruits)            # ['apple', 'banana', 'orange']

fruits.append([1, 2])
print(fruits) # ['apple', 'banana', 'orange', [1, 2]]
print(len(fruits)) # 4 - вложенный список это один элемент`} />

`extend` - поэлементное вливание итерируемого:

```python
fruits = ['apple', 'banana']
fruits.extend(['orange', 'grape'])
print(fruits)            # ['apple', 'banana', 'orange', 'grape']
print(len(fruits))       # 4
```

`extend` принимает любой итерируемый объект, не только список - строку, кортеж, множество, генератор:

```python
chars = ['a']
chars.extend('xy')       # строка итерируется по символам
print(chars)             # ['a', 'x', 'y']
```

## Подводные камни

- Оба метода работают in-place и возвращают `None`. Запись `fruits = fruits.append('x')` обнулит переменную - частая ошибка.
- `lst.extend('abc')` разложит строку на символы `'a', 'b', 'c'`, а не добавит её целиком. Чтобы положить строку одним элементом, нужен `append`.
- `lst.extend(5)` упадёт с `TypeError`: число не итерируемо. `append(5)` сработает.
- `a.extend(b)` эквивалентно `a += b` (для списков), но не `a = a + b`: второе создаёт новый список, а не меняет исходный на месте.
