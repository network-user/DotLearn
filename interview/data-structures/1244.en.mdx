# Recursion versus a loop: what is the fundamental difference?

## In short

A recursive function solves a problem by calling itself on reduced input until it reaches a base case. A loop solves the same problem by repeating a block of instructions without new calls. The main difference is in cost: recursion grows the call stack, while iteration works in a fixed amount of memory.

## How to answer in an interview

Recursion and iteration are interchangeable in their result but differ in implementation and resource use. Recursion describes a solution in terms of itself and must have a base case, otherwise it runs forever and hits the stack limit. Each nested call takes a frame on the stack, so deep recursion in Python fails with a `RecursionError` at around a thousand levels. Iteration keeps state in ordinary variables and does not grow the stack, so in terms of memory and constants it is almost always more efficient. I choose recursion where the structure of the problem is tree-like (tree traversal, divide-and-conquer splitting), and iteration for linear passes and when performance matters.

## How it works

Recursion breaks a problem down into similar subproblems of smaller size. The classic example is the factorial:

```python
def factorial(n):
    if n == 0:
        return 1
    return n * factorial(n - 1)
```

The call `factorial(3)` unfolds into `3 * factorial(2)`, then `2 * factorial(1)`, and so on down to the base case `n == 0`, after which the chain collapses back. Each unfinished call hangs on the stack, waiting for the result of the nested one.

The same result iteratively - without growing the stack:

```python
def factorial(n):
    result = 1
    for i in range(1, n + 1):
        result *= i
    return result
```

Here state accumulates in a single variable `result`, and memory does not depend on `n`.

## Pitfalls

- **No base case, or an unreachable one** - infinite recursion and a `RecursionError`. Always check that the argument moves strictly toward the base case.
- **Stack depth**. CPython limits recursion by default (`sys.getrecursionlimit()` ~ 1000). For long linear tasks, recursion is unusable without rewriting it as a loop.
- **Python does not do tail-call optimization**, so even tail recursion is not automatically turned into a loop.
- **Call overhead**. Each recursive call creates a stack frame; on hot paths this is noticeably slower than a loop.
