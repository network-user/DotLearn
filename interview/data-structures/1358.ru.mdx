# Поиск N-го узла с конца односвязного списка за один проход

## Суть

Задачу решает приём двух указателей. Один указатель уходит вперёд на N узлов, затем оба двигаются синхронно. Когда ведущий упирается в конец, второй стоит ровно на N-м элементе с конца - один проход, без подсчёта длины.

## Что ответить на собесе

В односвязном списке нет доступа по индексу, поэтому решаю за один проход двумя указателями.

- В лоб "пятый с конца" потребовал бы двух проходов: сначала узнать длину, потом отсчитать.
- Развожу два указателя на нужный зазор: ведущий `fast` уходит вперёд на 5 узлов, потом двигаю `fast` и `slow` вместе по одному шагу.
- Зазор сохраняется, поэтому когда `fast` достигает конца, `slow` оказывается на пятом элементе с конца.
- Сложность - O(n) по времени и O(1) по памяти; не забываю про случай, когда узлов меньше пяти.

## Как это работает

Создаём зазор в N узлов, затем двигаем пару указателей синхронно.

    def __init__(self, data):
        self.data = data
        self.next = None

def nth_from_end(head, n):
fast = head
slow = head

    for _ in range(n):
        if fast is None:
            return None  # узлов меньше n
        fast = fast.next

    while fast is not None:
        fast = fast.next
        slow = slow.next

    return slow.data if slow else None

head = Node(1)
node = head
for value in (2, 3, 4, 5, 6, 7):
node.next = Node(value)
node = node.next

print(nth_from_end(head, 5)) # 3`} />

Ведущий `fast` сначала отрывается на `n` узлов, затем пара движется вместе. Когда `fast` выходит за конец, `slow` указывает на искомый узел.

## Подводные камни

- Если в списке меньше `n` узлов, цикл разгона должен это отловить и вернуть `None`, иначе `fast.next` упадёт на `None`.
- Граничный случай "удалить N-й с конца" решается тем же приёмом, но удобнее завести фиктивную голову (dummy), чтобы корректно удалить самый первый узел.
- Зазор должен быть именно `n`, а не `n-1`: ошибка на единицу здесь самая частая.
- Приём работает только вперёд: в односвязном списке назад вернуться нельзя.
