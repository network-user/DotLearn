# What is the time cost of iterating over every element of a list?

## In short

A full traversal of a list of n elements costs O(n): each element is visited exactly once, so the time grows linearly with the list's length.

## How to answer in an interview

Iterating over a list is `O(n)`, where n is the number of elements, because the loop walks through every element once. Double the list's length and the time roughly doubles too. The key thing here is not to confuse the complexity of the traversal itself with the complexity of what happens inside the loop: if each iteration runs an `O(k)` operation, the overall complexity becomes `O(n*k)`. For example, `if x in other_list` inside the loop turns a harmless traversal into `O(n*m)`. But the traversal on its own, without nested heavy work, is honest `O(n)`.

## How it works

Linear complexity means the time is proportional to the input size: a list of 10 elements takes about 10 times as long to traverse as a list of one. In Python, traversal is usually done with a `for` loop:

```python
my_list = [1, 2, 3, 4, 5]

for element in my_list:
    print(element)
```

Each element is visited once, with no extra passes. The same O(n) applies to typical reduction tasks - for example, summing:

```python
grades = [85, 90, 78, 92, 88]

total = 0
for grade in grades:
    total += grade

average = total / len(grades)
print(f"Средняя оценка: {average}")
```

The loop walks through the grades exactly once, so the complexity is O(n).

```
n elements  ->  n steps  ->  time grows as a straight line
```

## Pitfalls

- A nested loop over the same list is already `O(n^2)`, not `O(n)`. The classic trap is checking for duplicates by comparing all pairs.
- The `in` operator on a list is itself `O(n)`. Inside a loop it gives `O(n*m)`; if you need frequent membership checks, use a `set` with its `O(1)`.
- `len(list)` in Python is `O(1)` - the length is stored ready to use, so calling it in a loop condition is harmless.
- Slicing or copying a list (`list[:]`, `list.copy()`) is also `O(n)`: an inconspicuous copy inside a loop can easily inflate the complexity.
