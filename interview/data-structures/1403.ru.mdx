# Три способа развернуть список: где копия, где итератор, где мутация?

## Суть

Развернуть список можно срезом `[::-1]` (новая копия), функцией `reversed()` (ленивый итератор) или методом `.reverse()` (меняет список на месте). Различаются они тем, что возвращают и трогают ли оригинал.

## Что ответить на собесе

Выбираю по тому, нужна ли копия; главная ловушка - `.reverse()` мутирует оригинал, а `[::-1]` и `reversed()` его не трогают.

- `lst[::-1]` создаёт новый перевёрнутый список, оригинал не меняется.
- `reversed(lst)` возвращает ленивый итератор - оборачиваю в `list()`, если нужен список; работает с любой последовательностью и не копирует данные сразу.
- `lst.reverse()` переворачивает на месте, ничего не возвращает (`None`) и экономит память.

## Как это работает

Срез - возвращает отдельный новый список:

```python
original = [1, 2, 3, 4, 5]
reversed_copy = original[::-1]
# reversed_copy == [5, 4, 3, 2, 1], original не изменился
```

`reversed()` - отдаёт итератор, который ничего не копирует, пока его не материализуют:

```python
original = [1, 2, 3, 4, 5]
it = reversed(original)        # это итератор, не список
as_list = list(it)             # [5, 4, 3, 2, 1]
```

`.reverse()` - переворачивает сам объект и возвращает `None`:

```python
original = [1, 2, 3, 4, 5]
result = original.reverse()
# original == [5, 4, 3, 2, 1], result is None
```

Что выбрать:

- **срез** - нужна копия, оригинал трогать нельзя;
- **`reversed()`** - удобно итерировать с конца или развернуть строку/кортеж без лишнего списка;
- **`.reverse()`** - надо изменить порядок прямо в исходном списке без копии.

## Подводные камни

- `lst.reverse()` возвращает `None`. Запись `lst = lst.reverse()` сотрёт список в `None` - классическая ошибка, как и с `sort()`.
- `reversed()` отдаёт одноразовый итератор: после полного прохода он исчерпан, второй раз по нему не пройти.
- `[::-1]` копирует весь список в память; на огромных данных это дороже, чем ленивый `reversed()` или `.reverse()` на месте.
- `reversed()` требует, чтобы объект знал свою длину и поддерживал индексацию (`__len__` и `__getitem__`) либо реализовал `__reversed__`; по произвольному генератору он не пройдёт.
