# Бинарное дерево: как оно устроено и зачем нужно?

## Суть

Бинарное дерево — иерархическая структура, где у каждого узла не более двух потомков: левого и правого. На его основе строят деревья поиска и кучи, которые дают быстрый поиск, вставку и удаление.

## Что ответить на собесе

Бинарное дерево — это узлы, у каждого из которых максимум два ребёнка; это фундамент для BST и куч.

- Узел без родителя — корень, узлы без детей — листья.
- На структуре строятся бинарные деревья поиска (BST), где слева от узла значения меньше, а справа больше, и кучи для очередей с приоритетом.
- В сбалансированном BST поиск, вставка и удаление идут за O(log n), потому что на каждом шаге отсекаем половину дерева.

## Как это работает

У каждого узла есть до двух потомков — левый и правый. Узел без детей — лист, единственный узел без родителя — корень.

Здесь 1 — корень, 2 и 3 — его потомки, 4 и 5 — листья и потомки узла 2.

В **бинарном дереве поиска (BST)** для каждого узла действует инвариант: все значения в левом поддереве меньше значения узла, в правом — больше. Это и даёт быстрый поиск.

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

def insert(root, value):
    if root is None:
        return Node(value)
    if value < root.value:
        root.left = insert(root.left, value)
    else:
        root.right = insert(root.right, value)
    return root

def search(root, target):
    if root is None or root.value == target:
        return root
    if target < root.value:
        return search(root.left, target)
    return search(root.right, target)
```

Операции в BST:

- **Поиск:** от корня идём влево, если цель меньше узла, и вправо, если больше — до совпадения или до пустого места.
- **Вставка:** спускаемся по тем же правилам и подвешиваем новый узел в найденную пустую позицию.
- **Удаление:** разбираем три случая — лист, узел с одним потомком и узел с двумя; в последнем заменяют узел минимальным элементом правого поддерева.

Применяют бинарные деревья в реализациях множеств и словарей, в кучах для пирамидальной сортировки и приоритетных очередей, а ещё в деревьях выражений компиляторов.

## Подводные камни

- O(log n) у BST — только для сбалансированного дерева. Если вставлять отсортированные данные, дерево выродится в список, и операции станут O(n). От этого спасают самобалансирующиеся деревья (AVL, красно-чёрные).
- Не путайте просто бинарное дерево и дерево поиска: инвариант «слева меньше, справа больше» есть только у BST.
- Рекурсивный обход глубокого дерева может упереться в лимит рекурсии Python — иногда нужен итеративный обход со стеком.
- Удаление узла с двумя потомками — самый частый источник ошибок: важно корректно выбрать замену (минимум правого или максимум левого поддерева).
