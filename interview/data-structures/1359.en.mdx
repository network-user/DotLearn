# Merging dictionaries in Python and the fate of duplicate keys

## In short

Dictionaries are merged with the `|` operator (Python 3.9+), the `update()` method, or unpacking `{**a, **b}`. On a key collision, the value from the right (last) dictionary wins: it overwrites the left one.

## How to answer in an interview

I pick the method by whether I need a new dictionary or an in-place edit. `a | b` and `{**a, **b}` create a new dictionary without touching the originals. `a.update(b)` changes `a` in place and returns `None`. In all variants the collision rule is the same: when a key matches, the value from the dictionary that comes further to the right wins. That is, in `a | b` the common key takes its value from `b`. This is a shallow merge: nested dictionaries are not merged recursively but replaced wholesale.

## How it works

```python
a = {'x': 1, 'y': 2}
b = {'y': 3, 'z': 4}

print(a | b)          # {'x': 1, 'y': 3, 'z': 4}  - new dictionary
print({**a, **b})     # {'x': 1, 'y': 3, 'z': 4}  - same via unpacking

a.update(b)
print(a)              # {'x': 1, 'y': 3, 'z': 4}  - a changed in place
```

The key `'y'` is present in both dictionaries, so the result keeps `3`, the value from `b`. The order of the arguments determines who overwrites whom: `b | a` would give `'y': 2`.

## Pitfalls

- `update()` returns `None` and changes the dictionary in place; writing `c = a.update(b)` is a mistake, `c` will be `None`.
- The merge is shallow: for nested structures (`{'cfg': {...}}`) the inner dictionary is replaced wholesale rather than merged; for a deep merge you need a manual traversal.
- The `|` operator is available only from Python 3.9; in older versions use `{**a, **b}` or `update()`.
- Order matters: on a collision the right operand always wins, and it is easy to confuse the direction of the overwrite.
