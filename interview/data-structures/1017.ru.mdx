# За какое время хеш-таблица находит значение по ключу?

## Суть

В среднем доступ по ключу амортизированно O(1): хеш-функция сразу даёт индекс, и значение читается напрямую. В вырожденном случае, когда всё стекается в одну корзину, время вырастает до O(n).

## Что ответить на собесе

Поиск по ключу в среднем O(1): хеш-функция переводит ключ в индекс массива, и мы попадаем в нужную ячейку за константу.

- Это среднее при равномерном распределении и адекватном коэффициенте заполнения.
- Если хеш-функция плохая или все ключи дают один хеш, корзина превращается в линейную структуру, и поиск деградирует до O(n).
- Поэтому на практике важны и качество хеш-функции, и своевременное расширение таблицы.

## Как это работает

Таблица хранит пары ключ-значение поверх обычного массива. Логика доступа сводится к трём шагам:

1. **Хеширование.** Ключ прогоняется через хеш-функцию, результат сворачивается в индекс массива (обычно через остаток от деления на размер). Хорошая функция распределяет ключи равномерно.
2. **Чтение корзины.** По индексу лежит либо само значение, либо корзина с несколькими элементами, если случилась коллизия.
3. **Разрешение коллизий.** При методе цепочек корзина - это список, по которому идёт линейный поиск нужного ключа. При открытой адресации проверяются соседние ячейки по заданной стратегии пробирования.

Пример мысленной модели - телефонная книга: имя превращается в индекс, по которому хранится номер. Если два имени дали один индекс, механизм разрешения коллизий укладывает оба в одну корзину.

## Подводные камни

- O(1) - это **среднее, амортизированное** значение, а не гарантия. На собесе легко ошибиться, назвав его худшим случаем.
- Худший случай O(n) реален: подобранные ключи с одинаковым хешем (hash flooding) умышленно ломают производительность - это вектор DoS-атаки.
- При высоком коэффициенте заполнения таблица должна расширяться (rehash). Само расширение разовое O(n), но оно и держит среднюю стоимость доступа константной.
- В Python `dict` и `set` - это хеш-таблицы, и их `in`/доступ по ключу работают по тем же правилам; зависеть от порядка обхода нельзя.
