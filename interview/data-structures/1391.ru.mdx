# Что делает хеш-таблица, когда у разных ключей совпадает хеш?

## Суть

Совпадение хешей у разных ключей называется коллизией. Это неизбежно, потому что входов бесконечно много, а хешей конечное число. Хеш-таблица разрешает коллизии методом цепочек или открытой адресацией, поэтому элементы по-прежнему хранятся и находятся корректно.

## Что ответить на собесе

Коллизия - это когда хеш-функция выдаёт одинаковый индекс для двух разных ключей; её обрабатывают, а не избегают.

- Полностью избежать коллизий нельзя, их можно только грамотно обрабатывать.
- Метод цепочек: в одной ячейке лежит список всех элементов с этим хешем.
- Открытая адресация: при занятой ячейке элемент кладётся в следующую свободную по правилу пробирования.
- В среднем доступ остаётся O(1), но при плохой хеш-функции деградирует до O(n).

## Как это работает

Хеширование превращает данные произвольного размера в значение фиксированной длины. Индекс ячейки обычно берут как `hash(key) % size`. Когда два ключа дают один индекс, нужна стратегия разрешения. Метод цепочек хранит в каждой ячейке список:

```python
class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]

    def hash_function(self, key):
        return len(key) % self.size

    def insert(self, key, value):
        index = self.hash_function(key)
        self.table[index].append((key, value))

    def search(self, key):
        index = self.hash_function(key)
        for k, v in self.table[index]:
            if k == key:
                return v
        return None
```

Ячейка - это список пар. При вставке элемент добавляется в список нужного индекса, при поиске список перебирается до совпадения ключа. Так несколько ключей с одинаковым хешем спокойно уживаются.

## Подводные камни

- Хеш-функция здесь учебная (`len(key)`): на практике берут функции с хорошим распределением, иначе все ключи свалятся в одну цепочку.
- При плохом распределении или высоком коэффициенте заполнения сложность падает с O(1) до O(n).
- Коллизия по хешу - не то же самое, что равенство ключей: всегда нужна финальная проверка `k == key`, поэтому объект обязан корректно реализовывать и `__hash__`, и `__eq__`.
- В открытой адресации удаление сложнее: нельзя просто «обнулить» ячейку, иначе сломается цепочка пробирования - используют маркеры удаления.
