# Почему доступ по индексу в массиве O(1), а в связном списке O(n)?

## Суть

В массиве элементы лежат подряд, поэтому адрес любого вычисляется арифметикой: это O(1). В связном списке узлы разбросаны по памяти, и до n-го приходится идти по ссылкам, отсюда O(n).

## Что ответить на собесе

Скорость доступа по индексу определяется раскладкой данных в памяти.

- Массив: непрерывный блок, адрес элемента считается как `база + индекс × размер_элемента`, за константное время.
- Список Python внутри устроен как динамический массив указателей, поэтому `lst[i]` тоже O(1).
- Связный список хранит узлы где попало, каждый знает лишь следующий, и дойти до индекса i можно только за i переходов по ссылкам, то есть за O(n).
- Зато связный список выигрывает на вставке/удалении в середине при наличии ссылки на узел: O(1) против O(n) у массива.

## Как это работает

Доступ в массиве или списке Python устроен напрямую:

```python
array = [10, 20, 30, 40, 50]
print(array[2])   # 30, адрес вычислен напрямую, O(1)
```

## Подводные камни

- «Список» неоднозначен: list в Python представляет собой динамический массив (доступ O(1)), а структура «связный список» - нечто другое (доступ O(n)). На собесе уточняйте, о чём речь.
- У массива O(1) доступ, но вставка/удаление в начале/середине стоят O(n) из-за сдвига; у связного списка наоборот. Выбор структуры зависит от того, какие операции преобладают.
- O(1) для массива справедливо именно для произвольного доступа по индексу, а не для поиска значения; поиск всё равно идёт за O(n).
- `collections.deque` не массив: индексирование к середине у него O(n), зато добавление/удаление с обоих концов O(1).
