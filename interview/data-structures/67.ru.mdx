# Что искать быстрее по ключу: список или словарь?

## Суть

Словарь ищет по ключу за амортизированное O(1) благодаря хеш-таблице, а список делает это за O(n), потому что перебирает элементы подряд. Если нужен быстрый доступ по ключу, выбор очевиден в пользу словаря.

## Что ответить на собесе

Для поиска по ключу словарь принципиально быстрее: O(1) против O(n) у списка.

- Ключ прогоняется через хеш-функцию, и Python сразу прыгает к ячейке, где лежит значение: это амортизированное O(1).
- Список такой роскоши не даёт: чтобы проверить наличие значения, он идёт от начала и сравнивает элементы по одному, то есть O(n).
- Списки беру там, где важен порядок или нужны повторы, а словари выбираю, когда нужно часто доставать данные по уникальному ключу.

## Как это работает

Список представляет собой упорядоченный динамический массив. Доступ по индексу мгновенный, O(1), но поиск значения (`x in my_list`) линейный: Python сравнивает элементы по очереди, пока не найдёт нужный или не дойдёт до конца. Добавление в конец амортизированно O(1), а вставка в середину стоит O(n) из-за сдвига хвоста.

Словарь хранит пары «ключ-значение» в хеш-таблице. Ключ обязан быть уникальным и хешируемым. При поиске ключ превращается хеш-функцией в число, которое указывает на позицию в таблице, поэтому доступ почти не зависит от размера словаря.

```python
my_list = [1, 2, 3, 4, 5]
my_dict = {1: "a", 2: "b", 3: "c", 4: "d", 5: "e"}

element_in_list = 3 in my_list  # O(n): перебор элементов
element_in_dict = 3 in my_dict  # O(1): проверка по хешу ключа
```

Разница в `in` тут ключевая: для списка это поиск по значениям, а для словаря это проверка наличия ключа в хеш-таблице.

## Подводные камни

- O(1) у словаря является амортизированным. При большом числе коллизий поиск может деградировать до O(n).
- `x in my_dict` проверяет ключи, а не значения. Чтобы искать по значениям, нужно `x in my_dict.values()`, и это снова O(n).
- Ключи словаря обязаны быть хешируемыми. Список или другой изменяемый объект ключом не станет: будет `TypeError`.
- Словарь тратит больше памяти, чем список той же длины, ради скорости поиска. Это сознательный размен.
