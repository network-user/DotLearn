# How do you merge two lists - keys and values - into a single dictionary?

## In short

Pair up the elements of the two lists with the `zip()` function and wrap the result in `dict()`. It all fits on one line: `dict(zip(keys, values))`.

## How to answer in an interview

The idiomatic way is `dict(zip(keys, values))`. `zip` walks both lists in parallel and emits `(key, value)` tuples, and `dict` assembles a dictionary from those pairs. The solution reads in a second and runs in O(n). A nuance interviewers like to ask about: `zip` stops at the shortest list, so with differing lengths the tail of the longer one is lost without an error.

## How it works

```python
keys = ['name', 'age', 'city']
values = ['Alice', 30, 'New York']

result = dict(zip(keys, values))
print(result)
```

Step by step:

1. `zip(keys, values)` creates a lazy iterator of pairs: `('name', 'Alice')`, `('age', 30)`, `('city', 'New York')`.
2. `dict(...)` walks over these pairs and builds a dictionary from them.
3. The output: `{'name': 'Alice', 'age': 30, 'city': 'New York'}`.

A dict comprehension gives the same result if you need to transform something along the way: `{k: v for k, v in zip(keys, values)}`.

## Pitfalls

- With lists of different lengths, `zip` silently truncates the result to the shorter one. If you need to catch the mismatch, use `zip(keys, values, strict=True)` (Python 3.10+), which raises a `ValueError`.
- Duplicates in `keys` collapse: the last value overwrites the previous ones, the dictionary stores the key only once.
- Keys must be hashable - a list in the role of a key gives a `TypeError`.
- `zip` returns a one-shot iterator: once you have walked over it, the second time you will get nothing.
