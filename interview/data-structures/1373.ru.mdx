# Как избавиться от дубликатов в списке и когда важен порядок

## Суть

Если порядок не нужен, прогоните список через `set`. Если порядок важен, используйте `dict.fromkeys`, который убирает повторы, но запоминает первое появление каждого элемента.

## Что ответить на собесе

Если порядок не важен, подойдёт `list(set(data))`; если важен - `list(dict.fromkeys(data))`.

- Самый короткий способ: `list(set(data))`, но `set` не хранит порядок, результат может перемешаться.
- Когда порядок нужно сохранить, беру `list(dict.fromkeys(data))`: словари с Python 3.7 держат порядок вставки, а ключи уникальны.
- Оба варианта работают за O(n) и требуют, чтобы элементы были хешируемыми.

## Как это работает

Множество хранит только уникальные значения, поэтому повторы отбрасываются на этапе вставки:

```python
data = [1, 2, 2, 3, 4, 4, 5]
unique = list(set(data))
print(unique)  # порядок не гарантирован, например [1, 2, 3, 4, 5]
```

`dict.fromkeys` строит словарь, где ключами становятся элементы списка. Повторный ключ не создаёт новую запись, а порядок первого появления сохраняется:

Если элементы не хешируемые (например, вложенные списки), оба способа не подойдут, и приходится отслеживать виденное вручную:

```python
seen = []
for item in [[1], [1], [2]]:
    if item not in seen:
        seen.append(item)
print(seen)  # [[1], [2]]
```

## Подводные камни

- `set` не сохраняет порядок, и для одних и тех же данных он может выглядеть стабильным, но полагаться на это нельзя.
- Оба быстрых способа требуют хешируемых элементов: `list` и `dict` внутрь не положить.
- Ручной вариант через `if item not in seen` для списков работает за O(n^2), что на больших данных медленно.
- `dict.fromkeys` всем ключам ставит значение `None`; нам нужны только ключи, поэтому оборачиваем в `list(...)`.
