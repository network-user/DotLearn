# Слияние словарей в Python и судьба совпадающих ключей

## Суть

Словари объединяют оператором `|` (Python 3.9+), методом `update()` или распаковкой `{**a, **b}`. При коллизии ключей выигрывает значение из правого (последнего) словаря - оно перезаписывает левое.

## Что ответить на собесе

Способ выбираю по тому, нужен ли мне новый словарь или правка на месте. `a | b` и `{**a, **b}` создают новый словарь, не трогая исходные. `a.update(b)` меняет `a` на месте и возвращает `None`. Во всех вариантах правило коллизии одинаковое: при совпадении ключа побеждает значение из словаря, который идёт правее. То есть в `a | b` для общего ключа возьмётся значение из `b`. Это поверхностное слияние: вложенные словари не сливаются рекурсивно, а заменяются целиком.

## Как это работает

```python
a = {'x': 1, 'y': 2}
b = {'y': 3, 'z': 4}

print(a | b)          # {'x': 1, 'y': 3, 'z': 4}  - новый словарь
print({**a, **b})     # {'x': 1, 'y': 3, 'z': 4}  - то же через распаковку

a.update(b)
print(a)              # {'x': 1, 'y': 3, 'z': 4}  - a изменён на месте
```

Ключ `'y'` есть в обоих словарях, поэтому в результате остаётся `3` - значение из `b`. Порядок аргументов определяет, кто кого перезаписывает: `b | a` дало бы `'y': 2`.

## Подводные камни

- `update()` возвращает `None` и меняет словарь на месте - писать `c = a.update(b)` ошибка, `c` окажется `None`.
- Слияние поверхностное: для вложенных структур (`{'cfg': {...}}`) внутренний словарь заменяется целиком, а не сливается; для глубокого слияния нужен ручной обход.
- Оператор `|` доступен только с Python 3.9; в более старых версиях берите `{**a, **b}` или `update()`.
- Порядок имеет значение: при коллизии всегда побеждает правый операнд, легко перепутать направление перезаписи.
