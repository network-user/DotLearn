# Three ways to drop an element from a list: where remove, where del, and where pop?

## In short

You can remove items from a list with three tools, and the choice depends on what you have on hand. `remove()` works by value, `del` and `pop()` work by index, but `pop()` also returns what it took out.

## How to answer in an interview

In Python there are three different removal tools for three different situations. If I know the value and want to drop its first occurrence - I use `list.remove(value)`. If I'm working with an index or a slice and don't need the returned value - I use the `del` operator, which can also delete the variable itself from the namespace. And if I need to both grab an element and remove it from the list at once - that's where only `list.pop(index)` fits; without an argument it pops the last element. The key thing to remember is the exceptions: `remove` raises `ValueError` when the value is absent, and `pop` raises `IndexError` when the index is out of bounds.

## How it works

`remove()` finds the first match by value and erases it. Duplicates after it remain.

```python
fruits = ['apple', 'banana', 'cherry', 'banana']
fruits.remove('banana')
print(fruits)  # ['apple', 'cherry', 'banana']
```

`del` is a language-level operator, not a method. It targets an index, a slice, and even an entire variable.

```python
numbers = [0, 1, 2, 3, 4, 5]
del numbers[1]
print(numbers)  # [0, 2, 3, 4, 5]

del numbers[1:3]
print(numbers)  # [0, 4, 5]
```

`pop()` removes an element by index and immediately hands back its value. Without an argument it takes the tail of the list - which is what makes it handy for a stack.

```python
letters = ['a', 'b', 'c', 'd']
last_letter = letters.pop()
print(last_letter)  # 'd'

second_letter = letters.pop(1)
print(second_letter)  # 'b'
print(letters)  # ['a', 'c']
```

## Pitfalls

- `remove()` deletes only the **first** occurrence. To clear out all duplicates, you need a loop or a list comprehension.
- If the value isn't in the list, `remove()` raises `ValueError` rather than silently returning something.
- `pop(i)` on an empty list or with an out-of-bounds index raises `IndexError`.
- `del list[i]` doesn't return the deleted item - if you need the value, that's `pop()` right away; otherwise you'll have to read the element before deleting it.
- Deleting by index inside a loop is dangerous: indices shift and you can skip elements. Iterate over a copy or from the end.
