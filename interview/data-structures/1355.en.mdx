# What is the difference between an array and a linked list

## In short

Both structures store a sequence, but they lay it out in memory differently. An array is a contiguous block with instant access by index in O(1). A linked list is a chain of nodes with references, cheap to insert into and delete from, but slow to access in O(n).

## How to answer in an interview

An array sits in memory contiguously, so the address of any element is computed by arithmetic: access by index in O(1). We pay for this with shifting on insertion and deletion in the middle (O(n)) and a less flexible size. A linked list stores nodes anywhere, linking them by references: insertion and deletion mean rewriting a couple of references in O(1) if the position is already known, but access to the k-th element requires a walk from the head in O(n) and spends memory on the references themselves. The choice is a trade-off: an array for frequent access by index, a list for frequent insertions and deletions.

## How it works

An array is addressed by index directly, without traversal:

```python
array = [1, 2, 3, 4, 5]
third = array[2]   # 3, access in O(1)
array[2] = 10      # [1, 2, 10, 4, 5]
```

A linked list is traversed by following the `next` reference from node to node:

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None


node1 = Node(1)
node2 = Node(2)
node3 = Node(3)
node1.next = node2
node2.next = node3

current = node1
while current is not None:
    print(current.data)   # 1, then 2, then 3
    current = current.next
```

## Pitfalls

- The O(1) complexity for insertion/deletion in a list holds only if the neighboring node has already been found; finding that spot is itself O(n).
- A list spends extra memory on references and is worse for the CPU cache: nodes are scattered, whereas an array sits contiguously and is read faster in practice.
- The built-in `list` in Python is a dynamic array, not a linked list: access by index is O(1), but insertion at the front (`insert(0, x)`) is O(n).
- "Fixed size" applies to low-level arrays (for example, from the `array` module or in C); Python's `list` grows on its own, adding capacity in an amortized fashion.
