# Какой набор операций доступен у отображений в Python?

## Суть

Отображение (в Python это словарь) - структура «ключ → значение». Поверх него работают вставка, обновление, удаление, проверка наличия ключа, доступ по ключу и три способа обхода: по ключам, по значениям и по парам.

## Что ответить на собесе

Словарь поддерживает полный набор операций отображения: запись по ключу (`d[k] = v`) одновременно добавляет и обновляет, `del d[k]` и `d.pop(k)` удаляют, оператор `in` проверяет наличие ключа за O(1), а методы `keys()`, `values()`, `items()` возвращают view-объекты для перебора. Важная деталь: все базовые операции - вставка, удаление, поиск - в среднем O(1) благодаря хеш-таблице под капотом.

## Как это работает

Вставка и обновление - это одна и та же операция присваивания по ключу:

```python
d = {}
d['key1'] = 'value1'
d['key1'] = 'new_value'
```

Если ключа не было - запись создаётся, если был - значение перезаписывается.

Удаление:

```python
del d['key1']
value = d.pop('key2', None)
```

`del` бросит `KeyError`, если ключа нет, а `pop` с дефолтом - безопасен и заодно вернёт удалённое значение.

Проверка наличия и обход:

```python
exists = 'key1' in d

for key in d:
    print(key)
for value in d.values():
    print(value)
for key, value in d.items():
    print(key, value)
```

`keys()`, `values()` и `items()` возвращают не списки, а динамические представления (views), которые отражают изменения словаря и при этом легко превращаются в список через `list()`.

## Подводные камни

- Обращение к отсутствующему ключу через `d[k]` бросает `KeyError`; для мягкого доступа используйте `d.get(k)` или `d.setdefault(k, default)`.
- View-объекты «живые»: если менять словарь во время итерации по `items()`, получите `RuntimeError`.
- Ключом может быть только хешируемый (неизменяемый) объект - список ключом не станет, кортеж из неизменяемых элементов станет.
- С Python 3.7 порядок вставки сохраняется, но полагаться на словарь как на упорядоченную структуру для сортировки не стоит - это деталь реализации применительно к порядку добавления, а не сортировки.
