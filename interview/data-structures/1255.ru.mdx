# Линейная сложность против квадратичной: где проходит граница

## Суть

O(n) обгоняет O(n²), потому что лучше держит рост входа. У линейного алгоритма время растёт пропорционально числу элементов, у квадратичного - как квадрат этого числа. На малых данных разница незаметна, но с ростом n квадрат разносит линейный алгоритм в пыль.

## Что ответить на собесе

Big-O описывает, как масштабируется время работы при росте размера входа n, игнорируя константы. O(n) означает, что удвоение входа примерно удваивает время. O(n²) - что удвоение входа учетверяет время. Поэтому на тысяче элементов линейный алгоритм делает ~1000 шагов, а квадратичный - ~1 000 000: разница в тысячу раз и дальше только растёт. На собесе важно подчеркнуть: O говорит про асимптотику, а не про конкретные секунды - на крошечных n квадратичный алгоритм с маленькой константой может быть даже быстрее, и решает именно поведение на больших объёмах.

- **O(n) - линейная.** Один проход по данным. Поиск максимума в массиве из n элементов требует n сравнений: время пропорционально n.
- **O(n²) - квадратичная.** Обычно это вложенный цикл, где для каждого элемента перебираются все остальные. Сортировка пузырьком сравнивает по сути каждый с каждым, отсюда n·n.

Почему линейное предпочтительнее на масштабе:

1. **Рост.** При увеличении n время O(n²) разгоняется кратно быстрее, чем O(n).
2. **Практика.** На больших массивах линейный алгоритм отдаёт результат заметно раньше, что прямо влияет на отзывчивость системы.
3. **Ресурсы.** Меньше шагов - меньше нагрузка на CPU; на мобильных и встроенных устройствах это критично.

### Линейный поиск - O(n)

```python
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1
```

В худшем случае проверяем все n элементов - O(n).

### Сортировка пузырьком - O(n²)

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
```

Вложенные циклы дают n·n сравнений - O(n²).

## Подводные камни

- **Big-O - про асимптотику, а не про секунды.** Алгоритм O(n) с огромной константой на маленьком n может проиграть O(n²) с крошечной. Константы скрыты, и для малых входов они иногда решают.
- **Худший, средний и лучший случай различаются.** У того же пузырька лучший случай на почти отсортированных данных можно довести до O(n) ранним выходом; голое «O(n²)» этого не отражает.
- **Сложность по памяти отдельна от времени.** Алгоритм бывает быстрым по времени, но прожорливым по памяти - оценивайте обе оси.
- **Не любую O(n²) можно бездумно заменить на O(n).** Иногда O(n log n) или O(n) требуют дополнительной памяти или предобработки; «меньшая сложность» не всегда означает «строго лучше для задачи».
