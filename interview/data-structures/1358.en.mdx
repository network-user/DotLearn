# Finding the N-th node from the end of a singly linked list in one pass

## In short

The two-pointer technique solves this. One pointer moves ahead by N nodes, then both move in lockstep. When the leading pointer hits the end, the second one stands exactly on the N-th element from the end — a single pass, without counting the length.

## How to answer in an interview

A singly linked list has no access by index, so "the fifth from the end" done naively would require two passes: first find the length, then count back. I do it in one pass with two pointers. I space them out by the required gap: the leading `fast` goes ahead by 5 nodes, then I move `fast` and `slow` together one step at a time. The gap between them is preserved, so the moment `fast` reaches the end, `slow` lands on the fifth element from the end. The complexity is O(n) in time and O(1) in memory. I do not forget the case where there are fewer than five nodes.

## How it works

Create a gap of N nodes, then move the pair of pointers in lockstep.

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

def nth_from_end(head, n):
    fast = head
    slow = head

    for _ in range(n):
        if fast is None:
            return None  # fewer than n nodes
        fast = fast.next

    while fast is not None:
        fast = fast.next
        slow = slow.next

    return slow.data if slow else None

head = Node(1)
node = head
for value in (2, 3, 4, 5, 6, 7):
    node.next = Node(value)
    node = node.next

print(nth_from_end(head, 5))  # 3
```

The leading `fast` first breaks away by `n` nodes, then the pair moves together. When `fast` runs off the end, `slow` points to the target node.

## Pitfalls

- If the list has fewer than `n` nodes, the run-up loop must catch this and return `None`, otherwise `fast.next` will fail on `None`.
- The edge case "delete the N-th from the end" is solved by the same technique, but it is more convenient to introduce a dummy head so that the very first node can be removed correctly.
- The gap must be exactly `n`, not `n-1`: an off-by-one error is the most common here.
- The technique only works forward: in a singly linked list you cannot go back.
