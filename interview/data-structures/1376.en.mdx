# What can and cannot serve as dictionary keys

## In short

A `dict` key can only be a hashable object, and in Python hashability almost always means immutability: strings, numbers, tuples of immutable elements. Lists and dictionaries are not suitable as keys.

## How to answer in an interview

Dictionary keys must be hashable. Access by key is built on a hash table: Python takes `hash(key)`, uses it to select a bucket, and there compares keys with `==`. So a key needs two properties — a stable hash for its entire lifetime and correct comparison. Immutable built-in types (str, int, float, bool, tuple, frozenset) satisfy this, while mutable ones (list, dict, set) do not, so trying to use them as a key raises `TypeError: unhashable type`.

Hashability is determined by the `__hash__` method. An object has a stable hash if `__hash__` returns the same number throughout the object's life, and equal objects produce equal hashes.

Strings and numbers work right away:

```python
prices = {"apple": 50, "pear": 70}
print(prices["apple"])  # 50

coords = {1: "one", 2.5: "two-and-half"}
print(coords[1])  # one
```

A tuple is hashable only if all of its elements are hashable — this is a recursive property:

```python
points = {(1, 2): "A", (3, 4): "B"}
print(points[(1, 2)])  # A
```

Comparing hashes directly looks, in simplified form, like this:

```python
print(hash("apple") == hash("apple"))  # True
print(hash((1, 2)) == hash((1, 2)))    # True
```

## Pitfalls

- A tuple with a list inside is no longer hashable: `hash((1, [2]))` raises `TypeError`, because the list element is mutable.
- `1`, `1.0`, and `True` are the same key for a dictionary, because they are equal and have equal hashes. `{1: "a", True: "b", 1.0: "c"}` collapses into a single element.
- A custom class is hashable by default (by `id`), but as soon as you define `__eq__`, Python sets `__hash__` to None, and the object becomes unhashable until you define `__hash__` explicitly.
- A list cannot be used as a key, but it can be turned into a `tuple`; a set into a `frozenset`.
