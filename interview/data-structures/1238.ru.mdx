# Какими способами хеш-таблица разрешает коллизии ключей?

## Суть

Два базовых семейства методов: метод цепочек (элементы с одинаковым хешем складывают в список рядом с ячейкой) и открытая адресация (всё хранится в самом массиве, а свободное место ищут пробированием - линейным, квадратичным или двойным хешированием).

## Что ответить на собесе

Коллизия - это когда два разных ключа дают один индекс; разрешают её двумя путями.

- Метод цепочек: каждая ячейка ведёт к связному списку (или дереву), совпавшие ключи дописываются туда; просто и устойчиво к высокой заполненности.
- Открытая адресация: всё лежит прямо в массиве, а при коллизии по детерминированной схеме ищется следующая свободная ячейка.
- Схемы пробирования: линейное (шаг +1), квадратичное (шаг растёт квадратично) и двойное хеширование (шаг задаёт вторая хеш-функция).
- Цепочки проще и терпят высокий load factor, открытая адресация дружелюбнее к кэшу, но страдает от кластеризации.

**Метод цепочек.** Ячейка массива - голова списка. При коллизии новый элемент дописывается в нужный список, поиск идёт по нему линейно.

```python
class HashTable:
    def __init__(self, size):
        self.size = size
        self.table = [[] for _ in range(size)]

    def insert(self, key, value):
        index = hash(key) % self.size
        bucket = self.table[index]
        for i, (k, _) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)
                return
        bucket.append((key, value))

    def get(self, key):
        index = hash(key) % self.size
        for k, v in self.table[index]:
            if k == key:
                return v
        return None
```

**Открытая адресация.** Данные лежат в самом массиве; при занятой ячейке ищется следующая по схеме пробирования.

```python
class OpenAddressingHashTable:
    def __init__(self, size):
        self.size = size
        self.table = [None] * size

    def insert(self, key, value):
        index = hash(key) % self.size
        while self.table[index] is not None and self.table[index][0] != key:
            index = (index + 1) % self.size
        self.table[index] = (key, value)

    def get(self, key):
        index = hash(key) % self.size
        while self.table[index] is not None:
            if self.table[index][0] == key:
                return self.table[index][1]
            index = (index + 1) % self.size
        return None
```

## Подводные камни

- **Кластеризация** при линейном пробировании: занятые ячейки слипаются в длинные участки, и пробирование удлиняется. Квадратичное и двойное хеширование смягчают это.
- При открытой адресации **удаление сложнее**: нельзя просто обнулить ячейку, иначе оборвётся цепочка пробирования для других ключей. Нужны "надгробия" (tombstones).
- У метода цепочек при плохой хеш-функции один список может вырасти до O(n) - доступ деградирует. В CPython корзины - не списки, а часть единого массива с открытой адресацией.
- **Load factor** критичен для обоих методов: при заполнении выше порога таблицу надо расширять и перехешировать, иначе число коллизий растёт.
- В наивной открытой адресации из примера обязательно нужен запас свободных ячеек, иначе вставка зациклится на полном массиве.
