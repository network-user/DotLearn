# What set of operations does the set type provide for working with unique elements?

## In short

`set` implements the classic set operations: union, intersection, difference, symmetric difference, plus subset and superset checks. This set of operations is enough for most tasks involving unique collections.

## How to answer in an interview

A set supports set-theoretic operations, each available both as an operator and as an identically named method: union `|` / `union`, intersection `&` / `intersection`, difference `-` / `difference`, symmetric difference `^` / `symmetric_difference`. Relationships are checked with comparison operators: `<=` / `issubset` and `>=` / `issuperset`. Since the elements are hashed, all these operations work fast, and the elements themselves must be hashable.

```python
set_a = {1, 2, 3}
set_b = {3, 4, 5}

set_a | set_b   # union: {1, 2, 3, 4, 5}
set_a & set_b   # intersection: {3}
set_a - set_b   # difference: {1, 2}
set_a ^ set_b   # symmetric difference: {1, 2, 4, 5}
```

Relationships between sets:

```python
set_c = {1, 2}
set_c <= set_a   # subset: True
set_a >= set_c   # superset: True
```

The operator form (`|`, `&`, ...) requires both operands to be sets, while the methods (`union`, `intersection`, ...) accept any iterable argument - which is more convenient when the second operand is, for example, a list.

## Pitfalls

- **Operators are stricter than methods.** `set_a | [1, 2]` raises a `TypeError`, whereas `set_a.union([1, 2])` works - the method accepts any iterable.
- **Difference is not symmetric.** `set_a - set_b` and `set_b - set_a` give different results; don't confuse it with `^`, which is symmetric.
- **A set is unordered.** The output order of elements is undefined and can't be relied on; for an ordered result, sort it separately.
- **Elements must be hashable.** You can't put a list inside a set; for nested immutable collections use `frozenset`.
- **`{}` is a dict, not a set.** An empty set is created only via `set()`.
