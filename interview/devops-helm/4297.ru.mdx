# Что такое Helm и chart?

## Суть

Helm это package manager для Kubernetes: устанавливает, обновляет и удаляет приложения одной командой. Chart это версионируемый пакет (templates + values + metadata), из которого Helm рендерит манифесты и создаёт release в namespace. Слой между сотней YAML и параметризованным deploy для dev/stage/prod.

## Что ответить на собесе

- Chart = `Chart.yaml` + `values.yaml` + `templates/` (+ optional `charts/` deps).
- `helm install <release> <chart>` создаёт revision 1 в cluster.
- `helm upgrade` меняет revision; `helm uninstall` снимает ресурсы (с учётом hooks/policy).
- OCI registry или HTTP repo (`helm repo add`) для distribution chart'ов.
- Честно: Helm не заменяет GitOps controller; в prod часто chart + Argo CD/Flux sync.

## Как это работает

Создание skeleton и install:

```bash
helm create payments-api
helm lint ./payments-api

helm install payments ./payments-api \
  -n prod --create-namespace \
  -f values-prod.yaml \
  --set image.tag=2.1.0 \
  --wait
```

Структура chart после `helm create`:

```bash
find payments-api -maxdepth 2 -type f | sort
# payments-api/Chart.yaml
# payments-api/values.yaml
# payments-api/templates/deployment.yaml
# payments-api/templates/service.yaml
# payments-api/templates/_helpers.tpl
```

Обновление и удаление:

```bash
helm upgrade payments ./payments-api -n prod -f values-prod.yaml --atomic --wait
helm list -n prod
helm uninstall payments -n prod
```

## Подводные камни

- **Release name collision.** Два install с одним release в namespace перезаписывают history; имена уникальны per namespace.
- **Orphan resources.** Аннотация `helm.sh/resource-policy: keep` оставляет PVC после uninstall.
- **Values merge surprise.** Множественные `-f` перекрывают ключи последним файлом; неочевидный override в CI.
- **Chart from random repo.** Без signature verification тянете supply-chain risk; pin OCI digest.
- **Tiller legacy confusion.** Helm 2 tiller vs Helm 3 client-only; в interview уточняю версию 3.x.
