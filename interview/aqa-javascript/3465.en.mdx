# Object.create vs the new operator: when to pick which

## In short

`Object.create(proto)` builds an empty object with the given prototype and does not run a constructor. The `new Constructor()` operator creates an object, links it to `Constructor.prototype`, runs the function body, and returns the instance (or an explicit object return). The first path suits pure inheritance without initialization; the second suits classes and factories with side effects on creation.

## How to answer in an interview

- For Page Objects I reuse base methods via `Object.create(basePO)` without rerunning the constructor.
- I use `new` for DOM fixture wrappers where the constructor attaches listeners.
- For test doubles without extra logic I create objects with `Object.create(null)` as a clean dictionary.
- I assert the chain with `Object.getPrototypeOf(instance) === Expected.prototype`.
- Honestly: `Object.create` does not call `constructor`; if you need invariants, use `new` or `Reflect.construct`.

## How it works

Prototype without a constructor:

```
const baseReporter = {
  header() {
    return "Report";
  },
};

const junitReporter = Object.create(baseReporter);
junitReporter.format = () => `${this.header()}: XML`;
```

Via constructor:

```
function HtmlReporter(theme) {
  this.theme = theme;
  this.header = () => `Report (${this.theme})`;
}

const dark = new HtmlReporter("dark");
```

Behavior comparison in pytest:

```python
class BasePage:
    def url(self) -> str:
        return "/login"


def make_page_without_init(proto: type) -> object:
    return type("Anonymous", (proto,), {})()


def make_page_with_init(cls: type, theme: str):
    return cls(theme)


class LoginPage(BasePage):
    def __init__(self, theme: str):
        self.theme = theme


def test_create_vs_new_semantics():
    bare = make_page_without_init(BasePage)
    assert bare.url() == "/login"
    assert not hasattr(bare, "theme")
    full = make_page_with_init(LoginPage, "dark")
    assert full.theme == "dark"
```

## Pitfalls

**Object.create(null) and JSON.** No `toString`; serialization breaks.

**Constructor without new.** Calling `Animal()` as a plain function loses `this`; `instanceof` tests fail.

**Returning a primitive from a constructor.** `new` ignores returned number/string.

**Object.create second argument.** Forgotten descriptors default writable/enumerable to false.

**Mocking the prototype chain.** Jest `mockImplementation` on the prototype is not copied by `Object.create`.
