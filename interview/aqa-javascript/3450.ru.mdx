# Стрелочные функции без своего this и arguments

## Суть

Стрелочная функция лексически наследует `this` и `arguments` из внешней функции, не создавая собственных. Обычная function получает `this` в момент вызова. В page object стрелки удобны в колбэках таймеров и обработчиков, но не подходят как методы с динамическим `this`.

## Что ответить на собесе

- `this` внутри `() =>` совпадает с enclosing scope, не с объектом слева от точки.
- `arguments` в стрелке отсутствует; беру rest `(...params)`.
- Стрелку не вызываю с `new`, у неё нет `prototype` для конструктора.
- В классе тестового fixture методы оставляю обычными function или class fields.
- Честно: стрелка как метод объекта `{ tick: () => this.count }` ломает ожидаемый `this` при передаче по ссылке.

## Как это работает

Проблема обычной function в таймере:

```
function Counter() {
    this.count = 0;
    setInterval(function () {
        this.count += 1;
    }, 100);
}
```

Стрелка сохраняет `this` Counter:

```
function Counter() {
    this.count = 0;
    setInterval(() => {
        this.count += 1;
    }, 100);
}
```

Rest вместо arguments:

```
const summarize = (...parts) => parts.join("|");
```

Проверяю binding в pytest через простую модель:

```python
class Counter:
    def __init__(self):
        self.count = 0

    def tick_lexical(self, delta):
        self.count += delta
        return self.count


def test_lexical_method_updates_state():
    c = Counter()
    assert c.tick_lexical(2) == 2
    assert c.tick_lexical(1) == 3
```

## Подводные камни

**Стрелка в addEventListener на class instance.** `this` не instance, если не bound.

**Метод в объекте литерале.** Потеря `this` при деструктуризации `{ tick }`.

**call/apply/bind бесполезны.** `this` стрелки не меняется.

**Наследование arguments из outer.** Вложенная стрелка читает arguments родителя, не свои параметры.

**Стрелка в prototype method.** Нельзя override `this` для subclass hook.
