# Chai и assertion-библиотеки: аналог для Python AQA

## Суть

Chai — assertion library для JavaScript (стили expect/should/assert), часто с Mocha/Jest/Cypress. Она выражает ожидания читаемо: тип, глубокое равенство, исключения. Python-эквивалент в pytest: `assert`, hamcrest, pytest-check или jsonschema для контрактов.

## Что ответить на собесе

- Chai BDD: `expect(x).to.equal(1)`; TDD: `assert.strictEqual(x, 1)`.
- plugins: chai-as-promised для async, chai-http для API (legacy).
- В Cypress assertions встроены поверх Chai; в Jest чаще `expect` из jest.
- На Python проектах использую pytest + pydantic/jsonschema вместо переноса Chai синтаксиса.
- Честно добавляю: смешивать Chai и Jest matchers в одном файле не стоит; разный fail output путает triage.

## Как это работает

Assertion библиотека сравнивает actual/expected и форматирует diff. В JS stack Mocha ловит throw; Chai добавляет chainable language.

```
// JavaScript (Mocha + Chai)
expect([1, 2]).to.be.an("array").with.lengthOf(2);
```

В pytest тот же intent:

```python
import pytest


def normalize_ids(rows):
    return sorted(row["id"] for row in rows)


def test_user_list_sorted_ids():
    payload = [{"id": 2, "name": "b"}, {"id": 1, "name": "a"}]
    assert normalize_ids(payload) == [1, 2]


def test_raises_on_invalid():
    with pytest.raises(ValueError, match="empty"):
        raise ValueError("empty input")
```

Для API сравниваю subset полей, а не весь JSON, чтобы не ловить лишние metadata поля.

## Подводные камни

**Deep equal на больших объектах.** Diff нечитаем; snapshot без фильтра.

**Floating compare без delta.** `0.1 + 0.2` ломает strict equal.

**chai-as-promised без return.** Promise assert не ждётся.

**Переиспользование expect на mutable object.** Side effect между assert.

**Assert message без контекста.** В CI непонятно, какой step упал в scenario.
