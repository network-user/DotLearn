# Two ways to declare a function in JavaScript

## In short

In JS, functions are defined via Function Declaration (a named statement) or Function Expression (a function as a variable value). Declarations are hoisted and callable before their line; expressions exist only after the assignment runs. For AQA, this affects hook and helper order in test files.

## How to answer in an interview

- Declaration: `function greet(name) { return ... }`, callable before the line in the same scope.
- Expression: `const greet = function(name) { ... }` or an arrow form; ReferenceError before assignment.
- In page objects I prefer expressions to avoid polluting the Playwright/Cypress global scope.
- A named function expression improves stack traces when a UI test fails.
- Honestly: declaration hoisting can hide circular dependencies between helper modules.

## How it works

A declaration is hoisted in full:

```
function buildUrl(path) {
    return `https://api.test${path}`;
}
```

An expression is bound to a constant:

```
const buildUrl = function(path) {
    return `https://api.test${path}`;
};
```

In pytest wrappers for JS API tests I mirror the ordering rule:

```python
import pytest


def test_declaration_available_before_line():
    source = """
    const result = greet('qa');
    function greet(name) { return `hi ${name}`; }
    """
    assert "greet" in source


def test_expression_requires_order():
    ordered = """
    const greet = (name) => `hi ${name}`;
    const result = greet('qa');
    """
    assert ordered.index("const greet") < ordered.index("greet('qa')")
```

CI lint forbids function declarations inside blocks; expressions pass.

## Pitfalls

**Hoisting inside conditionals.** In strict mode, a declaration inside `if` behaves inconsistently across engines.

**Anonymous expression in the stack.** A failing test shows `(anonymous)` instead of the helper name.

**Reassigning const.** Expressions cannot be redeclared; declarations conflict in the same scope.

**IIFE vs expression.** Teams confuse an immediately invoked wrapper with a normal assignment.

**Mixed styles in a suite.** Some files use declarations, others arrows, breaking a unified code-style gate.
