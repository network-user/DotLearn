# JavaScript data types for automated tests

## In short

JavaScript splits values into seven primitives (`undefined`, `null`, `boolean`, `number`, `string`, `bigint`, `symbol`) and the reference type `object` (objects, arrays, functions, dates). In API/UI tests, the type drives expected/actual comparison and JSON serialization.

## How to answer in an interview

- Primitives copy by value; objects pass by reference.
- `typeof null` is `"object"` (a historical bug); I use strict checks.
- I do not mix `bigint` with `number` without an explicit cast in assertions.
- In contract tests I validate the response schema, not only `typeof`.
- Honestly: `symbol` in a payload is rare but breaks deep equal without a custom replacer.

## How it works

Primitives live inline in the variable:

```
const passed = true;
const retries = 3;
const label = "smoke";
```

Reference types point at shared memory:

```
const payload = { userId: 42, roles: ["qa"] };
const copy = payload;
copy.roles.push("admin");
```

I verify deep equality in a Python runner that executes a JS snippet:

```python
import json


def normalize_api_body(raw: str) -> dict:
    data = json.loads(raw)
    if isinstance(data.get("id"), str) and data["id"].isdigit():
        data["id"] = int(data["id"])
    return data


def test_types_after_normalization():
    body = '{"id":"7","ok":true,"tags":[]}'
    parsed = normalize_api_body(body)
    assert isinstance(parsed["id"], int)
    assert isinstance(parsed["ok"], bool)
    assert isinstance(parsed["tags"], list)
```

`undefined` is omitted in JSON; `null` is serialized explicitly.

## Pitfalls

**Comparing arrays with `==`.** Two different arrays with the same content are not reference-equal.

**Number and NaN.** `typeof NaN` is `number`; a plain assert confuses it with a valid number.

**BigInt in REST.** `JSON.stringify` throws on bigint without a replacer.

**null vs undefined confusion.** The frontend sends `null` while the test expects a missing key.

**Date as object.** `typeof new Date()` is not `date`; you need an instance check.
