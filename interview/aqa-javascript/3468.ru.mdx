# Двойное отрицание !! для boolean coercion

## Суть

Оператор `!!` дважды применяет логическое NOT и превращает любое значение в строгий `true` или `false` по правилам truthy/falsy. Первый `!` даёт boolean-инверсию, второй возвращает «нормализованный» булев результат. Это короче, чем `Boolean(x)`, но семантика та же.

## Что ответить на собесе

- В guard для feature flag пишу `if (!!process.env.CI)` только если нужен именно boolean, иначе prefer явное сравнение.
- В expect для DOM: `expect(!!node.textContent).toBe(true)`.
- Falsy: `0`, `""`, `null`, `undefined`, `NaN`, `false`.
- Truthy: `{}`, `[]`, любая непустая строка, ненулевое число.
- Честно: `!![]` true сбивает новичков, в assert лучше `.toBeTruthy()` с комментарием intent.

## Как это работает

```
const hasCases = !!suite.cases?.length;
const idle = !!0;
const pending = !!"";

console.log(hasCases, idle, pending); // true false false
```

Эквивалент:

```
Boolean(suite.cases?.length);
```

Таблица в pytest:

```python
TRUTHY_SAMPLES = ["ok", 1, [1], {"a": 1}]
FALSY_SAMPLES = ["", 0, None, [], {}]


def to_bool(value) -> bool:
    return bool(value)


def test_double_negation_matches_bool():
    for sample in TRUTHY_SAMPLES:
        assert to_bool(sample) is True
    for sample in FALSY_SAMPLES:
        if sample == [] or sample == {}:
            assert to_bool(sample) is True  # Python: empty list/dict truthy differs
        else:
            assert to_bool(sample) is False
```

## Подводные камни

**Document.all.** Исторический exotic object ведёт себя как undefined и object одновременно.

**!! document.querySelector miss.** null -> false, но пустой div без текста тоже falsy для content check.

**Nested !! в template.** `#{!!flag}` в Ruby другой язык, в JS лишний шум.

**Optional chaining short.** `!!obj?.enabled` vs `obj?.enabled === true` разный смысл для `undefined`.

**Lint no-extra-boolean-cast.** ESLint ругается на `!!` в if condition, оставляю для API return type.
