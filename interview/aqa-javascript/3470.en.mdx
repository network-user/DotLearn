# Mocks, stubs, and spies in unit tests

## In short

A test double isolates the SUT from slow or unavailable dependencies. A stub replaces responses without verifying interaction. A mock sets call expectations and verifies parameters. A spy wraps a real function, records calls, and may still run the original. In Jest, `jest.fn` is closer to mock/spy; `jest.spyOn` keeps the implementation.

## How to answer in an interview

- I stub the HTTP client to return fixture JSON and never hit prod APIs in unit tests.
- With mocks I assert `toHaveBeenCalledWith` for audit log side effects.
- I spy on the logger to see errors recorded without silencing console.
- I do not mock what I am testing: only module boundaries.
- Honestly: over-mocking makes tests brittle on refactors that do not change behavior.

## How it works

Stub:

```
const api = { fetchUser: () => ({ id: 1, role: "qa" }) };
const profile = buildProfile(api.fetchUser());
expect(profile.label).toBe("qa");
```

Mock:

```
const notify = jest.fn();
service.onFail(notify);
expect(notify).toHaveBeenCalledTimes(1);
```

Spy:

```
const math = { sum: (a, b) => a + b };
const spy = jest.spyOn(math, "sum");
expect(math.sum(2, 3)).toBe(5);
expect(spy).toHaveBeenCalledWith(2, 3);
spy.mockRestore();
```

Python pytest with unittest.mock:

```python
from unittest.mock import MagicMock, patch


def test_stub_vs_mock():
    repo = MagicMock()
    repo.get_user.return_value = {"id": 1, "role": "qa"}
    assert repo.get_user(1)["role"] == "qa"
    repo.get_user.assert_called_once_with(1)


def test_spy_on_real():
    calls = []

    def emit(event: str) -> None:
        calls.append(event)

    with patch(__name__ + ".emit", wraps=emit) as spy:
        emit("started")
        assert spy.call_count == 1
```

## Pitfalls

**Mock without assert.** Silent pass while prod breaks.

**Spy without restore.** Leaks between tests; suite order matters.

**Stub too smart.** Duplicates prod logic and false confidence.

**Partial mock of a class.** Real constructor, mocked method: `new` side effects remain.

**jest.clearAllMocks vs reset.** clear wipes counters; reset drops implementation.
