# call, apply и bind: управление this

## Суть

`Function.prototype.call` и `apply` вызывают функцию немедленно, первым аргументом задавая `this`, дальше аргументы по списку или массивом. `bind` возвращает новую функцию с зафиксированным `this` и опциональным partial application. В тестах это лечит потерянный контекст у методов page object и legacy callback.

## Что ответить на собесе

- Перед `setTimeout(po.click, 100)` делаю `po.click.bind(po)` или стрелку.
- `apply` удобен, когда аргументы уже лежат в массиве из data provider.
- Borrow method: `[].slice.call(arguments)` в старых helper.
- Sinon stub сохраняет bind chain, проверяю `calledOn`.
- Честно: двойной bind не снять без `.bind` на оригинале, в teardown храню reference.

## Как это работает

call:

```
function logResult(label) {
  console.log(`${label}: ${this.status}`);
}

const run = { status: "passed" };
logResult.call(run, "smoke");
```

apply:

```
logResult.apply(run, ["regression"]);
```

bind:

```
const boundLog = logResult.bind(run, "nightly");
boundLog();
```

Pytest analogue for partial:

```python
from functools import partial


class RunContext:
    def __init__(self, status: str):
        self.status = status


def log_result(ctx: RunContext, label: str) -> str:
    return f"{label}: {ctx.status}"


def test_bind_like_partial():
    ctx = RunContext("passed")
    bound = partial(log_result, ctx, "smoke")
    assert bound() == "smoke: passed"
```

## Подводные камни

**bind в constructor.** `new (Fn.bind(obj))` игнорирует bound this.

**Arrow и bind.** Стрелка не имеет own this, bind бесполезен.

**apply vs spread.** `fn(...args)` заменил apply, но apply нужен для array-like.

**jest.fn bind.** Mock теряет call count если перепутать reference.

**Strict mode global.** call(null) даёт null this, не window.
