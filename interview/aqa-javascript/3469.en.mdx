# call, apply, and bind: controlling this

## In short

`Function.prototype.call` and `apply` invoke a function immediately, setting `this` as the first argument, then passing args individually or as an array. `bind` returns a new function with fixed `this` and optional partial application. In tests this fixes lost context on page object methods and legacy callbacks.

## How to answer in an interview

- Before `setTimeout(po.click, 100)` I use `po.click.bind(po)` or an arrow wrapper.
- `apply` helps when args already live in an array from a data provider.
- Borrowing methods: `[].slice.call(arguments)` in older helpers.
- Sinon stubs keep the bind chain; I assert with `calledOn`.
- Honestly: you cannot unwrap double bind without the original; I keep references for teardown.

## How it works

call:

```
function logResult(label) {
  console.log(`${label}: ${this.status}`);
}

const run = { status: "passed" };
logResult.call(run, "smoke");
```

apply:

```
logResult.apply(run, ["regression"]);
```

bind:

```
const boundLog = logResult.bind(run, "nightly");
boundLog();
```

Pytest analogue for partial:

```python
from functools import partial


class RunContext:
    def __init__(self, status: str):
        self.status = status


def log_result(ctx: RunContext, label: str) -> str:
    return f"{label}: {ctx.status}"


def test_bind_like_partial():
    ctx = RunContext("passed")
    bound = partial(log_result, ctx, "smoke")
    assert bound() == "smoke: passed"
```

## Pitfalls

**bind in constructor.** `new (Fn.bind(obj))` ignores the bound this.

**Arrow and bind.** Arrows have no own this; bind does nothing useful.

**apply vs spread.** `fn(...args)` replaced most apply, but apply still helps array-like objects.

**jest.fn bind.** Swapping references loses call count.

**Strict mode global.** call(null) yields null this, not window.
