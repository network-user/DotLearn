# Chai and assertion libraries: the Python AQA parallel

## In short

Chai is a JavaScript assertion library (expect/should/assert styles), often with Mocha/Jest/Cypress. It states expectations readably: type, deep equality, exceptions. The Python parallel in pytest is plain `assert`, hamcrest, pytest-check, or jsonschema for contracts.

## How to answer in an interview

- Chai BDD: `expect(x).to.equal(1)`; TDD: `assert.strictEqual(x, 1)`.
- Plugins: chai-as-promised for async, chai-http for API (legacy).
- Cypress bundles assertions on Chai; Jest usually uses its own `expect`.
- On Python projects I use pytest + pydantic/jsonschema instead of copying Chai syntax.
- I add honestly: mixing Chai and Jest matchers in one file hurts; different fail output confuses triage.

## How it works

The assertion library compares actual/expected and formats a diff. In the JS stack Mocha catches throws; Chai adds chainable language.

```
// JavaScript (Mocha + Chai)
expect([1, 2]).to.be.an("array").with.lengthOf(2);
```

The same intent in pytest:

```python
import pytest


def normalize_ids(rows):
    return sorted(row["id"] for row in rows)


def test_user_list_sorted_ids():
    payload = [{"id": 2, "name": "b"}, {"id": 1, "name": "a"}]
    assert normalize_ids(payload) == [1, 2]


def test_raises_on_invalid():
    with pytest.raises(ValueError, match="empty"):
        raise ValueError("empty input")
```

For APIs I compare a field subset, not the whole JSON, to ignore extra metadata.

## Pitfalls

**Deep equal on huge objects.** Diff is unreadable; unfiltered snapshots.

**Float compare without delta.** `0.1 + 0.2` breaks strict equal.

**chai-as-promised without return.** Promise assert never awaited.

**Reusing expect on mutable objects.** Side effects between asserts.

**Assert message without context.** CI cannot tell which step failed in a scenario.
