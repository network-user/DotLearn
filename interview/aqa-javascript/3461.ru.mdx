# == и === в assert автотестов на JavaScript

## Суть

`==` сравнивает с приведением типов (abstract equality), `===` без приведения (strict). Примеры: `0 == false` true, `0 === false` false. В AQA по умолчанию strict matcher и `===`, иначе строка `"5"` и число 5 дают ложный green.

## Что ответить на собесе

- В Jest/Playwright `toBe` это `Object.is`, ближе к `===`.
- `toEqual` глубокое сравнение, но типы leaf-значений тоже strict.
- `==` допустим только при явном legacy контракте и комментарии в тест-плане.
- Особые кейсы: `NaN`, `+0/-0` через `Object.is`.
- Честно: `null == undefined` true ломает тест, если ждали только null.

## Как это работает

```
console.log(5 == "5");
console.log(5 === "5");
console.log(null == undefined);
console.log(null === undefined);
```

Правильный assert:

```
expect(response.status).toBe(200);
expect(payload.count).toBe(0);
```

В pytest зеркалю strict policy:

```python
def strict_equal(left, right) -> bool:
    return type(left) is type(right) and left == right


def test_strict_rejects_cross_type():
    assert strict_equal(5, 5)
    assert not strict_equal(5, "5")
```

ESLint rule `eqeqeq` включён в CI для prod и test пакетов.

## Подводные камни

**Object.is vs ===.** `Object.is(NaN, NaN)` true, `===` false.

**Массивы через ==.** Сравнение ссылок, не содержимого.

**Coercion в template API.** Query param string `"true"` truthy в `if (flag)`.

**Snapshot string number.** Обновление snapshot маскирует type regression.

**== с объектами.** Приводит к `[object Object]`, редкий но confusing fail.
