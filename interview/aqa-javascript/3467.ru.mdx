# var, let и const: область видимости и hoisting

## Суть

`var` привязан к функции или глобальному объекту, hoisting инициализирует `undefined` до строки объявления. `let` и `const` блочные, попадают в temporal dead zone до объявления: обращение раньше даёт ReferenceError. `const` запрещает переназначение ссылки, но mutating полей объекта или массива допустим.

## Что ответить на собесе

- В новых тестах на Jest использую только `let`/`const`, `var` оставляю для legacy suite audit.
- Цикл с `var` и setTimeout даёт classic bug, фикс `let` или IIFE.
- `const` для page object selector map, `let` для mutable counter в loop.
- ESLint `no-var` и `prefer-const` в CI ловят регресс.
- Честно: `const` не делает объект immutable, для freeze нужен `Object.freeze` в fixture.

## Как это работает

Hoisting у var:

```
console.log(tag); // undefined
var tag = "smoke";
```

TDZ у let:

```
{
  // console.log(step); ReferenceError
  let step = 1;
}
```

const и мутация:

```
const selectors = { submit: "#send" };
selectors.submit = "#submit-btn"; // ok
// selectors = {} // TypeError
```

Таблица для pytest docstring:

```python
SCOPES = {
    "var": "function or global",
    "let": "block",
    "const": "block, binding frozen",
}


def test_scope_cheatsheet():
    assert SCOPES["let"] == "block"
    assert "frozen" in SCOPES["const"]
```

Loop trap:

```
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 3,3,3
}
for (let j = 0; j < 3; j++) {
  setTimeout(() => console.log(j), 0); // 0,1,2
}
```

## Подводные камни

**var в if без block до ES6.** Одна переменная на всю функцию.

**const в for-in.** Ошибка, нужен let.

**Shadowing параметра функции.** `let x` внутри дублирует arg, путает отладку.

**global var в jsdom.** Утечка между тестами без isolateModules.

**Re-declare var.** Второй var в той же функции legal, скрывает intent.
