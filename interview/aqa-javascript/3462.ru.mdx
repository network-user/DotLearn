# Как в JavaScript определяется this

## Суть

`this` задаётся способом вызова, не местом объявления (кроме стрелок и bind). В strict mode глобальный вызов даёт `undefined`, в браузере без strict ссылка на `window`. Метод `obj.run()` получает `this === obj`. `new Fn()` создаёт новый объект как `this`. `call/apply/bind` задают `this` явно.

## Что ответить на собесе

- Потерянный `this` при `const fn = obj.run; fn()` лечу bind или стрелкой-обёрткой.
- В class field arrow method `this` лексический, удобен в page object callback.
- В event listener обычная function меняет `this` на target element.
- Для теста spy проверяю, что метод вызван с ожидаемым контекстом (`toHaveBeenCalledWith`).
- Честно: двойной bind не откатить без сохранения оригинала, teardown в afterEach обязателен.

## Как это работает

Метод объекта:

```
const panel = {
    label: "Orders",
    title() {
        return this.label;
    },
};
panel.title();
```

Потеря контекста:

```
const read = panel.title;
read();
```

Восстановление:

```
const read = panel.title.bind(panel);
```

В pytest модель контекста:

```python
class Panel:
    def __init__(self, label: str):
        self.label = label

    def title(self) -> str:
        return self.label


def test_method_receives_instance_context():
    panel = Panel("Orders")
    assert panel.title() == "Orders"
```

Стрелка внутри метода наследует `this` метода, не DOM target.

## Подводные камни

**Callback без bind в setTimeout.** `this` global/undefined, assert на state падает.

**Destructuring method.** `const { title } = panel` отрывает от контекста.

**Strict vs sloppy global.** Разный `this` в helper, тест flaky между node и browser.

**jest.spyOn без restore.** Подменённый метод тащит чужой `this` в следующий spec.

**Class method as handler.** Нужен field arrow или bind в constructor.
