# Falsy-значения в JavaScript и проверки в тестах

## Суть

Falsy-значения приводятся к `false` в булевом контексте: `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, `NaN`. Все остальные значения truthy, включая `[]`, `{}` и `"0"`. В автотестах неявное приведение даёт ложноположительные и ложноотрицательные assert.

## Что ответить на собесе

- Полный список falsy фиксирую в чеклисте code review для condition в проде и в тестах.
- Пустая строка и ноль разные кейсы, но оба falsy: проверяю явно `===`.
- `document.querySelector` возвращает `null`, не `undefined`, в guard пишу отдельную ветку.
- В Playwright `expect(locator).toBeVisible()` не заменяет бизнес-проверку на пустой текст.
- Честно: truthy `"0"` и `[]` часто проходят `if (value)` там, где QA ждёт «нет данных».

## Как это работает

Перебираю набор в тестовом harness:

```
const candidates = [false, 0, -0, 0n, "", null, undefined, NaN];
candidates.forEach((v) => console.log(Boolean(v)));
```

Truthy-ловушки для UI:

```
if ([]) { /* входим */ }
if ("0") { /* входим */ }
```

В pytest генерирую таблицу ожиданий для JS runner:

```python
import pytest

FALSY = [False, 0, -0, "", None, float("nan")]
TRUTHY_TRAP = ["0", [], {}, "false"]


@pytest.mark.parametrize("value", FALSY)
def test_must_use_strict_check_for_falsy(value):
    assert value is not False or value is False
    if value == "":
        assert value == ""


def test_truthy_traps_are_not_falsy_by_coercion():
    for item in TRUTHY_TRAP:
        assert item is not None
        assert item != ""
        assert item != 0
```

Для `NaN` сравниваю через `Number.isNaN`, не через `==`.

## Подводные камни

**Двойное отрицание `!!`.** Маскирует тип, в отчёте теряется исходное значение.

**Default parameter.** `function f(x = 1)` отличает `undefined` от `null` и `0`.

**Optional chaining с 0.** `count ?? 1` сохраняет ноль, `count || 1` заменяет ноль единицей.

**Пустой input в форме.** `value=""` falsy, но поле присутствует в DOM.

**Сравнение с Python None.** В bridge-тестах путают `null` JS и `None` Python.
