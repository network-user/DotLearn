# Объект arguments и variadic-хелперы в тестах

## Суть

`arguments` это array-like объект внутри обычной (не стрелочной) функции: индексный доступ и `length`, но без методов массива. В legacy helper-коде через него суммируют или форматируют произвольное число параметров. В современных автотестах предпочитаю rest-параметр `...items`.

## Что ответить на собесе

- `arguments[i]` и `arguments.length` работают в function declaration/expression.
- В стрелочной функции `arguments` нет, беру `...args` из внешнего scope или rest.
- Для `map/filter` конвертирую через `Array.from(arguments)` или spread.
- В TypeScript типизирую rest, а не `arguments`, чтобы не терять проверку в CI.
- Честно: старые utility в node_modules ещё на `arguments`, при рефакторе ломаются моки.

## Как это работает

Классический сумматор:

```
function total() {
    let sum = 0;
    for (let i = 0; i < arguments.length; i++) {
        sum += arguments[i];
    }
    return sum;
}
```

Современная замена с rest:

```
const total = (...values) => values.reduce((acc, n) => acc + n, 0);
```

Проверяю контракт helper в pytest:

```python
def compute_total(*values: int) -> int:
    return sum(values)


def test_variadic_helper_matches_legacy_contract():
    assert compute_total(1, 2, 3) == 6
    assert compute_total() == 0
    assert compute_total(-1, 5) == 4
```

В UI wrapper передаю список селекторов как rest, а не через `arguments.callee`.

## Подводные камни

**Стрелка в callback.** Внутри `array.map(() => ...)` нет своего `arguments` родителя.

**Изменение `arguments` в non-strict.** Присваивание `arguments[0]` может синхронизироваться с параметром.

**Производительность цикла.** `for..of` по rest читаемее и быстрее в V8 на больших наборах.

**Связка с `callee`.** Удалена в strict, ломает рекурсивные legacy helper.

**Смешение rest и arguments.** Дублирование источника правды для arity-check.
