# Mocks, stubs и spies в unit-тестах

## Суть

Test double изолирует SUT от медленных или недоступных зависимостей. Stub подменяет ответ без проверки взаимодействия. Mock задаёт ожидания вызовов и verify параметров. Spy оборачивает реальную функцию, записывает вызовы, но может пропускать оригинал. В Jest `jest.fn` ближе к mock/spy, `jest.spyOn` сохраняет implementation.

## Что ответить на собесе

- Stub HTTP client возвращает fixture JSON, не хожу в prod API в unit.
- Mock проверяю `toHaveBeenCalledWith` для audit log side effect.
- Spy на logger смотрю, что error записан, но не глушу console.
- Не mock то, что тестируешь: только границы модуля.
- Честно: over-mocking делает тест хрупким к refactor без изменения behavior.

## Как это работает

Stub:

```
const api = { fetchUser: () => ({ id: 1, role: "qa" }) };
const profile = buildProfile(api.fetchUser());
expect(profile.label).toBe("qa");
```

Mock:

```
const notify = jest.fn();
service.onFail(notify);
expect(notify).toHaveBeenCalledTimes(1);
```

Spy:

```
const math = { sum: (a, b) => a + b };
const spy = jest.spyOn(math, "sum");
expect(math.sum(2, 3)).toBe(5);
expect(spy).toHaveBeenCalledWith(2, 3);
spy.mockRestore();
```

Python pytest с unittest.mock:

```python
from unittest.mock import MagicMock, patch


def test_stub_vs_mock():
    repo = MagicMock()
    repo.get_user.return_value = {"id": 1, "role": "qa"}
    assert repo.get_user(1)["role"] == "qa"
    repo.get_user.assert_called_once_with(1)


def test_spy_on_real():
    calls = []

    def emit(event: str) -> None:
        calls.append(event)

    with patch(__name__ + ".emit", wraps=emit) as spy:
        emit("started")
        assert spy.call_count == 1
```

## Подводные камни

**Mock без assert.** Silent pass, баг в production.

**Spy без restore.** Утечка между тестами, порядок suite важен.

**Stub слишком умный.** Дублирует prod logic, ложная уверенность.

**Partial mock class.** Constructor real, method mock: `new` side effects остаются.

**jest.clearAllMocks vs reset.** clear счётчик, reset снимает implementation.
