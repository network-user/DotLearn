# Double negation !! for boolean coercion

## In short

The `!!` operator applies logical NOT twice and turns any value into strict `true` or `false` using truthy/falsy rules. The first `!` inverts to boolean; the second yields the normalized boolean result. It is shorter than `Boolean(x)` with the same semantics.

## How to answer in an interview

- For a feature flag guard I write `if (!!process.env.CI)` only when I need a real boolean; otherwise I prefer an explicit comparison.
- For DOM expects: `expect(!!node.textContent).toBe(true)`.
- Falsy: `0`, `""`, `null`, `undefined`, `NaN`, `false`.
- Truthy: `{}`, `[]`, any non-empty string, non-zero number.
- Honestly: `!![]` being true confuses juniors; in asserts I prefer `.toBeTruthy()` with clear intent.

## How it works

```
const hasCases = !!suite.cases?.length;
const idle = !!0;
const pending = !!"";

console.log(hasCases, idle, pending); // true false false
```

Equivalent:

```
Boolean(suite.cases?.length);
```

Table in pytest:

```python
TRUTHY_SAMPLES = ["ok", 1, [1], {"a": 1}]
FALSY_SAMPLES = ["", 0, None, [], {}]


def to_bool(value) -> bool:
    return bool(value)


def test_double_negation_matches_bool():
    for sample in TRUTHY_SAMPLES:
        assert to_bool(sample) is True
    for sample in FALSY_SAMPLES:
        if sample == [] or sample == {}:
            assert to_bool(sample) is True  # Python: empty list/dict truthy differs
        else:
            assert to_bool(sample) is False
```

## Pitfalls

**Document.all.** A legacy exotic object behaves like both undefined and object.

**!! on querySelector miss.** null becomes false; an empty div without text is also falsy for content checks.

**Nested !! in templates.** `#{!!flag}` is Ruby; in JS it adds noise.

**Optional chaining shortcut.** `!!obj?.enabled` vs `obj?.enabled === true` differ when value is `undefined`.

**Lint no-extra-boolean-cast.** ESLint flags `!!` inside if conditions; I keep it when the API must return boolean.
