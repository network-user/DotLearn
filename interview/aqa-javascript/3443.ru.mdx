# Два способа объявить функцию в JavaScript

## Суть

В JS функции задают через Function Declaration (именованное объявление) или Function Expression (функция как значение переменной). Declaration поднимается (hoisting) и доступна до строки определения; expression создаётся в момент выполнения присваивания. Для AQA это влияет на порядок хуков и helper-функций в тестовых файлах.

## Что ответить на собесе

- Declaration: `function greet(name) { return ... }`, вызываю до объявления в том же scope.
- Expression: `const greet = function(name) { ... }` или стрелочная форма; до присваивания ReferenceError.
- В page object держу expression, чтобы не засорять глобальный scope Playwright/Cypress.
- Named function expression помогает в stack trace при падении UI-теста.
- Честно: hoisting declaration иногда маскирует циклические зависимости между helper-модулями.

## Как это работает

Declaration поднимается целиком:

```
function buildUrl(path) {
    return `https://api.test${path}`;
}
```

Expression привязываю к константе:

```
const buildUrl = function(path) {
    return `https://api.test${path}`;
};
```

В pytest-обвязке для JS API-тестов зеркалю различие через порядок импорта:

```python
import pytest


def test_declaration_available_before_line():
    source = """
    const result = greet('qa');
    function greet(name) { return `hi ${name}`; }
    """
    assert "greet" in source


def test_expression_requires_order():
    ordered = """
    const greet = (name) => `hi ${name}`;
    const result = greet('qa');
    """
    assert ordered.index("const greet") < ordered.index("greet('qa')")
```

В CI линтер запрещает function declaration внутри блоков, expression проходит.

## Подводные камни

**Hoisting в условных блоках.** В strict mode declaration внутри `if` ведёт себя непредсказуемо между движками.

**Анонимный expression в стеке.** Падающий тест показывает `(anonymous)` вместо имени helper.

**Перезапись const.** Expression нельзя переобъявить, declaration в том же scope конфликтует.

**IIFE vs expression.** Путают самовызывающуюся обёртку с обычным присваиванием.

**Смешение стилей в suite.** Часть файлов на declaration, часть на arrow ломает единый code style gate.
