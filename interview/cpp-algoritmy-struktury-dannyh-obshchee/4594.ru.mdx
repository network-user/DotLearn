# Что такое treap (декартово дерево)?

## Суть

Treap совмещает BST по ключу и heap по случайному приоритету. Ключи упорядочены как в дереве поиска, приоритеты родителя выше детей (max-heap). Вставка идёт по ключу, затем вращениями восстанавливают heap-order. Средняя высота O(log n) при случайных приоритетах.

## Что ответить на собесе

- Определяю пару (key, priority): BST invariant по key, heap invariant по priority.
- Объясняю вставку: спуск по ключу, затем rotate up если приоритет ребёнка больше.
- Сравниваю с AVL/rb-tree: проще код, рандомизация вместо строгих балансировок.
- Упоминаю split/merge по ключу или приоритету в конкурентном программировании.
- Честно добавляю: при плохих приоритетах высота деградирует; в STL treap нет, есть `std::map` и ручные структуры.

## Как это работает

Рекурсивная вставка размещает узел как в BST. Если приоритет нового узла выше родителя, выполняется поворот (как в AVL), поднимая узел и сохраняя порядок ключей.

```cpp
#include <iostream>
#include <cstdlib>

struct TreapNode {
    int key;
    int priority;
    TreapNode* left = nullptr;
    TreapNode* right = nullptr;
    TreapNode(int k, int p) : key(k), priority(p) {}
};

TreapNode* rotate_right(TreapNode* y) {
    TreapNode* x = y->left;
    y->left = x->right;
    x->right = y;
    return x;
}

TreapNode* rotate_left(TreapNode* x) {
    TreapNode* y = x->right;
    x->right = y->left;
    y->left = x;
    return y;
}

TreapNode* insert(TreapNode* root, int key, int priority) {
    if (!root) return new TreapNode(key, priority);
    if (key < root->key) {
        root->left = insert(root->left, key, priority);
        if (root->left->priority > root->priority) root = rotate_right(root);
    } else {
        root->right = insert(root->right, key, priority);
        if (root->right->priority > root->priority) root = rotate_left(root);
    }
    return root;
}

void inorder(TreapNode* root) {
    if (!root) return;
    inorder(root->left);
    std::cout << root->key << ' ';
    inorder(root->right);
}

int main() {
    TreapNode* root = nullptr;
    for (int k : {50, 30, 20, 40, 70, 60, 80}) {
        root = insert(root, k, std::rand());
    }
    inorder(root);
}
```

Inorder обход выводит ключи отсортированно, независимо от формы дерева по приоритетам.

## Подводные камни

- **Случайность.** `std::rand()` для продакшена слаб; нужен качественный RNG.
- **Дубликаты ключей.** Нужна явная политика (левое/правое поддерево для равных).
- **Указатели и память.** В реальном коде предпочитают RAII или implicit treap на индексах.
- **Конкурентность.** Без lock-free дизайна вращения сложно делать thread-safe.
- **Не путать с Cartesian tree по массиву.** Там приоритеты фиксированы позицией, не random.
