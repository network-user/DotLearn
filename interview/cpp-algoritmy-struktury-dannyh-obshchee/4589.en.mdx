# What collision resolution strategies exist?

## In short

When two keys map to one index, the hash table picks a collision strategy. The two main families are chaining (chains per bucket) and open addressing (all entries in a slot array with probing). Open addressing includes linear probing, quadratic probing, and double hashing.

## How to answer in an interview

- I list chaining and open addressing as the main branches; mainstream STL `std::unordered_map` uses chaining.
- For open addressing I name linear probing, quadratic probing, and double hashing.
- I compare trade-offs: chaining tolerates high load, open addressing saves pointers but clusters.
- I mention tombstones on delete in open addressing.
- I add honestly: production often inherits the standard library choice; interviews want the idea, not a libc rewrite.

## How it works

**Chaining:** a bucket holds a collection of entries sharing an index; insert appends, lookup scans the chain.

**Open addressing:** a slot array; on occupancy the next index follows a probe rule until an empty cell or the key appears.

```cpp
#include <iostream>
#include <list>
#include <optional>
#include <vector>

class ChainSet {
    std::vector<std::list<int>> buckets;
    int modulo;

public:
    explicit ChainSet(int m) : buckets(m), modulo(m) {}

    void add(int value) {
        auto& chain = buckets[value % modulo];
        for (int v : chain) if (v == value) return;
        chain.push_back(value);
    }

    bool has(int value) const {
        for (int v : buckets[value % modulo]) if (v == value) return true;
        return false;
    }
};

class LinearProbeSet {
    std::vector<std::optional<int>> cells;
    int modulo;
    static constexpr int kEmpty = -1;

public:
    explicit LinearProbeSet(int m) : cells(m), modulo(m) {}

    void add(int value) {
        int idx = value % modulo;
        for (int step = 0; step < modulo; ++step) {
            int slot = (idx + step) % modulo;
            if (!cells[slot].has_value() || cells[slot].value() == kEmpty) {
                cells[slot] = value;
                return;
            }
            if (cells[slot].value() == value) return;
        }
    }

    bool has(int value) const {
        int idx = value % modulo;
        for (int step = 0; step < modulo; ++step) {
            int slot = (idx + step) % modulo;
            if (!cells[slot].has_value() || cells[slot].value() == kEmpty) return false;
            if (cells[slot].value() == value) return true;
        }
        return false;
    }
};

int main() {
    ChainSet chained(8);
    LinearProbeSet probed(8);
    for (int v : {15, 23, 31}) {
        chained.add(v);
        probed.add(v);
    }
    std::cout << chained.has(23) << ' ' << probed.has(23) << '\n';
}
```

Quadratic probing steps as `1, 4, 9...`, easing primary clustering. Double hashing uses a second hash for the step size. Deletes in open addressing need tombstones or search stops too early.

## Pitfalls

- **Primary clustering.** Linear probing glues occupied runs; insert slows before paper load says so.
- **Load ceiling.** Open addressing needs empty slack; above ~0.7 without rehash performance collapses.
- **Delete in chaining.** Remove one node, not the whole bucket.
- **Probe/hash mismatch.** Double hashing needs an independent second function.
- **Robin Hood / cuckoo.** Interviews may mention advanced schemes; knowing the name and distance-balancing idea helps.
