# What is serialization in C++?

## In short

Serialization turns object state into a byte stream (file, network, buffer) for storage or transport. Deserialization rebuilds the object from that stream. C++ has no single standard: you craft a binary/text format or use a library (Protobuf, Boost.Serialization, JSON).

## How to answer in an interview

- I split encode (serialization) and decode (deserialization).
- I name use cases: persistence, IPC, snapshot caches, message queues.
- I stress schema versioning and endianness for binary formats.
- I note that `reinterpret_cast` writing a whole struct breaks on padding/architecture.
- I add honestly: public APIs usually pick JSON/Protobuf over a homemade binary layout.

## How it works

An explicit format writes metadata first (string length), then payload. Reading reverses the order. Nested structs serialize fields recursively.

```cpp
#include <fstream>
#include <string>

struct Profile {
    std::string name;
    int age = 0;

    void save(const std::string& path) const {
        std::ofstream out(path, std::ios::binary);
        auto len = name.size();
        out.write(reinterpret_cast<const char*>(&len), sizeof(len));
        out.write(name.data(), static_cast<std::streamsize>(len));
        out.write(reinterpret_cast<const char*>(&age), sizeof(age));
    }

    static Profile load(const std::string& path) {
        std::ifstream in(path, std::ios::binary);
        std::size_t len = 0;
        in.read(reinterpret_cast<char*>(&len), sizeof(len));
        Profile p;
        p.name.resize(len);
        in.read(p.name.data(), static_cast<std::streamsize>(len));
        in.read(reinterpret_cast<char*>(&p.age), sizeof(p.age));
        return p;
    }
};
```

Field order and type sizes become a contract across program versions.

## Pitfalls

- **Padding and alignment.** Binary struct dumps are not portable across compilers.
- **Endianness.** Raw `int` on the wire without `htonl` breaks across architectures.
- **Schema versions.** New fields without a version tag break old files.
- **Security.** Untrusted input: length overflow, zip-bomb style payloads.
- **Pointers and resources.** Serialize data, not addresses; object graphs need ids/refs.
