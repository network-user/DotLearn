# What is a treap?

## In short

A treap combines a BST on keys and a heap on random priorities. Keys stay search-tree ordered; parent priorities beat child priorities (max-heap). Insert by key, then rotations restore heap order. Random priorities yield O(log n) expected height.

## How to answer in an interview

- I define the (key, priority) pair: BST invariant on key, heap invariant on priority.
- I explain insert: descend by key, then rotate up if the child priority wins.
- I compare with AVL/red-black: simpler code, randomization instead of strict balance rules.
- I mention split/merge by key or priority in contest programming.
- I add honestly: bad priorities skew height; STL has no treap, only `std::map` or hand-rolled structures.

## How it works

Recursive insert places the node like a BST. If the new priority beats the parent, a rotation (as in AVL) lifts the node while preserving key order.

```cpp
#include <iostream>
#include <cstdlib>

struct TreapNode {
    int key;
    int priority;
    TreapNode* left = nullptr;
    TreapNode* right = nullptr;
    TreapNode(int k, int p) : key(k), priority(p) {}
};

TreapNode* rotate_right(TreapNode* y) {
    TreapNode* x = y->left;
    y->left = x->right;
    x->right = y;
    return x;
}

TreapNode* rotate_left(TreapNode* x) {
    TreapNode* y = x->right;
    x->right = y->left;
    y->left = x;
    return y;
}

TreapNode* insert(TreapNode* root, int key, int priority) {
    if (!root) return new TreapNode(key, priority);
    if (key < root->key) {
        root->left = insert(root->left, key, priority);
        if (root->left->priority > root->priority) root = rotate_right(root);
    } else {
        root->right = insert(root->right, key, priority);
        if (root->right->priority > root->priority) root = rotate_left(root);
    }
    return root;
}

void inorder(TreapNode* root) {
    if (!root) return;
    inorder(root->left);
    std::cout << root->key << ' ';
    inorder(root->right);
}

int main() {
    TreapNode* root = nullptr;
    for (int k : {50, 30, 20, 40, 70, 60, 80}) {
        root = insert(root, k, std::rand());
    }
    inorder(root);
}
```

Inorder prints keys sorted regardless of priority-driven shape.

## Pitfalls

- **Randomness.** `std::rand()` is weak for production; use a proper RNG.
- **Duplicate keys.** Pick a policy (left/right subtree for equals).
- **Pointers and memory.** Real code prefers RAII or implicit treap on indices.
- **Concurrency.** Rotations are hard to make thread-safe without a lock-free design.
- **Not the array Cartesian tree.** There priorities come from positions, not random draws.
