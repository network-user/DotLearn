# What is a hash map collision?

## In short

A collision happens when different keys hash to the same bucket (same slot index). That is expected: hashing maps a large key space into a finite bucket count. The container keeps multiple entries in one bucket and tells them apart with key comparison.

## How to answer in an interview

- I define it: equal bucket index, not equal keys.
- I explain collisions are inevitable by pigeonhole with any finite bucket count.
- I name strategies: chaining and open addressing.
- I show how a weak hash (e.g. `length % size`) creates misleading clusters.
- I add honestly: the goal is short chains and stable load factor, not zero collisions.

## How it works

The hash maps a key to an index. If the bucket already holds another key, the new entry joins a chain or probes for the next slot. Lookup picks the bucket first, then compares keys for equality.

```cpp
#include <iostream>
#include <list>
#include <string>
#include <vector>

class WordMap {
    std::vector<std::list<std::pair<std::string, int>>> slots;
    int modulo;

    int bucket(const std::string& word) const {
        return static_cast<int>(word.size() % modulo);
    }

public:
    explicit WordMap(int buckets) : slots(buckets), modulo(buckets) {}

    void put(const std::string& word, int score) {
        auto& chain = slots[bucket(word)];
        for (auto& item : chain) {
            if (item.first == word) {
                item.second = score;
                return;
            }
        }
        chain.emplace_back(word, score);
    }

    int get(const std::string& word) const {
        for (const auto& item : slots[bucket(word)]) {
            if (item.first == word) return item.second;
        }
        throw std::runtime_error("missing key");
    }
};

int main() {
    WordMap map(8);
    map.put("cat", 3);
    map.put("dog", 5);
    std::cout << map.get("cat") << '\n';
}
```

In the toy example, equal-length words share a bucket, but `get` still finds the right key via string compare.

## Pitfalls

- **Hash vs equality confusion.** Same hash or bucket does not mean same key.
- **Weak hash function.** Artificially inflates chains.
- **Ignoring rehash.** Growth without more buckets worsens collisions.
- **Collision counters alone.** The metric means little without load factor.
- **Security.** Crafted keys can target one bucket (HashDoS).
