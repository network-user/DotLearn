# Что такое хеш-таблица?

## Суть

Хеш-таблица хранит пары ключ/значение в массиве корзин. Хеш-функция переводит ключ в индекс; при коллизии несколько записей делят bucket. В среднем insert/find/erase близки к O(1). В C++ аналог: `std::unordered_map`.

## Что ответить на собесе

- Называю компоненты: hash, bucket array, collision policy, key_equal.
- Объясняю rehash при росте load factor.
- Сравниваю с `std::map`: hash быстрее в среднем, map даёт порядок ключей.
- Упоминаю применения: индексы, кэши, frequency tables.
- Честно добавляю: worst case O(n) при плохом hash или атаке; нужны reserve и качественный hasher.

## Как это работает

Вставка вычисляет bucket, добавляет узел в цепочку или слот. Поиск повторяет hash и сравнивает ключи. Удаление убирает узел из цепочки или помечает tombstone.

```cpp
#include <iostream>
#include <list>
#include <string>
#include <vector>

class IntDict {
    std::vector<std::list<std::pair<int, std::string>>> buckets;
    int modulo;

    int slot(int key) const { return key % modulo; }

public:
    explicit IntDict(int size) : buckets(size), modulo(size) {}

    void put(int key, const std::string& value) {
        auto& chain = buckets[slot(key)];
        for (auto& e : chain) {
            if (e.first == key) { e.second = value; return; }
        }
        chain.emplace_back(key, value);
    }

    std::string get(int key) const {
        for (const auto& e : buckets[slot(key)]) {
            if (e.first == key) return e.second;
        }
        return "missing";
    }

    void erase(int key) {
        auto& chain = buckets[slot(key)];
        for (auto it = chain.begin(); it != chain.end(); ++it) {
            if (it->first == key) { chain.erase(it); return; }
        }
    }
};

int main() {
    IntDict d(11);
    d.put(1, "alpha");
    d.put(12, "beta");
    std::cout << d.get(12) << '\n';
    d.erase(1);
}
```

Ключи 1 и 12 могут попасть в один bucket при modulo 11, но `get` различает их по полному ключу.

## Подводные камни

- **Слабый hash.** Главный враг производительности, не «магический» размер.
- **operator[] вставляет.** Для read-only lookup нужен `find`.
- **Итераторы и rehash.** Rehash инвалидирует итераторы.
- **Нестабильный порядок.** Нельзя полагаться на порядок обхода.
- **Сравнение с tree.** Нет range query «все ключи между A и B» без дополнительной структуры.
