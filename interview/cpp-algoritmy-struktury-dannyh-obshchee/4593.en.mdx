# What is a binary heap?

## In short

A binary heap is a complete binary tree with heap order: in a max-heap the parent is not less than its children, in a min-heap not greater. The root holds the extremum. It stores compactly in an array; `push`/`pop` are O(log n). It underpins `std::priority_queue`.

## How to answer in an interview

- I define complete tree plus heap property; I separate min and max variants.
- I explain vector indexing: parent `(i-1)/2`, children `2i+1`, `2i+2`.
- I name operations: insert via sift-up, extract via sift-down, O(log n).
- I tie it to priority queues and heap sort.
- I add honestly: arbitrary search is O(n); a heap does not sort the whole set, only fast extremum access.

## How it works

A new item goes at the array end (a leaf), then rises while heap order breaks. Max extract swaps root with the last leaf, drops the leaf, sifts the root down.

```cpp
#include <iostream>
#include <vector>
#include <stdexcept>

class MaxPQ {
    std::vector<int> heap;

    void sift_up(std::size_t idx) {
        while (idx > 0) {
            std::size_t parent = (idx - 1) / 2;
            if (heap[parent] >= heap[idx]) break;
            std::swap(heap[parent], heap[idx]);
            idx = parent;
        }
    }

    void sift_down(std::size_t idx) {
        const std::size_t n = heap.size();
        while (true) {
            std::size_t best = idx;
            std::size_t left = 2 * idx + 1;
            std::size_t right = 2 * idx + 2;
            if (left < n && heap[left] > heap[best]) best = left;
            if (right < n && heap[right] > heap[best]) best = right;
            if (best == idx) break;
            std::swap(heap[idx], heap[best]);
            idx = best;
        }
    }

public:
    void push(int value) {
        heap.push_back(value);
        sift_up(heap.size() - 1);
    }

    int pop_max() {
        if (heap.empty()) throw std::out_of_range("empty");
        int top = heap.front();
        heap.front() = heap.back();
        heap.pop_back();
        if (!heap.empty()) sift_down(0);
        return top;
    }
};

int main() {
    MaxPQ pq;
    for (int v : {3, 9, 1, 6, 5}) pq.push(v);
    std::cout << pq.pop_max() << '\n';
}
```

`std::make_heap`, `push_heap`, and `pop_heap` apply the same idea over an arbitrary range.

## Pitfalls

- **BST confusion.** Heap order does not sort an in-order walk; only the root is extremal.
- **0-based vs 1-based formulas.** Parent index mistakes are common in interviews.
- **Stability.** Equal keys may reorder during sift.
- **build_heap.** Linear build from an array beats n separate pushes.
- **Pairing with map.** Decrease-key often needs an auxiliary index per node.
