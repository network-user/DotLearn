# Что такое бинарная куча?

## Суть

Бинарная куча это полное бинарное дерево с heap-order: в max-heap родитель не меньше детей, в min-heap не больше. Корень даёт экстремум. На массиве хранится компактно; `push`/`pop` стоят O(log n). Основа `std::priority_queue`.

## Что ответить на собесе

- Даю определение complete tree + heap property; различаю min и max варианты.
- Объясняю индексацию в vector: parent `(i-1)/2`, children `2i+1`, `2i+2`.
- Называю операции: insert через sift-up, extract через sift-down, O(log n).
- Связываю с приоритетной очередью и heap sort.
- Честно добавляю: произвольный поиск O(n); куча не сортирует всё множество, только даёт быстрый extremum.

## Как это работает

Новый элемент кладут в конец массива (лист), затем поднимают, пока нарушен heap-order. Извлечение максимума: корень меняют с последним листом, удаляют лист, просеивают корень вниз.

```cpp
#include <iostream>
#include <vector>
#include <stdexcept>

class MaxPQ {
    std::vector<int> heap;

    void sift_up(std::size_t idx) {
        while (idx > 0) {
            std::size_t parent = (idx - 1) / 2;
            if (heap[parent] >= heap[idx]) break;
            std::swap(heap[parent], heap[idx]);
            idx = parent;
        }
    }

    void sift_down(std::size_t idx) {
        const std::size_t n = heap.size();
        while (true) {
            std::size_t best = idx;
            std::size_t left = 2 * idx + 1;
            std::size_t right = 2 * idx + 2;
            if (left < n && heap[left] > heap[best]) best = left;
            if (right < n && heap[right] > heap[best]) best = right;
            if (best == idx) break;
            std::swap(heap[idx], heap[best]);
            idx = best;
        }
    }

public:
    void push(int value) {
        heap.push_back(value);
        sift_up(heap.size() - 1);
    }

    int pop_max() {
        if (heap.empty()) throw std::out_of_range("empty");
        int top = heap.front();
        heap.front() = heap.back();
        heap.pop_back();
        if (!heap.empty()) sift_down(0);
        return top;
    }
};

int main() {
    MaxPQ pq;
    for (int v : {3, 9, 1, 6, 5}) pq.push(v);
    std::cout << pq.pop_max() << '\n';
}
```

`std::make_heap`, `push_heap`, `pop_heap` реализуют ту же идею над произвольным диапазоном.

## Подводные камни

- **Путаница с BST.** Heap-order не сортирует in-order обход; только корень экстремален.
- **0-based vs 1-based формулы.** На собесе легко ошибиться в индексах родителя.
- **Нестабильность.** Равные элементы могут менять порядок при sift.
- **build_heap.** Линейная сборка из массива быстрее n отдельных push.
- **pairing с map.** Для decrease-key часто нужен auxiliary индекс в узле.
