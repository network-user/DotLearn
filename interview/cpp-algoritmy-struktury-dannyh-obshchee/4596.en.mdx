# What is a balanced search tree?

## In short

A balanced search tree caps height at O(log n) so insert/find/erase stay logarithmic even on bad insert order. AVL keeps subtree height difference ≤ 1; red-black relaxes balance but rotates less often. `std::map`/`std::set` use rb-trees.

## How to answer in an interview

- I contrast with naive BST: sorted inserts become O(n) without balancing.
- I name AVL, red-black, B-trees (disks); C++ associative containers usually use rb-trees.
- I explain rotations after insert/erase to restore invariants.
- I mention ordered traversal as a map/set advantage over `unordered_map`.
- I add honestly: balancing adds constants and metadata; for tiny n, vector plus sort can win.

## How it works

Insert like a BST, then check balance on the way up. Skew triggers single or double rotations. AVL is stricter; rb-trees allow temporary skew but still guarantee log height.

```cpp
#include <algorithm>
#include <iostream>

struct AvlNode {
    int key;
    AvlNode* left = nullptr;
    AvlNode* right = nullptr;
    int height = 1;
};

int node_height(AvlNode* n) { return n ? n->height : 0; }

AvlNode* rotate_right(AvlNode* y) {
    AvlNode* x = y->left;
    AvlNode* t2 = x->right;
    x->right = y;
    y->left = t2;
    y->height = 1 + std::max(node_height(y->left), node_height(y->right));
    x->height = 1 + std::max(node_height(x->left), node_height(x->right));
    return x;
}

AvlNode* rotate_left(AvlNode* x) {
    AvlNode* y = x->right;
    AvlNode* t2 = y->left;
    y->left = x;
    x->right = t2;
    x->height = 1 + std::max(node_height(x->left), node_height(x->right));
    y->height = 1 + std::max(node_height(y->left), node_height(y->right));
    return y;
}

AvlNode* insert(AvlNode* node, int key) {
    if (!node) return new AvlNode{key};
    if (key < node->key) node->left = insert(node->left, key);
    else if (key > node->key) node->right = insert(node->right, key);
    else return node;

    node->height = 1 + std::max(node_height(node->left), node_height(node->right));
    int balance = node_height(node->left) - node_height(node->right);

    if (balance > 1 && key < node->left->key) return rotate_right(node);
    if (balance < -1 && key > node->right->key) return rotate_left(node);
    if (balance > 1 && key > node->left->key) {
        node->left = rotate_left(node->left);
        return rotate_right(node);
    }
    if (balance < -1 && key < node->right->key) {
        node->right = rotate_right(node->right);
        return rotate_left(node);
    }
    return node;
}
```

After inserts 10,20,30,40,50 an AVL does not degenerate into a list; height stays logarithmic.

## Pitfalls

- **Balanced vs complete.** A complete binary tree is not necessarily a BST.
- **AVL vs rb in practice.** AVL favors lookup; rb favors cheaper insert/delete on average.
- **Iterator invalidation.** Map erase invalidates only the removed iterator.
- **Key comparison.** Custom types need strict weak ordering.
- **B-trees on SSD.** On-disk indexes use different node models; do not mix with in-memory rb-trees.
