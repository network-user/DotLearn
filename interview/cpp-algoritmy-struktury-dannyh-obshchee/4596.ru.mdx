# Что такое сбалансированное дерево?

## Суть

Сбалансированное дерево поиска ограничивает высоту O(log n), чтобы insert/find/erase оставались логарифмическими даже при «плохом» порядке вставок. AVL держит разницу высот поддеревьев ≤ 1; красно-черное дерево ослабляет баланс, но дешевле ротаций. `std::map`/`std::set` используют rb-tree.

## Что ответить на собесе

- Контрастирую с наивным BST: отсортированная вставка даёт O(n) без балансировки.
- Называю AVL, red-black, B-tree (диски); для C++ чаще всего rb-tree в ассоциативных контейнерах.
- Объясняю ротации после insert/erase для восстановления инварианта.
- Упоминаю ordered traversal как плюс map/set против unordered_map.
- Честно добавляю: балансировка добавляет константу и память на цвет/высоту; для tiny n vector + sort может быть быстрее.

## Как это работает

Вставка идёт как в BST, затем проверяется баланс узлов на пути вверх. При перекосе выполняются single/double rotation. AVL строже, rb-tree допускает временный дисбаланс, но гарантирует log height.

```cpp
#include <algorithm>
#include <iostream>

struct AvlNode {
    int key;
    AvlNode* left = nullptr;
    AvlNode* right = nullptr;
    int height = 1;
};

int node_height(AvlNode* n) { return n ? n->height : 0; }

AvlNode* rotate_right(AvlNode* y) {
    AvlNode* x = y->left;
    AvlNode* t2 = x->right;
    x->right = y;
    y->left = t2;
    y->height = 1 + std::max(node_height(y->left), node_height(y->right));
    x->height = 1 + std::max(node_height(x->left), node_height(x->right));
    return x;
}

AvlNode* rotate_left(AvlNode* x) {
    AvlNode* y = x->right;
    AvlNode* t2 = y->left;
    y->left = x;
    x->right = t2;
    x->height = 1 + std::max(node_height(x->left), node_height(x->right));
    y->height = 1 + std::max(node_height(y->left), node_height(y->right));
    return y;
}

AvlNode* insert(AvlNode* node, int key) {
    if (!node) return new AvlNode{key};
    if (key < node->key) node->left = insert(node->left, key);
    else if (key > node->key) node->right = insert(node->right, key);
    else return node;

    node->height = 1 + std::max(node_height(node->left), node_height(node->right));
    int balance = node_height(node->left) - node_height(node->right);

    if (balance > 1 && key < node->left->key) return rotate_right(node);
    if (balance < -1 && key > node->right->key) return rotate_left(node);
    if (balance > 1 && key > node->left->key) {
        node->left = rotate_left(node->left);
        return rotate_right(node);
    }
    if (balance < -1 && key < node->right->key) {
        node->right = rotate_right(node->right);
        return rotate_left(node);
    }
    return node;
}
```

После серии вставок 10,20,30,40,50 AVL не вырождается в список, высота остаётся логарифмической.

## Подводные камни

- **Путаница «balanced» и «complete».** Complete binary tree не обязательно BST.
- **AVL vs rb на практике.** AVL быстрее на lookup, rb дешевле insert/delete в среднем.
- **Итератор invalidation.** Erase в map инвалидирует только удалённый итератор.
- **Сравнение ключей.** Для пользовательских типов нужен strict weak ordering.
- **B-tree на SSD.** Для on-disk индексов другая модель узлов; не смешивать с in-memory rb-tree.
