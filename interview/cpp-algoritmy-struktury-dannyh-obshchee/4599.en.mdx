# What is a hash table?

## In short

A hash table stores key/value pairs in a bucket array. A hash function maps a key to an index; on collision, multiple entries share a bucket. Average insert/find/erase stay near O(1). The C++ counterpart is `std::unordered_map`.

## How to answer in an interview

- I name parts: hash, bucket array, collision policy, key_equal.
- I explain rehash when load factor grows.
- I contrast with `std::map`: hash is faster on average, map orders keys.
- I mention uses: indexes, caches, frequency tables.
- I add honestly: worst case O(n) with a bad hash or attack; use reserve and a strong hasher.

## How it works

Insert hashes to a bucket, adds a node to a chain or slot. Lookup hashes again and compares keys. Erase removes from the chain or marks a tombstone.

```cpp
#include <iostream>
#include <list>
#include <string>
#include <vector>

class IntDict {
    std::vector<std::list<std::pair<int, std::string>>> buckets;
    int modulo;

    int slot(int key) const { return key % modulo; }

public:
    explicit IntDict(int size) : buckets(size), modulo(size) {}

    void put(int key, const std::string& value) {
        auto& chain = buckets[slot(key)];
        for (auto& e : chain) {
            if (e.first == key) { e.second = value; return; }
        }
        chain.emplace_back(key, value);
    }

    std::string get(int key) const {
        for (const auto& e : buckets[slot(key)]) {
            if (e.first == key) return e.second;
        }
        return "missing";
    }

    void erase(int key) {
        auto& chain = buckets[slot(key)];
        for (auto it = chain.begin(); it != chain.end(); ++it) {
            if (it->first == key) { chain.erase(it); return; }
        }
    }
};

int main() {
    IntDict d(11);
    d.put(1, "alpha");
    d.put(12, "beta");
    std::cout << d.get(12) << '\n';
    d.erase(1);
}
```

Keys 1 and 12 may share a bucket with modulo 11, but `get` tells them apart by full key.

## Pitfalls

- **Weak hash.** The main performance enemy, not a "magic" bucket count.
- **operator[] inserts.** Read-only lookup needs `find`.
- **Iterators and rehash.** Rehash invalidates iterators.
- **Unstable order.** Do not rely on traversal order.
- **Vs trees.** No native range query "all keys between A and B" without extra structure.
