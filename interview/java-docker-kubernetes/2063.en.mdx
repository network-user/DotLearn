# Docker container vs virtual machine

## In short

A container isolates processes on a shared host kernel via namespaces and cgroups: one OS kernel, separate PID/network/mount views. A VM emulates hardware with a hypervisor and runs a full guest OS per instance. Containers are lighter and start faster; VMs provide a stronger boundary between different OSes and kernels.

## How to answer in an interview

- A Docker container = process(es) with isolation, not a separate OS.
- A VM includes a guest kernel; you can run Windows next to Linux on one host.
- Containers share the host kernel: a kernel CVE affects all containers.
- Density: the same hardware usually fits more containers than VMs.
- Honestly: for legacy with custom kernel modules or a different OS, a VM is still simpler than a container.

## How it works

Running a Java service in a container:

```bash
docker run -d --name pricing-api \
  -p 8080:8080 \
  -e JAVA_OPTS="-Xms256m -Xmx512m" \
  pricing-api:3.0.1
```

Container stack:

```
┌─────────────────────────────┐
│  pricing-api (JVM process)  │  ← container
├─────────────────────────────┤
│  Docker engine / containerd │
├─────────────────────────────┤
│  Host Linux kernel          │
└─────────────────────────────┘
```

VM stack for comparison:

```
┌─────────────────────────────┐
│  Guest OS + JVM             │
├─────────────────────────────┤
│  Hypervisor (KVM/ESXi)      │
├─────────────────────────────┤
│  Host hardware              │
└─────────────────────────────┘
```

VM boot takes minutes (guest OS init); container cold start takes seconds (JVM + app only). VM memory overhead includes the entire guest OS; a container mainly pays app heap and metaspace.

## Pitfalls

- **"Container = VM".** Expecting full hardware isolation leads to compliance mistakes.
- **Host kernel coupling.** Containers on Windows/Linux/macOS behave differently (especially filesystem).
- **Privileged container.** `--privileged` nearly removes isolation; close to root on the host.
- **VM sprawl.** A full OS per microservice bloats RAM and patch surface.
- **Wrong tool for multi-OS.** A Linux container will not run Windows .NET without special runtimes.
