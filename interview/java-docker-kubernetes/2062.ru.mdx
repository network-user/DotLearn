# OpenShift: enterprise Kubernetes от Red Hat

## Суть

OpenShift это дистрибутив Kubernetes с дополнительными слоями для разработки и эксплуатации: веб-консоль, RBAC по проектам, встроенный registry, S2I-сборка образов из Git, Routes вместо «голого» Ingress. Ядро оркестрации совместимо с upstream API, но CLI и политики безопасности заточены под корпоративный контур.

## Что ответить на собесе

- OpenShift = Kubernetes + opinionated tooling (oc, console, built-in CI hooks).
- Project (namespace) изолирует команды; квоты и SCC ограничивают привилегии Pod.
- S2I собирает образ из репозитория без ручного Dockerfile в простых кейсах.
- Route публикует Service наружу с TLS и hostname на уровне платформы.
- Честно: vendor lock-in на Red Hat tooling; vanilla k8s манифесты иногда требуют адаптации (SCC, Routes).

## Как это работает

Типичный flow Java-сервиса:

```bash
oc new-project payments
oc new-app java:17~https://github.com/acme/payments.git
oc expose svc/payments
oc scale deployment/payments --replicas=3
```

Под капотом `new-app` создаёт BuildConfig (S2I или Docker strategy), ImageStream, Deployment и Service. `expose` генерирует Route с hostname вида `payments-payments.apps.cluster.example.com`.

Сравнение с plain Kubernetes:

| Аспект | Kubernetes | OpenShift |
|--------|------------|-----------|
| CLI | kubectl | oc (+ kubectl совместим) |
| Внешний доступ | Ingress + controller | Route (integrated) |
| Сборка образа | вне кластера (CI) | BuildConfig / S2I in-cluster |
| Security | PSA / custom | SecurityContextConstraints |

Java-разработчик получает единый путь от commit до running Pod без отдельного Jenkins, если команда приняла platform workflow.

## Подводные камни

- **SCC и arbitrary UID.** Образ с root может не стартовать без настройки SCC или USER в Dockerfile.
- **ResourceQuota.** Build Pod и runtime Pod делят квоту project; OOM при пике сборки.
- **ImagePullPolicy.** ImageStream tag `:latest` в dev != prod digest pinning.
- **Смешение oc/kubectl.** Некоторые OpenShift CR (Route, BuildConfig) kubectl не создаст без CRD awareness.
- **Upgrade cadence.** Minor OpenShift upgrade может deprecate API versions раньше, чем ожидает команда.
