# OpenShift: Red Hat enterprise Kubernetes

## In short

OpenShift is a Kubernetes distribution with extra layers for development and operations: web console, project RBAC, built-in registry, S2I image builds from Git, and Routes instead of bare Ingress. The orchestration core is upstream API compatible, but CLI and security policies target enterprise environments.

## How to answer in an interview

- OpenShift = Kubernetes + opinionated tooling (oc, console, built-in CI hooks).
- A Project (namespace) isolates teams; quotas and SCCs limit Pod privileges.
- S2I builds an image from a repo without a hand-written Dockerfile in simple cases.
- A Route exposes a Service externally with TLS and hostname at platform level.
- Honestly: vendor lock-in on Red Hat tooling; vanilla k8s manifests sometimes need adaptation (SCC, Routes).

## How it works

Typical Java service flow:

```bash
oc new-project payments
oc new-app java:17~https://github.com/acme/payments.git
oc expose svc/payments
oc scale deployment/payments --replicas=3
```

Under the hood, `new-app` creates BuildConfig (S2I or Docker strategy), ImageStream, Deployment, and Service. `expose` generates a Route with a hostname like `payments-payments.apps.cluster.example.com`.

Compared to plain Kubernetes:

| Aspect | Kubernetes | OpenShift |
|--------|------------|-----------|
| CLI | kubectl | oc (+ kubectl compatible) |
| External access | Ingress + controller | Route (integrated) |
| Image build | off-cluster (CI) | BuildConfig / S2I in-cluster |
| Security | PSA / custom | SecurityContextConstraints |

A Java developer gets one path from commit to running Pod without separate Jenkins if the team adopts the platform workflow.

## Pitfalls

- **SCC and arbitrary UID.** A root image may not start without SCC tuning or USER in the Dockerfile.
- **ResourceQuota.** Build Pods and runtime Pods share project quota; OOM during build peaks.
- **ImagePullPolicy.** ImageStream tag `:latest` in dev != prod digest pinning.
- **Mixing oc/kubectl.** Some OpenShift CRs (Route, BuildConfig) need CRD awareness with kubectl.
- **Upgrade cadence.** Minor OpenShift upgrades may deprecate API versions sooner than the team expects.
