# Architectural security patterns

## In short

Defense in depth: authn/authz (OAuth2/JWT/mTLS), encryption in transit (TLS) and at rest, network segmentation, WAF/rate limit, secrets manager, least privilege IAM, audit logs, zero trust service mesh. Security as an architecture constraint, not a bolt-on.

## How to answer in an interview

- mTLS or signed JWT between services; RBAC at API gateway.
- Secrets not in repo; rotate via vault.
- Edge rate limiting plus circuit breaker on auth failures.
- Honestly: compliance checklists (SOC2) drive many controls.

## How it works

```go
func authMiddleware(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		token := r.Header.Get("Authorization")
		claims, err := verifyJWT(token)
		if err != nil {
			http.Error(w, "unauthorized", http.StatusUnauthorized)
			return
		}
		ctx := context.WithValue(r.Context(), claimsKey, claims)
		next.ServeHTTP(w, r.WithContext(ctx))
	})
}
```

The gateway terminates TLS, validates the token, forwards internal mTLS to services with a scoped service account.

## Pitfalls

**JWT in localStorage XSS.** Prefer HttpOnly cookie or short-lived token.

**Shared service account for all pods.** Large blast radius.

**Flat network K8s.** Missing NetworkPolicy default deny.

**Log PII.** GDPR incident.

**Security only at perimeter.** Lateral movement unblocked inside.
