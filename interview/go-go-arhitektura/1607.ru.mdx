# Паттерны architectural security

## Суть

Defense in depth: authn/authz (OAuth2/JWT/mTLS), encryption in transit (TLS) and at rest, network segmentation, WAF/rate limit, secrets manager, least privilege IAM, audit logs, zero trust service mesh. Security as architecture constraint, not bolt-on.

## Что ответить на собесе

- mTLS or signed JWT between services; RBAC at API gateway.
- Secrets not in repo; rotate via vault.
- Edge rate limiting + circuit breaker on auth failures.
- Честно: compliance checklist (SOC2) drives many controls.

## Как это работает

```go
func authMiddleware(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		token := r.Header.Get("Authorization")
		claims, err := verifyJWT(token)
		if err != nil {
			http.Error(w, "unauthorized", http.StatusUnauthorized)
			return
		}
		ctx := context.WithValue(r.Context(), claimsKey, claims)
		next.ServeHTTP(w, r.WithContext(ctx))
	})
}
```

Gateway terminates TLS, validates token, forwards internal mTLS to services with scoped service account.

## Подводные камни

**JWT in localStorage XSS.** Prefer HttpOnly cookie or short-lived token.

**Shared service account all pods.** Blast radius.

**Flat network K8s.** NetworkPolicy default deny missing.

**Log PII.** GDPR incident.

**Security only at perimeter.** Lateral movement unblocked inside.
