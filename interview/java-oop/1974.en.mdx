# How do you safely copy an object in Java?

## In short

Cloning in Java means creating another object with the same state. Common approaches: `Cloneable` + `clone()` (often shallow), copy constructor/factory, serialization (deep but heavy), or manual field copying. The choice depends on mutable nested objects.

## How to answer in an interview

- Explain shallow vs deep copy: copy references or recursively copy contents.
- Critique `Object.clone()`: protected, Cloneable marker, easy to forget deep copy.
- Recommend an explicit copy constructor or static factory like `of`/`copyOf`.
- Mention `record` with a compact constructor for immutable copies.
- Be honest: JSON round-trip (Gson/Jackson) works but is slow and loses types/cycles; not the primary choice for domain models.

## How it works

A copy constructor keeps control:

```java
public final class RoutePlan {
    private final List<String> stops;

    public RoutePlan(List<String> stops) {
        this.stops = List.copyOf(stops);
    }

    public RoutePlan(RoutePlan source) {
        this.stops = List.copyOf(source.stops);
    }
}
```

The `Cloneable` approach (shallow by default):

```java
public class TeamMember implements Cloneable {
    private String name;
    private int level;

    @Override
    protected TeamMember clone() {
        try {
            return (TeamMember) super.clone();
        } catch (CloneNotSupportedException ex) {
            throw new AssertionError(ex);
        }
    }
}
```

For deep copies of mutable graphs, walk fields explicitly or use serialization only when the type is already `Serializable` and performance is acceptable.

## Pitfalls

- **Shallow clone of shared mutable list.** Both copies mutate one `ArrayList`.
- **Cloneable without override.** `clone()` throws `CloneNotSupportedException`.
- **Serialization copy.** Skips transient/static fields; security and performance risks.
- **Forgotten final field.** A copy constructor cannot reassign finals.
- **Missing defensive copies.** External mutable state still aliases through a "copy."
