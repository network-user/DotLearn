# Does Java support multiple inheritance?

## In short

Java forbids inheriting more than one class: only single inheritance along an `extends` chain. Multiple inheritance of behavior is achieved through several interfaces (`implements A, B`). That avoids the diamond problem for class implementation, while default-method conflicts on interfaces are resolved by explicit language rules.

## How to answer in an interview

- I answer: no multiple class inheritance, yes multiple interface implementation.
- I explain the diamond problem: two parents with the same method implementation from a common ancestor.
- I show an example class with two interfaces.
- I mention default-method conflict resolution (most specific, explicit override).
- I add honestly: composition is often cleaner than growing a list of interfaces on one type.

## How it works

```java
interface Durable {
    int maxCycles();
}

interface Portable {
    double weightKg();
}

final class FieldSensor implements Durable, Portable {
    private final int maxCycles;
    private final double weightKg;

    FieldSensor(int maxCycles, double weightKg) {
        this.maxCycles = maxCycles;
        this.weightKg = weightKg;
    }

    @Override
    public int maxCycles() {
        return maxCycles;
    }

    @Override
    public double weightKg() {
        return weightKg;
    }
}

void inspect(Durable durable, Portable portable) {
    System.out.println(durable.maxCycles());
    System.out.println(portable.weightKg());
}
```

`FieldSensor` cannot `extends` two classes, but it implements both contracts. A `Durable` reference sees only `maxCycles()`, a `Portable` reference only `weightKg()`. One object can be passed to APIs expecting different capabilities. If both interfaces inherited a default with the same signature, the class must override the method and choose `InterfaceName.super.method()`.

## Pitfalls

- **Trying to extend two classes.** The compiler rejects it at compile time.
- **Implicit diamond with defaults.** You forget to override conflicting default methods.
- **Interface explosion.** Ten marker interfaces on one class with no cohesion.
- **Confusing is-a with can-do.** Class inheritance for reuse, interfaces for roles.
- **Hidden hierarchy through interfaces.** Deep interface `extends` chains without documented rules.
