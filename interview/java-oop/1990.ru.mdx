# Что такое интерфейс в Java?

## Суть

Интерфейс в Java описывает контракт поведения без обязательного состояния экземпляра. Класс `implements` интерфейс и предоставляет реализации abstract-методов. С Java 8 интерфейс может содержать `default` и `static` методы; с Java 9+ private helper-методы. Интерфейсы дают полиморфизм и множественное «наследование» поведения.

## Что ответить на собесе

- Определяю интерфейс как capability contract: что объект умеет, не как устроен внутри.
- Объясняю отличие от abstract class: нет ctor state, multiple implements.
- Показываю полиморфный вызов через тип интерфейса.
- Упоминаю default methods для эволюции API без ломки implementors.
- Честно добавляю: интерфейс не заменяет модель данных; DTO и entity лучше держать отдельно от behavior contracts.

## Как это работает

```java
public interface AlertChannel {
    void deliver(String payload);

    default void deliverWithPrefix(String payload) {
        deliver("[ALERT] " + payload);
    }

    static AlertChannel noop() {
        return payload -> { };
    }
}

public final class PagerChannel implements AlertChannel {
    @Override
    public void deliver(String payload) {
        System.out.println("pager: " + payload);
    }
}

public final class AlertRouter {
    private final AlertChannel channel;

    AlertRouter(AlertChannel channel) {
        this.channel = channel;
    }

    void route(String text) {
        channel.deliverWithPrefix(text);
    }
}
```

`AlertRouter` зависит от `AlertChannel`, а не от `PagerChannel`. В runtime можно подставить mock, другую реализацию или `AlertChannel.noop()` в тестах. Default method `deliverWithPrefix` расширяет контракт без правки всех implementors. Static factory на интерфейсе даёт именованные варианты без отдельного utility class.

## Подводные камни

- **Fat interface.** Один интерфейс на 20 методов, implementor вынужден stub-ить половину.
- **Дублирование default в иерархии.** Diamond conflict при multiple inheritance интерфейсов.
- **Интерфейс как bag of constants.** Только `public static final` без поведения.
- **Leaky abstraction.** Метод возвращает concrete JDK type, привязывая callers.
- **Тест double без default.** Забыли override нового default после upgrade JDK/library.
