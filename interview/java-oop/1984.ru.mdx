# Что такое класс в Java и зачем он нужен?

## Суть

Класс в Java это тип, который описывает поля и методы будущих объектов. Он задаёт общий контракт и реализацию, а экземпляр (`new`) хранит конкретное состояние. Классы группируют данные и поведение, позволяя моделировать предметную область и переиспользовать код.

## Что ответить на собесе

- Определяю класс как blueprint: один класс, много независимых объектов с одинаковой структурой.
- Перечисляю состав: поля состояния, методы поведения, конструкторы, при необходимости static-члены.
- Подчёркиваю инкапсуляцию: `private` поля + публичный API через методы.
- Связываю с JVM: класс компилируется в `.class`, объект живёт в heap.
- Честно добавляю: record и enum тоже типы классов, но с ограниченной моделью изменения состояния.

## Как это работает

Класс объявляют ключевым словом `class`, затем создают экземпляры:

```java
public final class VehicleProfile {
    private String plateNumber;
    private int mileageKm;

    public VehicleProfile(String plateNumber, int mileageKm) {
        this.plateNumber = plateNumber;
        this.mileageKm = mileageKm;
    }

    public String plateNumber() {
        return plateNumber;
    }

    public void registerTrip(int deltaKm) {
        if (deltaKm > 0) {
            mileageKm += deltaKm;
        }
    }

    public void printSummary() {
        System.out.println("plate=" + plateNumber + ", km=" + mileageKm);
    }
}
```

`VehicleProfile` инкапсулирует `plateNumber` и `mileageKm`. Конструктор задаёт начальное состояние, `registerTrip` меняет пробег через контролируемый метод. Два объекта `new VehicleProfile(...)` независимы: изменение одного не затрагивает другой. Static-поля и методы принадлежат классу целиком и разделяются всеми экземплярами.

## Подводные камни

- **Anemic domain model.** Класс только с геттерами/сеттерами без инвариантов и поведения.
- **Mutable shared state.** Публичные поля или возврат внутренних коллекций без копии.
- **God class.** Один тип отвечает за всё: persistence, validation, HTTP, форматирование.
- **Нарушение LSP через наследование.** Подкласс ломает ожидания базового API.
- **Смешение уровней.** Бизнес-логика и инфраструктурные детали в одном классе без границ.
