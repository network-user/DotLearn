# Can an interface extend another interface?

## In short

Yes: a Java interface may extend one or more interfaces. The child merges ancestor contracts and may add new abstract, default, or static methods. A class implementing the child must satisfy the entire chain.

## How to answer in an interview

- Show syntax: `interface B extends A`.
- Clarify multiple extends applies to interfaces, not classes.
- Explain that default methods inherit and may be overridden.
- Relate to ISP: narrow interfaces compose into a broader role interface.
- Be honest: deep interface hierarchies are rarely needed; flat interfaces plus composition often suffice.

## How it works

```java
interface Eats { void consume(); }
interface Plays extends Eats { void frolic(); }

final class Terrier implements Plays {
    @Override
    public void consume() { System.out.println("kibble"); }

    @Override
    public void frolic() { System.out.println("fetch"); }
}
```

`Plays` requires both `consume` and `frolic`. Polymorphically, `Eats pet = new Terrier()` may call only `consume`.

## Pitfalls

- **Diamond defaults.** Two parents with the same default force explicit resolution in the implementor.
- **Breaking parent change.** A new abstract method in a base interface breaks downstream types unless you add a default.
- **Empty extends for naming only.** Adds confusion without new methods.
- **implements vs extends mix-up.** Classes implement; interfaces extend.
- **Over-wide super interface.** Clients depend on a fat contract instead of a narrow capability.
