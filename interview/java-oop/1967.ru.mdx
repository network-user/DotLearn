# Зачем в интерфейсах появились default-методы?

## Суть

Default-метод (Java 8+) несёт реализацию прямо в interface с модификатором `default`. Это позволяет расширять API интерфейса без поломки существующих implementors: новый метод получает поведение по умолчанию, которое класс может переопределить.

## Что ответить на собесе

- Объясняю мотивацию: эволюция JDK interfaces (`Collection`, `List`) без массовых правок реализаций.
- Показываю синтаксис `default void method() { ... }` внутри interface.
- Отличаю от static-методов interface: default участвует в полиморфизме экземпляра.
- Упоминаю конфликт diamond: класс должен явно выбрать реализацию, если два interface дают одинаковый default.
- Честно добавляю: злоупотребление default превращает interface в pseudo-base-class; для shared state по-прежнему нужен abstract class или композиция.

## Как это работает

```java
public interface Movable {
    void start();

    default void halt() {
        System.out.println("movement halted");
    }
}

public final class Scooter implements Movable {
    @Override
    public void start() {
        System.out.println("scooter rolling");
    }

    @Override
    public void halt() {
        System.out.println("scooter brake applied");
    }
}
```

`Scooter` обязан реализовать abstract `start`, а `halt` может унаследовать default или override. Вызов через ссылку `Movable m = new Scooter()` попадёт в переопределённую версию, если она есть.

Default-методы также позволяют добавлять utility поверх abstract API, например `Collection.stream()` без правки всех коллекций вручную.

## Подводные камни

- **Diamond conflict.** Два interface с одинаковым default требуют явного `InterfaceName.super.method()` в классе.
- **Override без @Override.** Легко случайно создать overload вместо override default-метода.
- **Visibility и inheritance.** Default public в interface; попытка сузить доступ в классе не компилируется.
- **Тестирование через mock.** Старые mock-фреймворки могли не знать про default; поведение нужно явно stub-ить.
- **Смешение ролей.** Interface с десятком default-методов дублирует abstract class и усложняет иерархию.
