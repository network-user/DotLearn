# Поддерживает ли Java множественное наследование?

## Суть

Java запрещает наследовать более одного класса: только single inheritance по цепочке `extends`. Множественное наследование поведения достигается через несколько интерфейсов (`implements A, B`). Это снимает diamond problem для реализации классов, но конфликты default-методов интерфейсов разрешаются явными правилами языка.

## Что ответить на собесе

- Отвечаю: multiple class inheritance нет, multiple interface implementation да.
- Объясняю diamond problem: два родителя с одной реализацией метода от общего предка.
- Показываю пример класса с двумя interfaces.
- Упоминаю default method conflict resolution (most specific, explicit override).
- Честно добавляю: composition часто чище, чем наращивание списка interfaces на одном типе.

## Как это работает

```java
interface Durable {
    int maxCycles();
}

interface Portable {
    double weightKg();
}

final class FieldSensor implements Durable, Portable {
    private final int maxCycles;
    private final double weightKg;

    FieldSensor(int maxCycles, double weightKg) {
        this.maxCycles = maxCycles;
        this.weightKg = weightKg;
    }

    @Override
    public int maxCycles() {
        return maxCycles;
    }

    @Override
    public double weightKg() {
        return weightKg;
    }
}

void inspect(Durable durable, Portable portable) {
    System.out.println(durable.maxCycles());
    System.out.println(portable.weightKg());
}
```

`FieldSensor` не может `extends` два класса, но реализует оба контракта. Переменная типа `Durable` видит только `maxCycles()`, типа `Portable` только `weightKg()`. Один объект можно передать в API, ожидающие разные capability. Если оба интерфейса унаследовали default с одинаковой сигнатурой, класс обязан переопределить метод и выбрать `InterfaceName.super.method()`.

## Подводные камни

- **Попытка extends двух классов.** Компилятор отклонит на этапе compile.
- **Неявный diamond с defaults.** Забыли override конфликтующих default methods.
- **Interface explosion.** Десять marker interfaces на одном классе без cohesion.
- **Путаница is-a и can-do.** Наследование класса для reuse, interfaces для ролей.
- **Скрытая иерархия через interfaces.** Глубокое extends interface без документированных правил.
