# Какие методы наследует каждый объект из Object?

## Суть

Класс `Object` даёт базовые операции: equality (`equals`, `hashCode`), строковое представление (`toString`), reflection (`getClass`), клонирование (`clone`), финализацию (`finalize`, deprecated) и координацию потоков (`wait`, `notify`, `notifyAll`). Большинство переопределяют для доменной семантики или оставляют дефолт осознанно.

## Что ответить на собесе

- Перечисляю ключевые методы и зачем они нужны в коллекциях и логах.
- Подчёркиваю контракт equals/hashCode для `HashMap`/`HashSet`.
- Уточняю: `getClass` final; `clone` protected и требует `Cloneable`.
- Отмечаю, что `finalize` deprecated since 9, вместо него `Cleaner`/`try-with-resources`.
- Честно добавляю: для value types предпочитаю `record`, где equals/hashCode/toString генерируются автоматически.

## Как это работает

```java
public final class SkuRef {
    private final String code;

    public SkuRef(String code) {
        this.code = code;
    }

    @Override
    public boolean equals(Object other) {
        if (this == other) return true;
        if (!(other instanceof SkuRef that)) return false;
        return code.equals(that.code);
    }

    @Override
    public int hashCode() {
        return code.hashCode();
    }

    @Override
    public String toString() {
        return "SkuRef(" + code + ")";
    }
}
```

`equals`/`hashCode` должны быть согласованы: equal objects share hash. `toString` помогает в логах и debugger. `getClass()` возвращает runtime class (полезно в equals вместо `instanceof` для strict type). Monitor methods (`wait`/`notify`) работают на intrinsic lock каждого instance.

## Подводные камни

- **equals без hashCode.** Ломает hash-based коллекции.
- **Наследование equals.** Symmetry ломается, если subclass добавляет поле без getClass check.
- **clone shallow copy.** Копируются ссылки на mutable поля; нужен deep copy или copy constructor.
- **toString с секретами.** Пароли и токены не должны попадать в лог через default или naive override.
- **wait/notify в бизнес-коде.** Легко получить deadlock; предпочитайте j.u.c.
