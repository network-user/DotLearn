# What is an interface in Java?

## In short

An interface in Java describes a behavioral contract without mandatory instance state. A class `implements` the interface and supplies implementations for abstract methods. Since Java 8, an interface may contain `default` and `static` methods; since Java 9+, private helper methods. Interfaces enable polymorphism and multiple inheritance of behavior.

## How to answer in an interview

- I define an interface as a capability contract: what an object can do, not how it is built inside.
- I explain the difference from an abstract class: no ctor state, multiple implements allowed.
- I show a polymorphic call through the interface type.
- I mention default methods for evolving APIs without breaking implementors.
- I add honestly: an interface does not replace a data model; DTOs and entities should stay separate from behavior contracts.

## How it works

```java
public interface AlertChannel {
    void deliver(String payload);

    default void deliverWithPrefix(String payload) {
        deliver("[ALERT] " + payload);
    }

    static AlertChannel noop() {
        return payload -> { };
    }
}

public final class PagerChannel implements AlertChannel {
    @Override
    public void deliver(String payload) {
        System.out.println("pager: " + payload);
    }
}

public final class AlertRouter {
    private final AlertChannel channel;

    AlertRouter(AlertChannel channel) {
        this.channel = channel;
    }

    void route(String text) {
        channel.deliverWithPrefix(text);
    }
}
```

`AlertRouter` depends on `AlertChannel`, not on `PagerChannel`. At runtime you can plug in a mock, another implementation, or `AlertChannel.noop()` in tests. The default method `deliverWithPrefix` extends the contract without editing every implementor. A static factory on the interface gives named variants without a separate utility class.

## Pitfalls

- **Fat interface.** One interface with 20 methods forces implementors to stub half of them.
- **Duplicated defaults in a hierarchy.** Diamond conflict with multiple interface inheritance.
- **Interface as a bag of constants.** Only `public static final` fields and no behavior.
- **Leaky abstraction.** A method returns a concrete JDK type and binds callers.
- **Test double without defaults.** You forget to override a new default after a JDK/library upgrade.
