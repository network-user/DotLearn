# Может ли interface extends другой interface?

## Суть

Да: interface в Java может наследовать один или несколько interfaces через `extends`. Дочерний interface объединяет контракты предков и может добавить новые abstract/default/static методы. Класс, реализующий дочерний interface, обязан покрыть всю цепочку.

## Что ответить на собесе

- Показываю синтаксис `interface B extends A`.
- Уточняю множественное extends только для interfaces, не для classes.
- Объясняю, что default methods наследуются и могут быть overridden.
- Связываю с ISP: узкие interfaces compose в более широкий role interface.
- Честно добавляю: глубокие interface hierarchies редко нужны; часто достаточно flat interfaces + composition.

## Как это работает

```java
interface Eats { void consume(); }
interface Plays extends Eats { void frolic(); }

final class Terrier implements Plays {
    @Override
    public void consume() { System.out.println("kibble"); }

    @Override
    public void frolic() { System.out.println("fetch"); }
}
```

`Plays` включает обязанность реализовать и `consume`, и `frolic`. Полиморфно можно принять `Eats pet = new Terrier()` и вызвать только `consume`.

## Подводные камни

- **Diamond default methods.** Два parent interface с одинаковым default требуют explicit resolve в implementor class.
- **Breaking change в parent.** Новый abstract method в base interface ломает все downstream interfaces и classes без default.
- **Interface inheritance ради naming.** Пустое extends без новых методов только путает типы.
- **implements vs extends путаница.** Class implements; interface extends.
- **Over-wide super interface.** Клиент зависит от fat contract вместо narrow capability.
