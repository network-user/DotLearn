# Как безопасно скопировать объект в Java?

## Суть

Клонирование в Java это создание копии объекта с тем же состоянием. Стандартные подходы: `Cloneable` + `clone()` (часто shallow), copy constructor/factory, сериализация (deep, но тяжёлая) или ручное копирование полей. Выбор зависит от того, есть ли mutable вложенные объекты.

## Что ответить на собесе

- Объясняю shallow vs deep copy: копируются ссылки или рекурсивно содержимое.
- Критикую `Object.clone()`: protected, Cloneable marker, легко забыть deep copy.
- Рекомендую явный copy constructor или static factory `of/copyOf`.
- Упоминаю `record` с компактным constructor для immutable copies.
- Честно добавляю: JSON round-trip (Gson/Jackson) работает, но медленный и теряет типы/циклы; для домена не primary choice.

## Как это работает

Copy constructor даёт контроль:

```java
public final class RoutePlan {
    private final List<String> stops;

    public RoutePlan(List<String> stops) {
        this.stops = List.copyOf(stops);
    }

    public RoutePlan(RoutePlan source) {
        this.stops = List.copyOf(source.stops);
    }
}
```

`Cloneable` approach (shallow по умолчанию):

```java
public class TeamMember implements Cloneable {
    private String name;
    private int level;

    @Override
    protected TeamMember clone() {
        try {
            return (TeamMember) super.clone();
        } catch (CloneNotSupportedException ex) {
            throw new AssertionError(ex);
        }
    }
}
```

Для deep copy mutable graph проще явно пройти поля или использовать serialization только если объект уже `Serializable` и performance приемлема.

## Подводные камни

- **Shallow clone shared mutable list.** Две копии меняют один `ArrayList`.
- **Cloneable без override.** `clone()` бросает `CloneNotSupportedException`.
- **Копирование через serialization.** Не копирует transient/static; security и performance риски.
- **Забытый final field.** Copy constructor не скомпилируется, если пытается reassigned final.
- **Defensive copy на входе и выходе.** Иначе «копия» всё равно alias на внешний mutable state.
