# Можно ли переопределить static-метод в Java?

## Суть

Static-методы в Java не переопределяются (override). Они привязаны к типу ссылки на этапе компиляции, а не к runtime-типу объекта. В подклассе можно объявить static-метод с той же сигнатурой, но это hiding, а не полиморфизм.

## Что ответить на собесе

- Чётко разделяю override (instance, virtual dispatch) и static hiding.
- Объясняю early binding: вызов `ref.staticMethod()` резолвится по declared type ссылки.
- Показываю, что `@Override` на static не компилируется.
- Сравниваю с instance-методом на той же иерархии для контраста.
- Честно добавляю: hiding легко спутать с override; в проде static в иерархиях лучше избегать.

## Как это работает

```java
class BaseNode {
    static void report() {
        System.out.println("base report");
    }

    void describe() {
        System.out.println("base describe");
    }
}

class LeafNode extends BaseNode {
    static void report() {
        System.out.println("leaf report");
    }

    @Override
    void describe() {
        System.out.println("leaf describe");
    }
}

class StaticDispatchDemo {
    public static void main(String[] args) {
        BaseNode ref = new LeafNode();

        BaseNode.report();
        LeafNode.report();

        ref.report();
        ref.describe();
    }
}
```

Вызов через класс (`BaseNode.report()`, `LeafNode.report()`) выбирает метод объявленного типа. Вызов `ref.report()` печатает `base report`, хотя объект `LeafNode`, потому что static не участвует в dynamic dispatch. `ref.describe()` печатает `leaf describe` благодаря override. Полиморфизм в Java работает для instance-методов; static остаётся class-level utility.

## Подводные камни

- **Ложное ожидание полиморфизма.** `parentRef.staticHelper()` всегда вызывает версию типа ссылки.
- **Скрытый баг при рефакторинге.** Переименовали static в родителе, в потомке остался старый hiding.
- **@Override на static.** Компилятор отклонит; это сигнал, что модель неверна.
- **Тесты через subclass instance.** Поведение static не проверяется полиморфным вызовом.
- **Static в deep hierarchy.** Сложно отследить, какая версия вызывается из generic API.
