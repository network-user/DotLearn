# Что такое объект в Java и из чего он состоит?

## Суть

Объект это экземпляр класса в heap: у него есть состояние (поля), поведение (методы) и идентичность (ссылка или адрес в памяти). Класс описывает чертёж; `new` создаёт конкретный экземпляр с собственными значениями полей.

## Что ответить на собесе

- Разделяю класс (описание типа) и объект (конкретный экземпляр в runtime).
- Называю три составляющие: state, behavior, identity.
- Показываю создание через конструктор и работу через ссылочную переменную.
- Уточняю, что два объекта с одинаковыми полями не равны по identity, если это разные экземпляры (если не переопределён equals).
- Честно добавляю: value objects (record, immutable types) намеренно стирают различие identity в пользу equals по значению.

## Как это работает

```java
public final class FleetVehicle {
    private String paintColor;
    private final String modelName;
    private final int modelYear;

    public FleetVehicle(String paintColor, String modelName, int modelYear) {
        this.paintColor = paintColor;
        this.modelName = modelName;
        this.modelYear = modelYear;
    }

    public void igniteEngine() {
        System.out.println(modelName + " engine on");
    }

    public void repaint(String paintColor) {
        this.paintColor = paintColor;
    }

    public String describe() {
        return modelYear + " " + modelName + " (" + paintColor + ")";
    }
}

public final class FleetDemo {
    public static void main(String[] args) {
        FleetVehicle unitA = new FleetVehicle("silver", "Courier", 2022);
        unitA.igniteEngine();
        unitA.repaint("navy");
        System.out.println(unitA.describe());
    }
}
```

Переменная `unitA` хранит ссылку на объект в heap. Методы получают неявный `this` и работают с полями этого экземпляра. JVM выделяет память, вызывает конструктор, а GC позже освобождает объект, когда на него не остаётся reachable-ссылок.

Объекты позволяют моделировать доменные сущности, инкапсулировать инварианты и переиспользовать класс для множества экземпляров с разным состоянием.

## Подводные камни

- **Путаница ссылка и объект.** Присваивание копирует ссылку, а не clone объекта.
- **== vs equals.** `==` сравнивает identity; для значений нужен контракт equals/hashCode.
- **Утечка this до конструктора.** Публикация `this` из конструктора даёт partially constructed object.
- **Mutable shared state.** Один объект, на который ссылаются несколько владельцев, требует синхронизации или immutability.
- **Слишком «анемичная» модель.** Объекты только с геттерами/сеттерами без поведения превращаются в DTO и теряют смысл ООП.
