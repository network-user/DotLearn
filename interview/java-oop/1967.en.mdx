# Why did interfaces get default methods?

## In short

A default method (Java 8+) carries implementation inside an interface with the `default` modifier. It lets you evolve interface APIs without breaking existing implementors: new methods ship with fallback behavior that classes may override.

## How to answer in an interview

- Explain the motivation: evolving JDK interfaces like `Collection` and `List` without mass edits to implementations.
- Show syntax: `default void method() { ... }` inside the interface.
- Contrast with interface static methods: defaults participate in instance polymorphism.
- Mention diamond conflicts: a class must pick explicitly when two interfaces supply the same default.
- Be honest: overusing defaults turns an interface into a pseudo base class; shared state still belongs in an abstract class or composition.

## How it works

```java
public interface Movable {
    void start();

    default void halt() {
        System.out.println("movement halted");
    }
}

public final class Scooter implements Movable {
    @Override
    public void start() {
        System.out.println("scooter rolling");
    }

    @Override
    public void halt() {
        System.out.println("scooter brake applied");
    }
}
```

`Scooter` must implement abstract `start`; for `halt` it can inherit the default or override. A call through `Movable m = new Scooter()` hits the override when present.

Defaults also add utilities atop abstract APIs, e.g. `Collection.stream()` without editing every collection by hand.

## Pitfalls

- **Diamond conflict.** Two interfaces with the same default force explicit `InterfaceName.super.method()` in the class.
- **Override without @Override.** Easy to create an overload instead of overriding a default.
- **Visibility rules.** Defaults are public in the interface; narrowing access in a class will not compile.
- **Mock frameworks.** Older mocks may ignore defaults; stub behavior explicitly in tests.
- **Role confusion.** An interface packed with defaults duplicates abstract class responsibilities.
