# What is a structure with prev and next links?

## In short

It is a **doubly linked list**: each node stores payload plus links to the previous and next element. You can traverse both directions and remove from the middle in O(1) when you already hold a reference to the node.

## How to answer in an interview

- Name the structure and contrast it with a singly linked list (next only).
- Describe node fields: value, prev, next; containers often track head and tail too.
- Relate to Java `LinkedList`: the JDK uses a doubly linked list with head and tail.
- Mention uses: LRU cache, deque, browser history.
- Be honest: random access is O(n); use `ArrayList` when you need indexed access.

## How it works

```java
final class HistoryNode {
    int pageId;
    HistoryNode previous;
    HistoryNode next;

    HistoryNode(int pageId) {
        this.pageId = pageId;
    }
}

final class BrowserHistory {
    private HistoryNode head;
    private HistoryNode tail;

    void visit(int pageId) {
        HistoryNode node = new HistoryNode(pageId);
        if (tail == null) {
            head = tail = node;
            return;
        }
        tail.next = node;
        node.previous = tail;
        tail = node;
    }

    void printForward() {
        for (HistoryNode cursor = head; cursor != null; cursor = cursor.next) {
            System.out.print(cursor.pageId + " ");
        }
    }
}
```

Tail insertion updates links on both sides. Middle removal rewires neighbors without shifting an array. Cost: two pointers per node plus boxing overhead if you store wrappers like `Integer`.

## Pitfalls

- **Null prev/next at edges.** Missing head/tail checks cause NPE when removing ends.
- **Concurrent modification.** Unsynchronized writers and iterators corrupt links.
- **Memory vs ArrayList.** Many small node objects hurt cache locality.
- **Accidental cycle.** Wrong rewiring creates an infinite traversal loop.
- **equals/hashCode on nodes.** Compare payload, not node reference identity.
