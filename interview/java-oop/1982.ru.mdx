# Что такое method overriding в Java?

## Суть

Override это предоставление подклассом новой реализации instance-метода с той же сигнатурой, что у supertype. JVM выбирает версию по runtime type объекта (virtual dispatch), что и даёт runtime polymorphism.

## Что ответить на собесе

- Отличаю override от overload: overload compile-time, override runtime.
- Требования: same name, params, compatible return (covariant), не weaker access.
- Рекомендую `@Override` для compile-time check.
- Упоминаю `super.method()` для расширения, не замены поведения.
- Честно добавляю: private/static/final methods не override; hiding static another story.

## Как это работает

```java
class Alert {
    void signal() {
        System.out.println("generic alert");
    }
}

class SirenAlert extends Alert {
    @Override
    void signal() {
        System.out.println("siren pulse");
    }
}

public final class OverrideDemo {
    public static void main(String[] args) {
        Alert base = new SirenAlert();
        base.signal();
    }
}
```

Static type `Alert`, runtime `SirenAlert` печатает siren. Без `@Override` опечатка в signature создаст overload и silently skip polymorphism.

## Подводные камни

- **Accidental overload.** Изменили parameter type думая, что override.
- **Weaker access.** protected/public mismatch не компилируется.
- **equals signature error.** `equals(OtherType)` не override `equals(Object)`.
- **Override static method.** Static hiding, не polymorphism; confusing behavior.
- **super call order.** Неверный порядок в constructor + override может видеть zero state.
