# What is a class in Java and why do we need it?

## In short

A class in Java is a type that describes the fields and methods of future objects. It defines a shared contract and implementation, while an instance (`new`) holds concrete state. Classes group data and behavior so you can model the domain and reuse code.

## How to answer in an interview

- I define a class as a blueprint: one class, many independent objects with the same structure.
- I list what it contains: state fields, behavior methods, constructors, and static members when needed.
- I stress encapsulation: `private` fields plus a public API through methods.
- I connect it to the JVM: the class compiles to a `.class` file, the object lives on the heap.
- I add honestly: records and enums are class-like types too, but with a limited model for changing state.

## How it works

You declare a class with the `class` keyword, then create instances:

```java
public final class VehicleProfile {
    private String plateNumber;
    private int mileageKm;

    public VehicleProfile(String plateNumber, int mileageKm) {
        this.plateNumber = plateNumber;
        this.mileageKm = mileageKm;
    }

    public String plateNumber() {
        return plateNumber;
    }

    public void registerTrip(int deltaKm) {
        if (deltaKm > 0) {
            mileageKm += deltaKm;
        }
    }

    public void printSummary() {
        System.out.println("plate=" + plateNumber + ", km=" + mileageKm);
    }
}
```

`VehicleProfile` encapsulates `plateNumber` and `mileageKm`. The constructor sets the initial state, and `registerTrip` updates mileage through a controlled method. Two `new VehicleProfile(...)` objects are independent: changing one does not affect the other. Static fields and methods belong to the class as a whole and are shared by every instance.

## Pitfalls

- **Anemic domain model.** A class with only getters/setters and no invariants or behavior.
- **Mutable shared state.** Public fields or returning internal collections without a copy.
- **God class.** One type handles everything: persistence, validation, HTTP, formatting.
- **LSP violation through inheritance.** A subclass breaks expectations of the base API.
- **Mixed layers.** Business logic and infrastructure details in one class with no boundaries.
