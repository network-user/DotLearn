# Как называется структура с prev и next ссылками?

## Суть

Это **двусвязный список** (doubly linked list): каждый узел хранит payload и ссылки на предыдущий и следующий элемент. Обход возможен в обе стороны, удаление из середины делается за O(1), если уже есть ссылка на узел.

## Что ответить на собесе

- Называю структуру и отличаю от singly linked list (только next).
- Объясняю поля узла: value, prev, next; часто ещё head/tail у контейнера.
- Сравниваю с `LinkedList` в Java: JDK реализует doubly linked list с head и tail.
- Упоминаю применения: LRU cache, deque, browser history.
- Честно добавляю: random access O(n); для индексного доступа лучше `ArrayList`.

## Как это работает

```java
final class HistoryNode {
    int pageId;
    HistoryNode previous;
    HistoryNode next;

    HistoryNode(int pageId) {
        this.pageId = pageId;
    }
}

final class BrowserHistory {
    private HistoryNode head;
    private HistoryNode tail;

    void visit(int pageId) {
        HistoryNode node = new HistoryNode(pageId);
        if (tail == null) {
            head = tail = node;
            return;
        }
        tail.next = node;
        node.previous = tail;
        tail = node;
    }

    void printForward() {
        for (HistoryNode cursor = head; cursor != null; cursor = cursor.next) {
            System.out.print(cursor.pageId + " ");
        }
    }
}
```

Вставка в конец обновляет `tail` и связывает `previous`/`next`. Удаление из середины перешивает соседей без сдвига массива. Память: два pointer на узел плюс boxing overhead, если хранить `Integer`.

## Подводные камни

- **Null prev/next на краях.** Забытые проверки head/tail дают NPE при удалении первого/последнего.
- **Concurrent modification.** Без синхронизации итератор и writer ломают ссылки.
- **Memory overhead vs ArrayList.** Много мелких node объектов бьёт по cache locality.
- **Цикл по ошибке.** Неверная перелинковка создаёт infinite loop при обходе.
- **equals/hashCode на node.** Сравнивать нужно payload, а не ссылочную идентичность узла.
