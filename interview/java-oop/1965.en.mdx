# Why does Java need an abstract class?

## In short

An abstract class provides partial implementation and mandatory hooks for subclasses. You cannot instantiate it directly: shared code lives in concrete methods while variable behavior is declared through abstract methods.

## How to answer in an interview

- Contrast with interface: an abstract class may hold state, constructors, and protected helpers.
- Explain that a subclass must implement every abstract method or stay abstract itself.
- Give a case: a template method plus hook methods in a hierarchy.
- Compare with interface (Java 8+ defaults/static): multiple type inheritance vs one code base.
- Be honest: when you only need a contract without shared state, an interface is usually cleaner; abstract class fits when there is real shared implementation.

## How it works

```java
abstract class ShipmentLabel {
    private final String trackingCode;

    protected ShipmentLabel(String trackingCode) {
        this.trackingCode = trackingCode;
    }

    abstract String carrierName();

    void printLabel() {
        System.out.println("[" + carrierName() + "] " + trackingCode);
    }
}

final class AirShipment extends ShipmentLabel {
    AirShipment(String trackingCode) {
        super(trackingCode);
    }

    @Override
    String carrierName() {
        return "AIR";
    }
}
```

`printLabel` inherits as ready-made logic; each subclass defines `carrierName`. You can hold `ShipmentLabel label = new AirShipment("X1")` and call it polymorphically.

Abstract classes often implement Template Method: the base defines the algorithm, subclasses override individual steps.

## Pitfalls

- **God abstract base.** Too much logic in the base makes every subclass fragile on change.
- **Confusion with interface.** Using interface + defaults where shared mutable state belongs in a base class.
- **Missing abstract implementation.** A concrete subclass will not compile until every abstract member is implemented.
- **Public abstract class with protected ctor.** Valid pattern, but without a documented factory, subclasses may be the only creation path.
- **Deep hierarchy.** Each new abstract level obscures which method actually runs.
