# How does a process differ from a thread?

## In short

A process is an isolated program instance with its own address space, handles, and OS accounting. A thread is a unit of execution inside a process: threads share the process memory and resources but have separate stacks and CPU context.

## How to answer in an interview

- A process crash stays contained; threads in one process share the address space and can take down the whole process.
- Spawning a process costs more (fork/exec, new PID); a thread is cheaper but needs synchronization for shared data.
- IPC uses pipes, sockets, shared memory with constraints; between threads, globals under a mutex often suffice.
- In C++ threads are `std::thread`, jthread; processes use OS APIs (`fork`, `CreateProcess`), not the STL.
- Honestly: Windows has no POSIX `fork`; I compare OS concepts, not a single API surface.

## How it works

A process gets its own virtual memory. Threads in one process see the same globals and heap.

```cpp
#include <iostream>
#include <thread>

int sharedTotal = 0;

void worker(int id) {
    sharedTotal += id;
    std::cout << "thread " << id << " sum=" << sharedTotal << '\n';
}

int main() {
    std::thread a(worker, 1);
    std::thread b(worker, 2);
    a.join();
    b.join();
    return 0;
}
```

Two threads write `sharedTotal` without protection (I would mention the race in an interview). Another process would not see that variable without IPC. Thread switches are cheaper than process context switches, but one thread's bug (heap corruption) affects all of them.

## Pitfalls

- **Races in shared memory.** Threads make data races easy; processes are isolated but IPC is harder.
- **fork and threads.** After `fork` in a multithreaded process, only async-signal-safe calls are safe until `exec`.
- **join and leaks.** A `std::thread` destroyed without `join` or `detach` calls `std::terminate`.
- **Priorities and scheduling.** Threads compete inside one process; fair scheduling is not guaranteed.
- **fork copy behavior.** Copy-on-write saves memory, but inherited handles and locks start in a fragile state.
