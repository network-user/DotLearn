# Два класса с одним FQN в одной JVM

## Суть

Identity класса в JVM это пара (fully qualified name, defining ClassLoader). Два `.class` с одинаковым `com.acme.Plugin` загруженные разными loaders считаются разными runtime types. Присваивание и `instanceof` между ними illegal. Это основа isolation в app servers, OSGi, plugin frameworks.

## Что ответить на собесе

- Bootstrap loads `java.*`; app classes via ApplicationClassLoader.
- Custom loader `defineClass` создаёт новый klass object даже для identical bytecode.
- Parent delegation: loader сначала спрашивает parent, но sibling loaders isolated.
- `Class.forName(name, init, loader)` явно выбирает loader.
- Честно: duplicate class = ClassCastException hell; prefer single classpath version.

## Как это работает

```java
public final class LoaderIsolationDemo {

    public static void main(String[] args) throws Exception {
        var jarA = Paths.get("plugins/v1/plugin.jar");
        var jarB = Paths.get("plugins/v2/plugin.jar");

        var loaderA = new JarClassLoader(jarA);
        var loaderB = new JarClassLoader(jarB);

        Class<?> handlerA = loaderA.loadClass("com.acme.Handler");
        Class<?> handlerB = loaderB.loadClass("com.acme.Handler");

        System.out.println(handlerA == handlerB);
        System.out.println(handlerA.getClassLoader());
        System.out.println(handlerB.getClassLoader());

        Object instanceA = handlerA.getDeclaredConstructor().newInstance();
        // handlerB.cast(instanceA); // ClassCastException
    }
}
```

`handlerA == handlerB` → `false`. Same name, different klass metadata in metaspace (possibly twice if not unloaded).

Tomcat: each webapp WAR → WebappClassLoader → two apps ship different Guava versions without JVM-wide conflict (within limits).

## Подводные камни

- **SPI with wrong TCCL.** ServiceLoader picks app loader vs plugin loader mismatch.
- **Serialization across loaders.** Deserialized object class not assignable to caller type.
- **Parent-last inversion.** Shading libs duplicate classes silently until runtime CCE.
- **Metaspace leak.** Reload plugin without dropping loader retains all klasses.
- **Assuming name uniqueness global.** Only per loader namespace guaranteed.
