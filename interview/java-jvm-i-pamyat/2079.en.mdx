# Java code path from source to execution

## In short

`.java` is compiled by `javac` into platform-independent bytecode (`.class`). JVM ClassLoader loads bytes, verifier checks safety, linker resolves symbols into Metaspace. The interpreter runs bytecode; HotSpot JIT (C1/C2) compiles hot methods to native code in the code cache. On Linux the result runs on the CPU as native instructions with GC and JVM runtime services.

## How to answer in an interview

- javac: source → bytecode + constant pool + metadata, not machine code.
- Loading hierarchy: Bootstrap → Platform → Application class loaders.
- Verification: stack map frames, type safety before execution.
- JIT tiers: interpreter → C1 → C2 on invocation counter thresholds.
- Honestly: AOT (GraalVM native-image) changes the pipeline; classic HotSpot path above.

## How it works

Source:

```java
public final class HashProbe {

    public static int mix(int seed, String key) {
        return seed * 31 + key.hashCode();
    }

    public static void main(String[] args) {
        System.out.println(mix(7, "sku-991"));
    }
}
```

Pipeline:

```
HashProbe.java
    │ javac
    ▼
HashProbe.class (bytecode)
    │ java HashProbe  →  Launcher  →  ClassLoader.loadClass
    ▼
Verify + Link (Metaspace: klass, methods)
    ▼
Interpreter runs main
    ▼
mix() becomes hot → JIT C2 → native code in Code Cache
    ▼
CPU executes machine instructions
```

The `java` command bootstraps through `sun.launcher.LauncherHelper`; classpath/module path defines Application ClassLoader search path.

Module system (JPMS): `module-info.java` limits exports/opens; illegal access blocked at compile and runtime.

## Pitfalls

- **Wrong classpath.** ClassNotFoundException before any bytecode runs.
- **VerifyError on bytecode manipulation.** ASM agent bug → class rejected at load.
- **Assuming javac version = runtime.** New language feature on old JRE fails at verify.
- **Decompile ≠ original source.** Lost generics/local names in bytecode view.
- **JIT deopt after class reload.** Agent redefine may invalidate compiled code mid-flight.
