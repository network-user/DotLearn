# Postman in API testing: when the GUI is enough and when you need code

## In short

Postman bundles HTTP requests, environments, and collections, lets you write JavaScript tests in a sandbox, and run them via Newman in CI. For AQA it is a fast exploratory layer and automation prototype, but mature regression usually moves to pytest/httpx for versioning and review.

## How to answer in an interview

- I list features: HTTP methods, pre-request scripts, tests tab, variables, environments, mock server, shared collections.
- I explain Newman/Postman CLI for headless pipeline runs.
- I compare to code: Postman starts faster, Python fits complex data-driven logic and typing.
- I mention exporting collections to git and syncing with OpenAPI.
- I add honestly: secrets in shared workspaces and drifting local variables often break "works in Postman" vs CI.

## How it works

A collection stores requests; an environment injects `{{baseUrl}}`, `{{token}}`. After the response a `pm.test` script validates JSON schema or status. Newman reads the same JSON collection and fails the pipeline on non-zero exit.

I mirror critical cases in pytest for a single report:

```python
import httpx
import pytest

BASE = "https://api.weather.test"
CITY = "London"


def test_current_weather_shape():
    with httpx.Client(base_url=BASE, timeout=5.0) as client:
        response = client.get("/v1/current", params={"city": CITY})
    assert response.status_code == 200
    body = response.json()
    assert body["city"] == CITY
    assert body["temperature"].endswith("C")
```

I keep Postman for manual debugging of new endpoints until the contract lands in OpenAPI.

## Pitfalls

**GUI only without Newman.** Regression never enters CI.

**Hardcoded prod URL in the collection.** Risky accidental run.

**Duplicate asserts in Postman and pytest without sync.** They diverge within a month.

**OAuth flow in the UI.** Refresh token never reaches automated runs.

**Large binary bodies in the collection.** Git diffs become unreadable; store fixtures separately.
