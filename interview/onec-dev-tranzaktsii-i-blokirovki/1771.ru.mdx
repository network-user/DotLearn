# Ручное управление транзакциями через НачатьТранзакцию и ЗафиксироватьТранзакцию

## Суть

Явные вызовы `НачатьТранзакцию`, `ЗафиксироватьТранзакцию` и `ОтменитьТранзакцию` нужны, когда несколько записей и регистров должны сохраниться атомарно вне стандартного проведения. Это прямой интерфейс к транзакции СУБД в сеансе 1С.

## Что ответить на собесе

- Я использую ручную транзакцию в обработках загрузки, пакетном переносе остатков и нетиповых интеграциях.
- На собеседовании подчёркиваю пару с `Попытка`/`Исключение`: фиксация только на успехе, откат на любой ошибке.
- Уточняю, что устаревшие имена `ТранзакцияНачать()` заменены на `НачатьТранзакцию()` в актуальном синтаксисе.
- Честно говорю: лишняя транзакция внутри уже проводимого документа обычно признак неверного дизайна.
- Не фиксирую транзакцию в общей библиотеке, если её открыл вызывающий код.

## Как это работает

Шаблон ручного блока:

```bsl
Процедура ЗагрузитьПакетНоменклатуры(ТаблицаДанных) Экспорт
    НачатьТранзакцию();
    Попытка
        Для Каждого Строка Из ТаблицаДанных Цикл
            Элемент = Справочники.Номенклатура.СоздатьЭлемент();
            Элемент.Наименование = Строка.Наименование;
            Элемент.Артикул = Строка.Артикул;
            Элемент.Записать();
        КонецЦикла;
        ЗафиксироватьТранзакцию();
    Исключение
        ОтменитьТранзакцию();
        ЗаписьЖурналаРегистрации("ЗагрузкаНоменклатуры", УровеньЖурналаРегистрации.Ошибка, , ,
            ПодробноеПредставлениеОшибки(ИнформацияОбОшибке()));
        ВызватьИсключение;
    КонецПопытки;
КонецПроцедуры
```

`НачатьТранзакцию()` открывает границу на соединении с СУБД. Все последующие `Записать()` до фиксации или отката видны только текущему сеансу. `ЗафиксироватьТранзакцию()` делает изменения постоянными; `ОтменитьТранзакцию()` возвращает базу к состоянию на момент начала блока.

Для длинных пакетов дроблю на части с отдельными транзакциями по N строк, чтобы не раздувать журнал SQL и не удерживать блокировки минутами. Атомарность всего файла тогда ослабляется, но доступность системы выше.

## Подводные камни

- **Фиксация внутри цикла.** Частичная загрузка при ошибке на середине файла.
- **Забытый НачатьТранзакцию.** Откат в Исключение без открытого блока даёт ошибку или неожиданное поведение.
- **Вложенная фиксация.** Ломает атомарность внешней операции.
- **Транзакция с HTTP-вызовом внутри.** Сеть ждёт, блокировки держатся.
- **Путаница с блокировкой данных.** `Заблокировать()` и транзакция SQL решают разные задачи.
