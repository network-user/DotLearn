# Когда платформа 1С открывает транзакцию автоматически

## Суть

Если разработчик не вызвал `НачатьТранзакцию()`, платформа сама оборачивает изменяющие операции в транзакцию: запись и удаление объектов, проведение и отмена проведения, запись наборов записей регистров. Явная транзакция нужна для пакета нескольких операций под одним commit.

## Что ответить на собесе

- Я говорю, что один вызов `Записать()` уже атомарен без ручного `НачатьТранзакцию()`.
- На собеседовании перечисляю триггеры: `Записать()`, `Удалить()`, `Записать(Проведение)`, запись `МенеджерЗаписи`.
- Подчёркиваю: явная транзакция не закрывается автоматически в конце метода, если была открыта вручную.
- Честно уточняю: чтение (`Прочитать()`, запрос на чтение) транзакцию не открывает.
- Для нескольких документов в одном commit явно вызываю `НачатьТранзакцию()`.

## Как это работает

Процедура записывает один документ. Платформа открывает транзакцию, пишет объект и движения, фиксирует или откатывает при ошибке.

```bsl
Процедура СохранитьНакладную(ДокументОбъект) Экспорт
    ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
КонецПроцедуры
```

Внутри `Записать(Проведение)` платформа вызывает обработчики, формирует движения, выполняет commit. Разработчик не видит явных вызовов, но блокировки и rollback работают.

Для пакета из двух связанных документов автоматической склейки нет:

```bsl
НачатьТранзакцию();
Попытка
    ДокументРасхода.Записать(РежимЗаписиДокумента.Проведение);
    ДокументПрихода.Записать(РежимЗаписиДокумента.Проведение);
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```

Без внешней транзакции каждый `Записать()` зафиксируется отдельно. Удаление объекта и пометка удаления также идут в автоматической транзакции. Запрос только на чтение транзакцию не начинает, если не было `НачатьТранзакцию()` ранее.

## Подводные камни

- **Два Записать() без общей транзакции.** Первый успел, второй упал, данные несогласованы.
- **Явная транзакция без фиксации.** Автозакрытие не сработает как у неявной.
- **Ожидание транзакции на чтении.** Лишний overhead не появляется, но и изоляции записи нет.
- **Вложенный вызов с откатом.** Откатывается весь внешний блок.
- **Серверный вызов с формы.** Граница автотранзакции совпадает с границей вызова.
