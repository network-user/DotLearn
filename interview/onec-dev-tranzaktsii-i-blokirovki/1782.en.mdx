# What lock types the 1C platform provides

## In short

The platform combines managed application locks, automatic locks on object write, and DBMS transactional locks. The developer works explicitly with `БлокировкаДанных`; the rest is applied on `Записать()`, posting, and SQL queries with locking.

## How to answer in an interview

- I list three layers: managed, pessimistic on write, SQL row/page/table.
- In an interview I name `Разделяемый` and `Исключительный` modes in `БлокировкаДанных`.
- I stress optimistic version check on write as separate from locking.
- I honestly say exclusive configuration and session modes are a special case of exclusive access.
- For diagnosis I check the 1C monitor and DBMS locks together.

## How it works

A managed lock is set explicitly before changing a register:

```bsl
Блокировка = Новый БлокировкаДанных;
Элемент = Блокировка.Добавить("Справочник.Номенклатура");
Элемент.Режим = РежимБлокировкиДанных.Разделяемый;
Элемент.УстановитьЗначение("Ссылка", СсылкаНоменклатуры);
Блокировка.Заблокировать();
```

Automatic locking applies when writing an object in a transaction: the platform locks the `_Reference*` row until commit. Posting adds locks on register movements.

The DBMS transactional layer uses shared/exclusive locks on rows and pages. Isolation level defines whether a transaction sees uncommitted changes from others.

```bsl
НачатьТранзакцию();
Попытка
    Объект = СсылкаКонтрагента.ПолучитьОбъект();
    Объект.Наименование = НовоеИмя;
    Объект.Записать();
    ЗафиксироватьТранзакцию();
Исключение
    ОтменитьТранзакцию();
    ВызватьИсключение;
КонецПопытки;
```

Exclusive access (configuration update, testing) blocks writes for all sessions. It does not replace managed locks in application code.

## Pitfalls

- **Mixing "hard" and "exclusive" terms.** In 1C docs "exclusive managed" is clearer.
- **Expecting a lock wait on optimistic write.** Version conflict rejects, it does not wait.
- **Ignoring the SQL layer.** Managed lock is released but the row is still locked in the DBMS.
- **Shared mode where writing happens.** Two sessions write at once.
- **Exclusive mode in production.** The maintenance window was left open.
