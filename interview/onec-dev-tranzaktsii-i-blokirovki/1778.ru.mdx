# Какие объекты чаще всего блокируют при проведении

## Суть

При проведении платформа блокирует движения регистров накопления и сведений, таблицы итогов, иногда справочники и документы-регистраторы. Основная конкуренция идёт за срезы остатков по складу, номенклатуре и периоду, а не за сам объект формы.

## Что ответить на собесе

- Я называю регистры накопления с остатками первым источником очередей при проведении.
- На собеседовании упоминаю регистры сведений с периодичностью и подчинённые регистратору.
- Подчёркиваю роль итогов регистра: их пересчёт блокирует широкий диапазон периодов.
- Честно говорю: два документа на один склад и одну номенклатуру почти всегда выстраиваются в очередь на движениях.
- Для анализа смотрю, какие регистры подключены в модуле проведения конкретного документа.

## Как это работает

Проведение реализации формирует движения по товарам, взаиморасчётам, себестоимости. Каждый набор движений блокирует ключ измерений в таблице `_AccumReg*`.

```bsl
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    Движения.ТоварыНаСкладах.Записывать = Истина;
    Движение = Движения.ТоварыНаСкладах.Добавить();
    Движение.ВидДвижения = ВидДвиженияНакопления.Расход;
    Движение.Период = Дата;
    Движение.Склад = Склад;
    Движение.Номенклатура = СтрокаТовара.Номенклатура;
    Движение.Количество = СтрокаТовара.Количество;
    Движения.Взаиморасчеты.Записывать = Истина;
    ЗаписьВзаиморасчетов = Движения.Взаиморасчеты.Добавить();
    ЗаписьВзаиморасчетов.Контрагент = Контрагент;
    ЗаписьВзаиморасчетов.Сумма = СуммаДокумента;
КонецПроцедуры
```

Регистр сведений с периодом «в пределах секунды» блокирует запись по ключу период + измерения. Последовательность номеров документов блокирует таблицу `_Document*` при записи.

Итоги накопления при проведении задним числом захватывают агрегаты по периодам. Параллельное проведение на один регистр без упорядочивания складов повышает deadlock.

Типовые «горячие» точки: `ТоварыНаСкладах`, `ВзаиморасчетыСКонтрагентами`, регистры себестоимости, курсы валют, статусы заказов.

## Подводные камни

- **Фокус только на документе.** Блокировка на регистре, а не на форме документа.
- **Несколько регистров в одном проведении.** Deadlock из-за разного порядка блокировки.
- **Перепроведение старых документов.** Тянет пересчёт итогов шире, чем кажется.
- **Авторегистрация лишних регистров.** Лишние движения и лишние блокировки.
- **Ночной регламент и дневные документы.** Конкуренция за одни таблицы `_AccumReg*`.
