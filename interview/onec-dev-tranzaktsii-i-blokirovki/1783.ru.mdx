# Какие блокировки возникают при проведении документа

## Суть

При проведении платформа открывает транзакцию, блокирует документ-регистратор, движения связанных регистров и затронутые итоги. Дополнительно разработчик может поставить управляемые блокировки в модуле проведения для защиты остатков до записи движений.

## Что ответить на собесе

- Я разделяю автоматические блокировки платформы и явные `БлокировкаДанных` в коде.
- На собеседовании перечисляю объекты: таблица документа, `_AccumReg*`, итоги, иногда последовательность номеров.
- Подчёркиваю, что отмена проведения снимает движения в той же транзакционной модели.
- Честно говорю: лишние регистры в движениях умножают блокировки без пользы.
- Смотрю `ОбработкаПроведения` и список `Движения.*.Записывать = Истина`.

## Как это работает

Платформа при `Записать(Проведение)` вызывает обработчики проведения, формирует движения и записывает их атомарно.

```bsl
Процедура ОбработкаПроведения(Отказ, Режим)
    Блокировка = Новый БлокировкаДанных;
    Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
    Элемент.Режим = РежимБлокировкиДанных.Исключительный;
    Элемент.УстановитьЗначение("Склад", Склад);
    Блокировка.Заблокировать();
    Движения.ТоварыНаСкладах.Записывать = Истина;
    Движение = Движения.ТоварыНаСкладах.Добавить();
    Движение.ВидДвижения = ВидДвиженияНакопления.Расход;
    Движение.Количество = КоличествоОтгрузки;
КонецПроцедуры
```

Автоматически блокируются строки регистра по ключу измерений и регистратор. При перепроведении сначала стираются старые движения, затем пишутся новые, блокировки удерживаются на весь интервал транзакции.

Итоги регистра накопления при движении «задним числом» блокируют диапазон периодов агрегатов. Регистры сведений с периодом блокируют срез период + измерения.

Если `Отказ = Истина`, транзакция откатывается, блокировки снимаются. При успехе commit освобождает ресурсы на стороне СУБД.

## Подводные камни

- **Блокировка после записи движений.** Остаток уже мог изменить параллельный сеанс.
- **Авторегистрация всех регистров документа.** Лишние таблицы в lock graph.
- **Проведение без управляемой блокировки на горячем регистре.** Гонки при списании.
- **Перепроведение в пик нагрузки.** Двойной объём блокировок на те же ключи.
- **Ручные движения вне стандартного проведения.** Блокировки не совпадают с ожиданиями платформы.
