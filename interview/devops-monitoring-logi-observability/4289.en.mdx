# What is service discovery in Prometheus?

## In short

Service discovery (SD) builds the scrape target list from an orchestrator or registry instead of static `static_configs`. Prometheus queries the Kubernetes API, Consul, EC2, DNS, and more, discovers endpoints, then `relabel_configs` filters and attaches labels. On scale-out, new pods enter scrape without manual reloads.

## How to answer in an interview

- In k8s I use `kubernetes_sd_configs` with role `pod`, `service`, or `endpoints`.
- `relabel_configs` keep/drop via `prometheus.io/scrape` annotations.
- Outside k8s: `file_sd` plus JSON generated by consul-template or Ansible.
- AWS: `ec2_sd_configs` filtered by tag `Environment=prod`.
- Honestly: SD without relabel scrapes the whole control plane and sidecar metrics; aggressive drop rules are mandatory.

## How it works

Kubernetes pod discovery with label filter:

```bash
cat <<'EOF' >> /etc/prometheus/prometheus.yml
scrape_configs:
  - job_name: kubernetes-pods
    kubernetes_sd_configs:
      - role: pod
    relabel_configs:
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: "true"
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_port]
        action: replace
        target_label: __address__
        regex: (.+)
        replacement: ${1}
      - source_labels: [__meta_kubernetes_namespace]
        target_label: namespace
EOF
```

File-based SD for bare metal:

```bash
cat <<'EOF' > /etc/prometheus/file_sd/apps.json
[
  { "targets": ["10.0.1.12:9100", "10.0.1.13:9100"], "labels": { "job": "node", "env": "prod" } }
]
EOF
```

Inspect active targets:

```bash
curl -s 'http://127.0.0.1:9090/api/v1/targets?state=active' | jq '.data.activeTargets[] | {job: .labels.job, instance: .labels.instance}'
```

## Pitfalls

- **Scraping all of kube-system.** Without drop rules Prometheus DDoS'es itself with thousands of cAdvisor series.
- **Stale targets.** Removed pod IPs linger until TTL; watch `up==0` noise and scrape timeouts.
- **Annotation typo.** `prometheus.io/port` without scrape=true is silently ignored; document the dev contract.
- **RBAC.** A Prometheus SA without `get/list/watch pods` yields an empty target list with no loud UI error.
- **File SD drift.** JSON from templates without atomic rename catches half-written files; write via temp + mv.
