# What is Fluentd and why use it in a pipeline?

## In short

Fluentd is an open-source log collector and router: it ingests events via input plugins, transforms them with filters, and ships via outputs to Elasticsearch, S3, Kafka, and more. A tag-based architecture (`<match>`) with on-disk buffering gives at-least-once delivery during short sink outages. In k8s you often deploy lighter Fluent Bit, but full Fluentd fits complex routing.

## How to answer in an interview

- Pipeline: `source` (tail, forward, systemd) → `filter` (parser, record_transformer) → `match` (es, s3).
- Tags route streams: `web.**` to ES, `audit.**` to an immutable S3 bucket.
- Buffers `file` or `memory` with `retry_forever` and exponential backoff.
- CNCF graduated project; unified JSON event model across sources.
- Honestly: on 512 Mi RAM edge nodes Fluentd is heavy; Fluent Bit covers 80% of cases.

## How it works

Config tailing nginx, enrich, dual output:

```bash
cat <<'EOF' > /etc/fluent/fluent.conf
<source>
  @type tail
  path /var/log/nginx/access.log
  pos_file /var/log/fluentd/nginx.pos
  tag web.nginx
  <parse>
    @type nginx
  </parse>
</source>

<filter web.nginx>
  @type record_transformer
  <record>
    host ${hostname}
    env prod
  </record>
</filter>

<match web.nginx>
  @type copy
  <store>
    @type elasticsearch
    host es.internal
    port 9200
    logstash_format true
    <buffer>
      @type file
      path /var/log/fluentd/buffer/es
    </buffer>
  </store>
  <store>
    @type s3
    aws_key_id "#{ENV['AWS_KEY']}"
    aws_sec_key "#{ENV['AWS_SECRET']}"
    s3_bucket logs-archive
    s3_object_key_format %{path}%{time_slice}_%{index}.gz
    <buffer time>
      @type file
      path /var/log/fluentd/buffer/s3
      timekey 3600
    </buffer>
  </store>
</match>
EOF
```

Verify offset and dry-run:

```bash
sudo cat /var/log/fluentd/nginx.pos
sudo fluentd --dry-run -c /etc/fluent/fluent.conf
```

## Pitfalls

- **Tag typo.** A typo in `match` silently drops the whole stream; add `@label @ERROR` routing.
- **POS file permissions.** Fluentd cannot write pos after rotate under another user; duplicates ship.
- **Memory buffer OOM.** Large bursts without file buffers kill the process; cap `chunk_limit_size`.
- **Plugin maintenance.** Obscure output plugins without security patches; pin versions and scan CVEs.
- **Timezone in S3 keys.** `timekey` without UTC yields odd archive boundaries on DST.
