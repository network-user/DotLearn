# Путь сообщения внутри RabbitMQ: от producer до consumer

## Суть

Сообщение в RabbitMQ никогда не попадает в очередь напрямую: оно приходит в обменник, тот по типу и ключу маршрутизации с помощью привязок раскидывает его по очередям, где сообщение ждёт, пока его не заберёт потребитель.

## Что ответить на собесе

Ключевая идея: producer публикует не в очередь напрямую, а в обменник, и полный путь сообщения такой: producer → exchange → binding → queue → consumer.

- Producer публикует в обменник (exchange), указывая ключ маршрутизации, а не пишет в очередь напрямую.
- Обменник по своему типу и привязкам (bindings) решает, в какие очереди положить копию сообщения; привязка связывает обменник с очередью по ключу маршрутизации или заголовкам.
- Дальше сообщение лежит в очереди, пока потребитель его не вытянет.
- Если ни одна привязка не подошла, сообщение либо отбрасывается, либо уходит в alternate exchange.

## Как это работает

1. **Публикация.** Producer открывает канал и публикует сообщение в обменник, указывая ключ маршрутизации. Используется протокол AMQP.
2. **Маршрутизация обменником.** Тип обменника определяет логику выбора очередей:
   - **direct** — точное совпадение ключа маршрутизации с ключом привязки;
   - **fanout** — копия во все привязанные очереди, ключ игнорируется;
   - **topic** — совпадение ключа с шаблоном привязки (`*`, `#`);
   - **headers** — маршрутизация по заголовкам, а не по ключу.
3. **Привязка.** Binding соединяет обменник с конкретной очередью и задаёт условие отбора.
4. **Хранение.** Сообщение лежит в очереди до тех пор, пока потребитель его не заберёт. Очередь и сообщение можно объявить durable, чтобы пережить рестарт брокера.
5. **Доставка.** Потребитель получает сообщения через подписку (push) либо сам опрашивает очередь (pull).

```python
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters('localhost'))
channel = connection.channel()

channel.exchange_declare(exchange='direct_logs', exchange_type='direct')

channel.basic_publish(
    exchange='direct_logs',
    routing_key='info',
    body='Hello RabbitMQ!',
)

connection.close()
```

## Подводные камни

- Producer публикует в обменник, а не в очередь напрямую. Очередь без привязки к подходящему обменнику просто не получит сообщения.
- При типе `fanout` ключ маршрутизации не имеет значения, новички часто пытаются им фильтровать и удивляются, что фильтр не работает.
- Дефолтный безымянный обменник (`""`) ведёт себя как direct, где routing_key равен имени очереди, отсюда иллюзия «публикации прямо в очередь».
- Без durable у очереди и `delivery_mode=2` у сообщения данные потеряются при перезапуске брокера.
- Если ни одна привязка не сматчилась, сообщение теряется молча, пока не настроен alternate exchange или `mandatory`-флаг.
