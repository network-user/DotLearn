# Цикл событий: кто на самом деле исполняет корутины

## Суть

Event loop (цикл событий): диспетчер асинхронной программы. Он по очереди исполняет готовые корутины, отслеживает завершение операций ввода-вывода и переключается между задачами в одном потоке без блокировки.

## Что ответить на собесе

Цикл событий - ядро `asyncio`: сами по себе `async def` и `await` ничего не запускают, реально крутит корутины именно loop.

- Он держит очередь готовых к исполнению задач, по очереди передаёт каждой управление до ближайшего `await`, а в этот момент корутина отдаёт управление обратно.
- Параллельно loop через системные механизмы (`epoll`/`kqueue`/`select`) следит, какие сокеты и файлы готовы, и будит соответствующие корутины.
- Получается однопоточная кооперативная многозадачность: пока одна задача ждёт I/O, исполняются другие.
- `asyncio.run(main())` как раз создаёт цикл, прогоняет в нём корутину и корректно закрывает loop.

## Как это работает

Цикл работает по схеме «событие → реакция»: непрерывно проверяет, не появилось ли готовых задач или завершённых операций, и вызывает соответствующие продолжения корутин.

```python
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

async def main():
    await asyncio.gather(say_hello(), say_hello())

asyncio.run(main())
```

Здесь `asyncio.gather` ставит две корутины в работу одновременно. Обе печатают `Hello`, обе уходят в `await asyncio.sleep(1)` и уступают управление циклу. Через секунду loop будит их, и обе печатают `World`. Времени в сумме уходит около одной секунды, а не двух: ожидания перекрылись, потому что ими управлял один цикл.

## Подводные камни

- В одном потоке активен только один цикл событий. Нельзя вызвать `asyncio.run` внутри уже работающего loop (например, в Jupyter): будет `RuntimeError`.
- Любой синхронный блокирующий вызов внутри корутины замораживает весь loop: пока он не вернётся, остальные задачи стоят.
- `gather` даёт конкурентность только при двух и более `await`-точках. Чистый CPU-расчёт без `await` всё равно выполнится последовательно.
- Не путать с потоками ОС: переключение здесь кооперативное и происходит только на `await`, а не вытесняется планировщиком.
