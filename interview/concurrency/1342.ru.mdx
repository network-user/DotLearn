# Global Interpreter Lock: почему потоки в CPython не считают параллельно

## Суть

GIL — это глобальный замок интерпретатора CPython, который пускает к исполнению байт-кода только один поток за раз. Он упрощает управление памятью, но лишает чисто вычислительный многопоточный код параллелизма на нескольких ядрах.

## Что ответить на собесе

GIL (Global Interpreter Lock) — мьютекс уровня интерпретатора в CPython: чтобы выполнять байт-код, поток должен держать этот замок, поэтому в один момент времени активен ровно один поток. Сделано это ради безопасности подсчёта ссылок при сборке мусора. Следствие: CPU-bound задачи многопоточностью не ускоришь, для них берут `multiprocessing`. А вот I/O-bound нагрузка выигрывает, потому что на время ожидания ввода-вывода поток отпускает GIL.

Замок завязан на счётчик ссылок: при изменении объекта обновляется его reference count, и без GIL два потока могли бы испортить этот счётчик одновременно. Поэтому поток захватывает GIL, исполняет порцию байт-кода, периодически отпускает замок и уступает очередь другому.

```python
import threading

def increment(counter):
    for _ in range(1_000_000):
        counter[0] += 1

counter = [0]
t1 = threading.Thread(target=increment, args=(counter,))
t2 = threading.Thread(target=increment, args=(counter,))

t1.start(); t2.start()
t1.join(); t2.join()

print(counter[0])  # 2000000
```

Здесь два потока крутят одну и ту же CPU-bound работу, но GIL не даёт им считать одновременно — на многоядерной машине ускорения не будет, скорее наоборот, добавятся накладные расходы на переключение.

## Подводные камни

- GIL не делает операции атомарными «бесплатно»: `counter[0] += 1` — это чтение, инкремент и запись, и при отпускании замка между ними возможна гонка. Для надёжности нужен `Lock`.
- «GIL мешает всегда» — миф. На I/O (сеть, диск) поток освобождает замок, и многопоточность реально помогает.
- GIL — деталь CPython, а не самого языка. Jython, IronPython и сборки без GIL (free-threaded build) ведут себя иначе; для CPU-bound в CPython стандартный обходной путь — процессы.
