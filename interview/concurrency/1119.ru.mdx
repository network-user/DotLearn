# Как одним вызовом стартовать пачку корутин в asyncio и собрать их результаты?

## Суть

Чтобы запустить несколько корутин и дождаться их всех разом, берут `asyncio.gather()`. Он планирует переданные корутины конкурентно и по завершении возвращает их результаты списком в том же порядке, в каком корутины были переданы.

## Что ответить на собесе

Самый частый инструмент - `asyncio.gather(*coros)`.

- Он оборачивает корутины в задачи, отдаёт их циклу событий и ждёт, пока завершатся все.
- Результат - список значений в порядке аргументов, а не в порядке завершения.
- Если нужен более структурированный контроль над группой задач, в Python 3.11+ есть `asyncio.TaskGroup`, который безопаснее по обработке ошибок и отмене.

`asyncio` даёт писать конкурентный код для задач ввода-вывода: пока одна корутина ждёт ответа сети или диска, цикл событий переключается на другую. Важно: это конкурентность в одном потоке, а не параллелизм на нескольких ядрах.

`gather` не запускает корутины «сразу и физически одновременно» - он отдаёт их планировщику, который чередует их на точках `await`. Поэтому выигрыш появляется именно на ожидании ввода-вывода.

```python
import asyncio

async def fetch_data(site):
    print(f"start {site}")
    await asyncio.sleep(2)  # имитация сетевого запроса
    print(f"done {site}")
    return f"data:{site}"

async def main():
    sites = ["site1", "site2", "site3"]
    results = await asyncio.gather(*(fetch_data(s) for s in sites))
    print(results)

asyncio.run(main())
```

Все три `fetch_data` запускаются конкурентно, поэтому общее время около 2 секунд, а не 6. Порядок в `results` совпадает с порядком `sites`, даже если какая-то корутина финишировала раньше.

## Подводные камни

- **Порядок результатов != порядок завершения.** `gather` сохраняет порядок аргументов; не полагайтесь на то, что первый результат пришёл от самой быстрой корутины.
- **Первое исключение прерывает ожидание.** По умолчанию `gather` пробрасывает первое возникшее исключение наверх, но остальные задачи при этом не отменяются автоматически и могут оставить «висящие» предупреждения. Используйте `return_exceptions=True`, чтобы собрать исключения как обычные элементы списка.
- **Передавать нужно корутины или future, а не результаты их вызова заранее.** `gather(fetch_data())` - корректно; `gather(fetch_data)` (без вызова) - ошибка.
- **Это не многопоточность.** CPU-bound вычисления внутри корутин всё равно блокируют единственный поток и убивают конкурентность.
