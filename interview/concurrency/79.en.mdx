# Coroutines in Python: a function you can pause

## In short

A coroutine is a callable whose execution can be suspended at a given point and later resumed from exactly the same place. All asynchronous code rests on this property: waiting on I/O no longer blocks the thread.

## How to answer in an interview

A coroutine is declared with `async def`. What sets it apart from an ordinary function is that calling it does not run the body right away - it returns a coroutine object; the event loop is what actually drives it. Inside, at `await` points, the coroutine voluntarily hands control back to the loop, which switches to other ready work, and once the awaited operation finishes it returns and resumes the coroutine from where it stopped. This is cooperative multitasking in a single thread: switches happen only at `await`, they are not preempted by the OS scheduler.

## How it works

Calling a coroutine executes nothing - it creates an object that has to be "spun up" through the event loop (`asyncio.run`, `await`, or `create_task`). An `await` point marks the place where the coroutine is ready to yield control until the awaited awaitable completes.

```python
import asyncio

async def say_hello():
    print("Hello")
    await asyncio.sleep(1)  # yield control for 1 second
    print("World")

async def main():
    await say_hello()

asyncio.run(main())
```

At the line `await asyncio.sleep(1)` the coroutine `say_hello` freezes, control goes to the event loop, and at that moment other coroutines can run. A second later the loop comes back and prints `World`.

Under the hood, coroutines are an evolution of generators: the same suspension mechanism through `yield`, but with their own `async`/`await` syntax. That is why you cannot "call and forget" a coroutine like an ordinary function - its result is always obtained through the event loop.

## Pitfalls

- A bare call `say_hello()` without `await` runs no code; it returns a coroutine object and usually triggers a `coroutine was never awaited` warning.
- `await` is only allowed inside `async def`. In synchronous code you start a coroutine with `asyncio.run(...)`.
- `await asyncio.sleep(1)` yields control to the loop, whereas `time.sleep(1)` blocks the entire thread and defeats the point of async.
- Coroutines give no CPU parallelism: heavy computation in a single thread still blocks the loop. The win is only on I/O waits.
