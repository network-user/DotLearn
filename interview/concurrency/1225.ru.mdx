# Что на самом деле делает asyncio.run(): три шага под капотом

## Суть

`asyncio.run(coro)` - точка входа в асинхронную программу. Она создаёт новый цикл событий, прогоняет в нём переданную корутину до конца и гарантированно закрывает цикл, прибирая за собой ресурсы.

## Что ответить на собесе

`asyncio.run()` появилась в 3.7 как удобная обёртка над ручной работой с циклом и делает внутри примерно три вещи.

- Поднимает свежий event loop через `new_event_loop()`.
- Выполняет корутину до завершения (`run_until_complete`).
- В `finally` отменяет оставшиеся задачи, доводит до конца асинхронные генераторы и закрывает цикл.

Важная деталь: она требует, чтобы активного цикла ещё не было, поэтому вызывать её вложенно из уже работающего цикла нельзя.

1. **Создание цикла.** Поднимается новый event loop - механизм, который следит, какие задачи готовы исполняться, и переключается между ними на точках `await`.
2. **Выполнение корутины.** В этом цикле запускается переданная корутина и работает до полного завершения.
3. **Закрытие и уборка.** После завершения цикл отменяет оставшиеся задачи, завершает асинхронные генераторы и закрывается - чтобы не текли ресурсы.

```python
import asyncio

async def main():
    print("Hello")
    await asyncio.sleep(1)
    print("World")

asyncio.run(main())
```

Здесь `asyncio.run(main())` сам поднимает цикл, печатает "Hello", пауза на секунду, затем "World", после чего закрывает цикл.

## Подводные камни

- **Нельзя вызывать вложенно.** Если цикл уже работает (например, внутри другой корутины или в Jupyter), `asyncio.run()` бросит `RuntimeError: asyncio.run() cannot be called from a running event loop`. Внутри корутины используйте `await`, а не повторный `run`.
- **Каждый вызов - новый цикл.** Два подряд `asyncio.run()` создают и закрывают два разных цикла; состояние между ними не переносится.
- **Аргумент - именно корутина.** Передавайте `main()` (вызов, дающий объект корутины), а не саму функцию `main`.
- **Только верхний уровень.** Это точка входа для скрипта/программы, а не способ «запустить корутину откуда угодно». Для подзадач внутри уже работающего цикла - `create_task` или `gather`.
