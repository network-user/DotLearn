# Упала одна корутина с исключением - что станет с остальными задачами?

## Суть

Само по себе исключение в одной корутине не убивает другие и не роняет цикл событий. Но поведение группы зависит от того, как вы их запускали: `gather` без флага пробросит первое исключение наверх, а `TaskGroup` отменит всю группу.

## Что ответить на собесе

«Корутины изолированы: сбой одной не выполняет код другой. Цикл событий продолжает крутиться. Дальше всё решает способ запуска. У `asyncio.gather()` по умолчанию первое исключение всплывает в точку `await gather(...)`, при этом остальные задачи продолжают работать, но их результаты теряются. С `return_exceptions=True` исключения возвращаются как обычные элементы. А `asyncio.TaskGroup` (3.11+) при падении любой задачи отменяет остальные и собирает ошибки в `ExceptionGroup` - это и есть structured concurrency.»

## Как это работает

Исключение внутри корутины ведёт себя как в обычной функции: если его не поймали внутри, оно всплывает в место, где корутину ожидали через `await`. Незаконченная корутина на этом обрывается, но соседние корутины - это отдельные объекты в цикле событий, их это напрямую не касается.

```python
import asyncio

async def task1():
    print("task1 start")
    await asyncio.sleep(1)
    raise ValueError("boom in task1")

async def task2():
    print("task2 start")
    await asyncio.sleep(2)
    print("task2 done")
    return "ok"

async def main():
    results = await asyncio.gather(task1(), task2(), return_exceptions=True)
    print(results)

asyncio.run(main())
```

С `return_exceptions=True` мы увидим, что `task2` спокойно дошла до конца, а ошибка `task1` вернулась объектом исключения в списке результатов, не прервав ожидание.

## Подводные камни

- **`gather` без `return_exceptions` маскирует часть проблем.** Первое исключение всплывает, но прочие задачи остаются работать в фоне; их ошибки могут вылиться в «Task exception was never retrieved».
- **Косвенное влияние через общие ресурсы.** Если корутины делят файл, соединение или общее состояние, сбой на середине операции в одной может оставить ресурс в некорректном виде - и это уже заденет других.
- **Цепочка `await`.** Если корутина A ждёт корутину B через `await`, исключение из B оборвёт и A. Это не «соседи», а прямая зависимость.
- **TaskGroup меняет правила.** В `asyncio.TaskGroup` падение одной задачи отменяет остальные - не путайте это поведение с поведением `gather`.
