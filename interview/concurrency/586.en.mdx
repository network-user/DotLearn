# What problem does async solve in Python

## In short

Async removes idle time on I/O operations: while one call waits for a response from the network, disk, or database, the program does other work in the same thread. The result is higher throughput and responsiveness without extra threads.

## How to answer in an interview

Async is for I/O-bound workloads - those where time is spent not on computation but on waiting for an external response. Classic synchronous code simply stalls on such a wait and keeps the thread busy. The `asyncio` model lets a single thread serve thousands of simultaneous connections: at every `await` the coroutine yields control to the event loop, which switches to ready work. This is cheaper than threads (no overhead from creating them or from OS context switches) and sidesteps the GIL, because the bottleneck is the wait, not the CPU. For CPU-bound tasks async is useless - there you need processes.

## How it works

The basic toolkit is the `asyncio` module and the `async`/`await` keywords. A coroutine is marked with `async def`, and `await` pauses it until the awaited operation completes, handing control to the loop.

```python
import asyncio

async def fetch_data():
    print("Starting data download...")
    await asyncio.sleep(2)  # simulate I/O delay
    print("Data loaded!")
    return "Data"

async def main():
    print("Starting the program")
    data = await fetch_data()
    print(f"Received data: {data}")

asyncio.run(main())
```

The real win shows up when there are many such operations and they run concurrently - for example via `asyncio.gather(...)`: the total time approaches that of the single longest operation rather than the sum of all of them. That is why async dominates web servers (`FastAPI`, `aiohttp`), async database drivers, and network services with many simultaneous connections.

## Pitfalls

- Async only speeds up I/O-bound code. A heavy computation inside a coroutine blocks the loop, and the whole service "hangs".
- A single synchronous blocking call (`requests.get`, `time.sleep`, an ordinary DB driver) stalls the entire event loop. Inside `async` you need asynchronous equivalents.
- Concurrency is not parallelism: everything runs in one thread. Multiple cores are engaged through processes, not `asyncio`.
- Sequential `await fetch(); await fetch()` gives no speedup. You must parallelize explicitly via `gather` or `create_task`.
