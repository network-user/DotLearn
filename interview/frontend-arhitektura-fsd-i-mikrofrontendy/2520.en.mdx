# Dependency Injection vs Service Locator

## In short

Dependency Injection supplies dependencies from outside (constructor/factory param): the class does not know where to get a logger. Service Locator offers a global registry `Locator.get('Logger')` inside the class. DI makes dependencies explicit in the signature; Locator hides them and complicates tests.

## How to answer in an interview

- Constructor DI: `new OrderService(paymentGateway, auditLog)` is visible at the composition root.
- React analogue: pass props/context providers, not `window.__services.fetch`.
- Service Locator fits legacy bootstrap but hides the graph from static analysis.
- Honestly: the frontend rarely needs an enterprise DI container; manual wiring in `main.tsx` is often enough.
- Tests: DI → inject fakes; Locator → reset global mock registry between tests.

## How it works

DI style:

```ts
export interface Mailer {
  send(to: string, body: string): Promise<void>;
}

export class InviteService {
  constructor(private readonly mailer: Mailer) {}

  async invite(email: string) {
    await this.mailer.send(email, 'You are invited');
  }
}

const mailer: Mailer = { send: async () => undefined };
const invites = new InviteService(mailer);
```

Service Locator style:

```ts
const services = new Map<string, unknown>();

export const ServiceLocator = {
  register<T>(key: string, instance: T) {
    services.set(key, instance);
  },
  resolve<T>(key: string) {
    return services.get(key) as T;
  },
};

export class InviteService {
  async invite(email: string) {
    const mailer = ServiceLocator.resolve<Mailer>('mailer');
    await mailer.send(email, 'You are invited');
  }
}
```

Locator dependencies are invisible in the constructor signature. DI keeps wiring in one composition root; Locator scatters lookup across the codebase.

## Pitfalls

- **Hidden deps.** Locator coupling is invisible to grep.
- **Global mutable registry.** Flaky parallel tests without reset.
- **Over-container.** Angular-style DI for five services on a landing page.
- **React anti-pattern.** Locator in render paths breaks pure component models.
- **Migration trap.** Locator plus DI together → duplicate source of truth.
