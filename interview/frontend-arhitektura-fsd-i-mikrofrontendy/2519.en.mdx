# SOLID principles on the frontend

## In short

SOLID is five OOP principles that apply to TS/React modules too: one module, one reason to change (SRP), extend without editing the base (OCP), subtypes are replaceable (LSP), narrow interfaces (ISP), depend on abstractions (DIP). On the frontend this shows up in hooks/services, not only classes.

## How to answer in an interview

- SRP: `UserCard` renders, `userApi` fetches, not both together.
- OCP: add a payment provider via a strategy object without editing a checkout `switch`.
- LSP: mock `StoragePort` in tests behaves like `LocalStorageAdapter`.
- ISP: do not force a component to implement a fat 12-method interface; split ports.
- Honestly: dogmatic SOLID in UI sometimes fights pragmatic colocation; balance beats the letter.

## How it works

DIP + ISP example:

```ts
export interface AnalyticsPort {
  track(event: string, payload?: Record<string, unknown>): void;
}

export class CheckoutService {
  constructor(private readonly analytics: AnalyticsPort) {}

  confirmOrder(orderId: string) {
    this.analytics.track('order_confirmed', { orderId });
  }
}
```

```ts
export class ConsoleAnalytics implements AnalyticsPort {
  track(event: string, payload?: Record<string, unknown>) {
    console.info(event, payload);
  }
}
```

OCP pricing rules:

```ts
type DiscountRule = (subtotal: number) => number;

const rules: DiscountRule[] = [
  (s) => (s > 100 ? s * 0.1 : 0),
  (s) => (s > 500 ? 50 : 0),
];

export function applyDiscounts(subtotal: number) {
  return rules.reduce((sum, rule) => sum + rule(subtotal), 0);
}
```

A new promo = a new rule in the array, not editing the core function.

## Pitfalls

- **SRP over-split.** A separate file for a 3-line helper without cohesion.
- **LSP ignored in hooks.** A custom hook silently changing contract breaks consumers.
- **Fat context ISP violation.** React context with 20 fields rerenders the whole subtree.
- **DIP ceremony.** An interface for every fetch in an MVP slows delivery.
- **OCP abuse.** Plugin architecture for two `if` branches.
