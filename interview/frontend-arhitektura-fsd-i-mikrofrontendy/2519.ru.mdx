# Принципы SOLID на фронтенде

## Суть

SOLID это пять OOP принципов, применимых и к TS/React модулям: один модуль — одна причина меняться (SRP), расширяй без правок базы (OCP), подтипы заменяемы (LSP), узкие интерфейсы (ISP), зависимость от абстракций (DIP). На фронте это про hooks/services, не только классы.

## Что ответить на собесе

- SRP: `UserCard` рендерит, `userApi` fetch, не вместе.
- OCP: новый payment provider через strategy object, без edit `switch` в checkout.
- LSP: mock `StoragePort` в тестах ведёт себя как `LocalStorageAdapter`.
- ISP: не заставляй компонент имплементить fat interface из 12 методов; split ports.
- Честно: dogmatic SOLID в UI иногда конфликтует с pragmatic colocation; баланс важнее буквы.

## Как это работает

DIP + ISP example:

```ts
export interface AnalyticsPort {
  track(event: string, payload?: Record<string, unknown>): void;
}

export class CheckoutService {
  constructor(private readonly analytics: AnalyticsPort) {}

  confirmOrder(orderId: string) {
    this.analytics.track('order_confirmed', { orderId });
  }
}
```

```ts
export class ConsoleAnalytics implements AnalyticsPort {
  track(event: string, payload?: Record<string, unknown>) {
    console.info(event, payload);
  }
}
```

OCP pricing rules:

```ts
type DiscountRule = (subtotal: number) => number;

const rules: DiscountRule[] = [
  (s) => (s > 100 ? s * 0.1 : 0),
  (s) => (s > 500 ? 50 : 0),
];

export function applyDiscounts(subtotal: number) {
  return rules.reduce((sum, rule) => sum + rule(subtotal), 0);
}
```

Новая акция = новый rule в массив, не правка core function.

## Подводные камни

- **SRP over-split.** Отдельный файл на 3 строки helper без cohesion.
- **LSP ignored in hooks.** Custom hook меняет contract silently ломает consumers.
- **Fat context ISP violation.** React context с 20 fields заставляет всё subtree rerender.
- **DIP ceremony.** Interface на каждый fetch в MVP замедляет delivery.
- **OCP abuse.** Plugin architecture для двух if веток.
