# How do you change network request parameters in Charles?

## In short

Charles lets you inspect and mutate traffic: Rewrite, Map Local/Remote, Breakpoints, Throttling. QA changes headers, query, body, or response status without rebuilding the client to verify error handling, retry logic, and UI on a slow network.

## How to answer in an interview

- Breakpoints: I pause request/response, edit JSON, forward.
- Rewrite rule: permanent swap of header `Accept-Language` or add `X-Debug-Trace`.
- Map Local: serve static JSON instead of backend for offline UI tests.
- Throttling preset 3G checks timeouts and skeleton loaders.
- Honestly: mocking responses on a prod-like stand can hide real API bugs; I note the test used a mock, not integration.

## How it works

```gherkin
Feature: Charles traffic modification for QA

  Scenario: Simulate 503 on payment endpoint
    Given Breakpoints enabled for POST /api/v1/charge
    When app submits payment
    Then Charles intercepts response
    And tester edits status to 503 with body service_unavailable
    And app shows retry banner without crash

  Scenario: Rewrite User-Agent for compatibility check
    Given Rewrite rule adds header User-Agent LegacyClient/2.1
    When mobile app sends catalog request
    Then server branch legacy path exercised without rebuild

  Scenario: Throttle checkout on slow network
    Given Throttling profile 3G 780kbps 300ms latency
    When user completes checkout
    Then UI keeps spinner under 15s and no double submit
```

Sample request edit at breakpoint (logical form):

```
POST /api/v1/cart/apply-coupon HTTP/1.1
Host: shop.stage.example
Content-Type: application/json

{"code":"SAVE10","cartId":"c-4419"}
→ tester changes code to EXPIRED99 to verify error toast
```

Map Remote redirects host to another stand; Map Local injects file `./fixtures/empty_cart.json`. Rule order matters: Rewrite vs Breakpoint changes the outcome.

## Pitfalls

- **Forgotten Breakpoint.** All requests hang; team thinks staging is down.
- **Rewrite on `*`.** Breaks OAuth redirect and CDN assets.
- **Map Local with stale JSON.** UI green against a dead contract.
- **Throttle on desktop Wi‑Fi only.** Real mobile network behaves differently.
- **Edited response without ticket logs.** Dev cannot reproduce; export the session.
