# What is Apache JMeter and when does QA use it?

## In short

JMeter is an open-source load and performance testing tool. It simulates many virtual users (threads), sends HTTP/JDBC/Kafka and other requests, and collects latency, throughput, and error rate. QA uses it to validate SLA under peak, find bottlenecks, and catch performance regressions after release.

## How to answer in an interview

- Test Plan: Thread Group, Samplers, Timers, Assertions, Listeners.
- Ramp-up spreads thread start so staging is not killed by one spike.
- Assertions on status 200 and p95 latency below threshold.
- Results: Aggregate Report, Backend Listener to Influx/Grafana.
- Honestly: one JMeter machine is CPU-bound; serious load needs distributed mode or cloud.

## How it works

```gherkin
Feature: JMeter load test plan

  Scenario: Checkout SLA under 200 users
    Given Thread Group 200 threads ramp 120 seconds
    And HTTP Sampler POST /api/checkout
    When loop 5 iterations
    Then p95 response time under 800 ms
    And error rate under 1 percent

  Scenario: Steady soak test
    Given 50 threads constant 2 hours
    When memory leak in API
    Then response time drift upward QA files stability bug

  Scenario: Parameterized credentials
    Given CSV Data Set user list 500 rows
    When each thread unique login
    Then no false 429 from single test account
```

Simplified JMX fragment (renamed fields):

```xml
<ThreadGroup>
  <stringProp name="ThreadGroup.num_threads">100</stringProp>
  <stringProp name="ThreadGroup.ramp_time">60</stringProp>
  <HTTPSamplerProxy>
    <stringProp name="HTTPSampler.domain">staging.shop.local</stringProp>
    <stringProp name="HTTPSampler.path">/api/catalog</stringProp>
    <stringProp name="HTTPSampler.method">GET</stringProp>
  </HTTPSamplerProxy>
</ThreadGroup>
```

## Pitfalls

- **Load on prod without approval.** Incident and legal risk.
- **No warm-up phase.** First minutes skew metrics.
- **CDN cache off on staging.** Numbers unlike prod.
- **View Results Tree on 10k samples.** GUI crashes, OOM.
- **Think time ignored.** Unrealistic burst, false API fail.
