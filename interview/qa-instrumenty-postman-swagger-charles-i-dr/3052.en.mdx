# How do you replace the server response body in Charles?

## In short

Yes: via Rewrite (body replace rules), Breakpoints (manual edit on the fly), or Map Local (file instead of remote response). QA injects empty lists, 500 errors, or trimmed JSON to verify empty states, error UI, and pagination without a backend deploy.

## How to answer in an interview

- Tools → Rewrite → Add rule: If URL contains `/api/catalog`, Body replace `"inStock":true` with `false`.
- Breakpoints on response: edit JSON, Execute.
- Map Local: `./fixtures/empty_catalog.json` for GET catalog.
- In the ticket I note the scenario used a mock, not integration.
- Honestly: gzip/br responses sometimes break text replace; disable compression on staging or use Map Local.

## How it works

```gherkin
Feature: Replace server response in Charles

  Scenario: Empty cart without backend change
    Given Map Local maps GET /api/v2/cart to fixtures/cart_empty.json
    When app opens cart screen
    Then UI shows empty state illustration and CTA shop

  Scenario: Simulate invalid token 401
    Given Rewrite replaces response status 200 with 401 on GET /profile
    And body {"error":"token_expired"}
    When app refreshes session
    Then login modal appears without white screen

  Scenario: Breakpoint one-off typo field
    Given Breakpoints enabled for POST /graphql
    When response contains "balance": "NaN"
    Then tester fixes to numeric 0 and verifies formatting
```

Rewrite rule (logical record):

```
Set: QA Catalog mocks
If: host shop.stage.example AND path contains /api/catalog
Response Body: Replace All
  from: "total": 42
  to:   "total": 0
```

Map Local binds a URL pattern to a disk file; Breakpoints suit one-off repro. Disable the rule set after testing or the next tester sees a "magic" defect.

## Pitfalls

- **Replace by substring in HTML.** Breaks markup; for UI prefer Map Local with full JSON.
- **HTTPS without SSL Proxying.** Rewrite never runs on encrypted tunnel.
- **Client cache.** App shows old response; force refresh or new session.
- **Binary/protobuf body.** Text replace is useless; need decoder or mock server.
- **Forgotten active Rewrite.** Hours debugging a phantom defect.
