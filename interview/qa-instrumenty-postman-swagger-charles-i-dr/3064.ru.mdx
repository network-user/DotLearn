# Как подменить исходящий запрос в Charles через Breakpoints?

## Суть

Breakpoint останавливает request (и опционально response) перед отправкой на server. QA правит URL, headers или JSON body, затем Execute. Так проверяют validation на server, RBAC, injection handling без изменения client build.

## Что ответить на собесе

- Proxy → Breakpoint Settings → enable Request breakpoint для host/path.
- Repro action → Charles Breakpoint window → Edit Request → Execute.
- Меняю amount на отрицательный, role header, expired token.
- Repeat для matrix негативных кейсов.
- Честно: breakpoint manual; для regression suite лучше automated API tests.

## Как это работает

```gherkin
Feature: Charles request substitution

  Scenario: Send invalid quantity to cart API
    Given breakpoint on POST /api/v2/cart/items
    When original body qty 2 intercepted
    And tester edits qty to -1
    And Execute forwards to server
    Then response 400 and UI shows validation message

  Scenario: Test admin endpoint as regular user
    Given breakpoint on GET /admin/users
    When Authorization header replaced with user token
    Then server returns 403 and app handles forbidden state

  Scenario: Omit required header
    Given breakpoint removes Idempotency-Key header
    When POST /payments executed
    Then 409 duplicate charge prevented or error surfaced QA verifies policy
```

Исходный и изменённый body (логическая форма):

```json
{"sku":"WB-991","qty":2,"warehouseId":"wh-east"}
→ {"sku":"WB-991","qty":-1,"warehouseId":"wh-east"}
```

Alternatives: Rewrite для permanent header inject; Edit & Repeat на уже захваченном request без breakpoint next time.

## Подводные камни

- **Breakpoint на все URLs.** Каждый tap в app hangs; narrow rule.
- **Edit wrong tab (Response).** Request unchanged; confusion in triage.
- **Pretty JSON invalid after edit.** Trailing comma → 400 opaque error.
- **Idempotency:** повтор Execute создаёт duplicate side effect on staging.
- **HTTPS host not in SSL list.** Breakpoint never fires; tunnel only.
