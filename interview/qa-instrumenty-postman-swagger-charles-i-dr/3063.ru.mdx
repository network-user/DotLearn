# Как выглядит пример запроса и ответа в Swagger UI?

## Суть

После Execute Swagger UI показывает curl/fetch snippet, request URL с подставленными path/query, response status, headers и body (pretty JSON). Пример в spec (Example Value) может отличаться от live response; QA сверяет оба и прикладывает curl к багу.

## Что ответить на собесе

- Request: method, `https://api.stage.example/v2/users/a1b2`, headers Authorization Accept.
- Response: `200`, body `{"id":"a1b2","email":"qa@corp.example","tier":"beta"}`.
- Copy curl → paste in terminal or Jira for repro.
- Schema tab показывает required fields; Responses → 404 example для negative.
- Честно: browser CORS может скрыть body; дублирую curl в Postman.

## Как это работает

```gherkin
Feature: Swagger UI request response sample

  Scenario: GET user by id Try it out
    Given operation GET /v2/users/{userId} with param userId a1b2
    When Execute clicked after Authorize bearer token
    Then request URL https://api.stage.example/v2/users/a1b2
    And response code 200
    And response body contains email field string

  Scenario: Attach curl to defect
    Given POST /refunds returns 422 validation error
    When tester copies curl from Swagger panel
    Then Jira bug includes curl reproducing same 422

  Scenario: Example value vs live mismatch
    Given documented example tier "standard"
    When live response tier "beta"
    Then QA files spec drift if product expects doc accuracy
```

Логический вывод Swagger UI после Execute:

```
Request URL
https://api.stage.example/v2/users/a1b2

Curl
curl -X GET "https://api.stage.example/v2/users/a1b2" \
  -H "accept: application/json" \
  -H "Authorization: Bearer eyJ..."

Server response
Code: 200
Body:
{
  "id": "a1b2",
  "email": "qa@corp.example",
  "tier": "beta"
}
```

Parameters section до Execute показывает path/query/header/body forms с schema types и required markers.

## Подводные камни

- **Скрин example вместо live Execute.** Доказательство бага должно быть real response.
- **Token в скриншоте curl.** Redact before attach.
- **Pretty print скрывает trailing spaces.** String assert fails in client.
- **204 No Content.** Swagger UI пустое body; не жди JSON.
- **Wrong server selected in dropdown.** Response from dev, bug filed against stage.
