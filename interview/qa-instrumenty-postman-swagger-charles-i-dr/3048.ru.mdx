# Как подключить Charles Proxy на desktop и mobile?

## Суть

Charles слушает порт 8888 (по умолчанию) и принимает HTTP(S) через системный или ручной proxy. Desktop: Proxy → macOS/Windows Proxy включает перехват браузера. Mobile: Wi‑Fi manual proxy на IP ноутбука + установка root cert для HTTPS. Без SSL Proxying settings host виден только как CONNECT tunnel.

## Что ответить на собесе

- Help → SSL Proxying → Install Charles Root Certificate на машине тестера.
- Proxy → SSL Proxying Settings → добавляю `host:443` или конкретный staging domain.
- Android/iOS: Wi‑Fi → proxy manual, IP desktop, port 8888; на телефоне chls.pro/ssl.
- iOS: Settings → General → About → Certificate Trust Enable Full Trust.
- Честно: VPN на телефоне обходит Wi‑Fi proxy; отключаю VPN на время сессии.

## Как это работает

```gherkin
Feature: Charles proxy setup

  Scenario: Desktop browser capture
    Given Charles running on 192.168.0.14:8888
    When macOS proxy enabled via Proxy menu
    And Chrome opens https://shop.stage.example
    Then Structure shows decrypted GET /api/config with JSON body

  Scenario: Android app through Charles
    Given phone Wi-Fi proxy 192.168.0.14:8888
    And Charles root cert installed as user CA
    When native app starts checkout flow
    Then HTTPS requests listed unless app uses certificate pinning

  Scenario: Allow mobile device popup
    Given first request from unknown IP
    When Charles shows Allow dialog
    Then tester clicks Allow to capture that device traffic
```

Checklist подключения:

```
Desktop:
  Charles → Proxy → Enable macOS/Windows Proxy
  Install root cert → trust in Keychain/Credential Manager
  SSL Proxying: *.stage.example:443

Mobile:
  Same LAN as laptop
  Wi-Fi proxy manual → laptop IP, port 8888
  Browse chls.pro/ssl → install profile → trust (iOS)
  Repeat SSL host list on Charles
```

После сессии отключаю system proxy, чтобы не ломать обычный браузинг. Export `.chls` session прикладываю к багу с чувствительными полями замазанными.

## Подводные камни

- **Firewall блокирует 8888.** Mobile видит «no internet»; открыть порт или другой listener.
- **Cert не trusted на iOS 16+.** Full Trust не включён; HTTPS пустой в Charles.
- **Android 7+ user cert ignored by app.** Нужен debug build или network security config.
- **Забытый system proxy overnight.** Утром почта не синкается; proxy off ritual.
- **Charles на corp VPN IP другой.** IP ноутбука меняется; обнови proxy на телефоне.
