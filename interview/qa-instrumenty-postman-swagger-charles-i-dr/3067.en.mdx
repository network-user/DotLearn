# What is curl and how does QA use it for APIs?

## In short

curl (client URL) is a CLI for HTTP/FTP and other protocols: send GET/POST, headers, body, cookies, TLS, proxy. QA copies curl from Swagger/Charles/DevTools for terminal repro, CI smoke scripts, and quick endpoint checks without a GUI.

## How to answer in an interview

- `curl -sS -D -` shows headers plus body; `-o` save, `-w "%{http_code}"` code only.
- POST JSON: `-H "Content-Type: application/json" -d '{"sku":"X"}'`.
- Auth: `-H "Authorization: Bearer $TOKEN"` from env, not hardcoded in tickets.
- `-k` insecure skip verify only on local mock, not a prod habit.
- Honestly: curl has no pretty asserts; suites use Postman/Newman or pytest+requests.

## How it works

```gherkin
Feature: curl for QA API checks

  Scenario: Smoke health after deploy
    Given staging URL https://api.stage.example/health
    When curl returns 200 within 5 seconds
    Then deploy gate passes before full regression

  Scenario: Reproduce bug from Swagger copy
    Given developer pasted curl in Jira SHOP-1001
    When QA runs same curl against staging
    Then identical 422 validation body confirms environment parity

  Scenario: Script loop idempotency keys
    Given bash script posts payment with unique Idempotency-Key header
    When curl executed twice with same key
    Then second response 409 or same 200 without double charge
```

Examples (renamed host):

```
curl -sS "https://api.stage.example/v2/catalog?limit=5" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer $STAGE_TOKEN"

curl -sS -X POST "https://api.stage.example/v2/cart/items" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $STAGE_TOKEN" \
  -d '{"sku":"WB-991","qty":1}'
```

Flags: `-v` verbose TLS/handshake debug, `-L` follow redirects, `-x http://127.0.0.1:8888` via Charles, `--compressed` gzip accept.

## Pitfalls

- **Secrets in shell history.** Token in history; use env var and `set +o history` for demos.
- **Windows quoting.** JSON `-d` breaks in cmd; use Git Bash or `@body.json`.
- **302 without -L.** Script shows "404" while live 200 after redirect.
- **Large response floods terminal.** `-o out.json` instead of blind pipe.
- **curl happy path only.** Negative matrix still belongs in automated tests.
