# Как прикреплять артефакты к баг-репорту, чтобы его быстро воспроизвели?

## Суть

Артефакты в тикете (скрин, видeo, HAR, лог, URL, конфиг) сокращают цикл «не воспроизводится». Каждый файл должен быть назван понятно, доступен команде и явно привязан к шагам, ожидаемому и фактическому результату. Описание в теле бага объясняет, что смотреть в каждом вложении.

## Что ответить на собесе

- Прикрепляю минимальный набор: шаги, expected/actual, окружение, build, скрин с аннотацией.
- Для API добавляю curl/Postman export или HAR; для UI короткое видео 30-60 секунд вместо десятка скринов.
- Имена файлов структурированы: `20250713_login_403_staging.png`, не `image(3).png`.
- Ссылки на S3/Confluence проверяю ACL: dev без VPN не должен получать 403.
- Честно: перегруз вложениями без навигации в описании хуже, чем один качественный скрин и лог.

## Как это работает

```gherkin
Feature: Bug report attachments

  Scenario: Login failure on staging
    Given tester on build 4.2.1 staging EU
    When invalid password entered three times
    Then UI shows generic error instead of lockout message
    And attachment login_lockout_missing.png highlights toast text
    And attachment auth-service.log contains ERROR brute_force_guard
    And attachment repro_steps.mp4 shows full flow under 45 seconds
    And ticket body references each file by name in steps 4-6

  Scenario: API regression 500 on checkout
    Given Postman collection export checkout_v3.json attached
    When developer imports collection into local Postman
    Then same 500 reproduced with requestId from server log snippet
```

Пример фрагмента описания в Jira:

```
Steps:
1. Open https://staging.shop.example/login (see repro_steps.mp4)
2. Enter user qa_demo / wrong password x3
Expected: account lockout banner after 3rd attempt
Actual: generic "Something went wrong" (login_lockout_missing.png)
Logs: auth-service.log lines 1842-1855, requestId=9f2a...
Environment: staging EU, build 4.2.1, Chrome 125
```

Что включать по типу бага: **скриншот** фиксирует UI-состояние; **видео** показывает тайминг и жесты; **логи** дают stack trace и correlation id; **URL/query** задаёт точку входа; **конфиг** нужен, если баг завязан на feature flag или tenant settings.

## Подводные камни

- **PII на скринах.** Паспорт, карта, email реального клиента: блюр или синтетические данные.
- **Истекающие ссылки.** Presigned URL через 7 дней мёртв; дублируй файл во вложения TMS.
- **Огромные видео без сжатия.** 200 МБ блокируют мобильного dev; режь до ключевого фрагмента.
- **Лог без таймзоны и build.** «Вчера вечером» не воспроизводится; указывай UTC и версию.
- **Скрин всего экрана 4K.** Обрезай до области ошибки, иначе теряется фокус.
