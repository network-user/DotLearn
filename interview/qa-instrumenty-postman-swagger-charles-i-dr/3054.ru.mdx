# Commit, branch и merge в Git: что это для QA?

## Суть

**Commit** фиксирует снимок изменений в локальной истории с message и hash. **Branch** это указатель на линию работы (feature/fix) параллельно main. **Merge** (или squash/rebase через PR) вливает commits из одной ветки в другую. QA использует те же операции для тестовых репозиториев и для review diff релиза.

## Что ответить на собесе

- Commit: атомарная единица «что изменилось и зачем»; message в imperative mood.
- Branch: `feature/QA-441-checkout-tests` изолирован от main до review.
- Merge через PR после green CI; конфликты в data fixtures решаю с dev.
- `git log --oneline`, `git diff main...branch` перед регрессом релиза.
- Честно: merge commit без rebase засоряет историю; в командах часто squash merge в main.

## Как это работает

```gherkin
Feature: Git commit branch merge for QA artifacts

  Scenario: Commit updated test data
    Given branch fix/QA-512-wallet-tests from main
    When tester edits fixtures/wallet_ok.json
    And runs git commit -m "test: align wallet fixture with v4 API"
    Then history shows single commit ahead of main

  Scenario: Merge via pull request
    Given PR opened fix/QA-512-wallet-tests
    When CI pipeline green and reviewer approved
    Then squash merge into main
    And nightly suite picks new fixture on next run

  Scenario: Resolve merge conflict
    Given main updated same yaml by developer
    When git merge main into feature branch
    Then conflict markers in tests/checkout.yaml resolved manually
    And commit merge records integration point
```

Команды (переименованный репо):

```
git checkout -b fix/QA-512-wallet-tests
git add tests/wallet/
git commit -m "test: cover insufficient funds path"
git push -u origin fix/QA-512-wallet-tests
git checkout main
git pull
git merge fix/QA-512-wallet-tests
```

Merge создаёт commit слияния (или fast-forward если main не ушёл вперёд). Rebase переписывает историю feature поверх main; QA редко rebase shared branch без согласования.

## Подводные камни

- **Commit без тестов.** CI красный на main; pre-commit hook и branch protection.
- **Long-lived branch.** Drift от main, merge hell перед релизом.
- **Merge без pull main first.** Конфликты в CI secrets и shared config.
- **Force push shared branch.** Сносит историю коллег; только personal feature.
- **Binary merge conflict.** Excel test data; договоритесь о source format yaml/json.
