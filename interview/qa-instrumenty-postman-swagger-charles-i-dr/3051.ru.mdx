# Плюсы и минусы Swagger для QA и команды

## Суть

OpenAPI/Swagger даёт единый контракт API: автодок, Swagger UI для Try it out, codegen клиентов и валидацию схем. QA выигрывает в скорости smoke и синхронизации с dev. Минусы: drift между spec и реализацией, сложность больших API, ограничения на нестандартный auth и бинарные payload, кривая обучения OpenAPI 3.x.

## Что ответить на собесе

- Плюс: один yaml/json как source of truth для dev, QA, frontend.
- Плюс: Swagger UI снижает порог входа новому тестировщику.
- Минус: без CI diff spec vs implementation документация врёт.
- Минус: крупные спеки неудобны в UI; нужны tags и split by domain.
- Честно: Swagger не заменяет exploratory и E2E; это контрактный слой, не пользовательский путь.

## Как это работает

```gherkin
Feature: Swagger pros and cons in practice

  Scenario: Fast onboarding QA on new microservice
    Given openapi.yaml published in repo docs/
    When new QA imports spec to Postman and Swagger UI
    Then smoke list built in one hour without reading controller code

  Scenario: Spec drift detected
    Given integration test compares response schema to openapi
    When field discountAmount missing in actual JSON
    Then CI fails and ticket assigned to API owner

  Scenario: Complex OAuth2 not fully described
    Given custom mTLS gateway not in openapi securitySchemes
    When tester relies only on Swagger Authorize button
    Then requests fail until wiki documents header X-Client-Cert
```

Мини-спека (переименованный сервис):

```yaml
openapi: 3.1.0
info:
  title: Returns API
  version: 1.2.0
paths:
  /returns/{id}:
    get:
      operationId: fetchReturn
      responses:
        "200":
          description: Return detail
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/ReturnDto"
```

Баланс: codegen ускоряет клиентов, но QA должен дополнять негативные кейсы и stateful flows, которых в spec нет.

## Подводные камни

- **Swagger 2 vs OpenAPI 3 mix.** Импорт ломает nullable и oneOf; держите одну версию.
- **Example value copied as expected.** Prod возвращает другие enum; assert по schema.
- **Гигантский monolith spec.** UI timeout; split на domain repos.
- **Try it out на prod write methods.** Инцидент с удалением данных.
- **Annotations only in code.** Spec генерится, но QA не видит PR с контрактом; review yaml в git.
