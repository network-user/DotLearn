# Что такое Apache JMeter и когда его использует QA?

## Суть

JMeter это open-source инструмент нагрузочного и performance-тестирования. Он имитирует множество виртуальных пользователей (threads), шлёт HTTP/JDBC/Kafka и другие запросы, собирает latency, throughput и error rate. QA использует его для проверки SLA под пиком, поиска bottleneck и регрессии производительности после релиза.

## Что ответить на собесе

- Test Plan: Thread Group, Samplers, Timers, Assertions, Listeners.
- Ramp-up размазывает старт threads, чтобы не убить staging одним spike.
- Assertions на код 200 и p95 latency меньше порога.
- Результаты: Aggregate Report, Backend Listener в Influx/Grafana.
- Честно: нагрузку с одной машины JMeter ограничена CPU; для serious load нужен distributed mode или cloud.

## Как это работает

```gherkin
Feature: JMeter load test plan

  Scenario: Checkout SLA under 200 users
    Given Thread Group 200 threads ramp 120 seconds
    And HTTP Sampler POST /api/checkout
    When loop 5 iterations
    Then p95 response time under 800 ms
    And error rate under 1 percent

  Scenario: Steady soak test
    Given 50 threads constant 2 hours
    When memory leak in API
    Then response time drift upward QA files stability bug

  Scenario: Parameterized credentials
    Given CSV Data Set user list 500 rows
    When each thread unique login
    Then no false 429 from single test account
```

Упрощённый фрагмент JMX-логики (переименованные поля):

```xml
<ThreadGroup>
  <stringProp name="ThreadGroup.num_threads">100</stringProp>
  <stringProp name="ThreadGroup.ramp_time">60</stringProp>
  <HTTPSamplerProxy>
    <stringProp name="HTTPSampler.domain">staging.shop.local</stringProp>
    <stringProp name="HTTPSampler.path">/api/catalog</stringProp>
    <stringProp name="HTTPSampler.method">GET</stringProp>
  </HTTPSamplerProxy>
</ThreadGroup>
```

## Подводные камни

- **Нагрузка на prod без approval.** Инцидент и юридические риски.
- **Нет warm-up phase.** Первые минуты skew metrics.
- **Кэш CDN на staging off.** Цифры не сравнимы с prod.
- **Один listener View Results Tree на 10k samples.** GUI падает, OOM.
- **Игнор think time.** Нереалистичный burst, ложный fail API.
