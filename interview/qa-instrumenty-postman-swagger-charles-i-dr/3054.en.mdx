# Git commit, branch, and merge: what do they mean for QA?

## In short

A **commit** snapshots changes in local history with a message and hash. A **branch** is a pointer to a line of work (feature/fix) parallel to main. **Merge** (or squash/rebase via PR) integrates commits from one branch into another. QA uses the same operations for test repos and for reviewing release diffs.

## How to answer in an interview

- Commit: atomic unit of "what changed and why"; message in imperative mood.
- Branch: `feature/QA-441-checkout-tests` isolated from main until review.
- Merge via PR after green CI; I resolve conflicts in data fixtures with dev.
- `git log --oneline`, `git diff main...branch` before release regression.
- Honestly: merge commits without rebase clutter history; many teams squash merge to main.

## How it works

```gherkin
Feature: Git commit branch merge for QA artifacts

  Scenario: Commit updated test data
    Given branch fix/QA-512-wallet-tests from main
    When tester edits fixtures/wallet_ok.json
    And runs git commit -m "test: align wallet fixture with v4 API"
    Then history shows single commit ahead of main

  Scenario: Merge via pull request
    Given PR opened fix/QA-512-wallet-tests
    When CI pipeline green and reviewer approved
    Then squash merge into main
    And nightly suite picks new fixture on next run

  Scenario: Resolve merge conflict
    Given main updated same yaml by developer
    When git merge main into feature branch
    Then conflict markers in tests/checkout.yaml resolved manually
    And commit merge records integration point
```

Commands (renamed repo):

```
git checkout -b fix/QA-512-wallet-tests
git add tests/wallet/
git commit -m "test: cover insufficient funds path"
git push -u origin fix/QA-512-wallet-tests
git checkout main
git pull
git merge fix/QA-512-wallet-tests
```

Merge creates a merge commit (or fast-forward if main did not move). Rebase rewrites feature history on top of main; QA rarely rebases a shared branch without agreement.

## Pitfalls

- **Commit without tests.** CI red on main; pre-commit hook and branch protection.
- **Long-lived branch.** Drift from main, merge hell before release.
- **Merge without pulling main first.** Conflicts in CI secrets and shared config.
- **Force push to shared branch.** Wipes teammates' history; personal feature only.
- **Binary merge conflict.** Excel test data; agree on yaml/json as source format.
