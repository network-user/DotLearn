# How to attach artifacts to a bug report so it gets reproduced quickly?

## In short

Artifacts in a ticket (screenshot, video, HAR, log, URL, config) shorten the "cannot reproduce" cycle. Each file needs a clear name, team access, and an explicit tie to steps, expected, and actual results. The ticket body explains what to look at in every attachment.

## How to answer in an interview

- I attach a minimal set: steps, expected/actual, environment, build, annotated screenshot.
- For API I add curl/Postman export or HAR; for UI a 30-60 second video beats a dozen screenshots.
- File names are structured: `20250713_login_403_staging.png`, not `image(3).png`.
- I verify ACL on S3/Confluence links so devs without VPN do not hit 403.
- Honestly: dumping attachments without navigation in the description is worse than one good screenshot and a log.

## How it works

```gherkin
Feature: Bug report attachments

  Scenario: Login failure on staging
    Given tester on build 4.2.1 staging EU
    When invalid password entered three times
    Then UI shows generic error instead of lockout message
    And attachment login_lockout_missing.png highlights toast text
    And attachment auth-service.log contains ERROR brute_force_guard
    And attachment repro_steps.mp4 shows full flow under 45 seconds
    And ticket body references each file by name in steps 4-6

  Scenario: API regression 500 on checkout
    Given Postman collection export checkout_v3.json attached
    When developer imports collection into local Postman
    Then same 500 reproduced with requestId from server log snippet
```

Sample Jira description fragment:

```
Steps:
1. Open https://staging.shop.example/login (see repro_steps.mp4)
2. Enter user qa_demo / wrong password x3
Expected: account lockout banner after 3rd attempt
Actual: generic "Something went wrong" (login_lockout_missing.png)
Logs: auth-service.log lines 1842-1855, requestId=9f2a...
Environment: staging EU, build 4.2.1, Chrome 125
```

What to include by bug type: a **screenshot** captures UI state; **video** shows timing and gestures; **logs** expose stack trace and correlation id; **URL/query** pinpoints entry; **config** matters when the bug depends on feature flags or tenant settings.

## Pitfalls

- **PII in screenshots.** Passport, card, real customer email: blur or use synthetic data.
- **Expiring links.** Presigned URLs die in 7 days; duplicate the file in TMS attachments.
- **Huge uncompressed video.** 200 MB blocks mobile devs; trim to the key fragment.
- **Log without timezone and build.** "Yesterday evening" does not reproduce; state UTC and version.
- **Full 4K desktop screenshot.** Crop to the error area or focus is lost.
