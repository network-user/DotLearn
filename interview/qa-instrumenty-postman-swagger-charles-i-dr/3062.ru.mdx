# Что такое Jira и как QA работает с тикетами?

## Суть

Jira (Atlassian) это TMS/issue tracker: Bug, Story, Task, Epic, workflow статусов, sprint board, связи blocks/relates, поля Environment, Priority, Components. QA создаёт баги с repro steps, actual/expected, attachments, линкует к story и release version; фильтрует open defects для regression и release sign-off.

## Что ответить на собесе

- Bug template: title, steps, expected, actual, env, build, severity/priority.
- Workflow: Open → In Progress → Ready for QA → Done / Reopen.
- JQL: `project = SHOP AND type = Bug AND status != Done AND fixVersion = 4.3.0`.
- Link issue: blocks, duplicates, relates to story SHOP-441.
- Честно: Jira не заменяет test case repo (TestRail/Zephyr); это tracking, не execution.

## Как это работает

```gherkin
Feature: Jira workflow for QA

  Scenario: File defect after checkout failure
    Given tester on staging build 4.3.0-rc1
    When payment returns 402 and UI silent
    Then bug SHOP-992 created with Charles session attached
    And fixVersion 4.3.0 and component Checkout set

  Scenario: Verify fix on Ready for QA
    Given developer moves SHOP-992 to Ready for QA
    When QA reruns steps on same build candidate
    Then status transitioned to Done or Reopen with comment

  Scenario: Release burndown query
    Given JQL filter open P1 bugs for fixVersion 4.3.0
    When count greater than zero
    Then release manager blocks prod deploy until triaged
```

Пример JQL:

```
project = SHOP AND issuetype = Bug AND priority in (Highest, High)
AND status in ("Open", "In Progress", "Reopened")
AND labels = regression-4.3.0
ORDER BY priority DESC, created ASC
```

Интеграции: Bitbucket commit `SHOP-992` in message, Confluence spec link, Slack via automation. Custom fields часто: Environment, Steps to Reproduce rich text, Found in Version.

## Подводные камни

- **Vague title «Не работает оплата».** Не searchable; конкретный symptom + endpoint.
- **Priority inflation.** Все Blocker; договоритесь о matrix severity vs priority.
- **Duplicate flood.** Search before create; use duplicate link.
- **Missing fixVersion.** Bug fixed но не в release notes scope.
- **Comment instead of status.** Workflow stale; transition with evidence.
