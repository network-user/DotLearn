# What QA tasks is Charles Proxy used for?

## In short

Charles covers tasks that need transparent HTTP(S) between client and server: API contract checks, mobile payload audit, security (tokens in cleartext), slow-network UX, mock/breakpoint scenarios, client vs server triage, HAR/session attachments to bugs. It complements Postman (manual API) and browser DevTools (web tab only).

## How to answer in an interview

- Mobile regression: proxy the phone, verify JSON fields and error codes.
- Payment/checkout: confirm PAN/CVV are not logged in body.
- Throttling 3G/Edge before retail app release.
- Breakpoint/Rewrite for negative paths without backend deploy.
- Repeat request with different Authorization for RBAC matrix.
- Honestly: not for load tests; JMeter/k6 on a dedicated stand.

## How it works

```gherkin
Feature: QA tasks with Charles

  Scenario: API contract on native app
    Given Charles captures iOS catalog requests
    When pagination cursor sent in query pageToken
    Then response items array length matches UI row count

  Scenario: Security review login
    Given SSL Proxy decrypts auth endpoint
    When POST /login inspected
    Then password field not repeated in analytics beacon POST

  Scenario: Bug report attachment
    Given failing checkout captured in Charles session
    When QA exports .chls with redacted tokens
    Then developer reproduces same 402 response locally
```

Typical workflow:

```
Enable proxy + SSL → repro bug → filter host api.stage.example
→ inspect request/response → optional Edit & Repeat
→ export session → link in Jira with steps
```

Charles shines when the client cannot be patched quickly (store build) and the server fix is not ready: Map Local unblocks UI QA.

## Pitfalls

- **Happy path only without throttling.** Prod users on 3G suffer; add a Throttling suite.
- **Charles without correlation id.** Dev cannot find log line; copy requestId from response.
- **Session from prod.** PII in .chls; test accounts only.
- **Forgotten Map Local.** False "bugs" for the next tester.
- **Ignoring WebSocket/gRPC.** HTTP-only view is incomplete; add other tools.
