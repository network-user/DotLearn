# В каких QA-задачах применяют Charles Proxy?

## Суть

Charles закрывает задачи, где нужен прозрачный HTTP(S) между client и server: API contract check, mobile payload audit, security (tokens in cleartext), slow network UX, mock/breakpoint сценарии, triage «клиент vs сервер», attachment HAR/session к багу. Это complement к Postman (ручной API) и browser DevTools (только web tab).

## Что ответить на собесе

- Mobile regression: proxy phone, verify JSON fields and error codes.
- Payment/checkout: убедиться, что PAN/CVV не логируются в body.
- Throttling 3G/Edge перед релизом retail app.
- Breakpoint/Rewrite для negative paths без backend deploy.
- Repeat request с другим Authorization для RBAC matrix.
- Честно: не для load test; JMeter/k6 на отдельном стенде.

## Как это работает

```gherkin
Feature: QA tasks with Charles

  Scenario: API contract on native app
    Given Charles captures iOS catalog requests
    When pagination cursor sent in query pageToken
    Then response items array length matches UI row count

  Scenario: Security review login
    Given SSL Proxy decrypts auth endpoint
    When POST /login inspected
    Then password field not repeated in analytics beacon POST

  Scenario: Bug report attachment
    Given failing checkout captured in Charles session
    When QA exports .chls with redacted tokens
    Then developer reproduces same 402 response locally
```

Типовой workflow:

```
Enable proxy + SSL → repro bug → filter host api.stage.example
→ inspect request/response → optional Edit & Repeat
→ export session → link in Jira with steps
```

Charles особенно ценен, когда клиент нельзя быстро patch (store build) а server ещё недоступен для fix: mock через Map Local unblock UI QA.

## Подводные камни

- **Только happy path без throttling.** Prod users на 3G страдают; добавь Throttling suite.
- **Charles без correlation id.** Dev не найдёт log line; копируй requestId из response.
- **Session from prod.** PII в .chls; только test accounts.
- **Забытый Map Local.** Ложные «баги» у следующего тестера.
- **Игнор WebSocket/gRPC.** HTTP-only вывод incomplete; дополняй другими tools.
