# How do you set up a response mock in Charles with Map Local?

## In short

Mocking in Charles replaces the remote response with a local file: capture a real response, save it, edit JSON/XML, bind the URL in Map Local. On the next request Charles serves the file to the client. Alternative: Map Remote to WireMock. QA validates UI and client logic without a live backend.

## How to answer in an interview

- Export Response → edit in `./mocks/catalog_empty.json`.
- Tools → Map Local → Add: host/path match plus local file path.
- Enable Map Local checkbox before the run.
- Document in the ticket: scenario mocked, not live API.
- Honestly: Map Local does not change latency and headers by default; for status 500 use Rewrite or Breakpoint.

## How it works

```gherkin
Feature: Charles Map Local mock

  Scenario: Empty product list UI
    Given captured GET /api/v1/products response saved
    When file edited to {"items":[],"total":0}
    And Map Local maps host api.stage.example path /api/v1/products
    Then mobile app shows empty catalog placeholder

  Scenario: Reuse mock across team
    Given mocks folder committed in qa-fixtures repo
    When teammate imports same Map Local xml settings
    Then identical behavior without re-export from Charles

  Scenario: Disable mock after test
    Given Map Local enabled for promotions endpoint
    When tester unchecks Enable Map Local
    Then app receives live staging data again
```

Workflow:

```
1. Repro GET /api/v1/wallet → Export Response → wallet_ok.json
2. Edit balance to 0 for insufficient funds UI
3. Map Local: https://api.stage.example/api/v1/wallet → wallet_ok.json
4. Reload app screen → verify banner "Top up required"
5. Disable Map Local before next case
```

Map Local matches protocol/host/port/path/query; with multiple rules the more specific one wins. Combine with Breakpoints for one-off edits without a file.

## Pitfalls

- **Wrong Content-Type.** Client parser fails; copy headers from export.
- **Absolute URL vs path.** Rule misses; duplicate host variants.
- **Cached ETag.** App does not refetch; hard reload or new user session.
- **Mock on write POST.** GET mock ok, POST still hits server; separate rule needed.
- **Committed PII in mock file.** Sanitize before git push.
