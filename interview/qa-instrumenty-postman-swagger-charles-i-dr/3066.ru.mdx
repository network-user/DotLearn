# Какие ключевые функции Charles знает QA?

## Суть

Основной набор: Structure/Sequence views, SSL Proxying, Breakpoints, Rewrite, Map Local/Remote, Throttling, Focus hosts, Repeat/Edit, WebSocket tab, Compose tool (optional), session export. QA комбинирует их для intercept, modify, slow down и document traffic.

## Что ответить на собесе

- SSL Proxying + root cert: decrypt HTTPS staging.
- Breakpoints: pause edit request/response.
- Rewrite/Map Local: automated mocks and header inject.
- Throttling: 3G/custom latency bandwidth.
- Focus: filter noise; Recording clear between cases.
- Честно: Viewer mode без edit на demo; license limits concurrent users.

## Как это работает

```gherkin
Feature: Charles feature set for QA

  Scenario: SSL Proxying decrypt checkout
    Given SSL Proxying enabled for *.payments.stage.example
    When POST charge captured
    Then JSON body visible not unknown tunnel

  Scenario: Throttling plus Breakpoint
    Given Throttling 3G profile active
    And breakpoint on GET /config
    When slow config response edited to 503
    Then app offline mode within 5 seconds

  Scenario: Session handoff
    Given 20 minute repro recorded
    When QA exports chls file to developer
    Then developer imports and sees same sequence without live repro
```

Краткий map функций:

```
Structure/Sequence     → find request, timing waterfall
SSL Proxying           → HTTPS decrypt
Breakpoints            → manual edit mid-flight
Rewrite / Map Local    → rules-based mock
Throttling             → network conditions
Repeat / Edit          → resend tweaked call
WebSocket tab          → frame inspect
Export session         → bug attachment
```

Advanced: External proxy chaining, DNS Spoofing (careful on shared network), Mirror to second Charles for pair debugging.

## Подводные камни

- **Все функции включены сразу.** Throttling + Breakpoint + Rewrite → flaky chaos; one knob per test.
- **Compose without understanding TCP.** Wrong tool for raw socket apps.
- **Export session with secrets.** Scrub Authorization before share.
- **WebSocket forgotten.** Only REST in report; incomplete triage.
- **Map Remote to prod by mistake.** Accidental live traffic mutation.
