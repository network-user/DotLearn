# How do you connect Charles Proxy on desktop and mobile?

## In short

Charles listens on port 8888 (default) and accepts HTTP(S) via system or manual proxy. Desktop: Proxy → macOS/Windows Proxy enables browser capture. Mobile: Wi‑Fi manual proxy to laptop IP plus root cert install for HTTPS. Without SSL Proxying settings you only see a CONNECT tunnel.

## How to answer in an interview

- Help → SSL Proxying → Install Charles Root Certificate on the tester machine.
- Proxy → SSL Proxying Settings → add `host:443` or a specific staging domain.
- Android/iOS: Wi‑Fi → manual proxy, desktop IP, port 8888; on phone open chls.pro/ssl.
- iOS: Settings → General → About → Certificate Trust Enable Full Trust.
- Honestly: phone VPN bypasses Wi‑Fi proxy; I disable VPN for the session.

## How it works

```gherkin
Feature: Charles proxy setup

  Scenario: Desktop browser capture
    Given Charles running on 192.168.0.14:8888
    When macOS proxy enabled via Proxy menu
    And Chrome opens https://shop.stage.example
    Then Structure shows decrypted GET /api/config with JSON body

  Scenario: Android app through Charles
    Given phone Wi-Fi proxy 192.168.0.14:8888
    And Charles root cert installed as user CA
    When native app starts checkout flow
    Then HTTPS requests listed unless app uses certificate pinning

  Scenario: Allow mobile device popup
    Given first request from unknown IP
    When Charles shows Allow dialog
    Then tester clicks Allow to capture that device traffic
```

Connection checklist:

```
Desktop:
  Charles → Proxy → Enable macOS/Windows Proxy
  Install root cert → trust in Keychain/Credential Manager
  SSL Proxying: *.stage.example:443

Mobile:
  Same LAN as laptop
  Wi-Fi proxy manual → laptop IP, port 8888
  Browse chls.pro/ssl → install profile → trust (iOS)
  Repeat SSL host list on Charles
```

After the session I disable system proxy so normal browsing works. I export `.chls` sessions to bugs with sensitive fields redacted.

## Pitfalls

- **Firewall blocks 8888.** Mobile shows "no internet"; open port or change listener.
- **Cert not trusted on iOS 16+.** Full Trust not enabled; HTTPS empty in Charles.
- **Android 7+ ignores user cert in app.** Needs debug build or network security config.
- **Forgotten system proxy overnight.** Mail stops syncing; make proxy-off a ritual.
- **Charles on corp VPN different IP.** Laptop IP changes; update phone proxy.
