# What does a sample request and response look like in Swagger UI?

## In short

After Execute, Swagger UI shows a curl/fetch snippet, request URL with substituted path/query, response status, headers, and body (pretty JSON). Spec examples may differ from live responses; QA compares both and attaches curl to bugs.

## How to answer in an interview

- Request: method, `https://api.stage.example/v2/users/a1b2`, Authorization and Accept headers.
- Response: `200`, body `{"id":"a1b2","email":"qa@corp.example","tier":"beta"}`.
- Copy curl → paste in terminal or Jira for repro.
- Schema tab shows required fields; Responses → 404 example for negative cases.
- Honestly: browser CORS may hide body; duplicate curl in Postman.

## How it works

```gherkin
Feature: Swagger UI request response sample

  Scenario: GET user by id Try it out
    Given operation GET /v2/users/{userId} with param userId a1b2
    When Execute clicked after Authorize bearer token
    Then request URL https://api.stage.example/v2/users/a1b2
    And response code 200
    And response body contains email field string

  Scenario: Attach curl to defect
    Given POST /refunds returns 422 validation error
    When tester copies curl from Swagger panel
    Then Jira bug includes curl reproducing same 422

  Scenario: Example value vs live mismatch
    Given documented example tier "standard"
    When live response tier "beta"
    Then QA files spec drift if product expects doc accuracy
```

Logical Swagger UI output after Execute:

```
Request URL
https://api.stage.example/v2/users/a1b2

Curl
curl -X GET "https://api.stage.example/v2/users/a1b2" \
  -H "accept: application/json" \
  -H "Authorization: Bearer eyJ..."

Server response
Code: 200
Body:
{
  "id": "a1b2",
  "email": "qa@corp.example",
  "tier": "beta"
}
```

The Parameters section before Execute shows path/query/header/body forms with schema types and required markers.

## Pitfalls

- **Screenshot of example instead of live Execute.** Bug evidence needs real response.
- **Token in curl screenshot.** Redact before attach.
- **Pretty print hides trailing spaces.** Client string assert fails.
- **204 No Content.** Swagger UI empty body; do not expect JSON.
- **Wrong server in dropdown.** Response from dev, bug filed against stage.
