# Extension limits: objects, methods, events

## In short

Extensions are powerful but bounded: you cannot change sealed platform objects, some metadata is adopt-only, procedure overrides are limited to annotations, and extension count and order affect build and performance. Knowing limits decides when main configuration changes are required.

## How to answer in an interview

- I list: cannot delete base objects, some adopted object properties are read-only, form change depth is limited.
- I talk about `&Instead` / `&Before` / `&After`: not every procedure can be annotated.
- I mention safe mode, blocked external resources, limits on subscriptions to system events.
- I honestly note attributes on base objects are added via adoption in extensions, not forbidden.
- I name organizational limits: multiple extensions, unified prefix, merge code review.

## How it works

Adopt a document and add string attribute "InternalCode". You cannot change an existing base attribute type or drop a tabular section; only additions and programmatic form changes.

Manager procedure override:

```bsl
&После("ПередЗаписью")
Процедура Расш1_ПередЗаписью(Отказ)

    Если ПустаяСтрока(ЭтотОбъект.Расш1_ВнутреннийКод) Тогда
        ЭтотОбъект.Расш1_ВнутреннийКод = СформироватьКодНаСервере();
    КонецЕсли;

КонецПроцедуры
```

If a procedure is not exported or the module is not adoptable, use an event subscription or SSL overridable module. Internal platform procedures and main subsystem composition cannot be overridden from an extension.

Event limits: subscriptions cannot replace all low-level DB events; call order across extensions must be agreed. When complexity exceeds extension fit (new posting model, different chart of accounts), branch configuration or agreed off-support work wins.

## Pitfalls

**"No attributes" myth.** Skipping extensions due to outdated belief; attributes are added with prefix and data migration.

**&Instead without merge discipline.** Full replacement of base procedures breaks upgrades.

**Adopting whole forms.** A base form copy in an extension needs manual merge every release.

**Performance.** Dozens of annotations on hot posting paths show up in the profiler.

**Version mismatch.** Extension built for 8.3.24 may not apply to an older platform without upgrade.
