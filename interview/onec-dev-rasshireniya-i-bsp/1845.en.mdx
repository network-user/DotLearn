# How to debug code in a configuration extension

## In short

Debug an extension from the designer on an infobase with cfe attached: set breakpoints in extension modules and start a session under the debugger bound to the right extension. Distinguish client and server context; annotated `&After` / `&Instead` procedures do run.

## How to answer in an interview

- I attach the extension to a production copy, enable "Debug → Attach" and "Start debugging".
- I set a breakpoint in the extension module and replay the scenario in user mode.
- For server code I verify the debugger is attached to rphost (file or cluster).
- I honestly note without cfe sources debugging works only if the vendor shipped cf/cfe with modules.
- I use the event log and targeted `Сообщить` when remote server attach is unavailable.

## How it works

Developer opens infobase "ERP_Test", activates extension "Warehouse_WMS", sets a stop in the document form module on `&After("AfterWriteAtServer")`.

```bsl
&После("ПриЗаписиНаСервере")
Процедура WMS_ПриЗаписиНаСервере(Отказ, ТекущийОбъект)

    Если Отказ Тогда
        Возврат;
    КонецЕсли;

    // <- breakpoint here
    WMS_ОтправитьЗадание(ТекущийОбъект.Ссылка);

КонецПроцедуры
```

Launch "1C:Enterprise" from the designer with debugging. On document save execution stops in the extension after base logic. The debugger shows `ТекущийОбъект` and call stack (base module → extension annotation).

For `&AtClient` code set client breakpoints; for `&AtServer` use server debug. With multiple extensions verify which one fired by temporarily disabling others. After cfe fix, update in the infobase and replay; record steps in Vanessa or a checklist for regression.

## Pitfalls

**Debugging production.** Breakpoints on live rphost are unacceptable; use a copy only.

**Wrong process.** Client breakpoint never hits server procedure and vice versa.

**Compiler optimization.** Rarely, non-executable lines are skipped; stop on a live statement.

**Third-party extensions.** Several cfe files change call order; debugging without disabling extras misleads.

**Build without symbols.** Shipping only .epf without cfe gives no sources for step-through.
