# How to add an attribute to a base object via extension

## In short

Add an attribute to a base catalog or document in an extension by adopting the object: it moves into cfe, you create a prefixed attribute, update the infobase configuration, and show the field on an adopted or programmatically changed form.

## How to answer in an interview

- I describe the chain: create extension → "Add to extension" on object → new attribute → update DB → form.
- I stress prefix (e.g. `CRM_`) to avoid collision with base names.
- I talk roles: include the attribute in extension roles or extend base roles.
- I honestly note you do not add attributes programmatically at runtime; only via metadata and DB update.
- I mention migration: fill values for existing items with a one-off processor.

## How it works

In the designer open extension "CRMCustomizations", on catalog "Counterparties" choose "Add to extension". In the extension branch add attribute `CRM_КодПартнера`, type String, length 36.

After "Configuration → Update database configuration" the field is available in API:

```bsl
Контрагент = Справочники.Контрагенты.НайтиПоНаименованию("ООО Альфа").ПолучитьОбъект();
Контрагент.CRM_КодПартнера = "AF-1024";
Контрагент.Записать();
```

Adopt the item form and drag the attribute to a tab, or add a control in `OnCreateAtServer`:

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)

    Если Элементы.Найти("CRM_КодПартнера") = Неопределено Тогда
        Поле = Элементы.Добавить("CRM_КодПартнера", Тип("ПолеФормы"), Элементы.ГруппаРеквизиты);
        Поле.ПутьКДанным = "Объект.CRM_КодПартнера";
    КонецЕсли;

КонецПроцедуры
```

Run a one-off fill processor for historical data. Before go-live check reports, exchange, and users without rights to the new attribute.

## Pitfalls

**No prefix.** Name `КодПартнера` may collide with a future base attribute.

**Missing form.** Attribute exists in metadata but users do not see it and assume the change "does not work".

**Base upgrade.** Adopted form conflict after ERP release needs manual merge.

**Indexing.** Index on a string field without need increases DB size and write time.

**Data exchange.** Add the attribute to XDTO/EnterpriseData conversion rules.
