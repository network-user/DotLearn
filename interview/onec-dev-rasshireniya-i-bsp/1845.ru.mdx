# Как отлаживать код в расширении конфигурации

## Суть

Отладка расширения выполняется из конфигуратора на базе с подключённым cfe: точки останова ставят в модули расширения, сеанс запускают под отладчиком с привязкой к нужному расширению. Важно различать клиентский и серверный контекст и помнить, что срабатывают аннотированные `&После` / `&Вместо` процедуры.

## Что ответить на собесе

- Я подключаю расширение к копии продуктивной базы, включаю «Отладка → Подключение» и запуск «Начать отладку».
- Ставлю breakpoint в модуле расширения, воспроизвожу сценарий в пользовательском режиме.
- Для серверного кода проверяю, что отладчик подключён к серверному процессу rphost (файловый или кластер).
- Честно уточняю: без исходников cfe отладка возможна только если поставщик передал cf/cfe с включёнными модулями.
- Использую журнал регистрации и `Сообщить` точечно, когда удалённый сервер недоступен для attach.

## Как это работает

Разработчик открывает базу «ERP_Тест», активирует расширение «Склад_WMS», в модуле формы документа ставит останов на процедуру с `&После("ПриЗаписиНаСервере")`.

```bsl
&После("ПриЗаписиНаСервере")
Процедура WMS_ПриЗаписиНаСервере(Отказ, ТекущийОбъект)

    Если Отказ Тогда
        Возврат;
    КонецЕсли;

    // <- breakpoint здесь
    WMS_ОтправитьЗадание(ТекущийОбъект.Ссылка);

КонецПроцедуры
```

Запуск «1С:Предприятие» из конфигуратора с отладкой. При записи документа выполнение останавливается в расширении после типовой логики. В окне отладчика смотрят `ТекущийОбъект`, стек вызовов (типовой модуль → аннотация расширения).

Если код в `&НаКлиенте`, breakpoint ставят на клиенте; для `&НаСервере` нужен server debug. При нескольких расширениях проверяют, какое из них реально сработало, временно отключая остальные. После правки cfe обновляют в базе и повторяют сценарий; для регрессии фиксируют шаги в Vanessa или чек-листе.

## Подводные камни

**Отладка на проде.** Breakpoint на боевом rphost недопустим; только копия.

**Не тот процесс.** Клиентский breakpoint не ловит серверную процедуру и наоборот.

**Оптимизация компилятора.** Редко, но строки без кода «перепрыгиваются»; останов ставят на исполняемую строку.

**Чужие расширения.** Несколько cfe меняют порядок вызова; отладка без отключения лишних вводит в заблуждение.

**Сборка без символов.** Поставка только .epf без cfe не даёт исходников для step-through.
