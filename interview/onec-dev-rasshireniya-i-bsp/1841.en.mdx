# What a 1C configuration extension is

## In short

An extension is a separate metadata and code layer attached to main configuration without taking it off vendor support. It adds objects, attributes, forms, roles, and procedure overrides while leaving vendor sources intact for upgrades.

## How to answer in an interview

- I define an extension as a cfe/cf package attached to base "UT", "ERP", "HRM", etc.
- I list capabilities: new catalogs, adopted documents with new attributes, `&After` annotations, subscriptions.
- I mention safe mode and extension compatibility with platform version.
- I honestly note extensions do not remove the need to test after every base upgrade.
- I distinguish extensions from external processors: extensions are part of infobase configuration.

## How it works

The team creates extension "CRM_Integration", adds an information register for message queue, and adopts catalog "Counterparties" with attribute "CRMIdentifier".

```bsl
&После("ПриЗаписи")
Процедура CRM_ПриЗаписиКонтрагента(Источник, Отказ)

    РегистрыСведений.ОчередьCRM.ДобавитьЗапись(Источник.Ссылка, Источник.CRM_ИдентификаторCRM);

КонецПроцедуры
```

Attachment happens in the designer or via `LoadCfg` / CI pipeline. The platform merges main configuration and active extension metadata at session start. Users see one UI; application code uses new attributes with extension prefix when set.

Base upgrade: new release arrives, compare report is checked, adopted conflicts fixed, posting and exchange scenarios rerun. An extension can be disabled to isolate a defect.

## Pitfalls

**Editing base instead of extension.** Takes configuration off support and removes the mechanism's benefits.

**Many extensions without order.** Several cfe files from different vendors conflict on annotations and subscriptions.

**Missed DB update.** New extension attribute without "Update database configuration" causes write errors.

**Prefixes and names.** Without prefix, name collision risk with base and other extensions grows.

**Safe mode.** Extension code is restricted like main code; bypass via external COM breaks security policy.
