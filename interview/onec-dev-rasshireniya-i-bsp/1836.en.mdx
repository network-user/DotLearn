# How overridable modules work in base configurations

## In short

An overridable module in an SSL-based product is an extension point: empty or minimal exported procedures called from vendor code. The implementer fills the procedure in main configuration or extension without editing the caller.

## How to answer in an interview

- I explain the pattern "base code calls `OverridableModule.Procedure()`".
- I show in a clean base the body is often empty and project logic lives in a separate common module.
- I mention SSL: dozens of overridable modules for print, access, exchange, additional attributes.
- I honestly note in extensions you override with `&After` / `&Instead` on adopted procedures when the module is adopted.
- I mention documenting which procedures are filled for the maintenance team.

## How it works

Base document posting code calls an overridable hook before writing movements:

```bsl
Процедура ПередЗаписьюДвижений(Документ, Отказ) Экспорт

    ПереопределяемыйПроведение.ПередЗаписьюДвижений(Документ, Отказ);

КонецПроцедуры
```

In the customer configuration `ПереопределяемыйПроведение` holds the implementation:

```bsl
Процедура ПередЗаписьюДвижений(Документ, Отказ) Экспорт

    Если Не ДопустимыйСклад(Документ.Склад) Тогда
        Отказ = Истина;
    КонецЕсли;

КонецПроцедуры
```

On base upgrade the caller may change while the hook signature stays stable. If a hook is removed in a new version, SSL release notes say so; the team moves logic to the new point.

In an extension you adopt the overridable module and add `&After("ПередЗаписьюДвижений")` without editing the base file.

## Pitfalls

**Copying the whole base module.** Duplicating instead of filling the hook loses upgrades.

**Signature changes.** Extra parameters in the project module break compatibility with the base call.

**Side effects in hooks.** Long work in an overridable procedure slows every document that calls it.

**Opaque to newcomers.** Without a map of overridable modules it is hard to find where a rule lives.

**Double fill.** Logic in both main configuration and extension may run twice after merge.
