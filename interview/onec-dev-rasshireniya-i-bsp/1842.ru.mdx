# Ограничения расширений: объекты, методы, события

## Суть

Расширения мощны, но не безграничны: нельзя менять закрытые объекты платформы, часть метаданных доступна только для заимствования, переопределение процедур ограничено аннотациями, а количество и порядок расширений влияют на сборку и производительность. Понимание лимитов определяет, когда нужна доработка основной конфигурации.

## Что ответить на собесе

- Я перечисляю: нельзя удалять типовые объекты, нельзя менять некоторые свойства заимствованного объекта без adopt, есть лимит на глубину изменений формы.
- Говорю про `&Вместо` / `&Перед` / `&После`: не каждая процедура доступна для аннотации.
- Упоминаю safe mode, запрет на часть внешних ресурсов, ограничения на подписки к системным событиям.
- Честно уточняю: реквизиты к типовому объекту как раз добавляют через заимствование в расширении, это не «запрет».
- Называю организационные лимиты: несколько расширений, единый префикс, code review merge.

## Как это работает

Заимствуют документ и добавляют реквизит «ВнутреннийКод» типа строка. Изменить тип существующего типового реквизита или удалить табличную часть нельзя: только добавление и программные изменения формы.

Переопределение процедуры менеджера:

```bsl
&После("ПередЗаписью")
Процедура Расш1_ПередЗаписью(Отказ)

    Если ПустаяСтрока(ЭтотОбъект.Расш1_ВнутреннийКод) Тогда
        ЭтотОбъект.Расш1_ВнутреннийКод = СформироватьКодНаСервере();
    КонецЕсли;

КонецПроцедуры
```

Если процедура не экспортная или модуль недоступен для заимствования, остаётся подписка на событие или переопределяемый модуль БСП. Нельзя переопределить внутренние процедуры платформы и изменить состав подсистем основной конфигурации из расширения.

Ограничения по событиям: подписка не может подменить все низкоуровневые события СУБД; порядок вызова подписок из разных расширений нужно согласовывать. При превышении сложности (новая модель проведения, другой план счетов) расширение уступает место ответвлению конфигурации или согласованному снятию с поддержки.

## Подводные камни

**Миф «реквизиты нельзя».** Отказ от расширения из-за устаревшего представления; реквизиты добавляют, но с префиксом и миграцией данных.

**&Вместо без merge-дискipline.** Полная замена типовых процедур ломает обновления.

**Заимствование всей формы.** Копия типовой формы в расширении требует ручного merge при каждом релизе.

**Производительность.** Десятки аннотаций на горячем пути проведения заметны в профилировщике.

**Несовместимость версий.** Расширение, собранное под 8.3.24, может не примениться к базе на более старой платформе без обновления.
