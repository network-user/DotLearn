# Подписки на события и правка модуля объекта: в чём разница

## Суть

Модуль объекта содержит обработчики жизненного цикла конкретного экземпляра и вызывается платформой напрямую. Подписка на событие вынесена в общий модуль и срабатывает для выбранного типа метаданных без изменения модуля объекта. Первый путь теснее связан с объектом, второй удобнее для сквозных правил и расширений типовой.

## Что ответить на собесе

- Я говорю: модуль объекта знает `ЭтотОбъект`, подписка получает `Источник` как параметр.
- Подписка подключается к типу документа/справочника целиком; модуль объекта только к одному объекту метаданных.
- Для типовой без снятия с поддержки предпочитаю подписку или `&После` в расширении, а не правку модуля объекта в основе.
- Честно уточняю: сложная логика проведения, завязанная на приватные процедуры модуля, в подписке дублируется или выносится в общий сервис.
- Упоминаю порядок: сначала модуль объекта, затем подписки (уточняю по версии платформы в проекте).

## Как это работает

В модуле документа «Заказ» процедура `ПередЗаписью` проверяет заполнение табличной части:

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи)

    Если Объект.Товары.Количество() = 0 Тогда
        Отказ = Истина;
        Сообщить("Табличная часть пуста");
    КонецЕсли;

КонецПроцедуры
```

Сквозной запрет редактирования закрытых периодов для нескольких документов оформляют подпиской:

```bsl
Процедура КонтрольЗакрытогоПериодаПередЗаписью(Источник, Отказ, РежимЗаписи, РежимПроведения) Экспорт

    Если Источник.Дата < Константы.ДатаЗапретаРедактирования.Получить() Тогда
        Отказ = Истина;
    КонецЕсли;

КонецПроцедуры
```

Подписка переиспользуется для «Заказа», «Реализации», «Поступления» одним обработчиком. Модуль объекта остаётся про специфику одного документа. В расширении модуль объекта заимствуют и дополняют аннотацией; подписку добавляют отдельным объектом метаданных.

## Подводные камни

**Дублирование.** Одна проверка и в модуле, и в подписке срабатывает дважды.

**Не видно из модуля.** Новый разработчик читает только модуль объекта и пропускает подписки.

**Производительность.** Несколько подписок на `ПередЗаписью` у всех документов добавляют latency.

**Контекст.** Подписка не видит локальные переменные модуля объекта; нужен явный API.

**Обновление типовой.** Правка модуля объекта в основе перезаписывается; подписка в cfe переживает релиз легче.
