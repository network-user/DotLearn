# Способы переопределения процедур и функций в расширениях

## Суть

В расширении конфигурации процедуры типовых модулей меняют через аннотации платформы: `&Перед`, `&После`, `&Вместо` и изменение заимствованного тела. Дополнительно подключают подписки на события и собственные общие модули, не трогая поставщика.

## Что ответить на собесе

- Я перечисляю аннотации и когда какую выбираю: `&После` для дополнения, `&Вместо` только при полной замене с осознанным риском.
- Показываю заимствование объекта метаданных или модуля перед переопределением.
- Упоминаю подписки как альтернативу, когда нельзя заимствовать модуль.
- Честно говорю: `&Вместо` без вызова базовой логики частая причина регрессий после обновления.
- Называю инструмент проверки: сравнение cfe, smoke-тест проведения, журнал регистрации.

## Как это работает

Заимствуют модуль менеджера документа и добавляют обработчик после типовой процедуры:

```bsl
&После("ОбработкаПроведения")
Процедура Расш1_ОбработкаПроведения(Отказ, РежимПроведения)

    ЗарегистрироватьДвижениеВWMS(ЭтотОбъект);

КонецПроцедуры
```

Для изменения результата функции используют `&Вместо` с вызовом исходной реализации через цепочку платформы или пересчёт только нужного фрагмента:

```bsl
&Вместо("ПолучитьСкидку")
Функция Расш1_ПолучитьСкидку(Партнер, Сумма)

    БазоваяСкидка = ПродолжитьВызов(Партнер, Сумма);
    Возврат БазоваяСкидка + ДополнительнаяСкидкаПартнера(Партнер);

КонецФункции
```

Если модуль недоступен для заимствования, создают подписку на `ПриПроведении` или заполняют переопределяемый модуль БСП. Порядок: сначала штатные точки расширения типовой, затем аннотации, подписки в последнюю очередь.

## Подводные камни

**&Вместо без `ПродолжитьВызов`.** Типовая логика молча исчезает; после обновления появляются «тихие» ошибки учёта.

**Конфликт аннотаций.** Два расширения с `&Вместо` на одну процедуру требуют согласования порядка подключения.

**Заимствование без необходимости.** Лишние заимствованные формы усложняют merge при обновлении.

**Клиентский контекст.** Аннотация на `&НаКлиенте` процедуре не переносит серверные вызовы автоматически.

**Тестовое покрытие.** Без автотеста проведения регрессию ловят только пользователи в проде.
