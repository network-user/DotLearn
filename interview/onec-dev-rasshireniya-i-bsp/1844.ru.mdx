# Как добавить реквизит в типовой объект через расширение

## Суть

Новый реквизит к типовому справочнику или документу добавляют в расширении через заимствование объекта: объект попадает в cfe, в нём создают реквизит с префиксом расширения, обновляют конфигурацию БД и выводят поле на заимствованную или программно изменённую форму.

## Что ответить на собесе

- Я описываю цепочку: создать расширение → «Добавить в расширение» на объект → новый реквизит → обновить БД → форма.
- Подчёркиваю префикс (например `CRM_`), чтобы не конфликтовать с типовой.
- Говорю про права: новый реквизит включают в роли расширения или дополняют типовые.
- Честно уточняю: программно в runtime реквизит не добавляют; только через метаданные и DDL обновления.
- Упоминаю миграцию: заполнение значения для существующих элементов отдельной обработкой.

## Как это работает

В конфигураторе открывают расширение «ДоработкиCRM», в дереве метаданных на справочнике «Контрагенты» выбирают «Добавить в расширение». В ветке расширения добавляют реквизит `CRM_КодПартнера`, тип «Строка», длина 36.

После «Конфигурация → Обновить конфигурацию базы данных» поле доступно в API:

```bsl
Контрагент = Справочники.Контрагенты.НайтиПоНаименованию("ООО Альфа").ПолучитьОбъект();
Контрагент.CRM_КодПартнера = "AF-1024";
Контрагент.Записать();
```

Форму элемента заимствуют и перетаскивают реквизит на вкладку или добавляют элемент в `ПриСозданииНаСервере`:

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)

    Если Элементы.Найти("CRM_КодПартнера") = Неопределено Тогда
        Поле = Элементы.Добавить("CRM_КодПартнера", Тип("ПолеФормы"), Элементы.ГруппаРеквизиты);
        Поле.ПутьКДанным = "Объект.CRM_КодПартнера";
    КонецЕсли;

КонецПроцедуры
```

Для исторических данных запускают одноразовую обработку заполнения. Перед выкладкой проверяют отчёты, обмен и права пользователей без роли на новый реквизит.

## Подводные камни

**Без префикса.** Имя `КодПартнера` может совпасть с будущим реквизитом типовой.

**Забытая форма.** Реквизит есть в метаданных, пользователь его не видит и считает, что доработка «не работает».

**Обновление типовой.** Конфликт заимствованной формы после релиза ERP требует ручного merge.

**Индексирование.** Включение индекса на строковом поле без необходимости увеличивает размер БД и время записи.

**Обмен данными.** Новый реквизит нужно добавить в правила конвертации XDTO/EnterpriseData.
