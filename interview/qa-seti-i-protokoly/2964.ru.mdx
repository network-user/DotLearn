# Структура IP-адреса: что проверяет QA в сетевых багах?

## Суть

IP-адрес на канальном уровне инкапсулирован в L3 заголовок: версия (4 или 6), source, destination, TTL/hop limit. IPv4 пишут четырьмя октетами, IPv6 восемью hextet блоками с правилами сжатия нулей. Понимание формата помогает читать логи балансировщика, трассировку иGeo-ограничения по подсети.

## Что ответить на собесе

- Маска / длина префикса определяет размер подсети: /24 в v4 это 256 адресов, в v6 /64 стандарт на линк.
- Loopback 127.0.0.1 и ::1 всегда локальные; докер-сервисы часто публикуют 127.0.0.1:port только на хосте.
- Link-local fe80::/10 и 169.254.0.0/16 не маршрутизируются в интернет.
- Честно: в багрепорте всегда пишу полный адрес и префикс; «не пингуется» без маски бесполезно.

## Как это работает

Сопоставление записей:

| Версия | Пример | Длина | Типичный тест |
|--------|--------|-------|---------------|
| IPv4 | 192.168.1.50/24 | 32 бит | home lab |
| IPv6 | 2001:db8::ac1:32/64 | 128 бит | cloud LB |

Gherkin валидации конфигурации:

```gherkin
Feature: Корректность IP в конфиге деплоя

  Scenario: Whitelist принимает только CIDR
    Given security group rule "198.51.100.0/24"
    When QA с IP 198.51.100.77 вызывает /internal
    Then доступ разрешён

  Scenario: Одиночный IP без маски отклоняется в UI админки
    Given поле "Allowed CIDR"
    When оператор вводит "203.0.113.5"
    Then форма показывает ошибку валидации
    And сохранение заблокировано

  Scenario: IPv6 literal в URL health-check
    Given health URL "https://[2001:db8::1]/ready"
    When оркестратор опрашивает pod
    Then probe успешен
    And в логе нет parse error
```

`traceroute`/`tracert` показывает, на каком hop теряется пакет; для v6 использую `traceroute6`.

## Подводные камни

- **Один адрес, две роли.** VIP балансировщика и pod IP в логах смешиваются.
- **Сжатие :: в IPv6.** Два `::` в одном адресе невалидно, конфиг не поднимается.
- **Reverse DNS.** Почтовые тесты падают без PTR, хотя HTTP OK.
- **GeoIP по устаревшей базе.** QA VPN в «не том» регионе.
- **Static vs DHCP lease.** После ребута другой IP, whitelist ломается.
