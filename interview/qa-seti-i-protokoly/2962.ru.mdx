# IPv4: зачем QA понимать 32-битный адрес?

## Суть

IPv4 задаёт 32-битный адрес узла в сети (четыре октета в десятичной записи). Маршрутизаторы смотрят destination IP в заголовке пакета и выбирают next hop. Для тестировщика важны private ranges (10/8, 172.16/12, 192.168/16), loopback и отличие «доступен ping» от «открыт нужный порт».

## Что ответить на собесе

- Адрес делится на сеть и хост маской; ACL и security group режут трафик по подсетям.
- NAT маскирует внутренние адреса: в логах приложения вижу 10.x, в access.log nginx публичный IP клиента.
- Исчерпание публичных IPv4 решают CGNAT и переход на v6; staging часто сидит в RFC1918.
- Честно: ping успешен, но HTTPS закрыт firewall; проверяю конкретный порт curl, а не только ICMP.

## Как это работает

Разбор `203.0.113.42/24`:

| Часть | Значение | Роль |
|-------|----------|------|
| Сеть | 203.0.113.0 | маршрут в интернет |
| Хост | .42 | конкретная VM |
| Broadcast | .255 | редко в облаке |

Gherkin connectivity checklist:

```gherkin
Feature: IPv4 доступность сервиса

  Scenario: API отвечает по публичному IPv4
    Given сервис слушает 203.0.113.42:443
    When QA выполняет curl https://203.0.113.42/health
    Then HTTP статус 200
    And TLS сертификат валиден для CN/SAN

  Scenario: Private IP недоступен с интернета
    Given backend только на 10.20.30.40:8080
    When QA curl с внешней машины без VPN
    Then соединение timeout или connection refused
    And в багрепорте указан ожидаемый ACL deny

  Scenario: Whitelist по подсети офиса
    Given ACL разрешает 198.51.100.0/24
    And QA машина 198.51.100.15
    When GET /admin
    Then статус не 403
```

Wireshark фильтр `ip.addr == 203.0.113.42` помогает отделить «пакет не дошёл» от «дошёл, RST от приложения».

## Подводные камни

- **Путают L3 и L7.** IP reachable, сертификат на другой hostname.
- **Docker bridge 172.17.0.0/16.** Локально работает, в CI другая сеть.
- **Multiple NIC.** Ответ уходит с «не того» source IP, asymmetric routing.
- **0.0.0.0 bind vs connect.** Сервис слушает все интерфейсы, firewall режет один.
- **Литералы в конфиге.** Staging IP захардкожен, DNS уже сменился.
