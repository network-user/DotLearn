# ESM and Commander.js: different jobs

## In short

ESM (ECMAScript Modules) is the standard JS module system: static `import`/`export`, tree shaking, native support in browsers and Node. Commander.js is an npm library for CLIs: it parses argv, defines flags/subcommands, and prints help. They solve orthogonal problems: code organization vs command-line UX.

## How to answer in an interview

- ESM files: `"type": "module"` in package.json or a `.mjs` extension.
- Commander builds `program.option('-p, --port <n>')` and `program.parse()`.
- Import Commander as ESM: `import { Command } from 'commander'`, not `require`.
- Honestly: small CLIs can use `node --experimental-strip-types` + `util.parseArgs`; Commander pays off with nested subcommands.
- Bundlers and test runners must understand ESM resolution (`exports` field).

## How it works

ESM utilities module:

```ts
export function sumValues(values: number[]) {
  return values.reduce((acc, n) => acc + n, 0);
}

export const DEFAULT_PORT = 4173;
```

CLI entry with Commander (ESM):

```ts
#!/usr/bin/env node
import { Command } from 'commander';
import { sumValues, DEFAULT_PORT } from './math-utils.js';

const program = new Command();

program
  .name('stats-cli')
  .description('Small demo CLI')
  .option('-p, --port <number>', 'preview port', String(DEFAULT_PORT))
  .argument('<nums...>', 'numbers to sum')
  .action((nums: string[], opts: { port: string }) => {
    const total = sumValues(nums.map(Number));
    console.log(`sum=${total} port=${opts.port}`);
  });

program.parse();
```

`import './math-utils.js'` needs an explicit extension in Node ESM. Commander generates `--help`, validates arity, and supports `.command('deploy')` nesting. ESM provides a static graph for bundlers; Commander stays a runtime dependency only in Node tooling, not in browser bundles.

## Pitfalls

- **CJS interop.** `require()` of ESM from legacy scripts fails; migrate entries to import.
- **Shebang + type module.** A missing `.js` extension on relative imports breaks execution.
- **Dual package hazard.** A lib with both CJS and ESM entries may resolve twice (duplicate state).
- **Commander version API.** v12 vs v11 differs in `program.opts()` typing.
- **Confusing interview question.** ESM is not an alternative to Commander; compare it to CommonJS, not CLI libs.
