# script type="module" в браузере

## Суть

`<script type="module">` говорит браузеру загрузить скрипт как ES module: strict mode, own scope, static `import`/`export`, deferred execution (после parse HTML). Relative imports требуют full URL path с extension. Classic `<script>` без type не может использовать `import` syntax.

## Что ответить на собесе

- Module auto defer: не блокирует HTML parse как blocking classic script без defer.
- Top-level `await` разрешён в module graph.
- CORS: module scripts treat cross-origin как CORS resource (нужен correct headers).
- Честно: prod почти всегда bundler (Vite/Webpack); native modules в dev и для tiny demos.
- `importmap` (`<script type="importmap">`) мапит bare specifiers на URLs без bundler.

## Как это работает

`index.html`:

```html
<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8" />
    <title>ESM demo</title>
  </head>
  <body>
    <script type="module" src="/src/bootstrap.js"></script>
  </body>
</html>
```

`src/math-helpers.js`:

```ts
export function mergeTotals(a: number, b: number) {
  return a + b;
}
```

`src/bootstrap.js`:

```ts
import { mergeTotals } from './math-helpers.js';

console.log(mergeTotals(4, 9));
```

Browser fetch chain: `bootstrap.js` → parse static import → fetch `math-helpers.js` → evaluate in dependency order. Inline module:

```html
<script type="module">
  import { mergeTotals } from './src/math-helpers.js';
</script>
```

## Подводные камни

- **Missing extension.** `./math-helpers` 404 в browser ESM, хотя bundler прощает.
- **File protocol.** `file://` ломает module CORS; нужен dev server.
- **Mixed classic + module order.** Side-effect classic script до module может surprise по init order.
- **Duplicate loads.** Два entry module импортируют один URL: browser cache dedupe, но осторожно с side effects.
- **No import in workers без type.** Worker нужен `type: 'module'` в constructor options.
