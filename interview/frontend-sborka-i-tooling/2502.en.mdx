# Loaders and plugins in Webpack

## In short

A loader transforms a single file into a JS module Webpack understands: TS → JS, SCSS → CSS string, assets → URLs. A plugin hooks the whole compiler lifecycle: split chunks, emit HTML, define env, analyze bundles. Rule of thumb: loader = per-file pipeline, plugin = global orchestration.

## How to answer in an interview

- `module.rules[].test` plus `use: ['style-loader', 'css-loader']` chains right to left.
- `babel-loader` with `@babel/preset-env` for syntax downlevel and polyfill policy.
- `HtmlWebpackPlugin` generates `index.html` with injected script tags.
- Honestly: Vite/Rollup plugins often replace loaders; Webpack 5 has built-in asset modules (`type: 'asset/resource'`).
- Misconfigured loader order breaks source maps (CSS before PostCSS).

## How it works

```ts
import path from 'node:path';
import HtmlWebpackPlugin from 'html-webpack-plugin';
import type { Configuration } from 'webpack';

const config: Configuration = {
  entry: './src/main.ts',
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve('dist'),
    clean: true,
  },
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        exclude: /node_modules/,
        use: {
          loader: 'ts-loader',
          options: { transpileOnly: true },
        },
      },
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './public/index.html',
      inject: 'body',
    }),
  ],
};

export default config;
```

Loader chain: `main.ts` → ts-loader → JS AST → webpack graph. Plugin `emit` hook adds `index.html` to assets. `MiniCssExtractPlugin` (prod) emits CSS files instead of style-loader runtime injection.

## Pitfalls

- **Loader reach.** A heavy loader on all of `node_modules` kills build cache.
- **Plugin order.** `DefinePlugin` must run before optimizations or dead branches will not fold.
- **Duplicate PostCSS.** Both css-loader and a separate postcss-loader may run PostCSS.
- **Asset inline limit.** `asset/inline` base64 bloats JS; large SVGs fit `resource` better.
- **Legacy vs Webpack 5.** Old tutorials with `file-loader`/`url-loader` are replaced by asset modules.
