# Роль package-lock.json

## Суть

`package-lock.json` фиксирует полный resolved dependency tree: exact version, tarball URL, integrity hash, nested requires. npm создаёт и обновляет его на `install`/`update`. Цель: одинаковый `node_modules` на laptop, CI и prod image при том же lock commit.

## Что ответить на собесе

- Коммитим lock в VCS; `.gitignore` lock только для publishable libraries (редко).
- `npm ci` строго по lock, удаляет node_modules, быстрее CI чем `install`.
- `lockfileVersion` 3 (npm 9+) другой формат; не смешивай генерацию разными npm major.
- Честно: security patch transitive dep требует `npm audit fix` или overrides, не ручной edit lock.
- pnpm/yarn используют свои locks; не дублируй три менеджера в одном repo.

## Как это работает

После `npm install lodash@^4.17`:

```json
{
  "name": "demo-app",
  "lockfileVersion": 3,
  "packages": {
    "node_modules/lodash": {
      "version": "4.17.21",
      "resolved": "https://registry.npmjs.org/lodash/-/lodash-4.17.21.tgz",
      "integrity": "sha512-v2Tx..."
    }
  }
}
```

CI pipeline:

```bash
npm ci
npm run build
npm test
```

`ci` падает, если `package.json` и lock расходятся. Overrides (npm 8.3+):

```json
{
  "overrides": {
    "lodash": "4.17.21"
  }
}
```

форсят версию во всём дереве без ручного merge thousand lines lock.

## Подводные камни

- **Deleted lock.** Fresh `npm install` может подтянуть breaking transitive major overnight.
- **Merge conflicts.** Blind accept theirs/lines corrupt integrity; regenerate lock.
- **Global npm update.** Разработчик с npm 6 vs 10 пишет разный lockfileVersion.
- **Optional deps.** Lock с `fsevents` на macOS fail на Linux CI без `--omit=optional` policy.
- **Phantom drift.** `npm install pkg` без commit lock ломает reproducibility для команды.
