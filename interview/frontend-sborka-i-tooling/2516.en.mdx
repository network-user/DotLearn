# Lazy loading on the frontend

## In short

Lazy loading defers loading a resource until it is needed: a code chunk per route, below-the-fold images, or the next data page on scroll. Goal: smaller initial payload, faster FCP/LCP, less bandwidth. Implemented natively (`loading="lazy"`), with Intersection Observer, or framework `React.lazy` / dynamic `import()`.

## How to answer in an interview

- Route-level split: admin panel chunk loads only after an auth redirect.
- Images: `loading="lazy"` plus `width/height` against CLS; do not lazy-load the hero LCP image.
- Lists: virtualize and fetch the next page when a sentinel nears view.
- Honestly: over-splitting causes chunk waterfalls; balance vendor and route chunks.
- Prefetch with `link rel="modulepreload"` for likely next navigation.

## How it works

React route lazy:

```ts
import { lazy, Suspense } from 'react';

const ReportsPage = lazy(() => import('./pages/ReportsPage'));

export function AppRouter() {
  return (
    <Suspense fallback={<p>Loading reports…</p>}>
      <ReportsPage />
    </Suspense>
  );
}
```

Native image lazy:

```html
<img
  src="/assets/diagram.webp"
  alt="Architecture diagram"
  width="640"
  height="360"
  loading="lazy"
  decoding="async"
/>
```

Intersection Observer data:

```ts
const sentinel = document.querySelector('#feed-sentinel');
const observer = new IntersectionObserver(([entry]) => {
  if (entry.isIntersecting) void loadNextFeedPage();
});
if (sentinel) observer.observe(sentinel);
```

The bundler emits a separate async chunk; the browser fetches on the first `import()` call.

## Pitfalls

- **Lazy LCP.** A lazy hero banner hurts Core Web Vitals.
- **Layout shift.** Placeholders without aspect-ratio jump on load.
- **SEO routes.** SSR/SSG routes need eager hydration of critical HTML.
- **Waterfall navigation.** No prefetch → blank Suspense on every click.
- **Observer threshold.** `threshold: 0` fires too early on long pages.
