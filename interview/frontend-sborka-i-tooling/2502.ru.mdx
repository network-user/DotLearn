# Loader и plugin в Webpack

## Суть

Loader трансформирует отдельный файл в JS module, который Webpack понимает: TS → JS, SCSS → CSS string, asset → URL. Plugin подключается к lifecycle compiler hooks всей сборки: split chunks, HTML emit, define env, bundle analyze. Правило: loader = per-file pipeline, plugin = global orchestration.

## Что ответить на собесе

- `module.rules[].test` + `use: ['style-loader', 'css-loader']` chain справа налево.
- `babel-loader` + `@babel/preset-env` для syntax downlevel и polyfill policy.
- `HtmlWebpackPlugin` генерит `index.html` с injected script tags.
- Честно: Vite/Rollup plugins часто заменяют loaders; в Webpack 5 asset modules встроены (`type: 'asset/resource'`).
- Misconfigured loader order ломает source maps (css before postcss).

## Как это работает

```ts
import path from 'node:path';
import HtmlWebpackPlugin from 'html-webpack-plugin';
import type { Configuration } from 'webpack';

const config: Configuration = {
  entry: './src/main.ts',
  output: {
    filename: '[name].[contenthash].js',
    path: path.resolve('dist'),
    clean: true,
  },
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        exclude: /node_modules/,
        use: {
          loader: 'ts-loader',
          options: { transpileOnly: true },
        },
      },
      {
        test: /\.css$/i,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './public/index.html',
      inject: 'body',
    }),
  ],
};

export default config;
```

Loader chain: `main.ts` → ts-loader → JS AST → webpack graph. Plugin `emit` hook добавляет `index.html` в assets. `MiniCssExtractPlugin` (prod) выносит CSS в файлы вместо style-loader runtime inject.

## Подводные камни

- **Loader pitch.** Слишком тяжёлый loader на всём `node_modules` убивает build cache.
- **Plugin order.** `DefinePlugin` должен быть до оптимизаций, иначе dead branch не схлопнется.
- **Duplicate postcss.** css-loader и отдельный postcss-loader оба запускают postcss.
- **Asset inline limit.** `asset/inline` base64 раздувает JS; крупные svg лучше `resource`.
- **Legacy vs Webpack 5.** Старые туториалы с `file-loader`/`url-loader` заменены asset modules.
