# npm: менеджер пакетов Node

## Суть

npm (Node Package Manager) это CLI + registry для публикации и установки JS пакетов. `npm install` читает `package.json`, резолвит semver ranges, пишет `node_modules` и lockfile. Registry по умолчанию `registry.npmjs.org`; scope packages `@org/name` для private/public monorepo artifacts.

## Что ответить на собесе

- `npm init` / `npm create vite@latest` scaffold проект.
- `npm install -D vitest` devDependency; без `-D` попадает в production deps.
- `npx` / `npm exec` одноразовый запуск бинаря без global install.
- Честно: в monorepo часто pnpm/yarn из-за disk dedup и strict deps; npm workspaces догнал, но hoisting отличается.
- `npm publish --access public` для scoped пакета; private через org billing или Verdaccio.

## Как это работает

```bash
npm init -y
npm install zod
npm install -D typescript vitest
```

```ts
import { z } from 'zod';

const Email = z.string().email();

export function parseSubscriber(input: unknown) {
  return Email.parse(input);
}
```

`package.json` получает:

```json
{
  "dependencies": { "zod": "^3.23.8" },
  "devDependencies": { "typescript": "^5.6.2", "vitest": "^2.0.5" }
}
```

`node_modules/.bin/vitest` symlink для scripts. Lock фиксирует transitive tree. CI: `npm ci && npm test`. Overrides/resolutions патчат уязвимую transitive без fork lib.

## Подводные камни

- **Global install drift.** `npm i -g` ломает reproducibility; prefer project-local bin.
- **Phantom deps.** Flat hoisting позволяет import пакета не в package.json (ломается при pnpm strict).
- **Left-pad incident lesson.** Unpublish policy; pin critical deps or vendor tarball.
- **Script lifecycle.** `postinstall` supply-chain risk; review deps scripts в enterprise.
- **Registry mirror lag.** Internal proxy отстаёт от npm → missing latest security patch.
