# Lazy loading на фронтенде

## Суть

Lazy loading откладывает загрузку ресурса до момента need: code chunk по route, image ниже fold, data page при scroll. Цель: меньше initial payload, быстрее FCP/LCP, экономия bandwidth. Реализуется нативно (`loading="lazy"`), Intersection Observer, или framework `React.lazy` / dynamic `import()`.

## Что ответить на собесе

- Route-level split: admin panel chunk грузится только после auth redirect.
- Images: `loading="lazy"` + `width/height` против CLS; hero LCP image не lazy.
- Lists: virtualize + fetch next page при приближении sentinel.
- Честно: over-splitting даёт chunk waterfall; балансируй vendor + route chunks.
- Prefetch `link rel="modulepreload"` для likely next navigation.

## Как это работает

React route lazy:

```ts
import { lazy, Suspense } from 'react';

const ReportsPage = lazy(() => import('./pages/ReportsPage'));

export function AppRouter() {
  return (
    <Suspense fallback={<p>Loading reports…</p>}>
      <ReportsPage />
    </Suspense>
  );
}
```

Native image lazy:

```html
<img
  src="/assets/diagram.webp"
  alt="Architecture diagram"
  width="640"
  height="360"
  loading="lazy"
  decoding="async"
/>
```

Intersection Observer data:

```ts
const sentinel = document.querySelector('#feed-sentinel');
const observer = new IntersectionObserver(([entry]) => {
  if (entry.isIntersecting) void loadNextFeedPage();
});
if (sentinel) observer.observe(sentinel);
```

Bundler эмитит отдельный async chunk; browser fetch по первому `import()` call.

## Подводные камни

- **Lazy LCP.** Hero banner с lazy hurt Core Web Vitals.
- **Layout shift.** Placeholder без aspect-ratio прыгает при load.
- **SEO-only routes.** SSR/SSG route должен eager hydrate critical HTML.
- **Waterfall navigation.** Нет prefetch → blank Suspense на каждый клик.
- **Observer threshold.** `threshold: 0` срабатывает слишком рано на длинных pages.
