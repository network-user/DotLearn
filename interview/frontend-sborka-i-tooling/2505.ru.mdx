# Babel: транспиляция современного JS

## Суть

Babel парсит современный JS/TS синтаксис в AST, прогоняет plugin pipeline и генерирует код под target environments (browserslist). Он не type-check: TypeScript types снимаются отдельно (`tsc` или `babel-plugin-transform-typescript`). Preset `@babel/preset-env` подбирает transforms по supported features matrix.

## Что ответить на собесе

- Plugins атомарны (`@babel/plugin-transform-arrow-functions`), presets группируют набор.
- `useBuiltIns: 'usage'` + `core-js` вставляет только нужные polyfills per file.
- В Vite/esbuild Babel часто только для JSX legacy или custom macros, не для full transpile.
- Честно: для зелёного field evergreen browsers многие команды убрали Babel из prod, оставив tsc/swc.
- Source maps: `sourceMaps: true` сохраняет связь output ↔ original для debug.

## Как это работает

Input (modern):

```ts
const greetVisitor = (name: string) => `Welcome, ${name}!`;

console.log(greetVisitor('Ada'));
```

Babel output (ES5 style):

```js
var greetVisitor = function greetVisitor(name) {
  return 'Welcome, ' + name + '!';
};

console.log(greetVisitor('Ada'));
```

Config:

```json
{
  "presets": [
    [
      "@babel/preset-env",
      {
        "targets": ">0.5%, not dead",
        "modules": false
      }
    ],
    "@babel/preset-typescript"
  ]
}
```

`modules: false` оставляет ESM для Rollup/Webpack tree shaking. CLI: `babel src --out-dir dist --extensions ".ts,.tsx"`. Integrates как `babel-loader` rule или `@vitejs/plugin-react` с optional babel stage for legacy decorators.

## Подводные камни

- **Double transpile.** Babel + tsc оба конвертят TS → медленный CI и confusing maps.
- **Polyfill bloat.** `useBuiltIns: 'entry'` тянет весь core-js если не аккуратен.
- **Plugin order.** Некоторые proposals конфликтуют; class properties vs decorators stage mismatch.
- **Runtime helpers.** `@babel/runtime` забытый → duplicated helper bytes в каждом file.
- **False compatibility.** Babel синтаксис не гарантирует platform APIs (fetch, Promise) без polyfill.
