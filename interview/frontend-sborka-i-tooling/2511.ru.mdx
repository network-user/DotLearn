# Profiler в Chrome DevTools

## Суть

Performance/React Profiler в DevTools записывает timeline выполнения: JS call stack, layout, paint, long tasks. React DevTools Profiler снимает commit phases компонентов (render vs layout effects). Цель: найти bottlenecks (лишние re-render, тяжёлый main thread) с evidence, не гадать.

## Что ответить на собесе

- Performance panel: Record → reproduce jank → анализ Main thread flame chart.
- Long task >50ms блокирует input; ищи forced sync layout (`offsetWidth` после write).
- React Profiler: highlight why component rendered (props/state/context change).
- Честно: профилируй на throttled CPU + production build; dev HMR искажает цифры.
- Memory tab отдельно для leak; Performance не заменяет heap snapshot.

## Как это работает

Suspect heavy loop:

```ts
export function computeScoreTable(size: number) {
  let acc = 0;
  for (let i = 0; i < size; i += 1) {
    acc += Math.hypot(i, size - i);
  }
  return acc;
}
```

Workflow:
1. DevTools → Performance → Enable Web Vitals overlays.
2. Start recording, trigger UI action calling `computeScoreTable(1_000_000)`.
3. Stop: видим жёлтый Long Task на Main, функция в Bottom-Up tab.
4. Fix: Web Worker offload или reduce input size + memoization.

React path: Components → Profiler → record → rank by "self duration" → inspect "why did this render". Flamegraph показывает parent chain, не только leaf.

## Подводные камни

- **Profiling overhead.** Instrumentation сама добавляет noise; короткие frames сравнивай относительно.
- **Anonymous functions.** Minified names бесполезны; включай source maps или name inference.
- **Idle misread.** GPU raster иногда в другом track; смотри Rendering + GPU вместе.
- **Single sample.** Один trace не статистика; повтори 3 раза warm + cold.
- **Fix wrong layer.** Оптимизация memo когда bottleneck network waterfall.
