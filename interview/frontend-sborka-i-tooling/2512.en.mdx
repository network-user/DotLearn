# npm: Node package manager

## In short

npm (Node Package Manager) is the CLI plus registry for publishing and installing JS packages. `npm install` reads `package.json`, resolves semver ranges, writes `node_modules`, and updates the lockfile. Default registry is `registry.npmjs.org`; scoped packages `@org/name` cover private/public monorepo artifacts.

## How to answer in an interview

- `npm init` / `npm create vite@latest` scaffolds a project.
- `npm install -D vitest` is a devDependency; without `-D` it lands in production deps.
- `npx` / `npm exec` runs a binary once without a global install.
- Honestly: monorepos often use pnpm/yarn for disk dedup and strict deps; npm workspaces caught up but hoisting differs.
- `npm publish --access public` for scoped packages; private via org billing or Verdaccio.

## How it works

```bash
npm init -y
npm install zod
npm install -D typescript vitest
```

```ts
import { z } from 'zod';

const Email = z.string().email();

export function parseSubscriber(input: unknown) {
  return Email.parse(input);
}
```

`package.json` gets:

```json
{
  "dependencies": { "zod": "^3.23.8" },
  "devDependencies": { "typescript": "^5.6.2", "vitest": "^2.0.5" }
}
```

`node_modules/.bin/vitest` symlinks for scripts. The lock pins the transitive tree. CI: `npm ci && npm test`. Overrides/resolutions patch vulnerable transitives without forking the lib.

## Pitfalls

- **Global install drift.** `npm i -g` breaks reproducibility; prefer project-local bins.
- **Phantom deps.** Flat hoisting lets you import packages not in package.json (breaks under pnpm strict).
- **Left-pad lesson.** Unpublish policy; pin critical deps or vendor tarballs.
- **Script lifecycle.** `postinstall` supply-chain risk; review dep scripts in enterprise.
- **Registry mirror lag.** Internal proxies lag npm and miss latest security patches.
