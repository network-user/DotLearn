# ESM и Commander.js: разные задачи

## Суть

ESM (ECMAScript Modules) это стандарт модульной системы JS: статические `import`/`export`, tree shaking, native support в браузере и Node. Commander.js это npm библиотека для CLI: парсит argv, описывает flags/subcommands, печатает help. Они решают ортогональные проблемы: организация кода vs интерфейс командной строки.

## Что ответить на собесе

- ESM файлы: `"type": "module"` в package.json или расширение `.mjs`.
- Commander строит `program.option('-p, --port <n>')` и `program.parse()`.
- ESM import Commander: `import { Command } from 'commander'`, не `require`.
- Честно: для лёгкого CLI часто хватает `node --experimental-strip-types` + `util.parseArgs`; Commander окупается на nested subcommands.
- Bundlers и test runners должны понимать ESM resolution (`exports` field).

## Как это работает

ESM utilities module:

```ts
export function sumValues(values: number[]) {
  return values.reduce((acc, n) => acc + n, 0);
}

export const DEFAULT_PORT = 4173;
```

CLI entry with Commander (ESM):

```ts
#!/usr/bin/env node
import { Command } from 'commander';
import { sumValues, DEFAULT_PORT } from './math-utils.js';

const program = new Command();

program
  .name('stats-cli')
  .description('Small demo CLI')
  .option('-p, --port <number>', 'preview port', String(DEFAULT_PORT))
  .argument('<nums...>', 'numbers to sum')
  .action((nums: string[], opts: { port: string }) => {
    const total = sumValues(nums.map(Number));
    console.log(`sum=${total} port=${opts.port}`);
  });

program.parse();
```

`import './math-utils.js'` требует explicit extension в Node ESM. Commander генерирует `--help`, валидирует arity, поддерживает `.command('deploy')` nesting. ESM даёт static graph для bundler; Commander остаётся runtime dependency только в node tooling, не в browser bundle.

## Подводные камни

- **CJS interop.** `require()` ESM из legacy script падает; мигрируй entry на import.
- **Shebang + type module.** Забытый `.js` extension в relative import ломает запуск.
- **Dual package hazard.** Lib с и CJS и ESM entry может резолвиться дважды (duplicate state).
- **Commander version API.** v12 vs v11 отличается `program.opts()` typing.
- **Confusing interview question.** ESM не «альтернатива» Commander; сравни с CommonJS, не с CLI libs.
