# Какой тип SSH-ключей выбрать сегодня?

## Суть

Для новых ключей выбираю Ed25519: короткий pubkey, быстрая генерация, сильная криптография при 256-bit curve. RSA 4096 оставляю только для legacy систем без ed25519 support. DSA и RSA 1024 считаю устаревшими и отключаю в sshd.

## Что ответить на собесе

- `ssh-keygen -t ed25519 -a 100` с KDF rounds для passphrase.
- RSA fallback: `-t rsa -b 4096` только если appliance требует.
- ECDSA nistp256 допустим, но Ed25519 проще и без curve preference drama.
- На сервере `PubkeyAcceptedAlgorithms` whitelist современных algos.
- Честно: «квантовая устойчивость» Ed25519 marketing oversell; просто современный best practice.

## Как это работает

Generate and audit:

```bash
ssh-keygen -t ed25519 -C "ops@corp" -f ~/.ssh/id_ed25519
ssh-keygen -lf ~/.ssh/id_ed25519.pub

ssh -o PubkeyAuthentication=yes -i ~/.ssh/id_ed25519 admin@host
```

Server hardening snippet:

```bash
grep PubkeyAcceptedAlgorithms /etc/ssh/sshd_config
sshd -T | grep pubkeyacceptedalgorithms
```

## Подводные камни

- **Legacy switch only rsa.** Ed25519 rejected; keep one RSA key for old gear only.
- **Too many key types offered.** ssh client tries all keys; server may rate-limit or lock account.
- **No passphrase on laptop key.** Theft = full access; use agent + touch ID or hardware token.
- **Same key everywhere.** Compromise one host leaks all; per-tier deploy keys.
- **Comment PII in pubkey.** Email in `-C` visible in authorized_keys audit export.
