# What levels make up the DNS hierarchy?

## In short

DNS is a distributed tree: root (`.`), TLD (`.com`, `.ru`), second-level domain (`example.com`), subdomains (`api.example.com`). A resolver walks root hints, TLD NS, then authoritative NS recursively. Each zone file or DNS provider holds records for its branch.

## How to answer in an interview

- Root servers (13 logical groups) delegate TLD.
- Registrar creates NS glue for the domain at registry.
- Authoritative NS answers SOA, A/AAAA, CNAME, MX, TXT.
- Recursive resolver (1.1.1.1, corp DNS) caches TTL.
- Honestly: "levels" sometimes confused with zone cuts; delegation NS records matter.

## How it works

Trace resolution:

```bash
dig +trace www.example.com A
dig NS com. +short
dig SOA example.com
```

Check delegation:

```bash
dig NS example.com @a.gtld-servers.net +short
dig A www.example.com @ns1.example.com
```

Typical path: stub resolver, recursive, root, TLD, authoritative, answer cached per TTL.

## Pitfalls

- **Stale NS at registrar.** Changed hosting but old NS at registry; partial outage.
- **CNAME at apex.** RFC conflict with SOA at zone apex; use ALIAS/ANAME at provider.
- **Long TTL during migration.** Lower TTL days before cutover.
- **Split-horizon mismatch.** Internal resolver other answer than public; debug wrong side.
- **DNSSEC not propagated.** DS record missing at parent; validators fail chain.
