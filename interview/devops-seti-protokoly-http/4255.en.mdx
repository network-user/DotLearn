# What SSH key formats appear in practice?

## In short

OpenSSH native format (PEM-like text with `BEGIN OPENSSH PRIVATE KEY`) is standard on Linux and macOS. Legacy PEM (PKCS#1/PKCS#8) is needed for OpenSSL and some SDKs. PPK v3 is used by PuTTY on Windows. Public keys appear as `ssh-ed25519 AAAA...` one-liners or `.pub` files.

## How to answer in an interview

- Generate: `ssh-keygen -t ed25519 -f ~/.ssh/deploy_key`.
- Convert to PEM: `ssh-keygen -p -m PEM -f ~/.ssh/id_rsa`.
- PPK: `puttygen id_ed25519 -o key.ppk -O private` or import in PuTTYgen GUI.
- I put only `.pub` on the server in `authorized_keys`; private never leaves CI secret store.
- Honestly: I prefer ed25519 over RSA 4096 for new keys; RSA still needed for legacy appliances.

## How it works

Ed25519 deploy key for CI:

```bash
ssh-keygen -t ed25519 -C "ci-deploy-prod" -f ~/.ssh/ci_prod -N ""

ssh-keygen -lf ~/.ssh/ci_prod.pub
cat ~/.ssh/ci_prod.pub >> ~/.ssh/authorized_keys

chmod 700 ~/.ssh
chmod 600 ~/.ssh/ci_prod
chmod 644 ~/.ssh/ci_prod.pub
```

Conversion for OpenSSL consumer:

```bash
ssh-keygen -p -m PEM -f ~/.ssh/legacy_rsa
openssl rsa -in ~/.ssh/legacy_rsa -pubout -out legacy_rsa_pub.pem
```

## Pitfalls

- **Permissions 644 on private key.** ssh refuses or warns; must be 600.
- **Wrong format in CI.** Jenkins expects PEM, key in OpenSSH format; convert or update plugin.
- **PPK version mismatch.** Old PuTTY vs PPK v3; upgrade client.
- **Comment line confusion.** authorized_keys options (`from=`, `command=`) syntax errors lock user out.
- **Shared deploy key.** One key on all hosts; compromise equals full fleet; per-env keys are better.
