# Что такое стек TCP/IP?

## Суть

TCP/IP это практический стек протоколов интернета: IP (L3) адресует и маршрутизирует datagrams, TCP (L4) даёт reliable byte stream с ports, UDP alternative без guarantees. Application protocols (HTTP, DNS, SSH) sit above. Модель из 4 layers (link, internet, transport, application) проще academic OSI.

## Что ответить на собесе

- IP best-effort; TCP adds ACK, reorder, congestion control.
- Socket = IP + port + protocol (TCP/UDP).
- IPv4/IPv6 at network layer; dual-stack common.
- ICMP helper for diagnostics (ping, unreachable).
- Честно: «TCP/IP» colloquially means whole internet stack including TLS and HTTP, not only two RFCs.

## Как это работает

Trace path:

```bash
ip route get 1.1.1.1
ss -tn state established | head
curl -4 -v https://example.com 2>&1 | grep -E 'Connected|IPv'
```

Minimal TCP listener test:

```bash
nc -l 127.0.0.1 9876 &
echo probe | nc 127.0.0.1 9876
```

Encapsulation: Ethernet frame -> IP packet -> TCP segment -> HTTP message.

## Подводные камни

- **Confuse TCP with IP.** Routing problem vs port firewalled need different tools.
- **UDP «unreliable» misuse.** VoIP/QUIC/DNS tuned for UDP; not «bad protocol».
- **NAT breaks end-to-end.** IP design assumption violated; need STUN/TURN for P2P.
- **Ignore MTU.** TCP/IP works until fragmentation or PMTUD black hole.
- **Memorize layers without tools.** `traceroute`, `ss`, `tcpdump` map theory to prod.
