# What might complement or replace SSH?

## In short

SSH remains the standard for shell access, but Zero Trust (BeyondCorp, Teleport, Cloudflare Access) replaces flat VPN plus SSH with identity-aware sessions and audit. WireGuard builds an L3 mesh instead of jump hosts. For automation, SSM Session Manager, kubectl exec, or CI deploy keys beat interactive shell.

## How to answer in an interview

- ZTNA: every session through IdP MFA, short-lived cert, no static SSH key on laptop.
- WireGuard: encrypted overlay, ssh inside WG or replace bastion entirely.
- AWS SSM / GCP OS Login: no open port 22, IAM-bound access.
- Kubernetes: `kubectl exec`, GitOps instead of ssh to node for app deploy.
- Honestly: "replace SSH" in enterprise is rarely total; mostly shrink blast radius and add audit.

## How it works

WireGuard point-to-site sketch:

```bash
sudo apt install wireguard
wg genkey | tee /etc/wireguard/server.key | wg pubkey > /etc/wireguard/server.pub
chmod 600 /etc/wireguard/server.key

cat <<EOF | sudo tee /etc/wireguard/wg0.conf
[Interface]
Address = 10.8.0.1/24
ListenPort = 51820
PrivateKey = $(sudo cat /etc/wireguard/server.key)

[Peer]
PublicKey = CLIENT_PUBLIC_KEY
AllowedIPs = 10.8.0.2/32
EOF

sudo systemctl enable --now wg-quick@wg0
```

Then ssh over `10.8.0.0/24` without public port 22.

## Pitfalls

- **ZTNA vendor lock-in.** Egress policy tied to one broker; plan break-glass SSH.
- **WG key rotation.** Static keys on laptops leak; automate expiry.
- **SSM agent missing.** Instance unreachable if agent down; monitor agent health.
- **kubectl exec as root.** Same risk as SSH; RBAC and audit logs mandatory.
- **Hype replaces basics.** Fancy mesh without patching sshd config leaves old path open.
