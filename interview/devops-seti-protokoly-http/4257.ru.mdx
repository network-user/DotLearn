# Что описывает модель OSI?

## Суть

OSI (Open Systems Interconnection) это семиуровневая reference model сетевого стека: от физической передачи бит до прикладных протоколов. Модель не протокол, а карта для диагностики: на каком уровне проблема (кабель, MAC, IP, TCP, TLS, HTTP).

## Что ответить на собесе

- L1 Physical: сигнал, кабель, NIC link up.
- L2 Data Link: MAC, Ethernet, switch, ARP в LAN.
- L3 Network: IP, routing, ICMP ping.
- L4 Transport: TCP/UDP, ports.
- L5 Session, L6 Presentation, L7 Application: часто сворачивают в «application layer» (HTTP, DNS, TLS).
- Честно: TCP/IP stack практичнее OSI, но OSI помогает объяснить layered troubleshooting.

## Как это работает

Email request path (упрощённо):

```
Browser (L7 HTTP) -> TLS (L6) -> TCP (L4) -> IP (L3) -> Ethernet (L2) -> fiber (L1)
```

Layered debug commands:

```bash
ip link show eth0
ip route get 8.8.8.8
ss -tn state established
curl -vI https://example.com 2>&1 | grep -E 'Connected|SSL|HTTP'
```

## Подводные камни

- **Strict layer purity.** Real stacks blur L5-L7; не спорю на собесе, объясняю mapping.
- **Ping is not L7.** Ping ICMP L3; app down при успешном ping.
- **Switch vs router confusion.** L2 vs L3 device in path changes debug tools.
- **Overlay networks.** VXLAN breaks naive L2 assumptions across DC.
- **Memorize without tools.** OSI без связи с tcpdump/ss бесполезна на практике.
