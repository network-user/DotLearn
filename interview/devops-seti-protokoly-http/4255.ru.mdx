# Какие форматы SSH-ключей встречаются на практике?

## Суть

OpenSSH native format (PEM-like text с `BEGIN OPENSSH PRIVATE KEY`) стандарт для Linux/macOS. Legacy PEM (PKCS#1/PKCS#8) нужен для OpenSSL и некоторых SDK. PPK v3 использует PuTTY на Windows. Публичные ключи: `ssh-ed25519 AAAA...` one-liner или `.pub` файл.

## Что ответить на собесе

- Генерация: `ssh-keygen -t ed25519 -f ~/.ssh/deploy_key`.
- Конвертация в PEM: `ssh-keygen -p -m PEM -f ~/.ssh/id_rsa`.
- PPK: `puttygen id_ed25519 -o key.ppk -O private` или import в PuTTYgen GUI.
- На сервер кладу только `.pub` в `authorized_keys`; private never leaves CI secret store.
- Честно: ed25519 предпочитаю RSA 4096 для новых ключей; RSA ещё нужен для legacy appliances.

## Как это работает

Ed25519 deploy key для CI:

```bash
ssh-keygen -t ed25519 -C "ci-deploy-prod" -f ~/.ssh/ci_prod -N ""

ssh-keygen -lf ~/.ssh/ci_prod.pub
cat ~/.ssh/ci_prod.pub >> ~/.ssh/authorized_keys

chmod 700 ~/.ssh
chmod 600 ~/.ssh/ci_prod
chmod 644 ~/.ssh/ci_prod.pub
```

Конвертация для OpenSSL consumer:

```bash
ssh-keygen -p -m PEM -f ~/.ssh/legacy_rsa
openssl rsa -in ~/.ssh/legacy_rsa -pubout -out legacy_rsa_pub.pem
```

## Подводные камни

- **Permissions 644 on private key.** ssh refuses or warns; must be 600.
- **Wrong format in CI.** Jenkins expects PEM, key in OpenSSH format; convert or update plugin.
- **PPK version mismatch.** PuTTY old version vs PPK v3; upgrade client.
- **Comment line confusion.** authorized_keys options (`from=`, `command=`) syntax errors lock user out.
- **Shared deploy key.** One key on all hosts; compromise = full fleet; per-env keys better.
