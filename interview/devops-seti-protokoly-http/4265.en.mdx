# Which addresses does the OSI network layer use?

## In short

The network layer uses logical addresses: IPv4 (32-bit) and IPv6 (128-bit). Routers forward by destination IP and longest prefix match in the routing table. IP identifies a host interface globally or in private space; NAT maps private to public at the edge.

## How to answer in an interview

- IPv4 classes are obsolete; CIDR `/24`, `/32` route semantics.
- Private RFC1918: 10/8, 172.16/12, 192.168/16; not routable on internet without NAT.
- IPv6 GUA, link-local fe80::/10, loopback ::1.
- Tools: `ip route`, `ip -4 addr`, traceroute.
- Honestly: "logical vs physical" in interviews: IP is logical, MAC is physical L2.

## How it works

Routing inspect:

```bash
ip -4 route show table main
ip -4 addr show scope global
traceroute -n 8.8.8.8
```

Add static route:

```bash
sudo ip route add 10.20.0.0/16 via 192.168.1.254 dev eth0
```

Cross-subnet flow: host ARPs gateway MAC, IP packet dst=remote, L2 dst=gateway MAC.

## Pitfalls

- **Asymmetric routing.** Forward and return path differ; stateful firewall drops reply.
- **/32 on wrong interface.** EC2 secondary IP needs correct iface or route.
- **NAT hairpin missing.** Internal client cannot reach public VIP on same LAN.
- **IPv6 ULA leaked to DNS.** Clients prefer broken ULA over working v4.
- **Default route compete.** Two equal cost defaults; flaky egress.
