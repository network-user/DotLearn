# Как на Linux посмотреть IP-адрес сервера?

## Суть

Современный способ: `ip addr show` (iproute2) для всех интерфейсов, IPv4 и IPv6. Для конкретного iface `ip -4 addr show eth0`. Legacy `ifconfig` и `hostname -I` ещё встречаются в скриптах, но iproute2 предпочтителен в minimal images без net-tools.

## Что ответить на собесе

- `ip addr` или сокращённо `ip a` полный список.
- `ip -4 route get 1.1.1.1` показывает source IP для egress.
- Cloud metadata: `curl -s http://169.254.169.254/latest/meta-data/local-ipv4` (AWS).
- Container: `hostname -i` может вернуть только pod IP в k8s.
- Честно: несколько интерфейсов (bond, vlan) требуют фильтра scope global.

## Как это работает

Inspect addresses:

```bash
ip -br addr show
ip -4 addr show dev eth0 scope global
ip -6 addr show dev eth0 scope global
```

Default egress source:

```bash
ip route get 8.8.8.8 | awk '{for(i=1;i<=NF;i++) if($i=="src") print $(i+1)}'
```

Legacy fallback if net-tools installed:

```bash
ifconfig eth0 2>/dev/null | awk '/inet /{print $2}'
```

## Подводные камни

- **Loopback only.** Service binds 127.0.0.1; `ip addr` shows eth0 but app unreachable remotely.
- **Secondary IPs missed.** grep wrong field; use `ip -j` for scripts.
- **Temporary DHCP.** Address changes; automation must not cache in config file.
- **namespace hidden.** `ip addr` in host ns misses veth inside container; `ip netns exec`.
- **hostname -I multiple.** Space-separated list; first not always default route source.
