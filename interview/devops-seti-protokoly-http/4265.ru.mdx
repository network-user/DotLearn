# Какие адреса использует сетевой уровень OSI?

## Суть

Network layer использует логические адреса: IPv4 (32-bit) и IPv6 (128-bit). Router forward по destination IP и longest prefix match в routing table. IP идентифицирует host interface в глобальной или private сети; NAT maps private→public на границе.

## Что ответить на собесе

- IPv4 classes устарели; CIDR `/24`, `/32` route semantics.
- Private RFC1918: 10/8, 172.16/12, 192.168/16; не routable в internet без NAT.
- IPv6 GUA, link-local fe80::/10, loopback ::1.
- Tools: `ip route`, `ip -4 addr`, traceroute.
- Честно: «logical vs physical» на собесе: IP logical, MAC physical L2.

## Как это работает

Routing inspect:

```bash
ip -4 route show table main
ip -4 addr show scope global
traceroute -n 8.8.8.8
```

Add static route:

```bash
sudo ip route add 10.20.0.0/16 via 192.168.1.254 dev eth0
```

Cross-subnet flow: host ARP gateway MAC, IP packet dst=remote, L2 dst=gateway MAC.

## Подводные камни

- **Asymmetric routing.** Forward and return path differ; stateful firewall drops reply.
- **/32 on wrong interface.** EC2 secondary IP needs correct iface or route.
- **NAT hairpin missing.** Internal client cannot reach public VIP on same LAN.
- **IPv6 ULA leaked to DNS.** Clients prefer broken ULA over working v4.
- **Default route compete.** Two equal cost default; flaky egress.
