# Из каких уровней состоит иерархия DNS?

## Суть

DNS это distributed tree: root (`.`), TLD (`.com`, `.ru`), second-level domain (`example.com`), subdomains (`api.example.com`). Resolver рекурсивно обходит цепочку root hints → TLD NS → authoritative NS. Каждый zone file или DNS provider хранит records для своей ветки.

## Что ответить на собесе

- Root servers (13 logical groups) делегируют TLD.
- Registrar создаёт NS glue для domain at registry.
- Authoritative NS отвечает SOA, A/AAAA, CNAME, MX, TXT.
- Recursive resolver (1.1.1.1, corp DNS) кеширует TTL.
- Честно: «levels» иногда путают с zone cuts; важно delegation NS records.

## Как это работает

Trace resolution:

```bash
dig +trace www.example.com A
dig NS com. +short
dig SOA example.com
```

Check delegation:

```bash
dig NS example.com @a.gtld-servers.net +short
dig A www.example.com @ns1.example.com
```

Typical path: stub resolver → recursive → root → TLD → authoritative → answer cached per TTL.

## Подводные камни

- **Stale NS at registrar.** Changed hosting but old NS at registry; partial outage.
- **CNAME at apex.** RFC conflict with SOA at zone apex; use ALIAS/ANAME at provider.
- **Long TTL during migration.** Lower TTL days before cutover.
- **Split-horizon mismatch.** Internal resolver other answer than public; debug wrong side.
- **DNSSEC not propagated.** DS record missing at parent; validators fail chain.
