# Чем IPv6 отличается от IPv4?

## Суть

IPv4 использует 32-bit адреса (~4.3 млрд), IPv6 128-bit (практически неисчерпаемый pool). IPv6 убирает NAT как необходимость, добавляет SLAAC, встроенный IPsec support, упрощённый header и обязательность dual-stack или translation на переходный период.

## Что ответить на собесе

- IPv4: dotted decimal `192.0.2.10`; IPv6: hex groups `2001:db8::1`.
- NAT44 продлевает жизнь IPv4; IPv6 end-to-end addressing снова возможен.
- SLAAC + DHCPv6 для адресов; NDP вместо ARP.
- Dual-stack: A и AAAA DNS records; app must handle both.
- Честно: «IPv6-only prod» редкость; чаще dual-stack с broken v6 path detection issues.

## Как это работает

Assign IPv6 on Linux:

```bash
sudo ip -6 addr add 2001:db8:acme::1/64 dev eth0
sudo ip link set eth0 up
ip -6 addr show dev eth0
ip -6 route show default
ping6 -c 2 2001:4860:4860::8888
```

Check DNS dual-stack:

```bash
dig A example.com +short
dig AAAA example.com +short
curl -6 -I https://example.com
curl -4 -I https://example.com
```

## Подводные камни

- **Broken IPv6 preference.** Client tries AAAA first, v6 path blackholed; long timeout before v4.
- **Firewall forgot v6.** ip6tables empty allow all or default drop breaks deploy.
- **ULA vs GUA.** `fd00::/8` not routable on internet; document prefix plan.
- **Literal address in config.** Brackets required in URL: `https://[2001:db8::1]:8443/`.
- **ICMPv6 filtered.** NDP broken; neighbor discovery fails silently.
