# How do you check the server IP address on Linux?

## In short

Modern approach: `ip addr show` (iproute2) for all interfaces, IPv4 and IPv6. For one iface use `ip -4 addr show eth0`. Legacy `ifconfig` and `hostname -I` still appear in scripts, but iproute2 is preferred on minimal images without net-tools.

## How to answer in an interview

- `ip addr` or short `ip a` for full list.
- `ip -4 route get 1.1.1.1` shows source IP for egress.
- Cloud metadata: `curl -s http://169.254.169.254/latest/meta-data/local-ipv4` (AWS).
- Container: `hostname -i` may return only pod IP in k8s.
- Honestly: multiple interfaces (bond, vlan) need scope global filter.

## How it works

Inspect addresses:

```bash
ip -br addr show
ip -4 addr show dev eth0 scope global
ip -6 addr show dev eth0 scope global
```

Default egress source:

```bash
ip route get 8.8.8.8 | awk '{for(i=1;i<=NF;i++) if($i=="src") print $(i+1)}'
```

Legacy fallback if net-tools installed:

```bash
ifconfig eth0 2>/dev/null | awk '/inet /{print $2}'
```

## Pitfalls

- **Loopback only.** Service binds 127.0.0.1; `ip addr` shows eth0 but app unreachable remotely.
- **Secondary IPs missed.** Wrong grep field; use `ip -j` for scripts.
- **Temporary DHCP.** Address changes; automation must not cache in config file.
- **Namespace hidden.** `ip addr` in host ns misses veth inside container; `ip netns exec`.
- **hostname -I multiple.** Space-separated list; first not always default route source.
