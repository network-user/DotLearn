# Оптимизация запросов динамического списка

## Суть

Динамический список выполняет основной запрос при открытии, прокрутке и автообновлении. Оптимизация: короткий список полей, жёсткий отбор по индексируемым параметрам, отказ от подзапросов в SELECT, вынос тяжёлых join в отчёт.

## Что ответить на собесе

- Я оставляю в списке только поля колонок, без лишних join.
- Параметры `&Период`, `&Организация` задаю при открытии формы.
- Вычисляемые поля с запросами в каждой строке убираю.
- Для поиска использую `СтрокаПоиска` по индексированным реквизитам.
- Честно: красивый список «как отчёт» на 50 join неприемлем для ежедневной работы.

## Как это работает

Упрощённый текст списка:

```bsl
"ВЫБРАТЬ
|    Док.Ссылка,
|    Док.Дата,
|    Док.Номер,
|    Док.Контрагент
|ИЗ
|    Документ.РеализацияТоваров КАК Док
|ГДЕ
|    Док.Дата >= &НачалоПериода
|    И Док.Организация = &Организация"
```

Детали подтягиваю в карточке документа отдельным лёгким запросом.

## Подводные камни

**Подзапрос в поле списка.** N+1 на каждую видимую строку.
**Отсутствие отбора по дате.** Полный скан документов.
**Сортировка по неиндексируемому выражению.** Filesort на больших таблицах.
**RLS без учёта в тесте.** Медленно только у ограниченных ролей.
**Пользовательские отборы без параметров.** Конкатенация ломает кэш планов.
