# Почему запросы в цикле недопустимы

## Суть

Каждый `Запрос.Выполнить()` в цикле это отдельный round-trip к СУБД и разбор плана. На N строк получаем N обращений вместо одного пакетного запроса или одного join.

## Что ответить на собесе

- Я заменяю цикл на `ГДЕ Ссылка В (&Массив)` или временную таблицу ключей.
- Для разных типов собираю соответствие и один запрос с UNION.
- Если нужен расчёт по строке, сначала выгружаю данные одним запросом в таблицу значений.
- Измеряю в ТЖ: сотни SDBL на один проведённый документ это сигнал.
- Честно: редкий цикл на 3 строки терпим; на тысячах строк валит сеанс.

## Как это работает

Плохо:

```bsl
Для Каждого Строка Из Таблица Цикл
    Запрос.УстановитьПараметр("Ссылка", Строка.Ссылка);
    // Выполнить() внутри цикла
КонецЦикла;
```

Хорошо:

```bsl
МассивСсылок = Таблица.ВыгрузитьКолонку("Ссылка");
Запрос.УстановитьПараметр("МассивСсылок", МассивСсылок);
" ... ГДЕ Т.Ссылка В (&МассивСсылок)"
```

## Подводные камни

**Скрытый запрос в функции.** Цикл вызывает функцию с запросом внутри.
**ПолучитьОбъект в цикле.** Ещё тяжелее чем лёгкий запрос.
**Слишком большой В().** Дроблю на пакеты по 1000 id.
**Параллельные циклы в фоне.** Умножают нагрузку на СУБД.
**Оптимизация без профиля.** Убрали цикл, но оставили cartesian join.
