# Why queries inside loops are unacceptable

## In short

Each `Запрос.Выполнить()` in a loop is a separate DBMS round-trip and plan handling. For N rows you get N hits instead of one batch query or join.

## How to answer in an interview

- I replace the loop with `ГДЕ Ссылка В (&Массив)` or a temp table of keys.
- For mixed types I build a map and one query with UNION.
- If per-row calc is needed I first load data in one query to a value table.
- I measure in tech log: hundreds of SDBL per posted document is a signal.
- I am honest: a rare loop over 3 rows is fine; over thousands it kills the session.

## How it works

Bad:

```bsl
Для Каждого Строка Из Таблица Цикл
    Запрос.УстановитьПараметр("Ссылка", Строка.Ссылка);
    // Выполнить() inside loop
КонецЦикла;
```

Good:

```bsl
МассивСсылок = Таблица.ВыгрузитьКолонку("Ссылка");
Запрос.УстановитьПараметр("МассивСсылок", МассивСсылок);
" ... ГДЕ Т.Ссылка В (&МассивСсылок)"
```

## Pitfalls

**Hidden query in function.** Loop calls function with query inside.
**ПолучитьОбъект in loop.** Heavier than a light query.
**Oversized В().** Split into batches of 1000 ids.
**Parallel loops in background.** Multiply DBMS load.
**Optimization without profiling.** Removed loop but left cartesian join.
