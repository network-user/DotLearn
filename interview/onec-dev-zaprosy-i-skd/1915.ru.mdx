# Диагностика ошибок выполнения запросов в 1С

## Суть

Ошибки запросов ловят в `Попытка/Исключение` при `Выполнить()`, смотрят текст с указанием строки запроса, параметры и ТЖ (`SDBL`, `EXCP`). Для отладки выводят сгенерированный SQL через консоль запросов.

## Что ответить на собесе

- Я воспроизвожу запрос в консоли с теми же параметрами.
- В лог пишу номер строки из `ОписаниеОшибки()` и значения параметров без ПДн.
- Различаю ошибки компиляции текста и ошибки СУБД (таймаут, deadlock).
- Для динамического списка снимаю текст из свойства `ТекстЗапроса`.
- Честно: `Сообщить(ТекстЗапроса)` в проде пользователям не показываю.

## Как это работает

```bsl
Попытка
    Результат = Запрос.Выполнить();
Исключение
    Инфо = ИнформацияОбОшибке();
    ЗаписьЖурналаРегистрации("Запрос", УровеньЖурналаРегистрации.Ошибка, , ,
        ПодробноеПредставлениеОшибки(Инфо) + Символы.ПС + Запрос.Текст);
    ВызватьИсключение;
КонецПопытки;
```

В ТЖ включаю длительные `DBMSSQL`/`DBPOSTGRS` события.

## Подводные камни

**Глотание исключения.** Пустой результат без причины.
**Параметры не логируются.** Невозможно воспроизвести.
**Путаница типов параметра.** Ошибка только на боевых данных.
**Таймаут как «ошибка данных».** Нужен тюнинг, не правка условия.
**Отладочный вывод пользователю.** Утечка структуры базы.
