# Diagnosing query execution errors in 1C

## In short

Catch query errors in `Попытка/Исключение` on `Выполнить()`, read text with line number, parameters, and tech log (`SDBL`, `EXCP`). For debug use generated SQL from the query console.

## How to answer in an interview

- I reproduce the query in the console with the same parameters.
- I log line number from `ОписаниеОшибки()` and parameter values without PII.
- I separate text compile errors from DBMS errors (timeout, deadlock).
- For dynamic lists I take text from `ТекстЗапроса`.
- I am honest: I do not `Сообщить(ТекстЗапроса)` to users in production.

## How it works

```bsl
Попытка
    Результат = Запрос.Выполнить();
Исключение
    Инфо = ИнформацияОбОшибке();
    ЗаписьЖурналаРегистрации("Запрос", УровеньЖурналаРегистрации.Ошибка, , ,
        ПодробноеПредставлениеОшибки(Инфо) + Символы.ПС + Запрос.Текст);
    ВызватьИсключение;
КонецПопытки;
```

In the tech log I enable long `DBMSSQL`/`DBPOSTGRS` events.

## Pitfalls

**Swallowed exception.** Empty result without cause.
**Parameters not logged.** Cannot reproduce.
**Parameter type confusion.** Error only on production data.
**Timeout as data error.** Needs tuning, not condition fix.
**Debug output to user.** Database structure leak.
