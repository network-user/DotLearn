# Auto-refresh in dynamic list and performance

## In short

Auto-refresh makes the dynamic list periodically reload from the server on database changes. Good for monitoring; on heavy queries it adds load and UI flicker.

## How to answer in an interview

- I disable auto-refresh on lists with heavy joins and weak indexes.
- For live queues I enable it with a longer interval or refresh on event.
- Main lever is simplifying the list query, not only the interval.
- I compare the dynamic list query plan with the report variant.
- I am honest: auto-refresh is not push; with hundreds of users load multiplies.

## How it works

In dynamic list properties: `АвтоФиксация = Истина` and refresh interval. I keep the list query filtered by period and indexed fields. On form open I set `&НачалоПериода` to narrow selection.

## Pitfalls

**Heavy query + auto-refresh.** Constant server full scans.
**Too short interval.** CPU and network scale with open forms.
**No company filter.** RLS and extra rows together.
**Hidden computed fields in list.** Recalc per row.
**Expecting instant updates.** Users miss others' changes without refresh.
