# Три вида регистров и их роль в учёте

## Суть

Регистры накопления фиксируют количественные и суммовые движения и остатки. Регистры сведений хранят произвольные срезы данных во времени (цены, курсы, настройки). Регистры расчёта ведут периодические начисления с зависимостями видов расчёта.

## Что ответить на собесе

- Я подбираю регистр под задачу: остатки товара → накопление; цена на дату → сведения; оклад → расчёт.
- Движения в накопление и расчёт обычно пишет проведение документа-регистратора.
- Регистр сведений может быть независимым или подчинённым регистратору.
- Виртуальные таблицы запроса дают срезы без прямого SQL к физическим таблицам.
- Честно: справочник не заменяет регистр, если нужна история изменений во времени.

## Как это работает

Проведение документа формирует движения по накоплению:

```bsl
Движения.ТоварыНаСкладах.Записывать = Истина;
Движение = Движения.ТоварыНаСкладах.Добавить();
Движение.ВидДвижения = ВидДвиженияНакопления.Приход;
Движение.Период = Дата;
Движение.Номенклатура = СтрокаТЧ.Номенклатура;
Движение.Склад = Склад;
Движение.Количество = СтрокаТЧ.Количество;
```

Запись цены в регистр сведений:

```bsl
Набор = РегистрыСведений.ЦеныНоменклатуры.СоздатьНаборЗаписей();
Запись = Набор.Добавить();
Запись.Период = ДатаЦены;
Запись.Номенклатура = Номенклатура;
Запись.Цена = НоваяЦена;
Набор.Записать();
```

Выбор регистра определяет модель отчётов и способ контроля остатков или периодов.

## Подводные камни

**Справочник вместо регистра.** Теряется история цен и курсов.
**Накопление для настроек.** Лишние движения и тяжёлые итоги без необходимости.
**Независимый регистр сведений без контроля.** Записи без регистратора сложнее аудировать.
**Смешение расчёта и накопления.** Неверная модель для зарплатных сценариев.
**Запросы без виртуальных таблиц.** Ручная агрегация движений медленнее платформенных срезов.
