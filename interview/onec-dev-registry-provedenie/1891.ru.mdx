# Виртуальная и временная таблица в запросе 1С

## Суть

Виртуальная таблица это встроенный срез регистра (`.Остатки`, `.СрезПоследних`). Временная таблица создаётся оператором `ПОМЕСТИТЬ` и живёт только в пакете запросов для промежуточных результатов.

## Что ответить на собесе

- Я не называю `ПОМЕСТИТЬ` виртуальной таблицей регистра.
- Виртуальную таблицу не создаю вручную; её имя фиксировано метаданными регистра.
- `ПОМЕСТИТЬ` применяю для тяжёлых join и поэтапной агрегации.
- Временные таблицы индексирую вторым запросом при необходимости.
- Честно: лишние `ПОМЕСТИТЬ` без выгоды усложняют план запроса.

## Как это работает

Виртуальная:

```bsl
"ИЗ РегистрНакопления.Остатки.Остатки(&Дата, ) КАК Ост"
```

Временная:

```bsl
"ВЫБРАТЬ ... ПОМЕСТИТЬ ВТОтбор
|ИЗ ...;
|ВЫБРАТЬ ... ИЗ ВТОтбор КАК ВТ"
```

## Подводные камни

**Синонимы в разных запросах пакета.** ВТ не видна между несвязанными пакетами без менеджера временных таблиц.
**Огромная ВТ без отбора.** Переполнение tempdb.
**Путаница терминов на собесе.** Неверная модель оптимизации.
**Дублирование среза регистра в ВТ.** Два способа получить одно с разной актуальностью.
**Забытый ИНДЕССИРОВАТЬ по ВТ.** Медленное соединение на следующем шаге.
