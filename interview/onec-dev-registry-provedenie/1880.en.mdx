# Information register record without recorder: when it is allowed

## In short

In an independent information register a row is created by dimension key and period without a recorder field. In a recorder-subordinate register every row must reference a document; standalone rows are not allowed.

## How to answer in an interview

- I first check the register write mode in metadata.
- For prices and rates I often use an independent register: I write a set without a recorder.
- For document statuses only a subordinate register with recorder filter.
- On unpost subordinate rows are removed with the document.
- I am honest: independent writes without source audit complicate incident analysis.

## How it works

Independent price write:

```bsl
Менеджер = РегистрыСведений.ЦеныНоменклатуры;
Менеджер.СоздатьМенеджерЗаписи();
Запись = Менеджер.СоздатьМенеджерЗаписи();
Запись.Период = ДатаЦены;
Запись.Номенклатура = НоменклатураСсылка;
Запись.Цена = 1500;
Запись.Записать();
```

A subordinate register is filled only from document `ОбработкаПроведения` with `Регистратор` set.

## Pitfalls

**Write without recorder in subordinate mode.** Platform error or corrupt data.
**Duplicate dimensions.** In an independent register a second slice with the same key overwrites the first.
**Periodicity confusion.** A non-periodic register does not accept date slices as expected.
**Manual edit without document.** No unpost chain.
**Mixed modes in one configuration.** Different registers behave differently with the same code.
