# Period parameters in accumulation register virtual tables

## In short

In virtual tables `.Остатки()`, `.Обороты()`, and `.ОстаткиИОбороты()` period bounds are set by date, point in time, or boundary (`Граница`, `МоментВремени`). Parameters filter movements by the register record `Период` field.

## How to answer in an interview

- I pass the end instant to `Остатки` and a start/end pair to `Обороты`.
- I use `КонецДня()`, `НачалоМесяца()` for an unambiguous bound.
- For "as of document moment" I use `МоментВремени(ДокументСсылка)`.
- An empty second parameter in `Остатки(&Дата, )` means filter only by dimensions in parentheses.
- I am honest: you cannot pass a string instead of a date; the parameter type must match.

## How it works

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |    Ост.Номенклатура,
    |    Ост.КоличествоОстаток
    |ИЗ
    |    РегистрНакопления.ТоварыНаСкладах.Остатки(&Момент, Склад = &Склад) КАК Ост";

Запрос.УстановитьПараметр("Момент", КонецДня(ДатаОтчета));
Запрос.УстановитьПараметр("Склад", СкладСсылка);
```

Turnovers for a month:

```bsl
"РегистрНакопления.Продажи.Обороты(&Начало, &Конец, , )"
```

`Граница` includes or excludes movements on the bound itself; `МоментВремени` respects posting order of documents in the same second.

## Pitfalls

**Date without time at day end.** End-of-day balance needs `КонецДня`.
**Swapped turnover parameters.** Start after end yields an empty selection.
**Ignoring point in time.** Two documents in one second give a wrong slice without `МоментВремени`.
**Stale totals.** `Остатки()` on large registers without totals rebuild is slow.
**String instead of date.** Query compile error or implicit cast issues.
