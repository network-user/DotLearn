# Когда использовать виртуальные таблицы регистров

## Суть

Виртуальные таблицы применяют для штатных срезов: остатки, обороты, срез последних. Прямое чтение таблицы движений оправдано только в специальных административных или отладочных сценариях.

## Что ответить на собесе

- Я всегда начинаю отчёт с `.Остатки` / `.Обороты` / `.СрезПоследних`.
- Прямой `РегистрНакопления.Имя` без виртуальной таблицы даёт сырые движения, не остаток.
- Временная таблица нужна после виртуальной, если дальше сложные join.
- Параметры периода в скобках обязательны для производительности.
- Честно: на микроскопическом регистре разница мала, на типовой ERP виртуальная таблица критична.

## Как это работает

Остаток на дату:

```bsl
"РегистрНакопления.Товары.Остатки(&Момент, Склад = &Склад)"
```

Цена на дату:

```bsl
"РегистрСведений.Цены.СрезПоследних(&Дата, Номенклатура = &Ном)"
```

## Подводные камни

**Сырые движения вместо остатка.** Неверные цифры в отчёте.
**Виртуальная таблица без отбора.** Полный скан.
**Двойная агрегация.** Остатки уже агрегированы, повторная сумма искажает.
**Устаревшие итоги.** Медленно или неверно на больших данных.
**Путаница с ПОМЕСТИТЬ.** Разные уровни абстракции.
