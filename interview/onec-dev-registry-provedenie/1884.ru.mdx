# Базовые виды расчёта в плане видов расчёта

## Суть

Базовый вид расчёта задаёт, от каких уже начисленных видов берётся база для процентов и формул. Связь настраивается в плане видов расчёта и определяет порядок расчёта в регистре расчёта.

## Что ответить на собесе

- Я объясняю: базовые виды это вход для зависимого начисления (например премия от оклада).
- Порядок расчёта платформа выстраивает по графу зависимостей.
- Без базового вида процентная строка получит нулевую или неверную базу.
- Базовые отличаются от ведущих и вытесняющих видов расчёта.
- Честно: циклические базовые связи ломают расчёт зарплаты до исправления плана.

## Как это работает

В плане видов расчёта для вида «Премия10» указываю базовые: «Оклад», «Надбавка». При проведении документа начисления регистр расчёта сначала получает записи базовых видов, затем производный вид читает их как базу.

```bsl
// упрощённо: результат зависит от записей базовых видов за тот же период действия
БазаПремии = ПолучитьБазу(Сотрудник, Период, МассивБазовыхВидов);
Результат = Окр(БазаПремии * 0.10, 2);
```

## Подводные камни

**Цикл в базовых видах.** Расчёт не завершается или даёт ошибку.
**Неверный период базы.** Премия считается от прошлого месяца.
**Путаница с ведущими.** Разные механизмы зависимости в платформе.
**Ручная запись без плана.** Формулы в коде расходятся с метаданными.
**Перерасчёт без обновления базы.** Старые базовые строки остаются в регистре.
