# Leading calculation kinds and accrual order

## In short

A leading calculation kind must be calculated before the dependent one in the same period. The platform uses the leading link to order calculation register rows and to recalculate when the leading accrual changes.

## How to answer in an interview

- I separate leading from base kinds: leading sets sequence, base supplies amount for formulas.
- When salary changes, dependent kinds with a leading link to salary are recalculated.
- Configuration is in the chart of calculation types on the dependencies tab.
- A wrong leading graph yields zero dependent rows in the accrual month.
- I am honest: leading does not replace displacement when kinds overlap in one period.

## How it works

If RegionalCoefficient leads NorthernAllowance, the platform writes the coefficient row first, then the allowance using the fixed leading result. When the leading row is reversed, dependents are recalculated by the recalculation routine.

## Pitfalls

**Cycle in leading links.** Impossible calculation order.
**Mixing with base kinds.** Correct order but zero base.
**Ignoring recalculation.** Dependent rows keep old amounts.
**Manual register input.** Bypasses the chart of calculation types.
**Different action periods.** Leading row in another month is not picked up.
