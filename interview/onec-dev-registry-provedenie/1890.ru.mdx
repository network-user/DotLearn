# Параметр периода в виртуальной таблице регистра сведений

## Суть

В `.СрезПоследних(&Период, )` и `.СрезПервых()` параметр периода задаёт границу актуальности записей. Допустимы дата, `КонецДня()`, `НачалоМесяца()` и граница момента времени для согласованности с документом.

## Что ответить на собесе

- Я передаю дату среза, а не произвольную строку.
- Для отчёта на конец дня использую `КонецДня(Дата)`.
- Второй параметр виртуальной таблицы задаёт отбор по измерениям.
- Непериодический регистр не имеет `.СрезПоследних()` по дате.
- Честно: срез без актуальных итогов на большом регистре медленный.

## Как это работает

```bsl
Запрос.УстановитьПараметр("Период", КонецДня(ДатаДокумента));
"РегистрСведений.СтавкиНДС.СрезПоследних(&Период, )"
```

Платформа выбирает последнюю запись с периодом не позже параметра для каждого набора измерений.

## Подводные камни

**Дата без времени.** Срез на начало дня вместо конца.
**Непериодический регистр.** Вызов среза по дате неверен.
**Пустой отбор в скобках.** Лишний объём данных.
**Игнор часового пояса сеанса.** Расхождение в ночных документах.
**Ручной MAX(Период) вместо среза.** Дублирование и ошибки в группировке.
