# Как выбрать тип регистра под задачу учёта

## Суть

Нужны остатки или обороты количества/суммы → регистр накопления. Нужен срез свойств на дату → регистр сведений. Периодические начисления с зависимостями → регистр расчёта. Бухгалтерские проводки → регистр бухгалтерии.

## Что ответить на собесе

- Я начинаю с вопроса: остаток, история значений или формула по периоду?
- Складские остатки: накопление остатков с контролем отрицательных.
- Цена номенклатуры на дату: сведения периодические независимые.
- Зарплата: расчёт с планом видов расчёта.
- Честно: универсальный «супер-регистр» с десятком измерений хуже двух специализированных.

## Как это работает

Остатки товаров:

```bsl
РегистрНакопления.ТоварыНаСкладах // вид Остатки
```

Курс валюты:

```bsl
РегистрСведений.КурсыВалют // периодический
```

Начисление оклада:

```bsl
РегистрРасчета.Начисления
```

Выбор влияет на проведение документов, отчёты и обмен.

## Подводные камни

**Сведения для остатков.** Нет прихода/расхода и контроля количества.
**Накопление для курсов валют.** Избыточные движения.
**Расчёт для складских остатков.** Неверная модель движений.
**Независимый регистр без регистратора для статусов документа.** Потеря связи с основанием.
**Слишком много измерений.** Медленные запросы и тяжёлые блокировки.
