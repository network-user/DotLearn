# How to pick a register type for an accounting task

## In short

Need quantity/amount balances or turnovers → accumulation register. Need property slices as of date → information register. Periodic accruals with dependencies → calculation register. Accounting entries → accounting register.

## How to answer in an interview

- I start with: balance, value history, or period formula?
- Warehouse stock: balance accumulation with negative control.
- Item price as of date: periodic independent information register.
- Payroll: calculation with chart of calculation types.
- I am honest: one universal super-register with many dimensions is worse than two focused ones.

## How it works

Stock balances:

```bsl
РегистрНакопления.ТоварыНаСкладах // Balance kind
```

Exchange rate:

```bsl
РегистрСведений.КурсыВалют // periodic
```

Salary accrual:

```bsl
РегистрРасчета.Начисления
```

Choice affects document posting, reports, and exchange.

## Pitfalls

**Information register for balances.** No receipt/issue and quantity control.
**Accumulation for exchange rates.** Redundant movements.
**Calculation register for warehouse stock.** Wrong movement model.
**Independent register without recorder for document status.** Lost link to source.
**Too many dimensions.** Slow queries and heavy locks.
