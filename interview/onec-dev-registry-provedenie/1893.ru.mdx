# Цепочка проведения: движения, итоги и блокировки

## Суть

Проведение документа в транзакции формирует движения по регистрам, обновляет итоги регистров остатков и использует управляемые блокировки по измерениям, чтобы параллельные сеансы не испортили остатки.

## Что ответить на собесе

- Я описываю порядок: `ПередЗаписью` → запись документа → `ОбработкаПроведения` → движения → итоги.
- Блокировки данных ставлю до чтения остатка, который буду менять.
- Контроль остатков опирается на актуальные итоги и режим блокировки.
- Отмена проведения удаляет движения и откатывает итоги в той же логике.
- Честно: длинная транзакция проведения увеличивает взаимные блокировки при массовом импорте.

## Как это работает

```bsl
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    Блокировка = Новый БлокировкаДанных;
    Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
    Элемент.Режим = РежимБлокировкиДанных.Исключительный;
    Элемент.УстановитьЗначение("Склад", Склад);
    Блокировка.Заблокировать();

    Движения.ТоварыНаСкладах.Записывать = Истина;
    // формирование прихода/расхода
КонецПроцедуры
```

Платформа записывает движения и пересчитывает затронутые итоги в рамках транзакции проведения.

## Подводные камни

**Чтение остатка без блокировки.** Два расхода проходят при одном остатке.
**Проведение без движений.** Учёт не меняется, пользователь думает что проведено.
**Игнор отказа в ОбработкаПроведения.** Документ помечен проведённым с ошибкой логики.
**Массовое проведение в один момент.** Очередь блокировок на популярных складах.
**Ручная правка итогов.** Расхождение с движениями до следующего полного пересчёта.
