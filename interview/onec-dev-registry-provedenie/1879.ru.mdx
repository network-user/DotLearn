# Подчинение регистра сведений регистратору: проверка в коде

## Суть

У регистра сведений в метаданных свойство «Подчинение регистратору» задаёт, обязательна ли связь записи с документом. Подчинённый регистр пишется только в составе проведения; независимый допускает запись без регистратора с полным набором измерений.

## Что ответить на собесе

- Я проверяю `Метаданные.РегистрыСведений.<Имя>.РежимЗаписи` и признак подчинения регистратору.
- Для подчинённого набора записей отбор по `Регистратор` обязателен при чтении и записи.
- В независимом регистре ключ это измерения плюс период (для периодического).
- Запись без регистратора в подчинённом регистре приведёт к ошибке или «висящим» данным.
- Честно: подчинение нельзя сменить в рантайме; только в конфигураторе с миграцией данных.

## Как это работает

```bsl
МетаРегистра = Метаданные.РегистрыСведений.СтатусыЗаказов;
ПодчиненРегистратору = (МетаРегистра.РежимЗаписи = Метаданные.СвойстваОбъектов.РежимЗаписиРегистраСведений.ПодчинениеРегистратору);

Если ПодчиненРегистратору Тогда
    Набор = РегистрыСведений.СтатусыЗаказов.СоздатьНаборЗаписей();
    Набор.Отбор.Регистратор.Установить(ДокументСсылка);
    Набор.Прочитать();
КонецЕсли;
```

При проведении документ формирует движения в подчинённый регистр; при отмене проведения записи регистратора удаляются автоматически.

## Подводные камни

**Запись независимого набора в подчинённый регистр.** Отсутствует регистратор, отчёты не находят источник.
**Чтение без отбора по регистратору.** Подтягиваются чужие записи других документов.
**Смена режима в проде.** Требует выгрузки и преобразования исторических данных.
**Периодический подчинённый.** Путаница периода записи и даты документа.
**Прямой SQL.** Обход механизма регистратора ломает отмену проведения.
