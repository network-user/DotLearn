# How to change register movements correctly

## In short

The standard path: change the recorder document and repost. The platform rewrites the movement set filtered by recorder. Direct edit of the movements table is only for service scenarios and breaks the link to the accounting document.

## How to answer in an interview

- I fix data via unpost, document edit, and post again.
- In code: `ПолучитьОбъект()`, change, `Записать(РежимЗаписиДокумента.Проведение)`.
- I read a register set with `Регистратор` filter, change, and `Записать()` only when there is no document.
- After movement changes I verify balance register totals.
- I am honest: manual SQL or processor edits "around the document" create mismatch with the recorder.

## How it works

Reposting an existing document:

```bsl
ДокументОбъект = ДокументСсылка.ПолучитьОбъект();
ДокументОбъект.Комментарий = "Корректировка количества";
ДокументОбъект.Товары[0].Количество = 15;
ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
```

The platform removes old recorder movements and writes new ones from `Движения` or `ОбработкаПроведения`. For an independent information register the record set is fully overwritten by key.

Unposting clears recorder movements without deleting the document.

## Pitfalls

**Editing movements without a document.** Recorder-based reports disagree with movements.
**Posting without rewriting the set.** Duplicate movements when rewriting a set without filter.
**Ignoring locks.** Parallel posting of two documents for one warehouse races.
**Skipped unpost.** Editing a posted document is blocked by settings.
**Totals not rebuilt.** Balances in reports are wrong after direct table edit.
