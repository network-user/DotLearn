# Блокировки при проведении документов в 1С

## Суть

Управляемые блокировки данных (`БлокировкаДанных`) ставятся перед чтением и изменением остатков в регистрах. Исключительная блокировка по измерениям предотвращает гонку при параллельном расходе со склада.

## Что ответить на собесе

- Я блокирую измерения регистра, которые меняю, до запроса остатка.
- Использую `РежимБлокировкиДанных.Исключительный` для списания.
- Блокировка живёт до конца транзакции проведения.
- Автоматические блокировки платформы не отменяют необходимость явных на горячих регистрах.
- Честно: слишком широкая блокировка (весь склад) сериализует всех пользователей.

## Как это работает

```bsl
Блокировка = Новый БлокировкаДанных;
Элемент = Блокировка.Добавить("РегистрНакопления.ТоварыНаСкладах");
Элемент.Режим = РежимБлокировкиДанных.Исключительный;
Элемент.УстановитьЗначение("Склад", Склад);
Элемент.УстановитьЗначение("Номенклатура", Номенклатура);
Блокировка.Заблокировать();
```

После блокировки читаю остаток и формирую расход.

## Подводные камни

**Нет блокировки.** Отрицательный остаток при параллельной продаже.
**Блокировка всего регистра.** Дедлоки и очереди.
**Блокировка после чтения.** Классическая гонка read-then-write.
**Длинная транзакция.** Удержание блокировки блокирует других.
**Прямой SQL в обход.** Блокировки платформы не сработают.
