# Physical tables of an accumulation register: balances and turnovers

## In short

A balance-type accumulation register stores movements in a records table and aggregated totals in a separate totals table (two physical tables). A turnovers-type register stores only movements in one table; virtual Balance and Turnover tables in the query language are not separate physical tables for every kind.

## How to answer in an interview

- I distinguish register kind Balance vs Turnovers when answering about table count.
- For balances: movements table plus totals table; totals speed up `Остатки()` in queries.
- For turnovers: one movements table; the platform does not accumulate balances.
- Virtual table `.Обороты(,,)` is built from movements; `.Остатки()` for a balance register relies on totals.
- I am honest: confusing query virtual tables with physical DBMS tables is a common interview mistake.

## How it works

Query to a balance register with virtual tables:

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |    Ост.Номенклатура,
    |    Ост.КоличествоОстаток,
    |    Обор.КоличествоПриход,
    |    Обор.КоличествоРасход
    |ИЗ
    |    РегистрНакопления.ТоварыНаСкладах.Остатки(&Граница, ) КАК Ост
    |        ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ТоварыНаСкладах.Обороты(&Начало, &Конец) КАК Обор
    |        ПО Ост.Номенклатура = Обор.Номенклатура";
```

On posting a document writes a record set to the movements table; for balance registers the platform updates the totals table. Balance control mode and totals freshness affect when parallel sessions see stock.

## Pitfalls

**One answer for all kinds.** A turnovers register has no balance totals table.
**Direct SQL to totals.** Service table names are not portable across DBMS engines and versions.
**Disabled totals.** `Остатки()` queries without totals rebuild become heavy.
**Slice last instead of balances.** Different mechanisms; you cannot substitute one for the other.
**Posting without movements.** An empty record set changes neither movements nor totals.
