# Запись регистра сведений без регистратора: когда возможно

## Суть

В независимом регистре сведений запись создаётся по ключу измерений и периода без поля регистратор. В подчинённом регистратору регистре каждая строка обязана ссылаться на документ; записи «сами по себе» недопустимы.

## Что ответить на собесе

- Я сначала смотрю режим записи регистра в метаданных.
- Для цен и курсов часто независимый регистр: пишу набор без регистратора.
- Для статусов по документу только подчинённый регистр с отбором по регистратору.
- При отмене проведения подчинённые записи удаляются вместе с документом.
- Честно: независимая запись без аудита источника усложняет разбор инцидентов.

## Как это работает

Независимая запись цены:

```bsl
Менеджер = РегистрыСведений.ЦеныНоменклатуры;
Менеджер.СоздатьМенеджерЗаписи();
Запись = Менеджер.СоздатьМенеджерЗаписи();
Запись.Период = ДатаЦены;
Запись.Номенклатура = НоменклатураСсылка;
Запись.Цена = 1500;
Запись.Записать();
```

Подчинённый регистр заполняется только из `ОбработкаПроведения` документа с установленным `Регистратор`.

## Подводные камни

**Запись без регистратора в подчинённом режиме.** Ошибка платформы или битые данные.
**Дубли по измерениям.** В независимом регистре второй срез с тем же ключом перезаписывает первый.
**Путаница периодичности.** Непериодический регистр не принимает срез по дате как ожидается.
**Ручная правка без документа.** Нет цепочки отмены проведения.
**Смешение режимов в одной конфигурации.** Разные регистры ведут себя по-разному при одинаковом коде.
