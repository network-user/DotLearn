# Как корректно менять движения регистров

## Суть

Штатный путь: изменить документ-регистратор и перепровести. Платформа перезапишет набор движений по отбору регистратора. Прямое редактирование таблицы движений допустимо только в служебных сценариях и ломает связь с учётным документом.

## Что ответить на собесе

- Я исправляю данные через отмену проведения, правку документа и повторное проведение.
- Программно: `ПолучитьОбъект()`, изменить, `Записать(РежимЗаписиДокумента.Проведение)`.
- Набор записей регистра читаю с отбором по `Регистратор`, меняю и `Записать()` только если нет документа.
- После правки движений проверяю итоги регистра остатков.
- Честно: ручная правка SQL или обработкой «в обход документа» создаёт расхождение с регистратором.

## Как это работает

Перепроведение существующего документа:

```bsl
ДокументОбъект = ДокументСсылка.ПолучитьОбъект();
ДокументОбъект.Комментарий = "Корректировка количества";
ДокументОбъект.Товары[0].Количество = 15;
ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
```

Платформа удаляет старые движения регистратора и записывает новые из `Движения` или `ОбработкаПроведения`. Для независимого регистра сведений набор записей перезаписывается целиком по ключу.

Отмена проведения очищает движения регистратора без удаления самого документа.

## Подводные камни

**Правка движений без документа.** Отчёты по регистратору не сходятся с движениями.
**Проведение без перезаписи набора.** Дубли движений при повторной записи набора без отбора.
**Игнор блокировок.** Параллельное проведение двух документов по одному складу даёт гонку.
**Забытая отмена проведения.** Изменение проведённого документа без отмены запрещено настройками.
**Итоги не пересчитаны.** Остатки в отчёте неверны после прямой правки таблицы.
