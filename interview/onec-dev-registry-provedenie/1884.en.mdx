# Base calculation kinds in the chart of calculation types

## In short

A base calculation kind defines which accrued kinds supply the base for percentages and formulas. The link is configured in the chart of calculation types and drives calculation order in the calculation register.

## How to answer in an interview

- I explain: base kinds are input for a dependent accrual (e.g. bonus from salary).
- The platform orders calculation by the dependency graph.
- Without a base kind a percentage line gets zero or wrong base.
- Base kinds differ from leading and displacing kinds.
- I am honest: cyclic base links break payroll until the chart is fixed.

## How it works

In the chart for kind Bonus10 I set base kinds Salary and Allowance. On payroll document posting the calculation register gets base rows first; the derived kind reads them as base.

```bsl
// simplified: result depends on base kind rows in the same action period
БазаПремии = ПолучитьБазу(Сотрудник, Период, МассивБазовыхВидов);
Результат = Окр(БазаПремии * 0.10, 2);
```

## Pitfalls

**Cycle in base kinds.** Calculation does not finish or errors.
**Wrong base period.** Bonus calculated from last month.
**Confusion with leading kinds.** Different dependency mechanisms in the platform.
**Manual write without chart.** Code formulas diverge from metadata.
**Recalculation without base refresh.** Old base rows remain in the register.
