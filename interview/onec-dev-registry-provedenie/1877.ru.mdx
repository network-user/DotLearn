# Параметры периода в виртуальных таблицах регистра накопления

## Суть

В виртуальных таблицах `.Остатки()`, `.Обороты()` и `.ОстаткиИОбороты()` границы периода задают датой, моментом времени или границей (`Граница`, `МоментВремени`). Параметры отсекают движения по полю `Период` записи регистра.

## Что ответить на собесе

- Я передаю в `Остатки` момент на конец интервала, в `Обороты` пару начало и конец.
- Использую `КонецДня()`, `НачалоМесяца()` для однозначной границы.
- Для среза «на момент документа» применяю `МоментВремени(ДокументСсылка)`.
- Пустой второй параметр в `Остатки(&Дата, )` означает отбор только по измерениям в скобках.
- Честно: подставить строку вместо даты нельзя; тип параметра должен совпадать с ожидаемым.

## Как это работает

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |    Ост.Номенклатура,
    |    Ост.КоличествоОстаток
    |ИЗ
    |    РегистрНакопления.ТоварыНаСкладах.Остатки(&Момент, Склад = &Склад) КАК Ост";

Запрос.УстановитьПараметр("Момент", КонецДня(ДатаОтчета));
Запрос.УстановитьПараметр("Склад", СкладСсылка);
```

Обороты за месяц:

```bsl
"РегистрНакопления.Продажи.Обороты(&Начало, &Конец, , )"
```

`Граница` позволяет включить или исключить движения на самой границе; `МоментВремени` учитывает порядок проведения документов в одну секунду.

## Подводные камни

**Дата без времени на границе дня.** Остаток на конец дня требует `КонецДня`.
**Перепутаны параметры оборотов.** Начало больше конца даёт пустую выборку.
**Игнор момента времени.** Два документа в одну секунду дают неверный срез без `МоментВремени`.
**Неактуальные итоги.** `Остатки()` на больших регистрах без пересчёта итогов тормозит.
**Строка вместо даты.** Ошибка компиляции запроса или неявное приведение.
