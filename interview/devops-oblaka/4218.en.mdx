# How are EC2 instance types classified?

## In short

EC2 families are named by letter: T/M (general purpose), C (compute), R/X (memory), I/D (storage), P/G (GPU/ML), F (FPGA). The generation number and suffix (a, g for Graviton, n for network) denote CPU generation and options. Pick by workload profile: CPU-bound, RAM-bound, disk I/O, or accelerators.

## How to answer in an interview

- T3/T4g for burstable dev and low traffic; I watch CPU credits.
- M7i/M7g for balanced web/app servers; Graviton is cheaper when ARM builds exist.
- C7 for batch, transcode, compile farms; R7 for in-memory DB and caches.
- I4 for NVMe local storage and high random I/O; P5 for GPU training/inference.
- Honestly: I find the right size via benchmarks plus Compute Optimizer, not memorizing every SKU.

## How it works

Instance type sets vCPU, RAM, network bandwidth, EBS limits, and sometimes local NVMe. Spot and Reserved/Savings Plans change economics, not hardware specs.

Launch via CLI with an explicit type:

```bash
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type m7g.large \
  --key-name deploy-key \
  --subnet-id subnet-private-app \
  --security-group-ids sg-app-tier \
  --iam-instance-profile Name=AppInstanceProfile \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=web-01}]'
```

CloudFormation pins the type in IaC:

```yaml
Resources:
  AppHost:
    Type: AWS::EC2::Instance
    Properties:
      InstanceType: m7g.large
      ImageId: ami-0c55b159cbfafe1f0
      SubnetId: !Ref PrivateSubnet
      SecurityGroupIds:
        - !Ref AppSecurityGroup
```

## Pitfalls

- **T-series CPU credits.** Sustained 100% load on T3 without unlimited mode throttles.
- **ARM compatibility.** Graviton needs arm64 images and binaries; verify CI matrix.
- **EBS vs instance store.** Local NVMe vanishes on stop/terminate; treat as ephemeral unless replicated.
- **Placement groups.** Cluster gives low latency but limits AZ capacity when scaling out.
- **Overprovisioning.** Oversized instances block capacity; right-size from CloudWatch metrics.
