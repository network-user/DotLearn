# Какие ключевые сервисы входят в экосистему AWS?

## Суть

AWS это набор из сотен сервисов, сгруппированных по доменам: compute (EC2, Lambda, ECS/EKS), storage (S3, EBS, EFS), databases (RDS, DynamoDB, Aurora), networking (VPC, Route 53, CloudFront, Direct Connect), security (IAM, KMS, WAF), observability (CloudWatch, X-Ray), IaC и CI/CD (CloudFormation, CDK, CodePipeline). На собесе важно показать карту доменов, а не заучивать весь каталог.

## Что ответить на собесе

- Compute: EC2 для долгоживущих VM, Lambda для event-driven, контейнеры через ECS/EKS/Fargate.
- Data: S3 для объектов, RDS/Aurora для SQL, DynamoDB для key-value с предсказуемым latency.
- Network: VPC как изолированная сеть, ALB/NLB, Route 53, PrivateLink для сервис-to-сервис.
- Ops: CloudWatch метрики/логи/алармы, Systems Manager для патчей и run command.
- Честно: полный стек AWS огромен; на проекте глубоко знаю 15-20 сервисов, остальное нахожу по документации.

## Как это работает

Типичный веб-стек: Route 53 указывает на ALB в public subnet; targets в private subnet; данные в RDS Multi-AZ; статика в S3 за CloudFront; секреты в Secrets Manager; деплой через CodePipeline + CloudFormation stack.

Быстрый обзор compute через CLI:

```bash
aws ec2 describe-instance-types \
  --filters Name=instance-type,Values=t3.micro,m5.large \
  --query 'InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Mem:MemoryInfo.SizeInMiB}' \
  --output table

aws lambda list-functions --query 'Functions[?Runtime!=`null`].[FunctionName,Runtime,MemorySize]' --output table
```

IAM связывает всё: роли для EC2/Lambda, policies с least privilege, SCP на уровне Organizations для guardrails.

## Подводные камни

- **Сервисный зоопарк.** Дублирующие решения (MSK vs Kinesis) плодят операционный долг; фиксирую стандарты в платформенной команде.
- **Региональные отличия.** Не каждый сервис во всех регионах; проверяю Service Quotas и regional endpoints до архитектуры.
- **Скрытые cost drivers.** NAT Gateway, cross-AZ traffic, idle EBS; FinOps начинается с тегов и Cost Explorer.
- **IAM complexity.** Wildcard policies накапливаются; периодический access analyzer и policy lint обязательны.
- **Vendor coupling.** Managed удобно, но миграция дороже; критичные контракты абстрагирую через portable formats (S3 API, Postgres).
