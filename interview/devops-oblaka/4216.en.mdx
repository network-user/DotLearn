# What are the core services in the AWS ecosystem?

## In short

AWS is hundreds of services grouped by domain: compute (EC2, Lambda, ECS/EKS), storage (S3, EBS, EFS), databases (RDS, DynamoDB, Aurora), networking (VPC, Route 53, CloudFront, Direct Connect), security (IAM, KMS, WAF), observability (CloudWatch, X-Ray), and IaC/CI/CD (CloudFormation, CDK, CodePipeline). In interviews, show a domain map rather than memorizing the full catalog.

## How to answer in an interview

- Compute: EC2 for long-lived VMs, Lambda for event-driven work, containers via ECS/EKS/Fargate.
- Data: S3 for objects, RDS/Aurora for SQL, DynamoDB for key-value with predictable latency.
- Network: VPC as the isolated network, ALB/NLB, Route 53, PrivateLink for service-to-service.
- Ops: CloudWatch metrics/logs/alarms, Systems Manager for patching and run command.
- Honestly: the full AWS stack is huge; on a project I know 15-20 services deeply and look up the rest in docs.

## How it works

A typical web stack: Route 53 points to an ALB in a public subnet; targets sit in a private subnet; data in RDS Multi-AZ; static assets in S3 behind CloudFront; secrets in Secrets Manager; deploy via CodePipeline plus a CloudFormation stack.

Quick compute overview via CLI:

```bash
aws ec2 describe-instance-types \
  --filters Name=instance-type,Values=t3.micro,m5.large \
  --query 'InstanceTypes[].{Type:InstanceType,vCPU:VCpuInfo.DefaultVCpus,Mem:MemoryInfo.SizeInMiB}' \
  --output table

aws lambda list-functions --query 'Functions[?Runtime!=`null`].[FunctionName,Runtime,MemorySize]' --output table
```

IAM ties it together: roles for EC2/Lambda, least-privilege policies, and Organization SCPs for guardrails.

## Pitfalls

- **Service sprawl.** Overlapping choices (MSK vs Kinesis) create ops debt; platform teams set standards.
- **Regional gaps.** Not every service exists in every region; check Service Quotas and endpoints before design.
- **Hidden cost drivers.** NAT Gateway, cross-AZ traffic, idle EBS; FinOps starts with tags and Cost Explorer.
- **IAM complexity.** Wildcard policies accumulate; periodic access analyzer and policy lint are mandatory.
- **Vendor coupling.** Managed is convenient but migration is costly; abstract critical contracts via portable formats (S3 API, Postgres).
