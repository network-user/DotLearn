# Что такое AWS Outposts и зачем он нужен?

## Суть

AWS Outposts это managed rack с AWS compute/storage на площадке клиента, управляемый теми же API и консолью, что и региональный cloud. Даёт гибрид: данные и обработка остаются on-prem при низкой latency и compliance, а control plane живёт в AWS. Форматы: Outposts rack и Outposts servers.

## Что ответить на собесе

- Нужен когда regulation или latency требуют локального размещения, но команда хочет EC2/VPC/EBS модель.
- Подключается к home region через локальную сеть и service link; без стабильного канала управление страдает.
- Поддерживает subset сервисов (EC2, EBS, ECS, RDS on Outposts и др.), не весь каталог AWS.
- Local Zones и Wavelength ближе к edge, Outposts ближе к full rack в ЦОД клиента.
- Честно: CapEx и операционная модель тяжелее pure cloud; оправдан при жёстких data residency требованиях.

## Как это работает

AWS ставит оборудование, создаёт Outpost ARN, привязывает к VPC extension в home region. Инстансы создаются как обычный EC2, но AZ это `outpost-id`. Subnet и route tables изолированы на площадке.

Проверка ресурсов Outpost:

```bash
aws outposts list-outposts \
  --query 'Outposts[].{Id:OutpostId,Site:Site,Status:LifeCycleStatus}' \
  --output table

aws ec2 run-instances \
  --image-id ami-local-outpost \
  --instance-type m5.large \
  --subnet-id subnet-on-outpost-01 \
  --placement "AvailabilityZone=op-0123456789abcdef"
```

Гибридный сценарий: latency-sensitive video processing on Outpost, metadata и billing в regional S3/DynamoDB через private connectivity.

## Подводные камни

- **Ограниченный service set.** Нельзя поднять любой managed сервис; проверяю matrix до закупки.
- **Capacity planning.** Расширение физического rack не мгновенное, как elastic scale в регионе.
- **Patching window.** AWS управляет инфраструктурой, но согласование maintenance с локальным ЦОД обязательно.
- **Network partition.** Потеря uplink не убивает локальные VM сразу, но API и backup в cloud блокируются.
- **Licensing и support.** Сторонние ISV лицензии on-prem могут отличаться от cloud marketplace.
