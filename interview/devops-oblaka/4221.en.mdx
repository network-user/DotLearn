# What is AWS Outposts and why use it?

## In short

AWS Outposts is a managed rack of AWS compute and storage on the customer premises, operated with the same APIs and console as the regional cloud. It enables hybrid deployments: data and processing stay on-prem for latency and compliance while the control plane lives in AWS. Form factors include Outposts rack and Outposts servers.

## How to answer in an interview

- I use it when regulation or latency demands local placement but the team wants the EC2/VPC/EBS model.
- It connects to the home region over local network and service link; unstable uplinks hurt management.
- It supports a subset of services (EC2, EBS, ECS, RDS on Outposts, etc.), not the full AWS catalog.
- Local Zones and Wavelength target edge; Outposts is closer to a full rack in the customer data center.
- Honestly: CapEx and operations are heavier than pure cloud; justified only with strict data residency needs.

## How it works

AWS installs hardware, creates an Outpost ARN, and extends a VPC in the home region. Instances launch like normal EC2 but the AZ is `outpost-id`. Subnets and route tables stay on the premises.

Inspect Outpost resources:

```bash
aws outposts list-outposts \
  --query 'Outposts[].{Id:OutpostId,Site:Site,Status:LifeCycleStatus}' \
  --output table

aws ec2 run-instances \
  --image-id ami-local-outpost \
  --instance-type m5.large \
  --subnet-id subnet-on-outpost-01 \
  --placement "AvailabilityZone=op-0123456789abcdef"
```

Hybrid pattern: latency-sensitive video processing on Outpost, metadata and billing in regional S3/DynamoDB over private connectivity.

## Pitfalls

- **Limited service set.** Not every managed service is available; verify the matrix before purchase.
- **Capacity planning.** Expanding a physical rack is not instant like regional elastic scale.
- **Patching window.** AWS operates the gear but local data center maintenance coordination is required.
- **Network partition.** Uplink loss does not instantly kill local VMs but cloud API and backup stall.
- **Licensing and support.** Third-party ISV licenses on-prem may differ from cloud marketplace terms.
