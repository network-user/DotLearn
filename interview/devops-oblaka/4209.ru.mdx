# RDS Single-AZ и Multi-AZ: в чём отличие?

## Суть

Single-AZ RDS живёт в одной зоне доступности: при сбое AZ база недоступна до ручного восстановления. Multi-AZ держит синхронную standby-копию в другой AZ того же региона; при падении primary AWS переключает DNS endpoint на standby без потери коммитов.

## Что ответить на собесе

- Single-AZ: dev/stage, некритичные сервисы, когда RTO измеряется часами.
- Multi-AZ: prod, где нужен автоматический failover за минуты и RPO ≈ 0.
- Standby в Multi-AZ не для read-трафика; чтение с реплик — отдельный Read Replica.
- Честно: Multi-AZ стоит почти вдвое дороже compute; для read-heavy лучше Replica + Single-AZ, а не Multi-AZ ради масштабирования чтения.

## Как это работает

Создание Multi-AZ MySQL через CLI:

```bash
aws rds create-db-instance \
  --db-instance-identifier app-db-prod \
  --db-instance-class db.m7g.large \
  --engine mysql \
  --engine-version 8.0.36 \
  --master-username dbadmin \
  --master-user-password "$(openssl rand -base64 24)" \
  --allocated-storage 100 \
  --storage-type gp3 \
  --multi-az \
  --backup-retention-period 14
```

Включить Multi-AZ на существующем инстансе:

```bash
aws rds modify-db-instance \
  --db-instance-identifier app-db-prod \
  --multi-az \
  --apply-immediately
```

При failover endpoint `app-db-prod.xxxx.region.rds.amazonaws.com` не меняется; приложение переподключается к тому же hostname. Репликация синхронная: транзакция подтверждается только после записи на standby. Время простоя обычно 60–120 секунд.

## Подводные камни

- **Путать Multi-AZ и Read Replica.** Standby не обслуживает SELECT; для read scale нужна отдельная реплика.
- **Failover без connection pool.** Пул с долгоживущими коннектами зависнет; нужен retry и короткий idle timeout.
- **Maintenance window и failover.** Плановый reboot Multi-AZ тоже переключает AZ; закладываю окно в SLO.
- **Cross-region DR.** Multi-AZ защищает от сбоя AZ, но не от падения всего региона; для этого нужен cross-region backup или replica.
- **Стоимость storage.** Оба инстанса платят за диски; gp3/io2 на Multi-AZ удваивает storage bill.
