# Assert introspection in PyTest

## In short

PyTest intercepts plain `assert` in tests and on failure shows intermediate subexpression values. Assert introspection parses the expression AST and injects actual variable values into the report. This replaces manual messages for most unit asserts.

## How to answer in an interview

- `assert resp.status == 200` shows `200 == 404` with `resp.status` unfolded.
- It works on standard assert; no unittest style required.
- With `-O` (optimize) asserts are stripped; introspection will not run.
- Complex side effects inside assert should move to a variable first.
- Honestly: with `assert foo() == bar()` introspection may call functions twice unless assigned upfront.

## How it works

Failing assert:

```python
def test_total():
    price = 19
    fee = 2
    expected = 25
    assert price + fee == expected
```

Report fragment:

```
assert 21 == 25
 +  where 21 = 19 + 2
```

## Pitfalls

**assert with side effects.** Re-evaluation for the report mutates state.

**Custom pytest_assertrepr_compare.** A plugin can override the output.

**Large dict comparison.** Diff text explodes; use `-vv` deliberately.

**Rewrite disabled.** `--assert=plain` removes introspection.

**Hamcrest assert.** Third-party matchers do not always get the same breakdown.
