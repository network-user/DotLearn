# The monkeypatch fixture

## In short

`monkeypatch` is PyTest's built-in fixture for temporarily replacing attributes, env vars, `sys.path`, and import paths. After the test PyTest rolls changes back automatically. I use it to isolate HTTP, clocks, and configuration without real external calls.

## How to answer in an interview

- `monkeypatch.setattr(module, "func", stub)` replaces a function.
- `monkeypatch.setenv("API_URL", "http://test")` for config.
- `monkeypatch.delitem` / `setitem` for dict-like env.
- I prefer `mocker` from pytest-mock, but monkeypatch needs no extra deps.
- Honestly: patching the wrong place (where imported vs where defined) yields "patch did nothing".

## How it works

HTTP client stub:

```python
import api_client


def fake_get(url: str) -> dict:
    return {"ok": True}


def test_fetch(monkeypatch):
    monkeypatch.setattr(api_client, "http_get", fake_get)
    assert api_client.fetch_status() == {"ok": True}
```

Env override:

```python
def test_mode(monkeypatch):
    monkeypatch.setenv("APP_MODE", "test")
    assert settings.mode() == "test"
```

## Pitfalls

**Patch target.** Patch the symbol in the tested module's namespace.

**Manual teardown forgotten.** monkeypatch rolls back; a manual patch without the fixture does not.

**setattr on immutable.** Named tuples and some C extensions cannot be patched.

**delenv without restore.** monkeypatch restores; manual `os.environ.pop` is risky.

**Session scope test + monkeypatch.** A function fixture does not cover neighbor worker state.
