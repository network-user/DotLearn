# Skipping tests in PyTest

## In short

Skip a test with `@pytest.mark.skip` / `skipif` or `pytest.skip()` at runtime. Skip differs from fail: reports show yellow skipped; CI may count it separately. `xfail` has different semantics: expected failure, not a skip.

## How to answer in an interview

- `@pytest.mark.skip(reason="JIRA-123")` hard-disables a case.
- `@pytest.mark.skipif(sys.platform == "win32")` conditionally.
- `pytest.skip("no docker")` inside the test when a dependency is missing.
- Module skip via `pytestmark = pytest.mark.skip(...)`.
- Honestly: a permanent skip without a ticket equals dead code; remove or use temporary xfail.

## How it works

Decorator:

```python
import pytest


@pytest.mark.skip(reason="feature behind flag")
def test_beta():
    assert beta()
```

Runtime:

```python
import shutil
import pytest


def test_docker_flow():
    if shutil.which("docker") is None:
        pytest.skip("docker binary missing")
    assert run_container()
```

Conditional:

```python
import sys
import pytest


@pytest.mark.skipif(sys.version_info < (3, 11), reason="needs 3.11")
def test_tomllib():
    assert True
```

## Pitfalls

**skip vs xfail.** xfail strict fails CI on unexpected pass.

**skipif at import time.** Conditions with import side effects are risky.

**skip in fixture setup.** The test is skipped but setup may have partially run.

**Hidden skipif false positive.** Platform strings are brittle.

**Report: skipped as success.** Coverage metrics do not grow; fullness is an illusion.
