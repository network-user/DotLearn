# Конфигурация PyTest

## Суть

PyTest читает настройки из `pytest.ini`, `pyproject.toml` (`[tool.pytest.ini_options]`) или `setup.cfg` (`[tool:pytest]`). Туда кладут `testpaths`, `addopts`, `markers`, `python_files`, `log_cli`, `filterwarnings`. Один канонический файл в корне, дубли создают drift между локальным и CI запуском.

## Что ответить на собесе

- `testpaths = tests` ограничивает collection root.
- `addopts = -ra --strict-markers` фиксирует флаги CI.
- markers registry в том же файле, не размазан по README.
- pyproject.toml предпочитаю в новых репозиториях.
- Честно: два активных конфига (ini + toml) дают «works on my machine» с разным addopts.

## Как это работает

pytest.ini:

```ini
[pytest]
testpaths = tests
addopts = -ra --strict-markers --tb=short
markers =
    smoke: gate suite
```

pyproject.toml:

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-ra --strict-markers"
```

## Подводные камни

**Относительный testpaths.** Запуск не из корня ломает discovery.

**addopts с абсолютным путём.** Ломает коллег на Windows/Linux.

**filterwarnings ignore all.** Скрывает реальные Deprecation в lib.

**ini_options typo.** Тихий fallback на дефолты.

**conftest vs ini.** ini не заменяет fixtures, только глобальные опции.
