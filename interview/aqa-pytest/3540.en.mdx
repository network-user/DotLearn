# Checking logs with caplog

## In short

The `caplog` fixture captures `logging` records during a test. Via `caplog.records`, `caplog.text`, and the `caplog.at_level` context you verify message, level, and logger name. This contracts observability: the test fails if production code stops logging an error.

## How to answer in an interview

- `with caplog.at_level(logging.ERROR):` narrows capture.
- `assert "payment failed" in caplog.text` is a simple smoke check.
- `caplog.records[0].levelname == "WARNING"` is more precise structurally.
- I set `log_cli` in ini for local debug; caplog for asserts.
- Honestly: a root logger handler outside the test may duplicate records; I filter by logger name.

## How it works

```python
import logging


logger = logging.getLogger("billing")


def charge():
    logger.warning("card declined")


def test_charge_logs(caplog):
    with caplog.at_level(logging.WARNING, logger="billing"):
        charge()
    assert "card declined" in caplog.text
    assert caplog.records[0].levelno == logging.WARNING
```

## Pitfalls

**Propagate=False.** A child logger without propagate will not reach caplog's root.

**Level below threshold.** INFO is invisible at WARNING caplog without at_level.

**Formatter changes text.** Compare message, not the full formatted line.

**caplog.clear().** Forgetting clear between calls mixes records.

**structlog.** Needs a bridge to stdlib logging or caplog stays empty.
