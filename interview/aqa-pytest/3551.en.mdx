# Overriding fixtures

## In short

A fixture with the same name in a narrower scope overrides the parent: a nearer conftest beats the root. Overrides live in `tests/integration/conftest.py` for a different API URL or mock backend. PyTest picks the nearest fixture by directory tree, not import order.

## How to answer in an interview

- Root `conftest.py` provides a prod-like `client`.
- `tests/unit/conftest.py` replaces `client` with InMemory.
- `@pytest.fixture(name="client")` pins the name when the function is aliased.
- `usefixtures` does not override; it only attaches.
- Honestly: an override without a note in conftest silently breaks a neighboring module.

## How it works

Root:

```python
import pytest


@pytest.fixture
def api_url():
    return "https://staging.example"
```

Unit package:

```python
import pytest


@pytest.fixture
def api_url():
    return "http://localhost:8000"
```

A test under `tests/unit/` gets localhost.

## Pitfalls

**Two conftest at one level.** Impossible, but duplicate fixture names in one file error.

**import fixture from another module.** Override fails; conftest scope is required.

**final fixture plugin.** Some plugins mark fixtures non-overridable.

**session override function.** Scope mismatch causes unexpected sharing.

**Copy-paste name typo.** The test quietly uses the parent fixture.
