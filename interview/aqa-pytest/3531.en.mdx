# PyTest fixtures

## In short

A fixture is a function with `@pytest.fixture` that prepares data or environment for a test and optionally cleans up afterward. The test requests a fixture by parameter name; PyTest injects the value and manages lifecycle by scope. `yield` splits setup and teardown in one function.

## How to answer in an interview

- `@pytest.fixture` plus an `api_client` argument in the test binds by name.
- `scope="session"` for an expensive browser, `function` for state isolation.
- `yield` after setup: code below runs as teardown.
- `conftest.py` exposes the fixture to the package without import.
- Honestly: a fixture without explicit teardown still runs yield after PyTest's finally when the test fails.

## How it works

Setup/teardown via yield:

```python
import pytest


@pytest.fixture
def sample_payload():
    payload = {"user": "qa", "role": "admin"}
    yield payload
    payload.clear()


def test_role(sample_payload):
    assert sample_payload["role"] == "admin"
```

## Pitfalls

**Parameter name ≠ fixture.** Typos yield "fixture not found", not AttributeError in the test.

**Mutable shared session fixture.** Parallel pytest-xdist breaks a shared dict.

**Fixture dependency cycle.** A→B→A errors at collection.

**return instead of yield.** No teardown; the browser stays open.

**autouse without control.** Hidden setup complicates debugging a failing test.
