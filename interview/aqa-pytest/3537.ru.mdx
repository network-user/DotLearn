# Ключевые опции CLI pytest

## Суть

Опции pytest управляют verbosity, фильтрацией, остановкой на fail, traceback, capture stdout и отчётами. В pipeline чаще всего комбинируют `-q`, `--maxfail`, `--tb=short`, `-m`, `--junitxml` и `-k`. Понимание capture важно, когда тест логирует в консоль и отчёт «пустой».

## Что ответить на собесе

- `-v` подробный список, `-q` тихий для CI.
- `-k expr` фильтр по substring имени node.
- `--maxfail=1` fail-fast на merge request.
- `--tb=line` короткий traceback в логе Jenkins.
- Честно: `--capture=no` шумит в parallel, но спасает при debug одного теста.

## Как это работает

Типичный pipeline вызов:

```bash
pytest tests \
  -q \
  -m "smoke and not slow" \
  --maxfail=3 \
  --tb=short \
  --junitxml=reports/junit.xml \
  -k "login"
```

Полезные одиночные флаги:

```bash
pytest --collect-only
pytest --lf
pytest --disable-warnings
```

## Подводные камни

**-k vs -m.** Путают substring имени и marker expression.

**--maxfail=0.** Ноль часто значит «без лимита», не «остановись сразу».

**junit family.** `--junitxml` без `--junit-prefix` даёт коллизии suite name.

**--lf без cache.** last-failed требует `.pytest_cache` в workspace.

**-x и --maxfail.** `-x` эквивалент `--maxfail=1`, но привычка смешивать оба.
