# Регистрация кастомных маркеров

## Суть

Пользовательские markers объявляют в `[pytest]` секции `pytest.ini`, `pyproject.toml` (`[tool.pytest.ini_options]`), `tox.ini` или `setup.cfg`. Строка `markers =` перечисляет имя и описание, чтобы PyTest не ругался на неизвестные теги и чтобы `pytest --markers` показывал документацию команды.

## Что ответить на собесе

- `markers = smoke: fast gate tests` в pytest.ini.
- `--strict-markers` в CI ловит опечатки в декораторе.
- Описание после двоеточия попадает в help, не в runtime.
- pyproject.toml предпочитаю в новых репо вместо legacy setup.cfg.
- Честно: без регистрации marker работает, но warning засоряет junit.

## Как это работает

pytest.ini:

```ini
[pytest]
markers =
    smoke: critical path checks
    slow: long running scenarios
    api: http contract tests
```

Использование:

```python
import pytest


@pytest.mark.api
def test_users_contract():
    assert fetch_users().status_code == 200
```

Запуск:

```bash
pytest --strict-markers -m api
```

## Подводные камни

**Отступы в ini.** Продолжение строки marker требует отступ, иначе parse error.

**Два конфиг файла.** pytest читает один канонический, дубли в tox.ini путают.

**Переименование marker.** Старые ветки CI с `-m oldname` дают 0 tests.

**Встроенные markers.** `skip`, `xfail` регистрировать не нужно.

**Описание с запятой.** Редкий parse edge case, держу текст простым.
