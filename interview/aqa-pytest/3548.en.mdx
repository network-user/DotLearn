# PyTest configuration files

## In short

PyTest reads settings from `pytest.ini`, `pyproject.toml` (`[tool.pytest.ini_options]`), or `setup.cfg` (`[tool:pytest]`). Typical entries: `testpaths`, `addopts`, `markers`, `python_files`, `log_cli`, `filterwarnings`. One canonical root file avoids drift between local and CI runs.

## How to answer in an interview

- `testpaths = tests` limits the collection root.
- `addopts = -ra --strict-markers` fixes CI flags.
- Marker registry lives in the same file, not scattered in README.
- I prefer pyproject.toml in new repositories.
- Honestly: two active configs (ini + toml) cause "works on my machine" with different addopts.

## How it works

pytest.ini:

```ini
[pytest]
testpaths = tests
addopts = -ra --strict-markers --tb=short
markers =
    smoke: gate suite
```

pyproject.toml:

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
addopts = "-ra --strict-markers"
```

## Pitfalls

**Relative testpaths.** Running outside the repo root breaks discovery.

**addopts with absolute paths.** Breaks teammates on Windows/Linux.

**filterwarnings ignore all.** Hides real Deprecation warnings in libs.

**ini_options typo.** Silent fallback to defaults.

**conftest vs ini.** ini does not replace fixtures, only global options.
