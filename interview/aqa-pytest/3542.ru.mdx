# Параллельный pytest-xdist

## Суть

`pytest-xdist` распределяет test items по worker процессам (`-n 4` или `-n auto`). Каждый worker это отдельный Python процесс со своим import state. Ускорение ограничено shared resources: БД, файлы, browser grid, session fixtures.

## Что ответить на собесе

- `pytest -n auto` на CI агенте по числу CPU.
- `--dist loadscope` группирует class/module в один worker, меньше shared collisions.
- Session fixture с mutable dict ломает изоляцию между workers.
- Уникальные DB schema per worker или transactional rollback.
- Честно: xdist не ускоряет один тяжёлый E2E, только много независимых.

## Как это работает

Запуск:

```bash
pip install pytest-xdist
pytest -n 4 tests/unit
pytest -n auto --dist loadscope
```

Изолированная fixture:

```python
import pytest


@pytest.fixture
def buffer():
    return []


def test_a(buffer):
    buffer.append(1)
    assert buffer == [1]


def test_b(buffer):
    buffer.append(2)
    assert buffer == [2]
```

## Подводные камни

**Глобальный singleton.** Counter в module level даёт race.

**Один файл логов.** Параллельная запись corrupts log.

**session browser.** Один chromedriver на всех workers недоступен.

**loadfile vs loadscope.** Неправильный dist режим смешивает class state.

**Coverage combine.** Забытый `coverage combine` после xdist врёт метрику.
