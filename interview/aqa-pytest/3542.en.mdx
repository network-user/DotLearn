# Parallel runs with pytest-xdist

## In short

`pytest-xdist` distributes test items across worker processes (`-n 4` or `-n auto`). Each worker is a separate Python process with its own import state. Speedup is limited by shared resources: databases, files, browser grids, and session fixtures.

## How to answer in an interview

- `pytest -n auto` on CI agents by CPU count.
- `--dist loadscope` groups class/module on one worker, fewer shared collisions.
- A session fixture with a mutable dict breaks isolation across workers.
- Unique DB schema per worker or transactional rollback.
- Honestly: xdist does not speed up one heavy E2E test, only many independent ones.

## How it works

Run:

```bash
pip install pytest-xdist
pytest -n 4 tests/unit
pytest -n auto --dist loadscope
```

Isolated fixture:

```python
import pytest


@pytest.fixture
def buffer():
    return []


def test_a(buffer):
    buffer.append(1)
    assert buffer == [1]


def test_b(buffer):
    buffer.append(2)
    assert buffer == [2]
```

## Pitfalls

**Global singleton.** A module-level counter races.

**Single log file.** Parallel writes corrupt the log.

**Session browser.** One chromedriver for all workers is not viable.

**loadfile vs loadscope.** Wrong dist mode mixes class state.

**Coverage combine.** Missing `coverage combine` after xdist lies about metrics.
