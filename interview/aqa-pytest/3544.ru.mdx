# Allure с PyTest

## Суть

Плагин `allure-pytest` пишет JSON results в `--alluredir`, затем `allure generate` или `allure serve` строит HTML. Steps, labels, links и attachments делают отчёт читаемым для QA и dev. В UI автотестах прикладываю screenshot, console log, HAR и trace на fail.

## Что ответить на собесе

- `pytest --alluredir=allure-results` в pipeline.
- `@allure.step` дробит сценарий для triage.
- `allure.attach` для PNG, TXT, JSON body.
- severity, epic, feature через dynamic API или декораторы.
- Честно: Allure без стабильных test id даёт «новый» кейс на каждый parametrize id shuffle.

## Как это работает

```bash
pip install allure-pytest
pytest --alluredir=allure-results
allure serve allure-results
```

```python
import allure


@allure.step("Open catalog")
def open_catalog(page):
    page.goto("/catalog")


def test_catalog(page):
    open_catalog(page)
    allure.attach(page.screenshot(), "screen", allure.attachment_type.PNG)
    assert page.title() == "Catalog"
```

## Подводные камни

**Пустой alluredir.** Забытый plugin не пишет файлы, generate падает.

**Большие attachments.** Видео раздувает артефакт storage CI.

**Step в helper без context.** Слишком мелкие steps шумят.

**history на flaky.** Rerun без quarantine засоряет trend.

**Локальный serve в CI.** Нужен publish HTML artifact, не только raw results.
