# pytest-cov и смысл coverage

## Суть

`pytest-cov` оборачивает `coverage.py` и показывает, какие строки исполнялись во время тестов. Метрика line coverage не равна качеству: можно вызвать функцию без проверки веток и edge cases. Использую coverage как карту дыр, не как KPI «100% или провал».

## Что ответить на собесе

- `pytest --cov=app --cov-report=term-missing` в CI.
- `--cov-fail-under=80` gate только на критичных пакетах.
- branch coverage (`--cov-branch`) дороже, но ловит непокрытые if.
- omit в `.coveragerc` для generated и `if TYPE_CHECKING`.
- Честно: 100% line coverage возможен с бессмысленными assert, метрика обманывает.

## Как это работает

```bash
pip install pytest-cov
pytest tests --cov=app --cov-report=html
```

Модуль и тест:

```python
def discount(price: int, vip: bool) -> int:
    if vip:
        return price // 2
    return price


def test_discount_regular():
    assert discount(100, False) == 100
```

Отчёт покажет непокрытую ветку `vip=True`, хотя функция вызывалась.

## Подводные камни

**xdist без combine.** Проценты занижены per worker.

**Исполнили, не проверили.** Вызов без assert всё равно зелёный coverage.

**Динамический import.** Строки lazy module не попадают в отчёт.

**pragma no cover.** Злоупотребление скрывает реальный dead code.

**Только unit в cov.** E2E в другом runner не входит в тот же отчёт.
