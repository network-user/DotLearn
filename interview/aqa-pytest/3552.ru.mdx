# Пропуск тестов в PyTest

## Суть

Тест пропускают декоратором `@pytest.mark.skip` / `skipif` или вызовом `pytest.skip()` в runtime. Skip отличается от fail: в отчёте yellow skipped, CI может считать отдельно. `xfail` другой семантики: ожидаемый fail, не пропуск.

## Что ответить на собесе

- `@pytest.mark.skip(reason="JIRA-123")` жёстко отключает кейс.
- `@pytest.mark.skipif(sys.platform == "win32")` условно.
- `pytest.skip("no docker")` внутри теста при missing dependency.
- skip на module через `pytestmark = pytest.mark.skip(...)`.
- Честно: вечный skip без ticket равен мёртвому коду, чистим или xfail временно.

## Как это работает

Декоратор:

```python
import pytest


@pytest.mark.skip(reason="feature behind flag")
def test_beta():
    assert beta()
```

Runtime:

```python
import shutil
import pytest


def test_docker_flow():
    if shutil.which("docker") is None:
        pytest.skip("docker binary missing")
    assert run_container()
```

Условный:

```python
import sys
import pytest


@pytest.mark.skipif(sys.version_info < (3, 11), reason="needs 3.11")
def test_tomllib():
    assert True
```

## Подводные камни

**skip vs xfail.** xfail fail strict ломает CI при неожиданном pass.

**skipif import time.** Условие на import side effect опасно.

**skip в fixture setup.** Весь тест skipped, но setup уже успел выполниться частично.

**Скрытый skipif False positive.** Platform string хрупок.

**Отчёт: skipped как success.** Метрики coverage не растут, иллюзия полноты.
