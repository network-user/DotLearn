# Фикстура monkeypatch

## Суть

`monkeypatch` это встроенная fixture PyTest для временной подмены атрибутов, env vars, `sys.path` и import paths. После теста PyTest откатывает изменения автоматически. Использую для изоляции HTTP, clock и конфигурации без реального внешнего вызова.

## Что ответить на собесе

- `monkeypatch.setattr(module, "func", stub)` подменяет функцию.
- `monkeypatch.setenv("API_URL", "http://test")` для config.
- `monkeypatch.delitem` / `setitem` для dict env.
- Предпочитаю `mocker` из pytest-mock, но monkeypatch без extra deps.
- Честно: patch не туда (где импортировали vs где определили) даёт «патч не сработал».

## Как это работает

Подмена HTTP клиента:

```python
import api_client


def fake_get(url: str) -> dict:
    return {"ok": True}


def test_fetch(monkeypatch):
    monkeypatch.setattr(api_client, "http_get", fake_get)
    assert api_client.fetch_status() == {"ok": True}
```

Env override:

```python
def test_mode(monkeypatch):
    monkeypatch.setenv("APP_MODE", "test")
    assert settings.mode() == "test"
```

## Подводные камни

**Patch target.** Патчить нужно символ в namespace тестируемого модуля.

**Забытый teardown вручную.** monkeypatch откатывает, ручной patch без fixture нет.

**setattr на immutable.** Namedtuple и некоторые C-ext не патчатся.

**delenv без restore.** monkeypatch восстанавливает, `os.environ.pop` вручную опасен.

**Session scope test + monkeypatch.** Function fixture не покрывает соседний worker state.
