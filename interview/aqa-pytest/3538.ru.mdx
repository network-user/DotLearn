# Assert introspection в PyTest

## Суть

PyTest перехватывает plain `assert` в тестах и при падении показывает промежуточные значения подвыражений. Механизм assert introspection разбирает AST выражения и подставляет фактические значения переменных в отчёт. Это заменяет ручные сообщения в большинстве unit assert.

## Что ответить на собесе

- `assert resp.status == 200` покажет `200 == 404` с раскрытием `resp.status`.
- Работает на стандартном assert, не нужен unittest-style.
- С `-O` (optimize) assert вырезается, introspection не сработает.
- Сложные side-effect в assert лучше вынести в переменную.
- Честно: при `assert foo() == bar()` introspection дважды вызовет функции, если не присвоить заранее.

## Как это работает

Падающий assert:

```python
def test_total():
    price = 19
    fee = 2
    expected = 25
    assert price + fee == expected
```

Фрагмент отчёта:

```
assert 21 == 25
 +  where 21 = 19 + 2
```

## Подводные камни

**assert с side effect.** Повторное вычисление в отчёте меняет state.

**Кастомный pytest_assertrepr_compare.** Плагин может переопределить вывод.

**Сравнение больших dict.** Текст diff огромен, использую `-vv` осознанно.

**Отключение rewrite.** `--assert=plain` убирает introspection.

**Hamcrest assert.** Сторонние matcher не всегда дают такой же разбор.
