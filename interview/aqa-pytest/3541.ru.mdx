# Плагины PyTest в практике

## Суть

Плагины расширяют pytest hooks: отчёты, coverage, parallel, mocking, asyncio. В CI обычно стоят `pytest-cov`, `pytest-xdist`, `pytest-html` или Allure, `pytest-mock`. Выбор плагина это trade-off между скоростью прогона, детализацией отчёта и сложностью поддержки.

## Что ответить на собесе

- `pytest-cov --cov=app` gate по минимальному покрытию.
- `pytest-xdist -n auto` режет wall time на многоядерном агенте.
- `pytest-mock` даёт `mocker.patch` с autouse teardown.
- `pytest-asyncio` для async def тестов с event loop fixture.
- Честно: десять плагинов удлиняют collection и повышают риск hook conflict.

## Как это работает

Coverage + parallel:

```bash
pip install pytest-cov pytest-xdist
pytest -n auto --cov=app --cov-report=term-missing
```

Mock:

```python
def test_token(mocker):
    mocker.patch("auth.fetch_token", return_value="fake")
    assert login() == "ok"
```

HTML отчёт:

```bash
pytest --html=report.html --self-contained-html
```

## Подводные камни

**cov + xdist.** Нужен совместимый режим combine, иначе заниженный percent.

**pytest-randomly без seed.** Flaky порядок зависимостей в плохо изолированных тестах.

**allure step без plugin.** Декоратор молча игнорируется без установки.

**Версия plugin vs pytest.** Major pytest ломает старые плагины на collection.

**Дубли junit.** `pytest-html` и `--junitxml` оба в CI без нужды.
