# pytest-cov and what coverage means

## In short

`pytest-cov` wraps `coverage.py` and shows which lines executed during tests. Line coverage is not quality: you can call a function without checking branches and edge cases. I use coverage as a gap map, not a KPI of "100% or fail".

## How to answer in an interview

- `pytest --cov=app --cov-report=term-missing` in CI.
- `--cov-fail-under=80` gate only on critical packages.
- Branch coverage (`--cov-branch`) costs more but catches uncovered if paths.
- omit in `.coveragerc` for generated code and `if TYPE_CHECKING`.
- Honestly: 100% line coverage is possible with meaningless asserts; the metric misleads.

## How it works

```bash
pip install pytest-cov
pytest tests --cov=app --cov-report=html
```

Module and test:

```python
def discount(price: int, vip: bool) -> int:
    if vip:
        return price // 2
    return price


def test_discount_regular():
    assert discount(100, False) == 100
```

The report shows the uncovered `vip=True` branch although the function was called.

## Pitfalls

**xdist without combine.** Percents are understated per worker.

**Executed, not verified.** A call without assert still counts as green coverage.

**Dynamic import.** Lazy module lines miss the report.

**pragma no cover.** Abuse hides real dead code.

**Unit-only cov.** E2E in another runner does not enter the same report.
