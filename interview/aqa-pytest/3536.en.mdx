# Registering custom markers

## In short

Declare custom markers in the `[pytest]` section of `pytest.ini`, `pyproject.toml` (`[tool.pytest.ini_options]`), `tox.ini`, or `setup.cfg`. The `markers =` line lists a name and description so PyTest does not warn on unknown tags and `pytest --markers` shows team documentation.

## How to answer in an interview

- `markers = smoke: fast gate tests` in pytest.ini.
- `--strict-markers` in CI catches typos in decorators.
- Text after the colon appears in help, not at runtime.
- I prefer pyproject.toml in new repos over legacy setup.cfg.
- Honestly: without registration the marker still works, but warnings pollute junit.

## How it works

pytest.ini:

```ini
[pytest]
markers =
    smoke: critical path checks
    slow: long running scenarios
    api: http contract tests
```

Usage:

```python
import pytest


@pytest.mark.api
def test_users_contract():
    assert fetch_users().status_code == 200
```

Run:

```bash
pytest --strict-markers -m api
```

## Pitfalls

**Indentation in ini.** Marker continuation lines need indentation or parse errors follow.

**Two config files.** pytest reads one canonical file; duplicates in tox.ini confuse.

**Renaming a marker.** Old CI branches with `-m oldname` select 0 tests.

**Built-in markers.** `skip` and `xfail` need no registration.

**Description with a comma.** Rare parse edge cases; keep text simple.
