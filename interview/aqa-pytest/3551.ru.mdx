# Переопределение fixture

## Суть

Fixture с тем же именем в более узкой области перекрывает родительскую: conftest ближе к тесту побеждает корневой. Переопределение делают в `tests/integration/conftest.py` для другого API URL или mock backend. PyTest выбирает nearest fixture по дереву каталогов, не по порядку импорта.

## Что ответить на собесе

- Корневой `conftest.py` даёт `client` prod-like.
- `tests/unit/conftest.py` подменяет `client` на InMemory.
- `@pytest.fixture(name="client")` явно фиксирует имя при alias функции.
- `usefixtures` не override, только подключение.
- Честно: override без комментария в conftest ломает соседний модуль молча.

## Как это работает

Корень:

```python
import pytest


@pytest.fixture
def api_url():
    return "https://staging.example"
```

Пакет unit:

```python
import pytest


@pytest.fixture
def api_url():
    return "http://localhost:8000"
```

Тест в `tests/unit/` получит localhost.

## Подводные камни

**Два conftest одного уровня.** Невозможно, но duplicate name fixture в одном файле ошибка.

**import fixture из другого модуля.** Override не сработает, нужен conftest scope.

**final fixture plugin.** Некоторые плагины помечают fixture непереопределяемой.

**session override function.** Scope mismatch даёт unexpected sharing.

**Copy paste имени с typo.** Тест тихо берёт parent fixture.
