# Маркеры и выборочный запуск

## Суть

Markers это теги на тестах (`@pytest.mark.smoke`) для группировки и фильтрации прогона. CLI `-m` принимает boolean выражение над именами маркеров. В CI smoke suite гоняет быстрый поднабор, regression ночью берёт полный набор с `not slow`.

## Что ответить на собесе

- `@pytest.mark.api` на контрактных тестах, `pytest -m api`.
- `-m "smoke and not ui"` комбинирует условия.
- Незарегистрированный marker даёт warning, в strict mode error.
- `pytest -m ""` пустое выражение трактуется как все тесты.
- Честно: слишком много overlapping markers путает, лучше 5-7 канонических.

## Как это работает

Разметка и запуск:

```python
import pytest


@pytest.mark.smoke
def test_health():
    assert ping()


@pytest.mark.regression
@pytest.mark.slow
def test_bulk_import():
    assert import_rows(10_000)
```

```bash
pytest -m smoke
pytest -m "smoke or regression"
pytest -m "regression and not slow"
```

## Подводные камни

**Typo в -m.** `pytest -m smoe` молча даст 0 selected без strict.

**Marker на module.** Нужен `pytestmark = pytest.mark.api` вверху файла.

**skip mark путают с filter.** `@pytest.mark.skip` не равно custom tag.

**xdist + markers.** Worker всё равно collect весь tree, filter после.

**Дубли marker имени.** `pytest.mark.api` и `pytest.mark.API` разные строки.
