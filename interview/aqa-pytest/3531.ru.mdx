# Fixture в PyTest

## Суть

Fixture это функция с `@pytest.fixture`, которая готовит данные или окружение для теста и при необходимости очищает после. Тест запрашивает fixture по имени параметра, PyTest внедряет значение и управляет жизненным циклом по scope. `yield` разделяет setup и teardown в одной функции.

## Что ответить на собесе

- `@pytest.fixture` + аргумент `api_client` в тесте связывает по имени.
- `scope="session"` для дорогого browser, `function` для изоляции state.
- `yield` после setup: код ниже выполняется как teardown.
- `conftest.py` делает fixture видимой пакету без импорта.
- Честно: fixture без teardown при падении теста всё равно отработает yield после finally PyTest.

## Как это работает

Setup/teardown через yield:

```python
import pytest


@pytest.fixture
def sample_payload():
    payload = {"user": "qa", "role": "admin"}
    yield payload
    payload.clear()


def test_role(sample_payload):
    assert sample_payload["role"] == "admin"
```

## Подводные камни

**Имя параметра ≠ fixture.** Typos дают «fixture not found», не AttributeError в тесте.

**Mutable shared session fixture.** Параллельный pytest-xdist ломает shared dict.

**Fixture dependency cycle.** A→B→A даёт error на collection.

**return вместо yield.** Нет teardown, браузер остаётся открыт.

**autouse без контроля.** Скрытый setup усложняет debug падающего теста.
