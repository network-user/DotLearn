# JSON-массив: синтаксис и типичные ошибки при тестировании API

## Суть

JSON-массив записывается в квадратных скобках, элементы через запятую: `["a", 1, true, null, {}]`. Индексация с нуля при парсинге в код. QA проверяет trailing comma rejection, nested arrays и schema minItems/maxItems.

## Что ответить на собесе

- Синтаксис: `[ elem, elem ]`; пустой массив `[]` валиден.
- Элементы любого JSON-типа, в том числе объекты и вложенные массивы.
- Вложенный массив в object: `{ "tags": ["qa", "api"] }`.
- Invalid JSON: trailing comma, single quotes, unquoted keys внутри array of objects still need quoted keys.
- Честно: Postman prettify скрывает invalid JSON; отправляю raw body и смотрю 400 parser error.

## Как это работает

```json
{
  "results": [
    { "id": "r1", "score": 98 },
    { "id": "r2", "score": 87 }
  ],
  "meta": { "page": 1 }
}
```

Gherkin:

```gherkin
Feature: JSON array syntax

  Scenario: Valid empty array
    When POST /api/clear body {"ids": []}
    Then статус 200
    And affected count 0

  Scenario: Trailing comma invalid
    When raw body [1, 2, ]
    Then parser rejects before request reaches business logic

  Scenario: Index access after parse
    Given response array length 3
    Then element at index 0 exists
    And index 3 out of range handled per client spec
```

RFC 8259: array grammar `[ value *("," value) ]`. В JavaScript `JSON.parse` строгий; Python `json.loads` аналогично.

## Подводные камни

- **Trailing comma from JS export.** API 400 на production, worked in browser console.
- **Mixed quote styles.** Invalid strict JSON.
- **Array serialized as string.** `"[1,2]"` double encoding.
- **maxItems not enforced.** DoS huge payload.
- **Order assumed stable without sort param.** Flaky list comparison.
