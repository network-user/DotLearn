# Key-value model: where does QA see it in HTTP and JSON?

## In short

Key-value links a unique identifier to data: fast lookup without scanning a list. In QA this is JSON bodies, query strings, HTTP headers, env vars, and cookie attributes. Understanding the format helps build parametrized tests and catch encoding bugs.

## How to answer in an interview

- JSON object: string keys, any value types; I validate against schema.
- Query params: `?filter=active&page=2`; I check URL encoding of special chars.
- Headers: `Content-Type: application/json`; names are case-insensitive in HTTP/2.
- Cookie: `session=abc; Path=/; HttpOnly` is also attribute pairs.
- Honestly: duplicate query keys (legacy servers) behave non-standardly; I record actual behavior.

## How it works

Examples from one GET:

```http
GET /search?query=manual+testing&sort=asc HTTP/1.1
Host: api.example.com
Accept: application/json
Authorization: Bearer eyJ...
```

JSON response:

```json
{
  "total": 12,
  "items": [{ "id": "q-1", "title": "RBAC basics" }]
}
```

Gherkin:

```gherkin
Feature: Key-value across HTTP layers

  Scenario: Query param passes filter
    When GET /search?query=RBAC
    Then each item in items contains "RBAC" in title or tags

  Scenario: Content-Type header affects parsing
    When POST /api/users with body without Content-Type application/json
    Then server returns 415 or correct fallback per contract

  Scenario: URL-encoded space in query
    When GET /search?query=manual%20testing
    Then backend decodes value as "manual testing"
```

In DevTools → Network each tab (Headers, Query, Payload) shows pairs. Postman collection variables are also a KV store for reuse.

## Pitfalls

- **Plus vs space in query.** `+` and `%20` decode differently in old libs.
- **Duplicate JSON keys.** Last wins; parser-dependent.
- **Header injection via CRLF in value.** Old proxies are vulnerable.
- **Case sensitivity: JSON keys vs headers.** JSON keys are case-sensitive.
- **Nested KV confused with flat env.** `DB__HOST` vs `DB.HOST` in config.
