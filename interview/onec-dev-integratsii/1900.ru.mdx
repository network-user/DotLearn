# HTTP-сервисы 1С: маршруты, форматы и ответы

## Суть

HTTP-сервис публикуется в веб-сервере 1С с корневым URL и шаблонами путей. Обработчик возвращает код, заголовки и тело (JSON, XML, текст). Входные параметры берут из сегментов URL, query string и тела запроса.

## Что ответить на собесе

- Я настраиваю публикацию в администрировании кластера и корневой URL сервиса.
- Шаблон `/orders/{id}` передаёт `id` в обработчик как параметр.
- JSON отдаю с `Content-Type: application/json` и UTF-8.
- Ошибки возвращаю с 400/404/500 и структурированным телом.
- Честно: без HTTPS и аутентификации сервис не вывожу в интернет.

## Как это работает

```bsl
Функция СоздатьЗаказ(Запрос)
    Тело = Запрос.ПолучитьТелоКакСтроку();
    Данные = ПрочитатьJSONВСтруктуру(Тело);
    Ссылка = СоздатьЗаказВБазе(Данные);
    Ответ = Новый HTTPСервисОтвет(201);
    Ответ.УстановитьТелоИзСтроки(ОтветJSON(Ссылка));
    Возврат Ответ;
КонецФункции
```

Версионирование: префикс `/api/v1/` в шаблонах.

## Подводные камни

**Нет публикации на веб-сервере.** 404 снаружи при рабочем коде.
**Один обработчик на все методы.** Сложная ветвление по `Запрос.HTTPМетод`.
**Кириллица без UTF-8.** Битые ответы клиенту.
**Долгая работа в обработчике.** Таймаут reverse proxy.
**Утечка стека в теле 500.** Информация для атакующего.
