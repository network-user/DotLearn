# Authentication when calling external API from 1C

## In short

Common schemes: API key header, OAuth2 bearer token, basic auth over TLS. Tokens refresh before expiry; secrets live in secure storage or role-restricted constants.

## How to answer in an interview

- I do not hardcode passwords; I use `БезопасноеХранилище` or scheduled refresh.
- Bearer goes in `Authorization`; API key in a separate header per contract.
- On 401 I do one refresh and retry, not an infinite loop.
- All calls over HTTPS with certificate validation.
- I am honest: client credentials in a common module without role guard are visible to all config developers.

## How it works

```bsl
Токен = ПолучитьТокенИзХранилища();
Запрос.Заголовки.Вставить("Authorization", "Bearer " + Токен);
Ответ = Соединение.ВызватьHTTPМетод("POST", Запрос);
Если Ответ.КодСостояния = 401 Тогда
    ОбновитьТокен();
    Запрос.Заголовки.Вставить("Authorization", "Bearer " + ПолучитьТокенИзХранилища());
    Ответ = Соединение.ВызватьHTTPМетод("POST", Запрос);
КонецЕсли;
```

## Pitfalls

**Token in log.** Leak during debug.
**HTTP without TLS.** Key interception.
**Infinite 401 retry.** Storm on auth server.
**Expired refresh.** Manual secret reissue needed.
**Secret in exchange text.** Ends up in configuration dump.
