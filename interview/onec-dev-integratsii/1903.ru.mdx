# Аутентификация при вызове внешнего API из 1С

## Суть

Типовые схемы: API-ключ в заголовке, OAuth2 bearer token, basic auth по TLS. Токены обновляют по refresh до истечения; секреты хранят в защищённом хранилище или константах с ограничением прав.

## Что ответить на собесе

- Я не хардкожу пароли; использую `БезопасноеХранилище` или регламент обновления.
- Bearer токен кладу в `Authorization`, ключ API в отдельный заголовок по контракту.
- При 401 делаю один refresh и повтор запроса, не бесконечный цикл.
- Все вызовы только по HTTPS с проверкой сертификата.
- Честно: client credentials в общем модуле без ролевого доступа видны всем разработчикам конфигурации.

## Как это работает

```bsl
Токен = ПолучитьТокенИзХранилища();
Запрос.Заголовки.Вставить("Authorization", "Bearer " + Токен);
Ответ = Соединение.ВызватьHTTPМетод("POST", Запрос);
Если Ответ.КодСостояния = 401 Тогда
    ОбновитьТокен();
    Запрос.Заголовки.Вставить("Authorization", "Bearer " + ПолучитьТокенИзХранилища());
    Ответ = Соединение.ВызватьHTTPМетод("POST", Запрос);
КонецЕсли;
```

## Подводные камни

**Токен в логе.** Утечка при отладке.
**HTTP без TLS.** Перехват ключей.
**Бесконечный retry на 401.** Шторм на auth-сервер.
**Истёкший refresh.** Нужен ручной перевыпуск секретов.
**Секрет в тексте обмена.** Попадает в выгрузку конфигурации.
