# Integrating 1C with external APIs: practical approach

## In short

Outbound calls use `HTTPСоединение` or `HTTPЗапрос`: build URL, headers, JSON body, handle status and timeouts. The API contract lives in an integration common module with retries and logging.

## How to answer in an interview

- I keep calls in a server common module, not from the client.
- I store keys and URLs in constants or secure storage, not in code.
- I handle 401/429/5xx: token refresh, backoff, alert.
- I log correlation id and error body without secrets.
- I am honest: synchronous call in `ПередЗаписью` blocks posting when the API is down.

## How it works

```bsl
Соединение = Новый HTTPСоединение(Хост, 443, , , , 30, Новый ЗащищенноеСоединениеOpenSSL);
Запрос = Новый HTTPЗапрос("/v1/items");
Запрос.Заголовки.Вставить("Authorization", "Bearer " + Токен);
Ответ = Соединение.Получить(Запрос);
Если Ответ.КодСостояния <> 200 Тогда
    ВызватьИсключение "API error: " + Ответ.КодСостояния;
КонецЕсли;
```

JSON via `ЧтениеJSON` into a structure or XDTO by schema.

## Pitfalls

**Secrets in repo.** Key leak on commit.
**No timeout.** Hung session blocks transaction.
**Ignoring pagination.** Incomplete catalog load.
**Write without idempotency.** Retry duplicates order.
**Parse without schema check.** Crash on unexpected API field.
