# Интеграция 1С со сторонними API: практический подход

## Суть

Исходящие вызовы выполняют через `HTTPСоединение` или `HTTPЗапрос`: формируют URL, заголовки, тело JSON, обрабатывают код ответа и таймауты. Контракт API фиксируют в общем модуле интеграции с повторными попытками и журналированием.

## Что ответить на собесе

- Я выношу вызовы в серверный общий модуль, не с клиента.
- Храню ключи и URL в константах или защищённом хранилище, не в коде.
- Обрабатываю 401/429/5xx: обновление токена, backoff, алерт.
- Логирую correlation id и тело ошибки без секретов.
- Честно: синхронный вызов в `ПередЗаписью` блокирует проведение при падении API.

## Как это работает

```bsl
Соединение = Новый HTTPСоединение(Хост, 443, , , , 30, Новый ЗащищенноеСоединениеOpenSSL);
Запрос = Новый HTTPЗапрос("/v1/items");
Запрос.Заголовки.Вставить("Authorization", "Bearer " + Токен);
Ответ = Соединение.Получить(Запрос);
Если Ответ.КодСостояния <> 200 Тогда
    ВызватьИсключение "API error: " + Ответ.КодСостояния;
КонецЕсли;
```

Десериализация JSON через `ЧтениеJSON` в структуру или XDTO по схеме.

## Подводные камни

**Секреты в репозитории.** Утечка ключей при коммите.
**Нет таймаута.** Зависший сеанс блокирует транзакцию.
**Игнор пагинации.** Неполная загрузка каталога.
**Запись без идемпотентности.** Повтор отправки дублирует заказ.
**Парсинг без проверки схемы.** Падение на неожиданном поле API.
