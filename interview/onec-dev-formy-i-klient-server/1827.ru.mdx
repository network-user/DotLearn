# Менеджер форм: открытие и параметры

## Суть

Менеджер форм это программный API платформы для получения экземпляра формы по имени метаданных, передачи параметров и выбора режима открытия (модально, владелец, уникальность ключа). В управляемом приложении основной вызов: `ОткрытьФорму` и `ПолучитьФорму`.

## Что ответить на собесе

- Я открываю формы через `ОткрытьФорму("Документ.Реализация.ФормаДокумента", ПараметрыФормы)`.
- На собеседовании объясняю `Параметры`, `ОписаниеОповещенияОЗакрытии`, `Уникальность` для повторного использования окна.
- Подчёркиваю разницу между формой объекта и общей формой без привязки к ссылке.
- Честно уточняю: хранить ссылку на форму в глобальной переменной долго не стоит, возможны утечки и закрытые окна.
- Для блокирующего ввода использую модальное открытие или `ПоказатьЗначение`.

## Как это работает

```bsl
&НаКлиенте
Процедура ОткрытьКарточкуНоменклатуры(СсылкаНоменклатуры)
    ПараметрыФормы = Новый Структура("Ключ", СсылкаНоменклатуры);
    ОткрытьФорму(
        "Справочник.Номенклатура.ФормаЭлемента",
        ПараметрыФормы,
        ЭтотОбъект,
        СсылкаНоменклатуры.УникальныйИдентификатор(),
        ,
        ,
        ,
        РежимОткрытияОкнаФормы.БлокироватьОкноВладельца);
КонецПроцедуры
```

`Уникальность` по UUID ссылки не плодит десять карточек одного элемента. Оповещение о закрытии возвращает результат подбора в форму-владелец.

```bsl
&НаКлиенте
Процедура ПослеВыбораКонтрагента(Результат, ДопПараметры) Экспорт
    Если Результат <> Неопределено Тогда
        Объект.Контрагент = Результат;
    КонецЕсли;
КонецПроцедуры
```

## Подводные камни

- **Неверное имя формы.** Строка пути должна совпадать с метаданными, иначе ошибка времени выполнения.
- **Параметры не объявлены.** В `ПриСозданииНаСервере` обращение к несуществующему ключу.
- **Модальность в веб-клиенте.** Поведение блокировки владельца отличается от тонкого клиента.
- **Утечка через замыкание.** Долгоживущий объект держит ссылку на закрытую форму.
- **Повторное открытие списка.** Без уникальности пользователь теряется в одинаковых окнах.
