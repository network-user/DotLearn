# Form manager: opening and parameters

## In short

The form manager is the platform API to get a form instance by metadata name, pass parameters, and choose open mode (modal, owner, key uniqueness). In managed application the main calls are `ОткрытьФорму` and `ПолучитьФорму`.

## How to answer in an interview

- I open forms via `ОткрытьФорму("Документ.Реализация.ФормаДокумента", ПараметрыФормы)`.
- In an interview I explain `Параметры`, `ОписаниеОповещенияОЗакрытии`, `Уникальность` for window reuse.
- I stress the difference between an object form and a common form without a reference binding.
- I honestly note keeping a form reference in a global long-lived variable risks leaks and closed windows.
- For blocking input I use modal open or `ПоказатьЗначение`.

## How it works

```bsl
&НаКлиенте
Процедура ОткрытьКарточкуНоменклатуры(СсылкаНоменклатуры)
    ПараметрыФормы = Новый Структура("Ключ", СсылкаНоменклатуры);
    ОткрытьФорму(
        "Справочник.Номенклатура.ФормаЭлемента",
        ПараметрыФормы,
        ЭтотОбъект,
        СсылкаНоменклатуры.УникальныйИдентификатор(),
        ,
        ,
        ,
        РежимОткрытияОкнаФормы.БлокироватьОкноВладельца);
КонецПроцедуры
```

`Уникальность` by reference UUID avoids ten cards for one item. Close notification returns picker result to the owner form.

```bsl
&НаКлиенте
Процедура ПослеВыбораКонтрагента(Результат, ДопПараметры) Экспорт
    Если Результат <> Неопределено Тогда
        Объект.Контрагент = Результат;
    КонецЕсли;
КонецПроцедуры
```

## Pitfalls

- **Wrong form name.** Path string must match metadata or runtime error.
- **Undeclared parameters.** Accessing a missing key in `ПриСозданииНаСервере`.
- **Modality in web client.** Owner blocking differs from thin client.
- **Leak via closure.** Long-lived object holds a closed form reference.
- **Repeated list open.** Without uniqueness users get lost in identical windows.
