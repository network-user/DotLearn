# Директивы &НаКлиенте, &НаСервере и &НаСервереБезКонтекста

## Суть

Директивы компиляции указывают, на какой стороне выполняется процедура модуля формы или общего модуля. `&НаКлиенте` работает с UI, `&НаСервере` имеет доступ к контексту формы и ИБ, `&НаСервереБезКонтекста` выполняется на сервере изолированно, без привязки к данным формы.

## Что ответить на собесе

- Я держу тонкий клиент: UI и валидация ввода на клиенте, запросы и запись на сервере.
- На собеседовании рисую цепочку: клиентский обработчик кнопки вызывает серверную функцию с параметрами.
- Подчёркиваю, что серверный код в модуле формы не попадает в клиентский бинарник при корректной разметке.
- Честно уточняю: лишние `&НаСервере` в цикле дают чрезмерные серверные вызовы.
- `&НаСервереБезКонтекста` ставлю для переиспользуемых функций без доступа к `Объект` формы.

## Как это работает

Клиент показывает сообщение:

```bsl
&НаКлиенте
Процедура ПоказатьПодсказку(Команда)
    ПоказатьПредупреждение(, "Действие выполнено на клиенте");
КонецПроцедуры
```

Сервер читает данные:

```bsl
&НаСервере
Функция ПолучитьОстатокНоменклатуры(Номенклатура)
    Запрос = Новый Запрос;
    Запрос.Текст =
    "ВЫБРАТЬ СУММА(Остаток) КАК Остаток
    |ИЗ РегистрНакопления.ТоварыНаСкладах.Остатки
    |ГДЕ Номенклатура = &Номенклатура";
    Запрос.УстановитьПараметр("Номенклатура", Номенклатура);
    Выборка = Запрос.Выполнить().Выбрать();
    Возврат ?(Выборка.Следующий(), Выборка.Остаток, 0);
КонецФункции
```

Без контекста для утилиты:

```bsl
&НаСервереБезКонтекста
Функция НормализоватьСтроку(ИсходнаяСтрока)
    Возврат СокрЛП(ВРег(ИсходнаяСтрока));
КонецФункции
```

## Подводные камни

- **Запросы на клиенте.** Прямой запрос в `&НаКлиенте` недоступен; нужен серверный вызов.
- **Передача больших ТЗ.** Каждый вызов тащит сериализацию; пакетируйте данные.
- **Смешение контекстов.** Вызов `&НаСервере` из `&НаСервереБезКонтекста` без параметров формы не видит `Объект`.
- **Веб-клиент.** Часть клиентских API недоступна; проверяйте совместимость.
- **Отладка.** Точка останова в серверном методе требует подключения к серверному процессу.
