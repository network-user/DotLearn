# Why form references must not be passed in parameters

## In short

A form is a client UI object with a limited lifetime. Passing a form reference in structures, common modules, or to the server breaks client-server boundaries, keeps closed windows in memory, and fails after close or reopen.

## How to answer in an interview

- I pass data (infobase references, structures, primitives), not a `–§–Њ—А–Љ–∞–Ъ–ї–Є–µ–љ—В—Б–Ї–Њ–≥–Њ–Я—А–Є–ї–Њ–ґ–µ–љ–Є—П` instance.
- In an interview I show `–Ю–њ–Є—Б–∞–љ–Є–µ–Ю–њ–Њ–≤–µ—Й–µ–љ–Є—П–Ю–Ч–∞–Ї—А—Л—В–Є–Є` instead of keeping a child form reference.
- I stress the server cannot accept a parameter of type "form".
- I honestly note a module global holding a form is the same anti-pattern.
- For owner access I use `–≠—В–Њ—В–Ю–±—К–µ–Ї—В` in the form module, not an external cache.

## How it works

Correct: pass a key and get the result via notification.

```bsl
&–Э–∞–Ъ–ї–Є–µ–љ—В–µ
–Я—А–Њ—Ж–µ–і—Г—А–∞ –Т—Л–±—А–∞—В—М–Ъ–Њ–љ—В—А–∞–≥–µ–љ—В–∞(–Ъ–Њ–Љ–∞–љ–і–∞)
    –Ю–њ–Њ–≤–µ—Й–µ–љ–Є–µ = –Э–Њ–≤—Л–є –Ю–њ–Є—Б–∞–љ–Є–µ–Ю–њ–Њ–≤–µ—Й–µ–љ–Є—П("–Я–Њ—Б–ї–µ–Т—Л–±–Њ—А–∞", –≠—В–Њ—В–Ю–±—К–µ–Ї—В);
    –Ю—В–Ї—А—Л—В—М–§–Њ—А–Љ—Г("–°–њ—А–∞–≤–Њ—З–љ–Є–Ї.–Ъ–Њ–љ—В—А–∞–≥–µ–љ—В—Л.–§–Њ—А–Љ–∞–Т—Л–±–Њ—А–∞", , , , , , –Ю–њ–Њ–≤–µ—Й–µ–љ–Є–µ);
–Ъ–Њ–љ–µ—Ж–Я—А–Њ—Ж–µ–і—Г—А—Л
```

Wrong: store the form in a processor attribute.

```bsl
–Я—А–Њ—Ж–µ–і—Г—А–∞ –Ч–∞–њ–Њ–Љ–љ–Є—В—М–§–Њ—А–Љ—Г(–§–Њ—А–Љ–∞–Т—Л–±–Њ—А–∞)
    –°–Њ—Е—А–∞–љ–µ–љ–љ–∞—П–§–Њ—А–Љ–∞ = –§–Њ—А–Љ–∞–Т—Л–±–Њ—А–∞;
–Ъ–Њ–љ–µ—Ж–Я—А–Њ—Ж–µ–і—Г—А—Л
```

After close `–°–Њ—Е—А–∞–љ–µ–љ–љ–∞—П–§–Њ—А–Љ–∞` points to a destroyed object; accessing `–≠–ї–µ–Љ–µ–љ—В—Л` errors or hangs. The platform serializes parameters on client-server calls; client references are not transferable.

## Pitfalls

- **Memory leak.** Object chains keep forms the user already closed.
- **Circular references.** Form A holds B, B holds A; memory is not released.
- **Async timing.** User closed the window while background code still calls `–°–Њ—Е—А–∞–љ–µ–љ–љ–∞—П–§–Њ—А–Љ–∞`.
- **Server call.** Passing a form to `&–Э–∞–°–µ—А–≤–µ—А–µ` fails with a type error.
- **Testability.** Code tied to a live form is not unit-testable.
