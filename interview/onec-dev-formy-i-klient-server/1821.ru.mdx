# Почему &НаСервереБезКонтекста не видит данные формы

## Суть

Процедура с `&НаСервереБезКонтекста` выполняется на сервере вне контекста конкретной формы и сеансового состояния вызова. Поэтому недоступны `Объект`, `Элементы`, реквизиты формы и клиентские переменные: они существуют только в связанном контексте формы на клиенте или в серверном контексте `&НаСервере`.

## Что ответить на собесе

- Я передаю нужные значения явными параметрами при вызове из клиента или `&НаСервере`.
- На собеседовании объясняю, что без контекста вызов дешевле и безопаснее для утилит и фоновой логики.
- Подчёркиваю ошибку при обращении к `Элементы.Поле.Значение` внутри безконтекстного метода.
- Честно уточняю: для работы с реквизитами формы нужен `&НаСервере`, не без контекста.
- Общие модули с `Сервер`, `Вызов сервера` похожи по изоляции, но правила вызова разные.

## Как это работает

```bsl
&НаСервереБезКонтекста
Функция УдвоитьЧисло(ВходноеЧисло)
    Возврат ВходноеЧисло * 2;
КонецФункции

&НаКлиенте
Процедура Рассчитать(Команда)
    Результат = УдвоитьЧисло(Число(Элементы.ПолеВвода.Значение));
    Элементы.ПолеРезультат.Значение = Результат;
КонецПроцедуры
```

Клиент читает UI, передаёт число на сервер, получает результат. Безконтекстная функция не может сама открыть `Элементы.ПолеВвода`: форма живёт на клиенте, контекст вызова не передан.

## Подводные камни

- **Скрытый доступ к форме.** Попытка использовать глобальную переменную формы из безконтекстного модуля даёт `Неопределено` или ошибку.
- **Лишние параметры.** Передача всего `Объект` в параметр тащит большой пакет и нивелирует выгоду без контекста.
- **Права.** Безконтекстный код всё равно подчиняется RLS; привилегированный режим включают осознанно.
- **Повторное использование.** Вынесли функцию в общий модуль, забыли убрать обращение к `Объект`.
- **Отладка.** Стек показывает безконтекстный кадр без локальных реквизитов формы.
