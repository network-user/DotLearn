# OnCreateAtServer vs OnOpen: difference and usage

## In short

`ПриСозданииНаСервере` runs once when the form is instantiated on the server: load object, filters, rights. `ПриОткрытии` runs on the client when the form is shown: title, element visibility, focus. Different contexts and roles.

## How to answer in an interview

- I put infobase queries and `Объект` fill in `ПриСозданииНаСервере`, UI polish in client `ПриОткрытии`.
- In an interview I stress order: server creation first, then client open.
- I note `ПриОткрытии` may run again on reshown forms without recreation.
- I honestly say heavy logic in `ПриОткрытии` via server call belongs in creation instead.
- Parameter `Отказ` in both handlers blocks open on init failure.

## How it works

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если Не ЗначениеЗаполнено(Параметры.Контрагент) Тогда
        Отказ = Истина;
        Возврат;
    КонецЕсли;
    Объект.Контрагент = Параметры.Контрагент;
    ЗаполнитьТабличнуюЧастьПоОстаткам();
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Заголовок = "Заказ для " + Строка(Объект.Контрагент);
    Элементы.СтраницаДоставка.Видимость = ИспользоватьДоставку;
КонецПроцедуры
```

The server prepares data before the window appears; the client adjusts presentation without an extra SCALL unless data is fetched again.

## Pitfalls

- **Duplicate load.** Same data loaded on create and again on open via server.
- **Client query.** Direct query in `ПриОткрытии` is not allowed.
- **Reject without message.** `Отказ = Истина` without `Сообщить` confuses users.
- **Parameter dependency.** Creation without `Параметры` check yields an empty form.
- **Repeated OnOpen.** One-time logic wrongly placed in `ПриОткрытии`.
