# Как сократить клиент-серверные вызовы в формах

## Суть

Каждый переход `&НаКлиенте` → `&НаСервере` это сетевой round-trip с сериализацией данных. Оптимизация сводится к пакетным серверным методам, клиентскому кэшу, отказу от серверных вызовов в циклах и передаче только нужных полей.

## Что ответить на собесе

- Я ищу в замере технологического журнала частые `SCALL` с малым временем: это типичный «чаттер».
- На собеседовании показываю один серверный метод, возвращающий структуру с несколькими результатами.
- Подчёркиваю кэш справочников на клиенте до конца сеанса, если данные редко меняются.
- Честно уточняю: агрессивный кэш даёт устаревшие остатки; нужна инвалидация по событию.
- Валидацию формата поля делаю на клиенте, бизнес-правила на сервере одним вызовом.

## Как это работает

Плохо: сервер в каждой строке таблицы.

```bsl
&НаКлиенте
Процедура ПересчитатьСтроки()
    Для Каждого ИдентификаторСтроки Из ИдентификаторыСтрок Цикл
        ОбновитьЦенуСтрокиНаСервере(ИдентификаторСтроки);
    КонецЦикла;
КонецПроцедуры
```

Лучше: один вызов с массивом.

```bsl
&НаСервере
Процедура ОбновитьЦеныПакетом(ТаблицаСтрок)
    Для Каждого СтрокаТаблицы Из ТаблицаСтрок Цикл
        СтрокаТаблицы.Цена = ПолучитьЦену(СтрокаТаблицы.Номенклатура);
    КонецЦикла;
КонецПроцедуры
```

Клиентский кэш:

```bsl
&НаКлиенте
Перем КэшВидовНоменклатуры;

Функция ВидыНоменклатуры()
    Если КэшВидовНоменклатуры = Неопределено Тогда
        КэшВидовНоменклатуры = ЗагрузитьВидыНаСервере();
    КонецЕсли;
    Возврат КэшВидовНоменклатуры;
КонецФункции
```

## Подводные камни

- **Ложная экономия.** Один гигантский вызов с полной ТЗ документа блокирует UI дольше round-trip.
- **Устаревший кэш.** Пользователь не видит свежие остатки после проведения в другом окне.
- **Сервер в `ПриИзменении`.** Каждый символ в поле дергает сервер; используйте отложенный пересчёт.
- **Веб-клиент.** Latency выше тонкого клиента; оптимизация критичнее.
- **Без замера.** Оптимизируют наугад без профилирования и ТЖ.
