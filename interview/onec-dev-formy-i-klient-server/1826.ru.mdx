# Источник данных формы и связь с сервером

## Суть

Источник данных формы это реквизит или динамический список, к которому привязаны элементы UI. Основной объект формы (`Объект`, `Запись`, `Список`) живёт на сервере; при открытии и изменениях платформа синхронизирует серверное состояние с клиентским представлением через сериализацию.

## Что ответить на собесе

- Я различаю реквизит формы типа `ДокументОбъект` и динамический список с произвольным запросом.
- На собеседовании объясняю путь: серверная инициализация → передача снимка на клиент → обратная запись при `Записать`.
- Подчёркиваю, что табличная часть в `Объект` редактируется на клиенте, но запись идёт на сервере.
- Честно уточняю: лишние реквизиты формы раздувают каждый round-trip.
- Для списков настраиваю отбор на сервере в `ПриСозданииНаСервере`, не тяну всю ТЗ вручную.

## Как это работает

Форма элемента справочника: главный реквизит `Объект` типа `СправочникОбъект.Номенклатура`. Элементы полей привязаны к `Объект.Наименование`, `Объект.Артикул`.

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если ЗначениеЗаполнено(Параметры.Ключ) Тогда
        Объект = Параметры.Ключ.ПолучитьОбъект();
    Иначе
        Объект = Справочники.Номенклатура.СоздатьЭлемент();
    КонецЕсли;
КонецПроцедуры

&НаКлиенте
Процедура Сохранить(Команда)
    ЗаписатьНаСервере();
КонецПроцедуры
```

Динамический список на форме списка: источник `Список`, запрос выполняется на сервере, клиент получает страницы и отборы. Изменение отбора инициирует серверное обновление набора данных.

## Подводные камни

- **Рассинхрон.** Долгая работа на форме без перечитывания: на сервере данные уже изменились.
- **Тяжёлый Объект.** Табличная часть на тысячи строк тормозит открытие; используйте постраничность или отдельные формы.
- **Динамический список без индексов.** Отбор по неиндексированному полю бьёт по SQL.
- **Временные реквизиты.** Забытые крупные ТЗ в реквизитах формы уезжают на клиент целиком.
- **Запись без проверки.** Клиент отправил устаревший `Объект`, сервер перезаписал чужие изменения.
