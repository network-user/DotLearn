# ПриСозданииНаСервере и ПриОткрытии: разница и применение

## Суть

`ПриСозданииНаСервере` выполняется один раз при инстанцировании формы на сервере: загрузка объекта, отборы, права. `ПриОткрытии` срабатывает на клиенте при показе формы пользователю: заголовок, видимость элементов, фокус. Разные контексты и разное назначение.

## Что ответить на собесе

- Я кладу запросы к ИБ и заполнение `Объект` в `ПриСозданииНаСервере`, косметику UI в `ПриОткрытии` на клиенте.
- На собеседовании подчёркиваю порядок: сначала создание на сервере, потом открытие на клиенте.
- Говорю, что `ПриОткрытии` может вызываться повторно при повторном показе без пересоздания формы.
- Честно уточняю: тяжёлую логику в `ПриОткрытии` через серверный вызов лучше перенести в создание.
- Параметр `Отказ` в обоих обработчиках блокирует открытие при ошибке инициализации.

## Как это работает

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если Не ЗначениеЗаполнено(Параметры.Контрагент) Тогда
        Отказ = Истина;
        Возврат;
    КонецЕсли;
    Объект.Контрагент = Параметры.Контрагент;
    ЗаполнитьТабличнуюЧастьПоОстаткам();
КонецПроцедуры

&НаКлиенте
Процедура ПриОткрытии(Отказ)
    Заголовок = "Заказ для " + Строка(Объект.Контрагент);
    Элементы.СтраницаДоставка.Видимость = ИспользоватьДоставку;
КонецПроцедуры
```

Сервер готовит данные до появления окна; клиент настраивает представление без лишнего SCALL, если не тянуть данные повторно.

## Подводные камни

- **Дублирование загрузки.** Одни данные грузят и при создании, и при открытии через сервер.
- **Клиентский запрос.** В `ПриОткрытии` нельзя выполнить запрос напрямую.
- **Отказ без сообщения.** `Отказ = Истина` без `Сообщить` оставляет пользователя в недоумении.
- **Зависимость от параметров.** Создание без проверки `Параметры` даёт пустую форму.
- **Повторное ПриОткрытии.** Код, который должен выполниться один раз, ошибочно кладут в `ПриОткрытии`.
