# Какие HTTP-методы существуют и за что каждый отвечает?

## Суть

HTTP-метод (он же глагол) задаёт, какое действие клиент хочет выполнить над ресурсом сервера. GET читает, POST создаёт, PUT заменяет целиком, PATCH правит частично, DELETE удаляет. Часть методов безопасна и идемпотентна, и это напрямую влияет на то, как их можно повторять.

## Что ответить на собесе

«Основных глаголов несколько. GET читает данные и ничего не меняет — он безопасный и идемпотентный. POST отправляет данные и обычно создаёт ресурс, повторный вызов даёт новый объект, поэтому он не идемпотентен. PUT заменяет ресурс целиком и идемпотентен: сколько раз ни повтори, состояние одно. PATCH меняет только переданные поля. DELETE удаляет и тоже идемпотентен. Ещё есть HEAD — как GET, но без тела, и OPTIONS — узнать, какие методы поддерживает сервер.»

## Как это работает

Ключевые свойства методов: *безопасность* (запрос не меняет состояние) и *идемпотентность* (повтор даёт тот же результат, что один вызов).

- **GET** — читает ресурс. Безопасен и идемпотентен. Браузер этим методом тянет HTML страницы.
- **POST** — отправляет данные, обычно создаёт новый ресурс. Не идемпотентен: два POST — два объекта. Данные лежат в теле.
- **PUT** — кладёт ресурс по известному адресу, заменяя его целиком или создавая. Идемпотентен.
- **PATCH** — частичное обновление: трогает только указанные поля, остальное не задевает.
- **DELETE** — удаляет ресурс. Идемпотентен (повторное удаление того же объекта не меняет итог).
- **HEAD** — как GET, но возвращает только заголовки без тела. Удобен для проверки существования и метаданных.
- **OPTIONS** — сообщает список поддерживаемых методов, участвует в CORS-preflight.

Работа с методами на Python через `requests`:

```python
import requests

base = "https://api.example.com"

response = requests.get(f"{base}/data")
print(response.status_code, response.json())

created = requests.post(f"{base}/data", json={"key": "value"})
print(created.status_code)

updated = requests.put(f"{base}/data/1", json={"key": "new_value"})
print(updated.status_code)

patched = requests.patch(f"{base}/data/1", json={"key": "patch"})
print(patched.status_code)

deleted = requests.delete(f"{base}/data/1")
print(deleted.status_code)
```

Каждый вызов посылает соответствующий глагол; сервер по нему понимает намерение клиента.

## Подводные камни

- GET не должен менять состояние: данные в query-строке кешируются и логируются, прятать туда побочные эффекты опасно.
- Путать PUT и PATCH — частая ошибка: PUT затирает ресурс целиком, и забытое поле станет пустым, а PATCH правит точечно.
- Идемпотентность — про итоговое состояние, а не про код ответа: второй DELETE может вернуть 404, но состояние сервера уже не меняется.
- POST идемпотентным не является, поэтому ретраи без ключа идемпотентности легко плодят дубликаты.
- Не всякий сервер реально поддерживает все методы — перед PATCH/OPTIONS стоит свериться с документацией API.
