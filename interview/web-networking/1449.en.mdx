# URI vs URL: what the difference is and why it is not symmetric

## In short

A URI is the general term for any resource identifier. A URL is a subtype of it that not only names a resource but also says where it lives and how to retrieve it. That is why every URL is a URI, but not every URI is a URL.

## How to answer in an interview

A URI (Uniform Resource Identifier) is the umbrella concept: anything that uniquely identifies a resource. Within URIs there are two kinds - URL and URN. A URL (Locator) specifies the location and access method: the scheme, host, and path. A URN (Name) specifies a stable name for the resource without tying it to a location, for example `urn:isbn:0451450523`. The key idea: the relationship is asymmetric - a URL is always a URI, but a URI can also be a URN, that is, not a URL. In practice on the web, almost everything we see in the address bar is a URL.

A URI splits into two subcategories:

- **URL (Uniform Resource Locator):** indicates where the resource is and how to reach it. Example: `https://www.example.com/index.html` - a page available over HTTPS.
- **URN (Uniform Resource Name):** a unique name with no location specified. Example: `urn:isbn:0451450523` identifies a book by its ISBN but does not say where to download it.

A URL is made of parts:

- **Scheme:** the access protocol - `http`, `https`, `ftp`.
- **Host:** the server's domain or IP - `www.example.com`.
- **Port:** an optional port number - `:443`.
- **Path:** the path to the resource - `/path/to/resource`.
- **Query:** optional parameters - `?id=123`.
- **Fragment:** an optional anchor within the resource - `#section1`.

Let's break down `https://www.example.com:443/path/to/resource?id=123#section1`:

- `https` - the scheme (the HTTPS protocol).
- `www.example.com` - the host.
- `:443` - the port (the default for HTTPS).
- `/path/to/resource` - the path.
- `?id=123` - the query parameter `id` with the value `123`.
- `#section1` - the fragment, a reference to a section within the resource.

## Pitfalls

- **Not every URI is a URL.** A URN is also a URI, but it has neither a host nor an access method - only a name.
- **The fragment `#section1` is processed by the browser, not the server.** It is not sent to the server, so you cannot influence the server's response through the fragment.
- **The default port is implied by the scheme.** For `http` it is 80, for `https` it is 443; it is rarely written explicitly.
- **"A URI and a URL are the same thing" is an imprecise answer.** Technically a URL is narrower, and in an interview it matters to show that you understand the hierarchy URI → {URL, URN}.
