# Что такое cookie и какую роль они играют в работе сайтов?

## Суть

Cookie - небольшой фрагмент данных, который сервер просит браузер сохранить и затем прикладывать к каждому запросу на тот же сайт. Так HTTP, который сам по себе без состояния, получает "память": сессию, авторизацию и настройки пользователя.

## Что ответить на собесе

Cookie решают проблему stateless-природы HTTP. Сервер шлёт заголовок `Set-Cookie`, браузер его запоминает и потом автоматически отправляет обратно в заголовке `Cookie` при каждом обращении к этому домену. Главное применение - сессии и аутентификация: после логина в cookie кладут идентификатор сессии или токен, и сервер по нему узнаёт пользователя. Ещё их используют для персонализации (язык, тема) и аналитики или рекламы. У cookie есть атрибуты безопасности: `HttpOnly` прячет cookie от JavaScript, `Secure` шлёт только по HTTPS, `SameSite` ограничивает кросс-доменную отправку и защищает от CSRF.

## Как это работает

Зайдя на сайт, браузер может получить от сервера `Set-Cookie`. Он сохраняет значение и автоматически добавляет его к каждому следующему запросу к этому же домену - сервер таким образом узнаёт клиента и отдаёт ему персональный ответ.

```python
from http.cookies import SimpleCookie

cookie = SimpleCookie()
cookie["language"] = "ru"
cookie["language"]["path"] = "/"
cookie["language"]["max-age"] = 3600
cookie["language"]["httponly"] = True

print(cookie.output())
# Set-Cookie: language=ru; HttpOnly; Max-Age=3600; Path=/
```

Что задают атрибуты:

- `path=/` - cookie доступен на всём сайте, а не только в одной директории.
- `max-age=3600` - срок жизни в секундах (здесь час); без него cookie сессионный и умирает при закрытии браузера.
- `httponly` - cookie не виден из JavaScript, что снижает риск кражи через XSS.

## Подводные камни

- HTTP без cookie не помнит клиента между запросами. Cookie - именно способ дать ему состояние, а не просто "файл с настройками".
- Чувствительные cookie без `HttpOnly` уязвимы к краже через XSS, а без `Secure` могут утечь по открытому HTTP. Для сессий эти флаги обязательны.
- `SameSite` важен против CSRF. По умолчанию в современных браузерах он `Lax`; для кросс-доменной отправки нужен `None`, но только вместе с `Secure`.
- Размер cookie ограничен (примерно 4 КБ), и они едут в каждом запросе. Хранить в них много данных - значит раздувать трафик; для этого есть серверные сессии и токены.
- Cookie привязаны к домену и пути. Cookie с `client.com` не уедет на `example.com` - частое заблуждение.
