# Что означают группы HTTP-статус-кодов и какие из них ключевые?

## Суть

Статус-код HTTP — трёхзначное число, которым сервер сообщает клиенту итог обработки запроса. Первая цифра задаёт класс: 1xx — информация, 2xx — успех, 3xx — перенаправление, 4xx — ошибка клиента, 5xx — ошибка сервера. По коду клиент решает, что делать дальше.

## Что ответить на собесе

Коды делятся на пять групп по первой цифре, и главное отличие — 4xx чинит клиент, а 5xx чинят на бэкенде.

- 2xx — всё хорошо: 200 OK, 201 Created при создании ресурса, 204 No Content без тела.
- 3xx — нужен ещё шаг: 301 переехало навсегда, 304 не изменилось с прошлого раза.
- 4xx — виноват клиент: 400 кривой запрос, 401 нужна аутентификация, 403 нет прав, 404 ресурс не найден.
- 5xx — сломалось на сервере: 500 общая ошибка, 502 кривой ответ от апстрима, 503 сервис временно недоступен.

Разбор по группам:

- **1xx — информационные**: запрос принят, обработка продолжается. В прикладном коде почти не встречаются (`100 Continue`, `101 Switching Protocols`).
- **2xx — успех**: `200 OK` — запрос выполнен; `201 Created` — создан новый ресурс; `204 No Content` — успех без тела ответа.
- **3xx — перенаправление**: `301 Moved Permanently` — ресурс навсегда переехал; `302 Found` — временно по другому адресу; `304 Not Modified` — с прошлого раза не менялся, можно брать из кеша.
- **4xx — ошибка клиента**: `400 Bad Request` — некорректный запрос; `401 Unauthorized` — нужна аутентификация; `403 Forbidden` — прав нет; `404 Not Found` — ресурса нет.
- **5xx — ошибка сервера**: `500 Internal Server Error` — общий сбой; `502 Bad Gateway` — апстрим вернул мусор; `503 Service Unavailable` — сервер временно не тянет.

Простой пример обмена:

```http
GET /index.html HTTP/1.1
Host: www.example.com
```

Ответ сервера:

```http
HTTP/1.1 200 OK
Content-Type: text/html

<html>
  <head><title>Example Page</title></head>
  <body><h1>Welcome to Example.com!</h1></body>
</html>
```

Клиент попросил страницу, сервер ответил `200 OK` и отдал HTML.

## Подводные камни

- 401 и 403 путают: 401 — «кто ты, представься», 403 — «я знаю, кто ты, но тебе сюда нельзя».
- На бизнес-ошибки нельзя слепо отдавать 200 с `{"error": ...}` в теле — клиенты и мониторинг ориентируются на код, ломается обработка ошибок и ретраи.
- 5xx обычно безопасно ретраить (особенно 503 с `Retry-After`), а 4xx ретраить бессмысленно — запрос так и останется кривым.
- 304 работает только в паре с условными заголовками (`If-None-Match`/`ETag`); без них кеш не сработает.
- 302 для постоянного переезда вредит SEO и кешам — для «навсегда» нужен именно 301.
