# URI против URL: в чём разница и почему она не симметрична

## Суть

URI - это общий термин для любого идентификатора ресурса. URL - его подвид, который не просто называет ресурс, но и говорит, где он лежит и как его получить. Поэтому любой URL - это URI, но не каждый URI является URL.

## Что ответить на собесе

URI (Uniform Resource Identifier) - зонтичное понятие: всё, что однозначно идентифицирует ресурс. Внутри URI есть два вида - URL и URN. URL (Locator) задаёт расположение и способ доступа: схему, хост, путь. URN (Name) задаёт устойчивое имя ресурса без привязки к местоположению, например `urn:isbn:0451450523`. Главная мысль: отношение асимметрично - URL всегда URI, но URI может быть и URN, то есть не URL. На практике в вебе почти всё, что мы видим в адресной строке, - это URL.

URI делится на две подкатегории:

- **URL (Uniform Resource Locator):** указывает, где ресурс и как до него добраться. Пример: `https://www.example.com/index.html` - страница, доступная по HTTPS.
- **URN (Uniform Resource Name):** уникальное имя без указания местоположения. Пример: `urn:isbn:0451450523` идентифицирует книгу по ISBN, но не говорит, где её скачать.

URL состоит из частей:

- **Схема:** протокол доступа - `http`, `https`, `ftp`.
- **Хост:** домен или IP сервера - `www.example.com`.
- **Порт:** необязательный номер порта - `:443`.
- **Путь:** путь к ресурсу - `/path/to/resource`.
- **Запрос:** необязательные параметры - `?id=123`.
- **Фрагмент:** необязательный якорь внутри ресурса - `#section1`.

Разберём `https://www.example.com:443/path/to/resource?id=123#section1`:

- `https` - схема (протокол HTTPS).
- `www.example.com` - хост.
- `:443` - порт (стандартный для HTTPS).
- `/path/to/resource` - путь.
- `?id=123` - параметр запроса `id` со значением `123`.
- `#section1` - фрагмент, ссылка на участок внутри ресурса.

## Подводные камни

- **Не каждый URI - URL.** URN тоже URI, но в нём нет ни хоста, ни способа доступа - только имя.
- **Фрагмент `#section1` обрабатывается браузером, а не сервером.** На сервер он не отправляется, поэтому повлиять на ответ сервера через фрагмент нельзя.
- **Порт по умолчанию подразумевается схемой.** Для `http` это 80, для `https` - 443; явно его пишут редко.
- **"URI и URL - это одно и то же" - неточный ответ.** Технически URL уже, и на собесе важно показать, что вы понимаете иерархию URI → {URL, URN}.
