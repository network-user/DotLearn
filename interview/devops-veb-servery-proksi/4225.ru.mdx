# Как проверить, установлен ли Nginx?

## Суть

Быстрая проверка: бинарь в PATH (`command -v nginx`), версия (`nginx -v`), пакет в менеджере (`dpkg -l nginx` / `rpm -q nginx`), unit systemd (`systemctl status nginx`). Для контейнера смотрю образ и процесс внутри namespace. Цель: отличить «не установлен» от «установлен, но не запущен».

## Что ответить на собесе

- `nginx -v` для версии; `nginx -V` ещё показывает compile flags и пути конфигов.
- `command -v nginx` или `which nginx` для пути бинаря.
- `systemctl is-active nginx` отдельно от наличия пакета.
- В Ansible/Chef проверяю факт пакета, не только curl на :80.
- Честно: nginx из snap/docker не виден в dpkg; смотрю runtime, не один способ.

## Как это работает

Пакетный nginx кладёт бинарь в `/usr/sbin/nginx`, unit `nginx.service`, конфиги в `/etc/nginx/`.

```bash
command -v nginx && nginx -v

nginx -V 2>&1 | head -5

systemctl status nginx --no-pager

dpkg -l 'nginx*' 2>/dev/null || rpm -qa 'nginx*'
```

Если бинарь есть, но сервис failed:

```bash
sudo nginx -t
journalctl -u nginx -n 30 --no-pager
```

## Подводные камни

- **Exit code nginx -v.** Версия пишется в stderr; в скриптах не путаю с ошибкой отсутствия.
- **Несколько установок.** Custom prefix `/opt/nginx` не в PATH оператора; ищу через `ps aux | grep nginx`.
- **Distro vs mainline.** Ubuntu default vs nginx.org repo дают разные модули; `-V` показывает отличия.
- **Права на nginx -t.** Без root не вижу все include; проверяю под тем же user, что systemd unit.
- **Контейнер sidecar.** Хост без nginx, прокси в pod; проверка на node вводит в заблуждение.
