# How does LEMP differ from LAMP?

## In short

LAMP is Linux + Apache + MySQL/MariaDB + PHP/Python/Perl. LEMP swaps Apache for Nginx (Engine-X). Database and language stay the same; the gap is the web server and PHP execution: Apache mod_php vs Nginx plus PHP-FPM over FastCGI. Nginx often wins on static files and high-concurrency proxying.

## How to answer in an interview

- LAMP: `.htaccess`, mod_php, prefork/event MPM; familiar for shared hosting and legacy apps.
- LEMP: Nginx terminates HTTP, PHP runs only in FPM pools; no per-request embed in the web server process.
- Static and TLS on Nginx offload CPU from Apache/PHP for typical CMS traffic.
- Migration means rewriting `.htaccess` rules into nginx config by hand.
- Honestly: for tiny traffic the gap is small; at high RPS LEMP usually saves RAM and latency.

## How it works

LEMP server block with PHP-FPM:

```nginx
server {
    listen 80;
    server_name blog.example.com;
    root /var/www/blog/public;
    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_pass unix:/run/php/php8.3-fpm.sock;
    }

    location ~ /\. {
        deny all;
    }
}
```

Check FPM socket:

```bash
ls -l /run/php/php8.3-fpm.sock
curl -sI -H 'Host: blog.example.com' http://127.0.0.1/
```

## Pitfalls

- **.htaccess does not port.** Every rewrite must land in nginx; missing rules break pretty URLs.
- **FPM pool sizing.** `pm.max_children` times RAM per worker can OOM under load.
- **Apache modules.** mod_security, LDAP auth lack 1:1 nginx equivalents; plan alternatives early.
- **Mixed stack risk.** Nginx proxying to Apache "for compatibility" adds hops without benefit.
- **Session file path.** PHP sessions on shared disk need sticky sessions or a centralized store.
