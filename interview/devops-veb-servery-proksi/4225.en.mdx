# How do you check if Nginx is installed?

## In short

Quick checks: binary in PATH (`command -v nginx`), version (`nginx -v`), package manager (`dpkg -l nginx` / `rpm -q nginx`), systemd unit (`systemctl status nginx`). In containers inspect the image and process inside the namespace. Goal: separate "not installed" from "installed but not running".

## How to answer in an interview

- `nginx -v` for version; `nginx -V` also shows compile flags and config paths.
- `command -v nginx` or `which nginx` for the binary path.
- `systemctl is-active nginx` separately from package presence.
- In Ansible/Chef I check the package fact, not only curl on :80.
- Honestly: snap/docker nginx may be absent from dpkg; inspect runtime, not a single method.

## How it works

Packaged nginx places the binary in `/usr/sbin/nginx`, unit `nginx.service`, configs under `/etc/nginx/`.

```bash
command -v nginx && nginx -v

nginx -V 2>&1 | head -5

systemctl status nginx --no-pager

dpkg -l 'nginx*' 2>/dev/null || rpm -qa 'nginx*'
```

If the binary exists but the service failed:

```bash
sudo nginx -t
journalctl -u nginx -n 30 --no-pager
```

## Pitfalls

- **nginx -v exit code.** Version prints to stderr; scripts must not treat it as missing binary.
- **Multiple installs.** Custom prefix `/opt/nginx` may be off PATH; find via `ps aux | grep nginx`.
- **Distro vs mainline.** Ubuntu default vs nginx.org repos differ in modules; `-V` reveals gaps.
- **nginx -t permissions.** Without root some includes fail; test as the systemd unit user.
- **Container sidecar.** No nginx on the host while the pod proxies; node checks mislead.
