# How do you sort a slice of structs?

## In short

Use the `sort` package: either implement `sort.Interface` (Len/Less/Swap) on a wrapper around `[]Struct`, or with Go 1.21+ use `slices.SortFunc` with a compare function. The sort key lives in `Less` or the comparator.

## How to answer in an interview

- For legacy code I show a wrapper type `type ByAge []Person` with interface methods.
- In new code I prefer `slices.SortFunc(people, func(a, b Person) int { ... })` from std `slices` (Go 1.21+).
- I note that sorting is in-place and mutates the original slice.
- I honestly add that stable ordering for equal keys needs `sort.Stable` or `slices.SortStableFunc`.

## How it works

`sort.Sort` runs introsort through your `Less`. The wrapper does not copy data; it swaps elements in the backing slice.

```go
package main

import (
	"cmp"
	"fmt"
	"slices"
)

type Member struct {
	Name string
	Age  int
}

func main() {
	team := []Member{
		{Name: "Ira", Age: 31},
		{Name: "Bob", Age: 27},
		{Name: "Cy", Age: 27},
	}

	slices.SortFunc(team, func(a, b Member) int {
		if c := cmp.Compare(a.Age, b.Age); c != 0 {
			return c
		}
		return cmp.Compare(a.Name, b.Name)
	})

	for _, m := range team {
		fmt.Printf("%s %d\n", m.Name, m.Age)
	}
}
```

Compare age first, then name. `cmp.Compare` returns -1/0/1 for ints and strings.

## Pitfalls

- **Sorting a copy header.** `sort.Sort(ByAge(people))` sorts the backing slice; a detached header with no elements does nothing.
- **Unstable sort.** Plain `sort.Sort` does not preserve order among equal keys.
- **Many fields.** A huge `Less` gets unreadable; extract a named comparator.
- **Concurrent sort.** Sorting the same slice from goroutines without sync is a data race.
- **Locale-aware strings.** `cmp.Compare` on strings is byte order, not Unicode collation.
