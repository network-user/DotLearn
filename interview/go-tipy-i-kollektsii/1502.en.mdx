# How do you convert types in Go?

## In short

Use explicit conversion `T(v)` between compatible numeric types and identical underlying types. There is no implicit C-style widening. Type assertions `x.(T)` are separate for interfaces.

## How to answer in an interview

- Syntax like `float64(count)` and `int64(u)` with compile-time constant checks.
- Named types with the same underlying type still convert explicitly: `UserID(42)`.
- Strings and bytes: `[]byte(s)` and `string(b)` copy data.
- I honestly add that int narrowing truncates silently; I validate ranges or use strconv.

## How it works

```go
package main

import (
	"fmt"
	"strconv"
)

type UserID int

func main() {
	var n int = 42
	f := float64(n)
	id := UserID(n)

	text := strconv.Itoa(n)
	back, _ := strconv.Atoi(text)

	fmt.Println(f, id, back)
}
```

`UserID` and `int` are distinct named types; conversion stays explicit.

## Pitfalls

- **Truncation.** float to int drops the fraction, no rounding.
- **Rune vs byte.** `string(rune)` differs from building UTF-8 carefully.
- **unsafe conversions.** Only for specialized perf paths.
- **Interface assertion panic.** Use the comma-ok form.
- **JSON numbers.** Unmarshaling may pass through float64 first.
