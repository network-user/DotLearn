# На сколько растёт capacity слайса при append?

## Суть

Когда `len` достигает `cap`, runtime выделяет новый массив и копирует данные. Пока `cap < 256` (исторически до 1024 в старых версиях) рост близок к удвоению; для больших слайсов прирост примерно 25% плюс существующая ёмкость. Точные пороги зависят от версии Go и размера элемента.

## Что ответить на собесе

- Формула из runtime: при малых cap удваиваем; при больших `newcap = oldcap + oldcap/4` с округлением под выравнивание.
- Подчёркиваю: после роста старый массив может остаться в памяти, если на него смотрят другие слайсы.
- Упоминаю `make([]T, 0, hint)` для снижения числа realloc при известном размере.
- Честно добавляю: цифры 1024/25% это эвристика runtime, не гарантия языка; в новых релизах детали могут меняться.

## Как это работает

Слайс это заголовок (pointer, len, cap) над массивом. `append` пишет в существующий массив, пока есть место. Иначе runtime считает `newcap`, аллоцирует массив, копирует элементы, обновляет заголовок.

```go
package main

import "fmt"

func trackGrowth() {
	var nums []int
	fmt.Printf("start len=%d cap=%d\n", len(nums), cap(nums))
	for i := 0; i < 12; i++ {
		nums = append(nums, i)
		fmt.Printf("i=%d len=%d cap=%d\n", i, len(nums), cap(nums))
	}
}

func aliasTrap() {
	base := make([]byte, 3, 8)
	chunk := base[:2]
	base = append(base, 99, 100, 101, 102, 103)
	fmt.Println("base", base)
	fmt.Println("chunk", chunk)
}

func main() {
	trackGrowth()
	aliasTrap()
}
```

Первый фрагмент печатает скачки `cap`. Второй показывает ловушку: `chunk` делит массив с `base` до realloc; после роста `base` `chunk` может указывать на старый массив или перезаписанные ячейки.

Планирование ёмкости важно в hot path: лишние копии при каждом append дают O(n²) на построение большого среза.

## Подводные камни

- **Зубчатая арифметика.** Ожидать ровно ×2 на каждом шаге неверно; runtime округляет под размер типа.
- **Общий backing array.** Subslice + append в соседний слайс перезаписывает данные без роста cap.
- **append(nil, x).** Первый append с nil создаёт массив с маленьким cap; серия append без hint дорога.
- **Срез struct с pointer.** Копирование при grow копирует указатели, не deep copy объектов.
- **Зависимость от версии.** В Go 1.18+ пороги роста менялись; на собесе говорю «удвоение на малых, ~25% на крупных».
