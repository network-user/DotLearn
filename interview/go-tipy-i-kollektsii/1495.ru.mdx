# Чем особенна built-in copy для slice?

## Суть

`copy(dst, src)` переносит min(len(dst), len(src)) элементов и возвращает их число. Длины slice не меняются; shallow copy элементов в overlapping regions корректен по spec.

## Что ответить на собесе

- Возвращаемое int показывает сколько реально скопировано.
- Типы dst/src должны совпадать (элемент один тип).
- Для полной копии slice делаю `append([]T(nil), src...)` или ` slices.Clone`.
- Честно добавляю: copy не deep copy; для `[]*T` или struct с pointers клонирую второй уровень отдельно.

## Как это работает

Built-in использует memmove по underlying array. Если dst и src overlap, поведение определено как для memmove.

```go
package main

import "fmt"

func main() {
	src := []int{10, 20, 30, 40}
	dst := make([]int, 2)

	n := copy(dst, src)
	fmt.Println(n, dst)

	buf := []int{1, 2, 3, 4, 5}
	copy(buf[1:], buf)
	fmt.Println(buf)
}
```

Первый copy переносит два элемента в короткий dst. Второй сдвигает overlapping region внутри одного slice.

## Подводные камни

- **Короткий dst silently truncates.** Без проверки return count теряются хвостовые элементы.
- **Nil dst/src.** copy(nil, src) returns 0 без panic.
- **Different cap same len.** copy не расширяет dst; нужен append или resize.
- **Shallow copy structs.** Pointer fields alias после copy.
- **String bytes.** Для []byte использую copy; string immutable, нужен []byte(src).
