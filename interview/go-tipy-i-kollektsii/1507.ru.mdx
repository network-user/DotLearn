# Как задать длину slice при создании?

## Суть

Длину slice задаю через `make([]T, length)` или `make([]T, length, capacity)`. Литерал `[]T{...}` фиксирует длину по числу элементов. У `var s []T` длина 0, пока не добавлю данные или не вызову `make`.

## Что ответить на собесе

- `make([]int, 5)` создаёт slice длиной 5 с zero-valued элементами; cap по умолчанию равен len.
- `make([]int, 0, 10)` длина 0, ёмкость 10: буфер под будущие `append` без лишних аллокаций.
- Срез существующего массива `arr[low:high]` задаёт длину как `high - low`.
- Честно уточняю: `make` с ненулевой длиной инициализирует элементы нулями, это не то же самое, что пустой slice с запасом cap.

## Как это работает

```go
package main

import "fmt"

func main() {
	fixed := make([]int, 4)
	buffered := make([]int, 0, 8)
	fromLiteral := []int{10, 20, 30}

	fmt.Println(len(fixed), cap(fixed))       // 4 4
	fmt.Println(len(buffered), cap(buffered)) // 0 8

	buffered = append(buffered, 1, 2)
	fmt.Println(len(buffered), cap(buffered)) // 2 8
	fmt.Println(fromLiteral)                    // [10 20 30]
}
```

`make` выделяет backing array и заполняет slice header указателем, len и cap. Срез `arr[i:j]` не аллоцирует новый массив, а переиспользует существующий с новой длиной относительно того же массива.

## Подводные камни

- **Путаница len и cap в make.** Второй аргумент это длина, третий (если есть) ёмкость, не наоборот.
- **Zero-filled vs empty.** `make([]T, n)` уже содержит `n` элементов; `append` увеличит длину с `n`, а не с 0.
- **Срез без копии.** Изменение элементов subslice меняет базовый массив для всех срезов на него ссылающихся.
- **make с length 0.** `make([]T, 0)` не `nil`, в отличие от `var s []T`.
- **Большой length.** `make([]byte, 1e9)` сразу резервирует память; для streaming лучше буфер с разумным cap.
