# How do you implement a Set in Go?

## In short

There is no built-in Set type. The idiom is `map[T]struct{}` or `map[T]bool` where the key is the element and the value is empty. Insert and membership are O(1) on average.

## How to answer in an interview

- I suggest `map[string]struct{}` for zero-size values and less memory than `bool`.
- Methods `Add`, `Contains`, and `Delete` wrap map operations.
- Union and intersect come from ranging over a second set.
- I honestly add that sorted sets or ordered iteration need more than a map alone, often a slice or tree.

## How it works

Map key uniqueness deduplicates automatically. `struct{}` stores no value bytes.

```go
package main

import "fmt"

type StringSet map[string]struct{}

func (s StringSet) Add(v string) {
	s[v] = struct{}{}
}

func (s StringSet) Has(v string) bool {
	_, ok := s[v]
	return ok
}

func main() {
	tags := make(StringSet)
	tags.Add("go")
	tags.Add("go")
	tags.Add("api")

	fmt.Println(tags.Has("go"), len(tags))
}
```

The second `Add("go")` does not increase len.

## Pitfalls

- **map[bool] zero value.** With `map[T]bool` you cannot tell missing keys from false without `ok`.
- **Concurrent sets.** Wrap the map with a mutex.
- **Non-comparable T.** Sets only work for comparable key types.
- **Iteration order.** A map-backed set is not sorted.
- **Generic sets since Go 1.18.** `type Set[T comparable] map[T]struct{}` is reusable.
