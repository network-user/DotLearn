# Чем int отличается от uint?

## Суть

`int` знаковый тип платформенной разрядности; хранит отрицательные и положительные значения. `uint` беззнаковый, только ≥0, верхняя граница в два раза выше для той же bit width.

## Что ответить на собесе

- На amd64 `int` и `uint` 64-bit; на 32-bit arch 32-bit.
- Смешивать int и uint в арифметике нельзя без explicit conversion.
- Для lengths и indexes stdlib почти всегда `int`, не uint.
- Честно добавляю: uint удобен для bit masks и hash-like значений, но overflow wrap silent и легко пропустить underflow через 0.

## Как это работает

Компилятор держит signed vs unsigned отдельно. Conversion `uint(x)` проверяет range at compile time only when constant.

```go
package main

import "fmt"

func main() {
	var signed int = -3
	var unsigned uint = 5

	fmt.Println(signed, unsigned)

	// mix := signed + unsigned // compile error

	sum := unsigned + uint(-signed)
	fmt.Println(sum)
}
```

Явное приведение `-signed` к uint для демонстрации требует осторожности: отрицательное к uint даст huge number.

## Подводные камни

- **Negative to uint cast.** `-1` as uint becomes max value.
- **Printf %d on uint.** Works but mixing verbs and types confusing.
- **JSON numbers.** Unmarshaling into uint rejects negative.
- **Architecture portability.** Code assuming 64-bit int breaks on 32-bit builds.
- **Loop counters.** `for i := uint(0); ...` rare; idiomatic Go uses int.
