# На чём реализована map в Go?

## Суть

Встроенная map это hash table: массив bucket, хеш ключа, цепочки overflow внутри bucket. Средняя сложность insert/lookup/delete амортизированно O(1).

## Что ответить на собесе

- Называю hash table с bucket и overflow chains, не linked list на верхнем уровне.
- Объясняю роль hash, load factor и grow при заполнении.
- Показываю базовый CRUD: make, assign, comma-ok, delete.
- Честно добавляю: детали bucket layout менялись между версиями Go; для собеса важна модель, не hex dump runtime.

## Как это работает

`make(map[K]V)` аллоцирует hmap structure. Ключ хешируется, выбирается bucket, внутри ищется slot. При коллизии используется overflow bucket. Когда load factor высок, таблица удваивается и rehash.

```go
package main

import "fmt"

func main() {
	ages := map[string]int{
		"Ada":   36,
		"Linus": 54,
	}

	ages["Grace"] = 85

	if age, ok := ages["Ada"]; ok {
		fmt.Println("Ada:", age)
	}

	delete(ages, "Linus")

	if _, ok := ages["Linus"]; !ok {
		fmt.Println("Linus removed")
	}
}
```

`delete` удаляет entry; обращение к удалённому ключу даёт zero value и `ok=false`.

## Подводные камни

- **Nil map write.** Запись в nil map panic; read возвращает zero.
- **Не thread-safe.** Параллельная запись без sync это data race.
- **Iteration order.** Обход bucket layout, не sorted keys.
- **Float keys.** NaN ломает dedup логику.
- **Taking address of map element.** `&m[k]` запрещено, потому что map может rehash и переместить data.
