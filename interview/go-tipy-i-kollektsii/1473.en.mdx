# What is the lookup complexity of a map?

## In short

Key lookup in a Go `map` is O(1) on average because it is backed by a hash table with buckets. In the worst case, when every key lands in the same bucket, search degrades to O(n).

## How to answer in an interview

- I explain that map lookup is amortized constant time: the key hash picks a bucket, then the key-value pair is found inside it.
- I note that the worst case is theoretically linear under massive collisions or adversarial keys.
- I show the `value, ok := m[key]` idiom to test presence without extra allocations.
- I honestly add that for normal keys and reasonable sizes a map behaves like O(1) in practice, but you must not rely on iteration order or magical stability.

## How it works

Go stores a map as an array of buckets. The key is hashed and the low bits of the hash select a bucket. Each bucket holds a small array of pairs and, when full, overflow chains. Reading `m[key]` computes the hash, locates the bucket, and compares keys inside it.

When the map grows, the runtime adds buckets and rebalances entries so average chain length stays small. That is why insert, delete, and lookup typically do not scale linearly with total map size.

Example read with an existence check:

```go
package main

import "fmt"

func main() {
	inventory := map[string]int{
		"apple":  12,
		"pear":   7,
		"orange": 3,
	}

	qty, found := inventory["pear"]
	if found {
		fmt.Printf("pear stock: %d\n", qty)
	}

	missing, ok := inventory["mango"]
	fmt.Printf("mango ok=%v qty=%d\n", ok, missing)
}
```

Here `found`/`ok` separates "key missing" from "key present with zero value", which matters for counters.

## Pitfalls

- **Zero value vs absence.** Without the second `ok` bool you cannot tell a missing key from a zero-valued entry.
- **Collisions and DoS.** Predictable or crafted keys can blow up one bucket; for untrusted keys you may need hardened maps or another structure.
- **Concurrent access.** Unsynchronized read/write causes a data race; for shared state I use `sync.Map` or a mutex.
- **Mutating while ranging.** Changing the same map during `range` is undefined and may panic.
- **Comparable keys.** Keys must support `==`; slices and maps cannot be keys and will not compile.
