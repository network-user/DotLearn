# Какими способами создают map в Go?

## Суть

Три основных пути: `make(map[K]V)` или с hint, composite literal `map[K]V{...}`, и объявление `var m map[K]V` с последующим `make`. Nil map нельзя писать до инициализации; чтение даёт zero value.

## Что ответить на собесе

- `make(map[string]int, 64)` задаёт hint по ожидаемому числу элементов.
- Литерал сразу кладёт стартовые пары и готов к записи.
- `var m map[K]V` это nil; `m["k"]=v` паникует, нужен `make` или литерал.
- Честно различаю: nil map при JSON маршалинге даёт `null`, пустой `make` даёт `{}`.

## Как это работает

```go
package main

import "fmt"

func main() {
	hinted := make(map[string]int, 4)
	hinted["cpu"] = 8

	preloaded := map[string]int{
		"rx": 1,
		"tx": 2,
	}

	var delayed map[string]int
	fmt.Println(delayed == nil) // true
	delayed = make(map[string]int)
	delayed["ok"] = 1

	fmt.Println(hinted, preloaded, delayed)
}
```

`make` аллоцирует hash table header и bucket массив. Литерал делает то же с начальным набором ключей. Пакетный `var` удобен для отложенной инициализации в конструкторах, но перед записью обязателен `make`.

## Подводные камни

- **Запись в nil map.** Runtime panic; отличаю от пустого, но инициализированного map.
- **Hint не hard limit.** `make(..., n)` не запрещает вставить больше n элементов.
- **Копирование map.** Присваивание копирует descriptor, не глубокие значения; для clone нужен явный обход.
- **Итерация и delete.** Безопасно удалять текущий ключ в `range`, но не добавлять произвольно во время обхода по старым правилам (в новых версиях ослаблено, но осторожность остаётся).
- **Concurrent init.** Двойной `make` из разных goroutine на одну переменную без sync это data race на уровне программы.
