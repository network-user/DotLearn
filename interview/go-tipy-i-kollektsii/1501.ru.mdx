# Что такое Big O?

## Суть

Big O описывает верхнюю асимптотическую границу роста времени или памяти алгоритма при увеличении размера входа. Константы и lower-order terms отбрасываются.

## Что ответить на собесе

- Фокус на dominant term при больших n.
- Примеры: index access O(1), linear scan O(n), nested loops O(n²).
- Отличаю time vs space complexity.
- Честно добавляю: на реальном Go-сервисе смотрю pprof, Big O не заменяет измерения.

## Как это работает

Считаем операции как функцию n, берём класс роста. `for range` один раз линейно, sort stdlib O(n log n).

```go
package main

import "fmt"

func contains(haystack []int, needle int) bool {
	for _, v := range haystack {
		if v == needle {
			return true
		}
	}
	return false
}

func main() {
	data := make([]int, 1000)
	fmt.Println(contains(data, 42))
}
```

`contains` O(n) time, O(1) extra memory.

## Подводные камни

- **Worst vs average.** Quicksort average O(n log n), worst O(n²) без care.
- **Multiple inputs.** O(n+m) для двух slice, не всегда один n.
- **Space recursion.** O(1) time может быть O(n) stack.
- **Misleading constants.** O(n) с tiny n может beat O(log n) with huge overhead.
- **Parallelism.** Big O rarely captures cores without extra model.
