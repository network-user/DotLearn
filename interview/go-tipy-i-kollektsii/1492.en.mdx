# What happens when you range over a map?

## In short

`for k, v := range m` visits every map pair and yields key and value each iteration. Visit order is unspecified and may differ between runs; the runtime may start from a pseudo-random bucket.

## How to answer in an interview

- I explain iteration is neither sorted nor insertion order.
- I can use `for k := range m` or `for _, v := range m` when only one part is needed.
- Deleting the current key during range is safe; adding entries is undefined behavior territory.
- I honestly add that deterministic output requires collecting and sorting keys first.

## How it works

The iterator walks the bucket array, skipping empty slots. Growth during range stays defined for the current snapshot, but order must never be relied on.

```go
package main

import "fmt"

func main() {
	rates := map[string]float64{
		"USD": 1.0,
		"EUR": 0.92,
		"GBP": 0.79,
	}

	for code, rate := range rates {
		fmt.Printf("%s=%.2f\n", code, rate)
	}

	for code := range rates {
		if code == "GBP" {
			delete(rates, code)
		}
	}
	fmt.Println(len(rates))
}
```

The first loop prints all pairs in arbitrary order. The second may delete the current key safely.

## Pitfalls

- **Mutating during range.** Inserts may perturb iteration; deleting the current key is ok, deleting others needs care.
- **Large values in loop.** `v` copies big structs; consider pointer values or indirection.
- **Break/continue.** Same as an ordinary for loop; no indexed order.
- **Concurrent range.** Parallel range plus writes is a data race.
- **Testing order.** Never assert stdout order without sorting keys.
