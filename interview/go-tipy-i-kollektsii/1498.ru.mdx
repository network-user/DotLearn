# Какие классы алгоритмической сложности знаешь?

## Суть

Big-O описывает рост времени или памяти при увеличении n. Базовые классы: O(1), O(log n), O(n), O(n log n), O(n²), O(2^n), O(n!). Различаю worst/average/amortized где уместно.

## Что ответить на собесе

- Перечисляю классы с примером: map get O(1), binary search O(log n), scan slice O(n), sort O(n log n).
- Уточняю space complexity: recursive DFS O(h) stack, merge sort O(n) extra.
- Для Go map говорю amortized O(1) insert, worst O(n) при pathological hash.
- Честно добавляю: константы matter в prod; Big-O это модель, не substitute для profiling.

## Как это работает

Анализирую dominant term при n→∞. Nested loops часто дают O(n²); divide-and-conquer sort O(n log n).

```go
package main

import "fmt"

func findFirst(ids []int, target int) (int, bool) {
	for i, v := range ids {
		if v == target {
			return i, true
		}
	}
	return 0, false
}

func main() {
	data := []int{4, 7, 1, 9}
	fmt.Println(findFirst(data, 7))
}
```

Linear scan O(n) time, O(1) extra space.

## Подводные камни

- **O vs Θ vs Ω.** Собес иногда требует уточнить tight bound.
- **Amortized vs worst.** append amortized O(1), single step может trigger O(n) copy.
- **Hidden n.** Graph alg с V+E, не один параметр.
- **Micro benchmarks.** Cache effects ломают теорию на малых n.
- **Space of recursion.** Depth без tail call может overflow stack.
