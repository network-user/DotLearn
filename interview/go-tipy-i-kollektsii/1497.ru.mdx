# Что такое ACT-дерево?

## Суть

ACT (Abstract Control Tree) это дерево потока управления в компиляторах: узлы представляют seq, if, loop, call, а не синтаксические детали AST. Нужно для анализа control flow и оптимизаций.

## Что ответить на собесе

- Отличаю ACT от AST: AST про синтаксис, ACT про control edges.
- Применение: dataflow, dead code elimination, loop transforms.
- В Go toolchain публично чаще говорят про SSA (`GOSSAFUNC`), но идея control tree знакома из compiler courses.
- Честно добавляю: на типичном Go-собесе deep ACT редок; достаточно связать с CFG и оптимизациями backend.

## Как это работает

Исходный код lowering в дерево, где листья это базовые блоки, внутренние узлы это комбинаторы control. Компилятор обходит ACT для построения CFG и дальше SSA.

```go
package main

import "fmt"

func classify(n int) {
	for i := 0; i < n; i++ {
		if i%2 == 0 {
			fmt.Println(i, "even")
		} else {
			fmt.Println(i, "odd")
		}
	}
}

func main() {
	classify(3)
}
```

Loop node содержит if внутри body; ACT абстрагирует это в иерархию control, не в строки fmt.

## Подводные камни

- **Путать с AST.** AST не equal ACT; разные passes.
- **SSA dominance.** ACT/CFG prerequisite для many opts.
- **Go escape analysis.** Работает на другом IR уровне, не ACT напрямую.
- **Interview overreach.** Не обещаю знание internal gcc/llvm unless claimed.
- **Runtime не видит ACT.** Это compile-time structure only.
