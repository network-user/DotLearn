# В каком порядке Go обходит map?

## Суть

Порядок элементов при `range` по map в Go не зафиксирован и может меняться между запусками. Это сознательное решение рантайма: map не упорядоченная коллекция, а хеш-таблица.

## Что ответить на собесе

- Объясняю, что `for k, v := range m` не гарантирует ни алфавит, ни порядок вставки.
- Если нужен стабильный порядок, собираю ключи в слайс, сортирую и иду по ним.
- Упоминаю, что рандомизация порядка помогает ловить скрытые зависимости от iteration order в тестах.
- Честно добавляю: внутри одного вызова `range` порядок фиксирован, но между запусками программы на него опираться нельзя.

## Как это работает

Map распределяет пары по корзинам через хеш ключа. При обходе runtime идёт по внутренним структурам таблицы, а не по логическому порядку ключей. Поэтому два последовательных `range` в одной программе могут совпасть, но после изменения map или в другом процессе порядок другой.

Для детерминированного вывода ключи выносят в слайс и сортируют:

```go
package main

import (
	"fmt"
	"sort"
)

func main() {
	scores := map[string]int{
		"Zara": 88,
		"Alex": 91,
		"Mila": 76,
	}

	names := make([]string, 0, len(scores))
	for name := range scores {
		names = append(names, name)
	}
	sort.Strings(names)

	for _, name := range names {
		fmt.Printf("%s: %d\n", name, scores[name])
	}
}
```

Сначала собираем все ключи, затем `sort.Strings` задаёт алфавитный порядок, и только потом читаем значения.

## Подводные камни

- **Тесты с hardcoded порядком.** Snapshot-тест, завязанный на порядок map, будет flaky между версиями Go или после rehash.
- **JSON-сериализация.** `encoding/json` сортирует ключи объектов, но это не меняет правило для `range` по map в вашем коде.
- **Параллельная сортировка больших map.** Сортировка ключей добавляет O(n log n); для горячих путей иногда хранят упорядоченную структуру отдельно.
- **Удаление во время range.** Удалять текущий ключ в `range` по map безопасно, добавлять новые bucket может менять порядок оставшихся итераций.
- **Сравнение map целиком.** Две map с одинаковым набором пар равны независимо от порядка обхода, но отображение в UI требует явной сортировки.
