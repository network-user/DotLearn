# Равен ли nil slice неинициализированному?

## Суть

`var s []int` и `var s []int = nil` это один zero value slice header: pointer=nil, len=0, cap=0. Оба дают `s == nil`. Отличие появляется только у empty non-nil slice `[]int{}`.

## Что ответить на собесе

- Zero value slice уже nil; явное `= nil` ничего не меняет.
- `append` на nil slice работает и аллоцирует backing array.
- `len`/`cap` nil slice равны 0; `range` безопасен и не итерирует.
- Честно добавляю: для JSON/API nil slice и `[]T{}` сериализуются по-разному (`null` vs `[]`), хотя оба «пустые» для len.

## Как это работает

Slice header без underlying array это nil slice. После первого успешного `append` pointer становится non-nil.

```go
package main

import (
	"encoding/json"
	"fmt"
)

func main() {
	var declared []int
	var explicit []int = nil
	empty := []int{}

	fmt.Println(declared == nil, explicit == nil, empty == nil)

	declared = append(declared, 1)
	fmt.Println(declared, len(declared))

	jNil, _ := json.Marshal([]int(nil))
	jEmpty, _ := json.Marshal([]int{})
	fmt.Println(string(jNil), string(jEmpty))
}
```

`declared` и `explicit` эквивалентны до и после присваивания nil. `empty` не nil header; JSON различает null и пустой массив.

## Подводные камни

- **nil vs empty в JSON.** API контракты ломаются, если клиент ждёт всегда массив.
- **reflect.DeepEqual.** `[]int(nil)` не равен `[]int{}`.
- **SQL/array drivers.** NULL vs empty array в БД.
- **Отправка nil slice в функцию.** Это нормально: `len=0`, range безопасен.
- **Указатель на slice.** `var p *[]int` это другой тип, не путать с nil slice value.
