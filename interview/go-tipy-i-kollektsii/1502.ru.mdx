# Как в Go преобразуют типы?

## Суть

Между совместимыми numeric types и одинаковыми underlying types использую explicit conversion `T(v)`. Неявного widening/narrowing как в C нет. Type assertion `x.(T)` отдельно для interfaces.

## Что ответить на собесе

- Синтаксис `float64(count)`, `int64(u)` с compile-time checks для constants.
- Named types с same underlying type convert явно: `UserID(42)`.
- Строки и bytes: `[]byte(s)`, `string(b)` копируют данные.
- Честно добавляю: overflow при int→ smaller int silent; проверяю range вручную или strconv.

## Как это работает

```go
package main

import (
	"fmt"
	"strconv"
)

type UserID int

func main() {
	var n int = 42
	f := float64(n)
	id := UserID(n)

	text := strconv.Itoa(n)
	back, _ := strconv.Atoi(text)

	fmt.Println(f, id, back)
}
```

`UserID` и `int` разные named types, conversion explicit даже при same underlying type.

## Подводные камни

- **Truncation.** float→int отбрасывает дробную часть, не округляет.
- **Rune vs byte.** `string(rune)` не то же что string from int code point without care.
- **Unsafe conversion.** `unsafe` package for perf, не для обычного кода.
- **Interface assertion panic.** Без ok-form.
- **JSON numbers to int.** Unmarshal float64 промежуточно.
