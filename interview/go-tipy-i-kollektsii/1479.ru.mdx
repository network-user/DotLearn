# Можно ли положить функцию в any?

## Суть

Да: функции в Go это значения первого класса, и `any` (`interface{}`) принимает любой конкретный тип, включая func. Чтобы вызвать её, нужен type assertion или type switch к точной сигнатуре.

## Что ответить на собесе

- Подтверждаю, что func можно присвоить переменной типа `any`, как int или struct.
- Объясняю, что без assertion компилятор не знает сигнатуру, поэтому вызов только через приведение типа.
- Показываю паттерн `if fn, ok := v.(func(int) int); ok { ... }`.
- Честно добавляю: хранить callbacks в `any` редко оправдано; лучше typed func или маленький interface с одним методом.

## Как это работает

`any` хранит dynamic type функции целиком, включая параметры и результаты. Два `func()` и `func(int)` это разные типы для assertion. Замыкания тоже упаковываются в `any`.

```go
package main

import "fmt"

func runIfCallable(box any) {
	switch fn := box.(type) {
	case func():
		fn()
	case func(string) string:
		fmt.Println(fn("ping"))
	default:
		fmt.Println("unsupported callable")
	}
}

func main() {
	greet := func() { fmt.Println("hi from closure") }
	runIfCallable(greet)

	upper := func(s string) string { return s + "!" }
	runIfCallable(upper)

	runIfCallable(42)
}
```

Первый вызов срабатывает через `case func()`, второй через другую сигнатуру, число попадает в default.

## Подводные камни

- **Nil func в any.** Typed nil function inside interface не равен nil interface; вызов может panic.
- **Разные сигнатуры.** Нельзя один раз assert к `func()` и передать `func(int)` без отдельной ветки.
- **Сравнение func.** Два func value сравнимы только если оба nil; иначе `==` не компилируется.
- **Reflect overhead.** Если сигнатуры заранее неизвестны, приходится идти в reflect, что медленно и хрупко.
- **Generics альтернатива.** Для коллекции колбэков одного типа `[]func(Context) error` типобезопаснее `[]any`.
