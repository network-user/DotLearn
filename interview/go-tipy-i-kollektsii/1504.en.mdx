# What does len() count for a Go string?

## In short

`len(s)` on a string returns the number of bytes in its UTF-8 encoding, not the number of Unicode code points. ASCII is one byte per character; Cyrillic and emoji use more bytes per visible character.

## How to answer in an interview

- A Go string is an immutable UTF-8 byte sequence; `len` measures bytes.
- To count characters (code points), I convert to `[]rune` or iterate with `for range` over the string.
- I honestly add: for user-facing limits (form field length), `len` alone is not enough without Unicode normalization.

## How it works

```go
package main

import "fmt"

func main() {
	ascii := "Hello"
	cyrillic := "Привет"

	fmt.Println(len(ascii))    // 5 bytes, 5 characters
	fmt.Println(len(cyrillic)) // 12 bytes, 6 characters

	runes := []rune(cyrillic)
	fmt.Println(len(runes)) // 6 code points
}
```

`for i, r := range cyrillic` yields byte indices and one rune at a time; that is the safe way to walk characters without copying the whole string into `[]rune`.

## Pitfalls

- **Surrogate pairs and combining marks.** `len` and `[]rune` do not count grapheme clusters; visible character counts need normalization.
- **String slicing by byte index.** `s[1:3]` can split a UTF-8 rune and produce an invalid string.
- **Locale length comparison.** The same text in NFC vs NFD can have different byte `len`.
- **Performance.** `[]rune(s)` allocates; for long strings prefer `range` or `utf8.RuneCountInString`.
- **Confusion with []byte.** `len` matches for `string` and the corresponding `[]byte`, but character semantics are still byte-based.
