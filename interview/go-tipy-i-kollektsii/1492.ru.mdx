# Что делает range по map?

## Суть

`for k, v := range m` обходит все пары map, возвращая ключ и значение на каждой итерации. Порядок обхода не определён и может отличаться между запусками; runtime может стартовать с pseudo-random bucket.

## Что ответить на собесе

- Объясняю, что iteration не sorted и не insertion order.
- Можно `for k := range m` или `for _, v := range m` если нужна только часть.
- Удаление текущего ключа во время range безопасно; добавление entries undefined.
- Честно добавляю: для детерминированного вывода сначала собираю и сортирую ключи.

## Как это работает

Iterator проходит bucket array, пропуская пустые slots. При grow во время range поведение остаётся defined для текущего snapshot, но полагаться на порядок нельзя.

```go
package main

import "fmt"

func main() {
	rates := map[string]float64{
		"USD": 1.0,
		"EUR": 0.92,
		"GBP": 0.79,
	}

	for code, rate := range rates {
		fmt.Printf("%s=%.2f\n", code, rate)
	}

	for code := range rates {
		if code == "GBP" {
			delete(rates, code)
		}
	}
	fmt.Println(len(rates))
}
```

Первый цикл печатает все пары в произвольном порядке. Во втором удаление текущего ключа допустимо.

## Подводные камни

- **Modify map during range.** Insert может сдвинуть итерацию; delete текущего ok, delete других осторожно.
- **Large map allocation in loop.** `v` копируется если value type big struct; беру pointer values или index.
- **Break/continue semantics.** Как в обычном for; нет indexed order.
- **Concurrent range.** Параллельный range + write это data race.
- **Testing order.** Нельзя assert порядок stdout без сортировки ключей.
