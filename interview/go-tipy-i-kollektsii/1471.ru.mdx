# Чем отличается var m map от m := make(map)?

## Суть

`var m map[K]V` создаёт nil map: читать можно (вернёт zero value), писать нельзя (panic). `m := make(map[K]V)` или литерал `map[K]V{}` выделяет пустую готовую таблицу.

## Что ответить на собесе

- Nil map ведёт себя как пустая при lookup/delete, но `m[k]=v` паникует.
- `make` и литерал `{}` эквивалентны для пустой map; `make` удобен с capacity hint.
- Упоминаю JSON unmarshal: в пустую struct поле map остаётся nil до первой записи unmarshaler создаёт map.
- Честно добавляю: `==` между map сравнивает ссылки на runtime-структуру; две пустые инициализированные map не «равны» по содержимому через `==`, только nil vs non-nil.

## Как это работает

Zero value для map это nil pointer на `hmap`. Операции read/delete безопасны и возвращают «нет ключа». Insert требует аллоцированного `hmap`, что делает `make` или присваивание из уже созданной map.

```go
package main

import "fmt"

func main() {
	var idle map[string]int
	fmt.Println(idle == nil)
	if v, ok := idle["missing"]; ok {
		fmt.Println(v)
	} else {
		fmt.Println("lookup on nil map is safe")
	}

	active := make(map[string]int, 4)
	active["go"] = 1
	fmt.Println(active["go"])

	literal := map[string]int{}
	literal["rust"] = 2
	fmt.Println(literal["rust"])

	// idle["x"] = 1 // panic: assignment to entry in nil map
}
```

`make(..., 4)` резервирует место под ~4 элемента без обязательного заполнения. Литерал `map[string]int{}` сразу даёт writable map без hint.

Паттерн «ленивая инициализация»: если map опциональна, держу nil и создаю при первой вставке через helper, чтобы не аллоцировать зря.

## Подводные камни

- **Путаница nil и empty map.** `len(nil map)` и `len(empty map)` оба 0, но только non-nil принимает insert.
- **Передача nil map в функцию, которая пишет.** Callee должен `make` если `m == nil`.
- **range по nil map.** Безопасен, ноль итераций; не путать с nil slice, который тоже range-safe.
- **Сериализация.** `json.Marshal(nil map)` даёт `null`, пустая map даёт `{}`.
- **Сравнение map через ==.** Работает только с nil; иначе compile error кроме сравнения с nil.
