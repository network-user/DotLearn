# Может ли интерфейс добавить поля в структуру?

## Суть

Нет. Интерфейс в Go описывает только методы, данных в нём нет. Поля живут в struct (или другом конкретном типе); интерфейс лишь требует поведение, которое эти типы должны предоставить.

## Что ответить на собесе

- Отвечаю коротко «нет» и контрастирую с Java/C# abstract class, где могут быть поля и реализованные методы.
- Объясняю композицию: общие поля выношу в embedded struct, общее поведение в интерфейс.
- Упоминаю embedding: встроенный тип продвигает методы наружу, но это не «поля интерфейса».
- Честно добавляю: если нужен и контракт, и shared state, использую struct + interface вместе, а не ожидаю магии от interface.

## Как это работает

Объявление `type Writer interface { Write([]byte) (int, error) }` не резервирует память под поля. Значение интерфейса хранит указатель на concrete value и таблицу методов. Поля `Title`, `Body` объявляются только в struct.

Паттерн «базовая структура + интерфейс»:

```go
package main

import "fmt"

type Document struct {
	Title string
	Body  string
}

func (d Document) WordCount() int {
	return len(d.Body)
}

type Measurable interface {
	WordCount() int
}

type Report struct {
	Document
	Author string
}

func summarize(m Measurable) {
	fmt.Printf("words=%d\n", m.WordCount())
}

func main() {
	r := Report{
		Document: Document{Title: "Q1", Body: "revenue growth"},
		Author:   "finance",
	}
	summarize(r)
}
```

`Report` встраивает `Document` и автоматически получает `WordCount`, удовлетворяя `Measurable`. Интерфейс не добавил поле `Title`; оно пришло из embedding struct.

Если нужен полиморфный доступ к полям, в Go используют type assertion, reflection или отдельные геттеры в интерфейсе (что раздувает контракт и обычно нежелательно).

## Подводные камни

- **Ожидание наследования полей.** Кандидаты из ООП-языков ищут `extends`; в Go только композиция.
- **Embedding и shadowing.** Вложенное поле с тем же именем, что и внешнее, скрывает доступ; методы продвигаются, но не всегда очевидно какое поле читается.
- **Интерфейс с геттерами вместо полей.** `Name() string` в интерфейсе работает, но превращает данные в API и усложняет эволюцию.
- **Пустой interface как «любая struct».** `any` не даёт доступа к полям без assertion.
- **Pointer embedding nil.** Встроенный указатель nil не продвигает методы value receiver; вызов паникует.
