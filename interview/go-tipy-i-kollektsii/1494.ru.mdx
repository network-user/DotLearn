# Как реализовать Set в Go?

## Суть

Отдельного типа Set в stdlib нет. Идиома: `map[T]struct{}` или `map[T]bool`, где ключ это элемент множества, а значение пустое. Insert и membership O(1) в среднем.

## Что ответить на собесе

- Предлагаю `map[string]struct{}` для zero-size values и экономии памяти против `bool`.
- Методы `Add`, `Contains`, `Delete` оборачивают map операции.
- Union/intersect делаю через range и второй set.
- Честно добавляю: если нужен sorted set или range по порядку, map alone недостаточен, добавляю slice или tree.

## Как это работает

Уникальность ключей map автоматически dedup. `struct{}` не занимает байт значения.

```go
package main

import "fmt"

type StringSet map[string]struct{}

func (s StringSet) Add(v string) {
	s[v] = struct{}{}
}

func (s StringSet) Has(v string) bool {
	_, ok := s[v]
	return ok
}

func main() {
	tags := make(StringSet)
	tags.Add("go")
	tags.Add("go")
	tags.Add("api")

	fmt.Println(tags.Has("go"), len(tags))
}
```

Второй `Add("go")` не увеличивает len.

## Подводные камни

- **map bool zero value.** `map[T]bool` не отличить «нет ключа» от false без ok.
- **Concurrent set.** Нужен mutex вокруг map.
- **Non-comparable T.** Set только для comparable ключей.
- **Iteration order.** Set из map не sorted.
- **Generic set Go 1.18+.** `type Set[T comparable] map[T]struct{}` переиспользуем.
