# How do you set slice length at creation?

## In short

Set slice length with `make([]T, length)` or `make([]T, length, capacity)`. A literal `[]T{...}` fixes length from the element count. `var s []T` stays at length 0 until you add data or call `make`.

## How to answer in an interview

- `make([]int, 5)` builds a length-5 slice with zero-valued elements; cap defaults to len.
- `make([]int, 0, 10)` has length 0 and capacity 10: headroom for `append` without extra allocations.
- Slicing an array `arr[low:high]` sets length to `high - low`.
- I honestly add: `make` with non-zero length zero-fills elements; that is not the same as an empty slice with reserved cap.

## How it works

```go
package main

import "fmt"

func main() {
	fixed := make([]int, 4)
	buffered := make([]int, 0, 8)
	fromLiteral := []int{10, 20, 30}

	fmt.Println(len(fixed), cap(fixed))       // 4 4
	fmt.Println(len(buffered), cap(buffered)) // 0 8

	buffered = append(buffered, 1, 2)
	fmt.Println(len(buffered), cap(buffered)) // 2 8
	fmt.Println(fromLiteral)                    // [10 20 30]
}
```

`make` allocates a backing array and fills the slice header with pointer, len, and cap. A slice expression `arr[i:j]` does not allocate a new array; it reuses the existing one with a new length on the same backing store.

## Pitfalls

- **len vs cap in make.** The second argument is length; the optional third is capacity, not the reverse.
- **Zero-filled vs empty.** `make([]T, n)` already holds `n` elements; `append` grows length from `n`, not from 0.
- **Non-copying subslices.** Mutating a subslice updates the backing array for every slice that shares it.
- **make with length 0.** `make([]T, 0)` is not `nil`, unlike `var s []T`.
- **Huge length.** `make([]byte, 1e9)` reserves memory immediately; for streaming prefer a reasonably sized cap buffer.
