# Как работает поиск в BST?

## Суть

Ищем значение от root: меньше текущего узла идём влево, больше вправо, равно нашли. Заканчиваем на nil если пути нет. Итеративная версия предпочтительнее из-за stack safety.

## Что ответить на собесе

- Описываю invariant BST: left < node < right.
- Даю и recursive, и loop вариант; на prod беру loop.
- Сравниваю с map lookup O(1) average vs BST O(log n) ordered traversal bonus.
- Честно добавляю: без балансировки worst case linear; для ordered set в Go чаще sort + slice.

## Как это работает

```go
package main

import "fmt"

type Node struct {
	Value int
	Left  *Node
	Right *Node
}

func find(root *Node, needle int) (*Node, bool) {
	cur := root
	for cur != nil {
		switch {
		case needle == cur.Value:
			return cur, true
		case needle < cur.Value:
			cur = cur.Left
		default:
			cur = cur.Right
		}
	}
	return nil, false
}

func main() {
	tree := &Node{
		Value: 8,
		Left:  &Node{Value: 3},
		Right: &Node{Value: 10},
	}
	if _, ok := find(tree, 3); ok {
		fmt.Println("found")
	}
}
```

Каждый hop отсекает subtree по invariant.

## Подводные камни

- **Not a BST.** Нарушен invariant даёт wrong answer без ошибки.
- **Nil root.** Возвращаю false, не panic.
- **Duplicates policy.** Нужно заранее решить куда класть equal keys.
- **Unbalanced tree.** Performance degrades silently.
- **Comparator types.** Generic BST needs consistent less function.
