# Какое value в map, если нужен только набор ключей?

## Суть

Для set-подобной map значение беру `struct{}`: нулевой размер, не хранит данных. Идиома `map[K]struct{}` экономит память по сравнению с `map[K]bool` и явно сигнализирует, что value не используется.

## Что ответить на собесе

- Вставка: `set[key] = struct{}{}`; проверка: `_, ok := set[key]`.
- `struct{}` занимает 0 байт; в map всё равно есть накладные расходы на bucket.
- Альтернатива `map[K]bool` проще для новичков, но каждый bool это минимум байт в value-слоте.
- Честно добавляю: для больших set в Go 1.23+ смотрю на `map` vs sorted slice vs специализированные структуры по профилю.

## Как это работает

```go
package main

import "fmt"

func main() {
	visited := make(map[string]struct{})

	for _, id := range []string{"a", "b", "a", "c"} {
		if _, seen := visited[id]; seen {
			continue
		}
		visited[id] = struct{}{}
		fmt.Println("new:", id)
	}

	fmt.Println(len(visited)) // 3
}
```

`struct{}{}` это единственное значение типа `struct{}`. При итерации `for k := range visited` value игнорирую. Удаление из set: `delete(visited, id)`.

## Подводные камни

- **Память не нулевая.** Bucket overhead остаётся; `struct{}` убирает только payload value.
- **bool как value.** `map[K]bool` позволяет хранить «выключен» явно, но легко забыть проверять `ok`.
- **Concurrent set.** Нужен mutex или `sync.Map`; обычная map не потокобезопасна.
- **Сериализация.** JSON map с `struct{}` неудобен; для API экспортирую `[]K`.
- **Ключи-указатели.** Set указателей сравнивает адреса; для value-semantics ключ должен быть value type.
