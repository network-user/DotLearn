# How does search work in a BST?

## In short

Start at the root: go left if smaller, right if larger, stop when equal. A nil pointer means missing. Prefer an iterative loop for stack safety.

## How to answer in an interview

- I state the BST invariant: left < node < right.
- I show recursive and loop forms; production code uses the loop.
- I compare map lookup O(1) average vs BST O(log n) with ordered traversal as a bonus.
- I honestly add that without balancing worst case is linear; ordered sets in Go often use sort plus slice.

## How it works

```go
package main

import "fmt"

type Node struct {
	Value int
	Left  *Node
	Right *Node
}

func find(root *Node, needle int) (*Node, bool) {
	cur := root
	for cur != nil {
		switch {
		case needle == cur.Value:
			return cur, true
		case needle < cur.Value:
			cur = cur.Left
		default:
			cur = cur.Right
		}
	}
	return nil, false
}

func main() {
	tree := &Node{
		Value: 8,
		Left:  &Node{Value: 3},
		Right: &Node{Value: 10},
	}
	if _, ok := find(tree, 3); ok {
		fmt.Println("found")
	}
}
```

Each hop discards a subtree using the invariant.

## Pitfalls

- **Broken invariant.** Wrong answers without errors if the tree is not a BST.
- **Nil root.** Return false, do not panic.
- **Duplicate policy.** Decide where equal keys go up front.
- **Unbalanced tree.** Performance degrades silently.
- **Generic BST.** Needs a consistent less function.
