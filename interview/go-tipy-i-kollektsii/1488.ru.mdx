# Можно ли использовать array как ключ map?

## Суть

Фиксированный array (`[N]T`) может быть ключом map, если элементы comparable. Slice (`[]T`) нельзя: slice не comparable. На собесах часто путают эти два случая.

## Что ответить на собесе

- Отвечаю: array да, slice нет, потому что только array поддерживает `==` целиком.
- Пример: `map[[2]int]string` компилируется, `map[[]int]string` нет.
- Если нужен динамический ключ, сериализую в string или использую hash от canonical bytes.
- Честно добавляю: array-key неудобен для больших N; на практике чаще string/id.

## Как это работает

Array сравнивается поэлементно, когда тип элемента comparable. Две переменные array с одинаковым содержимым равны и попадают в один bucket.

```go
package main

import "fmt"

func main() {
	grid := make(map[[2]int]string)

	pos := [2]int{3, 4}
	grid[pos] = "start"

	lookup := [2]int{3, 4}
	fmt.Println(grid[lookup])

	// grid2 := make(map[[]int]string) // compile error
}
```

`lookup` совпадает с `pos` по значению, запись находится без указателей.

## Подводные камни

- **Slice рядом с array.** Кандидаты часто отвечают «массивы нельзя»; уточняю терминологию Go.
- **Array с incomparable элементом.** `[3][]int` не годится как ключ.
- **Большой array-key.** Копирование при insert и сравнение дорогие.
- **Мутация локального array.** Изменение байт после insert не обновляет ключ в map.
- **Convert slice to array.** Go 1.20+ `array := [N]T(slice)` помогает, если длина известна заранее.
