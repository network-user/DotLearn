# What map value type when you only need keys?

## In short

For a set-like map I use `struct{}` as the value: zero size, no stored data. The idiom `map[K]struct{}` saves memory versus `map[K]bool` and signals clearly that values are unused.

## How to answer in an interview

- Insert with `set[key] = struct{}{}`; membership with `_, ok := set[key]`.
- `struct{}` occupies 0 bytes; maps still pay bucket overhead.
- `map[K]bool` is friendlier to beginners but each bool costs at least a byte in the value slot.
- I honestly add: for large sets since Go 1.23+ I compare map vs sorted slice vs specialized structures on profiles.

## How it works

```go
package main

import "fmt"

func main() {
	visited := make(map[string]struct{})

	for _, id := range []string{"a", "b", "a", "c"} {
		if _, seen := visited[id]; seen {
			continue
		}
		visited[id] = struct{}{}
		fmt.Println("new:", id)
	}

	fmt.Println(len(visited)) // 3
}
```

`struct{}{}` is the sole value of type `struct{}`. On `for k := range visited` I ignore values. Set removal is `delete(visited, id)`.

## Pitfalls

- **Memory is not zero.** Bucket overhead remains; `struct{}` only removes value payload.
- **bool values.** `map[K]bool` can represent explicit false, but it is easy to skip the `ok` check.
- **Concurrent sets.** Use a mutex or `sync.Map`; a plain map is not thread-safe.
- **Serialization.** JSON with `struct{}` values is awkward; APIs export `[]K` instead.
- **Pointer keys.** A set of pointers compares addresses; value semantics need value-typed keys.
