# Относительный путь: как браузер находит файл рядом с HTML

## Суть

Относительный путь указывает ресурс от текущего документа без домена и протокола. Браузер склеивает его с URL страницы, поэтому `css/app.css` и `../img/icon.svg` работают одинаково локально и на сервере, пока структура папок сохранена.

## Что ответить на собесе

- Путь без `http://` и без ведущего `/` считается относительным к location текущего файла.
- `styles/main.css` ищет папку `styles` рядом с HTML; `../` поднимается на уровень вверх.
- В CSS `url()` базой служит файл стилей, не HTML.
- Относительные ссылки упрощают перенос проекта между доменами и staging.
- Честно добавляю: в SPA с history API относительный путь считается от URL в адресной строке, что часто ломает наивные ссылки на глубоких роутах.

## Как это работает

Алгоритм резолва берёт **base URL** (обычно адрес страницы) и применяет сегменты пути.

Структура:

```
app/
  index.html
  assets/logo.svg
  styles/theme.css
```

В `index.html`:

```html
<link rel="stylesheet" href="styles/theme.css" />
<img src="./assets/logo.svg" alt="Logo" />
```

В `theme.css`:

```css
.hero {
  background-image: url("../assets/logo.svg");
}
```

Из CSS нужно выйти из `styles/` в `app/`, затем зайти в `assets/`.

Сборщик (Vite, Webpack) на этапе билда может переписать пути и положить файлы в `/assets/hash.ext`, но в исходниках относительные ссылки остаются читаемыми.

Относительный путь не привязан к домену: `href="docs/guide.html"` откроется и на `localhost:5173`, и на проде при той же иерархии.

## Подводные камни

**Разная база в CSS и HTML.** Ошибка `url(img/x.png)` в CSS лежащего в `/css/` ищет `/css/img/x.png`.

**Глубокие роуты SPA.** Страница `/topics/css/grid` превратит `href="favicon.ico"` в `/topics/css/favicon.ico`.

**Относительный vs protocol-relative.** `//host/file` не то же самое, что `file` или `./file`.

**Символы и кодировка.** Пробелы и кириллица в именах файлов требуют percent-encoding в URL.

**Жёсткие пути в email.** Письма требуют абсолютных URL; относительные там не работают.
