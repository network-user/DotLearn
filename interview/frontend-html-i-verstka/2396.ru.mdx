# Какие типы input появились в HTML5

## Суть

Атрибут `type` у `<input>` задаёт семантику поля, встроенную валидацию и нативный UI: email, url, number, date, range, color и другие. Браузер подбирает клавиатуру на мобиле и базовые проверки до отправки формы.

## Что ответить на собесе

- Называю ключевые типы: `email`, `url`, `tel`, `search`, `number`, `range`, `date`, `time`, `datetime-local`, `month`, `week`, `color`, плюс классические `text`, `password`, `checkbox`, `radio`, `file`, `hidden`.
- Объясняю, что тип влияет на constraint validation API и на подсказки UI (календарь, ползунок, палитра).
- Упоминаю атрибуты `min`, `max`, `step`, `multiple`, `list` как дополнение к типу.
- Подчёркиваю, что серверная валидация всё равно обязательна.
- Честно добавляю: виджеты и правила различаются между браузерами, особенно `date` в Safari.

## Как это работает

До HTML5 почти всё сводилось к `text` и ручным маскам на JS. Новые типы переносят часть логики в платформу.

**Текстовые и контактные:** `email` проверяет наличие `@` и домена; `url` требует схему; `tel` не валидирует формат, но открывает цифровую клавиатуру; `search` может показать кнопку очистки.

**Числовые:** `number` даёт спиннер и проверяет число; `range` рисует ползунок в пределах `min`/`max` с шагом `step`.

**Дата и время:** `date`, `time`, `datetime-local`, `month`, `week` открывают нативные пикеры там, где браузер их поддерживает.

**Прочие:** `color` показывает выбор цвета; `checkbox`/`radio` для булева и взаимоисключающего выбора; `file` для загрузки с `accept` и `multiple`.

Пример формы регистрации:

```html
<form>
  <label>
    Email
    <input type="email" name="email" required autocomplete="email" />
  </label>
  <label>
    Возраст
    <input type="number" name="age" min="18" max="120" />
  </label>
  <label>
    Дата визита
    <input type="date" name="visit" />
  </label>
  <label>
    Уровень громкости
    <input type="range" name="volume" min="0" max="100" step="5" />
  </label>
  <label>
    Акцентный цвет
    <input type="color" name="accent" value="#3366cc" />
  </label>
</form>
```

В JS можно читать `input.validity` и `input.validationMessage`; при `type="email"` поле `typeMismatch` станет true для строки без домена.

На мобильных `inputmode` и тип вместе влияют на раскладку: для телефона удобнее `tel`, для кода из цифр иногда `text` + `inputmode="numeric"`.

## Подводные камни

**Полагаться только на клиент.** Любой `type` обходится через DevTools; бэкенд должен проверять данные.

**Несовместимый UI дат.** `date` в одном браузере календарь, в другом почти `text`; нужен fallback или библиотека.

**number для кодов и телефонов.** Ведущие нули и плюс в `number` теряются; лучше `text` с маской.

**Слабая валидация email/url.** Браузер проверяет форму грубо, не RFC полностью.

**Скрытый type через CSS.** Нативный виджет исчезает, но constraint validation остаётся: тестируйте реальный UX.
