# Resource paths: relative, root-relative, and absolute URLs

## In short

A path in `src`/`href` can be relative (from the current file), root-relative from `/` (from the site origin), or absolute (full URL with protocol and host). The tokens `./`, `../`, and a leading `/` set different reference points.

## How to answer in an interview

- A relative path has no domain; the browser resolves it from the current page URL.
- `./file.png` looks in the same folder; `../assets/logo.svg` goes one level up.
- A leading `/` (`/static/app.js`) is from the host root, not the HTML folder.
- Absolute `https://cdn.example.com/lib.js` is for external CDNs and email templates.
- I honestly add that when deployed under a subpath (`example.com/app/`), root `/...` paths break without `base href` or a bundler public path prefix.

## How it works

The browser applies the WHATWG **URL resolution** algorithm: it takes the page base URL and merges the relative reference.

Project layout:

```
/site
  index.html
  css/main.css
  img/hero.webp
```

From `index.html`:

```html
<link rel="stylesheet" href="./css/main.css" />
<img src="img/hero.webp" alt="Hero" />
<img src="/img/hero.webp" alt="Hero from site root" />
```

The first two are equivalent when base is at the root. The third always requests `/img/hero.webp` on the current origin, even if HTML lives at `/blog/post/`.

External resource:

```html
<script src="https://esm.sh/react@18" type="module"></script>
```

Host and scheme are fixed; the page on `localhost` and in production load the same URL.

In an SPA with client-side routing, relative paths in markup resolve from the **address bar**, not the physical file: a deep route `/topics/css` breaks `href="img/x.png"` without a leading `/` or a bundler public path.

## Pitfalls

**Broken ../ when files move.** Deep HTML nesting needs more `../`; bundlers usually rewrite paths in the build.

**Root vs subdirectory.** `/favicon.ico` on a GitHub project site without base hits the wrong origin path.

**Protocol-relative URLs.** `//cdn.example.com/x.js` inherits http/https; prefer explicit `https:` today.

**Relative paths in CSS url().** `url(../fonts/a.woff2)` resolves from the CSS file, not the HTML.

**Mixing CDN and same-origin.** Absolute third-party scripts need SRI and CSP policies, or supply-chain risk grows.
