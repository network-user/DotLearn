# Table markup: thead, tbody, tfoot, th, and caption

## In short

An HTML table is for tabular data, not page layout. Sections `thead`, `tbody`, `tfoot`, cells `th`/`td`, and `caption` define structure that screen readers and the browser understand when printing.

## How to answer in an interview

- I list roles: `caption` (table title), `thead` (header), `tbody` (data), `tfoot` (footer/summary), `th` (header cell), `td` (value).
- I explain that `th` carries `scope` (`col`/`row`) to tie headers to data.
- I mention `colspan`/`rowspan` for merged cells and keeping header logic intact.
- I say CSS `display: table` for page layout loses to Flex/Grid; I reserve `<table>` for data.
- I honestly add that on very large tables, render virtualization matters more than perfect semantics on every row.

## How it works

The browser builds a **table layout model**: columns share width, `thead` may repeat when printing, and a screen reader announces "column header" when moving across `td` cells.

Workshop schedule example:

```html
<table>
  <caption>March workshop schedule</caption>
  <thead>
    <tr>
      <th scope="col">Date</th>
      <th scope="col">Topic</th>
      <th scope="col">Speaker</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Mar 12</td>
      <td>CSS Grid</td>
      <td>A. Ivanova</td>
    </tr>
    <tr>
      <td>Mar 19</td>
      <td>Form accessibility</td>
      <td>B. Petrov</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">All sessions on Zoom; link in the invite</td>
    </tr>
  </tfoot>
</table>
```

`scope="col"` links a header to cells below in that column. Complex headers with merged cells use `headers`/`id` on `th` and `td`.

`tfoot` may appear before `tbody` in markup, but the browser still paints it at the bottom. The parser knows totals before the body finishes streaming.

## Pitfalls

**Table as layout.** Nested layout tables break AT navigation and resist responsive design.

**Empty thead.** A header row of `td` instead of `th` removes column context for users.

**Colspan without a plan.** Random merges break header-to-data relationships.

**Missing caption.** An unnamed table is harder to index and is announced as "table" without a title.

**Hundreds of DOM rows.** Analytics with thousands of rows needs virtualization or another UI pattern, not a bare `<table>`.
