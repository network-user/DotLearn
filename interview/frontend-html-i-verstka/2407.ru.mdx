# Пути к ресурсам: относительные, корневые и абсолютные URL

## Суть

Путь в `src`/`href` может быть относительным (от текущего файла), корневым от `/` (от origin сайта) или абсолютным (полный URL с протоколом и хостом). Символы `./`, `../` и ведущий `/` задают разную точку отсчёта.

## Что ответить на собесе

- Относительный путь не содержит домен; браузер резолвит его от URL текущей страницы.
- `./file.png` ищет файл в той же папке; `../assets/logo.svg` поднимается на уровень выше.
- Путь с ведущим `/` (`/static/app.js`) считается от корня хоста, не от папки HTML.
- Абсолютный `https://cdn.example.com/lib.js` нужен для внешних CDN и email-шаблонов.
- Честно добавляю: при деплое в подпапку (`example.com/app/`) корневые `/...` пути ломаются без `base href` или префикса сборщика.

## Как это работает

Браузер применяет алгоритм **URL resolution** из WHATWG: берёт base URL страницы и сливает с относительной ссылкой.

Структура проекта:

```
/site
  index.html
  css/main.css
  img/hero.webp
```

Из `index.html`:

```html
<link rel="stylesheet" href="./css/main.css" />
<img src="img/hero.webp" alt="Hero" />
<img src="/img/hero.webp" alt="Hero from site root" />
```

Первые два эквивалентны при base в корне. Третий всегда ищет `/img/hero.webp` на текущем origin, даже если HTML лежит в `/blog/post/`.

Внешний ресурс:

```html
<script src="https://esm.sh/react@18" type="module"></script>
```

Здесь хост и схема фиксированы; страница на `localhost` и в проде грузит один и тот же URL.

В SPA с client-side routing относительные пути в разметке считаются от **адресной строки**, а не от физического файла: глубокий роут `/topics/css` ломает `href="img/x.png"` без ведущего `/` или без bundler, который подставляет public path.

## Подводные камни

**Сломанный ../ при переезде файлов.** Глубокая вложенность HTML требует больше `../`; сборщик обычно хеширует пути в билде.

**Корень vs поддиректория.** `/favicon.ico` на GitHub Pages в project site без base ведёт на неверный origin path.

**Протокол-relative URL.** `//cdn.example.com/x.js` наследует http/https; сегодня лучше явный `https:`.

**Относительный путь в CSS url().** `url(../fonts/a.woff2)` резолвится от файла CSS, не от HTML.

**Смешение CDN и same-origin.** Абсолютные внешние скрипты требуют SRI и политики CSP, иначе растёт поверхность supply-chain атак.
