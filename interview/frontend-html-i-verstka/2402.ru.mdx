# Как открыть ссылку в новой вкладке безопасно

## Суть

Атрибут `target="_blank"` на `<a>` открывает URL в новой вкладке или окне, не выгружая текущую страницу. Для внешних ссылок добавляют `rel="noopener noreferrer"`, чтобы новая вкладка не получила доступ к `window.opener`.

## Что ответить на собесе

- Называю пару `target="_blank"` + `rel="noopener noreferrer"` как стандарт для внешних ресурсов.
- Объясняю tabnabbing: чужая страница могла бы подменить `location` исходной вкладки через `opener`.
- Уточняю, что современные браузеры изолируют opener по умолчанию, но явный `rel` остаётся best practice.
- Говорю про UX: новые вкладки уместны для PDF, внешних доков; внутренняя навигация остаётся в той же вкладке.
- Честно добавляю: принудительное `_blank` на все ссылки раздражает пользователей и ломает историю «назад».

## Как это работает

Гиперссылка по умолчанию (`target="_self"`) заменяет текущий документ в той же вкладке. Значение **`_blank`** просит браузер создать новый browsing context.

```html
<a
  href="https://docs.vendor.example/guide"
  target="_blank"
  rel="noopener noreferrer"
>
  Документация поставщика
</a>
```

`noopener` запрещает новой вкладке ссылаться на исходную через `window.opener`. `noreferrer` дополнительно не отправляет Referer и в ряде браузеров тоже отключает opener.

В React/TSX те же атрибуты передаются на `<a>`; для внутренних маршрутов Router обычно не ставит `_blank`.

Другие значения `target` реже на фронте: `_parent`, `_top` встречаются во фреймах; именованные окна (`target="print"`) почти не используют.

С точки зрения доступности стоит предупредить пользователя, что ссылка внешняя: видимый текст или `aria-label` «откроется в новой вкладке», иконка с подписью, не только title.

Для скачивания файлов иногда комбинируют `download` (same-origin) или оставляют обычную навигацию без `_blank`, если пользователь должен вернуться к форме.

## Подводные камни

**Только `_blank` без rel.** Старые браузеры и корпоративные политики всё ещё уязвимы к opener-атакам.

**`_blank` на каждой ссылке.** Пользователь теряет контроль над навигацией.

**Скрытый внешний URL.** Открытие в новой вкладке не снимает ответственность за фишинг.

**popup-blocker на window.open.** Программное открытие без жеста пользователя блокируется; `<a target="_blank">` от клика работает.

**Забытый `href` на кнопке-ссылке.** `button` с JS `window.open` хуже для SEO и a11y, чем настоящий `<a>`.
