# div vs span: block container and inline wrapper

## In short

`<div>` is a generic block container: it spans the parent width and starts a new line. `<span>` is an inline text wrapper: it does not break the line flow and fits local styling or JS hooks.

## How to answer in an interview

- I compare display behavior: block for div, inline for span (until CSS changes it).
- I say div groups large layout sections, span highlights words and phrases.
- I stress both are non-semantic; prefer `section`, `p`, `button`, `strong` when possible.
- I note flex/grid on div is common; span sometimes needs `display: inline-block`.
- I add honestly: in modern markup, div-only layouts lose to semantics and components.

## How it works

A block **div** creates a rectangular box in the document flow. It may contain both block and inline children. Typical cases: card wrapper, grid column, React component root.

```html
<div class="card">
  <h2>Card title</h2>
  <p>Product description across multiple lines.</p>
</div>
```

A **span** lives inside a paragraph and does not force neighboring text onto a new line:

```html
<p>
  Order status:
  <span class="badge badge--paid">Paid</span>
</p>
```

CSS can override the model: `display: inline-block` on span enables sizing and margins; `display: flex` on div creates a flex formatting context.

For accessibility an empty div without a role gives screen readers no landmark. A bare `class` on span is silent too unless meaningful text sits in `<strong>` or a justified `aria-label` is added.

In interviews it helps to show tag choice starts with semantics: navigation in `<nav>`, actions in `<button>`, and div/span only when no semantic element fits.

## Pitfalls

**div-only layout.** Hundreds of wrappers without landmarks hurt a11y and SEO.

**span as a button.** A clickable span without role and keyboard handling is inaccessible.

**div inside p.** The parser closes the paragraph and markup breaks.

**display changes without flow awareness.** flex on span inside running text breaks the expected line.

**Extra wrappers for BEM.** Not every block needs three nested divs.
