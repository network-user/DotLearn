# Каркас HTML-документа: doctype, html, head и body

## Суть

Любая страница строится из объявления типа документа, корневого `<html>`, блока метаданных `<head>` и видимой части `<body>`. `DOCTYPE` переключает браузер в стандартный режим рендеринга и задаёт ожидаемую версию разметки.

## Что ответить на собесе

- Перечисляю четыре уровня: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>` и кратко назначение каждого.
- Подчёркиваю, что без корректного doctype возможен quirks mode и «плавающая» вёрстка.
- Упоминаю `lang` на `<html>` для SEO, озвучки скринридерами и выбора шрифта.
- Называю типичное содержимое head: `title`, `meta charset`, `viewport`, стили, скрипты.
- Честно добавляю: doctype не валидирует HTML, он только сообщает браузеру режим парсинга.

## Как это работает

Парсер начинаёт с **DOCTYPE**. В HTML5 достаточно короткой формы `<!DOCTYPE html>` без публичного идентификатора. Браузер понимает, что документ следует современным правилам, и не эмулирует старые баги Netscape.

Корневой **`<html>`** оборачивает всё дерево. Атрибут `lang="ru"` задаёт язык по умолчанию для всей страницы: влияет на произношение у VoiceOver/NVDA и на селекторы вроде `:lang()`.

**`<head>`** не рисуется на экране, но управляет поведением вкладки и загрузкой ресурсов:

- `<title>` попадает на вкладку и в сниппет поиска.
- `<meta charset="utf-8">` должен стоять как можно раньше.
- `<meta name="viewport" ...>` нужен для адаптивной вёрстки.
- `<link rel="stylesheet">` и `<script>` подключают CSS и JS; критичный JS иногда кладут в head с `defer`.

**`<body>`** содержит весь видимый UI: заголовки, текст, медиа, формы. Семантические секции (`header`, `main`, `footer`) живут именно здесь.

Минимальный скелет страницы:

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Каталог товаров</title>
    <link rel="stylesheet" href="/styles/base.css" />
  </head>
  <body>
    <header><h1>Магазин</h1></header>
    <main><p>Контент страницы</p></main>
  </body>
</html>
```

Без doctype тот же CSS может дать другие отступы у `body` и иначе считать `box-sizing`, потому что движок включает режим совместимости.

## Подводные камни

**Пропущенный DOCTYPE.** Страница «ломается» только в старых корпоративных браузерах или при копировании фрагментов без шаблона.

**Неверный порядок в head.** `charset` после большого скрипта задерживает определение кодировки.

**Дублирование title и h1.** Пустой или технический `<title>` ухудшает доступность вкладок и поиск.

**Скрипты без defer/async в head.** Блокируют первый рендер, хотя проблема не в структуре, а в размещении ресурсов.

**lang не задан.** Скринридер читает английский текст с русской интонацией и наоборот.
