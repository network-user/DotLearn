# HTML lists: ul, ol, and dl and when to use each

## In short

HTML provides three list types: bulleted `ul`, numbered `ol`, and definition list `dl`. The choice depends on whether order matters and whether you need term/definition pairs.

## How to answer in an interview

- `ul` for equivalent items without numbering (menus, tags, ingredients).
- `ol` when sequence carries meaning (recipe steps, rankings, instructions).
- `dl` for glossaries: `dt` is the term, `dd` is the definition; multiple `dd` per `dt` is allowed.
- I mention nested lists and styling markers with CSS (`list-style`, `::marker`).
- I honestly add that for long FAQs, `section` plus headings can be easier, but `dl` stays semantically right for term/value pairs.

## How it works

Each list renders as a **group of items** with list role. A screen reader reports item count and nesting depth.

Bulleted list:

```html
<ul>
  <li>TypeScript</li>
  <li>React</li>
  <li>Vite</li>
</ul>
```

Numbered steps:

```html
<ol>
  <li>Install dependencies: <code>pnpm install</code></li>
  <li>Start the dev server: <code>pnpm dev:web</code></li>
  <li>Open <code>localhost:5173</code></li>
</ol>
```

The `start` or `reversed` attribute changes numbering; `value` on `li` sets a specific item number.

Definition list:

```html
<dl>
  <dt>CLS</dt>
  <dd>Layout shift while content loads.</dd>
  <dt>LCP</dt>
  <dd>Time until the largest visible block appears.</dd>
</dl>
```

Navigation is often `nav > ul > li > a`, keeping list semantics for links.

## Pitfalls

**br instead of li.** Line breaks do not form a list; AT does not hear separate items.

**ol for decorative numbers.** If order does not matter, numbering misleads users.

**dl as arbitrary columns.** dt/dd pairs should be glossary terms, not a generic label/value grid.

**Deep nesting.** A fifth-level `ul` hurts narration; simplify structure or split content.

**Resetting list styles.** `list-style: none` on nav is fine, but keep roles correct (native `ul`/`li` preserves list semantics).
