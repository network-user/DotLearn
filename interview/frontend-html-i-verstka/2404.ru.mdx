# Связка label и input: for/id, вложение и доступность форм

## Суть

Метка `<label>` должна быть программно связана с полем ввода: через пару `for`/`id` или вложением. Тогда клик по тексту фокусирует контрол, а скринридер озвучивает подпись вместе с полем.

## Что ответить на собесе

- Называю два способа: `for="email"` на label и `id="email"` на input, либо input внутри label.
- Подчёркиваю рост зоны клика на мобиле: тап по тексту удобнее, чем по узкому чекбоксу.
- Упоминаю, что для `textarea`, `select` и кастомных контролов связь обязательна по тем же правилам.
- Говорю про `aria-labelledby`, если визуальная подпись не `<label>` (но нативная связь предпочтительнее).
- Честно добавляю: при сложной вёрстке (label и input в разных flex-ячейках) вложение не сработает, нужен `for`/`id`.

## Как это работает

Браузер строит **accessible name** для контрола из связанной метки. При клике на label событие перенаправляется на ассоциированный элемент; для checkbox/radio переключается состояние.

Явная связь по id:

```html
<label for="user-email">Рабочая почта</label>
<input
  type="email"
  id="user-email"
  name="email"
  autocomplete="email"
  required
/>
```

Вложение без id:

```html
<label>
  Запомнить меня
  <input type="checkbox" name="remember" />
</label>
```

Во втором варианте первый вложенный labelable-элемент (`input`, `select`, `textarea`) получает связь автоматически. Если внутри несколько полей, поведение непредсказуемо: оставляют одно поле или переходят на `for`/`id`.

Для групп радиокнопок каждый `input` получает свой `label`; общий заголовок группы оформляют через `<fieldset>` + `<legend>`.

## Подводные камни

**Несовпадение for и id.** Опечатка в id ломает связь; в DevTools во вкладке Accessibility имя поля станет пустым.

**Дублирующиеся id.** Два input с одним id нарушают HTML и ломают привязку label.

**Label только для placeholder.** Placeholder не заменяет метку: он исчезает при вводе и часто не читается как имя поля.

**Кликабельный label с button внутри.** Вложенный `<button>` внутри label может перехватывать активацию; разносите элементы.

**Скрытый input.** Визуально кастомный чекбокс требует сохранить нативный input и связанный label, иначе клавиатура и AT не увидят контрол.
