# Relative paths: how the browser resolves files next to HTML

## In short

A relative path locates a resource from the current document without a domain or protocol. The browser merges it with the page URL, so `css/app.css` and `../img/icon.svg` work locally and on the server while folder structure stays the same.

## How to answer in an interview

- A path without `http://` and without a leading `/` is relative to the current file location.
- `styles/main.css` looks for a `styles` folder beside the HTML; `../` goes one level up.
- In CSS, `url()` resolves from the stylesheet file, not the HTML.
- Relative links simplify moving a project between domains and staging.
- I honestly add that in SPAs with the history API, relative paths use the address bar URL, which often breaks naive links on deep routes.

## How it works

Resolution takes the **base URL** (usually the page address) and applies path segments.

Layout:

```
app/
  index.html
  assets/logo.svg
  styles/theme.css
```

In `index.html`:

```html
<link rel="stylesheet" href="styles/theme.css" />
<img src="./assets/logo.svg" alt="Logo" />
```

In `theme.css`:

```css
.hero {
  background-image: url("../assets/logo.svg");
}
```

From CSS you leave `styles/`, enter `app/`, then `assets/`.

Bundlers (Vite, Webpack) may rewrite paths at build time to `/assets/hash.ext`, but source trees stay readable with relative links.

A relative path is not tied to a domain: `href="docs/guide.html"` works on `localhost:5173` and production with the same hierarchy.

## Pitfalls

**Different bases in CSS vs HTML.** `url(img/x.png)` in `/css/theme.css` requests `/css/img/x.png`.

**Deep SPA routes.** `/topics/css/grid` turns `href="favicon.ico"` into `/topics/css/favicon.ico`.

**Relative vs protocol-relative.** `//host/file` is not the same as `file` or `./file`.

**Characters and encoding.** Spaces and non-ASCII filenames need percent-encoding in URLs.

**Hard-coded paths in email.** Messages need absolute URLs; relatives fail there.
