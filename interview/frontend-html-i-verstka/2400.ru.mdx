# Структурированные данные: microdata, schema.org и JSON-LD

## Суть

Структурированные данные описывают сущности страницы (товар, статья, событие) в машиночитаемом виде. Schema.org задаёт общий словарь типов и свойств; microdata вшивает его в HTML-атрибуты, JSON-LD выносит описание в отдельный JSON-блок.

## Что ответить на собесе

- Объясняю цель: поисковики и ассистенты точнее понимают контент и могут показать rich results.
- Различаю форматы: microdata (itemscope/itemprop), RDFa, JSON-LD (`application/ld+json`).
- Называю типичные типы: `Product`, `Article`, `FAQPage`, `Organization`, `BreadcrumbList`.
- Уточняю, что JSON-LD сейчас предпочтительнее: не ломает вёрстку и проще генерировать на сборке.
- Честно добавляю: разметка не гарантирует сниппет; Google валидирует и может отклонить некорректные данные.

## Как это работает

**Schema.org** это открытый набор схем, согласованный крупными поисковиками. Вы выбираете `@type` и заполняете обязательные поля для выбранного rich result.

**Microdata** аннотирует живой DOM:

```html
<article itemscope itemtype="https://schema.org/Article">
  <h1 itemprop="headline">Гайд по flexbox</h1>
  <p itemprop="author" itemscope itemtype="https://schema.org/Person">
    <span itemprop="name">Анна К.</span>
  </p>
  <time itemprop="datePublished" datetime="2026-03-01">1 марта 2026</time>
</article>
```

Плюс: данные рядом с видимым текстом. Минус: смешение презентации и метаданных, сложнее шаблонизировать.

**JSON-LD** кладут в `<script type="application/ld+json">` в `<head>` или `<body>`:

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Беспроводные наушники Aero",
  "sku": "ARO-42",
  "offers": {
    "@type": "Offer",
    "price": "7990",
    "priceCurrency": "RUB",
    "availability": "https://schema.org/InStock"
  }
}
</script>
```

На фронте JSON-LD часто собирают на этапе SSR или prerender вместе с SEO-мета. В SPA без SSR блок можно вставлять при смене маршрута, но краулер должен увидеть финальный HTML.

Проверка через Rich Results Test и Search Console снижает риск ошибок в nested-объектах (`offers`, `aggregateRating`).

Rich snippets (звёзды, цена, FAQ-раскрывашки) появляются только при соответствии контенту на странице: «фейковый» рейтинг приводит к санкциям.

## Подводные камни

**Несоответствие видимому тексту.** Цена в JSON-LD другая, чем на экране.

**Устаревший http://schema.org в примерах.** Лучше https и актуальные типы.

**Дублирование microdata и JSON-LD.** Два противоречащих описания путают парсер.

**Отсутствие обязательных полей.** Product без `offers` или `image` не даст карточку.

**Генерация на клиенте без SSR.** Бот видит пустую страницу без LD-JSON.
