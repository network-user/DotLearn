# Семантические теги HTML5: полный набор для собеса

## Суть

HTML5 добавил теги, которые описывают роль блока: шапка, навигация, основной контент, статья, секция, боковая колонка, подвал, подпись к медиа, выделение, время и раскрывающиеся блоки. Они улучшают landmarks, SEO и поддержку кода.

## Что ответить на собесе

- Структурные: `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`.
- Медиа и текст: `figure`/`figcaption`, `mark`, `time` с `datetime`.
- Интерактив: `details`/`summary` для нативного accordion без JS.
- Подчёркиваю один `main` на страницу и заголовок у каждой `section`.
- Честно добавляю: список тегов без дисциплины в команде не даёт пользы, если всё равно верстают div-ами.

## Как это работает

Landmark-теги попадают в **дерево доступности**; скринридер предлагает быстрый переход по регионам.

```html
<header>
  <h1>DotLearn</h1>
  <nav aria-label="Каталог">
    <a href="/topics">Темы</a>
  </nav>
</header>
<main>
  <article>
    <header>
      <h2>CSS Grid на практике</h2>
      <time datetime="2026-03-01">1 марта 2026</time>
    </header>
    <p>Текст урока...</p>
    <figure>
      <img src="/grid.png" alt="Сетка из трёх колонок" />
      <figcaption>Равные колонки через repeat(3, 1fr)</figcaption>
    </figure>
    <p>Это <mark>ключевой</mark> приём для карточек.</p>
  </article>
  <aside>
    <h2>См. также</h2>
  </aside>
</main>
<footer>
  <details>
    <summary>Лицензия контента</summary>
    <p>CC BY-SA...</p>
  </details>
</footer>
```

`article` подходит для самостоятельного фрагмента (пост, карточка товара). `section` группирует тему с собственным заголовком. `aside` для tangential content: реклама, ссылки, виджет.

`time datetime` даёт машиночитаемую дату для микроразметки и сортировки. `details` открывается/закрывается кликом по `summary`, состояние можно стилизовать через `details[open]`.

## Подводные камни

**header/footer только в корне.** Их можно вкладывать в `article`/`section` для локальной шапки блока.

**nav на каждую ссылку.** `nav` для крупных блоков навигации, не для одной ссылки в футере.

**article для любого div.** Не каждый виджет является самостоятельной публикацией.

**details без fallback.** Старые браузеры покажут контент развёрнутым; критичный текст не прячу только внутрь.

**mark как подсветка поиска.** Семантика «важно в контексте», не замена `<strong>`.
