# Табличная разметка: thead, tbody, tfoot, th и caption

## Суть

Таблица в HTML предназначена для табличных данных, не для сетки страницы. Секции `thead`, `tbody`, `tfoot`, ячейки `th`/`td` и `caption` задают структуру, которую понимают скринридеры и браузер при печати.

## Что ответить на собесе

- Перечисляю роли: `caption` (заголовок таблицы), `thead` (шапка), `tbody` (данные), `tfoot` (итоги), `th` (заголовок ячейки), `td` (значение).
- Объясняю, что `th` несёт scope (`col`/`row`) для связи с данными.
- Упоминаю `colspan`/`rowspan` для объединения ячеек и необходимость не ломать логику заголовков.
- Говорю, что CSS `display: table` для лейаута страницы уступает Flex/Grid, а `<table>` оставляю для данных.
- Честно добавляю: на очень больших таблицах виртуализация рендера важнее идеальной семантики каждой строки.

## Как это работает

Браузер формирует **модель таблицы**: колонки выравниваются по ширине, `thead` может повторяться при печати, скринридер объявляет «заголовок столбца» при переходе по `td`.

Пример расписания:

```html
<table>
  <caption>Расписание воркшопов на март</caption>
  <thead>
    <tr>
      <th scope="col">Дата</th>
      <th scope="col">Тема</th>
      <th scope="col">Спикер</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>12.03</td>
      <td>CSS Grid</td>
      <td>А. Иванова</td>
    </tr>
    <tr>
      <td>19.03</td>
      <td>Доступность форм</td>
      <td>Б. Петров</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">Все сессии в Zoom, ссылка в письме</td>
    </tr>
  </tfoot>
</table>
```

`scope="col"` связывает заголовок с ячейками ниже в столбце. Для сложных шапок с объединёнными ячейками используют `headers`/`id` на `th` и `td`.

`tfoot` в разметке может стоять перед `tbody`, но визуально браузер отрисует его внизу. Это ускоряет отображение: итоги известны парсеру до конца тела.

## Подводные камни

**Таблица как лейаут.** Вложенные таблицы для вёрстки ломают навигацию AT и усложняют адаптив.

**Пустой thead.** Шапка из `td` вместо `th` лишает пользователя контекста столбца.

**Colspan без плана.** Случайное объединение ячеек сбивает связь заголовков с данными.

**Нет caption.** Таблица без названия хуже индексируется и хуже озвучивается («таблица без подписи»).

**Сотни строк в DOM.** Для аналитики с тысячами строк нужна виртуализация или другой UI-паттерн, не голый `<table>`.
