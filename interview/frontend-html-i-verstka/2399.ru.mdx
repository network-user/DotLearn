# Семантическая разметка HTML-формы

## Суть

Форма собирает пользовательский ввод через связку `<form>`, подписей `<label>`, контролов (`input`, `textarea`, `select`) и групп `<fieldset>` с `<legend>`. Правильные связи `for`/`id` и имена полей дают доступность, автозаполнение и предсказуемую отправку данных.

## Что ответить на собесе

- Описываю роль `<form>`: контейнер, `action`, `method`, событие submit.
- Объясняю `<label>`: клик по тексту фокусирует поле, скринридер озвучивает связь.
- Различаю однострочный `input`, многострочный `textarea`, выбор из списка `select`/`option`.
- Показываю `<fieldset>` + `<legend>` для логических групп (адрес, способ оплаты).
- Честно добавляю: в SPA часто `method` не используется, но семантика label и fieldset остаётся обязательной.

## Как это работает

**form** объявляет интерактивную область. Атрибуты `action` и `method` задают HTTP-цель; в React/Vue отправку обычно перехватывают, но нативный submit и валидация всё равно работают.

**label** связывается с контролом двумя способами: `for="email"` + `id="email"` или вложением `<label><input /></label>`. Второй вариок удобен для чекбоксов.

```html
<form action="/api/profile" method="post">
  <fieldset>
    <legend>Контактные данные</legend>

    <label for="email">Email</label>
    <input id="email" name="email" type="email" autocomplete="email" required />

    <label for="bio">О себе</label>
    <textarea id="bio" name="bio" rows="4" maxlength="500"></textarea>

    <label for="country">Страна</label>
    <select id="country" name="country">
      <option value="">Выберите...</option>
      <option value="ru">Россия</option>
      <option value="de">Германия</option>
    </select>
  </fieldset>

  <button type="submit">Сохранить</button>
</form>
```

**input** покрывает текст, пароль, чекбоксы, радио, файлы. Имя `name` попадает в `FormData` и query/body при submit.

**textarea** хранит многострочный текст; значение по умолчанию кладут между тегами, не в `value`.

**select** + **option** дают выпадающий список; `value` уходит на сервер, текст option видит пользователь.

**fieldset** группирует радиокнопки и связанные поля; **legend** озвучивается как заголовок группы в accessibility tree.

Для радиогруппы:

```html
<fieldset>
  <legend>Способ доставки</legend>
  <label><input type="radio" name="ship" value="pickup" /> Самовывоз</label>
  <label><input type="radio" name="ship" value="courier" /> Курьер</label>
</fieldset>
```

## Подводные камни

**label без связи.** Пользователь кликает текст, поле не фокусируется.

**Дублирующиеся id.** Связь label ломается, автотесты падают.

**Кнопка без type.** Внутри form `button` по умолчанию submit и случайно отправляет форму.

**placeholder вместо label.** Подсказка исчезает при вводе, a11y страдает.

**fieldset только ради CSS.** Без legend группа не озвучивается осмысленно.
