# Semantic markup for HTML forms

## In short

A form collects user input through `<form>`, captions `<label>`, controls (`input`, `textarea`, `select`), and groups `<fieldset>` with `<legend>`. Correct `for`/`id` links and field names enable accessibility, autofill, and predictable submission.

## How to answer in an interview

- I describe `<form>`: container, `action`, `method`, submit event.
- I explain `<label>`: clicking text focuses the control; screen readers announce the link.
- I separate single-line `input`, multiline `textarea`, and `select`/`option` pickers.
- I show `<fieldset>` + `<legend>` for logical groups (address, payment method).
- I add honestly: in SPAs `method` is often unused, but label and fieldset semantics still matter.

## How it works

A **form** declares an interactive region. `action` and `method` set the HTTP target; in React/Vue you usually intercept submit, but native submit and validation still apply.

A **label** links to a control via `for="email"` + `id="email"` or by nesting `<label><input /></label>`. Nesting fits checkboxes well.

```html
<form action="/api/profile" method="post">
  <fieldset>
    <legend>Contact details</legend>

    <label for="email">Email</label>
    <input id="email" name="email" type="email" autocomplete="email" required />

    <label for="bio">About you</label>
    <textarea id="bio" name="bio" rows="4" maxlength="500"></textarea>

    <label for="country">Country</label>
    <select id="country" name="country">
      <option value="">Choose...</option>
      <option value="ru">Russia</option>
      <option value="de">Germany</option>
    </select>
  </fieldset>

  <button type="submit">Save</button>
</form>
```

**input** covers text, password, checkboxes, radio, files. The `name` lands in `FormData` and the query/body on submit.

**textarea** holds multiline text; the default value goes between tags, not in `value`.

**select** + **option** provide a dropdown; `value` goes to the server, option text is shown to the user.

**fieldset** groups radio buttons and related fields; **legend** is announced as the group title in the accessibility tree.

For a radio group:

```html
<fieldset>
  <legend>Delivery method</legend>
  <label><input type="radio" name="ship" value="pickup" /> Pickup</label>
  <label><input type="radio" name="ship" value="courier" /> Courier</label>
</fieldset>
```

## Pitfalls

**Unlinked label.** Clicking copy does not focus the field.

**Duplicate ids.** Label association breaks and tests fail.

**Button without type.** Inside a form `button` defaults to submit and sends accidentally.

**placeholder instead of label.** Hint disappears on input and a11y suffers.

**fieldset only for CSS.** Without legend the group is not announced meaningfully.
