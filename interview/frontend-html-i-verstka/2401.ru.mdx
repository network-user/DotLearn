# SVG, JPEG и PNG: когда какой формат выбрать

## Суть

SVG хранит векторную графику в XML и масштабируется без пикселизации. JPEG сжимает фото с потерями и даёт малый вес. PNG хранит растр без потерь и поддерживает прозрачность альфа-канала.

## Что ответить на собесе

- Сравниваю природу: вектор (SVG) против растра (JPEG/PNG).
- Говорю, что SVG беру для иконок, логотипов, диаграмм; JPEG для фотографий; PNG для UI с прозрачностью и резкими краями.
- Упоминаю современные альтернативы: WebP/AVIF для фото, но базовые три формата всё ещё на собесах.
- Отмечаю влияние на перформанс: тяжёлый SVG с тысячами path хуже лёгкого PNG.
- Честно добавляю: один «универсальный» формата нет, решает контент и бюджет байтов.

## Как это работает

**SVG** описывает фигуры формулами. На ретина-экранах иконка 24px остаётся чёткой; цвет меняют через CSS `currentColor` или `fill`.

```html
<svg width="48" height="48" viewBox="0 0 48 48" aria-hidden="true">
  <circle cx="24" cy="24" r="18" fill="currentColor" />
</svg>
```

SVG уместен inline в React/Vue, в спрайтах и как `background-image`. Сложные иллюстрации из редактора иногда выгоднее растеризовать.

**JPEG** применяют к фотографиям и градиентам с плавными переходами. Качество задают параметром сжатия; повторное сохранение накапливает артефакты. Прозрачности нет.

**PNG** сохраняет каждый пиксель; PNG-8 для плоских палитр, PNG-24/32 для полупрозрачных теней и скриншотов. Файл тяжелее JPEG при том же размере кадра.

В `<picture>` можно отдать браузеру выбор:

```html
<picture>
  <source srcset="/hero.avif" type="image/avif" />
  <source srcset="/hero.webp" type="image/webp" />
  <img src="/hero.jpg" alt="Интерьер студии" width="1200" height="800" />
</picture>
```

На фронте важны `width`/`height` против CLS, lazy-loading для ниже сгиба и сжатие на CDN.

## Подводные камни

**JPEG для текста и UI.** Размытые края у кнопок и скриншотов.

**Огромный SVG из Figma.** Десятки килобайт path медленнее оптимизированного PNG.

**PNG для больших фото.** Мегабайты там, где JPEG/WebP в разы легче.

**Отсутствие alt у значимого img.** Формат не спасает доступность.

**Игнорирование vector security.** Inline SVG с пользовательским контентом требует санитизации.
