# HTML5 semantic tags: the full interview checklist

## In short

HTML5 added tags that describe block roles: header, navigation, main content, article, section, sidebar, footer, media captions, highlights, time, and disclosure widgets. They improve landmarks, SEO, and maintainability.

## How to answer in an interview

- Structural: `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`.
- Media and text: `figure`/`figcaption`, `mark`, `time` with `datetime`.
- Interactive: `details`/`summary` for a native accordion without JS.
- I stress one `main` per page and a heading for every `section`.
- I honestly add that knowing tag names helps little if the team still ships div soup.

## How it works

Landmark tags enter the **accessibility tree**; screen readers offer quick region jumps.

```html
<header>
  <h1>DotLearn</h1>
  <nav aria-label="Catalog">
    <a href="/topics">Topics</a>
  </nav>
</header>
<main>
  <article>
    <header>
      <h2>CSS Grid in practice</h2>
      <time datetime="2026-03-01">March 1, 2026</time>
    </header>
    <p>Lesson text...</p>
    <figure>
      <img src="/grid.png" alt="Three-column grid" />
      <figcaption>Equal columns via repeat(3, 1fr)</figcaption>
    </figure>
    <p>This is a <mark>key</mark> technique for cards.</p>
  </article>
  <aside>
    <h2>See also</h2>
  </aside>
</main>
<footer>
  <details>
    <summary>Content license</summary>
    <p>CC BY-SA...</p>
  </details>
</footer>
```

`article` fits standalone fragments (posts, product cards). `section` groups a topic with its own heading. `aside` is tangential: promos, links, widgets.

`time datetime` exposes a machine-readable date for markup and sorting. `details` toggles via `summary`; style open state with `details[open]`.

## Pitfalls

**header/footer only at root.** They can nest inside `article`/`section` for local block headers.

**nav on every link group.** Reserve `nav` for major navigation blocks, not a single footer link.

**article for any div.** Not every widget is a syndicatable publication.

**details without fallback.** Legacy browsers show content expanded; do not hide critical text only inside.

**mark as generic highlight.** It means "relevant in context", not a replacement for `<strong>`.
