# How to open a link in a new tab safely

## In short

The `target="_blank"` attribute on `<a>` opens a URL in a new tab or window without unloading the current page. For external links add `rel="noopener noreferrer"` so the new tab cannot access `window.opener`.

## How to answer in an interview

- I name `target="_blank"` + `rel="noopener noreferrer"` as the standard for external resources.
- I explain tabnabbing: a foreign page could replace the original tab's `location` via `opener`.
- I note modern browsers isolate opener by default, but explicit `rel` remains best practice.
- I cover UX: new tabs fit PDFs and external docs; internal navigation stays in the same tab.
- I add honestly: forcing `_blank` on every link annoys users and breaks "back" expectations.

## How it works

A hyperlink defaults (`target="_self"`) to replacing the current document in the same tab. **`_blank`** asks the browser for a new browsing context.

```html
<a
  href="https://docs.vendor.example/guide"
  target="_blank"
  rel="noopener noreferrer"
>
  Vendor documentation
</a>
```

`noopener` blocks the new tab from referencing the opener through `window.opener`. `noreferrer` also omits the Referer header and in some browsers disables opener as well.

In React/TSX pass the same attributes to `<a>`; internal router links usually skip `_blank`.

Other `target` values are rare on the frontend: `_parent`, `_top` appear with frames; named windows (`target="print"`) are uncommon.

For accessibility warn that the link is external: visible copy or `aria-label` "opens in a new tab", an icon with text, not title alone.

For downloads you may combine `download` (same-origin) or skip `_blank` when the user must return to a form.

## Pitfalls

**`_blank` without rel.** Older browsers and corporate policies remain exposed to opener attacks.

**`_blank` on every link.** Users lose control of navigation.

**Hidden external URL.** A new tab does not remove phishing responsibility.

**popup-blocker on window.open.** Programmatic open without a user gesture is blocked; `<a target="_blank">` from a click works.

**Button pretending to be a link.** `button` + `window.open` is worse for SEO and a11y than a real `<a>`.
