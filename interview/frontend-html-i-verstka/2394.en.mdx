# HTML document skeleton: doctype, html, head, and body

## In short

Every page is built from a document type declaration, the root `<html>`, a metadata block `<head>`, and the visible `<body>`. `DOCTYPE` switches the browser into standards rendering mode and sets the expected markup version.

## How to answer in an interview

- I list four layers: `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>` and the role of each.
- I stress that without a correct doctype you may get quirks mode and unstable layout.
- I mention `lang` on `<html>` for SEO, screen reader pronunciation, and font selection.
- I name typical head content: `title`, `meta charset`, `viewport`, styles, scripts.
- I add honestly: doctype does not validate HTML; it only tells the browser which parsing mode to use.

## How it works

The parser starts with **DOCTYPE**. In HTML5 the short form `<!DOCTYPE html>` is enough without a public identifier. The browser knows the document follows modern rules and does not emulate legacy Netscape bugs.

The root **`<html>`** wraps the whole tree. `lang="en"` sets the default page language: it affects VoiceOver/NVDA pronunciation and selectors like `:lang()`.

**`<head>`** is not painted on screen but drives tab behavior and resource loading:

- `<title>` appears on the tab and in search snippets.
- `<meta charset="utf-8">` should appear as early as possible.
- `<meta name="viewport" ...>` is required for responsive layout.
- `<link rel="stylesheet">` and `<script>` load CSS and JS; critical JS sometimes lives in head with `defer`.

**`<body>`** holds all visible UI: headings, text, media, forms. Semantic sections (`header`, `main`, `footer`) live here.

Minimal page skeleton:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Product catalog</title>
    <link rel="stylesheet" href="/styles/base.css" />
  </head>
  <body>
    <header><h1>Store</h1></header>
    <main><p>Page content</p></main>
  </body>
</html>
```

Without a doctype the same CSS can yield different `body` margins and `box-sizing` behavior because the engine enables compatibility mode.

## Pitfalls

**Missing DOCTYPE.** The page "breaks" mainly in older corporate browsers or when pasting fragments without a template.

**Wrong order in head.** `charset` after a large script delays encoding detection.

**Mismatched title and h1.** Empty or technical `<title>` hurts tab accessibility and search.

**Scripts without defer/async in head.** They block first paint even though the issue is resource placement, not document structure.

**Missing lang.** Screen readers pronounce English text with the wrong voice and vice versa.
