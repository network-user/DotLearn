# Семантическая разметка: зачем теги несут смысл, а не только стиль

## Суть

Семантическая вёрстка выбирает HTML-теги по роли контента, а не по внешнему виду. Браузер, поисковик и скринридер получают явную структуру документа: шапка, основной блок, статья, навигация, подвал.

## Что ответить на собесе

- Объясняю, что `<div class="header">` рисует блок, а `<header>` сообщает машинам «это шапка страницы».
- Перечисляю выигрыши: a11y (landmarks, озвучка), SEO, читаемость кода для команды.
- Привожу набор тегов: `header`, `nav`, `main`, `article`, `section`, `aside`, `footer`.
- Уточняю, что семантика не заменяет ARIA, но снижает потребность в костылях.
- Честно добавляю: без единых договорённостей в команде семантика быстро превращается в `<section>` на каждый блок.

## Как это работает

Парсер строит **дерево доступности** из DOM. Семантический тег попадает в роль landmark: скринридер предлагает прыгнуть к `main` или `nav`. Поисковик использует иерархию заголовков и обёртки `article` для выделения самостоятельного контента.

Типичный каркас лендинга:

```html
<body>
  <header>
    <a href="/">DotLearn</a>
    <nav aria-label="Основное меню">
      <ul>
        <li><a href="/topics">Темы</a></li>
        <li><a href="/progress">Прогресс</a></li>
      </ul>
    </nav>
  </header>
  <main>
    <article>
      <h1>Как учить фронтенд</h1>
      <p>Текст урока...</p>
    </article>
    <aside>
      <h2>Похожие темы</h2>
    </aside>
  </main>
  <footer>
    <p>© 2026 DotLearn</p>
  </footer>
</body>
```

**`<header>`** может жить и внутри `article` (заголовок поста). **`<section>`** группирует тематический фрагмент с собственным заголовком; пустой `section` без `h2`/`h3` теряет смысл. **`<article>`** подходит для RSS-элемента: его можно вынести в другой контекст без потери смысла.

CSS по-прежнему задаёт внешний вид: семантика отвечает за **интерпретацию**, не за пиксели.

## Подводные камни

**Div-суп.** Замена всех тегов на `<div>` ломает landmarks; приходится дублировать `role` и `aria-label`.

**Section-спам.** Каждый виджет в отдельном `section` без заголовка не даёт навигации ни людям, ни AT.

**Неверный main.** Два `main` на странице или `main` внутри `aside` нарушают спецификацию и путают скринридеры.

**Article vs section.** Статья в блоге — `article`; блок «Наши преимущества» на лендинге чаще `section`, не `article`.

**Семантика без иерархии заголовков.** Правильные теги не спасут, если уровни `h1`–`h6` скачут или дублируются.
