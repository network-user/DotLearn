# What the metadata object "modified" status means

## In short

The "modified" status in the configuration tree means the metadata object's structure was edited in the configurator but not yet applied to the database. Until configuration database update runs, the physical table and index schema does not match the metadata.

## How to answer in an interview

- I explain the difference between saved configuration in the metadata repository and the actual SQL structure.
- In an interview I say a yellow/modified object requires "Update database configuration".
- I stress that code can reference a new attribute before the column exists in the DBMS.
- I honestly note that in file mode and Git export the status appears differently in EDT vs configurator, but the idea is the same.
- Before updating production I test on a copy, especially when removing attributes.

## How it works

You add attribute `Производитель` to catalog "Товары". The object is marked modified in the tree. Users in 1C:Enterprise mode do not see the field in data yet; queries to the new column fail.

Action sequence:

1. Save configuration (Ctrl+S).
2. Configuration → Update database configuration.
3. Accept the change plan and wait for table restructure.

```bsl
Процедура ПроверитьНаличиеРеквизита()
    Мета = Метаданные.Справочники.Товары.Реквизиты;
    Если Мета.Найти("Производитель") <> Неопределено Тогда
        Сообщить("Реквизит есть в метаданных");
    КонецЕсли;
КонецПроцедуры
```

Checking metadata does not guarantee the column is already in SQL; confirm via database update result and the event log during restructure.

## Pitfalls

- **Schema mismatch.** Running a report with a new field before DB update yields "field not found".
- **User lockout.** Long restructure on a large base needs a maintenance window.
- **Attribute removal.** Data in the column is lost without a separate migration.
- **Multiple developers.** One updated the DB while another uses the old schema; align rollout rules.
- **Extensions.** Extension changes need their own update; main configuration status does not reflect them.
