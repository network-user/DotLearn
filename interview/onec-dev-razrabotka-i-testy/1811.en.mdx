# Watch expressions in the 1C debugger

## In short

Watch expressions are a debugger panel where you list variables and arbitrary BSL expressions. When stopped at a breakpoint or stepping, the platform recomputes them and shows current values without changing code.

## How to answer in an interview

- I set a breakpoint, add `Сумма`, `Элемент`, `Массив.Количество()` to the watch, and step through.
- In an interview I distinguish watch from the Evaluate Expression window for one-off checks.
- I stress expressions are evaluated in the context of the current module and call stack.
- I honestly note heavy expressions with queries in watch slow debugging down.
- For collections I inspect nested fields via dot notation or index.

## How it works

A function sums an array; debug on the accumulation line:

```bsl
Функция СуммаМассива(ИсходныйМассив)
    Итог = 0;
    Для Каждого ЗначениеЭлемента Из ИсходныйМассив Цикл
        Итог = Итог + ЗначениеЭлемента;
    КонецЦикла;
    Возврат Итог;
КонецФункции
```

In the watch panel add `Итог`, `ЗначениеЭлемента`, `ИсходныйМассив.Количество()`. Each loop step shows how the sum grows and which element is processed. If the result is wrong, the mismatch is visible on a specific iteration.

For server code attach the debugger to the server session; watch behaves the same but respects server context of the form or common module.

## Pitfalls

- **Side effects.** Calling a procedure that writes to the infobase inside watch changes data.
- **Unavailable context.** Expressions from another module without exported variables will not evaluate.
- **Platform optimization.** In a release build without debugging, watch is unavailable by design.
- **Stale value.** After continue without step, watch updates only on the next stop.
- **Large value tables.** Expanding a huge value table in watch freezes the UI.
