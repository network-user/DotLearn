# Object auto-numbering: where to configure and how it works

## In short

Auto-numbering in 1C assigns unique numbers to documents and other numbered objects on creation. Parameters are set in the configurator under the metadata object's Numbering properties: length, prefix, periodicity, and uniqueness within a period.

## How to answer in an interview

- I explain that auto-numbering is enabled at the metadata level, not in every document's code.
- In an interview I list settings: number length, prefix, uniqueness by year/month/day.
- I stress that the number is assigned on write, not when the form opens.
- I honestly note that manual entry or numbering failures can cause conflicts; handlers and uniqueness checks catch those.
- For non-standard numbering schemes I add logic in the object or manager module.

## How it works

In the configurator, open the document properties, the Numbering tab, enable auto-numbering, and set the format. On programmatic creation the platform assigns the next number within the rules:

```bsl
&НаСервере
Процедура СоздатьЗаказПокупателя()
    НовыйЗаказ = Документы.ЗаказыПокупателей.СоздатьДокумент();
    НовыйЗаказ.Дата = ТекущаяДатаСеанса();
    НовыйЗаказ.Записать();
КонецПроцедуры
```

After `Записать()`, the `Номер` field is filled per metadata settings. A prefix helps distinguish document kinds in one infobase; periodicity resets the counter at year or month boundaries. For a custom template, override number retrieval in the manager module, but the base mechanism stays the same.

## Pitfalls

- **Uniqueness conflicts.** Two sessions can get the same number with weak settings; use transactional writes and correct periodicity.
- **Manual entry.** Users can type a number and break the sequence; restrict rights and validate in `ПередЗаписью`.
- **Data migration.** Numbers from external systems may not fit the format; plan migration separately.
- **Prefixes and length.** Changing number length after data accumulates requires careful conversion of existing records.
- **Exchange and branches.** In distributed infobases numbering must be aligned across nodes or duplicates appear.
