# Какие типы модулей есть в конфигурации 1С

## Суть

Код в 1С разнесён по модулям с разным контекстом выполнения: общие, объектные, менеджера, формы, команды, сеанса, приложения, внешние обработки. Выбор модуля определяет доступность клиент/сервер, событий и области видимости процедур.

## Что ответить на собесе

- Я перечисляю минимум семь типов и связываю каждый с местом вызова.
- На собеседовании объясняю разницу модуля объекта (экземпляр) и модуля менеджера (тип целиком).
- Подчёркиваю флаги общего модуля: сервер, клиент, внешнее соединение, привилегированный.
- Честно говорю: модуль формы делится директивами `&НаКлиенте` / `&НаСервере`.
- Для переиспользуемой логики без привязки к метаданным выбираю общий модуль.

## Как это работает

Общий модуль `РаботаСоСтроками` с флагом «Сервер»:

```bsl
Функция НормализоватьКод(ИсходныйКод) Экспорт
    Возврат ВРег(СокрЛП(ИсходныйКод));
КонецФункции
```

Модуль менеджера справочника содержит фабричные методы:

```bsl
Функция НайтиПоАртикулу(Артикул) Экспорт
    Возврат Справочники.Номенклатура.НайтиПоРеквизиту("Артикул", Артикул);
КонецФункции
```

Модуль объекта документа реагирует на события экземпляра:

```bsl
Процедура ОбработкаПроведения(Отказ, Режим)
    СформироватьДвиженияПоСкладу(Отказ);
КонецПроцедуры
```

Модуль формы обрабатывает UI-события:

```bsl
&НаКлиенте
Процедура СуммаПриИзменении(Элемент)
    РассчитатьИтогНаСервере();
КонецПроцедуры
```

Модули `Сеанса`, управляемого и обычного приложения инициализируют параметры сеанса и глобальные обработчики старта. Модуль команды связывает кнопку интерфейса с процедурой. Модуль набора записей регистра сведений обслуживает табличную часть набора.

## Подводные камни

- **Общий модуль без нужного контекста.** Вызов с клиента процедуры «только сервер».
- **Бизнес-логика в модуле формы.** Дублирование и невозможность вызова из регламента.
- **Проведение в модуле менеджера.** Событие живёт в модуле объекта.
- **Привилегированный модуль без необходимости.** Обход RLS.
- **Путаница модуля объекта и формы.** `ПриЗаписи` на форме не заменяет объектный обработчик.
