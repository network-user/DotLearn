# Разница между position relative, absolute и fixed

## Суть

`relative` смещает элемент от normal position, место в потоке сохраняется и создаёт positioning context. `absolute` вырывает из потока и позиционируется относительно ближайшего non-static предка. `fixed` якорится к viewport (или transform ancestor) и не скроллится с page.

## Что ответить на собесе

- relative: offset без collapse flow slot; часто якорь для absolute children.
- absolute: tooltip/popover внутри `.shell { position:relative }`.
- fixed: sticky header, FAB; учитываю mobile viewport units (`100dvh`).
- sticky это отдельное значение: relative до threshold, потом fixed-like.
- Честно: `fixed` внутри modal с `transform` на предке не к viewport.

## Как это работает

```css
.shell {
  position: relative;
  padding: 1rem;
  border: 1px solid #cbd5e1;
}

.badge {
  position: absolute;
  top: 0.5rem;
  right: 0.5rem;
}

.nudge {
  position: relative;
  top: 0.25rem;
  left: 0.25rem;
}

.fab {
  position: fixed;
  bottom: 1rem;
  right: 1rem;
  z-index: 40;
}
```

```ts
const fab = document.querySelector<HTMLElement>(".fab");

window.addEventListener(
  "scroll",
  () => {
    if (!fab) return;
    fab.dataset.scrolled = String(window.scrollY > 120);
  },
  { passive: true }
);
```

## Подводные камни

**absolute без positioned parent.** Элемент уезжает к viewport или `<html>`.

**fixed + overflow hidden body.** iOS Safari address bar сдвигает `bottom` inset.

**relative только ради z-index.** Иногда достаточно isolation/isolation stack.

**stacking context.** fixed не всегда «поверх всего» без z-index discipline.

**print styles.** fixed header дублируется на каждой странице PDF.
