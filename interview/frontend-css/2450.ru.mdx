# За что отвечает align-content во flex и grid

## Суть

`align-content` распределяет свободное место между flex-lines или grid tracks по поперечной оси, когда строк больше одной и в контейнере есть extra space. На одиночную линию не влияет; для элементов внутри линии есть `align-items`.

## Что ответить на собесе

- Работает только при `flex-wrap: wrap` (или multi-row grid) и лишней высоте/ширине контейнера.
- Значения как у `justify-content`, но по cross axis: `center`, `space-between`, `stretch` и т.д.
- Путают с `align-items`: items внутри одной линии vs сами линии как группа.
- В grid `align-content` двигает track bands, не отдельные cells (если не align-self).
- Честно: если контейнер без фиксированной cross size, extra space может не появиться и свойство «не видно».

## Как это работает

```css
.dashboard {
  display: flex;
  flex-wrap: wrap;
  align-content: space-between;
  gap: 0.75rem;
  height: 20rem;
  padding: 1rem;
}

.widget {
  flex: 1 1 8rem;
  min-height: 5rem;
  border-radius: 10px;
  background: #f1f5f9;
}
```

Grid вариант:

```css
.board {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: 4rem;
  align-content: center;
  height: 24rem;
  gap: 0.5rem;
}
```

```ts
const board = document.querySelector<HTMLElement>(".board");
if (board) {
  board.style.alignContent = "space-evenly";
}
```

Схема: несколько линий внутри контейнера с gap между линиями, не между items в одной линии (это gap/align-items).

## Подводные камни

**align-content без wrap.** В flex однострочник свойство не проявляется.

**height:auto родителя.** Нет свободной cross size, align-content noop.

**Смешение justify и align.** В column flex main/cross оси меняются местами.

**grid align-content vs place-content shorthand.** `place-content` перезаписывает оба.

**Safari старые баги multi-line flex.** Проверять space-* значения на iOS.
