# CSS-переменные: объявление, var() и темизация

## Суть

Пользовательские свойства объявляют как `--token-name: value` и читают через `var(--token-name)`. Они наследуются по DOM и живут в рантайме, в отличие от переменных препроцессора.

## Что ответить на собесе

- Синтаксис: `--color-accent: #3b82f6;` внутри селектора, чаще в `:root` для глобальных токенов.
- Использование: `color: var(--color-accent);` с fallback `var(--x, #000)`.
- Темы: переопределяю переменные на `[data-theme="dark"]` или `.dark`, не дублирую все правила.
- Переменные можно анимировать и передавать в calc()/clamp().
- Честно добавляю: нельзя использовать в селекторах и media queries как имя; для брейкпоинтов остаются обычные @media.

## Как это работает

Токены дизайн-системы в корне:

```css
:root {
  --space-md: 1rem;
  --font-body: system-ui, sans-serif;
  --surface: #ffffff;
  --text: #0f172a;
}

[data-theme="dark"] {
  --surface: #0f172a;
  --text: #e2e8f0;
}

.card {
  font-family: var(--font-body);
  padding: var(--space-md);
  background: var(--surface);
  color: var(--text);
  border-radius: var(--radius, 12px);
}
```

`var(--radius, 12px)` подставляет запасное значение, если `--radius` не задан.

Переменные **наследуются**: дочерний элемент видит переопределение на родителе. Для локального скоупа объявляю токены на компоненте:

```css
.metric {
  --bar-color: #22c55e;
}

.metric__bar {
  background: var(--bar-color);
}
```

В JS можно менять тему: `document.documentElement.style.setProperty('--accent', userColor)`.

## Подводные камни

**Путаница с Sass $vars.** `$primary` подставляется на этапе сборки; `--primary` доступен в браузере и в DevTools.

**Тип без единиц в calc.** `calc(var(--gap) + 4px)` требует, чтобы `--gap` включал единицу (`8px`, не `8`).

**Наследование нежелательных значений.** Забытый reset токена на вложенном блоке тянет цвет родителя.

**Производительность глубоких деревьев.** Тысячи элементов с уникальными inline custom properties усложняют стиль-рекалькуляцию.

**Invalid at computed value.** Ошибка в `var()` наследуется; fallback спасает только при отсутствии свойства, не при невалидном значении.
