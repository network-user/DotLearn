# Разница между :focus-visible и :focus-within

## Суть

`:focus-visible` стилизует элемент, когда браузер считает фокус нужным показать (обычно keyboard), скрывая ring после mouse click. `:focus-within` подсвечивает предка, если фокус внутри любого потомка.

## Что ответить на собесе

- `:focus` всегда при фокусе; `:focus-visible` убирает лишний outline после клика мышью.
- `:focus-within` на `.field` когда активен input внутри, удобно для групп форм.
- Не удаляю outline глобально; заменяю на `:focus-visible` стиль.
- Комбинирую с `:focus-within` для card border, не только input ring.
- Честно: старые браузеры без `:focus-visible` нужен polyfill или fallback `:focus`.

## Как это работает

```css
.btn {
  border: none;
  border-radius: 8px;
  padding: 0.5rem 1rem;
  background: #2563eb;
  color: #fff;
}

.btn:focus {
  outline: none;
}

.btn:focus-visible {
  outline: 2px solid #fbbf24;
  outline-offset: 2px;
}

.field:focus-within {
  border-color: #2563eb;
  box-shadow: 0 0 0 3px rgb(37 99 235 / 25%);
}
```

```html
<label class="field">
  Email
  <input type="email" autocomplete="email" />
</label>
```

```ts
document.addEventListener("keydown", (event) => {
  if (event.key === "Tab") {
    document.body.classList.add("kbd-nav");
  }
});
```

```css
body:not(.kbd-nav) .btn:focus:not(:focus-visible) {
  outline: none;
}
```

## Подводные камни

**outline:none без замены.** Ломает keyboard a11y и WCAG.

**focus-within на большом container.** Вся карточка мигает при tab через 20 controls.

**Custom element shadow DOM.** Focus inside shadow не всегда bubble для :focus-within light DOM.

**Programmatic focus.** `focus()` может не показать :focus-visible в некоторых UA.

**Путаница с :has().** `:has(:focus)` ≠ `:focus-within` по support и semantics.
