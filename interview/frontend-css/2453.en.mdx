# What values does CSS position support

## In short

Five main values: `static` (default), `relative`, `absolute`, `fixed`, `sticky`. They define flow participation, offset reference, and whether a containing block is created for descendants.

## How to answer in an interview

- static: normal flow, offsets ignored.
- relative: offset + positioning context, flow slot kept.
- absolute/fixed: out of flow; absolute from non-static ancestor, fixed from viewport (with transform caveats).
- sticky: relative until scroll threshold, then sticks in the scroll container.
- Honestly: `position: sticky` breaks with `overflow:hidden` on an ancestor.

## How it works

```css
.toolbar { position: sticky; top: 0; z-index: 10; }
.drawer { position: fixed; inset: 0; }
.popover {
  position: absolute;
  top: calc(100% + 0.25rem);
  right: 0;
}
.anchor { position: relative; }
.badge { position: absolute; top: -0.25rem; right: -0.25rem; }
```

```ts
type PositionMode =
  | "static"
  | "relative"
  | "absolute"
  | "fixed"
  | "sticky";

function setPosition(el: HTMLElement, mode: PositionMode) {
  el.style.position = mode;
}
```

Behavior sketch:

```
static   · flow, no offsets
relative · flow + offset + context
absolute · out of flow, ancestor anchor
fixed    · out of flow, viewport anchor
sticky   · hybrid within scrollport
```

## Pitfalls

**sticky without top/left.** No threshold, sticky never activates.

**fixed inside transformed modal.** Containing block is not the viewport.

**absolute height collapse.** Parent without height collapses.

**z-index only on positioned elements.** static ignores z-index.

**sticky vs fixed header confusion.** sticky is section-scoped, fixed is global.
