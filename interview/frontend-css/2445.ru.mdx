# Какой селектор побеждает при конфликте стилей

## Суть

Победитель определяется цепочкой: importance (`!important`), затем специфичность селектора, затем порядок объявления в cascade. Inline style сильнее класса, ID сильнее одного класса, но слабее `!important` в sheet при равных условиях.

## Что ответить на собесе

- Сравниваю веса (inline, id, class/pseudo/attr, type/pseudo-element) по компонентам.
- При равной специфичности побеждает правило, объявленное позже.
- `!important` author > normal author; user `!important` (accessibility) выше author.
- Не лечу конфликты ещё одним длинным селектором; понижаю сложность или layers.
- Честно: два одинаковых веса из разных bundle порядок зависит от порядка `<link>`.

## Как это работает

```html
<p id="badge" class="tag warn">Статус</p>
```

```css
.warn { color: #b45309; }
#badge { color: #1d4ed8; }
p { color: #64748b; }
```

Победит `#badge` (0,1,0,1) над `.warn` (0,0,1,0).

```ts
const badge = document.querySelector<HTMLElement>("#badge");
badge?.style.setProperty("color", "#7c3aed");
```

Inline перекрывает stylesheet до `!important`:

```css
.tag { color: #0f766e !important; }
```

@layer для контролируемого порядка без гонки специфичности:

```css
@layer base, components;

@layer base {
  .tag { color: #64748b; }
}

@layer components {
  .tag { color: #0f172a; }
}
```

## Подводные камни

**Гонка специфичности в legacy.** Каждый фикс добавляет `#id .class`.

**!important как default.** Потом невозможно переопределить theme toggle.

**:is() берёт max.** `:is(#a, .b)` вес как у `#a`.

**Сторонний widget CSS.** Загружается последним и «ломает» brand colors.

**Путаница source order vs specificity.** Новички добавляют `!important`, хотя достаточно порядка.
