# Difference between relative, absolute, and fixed positioning

## In short

`relative` offsets from normal position, keeps flow space, and creates a positioning context. `absolute` leaves flow and positions against the nearest non-static ancestor. `fixed` anchors to the viewport (or a transform ancestor) and does not scroll with the page.

## How to answer in an interview

- relative: offset without collapsing flow slot; often an anchor for absolute children.
- absolute: tooltip/popover inside `.shell { position:relative }`.
- fixed: sticky header, FAB; account for mobile viewport units (`100dvh`).
- sticky is separate: relative until threshold, then fixed-like.
- Honestly: `fixed` inside a modal with `transform` on an ancestor is not viewport-fixed.

## How it works

```css
.shell {
  position: relative;
  padding: 1rem;
  border: 1px solid #cbd5e1;
}

.badge {
  position: absolute;
  top: 0.5rem;
  right: 0.5rem;
}

.nudge {
  position: relative;
  top: 0.25rem;
  left: 0.25rem;
}

.fab {
  position: fixed;
  bottom: 1rem;
  right: 1rem;
  z-index: 40;
}
```

```ts
const fab = document.querySelector<HTMLElement>(".fab");

window.addEventListener(
  "scroll",
  () => {
    if (!fab) return;
    fab.dataset.scrolled = String(window.scrollY > 120);
  },
  { passive: true }
);
```

## Pitfalls

**absolute without positioned parent.** Element attaches to viewport or `<html>`.

**fixed + overflow hidden body.** iOS Safari URL bar shifts `bottom` inset.

**relative only for z-index.** Sometimes `isolation` is enough.

**stacking context.** fixed is not "above everything" without z-index discipline.

**print styles.** fixed headers repeat on every PDF page.
