# За что отвечает flex-wrap

## Суть

`flex-wrap` решает, могут ли flex-элементы перейти на новую линию (или колонку при `flex-direction: column`), когда суммарный flex-basis не помещается в контейнер. Значения: `nowrap`, `wrap`, `wrap-reverse`.

## Что ответить на собесе

- По умолчанию `nowrap`: элементы сжимаются (`flex-shrink`), overflow возможен.
- `wrap` создаёт дополнительные flex-lines; `align-content` выравнивает уже линии, не один item.
- `wrap-reverse` меняет порядок линий, не зеркалит элементы внутри строки.
- Для карточной сетки комбинирую `flex-wrap: wrap` + `flex: 1 1 <basis>` + `gap`.
- Честно: для строгих двумерных сеток Grid предсказуемее, wrap х хорош для «резиновых» рядов.

## Как это работает

```css
.gallery {
  display: flex;
  flex-wrap: wrap;
  gap: 0.75rem;
  width: 320px;
}

.thumb {
  flex: 1 1 96px;
  max-width: 120px;
  aspect-ratio: 1;
  border-radius: 8px;
  background: #e2e8f0;
  display: grid;
  place-items: center;
}
```

```ts
const gallery = document.querySelector<HTMLElement>(".gallery");
const items = gallery?.querySelectorAll<HTMLElement>(".thumb");

console.log(items?.length ?? 0);
```

## Подводные камни

**align-items vs align-content.** На многострочном контейнере нужен `align-content`, иначе линии слипаются.

**flex-shrink: 0 + nowrap.** Горизонтальный overflow без scroll container.

**wrap-reverse и tab order.** Визуальный порядок не совпадает с DOM focus order.

**Процентный flex-basis.** Процент считается от контейнера, wrap меняет доступную ширину линии.

**Ожидание равных рядов.** Wrap не выравнивает «кирпичную кладку» без фиксированного basis.
