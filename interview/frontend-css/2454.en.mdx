# How fixed differs from sticky

## In short

`fixed` always anchors to the viewport (or a transform ancestor) and does not scroll with the page. `sticky` acts like relative inside a scroll container until it hits an offset (`top`), then sticks until the container ends.

## How to answer in an interview

- fixed: global header/FAB; leaves document flow from frame one.
- sticky: section heading or table thead inside a scrollport.
- sticky needs `top`/`bottom` and no `overflow:hidden` on the ancestor chain.
- fixed overlays content; sticky keeps flow space until it sticks.
- Honestly: older iOS sticky bugs in nested scroll; test real devices.

## How it works

```css
.site-header {
  position: fixed;
  top: 0;
  inset-inline: 0;
  z-index: 50;
}

.article {
  overflow: auto;
  max-height: 70vh;
}

.article h2 {
  position: sticky;
  top: 0;
  background: #fff;
  padding-block: 0.5rem;
}
```

```ts
const header = document.querySelector<HTMLElement>(".site-header");
const height = header?.getBoundingClientRect().height ?? 0;
document.documentElement.style.setProperty(
  "--header-h",
  `${height}px`
);
```

```css
main {
  padding-top: var(--header-h, 3.5rem);
}
```

fixed sits outside the section; sticky releases when `.article` leaves the viewport.

## Pitfalls

**sticky without scrollport height.** Nothing to stick within.

**overflow:hidden on body/html.** Breaks the sticky chain.

**fixed + 100vh on mobile.** URL bar jumps; use `dvh`.

**Two sticky elements with same top.** Overlap without a z-index plan.

**IntersectionObserver instead of sticky.** Sometimes simpler for complex tables.
