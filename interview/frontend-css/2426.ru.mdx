# CSS Modules: изоляция классов, плюсы и ограничения

## Суть

CSS Modules компилируют локальные классы в уникальные имена при сборке. Импорт `styles.title` в TS/React даёт строку вроде `Hero_title_x3k9`, конфликты глобальных имён исчезают без runtime CSS-in-JS.

## Что ответить на собесе

- Файл `*.module.css`; только явно экспортированные классы доступны в JS.
- Глобальные стили через `:global(.reset)` или отдельный `global.css`.
- Композиция: `composes: btn from './Button.module.css'`.
- TypeScript: генерация `*.module.css.d.ts` для автодополнения.
- Честно добавляю: динамические классы только через шаблонные строки с известными ключами, произвольные имена из API в Modules не сработают.

## Как это работает

`TopicCard.module.css`:

```css
.root {
  display: grid;
  gap: 0.75rem;
  padding: 1rem;
  border-radius: 12px;
}

.title {
  font-size: 1.125rem;
  font-weight: 600;
}

:global(.dark) .root {
  background: #0f172a;
}
```

`TopicCard.tsx`:

```ts
import styles from "./TopicCard.module.css";

export function TopicCard({ title }: { title: string }) {
  return (
    <article className={styles.root}>
      <h3 className={styles.title}>{title}</h3>
    </article>
  );
}
```

Сборщик (Vite css modules) переименует `.root` в уникальный идентификатор. Tree-shaking неиспользуемых классов зависит от плагина.

Условный модификатор:

```ts
const variant = done ? styles.done : styles.pending;
return <span className={`${styles.badge} ${variant}`}>...</span>;
```

## Подводные камни

**Селекторы по тегу без локального класса.** `div { }` в module всё равно локализуется странно; стилизую через класс.

**:global без дисциплины.** Утечка глобальных правил возвращает конфликты имён.

**Третьи библиотеки.** Bootstrap классы не проходят через module; оборачиваю или использую wrapper.

**composes и порядок CSS.** Зависимость от порядка chunk в extract CSS может менять каскад.

**SSR + HMR.** Рассинхрон имён классов между server HTML и client при hot reload в dev (известный dev-only грабль).
