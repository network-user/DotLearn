# От чего зависят размеры блочного элемента

## Суть

Итоговая «коробка» складывается из content, padding, border и иногда margin в потоке. На неё влияют явные width/height, box-sizing, содержимое, проценты от родителя и formatting context.

## Что ответить на собесе

- Блок по умолчанию тянется на 100% ширины родителя; высота от контента, если не задана.
- `box-sizing: border-box` включает padding и border в width/height; `content-box` добавляет их снаружи.
- Проценты по ширине считаются от ширины containing block; по высоте работают только если у предка есть определённая высота.
- min/max-width/height, overflow и float/flex/grid меняют фактический размер.
- Честно: margin collapse не увеличивает border-box, но визуально «съедает» вертикальные отступы между соседями.

## Как это работает

Карточка в контейнере с border-box:

```css
.shell {
  width: min(100%, 360px);
  margin-inline: auto;
}

.panel {
  width: 100%;
  padding: 1rem;
  border: 2px solid #334155;
  box-sizing: border-box;
  background: #f8fafc;
}
```

```ts
const shell = document.querySelector<HTMLElement>(".shell");
const panel = document.querySelector<HTMLElement>(".panel");

if (shell && panel) {
  const outer = panel.getBoundingClientRect().width;
  const content = panel.clientWidth;
  console.log({ outer, content });
}
```

Схема box model (border-box):

<figure aria-label="width покрывает content padding и border">
<svg viewBox="0 0 200 100" width="200" height="100" fill="none" stroke="currentColor" stroke-width="1.2">
  <rect x="10" y="15" width="180" height="70" rx="3"/>
  <rect x="28" y="33" width="144" height="34" rx="2" stroke-dasharray="4 3"/>
  <text x="100" y="12" text-anchor="middle" font-size="9" fill="currentColor">width</text>
  <text x="100" y="54" text-anchor="middle" font-size="9" fill="currentColor">content</text>
</svg>
</figure>

В flex-ряду `flex-basis` и `min-width: 0` часто важнее сырого `width: 100%`.

## Подводные камни

**height: 100% без высоты предка.** Ребёнок остаётся «auto» по факту.

**width: 100% + padding в content-box.** Горизонтальный скролл на мобиле.

**min-content и длинные слова.** Блок раздувается шире viewport без `overflow-wrap`.

**Margin collapse.** Соседние блоки «склеивают» вертикальные margin, отладка путает.

**Замена width на flex.** `width: 100%` внутри flex-item ведёт себя иначе, чем в block flow.
