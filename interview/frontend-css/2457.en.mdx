# CSS pseudo-elements: purpose and common types

## In short

Pseudo-elements (`::before`, `::after`, `::first-line`, `::marker`, etc.) style parts of a box or add decorative content without extra HTML. Double-colon `::` syntax distinguishes them from pseudo-classes.

## How to answer in an interview

- `::before`/`::after` need `content` (even `""`) to render.
- Decorative icons and dividers belong in pseudo-elements, not DOM, for a11y.
- `::placeholder`, `::selection`, `::marker` cover input/list niches.
- Pseudos are not in the accessibility tree; meaningful text stays in HTML.
- Honestly: legacy single `:before` still appears, but I write `::`.

## How it works

```css
.quote {
  position: relative;
  padding-inline-start: 1.25rem;
}

.quote::before {
  content: "";
  position: absolute;
  inset-block: 0.25rem auto;
  inset-inline-start: 0;
  width: 3px;
  border-radius: 2px;
  background: var(--accent, #2563eb);
}

.tag::after {
  content: " · draft";
  font-size: 0.75em;
  color: #64748b;
}

::selection {
  background: #fde68a;
  color: #422006;
}
```

```ts
const quote = document.querySelector<HTMLElement>(".quote");
quote?.classList.toggle("is-highlighted");
```

```css
input::placeholder {
  color: #94a3b8;
}
```

## Pitfalls

**Content from attr() for UI text.** Bad for i18n and screen readers.

**Pseudo without positioning context.** absolute ::before lands in the wrong place.

**Interaction on ::after.** Hit area is unclear; pointer-events are limited.

**Emoji abuse in content.** Breaks a consistent icon system.

**Pseudo + class specificity.** Easy to beat utilities without an @layer plan.
