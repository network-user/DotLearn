# Специфичность CSS: как считать вес селектора

## Суть

Специфичность это вес селектора при разрешении конфликта свойств. Сравнивают столбцы: inline, id, класс/атрибут/псевдокласс, тип/псевдоэлемент; универсальный `*` и комбинаторы вес не добавляют.

## Что ответить на собесе

- Запись (a, b, c, d): inline style; id; class/attr/:pseudo-class; tag/::pseudo-element.
- `#nav .link:hover` → (0, 1, 2, 1): один id, класс + :hover, элемент `a` в селекторе не всегда явен.
- `:where(.btn)` обнуляет вклад класса; `:is(.a, #b)` берёт самый тяжёлый аргумент.
- Inline `style=""` бьёт id, кроме `!important` в author stylesheet в современных правилах слоёв.
- Честно добавляю: калькулятор специфичности в DevTools полезнее заучивания цифр наизусть.

## Как это работает

Набор правил:

```css
p { color: #64748b; }                    /* 0,0,0,1 */
.text-muted { color: #94a3b8; }           /* 0,0,1,0 */
p.intro { color: #0f172a; }               /* 0,0,1,1 */
#hero p.intro { color: #dc2626; }          /* 0,1,1,1 */
```

```html
<p id="hero" class="intro" style="color: #2563eb">Текст</p>
```

Победит inline `#2563eb`, пока нет `!important` в author rule с большим весом в том же cascade layer.

Псевдоклассы `:not()` считают аргумент внутри:

```css
li:not(.skip) { }  /* класс внутри not даёт +0,0,1,0 плюс li +0,0,0,1 */
```

`:has()` включает селектор в скобках полностью.

Снижение веса библиотекой:

```css
:where(.ds-button) {
  padding: 0.5rem 1rem;
}
```

Потребители переопределяют одним классом без гонки id.

## Подводные камни

**Считать только последний селектор.** В `.sidebar .nav a` суммируются все части цепочки.

**Путать специфичность и порядок.** При равном весе побеждает последнее правило.

**!important ломает таблицу.** Важность сравнивается до специфичности внутри origin.

**Наследование не специфичность.** `color` на `body` не конкурирует с классом на `p` как равный селектор.

**0,0,0,0 от *.** Универсальный селектор не помогает перебить класс.
