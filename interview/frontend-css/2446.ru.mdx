# Что приоритетнее: id или class в CSS

## Суть

ID-селектор весит выше класса: `(0,1,0,0)` против `(0,0,1,0)`. При конфликте на одном элементе победит правило с `#id`, если нет более сильного inline или `!important`.

## Что ответить на собесе

- На `<p id="x" class="y">` правило `#x { color }` перебьёт `.y { color }`.
- Несколько классов суммируются: `.a.b` (0,0,2,0) всё ещё ниже одного `#id`.
- В компонентах избегаю styling через id; один id на странице и высокий вес.
- Для override utility использую source order или layer, не новый id.
- Честно: accessibility tools и anchor links нуждаются в id, но не в id-based CSS.

## Как это работает

```html
<button id="save" class="btn primary" type="button">Сохранить</button>
```

```css
.btn { color: #334155; background: #e2e8f0; }
.primary { background: #2563eb; color: #fff; }
#save { outline-offset: 2px; }
```

Цвет задаёт `.primary`, не `#save`, если в `#save` нет color.

```ts
const save = document.querySelector<HTMLButtonElement>("#save");
save?.classList.toggle("primary");
```

Плохой паттерн:

```css
#save.primary { background: #16a34a; }
```

Лучше модификатор:

```css
.btn.is-success { background: #16a34a; }
```

## Подводные камни

**ID в CMS контенте.** Редактор добавляет id, ломая глобальные стили.

**React/Vue auto id на input.** Совпадение с CSS `#email` случайно.

**Специфичность якоря vs utility.** `#nav .link.active` блокирует redesign.

**Duplicate id в DOM.** Невалидно; querySelector берёт первый.

**hash routing vs id.** `#section` в URL не связан с CSS id, но путает команду.
