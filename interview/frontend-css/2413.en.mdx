# Debugging CSS Modules in production: source maps and DevTools

## In short

In production, CSS Modules hash class names (`Button_primary_x7k2`). Source maps link minified CSS back to original `.module.css` files so DevTools show real selectors and paths.

## How to answer in an interview

- I enable `devtool: 'source-map'` (or `hidden-source-map` for public deploys) in the bundler config.
- For css-loader/modules: `sourceMap: true` during CSS extraction.
- In Chrome DevTools Styles, clicking a rule jumps to the source module, not `app.[hash].css`.
- For Vite: `build.sourcemap: true` and ensure `.map` files are only exposed to the team.
- I honestly add that public source maps reveal project structure; production often uses hidden maps for authorized debugging only.

## How it works

CSS Modules compile imports like:

```ts
import styles from "./Card.module.css";

export function Card() {
  return <article className={styles.root}>...</article>;
}
```

The bundle renames the class to a unique hash. A **source map** stores mappings: `Card_root_a1b2` → `Card.module.css`, line 3.

Webpack config sketch:

```ts
export default {
  mode: "production",
  devtool: "hidden-source-map",
  module: {
    rules: [
      {
        test: /\.module\.css$/,
        use: [
          MiniCssExtractPlugin.loader,
          {
            loader: "css-loader",
            options: {
              modules: { localIdentName: "[name]__[local]___[hash:base64:5]" },
              sourceMap: true,
            },
          },
        ],
      },
    ],
  },
};
```

The browser loads `.css.map` only when DevTools are open (hidden mode). The inspector shows `Card.module.css` and the original `.root` rule.

Without maps: temporarily set `localIdentName: '[local]'` on staging or trace classes via Computed styles and repo grep.

## Pitfalls

**Maps not deployed.** CI ships only `.css`; DevTools show one long line with no file link.

**Public maps and IP.** Even without secrets, maps ease UI reverse engineering.

**SSR + extracted CSS.** Server chunks need maps too, or hydration class names diverge.

**PostCSS pipeline.** If the map breaks at autoprefixer, enable `sourceMap: true` on every loader.

**JS maps only.** CSS maps are separate; TS `sourcemap` does not cover styles automatically.
