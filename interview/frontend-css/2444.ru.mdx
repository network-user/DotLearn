# Что такое critical CSS и как ускорить first paint

## Суть

Critical CSS это минимальный набор правил для above-the-fold без FOUC. Его инлайнят в `<head>`, полный stylesheet грузят асинхронно, чтобы не блокировать первый paint.

## Что ответить на собесе

- Выделяю стили hero, typography, layout shell; остальное откладываю.
- Inline `<style>` в head + `link` с preload/async trick для полного CSS.
- Инструменты: critical, penthouse, vite-plugin-critical; проверяю на реальных URL.
- Баланс: inline >14KB съедает TTFB budget на медленных сетях.
- Честно: на SPA route-level CSS часто заменяет ручной critical per page.

## Как это работает

```html
<!doctype html>
<html lang="ru">
  <head>
    <meta charset="utf-8" />
    <style>
      body { margin: 0; font-family: system-ui, sans-serif; }
      .hero { padding: 2rem 1rem; max-width: 48rem; margin-inline: auto; }
      .hero h1 { font-size: clamp(1.75rem, 4vw, 2.5rem); line-height: 1.15; }
    </style>
    <link rel="preload" href="/assets/app.css" as="style" />
    <link rel="stylesheet" href="/assets/app.css" media="print" onload="this.media='all'" />
    <noscript><link rel="stylesheet" href="/assets/app.css" /></noscript>
  </head>
  <body>
    <header class="hero"><h1>Быстрый первый экран</h1></header>
  </body>
</html>
```

```ts
const deferred = document.querySelector<HTMLLinkElement>(
  'link[href="/assets/app.css"]'
);

deferred?.addEventListener("load", () => {
  document.documentElement.dataset.styles = "ready";
});
```

Extract pipeline (build):

```ts
const criticalTasks = ["parse-html", "match-selectors", "inline-head"] as const;
```

## Подводные камни

**Critical без route split.** Одна простыня critical на все SPA-страницы раздувает HTML.

**FOUC при onload trick.** Пользователь видит смену шрифта/цвета.

**@font-face в critical.** Блокирует render до загрузки шрифта.

**Duplicate rules.** Inline + external с одинаковыми селекторами усложняют cascade.

**Автоген без safelist.** Динамические классы выпадают из critical extract.
