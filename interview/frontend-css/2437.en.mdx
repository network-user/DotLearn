# What flex-wrap controls

## In short

`flex-wrap` decides whether flex items move to a new line (or column when `flex-direction: column`) when their flex bases no longer fit. Values: `nowrap`, `wrap`, `wrap-reverse`.

## How to answer in an interview

- Default `nowrap`: items shrink (`flex-shrink`); overflow is possible.
- `wrap` creates extra flex lines; `align-content` aligns those lines, not a single item.
- `wrap-reverse` reverses line order, not item order inside a row.
- For card grids I combine `flex-wrap: wrap` + `flex: 1 1 <basis>` + `gap`.
- Honestly: strict two-dimensional grids are clearer in Grid; wrap fits rubber rows.

## How it works

```css
.gallery {
  display: flex;
  flex-wrap: wrap;
  gap: 0.75rem;
  width: 320px;
}

.thumb {
  flex: 1 1 96px;
  max-width: 120px;
  aspect-ratio: 1;
  border-radius: 8px;
  background: #e2e8f0;
  display: grid;
  place-items: center;
}
```

```ts
const gallery = document.querySelector<HTMLElement>(".gallery");
const items = gallery?.querySelectorAll<HTMLElement>(".thumb");

console.log(items?.length ?? 0);
```

With column direction, wrap adds a new column along the horizontal axis.

## Pitfalls

**align-items vs align-content.** Multi-line containers need `align-content` or lines collapse.

**flex-shrink: 0 + nowrap.** Horizontal overflow without a scroll container.

**wrap-reverse and tab order.** Visual order may not match DOM focus order.

**Percentage flex-basis.** Percent is of the container; wrap changes line width.

**Expecting masonry rows.** Wrap does not brick-layout without a fixed basis.
