# Универсальный селектор * в CSS: когда уместен

## Суть

Селектор `*` выбирает каждый элемент в документе. Его чаще всего применяют в reset/normalize для выравнивания дефолтов браузеров или для глобальных правил вроде `box-sizing`, но массовое использование влияет на производительность селекторного движка.

## Что ответить на собесе

- В modern reset ставлю `box-sizing: border-box` через `*, *::before, *::after`.
- Не использую `*` для типографики и цветов: правила слишком широкие и ломают компоненты.
- Комбинирую с потомками (`* > *`) для margin-collapse reset между flex/grid детьми.
- Честно: на очень больших DOM глобальный `*` может замедлить recalc style; предпочитаю `:where()` для нулевой специфичности.
- Для a11y не трогаю `*` на focusable элементах без явного `:focus-visible`.

## Как это работает

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

.layout > * {
  margin: 0;
}

.card * {
  /* избегаю: слишком широко, бьёт по вложенным виджетам */
}
```

Универсальный селектор не добавляет специфичности сверх того, что уже есть в цепочке. В cascade он участвует как обычный type-like matcher: если правило `* { color: red }` конфликтует с `.title { color: blue }`, класс побеждает. На практике reset через `*` удобен, потому что не нужно перечислять `html, body, div, p, ...`.

Performance: браузер всё равно обходит дерево, но именно `*` на корне документа заставляет проверять каждый узел при изменении правила. Поэтому в production CSS reset ограничивают минимальным набором свойств.

## Подводные камни

- **Специфичность ловушек.** `*.active` и `.active` ведут себя одинаково по весу, но `*` в середине селектора (`div * span`) резко расширяет область матчинга.
- **Наследование.** `* { font-family }` не всегда дойдёт до pseudo без явного `::before/::after`.
- **Shadow DOM.** Глобальный `*` не проникает в shadow roots; web components живут своей cascade.
- **Third-party виджеты.** Reset через `*` может сбросить стили встроенного iframe/widget контента на странице хоста.
- **Отладка.** В DevTools правило на `*` труднее отследить: Sources показывают десятки совпадений на один узел.
