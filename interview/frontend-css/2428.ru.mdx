# Какие ограничения у CSS Modules на практике

## Суть

CSS Modules дают локальные классы через сборку, но глобальные reset/theme, динамика по состоянию и интеграция со сторонними UI-kit требуют отдельных паттернов. Плюс зависимость от конфигурации бандлера.

## Что ответить на собесе

- Глобальные стили выношу в `:global` или отдельный entry, иначе reset и typography не доходят до всего приложения.
- Динамику делаю через условные классы, `style` или CSS variables, а не «магию» внутри `.module.css`.
- Сборка: нужен loader (Vite/webpack), без него `.module.css` не превращается в map классов.
- Сторонние библиотеки часто ждут строковые className, а не объект из модуля.
- Честно: для design tokens и тем иногда проще CSS variables или CSS-in-JS, чем плодить глобальные исключения в modules.

## Как это работает

Loader читает файл и хеширует имена классов:

```ts
import styles from "./PrimaryAction.module.css";

type Variant = "solid" | "ghost";

interface ActionProps {
  variant?: Variant;
  disabled?: boolean;
}

export function PrimaryAction({ variant = "solid", disabled }: ActionProps) {
  const tone =
    variant === "solid" ? styles.solid : styles.ghost;

  return (
    <button
      type="button"
      disabled={disabled}
      className={`${styles.root} ${tone}`}
    >
      Сохранить
    </button>
  );
}
```

```css
.root {
  border-radius: 8px;
  padding: 0.625rem 1rem;
  font: inherit;
  cursor: pointer;
}

.solid {
  background: var(--accent, #2563eb);
  color: #fff;
  border: none;
}

.ghost {
  background: transparent;
  color: var(--accent, #2563eb);
  border: 1px solid currentColor;
}
```

Глобальный слой подключаю отдельно:

```css
:global(html) {
  box-sizing: border-box;
}

:global(*, *::before, *::after) {
  box-sizing: inherit;
}
```

Динамический цвет через custom property, не через импорт нового CSS на лету:

```ts
document.documentElement.style.setProperty("--accent", "#16a34a");
```

## Подводные камни

**Забытый :global.** Любой класс без него станет хешированным и не попадёт на `body`/portal.

**Composition через composes.** Работает не везде одинаково; в миксах Vite/webpack поведение расходится.

**SSR и FOUC.** Если class map не совпал между сервером и клиентом, стили «мигают».

**Tree-shaking не спасает.** Неиспользуемые селекторы внутри одного модуля всё равно попадают в бандл модуля.

**Переусложнение BEM в modules.** Дублирование методологии и хешей даёт длинные имена в DevTools без выигрыша.
