# Единицы измерения CSS: px, rem, %, vw и когда что брать

## Суть

CSS делит единицы на абсолютные (px, cm, pt) и относительные (%, em, rem, vw, vh, ch, fr). Относительные привязаны к родителю, корню, viewport или контексту grid/flex и делают интерфейс масштабируемым.

## Что ответить на собесе

- `px` в вебе это CSS-пиксель (1/96 дюйма логически), удобен для бордеров и теней.
- `rem` от `font-size` на `html`; предпочитаю для типографики и отступов дизайн-системы.
- `em` от шрифта текущего элемента; полезен для компонентов, масштабирующихся с локальным `font-size`.
- `%` от соответствующей оси родителя (width, height при заданной высоте родителя).
- Честно добавляю: `vw`/`vh` на мобиле включают высоту адресной строки; для полноэкранных блоков часто беру `dvh`/`svh`.

## Как это работает

Базовая шкала:

```css
html {
  font-size: 100%;
}

body {
  font-size: 1rem;
  line-height: 1.5;
}

h1 {
  font-size: clamp(1.75rem, 2vw + 1rem, 2.5rem);
}

.sidebar {
  width: min(320px, 90vw);
  padding: 1.25rem;
}

.hero {
  min-height: 100dvh;
}
```

`clamp` смешивает rem и vw: заголовок растёт с viewport, но в границах. `min()` не даёт сайдбару вылезти за экран.

**Viewport units:** `1vw` = 1% ширины viewport; `vmin`/`vmax` от меньшей/большей стороны. **Dynamic viewport** (`dvh`, `svh`, `lvh`) учитывает UI браузера на телефоне.

**Grid fr** делит свободное пространство трека, не процент от родителя:

```css
.layout {
  display: grid;
  grid-template-columns: 240px 1fr 20rem;
  gap: 1rem;
}
```

**ch** ширина символа «0»; удобно для моноширинных полей ввода фиксированной ширины в символах.

## Подводные камни

**em каскадом.** Вложенные `font-size: 1.2em` на трёх уровнях дают не 1.2, а 1.728 от корня.

**% height без высоты родителя.** `height: 50%` часто игнорируется, если родитель auto.

**100vw и горизонтальный скролл.** `100vw` включает ширину полосы прокрутки; иногда появляется overflow.

**px для accessibility.** Пользовательский zoom в браузере масштабирует px; но `rem` проще синхронизировать с user font settings через `html`.

**fr вне grid.** `fr` работает только в grid-треках, не в обычном `width`.
