# What are the practical limits of CSS Modules

## In short

CSS Modules scope classes at build time, but global resets/themes, state-driven styling, and third-party UI kits need separate patterns. You also depend on bundler configuration.

## How to answer in an interview

- I put global styles in `:global` or a separate entry; otherwise reset and typography never reach the whole app.
- For dynamics I use conditional classes, inline `style`, or CSS variables, not magic inside `.module.css`.
- Build step: a loader (Vite/webpack) must turn `.module.css` into a class map.
- Third-party libs often expect string class names, not a module object.
- Honestly: for design tokens and theming, CSS variables or CSS-in-JS can be simpler than many global exceptions in modules.

## How it works

The loader reads the file and hashes class names:

```ts
import styles from "./PrimaryAction.module.css";

type Variant = "solid" | "ghost";

interface ActionProps {
  variant?: Variant;
  disabled?: boolean;
}

export function PrimaryAction({ variant = "solid", disabled }: ActionProps) {
  const tone =
    variant === "solid" ? styles.solid : styles.ghost;

  return (
    <button
      type="button"
      disabled={disabled}
      className={`${styles.root} ${tone}`}
    >
      Save
    </button>
  );
}
```

```css
.root {
  border-radius: 8px;
  padding: 0.625rem 1rem;
  font: inherit;
  cursor: pointer;
}

.solid {
  background: var(--accent, #2563eb);
  color: #fff;
  border: none;
}

.ghost {
  background: transparent;
  color: var(--accent, #2563eb);
  border: 1px solid currentColor;
}
```

I attach the global layer separately:

```css
:global(html) {
  box-sizing: border-box;
}

:global(*, *::before, *::after) {
  box-sizing: inherit;
}
```

Dynamic color via a custom property, not by importing new CSS at runtime:

```ts
document.documentElement.style.setProperty("--accent", "#16a34a");
```

## Pitfalls

**Forgotten :global.** Any class without it gets hashed and never hits `body`/portals.

**Composition via composes.** Not identical across Vite/webpack setups.

**SSR and FOUC.** If the class map differs between server and client, styles flash.

**Tree-shaking does not help much.** Unused selectors inside one module still ship with that module.

**Over-BEM in modules.** Methodology plus hashes yields long DevTools names with little benefit.
