# Что такое CSSOM и где он в pipeline рендеринга

## Суть

CSSOM (CSS Object Model) это дерево стилей, которое браузер строит после парсинга CSS. Его сливают с DOM в render tree, откуда считаются layout и paint; через `element.style` и CSSOM API стили можно менять из JS.

## Что ответить на собесе

- Порядок: HTML → DOM, CSS → CSSOM, затем render tree (только видимые узлы со стилями).
- CSSOM хранит каскад, специфичность и computed values для match с селекторами.
- Inline `style` и изменения через JS попадают в CSSOM и могут инвалидировать layout.
- DevTools Styles panel показывает computed цепочку из CSSOM + DOM.
- Честно: чтение `offsetWidth` после записи style форсирует sync layout между JS и CSSOM.

## Как это работает

```html
<p id="status" class="badge">Онлайн</p>
```

```css
.badge {
  color: #15803d;
  font-size: 0.875rem;
  font-weight: 600;
}
```

```ts
const status = document.querySelector<HTMLElement>("#status");
if (!status) throw new Error("missing status");

status.style.color = "#b45309";
status.style.fontSize = "1rem";

const computed = getComputedStyle(status);
console.log(computed.color, computed.fontSize);
```

`getComputedStyle` читает итог каскада из CSSOM, не только inline.

## Подводные камни

**document.styleSheets и CORS.** Правила cross-origin листа недоступны для чтения из JS.

**Массовые style.* в цикле.** Каждая запись может триггерить recalc; лучше classList или CSS variables.

**Путаница DOM vs CSSOM.** DOM это структура; цвет и размер живут в computed styles.

**insertRule в runtime.** Динамический CSSOM растёт; забытый `<style>` в SPA не удаляется.

**getComputedStyle в hot path.** Дорого на тысячах элементов каждый кадр.
