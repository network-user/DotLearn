# Which CSS properties trigger reflow and repaint

## In short

Reflow touches geometry: sizes, spacing, positioning, fonts, display. Repaint redraws pixels without layout changes (color, background; visibility is nuanced). Compositor properties like transform and opacity usually skip layout.

## How to answer in an interview

- Layout triggers: width/height, margin/padding/border, top/left, font-size, line-height, display, position, float, overflow, content.
- DOM ops: insert/remove, text node changes, layout reads after writes.
- Paint mostly: color, outline, background, box-shadow (without size change).
- Composite: transform, opacity, filter (watch fixed descendants).
- I honestly add that trigger tables are simplified; browsers batch work, but forced sync layout is still costly.

## How it works

Pipeline: style → layout → paint → composite.

```ts
const box = document.querySelector<HTMLElement>(".panel");
if (!box) return;

box.style.width = "320px";
box.style.padding = "1rem";
box.style.transform = "scale(1.02)";
box.style.color = "#0f172a";
```

The first two may reflow siblings; `transform` and `color` stay in composite/paint without recomputing grid width.

Read-after-write forces layout:

```ts
box.style.width = "400px";
const w = box.offsetWidth;
box.style.height = `${w / 2}px`;
```

The browser must synchronously layout between those lines.

Common layout properties for interviews:

```
width, height, min-/max-*, margin, padding, border-width,
top, left, right, bottom, font-size, line-height,
display, position, float, clear, overflow, vertical-align
```

DOM insert/remove and `document.fonts` changes also invalidate layout.

## Pitfalls

**offsetWidth in scroll handlers.** Scroll + read per frame causes jank.

**classList.toggle on body.** Modal `overflow: hidden` reflows the whole page.

**Huge box-shadow blur.** Expensive paint even without layout.

**filter: blur() on large surfaces.** GPU fill cost, not layout, but FPS drops.

**False safety of translate.** `transform` avoids reflow, but you cannot animate neighboring `height: auto` the same way.
