# Что такое CSS Modules и зачем они в компонентах

## Суть

CSS Modules это файлы `.module.css`, где классы на этапе сборки получают уникальные имена и импортируются в JS как объект. Стили остаются локальными к компоненту и не конфликтуют глобально.

## Что ответить на собесе

- Имена классов хешируются bundler'ом; в JSX использую `styles.root`, не строку `"root"`.
- Изоляция снижает коллизии в больших командах и монорепо с legacy CSS.
- Глобальные правила оформляю через `:global` или отдельный `global.css`.
- Работает с React/Vue/Svelte при поддержке loader; Vite включает из коробки.
- Честно: это не runtime-фреймворк, а конвенция сборки; без bundler modules не существуют.

## Как это работает

```css
.card {
  display: grid;
  gap: 0.75rem;
  padding: 1rem;
  border-radius: 12px;
  background: #fff;
  box-shadow: 0 1px 2px rgb(15 23 42 / 8%);
}

.title {
  font-size: 1.125rem;
  font-weight: 600;
}
```

```ts
import styles from "./ProductCard.module.css";

interface ProductCardProps {
  name: string;
  price: string;
}

export function ProductCard({ name, price }: ProductCardProps) {
  return (
    <article className={styles.card}>
      <h2 className={styles.title}>{name}</h2>
      <p>{price}</p>
    </article>
  );
}
```

После сборки класс может стать `ProductCard_card_x7k2f`, но API в коде не меняется.

TypeScript типизация (Vite plugin / d.ts):

```ts
declare module "*.module.css" {
  const classes: Record<string, string>;
  export default classes;
}
```

## Подводные камни

**Строковые className из props.** `className="card"` не совпадёт с хешем modules.

**Дубли имён между файлами безопасны, внутри файла нет.** Два `.title` в одном module.css сольются.

**composes и dead code.** Неиспользуемые классы всё равно попадают в CSS chunk модуля.

**CSS variables vs modules.** Tokens лучше в `:root`, modules для компонентной геометрии.

**Тесты snapshot.** Хеши меняются при rename файла; snapshot class names хрупкие.
