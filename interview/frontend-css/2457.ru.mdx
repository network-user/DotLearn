# Псевдоэлементы в CSS: зачем и какие бывают

## Суть

Псевдоэлементы (`::before`, `::after`, `::first-line`, `::marker` и др.) стилизуют части box или добавляют декоративный content без лишнего HTML. Синтаксис с двойным двоеточием `::` отличает их от псевдоклассов.

## Что ответить на собесе

- `::before`/`::after` требуют `content` (хотя бы `""`) для отображения.
- Декоративные иконки и divider лучше в pseudo, не в DOM ради a11y.
- `::placeholder`, `::selection`, `::marker` покрывают input/list niches.
- Pseudo не появляются в accessibility tree; важный текст только в HTML.
- Честно: один `:before` legacy синтаксис ещё встречается, но пишу `::`.

## Как это работает

```css
.quote {
  position: relative;
  padding-inline-start: 1.25rem;
}

.quote::before {
  content: "";
  position: absolute;
  inset-block: 0.25rem auto;
  inset-inline-start: 0;
  width: 3px;
  border-radius: 2px;
  background: var(--accent, #2563eb);
}

.tag::after {
  content: " · draft";
  font-size: 0.75em;
  color: #64748b;
}

::selection {
  background: #fde68a;
  color: #422006;
}
```

```ts
const quote = document.querySelector<HTMLElement>(".quote");
quote?.classList.toggle("is-highlighted");
```

```css
input::placeholder {
  color: #94a3b8;
}
```

## Подводные камни

**Content из attr() для UI text.** Плохо для i18n и screen readers.

**Pseudo без position context.** absolute ::before «улетает» не туда.

**Interaction on ::after.** Hit area неочевидна; pointer-events limited.

**Злоупотребление emoji in content.** Ломает consistent icon system.

**Specificity pseudo + class.** Легко перебить utility без @layer plan.
