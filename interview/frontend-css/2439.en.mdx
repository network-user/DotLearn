# How to declare and use CSS variables

## In short

Custom properties are set as `--name: value` and read with `var(--name, fallback)`. They cascade and inherit, so themes and spacing fit well on `:root` or `[data-theme]`.

## How to answer in an interview

- Declaration: `--token: value` on any selector; scope matches normal properties.
- Usage: `property: var(--token, fallback)`.
- Theming: swap variable sets on `html[data-theme="dark"]`.
- In JS: `setProperty`/`removeProperty` on `document.documentElement`.
- Honestly: variables do not work inside `@media` queries as numbers without `@custom-media`; breakpoints stay plain media queries.

## How it works

```css
:root {
  --surface: #ffffff;
  --ink: #0f172a;
  --accent: #2563eb;
  --radius: 10px;
}

:root[data-theme="dark"] {
  --surface: #0b1220;
  --ink: #e2e8f0;
  --accent: #60a5fa;
}

.card {
  background: var(--surface);
  color: var(--ink);
  border-radius: var(--radius);
  border: 1px solid color-mix(in srgb, var(--ink) 12%, transparent);
}

.link {
  color: var(--accent, #1d4ed8);
}
```

```ts
const root = document.documentElement;

function applyTheme(mode: "light" | "dark") {
  root.dataset.theme = mode;
  root.style.setProperty("--accent", mode === "dark" ? "#60a5fa" : "#2563eb");
}
```

Local variable on a component:

```css
.stat {
  --bar: 40%;
  background: linear-gradient(
    to top,
    var(--accent) var(--bar),
    transparent var(--bar)
  );
}
```

## Pitfalls

**var() in shorthand without fallback.** One invalid value breaks the whole property.

**Inheritance in shadow DOM.** Light DOM tokens need `:host` or `@property` bridging.

**Animating non-numeric tokens.** `--color` does not interpolate; use `@property` for registered props.

**Typo in the name.** Silent fallback hides the bug.

**Too many tokens.** Without naming (`--space-4`) `:root` becomes a dump.
