# Какие CSS-свойства вызывают repaint

## Суть

Repaint перерисовывает пиксели без пересчёта геометрии layout. Типичные триггеры: color, background, border-color, outline, box-shadow, visibility (частично). Дешевле reflow, но массовый repaint в анимации бьёт по FPS.

## Что ответить на собесе

- Paint-only: `color`, `background`, `border-color`, `outline`, `text-decoration`, `box-shadow` без изменения размеров.
- `visibility:hidden` часто только paint; `display:none` убирает из render tree (layout).
- Compositor-friendly: `transform`, `opacity` минуют полный repaint layout area.
- Батчу DOM writes; read layout после write форсирует лишние стадии.
- Честно: большой blur shadow repaint дорогой, хотя layout не трогает.

## Как это работает

```ts
const chip = document.querySelector<HTMLElement>(".chip");
if (!chip) return;

chip.style.backgroundColor = "#fef3c7";
chip.style.color = "#92400e";
chip.style.boxShadow = "0 2px 8px rgb(146 64 14 / 20%)";
chip.style.borderColor = "#f59e0b";
```

```css
.chip {
  padding: 0.35rem 0.65rem;
  border: 1px solid transparent;
  border-radius: 6px;
  transition: background-color 160ms ease, color 160ms ease;
}

.chip.is-active {
  background: #dbeafe;
  color: #1e40af;
}
```

## Подводные камни

**background-image swap.** Decode + paint тяжелее solid color.

**filter: blur на большой области.** GPU fill, не layout, но jank.

**box-shadow spread анимация.** Каждый кадр repaint.

**visibility vs opacity.** opacity часто compositor; visibility может paint+composite.

**DevTools disable cache.** Paint flashing overlay показывает лишнее на prod hardware.
