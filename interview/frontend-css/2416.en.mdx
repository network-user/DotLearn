# Box model: content, padding, border, margin, and border-box

## In short

Every element is a rectangle of content, padding, border, and margin. By default, `width`/`height` set the content box only; `box-sizing: border-box` includes padding and border in the stated size.

## How to answer in an interview

- Content holds text and media; padding sits inside the border; border frames the box; margin separates neighbors.
- In content-box, total width = width + padding + border.
- `border-box` makes width the full inner box including padding and border.
- Margin is outside the element size but participates in margin collapse.
- I honestly add that global `* { box-sizing: border-box }` helps layout but watch replaced elements (`img`, `input`).

## How it works

Two modes for a 200px block:

```css
.box-content {
  box-sizing: content-box;
  width: 200px;
  padding: 16px;
  border: 4px solid #cbd5e1;
}

.box-border {
  box-sizing: border-box;
  width: 200px;
  padding: 16px;
  border: 4px solid #cbd5e1;
}
```

`.box-content` paints roughly 200 + 32 + 8 = 240px wide. `.box-border` stays 200px outer width; content shrinks.

Project baseline:

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

Flex/Grid place **border-box** sizes on tracks; percentage width is relative to the parent without padding blowing past 100%.

DevTools Layout overlays margin (orange), border, padding, and content to spot stray spacing.

## Pitfalls

**100% width + padding in content-box.** The box overflows the parent without `border-box`.

**Margin collapse.** Vertical margins between siblings collapse; parent `padding`/`border` or `display: flow-root` breaks it.

**min-height and percentages.** Percent height needs a defined parent height or behavior surprises you.

**outline is outside the box model.** Outline does not affect layout but draws over neighbors.

**box-sizing on replaced elements.** Fixed-width `img` with border-box behaves more predictably in bordered cards.
