# The CSS cascade: origin, specificity, and rule order

## In short

The cascade picks the winning declaration when multiple rules target one element. The modern algorithm weighs `@layer`, importance (`!important`), selector specificity, and source order; inline styles and user `!important` beat author rules in special cases.

## How to answer in an interview

- I compare origin first: user agent, user, author; `@layer` ranks layers before specificity.
- Specificity: inline > id > class/attribute/pseudo-class > type; universal `*` adds no weight.
- Equal specificity: the later rule in the same layer wins.
- `!important` inverts origin order within a level but does not fully bypass layers.
- I honestly add that `!important` and id selectors fix symptoms, not style architecture.

## How it works

Conflict example:

```css
@layer base, components;

@layer base {
  p { color: #334155; }
}

@layer components {
  .note { color: #0ea5e9; }
}

#lead { color: #dc2626; }

p.note#lead {
  color: #16a34a !important;
}
```

```html
<p class="note" id="lead">Paragraph</p>
```

Without `!important`, `#lead` (0,1,0,1) beats `.note` (0,0,1,1). With `p.note#lead !important`, green wins unless a more specific `!important` exists in the same layer.

**Inline** `style="color: …"` yields specificity (1,0,0,0) for that property. Inherited `color` on a parent loses to a direct assignment on the element.

`<link>` vs `<style>` order matters only when all else is equal. `@import` delays imported rules inside a file.

## Pitfalls

**Specificity arms race.** `#header .nav .link` is painful to override without an even longer selector.

**!important utilities.** Tailwind's `!` modifier helps in a pinch but breaks cascade predictability.

**Pseudo-classes and :where().** `:where(.a)` zeroes class contribution; handy for libraries.

**Inheritance vs cascade.** `font-size` on `html` inherits; two classes on one element are resolved by cascade, not inheritance.

**Shadow DOM styles.** Shadow roots have their own rule trees; global CSS does not pierce without `::part` or variables.
