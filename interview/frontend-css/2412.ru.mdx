# Reset.css: зачем сбрасывать браузерные стили по умолчанию

## Суть

У каждого браузера свой user-agent stylesheet: отступы у `body`, маркеры списков, размеры заголовков. Reset обнуляет или уравнивает эти значения, чтобы дизайн-система начинала с предсказуемой базы.

## Что ответить на собесе

- Reset убирает margin/padding/border у типового набора тегов, часто ставит `box-sizing: border-box`.
- Цель: одинаковый старт в Chrome, Firefox, Safari, не «красота из коробки».
- Альтернатива: normalize.css сохраняет полезные дефолты (например, `display: block` у `main`) и лишь выравнивает расхождения.
- Сегодня часто вместо классического reset используют `@layer reset` в дизайн-токенах или пресет Tailwind Preflight.
- Честно добавляю: агрессивный reset ломает нативную доступность форм, если потом не восстановить focus/outline и размеры контролов.

## Как это работает

Классический **Eric Meyer Reset** обнуляет отступы у длинного списка селекторов:

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

body,
h1,
h2,
h3,
p,
ul,
ol,
figure {
  margin: 0;
  padding: 0;
}

ul,
ol {
  list-style: none;
}

img {
  max-width: 100%;
  display: block;
}
```

После reset проект накладывает **base styles**: типографика, цвета, `line-height`, стили ссылок и кнопок.

**Normalize** другой философии: не обнуляет всё, а чинит известные баги UA (например, `line-height` в формах, `hidden` у `[hidden]`).

В современных сборках reset подключают первым слоем:

```css
@layer reset, base, components, utilities;
```

Так специфичность предсказуема, и utilities не борются с голым `margin: 0` на каждом теге.

## Подводные камни

**Сброс list-style без восстановления.** Навигация теряет семантику списка визуально, но AT всё ещё видит `ul`; нужны осознанные стили.

**Удаление outline на :focus.** Частая ошибка в старых reset/snippet; ломает клавиатурную навигацию.

**font: inherit на button/input.** Без последующих правил кнопки выглядят как системные и не совпадают с UI-kit.

**Двойной reset.** Подключили Meyer + Tailwind Preflight + свои нули: специфичность и размер CSS растут.

**Reset вместо дизайн-токенов.** Обнуление без системы отступов и типографики даёт «плоский» UI, а не консистентный.
