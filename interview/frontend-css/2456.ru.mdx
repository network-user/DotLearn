# display:none и visibility:hidden: в чём разница

## Суть

`display:none` убирает элемент из render tree и потока: места нет, layout пересчитывается. `visibility:hidden` скрывает отрисовку, но box остаётся и занимает место; для a11y оба обычно исключают элемент из accessibility tree, если не aria override.

## Что ответить на собесе

- display:none: табы, conditional blocks, элемент «как будто удалён».
- visibility:hidden: резерв места, skeleton swap, иногда анимация opacity+visibility.
- Tab focus не должен попадать в скрытые controls; использую `hidden` attr или `inert`.
- visibility: hidden + opacity:0 часто для fade без layout shift.
- Честно: для screen readers лучше `hidden`, `aria-hidden`, conditional render, не visibility alone.

## Как это работает

```css
.panel.is-collapsed {
  display: none;
}

.panel.is-loading {
  visibility: hidden;
}

.panel.is-loading::after {
  visibility: visible;
  content: "…";
}
```

```ts
function toggleTab(panel: HTMLElement, show: boolean) {
  panel.style.display = show ? "block" : "none";
  panel.toggleAttribute("hidden", !show);
}

function reserveSlot(node: HTMLElement, visible: boolean) {
  node.style.visibility = visible ? "visible" : "hidden";
}
```

```html
<section hidden id="settings">…</section>
```

`hidden` attribute maps to display:none in UA stylesheet.

## Подводные камни

**visibility:hidden кликабельно.** Events могут проходить; нужен pointer-events:none.

**display toggle layout thrashing.** Массовое show/hide grid reflow дорого.

**SEO и display:none.** Скрытый текст может считаться cloaking если злоупотреблять.

**opacity:0 vs visibility.** opacity оставляет focusable элементы.

**Анимация display.** Не анимируется; использую grid 0fr trick или class swap.
