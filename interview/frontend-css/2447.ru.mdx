# Какое значение position по умолчанию

## Суть

По умолчанию `position: static`. Элемент живёт в normal flow, `top/right/bottom/left` и `z-index` на нём не действуют. Любое не-static значение создаёт positioning context для absolute descendants.

## Что ответить на собесе

- Default static: без смещения и без нового containing block для absolute.
- `relative` смещает визуально, место в потоке сохраняется.
- `absolute` вырывает из потока, якорь у ближайшего non-static предка.
- `fixed` якорь viewport (или transform ancestor in modern browsers).
- Честно: `sticky` тоже не static, но ведёт себя как hybrid relative/fixed.

## Как это работает

```css
.tile {
  width: 6rem;
  height: 6rem;
  background: #bfdbfe;
}

.tile.is-offset {
  position: relative;
  top: 0.5rem;
  left: 0.5rem;
}
```

Static box: offset properties ignored:

```css
.tile.static-try {
  top: 2rem;
  left: 2rem;
}
```

```ts
const tile = document.querySelector<HTMLElement>(".tile");
const style = tile ? getComputedStyle(tile).position : "";
console.log(style);
```

Containing block для absolute:

```css
.shell { position: relative; }
.badge {
  position: absolute;
  top: 0.25rem;
  right: 0.25rem;
}
```

## Подводные камни

**z-index на static.** Игнорируется; кажется «баг stacking».

**absolute без positioned parent.** Привязка к viewport, ломает модалки.

**transform на предке.** Создаёт containing block для fixed (не viewport).

**sticky без overflow check.** Родитель с `overflow:hidden` ломает sticky.

**position через class toggle.** Layout shift если не резервировать место.
