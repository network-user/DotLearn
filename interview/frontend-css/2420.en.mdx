# z-index and stacking context: why 9999 is not always on top

## In short

`z-index` orders painting only inside one stacking context. It applies to positioned elements (`relative`, `absolute`, `fixed`, `sticky`) and flex/grid children with z-index; contexts also come from opacity, transform, filter, and isolation.

## How to answer in an interview

- Without non-static `position` or flex/grid placement, a child's z-index does not fight an external overlay.
- Each new stacking context isolates child z-index values from sibling DOM branches.
- DOM order breaks ties when z-index is auto in the same context.
- I keep a token scale: base 0, dropdown 100, modal 200, toast 300.
- I honestly add that ever-larger numbers (`99999`) cannot escape a parent context with a lower root z-index.

## How it works

Three layers in one context:

```css
.layer-a {
  position: relative;
  z-index: 1;
}

.layer-b {
  position: relative;
  z-index: 2;
}

.layer-c {
  position: relative;
  z-index: 3;
}
```

Nested context:

```css
.sidebar {
  position: relative;
  z-index: 10;
  isolation: isolate;
}

.sidebar .tooltip {
  z-index: 1000;
}
```

A tooltip at `1000` still sits under a modal at `z-index: 50` on `.app-header` if the header formed a separate context with a higher root stack.

Context creators (partial): `opacity < 1`, `transform`, `filter`, `will-change: transform`, `contain: paint`, `mix-blend-mode`.

Modal pattern:

```css
.modal-backdrop {
  position: fixed;
  inset: 0;
  z-index: var(--z-modal, 200);
}

.modal-panel {
  position: relative;
  z-index: 1;
}
```

Backdrop and panel share one portal; the panel stacks above the backdrop locally.

## Pitfalls

**z-index on static elements.** Ignored; set position or use flex/grid stacking.

**Parent transform context.** `transform: translateZ(0)` on a wrapper breaks child `position: fixed` fullscreen overlays.

**Negative z-index.** Sends content under the parent background unexpectedly.

**Cross-context comparison.** z-index 1 inside a context whose root is 100 beats z-index 9999 inside a context whose root is 1.

**React portals.** Rendering modals to `document.body` lifts overlays to the root context; without a portal, modals stay under the header.
