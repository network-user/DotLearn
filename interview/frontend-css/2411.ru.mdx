# Изоляция в CSS: stacking context, contain и Shadow DOM

## Суть

Изоляция ограничивает, как стили и слои одного блока влияют на остальную страницу. `isolation`, `contain` и инкапсуляция Shadow DOM решают разные задачи: z-index, производительность рендера и конфликты селекторов в компонентах.

## Что ответить на собесе

- `isolation: isolate` создаёт новый stacking context, чтобы `z-index` детей не соревновался с внешним UI.
- `contain: layout style paint` подсказывает браузеру, что изменения внутри блока локальны (меньше перерасчётов).
- Shadow DOM режет каскад: стили внутри shadow root не протекают наружу и обратно.
- CSS Modules / scoped в SPA дают изоляцию на уровне классов, без нативного shadow.
- Честно добавляю: `contain` не заменяет архитектуру компонентов; неправильный contain может обрезать overflow и сломать dropdown.

## Как это работает

**Stacking context.** Свойства вроде `position` + `z-index`, `opacity < 1`, `transform`, `filter` создают контекст. `isolation: isolate` форсирует его без побочных визуальных эффектов:

```css
.modal-host {
  isolation: isolate;
  position: relative;
  z-index: 100;
}

.modal-host .tooltip {
  z-index: 1;
}
```

Tooltip соревнуется только внутри `.modal-host`, не с header сайта.

**Containment:**

```css
.dashboard-card {
  contain: layout style paint;
  content-visibility: auto;
  contain-intrinsic-size: 320px 240px;
}
```

Браузер может не пересчитывать layout всей страницы при изменении текста внутри карточки.

**Shadow DOM** (Web Components):

```ts
class MetricCard extends HTMLElement {
  constructor() {
    super();
    const root = this.attachShadow({ mode: "open" });
    root.innerHTML = `
      <style>
        .value { font-size: 2rem; color: var(--accent, #2563eb); }
      </style>
      <p class="value"><slot></slot></p>
    `;
  }
}
customElements.define("metric-card", MetricCard);
```

Глобальный селектор `p { margin: 0 }` не переопределит стиль внутри компонента без `::part` или CSS custom properties.

## Подводные камни

**isolation без z-index плана.** Контекст создаётся, но порядок слоёв всё равно нужно проектировать.

**contain: paint и тени.** `box-shadow` и выпадающие меню могут обрезаться родителем с paint containment.

**Закрытый shadow mode.** Отладка и тесты сложнее; сторонние стили не проникнут даже когда нужно.

**Дублирование изоляции.** CSS Modules + Shadow DOM + BEM в одном виджете усложняют без выгоды.

**Производительность contain-intrinsic-size.** Неверные размеры-заглушки дают скачки layout при появлении контента.
