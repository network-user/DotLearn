# CSS isolation: stacking context, contain, and Shadow DOM

## In short

Isolation limits how one block's styles and layers affect the rest of the page. `isolation`, `contain`, and Shadow DOM encapsulation solve different problems: z-index, render performance, and selector collisions in components.

## How to answer in an interview

- `isolation: isolate` creates a new stacking context so children's `z-index` does not fight external UI.
- `contain: layout style paint` hints that changes inside the box stay local (fewer recalculations).
- Shadow DOM cuts the cascade: styles inside the shadow root do not leak in or out.
- CSS Modules / scoped styles in SPAs isolate at the class level without native shadow.
- I honestly add that `contain` does not replace component architecture; wrong contain can clip overflow and break dropdowns.

## How it works

**Stacking context.** Properties like `position` + `z-index`, `opacity < 1`, `transform`, and `filter` create contexts. `isolation: isolate` forces one without extra visual side effects:

```css
.modal-host {
  isolation: isolate;
  position: relative;
  z-index: 100;
}

.modal-host .tooltip {
  z-index: 1;
}
```

The tooltip competes only inside `.modal-host`, not with the site header.

**Containment:**

```css
.dashboard-card {
  contain: layout style paint;
  content-visibility: auto;
  contain-intrinsic-size: 320px 240px;
}
```

The browser may skip relayout of the whole page when text inside the card changes.

**Shadow DOM** (Web Components):

```ts
class MetricCard extends HTMLElement {
  constructor() {
    super();
    const root = this.attachShadow({ mode: "open" });
    root.innerHTML = `
      <style>
        .value { font-size: 2rem; color: var(--accent, #2563eb); }
      </style>
      <p class="value"><slot></slot></p>
    `;
  }
}
customElements.define("metric-card", MetricCard);
```

A global `p { margin: 0 }` rule will not override styles inside the component without `::part` or CSS custom properties.

## Pitfalls

**isolation without a z-index plan.** Context exists, but layer order still needs design.

**contain: paint and shadows.** `box-shadow` and dropdowns may clip under a parent with paint containment.

**Closed shadow mode.** Debugging and testing get harder; external styles cannot pierce even when you want them to.

**Stacked isolation mechanisms.** CSS Modules + Shadow DOM + BEM in one widget adds complexity without benefit.

**contain-intrinsic-size accuracy.** Wrong placeholders cause layout jumps when content appears.
