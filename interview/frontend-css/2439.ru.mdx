# Как объявить и использовать CSS-переменные

## Суть

Custom properties задаются как `--name: value` и читаются через `var(--name, fallback)`. Они каскадируют и наследуются, поэтому темы и spacing удобно вешать на `:root` или `[data-theme]`.

## Что ответить на собесе

- Объявление: `--token: value` на любом селекторе; область видимости как у обычных свойств.
- Использование: `property: var(--token, fallback)`.
- Темизация: переключаю набор переменных на `html[data-theme="dark"]`.
- В JS: `setProperty`/`removeProperty` на `document.documentElement`.
- Честно: переменные не работают в `@media` query как числа без `@custom-media` polyfill; для breakpoints остаются обычные media.

## Как это работает

```css
:root {
  --surface: #ffffff;
  --ink: #0f172a;
  --accent: #2563eb;
  --radius: 10px;
}

:root[data-theme="dark"] {
  --surface: #0b1220;
  --ink: #e2e8f0;
  --accent: #60a5fa;
}

.card {
  background: var(--surface);
  color: var(--ink);
  border-radius: var(--radius);
  border: 1px solid color-mix(in srgb, var(--ink) 12%, transparent);
}

.link {
  color: var(--accent, #1d4ed8);
}
```

```ts
const root = document.documentElement;

function applyTheme(mode: "light" | "dark") {
  root.dataset.theme = mode;
  root.style.setProperty("--accent", mode === "dark" ? "#60a5fa" : "#2563eb");
}
```

Локальная переменная на компоненте:

```css
.stat {
  --bar: 40%;
  background: linear-gradient(
    to top,
    var(--accent) var(--bar),
    transparent var(--bar)
  );
}
```

## Подводные камни

**var() в shorthand без fallback.** Одно невалидное значение ломает всё свойство.

**Наследование в shadow DOM.** Light DOM tokens нужно пробрасывать через `:host` или `@property`.

**Анимация не числовых tokens.** `--color` не интерполируется; использую `@property` для registered custom props.

**Опечатка в имени.** Тихий fallback маскирует баг.

**Слишком many tokens.** Без naming convention (`--space-4`) файл `:root` становится свалкой.
