# What is the default value of position

## In short

The default is `position: static`. The element stays in normal flow; `top/right/bottom/left` and `z-index` do not apply. Any non-static value creates a positioning context for absolute descendants.

## How to answer in an interview

- Static: no offset and no new containing block for absolute children.
- `relative` shifts visually but keeps flow space.
- `absolute` leaves flow; anchor is the nearest non-static ancestor.
- `fixed` anchors to the viewport (or a transform ancestor in modern browsers).
- Honestly: `sticky` is not static either; it behaves like relative/fixed hybrid.

## How it works

```css
.tile {
  width: 6rem;
  height: 6rem;
  background: #bfdbfe;
}

.tile.is-offset {
  position: relative;
  top: 0.5rem;
  left: 0.5rem;
}
```

On a static box, offsets are ignored:

```css
.tile.static-try {
  top: 2rem;
  left: 2rem;
}
```

```ts
const tile = document.querySelector<HTMLElement>(".tile");
const style = tile ? getComputedStyle(tile).position : "";
console.log(style);
```

Containing block for absolute:

```css
.shell { position: relative; }
.badge {
  position: absolute;
  top: 0.25rem;
  right: 0.25rem;
}
```

## Pitfalls

**z-index on static.** Ignored; looks like a stacking bug.

**absolute without positioned parent.** Anchors to viewport; breaks modals.

**transform on an ancestor.** Creates a containing block for fixed (not viewport).

**sticky without overflow check.** Parent `overflow:hidden` breaks sticky.

**position via class toggle.** Layout shift if space is not reserved.
