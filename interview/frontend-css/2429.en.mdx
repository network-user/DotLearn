# What flexbox properties change on the container and items

## In short

Flexbox splits the container into main and cross axes. Container properties set direction, wrapping, and group alignment; item properties distribute free space and base size.

## How to answer in an interview

- On the container: `display:flex`, `flex-direction`, `flex-wrap`, `justify-content` (main axis), `align-items`/`align-content` (cross).
- On items: `flex-grow`, `flex-shrink`, `flex-basis`, shorthand `flex`, plus `align-self` and `order`.
- `justify-content` follows the main axis; with `flex-direction: column`, horizontal becomes cross.
- For equal columns without percentages I use `flex: 1 1 0` or `flex-basis: 0`.
- Honestly: flexbox fits one-dimensional rows; two-dimensional layouts often belong in Grid.

## How it works

Toolbar with wrap and even spacing:

```ts
const toolbarStyle = {
  display: "flex",
  flexDirection: "row" as const,
  flexWrap: "wrap" as const,
  justifyContent: "space-between",
  alignItems: "center",
  gap: "0.75rem",
};

const chipStyle = {
  flex: "1 1 120px",
  minWidth: "6rem",
};
```

```css
.toolbar {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
  justify-content: space-between;
  align-items: center;
  gap: 0.75rem;
}

.chip {
  flex: 1 1 120px;
  min-width: 6rem;
  padding: 0.5rem 0.75rem;
  border-radius: 999px;
  background: color-mix(in srgb, currentColor 8%, transparent);
}
```

`flex-grow: 1` takes leftover row space; `flex-shrink: 0` protects a minimum icon width.

## Pitfalls

**margin:auto on a flex item.** Consumes free space in surprising ways, especially in a column.

**align-items: stretch with fixed child height.** Children stretch but inner content may not center.

**flex-basis: auto and min-width.** The item will not shrink below content without `min-width: 0`.

**gap vs margin.** Older Safari without gap breaks card grids if support is unchecked.

**100% height in column flex.** A parent without height never lets the child grow vertically.
