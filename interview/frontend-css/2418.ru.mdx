# Адаптивная сетка на Flexbox и CSS Grid

## Суть

Flexbox раскладывает элементы в одном направлении с переносом строк; Grid задаёт двумерную сетку колонок и рядов. Для карточек без жёсткого макета часто хватает `auto-fill`/`minmax`; для сложных лейаутов страницы предпочитаю Grid.

## Что ответить на собесе

- Flex: `display:flex`, `flex-wrap:wrap`, `flex: 1 1 280px` или `flex-basis` + `min-width` для равных карточек.
- Grid: `grid-template-columns: repeat(auto-fit, minmax(16rem, 1fr))`, `gap` вместо margin-хаков.
- Медиа-запросы дополняют, но `minmax`/`auto-fit` снимают часть брейкпоинтов.
- `subgrid` (где поддерживается) выравнивает вложенные карточки по родительской сетке.
- Честно добавляю: одна и та же сетка на Flex и Grid одновременно усложняет код; выбираю один инструмент под задачу.

## Как это работает

**Flex-каталог:**

```css
.card-grid {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card-grid > * {
  flex: 1 1 18rem;
  max-width: 100%;
}
```

Элементы тянутся от ~288px, при нехватке места переносятся. `gap` задаёт ритм без отрицательных margin.

**Grid с auto-fit:**

```css
.topic-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(14rem, 1fr));
  gap: 1.25rem;
}
```

`auto-fit` схлопывает пустые треки; `minmax` не даёт колонке стать уже минимума.

Комбинированный лейаут страницы:

```css
.app-shell {
  display: grid;
  grid-template-columns: minmax(12rem, 240px) 1fr;
  min-height: 100dvh;
}

@media (max-width: 48rem) {
  .app-shell {
    grid-template-columns: 1fr;
  }
}
```

Sidebar и main на Grid; внутри main карточки снова на `topic-grid`.

## Подводные камни

**flex-basis в процентах без gap.** Два по 50% с margin ломают строку; `gap` или `calc` обязательны.

**auto-fill vs auto-fit.** `auto-fill` оставляет пустые колонки; визуально «дырявый» ряд.

**Grid без minmax на узком экране.** Фиксированные `200px 200px 200px` дают горизонтальный скролл.

**Выравнивание высоты карточек.** Flex/Grid не выравнивают контент внутри; нужен внутренний flex column с `margin-top: auto` на футере карточки.

**Порядок в DOM для a11y.** `order` в Flex меняет визуальный порядок, не фокусный; осторожно с клавиатурой.
