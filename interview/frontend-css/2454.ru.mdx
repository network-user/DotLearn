# Чем fixed отличается от sticky

## Суть

`fixed` всегда привязан к viewport (или transform ancestor) и не скроллится со страницей. `sticky` ведёт себя как relative внутри scroll container, пока не достигнет offset (`top`), затем «прилипает» до конца контейнера.

## Что ответить на собесе

- fixed: глобальный header/FAB; выходит из document flow с первого кадра.
- sticky: section heading или table thead внутри прокручиваемой области.
- sticky требует заданный `top`/`bottom` и предка без `overflow:hidden` на цепочке.
- fixed перекрывает контент; sticky освобождает место в потоке до stick.
- Честно: на iOS старые баги sticky в nested scroll; тестирую реальные devices.

## Как это работает

```css
.site-header {
  position: fixed;
  top: 0;
  inset-inline: 0;
  z-index: 50;
}

.article {
  overflow: auto;
  max-height: 70vh;
}

.article h2 {
  position: sticky;
  top: 0;
  background: #fff;
  padding-block: 0.5rem;
}
```

```ts
const header = document.querySelector<HTMLElement>(".site-header");
const height = header?.getBoundingClientRect().height ?? 0;
document.documentElement.style.setProperty(
  "--header-h",
  `${height}px`
);
```

```css
main {
  padding-top: var(--header-h, 3.5rem);
}
```

fixed живёт вне section; sticky отпускает элемент при выходе `.article` из viewport.

## Подводные камни

**sticky без height у scrollport.** Некуда «липнуть», эффект не виден.

**overflow:hidden на body/html.** Ломает sticky цепочку.

**fixed + 100vh на mobile.** Address bar прыгает; нужен `dvh`.

**Два sticky с одним top.** Налезают друг на друга без z-index plan.

**IntersectionObserver вместо sticky.** Иногда проще для complex tables.
