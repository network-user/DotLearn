# Which CSS properties trigger repaint

## In short

Repaint redraws pixels without layout geometry recalc. Typical triggers: color, background, border-color, outline, box-shadow, visibility (partially). Cheaper than reflow, but mass repaint in animation hurts FPS.

## How to answer in an interview

- Paint-only: `color`, `background`, `border-color`, `outline`, `text-decoration`, `box-shadow` without size change.
- `visibility:hidden` is often paint-only; `display:none` drops from the render tree (layout).
- Compositor-friendly: `transform`, `opacity` skip full layout repaint.
- Batch DOM writes; layout reads after writes force extra stages.
- Honestly: large blurred shadows are expensive repaint even without layout.

## How it works

```ts
const chip = document.querySelector<HTMLElement>(".chip");
if (!chip) return;

chip.style.backgroundColor = "#fef3c7";
chip.style.color = "#92400e";
chip.style.boxShadow = "0 2px 8px rgb(146 64 14 / 20%)";
chip.style.borderColor = "#f59e0b";
```

```css
.chip {
  padding: 0.35rem 0.65rem;
  border: 1px solid transparent;
  border-radius: 6px;
  transition: background-color 160ms ease, color 160ms ease;
}

.chip.is-active {
  background: #dbeafe;
  color: #1e40af;
}
```

## Pitfalls

**background-image swap.** Decode + paint beats solid color.

**filter: blur on large surfaces.** GPU fill, not layout, but jank.

**Animated box-shadow spread.** Repaint every frame.

**visibility vs opacity.** Opacity often compositor-only; visibility may paint+composite.

**DevTools disable cache.** Paint flashing overlay misleads vs prod hardware.
