# Flexbox: container and item properties

## In short

Flexbox lays out direct children along main and cross axes. The container sets direction, wrapping, and alignment; items control growth, shrink, and self-alignment.

## How to answer in an interview

- Container: `display:flex`, `flex-direction`, `flex-wrap`/`flex-flow`, `justify-content`, `align-items`, `align-content`, `gap`.
- Items: `flex-grow`, `flex-shrink`, `flex-basis` (shorthand `flex`), `align-self`, `order`.
- `justify-*` is on the main axis, `align-*` on the cross axis; with `column`, axes swap.
- `flex: 1` often means `1 1 0%` and shares free space evenly.
- I honestly add that `order` changes visual order, not tab focus; risky for a11y without careful DOM.

## How it works

Toolbar layout:

```css
.toolbar {
  display: flex;
  flex-wrap: wrap;
  align-items: center;
  justify-content: space-between;
  gap: 0.75rem;
}

.toolbar__actions {
  display: flex;
  gap: 0.5rem;
  margin-left: auto;
}

.toolbar__btn {
  flex: 0 0 auto;
}
```

`margin-left: auto` on the action group pushes it to the end of the main axis without `justify-content: flex-end` on the whole bar.

Card with growing body:

```css
.card {
  display: flex;
  flex-direction: column;
  min-height: 12rem;
}

.card__body {
  flex: 1 1 auto;
}

.card__footer {
  flex-shrink: 0;
}
```

`flex-basis` sets the starting size before free space is distributed. `flex-shrink: 0` keeps the footer from collapsing when height is tight.

## Pitfalls

**min-width: auto on flex items.** Long text refuses to shrink; fix with `min-width: 0`.

**align-items: stretch and nested images.** Images distort unless `align-self: flex-start`.

**flex-basis: 100% with wrap.** One item takes the full row and breaks multi-column rows.

**gap in older Safari.** Keep a margin fallback for critical layouts or verify browser support.

**column + justify-content: center on short viewports.** Overflow clips the top; prefer stretch and inner scroll.
