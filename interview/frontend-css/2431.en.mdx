# What determines a block element's size

## In short

The final box combines content, padding, border, and sometimes margin in flow. Explicit width/height, box-sizing, content, percentages from the parent, and formatting context all matter.

## How to answer in an interview

- A block defaults to full parent width; height follows content unless set.
- `box-sizing: border-box` includes padding and border in width/height; `content-box` adds them outside.
- Width percentages use the containing block width; height percentages need a defined parent height.
- min/max-width/height, overflow, and float/flex/grid change the used size.
- Honestly: margin collapse does not grow the border box but visually merges vertical gaps between siblings.

## How it works

Card in a container with border-box:

```css
.shell {
  width: min(100%, 360px);
  margin-inline: auto;
}

.panel {
  width: 100%;
  padding: 1rem;
  border: 2px solid #334155;
  box-sizing: border-box;
  background: #f8fafc;
}
```

```ts
const shell = document.querySelector<HTMLElement>(".shell");
const panel = document.querySelector<HTMLElement>(".panel");

if (shell && panel) {
  const outer = panel.getBoundingClientRect().width;
  const content = panel.clientWidth;
  console.log({ outer, content });
}
```

In a flex row, `flex-basis` and `min-width: 0` often beat raw `width: 100%`.

## Pitfalls

**height: 100% without parent height.** The child stays effectively auto.

**width: 100% + padding in content-box.** Horizontal scroll on mobile.

**min-content and long words.** The block blows past the viewport without `overflow-wrap`.

**Margin collapse.** Vertical margins merge; debugging gets confusing.

**Swapping width for flex.** `width: 100%` inside a flex item behaves unlike block flow.
