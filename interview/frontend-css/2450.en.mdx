# What align-content does in flex and grid

## In short

`align-content` distributes free space between flex lines or grid tracks on the cross axis when there is more than one line and extra room in the container. It does not affect a single line; use `align-items` for items inside a line.

## How to answer in an interview

- Works only with `flex-wrap: wrap` (or multi-row grid) and extra cross size on the container.
- Values mirror `justify-content` on the cross axis: `center`, `space-between`, `stretch`, etc.
- Do not confuse with `align-items`: items in one line vs lines as a group.
- In grid, `align-content` moves track bands, not individual cells (unless `align-self`).
- Honestly: without a definite cross size on the container, extra space may not exist and the property looks inert.

## How it works

```css
.dashboard {
  display: flex;
  flex-wrap: wrap;
  align-content: space-between;
  gap: 0.75rem;
  height: 20rem;
  padding: 1rem;
}

.widget {
  flex: 1 1 8rem;
  min-height: 5rem;
  border-radius: 10px;
  background: #f1f5f9;
}
```

Grid variant:

```css
.board {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-auto-rows: 4rem;
  align-content: center;
  height: 24rem;
  gap: 0.5rem;
}
```

```ts
const board = document.querySelector<HTMLElement>(".board");
if (board) {
  board.style.alignContent = "space-evenly";
}
```

Sketch: multiple lines inside the container with gaps between lines, not between items on one line (that is gap/align-items).

## Pitfalls

**align-content without wrap.** On a single flex line it does nothing.

**height:auto parent.** No free cross size, align-content is a no-op.

**Mixing justify and align.** In column flex, main and cross axes swap.

**grid align-content vs place-content.** Shorthand overwrites both axes.

**Older Safari multi-line flex bugs.** Verify space-* values on iOS.
