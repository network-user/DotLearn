# Почему в CSS нельзя стилизовать родителя от потомка

## Суть

CSS применяется сверху вниз по дереву: правило выбирает предка и влияет на потомков, но не наоборот. Так сохраняются предсказуемый каскад, отсутствие циклов и дешёвый пересчёт стилей.

## Что ответить на собесе

- Каскад однонаправленный: селектор «родитель потомок», не «потомок родитель».
- Обратная связь создала бы циклы специфичности и нестабильный порядок применения правил.
- Для «подсвети карточку, если внутри ошибка» использую `:has()`, класс через JS или data-атрибут на родителе.
- Flex/Grid решают часть задач выравнивания без стилизации родителя изнутри.
- Честно: `:has()` уже в baseline, но старые корпоративные браузеры могут потребовать fallback.

## Как это работает

Классический обход через JS:

```ts
function syncParentHighlight(root: HTMLElement) {
  const invalid = root.querySelector<HTMLInputElement>(
    "input:user-invalid, input:invalid"
  );

  root.classList.toggle("has-field-error", Boolean(invalid));
}

document.querySelectorAll<HTMLElement>(".field-group").forEach((group) => {
  group.addEventListener("input", () => syncParentHighlight(group));
  syncParentHighlight(group);
});
```

```css
.field-group {
  border: 1px solid #cbd5e1;
  border-radius: 8px;
  padding: 0.75rem;
}

.field-group.has-field-error {
  border-color: #dc2626;
  background: color-mix(in srgb, #dc2626 6%, transparent);
}
```

С `:has()` родитель реагирует на потомка без скрипта:

```css
.field-group:has(input:user-invalid) {
  border-color: #dc2626;
}
```

Layout-задачи часто закрываются flex на родителе:

```css
.stack {
  display: flex;
  flex-direction: column;
  gap: 0.5rem;
}

.stack > .grow {
  flex: 1 1 auto;
}
```

## Подводные камни

**:has() и производительность.** Широкий селектор на большом DOM может стоить дороже простого класса.

**Порталы и модалки.** Потомок вне DOM-ветки родителя, `:has()` и наследование не сработают.

**Специфичность :has().** Легко перебить utility-классы и получить «липкие» стили.

**JS-only класс без debounce.** На каждый input пересчитывать всё дерево формы не нужно.

**Ожидание «parent selector».** Его нет; только `:has()` с ограничениями или состояние на предке.
