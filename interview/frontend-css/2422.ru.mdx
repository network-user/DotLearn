# Анимация в CSS: transitions, keyframes и Web Animations API

## Суть

Простые смены состояния делают через `transition` между двумя значениями свойства. Сценарии с ключевыми кадрами и циклами описывают `@keyframes` + `animation`. Сложная логика и синхронизация с JS идут через Web Animations API или requestAnimationFrame.

## Что ответить на собесе

- Transition: задать свойство, длительность, easing; триггер через `:hover`, класс или `prefers-reduced-motion`.
- Animation: `@keyframes`, `animation-name`, duration, iteration, direction, fill-mode.
- Анимирую преимущественно `transform` и `opacity` для композитного слоя и 60fps.
- `transition-behavior: allow-discrete` для display/visibility в современных браузерах.
- Честно добавляю: тяжёлые свойства (`width`, `top`) грузят layout; на мобиле это заметно.

## Как это работает

Кнопка с transition:

```css
.btn {
  background: var(--brand);
  transform: translateY(0);
  transition:
    background-color 200ms ease,
    transform 200ms ease;
}

.btn:hover {
  transform: translateY(-2px);
}

@media (prefers-reduced-motion: reduce) {
  .btn { transition: none; }
}
```

Keyframes для индикатора загрузки:

```css
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.4; }
}

.skeleton {
  animation: pulse 1.2s ease-in-out infinite;
}
```

Web Animations API из TypeScript:

```ts
const panel = document.querySelector<HTMLElement>(".panel");
if (!panel) return;

panel.animate(
  [
    { transform: "translateX(-100%)" },
    { transform: "translateX(0)" },
  ],
  { duration: 280, easing: "cubic-bezier(0.2, 0.8, 0.2, 1)", fill: "forwards" }
);
```

WAAPI даёт pause/reverse/currentTime и Promise на `finished` для цепочек без CSS class toggle.

## Подводные камни

**Анимация width/height.** Вызывает reflow каждый кадр; заменяю на `transform: scale` или `max-height` с осторожностью.

**transition: all.** Неожиданно анимирует новые свойства; перечисляю явно.

**Бесконечный animation без reduced-motion.** Укачивает чувствительных пользователей.

**z-index во время animation.** Новый stacking context от `transform` может перекрыть dropdown.

**Дублирование логики CSS + JS.** Две системы анимации одного элемента рассинхронизируются; выбираю один источник правды.
