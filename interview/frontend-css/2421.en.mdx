# CSS specificity: how to weigh a selector

## In short

Specificity is selector weight when properties conflict. Compare columns: inline, ids, classes/attributes/pseudo-classes, types/pseudo-elements; universal `*` and combinators add no weight.

## How to answer in an interview

- Tuple (a, b, c, d): inline style; ids; classes/attrs/:pseudo-class; tags/::pseudo-elements.
- `#nav .link:hover` → (0, 1, 2, 1): one id, class plus :hover, plus the element if present.
- `:where(.btn)` zeroes class contribution; `:is(.a, #b)` takes the heaviest argument.
- Inline `style=""` beats ids unless author `!important` wins in modern layer rules.
- I honestly add that DevTools specificity view beats memorizing digits.

## How it works

Rule set:

```css
p { color: #64748b; }                    /* 0,0,0,1 */
.text-muted { color: #94a3b8; }           /* 0,0,1,0 */
p.intro { color: #0f172a; }               /* 0,0,1,1 */
#hero p.intro { color: #dc2626; }          /* 0,1,1,1 */
```

```html
<p id="hero" class="intro" style="color: #2563eb">Text</p>
```

Inline `#2563eb` wins unless a heavier author `!important` exists in the same cascade layer.

`:not()` counts its argument:

```css
li:not(.skip) { }  /* .skip adds 0,0,1,0 plus li 0,0,0,1 */
```

`:has()` includes the full selector inside.

Lowering library weight:

```css
:where(.ds-button) {
  padding: 0.5rem 1rem;
}
```

Consumers override with a single class without an id arms race.

## Pitfalls

**Only counting the last token.** `.sidebar .nav a` sums the whole chain.

**Mixing specificity with order.** Equal weight goes to the last rule.

**!important breaks the table.** Importance compares before specificity within an origin.

**Inheritance is not specificity.** `color` on `body` does not duel a class on `p` as peers.

*** selector is 0,0,0,0.** Universal selectors do not beat a class.
