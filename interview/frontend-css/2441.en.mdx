# Media queries: purpose and usage

## In short

Media queries apply CSS when device or viewport conditions match: width, height, orientation, prefers-reduced-motion, pointer type. They underpin responsive and accessibility tuning without separate URLs.

## How to answer in an interview

- Syntax: `@media (condition) { ... }`; combine with `and`, `not`, comma (or).
- Mobile-first: base styles without a query, extensions via `min-width`.
- Beyond width I check `prefers-reduced-motion`, `hover: hover`, `prefers-color-scheme`.
- Container queries (`@container`) complement viewport media when a component sits in a sidebar.
- Honestly: rigid popular px breakpoints age poorly; fluid type/spacing + containers scale better.

## How it works

```css
.page {
  padding: 1rem;
  font-size: 1rem;
  background: #fff;
  color: #0f172a;
}

@media (min-width: 40rem) {
  .page {
    padding: 1.5rem 2rem;
    font-size: 1.0625rem;
  }
}

@media (max-width: 40rem) {
  .page {
    background: #f8fafc;
    font-size: 0.9375rem;
  }
}

@media (prefers-reduced-motion: reduce) {
  .page * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

```ts
const narrow = window.matchMedia("(max-width: 40rem)");

function syncLayout(event: MediaQueryList | MediaQueryListEvent) {
  document.body.dataset.layout = event.matches ? "compact" : "wide";
}

syncLayout(narrow);
narrow.addEventListener("change", syncLayout);
```

Print stylesheet:

```css
@media print {
  .nav {
    display: none;
  }
}
```

## Pitfalls

**Desktop-first max-width sprawl.** Overlapping rules are harder to maintain.

**matchMedia without cleanup.** Leaks in SPAs on mount/unmount.

**Orientation alone.** Tablet rotation is not a mobile layout signal.

**Hover media on touch.** `:hover` styles stick without `@media (hover: hover)`.

**Container vs viewport confusion.** `@media (min-width)` does not fix a narrow sidebar in a wide window.
