# What FLIP is and why it helps smooth motion

## In short

FLIP (First, Last, Invert, Play) animates layout shifts via transform: record geometry before and after a DOM change, compensate with translate, then transition transform away. Layout updates once; motion runs on the compositor.

## How to answer in an interview

- First: `getBoundingClientRect()` before DOM/CSS layout change.
- Last: apply the new state (reorder, grid area), measure again.
- Invert: `translate(deltaX, deltaY)` visually returns to the start.
- Play: clear transform with a transition on transform/opacity.
- Honestly: FLIP fails if content width/height changes between First and Last without stabilization.

## How it works

Reordering a card in a list:

```ts
function flipMove(node: HTMLElement, mutate: () => void) {
  const first = node.getBoundingClientRect();

  mutate();

  const last = node.getBoundingClientRect();
  const dx = first.left - last.left;
  const dy = first.top - last.top;

  node.style.transform = `translate(${dx}px, ${dy}px)`;
  node.style.transition = "transform 0s";

  requestAnimationFrame(() => {
    node.style.transition = "transform 280ms ease";
    node.style.transform = "translate(0, 0)";
  });
}

const tile = document.querySelector<HTMLElement>(".tile");
if (tile) {
  flipMove(tile, () => {
    tile.style.order = "3";
  });
}
```

```css
.tile {
  will-change: transform;
}

@media (prefers-reduced-motion: reduce) {
  .tile {
    transition: none !important;
  }
}
```

For lists, measure every item, not one:

```ts
type RectMap = Map<Element, DOMRect>;

function capture(nodes: Element[]): RectMap {
  return new Map(nodes.map((n) => [n, n.getBoundingClientRect()]));
}
```

## Pitfalls

**Subpixel rounding.** Fractional translate wobbles on retina screens.

**Scroll during FLIP.** Rect shifts; pin the scroll container or FLIP only in view.

**Missing transitionend.** Transform sticks and breaks hit-testing.

**height:auto animation.** FLIP is about position; max-height or grid tricks are separate.

**will-change everywhere.** GPU memory grows; remove it after the animation.
