# Media queries: зачем и как применять

## Суть

Media queries включают блок CSS при совпадении условий устройства или viewport: ширина, высота, orientation, prefers-reduced-motion, pointer type. Это основа responsive и accessibility адаптации без отдельных URL.

## Что ответить на собесе

- Синтаксис: `@media (condition) { ... }`, условия комбинирую через `and`, `not`, запятую (or).
- Mobile-first: базовые стили без query, расширения через `min-width`.
- Помимо width смотрю `prefers-reduced-motion`, `hover: hover`, `prefers-color-scheme`.
- Container queries (`@container`) дополняют viewport media, когда компонент живёт в sidebar.
- Честно: жёсткие breakpoints по популярным px устаревают; лучше fluid type/spacing + container.

## Как это работает

```css
.page {
  padding: 1rem;
  font-size: 1rem;
  background: #fff;
  color: #0f172a;
}

@media (min-width: 40rem) {
  .page {
    padding: 1.5rem 2rem;
    font-size: 1.0625rem;
  }
}

@media (max-width: 40rem) {
  .page {
    background: #f8fafc;
    font-size: 0.9375rem;
  }
}

@media (prefers-reduced-motion: reduce) {
  .page * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

```ts
const narrow = window.matchMedia("(max-width: 40rem)");

function syncLayout(event: MediaQueryList | MediaQueryListEvent) {
  document.body.dataset.layout = event.matches ? "compact" : "wide";
}

syncLayout(narrow);
narrow.addEventListener("change", syncLayout);
```

Print stylesheet:

```css
@media print {
  .nav {
    display: none;
  }
}
```

## Подводные камни

**Desktop-first max-width лес.** Много перекрывающихся правил сложнее поддерживать.

**matchMedia без removeListener.** Утечки в SPA при mount/unmount.

**orientation alone.** На планшетах orientation смена не равна «mobile layout».

**Hover media на touch.** `:hover` стили «залипают» без `@media (hover: hover)`.

**Container vs viewport путаница.** `@media (min-width)` не спасает узкий sidebar с широким окном.
