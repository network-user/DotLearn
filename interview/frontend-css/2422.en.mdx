# Animating CSS: transitions, keyframes, and the Web Animations API

## In short

Simple state changes use `transition` between two property values. Multi-step loops use `@keyframes` plus `animation`. Complex timing and orchestration use the Web Animations API or requestAnimationFrame.

## How to answer in an interview

- Transition: set property, duration, easing; trigger via `:hover`, a class, and respect `prefers-reduced-motion`.
- Animation: `@keyframes`, `animation-name`, duration, iteration, direction, fill-mode.
- I animate mostly `transform` and `opacity` for compositor-friendly 60fps.
- `transition-behavior: allow-discrete` helps display/visibility in modern browsers.
- I honestly add that heavy properties (`width`, `top`) trigger layout every frame; mobile feels it.

## How it works

Button transition:

```css
.btn {
  background: var(--brand);
  transform: translateY(0);
  transition:
    background-color 200ms ease,
    transform 200ms ease;
}

.btn:hover {
  transform: translateY(-2px);
}

@media (prefers-reduced-motion: reduce) {
  .btn { transition: none; }
}
```

Loading skeleton keyframes:

```css
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.4; }
}

.skeleton {
  animation: pulse 1.2s ease-in-out infinite;
}
```

Web Animations API from TypeScript:

```ts
const panel = document.querySelector<HTMLElement>(".panel");
if (!panel) return;

panel.animate(
  [
    { transform: "translateX(-100%)" },
    { transform: "translateX(0)" },
  ],
  { duration: 280, easing: "cubic-bezier(0.2, 0.8, 0.2, 1)", fill: "forwards" }
);
```

WAAPI exposes pause/reverse/currentTime and `finished` promises for chains without class toggles.

## Pitfalls

**Animating width/height.** Reflows every frame; prefer `transform: scale` or careful `max-height`.

**transition: all.** Surprises when new properties change; list targets explicitly.

**Infinite animation without reduced-motion.** Harms motion-sensitive users.

**z-index during animation.** `transform` may create a stacking context over dropdowns.

**Duplicate CSS + JS animation.** Two sources of truth desync; pick one driver.
