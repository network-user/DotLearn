# What a media query is in CSS and JS

## In short

A media query is a condition on environment traits (viewport width, pointer type, reduced motion). In CSS it wraps rules in `@media`; in JS the same expression is checked with `window.matchMedia`.

## How to answer in an interview

- CSS: `@media (min-width: 48rem) { ... }` applies the block when true.
- JS: `matchMedia("(prefers-reduced-motion: reduce)").matches` for logic, not just styles.
- I listen to `change` on `MediaQueryList`, not poll resize every frame for breakpoints.
- Combine with `and`, comma (OR), and `not` for compound conditions.
- Honestly: media queries are viewport-based, not card width; use `@container` for the latter.

## How it works

```css
.nav {
  display: none;
}

@media (min-width: 48rem) {
  .nav {
    display: flex;
    gap: 1rem;
  }
}

@media (prefers-color-scheme: dark) {
  :root {
    color-scheme: dark;
  }
}
```

```ts
const coarsePointer = window.matchMedia("(pointer: coarse)");

function updateTapTargets(mql: MediaQueryList | MediaQueryListEvent) {
  document.body.dataset.pointer = mql.matches ? "coarse" : "fine";
}

updateTapTargets(coarsePointer);
coarsePointer.addEventListener("change", updateTapTargets);
```

A link with a media attribute loads CSS only when matched:

```html
<link rel="stylesheet" href="/print.css" media="print" />
```

## Pitfalls

**Resize storm.** Using `window.onresize` for breakpoints instead of matchMedia.

**Legacy addListener.** Prefer `addEventListener("change")`.

**Orientation as mobile proxy.** Unreliable on foldables and narrow desktop.

**max-device-width.** Deprecated; viewport meta + width is safer.

**Duplicate CSS/JS breakpoints.** rem/px numbers drift between files.
