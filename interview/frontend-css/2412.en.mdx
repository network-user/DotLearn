# Reset.css: why zero out browser default styles

## In short

Every browser ships a user-agent stylesheet: `body` margins, list markers, heading sizes. A reset zeros or equalizes those values so your design system starts from a predictable base.

## How to answer in an interview

- A reset clears margin/padding/border on a long selector list and often sets `box-sizing: border-box`.
- The goal is the same starting point in Chrome, Firefox, and Safari, not pretty defaults.
- An alternative is normalize.css, which keeps useful defaults (like `display: block` on `main`) and only fixes gaps.
- Today teams often use `@layer reset` in design tokens or Tailwind Preflight instead of a classic reset file.
- I honestly add that an aggressive reset breaks native form accessibility unless you restore focus/outline and control sizing.

## How it works

The classic **Eric Meyer reset** zeros spacing on many tags:

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

body,
h1,
h2,
h3,
p,
ul,
ol,
figure {
  margin: 0;
  padding: 0;
}

ul,
ol {
  list-style: none;
}

img {
  max-width: 100%;
  display: block;
}
```

After the reset, the project applies **base styles**: typography, colors, `line-height`, links, and buttons.

**Normalize** takes a different path: it does not zero everything; it patches known UA quirks (`line-height` in forms, `[hidden]` behavior).

Modern builds load reset as the first layer:

```css
@layer reset, base, components, utilities;
```

Specificity stays predictable and utilities do not fight a bare `margin: 0` on every tag.

## Pitfalls

**Stripping list-style without a plan.** Navigation loses visual bullets but AT still sees `ul`; style it deliberately.

**Removing :focus outline.** A common old reset mistake; keyboard users lose visibility.

**font: inherit on button/input.** Without follow-up rules, buttons look system-native and mismatch the UI kit.

**Double reset.** Meyer + Tailwind Preflight + custom zeros bloat CSS and specificity.

**Reset without design tokens.** Zeroing without spacing and type scales yields flat UI, not consistency.
