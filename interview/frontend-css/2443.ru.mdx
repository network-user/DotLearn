# Что такое media query в CSS и JS

## Суть

Media query это условие на характеристики среды (ширина viewport, тип pointer, reduced motion). В CSS оно оборачивает правила `@media`; в JS то же выражение проверяет `window.matchMedia`.

## Что ответить на собесе

- CSS: `@media (min-width: 48rem) { ... }` применяет блок при истинном условии.
- JS: `matchMedia("(prefers-reduced-motion: reduce)").matches` для логики, не только стилей.
- Слушаю `change` на `MediaQueryList`, не poll resize каждый кадр для breakpoint логики.
- Комбинирую `and`, запятую (OR), `not` для составных условий.
- Честно: media query про viewport, не про ширину карточки; для последнего нужен `@container`.

## Как это работает

```css
.nav {
  display: none;
}

@media (min-width: 48rem) {
  .nav {
    display: flex;
    gap: 1rem;
  }
}

@media (prefers-color-scheme: dark) {
  :root {
    color-scheme: dark;
  }
}
```

```ts
const coarsePointer = window.matchMedia("(pointer: coarse)");

function updateTapTargets(mql: MediaQueryList | MediaQueryListEvent) {
  document.body.dataset.pointer = mql.matches ? "coarse" : "fine";
}

updateTapTargets(coarsePointer);
coarsePointer.addEventListener("change", updateTapTargets);
```

Link с media attr загружает CSS только при match:

```html
<link rel="stylesheet" href="/print.css" media="print" />
```

## Подводные камни

**Resize storm.** `window.onresize` для breakpoint вместо matchMedia.

**Устаревший addListener.** Использую `addEventListener("change")`.

**orientation как proxy mobile.** Ненадёжно на foldable и desktop narrow.

**max-device-width.** Deprecated; viewport meta + width надёжнее.

**Дубли CSS и JS breakpoints.** Расходятся числа rem/px между файлами.
