# The CSS universal selector *: when it fits

## In short

The `*` selector matches every element in the document. Teams mostly use it in resets or normalize sheets to align browser defaults or set global rules like `box-sizing`, but broad use adds work for the selector engine.

## How to answer in an interview

- In a modern reset I set `box-sizing: border-box` via `*, *::before, *::after`.
- I do not use `*` for typography or colors: the rules are too wide and break components.
- I combine it with child selectors (`* > *`) to reset margins between flex/grid children.
- Honestly: on very large DOMs a global `*` can slow style recalc; I prefer `:where()` for zero specificity.
- For a11y I do not override focus styles on focusable elements without explicit `:focus-visible`.

## How it works

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

.layout > * {
  margin: 0;
}

.card * {
  /* avoid: too broad, hits nested widgets */
}
```

The universal selector does not add specificity beyond what is already in the chain. In the cascade it behaves like a type-like matcher: if `* { color: red }` conflicts with `.title { color: blue }`, the class wins. In practice a `*` reset is handy because you do not list `html, body, div, p, ...`.

Performance: the engine still walks the tree, but a root-level `*` forces every node to be checked when the rule changes. Production resets therefore keep the property set minimal.

## Pitfalls

- **Specificity traps.** `*.active` and `.active` weigh the same, but `*` in the middle (`div * span`) widens matching fast.
- **Inheritance.** `* { font-family }` may not reach pseudo-elements unless you target `::before/::after`.
- **Shadow DOM.** A global `*` does not pierce shadow roots; web components have their own cascade.
- **Third-party widgets.** A `*` reset can strip styles from embedded iframe/widget content on the host page.
- **Debugging.** In DevTools a rule on `*` is hard to trace: Sources lists many matches per node.
