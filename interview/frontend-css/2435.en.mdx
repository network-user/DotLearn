# What CSS-in-JS is and when teams pick it

## In short

CSS-in-JS writes styles inside JavaScript/TSX: styled components, object styles, or runtime injection. Styles bind to props and state; class names are often generated automatically.

## How to answer in an interview

- Pros: colocation with the component, prop-driven dynamics, scoped classes without hand-rolled modules.
- Cons: runtime cost, SSR setup, sometimes opaque DevTools, risk of duplicated CSS in the bundle.
- Libraries: Emotion, styled-components, Stitches; zero-runtime: Linaria, Vanilla Extract.
- For design tokens I combine a theme object with CSS variables.
- Honestly: many teams moved back to modules + tokens; CSS-in-JS is not the default everywhere.

## How it works

```ts
import styled from "@emotion/styled";

interface PillProps {
  tone?: "neutral" | "danger";
}

const Pill = styled.span<PillProps>`
  display: inline-flex;
  align-items: center;
  padding: 0.25rem 0.625rem;
  border-radius: 999px;
  font-size: 0.8125rem;
  font-weight: 600;
  background: ${({ tone }) =>
    tone === "danger" ? "#fee2e2" : "#e2e8f0"};
  color: ${({ tone }) =>
    tone === "danger" ? "#991b1b" : "#334155"};
`;

export function StatusBadge({ failed }: { failed: boolean }) {
  return <Pill tone={failed ? "danger" : "neutral"}>Status</Pill>;
}
```

Object style (inline object libs):

```ts
const rowStyle = {
  display: "grid",
  gridTemplateColumns: "1fr auto",
  gap: "0.5rem",
  alignItems: "center",
} as const;
```

Zero-runtime compile time:

```ts
import { css } from "@linaria/core";

export const shell = css`
  max-width: 48rem;
  margin-inline: auto;
  padding: 1rem;
`;
```

## Pitfalls

**Specificity wars with global CSS.** Injection order changes overrides.

**SSR hydration mismatch.** Different class hashes on server vs client trigger warnings.

**List performance.** A styled component per item multiplies injection work.

**No static extract.** Runtime libs inflate JS on mobile.

**Migration.** Mixing modules + CSS-in-JS in one UI kit complicates theming.
