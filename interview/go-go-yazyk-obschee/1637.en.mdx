# Ways to declare variables in Go

## In short

At package level: `var` with explicit type or a grouped block. Inside a function: `var` or short `:=` with type inference. Constants: `const` and `iota` for enumerations. Zero value applies automatically on `var x int` without initialization.

## How to answer in an interview

- `:=` only inside functions; package level needs `var`.
- `var (` block for a group of related variables.
- `const` for compile-time constants; not all types are allowed.
- I prefer `:=` for locals with immediate initialization.
- Honestly: `:=` does not create a new variable if at least one name in the block is new (gotcha).

## How it works

```go
var (
	buildVersion = "dev"
	maxRetries   = 3
)

const (
	StatusNew = iota
	StatusActive
	StatusDone
)

func process(limit int) {
	var buf bytes.Buffer
	deadline := time.Now().Add(time.Minute)
	count := 0

	for count < limit {
		count++
	}
	_ = buf
	_ = deadline
}
```

`x := 1` inside `if` creates a variable in the `if` scope. In an outer block an existing `err` in `foo, err := bar()` is reused; no new `err` is created.

## Pitfalls

**:= shadows outer err.** Accidentally skip handling a new error.

**var without type and non-obvious literal.** `var x = 3` is int, but `3.0` depends on context.

**const slice/map.** Immutable header, mutable backing array/map.

**Package-level vars without synchronization.** Data race on lazy init.

**Too many package-level vars.** Hard to isolate tests, hidden dependencies.
