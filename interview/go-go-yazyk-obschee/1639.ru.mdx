# Каким образом аргументы передаются в функцию по умолчанию в Go

## Суть

В Go аргументы всегда передаются по значению: функция получает копию. Изменения параметра внутри функции не затрагивают оригинал. Чтобы мутировать снаружи, передаём указатель `*T` или слайс/map/channel (header копируется, но backing/shared state общий).

## Что ответить на собесе

- По умолчанию — pass-by-value для всех типов, включая struct.
- Указатель даёт общий адрес; `*p = v` видно снаружи.
- Слайс: копируется descriptor (ptr,len,cap), элементы общие.
- Нет `ref`/`out` как в C#; явные указатели или return.
- Честно: большие struct копируются целиком — иногда выгоднее `*BigStruct`.

## Как это работает

```go
func increment(n int) {
	n++
}

func grow(s []int) {
	s = append(s, 99)
}

func main() {
	x := 10
	increment(x)
	fmt.Println(x)

	data := []int{1, 2}
	grow(data)
	fmt.Println(data)

	var cfg Config
	update(&cfg)
}
```

`increment` не меняет `x`. `grow` меняет shared array, но reassignment `s = append` не расширяет слайс caller без `return` или `*[]int`. Для struct-mutation — `func update(c *Config)`.

## Подводные камни

**Думать, что slice всегда immutable.** Append может перезаписать shared backing.

**Передавать `&largeStruct` везде «на всякий случай».** Усложняет ownership и nil-checks.

**Копировать mutex в struct by value.** Data race и undefined behavior.

**Путать value receiver и pointer receiver.** Метод с value receiver не мутирует оригинал.

**Ожидать copy-on-write.** Go не делает COW для slice/map автоматически.
