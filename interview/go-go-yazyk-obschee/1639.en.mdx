# How are function arguments passed by default in Go

## In short

In Go, arguments are always passed by value: the function receives a copy. Changes to the parameter inside the function do not affect the original. To mutate from outside, pass a pointer `*T` or a slice/map/channel (the header is copied, but the backing/shared state is shared).

## How to answer in an interview

- By default — pass-by-value for all types, including structs.
- A pointer shares the address; `*p = v` is visible outside.
- Slice: the descriptor (ptr, len, cap) is copied, elements are shared.
- No `ref`/`out` like C#; use explicit pointers or return values.
- Honestly: large structs are copied wholesale — sometimes `*BigStruct` is cheaper.

## How it works

```go
func increment(n int) {
	n++
}

func grow(s []int) {
	s = append(s, 99)
}

func main() {
	x := 10
	increment(x)
	fmt.Println(x)

	data := []int{1, 2}
	grow(data)
	fmt.Println(data)

	var cfg Config
	update(&cfg)
}
```

`increment` does not change `x`. `grow` may mutate the shared array, but reassignment `s = append` does not extend the caller's slice without `return` or `*[]int`. For struct mutation — `func update(c *Config)`.

## Pitfalls

**Assume slices are always immutable.** Append can overwrite shared backing.

**Pass `&largeStruct` everywhere "just in case".** Complicates ownership and nil checks.

**Copy a mutex inside a struct by value.** Data race and undefined behavior.

**Confuse value and pointer receivers.** A value receiver method does not mutate the original.

**Expect copy-on-write.** Go does not do COW for slices/maps automatically.
