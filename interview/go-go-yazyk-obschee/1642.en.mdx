# What is scope in Go

## In short

Scope defines where an identifier is visible. In Go: universe (predeclared), package (files of one package), file (init order), function/block (local `:=`/`var`). Export is by the first letter: `Public` is visible outside the package, `private` only inside.

## How to answer in an interview

- No "global" variables across packages without export.
- Package-level `var`/`const`/`func` — shared scope of all files in the package.
- Block scope: `{}`, `if`, `for`, `switch`, `select`.
- Imported names — only via qualifier `pkg.Name`.
- Honestly: "file scope" in Go does not isolate symbols — only the package boundary.

## How it works

```go
package service

const MaxBatch = 100

var registry = map[string]Handler{}

type handler struct {
	name string
}

func Register(name string, h Handler) {
	registry[name] = h
}
```

`MaxBatch` and `Register` are exported. `registry` and `handler` — only inside the package.

```go
func run(ctx context.Context) error {
	deadline, ok := ctx.Deadline()
	if !ok {
		return errors.New("no deadline")
	}

	for attempt := 0; attempt < 3; attempt++ {
		if err := try(deadline); err == nil {
			return nil
		}
	}
	return errors.New("exhausted")
}
```

`attempt` is visible only in the loop. `deadline` — in `run`'s body, not outside.

## Pitfalls

**Assume unexported symbols are inaccessible from another file in the same package.** They are accessible, but not from another package.

**Package-level mutable state without sync.** Hidden data races.

**init() order across files.** Depends on linking, not file name.

**Dot import or blank `_`.** Side-effect init without explicit use.

**Shadow predeclared names (`error`, `copy`).** Rare, but the compiler may not warn about confusion.
