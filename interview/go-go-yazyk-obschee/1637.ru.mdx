# Способы объявления переменных в Go

## Суть

На уровне пакета: `var` с явным типом или группой. Внутри функции: `var` или короткая `:=` с выводом типа. Константы: `const` и `iota` для перечислений. Нулевое значение применяется автоматически при `var x int` без инициализации.

## Что ответить на собесе

- `:=` только в функциях; на package level нужен `var`.
- `var (` блок для группы связанных переменных.
- `const` для compile-time констант; не все типы допустимы.
- Предпочитаю `:=` для локальных с немедленной инициализацией.
- Честно: `:=` не создаёт новую переменную, если хотя бы одна имя новое в блоке (gotcha).

## Как это работает

```go
var (
	buildVersion = "dev"
	maxRetries   = 3
)

const (
	StatusNew = iota
	StatusActive
	StatusDone
)

func process(limit int) {
	var buf bytes.Buffer
	deadline := time.Now().Add(time.Minute)
	count := 0

	for count < limit {
		count++
	}
	_ = buf
	_ = deadline
}
```

`x := 1` внутри `if` создаёт переменную в scope `if`. Во внешнем блоке уже существующая `err` при `foo, err := bar()` переиспользуется, новая не создаётся.

## Подводные камни

**:= затеняет outer err.** Случайно пропускаю новую ошибку.

**var без типа с неочевидным literal.** `var x = 3` int, но `3.0` другой контекст.

**const slice/map.** Неизменяемый header, но mutable backing array/map.

**Глобальные var без синхронизации.** Data race при lazy init.

**Слишком много package-level var.** Тесты сложно изолировать, скрытые зависимости.
