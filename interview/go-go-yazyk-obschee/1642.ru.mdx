# Что является областью видимости в Go

## Суть

Scope определяет, где виден идентификатор. В Go: universe (predeclared), package (файлы одного package), file (порядок init), function/block (локальные `:=`/`var`). Экспорт — по первой букве имени: `Public` виден снаружи package, `private` — только внутри.

## Что ответить на собесе

- Нет «глобальных» переменных между packages без export.
- Package-level `var`/`const`/`func` — общий scope всех файлов package.
- Блочный scope: `{}`, `if`, `for`, `switch`, `select`.
- Импортированные имена — только через qualifier `pkg.Name`.
- Честно: «файловая область» в Go не изолирует символы — только package boundary.

## Как это работает

```go
package service

const MaxBatch = 100

var registry = map[string]Handler{}

type handler struct {
	name string
}

func Register(name string, h Handler) {
	registry[name] = h
}
```

`MaxBatch` и `Register` экспортируются. `registry` и `handler` — только внутри package.

```go
func run(ctx context.Context) error {
	deadline, ok := ctx.Deadline()
	if !ok {
		return errors.New("no deadline")
	}

	for attempt := 0; attempt < 3; attempt++ {
		if err := try(deadline); err == nil {
			return nil
		}
	}
	return errors.New("exhausted")
}
```

`attempt` виден только в цикле. `deadline` — в теле `run`, но не снаружи.

## Подводные камни

**Считать unexported символ доступным из другого файла того же package.** Да, но не из другого package.

**Package-level mutable state без sync.** Скрытые data races.

**init() порядок между файлами.** Зависит от линковки, не от имени файла.

**Импорт dot или blank `_`.** Side-effect init без явного использования.

**Затенение predeclared (`error`, `copy`).** Редко, но компилятор не всегда предупредит о путанице.
