# Разрешённая и запрещённая авторегистрация в плане обмена

## Суть

Запрещённая авторегистрация: объект попадает в обмен только при явном `ЗарегистрироватьИзменения`. Разрешённая: платформа регистрирует каждое изменение автоматически. В типовых ERP преобладает запрет с точечной регистрацией.

## Что ответить на собесе

- Я настраиваю запрет для документов и тяжёлых справочников.
- Разрешаю авторегистрацию для небольших стабильных НСИ с фильтром узла.
- Явная регистрация после проведения даёт контроль состава пакета.
- При миграции с разрешённой на запрет нужен план первичной синхронизации.
- Честно: разрешённая авторегистрация на журналах регистрации не нужна никогда.

## Как это работает

В свойствах плана обмена для метаданных: флаг авторегистрации. В коде:

```bsl
Если ОбменДанными.Загрузка Тогда
    Возврат;
КонецЕсли;
ПланыОбмена.ЗарегистрироватьИзменения(Узел, Ссылка);
```

## Подводные камни

**Авторегистрация на всём.** Огромные пакеты и шум.
**Забытая регистрация при запрете.** Узел отстаёт.
**Регистрация в загрузке обмена.** Рекурсия и пинг-понг.
**Разные правила в узлах.** Один объект уезжает не туда.
**Тест только на пустой базе.** Не видно объём пакета.
