# Ограничения RLS в управляемых формах

## Суть

Динамический список и выбор из справочника на форме автоматически получают RLS. Прямое чтение по ссылке в модуле формы без сервера может показать данные, которые список скрывает, если нет серверной проверки.

## Что ответить на собесе

- Я настраиваю отбор списка выбора через параметры и стандартный механизм выбора.
- Открытие объекта по ссылке проверяет право чтения с учётом RLS.
- Не передаю на клиент ссылки на «чужие» документы из гиперссылок без проверки.
- Отчёты на форме выполняю на сервере в контексте пользователя.
- Честно: скрытие поля на форме не равно RLS; это только UI.

## Как это работает

`ОткрытьФорму(Ссылка)` вызывает проверку доступа. Динамический список подставляет ограничения в запрос. Для `СписокВыбора` задаю `СвязиПараметровВыбора` и отборы.

## Подводные камни

**Гиперссылка без проверки.** Пользователь открывает чужой документ по URL.
**Привилегированный серверный вызов из формы.** Утечка в обработчике.
**Кэш списка без RLS в фоне.** Неверные подсказки.
**Поле видимо, данных нет.** Плохой UX без пояснения ограничения.
**Обход через API.** Форма чистая, HTTP сервис без RLS.
