# Allowed vs denied auto-registration in exchange plan

## In short

Denied auto-registration: object enters exchange only on explicit `ЗарегистрироватьИзменения`. Allowed: platform registers every change automatically. Standard ERP mostly uses deny with selective registration.

## How to answer in an interview

- I deny for documents and heavy catalogs.
- I allow auto-registration for small stable master data with node filter.
- Explicit registration after posting controls package content.
- Migrating from allowed to deny needs initial sync plan.
- I am honest: allowed auto-registration on log registers is never needed.

## How it works

In exchange plan metadata properties: auto-registration flag. In code:

```bsl
Если ОбменДанными.Загрузка Тогда
    Возврат;
КонецЕсли;
ПланыОбмена.ЗарегистрироватьИзменения(Узел, Ссылка);
```

## Pitfalls

**Auto-registration on everything.** Huge noisy packages.
**Missed registration when denied.** Node falls behind.
**Registration during exchange load.** Recursion and ping-pong.
**Different rules per node.** Object ships to wrong place.
**Test only on empty base.** Package volume invisible.
