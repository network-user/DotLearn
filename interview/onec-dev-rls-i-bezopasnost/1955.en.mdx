# RLS limits in managed forms

## In short

Dynamic list and catalog pickers on a form get RLS automatically. Direct read by reference in the form module without server check may show data the list hides unless server validation exists.

## How to answer in an interview

- I configure choice list filter via parameters and standard choice mechanism.
- Opening an object by reference checks read right with RLS.
- I do not pass foreign document links to the client in hyperlinks without check.
- Form reports run on server in user context.
- I am honest: hiding a field on the form is not RLS; it is UI only.

## How it works

`ОткрытьФорму(Ссылка)` triggers access check. Dynamic list injects limits into the query. For `СписокВыбора` I set `СвязиПараметровВыбора` and filters.

## Pitfalls

**Hyperlink without check.** User opens foreign document by URL.
**Privileged server call from form.** Leak in handler.
**Background list cache without RLS.** Wrong suggestions.
**Field visible, no data.** Poor UX without limit explanation.
**Bypass via API.** Clean form, HTTP service without RLS.
