# RLS and predefined catalog items

## In short

Predefined items exist in all infobases with a fixed name; catalog RLS applies to them like regular items. Code often references them by name without checking user visibility.

## How to answer in an interview

- I do not assume a predefined item is readable by all roles.
- On forms I hide links to inaccessible predefined items via RLS and roles.
- Server procedures in privileged mode read service predefined items.
- In exchange predefined sync separately from RLS.
- I am honest: "item not found" is often RLS, not missing data.

## How it works

```bsl
Ссылка = Справочники.ВидыКонтактнойИнформации.НайтиПоНаименованию("Email");
```

If a profile limits the catalog by company and the item lacks that dimension, the list may be empty for the user.

## Pitfalls

**Hard reference in client code.** Fails for restricted role.
**Privileged mode everywhere.** RLS bypass without audit.
**Predefined duplicates in extension.** Name clash.
**Admin-only test.** Miss empty UI lists.
**Exchange breaks predefined.** Unrelated to RLS but similar symptom.
