# HTTP security with TLS

## In short

HTTPS wraps HTTP in TLS: the handshake negotiates cipher, the server proves identity via an X.509 cert signed by a CA, a symmetric session key encrypts the payload. Protects confidentiality, integrity, and server authentication. HSTS, secure cookies, TLS 1.2+ required in prod.

## How to answer in an interview

- TLS handshake: ClientHello, cert chain, key exchange, Finished.
- Cert from Let's Encrypt or corp CA; rotate before expiry.
- Go `ListenAndServeTLS` or reverse proxy termination.
- Honestly: TLS at the edge is enough only if the internal mesh is also encrypted.

## How it works

```go
package main

import (
	"log"
	"net/http"
)

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		w.Write([]byte("ok"))
	})
	log.Fatal(http.ListenAndServeTLS(":443", "cert.pem", "key.pem", mux))
}
```

The browser validates the cert chain to a trusted root. The application sets the `Strict-Transport-Security` header.

## Pitfalls

**Self-signed in prod without pin.** MITM risk.

**Mixed content HTTP assets.** Bypasses HTTPS.

**Weak cipher suite enabled.** Compliance fail.

**Cert expiry no alert.** Midnight outage.

**Terminate TLS twice misconfig.** Protocol errors.
