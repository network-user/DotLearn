# HTTP security with TLS

## Суть

HTTPS wraps HTTP in TLS: handshake negotiates cipher, server proves identity via X.509 cert signed by CA, symmetric session key encrypts payload. Protects confidentiality, integrity, server authentication. HSTS, secure cookies, TLS 1.2+ required in prod.

## Что ответить на собесе

- TLS handshake: ClientHello, cert chain, key exchange, Finished.
- Cert from Let's Encrypt or corp CA; rotate before expiry.
- Go `ListenAndServeTLS` or reverse proxy termination.
- Честно: TLS on edge enough if internal mesh also encrypted.

## Как это работает

```go
package main

import (
	"log"
	"net/http"
)

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		w.Write([]byte("ok"))
	})
	log.Fatal(http.ListenAndServeTLS(":443", "cert.pem", "key.pem", mux))
}
```

Browser validates cert chain to trusted root. Application sets `Strict-Transport-Security` header.

## Подводные камни

**Self-signed in prod without pin.** MITM risk.

**Mixed content HTTP assets.** Bypasses HTTPS.

**Weak cipher suite enabled.** Compliance fail.

**Cert expiry no alert.** Outage at midnight.

**Terminate TLS twice misconfig.** Protocol errors.
