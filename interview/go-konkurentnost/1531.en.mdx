# How do you read from an unbuffered channel?

## In short

An unbuffered channel synchronizes a send/receive pair: both sides rendezvous at one transfer point. Read with `<-ch`, `v, ok := <-ch`, `for range`, or `select` plus `ctx.Done()` for cancel. Without a ready sender, receive blocks.

## How to answer in an interview

- Producer and consumer must be running; otherwise the first operation deadlocks.
- Handoff gives happens-before between send and receive of the same value.
- For shutdown I combine post-close `ok` with `select` on a done channel.
- I honestly add: unbuffered is not "slower buffered" by nature; it is a different sync contract.

## How it works

```go
package main

import "fmt"

func main() {
	ping := make(chan string)

	go func() {
		ping <- "pong"
	}()

	msg := <-ping
	fmt.Println(msg)
}
```

Send blocks until main enters receive. The value copies directly from sender to receiver without a buffer.

## Pitfalls

- **Deadlock in main.** Send and receive in one goroutine on unbuffered channel.
- **Missing receiver.** Sender blocks forever; goroutine leak.
- **Select default.** An eager default hides a ready channel.
- **Nil channel in select.** That case never becomes ready; used to disable a branch.
- **Range without close.** Waits forever if the producer neither closes nor sends.
