# What operations can you perform on channels?

## In short

Create with `make(chan T)` or buffered `make(chan T, n)`. Operations: send `ch <- v`, receive `v := <-ch`, close `close(ch)`, check `v, ok := <-ch`, multiplex with `select`. Blocking depends on buffer space and the paired goroutine.

## How to answer in an interview

- Unbuffered channels synchronize handoff: send and receive rendezvous.
- Buffered send blocks only when full; receive blocks when empty.
- After close: zero value plus `ok=false` once drained; send on closed panics.
- I honestly add: for signal-only cases I use `chan struct{}` or `close` as broadcast.

## How it works

```go
package main

import "fmt"

func main() {
	jobs := make(chan int, 2)
	done := make(chan struct{})

	jobs <- 1
	jobs <- 2
	close(jobs)

	for {
		select {
		case v, ok := <-jobs:
			if !ok {
				done <- struct{}{}
				return
			}
			fmt.Println(v)
		case <-done:
			return
		}
	}
}
```

`select` picks a ready case; if several are ready, choice is pseudo-random. `default` makes a non-blocking attempt.

## Pitfalls

- **Send on nil channel.** Blocks forever.
- **Receive on nil channel.** Same permanent block.
- **Double close.** Panics; one owner, usually the producer.
- **Goroutine leak.** Send without receive (or vice versa) blocks forever.
- **Range without close.** `for v := range ch` waits for sender close; a forgotten close hangs.
