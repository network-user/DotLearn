# Как параллельно отправить 5 HTTP-запросов?

## Суть

Fan-out: goroutine per URL, shared `http.Client`, `WaitGroup` или errgroup для sync. Передаю request-scoped ctx с timeout; собираю results через chan или mutex slice. Client connection pool reuse TCP.

## Что ответить на собесе

- errgroup.WithContext: первый error cancel siblings.
- Reuse http.Client, не NewClient per request.
- Limit concurrency sem если URLs >> 5.
- Честно: «5 parallel» ok bare goroutines; production добавляет timeout и max idle conns.

## Как это работает

```go
package main

import (
	"context"
	"fmt"
	"net/http"
	"sync"
	"time"
)

func fetchAll(ctx context.Context, urls []string) []int {
	client := &http.Client{Timeout: 3 * time.Second}
	out := make([]int, len(urls))
	var wg sync.WaitGroup
	for i, url := range urls {
		wg.Add(1)
		go func(idx int, u string) {
			defer wg.Done()
			req, err := http.NewRequestWithContext(ctx, http.MethodGet, u, nil)
			if err != nil {
				return
			}
			resp, err := client.Do(req)
			if err != nil {
				return
			}
			resp.Body.Close()
			out[idx] = resp.StatusCode
		}(i, url)
	}
	wg.Wait()
	return out
}

func main() {
	ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
	defer cancel()
	urls := []string{
		"https://example.com",
		"https://example.org",
		"https://example.net",
		"https://example.edu",
		"https://example.gov",
	}
	fmt.Println(fetchAll(ctx, urls))
}
```

Пять goroutines, один client, indexed slice avoids race on append.

## Подводные камни

**No ctx on request.** Hang past shutdown.

**Shared resp without Close.** FD leak.

**Race on result slice append.** Preallocate by index or use chan.

**Unbounded parallel to one host.** Rate limit or sem; respect server.

**Default client no timeout.** Stuck forever on bad network.
