# Range по channel в Go

## Суть

`for v := range ch` читает до close(ch). После close drained values возвращаются, затем loop exit. Без close range блокирует навсегда. Закрывать должен sender, не consumer.

## Что ответить на собесе

- Range удобнее ручного `v, ok := <-ch`.
- Close сигнализирует EOF; ok=false на последнем zero value если chan drained.
- Отправка после close panic.
- Честно: range не подходит если writer живёт вечно без close strategy.

## Как это работает

```go
package main

import "fmt"

func emit(out chan<- int) {
	for i := 1; i <= 3; i++ {
		out <- i
	}
	close(out)
}

func main() {
	stream := make(chan int)
	go emit(stream)
	for n := range stream {
		fmt.Println(n)
	}
}
```

Producer закрывает после последнего send. Consumer range получает 1,2,3 и выходит без ok-check.

## Подводные камни

**Forgot close.** Range deadlock.

**Double close.** Panic.

**Close на receive-only side.** Compile error if typed `<-chan` only in wrong place; design smell.

**Buffered leftovers.** Close разрешает drain без новых sends.

**Break из range без drain.** Goroutine leak если writer blocked on send.
