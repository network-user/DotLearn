# Как синхронно дождаться нескольких goroutine?

## Суть

Основные паттерны: `sync.WaitGroup` для fork-join без результатов, `errgroup` для ошибок, закрытие `done` channel как broadcast, `select` на нескольких channel, `context` для отмены. Выбор зависит от того, нужны ли ошибки, результаты и дедлайн.

## Что ответить на собесе

- WaitGroup: `Add` перед `go`, `Done` в worker, `Wait` в coordinator.
- errgroup из `golang.org/x/sync/errgroup` отменяет сиблингов при первой ошибке.
- Channel: N senders, один receiver считает N сообщений или ждёт `close`.
- Честно говорю: `time.Sleep` в main для ожидания goroutine это антипаттерн в production.

## Как это работает

```go
package main

import (
	"fmt"
	"sync"
)

func worker(id int, out chan<- int, wg *sync.WaitGroup) {
	defer wg.Done()
	out <- id * 10
}

func main() {
	const n = 3
	out := make(chan int, n)
	var wg sync.WaitGroup

	wg.Add(n)
	for i := 1; i <= n; i++ {
		go worker(i, out, &wg)
	}

	wg.Wait()
	close(out)

	sum := 0
	for v := range out {
		sum += v
	}
	fmt.Println(sum)
}
```

WaitGroup гарантирует, что все send завершены до `close(out)`. Без `Wait` ранний `close` гоняется с send и даёт panic.

## Подводные камни

- **Close до Wait.** Race между send и close на channel.
- **WaitGroup Add после Wait.** Panic; планирую счётчик заранее.
- **errgroup без context.** Зависшие goroutine не отменяются при ошибке соседа.
- **Смешение WaitGroup и channel.** Два источника правды о завершении путают поддержку.
- **Блокировка main навсегда.** Забытый `Done` или deadlock на unbuffered channel.
