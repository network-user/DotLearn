# Which commands create a file in Linux?

## In short

In Linux a file appears on first write to a path: empty marker via `touch`, stdout redirect with `>`/`>>`, heredoc, or an editor. The choice depends on whether you need an empty placeholder, a one-line config, or interactive editing.

## How to answer in an interview

- I name `touch` for empty files and mtime updates, `echo`/`printf` with redirect for content.
- I mention `install -D`, `mktemp`, `tee` for scripts and automation.
- I stress permissions: umask, owner, SELinux context on prod hosts.
- I add honestly: in Ansible/Terraform we usually create files idempotently via modules, not manual `touch` on servers.

## How it works

`touch path` creates a zero-byte file or updates timestamps. `>` overwrites, `>>` appends. Heredoc fits multi-line configs in bash.

```bash
install -d -m 0750 /var/log/myapp
touch /var/log/myapp/audit.log
chmod 0640 /var/log/myapp/audit.log
chown app:app /var/log/myapp/audit.log

printf '%s\n' 'DB_HOST=127.0.0.1' 'DB_PORT=5432' > /etc/myapp/env.conf

cat <<'EOF' >> /etc/myapp/env.conf
METRICS_ENABLED=true
EOF

TMP=$(mktemp /tmp/deploy.XXXXXX)
echo "staging marker" > "$TMP"
mv "$TMP" /var/log/myapp/bootstrap.done
```

`install -d` creates a directory with mode atomically. `mktemp` avoids races for temp files. Redirect is handled by the shell before exec; new file mode = `666 & ~umask`.

## Pitfalls

- **`>` without path check.** A typo overwrites a system file or creates junk in `/`.
- **Missing directory.** Redirect to `dir/file` fails if `dir` is absent; use `mkdir -p`.
- **Unquoted heredoc.** `<<EOF` expands variables; for literal config use `<<'EOF'`.
- **NFS and attributes.** `touch` on NFS may behave oddly on timestamps; prefer `flock` for lock files.
- **Immutable flag.** `chattr +i` blocks even root overwrite until `chattr -i`.
