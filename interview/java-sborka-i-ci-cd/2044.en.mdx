# How to set the build stage for a Maven library

## In short

The usage stage is set with the `<scope>` attribute inside `<dependency>` in `pom.xml`. Scope decides whether the jar lands on compile classpath, test classpath, runtime, and in the production artifact. Without scope Maven treats the dependency as `compile` and packages it.

## How to answer in an interview

- `<scope>test</scope>` for JUnit/Mockito: visible only during `mvn test`.
- `<scope>provided</scope>` for servlet API: compile yes, omit from WAR.
- `<scope>runtime</scope>` for JDBC driver: main code compiles without the driver, prod needs it.
- I verify with `mvn dependency:tree` and `dependency:analyze`.
- Honestly: wrong scope often shows up only after deploy, not on `mvn compile`.

## How it works

Test dependency:

```xml
<dependency>
  <groupId>org.mockito</groupId>
  <artifactId>mockito-core</artifactId>
  <version>5.12.0</version>
  <scope>test</scope>
</dependency>
```

On `mvn package` mockito does not enter `target/myapp.jar`, but Surefire sees it on the test classpath.

Production + runtime driver:

```xml
<dependency>
  <groupId>com.zaxxer</groupId>
  <artifactId>HikariCP</artifactId>
  <version>5.1.0</version>
</dependency>
<dependency>
  <groupId>org.postgresql</groupId>
  <artifactId>postgresql</artifactId>
  <version>42.7.3</version>
  <scope>runtime</scope>
</dependency>
```

`HikariCP` (compile) goes into the fat jar; `postgresql` too, because runtime scope is packaged.

WAR on external Tomcat:

```xml
<dependency>
  <groupId>jakarta.servlet</groupId>
  <artifactId>jakarta.servlet-api</artifactId>
  <scope>provided</scope>
</dependency>
```

Maven compiler sees the API at build time; Tomcat supplies the implementation at runtime.

## Pitfalls

- **Scope does not inherit from BOM automatically.** Child POM must declare test/provided explicitly.
- **Fat jar plugin.** `maven-shade-plugin` can pull test scope with wrong configuration.
- **Optional vs scope.** `optional=true` cuts transitives; scope cuts classpath stages.
- **systemPath.** Bypasses the repository; CI fails without a local jar.
- **import scope in dependencies.** Only in `dependencyManagement`, otherwise invalid POM.
