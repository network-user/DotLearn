# Maven pros and cons in Java development

## In short

Maven offers predictable project layout, centralized dependency management, a rich plugin ecosystem, and native CI integration. Cons: verbose XML, less flexibility than Gradle, full rebuilds without incremental tricks, reliance on remote repositories (mitigated by Nexus mirror). Corporate Java monorepos still lean heavily on Maven.

## How to answer in an interview

- Pros: convention over configuration, reactor multi-module, BOM/`dependencyManagement`, mature plugins (Surefire, Failsafe, Enforcer).
- Cons: custom workflows harder than Gradle DSL, slower on huge repos without cache.
- Parent POM + CI `.m2` cache covers most pain.
- `mvnw` wrapper pins Maven version for every developer.
- Honestly: Maven XML copy-paste without a BOM leads to version hell.

## How it works

Standard layout:

```
src/main/java
src/test/java
pom.xml
```

Adding a dependency:

```xml
<dependency>
  <groupId>org.apache.commons</groupId>
  <artifactId>commons-collections4</artifactId>
  <version>4.4</version>
</dependency>
```

Maven downloads from Central (or corporate mirror) into `~/.m2/repository`.

CI-friendly wrapper:

```bash
./mvnw -B verify -DskipITs
```

Enforcer in parent POM:

```xml
<plugin>
  <artifactId>maven-enforcer-plugin</artifactId>
  <executions>
    <execution>
      <goals><goal>enforce</goal></goals>
      <configuration>
        <rules>
          <dependencyConvergence/>
          <requireJavaVersion><version>[21,)</version></requireJavaVersion>
        </rules>
      </configuration>
    </execution>
  </executions>
</plugin>
```

| Pro | Con |
|------|-------|
| Same structure across projects | XML boilerplate |
| Strong plugin documentation | Complex custom pipelines |
| Offline after first resolve | First resolve needs network |
| Jenkins/GitLab templates | Slower than Gradle on incremental |

## Pitfalls

- **SNAPSHOT in production.** Non-reproducible builds without locked versions.
- **Parent POM unreachable.** CI without VPN to internal Nexus fails on inherit.
- **Plugin without version.** Super-POM default may not support JDK 21.
- **Local `settings.xml` mirror.** Dev works, CI uses another mirror: different artifacts.
- **multi-module without `-am`.** Built child without dependent modules: runtime compile errors.
