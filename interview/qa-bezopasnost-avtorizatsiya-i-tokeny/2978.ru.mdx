# Тестирование аутентификации: чеклист QA инженера

## Суть

Аутентификация подтверждает личность до выдачи токена или сессии. QA покрывает позитивный login, негативные credentials, блокировку brute force, MFA, password reset, session timeout и transport security. Фокус на server-side: клиент может обойти UI, но не hash-check на backend.

## Что ответить на собесе

- Equivalence classes: пустые поля, wrong password, locked account, case sensitivity login, unicode password.
- Session fixation: после login session id меняется; cookie flags Secure, HttpOnly, SameSite.
- MFA: OTP window, backup codes, rate limit на resend SMS.
- Честно: полный OWASP ASVS не в каждом спринте; приоритет по risk matrix продукта.

## Как это работает

Области чеклиста:

| Область | Примеры тестов |
|---------|----------------|
| Login | valid/invalid, lockout |
| Logout | invalidate server session |
| Reset | token one-time, TTL |
| MFA | TOTP drift, bypass |

Gherkin smoke auth:

```gherkin
Feature: Базовая аутентификация

  Scenario: Успешный login
    Given активный пользователь qa@example.com
    When POST /auth/login с верным паролем
    Then статус 200
    And Set-Cookie session с Secure и HttpOnly

  Scenario: Brute force lockout
    Given лимит 5 неудачных попыток
    When 6-й POST с неверным паролем
    Then статус 429 или 401 account_locked
    And 7-й с верным паролем тоже отклонён до таймаута

  Scenario: Password reset one-time link
    Given reset token из email
    When смена пароля успешна
    And повторное использование ссылки
    Then статус 400 token_used
```

Для API использую Postman collection с negative data; для UI параллельно проверяю сообщения без user enumeration.

## Подводные камни

- **User enumeration.** Разные сообщения «user not found» vs «wrong password».
- **Remember me вечный cookie.** Logout не инвалидирует на сервере.
- **OAuth social только на UI.** API login без password не закрыт.
- **Captcha только на 10-й попытке.** Скрипт успевает 9.
- **Тестовый bypass header в staging.** Забыли выключить в prod.
