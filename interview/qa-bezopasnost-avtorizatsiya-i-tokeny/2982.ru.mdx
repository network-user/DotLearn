# Какие виды аутентификации QA должен уметь сравнить?

## Суть

Аутентификация подтверждает личность через один или несколько факторов: знание (пароль), владение (OTP, push), биометрия, сертификат или делегирование через OAuth. QA оценивает не название метода, а риски компрометации, UX fallback и покрытие негативных сценариев.

## Что ответить на собесе

- SFA (пароль): быстро, но уязвимо к фишингу и reuse; MFA обязательна для admin.
- 2FA/MFA: TOTP/SMS/push; SMS слабее TOTP из-за SIM-swap.
- Certificate/mTLS: для service-to-service; проверяю expiry и chain of trust.
- OAuth social login: удобно, но зависимость от IdP; тестирую account linking.
- Честно: биометрия на клиенте не заменяет server-side auth; это локальный gate.

## Как это работает

Классификация по числу факторов:

| Метод | Факторы | Типичный риск |
|-------|---------|---------------|
| Password only | 1 | Credential stuffing |
| Password + TOTP | 2 | Phishing with realtime relay |
| Passkey/WebAuthn | 2 (device + biometric) | Lost device without backup |
| Client cert | possession | Revocation lag |
| OAuth code flow | delegated | Misconfigured redirect_uri |

Gherkin:

```gherkin
Feature: Сравнение методов login

  Scenario: SFA блокирует после N неудач
    When пользователь 5 раз вводит неверный пароль
    Then аккаунт временно locked
    And email уведомление отправлено

  Scenario: TOTP обязателен для admin
    Given пользователь с ролью admin и включённым TOTP
    When login только паролем без OTP
    Then статус 401 и step-up required

  Scenario: OAuth redirect_uri whitelist
    When authorize запрос с redirect_uri=https://evil.example/callback
    Then IdP возвращает error invalid_redirect
```

OTP apps (Google Authenticator, Authy) генерируют 30-секундные коды по shared secret. OAuth переносит trust на Google/GitHub: приложение получает code, обменивает на token у IdP.

## Подводные камни

- **SMS как «настоящий» 2FA.** SS7/SIM-swap обходят доставку кода.
- **Backup codes без rotation.** Одноразовые коды живут в plaintext в email.
- **Remember device forever.** Cookie обходит MFA на months.
- **Biometric false accept rate.** Не тестируют на accessibility edge cases.
- **OAuth implicit flow в SPA legacy.** Token в URL fragment утекает через Referer.
