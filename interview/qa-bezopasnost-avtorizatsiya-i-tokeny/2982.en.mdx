# Which authentication types should QA be able to compare?

## In short

Authentication proves identity through one or more factors: knowledge (password), possession (OTP, push), biometrics, certificates, or delegation via OAuth. QA evaluates not the label but compromise risk, UX fallback, and negative scenario coverage.

## How to answer in an interview

- SFA (password): fast but vulnerable to phishing and reuse; MFA is mandatory for admin.
- 2FA/MFA: TOTP/SMS/push; SMS is weaker than TOTP due to SIM swap.
- Certificate/mTLS: for service-to-service; I check expiry and chain of trust.
- OAuth social login: convenient but IdP dependency; I test account linking.
- Honestly: client biometrics do not replace server-side auth; it is a local gate.

## How it works

Classification by factor count:

| Method | Factors | Typical risk |
|--------|---------|--------------|
| Password only | 1 | Credential stuffing |
| Password + TOTP | 2 | Phishing with realtime relay |
| Passkey/WebAuthn | 2 (device + biometric) | Lost device without backup |
| Client cert | possession | Revocation lag |
| OAuth code flow | delegated | Misconfigured redirect_uri |

Gherkin:

```gherkin
Feature: Comparing login methods

  Scenario: SFA locks after N failures
    When user enters wrong password 5 times
    Then account is temporarily locked
    And notification email is sent

  Scenario: TOTP required for admin
    Given user with admin role and TOTP enabled
    When login with password only without OTP
    Then status is 401 and step-up required

  Scenario: OAuth redirect_uri whitelist
    When authorize request with redirect_uri=https://evil.example/callback
    Then IdP returns error invalid_redirect
```

OTP apps (Google Authenticator, Authy) generate 30-second codes from a shared secret. OAuth shifts trust to Google/GitHub: the app gets a code and exchanges it for a token at the IdP.

## Pitfalls

- **SMS as "real" 2FA.** SS7/SIM swap bypass code delivery.
- **Backup codes without rotation.** One-time codes live in plaintext email.
- **Remember device forever.** Cookie skips MFA for months.
- **Biometric false accept rate.** Accessibility edge cases are not tested.
- **OAuth implicit flow in legacy SPA.** Token in URL fragment leaks via Referer.
