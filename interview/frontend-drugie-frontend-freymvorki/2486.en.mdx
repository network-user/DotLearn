# Angular application architecture

## In short

An Angular app is a tree of components with templates, orchestrated by modules (legacy) or standalone APIs, services with DI, and directives/pipes. The router lazy-loads feature areas. RxJS streams often glue async data. The CLI generates consistent structure for enterprise teams.

## How to answer in an interview

- Component = template + class + styles; `@Component` metadata.
- `@Injectable` services hold business logic; DI injects via constructors.
- Feature modules (or standalone routes) isolate domains: auth, billing.
- Honestly: NgModules are gradually replaced by standalone components in v17+.
- Strict TS, eslint, and schematics keep Nx-friendly monorepo layouts.

## How it works

```ts
import { Component, inject } from '@angular/core';
import { CatalogService } from './catalog.service';

@Component({
  selector: 'app-product-card',
  standalone: true,
  template: `
    <h2>{{ item.title }}</h2>
    <p>{{ item.price | currency }}</p>
    <button type="button" (click)="add()">Add</button>
  `,
})
export class ProductCardComponent {
  private catalog = inject(CatalogService);
  item = { title: 'Keyboard', price: 120 };

  add() {
    this.catalog.add(this.item);
  }
}
```

Bootstrap: `main.ts` → `bootstrapApplication(AppComponent, { providers: [...] })`. Router maps paths to lazy `loadComponent`. HTTP via `HttpClient` plus interceptors (auth header, error mapping).

Layers: smart containers fetch; dumb presentational components take `@Input()` and emit `@Output()`.

## Pitfalls

- **God modules.** Everything in AppModule → huge initial bundle.
- **Shared mutable service state.** Races without observable discipline.
- **Template-driven forms at scale.** Reactive typed FormGroup is safer.
- **Zone.js overhead.** Heavy third-party libs trigger extra CD cycles.
- **Upgrade lag.** Enterprise teams two majors behind; migration cost is real.
