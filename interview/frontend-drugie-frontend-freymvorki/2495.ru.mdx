# Слоты и scoped-slots во Vue

## Суть

Slots это API композиции: родитель передаёт разметку внутрь дочернего компонента, не ломая инкапсуляцию оболочки. Default slot принимает произвольный контент, named slots раскладывают его по зонам layout. Scoped slot (slot props) отдаёт данные из child наружу, чтобы parent решал, как их отрисовать.

## Что ответить на собесе

- `<slot />` placeholder в child; fallback контент внутри тега показывается, если parent ничего не передал.
- `v-slot:header` / `#footer` мапят named regions в card, modal, page shell.
- Scoped: child `<slot :row="item" />`, parent `#default="{ row }"` кастомизирует cell renderer.
- Честно: для headless logic предпочитаю composables + renderless pattern; slots остаются для layout composition.
- Vue 3 unified `v-slot` syntax; `slot-scope` из Vue 2 deprecated.

## Как это работает

Named slots:

```vue
<!-- PanelShell.vue -->
<template>
  <section class="panel">
    <header><slot name="toolbar" /></header>
    <main><slot /></main>
    <footer><slot name="actions" /></footer>
  </section>
</template>
```

```vue
<!-- ParentView.vue -->
<template>
  <PanelShell>
    <template #toolbar>
      <h2>Filters</h2>
    </template>
    <p>Main body content</p>
    <template #actions>
      <button type="button">Save</button>
    </template>
  </PanelShell>
</template>
```

Scoped slot для таблицы:

```vue
<!-- DataGrid.vue -->
<script setup lang="ts">
const rows = [{ id: 1, label: 'Alpha' }];
</script>

<template>
  <ul>
    <li v-for="row in rows" :key="row.id">
      <slot :row="row" />
    </li>
  </ul>
</template>
```

```vue
<template>
  <DataGrid v-slot="{ row }">
    <strong>{{ row.label }}</strong>
  </DataGrid>
</template>
```

Child владеет данными и структурой списка, parent владеет presentation каждой строки. Паттерн близок к render props в React, но декларативен в template.

## Подводные камни

- **Implicit default slot.** Несколько root nodes без `#default` могут попасть не туда в сложных wrapper.
- **Slot prop typing.** В TS нужен `defineSlots` или generic wrapper для автодополнения.
- **Performance.** Тысячи scoped slots в virtualized list без `memo` дают лишние vnode allocations.
- **Fallback misuse.** Тяжёлый fallback в slot скрывает missing content bug до production.
- **Teleport + slots.** Modal teleported наружу теряет контекст scoped slot, если не продумать структуру.
