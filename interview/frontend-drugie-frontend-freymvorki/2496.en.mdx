# watch and watchEffect in Vue 3

## In short

`watch` and `watchEffect` run side effects when reactive state changes. `watch` explicitly names a source (ref, getter, reactive field) and provides `(newVal, oldVal)`. `watchEffect` auto-collects deps inside the callback and re-runs when any of them change. Both live in the Composition API and complement `computed`, which only derives data without IO.

## How to answer in an interview

- `watch(source, cb, { immediate, deep, flush })` for precise control: debounced search, persisting form drafts.
- `watchEffect` for synced side effects: `document.title`, localStorage mirror, analytics beacons.
- `flush: 'post'` waits for DOM updates; useful when reading layout after a mutation.
- Honestly: excess watches are often replaced by `computed` + template; I keep watch for async and imperative APIs.
- Cleanup: `watchEffect` returns a stop handle; `onInvalidate` cancels in-flight fetch when an id changes.

## How it works

Explicit `watch`:

```ts
import { ref, watch } from 'vue';

const query = ref('');
const results = ref<string[]>([]);

watch(
  query,
  async (next, prev) => {
    if (next.length < 2) {
      results.value = [];
      return;
    }
    const payload = await fetch(`/api/search?q=${encodeURIComponent(next)}`).then((r) =>
      r.json()
    );
    results.value = payload.items;
  },
  { flush: 'post' }
);
```

Auto-tracked `watchEffect`:

```ts
import { ref, watchEffect } from 'vue';

const quantity = ref(1);
const lineTotal = ref(0);

watchEffect(() => {
  lineTotal.value = quantity.value * 19.5;
  console.info('line recalculated', lineTotal.value);
});
```

`watch` does not run until the first change unless `immediate: true`. `watchEffect` runs immediately, then on dep changes. For multiple sources use `watch([a, b], ([aN, bN], [aO, bO]) => ...)`. In `<script setup>` use auto-import or explicit import from `vue`.

## Pitfalls

- **Deep watch cost.** `{ deep: true }` on a large reactive tree is expensive; watch a specific field instead.
- **Async race.** Without abort/`onInvalidate`, an older fetch overwrites a newer result.
- **watchEffect loop.** Writing the same ref inside the effect without a guard causes an infinite loop.
- **SSR.** Side effects in `watchEffect` on the server duplicate; wrap in `onMounted` when needed.
- **vs React useEffect.** No manual dependency array, but implicit deps are harder to review.
