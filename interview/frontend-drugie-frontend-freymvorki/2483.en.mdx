# React vs Vue and Angular: key differences

## In short

React is a UI library with JSX and one-way data flow; routing/state live in the ecosystem. Vue is a progressive framework with SFCs and built-in reactivity. Angular is a full TypeScript platform with DI, RxJS, and opinionated structure. The choice depends on team skills, enterprise constraints, and migration paths.

## How to answer in an interview

- React: flexibility, hooks, virtual DOM, largest hiring pool; more decisions yourself.
- Vue: lower barrier, SFC ergonomics, official router/Pinia; strong for incremental adoption.
- Angular: batteries included, strict architecture, RxJS everywhere; enterprise long-lived apps.
- Honestly: all three render component trees; perf gaps usually come from app design, not framework religion.
- Microfrontends work on each; module federation is common in React/Webpack ecosystems.

## How it works

Counter in React (hooks):

```tsx
import { useState } from 'react';

export function Counter() {
  const [total, setTotal] = useState(0);
  return (
    <div>
      <p>Count: {total}</p>
      <button type="button" onClick={() => setTotal((n) => n + 1)}>
        +1
      </button>
    </div>
  );
}
```

Vue 3 script setup:

```vue
<script setup lang="ts">
import { ref } from 'vue';
const total = ref(0);
</script>

<template>
  <p>Count: {{ total }}</p>
  <button type="button" @click="total++">+1</button>
</template>
```

Angular standalone component:

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  standalone: true,
  template: `<p>Count: {{ total }}</p><button (click)="total = total + 1">+1</button>`,
})
export class CounterComponent {
  total = 0;
}
```

React updates via setState/hook batching. Vue tracks reactive dependencies automatically. Angular change detection zones (or signals in v16+) scan the component tree.

## Pitfalls

- **Resume-driven choice.** Framework mismatch hurts velocity more than benchmark decks.
- **React without routing discipline.** SPAs grow into unstructured folders.
- **Vue Options vs Composition.** Mixed styles in one codebase confuse juniors.
- **Angular bundle size.** Heavy initial payload; lazy routes are mandatory.
- **SSR story.** Next/Nuxt/Analog differ; meta-framework features are not identical.
