# Composition API в Vue 3

## Суть

Composition API группирует связанную logic по feature через functions (`ref`, `computed`, `watch`, lifecycle hooks) вместо разнесения по Options (`data`, `methods`, `mounted`). `<script setup>` sugar auto-exposes bindings в template. Composables (`useX`) переиспользуют logic между компонентами.

## Что ответить на собесе

- `ref` для primitives, `reactive` для objects; destructuring reactive теряет reactivity без `toRefs`.
- `setup()` или `<script setup>` выполняется до render, один раз per instance.
- Composables extract domain logic: `usePagination`, `useAuth`.
- Честно: Options API не deprecated; Composition wins on large components and TS inference.
- React hooks analogy close, но Vue auto tracks deps без dependency arrays.

## Как это работает

```ts
import { ref, computed, onMounted } from 'vue';

export function useCounter(initial = 0) {
  const count = ref(initial);
  const doubled = computed(() => count.value * 2);

  function increment(step = 1) {
    count.value += step;
  }

  onMounted(() => {
    console.info('counter ready', count.value);
  });

  return { count, doubled, increment };
}
```

```vue
<script setup lang="ts">
import { useCounter } from './useCounter';
const { count, doubled, increment } = useCounter(1);
</script>

<template>
  <p>{{ count }} · x2 = {{ doubled }}</p>
  <button type="button" @click="increment()">+</button>
</template>
```

Lifecycle: `onMounted` replaces `mounted` option. `watch(source, cb)` reacts to refs/reactive getters. `provide/inject` works across Composition trees for DI-like patterns.

## Подводные камни

- **Reactivity loss.** `const { x } = reactive(obj)` breaks; use `toRefs`.
- **Async setup.** Top-level await in `<script setup>` wraps component in Suspense boundary.
- **Composable naming.** Must start with `use` convention for lint/review clarity.
- **Over-extraction.** Tiny composable for one line adds indirection.
- **Mixing Options + setup.** Legacy migration period confusion for team.
