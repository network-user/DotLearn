# Vue 3 Composition API

## In short

The Composition API groups related logic by feature via functions (`ref`, `computed`, `watch`, lifecycle hooks) instead of scattering across Options (`data`, `methods`, `mounted`). `<script setup>` auto-exposes bindings to the template. Composables (`useX`) reuse logic across components.

## How to answer in an interview

- `ref` for primitives, `reactive` for objects; destructuring reactive loses tracking without `toRefs`.
- `setup()` or `<script setup>` runs before render, once per instance.
- Composables extract domain logic: `usePagination`, `useAuth`.
- Honestly: Options API is not deprecated; Composition wins on large components and TS inference.
- Close to React hooks, but Vue auto-tracks deps without dependency arrays.

## How it works

```ts
import { ref, computed, onMounted } from 'vue';

export function useCounter(initial = 0) {
  const count = ref(initial);
  const doubled = computed(() => count.value * 2);

  function increment(step = 1) {
    count.value += step;
  }

  onMounted(() => {
    console.info('counter ready', count.value);
  });

  return { count, doubled, increment };
}
```

```vue
<script setup lang="ts">
import { useCounter } from './useCounter';
const { count, doubled, increment } = useCounter(1);
</script>

<template>
  <p>{{ count }} · x2 = {{ doubled }}</p>
  <button type="button" @click="increment()">+</button>
</template>
```

Lifecycle: `onMounted` replaces the `mounted` option. `watch(source, cb)` reacts to refs/reactive getters. `provide/inject` works across Composition trees for DI-like patterns.

## Pitfalls

- **Reactivity loss.** `const { x } = reactive(obj)` breaks tracking; use `toRefs`.
- **Async setup.** Top-level await in `<script setup>` wraps the component in Suspense.
- **Composable naming.** Use the `use` prefix for lint/review clarity.
- **Over-extraction.** One-line composables add indirection.
- **Mixing Options + setup.** Legacy migration confuses teams.
