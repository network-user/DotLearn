# Angular vs React/Vue: плюсы и минусы для поддержки

## Суть

Angular даёт opinionated platform (router, forms, HTTP, DI, CLI) и единые conventions для больших команд. React library-first: freedom + ecosystem choices. Vue балансирует ergonomics и batteries. Maintenance tradeoff: consistency vs flexibility and hiring pool.

## Что ответить на собесе

- Angular: лучше для regulated enterprise, long lifecycle, strict TS; дороже onboarding.
- React: max ecosystem/jobs; architecture drift без lint/architecture guild.
- Vue: быстрый старт, SFC clarity; меньше enterprise playbook чем Angular.
- Честно: migration cost между ними rarely worth it; pick for team + product horizon.
- Shared design system может быть framework-agnostic (web components).

## Как это работает

Decision matrix (упрощённо):

| Критерий | Angular | React | Vue |
|----------|---------|-------|-----|
| Structure | CLI schematics | BYO | Official router/pinia |
| Learning curve | Steep | Medium | Gentle |
| Bundle default | Heavier | Depends | Medium |
| Corporate adoption | Banks, gov | Startups+FAANG | Agencies, Asia |

Maintenance practices общие: semver upgrades, codemods, e2e smoke, module boundaries. Angular `ng update` orchestrates migrations. React relies on `react-codemod` community. Vue `vue-migration-helper`.

Cross-team support: document ADRs, enforce state pattern (NgRx vs Redux vs Pinia), visual regression, dependency renovate bot.

## Подводные камни

- **Resume hiring.** Framework without bench slows fixes on vacation gaps.
- **Microfrontend mash.** Angular + React shell integration tax high.
- **Version skew.** Multiple majors in monorepo if apps independent.
- **Design system coupling.** Wrapping MUI vs Angular Material vs Vuetify not portable.
- **Performance blame game.** Issues often data-fetch architecture not framework.
