# Реактивность в UI-фреймворках

## Суть

Реактивность это автоматическая синхронизация state и DOM при изменении данных. Фреймворки реализуют её по-разному: push-based subscriptions (Vue, Angular signals), pull + virtual tree diff (React), или compile-time tracking (Svelte). Цель одна: убрать ручной `document.querySelector` и imperative updates.

## Что ответить на собесе

- React: `setState` / hooks триггерят re-render, reconciler diff'ит virtual tree и патчит DOM.
- Vue 3: Proxy на reactive objects, effect runner пересчитывает render function.
- Angular: zone.js (legacy) или signals + explicit `markDirty` в новых версиях.
- Честно: fine-grained reactivity (Vue/Svelte) иногда эффективнее, чем full component re-render, но benchmark зависит от паттерна UI.
- Реактивность не заменяет продуманную модель state: derived data лучше в `computed`, side effects в watchers.

## Как это работает

React hooks model:

```ts
import { useState } from 'react';

export function TallyButton() {
  const [total, setTotal] = useState(0);

  const bump = () => setTotal((prev) => prev + 1);

  return (
    <div>
      <p>Current total: {total}</p>
      <button type="button" onClick={bump}>
        Increment
      </button>
    </div>
  );
}
```

Vue 3 reactivity:

```ts
import { ref } from 'vue';

export function useTally(initial = 0) {
  const total = ref(initial);

  function bump(step = 1) {
    total.value += step;
  }

  return { total, bump };
}
```

```vue
<script setup lang="ts">
import { useTally } from './useTally';
const { total, bump } = useTally(0);
</script>

<template>
  <p>Current total: {{ total }}</p>
  <button type="button" @click="bump()">Increment</button>
</template>
```

Dependency tracking: Vue собирает deps при первом run effect. React сравнивает новый element tree со старым. Batch updates (React 18 `automatic batching`, Vue scheduler queue) группируют несколько мутаций в один paint.

## Подводные камни

- **Stale closures.** React handler захватывает старый state без functional updater.
- **Deep reactivity cost.** Большие nested objects в Vue reactive дороги; нормализуй или используй `shallowRef`.
- **Mutation vs immutability.** Прямая мутация props/state ломает предсказуемость во всех фреймворках.
- **Over-render.** React без memoization перерисовывает всё subtree; Vue без `key` на list даёт wrong DOM reuse.
- **Debugging.** Reactive graph скрыт; DevTools hooks обязательны для расследования лишних updates.
