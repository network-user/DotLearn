# React vs Vue и Angular: ключевые различия

## Суть

React это UI library с JSX и однонаправленным data flow; экосистема routing/state снаружи. Vue progressive framework с SFC и реактивностью из коробки. Angular full platform на TypeScript с DI, RxJS, opinionated structure. Выбор зависит от team skills, enterprise constraints, migration path.

## Что ответить на собесе

- React: гибкость, hooks, virtual DOM, largest hiring pool; больше решений самому.
- Vue: lower barrier, SFC ergonomics, official router/pinia; силён в incremental adoption.
- Angular: batteries included, strict architecture, RxJS everywhere; enterprise long-lived apps.
- Честно: все три рендерят component trees; perf differences чаще от app design not framework gospel.
- Microfrontends possible на каждом, но module federation чаще в React/Webpack ecosystem.

## Как это работает

Counter в React (hooks):

```tsx
import { useState } from 'react';

export function Counter() {
  const [total, setTotal] = useState(0);
  return (
    <div>
      <p>Count: {total}</p>
      <button type="button" onClick={() => setTotal((n) => n + 1)}>
        +1
      </button>
    </div>
  );
}
```

Vue 3 script setup:

```vue
<script setup lang="ts">
import { ref } from 'vue';
const total = ref(0);
</script>

<template>
  <p>Count: {{ total }}</p>
  <button type="button" @click="total++">+1</button>
</template>
```

Angular standalone component:

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  standalone: true,
  template: `<p>Count: {{ total }}</p><button (click)="total = total + 1">+1</button>`,
})
export class CounterComponent {
  total = 0;
}
```

React updates via setState/hooks batching. Vue tracks reactive deps automatically. Angular change detection zones (or signals in v16+) scan component tree.

## Подводные камни

- **Resume-driven choice.** Framework mismatch with team hurts velocity more than benchmark slides.
- **React without router discipline.** SPA grows into unstructured folders.
- **Vue Options vs Composition.** Mixed styles in one codebase confuse juniors.
- **Angular bundle size.** Initial payload heavy; lazy routes mandatory.
- **SSR story.** Next/Nuxt/Analog differ; do not assume identical meta-framework features.
