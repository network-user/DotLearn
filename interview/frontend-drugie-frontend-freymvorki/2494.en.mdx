# Reactivity in UI frameworks

## In short

Reactivity is automatic synchronization between state and the DOM when data changes. Frameworks implement it differently: push-based subscriptions (Vue, Angular signals), pull plus virtual tree diff (React), or compile-time tracking (Svelte). The goal is the same: remove manual `document.querySelector` and imperative updates.

## How to answer in an interview

- React: `setState` / hooks trigger re-render; the reconciler diffs the virtual tree and patches the DOM.
- Vue 3: Proxy on reactive objects; an effect runner recomputes the render function.
- Angular: zone.js (legacy) or signals plus explicit `markDirty` in newer versions.
- Honestly: fine-grained reactivity (Vue/Svelte) can beat full component re-render in some cases, but benchmarks depend on UI patterns.
- Reactivity does not replace a sound state model: keep derived data in `computed` and side effects in watchers.

## How it works

React hooks model:

```ts
import { useState } from 'react';

export function TallyButton() {
  const [total, setTotal] = useState(0);

  const bump = () => setTotal((prev) => prev + 1);

  return (
    <div>
      <p>Current total: {total}</p>
      <button type="button" onClick={bump}>
        Increment
      </button>
    </div>
  );
}
```

Vue 3 reactivity:

```ts
import { ref } from 'vue';

export function useTally(initial = 0) {
  const total = ref(initial);

  function bump(step = 1) {
    total.value += step;
  }

  return { total, bump };
}
```

```vue
<script setup lang="ts">
import { useTally } from './useTally';
const { total, bump } = useTally(0);
</script>

<template>
  <p>Current total: {{ total }}</p>
  <button type="button" @click="bump()">Increment</button>
</template>
```

Dependency tracking: Vue collects deps on the first effect run. React compares the new element tree with the old one. Batch updates (React 18 automatic batching, Vue scheduler queue) group several mutations into one paint.

## Pitfalls

- **Stale closures.** A React handler captures old state without a functional updater.
- **Deep reactivity cost.** Large nested objects in Vue reactive are expensive; normalize or use `shallowRef`.
- **Mutation vs immutability.** Direct mutation of props/state breaks predictability in every framework.
- **Over-render.** React without memoization redraws the whole subtree; Vue lists without `key` reuse the wrong DOM nodes.
- **Debugging.** The reactive graph is hidden; DevTools hooks are required to trace extra updates.
