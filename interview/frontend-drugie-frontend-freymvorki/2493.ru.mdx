# Нужен ли JSX в шаблонах Vue 3

## Суть

Vue 3 по умолчанию использует SFC с блоком `<template>`: директивы, интерполяция и compiler оптимизации заточены под HTML-like синтаксис. JSX не обязателен: это опциональный render path через `@vitejs/plugin-vue-jsx` или `render()` функцию для случаев, где template DSL ограничивает динамику.

## Что ответить на собесе

- 90% UI покрываются `<template>` + `v-if` / `v-for` / slots без JSX.
- JSX уместен для highly dynamic render functions (таблицы с programmatic columns, HOC wrappers).
- Команда с React background может выбрать JSX для единого стиля, но теряет template compiler hints.
- Честно: смешивать template и JSX в одном проекте неудобно для code review и tooling.
- TypeScript inference в `<script setup>` сегодня сильнее, чем мотивация переходить на JSX ради типов.

## Как это работает

Template-first (рекомендуемый путь):

```vue
<script setup lang="ts">
import { ref } from 'vue';

const headline = ref('Hello, Vue');
const caption = ref('Template syntax is enough for most UIs.');
</script>

<template>
  <article>
    <h1>{{ headline }}</h1>
    <p>{{ caption }}</p>
  </article>
</template>
```

JSX path (нужен плагин Babel/Vite):

```ts
import { defineComponent, ref } from 'vue';

export default defineComponent({
  setup() {
    const headline = ref('Hello, Vue with JSX');
    const caption = ref('Use when template DSL feels awkward.');

    return () => (
      <article>
        <h1>{headline.value}</h1>
        <p>{caption.value}</p>
      </article>
    );
  },
});
```

Compiler template поднимает static hoisting и patch flags. JSX компилируется в `h()` calls, гибче для JavaScript control flow, но без некоторых compile-time оптимизаций. Выбор зависит от команды и паттерна UI, не от «современности» стека.

## Подводные камни

- **Extra build config.** Забытый `vue-jsx` plugin даёт cryptic compile errors.
- **Event modifiers.** `v-on:click.prevent` в template; в JSX пишут `onClick` handlers вручную.
- **Scoped styles.** SFC `<style scoped>` не применяется к чистому `.tsx` без дополнительной настройки.
- **Design system drift.** JSX компоненты выбиваются из Vue ecosystem examples (документация template-first).
- **Overuse.** Простой список карточек в JSX добавляет шум без выигрыша в гибкости.
