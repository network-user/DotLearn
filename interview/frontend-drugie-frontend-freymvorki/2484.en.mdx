# Vue Single File Components: .vue file structure

## In short

An SFC (.vue) colocates template, script, and style for one component. `<template>` describes DOM, `<script>` holds logic (Options or Composition API), `<style scoped>` adds locally scoped styles via attribute hash. Vite/vue-loader compile blocks into JS render functions and CSS.

## How to answer in an interview

- Three blocks: template, script, style; optional `<script setup>` and `<style scoped>`.
- Scoped adds data-v-hash on nodes; `:deep()` pierces child component boundaries.
- Supports lang="ts", lang="scss", custom blocks for i18n/GraphQL with plugins.
- Honestly: large 400+ line SFCs should split into composables and child components.
- SFCs are tree-shaking friendly: unused components drop from bundles.

## How it works

```vue
<script setup lang="ts">
import { ref } from 'vue';

const headline = ref('Hello SFC');

function rotateTitle() {
  headline.value = headline.value === 'Hello SFC' ? 'Updated' : 'Hello SFC';
}
</script>

<template>
  <section class="hero">
    <h1>{{ headline }}</h1>
    <button type="button" @click="rotateTitle">Toggle</button>
  </section>
</template>

<style scoped>
.hero {
  display: grid;
  gap: 0.75rem;
}

button {
  inline-size: fit-content;
}
</style>
```

The compiler turns the template into a render function reading reactive refs. Scoped CSS rewrites `.hero` → `.hero[data-v-7a7a]`. Global styles live in separate entries or unscoped `<style>` blocks.

SFCs improve colocation vs React JSX + CSS modules split files; tradeoff: tooling must understand `.vue` (Vite does).

## Pitfalls

- **Scoped leaking.** `:deep(.child)` is required for third-party widget internals.
- **Two script blocks.** Mixing `<script>` and `<script setup>` needs macro rules.
- **SSR hydration mismatch.** Dates/locale in templates without server sync.
- **CSS order.** Import order across SFCs affects cascade without modules.
- **Testing.** Mount SFCs via @vue/test-utils; stub child SFC boundaries.
