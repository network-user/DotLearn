# Change Detection в Angular

## Суть

Change Detection синхронизирует component state с DOM. Default strategy проверяет все components при async tick (Zone.js patches promises/events). OnPush проверяет component только при input reference change, event in component, или explicit markForCheck.

## Что ответить на собесе

- Zone.js triggers CD after macrotasks/microtasks; `NgZone.runOutsideAngular` for high-frequency handlers.
- OnPush + immutable inputs снижают work в large tables.
- Signals (v16+) fine-grained updates альтернатива manual OnPush discipline.
- Честно: async pipe auto marks dirty; manual subscribe needs `ChangeDetectorRef.markForCheck`.
- Detach CD for off-screen portals rare advanced optimization.

## Как это работает

```ts
import { ChangeDetectionStrategy, Component, Input } from '@angular/core';

@Component({
  selector: 'app-metric',
  standalone: true,
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `<span>{{ value }}</span>`,
})
export class MetricComponent {
  @Input({ required: true }) value = 0;
}
```

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  standalone: true,
  template: `
    <p>{{ total }}</p>
    <button type="button" (click)="bump()">+1</button>
  `,
})
export class CounterComponent {
  total = 0;

  bump() {
    this.total += 1;
  }
}
```

Click runs in zone → CD tree walk → updates text. OnPush child `MetricComponent` skipped until `@Input()` reference changes.

## Подводные камни

- **Mutate input object.** OnPush won't update if parent mutates fields in place.
- **setTimeout without zone.** UI stale until next external event unless manual detect.
- **Third-party widgets.** Non-zone APIs need `NgZone.run` wrapper.
- **ExpressionChangedAfterItHasBeenCheckedError.** Sync parent/child update order in dev mode double check.
- **Over-OnPush.** Premature optimization breaks when devs forget immutability.
