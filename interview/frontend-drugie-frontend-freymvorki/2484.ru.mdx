# Single File Component в Vue: структура .vue файла

## Суть

SFC (.vue) colocate template, script и style одного компонента. `<template>` описывает DOM, `<script>` logic (Options или Composition API), `<style scoped>` локальные стили с attribute selector hash. Vite/vue-loader компилирует блоки в JS render functions и CSS modules.

## Что ответить на собесе

- Три блока: template, script, style; optional `<script setup>` и `<style scoped>`.
- Scoped добавляет data-v-hash на nodes; deep selector `:deep()` для child components.
- Можно lang="ts", lang="scss", custom blocks для i18n/graphQL при plugins.
- Честно: большие SFC 400+ строк split на composables + presentational child.
- SFC tree-shaking friendly: unused components drop from bundle.

## Как это работает

```vue
<script setup lang="ts">
import { ref } from 'vue';

const headline = ref('Hello SFC');

function rotateTitle() {
  headline.value = headline.value === 'Hello SFC' ? 'Updated' : 'Hello SFC';
}
</script>

<template>
  <section class="hero">
    <h1>{{ headline }}</h1>
    <button type="button" @click="rotateTitle">Toggle</button>
  </section>
</template>

<style scoped>
.hero {
  display: grid;
  gap: 0.75rem;
}

button {
  inline-size: fit-content;
}
</style>
```

Compiler transforms template to render function accessing reactive refs. Scoped CSS rewrites `.hero` → `.hero[data-v-7a7a]`. Global styles live in separate entry or unscoped `<style>` block.

SFC improves colocation vs React JSX+CSS modules split files; tradeoff: non-Vue tooling must understand `.vue` (Vite handles).

## Подводные камни

- **Scoped leaking.** `:deep(.child)` needed for third-party widget internals.
- **Two script blocks.** `<script>` + `<script setup>` mix requires knowing macro rules.
- **SSR hydration mismatch.** Date/locale in template without server sync.
- **CSS order.** Import order across SFC affects cascade unpredictably without modules.
- **Testing.** Mount SFC via @vue/test-utils; stub child SFC boundaries.
