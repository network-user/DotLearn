# Slots and scoped slots in Vue

## In short

Slots are a composition API: the parent passes markup into a child without breaking the shell's encapsulation. The default slot accepts arbitrary content; named slots place it in layout regions. A scoped slot (slot props) exposes data from the child so the parent decides how to render it.

## How to answer in an interview

- `<slot />` is a placeholder in the child; fallback content inside the tag shows when the parent passes nothing.
- `v-slot:header` / `#footer` map named regions in cards, modals, and page shells.
- Scoped: child `<slot :row="item" />`, parent `#default="{ row }"` customizes the cell renderer.
- Honestly: for headless logic I prefer composables plus a renderless pattern; slots stay for layout composition.
- Vue 3 unified `v-slot` syntax; Vue 2 `slot-scope` is deprecated.

## How it works

Named slots:

```vue
<!-- PanelShell.vue -->
<template>
  <section class="panel">
    <header><slot name="toolbar" /></header>
    <main><slot /></main>
    <footer><slot name="actions" /></footer>
  </section>
</template>
```

```vue
<!-- ParentView.vue -->
<template>
  <PanelShell>
    <template #toolbar>
      <h2>Filters</h2>
    </template>
    <p>Main body content</p>
    <template #actions>
      <button type="button">Save</button>
    </template>
  </PanelShell>
</template>
```

Scoped slot for a table:

```vue
<!-- DataGrid.vue -->
<script setup lang="ts">
const rows = [{ id: 1, label: 'Alpha' }];
</script>

<template>
  <ul>
    <li v-for="row in rows" :key="row.id">
      <slot :row="row" />
    </li>
  </ul>
</template>
```

```vue
<template>
  <DataGrid v-slot="{ row }">
    <strong>{{ row.label }}</strong>
  </DataGrid>
</template>
```

The child owns data and list structure; the parent owns each row's presentation. The pattern is close to React render props but stays declarative in templates.

## Pitfalls

- **Implicit default slot.** Multiple root nodes without `#default` may land in the wrong place in complex wrappers.
- **Slot prop typing.** TS needs `defineSlots` or a generic wrapper for autocomplete.
- **Performance.** Thousands of scoped slots in a virtualized list without `memo` cause extra vnode allocations.
- **Fallback misuse.** Heavy slot fallback hides a missing-content bug until production.
- **Teleport + slots.** A teleported modal can lose scoped slot context if structure is not planned.
