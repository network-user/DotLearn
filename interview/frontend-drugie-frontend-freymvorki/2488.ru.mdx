# Шаблоны Angular: binding и директивы

## Суть

Template это HTML расширенный Angular syntax: interpolation, property/event binding, two-way `[(ngModel)]`, structural directives (`*ngIf`, `*ngFor`), attribute directives (`ngClass`). Compiler генерирует instructions для change detection updates.

## Что ответить на собесе

- `{{ expr }}` display; `[prop]` down; `(event)` up; `[(ngModel)]` banana in box для forms.
- `*ngFor trackBy` обязателен для stable lists performance.
- Structural directives создают `<ng-template>` under hood.
- Честно: prefer `@if` / `@for` control flow (v17+) over legacy `*ngIf`.
- Strict templates (`strictTemplates`) ловят null errors compile-time.

## Как это работает

```ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-todo-panel',
  standalone: true,
  template: `
    <h1>{{ heading }}</h1>
    <button type="button" (click)="toggle()">Toggle list</button>
    @if (open) {
      <ul>
        @for (task of tasks; track task.id) {
          <li [class.done]="task.done">{{ task.label }}</li>
        }
      </ul>
    }
  `,
})
export class TodoPanelComponent {
  heading = 'Tasks';
  open = true;
  tasks = [
    { id: 1, label: 'Ship feature', done: false },
    { id: 2, label: 'Write tests', done: true },
  ];

  toggle() {
    this.open = !this.open;
  }
}
```

Binding grammar keeps view declarative; logic stays in class. Pipes (`| date`) format display without extra methods. Custom directives encapsulate DOM behavior (tooltip, autofocus).

## Подводные камни

- **Two-way on custom components.** Need `@Input` + `@Output` with same name + `[(item)]` convention.
- **ngModel without FormsModule.** Standalone requires `import { FormsModule }`.
- **Heavy logic in template.** Complex expressions rerun each CD cycle.
- **Security.** BypassSecurityTrustHtml only with sanitized source.
- **i18n.** Mark strings `@`/`$localize` early or retrofit costly.
