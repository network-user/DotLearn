# Angular pros and cons for enterprise

## In short

Angular is an opinionated platform from Google: modules, DI, router, forms, HTTP client, and CLI ship together. TypeScript is mandatory and conventions stay consistent across large teams. The tradeoff is a steep learning curve, heavier bundles, and major-version migrations.

## How to answer in an interview

- Pros: structure out of the box, RxJS streams, built-in testing tools, long LTS cadence.
- Cons: boilerplate, change detection nuances without OnPush, larger initial payload.
- Banks and regulated domains often pick Angular for predictable architecture.
- Honestly: for a small MVP React/Vue start faster; Angular pays off with 50+ developers and a 5+ year product horizon.
- Ivy and standalone components improved tree shaking, but the DI mental model stays complex.

## How it works

```ts
import { Component, inject } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Component({
  selector: 'app-greeting',
  standalone: true,
  template: `<h1>{{ headline }}</h1>`,
  styles: [`h1 { font-family: system-ui, sans-serif; }`],
})
export class GreetingComponent {
  private readonly http = inject(HttpClient);
  headline = 'Hello, Angular';

  ngOnInit() {
    this.http.get<{ title: string }>('/api/title').subscribe((res) => {
      this.headline = res.title;
    });
  }
}
```

Modularity: feature modules group components, pipes, and directives. The DI container resolves services by token (`providedIn: 'root'` singleton). Two-way binding `[(ngModel)]` links template and class, but large forms usually prefer reactive forms.

The CLI (`ng generate`, `ng build`, `ng test`) standardizes scaffolding. Jasmine + Karma (or Jest) cover unit tests; Protractor/Cypress handle e2e. Major upgrades run through `ng update` with schematic migrations.

## Pitfalls

- **Default change detection.** Without OnPush, extra digest cycles hurt performance in large tables.
- **RxJS overuse.** Streams on every click complicate debugging; sometimes async pipe + signals are enough.
- **Bundle weight.** The full platform is heavier than library-first stacks; lazy routes are mandatory.
- **Breaking migrations.** The Angular 2→15 history scares stakeholders; plan upgrade cadence early.
- **Hiring pool.** Fewer candidates than React and a higher bar for juniors.
