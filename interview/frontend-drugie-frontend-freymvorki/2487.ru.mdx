# Dependency Injection в Angular

## Суть

Angular DI container resolves dependencies by token (usually class) and injects instances into constructors или `inject()`. Providers register factories/singletons at root, module, component, or route level. Это decouple components from concrete HTTP/storage implementations и упрощает unit tests с mocks.

## Что ответить на собесе

- `@Injectable({ providedIn: 'root' })` singleton tree-shakeable service.
- Constructor injection: `constructor(private api: ApiService)`.
- `inject(ApiService)` в functional guards/interceptors.
- Честно: hierarchical injectors mean component-level provider shadows root.
- Testing: `TestBed.configureTestingModule({ providers: [{ provide: ApiService, useValue: mock }] })`.

## Как это работает

```ts
import { Injectable, inject, Component } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Injectable({ providedIn: 'root' })
export class ProfileApi {
  private http = inject(HttpClient);

  loadProfile() {
    return this.http.get<{ name: string }>('/api/me');
  }
}

@Component({
  selector: 'app-profile',
  standalone: true,
  template: `<p>{{ name }}</p>`,
})
export class ProfileComponent {
  private profileApi = inject(ProfileApi);
  name = '';

  constructor() {
    this.profileApi.loadProfile().subscribe((p) => {
      this.name = p.name;
    });
  }
}
```

Injector hierarchy: root → route → component. Child can override provider for scoped state (rare). Multi providers for tokens like `HTTP_INTERCEPTORS`.

## Подводные камни

- **Circular deps.** Service A injects B injects A; fix with `forwardRef` or split.
- **providedIn: 'any'.** Multiple instances per lazy module surprise.
- **Manual `new Service()`.** Bypasses DI and interceptors.
- **Environment providers.** Forgetting `provideHttpClient()` in bootstrap breaks inject.
- **SSR tokens.** PLATFORM_ID checks for browser-only APIs.
