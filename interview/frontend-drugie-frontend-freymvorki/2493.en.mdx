# Is JSX required for Vue 3 templates

## In short

Vue 3 defaults to SFCs with a `<template>` block: directives, interpolation, and compiler optimizations target HTML-like syntax. JSX is optional: an alternate render path via `@vitejs/plugin-vue-jsx` or a `render()` function when the template DSL limits dynamic UI.

## How to answer in an interview

- About 90% of UI is covered by `<template>` + `v-if` / `v-for` / slots without JSX.
- JSX fits highly dynamic render functions (tables with programmatic columns, HOC wrappers).
- A React-background team may pick JSX for one style, but loses some template compiler hints.
- Honestly: mixing template and JSX in one project hurts code review and tooling.
- TypeScript inference in `<script setup>` is strong today, so JSX is not needed just for types.

## How it works

Template-first (recommended):

```vue
<script setup lang="ts">
import { ref } from 'vue';

const headline = ref('Hello, Vue');
const caption = ref('Template syntax is enough for most UIs.');
</script>

<template>
  <article>
    <h1>{{ headline }}</h1>
    <p>{{ caption }}</p>
  </article>
</template>
```

JSX path (requires a Babel/Vite plugin):

```ts
import { defineComponent, ref } from 'vue';

export default defineComponent({
  setup() {
    const headline = ref('Hello, Vue with JSX');
    const caption = ref('Use when template DSL feels awkward.');

    return () => (
      <article>
        <h1>{headline.value}</h1>
        <p>{caption.value}</p>
      </article>
    );
  },
});
```

The template compiler enables static hoisting and patch flags. JSX compiles to `h()` calls, which is more flexible for JavaScript control flow but skips some compile-time optimizations. The choice depends on team and UI patterns, not on stack fashion.

## Pitfalls

- **Extra build config.** A missing `vue-jsx` plugin causes cryptic compile errors.
- **Event modifiers.** `v-on:click.prevent` in templates; JSX uses manual `onClick` handlers.
- **Scoped styles.** SFC `<style scoped>` does not apply to plain `.tsx` without extra setup.
- **Design system drift.** JSX components diverge from template-first Vue docs and examples.
- **Overuse.** A simple card list in JSX adds noise without flexibility gains.
