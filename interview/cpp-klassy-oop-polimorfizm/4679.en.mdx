# Why was override added in C++11?

## In short

`override` makes the compiler verify that a method actually overrides a virtual base function. A typo in the name or a const/ref mismatch becomes an error instead of a silent new overload.

## How to answer in an interview

- Without override, a typo like `vritual void draw()` adds a new method and polymorphism breaks quietly.
- `void draw() override` requires an exact match (with covariant return rules).
- Pairs with `final` to block further overrides.
- Applies only to virtual (or virtually inherited) candidates.
- Honestly: `using Base::foo;` plus override helps with overload sets in the base.

## How it works

```cpp
#include <iostream>

struct Canvas {
    virtual void render() const {
        std::cout << "Canvas\n";
    }
};

struct Sprite : Canvas {
    void render() const override {
        std::cout << "Sprite\n";
    }
};

void paint(const Canvas& item) {
    item.render();
}

int main() {
    Sprite hero;
    paint(hero);
}
```

What override catches:

```cpp
struct Buggy : Canvas {
    void render() override; // error: no matching virtual in Canvas (missing const)
};
```

Combination:

```cpp
struct SealedDraw : Canvas {
    void render() const override final {}
};
```

## Pitfalls

- **Parameter mismatch.** `draw(int)` vs `draw()` is not an override; the keyword exposes it.
- **Covariant return.** Narrow rules; pointer/reference covariance allowed with limits.
- **Multiple bases.** Ambiguous override without `using` and qualified names.
- **Hiding vs overriding.** Without a virtual base method, override fails compile (good).
- **Templates.** Override on template methods has complex matching; verify with compile tests.
