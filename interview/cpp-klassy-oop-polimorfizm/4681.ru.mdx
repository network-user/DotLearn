# Когда вызывается оператор копирующего присваивания?

## Суть

Copy assignment (`T& operator=(const T&)`) вызывается, когда уже существующему объекту присваивают другой объект того же типа. Это не construction, а перезапись состояния с release старых ресурсов.

## Что ответить на собесе

- `a = b;` после оба объекта constructed.
- Отличаю от copy ctor: ctor создаёт новый объект, assignment меняет живой.
- Strong exception safety: copy-and-swap idiom через by-value parameter.
- Проверка self-assignment `if (this != &other)` или swap handles it.
- Честно: на собесах «конструктор присваивания» обычно mean copy assignment operator, не ctor.

## Как это работает

```cpp
#include <iostream>

struct Pod {
    int value = 0;
    Pod& operator=(const Pod& other) {
        value = other.value;
        return *this;
    }
};

int main() {
    Pod x;
    Pod y;
    y = x; // copy assignment
}
```

RAII с copy-and-swap:

```cpp
#include <algorithm>
#include <utility>

struct Blob {
    int* data = nullptr;
    std::size_t len = 0;

    Blob& operator=(Blob other) {
        swap(*this, other);
        return *this;
    }
    friend void swap(Blob& a, Blob& b) {
        std::swap(a.data, b.data);
        std::swap(a.len, b.len);
    }
    ~Blob() { delete[] data; }
};
```

Move assignment `operator=(T&&)` вызывается для rvalue: `a = std::move(b);`.

## Подводные камни

- **Uninitialized assign.** `T x; T y; y = x;` OK; `T* p; *p = x;` UB.
- **Base/derived assign.** `Base& b = d1; b = d2;` slicing без virtual assign.
- **Container reassign.** `vec[i] = vec[j]` может copy assign elements.
- **Deleted copy assign.** `unique_ptr` forbids; use move assign.
- **Exception mid-assign.** Old state must stay valid; copy-and-swap helps.
