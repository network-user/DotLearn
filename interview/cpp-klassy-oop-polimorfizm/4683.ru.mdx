# Может ли конструктор быть шаблоном?

## Суть

Member constructor может быть function template (`template<typename U> Widget(U&& x)`). Нельзя объявить «шаблонную функцию-конструктор» вне класса как free template ctor. Часто шаблонным делают весь класс `template<typename T> Box`.

## Что ответить на собесе

- Class template: каждый `Box<T>` получает ctor для конкретного T.
- Templated ctor в non-template class: forwarding/universal reference, SFINAE, constraints.
- Constructor template не может быть virtual.
- Deduction guides (C++17) влияют на CTAD для class templates.
- Честно: templated ctor может захватить copy ctor role и ломать implicit copy; аккуратно с `T&&`.

## Как это работает

Class template:

```cpp
template<typename T>
struct Cell {
    explicit Cell(T value) : payload_(std::move(value)) {}
    T payload_;
};

Cell<int> a{7};
Cell<std::string> b{"hi"};
```

Templated constructor in ordinary class:

```cpp
#include <utility>

struct Wrapper {
    template<typename U>
    explicit Wrapper(U&& item) : stored_(std::forward<U>(item)) {}

    int stored_;
};
```

Ограничение copy:

```cpp
template<typename U>
Wrapper(U&&) -> std::enable_if_t<!std::is_same_v<std::decay_t<U>, Wrapper>>;
```

## Подводные камни

- **Copy ctor hijack.** `Wrapper(const Wrapper&)` may not trigger if templated ctor matches.
- **explicit on forwarding ctor.** Prevents silent conversions from int to Wrapper.
- **Virtual ctor impossible.** Runtime polymorphic construction uses factory/virtual clone.
- **Template ctor in derived.** Does not inherit as override; separate overload set.
- **CTAD pitfalls.** Guides must match intent or deduction surprises at call site.
