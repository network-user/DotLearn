# Чем struct отличается от class в C++?

## Суть

В C++ `struct` и `class` почти идентичны: различие по умолчанию в access (struct public, class private) и при default inheritance (struct public inherit, class private). Всё остальное одинаково: methods, ctors, inheritance, templates.

## Что ответить на собесе

- Default member access: struct public, class private.
- Default base class access: struct public, class private.
- Семантика: struct для aggregates/POD-like data carriers, class для инвариантов.
- В C struct нет methods; C++ struct полноценный class keyword.
- Честно: style guide часто mandates struct only for passive data; compiler не enforce.

## Как это работает

```cpp
struct Point {
    int x;
    int y;
};

class Account {
    int balance = 0;
public:
    void deposit(int amount) { balance += amount; }
};

struct Secret : Account {
    // default public inheritance from Account if Account were struct-like base
};
```

Default inheritance:

```cpp
struct DerivedStruct : Base {}; // public inheritance
class DerivedClass : Base {};   // private inheritance
```

Оба могут иметь virtual functions, static members, operators.

## Подводные камни

- **POD myth.** Modern C++ POD/trivial differs; struct keyword alone не делает type POD.
- **ABI.** Layout одинаков при одинаковых members/access; keyword не меняет layout.
- **std library convention.** `pair`, iterator traits use struct for public defaults.
- **Encapsulation leak.** Public struct fields bypass invariants.
- **Mixing style.** Team convention важнее literal keyword choice.
