# How does struct differ from class in C++?

## In short

In C++ `struct` and `class` are nearly identical: defaults differ for access (struct public, class private) and default inheritance (struct public, class private). Everything else matches: methods, ctors, inheritance, templates.

## How to answer in an interview

- Default member access: struct public, class private.
- Default base class access: struct public, class private.
- Style: struct for aggregates/data carriers, class for invariants.
- C structs have no methods; C++ struct is a full class keyword.
- Honestly: many style guides reserve struct for passive data; the compiler does not enforce it.

## How it works

```cpp
struct Point {
    int x;
    int y;
};

class Account {
    int balance = 0;
public:
    void deposit(int amount) { balance += amount; }
};

struct Secret : Account {
    // would be public inheritance if Base were used with struct defaults
};
```

Default inheritance:

```cpp
struct DerivedStruct : Base {}; // public inheritance
class DerivedClass : Base {};   // private inheritance
```

Both may have virtual functions, static members, and operators.

## Pitfalls

- **POD myth.** Modern trivial/standard-layout rules differ; struct alone does not make a POD.
- **ABI.** Layout matches with the same members/access; keyword does not change layout.
- **Standard library convention.** `pair` and iterator traits use struct for public defaults.
- **Encapsulation leak.** Public struct fields bypass invariants.
- **Mixed style.** Team convention beats literal keyword choice.
