# When is the copy assignment operator called?

## In short

Copy assignment (`T& operator=(const T&)`) runs when an already existing object is assigned from another object of the same type. This is not construction; it overwrites state and releases old resources.

## How to answer in an interview

- `a = b;` after both objects are constructed.
- Distinguish from copy ctor: ctor creates a new object, assignment mutates a live one.
- Strong exception safety: copy-and-swap idiom with a by-value parameter.
- Self-assignment check `if (this != &other)` or swap handles it.
- Honestly: in interviews «copy assignment constructor» usually means copy assignment operator, not a constructor.

## How it works

```cpp
#include <iostream>

struct Pod {
    int value = 0;
    Pod& operator=(const Pod& other) {
        value = other.value;
        return *this;
    }
};

int main() {
    Pod x;
    Pod y;
    y = x; // copy assignment
}
```

RAII with copy-and-swap:

```cpp
#include <algorithm>
#include <utility>

struct Blob {
    int* data = nullptr;
    std::size_t len = 0;

    Blob& operator=(Blob other) {
        swap(*this, other);
        return *this;
    }
    friend void swap(Blob& a, Blob& b) {
        std::swap(a.data, b.data);
        std::swap(a.len, b.len);
    }
    ~Blob() { delete[] data; }
};
```

Move assignment `operator=(T&&)` runs for rvalues: `a = std::move(b);`.

## Pitfalls

- **Uninitialized assign.** `T x; T y; y = x;` is OK; `T* p; *p = x;` is UB.
- **Base/derived assign.** `Base& b = d1; b = d2;` slices without virtual assign.
- **Container reassign.** `vec[i] = vec[j]` may copy-assign elements.
- **Deleted copy assign.** `unique_ptr` forbids it; use move assign.
- **Exception mid-assign.** Old state must remain valid; copy-and-swap helps.
