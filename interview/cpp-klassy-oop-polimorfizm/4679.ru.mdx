# Зачем в C++11 появился override?

## Суть

`override` заставляет компилятор проверить, что метод действительно переопределяет virtual base function. Опечатка в имени или const/ref mismatch превращается в ошибку, а не в скрытый новый overload.

## Что ответить на собесе

- Без override опечатка `vritual void draw()` создаёт новый метод, polymorphism молча ломается.
- `void draw() override` требует exact match (с учётом covariant return).
- Пара с `final` для запрета дальнейших override.
- Работает только на virtual (или virtual inherited) candidates.
- Честно: `using Base::foo;` plus override помогает при overload sets в base.

## Как это работает

```cpp
#include <iostream>

struct Canvas {
    virtual void render() const {
        std::cout << "Canvas\n";
    }
};

struct Sprite : Canvas {
    void render() const override {
        std::cout << "Sprite\n";
    }
};

void paint(const Canvas& item) {
    item.render();
}

int main() {
    Sprite hero;
    paint(hero);
}
```

Ошибка, которую ловит override:

```cpp
struct Buggy : Canvas {
    void render() override; // error: no matching virtual in Canvas (missing const)
};
```

Комбинация:

```cpp
struct SealedDraw : Canvas {
    void render() const override final {}
};
```

## Подводные камни

- **Parameter mismatch.** `draw(int)` vs `draw()` не override, override exposes it.
- **Return type covariance.** Narrow rules; pointer/reference covariance allowed with limits.
- **Multiple bases.** Ambiguous override without `using` and qualified names.
- **Hiding vs overriding.** Without virtual base method, override keyword fails compile (good).
- **Templates.** override on template methods follows complex matching rules; test compile.
