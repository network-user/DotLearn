# Как запретить наследование от класса в C++?

## Суть

Класс помечают `final`, чтобы от него нельзя было наследоваться. Отдельный метод `final` на virtual function запрещает дальнейший override в derived.

## Что ответить на собесе

- `class Widget final { ... };` блокирует `class Child : Widget`.
- `void run() final;` в derived закрывает цепочку override.
- До C++11 использовали private ctor + friend или документацию; сейчас `final` каноничен.
- Полезно для value types, sealed API, security-sensitive classes.
- Честно: `final` не защищает от composition или wrapping; это compile-time запрет наследования.

## Как это работает

```cpp
class Token final {
public:
    explicit Token(int id) : id_(id) {}
    int id() const { return id_; }

private:
    int id_;
};

// struct ExtendedToken : Token {}; // ошибка компиляции
```

Запрет override:

```cpp
struct Engine {
    virtual void spin() = 0;
};

struct Turbo : Engine {
    void spin() override final {}
};

// struct Ultra : Turbo { void spin() override {} }; // ошибка
```

Компилятор знает, что `Token` не имеет heirs, и может devirtualize вызовы в том же translation unit при LTO.

## Подводные камни

- **Mocking in tests.** `final` ломает gmock наследование; нужны interfaces без final.
- **Library evolution.** Sealed class нельзя расширить без breaking API v2.
- **Путаница final class vs final method.** Разные цели: sealed type vs sealed vfunc.
- **Private inheritance workaround.** Нельзя public inherit, но можно embed и делегировать.
- **ABI.** `final` в header меняет optimization, не layout, но влияет на design downstream.
