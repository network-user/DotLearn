# В каком порядке вызываются деструкторы?

## Суть

Деструкторы идут в обратном порядке к construction: последний созданный local уничтожается первым; members class уничтожаются в reverse order of declaration; inheritance chain от most derived к bases.

## Что ответить на собесе

- Stack locals: LIFO по scope.
- Members: reverse declaration order in class body.
- Derived then base destructors.
- Static/global destructors at program exit в reverse init order (implementation phase).
- Честно: union and trivial members follow same rules; destructor may be trivial = no call.

## Как это работает

```cpp
#include <iostream>
#include <string>

struct Tag {
    explicit Tag(std::string name) : name_(std::move(name)) {
        std::cout << "ctor " << name_ << '\n';
    }
    ~Tag() { std::cout << "dtor " << name_ << '\n'; }
    std::string name_;
};

int main() {
    Tag first("first");
    Tag second("second");
}
```

Output dtor: second, first.

Members:

```cpp
struct Bundle {
    Tag a{"member-a"};
    Tag b{"member-b"};
};
```

dtor Bundle: member-b, member-a, then ~Bundle.

Inheritance:

```cpp
struct Base { ~Base(); };
struct Derived : Base { ~Derived(); };
```

delete Derived: ~Derived, ~Base.

## Подводные камни

- **Virtual bases.** Most derived drives virtual base subobject destruction once.
- **Exception during dtor.** Other dtors still run for already destroyed parts; std::terminate if throws.
- **Manual placement new.** Must call dtor explicitly; order your responsibility.
- **Thread-local static.** Destroyed on thread exit, not necessarily program exit order.
- **std::exit.** Skips destructors for locals in that thread unless atexit handlers.
