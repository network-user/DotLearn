# Когда вызывается copy constructor?

## Суть

Copy constructor создаёт новый объект как копию существующего: `T(const T&)`. Он срабатывает при инициализации из другого `T`, а не при присваивании уже живому объекту.

## Что ответить на собесе

- `T b = a;`, `T b(a);`, pass-by-value, return-by-value (до NRVO), insert в контейнер.
- Не путаю с copy assignment `operator=` для уже constructed object.
- Compiler-generated copy memberwise; для ресурсов нужен Rule of Three/Five.
- C++11 move ctor забирает rvalue; copy ctor для lvalue.
- Честно: NRVO/RVO часто elide copy ctor даже для non-trivial T; не полагайся на side effect в copy ctor для «счётчика вызовов» в optimized build.

## Как это работает

```cpp
#include <iostream>

struct Buffer {
    Buffer() { std::cout << "default\n"; }
    Buffer(const Buffer&) { std::cout << "copy ctor\n"; }
};

Buffer make() {
    Buffer tmp;
    return tmp; // may elide copy
}

void consume(Buffer b) {
    (void)b;
}

int main() {
    Buffer a;
    Buffer b = a;     // copy ctor
    Buffer c(a);      // copy ctor
    consume(a);       // copy ctor if no move
    Buffer d = make(); // maybe elided
}
```

Для RAII:

```cpp
struct Handle {
    Handle(const Handle& other) : fd_(other.fd_) { /* dup or share */ }
    int fd_;
};
```

Deleted copy:

```cpp
struct UniqueOwner {
    UniqueOwner(const UniqueOwner&) = delete;
};
```

## Подводные камни

- **Copy vs assign.** `Buffer b; b = a;` это assignment, не copy ctor.
- **Initializer list.** `vector<Buffer> v(1)` value-initializes elements, may copy/move.
- **Slicing.** `Derived d; Base b = d;` copy ctor Base slices.
- **Polymorphic copy.** Нужен virtual clone(), copy ctor base не копирует derived part через Base&.
- **Self copy ctor.** Must handle `T(const T&)` if aliasing possible (rare in ctor).
