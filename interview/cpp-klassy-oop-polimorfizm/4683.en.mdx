# Can a constructor be a template?

## In short

A member constructor can be a function template (`template<typename U> Widget(U&& x)`). You cannot declare a standalone template constructor outside the class. Often the whole class is a template: `template<typename T> Box`.

## How to answer in an interview

- Class template: each `Box<T>` gets a ctor for a concrete T.
- Templated ctor in a non-template class: forwarding/universal references, SFINAE, constraints.
- A constructor template cannot be virtual.
- Deduction guides (C++17) affect CTAD for class templates.
- Honestly: a templated ctor can steal the copy ctor role and break implicit copy; be careful with `T&&`.

## How it works

Class template:

```cpp
template<typename T>
struct Cell {
    explicit Cell(T value) : payload_(std::move(value)) {}
    T payload_;
};

Cell<int> a{7};
Cell<std::string> b{"hi"};
```

Templated constructor in an ordinary class:

```cpp
#include <utility>

struct Wrapper {
    template<typename U>
    explicit Wrapper(U&& item) : stored_(std::forward<U>(item)) {}

    int stored_;
};
```

Restricting copies:

```cpp
template<typename U>
Wrapper(U&&) -> std::enable_if_t<!std::is_same_v<std::decay_t<U>, Wrapper>>;
```

## Pitfalls

- **Copy ctor hijack.** `Wrapper(const Wrapper&)` may not run if the templated ctor matches.
- **explicit on forwarding ctor.** Stops silent int-to-Wrapper conversions.
- **Virtual ctor impossible.** Runtime polymorphic construction uses factory/virtual clone.
- **Template ctor in derived.** Does not override; separate overload set.
- **CTAD pitfalls.** Guides must match intent or deduction surprises at the call site.
