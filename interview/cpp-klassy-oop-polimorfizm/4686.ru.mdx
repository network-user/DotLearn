# Зачем писать override у virtual-метода?

## Суть

`override` документирует intent и включает compile-time check: метод в derived must match virtual base signature. Опечатки и const mismatch становятся ошибкой компиляции вместо silent overload.

## Что ответить на собесе

- Явно показываю, что это override, не новый overload.
- Компилятор ловит `draw()` vs `draw() const`.
- С `final` закрываю дальнейшие override.
- Не заменяет `virtual` в base; только на derived side.
- Честно: 4679 и 4686 в банке близки; на собесе достаточно одной формулировки про safety net.

## Как это работает

```cpp
struct Device {
    virtual void ping() const;
};

struct Sensor : Device {
    void ping() const override { /* ... */ }
};

void check(const Device& d) {
    d.ping();
}
```

Fail compile:

```cpp
struct Broken : Device {
    void ping() override; // error: base has const, derived omitted const
};
```

With final:

```cpp
struct FixedSensor : Device {
    void ping() const override final {}
};
```

## Подводные камни

- **Missing virtual in base.** override fails compile; fix base or remove override.
- **Overload sets.** `using Base::foo;` needed before override specific overload.
- **Return type drift.** Narrow covariant rules; mismatch fails override.
- **Hiding without override.** Without keyword compiler accepts unrelated method silently.
- **Maintenance.** Base signature change breaks all overrides loudly (feature, not bug).
