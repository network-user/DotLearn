# How do you forbid inheritance from a class in C++?

## In short

Mark a class `final` so it cannot be derived from. A `final` specifier on a virtual function blocks further overrides in derived classes.

## How to answer in an interview

- `class Widget final { ... };` blocks `class Child : Widget`.
- `void run() final;` in derived closes the override chain.
- Before C++11 people used private ctors plus friends or documentation; `final` is canonical now.
- Useful for value types, sealed APIs, security-sensitive classes.
- Honestly: `final` does not stop composition or wrapping; it is a compile-time inheritance ban.

## How it works

```cpp
class Token final {
public:
    explicit Token(int id) : id_(id) {}
    int id() const { return id_; }

private:
    int id_;
};

// struct ExtendedToken : Token {}; // compile error
```

Blocking override:

```cpp
struct Engine {
    virtual void spin() = 0;
};

struct Turbo : Engine {
    void spin() override final {}
};

// struct Ultra : Turbo { void spin() override {} }; // error
```

The compiler knows `Token` has no heirs and may devirtualize calls in the same translation unit with LTO.

## Pitfalls

- **Test mocking.** `final` breaks gmock-style inheritance; use interfaces without `final`.
- **Library evolution.** A sealed class cannot be extended without a breaking API v2.
- **final class vs final method.** Different goals: sealed type vs sealed virtual function.
- **Private inheritance workaround.** No public inherit, but embedding and delegation still work.
- **ABI.** `final` in headers changes optimization, not layout, but affects downstream design.
