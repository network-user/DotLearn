# When is the copy constructor called?

## In short

The copy constructor creates a new object as a copy of an existing one: `T(const T&)`. It runs on initialization from another `T`, not when assigning to an already live object.

## How to answer in an interview

- `T b = a;`, `T b(a);`, pass-by-value, return-by-value (before NRVO), container insert.
- Do not confuse with copy assignment `operator=` on an already constructed object.
- Compiler-generated copy is memberwise; resources need Rule of Three/Five.
- C++11 move ctor takes rvalues; copy ctor handles lvalues.
- Honestly: NRVO/RVO often elides the copy ctor even for non-trivial T; do not rely on copy-ctor side effects as a call counter in optimized builds.

## How it works

```cpp
#include <iostream>

struct Buffer {
    Buffer() { std::cout << "default\n"; }
    Buffer(const Buffer&) { std::cout << "copy ctor\n"; }
};

Buffer make() {
    Buffer tmp;
    return tmp; // may elide copy
}

void consume(Buffer b) {
    (void)b;
}

int main() {
    Buffer a;
    Buffer b = a;     // copy ctor
    Buffer c(a);      // copy ctor
    consume(a);       // copy ctor if no move
    Buffer d = make(); // maybe elided
}
```

For RAII:

```cpp
struct Handle {
    Handle(const Handle& other) : fd_(other.fd_) { /* dup or share */ }
    int fd_;
};
```

Deleted copy:

```cpp
struct UniqueOwner {
    UniqueOwner(const UniqueOwner&) = delete;
};
```

## Pitfalls

- **Copy vs assign.** `Buffer b; b = a;` is assignment, not copy ctor.
- **Initializer list.** `vector<Buffer> v(1)` value-initializes elements; may copy/move.
- **Slicing.** `Derived d; Base b = d;` uses Base copy ctor and slices.
- **Polymorphic copy.** Need virtual clone(); base copy ctor does not copy derived through `Base&`.
- **Self copy.** Must handle aliasing in `T(const T&)` if possible (rare at construction).
