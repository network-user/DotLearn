# Ленивая загрузка против жадной в Django ORM: select_related и prefetch_related

## Суть

QuerySet в Django ленив: SQL уходит в базу только в момент, когда данные реально потребовались. Жадная загрузка заранее подтягивает связанные объекты - через `select_related()` (JOIN для FK и one-to-one) и `prefetch_related()` (отдельные запросы для M2M и обратных FK), что спасает от проблемы N+1.

## Что ответить на собесе

По умолчанию ORM ленивый: `Model.objects.all()` не идёт в базу сразу, а возвращает QuerySet, который выполняется лишь при итерации, срезе или `list()`.

- При обходе связанных объектов рождается проблема N+1: на каждый объект - отдельный запрос за связью; лечится жадной загрузкой.
- `select_related()` делает SQL JOIN и тащит связи "к одному" (ForeignKey, OneToOne) за один запрос.
- `prefetch_related()` берёт связи "ко многим" (ManyToMany, обратные FK) отдельным запросом и сшивает данные на стороне Python.
- Выбор зависит от типа связи.

Ленивый QuerySet откладывает обращение к БД:

```python
queryset = MyModel.objects.all()  # SQL ещё НЕ выполнен

for obj in queryset:              # вот здесь уходит запрос
    print(obj.name)
```

Жадная загрузка убирает лишние запросы при работе со связями:

```python
# JOIN: одна строка SQL на FK / OneToOne
books = Book.objects.select_related('author').all()

# Отдельный запрос + сшивка в Python: для M2M и обратных FK
authors = Author.objects.prefetch_related('book_set').all()
```

Без жадной загрузки цикл `for book in books: print(book.author.name)` выполнил бы один запрос на список книг плюс по запросу на автора каждой книги - та самая N+1. С `select_related('author')` всё уходит одним JOIN.

## Подводные камни

- `select_related` нельзя применять к ManyToMany и обратным FK - там нет единственной строки для JOIN; для них только `prefetch_related`.
- `prefetch_related` делает минимум два запроса, зато не раздувает результат декартовым произведением, как глубокий JOIN. На больших M2M это часто выгоднее.
- Жадная загрузка не для каждого случая: если связанные объекты не используются, она лишь зря тянет данные. Профилируйте по `django-debug-toolbar` или `connection.queries`.
- Цепочка `.filter()` после жадной загрузки может потребовать `Prefetch()` с собственным QuerySet, иначе фильтрация связи не применится так, как ожидаешь.
- QuerySet кэширует результат после первого выполнения, но новый вызов метода (`.filter()`, `.exclude()`) создаёт новый QuerySet и новый запрос.
