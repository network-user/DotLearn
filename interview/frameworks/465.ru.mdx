# Создание записи в Django ORM: create() против конструктора и save()

## Суть

Новую строку в таблице через Django ORM добавляют двумя путями: `Model.objects.create(...)` сразу создаёт объект и пишет его в базу, а `Model(...)` плюс `save()` разносит создание объекта в памяти и его сохранение на два шага.

## Что ответить на собесе

Если объект нужно создать и тут же сохранить, я беру `Model.objects.create(**fields)` - это одна операция и один запрос INSERT. Когда между созданием и записью нужно что-то сделать с объектом (досчитать поле, провалидировать, проставить связи), я создаю экземпляр конструктором `Model(**fields)`, а `save()` вызываю позже. Важный нюанс: `create()` обязательно идёт в базу, а просто `Model(...)` объект в БД не пишет - до `save()` его там нет.

## Как это работает

Каждая модель отображается на таблицу, а экземпляр - на строку. `create()` под капотом и есть конструктор плюс `save()`, объединённые в один вызов менеджера.

Создание и запись одной операцией:

```python
from myapp.models import MyModel

new_object = MyModel.objects.create(field1='value1', field2='value2')
```

Создание в памяти и отложенное сохранение:

```python
from myapp.models import MyModel

new_object = MyModel(field1='value1', field2='value2')
new_object.save()
```

Второй способ удобен, когда до записи объект нужно подправить:

```python
from library.models import Book

book = Book(title='Two Scoops of Django', author='Daniel Roy Greenfeld')
book.author = book.author.strip()
book.save()
```

## Подводные камни

- `Model(...)` без `save()` в базу ничего не пишет - частая ошибка новичков: «создал объект, а в БД пусто».
- Массовую вставку не делают циклом из `create()`: это N запросов. Для пачки берут `Model.objects.bulk_create([...])` - один запрос, но он не вызывает `save()` и сигналы `pre_save`/`post_save`.
- `create()` всегда делает INSERT и упадёт при нарушении уникальности. Если нужно «создать или взять существующий», есть `get_or_create()`; для «создать или обновить» - `update_or_create()`.
- `save()` по умолчанию не запускает `full_clean()`, то есть валидаторы полей и `clean()` не отрабатывают - валидацию нужно звать явно.
- У `create()` нет аргументов `update_fields`/`force_insert` для тонкого контроля - если нужно управлять тем, какие поля писать, используйте `save(update_fields=[...])`.
