# Как сериализатор DRF проверяет входные данные на двух уровнях валидации?

## Суть

Сериализатор проверяет данные в два прохода: сначала пополевая валидация через методы `validate_<поле>`, затем общая через метод `validate(self, data)`, которому доступны все поля сразу. Это позволяет отлавливать как ошибки в отдельном поле, так и противоречия между полями.

## Что ответить на собесе

Валидация запускается вызовом `serializer.is_valid()` и идёт в несколько проходов.

- Сначала DRF прогоняет встроенные проверки типов и ограничений полей (`EmailField`, `max_length` и т.д.).
- Потом — методы `validate_<имя_поля>` для отдельных полей, и в конце — метод `validate(self, data)` для кросс-полевых правил.
- Каждый метод либо возвращает (возможно, изменённое) значение, либо бросает `serializers.ValidationError`; все собранные ошибки оказываются в `serializer.errors`.
- Важно, что `validate(self, data)` видит только те поля, что прошли предыдущие этапы.

Метод `validate_<поле>` отвечает за одно поле и получает уже его очищенное значение:

```python
from rest_framework import serializers

class UserSerializer(serializers.Serializer):
    email = serializers.EmailField()

    def validate_email(self, value):
        if not value.endswith("@example.com"):
            raise serializers.ValidationError("Допустим только домен example.com")
        return value
```

Метод `validate(self, data)` видит весь набор и нужен для правил, завязанных на несколько полей:

```python
class UserSerializer(serializers.Serializer):
    username = serializers.CharField(max_length=100)
    email = serializers.EmailField()

    def validate(self, data):
        if data['username'] == data['email']:
            raise serializers.ValidationError("Логин и email не должны совпадать")
        return data
```

На практике оба уровня сочетаются — например, в регистрации: проверяем уникальность email и длину пароля по отдельности.

```python
from rest_framework import serializers

class RegistrationSerializer(serializers.Serializer):
    username = serializers.CharField(max_length=100)
    email = serializers.EmailField()
    password = serializers.CharField(write_only=True)

    def validate_email(self, value):
        if User.objects.filter(email=value).exists():
            raise serializers.ValidationError("Email уже занят")
        return value

    def validate_password(self, value):
        if len(value) < 8:
            raise serializers.ValidationError("Пароль короче 8 символов")
        return value
```

## Подводные камни

- Метод обязан называться `validate_<имя_поля>` точно как поле, иначе он просто не вызовется — опечатка в имени тихо отключает проверку.
- Каждый `validate_<поле>` должен вернуть значение. Забудете `return` — поле станет `None` в `validated_data`.
- В `validate(self, data)` поле может отсутствовать в `data`, если оно опциональное или не прошло раньше; обращайтесь через `data.get('field')`.
- Валидация поля выполняется до `validate()`; если поле упало на своём уровне, до общей проверки оно не дойдёт.
- Чтобы привязать ошибку к конкретному полю в общем `validate`, кидайте `ValidationError({'field': '...'})`, а не голую строку.
