# Сериализаторы DRF: мост между Python-объектами и JSON в обе стороны

## Суть

Сериализатор в Django REST Framework переводит сложные объекты (модели, QuerySet) в примитивы Python, готовые к рендеру в JSON, и выполняет обратную операцию - разбор входного JSON в Python-данные с валидацией. По сути это двусторонний контракт между внутренним представлением и внешним API.

## Что ответить на собесе

Сериализатор решает две задачи - сериализацию на выход и десериализацию на вход.

- На выход он превращает Django-объект в словарь примитивов, который рендерер DRF отдаёт клиенту как JSON.
- На вход он принимает сырые данные клиента, прогоняет их через `is_valid()`, и при успехе кладёт чистые данные в `validated_data`, откуда их берут `create()` или `update()`.
- Ключевой момент: невалидные данные не доходят до модели - валидация встроена в сам сериализатор.
- Чтение объекта - через `serializer.data`, запись - через `serializer(data=...)` и `is_valid()`.

## Как это работает

Объявляем поля и их типы, дальше DRF берёт преобразование и проверки на себя:

```python
from rest_framework import serializers
from myapp.models import Book

class BookSerializer(serializers.Serializer):
    title = serializers.CharField(max_length=100)
    author = serializers.CharField(max_length=100)
    published_date = serializers.DateField()
```

Сериализация - объект в примитивы (доступ через `.data`):

```python
book_instance = Book.objects.get(id=1)
serializer = BookSerializer(book_instance)
serializer.data
# {'title': 'Django for Beginners', 'author': 'William S. Vincent', 'published_date': '2020-01-01'}
```

Десериализация - входной словарь в проверенные данные:

```python
data = {'title': 'New Book', 'author': 'John Doe', 'published_date': '2023-10-10'}
serializer = BookSerializer(data=data)
if serializer.is_valid():
    clean = serializer.validated_data
    # clean используем для create()/update()
```

## Подводные камни

- `serializer.data` доступен у сериализатора для записи только после успешного `is_valid()` - до этого обращение к `.data` на инстансе с `data=...` бросит ошибку (используйте `validated_data` либо сначала валидируйте).
- `is_valid()` сам не выбрасывает исключение по умолчанию - он возвращает `False`, а ошибки кладёт в `serializer.errors`. Чтобы получить 400 автоматически, передают `is_valid(raise_exception=True)`.
- Базовый `Serializer` не умеет сам сохранять в БД: методы `create()` и `update()` нужно писать руками (в отличие от `ModelSerializer`).
- Для коллекции объектов нужен `many=True`, иначе сериализатор попытается обработать QuerySet как один объект.
- Валидация уровня поля (`validate_<field>`) и объекта (`validate`) - разные хуки; не путайте их область видимости.
