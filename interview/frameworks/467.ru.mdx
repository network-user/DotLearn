# Какую роль играет manage.py в Django-проекте

## Суть

`manage.py` - это исполняемый Python-скрипт, который Django кладёт в корень проекта при его создании. Он работает как единая точка входа в командную строку проекта: через него запускают сервер разработки, применяют миграции, заводят приложения и создают суперпользователя. Главное его отличие от голого `django-admin` в том, что он заранее знает, где лежат настройки вашего проекта.

## Что ответить на собесе

Коротко: `manage.py` - это обёртка над `django-admin`, привязанная к конкретному проекту.

- Он выставляет переменную окружения `DJANGO_SETTINGS_MODULE`, указывающую на ваш модуль настроек, и передаёт управление дальше.
- Поэтому команды вида `python manage.py <команда>` выполняются уже в контексте именно вашего проекта - с правильной базой, приложениями и конфигом.
- Дежурный список команд, который полезно назвать: `runserver`, `migrate`, `makemigrations`, `startapp`, `createsuperuser`, `shell`.

## Как это работает

Когда вы вводите команду, `manage.py` сначала прописывает в окружение путь к модулю настроек, а затем зовёт `execute_from_command_line` из `django.core.management`. Дальше Django ищет подходящий класс команды (свой встроенный или объявленный в приложении) и запускает его. То есть `manage.py` сам по себе почти ничего не делает - он лишь готовит контекст и делегирует работу диспетчеру команд.

```python
import os
import sys


def main():
    os.environ.setdefault("DJANGO_SETTINGS_MODULE", "myproject.settings")
    from django.core.management import execute_from_command_line
    execute_from_command_line(sys.argv)


if __name__ == "__main__":
    main()
```

Типичные вызовы:

```bash
python manage.py runserver
python manage.py makemigrations
python manage.py migrate
python manage.py startapp myapp
python manage.py createsuperuser
```

## Подводные камни

- `manage.py` и `django-admin` - не одно и то же. `django-admin` универсален и не знает про настройки; чтобы он работал с проектом, нужно вручную задать `DJANGO_SETTINGS_MODULE`. `manage.py` делает это за вас.
- Запускать команды нужно из каталога, где лежит `manage.py`, иначе Python не найдёт модуль проекта и настройки.
- Файл не магический: это обычный скрипт, и путь к настройкам в нём прописан строкой. Если переименовать пакет проекта или переместить `settings.py`, строку `DJANGO_SETTINGS_MODULE` придётся поправить.
- Своя команда добавляется не правкой `manage.py`, а файлом в `<app>/management/commands/`. Сам `manage.py` трогать для этого не нужно.
