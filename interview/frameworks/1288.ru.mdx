# select_related или prefetch_related: одним JOIN или вторым запросом

## Суть

Оба метода Django ORM борются с проблемой N+1, но разными механизмами. `select_related` подтягивает связанные объекты одним SQL-запросом через JOIN и подходит для `ForeignKey` и `OneToOne`. `prefetch_related` делает отдельный запрос за связанными строками и сшивает их в Python, поэтому покрывает `ManyToMany` и обратные связи «один-ко-многим».

## Что ответить на собесе

«Разница в том, кто соединяет данные. `select_related` перекладывает работу на базу: один JOIN, один запрос, годится для связей, где с одной стороны ровно один объект - то есть `ForeignKey` и `OneToOneField`. `prefetch_related` делает два запроса и соединяет результаты уже на стороне Python по полю-ключу; его берут для `ManyToManyField` и обратных `FK`, где JOIN раздул бы выборку дубликатами. На практике сначала ловлю N+1, потом подбираю метод по типу связи, а для вложенных связей комбинирую оба и использую `Prefetch` для тонкой настройки вложенного запроса».

Возьмём авторов и книги. С `ForeignKey` книга ссылается на одного автора - это случай `select_related`.

```python
class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=100)
    author = models.ForeignKey(Author, on_delete=models.CASCADE)

books = Book.objects.select_related("author").all()
for book in books:
    print(book.title, book.author.name)
```

Здесь ORM строит один SQL с `JOIN` к `Author`, и обращение к `book.author` уже не идёт в базу.

Для связи «многие-ко-многим» один JOIN не подходит - используем `prefetch_related`.

```python
class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=100)
    authors = models.ManyToManyField(Author)

books = Book.objects.prefetch_related("authors").all()
for book in books:
    print(book.title, [a.name for a in book.authors.all()])
```

Django выполнит запрос за книгами, затем отдельный запрос за всеми нужными авторами и соберёт связь в памяти.

## Подводные камни

- `select_related` на связи «многие» через JOIN размножает строки и раздувает выборку, поэтому для `ManyToMany` он не работает - там только `prefetch_related`.
- `prefetch_related` кешируется в момент вызова. Если после префетча применить к связи новый фильтр (`book.authors.filter(...)`), Django сделает дополнительный запрос и весь смысл потеряется - фильтруйте через объект `Prefetch`.
- Оба метода помогают только при реальном обращении к связанным объектам. Если связь в коде не используется, префетч просто добавит лишний запрос.
- Чрезмерный `select_related` по длинной цепочке связей даёт тяжёлый JOIN с множеством колонок; иногда выгоднее `only()`/`defer()` или разбиение на `prefetch_related`.
