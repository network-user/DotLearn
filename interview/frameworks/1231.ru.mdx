# Когда тянуть из ORM словари через values(), а когда кортежи через values_list()?

## Суть

Оба метода заставляют ORM вытащить из таблицы только нужные колонки, а не полные объекты модели. Разница в форме результата: `values()` даёт словари `{поле: значение}`, а `values_list()` даёт кортежи значений.

## Что ответить на собесе

`values()` и `values_list()` превращают `QuerySet` модельных объектов в облегчённый набор данных.

- `values()` беру, когда удобнее обращаться к полям по имени, например, сразу отдать данные в JSON или шаблон.
- `values_list()` беру, когда имена не важны и нужны только сами значения.
- Для одной колонки добавляю `flat=True`: получаю плоский список вместо списка одноэлементных кортежей.
- Главный мотив: не таскать из БД лишние поля и не инстанцировать тяжёлые объекты модели там, где они не нужны.

## Как это работает

`values()` отдаёт `QuerySet`, каждый элемент которого является словарём. Ключи совпадают с именами запрошенных полей:

```python
from myapp.models import Book

books = Book.objects.values('title', 'author')
for book in books:
    print(book)  # {'title': 'Чистый код', 'author': 'Мартин'}
```

`values_list()` отдаёт те же данные кортежами; порядок элементов соответствует порядку аргументов:

```python
books = Book.objects.values_list('title', 'author')
for book in books:
    print(book)  # ('Чистый код', 'Мартин')
```

Когда нужна ровно одна колонка, `flat=True` убирает обёртку-кортеж и возвращает плоский список значений:

```python
titles = Book.objects.values_list('title', flat=True)
list(titles)  # ['Чистый код', 'Рефакторинг', ...]
```

Типичным сценарием является сбор списка уникальных id без выгрузки целых строк:

```python
from myapp.models import Order

customer_ids = Order.objects.values_list('customer_id', flat=True).distinct()
```

## Подводные камни

- `flat=True` работает только с одним полем. Если передать два, получите `TypeError`.
- Результат остаётся ленивым `QuerySet`, а не списком. Запрос к БД уходит при первой итерации; не путайте «облегчённый» с «уже загруженным».
- Словари и кортежи не являются объектами модели: у них нет методов модели, свойств (`@property`) и доступа к связанным объектам через атрибуты. Если в коде дальше дёргается `obj.get_absolute_url()`, `values()` его сломает.
- Для обхода связей пишите путь через двойное подчёркивание: `values('author__name')`. Точечная нотация тут не сработает.
