# Фикстуры в pytest: как готовить окружение для тестов без дублирования?

## Суть

Фикстура — это функция, которая готовит данные или окружение перед тестом и при необходимости прибирает за собой после. Тест получает её результат, просто объявив одноимённый аргумент, а сама подготовка выносится из тела теста и переиспользуется многими тестами.

## Что ответить на собесе

Фикстуры решают проблему повторяющейся подготовки и очистки. Например, тестам нужна чистая база перед каждым прогоном — вместо того чтобы дублировать setup и teardown в каждом тесте, я выношу это в функцию с декоратором `@pytest.fixture`. Тест объявляет имя фикстуры как аргумент, pytest сам её вызывает и подставляет результат. Через `yield` фикстура умеет ещё и освобождать ресурсы после теста. Сверху — управление частотой пересоздания через `scope`, параметризация и автоподключение через `autouse`. Итог: тесты чище, без копипасты, а подготовка данных живёт в одном месте.

## Как это работает

Базовая фикстура объявляется декоратором и возвращает данные. Тест получает их, назвав фикстуру в аргументах:

```python
import pytest

@pytest.fixture
def sample_data():
    return {"key": "value"}

def test_example(sample_data):
    assert sample_data["key"] == "value"
```

Чтобы прибраться после теста, используют `yield`: код до него — подготовка, код после — очистка, которая выполнится даже если тест упал.

```python
@pytest.fixture
def db_connection():
    conn = open_connection()
    yield conn
    conn.close()
```

Дополнительные рычаги:

```python
@pytest.fixture(params=[1, 2, 3])
def number(request):
    return request.param

@pytest.fixture(scope="module")
def setup_module():
    return "module setup"

@pytest.fixture(autouse=True)
def auto_setup():
    pass
```

`params` размножает тест по наборам данных, `scope` (`function`, `class`, `module`, `session`) задаёт, как часто фикстура пересоздаётся, а `autouse=True` подключает её ко всем тестам без явного аргумента.

## Подводные камни

- `return` и `yield` — не одно и то же. Возврат через `return` не даёт фазы очистки; для освобождения ресурсов нужен `yield` с кодом после него.
- `scope` напрямую влияет на изоляцию. `session`-фикстура с изменяемым состоянием может протащить «грязь» между тестами, и они начнут зависеть от порядка запуска.
- `autouse=True` удобен, но коварен: фикстура срабатывает скрытно для всех тестов, и причину неожиданного состояния потом тяжело найти. Применяйте экономно.
- Цепочки фикстур, зависящих друг от друга, легко превращаются в запутанный граф. Держите их плоскими и явными, чтобы тест читался без раскопок.
