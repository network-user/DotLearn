# Чем проверять корутины: организация тестов для async-функций

## Суть

Корутину нельзя просто вызвать и сравнить результат - её нужно прогнать через событийный цикл. Поэтому для async-кода берут инструмент, который умеет запускать `await` внутри теста: плагин `pytest-asyncio` или встроенный `unittest.IsolatedAsyncioTestCase`.

## Что ответить на собесе

Асинхронную функцию тестируют так же, как обычную, но тест-фреймворку нужно дать event loop. На практике я использую `pytest` с плагином `pytest-asyncio`: помечаю тест `@pytest.mark.asyncio` и пишу `await` прямо в теле. Если хочется обойтись стандартной библиотекой, есть `unittest.IsolatedAsyncioTestCase` - он даёт каждому тесту изолированный цикл. Главное - не забыть дождаться корутины: голый вызов `async`-функции возвращает объект корутины, а не результат, и assert на нём ничего не проверит.

## Как это работает

Вызов `async def`-функции не запускает её тело - он создаёт объект корутины, который надо прокрутить в событийном цикле. Тест-раннер для async берёт на себя создание и закрытие этого цикла.

Вариант с `pytest-asyncio`:

```python
import pytest
import asyncio

async def async_function():
    await asyncio.sleep(0)
    return 42

@pytest.mark.asyncio
async def test_async_function():
    result = await async_function()
    assert result == 42
```

Метка `@pytest.mark.asyncio` говорит плагину обернуть тест в цикл. Внутри можно свободно ставить `await`.

Тот же сценарий на стандартном `unittest`:

```python
import unittest
import asyncio

class TestAsyncFunction(unittest.IsolatedAsyncioTestCase):
    async def produce(self):
        await asyncio.sleep(0)
        return 42

    async def test_async_function(self):
        result = await self.produce()
        self.assertEqual(result, 42)

if __name__ == "__main__":
    unittest.main()
```

`IsolatedAsyncioTestCase` создаёт свежий цикл на каждый тест и поддерживает `asyncSetUp` / `asyncTearDown` для асинхронной подготовки и очистки.

## Подводные камни

- Самая частая ошибка - забыть `await`. Тогда тест получает объект корутины, assert проходит впустую, а в логах висит предупреждение `coroutine was never awaited`.
- Без `@pytest.mark.asyncio` (или нужного `asyncio_mode`) pytest вернёт корутину как обычное значение и тест ничего не проверит.
- Не зашивайте реальные таймауты: `asyncio.sleep(1)` лишь тормозит прогон. Для проверки логики хватает `sleep(0)`, а ожидание лучше мокать.
- Исключения внутри корутины всплывают только после `await`, поэтому проверять их надо через `pytest.raises` вокруг самого `await`, а не вокруг создания корутины.
