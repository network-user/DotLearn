# Подмена кода на лету: для чего monkey patching нужен в тестах

## Суть

Monkey patching — это замена поведения уже написанного кода прямо во время выполнения. В тестах так отрезают проверяемый код от внешнего мира: вместо настоящей функции временно подставляют свою.

## Что ответить на собесе

Monkey patching — это динамическая подмена атрибута, функции или метода в рантайме.

- В тестах применяю его, чтобы изолировать модуль от сети, БД или сторонних библиотек: подставляю заглушку с заранее известным ответом.
- Так тесты становятся быстрыми, повторяемыми и не зависят от внешних сервисов.
- На практике обычно делаю это через `unittest.mock.patch`, чтобы подмена автоматически откатывалась после теста.

## Как это работает

В Python имена — это ссылки в пространстве имён модуля, и их можно переопределить во время работы программы. Подмена живёт только в памяти текущего процесса и действует ограниченное время.

Возьмём функцию, которая ходит во внешний API:

```python
import requests

def get_data_from_api(url):
    response = requests.get(url)
    return response.json()
```

Чтобы проверить её без реального запроса, временно подменим `requests.get`:

```python
from unittest.mock import patch

def fake_get(url):
    class FakeResponse:
        def json(self):
            return {"key": "value"}
    return FakeResponse()

with patch("requests.get", fake_get):
    data = get_data_from_api("http://example.com")
    print(data)  # {'key': 'value'}
```

Внутри блока `with` имя `requests.get` указывает на `fake_get`. Как только блок завершается, оригинал возвращается на место автоматически.

## Подводные камни

- Патчить нужно там, где имя используется, а не там, где оно определено. Если в модуле сделан `from requests import get`, то подменять надо `мой_модуль.get`, а не `requests.get`.
- Ручная подмена без `patch` опасна: если тест упадёт до отката, грязное состояние утечёт в соседние тесты. Контекстный менеджер или декоратор откатывают всё сами.
- Чрезмерное мокирование скрывает реальные баги интеграции — подменяйте границу системы, а не внутреннюю логику.
