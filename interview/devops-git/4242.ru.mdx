# Где в GitHub хранятся SSH и GPG ключи?

## Суть

Публичные SSH и GPG ключи лежат в настройках аккаунта: Settings → SSH and GPG keys. Приватные ключи никогда не загружаются на GitHub, только на клиенте (`~/.ssh`) или в CI secret store. Deploy keys привязаны к одному репозиторию (Settings → Deploy keys). Org-level SSH keys управляют доступом для automation bots.

## Что ответить на собесе

- User SSH key для git@github.com clone/push от разработчика.
- Deploy key read-only или write для одного repo в CI; не reuse между проектами.
- GPG public key в профиле для verified commits; private key в локальном gpg agent.
- GitHub Actions secrets отдельно: Settings → Secrets and variables (не путать с SSH keys UI).
- Честно: deploy key с write на prod repo это риск; prefer OIDC или fine-grained PAT с scope.

## Как это работает

Генерация SSH и добавление pubkey:

```bash
ssh-keygen -t ed25519 -C "alex@company.example" -f ~/.ssh/id_ed25519_github

cat ~/.ssh/id_ed25519_github.pub
ssh -T git@github.com
```

GPG для подписи коммитов:

```bash
gpg --full-generate-key
gpg --list-secret-keys --keyid-format long
gpg --armor --export ABCD1234EFGH5678

git config --global user.signingkey ABCD1234EFGH5678
git config --global commit.gpgsign true
```

Deploy key на repo даёт доступ только этому repository; org secrets для Actions лежат в Organization → Settings → Secrets.

## Подводные камни

- **Приватный ключ в репозитории.** Scanning поймает, но ротация болезненна; pre-commit hook блокирует.
- **Один SSH key на всех.** Компрометация открывает все repos пользователя; per-machine keys.
- **Deploy key write на fork PR.** RCE в CI + write key = supply chain; read-only по умолчанию.
- **GPG subkey expiry.** Подпись внезапно ломается; calendar reminder на rotate.
- **Passphrase-less CI keys.** Храню в vault с audit; never в plaintext workflow yaml.
