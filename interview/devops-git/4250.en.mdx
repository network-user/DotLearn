# Which tool is commonly used for GitOps on Kubernetes?

## In short

Argo CD and Flux CD dominate GitOps controllers for Kubernetes. Argo CD provides UI, multi-cluster, and ApplicationSet; Flux integrates natively with the CNCF ecosystem and Helm/Kustomize controllers. Both watch Git and reconcile cluster state.

## How to answer in an interview

- Argo CD: Application CRD, sync policy, health checks, RBAC via SSO.
- Flux: GitRepository plus Kustomization/HelmRelease; often paired with Flagger for canary.
- Choice depends on UI needs, multi-tenancy, and existing platform (OpenShift vs vanilla k8s).
- Secrets via SOPS/External Secrets, not plain in Git, regardless of tool.
- Honestly: "most popular" depends on the survey; enterprise often picks Argo CD, platform teams see Flux growing.

## How it works

Minimal Argo CD Application:

```bash
kubectl apply -f - <<'EOF'
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: storefront
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/org/k8s-manifests.git
    targetRevision: main
    path: apps/storefront
  destination:
    server: https://kubernetes.default.svc
    namespace: storefront
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
EOF

argocd app get storefront
argocd app sync storefront
```

Flux GitRepository plus Kustomization:

```bash
flux create source git platform \
  --url=https://github.com/org/k8s-manifests \
  --branch=main

flux create kustomization storefront \
  --source=platform \
  --path=./apps/storefront \
  --prune=true \
  --interval=1m
```

## Pitfalls

- **Auto-sync in prod.** selfHeal rolls back a manual debug patch without warning.
- **Helm chart drift.** Git pins chart version while values override lives in Argo UI.
- **CRD race.** Application sync runs before CRD operator install; ordering via sync waves.
- **Multi-cluster credentials.** Argo CD cluster secret rotation without downtime.
- **Observability gap.** Sync OK but pods CrashLoop; health checks and alerts are mandatory.
