# Как посмотреть изменения в Git?

## Суть

`git diff` сравнивает деревья: working tree vs index, index vs HEAD, произвольные коммиты и ветки. `git status` показывает что изменено; `git log -p` историю с патчами. Перед коммитом смотрю staged (`--staged`) и unstaged отдельно, чтобы не утащить debug в репозиторий.

## Что ответить на собесе

- `git diff` unstaged; `git diff --staged` то, что попадёт в следующий commit.
- `git diff main...feature` показывает изменения feature от точки расхождения (трехточечный diff для PR).
- `git diff --stat` для обзора объёма; `git log --oneline --graph` для структуры.
- Для бинарников `git diff` краткий; смотрю `git show SHA -- path`.
- Честно: огромный diff в терминале нечитаем; для review использую platform UI или `delta/difftastic`.

## Как это работает

```bash
git status -sb

git diff
git diff --staged

git diff --stat origin/main...HEAD

git log -5 --oneline --decorate
git show HEAD --stat
```

Сравнение конкретного файла между коммитами:

```bash
git diff a1b2c3d e4f5g6h -- src/deploy.sh
```

Перед коммитом:

```bash
git add -p
git diff --staged
git commit -m "fix(deploy): tighten health check timeout"
```

## Подводные камни

- **Двухточечный vs трёхточечный diff.** `main..feature` и `main...feature` разные базы; PR review путают.
- **Line endings CRLF.** Diff шумит на Windows; `.gitattributes` eol=lf для кода.
- **Submodules.** Обычный diff показывает только SHA submodule, не содержимое.
- **Ignored files.** `git diff` не видит untracked; `git status -u` и явный add.
- **Merge commit diff.** `git diff parent1..merge` неочевиден; для merge использую `git show -m`.
