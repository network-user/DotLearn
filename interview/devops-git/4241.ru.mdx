# Как создать и опубликовать Git-тег?

## Суть

Тег ставится на коммит: lightweight (`git tag v1.0.0`) или annotated (`git tag -a -m`). На remote теги не едут с обычным push branch; нужен `git push origin v1.0.0` или `--tags`. Удаление: локально `-d`, на remote `git push origin --delete tag`.

## Что ответить на собесе

- Релизы только annotated: `git tag -a v3.2.1 -m "..."`.
- Тег на конкретный SHA: `git tag -a v3.2.1 <sha>`.
- После push тега CI job `on: push tags: v*` собирает артефакт.
- Semver: breaking/minor/patch согласован с changelog.
- Честно: `git push --tags` пушит все локальные теги, включая черновики; пушу один тег явно.

## Как это работает

```bash
git switch main
git pull --rebase origin main

git tag -a v3.2.1 -m "Release 3.2.1"
git show v3.2.1

git push origin v3.2.1
```

Список и удаление ошибочного:

```bash
git tag -l 'v3.*'
git tag -d v3.2.1-wrong
git push origin --delete v3.2.1-wrong
```

Тег на прошлый коммит:

```bash
git log --oneline -10
git tag -a v3.1.2-hotfix abc1234 -m "Hotfix CVE-2024-xxx"
git push origin v3.1.2-hotfix
```

## Подводные камни

- **Тег не на том коммите.** CI деплоит wrong tree; double-check SHA в release checklist.
- **Локальный тег без push.** Релиз «выпущен» в чате, артефакта нет на remote.
- **Перезапись remote tag.** `--force` на tag ломает воспроизводимость; immutable policy.
- **Prerelease naming.** `v1.0.0-rc1` и `v1.0.0` разные pipelines; документирую regex триггера.
- **Signed tag без pubkey в GitHub.** Verified badge не появится; GPG key в account settings.
