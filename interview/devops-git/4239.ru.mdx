# Как изменить глобальные настройки Git?

## Суть

`git config --global key value` пишет в `~/.gitconfig` и применяется ко всем репозиториям пользователя, пока локальный `.git/config` не переопределит. Уровни: `--system`, `--global`, `--local` (приоритет растёт к local). Типичные ключи: `user.name`, `user.email`, `init.defaultBranch`, `pull.rebase`, `core.editor`.

## Что ответить на собесе

- Identity для коммитов: `user.name` и `user.email` глобально, корпоративный email переопределяю `--local` в work repos.
- `git config --global --list` и `git config --show-origin --get user.email` для аудита откуда значение.
- `includeIf` в gitconfig для разных профилей по пути (`~/work/` vs `~/oss/`).
- Не коммичу секреты в gitconfig; credential helper настраиваю осознанно.
- Честно: глобальный `user.email` с личной почтой на работе палит автора в корпоративных репо; фиксирую local override.

## Как это работает

```bash
git config --global user.name "Alex Devops"
git config --global user.email "alex@company.example"
git config --global init.defaultBranch main
git config --global pull.rebase true

git config --global --list
git config --show-origin --get-regexp '^user\.'
```

Локальное переопределение в проекте:

```bash
cd ~/work/platform
git config --local user.email "alex@company.example"
```

Фрагмент `~/.gitconfig`:

```ini
[user]
    name = Alex Devops
    email = alex.personal@example.com

[pull]
    rebase = true
```

## Подводные камни

- **Неверный email в коммитах.** Исправление через `git commit --amend` не меняет старые SHA; для истории нужен filter-repo.
- **credential.helper store.** Пароль в plaintext; prefer manager/cache с TTL.
- **Глобальный insteadOf URL.** Ломает clone других хостов; тестирую на sandbox remote.
- **core.autocrlf на Linux.** Глобальный true шумит в shell scripts; `input` или eol в attributes.
- **includeIf typo.** Тихо игнорируется; проверяю `git config --list --show-origin`.
