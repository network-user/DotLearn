# Как выразить логическое ИЛИ в bash?

## Суть

Логическое ИЛИ в bash: оператор `||` между командами или тестами, внутри `[[ ]]` можно `||` и `&&` без `-o`. Устаревший `test -o` в POSIX `[ ]` лучше не использовать в новых скриптах.

## Что ответить на собесе

- Между командами: `ping -c1 host-a || ping -c1 host-b`.
- В if: `if [[ "$ENV" == "prod" || "$ENV" == "staging" ]]; then ...`.
- В `[ ]`: `if [ "$x" -eq 1 ] || [ "$y" -eq 2 ]; then` (два отдельных test).
- Честно: `-o` в `[` легко сломать приоритетом; предпочитаю `[[` или два теста с `||`.

## Как это работает

Проверка, что хотя бы один health endpoint жив:

```bash
curl -fsS https://api.internal/health >/dev/null || \
curl -fsS https://api-backup.internal/health >/dev/null || \
{ echo "both health checks failed" >&2; exit 1; }
```

Условие на окружение:

```bash
if [[ "$DEPLOY_ENV" == "prod" || "$DEPLOY_ENV" == "staging" ]]; then
  require_approval=true
fi
```

Числовое сравнение в POSIX test:

```bash
if [ "$retry" -ge 3 ] || [ "$force" = "yes" ]; then
  echo "aborting rollout"
fi
```

Short-circuit: вторая часть `||` не выполняется, если первая успешна.

## Подводные камни

- **`test -o` и смешанные типы.** `[ "$a" = x -o "$b" -eq 1 ]` парсится не так, как ожидают.
- **Незаквотированные переменные.** Пустой `$var` ломает `[`.
- **`||` после pipeline без pipefail.** Успех маскируется последней командой pipe.
- **Двойное отрицание в CI.** `cmd || exit 0` скрывает падение job.
- **Приоритет с `&&`.** `a && b || c` не эквивалентен полному if без скобок.
