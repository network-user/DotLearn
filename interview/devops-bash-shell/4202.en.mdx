# How do [ ] and [[ ]] differ in bash?

## In short

`[` is the POSIX `test` builtin with strict syntax and separate words. `[[` is a bash keyword with safer quoting, regex `=~`, and inline `&&`/`||` inside one condition.

## How to answer in an interview

- `[` is portable to `/bin/sh`; `[[` is bash/ksh only.
- In `[[`, unquoted variables do not break the test on emptiness.
- Regex: `[[ "$host" =~ ^api-[0-9]+\.internal$ ]]`.
- Honestly: on Alpine init or dash shebang `[[` fails; for portable scripts I stay on `[` or explicit bash.

## How it works

POSIX test:

```bash
if [ -f /etc/os-release ] && [ "$(id -u)" -eq 0 ]; then
  echo "root on linux"
fi
```

Bash extended test:

```bash
if [[ -f /etc/os-release && $(id -u) -eq 0 ]]; then
  echo "root on linux"
fi

if [[ "$IMAGE_TAG" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
  echo "semver tag"
fi
```

String compare: use `==` in `[[`; in `[`, `-eq` is numeric only, strings use `=`. Filename globs in `[[` without a separate `case`.

## Pitfalls

- **Spaces after `[`.** `[ -f file]` is invalid; a space is required.
- **Closing `]` as an argument.** `]` must be its own token.
- **Regex in `[[` and locale.** Character classes differ under another LC_ALL.
- **Pasting `==` into `[`.** Fails on strict sh.
- **Command substitution in `[[` unquoted.** Word split can still happen outside.
