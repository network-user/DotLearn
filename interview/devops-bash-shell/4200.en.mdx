# How do you express logical OR in bash?

## In short

Logical OR in bash uses `||` between commands or tests; inside `[[ ]]` you can use `||` and `&&` without `-o`. Legacy `test -o` in POSIX `[ ]` is best avoided in new scripts.

## How to answer in an interview

- Between commands: `ping -c1 host-a || ping -c1 host-b`.
- In if: `if [[ "$ENV" == "prod" || "$ENV" == "staging" ]]; then ...`.
- In `[ ]`: `if [ "$x" -eq 1 ] || [ "$y" -eq 2 ]; then` (two separate tests).
- Honestly: `-o` in `[` breaks easily on precedence; I prefer `[[` or two tests with `||`.

## How it works

Check that at least one health endpoint is alive:

```bash
curl -fsS https://api.internal/health >/dev/null || \
curl -fsS https://api-backup.internal/health >/dev/null || \
{ echo "both health checks failed" >&2; exit 1; }
```

Environment condition:

```bash
if [[ "$DEPLOY_ENV" == "prod" || "$DEPLOY_ENV" == "staging" ]]; then
  require_approval=true
fi
```

Numeric compare in POSIX test:

```bash
if [ "$retry" -ge 3 ] || [ "$force" = "yes" ]; then
  echo "aborting rollout"
fi
```

Short-circuit: the second part of `||` does not run if the first succeeds.

## Pitfalls

- **`test -o` and mixed types.** `[ "$a" = x -o "$b" -eq 1 ]` does not parse as expected.
- **Unquoted variables.** An empty `$var` breaks `[`.
- **`||` after a pipeline without pipefail.** Success is masked by the last pipe command.
- **Double negation in CI.** `cmd || exit 0` hides job failure.
- **Precedence with `&&`.** `a && b || c` is not a full if without parentheses.
