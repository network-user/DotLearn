# Как объединить несколько команд bash в одну строку?

## Суть

В одной строке команды связывают `;` (всегда дальше), `&&` (дальше при успехе), `||` (дальше при ошибке) и `&` (фон). Скобки `()` и `{ }` группируют для pipeline и subshell.

## Что ответить на собесе

- `cmd1; cmd2` выполняет обе независимо от exit code.
- `cmd1 && cmd2` типичный guard перед destructive step.
- `cmd1 || cmd2` fallback или alert при сбое.
- Честно: длинные one-liner в CI плохо читаются и дебажатся; в скрипте раскладываю на строки с `set -e`.

## Как это работает

Создать каталог и перейти только при успехе:

```bash
mkdir -p /opt/releases/v2.4 && cd /opt/releases/v2.4
```

Fallback при недоступном primary mirror:

```bash
curl -fsS https://mirror-a/pkg.tgz -o pkg.tgz || \
  curl -fsS https://mirror-b/pkg.tgz -o pkg.tgz
```

Группа для pipeline:

```bash
( kubectl get pods -n prod | grep -v Running ) || echo "all pods running"
```

Фоновый job с ожиданием:

```bash
long_task.sh & wait $!
```

В Makefile и GitHub Actions те же операторы; помню, что `set -e` в скрипте не всегда эквивалентен ручному `&&` в интерактивной сессии.

## Подводные камни

- **`mkdir && cd` без `-p`.** Вторая команда не выполнится, но первая могла частично изменить FS.
- **`||` глотает код.** Конвейер `deploy || true` скрывает failure в мониторинге.
- **Subshell `( )`.** Переменные cd не видны родителю.
- **Приоритет `&&` и `||`.** Нужны скобки, иначе логика не та, что ожидал.
- **Background без wait.** CI завершается до окончания фоновой задачи.
