# Как записать if в одну строку bash?

## Суть

Однострочный `if` в bash разделяет `then`, `else` и `fi` точкой с запятой: `if условие; then cmd1; else cmd2; fi`. Для коротких guard-проверок в CI и ad-hoc shell это компактная альтернатива многострочному блоку.

## Что ответить на собесе

- Классика: `if [ -f file ]; then echo ok; else echo miss; fi`.
- С `[[` удобнее строки и regex: `if [[ -n "$VAR" ]]; then ...; fi`.
- Тернарный стиль через `&&`/`||`: `[[ -d dir ]] && cd dir || exit 1`.
- Честно: в production-скриптах длинный one-line if ухудшает review; раскладываю на блок при более чем одной команде в ветке.

## Как это работает

Проверка kubeconfig перед apply:

```bash
if [[ -f "$HOME/.kube/config" ]]; then kubectl get nodes; else echo "no kubeconfig" >&2; exit 1; fi
```

Проверка exit code предыдущей команды:

```bash
kubectl rollout status deploy/api -n prod; if [[ $? -ne 0 ]]; then logger -t deploy "rollout failed"; exit 1; fi
```

Короткий guard без else:

```bash
[[ -w /var/log ]] || { echo "log dir not writable" >&2; exit 1; }
```

В GitHub Actions и SSH one-liner экономят escape; помню кавычки вокруг переменных в `[ ]`.

## Подводные камни

- **Пустая переменная в `[`.** `[ $VAR = x ]` даёт syntax error; нужны кавычки или `[[`.
- **Отсутствие `fi`.** Shell парсит остаток строки как мусор.
- **`then` после newline без `;`.** В однострочнике `;` обязателен перед `then`.
- **Смешение `[` и `[[` синтаксиса.** `==` в `[` не POSIX на всех системах.
- **Нечитаемый else с вложенным if.** Лучше early return в функции.
