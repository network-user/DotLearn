# Чем отличаются [ ] и [[ ]] в bash?

## Суть

`[` это POSIX builtin `test` с жёстким синтаксисом и отдельными словами. `[[` это bash keyword с безопасным quoting, regex `=~` и встроенными `&&`/`||` внутри одного условия.

## Что ответить на собесе

- `[` переносим на `/bin/sh`; `[[` только bash/ksh.
- В `[[` переменные без кавычек не ломают test на пустоте.
- Regex: `[[ "$host" =~ ^api-[0-9]+\.internal$ ]]`.
- Честно: в Alpine init или dash shebang `[[` не работает; для portable скриптов остаюсь на `[` или явный bash.

## Как это работает

POSIX test:

```bash
if [ -f /etc/os-release ] && [ "$(id -u)" -eq 0 ]; then
  echo "root on linux"
fi
```

Bash extended test:

```bash
if [[ -f /etc/os-release && $(id -u) -eq 0 ]]; then
  echo "root on linux"
fi

if [[ "$IMAGE_TAG" =~ ^v[0-9]+\.[0-9]+\.[0-9]+$ ]]; then
  echo "semver tag"
fi
```

Сравнение строк: в `[[` использую `==`, в `[` для POSIX `-eq` только для чисел, строки через `=`. Filename glob в `[[` без отдельного `case`.

## Подводные камни

- **Пробелы после `[`.** `[ -f file]` invalid; нужен пробел.
- **Закрывающая `]` как аргумент.** `]` должен быть отдельным токеном.
- **Regex в `[[` и locale.** Классы символов ведут себя иначе при другой LC_ALL.
- **Копипаст `==` в `[`.** На строгом sh падает.
- **Command substitution в `[[` без кавычек.** Word split всё ещё возможен снаружи.
