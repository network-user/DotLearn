# Как описать собственный мини-язык и вычислять выражения по его грамматике?

## Суть

Interpreter (Интерпретатор) представляет грамматику языка как дерево объектов-выражений, где у каждого узла есть метод `interpret`. Вычисление выражения - это рекурсивный обход этого дерева. Подходит для небольших DSL: формул, правил, конфигов.

## Что ответить на собесе

Interpreter - поведенческий паттерн для задачи "разобрать и вычислить выражение по заданной грамматике".

- Каждому правилу грамматики соответствует свой класс выражения с общим интерфейсом `interpret(context)`.
- Терминальные выражения - это листья (число, переменная), нетерминальные - составные узлы (сложение, умножение), которые рекурсивно вызывают `interpret` у детей; контекст несёт общее состояние, например значения переменных.
- Дерево выражений собирается из этих узлов, и один вызов `interpret` на корне запускает рекурсию вниз.
- Применяю, когда нужен маленький собственный язык; для серьёзных языков беру готовый парсер, потому что классов на каждое правило становится слишком много.

## Как это работает

Грамматика отображается на иерархию классов. Терминалы возвращают значение напрямую, нетерминалы комбинируют результаты детей.

Узлы дерева - это и есть структура выражения `3 + x`. Корневой `Add` спрашивает значение у левого и правого ребёнка и складывает. Типичные применения: парсеры конфигов, движки правил, вычисление математических формул, простые скриптовые языки.

## Подводные камни

- Паттерн масштабируется плохо: на каждое правило грамматики - отдельный класс, и для сложного языка их становятся десятки. Тогда лучше готовый генератор парсеров или библиотека.
- Interpreter описывает только вычисление готового дерева. Превратить строку `"3 + x"` в это дерево (парсинг/лексинг) - отдельная задача, паттерн её не решает.
- Глубокая рекурсия на больших выражениях может упереться в лимит стека Python; для длинных цепочек нужен итеративный обход или увеличение лимита.
- Контекст легко превращается в общую свалку глобального состояния - держите в нём только то, что реально нужно выражениям.
