# Метод __init__: что инициализирует объект и зачем он нужен

## Суть

`__init__` - это инициализатор объекта: метод, который Python автоматически зовёт сразу после создания экземпляра, чтобы проставить его атрибутам стартовые значения. К моменту, когда вы получаете объект, он уже готов к работе.

## Что ответить на собесе

`__init__` - специальный метод класса, который запускается при создании каждого экземпляра. Первым аргументом он получает сам новый объект (`self`), дальше - любые параметры для настройки. Внутри мы навешиваем на `self` атрибуты. Строго говоря, объект создаёт `__new__` (он выделяет память и возвращает экземпляр), а `__init__` только наполняет уже готовый объект данными и ничего не возвращает. Поэтому в быту `__init__` называют конструктором, хотя конструирование - это пара `__new__` + `__init__`.

## Как это работает

При вызове `Car(...)` Python сначала зовёт `Car.__new__`, получает объект, затем передаёт его и аргументы в `__init__`. Тот заполняет атрибуты:

```python
class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    def display_info(self):
        print(f"{self.year} {self.make} {self.model}")


my_car = Car("Toyota", "Corolla", 2020)
my_car.display_info()
```

Здесь `__init__` принимает `make`, `model`, `year` и записывает их в атрибуты экземпляра. Без него пришлось бы выставлять поля вручную после создания, что многословно и легко привести к объекту в недонастроенном состоянии.

## Подводные камни

- **`__init__` не должен ничего возвращать.** Любой `return` со значением, кроме `None`, поднимет `TypeError`.
- **Изменяемые значения по умолчанию - ловушка.** `def __init__(self, items=[])` создаёт список один раз при определении функции, и он будет общим для всех экземпляров. Правильно: `items=None`, а внутри `self.items = items or []`.
- **`__init__` - не точка создания объекта.** Сам экземпляр рождает `__new__`; для неизменяемых типов и метаклассов это различие принципиально.
- **При наследовании не забывайте `super().__init__(...)`.** Если переопределили `__init__` и не позвали родительский, часть состояния останется неинициализированной.
- **Тяжёлую логику (запросы в сеть, открытие файлов) в `__init__` лучше не класть** - это усложняет тестирование и создание объектов; для альтернативных способов создания используют classmethod-фабрики.
