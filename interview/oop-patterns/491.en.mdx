# Prototype: when cloning a new object beats building it from scratch

## In short

Prototype is a creational pattern that produces a new object as a copy of a ready-made specimen instead of building it from scratch. It pays off when object initialization is expensive or cumbersome.

## How to answer in an interview

Prototype gives objects a cloning method (usually `clone()`), and new instances are produced by copying existing ones. This saves on complex initialization and makes it easy to spawn objects with a similar configuration. In Python the `copy` module is the canonical tool: `copy.copy` makes a shallow copy, `copy.deepcopy` makes a deep one. The key interview nuance is the difference between them: with a shallow copy, nested mutable objects remain shared.

## How it works

The specimen implements a method that returns its own copy. Internally it usually uses `copy`.

```python
import copy

class Prototype:
    def __init__(self, value, tags):
        self.value = value
        self.tags = tags

    def clone(self):
        return copy.copy(self)

original = Prototype(42, ["a"])
clone = original.clone()

print(original.value, clone.value)   # 42 42
print(original is clone)             # False
```

Here `clone` is a different object (`is` returns `False`), but with a shallow copy the `tags` list is the same for the original and the clone. If you need full independence, use `copy.deepcopy`.

```python
clone.tags.append("b")
print(original.tags)   # ['a', 'b']  - the list is shared with copy.copy
```

An analogy is a print shop: there is a business card template, and you print as many copies as you want, changing only minor details. You do not design the card anew each time, you clone a ready-made specimen.

## Pitfalls

- The main trap is the shallow copy. `copy.copy` duplicates the object, but nested mutable fields (lists, dicts) remain shared references.
- `copy.deepcopy` solves this, but it is more expensive and can loop on recursive structures (there is a memo dictionary for that).
- `clone` does not call `__init__` again - it copies the already-built state. If initialization has side effects, they will not be repeated.
- Prototype is a creational pattern, not a structural one; interviewers like to check this.
