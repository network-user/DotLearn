# Виды наследования в Python: от одиночного до гибридного

## Суть

По форме иерархии наследование делят на одиночное, множественное, многоуровневое, иерархическое и гибридное. Это просто способы описать, сколько у класса родителей и как они выстроены в дерево.

## Что ответить на собесе

Различают пять схем по форме иерархии.

- **Одиночное** - один родитель; **множественное** - сразу несколько родителей у одного класса.
- **Многоуровневое** - цепочка A → B → C, где потомок наследует от потомка; **иерархическое** - один родитель и несколько прямых потомков.
- **Гибридное** - комбинация предыдущих, например ромбовидная схема, где два предка сходятся в общем прародителе.
- Python поддерживает множественное наследование, а порядок поиска атрибутов и методов по этому графу классов определяет MRO (Method Resolution Order), вычисляемый алгоритмом C3.

## Как это работает

**Одиночное** - база переиспользования: потомок получает методы родителя.

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def bark(self):
        print("Dog barks")
```

**Множественное** - класс собирает поведение из нескольких источников.

```python
class Flyable:
    def fly(self):
        print("Can fly")

class Swimmable:
    def swim(self):
        print("Can swim")

class Duck(Flyable, Swimmable):
    pass
```

**Многоуровневое** - цепочка, где `Dog` тянет методы и из `Mammal`, и из `Animal`.

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Mammal(Animal):
    def walk(self):
        print("Mammal walks")

class Dog(Mammal):
    def bark(self):
        print("Dog barks")
```

**Иерархическое** - один родитель `Animal`, несколько потомков `Dog` и `Cat`. **Гибридное** - смесь схем; ниже `Bat` наследует от `Mammal` и `Bird`, которые оба идут от `Animal` (ромб):

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Mammal(Animal):
    def walk(self):
        print("Mammal walks")

class Bird(Animal):
    def fly(self):
        print("Bird flies")

class Bat(Mammal, Bird):
    pass

print([c.__name__ for c in Bat.__mro__])
```

При множественном и гибридном наследовании именно MRO решает, чей метод будет вызван, если имя совпадает у нескольких предков.

## Подводные камни

- **Ромбовидная проблема (diamond).** Если общий прародитель встречается через двух предков, наивный поиск зашёл бы в него дважды. C3-линеаризация в Python гарантирует, что каждый класс в MRO встречается ровно один раз.
- **Порядок родителей важен.** `class C(A, B)` и `class C(B, A)` дают разный MRO и потенциально разное поведение при совпадающих методах.
- **`super()` идёт по MRO, а не "к прямому родителю".** В ромбе `super()` может увести в соседнюю ветку, а не туда, где написан класс. Это и есть кооперативное наследование.
- **Несовместимый порядок баз** вызовет `TypeError: Cannot create a consistent method resolution order` ещё на этапе определения класса.
- **Гибридное наследование легко превращается в кашу.** Часто чище заменить его композицией или миксинами с узкой ответственностью.
