# Instance method, classmethod and staticmethod: what's the difference

## In short

The difference is in what the method receives as its first argument. A regular method receives `self` (the instance), a `classmethod` receives `cls` (the class itself), and a `staticmethod` receives nothing: it's just a function living in the class namespace.

## How to answer in an interview

An instance method works with a specific object through `self` - it reads and changes its attributes. A `classmethod` works with the class through `cls`: it changes class attributes or serves as an alternative constructor (`from_string`, `from_dict`), and `cls` correctly picks up the subclass under inheritance. A `staticmethod` is bound to neither the instance nor the class - it's a helper function that logically lives next to the class but doesn't depend on its state.

## How it works

An instance method is the default. The first parameter `self` gives access to the object's data.

```python
class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        return f"{self.name} says woof!"
```

A `classmethod` is declared with the `@classmethod` decorator, and its first parameter is `cls`. A typical use is an alternative constructor:

```python
class Dog:
    species = "Canis lupus familiaris"

    def __init__(self, name):
        self.name = name

    @classmethod
    def stray(cls):
        return cls("Nameless")

    @classmethod
    def get_species(cls):
        return cls.species
```

`cls` points to the class through which the method was called, so `stray` on a subclass returns an instance of the subclass, not of the base `Dog`.

A `staticmethod` takes neither `self` nor `cls`:

```python
class MathOps:
    @staticmethod
    def add(x, y):
        return x + y
```

When to use which:

- **instance method** - for working with a specific object's data;
- **classmethod** - for operations on the class and for factory constructors;
- **staticmethod** - for utilities logically tied to the class but independent of its state.

## Pitfalls

- The main advantage of `classmethod` over a hardcoded class name is polymorphism: `cls` picks up the subclass, and the factory method returns the correct type.
- A `staticmethod` differs from a plain function outside the class only in where it lives. If the body needs neither `self` nor `cls`, choose it rather than making an "empty" instance method.
- A forgotten `@classmethod`/`@staticmethod` decorator turns the method into a regular one: the first positional argument then unexpectedly captures the instance.
- You can't call an instance method without an instance (`Dog.bark()`) - `self` won't be supplied; whereas `classmethod` and `staticmethod` are called from the class without issue.
