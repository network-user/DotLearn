# Какие принципы ООП поддерживает Python и как они выглядят на практике

## Суть

Python - полноценный объектно-ориентированный язык: в нём есть инкапсуляция, наследование, полиморфизм и абстракция, а композицию используют как альтернативу глубоким иерархиям. Динамическая типизация и множественное наследование делают эти механизмы особенно гибкими.

## Что ответить на собесе

Назовите четыре классических столпа - инкапсуляция, наследование, полиморфизм, абстракция - и добавьте композицию как практичный приём. В Python полиморфизм во многом строится на утиной типизации: важно не наследование, а наличие нужного метода. Абстракцию задают через модуль `abc`, инкапсуляция держится на соглашениях (`_`, `__`), а не на жёстком сокрытии.

Инкапсуляция связывает данные и методы в одном объекте. Наследование строит новый класс поверх существующего. Полиморфизм даёт единый вызов для разных типов. Абстракция прячет детали за интерфейсом. Композиция собирает объект из других объектов.

## Подводные камни

- В Python нет настоящих приватных полей. `__name` лишь искажается через name mangling (`_Class__name`), но доступ к нему сохраняется. Инкапсуляция - это соглашение, а не запрет.
- Полиморфизм в Python чаще про утиную типизацию: объекту не обязательно наследоваться от общего класса, достаточно иметь нужный метод.
- Глубокие иерархии наследования хрупки; часто композиция («has-a») предпочтительнее наследования («is-a»).
- Абстрактный класс с `@abstractmethod` нельзя инстанцировать, пока наследник не реализует все абстрактные методы - иначе `TypeError` при создании объекта.
