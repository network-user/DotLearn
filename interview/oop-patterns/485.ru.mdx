# Как Адаптер подружит несовместимые интерфейсы без правки их кода

## Суть

Адаптер - переходник между двумя несовместимыми интерфейсами. Он оборачивает «чужой» объект и переводит вызовы в формат, который ждёт клиент, не меняя ни одну из сторон.

## Что ответить на собесе

Адаптер - структурный паттерн. Когда у меня есть готовый класс с одним интерфейсом, а клиент ждёт другой, я не переписываю ни тот, ни другой. Вместо этого делаю класс-обёртку, который реализует ожидаемый интерфейс, а внутри держит ссылку на адаптируемый объект и транслирует вызовы в его методы. Это спасает, когда исходный код трогать нельзя - сторонняя библиотека, легаси, изменившийся внешний API.

## Как это работает

Адаптер реализует целевой интерфейс и делегирует работу обёрнутому объекту, переименовывая и при необходимости преобразуя вызовы.

```python
class OldPrinter:
    def print_text(self, text):
        print(f"Old Printer: {text}")

class NewPrinterInterface:
    def print(self, text):
        pass

class PrinterAdapter(NewPrinterInterface):
    def __init__(self, old_printer):
        self.old_printer = old_printer

    def print(self, text):
        self.old_printer.print_text(text)

old_printer = OldPrinter()
adapter = PrinterAdapter(old_printer)
adapter.print("Hello, World!")
```

`PrinterAdapter` выставляет метод `print()`, которого ждёт новый код, а внутри зовёт `print_text()` у старого принтера. Теперь `OldPrinter` подставляется туда, где требуется `NewPrinterInterface`. Бытовая аналогия: переходник между HDMI-консолью и аналоговым телевизором.

## Подводные камни

- Не путайте с Фасадом: Фасад упрощает доступ к целой подсистеме новым интерфейсом, а Адаптер согласует один конкретный несовместимый интерфейс с ожидаемым.
- Адаптер только перенаправляет вызовы - он не должен тащить в себя бизнес-логику. Если внутри появляются нетривиальные правила, это уже не адаптер.
- При сложных преобразованиях данных адаптер легко раздувается и становится местом скрытых багов: следите, чтобы трансляция оставалась тонкой.
- Бывает объектный адаптер (через композицию, как в примере) и классовый (через множественное наследование). В Python чаще берут объектный - он гибче и не зависит от того, что адаптируемый класс можно наследовать.
