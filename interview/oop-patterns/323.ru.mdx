# Template Method: фиксируем скелет алгоритма, отдаём шаги наследникам

## Суть

Template Method закрепляет последовательность шагов алгоритма в методе базового класса, а сами шаги делает переопределяемыми в наследниках. Общая логика живёт в одном месте, различия - в подклассах.

## Что ответить на собесе

Поведенческий паттерн: в базовом классе есть один публичный метод-каркас, который вызывает набор более мелких шагов в нужном порядке.

- Часть шагов реализована в базе и общая для всех, часть объявлена как абстрактная или как заглушка и переопределяется наследниками.
- Клиент дёргает только каркасный метод, а варьируется внутреннее наполнение.
- Паттерн убирает дублирование общей части и не даёт наследникам сломать порядок шагов - инверсия управления в духе "не звони нам, мы позвоним тебе".

Каркасный метод (`prepare_recipe`) задаёт жёсткий порядок. Шаги `boil_water` и `pour_in_cup` одинаковы для всех, а `brew` и `add_condiments` обязан определить каждый наследник.

```python
from abc import ABC, abstractmethod

class CaffeineBeverage(ABC):
    def prepare_recipe(self):
        self.boil_water()
        self.brew()
        self.pour_in_cup()
        if self.wants_condiments():
            self.add_condiments()

    def boil_water(self):
        print("Кипятим воду")

    def pour_in_cup(self):
        print("Наливаем в чашку")

    def wants_condiments(self):
        return True

    @abstractmethod
    def brew(self):
        ...

    @abstractmethod
    def add_condiments(self):
        ...

class Tea(CaffeineBeverage):
    def brew(self):
        print("Завариваем чай")

    def add_condiments(self):
        print("Добавляем лимон")

class Coffee(CaffeineBeverage):
    def brew(self):
        print("Проливаем кофе через фильтр")

    def add_condiments(self):
        print("Добавляем сахар и молоко")

Tea().prepare_recipe()
Coffee().prepare_recipe()
```

Метод `wants_condiments` - это hook: необязательный шаг с разумным умолчанием, который наследник может, но не обязан переопределять.

## Подводные камни

- Паттерн строится на наследовании, а оно жёстко связывает наследника с базой. Если вариативности много, гибче оказывается Strategy с композицией.
- Каркасный метод стоит защищать от переопределения (по смыслу он `final`); в Python это лишь соглашение, язык не запрещает его перекрыть.
- Легко перегрузить базу абстрактными шагами - наследник вынуждают реализовывать то, что ему не нужно. Помогают hook-методы с дефолтом.
- Не путайте hook (необязательный, с реализацией по умолчанию) и абстрактный шаг (обязательный к реализации).
