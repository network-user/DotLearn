# Какую задачу решает super() и почему имя родителя в коде лучше не писать руками

## Суть

`super()` даёт доступ к реализации метода выше по иерархии наследования, не привязываясь к конкретному имени родителя. Опора на MRO вместо жёстко зашитого имени класса делает иерархию переносимой и спасает при множественном наследовании.

## Что ответить на собесе

`super()` возвращает прокси-объект, который делегирует вызов следующему классу в MRO текущего объекта, а не «родителю» в наивном понимании. Главный плюс — отвязка от конкретного имени класса: при переименовании или перестановке базовых классов код продолжает работать. В кооперативном множественном наследовании именно `super()` гарантирует, что каждый класс в ромбовидной схеме отработает ровно один раз и общий предок не вызовется дважды.

## Как это работает

Базовое наследование тривиально — подкласс просто получает методы предка:

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def bark(self):
        print("Dog barks")

dog = Dog()
dog.speak()  # Animal speaks
dog.bark()   # Dog barks
```

Интересное начинается при множественном наследовании, где возникает ромбовидная схема: два класса наследуют общего предка, а третий — оба этих класса. Если каждый метод зовёт `super().method()`, Python проходит классы строго в порядке MRO:

```python
class A:
    def do_something(self):
        print("A")

class B(A):
    def do_something(self):
        print("B")
        super().do_something()

class C(A):
    def do_something(self):
        print("C")
        super().do_something()

class D(B, C):
    def do_something(self):
        print("D")
        super().do_something()

D().do_something()
```

Вывод — `D`, `B`, `C`, `A`: MRO выстраивает классы в линейный порядок (его можно посмотреть в `D.__mro__`), и `super()` в `B` указывает не на `A`, а на следующий за `B` класс в этом порядке, то есть `C`. Так общий предок `A` вызывается один раз, а не дважды.

## Подводные камни

- `super()` идёт по MRO **экземпляра**, а не по статической паре «класс — его база». Поэтому `super()` внутри `B` может привести в `C`, хотя `B` напрямую от `C` не наследует.
- Если в кооперативной цепочке хоть один класс забудет вызвать `super()`, дальнейшие методы просто не отработают — цепочка оборвётся молча.
- Сигнатуры методов в кооперативной цепочке должны быть совместимы, иначе передача аргументов через `super()` сломается.
- Жёсткое `A.do_something(self)` вместо `super()` ломает кооперативность: общий предок может вызваться повторно.
