# Metaclasses: who controls class creation, and why?

## In short

A metaclass is a class whose instances are themselves classes. It intercepts the moment a class is born and lets you reshape it: add methods and attributes, validate the structure, enforce shared rules.

## How to answer in an interview

In Python, classes are objects too, and every object has a type. The type of a class is its metaclass; by default that is `type`. When the interpreter executes a `class` statement, it effectively calls the metaclass, passing the name, the bases, and the body namespace, and gets back a finished class. By swapping in a custom metaclass, I hook into this process: I add or validate attributes, register the class, or enforce an invariant across an entire hierarchy. In practice metaclasses are rarely needed - class decorators or `__init_subclass__` usually suffice - but in an interview it is important to show that a class is created by a call to its metaclass.

## How it works

Class creation goes through the metaclass's `__new__`/`__init__`. By overriding `__new__`, you can modify the attribute namespace before the class is assembled. In the example below, the metaclass adds a `hello` method to each of its classes:

```python
class HelloMeta(type):
    def __new__(mcs, name, bases, namespace):
        namespace["hello"] = lambda self: f"Hello from {self.__class__.__name__}"
        return super().__new__(mcs, name, bases, namespace)


class Widget(metaclass=HelloMeta):
    pass


obj = Widget()
print(obj.hello())  # Hello from Widget
```

Here is what happens step by step:

1. `HelloMeta` inherits from `type` - that is what makes it a metaclass.
2. `__new__` receives the metaclass itself `mcs`, the name of the future class `name`, the tuple of bases `bases`, and the body namespace `namespace`.
3. The `hello` method is placed into `namespace`.
4. `super().__new__` assembles the class with that method already present.
5. `Widget` is declared with `metaclass=HelloMeta`, so it gets `hello` automatically.

## Pitfalls

- Metaclasses are inherited: a subclass uses its parent's metaclass by default, and a metaclass conflict in multiple inheritance raises `TypeError`.
- For most tasks (field validation, subclass registration), `__init_subclass__` or a class decorator is simpler - they are clearer and do not break the hierarchy.
- Do not confuse the levels: the metaclass's `__new__` controls class creation, while a regular class's `__new__`/`__init__` controls instance creation.
- A metaclass runs once per class declaration, not per instance; instance-level logic does not belong there.
