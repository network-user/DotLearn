# Метод __call__: как превратить экземпляр в вызываемый объект

## Суть

`__call__` делает экземпляр класса вызываемым: запись `obj(...)` под капотом превращается в `obj.__call__(...)`. Это позволяет объекту вести себя как функция, но при этом помнить своё состояние между вызовами.

## Что ответить на собесе

В Python вызываемость - это просто наличие метода `__call__`. Любая функция вызываема, потому что у неё этот метод есть; и наоборот, обычный объект станет вызываемым, если определить `__call__` в его классе. Запись `obj(5)` Python разворачивает в `type(obj).__call__(obj, 5)`. Это удобно для объектов-функций, которые держат настройку или накопленное состояние: фабрики функций, классы-декораторы, конфигурируемые предикаты, модели, которые "вызывают" ради предсказания. Проверить вызываемость можно через `callable(obj)`.

## Как это работает

```python
class Multiplier:
    def __init__(self, factor):
        self.factor = factor

    def __call__(self, value):
        return value * self.factor


double = Multiplier(2)
print(double(5))          # 10
print(double.__call__(5)) # то же самое
print(callable(double))   # True
```

Конструктор фиксирует `factor`, а `__call__` использует его при каждом вызове. Объект `double` одновременно и хранит состояние (`factor`), и вызывается как функция - в этом его отличие от обычной функции, у которой состояние пришлось бы тащить через замыкание или глобали.

## Подводные камни

- **`__call__` ищется на типе, а не на экземпляре.** Присвоить `obj.__call__ = lambda ...` недостаточно: при вызове `obj()` Python смотрит `type(obj).__call__`, поэтому метод должен быть объявлен в классе.
- **`callable()` проверяет только наличие метода, а не корректность сигнатуры.** Объект может быть `callable`, но падать при вызове из-за неверных аргументов.
- **Не путать `__call__` с `__init__`.** `__init__` срабатывает один раз при создании, `__call__` - каждый раз, когда вы вызываете уже готовый объект.
- **Не злоупотребляйте.** Если объекту нужна одна операция, часто понятнее обычная функция или замыкание; `__call__` оправдан, когда важны состояние, конфигурация или единый интерфейс "вызвать".
