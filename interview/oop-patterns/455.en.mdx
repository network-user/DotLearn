# A stand-in object in front of the real object (the Proxy pattern)

## In short

Proxy substitutes a surrogate with the same interface in place of the real object. The client works with the surrogate, which controls access to the original and adds logic around the call without changing the object itself.

## How to answer in an interview

Proxy is a structural pattern. The surrogate implements the same interface as the real object and holds a reference to it. On a method call it can do something before or after delegating: check permissions, return a cached result, defer the expensive creation of an object, or write a log. To the client, the surrogate and the real object are indistinguishable — that's the whole point.

## How it works

The surrogate solves one of the typical tasks:

- **Protection** — checks permissions before the call.
- **Caching** — stores the result and returns the ready one on a repeat.
- **Virtual (lazy)** — creates a heavy object only on first access.
- **Logging** — records calls for debugging and monitoring.

```python
class RealSubject:
    def request(self):
        print("RealSubject: running an expensive request")
        return 42

class CachingProxy:
    def __init__(self, real_subject):
        self._real_subject = real_subject
        self._cached = None

    def request(self):
        if self._cached is None:
            print("Proxy: computing and caching")
            self._cached = self._real_subject.request()
        else:
            print("Proxy: returning from cache")
        return self._cached

proxy = CachingProxy(RealSubject())
proxy.request()   # computes
proxy.request()   # from cache
```

The key point: `CachingProxy` repeats the signature of `request`, so the client doesn't know whether it holds the real object or a surrogate.

## Pitfalls

- **Caching on `None` breaks** if the real result itself can be `None`: the check `if self._cached is None` will treat it as "not yet computed" and call the original every time. Use a separate flag or a sentinel value.
- **Proxy gets confused with Decorator and Adapter.** Adapter changes the interface, Decorator adds behavior, Proxy controls access while keeping the interface unchanged.
- **The surrogate is a point of failure and latency**: an extra level on every call. Don't pile up proxy chains without need.
- **A cache needs invalidation.** A cached result can go stale — think through its lifetime and how to reset it.
