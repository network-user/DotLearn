# Строитель: пошаговая сборка сложного объекта без конструктора-монстра

## Суть

Строитель - порождающий паттерн для пошаговой сборки сложного объекта. Он отделяет процесс конструирования от итогового представления, поэтому одной последовательностью шагов можно получать разные варианты объекта.

## Что ответить на собесе

Строитель спасает от «телескопического конструктора» - длинного списка необязательных аргументов, который невозможно читать.

- Объект собирается по шагам: отдельные методы задают части, а `build()` возвращает готовый продукт.
- Часто методы возвращают `self`, что даёт текучий интерфейс (method chaining).
- Опционально есть Директор - он инкапсулирует типовой порядок шагов, чтобы клиент не повторял рецепт сборки.

Продукт собирается через строитель, каждый шаг настраивает свою часть.

```python
class Car:
    def __init__(self):
        self.engine = None
        self.wheels = None
        self.color = None

    def __str__(self):
        return f"Car({self.engine}, {self.wheels}, {self.color})"

class CarBuilder:
    def __init__(self):
        self.car = Car()

    def set_engine(self, engine):
        self.car.engine = engine
        return self

    def set_wheels(self, wheels):
        self.car.wheels = wheels
        return self

    def set_color(self, color):
        self.car.color = color
        return self

    def build(self):
        return self.car
```

Каждый сеттер возвращает `self`, поэтому шаги выстраиваются в цепочку.

```python
class Director:
    def __init__(self, builder):
        self.builder = builder

    def construct_sports_car(self):
        return self.builder.set_engine("V8").set_wheels(4).set_color("Red").build()

car = Director(CarBuilder()).construct_sports_car()
print(car)   # Car(V8, 4, Red)
```

Директор знает рецепт «спорткар» и применяет его к любому строителю с тем же интерфейсом. Сам процесс сборки отделён от того, каким получится продукт.

## Подводные камни

- Не путайте с Фабричным методом: фабрика отдаёт готовый объект за один вызов, Строитель собирает его по шагам, когда конфигурация сложная.
- Директор необязателен. Если порядок шагов простой, текучего строителя достаточно, а Директор - лишний слой.
- Возврат `self` из методов нужен только для цепочки; если его забыть, chaining сломается с `AttributeError`.
- Для неизменяемых продуктов собирайте поля во временных переменных и создавайте объект целиком в `build()`, а не мутируйте готовый.
- В Python для простых случаев нередко хватает `dataclass` с дефолтами или именованных аргументов - не тащите Строитель туда, где он избыточен.
