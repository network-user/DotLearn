# Как переиспользовать общее состояние множества объектов (паттерн Flyweight)

## Суть

Flyweight экономит память, когда объектов очень много. Он разделяет состояние на общее (внутреннее) и уникальное (внешнее), хранит общее в единственном переиспользуемом экземпляре, а уникальное передаёт снаружи в момент вызова.

## Что ответить на собесе

Flyweight представляет собой структурный паттерн для случаев, когда тысячи или миллионы объектов отличаются лишь малой частью данных.

- Неизменяемую общую часть (внутреннее состояние) выношу в один объект-легковес и шарю его между всеми.
- Изменяемую уникальную часть (внешнее состояние) не храню внутри, а передаю аргументом при использовании.
- Фабрика с кэшем гарантирует, что под одинаковое внутреннее состояние создаётся только один легковес.

## Как это работает

1. **Разделите состояние** на внутреннее (общее, неизменное) и внешнее (уникальное для каждого использования).
2. **Сделайте легковес**: объект, несущий только внутреннее состояние.
3. **Передавайте внешнее состояние** в методы извне, чтобы один легковес обслуживал множество вызовов.
4. **Кэшируйте легковесы в фабрике**: повторный запрос с теми же параметрами возвращает существующий объект.

## Подводные камни

- **Внутреннее состояние обязано быть неизменяемым.** Легковес шарится, поэтому мутация одного места отразится на всех, кто его использует. Меняем только внешнее.
- **Экономия мнимая, если уникального состояния много.** Когда «общей» части почти нет, легковес не окупается, а только добавляет сложности.
- **Усложнение кода.** Появляется фабрика, кэш и разделение состояния; это оправдано лишь при действительно большом числе объектов.
- **Кэш фабрики живёт долго.** Это глобальный словарь; в долгоживущем процессе он может бесконтрольно расти, если ключей много.
