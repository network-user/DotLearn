# How to treat an element and a group uniformly via the Composite

## In short

The Composite arranges objects into a part-whole tree and lets you treat a group the same way as a single element. The client calls the same method without figuring out whether it's facing a leaf or a branch.

## How to answer in an interview

The Composite is a structural pattern for tree-shaped hierarchies. I define a common component interface that both the leaf (the terminal element) and the container (which holds other components) inherit from. The container implements the operation by delegating it to its children recursively. Thanks to the shared interface, client code is identical for a single object and for an entire subtree - this removes the "if it's a group, then..." branching.

## How it works

Three roles: the component defines the common interface, the leaf implements behavior with no children, and the container holds children and forwards the call down the tree.

```python
from abc import ABC, abstractmethod

class Graphic(ABC):
    @abstractmethod
    def draw(self):
        pass

class Circle(Graphic):
    def draw(self):
        print("Drawing a Circle")

class CompositeGraphic(Graphic):
    def __init__(self):
        self._children = []

    def add(self, graphic):
        self._children.append(graphic)

    def remove(self, graphic):
        self._children.remove(graphic)

    def draw(self):
        for child in self._children:
            child.draw()

circle1 = Circle()
circle2 = Circle()

composite = CompositeGraphic()
composite.add(circle1)
composite.add(circle2)

composite.draw()
```

`Circle` is a leaf, `CompositeGraphic` is a container. The `draw` call works the same for both, and the container simply walks its children recursively. This is how file systems work (a folder contains files and folders), as well as UI element trees and org charts.

## Pitfalls

- A leaf can't hold children: calling `add`/`remove` on it is a design error. This is resolved either by the "transparent" variant (the methods live in the common interface, but the leaf raises an exception) or by the "safe" variant (the methods live only in the container, but full uniformity is lost).
- A recursive traversal of a deep tree can hit Python's recursion limit or create a noticeable load - for large structures, consider an iterative traversal.
- An overly general interface blurs responsibility: if you pull in operations that make sense only for containers, the leaves get cluttered with empty stubs.
- The Composite doesn't watch for cycles: if you accidentally add a container to itself, the traversal goes into an infinite loop.
