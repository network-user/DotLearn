# Three ways to declare a class in Python: class, type(), and metaclasses

## In short

Usually a class is declared with the `class` keyword. But under the hood the class is created by the callable `type`, so the same class can be assembled dynamically through `type(name, bases, dict)` or its creation can be controlled through a metaclass.

## How to answer in an interview

The standard path is the `class` syntax. It is just syntactic sugar: the interpreter still calls the metaclass (by default `type`), passing it the name, a tuple of bases, and the namespace dictionary. This means a class can be created by hand through `type('Name', (Base,), {...})` for dynamic generation, and if you need to intervene at the moment any class is created, you attach a metaclass by overriding `__new__` or `__init__`.

All three ways ultimately go through `type`. The difference is the level of control.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} издаёт звук"
```

The same thing dynamically through `type()`:

```python
def init(self, name):
    self.name = name

Animal = type("Animal", (object,), {
    "__init__": init,
    "speak": lambda self: f"{self.name} издаёт звук",
})

print(Animal("Dog").speak())
```

A metaclass intercepts the class creation and can edit its namespace:

```python
class Meta(type):
    def __new__(mcls, name, bases, ns):
        ns["category"] = "Animal"
        return super().__new__(mcls, name, bases, ns)

class Cat(metaclass=Meta):
    pass

print(Cat.category)
```

## Pitfalls

- `type` with one argument returns the type of an object (`type(5)`), and with three arguments it creates a class. These are two different operations of the same function.
- Metaclasses are a heavy tool. For most tasks class decorators, `__init_subclass__`, or descriptors are enough; you reach for a metaclass when you need to influence the class creation itself.
- A subclass's metaclass must be compatible (a subclass) with the parent's metaclass, otherwise you get a `TypeError` about a metaclass conflict.
- Dynamic classes through `type()` are hard to read and debug - use them only where the structure really is unknown in advance.
