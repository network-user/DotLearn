# Why structural patterns exist and what problems they solve

## In short

Structural patterns answer the question of how to connect classes and objects into larger constructs while keeping the system flexible. They deal with composition and relationships rather than object creation itself.

## How to answer in an interview

This is one of the three GoF pattern families (alongside creational and behavioral). Structural patterns describe how to assemble objects into composite structures: Adapter bridges incompatible interfaces, Composite lets you treat a tree of objects as a single element, and Facade hides a complex subsystem behind a simple entry point. Their main goal is to reduce coupling and simplify maintenance without changing the behavior of the individual parts.

## How it works

Each structural pattern solves its own composition problem.

**Adapter** wraps a foreign object and translates its calls into the expected interface.

```python
class EuropeanSocket:
    def voltage(self):
        return 230

class Adapter:
    def __init__(self, socket):
        self.socket = socket

    def voltage(self):
        return self.socket.voltage()

adapter = Adapter(EuropeanSocket())
print(adapter.voltage())  # 230
```

**Composite** lets you treat a single object and a group of objects uniformly.

```python
class Leaf:
    def operation(self):
        return "Leaf"

class Composite:
    def __init__(self):
        self.children = []

    def add(self, component):
        self.children.append(component)

    def operation(self):
        inner = "+".join(child.operation() for child in self.children)
        return f"Composite({inner})"

composite = Composite()
composite.add(Leaf())
composite.add(Leaf())
print(composite.operation())  # Composite(Leaf+Leaf)
```

**Facade** exposes a single simple entry point to a set of subsystems.

```python
class SubsystemA:
    def operation_a(self):
        return "A"

class SubsystemB:
    def operation_b(self):
        return "B"

class Facade:
    def __init__(self):
        self.a = SubsystemA()
        self.b = SubsystemB()

    def operation(self):
        return f"Facade({self.a.operation_a()}+{self.b.operation_b()})"

print(Facade().operation())  # Facade(A+B)
```

## Pitfalls

- People confuse the pattern families: Factory and Prototype are creational, not structural; Observer is behavioral.
- Adapter adds no new behavior, it only translates calls. If you actually want to extend behavior, that is Decorator.
- Facade does not forbid direct access to subsystems, it merely provides a convenient front entrance; do not turn it into a "god object".
- Composite works where a leaf and a container share a common interface; without that, the uniformity is lost.
