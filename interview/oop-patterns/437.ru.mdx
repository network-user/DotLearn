# Как откатить объект к прошлому состоянию, не вскрывая его внутренности?

## Суть

Memento (Снимок) фиксирует состояние объекта в отдельном объекте-снимке и позволяет позже восстановиться из него, не раскрывая приватные поля наружу. Это базовый кирпич для undo/redo и отката изменений.

## Что ответить на собесе

Memento - поведенческий паттерн, который решает задачу отмены действий, сохраняя инкапсуляцию.

- Originator хранит данные и сам умеет создавать и принимать снимок.
- Memento - неизменяемый контейнер состояния, который наружу свою кухню не показывает.
- Caretaker ведёт историю снимков, но не лезет в их содержимое: при откате он просто отдаёт Originator нужный снимок, и тот восстанавливает себя сам.
- Ключевая мысль: только Originator знает, как читать и применять снимок, поэтому инкапсуляция не ломается.

## Как это работает

Логика держится на разделении ответственности. Originator не отдаёт свои поля наружу - он отдаёт запечатанный снимок. Caretaker умеет только складывать снимки в историю и доставать их по индексу, но не интерпретирует их.

```python
class Memento:
    def __init__(self, state):
        self._state = state

    def get_state(self):
        return self._state


class Originator:
    def __init__(self):
        self._state = None

    def set_state(self, state):
        self._state = state

    def save(self):
        return Memento(self._state)

    def restore(self, memento):
        self._state = memento.get_state()


class Caretaker:
    def __init__(self):
        self._history = []

    def push(self, memento):
        self._history.append(memento)

    def get(self, index):
        return self._history[index]


originator = Originator()
caretaker = Caretaker()

originator.set_state("State1")
caretaker.push(originator.save())

originator.set_state("State2")
caretaker.push(originator.save())

originator.set_state("State3")
originator.restore(caretaker.get(0))
print(originator._state)  # State1
```

## Подводные камни

- Снимки могут съесть много памяти: каждое состояние - отдельный объект. Для тяжёлых состояний используйте инкрементальные дельты, а не полные копии.
- Если Memento хранит ссылку, а не копию изменяемых данных, восстановление вернёт уже испорченное состояние. Снимок должен быть фактически неизменяемым.
- Caretaker не должен читать или менять содержимое снимка - иначе инкапсуляция, ради которой всё затевалось, рушится.
- Глубокое и поверхностное копирование путают: для вложенных структур нужен `copy.deepcopy`, иначе снимок и оригинал делят одни и те же вложенные объекты.
