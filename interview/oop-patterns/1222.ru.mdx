# Один интерфейс — много типов: полиморфизм в стандартной библиотеке Python

## Суть

Полиморфизм — это когда один и тот же вызов работает с объектами разных типов. В Python он держится не на общем базовом классе, а на наличии нужного метода (утиная типизация). Классика — `len()`: одинаково считает длину строки, списка и любого объекта с `__len__`.

## Что ответить на собесе

Полиморфизм даёт единый интерфейс для разных типов. В Python он опирается на динамическую типизацию: важно не наследование, а наличие подходящего метода у объекта. Встроенные функции вроде `len()`, `sorted()`, `iter()` полиморфны, потому что делегируют работу дандер-методам (`__len__`, `__iter__`, сравнениям). Дандер `__str__` тоже полиморфен: каждый класс задаёт своё строковое представление. Благодаря этому можно писать обобщённые функции, не привязываясь к конкретному классу аргумента.

## Как это работает

Полиморфизм бывает через наследование (переопределённый метод подкласса) и без него — лишь бы у объекта был метод с нужным именем. Встроенные функции обращаются к протоколу:

```python
print(len("Hello"))      # 5
print(len([1, 2, 3, 4])) # 4
print(sorted([3, 1, 2])) # [1, 2, 3]
```

`len(x)` под капотом вызывает `type(x).__len__(x)`, поэтому ваш класс с `__len__` тоже подойдёт. То же с `__str__`:

```python
class Dog:
    def __str__(self):
        return "это собака"


class Cat:
    def __str__(self):
        return "это кошка"


print(Dog())  # это собака
print(Cat())  # это кошка
```

Обобщённая функция не проверяет тип — она просто рассчитывает на наличие метода:

```python
class Dog:
    def speak(self):
        return "Woof!"


class Cat:
    def speak(self):
        return "Meow!"


def animal_sound(animal):
    return animal.speak()


print(animal_sound(Dog()))  # Woof!
print(animal_sound(Cat()))  # Meow!
```

`Dog` и `Cat` даже не связаны общим предком — `animal_sound` работает с любым объектом, у которого есть `speak`.

## Подводные камни

- Утиная типизация не проверяется заранее: если у объекта нет `speak`, ошибка вылетит только в момент вызова (`AttributeError`).
- `len()` требует именно `__len__`, который возвращает неотрицательное целое; наличие `__iter__` тут не поможет.
- `sorted()` возвращает новый список и для строки даёт список символов, а не строку — это удивляет.
- Полиморфизм через наследование требует совпадения сигнатуры метода; если подкласс меняет параметры, обобщённый код может сломаться.
