# Decorator as a way to extend an object's behavior without editing its code

## In short

The Decorator (also known as Wrapper) is a structural pattern: it attaches new behavior to an object at runtime by wrapping it in another wrapper object. The original code stays untouched.

## How to answer in an interview

The Decorator is a structural pattern that dynamically adds responsibilities to an object without changing its class or spawning subclasses. The object is placed inside a wrapper with the same interface, and the wrapper intercepts calls: it does something of its own before and after, and delegates the actual work to the original object. In Python this is most often implemented through function decorators with the `@` syntax, and used for logging, caching, permission checks and similar cross-cutting concerns.

## How it works

A function decorator in Python is a callable that takes a function and returns a new function. The new one usually calls the original, adding behavior around it.

```python
def my_decorator(func):
    def wrapper():
        print("Actions before the call")
        func()
        print("Actions after the call")
    return wrapper

def say_hello():
    print("Hello!")

decorated = my_decorator(say_hello)
decorated()
```

Result:

```
Actions before the call
Hello!
Actions after the call
```

The `@` notation is syntactic sugar: `@my_decorator` above `say_hello` means exactly `say_hello = my_decorator(say_hello)`.

```python
@my_decorator
def say_hello():
    print("Hello!")

say_hello()
```

To make the wrapper pass any arguments straight through, give it `*args` and `**kwargs` and return the result of the original call:

```python
def log_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}: args={args}, kwargs={kwargs}")
        result = func(*args, **kwargs)
        print(f"{func.__name__} returned {result}")
        return result
    return wrapper

@log_decorator
def add(a, b):
    return a + b

add(3, 5)
```

Output:

```
Calling add: args=(3, 5), kwargs={}
add returned 8
```

## Pitfalls

- Without `return result` in the wrapper, the decorated function will silently start returning `None` - a common and nasty bug.
- The wrapper overwrites metadata: `add.__name__` becomes `wrapper`. To preserve it, wrap `wrapper` with `functools.wraps(func)`.
- If you forget `*args, **kwargs`, the decorator will break any function that takes arguments.
- A decorator function and the GoF "Decorator" pattern are related but not the same: the classic pattern wraps objects, not functions, and preserves a common interface.
