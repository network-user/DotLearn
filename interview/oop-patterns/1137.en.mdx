# How Python decides whose method to call when there are many parents

## In short

The MRO (Method Resolution Order) is a precomputed linear list of classes that Python walks when looking for a method or attribute. With multiple inheritance it removes any ambiguity: lookup always goes through this list from left to right. The list itself is built by the C3 linearization algorithm.

## How to answer in an interview

The MRO answers the question "in what order does Python walk classes when looking for an attribute." For each class this order is computed once when the class is created, using the C3 algorithm, and stored in `__mro__`. C3 gives three guarantees: a class comes before its ancestors, locally closer classes take priority over more distant ones, and the specified inheritance order is not violated. When I call a method on an object, Python takes the MRO of its class and returns the first implementation found. You can view the order through `Cls.mro()` or `Cls.__mro__`.

C3 builds a chain with three properties:

- **Local precedence**: classes located closer in the hierarchy come first.
- **Consistency**: the order in which parents are listed is preserved.
- **Monotonicity**: if A inherits from B, then A always comes before B.

The classic diamond example:

```python
class A:
    def method(self):
        print("Method in A")

class B(A):
    def method(self):
        print("Method in B")

class C(A):
    def method(self):
        print("Method in C")

class D(B, C):
    pass

D().method()  # Method in B
```

The MRO for `D` comes out as `D -> B -> C -> A -> object`. Python walks the list, finds `method` on `B`, and stops. That is why "Method in B" is printed.

The order is available two ways:

```python
print(D.mro())
print(D.__mro__)
# [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

## Pitfalls

- `super()` does not mean "parent." It means "the next class in the MRO of the current object." In a diamond, `super()` inside `B` can lead to `C`, not `A` - and that is exactly what makes cooperative inheritance possible.
- The MRO depends on the specific object, not only on the class where the call is written. The same `super().method()` in `B` will go differently for an instance of `B` and an instance of `D`.
- `object` always closes the list last: it is the common ancestor of all classes.
- If C3 cannot reconcile the parents' orders, the class simply won't be created - Python raises `TypeError: Cannot create a consistent method resolution order`.
