# __new__ против __init__: кто создаёт объект, а кто его настраивает

## Суть

`__new__` отвечает за выделение и возврат самого экземпляра, а `__init__` лишь заполняет уже созданный объект данными. Порядок жёсткий: сперва `__new__`, затем `__init__`.

## Что ответить на собесе

Когда вы пишете `MyClass(10)`, создание объекта проходит в два жёстко упорядоченных шага: сначала `__new__`, потом `__init__`.

- `__new__(cls, ...)` — статический метод, который создаёт и возвращает сам экземпляр; его задача выдать объект.
- `__init__(self, ...)` запускается, только если `__new__` вернул экземпляр этого класса; он ничего не возвращает (обязан вернуть `None`) и лишь настраивает состояние через `self`.
- В 99% кода переопределяют только `__init__`.
- `__new__` трогают редко: при наследовании от неизменяемых типов (`int`, `str`, `tuple`), при реализации синглтона или кеширования экземпляров, в метаклассах.

`__new__` получает первым аргументом класс `cls` и обычно делегирует создание родителю:

<PyStepper title="Порядок: сначала __new__, потом __init__" code={`class MyClass:
    def __new__(cls, *args, **kwargs):
        instance = super().__new__(cls)
        print("создан экземпляр")
        return instance

    def __init__(self, value):
        self.value = value
        print("инициализирован")

obj = MyClass(10)
# создан экземпляр
# инициализирован`} />

Если же класс простой, переопределять `__new__` не нужно — хватает `__init__`:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
print(p.x, p.y)  # 1 2
```

## Подводные камни

- `__init__` не должен ничего возвращать, кроме `None`. `return self.value` из `__init__` приведёт к `TypeError`.
- `__init__` запустится только если `__new__` вернул экземпляр того же класса. Вернёте чужой объект — `__init__` молча пропустится.
- `__new__` — статический метод и получает `cls` явным первым аргументом; забыть передать `cls` в `super().__new__(cls)` — частая ошибка.
- При наследовании от неизменяемого типа значение задают именно в `__new__`: `int.__new__(cls, value)`, потому что в `__init__` менять уже поздно — объект неизменяем.
