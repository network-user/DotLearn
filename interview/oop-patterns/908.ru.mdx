# Три способа объявить класс в Python: class, type() и метаклассы

## Суть

Обычно класс объявляют ключевым словом `class`. Но под капотом класс создаёт вызываемый `type`, поэтому тот же класс можно собрать динамически через `type(name, bases, dict)` или управлять его созданием через метакласс.

## Что ответить на собесе

Стандартный путь - синтаксис `class`, но это просто синтаксический сахар.

- Интерпретатор всё равно зовёт метакласс (по умолчанию `type`), передавая ему имя, кортеж баз и словарь пространства имён.
- Значит, класс можно создать руками через `type('Name', (Base,), {...})` для динамической генерации.
- А если нужно вмешаться в момент создания любого класса - подключают метакласс, переопределяя `__new__` или `__init__`.

Все три способа в итоге проходят через `type`. Различие - в уровне контроля.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} издаёт звук"
```

То же самое динамически через `type()`:

Метакласс перехватывает создание класса и может править его пространство имён:

```python
class Meta(type):
    def __new__(mcls, name, bases, ns):
        ns["category"] = "Animal"
        return super().__new__(mcls, name, bases, ns)

class Cat(metaclass=Meta):
    pass

print(Cat.category)
```

## Подводные камни

- `type` с одним аргументом возвращает тип объекта (`type(5)`), а с тремя - создаёт класс. Это две разные операции одной функции.
- Метаклассы - тяжёлый инструмент. В большинстве задач хватает декораторов класса, `__init_subclass__` или дескрипторов; метакласс берут, когда надо влиять на само создание класса.
- Метакласс наследника должен быть совместим (подклассом) с метаклассом родителя, иначе будет `TypeError` о конфликте метаклассов.
- Динамические классы через `type()` трудно читать и отлаживать - применяйте их только там, где структура действительно неизвестна заранее.
