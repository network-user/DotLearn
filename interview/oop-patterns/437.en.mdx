# How do you roll an object back to a past state without exposing its internals?

## In short

Memento captures the state of an object in a separate snapshot object and lets you later restore from it without exposing private fields to the outside. It is the basic building block for undo/redo and rolling back changes.

## How to answer in an interview

Memento is a behavioral pattern that solves the problem of undoing actions while preserving encapsulation. I distinguish three roles: the Originator holds the data and itself knows how to create and accept a snapshot; the Memento is an immutable state container that does not reveal its internals to the outside; the Caretaker keeps a history of snapshots but does not poke into their contents. When a rollback is needed, the Caretaker hands the Originator the required snapshot, and the Originator restores itself. The key idea: only the Originator knows how to read and apply a snapshot, so encapsulation is not broken.

## How it works

The logic rests on a separation of responsibilities. The Originator does not give out its fields - it gives out a sealed snapshot. The Caretaker can only put snapshots into the history and pull them out by index, but it does not interpret them.

```python
class Memento:
    def __init__(self, state):
        self._state = state

    def get_state(self):
        return self._state


class Originator:
    def __init__(self):
        self._state = None

    def set_state(self, state):
        self._state = state

    def save(self):
        return Memento(self._state)

    def restore(self, memento):
        self._state = memento.get_state()


class Caretaker:
    def __init__(self):
        self._history = []

    def push(self, memento):
        self._history.append(memento)

    def get(self, index):
        return self._history[index]


originator = Originator()
caretaker = Caretaker()

originator.set_state("State1")
caretaker.push(originator.save())

originator.set_state("State2")
caretaker.push(originator.save())

originator.set_state("State3")
originator.restore(caretaker.get(0))
print(originator._state)  # State1
```

Scenarios where the pattern feels natural: text editors (undo/redo), games (save points), database transactions (rollback), any complex object that should be able to return to its past states.

## Pitfalls

- Snapshots can eat a lot of memory: each state is a separate object. For heavy states, use incremental deltas rather than full copies.
- If a Memento stores a reference rather than a copy of mutable data, restoration will bring back already-corrupted state. A snapshot must be effectively immutable.
- The Caretaker must not read or change a snapshot's contents - otherwise the encapsulation the whole thing was built for collapses.
- Deep and shallow copying get confused: nested structures need `copy.deepcopy`, otherwise the snapshot and the original share the same nested objects.
