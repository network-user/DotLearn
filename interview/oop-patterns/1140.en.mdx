# Where does a variable live: on the class or the object, and why it matters

## In short

A class attribute is stored in the class itself and is shared by all its instances. An instance attribute is stored in a specific object, and each object has its own. The difference between them determines whether all objects will see one change or only one of them.

## How to answer in an interview

A class attribute is declared in the class body outside methods and exists as a single copy - it is shared by all objects. An instance attribute is usually created in `__init__` through `self.attr` and belongs to a specific object. The key point is the lookup order when reading: Python first looks in the object's `__dict__`, and only if it is empty there - in the class and its MRO. That is why an assignment `obj.attr = ...` does not change the class attribute but creates a new attribute on the object that "shadows" it.

A class attribute is visible to all instances, and a change at the class level is reflected on each one:

```python
class Car:
    wheels = 4  # атрибут класса

car1 = Car()
car2 = Car()
print(car1.wheels, car2.wheels)  # 4 4

Car.wheels = 6
print(car1.wheels, car2.wheels)  # 6 6
```

An instance attribute is unique to the object:

```python
class Car:
    def __init__(self, color):
        self.color = color  # атрибут экземпляра

car1 = Car("red")
car2 = Car("blue")
print(car1.color, car2.color)  # red blue

car1.color = "green"
print(car1.color, car2.color)  # green blue
```

When to choose which:

- **Class attribute** - for something shared by everyone: constants, settings, a counter of created objects.
- **Instance attribute** - for the individual state of each object.

## Pitfalls

- Assigning through the object does not touch the class. `car1.wheels = 6` creates an instance attribute that obscures the class one. `car2.wheels` will keep the old value.
- A mutable class attribute is a common trap. If the class holds `items = []` and a method does `self.items.append(x)`, all objects write into the same list, because there was no assignment - the shared object was mutated. Initialize lists and dicts in `__init__`.
- Reading and writing are asymmetric. `obj.attr` may be read from the class, but `obj.attr = ...` always writes to the instance.
- To deliberately change a class attribute, address it through the class: `Car.wheels = 6`, not through an instance.
