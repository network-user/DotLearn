# Как Python решает, чей метод вызвать, когда родителей много?

## Суть

MRO (Method Resolution Order) - это заранее посчитанный линейный список классов, по которому Python ищет метод или атрибут. При множественном наследовании он убирает любую двусмысленность: поиск всегда идёт по этому списку слева направо. Сам список строит алгоритм C3-линеаризации.

## Что ответить на собесе

MRO отвечает на вопрос "в каком порядке Python обходит классы при поиске атрибута".

- Для каждого класса этот порядок вычисляется один раз при его создании по алгоритму C3 и хранится в `__mro__`.
- C3 даёт три гарантии: класс идёт раньше своих предков, локально близкие классы приоритетнее дальних, а заданный порядок наследования не нарушается.
- Когда я зову метод на объекте, Python берёт MRO его класса и возвращает первую найденную реализацию; посмотреть порядок можно через `Cls.mro()` или `Cls.__mro__`.

C3 строит цепочку с тремя свойствами:

- **Локальный приоритет**: классы, расположенные ближе в иерархии, идут раньше.
- **Согласованность**: порядок перечисления родителей сохраняется.
- **Монотонность**: если A наследуется от B, то A всегда стоит впереди B.

Классический пример с ромбом:

<PyStepper title="MRO: поиск метода по ромбу" code={`class A:
    def method(self):
        print("Method in A")

class B(A):
def method(self):
print("Method in B")

class C(A):
def method(self):
print("Method in C")

class D(B, C):
pass

D().method() # Method in B`} />

MRO для `D` получается `D -> B -> C -> A -> object`. Python идёт по списку, на `B` находит `method` и останавливается. Поэтому печатается "Method in B".

Порядок доступен двумя способами:

```python
print(D.mro())
print(D.__mro__)
# [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

## Подводные камни

- `super()` не означает "родитель". Он означает "следующий класс в MRO текущего объекта". В ромбе `super()` внутри `B` может привести в `C`, а не в `A`, - это и делает кооперативное наследование возможным.
- MRO зависит от конкретного объекта, а не только от класса, где написан вызов. Один и тот же `super().method()` в `B` пойдёт по-разному для экземпляра `B` и экземпляра `D`.
- `object` всегда замыкает список последним: это общий предок всех классов.
- Если C3 не может согласовать порядки родителей, класс просто не создастся - Python бросит `TypeError: Cannot create a consistent method resolution order`.
