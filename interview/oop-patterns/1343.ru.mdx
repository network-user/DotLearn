# classmethod и staticmethod: кому достаётся cls, а кому ничего

## Суть

`@classmethod` получает первым аргументом сам класс (`cls`) и работает с состоянием класса, в том числе как фабрика новых экземпляров. `@staticmethod` не получает ни класса, ни экземпляра — это просто функция, живущая в неймспейсе класса.

## Что ответить на собесе

Оба декоратора позволяют вызывать метод без создания экземпляра, но различаются по тому, что они получают первым аргументом.

- `@classmethod` связывается с классом: через `cls` читает и меняет атрибуты класса, удобен для альтернативных конструкторов (`from_string`, `from_dict`).
- `@staticmethod` не связывается ни с чем — нет ни `self`, ни `cls`; это вспомогательная логика, которую логично держать рядом с классом.
- Обычный метод против них получает `self`, то есть конкретный экземпляр.

## Как это работает

Классметод видит класс и потому может менять общее для всех экземпляров состояние:

```python
class Counter:
    total = 0

    @classmethod
    def bump(cls):
        cls.total += 1
        return cls.total

    @classmethod
    def from_start(cls, start):
        obj = cls()
        cls.total = start
        return obj
```

`cls` важен для наследования: если подкласс вызовет `bump()` или `from_start()`, `cls` будет ссылаться на подкласс, и фабрика создаст объект правильного типа. Статикметод же ничего о классе не знает:

```python
class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b
```

`MathUtils.add(2, 3)` вернёт `5`; внутри нет доступа к атрибутам класса, потому что ни `cls`, ни `self` не передаются.

## Подводные камни

- Если в статикметоде вдруг понадобился класс или экземпляр — выбран не тот декоратор; нужен `@classmethod` или обычный метод.
- Классметод-фабрика должна использовать `cls()`, а не имя класса напрямую (`Counter()`), иначе в подклассе фабрика вернёт объект родителя, а не наследника.
- Статикметод — это не «метод быстрее обычного»; разница не в скорости, а в том, какие аргументы привязываются. Выбор диктуется тем, нужно ли методу состояние.
