# Abstract base classes: how to pin down a mandatory interface

## In short

An abstract base class (ABC) is a contract: it lists the methods a descendant must implement but does not implement them itself. Trying to create an instance of a class with an unimplemented abstract method fails with an error before the main logic even runs.

## How to answer in an interview

An ABC defines an interface and guarantees that all descendants fulfill it. In Python this is the `abc` module: the class inherits from `ABC`, and mandatory methods are marked with the `@abstractmethod` decorator. Until a subclass implements all such methods, it cannot be instantiated - the error appears at object creation, not somewhere deep in the middle of execution. ABCs are useful in large projects and team work: you can see what a class must implement, and behavior becomes predictable. An alternative for checking "does it have the required methods" is `typing.Protocol` and structural typing.

## How it works

```python
from abc import ABC, abstractmethod


class Animal(ABC):
    @abstractmethod
    def make_sound(self):
        ...


class Dog(Animal):
    def make_sound(self):
        return "Woof!"


class Cat(Animal):
    def make_sound(self):
        return "Meow!"


print(Dog().make_sound())  # Woof!
print(Cat().make_sound())  # Meow!
```

`Animal` declares the abstract `make_sound` and is itself non-instantiable. `Dog` and `Cat` provide their own implementations and work fine. An empty descendant, however, will fail:

```python
class Mute(Animal):
    pass


Mute()  # TypeError: Can't instantiate abstract class Mute
        # with abstract method make_sound
```

## Pitfalls

- Merely inheriting from `ABC` isn't enough: the contract kicks in only via the `@abstractmethod` decorator on specific methods.
- An abstract method may have a body of its own - the descendant must still override it, otherwise the instance can't be created.
- The error occurs at instance creation, not at subclass declaration - a common point of confusion in interviews.
- `@abstractmethod` is combined with `@property`, `@classmethod`, `@staticmethod`, but order matters: `@abstractmethod` goes as the innermost decorator.
- If you only need to check "the object has method X" without a rigid hierarchy, `typing.Protocol` is usually better.
