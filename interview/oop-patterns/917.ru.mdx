# Зачем классу несколько родителей и чем за это приходится платить?

## Суть

Класс в Python волен наследоваться от любого числа родителей сразу. Это собирает поведение из независимых кусочков, но взамен делает поиск метода неочевидным: когда родители пересекаются по предкам, надо строго задать, чья реализация победит. Этот порядок Python считает алгоритмом C3 и называет MRO.

## Что ответить на собесе

Множественное наследование - это перечисление нескольких базовых классов в скобках при объявлении.

- Главный риск - ромбовидная иерархия: два родителя имеют общего предка, и без правил было бы непонятно, какой метод вызвать.
- Python решает это детерминированно через C3-линеаризацию, которая выстраивает линейный список классов (MRO).
- Поиск атрибута идёт по этому списку слева направо, и побеждает первый, у кого нужный метод нашёлся; посмотреть порядок можно через `ClassName.mro()`.

## Как это работает

Родителей перечисляют через запятую. Поиск идёт по MRO, поэтому при конфликте имён выигрывает тот родитель, что стоит в списке раньше.

```python
class Parent1:
    def greet(self):
        print("Hello from Parent1")

class Parent2:
    def greet(self):
        print("Hello from Parent2")

class Child(Parent1, Parent2):
    pass

Child().greet()  # Hello from Parent1
```

Когда у родителей общий предок, возникает та самая ромбовидная схема:

C3 гарантирует два свойства: предок никогда не идёт раньше потомка, а взаимный порядок родителей повторяет порядок их перечисления. Результат можно увидеть напрямую:

```python
print(D.mro())
# [<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]
```

Когда родители не пересекаются, множественное наследование работает как сборка способностей из миксинов:

```python
class Flyable:
    def fly(self):
        print("Flying")

class Swimmable:
    def swim(self):
        print("Swimming")

class Duck(Flyable, Swimmable):
    pass

duck = Duck()
duck.fly()   # Flying
duck.swim()  # Swimming
```

## Подводные камни

- Порядок родителей значим: `class Child(Parent1, Parent2)` и `class Child(Parent2, Parent1)` дадут разный MRO и разный результат при конфликте имён.
- Несогласованная иерархия не скомпилируется вовсе: если C3 не может построить корректную линеаризацию, Python бросит `TypeError` прямо при объявлении класса.
- В кооперативных иерархиях вызывайте предков через `super()`, а не по имени класса. Прямой вызов `Parent.method(self)` ломает порядок MRO и может пропустить часть классов или вызвать их дважды.
- Множественное наследование легко превращается в запутанную сеть зависимостей. Часто чище взять композицию или миксины с узкой ответственностью.
