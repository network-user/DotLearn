# Магические методы: как научить свой класс вести себя как встроенный тип

## Суть

Dunder-методы (от double underscore — двойное подчёркивание по краям имени) — это хуки, которые Python зовёт сам в ответ на синтаксис и встроенные функции: `+`, `==`, `len()`, печать. Переопределяя их, вы встраиваете свой объект в язык, и он начинает вести себя как родной тип.

## Что ответить на собесе

Магические методы — это протокол между вашим классом и интерпретатором: язык сам зовёт их в ответ на синтаксис и встроенные функции.

- Когда я пишу `a + b`, Python зовёт `a.__add__(b)`; `len(x)` под капотом — это `x.__len__()`; `print(x)` обращается к `__str__`.
- Реализовав нужные dunder-методы, я делаю объект совместимым с операторами, контейнерным протоколом, итерацией и форматированием — без специальных API, чисто через привычный синтаксис.
- Это и есть «питоничность»: код с моими объектами читается так же, как со списками и числами.

## Как это работает

`__init__` настраивает объект при создании:

```python
class Dog:
    def __init__(self, name):
        self.name = name

print(Dog("Buddy").name)  # Buddy
```

`__str__` и `__repr__` управляют строковым видом: первый — для человека (`print`), второй — для отладки (`repr`):

```python
class Dog:
    def __init__(self, name):
        self.name = name
    def __str__(self):
        return f"Dog named {self.name}"
    def __repr__(self):
        return f"Dog('{self.name}')"

d = Dog("Buddy")
print(d)        # Dog named Buddy
print(repr(d))  # Dog('Buddy')
```

`__add__` подключает оператор `+`, а `__eq__` — сравнение на равенство:

А `__len__` встраивает объект в `len()` и в проверки на «пустоту»:

```python
class Bag:
    def __init__(self, items):
        self.items = items
    def __len__(self):
        return len(self.items)

print(len(Bag([1, 2, 3, 4])))  # 4
```

## Подводные камни

- `__str__` отсутствует — `print` падает обратно на `__repr__`; поэтому осмысленный `__repr__` стоит писать всегда.
- Определив `__eq__`, вы делаете объект нехешируемым: Python обнуляет `__hash__`, и объект нельзя класть в `set`/ключ `dict`, пока не добавите свой `__hash__`.
- `__add__` должен возвращать новый объект, а не мутировать `self` — иначе ломается ожидание о том, что `+` не имеет побочных эффектов.
- Если у `other` нет нужных атрибутов, операторные dunder-методы упадут; в продакшене стоит вернуть `NotImplemented`, чтобы Python попробовал отражённую операцию.
