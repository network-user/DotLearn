# Зачем нужны структурные паттерны и какие задачи они решают

## Суть

Структурные паттерны отвечают на вопрос, как соединять классы и объекты в крупные конструкции так, чтобы система оставалась гибкой. Они оперируют композицией и связями, а не самим созданием объектов.

## Что ответить на собесе

Это одна из трёх семей паттернов GoF (наряду с порождающими и поведенческими). Структурные описывают сборку объектов в составные структуры: Адаптер мостит несовместимые интерфейсы, Компоновщик даёт работать с деревом объектов как с одним элементом, Фасад прячет сложную подсистему за простым входом. Их главная задача - снизить связанность и упростить поддержку, не меняя поведение отдельных частей.

## Как это работает

Каждый структурный паттерн решает свою задачу композиции.

**Адаптер (Adapter)** оборачивает чужой объект и переводит его вызовы в ожидаемый интерфейс.

```python
class EuropeanSocket:
    def voltage(self):
        return 230

class Adapter:
    def __init__(self, socket):
        self.socket = socket

    def voltage(self):
        return self.socket.voltage()

adapter = Adapter(EuropeanSocket())
print(adapter.voltage())  # 230
```

**Компоновщик (Composite)** позволяет единообразно обращаться к одиночному объекту и к группе.

**Фасад (Facade)** выставляет одну простую точку входа к набору подсистем.

```python
class SubsystemA:
    def operation_a(self):
        return "A"

class SubsystemB:
    def operation_b(self):
        return "B"

class Facade:
    def __init__(self):
        self.a = SubsystemA()
        self.b = SubsystemB()

    def operation(self):
        return f"Facade({self.a.operation_a()}+{self.b.operation_b()})"

print(Facade().operation())  # Facade(A+B)
```

## Подводные камни

- Путают семьи паттернов: Фабрика и Прототип - порождающие, а не структурные; Наблюдатель - поведенческий.
- Адаптер не добавляет нового поведения, он только переводит вызовы. Если хочется именно расширить поведение, это уже Декоратор.
- Фасад не запрещает прямой доступ к подсистемам, он лишь даёт удобный фасадный вход; не делайте из него «божественный объект».
- Компоновщик работает там, где у листа и контейнера общий интерфейс; без этого единообразие теряется.
