# Фабричный метод: как отдать выбор класса подклассам

## Суть

Фабричный метод - порождающий паттерн: суперкласс объявляет интерфейс для создания объекта, а конкретный тип выбирают подклассы. Код, которому нужен объект, не знает его точного класса.

## Что ответить на собесе

Вместо прямого вызова конструктора код обращается к методу-фабрике, который возвращает объект нужного типа. Подклассы переопределяют этот метод и подменяют создаваемый класс. Выгода - слабая связанность и расширяемость: новый тип добавляется отдельным подклассом, а потребитель остаётся нетронутым (привет принципу открытости/закрытости). Важно не путать «фабричный метод» с простой «фабрикой-свитчем»: классический паттерн выносит выбор именно в переопределяемый метод подкласса.

Базовый класс задаёт абстрактный интерфейс продукта, наследники реализуют конкретику.

```python
from abc import ABC, abstractmethod

class Transport(ABC):
    @abstractmethod
    def deliver(self):
        ...

class Car(Transport):
    def deliver(self):
        return "Delivering by car"

class Bike(Transport):
    def deliver(self):
        return "Delivering by bike"
```

Дальше создание объекта прячется за методом. В канонической форме его переопределяют подклассы-создатели.

```python
class Logistics(ABC):
    @abstractmethod
    def create_transport(self) -> Transport:
        ...

    def plan_delivery(self):
        transport = self.create_transport()
        return transport.deliver()

class RoadLogistics(Logistics):
    def create_transport(self):
        return Car()

class BikeLogistics(Logistics):
    def create_transport(self):
        return Bike()

print(RoadLogistics().plan_delivery())   # Delivering by car
print(BikeLogistics().plan_delivery())   # Delivering by bike
```

Метод `plan_delivery` работает с абстрактным `Transport` и не зависит от того, какой именно класс создаст подкласс. Чтобы добавить грузовик, достаточно нового подкласса `Logistics` - существующий код не трогаем.

## Подводные камни

- Не путайте с Абстрактной фабрикой: фабричный метод порождает один продукт, абстрактная фабрика - семейство связанных продуктов.
- Простой `if/elif` в одном методе - это упрощённая «параметризованная фабрика». Классический Фабричный метод выносит решение в переопределяемый метод подкласса.
- Паттерн оправдан, когда точный класс неизвестен до рантайма или система должна расти подклассами. Для двух стабильных типов он может оказаться лишним усложнением.
- Возвращаемые продукты должны иметь общий интерфейс, иначе полиморфизм у потребителя ломается.
