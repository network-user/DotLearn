# Прототип: когда новый объект выгоднее клонировать, чем строить заново

## Суть

Прототип - порождающий паттерн, который делает новый объект копией готового образца вместо построения с нуля. Это окупается, когда инициализация объекта дорогая или громоздкая.

## Что ответить на собесе

Прототип задаёт у объектов метод клонирования (обычно `clone()`), и новые экземпляры получаются копированием существующих.

- Так мы экономим на сложной инициализации и легко плодим объекты с похожей конфигурацией.
- В Python образцом служит модуль `copy`: `copy.copy` делает поверхностную копию, `copy.deepcopy` - глубокую.
- Ключевой нюанс собеса - разница между ними: при поверхностной копии вложенные изменяемые объекты остаются общими.

class Prototype:
    def __init__(self, value, tags):
        self.value = value
        self.tags = tags

    def clone(self):

Здесь `clone` - другой объект (`is` даёт `False`), но при поверхностной копии список `tags` у оригинала и клона один и тот же. Если нужна полная независимость, берут `copy.deepcopy`.

```python
clone.tags.append("b")
print(original.tags)   # ['a', 'b']  - список общий при copy.copy
```

## Подводные камни

- Главная ловушка - поверхностная копия. `copy.copy` дублирует объект, но вложенные изменяемые поля (списки, словари) остаются общими ссылками.
- `copy.deepcopy` решает это, но обходится дороже и может зациклиться на рекурсивных структурах (на это есть memo-словарь).
- `clone` не вызывает `__init__` заново - копируется уже готовое состояние. Если в инициализации есть побочные эффекты, они не повторятся.
- Прототип - порождающий паттерн, а не структурный; на собесе это любят проверять.
