# Метод экземпляра, classmethod и staticmethod: в чём разница

## Суть

Разница - в том, что метод получает первым аргументом. Обычный метод получает `self` (экземпляр), `classmethod` - `cls` (сам класс), а `staticmethod` не получает ничего: это просто функция, живущая в пространстве имён класса.

## Что ответить на собесе

Разница - в том, что метод получает первым аргументом.

- Метод экземпляра работает с конкретным объектом через `self` - читает и меняет его атрибуты.
- `classmethod` через `cls` работает с классом: меняет классовые атрибуты или служит альтернативным конструктором (`from_string`, `from_dict`), причём `cls` корректно подхватывает подкласс при наследовании.
- `staticmethod` ни к экземпляру, ни к классу не привязан - это вспомогательная функция, которую держат рядом с классом, но которая не зависит от его состояния.

## Как это работает

Метод экземпляра - дефолт. Первый параметр `self` даёт доступ к данным объекта.

```python
class Dog:
    def __init__(self, name):
        self.name = name

    def bark(self):
        return f"{self.name} говорит гав!"
```

`classmethod` объявляют декоратором `@classmethod`, первый параметр - `cls`. Типичное применение - альтернативный конструктор:

```python
class Dog:
    species = "Canis lupus familiaris"

    def __init__(self, name):
        self.name = name

    @classmethod
    def stray(cls):
        return cls("Безымянный")

    @classmethod
    def get_species(cls):
        return cls.species
```

`cls` указывает на тот класс, через который позвали метод, поэтому `stray` у подкласса вернёт экземпляр подкласса, а не базового `Dog`.

`staticmethod` не принимает ни `self`, ни `cls`:

```python
class MathOps:
    @staticmethod
    def add(x, y):
        return x + y
```

Когда что брать:

- **метод экземпляра** - для работы с данными конкретного объекта;
- **classmethod** - для операций над классом и для фабричных конструкторов;
- **staticmethod** - для утилит, логически связанных с классом, но не зависящих от его состояния.

## Подводные камни

- Главное преимущество `classmethod` над захардкоженным именем класса - полиморфизм: `cls` подхватывает подкласс, и фабричный метод вернёт правильный тип.
- `staticmethod` от обычной функции вне класса отличается только местом размещения. Если в теле не нужны ни `self`, ни `cls`, выбирайте именно его, а не делайте «пустой» метод экземпляра.
- Забытый декоратор `@classmethod`/`@staticmethod` превращает метод в обычный: первый позиционный аргумент тогда внезапно займёт экземпляр.
- Вызвать метод экземпляра без экземпляра (`Dog.bark()`) нельзя - `self` не подставится; а `classmethod` и `staticmethod` спокойно зовутся от класса.
