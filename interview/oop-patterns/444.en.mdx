# How do you describe your own mini-language and evaluate expressions by its grammar?

## In short

Interpreter represents a language's grammar as a tree of expression objects, where each node has an `interpret` method. Evaluating an expression is a recursive traversal of this tree. It suits small DSLs: formulas, rules, configs.

## How to answer in an interview

Interpreter is a behavioral pattern for the task of "parse and evaluate an expression according to a given grammar." Each grammar rule corresponds to its own expression class with a shared `interpret(context)` interface. Terminal expressions are leaves (a number, a variable), non-terminal ones are composite nodes (addition, multiplication) that recursively call `interpret` on their children. The context carries shared state, for example variable values. An expression tree is assembled from these nodes, and a single `interpret` call on the root launches the recursion downward. I use it when a small custom language is needed; for serious languages I take a ready-made parser, because the number of classes per rule becomes too large.

## How it works

The grammar maps onto a class hierarchy. Terminals return a value directly, non-terminals combine the results of their children.

```python
class Context:
    def __init__(self):
        self.variables = {}

    def set(self, name, value):
        self.variables[name] = value

    def get(self, name):
        return self.variables.get(name, 0)


class Expression:
    def interpret(self, context):
        raise NotImplementedError


class Number(Expression):
    def __init__(self, value):
        self.value = value

    def interpret(self, context):
        return self.value


class Variable(Expression):
    def __init__(self, name):
        self.name = name

    def interpret(self, context):
        return context.get(self.name)


class Add(Expression):
    def __init__(self, left, right):
        self.left = left
        self.right = right

    def interpret(self, context):
        return self.left.interpret(context) + self.right.interpret(context)


context = Context()
context.set('x', 5)

tree = Add(Number(3), Variable('x'))
print(tree.interpret(context))  # 8
```

The tree's nodes are exactly the structure of the expression `3 + x`. The root `Add` asks the left and right child for their values and adds them. Typical uses: config parsers, rule engines, evaluating mathematical formulas, simple scripting languages.

## Pitfalls

- The pattern scales poorly: a separate class per grammar rule, and for a complex language they grow into dozens. In that case a ready-made parser generator or library is better.
- Interpreter only describes evaluating a ready-made tree. Turning the string `"3 + x"` into that tree (parsing/lexing) is a separate task that the pattern does not solve.
- Deep recursion on large expressions can hit Python's stack limit; long chains need an iterative traversal or a higher limit.
- The context easily turns into a general dumping ground of global state - keep in it only what the expressions actually need.
