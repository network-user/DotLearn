# Паттерны, которые управляют взаимодействием объектов (поведенческие)

## Суть

Поведенческие паттерны описывают, как объекты распределяют ответственность и обмениваются сообщениями. Их забота — алгоритмы и связи между компонентами, а не способ создания или компоновки объектов.

## Что ответить на собесе

Это одна из трёх групп паттернов GoF наряду с порождающими и структурными. Порождающие отвечают за создание объектов, структурные — за их компоновку, а поведенческие — за коммуникацию и распределение обязанностей в рантайме. Они снижают связанность: объекты общаются через абстракции, а не напрямую. Из самых ходовых я назову Observer, Strategy и Command.

## Как это работает

Разберу три представителя группы.

**Observer (Наблюдатель)** — объект-субъект рассылает уведомления подписчикам при изменении состояния. Подписчики не опрашивают субъект, а получают push.

```python
class Subject:
    def __init__(self):
        self._observers = []

    def attach(self, observer):
        self._observers.append(observer)

    def notify(self, message):
        for observer in self._observers:
            observer.update(message)


class Observer:
    def update(self, message):
        print(f"Получено: {message}")


subject = Subject()
subject.attach(Observer())
subject.attach(Observer())
subject.notify("Погода: солнечно")
```

**Strategy (Стратегия)** — семейство взаимозаменяемых алгоритмов за единым интерфейсом. Контекст хранит ссылку на стратегию и переключает её на лету.

**Command (Команда)** — действие превращается в объект. Запросы можно ставить в очередь, логировать и откатывать.

```python
class Command:
    def execute(self):
        raise NotImplementedError


class CopyCommand(Command):
    def execute(self):
        print("Копирование")


class Editor:
    def __init__(self):
        self._history = []

    def run(self, command):
        self._history.append(command)
        command.execute()


Editor().run(CopyCommand())
```

## Подводные камни

- **Не путайте группы паттернов.** Singleton и Factory — порождающие, Adapter и Proxy — структурные; поведенческими их называют ошибочно.
- **Observer легко превращается в утечку памяти**: субъект держит ссылки на подписчиков. Не забывайте отписываться (`detach`).
- **Strategy и State структурно похожи**, но различаются намерением: Strategy выбирает алгоритм, State меняет поведение по внутреннему состоянию.
- **Паттерн ради паттерна — антипаттерн.** Behavioral-паттерны оправданы, когда взаимодействие реально сложное; для простого вызова метода они избыточны.
