# Calling base class methods from a subclass: why super() is needed

## In short

So that a child class can call the parent's code, Python uses `super()`. This function returns a proxy to the next class in the inheritance chain and lets you reuse or extend the parent's behavior.

## How to answer in an interview

From a subclass method you reach the parent through `super()`. Most often it is called in `__init__`, so that the parent initializes its part of the state and the subclass adds its own. `super()` does not simply "go to the parent" - it follows the method resolution order (MRO), so it works correctly even with multiple inheritance, visiting each class exactly once.

## How it works

`super()` inside a method gives a proxy object that looks for an attribute not in the current class but in the next one along the MRO. That is why `super().__init__(...)` calls the parent's constructor and `super().speak()` calls the parent's version of an overridden method. The MRO is computed by the C3 algorithm and is available through `Class.__mro__`.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return "звук"

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)
        self.breed = breed

    def speak(self):
        base = super().speak()
        return f"{base} -> Гав!"

dog = Dog("Бадди", "лабрадор")
print(dog.name)
print(dog.speak())
```

Here `Dog.__init__` delegates initializing the name to the parent and stores only its own `breed` field. And `speak` first takes the parent's result through `super().speak()`, then extends it.

## Pitfalls

- `super()` with no arguments only works inside a class method - it relies on the implicit `__class__` and the first argument. Outside a class it won't work that way.
- `super()` follows the MRO, not "the physical parent." With multiple inheritance the next class may not be the one you expect - check `__mro__`.
- If you forget to call `super().__init__()`, the parent's part of the state won't be initialized, and the parent's attributes may be missing.
- In cooperative multiple inheritance, every participant in the chain must call `super()`, otherwise some classes will be skipped.
