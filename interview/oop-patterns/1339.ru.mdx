# Метаклассы: кто и зачем управляет созданием классов?

## Суть

Метакласс - это класс, экземплярами которого являются другие классы. Он перехватывает момент рождения класса и позволяет менять его устройство: добавлять методы и атрибуты, проверять структуру, навязывать единые правила.

## Что ответить на собесе

В Python классы - тоже объекты, и у каждого объекта есть тип. Тип класса и есть его метакласс; по умолчанию это `type`. Когда интерпретатор выполняет `class`, он фактически вызывает метакласс, передавая имя, базы и словарь тела, - и получает готовый класс. Подменив метакласс, я вклиниваюсь в этот процесс: дописываю или проверяю атрибуты, регистрирую класс, обеспечиваю инвариант для целой иерархии. На практике метаклассы нужны редко - чаще хватает декораторов классов или `__init_subclass__`, - но на собеседовании важно показать, что класс создаётся вызовом метакласса.

## Как это работает

Создание класса проходит через `__new__`/`__init__` метакласса. Переопределив `__new__`, можно изменить словарь атрибутов до того, как класс будет собран. В примере метакласс добавляет каждому своему классу метод `hello`:

```python
class HelloMeta(type):
    def __new__(mcs, name, bases, namespace):
        namespace["hello"] = lambda self: f"Hello from {self.__class__.__name__}"
        return super().__new__(mcs, name, bases, namespace)


class Widget(metaclass=HelloMeta):
    pass


obj = Widget()
print(obj.hello())  # Hello from Widget
```

Что здесь происходит по шагам:

1. `HelloMeta` наследуется от `type` - это и делает его метаклассом.
2. `__new__` получает сам метакласс `mcs`, имя будущего класса `name`, кортеж баз `bases` и словарь тела `namespace`.
3. В `namespace` кладётся метод `hello`.
4. `super().__new__` собирает класс уже с этим методом.
5. `Widget` объявлен с `metaclass=HelloMeta`, поэтому получает `hello` автоматически.

## Подводные камни

- Метаклассы наследуются: подкласс по умолчанию использует метакласс родителя, и конфликт метаклассов в множественном наследовании даёт `TypeError`.
- Для большинства задач (валидация полей, регистрация подклассов) проще `__init_subclass__` или декоратор класса - они понятнее и не ломают иерархию.
- Не путайте уровни: `__new__` метакласса управляет созданием класса, а `__new__`/`__init__` обычного класса - созданием экземпляра.
- Метакласс выполняется один раз на объявление класса, а не на каждый экземпляр; логику уровня экземпляра туда класть нельзя.
