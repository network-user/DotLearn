# Как развязать кучу объектов, которые тянут друг к другу прямые связи?

## Суть

Mediator (Посредник) убирает прямые ссылки объектов друг на друга: они общаются через единый центральный объект. Вместо паутины связей "каждый-с-каждым" получается звезда, где центр - посредник.

## Что ответить на собесе

Mediator - поведенческий паттерн, который борется с сильной связанностью.

- Проблема: когда N компонентов общаются напрямую, число связей растёт квадратично, и любое изменение тянет за собой правки во всех соседях.
- Посредник вводит один объект, инкапсулирующий протокол взаимодействия: компоненты больше не знают друг о друге, они знают только посредника и шлют ему уведомления о событиях, а он решает, кого и как дёрнуть в ответ.
- Это снижает связанность и делает логику взаимодействия централизованной и тестируемой.
- Классическая аналогия - диспетчер аэропорта: пилоты не переговариваются между собой, всё идёт через башню.

## Как это работает

Компоненты держат ссылку на посредника и при действии вызывают `notify`. Вся логика "что за чем следует" живёт в одном месте - в конкретном посреднике.

```python
class Mediator:
    def notify(self, sender, event):
        raise NotImplementedError


class ConcreteMediator(Mediator):
    def __init__(self):
        self.c1 = None
        self.c2 = None

    def notify(self, sender, event):
        if event == "A":
            self.c2.do_c()
        elif event == "D":
            self.c1.do_b()
            self.c2.do_c()


class BaseComponent:
    def __init__(self, mediator=None):
        self._mediator = mediator


class Component1(BaseComponent):
    def do_a(self):
        print("C1 does A")
        self._mediator.notify(self, "A")

    def do_b(self):
        print("C1 does B")


class Component2(BaseComponent):
    def do_c(self):
        print("C2 does C")

    def do_d(self):
        print("C2 does D")
        self._mediator.notify(self, "D")


mediator = ConcreteMediator()
mediator.c1 = Component1(mediator)
mediator.c2 = Component2(mediator)

mediator.c1.do_a()
mediator.c2.do_d()
```

Где это естественно: координация виджетов в GUI (изменение чекбокса включает кнопку), чат-серверы (клиенты шлют сообщения серверу, а не друг другу напрямую), системы согласования событий между независимыми модулями.

## Подводные камни

- Посредник склонен превращаться в "божественный объект": вся логика взаимодействия стекается в него и он распухает. Если посредник стал огромным, это сигнал разбить его или пересмотреть границы.
- Паттерн снижает связанность компонентов, но повышает связанность с самим посредником - компоненты бесполезны без него.
- Легко спутать с Observer: Observer - это широковещательная подписка один-ко-многим, а Mediator централизует двунаправленный протокол между конкретными участниками.
- Не вводите посредника там, где связей мало: для двух-трёх объектов прямые вызовы проще и читаются лучше.
