# Как единообразно работать с элементом и группой через Компоновщик

## Суть

Компоновщик выстраивает объекты в дерево «часть-целое» и даёт обращаться с группой так же, как с одиночным элементом. Клиент вызывает один и тот же метод, не разбираясь, перед ним лист или ветка.

## Что ответить на собесе

Компоновщик - структурный паттерн для древовидных иерархий. Я задаю общий интерфейс компонента, от которого наследуются и лист (конечный элемент), и контейнер (хранит другие компоненты). Контейнер реализует операцию, делегируя её детям рекурсивно. За счёт общего интерфейса клиентский код одинаков для одиночного объекта и для целого поддерева - это убирает ветвления «если это группа, то...».

## Как это работает

Три роли: компонент задаёт общий интерфейс, лист реализует поведение без потомков, контейнер хранит детей и пробрасывает вызов вниз по дереву.

class Graphic(ABC):
    @abstractmethod
    def draw(self):
        pass

class Circle(Graphic):
    def draw(self):
        print("Drawing a Circle")

class CompositeGraphic(Graphic):
    def __init__(self):
        self._children = []

    def add(self, graphic):
        self._children.append(graphic)

    def remove(self, graphic):
        self._children.remove(graphic)

    def draw(self):
        for child in self._children:
            child.draw()

circle1 = Circle()
circle2 = Circle()

composite = CompositeGraphic()
composite.add(circle1)
composite.add(circle2)

composite.draw()`} />

`Circle` - лист, `CompositeGraphic` - контейнер. Вызов `draw` работает одинаково для обоих, а контейнер просто рекурсивно обходит детей. Так устроены файловые системы (папка содержит файлы и папки), деревья UI-элементов, орг-структуры.

## Подводные камни

- Лист не умеет хранить детей: вызов `add`/`remove` на нём - ошибка дизайна. Решают либо «прозрачным» вариантом (методы в общем интерфейсе, но у листа кидают исключение), либо «безопасным» (методы только в контейнере, зато теряется полное единообразие).
- Рекурсивный обход глубокого дерева может упереться в лимит рекурсии Python или дать заметную нагрузку - на больших структурах думайте об итеративном обходе.
- Слишком общий интерфейс размывает ответственность: если в него тянут операции, осмысленные только для контейнеров, листья обрастают пустыми заглушками.
- Компоновщик не следит за циклами: если по ошибке добавить контейнер сам в себя, обход уйдёт в бесконечность.
