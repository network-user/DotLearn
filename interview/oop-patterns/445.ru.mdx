# Как превратить вызов действия в объект, чтобы его можно было хранить и отменять?

## Суть

Command (Команда) упаковывает запрос в самостоятельный объект с методом `execute`. Раз действие стало объектом, его можно передавать как аргумент, складывать в очередь, логировать, ставить в историю и отменять. Паттерн разводит того, кто инициирует операцию, и того, кто её выполняет.

## Что ответить на собесе

Command - поведенческий паттерн, который овеществляет запрос: вместо прямого вызова `receiver.do()` я создаю объект-команду, где запечатано всё нужное для действия - ссылка на исполнителя и параметры.

- Роли: клиент создаёт команду, инвокер хранит и запускает её через единый `execute`, получатель (receiver) делает реальную работу.
- Выгоды: развязка инициатора и исполнителя, добавление новых команд без правки старого кода, undo/redo (команда хранит данные для отмены), сборка макрокоманд из простых.
- Классические применения - кнопки и хоткеи в UI, очереди задач, транзакции.

## Как это работает

Все команды реализуют общий интерфейс с `execute` (и при необходимости `undo`). Инвокер работает с ними единообразно, не зная деталей.

Инвокер хранит историю команд, поэтому отмена сводится к снятию последней команды со стека и вызову её `undo`. Применения: undo/redo в редакторах, очереди фоновых задач, действия кнопок и горячих клавиш, транзакционные операции.

## Подводные камни

- Для каждой операции - отдельный класс команды; на простых действиях это лишний бойлерплейт. Не плодите команды там, где хватает прямого вызова.
- `undo` корректен, только если команда сохранила достаточно данных для отката. Часто этого мало без снимка прежнего состояния - тогда Command сочетают с Memento.
- Не путайте с Strategy: Strategy подменяет алгоритм одной операции, Command овеществляет сам запрос ради хранения, очереди и отмены.
- Если команда держит ссылку на тяжёлый объект, длинная история может удерживать память от сборки - ограничивайте глубину истории undo.
