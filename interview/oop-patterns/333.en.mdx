# Types of inheritance in Python: from single to hybrid

## In short

By the shape of the hierarchy, inheritance is split into single, multiple, multilevel, hierarchical, and hybrid. These are simply ways to describe how many parents a class has and how they are arranged into a tree.

## How to answer in an interview

Five schemes are distinguished. **Single** - one parent. **Multiple** - several parents at once for a single class. **Multilevel** - a chain A → B → C, where a descendant inherits from a descendant. **Hierarchical** - one parent with several direct descendants. **Hybrid** - a combination of the previous ones, for example a diamond scheme where two ancestors converge in a common forebear. Python supports multiple inheritance, and the order in which attributes and methods are looked up across this class graph is determined by the MRO (Method Resolution Order), computed by the C3 algorithm.

## How it works

**Single** is the foundation of reuse: a descendant gets the parent's methods.

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def bark(self):
        print("Dog barks")
```

**Multiple** - a class assembles behavior from several sources.

```python
class Flyable:
    def fly(self):
        print("Can fly")

class Swimmable:
    def swim(self):
        print("Can swim")

class Duck(Flyable, Swimmable):
    pass
```

**Multilevel** - a chain where `Dog` pulls methods from both `Mammal` and `Animal`.

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Mammal(Animal):
    def walk(self):
        print("Mammal walks")

class Dog(Mammal):
    def bark(self):
        print("Dog barks")
```

**Hierarchical** - one parent `Animal` with several descendants `Dog` and `Cat`. **Hybrid** - a mix of schemes; below, `Bat` inherits from `Mammal` and `Bird`, which both derive from `Animal` (a diamond):

```python
class Animal:
    def speak(self):
        print("Animal speaks")

class Mammal(Animal):
    def walk(self):
        print("Mammal walks")

class Bird(Animal):
    def fly(self):
        print("Bird flies")

class Bat(Mammal, Bird):
    pass

print([c.__name__ for c in Bat.__mro__])
```

With multiple and hybrid inheritance, it is precisely the MRO that decides whose method gets called when the name coincides across several ancestors.

## Pitfalls

- **The diamond problem.** If a common forebear is reached through two ancestors, a naive lookup would visit it twice. C3 linearization in Python guarantees that each class appears in the MRO exactly once.
- **The order of parents matters.** `class C(A, B)` and `class C(B, A)` produce different MROs and potentially different behavior when methods coincide.
- **`super()` follows the MRO, not "the direct parent".** In a diamond, `super()` may lead into a sibling branch rather than where the class is written. This is exactly cooperative inheritance.
- **An incompatible order of bases** raises `TypeError: Cannot create a consistent method resolution order` already at the class-definition stage.
- **Hybrid inheritance easily turns into a mess.** It is often cleaner to replace it with composition or mixins with narrow responsibility.
