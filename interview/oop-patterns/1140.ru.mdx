# Где живёт переменная: на классе или на объекте, и почему это важно?

## Суть

Атрибут класса хранится в самом классе и общий для всех его экземпляров. Атрибут экземпляра хранится в конкретном объекте, и у каждого объекта он свой. Разница между ними определяет, увидят ли все объекты одно изменение или только один.

## Что ответить на собесе

Атрибут класса объявляется в теле класса вне методов и существует в единственном экземпляре - его разделяют все объекты. Атрибут экземпляра обычно создаётся в `__init__` через `self.attr` и принадлежит конкретному объекту. Ключевой момент - порядок поиска при чтении: Python сначала смотрит в `__dict__` объекта, и только если там пусто - в классе и его MRO. Поэтому присваивание `obj.attr = ...` не меняет атрибут класса, а создаёт новый атрибут на объекте, который его "затеняет".

Атрибут класса виден всем экземплярам, и правка на уровне класса отражается на каждом:

```python
class Car:
    wheels = 4  # атрибут класса

car1 = Car()
car2 = Car()
print(car1.wheels, car2.wheels)  # 4 4

Car.wheels = 6
print(car1.wheels, car2.wheels)  # 6 6
```

Атрибут экземпляра уникален для объекта:

```python
class Car:
    def __init__(self, color):
        self.color = color  # атрибут экземпляра

car1 = Car("red")
car2 = Car("blue")
print(car1.color, car2.color)  # red blue

car1.color = "green"
print(car1.color, car2.color)  # green blue
```

Когда выбирать что:

- **Атрибут класса** - для общего на всех: констант, настроек, счётчика созданных объектов.
- **Атрибут экземпляра** - для индивидуального состояния каждого объекта.

## Подводные камни

- Присваивание через объект не трогает класс. `car1.wheels = 6` создаёт атрибут экземпляра, который заслоняет классовый. У `car2.wheels` останется старое значение.
- Изменяемый атрибут класса - частая ловушка. Если на классе лежит `items = []` и метод делает `self.items.append(x)`, все объекты пишут в один и тот же список, потому что присваивания не было, изменился общий объект. Списки и словари инициализируйте в `__init__`.
- Чтение и запись несимметричны. `obj.attr` может прочитаться из класса, но `obj.attr = ...` всегда пишет в экземпляр.
- Чтобы намеренно поменять атрибут класса, обращайтесь через класс: `Car.wheels = 6`, а не через экземпляр.
