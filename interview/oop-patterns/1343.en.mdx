# classmethod and staticmethod: who gets cls, and who gets nothing

## In short

`@classmethod` receives the class itself (`cls`) as its first argument and works with class-level state, including acting as a factory for new instances. `@staticmethod` receives neither the class nor an instance — it's just a function living in the class namespace.

## How to answer in an interview

Both decorators let you call a method without creating an instance, but they differ in what they receive. A classmethod is bound to the class: through `cls` it reads and modifies class attributes, and it's handy for alternative constructors (`from_string`, `from_dict`). A staticmethod is bound to nothing — it has neither `self` nor `cls`, so it fits helper logic that makes sense to keep next to the class. A regular method, by contrast, receives `self`, that is, a concrete instance.

## How it works

A classmethod sees the class and can therefore modify state shared across all instances:

```python
class Counter:
    total = 0

    @classmethod
    def bump(cls):
        cls.total += 1
        return cls.total

    @classmethod
    def from_start(cls, start):
        obj = cls()
        cls.total = start
        return obj
```

`cls` matters for inheritance: if a subclass calls `bump()` or `from_start()`, `cls` refers to the subclass, and the factory creates an object of the right type. A staticmethod, on the other hand, knows nothing about the class:

```python
class MathUtils:
    @staticmethod
    def add(a, b):
        return a + b
```

`MathUtils.add(2, 3)` returns `5`; inside it there's no access to class attributes, because neither `cls` nor `self` is passed.

## Pitfalls

- If a staticmethod suddenly needs the class or an instance, you picked the wrong decorator; you need `@classmethod` or a regular method.
- A classmethod factory must use `cls()`, not the class name directly (`Counter()`), otherwise in a subclass the factory returns the parent's object rather than the subclass's.
- A staticmethod is not "a faster method"; the difference isn't speed but which arguments get bound. The choice is driven by whether the method needs state.
