# Фабрика семейств объектов: как устроена Abstract Factory

## Суть

Abstract Factory - порождающий паттерн, который прячет создание целого семейства связанных объектов за общим интерфейсом. Клиент получает готовые продукты, не зная, какие именно конкретные классы стоят за фабрикой.

## Что ответить на собесе

Абстрактная фабрика задаёт интерфейс для создания группы взаимосвязанных продуктов так, чтобы клиентский код не зависел от конкретных классов. Каждая конкретная фабрика выпускает свой согласованный набор продуктов (например, всё под Windows или всё под macOS), а клиент работает только через абстракции. Это даёт лёгкую подмену целого семейства и защищает от смешивания несовместимых элементов.

Паттерн собирается из четырёх ролей. Абстрактные продукты задают интерфейсы (`Button`, `Checkbox`). Конкретные продукты реализуют их под платформу. Абстрактная фабрика объявляет методы создания каждого продукта, а конкретные фабрики возвращают согласованный набор. Клиент держит ссылку на абстрактную фабрику и не видит реализаций.

Паттерн уместен, когда система не должна зависеть от способа создания продуктов, когда её настраивают одним из нескольких семейств и когда важно гарантировать совместность объектов между собой.

<PyDemo title="Abstract Factory: согласованные семейства" autoRun code={`from abc import ABC, abstractmethod

class Button(ABC):
@abstractmethod
def paint(self) -> str: ...

class Checkbox(ABC):
@abstractmethod
def paint(self) -> str: ...

class WindowsButton(Button):
def paint(self) -> str:
return "Windows button"

class WindowsCheckbox(Checkbox):
def paint(self) -> str:
return "Windows checkbox"

class MacOSButton(Button):
def paint(self) -> str:
return "macOS button"

class MacOSCheckbox(Checkbox):
def paint(self) -> str:
return "macOS checkbox"

class GUIFactory(ABC):
@abstractmethod
def create_button(self) -> Button: ...

    @abstractmethod
    def create_checkbox(self) -> Checkbox: ...

class WindowsFactory(GUIFactory):
def create_button(self) -> Button:
return WindowsButton()

    def create_checkbox(self) -> Checkbox:
        return WindowsCheckbox()

class MacOSFactory(GUIFactory):
def create_button(self) -> Button:
return MacOSButton()

    def create_checkbox(self) -> Checkbox:
        return MacOSCheckbox()

def render(factory: GUIFactory) -> None:
print(factory.create_button().paint())
print(factory.create_checkbox().paint())

render(WindowsFactory())
render(MacOSFactory())`} />

## Подводные камни

- Не путайте с Factory Method: тот создаёт один продукт, а Abstract Factory - целое семейство согласованных продуктов.
- Добавить новый продукт в семейство дорого: придётся менять интерфейс фабрики и все конкретные фабрики. Паттерн хорошо расширяется новыми семействами, но плохо - новыми видами продуктов.
- Смысл паттерна теряется, если конкретные фабрики смешивают продукты разных семейств (например, виндовая кнопка с маковским чекбоксом) - именно от этого паттерн и защищает.
- Для одного-двух продуктов паттерн часто избыточен и только утяжеляет код.
