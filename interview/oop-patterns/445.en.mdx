# How do you turn an action call into an object so it can be stored and undone?

## In short

Command wraps a request into a standalone object with an `execute` method. Once an action becomes an object, you can pass it as an argument, queue it, log it, put it in a history, and undo it. The pattern separates the one who initiates an operation from the one who performs it.

## How to answer in an interview

Command is a behavioral pattern that reifies a request. Instead of calling `receiver.do()` directly, I create a command object that seals everything needed for the action: a reference to the executor and the parameters. Roles emerge: the client creates the command, the invoker stores and runs it through a unified `execute`, the receiver does the real work. This gives four benefits: decoupling the initiator from the executor, easily adding new commands without touching old code, undo/redo (the command stores the data for undoing), and assembling macro-commands from simple ones. Classic uses are buttons and hotkeys in a UI, task queues, transactions.

## How it works

All commands implement a common interface with `execute` (and `undo` if needed). The invoker works with them uniformly, without knowing the details.

```python
class Command:
    def execute(self):
        raise NotImplementedError

    def undo(self):
        raise NotImplementedError


class Editor:
    def __init__(self):
        self.text = ""

    def add(self, s):
        self.text += s

    def remove(self, n):
        self.text = self.text[:-n]


class AppendCommand(Command):
    def __init__(self, editor, payload):
        self.editor = editor
        self.payload = payload

    def execute(self):
        self.editor.add(self.payload)

    def undo(self):
        self.editor.remove(len(self.payload))


class Invoker:
    def __init__(self):
        self.history = []

    def run(self, command):
        command.execute()
        self.history.append(command)

    def undo_last(self):
        if self.history:
            self.history.pop().undo()


editor = Editor()
invoker = Invoker()

invoker.run(AppendCommand(editor, "Hello"))
invoker.run(AppendCommand(editor, " World"))
print(editor.text)   # Hello World
invoker.undo_last()
print(editor.text)   # Hello
```

The invoker keeps a history of commands, so undoing reduces to popping the last command off the stack and calling its `undo`. Uses: undo/redo in editors, background task queues, button and hotkey actions, transactional operations.

## Pitfalls

- A separate command class for each operation; on simple actions this is needless boilerplate. Do not breed commands where a direct call is enough.
- `undo` is correct only if the command saved enough data for the rollback. Often that is not enough without a snapshot of the previous state - then Command is combined with Memento.
- Do not confuse it with Strategy: Strategy swaps the algorithm of a single operation, Command reifies the request itself for storage, queuing, and undoing.
- If a command holds a reference to a heavy object, a long history can keep memory from being collected - limit the depth of the undo history.
