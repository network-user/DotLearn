# Metaclass in Python: who creates the class itself, and how

## In short

A metaclass is the "class of a class": a type that produces other classes at the moment they are declared. A regular class makes instances; a metaclass makes the classes themselves and can intervene in their assembly.

## How to answer in an interview

A metaclass defines how a class is created. By default all classes are created by the type `type`, and `type` is its own metaclass. You write your own metaclass by inheriting from `type` and overriding `__new__` or `__init__` to intercept class creation: validate the structure, add methods, register the class somewhere. It should be used rarely: more often the task is solved by class decorators or `__init_subclass__`. Real-world examples are building models in Django and ORMs, and plugin registration.

## How it works

When `class Foo(...)` runs, Python gathers the name, the tuple of bases and the body's dictionary, then calls the metaclass roughly as `Meta(name, bases, namespace)`. By intercepting `__new__`, you hold this dictionary in your hands before the class is finally assembled.

```python
class MyMeta(type):
    def __new__(mcs, name, bases, namespace):
        namespace["hello"] = lambda self: f"Hello from {type(self).__name__}"
        return super().__new__(mcs, name, bases, namespace)


class MyClass(metaclass=MyMeta):
    pass


obj = MyClass()
print(obj.hello())  # Hello from MyClass
```

Here the metaclass writes the `hello` method into the namespace of `MyClass` before assembling it, so it appears on every instance without being declared by hand.

## Pitfalls

- Metaclass conflict: if the bases have different, incompatible metaclasses, Python raises `TypeError`. A metaclass is inherited, so it must be a subtype of the metaclasses of all parents.
- `__new__` returns the already-built class object, while the metaclass's `__init__` only fine-tunes it - don't mix up where each belongs.
- A metaclass is a heavy tool. For validating or registering subclasses, `__init_subclass__` or a class decorator is almost always enough and reads more simply.
- In the example, `type(self).__name__` matters, not `self.__class__.__name__` reached directly through the metaclass: the name is taken from the instance's class, otherwise subclasses would get someone else's name.
