# Декоратор как способ нарастить поведение объекта без правки его кода

## Суть

Декоратор (он же Wrapper) - структурный паттерн: он навешивает на объект новое поведение во время выполнения, оборачивая его в другой объект-обёртку. Исходный код при этом остаётся нетронутым.

## Что ответить на собесе

Декоратор - структурный паттерн, который динамически добавляет объекту обязанности, не меняя его класс и не плодя подклассы.

- Объект кладут внутрь обёртки с тем же интерфейсом, и обёртка перехватывает вызовы.
- Обёртка делает что-то своё до и после, а нужную работу делегирует исходному объекту.
- В Python это чаще всего реализуют через декораторы функций с синтаксисом `@` и применяют для логирования, кэширования, проверки прав и подобных сквозных задач.

## Как это работает

Декоратор функции в Python - это вызываемый объект, который принимает функцию и возвращает новую функцию. Новая обычно вызывает исходную, дописывая поведение вокруг неё.

```python
def my_decorator(func):
    def wrapper():
        print("Действия до вызова")
        func()
        print("Действия после вызова")
    return wrapper

def say_hello():
    print("Привет!")

decorated = my_decorator(say_hello)
decorated()
```

Результат:

```
Действия до вызова
Привет!
Действия после вызова
```

Запись через `@` - синтаксический сахар: `@my_decorator` над `say_hello` означает ровно `say_hello = my_decorator(say_hello)`.

```python
@my_decorator
def say_hello():
    print("Привет!")

say_hello()
```

Чтобы обёртка пропускала любые аргументы насквозь, ей дают `*args` и `**kwargs` и возвращают результат исходного вызова:

## Подводные камни

- Без `return result` в обёртке декорированная функция начнёт молча возвращать `None` - частая и неприятная ошибка.
- Обёртка подменяет метаданные: `add.__name__` станет `wrapper`. Чтобы их сохранить, оборачивайте `wrapper` в `functools.wraps(func)`.
- Если забыть `*args, **kwargs`, декоратор сломает любую функцию с аргументами.
- Декоратор-функция и паттерн «Декоратор» из GoF - родственные, но не одно и то же: классический паттерн оборачивает объекты, а не функции, и сохраняет общий интерфейс.
