# Что даёт декоратор @dataclass и от какого шаблонного кода он избавляет

## Суть

`@dataclass` смотрит на аннотации полей класса и сам пишет за вас `__init__`, `__repr__` и `__eq__`. Класс-контейнер для данных получается без ручного шаблонного кода.

## Что ответить на собесе

`@dataclass` из модуля `dataclasses` (с Python 3.7) автоматически генерирует служебные методы по объявленным полям с аннотациями типов.

- Вместо ручного конструктора, сравнения и строкового представления я просто перечисляю поля с аннотациями.
- Декоратор сам пишет `__init__`, `__repr__` и `__eq__`.
- По флагам можно дополнительно включить сортировку (`order=True`), неизменяемость (`frozen=True`) и другие методы.

## Как это работает

Объявляем поля с аннотациями — методы появляются сами:

```python
from dataclasses import dataclass

@dataclass
class Point:
    x: int
    y: int
```

`__init__` теперь принимает `x` и `y`, `__repr__` даёт читаемую строку, а `__eq__` сравнивает по значениям полей:

```python
p1 = Point(1, 2)
p2 = Point(1, 2)
p3 = Point(3, 4)

print(p1)          # Point(x=1, y=2)
print(p1 == p2)    # True  — поля совпадают
print(p1 == p3)    # False
```

Значения по умолчанию задаются прямо в аннотации, а для изменяемых значений используют `field(default_factory=...)`, иначе все объекты разделили бы один и тот же список:

```python
from dataclasses import dataclass, field

@dataclass
class Cart:
    items: list = field(default_factory=list)
```

## Подводные камни

- Изменяемое значение по умолчанию (`items: list = []`) запрещено — Python бросит ошибку; нужен `field(default_factory=list)`.
- По умолчанию объект изменяемый и не хешируемый; чтобы класть его в `set` или использовать как ключ `dict`, добавьте `frozen=True`.
- Поля с дефолтами должны идти после полей без дефолтов, иначе будет ошибка определения класса.
- `@dataclass` не делает глубокого копирования и не валидирует типы: аннотации — это подсказки, а не проверки во время выполнения.
