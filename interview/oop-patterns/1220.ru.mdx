# Метакласс в Python: кто и как создаёт сам класс

## Суть

Метакласс — это «класс класса»: тип, который порождает другие классы в момент их объявления. Обычный класс делает экземпляры, метакласс делает сами классы и может вмешаться в их сборку.

## Что ответить на собесе

Метакласс задаёт, как создаётся класс. По умолчанию все классы создаются типом `type`, и `type` сам является своим метаклассом.

- Свой метакласс наследуют от `type` и переопределяют `__new__` или `__init__`, чтобы перехватить создание класса: проверить структуру, дописать методы, зарегистрировать класс где-то.
- Применять стоит редко: чаще задачу решают декораторы классов или `__init_subclass__`.
- Реальные примеры — построение моделей в Django и ORM, регистрация плагинов.

## Как это работает

При выполнении `class Foo(...)` Python собирает имя, кортеж баз и словарь тела, после чего зовёт метакласс примерно как `Meta(name, bases, namespace)`. Перехватив `__new__`, вы держите в руках этот словарь до того, как класс окончательно собран.

```python
class MyMeta(type):
    def __new__(mcs, name, bases, namespace):
        namespace["hello"] = lambda self: f"Hello from {type(self).__name__}"
        return super().__new__(mcs, name, bases, namespace)


class MyClass(metaclass=MyMeta):
    pass


obj = MyClass()
print(obj.hello())  # Hello from MyClass
```

Здесь метакласс перед сборкой `MyClass` вписывает в его пространство имён метод `hello`, поэтому он появляется у каждого экземпляра без ручного объявления.

## Подводные камни

- Конфликт метаклассов: если базы имеют разные несовместимые метаклассы, Python бросит `TypeError`. Метакласс наследуется, так что он должен быть подтипом метаклассов всех родителей.
- `__new__` возвращает уже готовый объект класса, а `__init__` метакласса лишь донастраивает его — путать их места нельзя.
- Метакласс — тяжёлый инструмент. Для проверки или регистрации подклассов почти всегда хватает `__init_subclass__` или декоратора класса, которые читаются проще.
- В примере важно `type(self).__name__`, а не `self.__class__.__name__` напрямую через метакласс: имя берётся у класса экземпляра, иначе для подклассов получите чужое имя.
