# Может ли шаблонный метод стать виртуальным?

## Суть

Функция-шаблон с собственными параметрами шаблона не может быть объявлена `virtual`: vtable требует одну фиксированную сигнатуру на слот, а шаблон порождает множество перегрузок при подстановке типов. Виртуальный метод внутри шаблонного класса допустим, если он не является шаблоном сам по себе.

## Что ответить на собесе

- Уточняю формулировку: речь о `template <typename U> virtual void foo(U)` в одном классе, а не о методе шаблонного класса `Base<T>`.
- Отвечаю: такой член нельзя сделать виртуальным, компилятор отклоняет комбинацию `virtual` и `template` на одной функции.
- Показываю рабочий вариант: `template <typename T> struct Base { virtual void handle(const T&); }` без дополнительных template-параметров у метода.
- Объясняю, что полиморфизм здесь ограничен одним инстанцированием `Base<int>`, а не всеми типами сразу.
- Честно добавляю: путают «шаблонный класс» и «шаблонный метод»; на собесе важно развести эти случаи явно.

## Как это работает

Компилятор строит vtable при определении класса. Виртуальный слот указывает на одну функцию с конкретной сигнатурой. Шаблонная функция-член с параметром `U` генерирует отдельную функцию для каждого `U`, и единый полиморфный слот для всего семейства невозможен. Поэтому стандарт запрещает `virtual template`.

```cpp
#include <iostream>

template <typename Item>
struct Handler {
    virtual ~Handler() = default;
    virtual void dispatch(const Item& value) {
        std::cout << "base: " << value << '\n';
    }
};

template <typename Item>
struct LoudHandler : Handler<Item> {
    void dispatch(const Item& value) override {
        std::cout << "loud: " << value << "!\n";
    }
};

int main() {
    Handler<int>* ptr = new LoudHandler<int>();
    ptr->dispatch(42);
    delete ptr;
}
```

Метод `dispatch` виртуален, но тип `Item` зафиксирован параметром класса, а не отдельным шаблоном метода. Полиморфный вызов через `Handler<int>*` работает среди наследников `Handler<int>`. Если бы `dispatch` сам был шаблоном по второму типу, компилятор не смог бы выбрать один указатель в vtable.

Для разных типов в одном runtime-интерфейсе используют type erasure, `std::variant` или отдельные иерархии на каждый инстанцированный класс.

## Подводные камни

- **Путаница с примером из учебников.** `virtual void process(T)` в `template <typename T> class Base` не делает метод шаблоном.
- **Попытка обойти через макросы.** Генерация множества виртуальных методов не даёт настоящего шаблонного полиморфизма.
- **Чисто виртуальный шаблонный метод.** Даже без `template` у метода `= 0` в шаблонном классе требует осторожности с инстанцированием.
- **Смешение инстансов.** `Handler<int>` и `Handler<double>` не полиморфны друг относительно друга.
- **Ожидание CRTP.** Статический полиморфизм решает другую задачу, не заменяя runtime vtable.
