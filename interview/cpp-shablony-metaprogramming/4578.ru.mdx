# Что такое SFINAE и зачем оно в шаблонах?

## Суть

SFINAE (Substitution Failure Is Not An Error) означает: если при подстановке аргументов в шаблон возникает ошибка в сигнатуре или возвращаемом типе, кандидат просто отбрасывается, а не ломает компиляцию. Это основа условной перегрузки и compile-time ветвления до появления `constexpr if` и concepts.

## Что ответить на собесе

- Даю расшифровку: неудачная подстановка в шаблон не считается ошибкой, если кандидат отсеивается на этапе overload resolution.
- Показываю классический приём `decltype` + ellipsis fallback для проверки наличия метода у типа.
- Упоминаю `std::enable_if`, `void_t` и современную замену через C++20 concepts.
- Объясняю границу: SFINAE работает в immediate context сигнатуры, а не в теле функции.
- Честно добавляю: в новом коде предпочитаю concepts, но SFINAE всё ещё встречается в legacy и стандартной библиотеке.

## Как это работает

При вызове шаблонной функции компилятор перебирает кандидатов. Для каждого подставляет фактические типы. Если подстановка делает сигнатуру невалидной (например, `decltype(x.missing())` не существует), этот шаблон исключается из рассмотрения. Остаётся следующая перегрузка или срабатывает ошибка «нет подходящей функции».

```cpp
#include <iostream>

template <typename T>
auto invoke_hook(T& obj) -> decltype(obj.on_event(), void()) {
    obj.on_event();
}

template <typename T>
void invoke_hook(...) {
    std::cout << "no on_event hook\n";
}

struct WithHook {
    void on_event() { std::cout << "event fired\n"; }
};

struct Plain {};

int main() {
    WithHook wired;
    Plain bare;
    invoke_hook(wired);
    invoke_hook(bare);
}
```

Первая перегрузка участвует только если у `T` есть вызываемый `on_event()`. Для `Plain` подстановка в `decltype` проваливается, кандидат снимается по SFINAE, выбирается вариадическая заглушка. В C++11+ тот же смысл выражают через `std::void_t<decltype(std::declval<T>().on_event())>` в trait-классах.

В теле функции ошибка типа уже не SFINAE: компиляция остановится. Поэтому проверки выносят в return type, параметры по умолчанию или `enable_if_t` в списке template-параметров.

## Подводные камни

- **Ошибка в теле, не в сигнатуре.** SFINAE не спасёт, если невалидный код внутри `{ }`.
- **Неоднозначность перегрузок.** Два кандидата могут одновременно подойти и дать hard error.
- **Читаемость.** Цепочки `enable_if` быстро становятся непрозрачными.
- **C++20 concepts.** `requires` выражает те же ограничения явнее; знать SFINAE всё равно нужно для чтения старого кода.
- **Зависимые имена.** В шаблонах `typename` и контекст подстановки влияют на то, сработает ли отсев кандидата.
