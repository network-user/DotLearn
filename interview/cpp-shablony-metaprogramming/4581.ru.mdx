# Зачем в C++ нужны шаблоны?

## Суть

Шаблоны параметризуют функции и классы по типам и значениям, позволяя писать один обобщённый код вместо копий под каждый тип. Проверка типов и генерация реализаций происходят на этапе компиляции, без потери производительности относительно ручных дубликатов.

## Что ответить на собесе

- Даю определение: механизм compile-time параметризации с подстановкой аргументов при использовании.
- Называю выгоды: меньше дублирования, статическая типобезопасность, возможность метапрограммирования.
- Показываю шаблон функции и шаблон класса на одном примере контейнера.
- Сравниваю с макросами: шаблоны проходят проверку типов, макросы нет.
- Честно добавляю: ошибки в шаблонах часто длинные; в горячих API иногда оставляют нетипизированные C-интерфейсы ради стабильности ABI.

## Как это работает

Параметры шаблона объявляют «дыры» в сигнатуре. При вызове `sort(v)` компилятор выводит тип элементов вектора и подставляет его в определение `sort<T>`. Для класса `Buffer<T>` создание `Buffer<std::string>` генерирует отдельный тип с методами, работающими со `std::string`.

```cpp
#include <iostream>
#include <vector>

template <typename Elem>
class RingBuffer {
    std::vector<Elem> slots;
    std::size_t head = 0;

public:
    explicit RingBuffer(std::size_t cap) : slots(cap) {}

    void push(const Elem& item) {
        slots[head] = item;
        head = (head + 1) % slots.size();
    }

    const Elem& peek() const { return slots[head]; }
};

template <typename Elem>
Elem pick_larger(const Elem& lhs, const Elem& rhs) {
    return (lhs > rhs) ? lhs : rhs;
}

int main() {
    RingBuffer<int> ints(4);
    ints.push(10);
    std::cout << pick_larger(3, 7) << ' ' << ints.peek() << '\n';
}
```

Одна реализация `pick_larger` обслуживает любой тип с оператором `>`. `RingBuffer` переиспользует логику кольца для `int`, `double` или пользовательских структур без изменения исходника. Стандартная библиотека построена на этом принципе: `std::vector`, алгоритмы, умные указатели.

Шаблоны также поддерживают нетиповые параметры (`template <int N>`, `template <typename T, std::size_t N>`), что лежит в основе `std::array` и многих оптимизаций.

## Подводные камни

- **Диагностика ошибок.** Сообщение указывает на внутренности шаблона, а не на строку вызова.
- **Размер бинарника.** Каждое инстанцирование добавляет код.
- **Заголовочная модель.** Определения шаблонов обычно в `.h`, иначе линковка ломается.
- **ABI стабильность.** Шаблоны в публичном API меняют layout при смене аргументов; для границ DLL часто избегают.
- **Смешение с виртуальностью.** Шаблоны дают статический полиморфизм, vtable даёт динамический; выбор влияет на дизайн.
