# How does partial template specialization work?

## In short

Partial specialization lets you supply an alternate class template implementation when only some template arguments are known and the rest stay parameters. Function templates have no direct equivalent; overloads and SFINAE fill that role.

## How to answer in an interview

- I separate full specialization (`vector<bool>` as its own type) from partial (`pair<T, T>`, `vector<T*>`).
- I stress partial specialization applies only to class templates, not function templates.
- I show an example fixing the first parameter while leaving the second free.
- I cite standard cases: `std::unique_ptr<T[]>`, `std::tuple<T&>`.
- I add honestly: for similar goals with functions I write overloads or concepts instead of hunting for "partial function specialization".

## How it works

The primary template states the general case. A partial specialization narrows the argument pattern: the compiler picks the most specific match at instantiation. When `Storage<int, U>` fits, it wins over `Storage<T1, T2>` for `Storage<int, double>`.

```cpp
#include <iostream>

template <typename A, typename B>
struct PairBox {
    void describe() const {
        std::cout << "generic pair box\n";
    }
};

template <typename T>
struct PairBox<T, T> {
    void describe() const {
        std::cout << "homogeneous pair box\n";
    }
};

template <>
struct PairBox<int, int> {
    void describe() const {
        std::cout << "both ints, full specialization\n";
    }
};

int main() {
    PairBox<char, double> mixed;
    PairBox<float, float> same;
    PairBox<int, int> ints;
    mixed.describe();
    same.describe();
    ints.describe();
}
```

`PairBox<float, float>` lands in the homogeneous partial specialization. `PairBox<int, int>` takes the full specialization as the tighter match. `PairBox<char, double>` stays on the primary template.

Declaration order matters: primary, then partial, then full. A specialization does not inherit members from the primary; you rewrite the body.

## Pitfalls

- **Partial function specialization.** The language forbids it; use overloads.
- **Code duplication.** Each specialization carries a full implementation.
- **Ambiguous selection.** Two partial specializations may match at once.
- **Dependent types in the pattern.** Errors in `<typename T> Foo<std::vector<T>>` surface at instantiation.
- **Primary template confusion.** You cannot partially specialize the same parameter list as the primary without narrowing the pattern.
