# Как устроена частичная специализация шаблона?

## Суть

Частичная специализация позволяет задать альтернативную реализацию шаблона класса, когда известна только часть template-аргументов, остальные остаются параметрами. Для функций полной аналогии нет: вместо этого используют перегрузки и SFINAE.

## Что ответить на собесе

- Развожу полную специализацию (`vector<bool>` по сути отдельный тип) и частичную (`pair<T, T>`, `vector<T*>`).
- Подчёркиваю: частичная специализация только у class templates, не у function templates.
- Показываю пример с фиксацией первого параметра и свободным вторым.
- Упоминаю стандартные примеры: `std::unique_ptr<T[]>`, `std::tuple<T&>`.
- Честно добавляю: для функций с похожей целью пишу несколько перегрузок или concepts, а не ищу «частичную специализацию функции».

## Как это работает

Первичный шаблон объявляет общий случай. Частичная специализация сужает паттерн аргументов: компилятор выбирает наиболее специфичную версию при инстанцировании. Если подходит `Storage<int, U>`, она побеждает над `Storage<T1, T2>` для `Storage<int, double>`.

```cpp
#include <iostream>

template <typename A, typename B>
struct PairBox {
    void describe() const {
        std::cout << "generic pair box\n";
    }
};

template <typename T>
struct PairBox<T, T> {
    void describe() const {
        std::cout << "homogeneous pair box\n";
    }
};

template <>
struct PairBox<int, int> {
    void describe() const {
        std::cout << "both ints, full specialization\n";
    }
};

int main() {
    PairBox<char, double> mixed;
    PairBox<float, float> same;
    PairBox<int, int> ints;
    mixed.describe();
    same.describe();
    ints.describe();
}
```

`PairBox<float, float>` попадает в частичную специализацию с одинаковыми типами. `PairBox<int, int>` выбирает полную специализацию как более узкую. `PairBox<char, double>` остаётся на первичном шаблоне.

Порядок объявлений важен: первичный шаблон, затем частичные, затем полные. Специализация не наследует автоматически члены первичного шаблона: тело задают заново.

## Подводные камни

- **Частичная специализация функции.** Язык не позволяет; только перегрузки.
- **Дублирование кода.** Каждая специализация несёт полную реализацию.
- **Неоднозначный выбор.** Две частичные специализации могут подойти одновременно.
- **Зависимые типы в паттерне.** Ошибки в `<typename T> Foo<std::vector<T>>` всплывают при инстанцировании.
- **Путаница с primary template.** Нельзя частично специализировать тот же набор параметров, что у первичного, без сужения паттерна.
