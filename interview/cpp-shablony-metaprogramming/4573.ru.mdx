# Зачем в шаблонах нужно ключевое слово typename?

## Суть

`typename` сообщает компилятору, что зависимое имя внутри шаблона обозначает тип, а не значение или функцию. Без этой пометки парсер не может однозначно разобрать конструкции вроде `T::value_type` до инстанцирования шаблона.

## Что ответить на собесе

- Говорю, что речь о dependent names: имена, смысл которых известен только после подстановки параметра шаблона.
- Показываю типичный случай: `typedef typename T::value_type Item;` внутри обёртки над контейнером.
- Упоминаю второй контекст: `typename` перед именем в шаблоне параметра (`template <typename U>`).
- Подчёркиваю, что для независимых имён (например, `int` или `std::string`) `typename` не нужен.
- Честно добавляю: в C++20 многие случаи упрощаются concepts и `requires`, но на legacy-коде `typename` всё ещё обязателен.

## Как это работает

Когда шаблон ещё не инстанцирован, компилятор не знает, что именно спрятано за `T::something`: вложенный тип, статическое поле или метод. Ключевое слово `typename` снимает двусмысленность на этапе разбора и разрешает использовать имя как тип.

```cpp
template <typename Seq>
class FrontView {
public:
    using Item = typename Seq::value_type;

    explicit FrontView(Seq& seq) : seq_(seq) {}

    Item peek() const {
        return seq_.front();
    }

private:
    Seq& seq_;
};
```

В `using Item = typename Seq::value_type` компилятор трактует `Seq::value_type` как тип. Если убрать `typename`, разбор может завершиться ошибкой «expected a type». Аналогично `typename` требуется в возвращаемых типах и параметрах функций, где фигурируют зависимые имена.

Правило двухфазного поиска (two-phase lookup) делит имена на зависимые и независимые. Независимые проверяются при первом проходе, зависимые откладываются до инстанцирования. `typename` относится именно ко второй группе. В C++11+ `using` и `auto` не отменяют необходимость помечать зависимые типы.

## Подводные камни

- **Забытый typename.** Самая частая ошибка новичков в шаблонных адаптерах STL-подобных интерфейсов.
- **Путаница с template.** Для зависимых шаблонов нужна связка `template` + `typename`, например `T::template nest<U>`.
- **Значение vs тип.** `T::count` может оказаться статическим членом; `typename` здесь неуместен и даст неверный код.
- **typedef в старом коде.** В legacy-проектах встречается `typedef typename ...` вместо `using`; семантика та же, стиль разный.
- **Концепты не везде.** Даже с concepts проверка зависимых имён в теле шаблона остаётся; не стоит считать keywords устаревшими.
