# Mobile app logs: Logcat, Xcode, and remote logging

## In short

Android: USB debugging + Android Studio Logcat or `adb logcat` filtered by package/tag. iOS: Xcode Devices and Simulators console or `Console.app`. QA reproduces bug on device, saves log snippet and build number; prod uses Firebase/Sentry breadcrumbs.

## How to answer in an interview

- Android filter: `adb logcat --pid=$(adb shell pidof com.app)`.
- iOS: run from Xcode, watch debug area; release needs OSLog export.
- Log levels Verbose/Debug/Info/Warn/Error in Logcat colors.
- Charles/mitmproxy supplements network, does not replace native crash log.
- Honestly: without debug build many logs stripped; ask for internal beta with logging flag.

## How it works

```gherkin
Feature: Mobile log capture

  Scenario: Android reproduce payment fail
    Given USB debugging enabled
    When QA clears logcat and taps Pay
    Then Logcat shows tag Checkout error line
    And screenshot plus exported log attached to ticket

  Scenario: iOS filter noise
    Given Xcode console open
    When filter subsystem com.myapp.checkout
    Then only checkout logs visible during repro

  Scenario: Remote crash from TestFlight
    Given Firebase Crashlytics enabled
    When tester reproduces on beta 3.2.0
    Then crash appears with device model and stack
```

Tag convention `MyApp/Network` speeds filter. Clear log before steps reduces noise.

## Pitfalls

- **Log buffer overflow.** Early steps rotated out before save.
- **PII in Log.d.** Card number in debug only on tester device leak.
- **Proguard obfuscates line numbers.** Need mapping file.
- **Simulator vs device difference.** Bug only on real OEM skin.
- **No network permission logs silent fail.** Looks like app hang.
