# Defined types in 1C: composite type without duplication

## In short

A defined type in metadata combines a set of value types under one name. You use it in attributes, parameters, and dimensions instead of a long list of references. It reduces duplication and lets you change the type set in one place.

## How to answer in an interview

- I say a defined type is a named composite type, not a separate primitive with regex validation.
- I use it for fields like counterparty or person, or a base document from several kinds.
- When the type set changes I update one defined type, not dozens of attributes.
- I clarify that allowed-value checks stay at the platform type level and in code.
- I am honest: a defined type does not replace a strict domain model; sometimes separate attributes are better.

## How it works

In the configurator I create a Defined type, e.g. `УчастникСделки`, and include `СправочникСсылка.Контрагенты` and `СправочникСсылка.ФизическиеЛица`. In a document the `Участник` attribute gets type `УчастникСделки`.

Reading and writing a composite value:

```bsl
Процедура ЗаполнитьУчастника(ДокументОбъект, Участник)
    Если ТипЗнч(Участник) = Тип("СправочникСсылка.Контрагенты") Тогда
        ДокументОбъект.Участник = Участник;
    ИначеЕсли ТипЗнч(Участник) = Тип("СправочникСсылка.ФизическиеЛица") Тогда
        ДокументОбъект.Участник = Участник;
    Иначе
        ВызватьИсключение "Недопустимый тип участника";
    КонецЕсли;
КонецПроцедуры
```

In queries a defined type behaves as a union of allowed types; when you extend the set you update configuration and check forms where the field is chosen from a list.

## Pitfalls

**Expecting format validation.** A defined type does not add regex; you need separate code or a constrained attribute.
**Too wide a set.** Extra references clutter the field and confuse users.
**Missed dependent attributes.** After narrowing the set, old data values may become invalid.
**Exchange and extensions.** An extension cannot always change vendor defined types.
**Confusion with standard configs.** Standard configurations already use defined type names; duplicates break comparison on update.
