# Reading an attribute from a reference without full object load

## In short

A reference identifies a database row. For one field you do not have to call `ПолучитьОбъект()`: a query with a reference parameter or the SSL function `ЗначениеРеквизитаОбъекта` is enough. Full load is justified when you change many fields.

## How to answer in an interview

- For a caption in a list I use a query or `ЗначениеРеквизитаОбъекта(Ссылка, "Наименование")`.
- I use `ПолучитьОбъект()` when I need to change attributes and write the object.
- In a query I access fields via a table alias: `Контрагент.ИНН`.
- I handle an empty reference: `ЗначениеЗаполнено(Ссылка)` before read.
- I am honest: `ПолучитьОбъект()` for one field inside a loop over hundreds of rows kills performance.

## How it works

Light read via SSL:

```bsl
Наименование = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(КонтрагентСсылка, "Наименование");
```

Read several fields in one query:

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |    К.Наименование,
    |    К.ИНН
    |ИЗ
    |    Справочник.Контрагенты КАК К
    |ГДЕ
    |    К.Ссылка = &Ссылка";
Запрос.УстановитьПараметр("Ссылка", КонтрагентСсылка);
Выборка = Запрос.Выполнить().Выбрать();
```

Full load for change:

```bsl
ОбъектКонтрагента = КонтрагентСсылка.ПолучитьОбъект();
ОбъектКонтрагента.Наименование = НовоеИмя;
ОбъектКонтрагента.Записать();
```

## Pitfalls

**ПолучитьОбъект in a loop.** Hundreds of DBMS hits instead of one query.
**Read without reference check.** An empty reference errors or returns unpredictably.
**Tabular section attribute.** Cannot read via header reference; need a query to the tabular section.
**Stale data.** Form cache may disagree with the database after a write in another session.
**Bypassing privileges.** `ЗначениеРеквизитаОбъекта` in privileged mode reads what RLS denies the user.
