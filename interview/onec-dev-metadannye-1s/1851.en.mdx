# How to put a new attribute on a 1C object form

## In short

First declare the attribute in catalog or document metadata, then add it to the form attributes and place a control in the form designer. In code you create the field with `Элементы.Добавить` and bind `Данные` to the attribute name.

## How to answer in an interview

- I describe the chain: object metadata, form attribute, layout control.
- I clarify that a form attribute can be computed and not stored on the object.
- I note that in managed forms I usually use the designer, not only code.
- I mention updating the infobase configuration after adding metadata attributes.
- I am honest: in an extension you must be careful with standard forms; sometimes programmatic placement is required.

## How it works

In the configurator I open object properties, create a field on the Attributes tab with type and synonym. In the object form I open the designer, drag the attribute onto the layout, and set visibility, availability, and caption.

Programmatic input field on form open:

```bsl
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если НЕ Элементы.Найти("ДатаСтарта") Тогда
        ПолеДаты = Элементы.Добавить("ДатаСтарта", Тип("ПолеФормы"));
        ПолеДаты.Вид = ВидПоляФормы.ПолеВвода;
        ПолеДаты.ПутьКДанным = "Объект.ДатаСтарта";
        ПолеДаты.Заголовок = "Дата старта";
        ПолеДаты.Родитель = Элементы.ГруппаШапка;
    КонецЕсли;
КонецПроцедуры
```

A form attribute without object storage is declared in the form tree and tied to an expression or an `ПриИзменении` handler. After metadata changes I update the database structure; otherwise the column will not exist in the DBMS tables.

## Pitfalls

**Attribute without a control.** The field exists on the object but the user sees nothing until a form element is added.
**Control without a form attribute.** A `ПутьКДанным` binding to a missing attribute crashes the form on open.
**Skipped DB update.** A new metadata attribute does not create a column until configuration is updated.
**Name clash.** Element and attribute names must be unique within the form.
**Extension of a standard form.** Adding elements to a borrowed form needs compatibility checks when the vendor configuration is updated.
