# Структура и соответствие в 1С: когда что использовать

## Суть

Структура хранит пары ключ-значение со строковыми ключами и доступом через свойство (`Данные.Ключ`). Соответствие принимает ключи любого типа и подходит для lookup по ссылкам, числам, перечислениям. В обоих коллекциях ключи уникальны; повторная вставка заменяет значение.

## Что ответить на собесе

- Я говорю: структура для параметров процедур, DTO, `ЗаполнитьЗначенияСвойств`; соответствие для кэша «ссылка → данные».
- Подчёркиваю: ключ структуры это идентификатор, его нельзя сделать «1С:Ссылка» без соответствия.
- Упоминаю итерацию: `Для Каждого КлючИЗначение Из Соответствие` vs обход структуры.
- Честно уточняю: ни структура, ни соответствие не потокобезопасны; в одном сеансе 1С это не проблема.
- Не путаю с `ФиксированнаяСтруктура` / `ФиксированноеСоответствие` для передачи на клиент без мутаций.

## Как это работает

Параметры открытия формы передают структурой:

```bsl
ПараметрыФормы = Новый Структура;
ПараметрыФормы.Вставить("ДокументОснование", СсылкаНаЗаказ);
ПараметрыФормы.Вставить("РежимКопирования", Истина);

ОткрытьФорму("Документ.РеализацияТоваров.ФормаОбъекта", ПараметрыФормы);
```

Кэш дополнительных сведений по номенклатуре удобнее в соответствии:

```bsl
КэшЕдиниц = Новый Соответствие;

Для Каждого Строка Из ТаблицаТоваров Цикл
    Если КэшЕдиниц[Строка.Номенклатура] = Неопределено Тогда
        КэшЕдиниц.Вставить(Строка.Номенклатура, Строка.Номенклатура.ЕдиницаИзмерения);
    КонецЕсли;
    Строка.Единица = КэшЕдиниц[Строка.Номенклатура];
КонецЦикла;
```

Структура сериализуется в JSON удобнее для простых наборов полей. Соответствие используют, когда ключ не строка или нужен частый поиск по ссылке без `Строка(Ссылка)` в качестве ключа.

## Подводные камни

**Ключ-ссылка в структуре.** Нельзя: только строковые имена свойств.

**Путаница с уникальностью.** И в структуре, и в соответствии второй `Вставить` с тем же ключом перезаписывает значение.

**Передача на клиент.** Мutable структура может быть изменена на клиенте; для read-only используют фиксированные варианты.

**Производительность.** Очень большие соответствия в памяти хуже, чем запрос с временной таблицей.

**Сравнение ключей.** Ссылочные ключи сравниваются по ссылке; дубли разных объектов с одинаковым GUID невозможны, но пустая ссылка требует отдельной проверки.
