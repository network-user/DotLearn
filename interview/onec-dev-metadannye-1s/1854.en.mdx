# How to design a subsystem with new documents in 1C

## In short

A subsystem groups metadata objects for the UI and access rights. A new area starts with a data model (document, catalogs, registers), then you create the subsystem, include objects in it, and set up posting and forms.

## How to answer in an interview

- I describe a subsystem as a logical module: menu, roles, visibility in the configurator.
- First I fix movements: which registers the document writes on posting.
- I show the document plus catalogs plus accumulation register chain for stock accounting.
- I mention including objects in the subsystem; otherwise users will not see them in the UI.
- I am honest: without scheduled jobs and reports the subsystem is incomplete; a document alone does not close the process.

## How it works

I create a subsystem in the configuration tree, set synonym and picture. I add a document with header attributes and a tabular section, counterparty and item catalogs, and an accumulation register for quantities. I include all objects in the subsystem.

Programmatic create and post of a sales order:

```bsl
ДокументОбъект = Документы.ЗаказКлиента.СоздатьДокумент();
ДокументОбъект.Дата = ТекущаяДатаСеанса();
ДокументОбъект.Контрагент = Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка");

СтрокаТоваров = ДокументОбъект.Товары.Добавить();
СтрокаТоваров.Номенклатура = Справочники.Номенклатура.НайтиПоРеквизиту("Артикул", "A-100");
СтрокаТоваров.Количество = 5;
СтрокаТоваров.Цена = 1200;

ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
```

In the object module I implement `ОбработкаПроведения`: build register movements, check balances in `ПередЗаписью`. Roles on the subsystem restrict access to the new menu section.

## Pitfalls

**Objects outside the subsystem.** The document exists but is hidden unless users search manually.
**Posting without registers.** The document does not change accounting data; the business goal fails.
**Duplicating standard objects.** Creating a parallel document instead of an extension breaks exchange.
**Missing numbering and dates.** Forgotten numerator settings cause number conflicts.
**Weak role model.** The subsystem is visible to everyone although the document should be restricted.
