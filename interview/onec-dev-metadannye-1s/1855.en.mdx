# Command group in 1C: purpose and structure

## In short

A command group bundles UI commands into a hierarchy for panels, submenus, and navigation sections. It is a metadata object, not a programmatic class: commands and nested groups are defined in the configurator and feed the configuration command interface.

## How to answer in an interview

- I explain that a command group structures navigation; it does not run business logic.
- Commands reference data processors, reports, common forms, or standard actions.
- Subsystems tie group visibility to user roles.
- In the managed interface groups build the ribbon and section panels.
- I am honest: do not confuse a command group with `КомандыФормы`; they are different levels (configuration UI vs form).

## How it works

In the configurator I create a Command group object, add child commands with presentation and action. I nest subgroups for multi-level menus. I bind the group to a subsystem so the section appears for roles with access to that subsystem.

On an object form commands are declared separately and placed on the `КоманднаяПанель`:

```bsl
&НаКлиенте
Процедура ПечатьСпецификации(Команда)
    ПечатьСпецификацииНаСервере();
КонецПроцедуры

&НаСервере
Процедура ПечатьСпецификацииНаСервере()
    ТабДок = Документы.РеализацияТоваров.ПечатьСпецификации(Объект.Ссылка);
    ТабДок.Показать("Спецификация");
КонецПроцедуры
```

A configuration command group defines which global actions are available in the UI; `КомандыФормы` live in a specific form module and handle local scenarios.

## Pitfalls

**Confusion with form commands.** A command group does not replace `КомандыФормы` on a managed form.
**Command without a handler.** The user sees a button but the action is not bound to a module or metadata.
**Roles not configured.** The group exists but is hidden because of missing subsystem rights.
**Excessive nesting.** Deep menus hurt UX in thin and web clients.
**Duplicate commands.** The same processor in several groups complicates maintenance of translations and rights.
