# Определяемые типы в 1С: составной тип без дублирования

## Суть

Определяемый тип в метаданных объединяет набор типов значений под одним именем. Его подставляют в реквизиты, параметры и измерения вместо длинного перечисления ссылок. Это сокращает дублирование и упрощает изменение состава типов в одном месте.

## Что ответить на собесе

- Я говорю: определяемый тип это именованный «составной тип», а не отдельный примитив с валидацией regex.
- Использую для полей «Контрагент или Физлицо», «Документ-основание» из нескольких видов.
- При смене состава типов обновляю один определяемый тип, а не десятки реквизитов.
- Уточняю: проверка допустимых значений остаётся на уровне типа платформы и кода.
- Честно: определяемый тип не заменяет строгую доменную модель; иногда лучше отдельные реквизиты.

## Как это работает

В конфигураторе создаю «Определяемый тип», например `УчастникСделки`, и включаю `СправочникСсылка.Контрагенты` и `СправочникСсылка.ФизическиеЛица`. В документе реквизит `Участник` получает тип `УчастникСделки`.

Чтение и запись значения составного типа:

```bsl
Процедура ЗаполнитьУчастника(ДокументОбъект, Участник)
    Если ТипЗнч(Участник) = Тип("СправочникСсылка.Контрагенты") Тогда
        ДокументОбъект.Участник = Участник;
    ИначеЕсли ТипЗнч(Участник) = Тип("СправочникСсылка.ФизическиеЛица") Тогда
        ДокументОбъект.Участник = Участник;
    Иначе
        ВызватьИсключение "Недопустимый тип участника";
    КонецЕсли;
КонецПроцедуры
```

В запросах определяемый тип ведёт себя как объединение допустимых типов; при расширении состава нужно обновить конфигурацию и проверить формы, где поле выбирается из списка.

## Подводные камни

**Ожидание валидации формата.** Определяемый тип не добавляет regex; нужен отдельный код или реквизит с ограничением.
**Слишком широкий состав.** В поле попадают лишние ссылки, усложняется выбор пользователю.
**Забытые зависимые реквизиты.** После сужения состава старые значения в данных могут стать невалидными.
**Обмен и расширения.** Расширение не всегда может менять определяемые типы поставщика.
**Путаница с типовыми.** В типовых конфигурациях имена определяемых типов уже заняты; дублирование ломает сравнение при обновлении.
