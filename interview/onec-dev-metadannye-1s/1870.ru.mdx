# Чтение реквизита по ссылке без полной загрузки объекта

## Суть

Ссылка идентифицирует запись в базе. Для одного поля не обязательно вызывать `ПолучитьОбъект()`: достаточно запроса с параметром ссылки или функции БСП `ЗначениеРеквизитаОбъекта`. Полная загрузка объекта оправдана при массовом изменении данных.

## Что ответить на собесе

- Я для отображения наименования в списке использую запрос или `ЗначениеРеквизитаОбъекта(Ссылка, "Наименование")`.
- `ПолучитьОбъект()` применяю, когда нужно менять реквизиты и писать объект.
- В запросе обращаюсь к полям через псевдоним таблицы: `Контрагент.ИНН`.
- Учитываю пустую ссылку: `ЗначениеЗаполнено(Ссылка)` перед чтением.
- Честно: цепочка `ПолучитьОбъект()` ради одного поля в цикле из сотен строк убивает производительность.

## Как это работает

Лёгкое чтение через БСП:

```bsl
Наименование = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(КонтрагентСсылка, "Наименование");
```

Чтение нескольких полей одним запросом:

```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    "ВЫБРАТЬ
    |    К.Наименование,
    |    К.ИНН
    |ИЗ
    |    Справочник.Контрагенты КАК К
    |ГДЕ
    |    К.Ссылка = &Ссылка";
Запрос.УстановитьПараметр("Ссылка", КонтрагентСсылка);
Выборка = Запрос.Выполнить().Выбрать();
```

Полная загрузка для изменения:

```bsl
ОбъектКонтрагента = КонтрагентСсылка.ПолучитьОбъект();
ОбъектКонтрагента.Наименование = НовоеИмя;
ОбъектКонтрагента.Записать();
```

## Подводные камни

**ПолучитьОбъект в цикле.** Сотни обращений к СУБД вместо одного запроса.
**Чтение без проверки ссылки.** Пустая ссылка даёт ошибку или пустой результат непредсказуемо.
**Реквизит табличной части.** Через ссылку шапки не прочитать; нужен запрос к ТЧ.
**Устаревшие данные.** Кэш формы может не совпадать с базой после записи в другом сеансе.
**Обход привилегий.** `ЗначениеРеквизитаОбъекта` в привилегированном режиме читает то, что пользователю запрещено RLS.
