# Как проектировать подсистему с новыми документами в 1С

## Суть

Подсистема группирует объекты метаданных для интерфейса и прав доступа. Новый контур начинают с модели данных (документ, справочники, регистры), затем создают подсистему, включают в неё объекты, настраивают проведение и формы.

## Что ответить на собесе

- Я описываю подсистему как логический модуль: меню, роли, видимость в конфигураторе.
- Сначала фиксирую движения: какие регистры пишет документ при проведении.
- Показываю связку документ + справочники + регистр накопления для учёта остатков.
- Упоминаю включение объектов в подсистему, иначе пользователь не увидит их в интерфейсе.
- Честно: без регламентных заданий и отчётов подсистема «висящая»; документ сам по себе не закрывает процесс.

## Как это работает

Создаю подсистему в дереве конфигурации, задаю синоним и картинку. Добавляю документ с реквизитами шапки и табличной частью, справочники контрагентов и номенклатуры, регистр накопления для количественного учёта. Включаю все объекты в состав подсистемы.

Программное создание и проведение документа заказа:

```bsl
ДокументОбъект = Документы.ЗаказКлиента.СоздатьДокумент();
ДокументОбъект.Дата = ТекущаяДатаСеанса();
ДокументОбъект.Контрагент = Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка");

СтрокаТоваров = ДокументОбъект.Товары.Добавить();
СтрокаТоваров.Номенклатура = Справочники.Номенклатура.НайтиПоРеквизиту("Артикул", "A-100");
СтрокаТоваров.Количество = 5;
СтрокаТоваров.Цена = 1200;

ДокументОбъект.Записать(РежимЗаписиДокумента.Проведение);
```

В модуле объекта описываю `ОбработкаПроведения`: формирую движения по регистру, проверяю остатки в `ПередЗаписью`. Роли на подсистему ограничивают доступ к новому разделу меню.

## Подводные камни

**Объекты вне подсистемы.** Документ создан, но скрыт от пользователей без ручного поиска.
**Проведение без регистров.** Документ не меняет учётные данные, бизнес-цель не достигнута.
**Дублирование типовых объектов.** Вместо расширения создают параллельный документ и ломают обмен.
**Нет нумерации и дат.** Забытые настройки нумераторов дают конфликты номеров.
**Слабая модель ролей.** Подсистема видна всем, хотя документ должен быть ограничен.
