# Где задаётся начальное значение параметра сеанса

## Суть

Параметры сеанса объявляют в конфигурации (ветка «Параметры сеанса»), а начальные значения устанавливают в модуле сеанса в процедуре `УстановкаПараметровСеанса`. Она вызывается платформой при старте сеанса до основной прикладной логики.

## Что ответить на собесе

- Я называю два места: метаданные параметра (имя, тип) и модуль сеанса с `УстановкаПараметровСеанса`.
- Показываю вызов `ПараметрыСеанса.ИмяПараметра = ...` или `УстановитьПараметрыСеанса()` в зависимости от версии API.
- Отличаю от `ПриНачалеРаботыСистемы` в модуле управляемого приложения: там UI и предупреждения, не первичная инициализация параметров.
- Честно уточняю: параметр сеанса доступен и на клиенте, и на сервере в рамках одного сеанса, но инициализация выполняется по правилам платформы на сервере.
- Упоминаю БСП: часть параметров заполняет подсистема «Текущий сеанс» / стандартные библиотеки.

## Как это работает

В конфигураторе добавляют параметр сеанса `ТекущаяОрганизация` типа `СправочникСсылка.Организации`. В модуле сеанса:

```bsl
Процедура УстановкаПараметровСеанса(ИменаПараметровСеанса)

    Если ИменаПараметровСеанса = Неопределено Тогда
        ПараметрыСеанса.ТекущаяОрганизация = ОрганизацияПоУмолчанию();
    Иначе
        Для Каждого Имя Из ИменаПараметровСеанса Цикл
            Если Имя = "ТекущаяОрганизация" Тогда
                ПараметрыСеанса.ТекущаяОрганизация = ОрганизацияПоУмолчанию();
            КонецЕсли;
        КонецЦикла;
    КонецЕсли;

КонецПроцедуры

Функция ОрганизацияПоУмолчанию()

    Возврат Справочники.Организации.ОрганизацияПоУмолчанию();

КонецФункции
```

Платформа вызывает процедуру при создании сеанса и при точечном сбросе параметра. Прикладной код далее читает `ПараметрыСеанса.ТекущаяОрганизация` в запросах и формах без повторного поиска в ИБ на каждый вызов.

Параметры сеанса не сохраняются между сеансами и не заменяют константы: это кэш сеанса с типом из метаданных.

## Подводные камни

**Путаница с модулем приложения.** Инициализация в `ПриНачалеРаботыСистемы` может выполниться позже или на клиенте без нужного контекста.

**Тяжёлый код в установке.** Длинные запросы при старте сеанса замедляют вход для всех пользователей.

**Не объявлен тип в метаданных.** Обращение к несуществующему параметру даёт ошибку времени выполнения.

**Привилегированный режим.** Чтение данных при установке может требовать `УстановитьПривилегированныйРежим` для системных настроек.

**Фоновые задания.** У фонового сеанса свой набор параметров; значения интерактивного сеанса не переносятся автоматически.
