# Модули объекта, формы и менеджера: разделение ответственности

## Суть

Модуль объекта обслуживает экземпляр данных (запись, проведение, валидация). Модуль формы управляет UI и клиент-серверным обменом. Модуль менеджера содержит статические методы типа метаданных: выборки, фабрики, печать, общие алгоритмы без конкретного `ЭтотОбъект`.

## Что ответить на собесе

- Я кладу бизнес-инварианты в модуль объекта: `ПередЗаписью`, `ОбработкаПроведения`.
- В модуле формы оставляю видимость полей, обработчики кнопок, вызовы сервера.
- В менеджере держу `НайтиПоРеквизиту`, создание по умолчанию, общие запросы.
- Подчёркиваю контекст: объектный модуль выполняется на сервере при работе с данными.
- Честно: тяжёлые запросы в модуле формы недопустимы; их выношу в менеджер или общий модуль.

## Как это работает

При записи документа платформа вызывает обработчики модуля объекта:

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)
    Если СуммаДокумента < 0 Тогда
        Отказ = Истина;
        ВызватьИсключение "Сумма не может быть отрицательной";
    КонецЕсли;
КонецПроцедуры
```

Модуль управляемой формы реагирует на действия пользователя:

```bsl
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    ОбновитьДоступностьСклада();
КонецПроцедуры

&НаСервере
Процедура ОбновитьДоступностьСклада()
    Элементы.Склад.Доступность = ЗначениеЗаполнено(Объект.Контрагент);
КонецПроцедуры
```

Модуль менеджера справочника предоставляет API без загрузки элемента:

```bsl
Функция АктуальнаяЦена(Номенклатура, ДатаЦены) Экспорт
    Возврат РегистрыСведений.ЦеныНоменклатуры.ПолучитьПоследнее(ДатаЦены, Новый Структура("Номенклатура", Номенклатура)).Цена;
КонецФункции
```

## Подводные камни

**Логика в форме.** Дублирование проверок в форме и объекте расходится при программной записи.
**Тяжёлый код в объекте.** Длительные операции в `ПередЗаписью` блокируют транзакцию.
**Менеджер как свалка.** Без структуры модуль менеджера превращается в нечитаемый монолит.
**Клиентский доступ к данным.** Прямой запрос с клиента нарушает границы управляемого приложения.
**Путаница с модулем набора записей.** У регистров отдельный модуль набора; не смешиваю с модулем объекта документа.
