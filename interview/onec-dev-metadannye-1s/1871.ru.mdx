# Автозаполнение «Ответственный» для нового документа

## Суть

Для нового объекта реквизит «Ответственный» заполняют до записи: в `ПередЗаписью` или при создании на сервере формы. Источник значения: текущий пользователь ИБ или справочник пользователей/сотрудников по связке с сеансом.

## Что ответить на собесе

- Я ставлю ответственного в `ПередЗаписью`, если поле пустое и `ЭтоНовый()`.
- На форме дублирую в `ПриСозданииНаСервере`, чтобы пользователь видел значение сразу.
- Использую `Пользователи.ТекущийПользователь()` или типовой механизм БСП, не устаревший `ТекущийПользователь()` без контекста ИБ.
- Не перезаписываю ответственного при повторной записи существующего документа.
- Честно: при записи из обмена или регламента подставляю технического пользователя или оставляю переданное значение.

## Как это работает

В модуле объекта:

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)
    Если ЭтоНовый() И НЕ ЗначениеЗаполнено(Ответственный) Тогда
        Ответственный = Пользователи.ТекущийПользователь();
    КонецЕсли;
КонецПроцедуры
```

На форме при открытии нового:

```bsl
&НаСервере
Процедура ПриСозданииНаСервере(Отказ, СтандартнаяОбработка)
    Если НЕ ЗначениеЗаполнено(Объект.Ответственный) Тогда
        Объект.Ответственный = Пользователи.ТекущийПользователь();
    КонецЕсли;
КонецПроцедуры
```

Если реквизит ссылается на справочник сотрудников, получаю сотрудника по пользователю через регистр сведений или типовую функцию конфигурации.

## Подводные камни

**ПриЗаписи вместо ПередЗаписью.** Слишком поздно для влияния на проведение и блокировки.
**Перезапись при каждой записи.** Теряется исходный автор при копировании документа.
**Несовпадение типов.** Реквизит «Пользователь» vs «Сотрудник» требует разных функций.
**Фоновый сеанс.** Текущий пользователь регламентного задания не равен инициатору в UI.
**Только на форме.** Программная запись без модуля объекта оставляет поле пустым.
