# Structure vs Map in 1C: when to use which

## In short

Structure stores key-value pairs with string keys and property access (`Data.Key`). Map accepts keys of any type and fits lookup by references, numbers, or enums. In both collections keys are unique; a second insert replaces the value.

## How to answer in an interview

- I say structure for procedure parameters, DTOs, `FillPropertyValues`; map for "reference → data" caches.
- I stress structure keys are identifiers, not `CatalogRef` without using a map.
- I mention iteration: `For Each KeyAndValue In Map` vs structure walk.
- I honestly note neither is thread-safe; in a single 1C session that is fine.
- I do not confuse with `FixedStructure` / `FixedMap` for client transfer without mutation.

## How it works

Form open parameters go in a structure:

```bsl
ПараметрыФормы = Новый Структура;
ПараметрыФормы.Вставить("ДокументОснование", СсылкаНаЗаказ);
ПараметрыФормы.Вставить("РежимКопирования", Истина);

ОткрытьФорму("Документ.РеализацияТоваров.ФормаОбъекта", ПараметрыФормы);
```

A cache of item details by nomenclature reference fits a map:

```bsl
КэшЕдиниц = Новый Соответствие;

Для Каждого Строка Из ТаблицаТоваров Цикл
    Если КэшЕдиниц[Строка.Номенклатура] = Неопределено Тогда
        КэшЕдиниц.Вставить(Строка.Номенклатура, Строка.Номенклатура.ЕдиницаИзмерения);
    КонецЕсли;
    Строка.Единица = КэшЕдиниц[Строка.Номенклатура];
КонецЦикла;
```

Structure maps to JSON more easily for simple field sets. Use a map when the key is not a string or you need frequent lookup by reference without `String(Reference)` as key.

## Pitfalls

**Reference key in structure.** Not allowed: only string property names.

**Uniqueness confusion.** In both structure and map a second `Insert` with the same key overwrites.

**Client transfer.** Mutable structure can change on client; use fixed variants for read-only.

**Performance.** Very large in-memory maps lose to a query with a temp table.

**Key comparison.** Reference keys compare by reference; empty reference needs an explicit check.
