# Object, form, and manager modules: separation of concerns

## In short

The object module serves a data instance (write, posting, validation). The form module controls the UI and client-server exchange. The manager module holds static metadata-type methods: queries, factories, print, shared algorithms without a specific `ЭтотОбъект`.

## How to answer in an interview

- I put business invariants in the object module: `ПередЗаписью`, `ОбработкаПроведения`.
- In the form module I keep field visibility, button handlers, and server calls.
- In the manager I keep `НайтиПоРеквизиту`, default creation, and shared queries.
- I stress context: the object module runs on the server when working with data.
- I am honest: heavy queries in the form module are wrong; I move them to the manager or a common module.

## How it works

On document write the platform calls object module handlers:

```bsl
Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)
    Если СуммаДокумента < 0 Тогда
        Отказ = Истина;
        ВызватьИсключение "Сумма не может быть отрицательной";
    КонецЕсли;
КонецПроцедуры
```

The managed form module reacts to user actions:

```bsl
&НаКлиенте
Процедура КонтрагентПриИзменении(Элемент)
    ОбновитьДоступностьСклада();
КонецПроцедуры

&НаСервере
Процедура ОбновитьДоступностьСклада()
    Элементы.Склад.Доступность = ЗначениеЗаполнено(Объект.Контрагент);
КонецПроцедуры
```

The catalog manager module exposes API without loading an item:

```bsl
Функция АктуальнаяЦена(Номенклатура, ДатаЦены) Экспорт
    Возврат РегистрыСведений.ЦеныНоменклатуры.ПолучитьПоследнее(ДатаЦены, Новый Структура("Номенклатура", Номенклатура)).Цена;
КонецФункции
```

## Pitfalls

**Logic in the form.** Duplicated checks in form and object diverge on programmatic write.
**Heavy code in the object.** Long work in `ПередЗаписью` blocks the transaction.
**Manager as a dump.** Without structure the manager module becomes an unreadable monolith.
**Client data access.** Direct queries from the client break managed application boundaries.
**Confusion with record set module.** Registers have a separate set module; do not mix it with a document object module.
