# Что такое dynamic_cast?

## Суть

`dynamic_cast` это runtime-checked cast для полиморфных классов: downcast и crosscast с проверкой dynamic type через RTTI. Для указателей неудача даёт `nullptr`, для ссылок бросает `std::bad_cast`.

## Что ответить на собесе

- Работает только если в иерархии есть virtual function (обычно virtual ~Base).
- Безопаснее `static_cast` там, где static type базы не гарантирует derived.
- Pointer form: `if (auto* d = dynamic_cast<Derived*>(b))`.
- Reference form редко: исключения или try/catch; чаще redesign.
- Честно: при `-fno-rtti` недоступен; embedded код использует tagged union или vtable hook.

## Как это работает

Компiler/runtime сверяет dynamic type объекта с целевым. При multiple inheritance может скорректировать pointer offset.

```cpp
#include <iostream>

struct Animal {
    virtual ~Animal() = default;
    virtual void speak() const { std::cout << "...\n"; }
};

struct Dog : Animal {
    void speak() const override { std::cout << "woof\n"; }
    void fetch() const { std::cout << "fetch\n"; }
};

void interact(Animal* pet) {
    if (auto* dog = dynamic_cast<Dog*>(pet)) {
        dog->fetch();
    } else {
        pet->speak();
    }
}

int main() {
    Dog d;
    interact(&d);
    return 0;
}
```

Успешный cast вызывает derived-only API; иначе fallback на базовое поведение без UB.

## Подводные камни

- **Отсутствие virtual в базе.** Cast ill-formed или meaningless.
- **Performance в hot loop.** RTTI дороже virtual call; кэширую тип один раз.
- **Ссылочный cast.** Exception flow тяжелее nullptr check.
- **Смешение с const.** `dynamic_cast` уважает const; для снятия const нужен другой путь.
- **Design smell.** Частые downcast → visitor или variant refactor.
