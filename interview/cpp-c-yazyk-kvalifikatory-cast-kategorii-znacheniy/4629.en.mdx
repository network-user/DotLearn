# What access modifiers exist in C++?

## In short

Three levels: `public` (accessible anywhere), `protected` (class plus derived classes), `private` (the class only; default private for `class`, default public for `struct`). They implement encapsulation and a stable API over implementation details.

## How to answer in an interview

- `public`: interface; enforce invariants through methods.
- `protected`: hooks for derived types, not external clients; many style guides minimize it.
- `private`: data members almost always private; Law of Demeter.
- `friend` punctures encapsulation for free functions/operators.
- Honestly: protected data often breaks base invariants; prefer a protected virtual interface.

## How it works

Access control is compile-time; violations are compile errors.

```cpp
class Account {
    int balance = 0;

protected:
    void apply_delta(int delta) { balance += delta; }

public:
    void deposit(int amount) {
        if (amount > 0) apply_delta(amount);
    }

    int funds() const { return balance; }
};

class BonusAccount : public Account {
public:
    void bonus(int amount) { apply_delta(amount); }
};

int main() {
    BonusAccount user;
    user.deposit(100);
    user.bonus(10);
    return user.funds();
}
```

`apply_delta` is protected: visible in `BonusAccount`, not externally. `balance` stays private to block direct derived access.

## Pitfalls

- **protected data members.** Tight coupling; base refactors break heirs.
- **public fields.** Skip validation; hard to change representation.
- **friend abuse.** Over-broad friends blur boundaries.
- **struct vs class defaults.** Forgetting struct's default public access.
- **Inherited access specifiers.** `class D : private Base` makes Base public members private in D's interface.
