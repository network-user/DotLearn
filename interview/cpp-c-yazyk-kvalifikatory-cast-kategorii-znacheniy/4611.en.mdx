# What specifiers can C++ member functions have?

## In short

Member specifiers and qualifiers define the contract: `const`/`volatile` on `this`, virtual dispatch (`virtual`, `override`, `final`), class scope (`static`), inline hints, `explicit`, `noexcept`, and `constexpr`. They affect overload resolution, vtables, optimizations, and API safety.

## How to answer in an interview

- A `const` member does not change logical state; it is callable on const objects.
- `virtual` with `override`/`final` controls polymorphism and blocks further overrides.
- A `static` member has no `this`; it is invoked through the class name.
- `noexcept` and `constexpr` affect moves in containers and compile-time evaluation.
- Honestly: `inline` today is mostly an ODR/linkage hint; the optimizer decides actual inlining.

## How it works

The compiler emits const/non-const overload pairs where needed. `override` verifies a matching virtual base signature.

```cpp
#include <iostream>

struct Shape {
    virtual void draw() const { std::cout << "shape\n"; }
    virtual ~Shape() = default;
};

struct Circle final : Shape {
    void draw() const override { std::cout << "circle\n"; }

    static int count() { return live; }

    constexpr int areaScale() const noexcept { return radius * radius; }

private:
    static inline int live = 0;
    int radius = 1;
};

void render(const Shape& item) noexcept {
    item.draw();
}

int main() {
    Circle c;
    render(c);
    return Circle::count();
}
```

`draw() const override` locks the base contract. `static count()` is shared by the type. `constexpr areaScale()` may participate in constant evaluation in a const context.

## Pitfalls

- **override without a virtual base.** Compile error or wrong behavior.
- **const correctness holes.** `mutable` or non-const pointers inside const methods hide mutation.
- **static and virtual.** No static virtual in C++; polymorphism uses instance methods.
- **False noexcept.** Throwing breaks the contract; some paths call `std::terminate`.
- **constexpr with side effects.** Not every body is constexpr-evaluable; overloads may be runtime-only.
