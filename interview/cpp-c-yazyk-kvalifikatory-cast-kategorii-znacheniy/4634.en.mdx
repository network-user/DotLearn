# How does simple assignment differ from compound assignment?

## In short

The `=` operator copies or moves the right-hand side into the left operand. Compound forms (`+=`, `-=`, `*=`, `/=`, `%=`, bitwise, shifts) first combine the current left-hand value with the right-hand side, then store the result back. For user-defined types, the two forms may invoke different overloads.

## How to answer in an interview

- `a = b` always replaces `a`'s value; `a += b` semantically means "mutate `a` in place."
- For builtins, `a += b` equals `a = a + b`, but heavy types may avoid an extra copy.
- For classes, `operator+=` is often cheaper than `operator=` after a temporary `a + b`.
- Compound operators also work through overloads on custom types (`std::string`, containers).
- Honestly: for integers the compiler usually emits the same code; gains show up on strings and containers.

## How it works

```cpp
int score = 10;
score = 25;
score += 5;

std::string label = "dot";
label += ".learn";

class Counter {
public:
    Counter& operator+=(int delta) {
        value_ += delta;
        return *this;
    }
    Counter& operator=(int v) {
        value_ = v;
        return *this;
    }
    int value() const { return value_; }
private:
    int value_ = 0;
};

Counter tally;
tally = 3;
tally += 2;
```

`score = 25` overwrites entirely. `score += 5` adds to the current value. For `std::string`, `+=` appends without building an intermediate for `label + ".learn"`. In `Counter`, `operator+=` mutates and returns a reference for chaining; `operator=` sets fresh state.

## Pitfalls

- **Reference return.** Compound operators typically return `T&`; simple `operator=` should too since C++11, or chaining breaks.
- **Self-assignment.** `a = a` is safe with a correct `operator=`; `a += a` as well, but custom logic must account for it.
- **Evaluation order.** In `a[i++] = ...` and similar expressions, operand order was not fully specified before C++17 in some cases.
- **Bitwise forms.** `&=`, `|=`, `^=` apply to integers; do not confuse them with logical `&&`/`||`.
- **const left-hand side.** Neither form can assign into a `const` object; the compiler rejects it.
