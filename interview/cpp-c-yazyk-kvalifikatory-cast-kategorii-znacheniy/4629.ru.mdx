# Какие модификаторы доступа есть в C++?

## Суть

Три уровня: `public` (доступ откуда угодно), `protected` (класс + наследники), `private` (только сам класс; для class default private, для struct default public). Они реализуют encapsulation и stable API поверх implementation details.

## Что ответить на собесе

- `public`: interface, invariants через методы.
- `protected`: hook для derived, но не external clients; многие style guides минимизируют.
- `private`: data members почти всегда private; Law of Demeter.
- `friend` пробивает encapsulation точечно для free functions/operators.
- Честно: protected data часто ломает invariant base; prefer protected virtual interface.

## Как это работает

Access control compile-time; нарушение ошибка компиляции.

```cpp
class Account {
    int balance = 0;

protected:
    void apply_delta(int delta) { balance += delta; }

public:
    void deposit(int amount) {
        if (amount > 0) apply_delta(amount);
    }

    int funds() const { return balance; }
};

class BonusAccount : public Account {
public:
    void bonus(int amount) { apply_delta(amount); }
};

int main() {
    BonusAccount user;
    user.deposit(100);
    user.bonus(10);
    return user.funds();
}
```

`apply_delta` protected: доступен в `BonusAccount`, но не снаружи. `balance` private скрыт от derived direct access.

## Подводные камни

- **protected data members.** Derived tight coupling; refactor base ломает heirs.
- **public fields.** Bypass validation; сложно менять representation.
- **friend abuse.** Слишком широкий friend размывает границы.
- **struct vs class default.** Забытый default public в struct.
- **Inherited access specifiers.** `class D : private Base` меняет public Base members to private in D interface.
