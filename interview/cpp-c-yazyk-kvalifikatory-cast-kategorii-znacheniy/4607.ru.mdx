# Зачем лямбде нужен захват переменных?

## Суть

Лямбда в C++ компилируется в объект с оператором `()`, а «поля» появляются из списка захвата: внешние переменные становятся членами этого объекта. Без захвата лямбда видит только свои параметры; с захватом она читает или меняет контекст вызова и может хранить состояние между вызовами.

## Что ответить на собесе

- Объясняю, что захват превращает лямбду в closure с собственным состоянием.
- Различаю `[=]`, `[&]`, `[x]`, `[&x]` и `mutable` для изменения копий.
- Привожу пример с `std::count_if` и порогом из внешней переменной.
- Уточняю: захват по ссылке опасен, если лямбда переживёт объект; для async предпочитаю value или `std::move`.
- Честно: C++14 generic lambda и init-capture `[v = std::move(x)]` часто чище, чем голый `[&]`.

## Как это работает

Компилятор генерирует уникальный functor. Элементы списка захвата становятся полями: копия при `[var]`, ссылка при `[&var]`. Generic lambda (auto параметры) всё равно требует явного захвата внешних имён.

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3, 4, 5};
    int cutoff = 3;

    auto above_cutoff = [cutoff](int item) {
        return item > cutoff;
    };

    const int matched = std::count_if(values.begin(), values.end(), above_cutoff);
    std::cout << "Above " << cutoff << ": " << matched << '\n';
    return 0;
}
```

`cutoff` копируется в closure один раз; алгоритм получает callable с уже «зашитым» порогом. С `[&cutoff]` поведение то же для чтения, но lifetime должен покрывать все отложенные вызовы. `mutable` позволяет менять копии захваченных by-value переменных внутри оператора.

## Подводные камни

- **Dangling при `[&]`.** Лямбда, ушедшая в очередь или поток, держит ссылку на локальную переменную стека.
- **Неявный захват всего.** `[=]` и `[&]` удобны, но скрывают зависимости; в code review просят явные имена.
- **Захват `this`.** `[=]` копирует указатель, не объект; для data members часто нужен `[*this]` (C++17).
- **Init-capture и move.** Без `[buf = std::move(buf)]` дорогие объекты копируются дважды.
- **constexpr лямбды.** Захват ограничен: не всё можно вычислить на этапе компиляции.
