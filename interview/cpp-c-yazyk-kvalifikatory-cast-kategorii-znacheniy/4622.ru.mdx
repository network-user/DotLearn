# Что такое лямбда-выражение в C++?

## Суть

Лямбда это синтаксическая форма безымянного function object: capture list, parameters, optional trailing return type и body. Компилятор создаёт closure type с operator(), пригодный для STL algorithms и std::function.

## Что ответить на собесе

- Синтаксис: `[capture](params) -> Ret { body }`.
- Capture: `[]`, `[=]`, `[&]`, `[x,&y]`, init-capture C++14.
- Generic lambda: `[](auto a, auto b){}`.
- Можно хранить в `auto`, `std::function`, template callable.
- Честно: каждая лямбда имеет unique тип; `std::function` добавляет type erasure и alloc.

## Как это работает

Лямбда без capture converts to function pointer если нет state. С capture генерируется struct.

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> data{5, 2, 8, 1, 3};

    std::sort(data.begin(), data.end(), [](int lhs, int rhs) {
        return lhs > rhs;
    });

    int scale = 2;
    auto scaled = [scale](int v) { return v * scale; };

    for (int n : data) {
        std::cout << scaled(n) << ' ';
    }
    return 0;
}
```

Первая лямбда stateless comparator; вторая захватывает `scale` by value для map step в том же scope.

## Подводные камни

- **Default capture `[&]`.** Скрывает lifetime dependencies.
- **mutable lambda.** Меняет копии capture, не originals при `[=]`.
- **IIFE.** Immediately invoked lambda для const init в функции.
- **Template instantiation bloat.** Много разных lambdas в headers раздувают код.
- **Exception spec.** Лямбда наследует noexcept rules как обычный operator().
