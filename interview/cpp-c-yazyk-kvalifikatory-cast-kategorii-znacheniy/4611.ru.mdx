# Какие спецификаторы бывают у методов C++?

## Суть

Спецификаторы и квалификаторы методов задают контракт: `const`/`volatile` на `this`, виртуальность (`virtual`, `override`, `final`), принадлежность классу (`static`), inline-подсказку, `explicit`, `noexcept`, `constexpr`. Они влияют на overload resolution, vtable, оптимизации и безопасность API.

## Что ответить на собесе

- `const` member не меняет логическое состояние; вызывается на const объектах.
- `virtual` + `override`/`final` управляют полиморфизмом и запретом дальнейших override.
- `static` member не имеет `this`; вызывается через имя класса.
- `noexcept` и `constexpr` влияют на move в контейнерах и compile-time вычисления.
- Честно: `inline` сегодня скорее ODR/linkage hint; реальную inlining решает оптимизатор.

## Как это работает

Компилятор генерирует две версии const/non-const overload там, где нужно. `override` проверяет наличие virtual базового метода с той же сигнатурой.

```cpp
#include <iostream>

struct Shape {
    virtual void draw() const { std::cout << "shape\n"; }
    virtual ~Shape() = default;
};

struct Circle final : Shape {
    void draw() const override { std::cout << "circle\n"; }

    static int count() { return live; }

    constexpr int areaScale() const noexcept { return radius * radius; }

private:
    static inline int live = 0;
    int radius = 1;
};

void render(const Shape& item) noexcept {
    item.draw();
}

int main() {
    Circle c;
    render(c);
    return Circle::count();
}
```

`draw() const override` фиксирует контракт базы. `static count()` общий для типа. `constexpr areaScale()` может участвовать в constant evaluation при const контексте.

## Подводные камни

- **override без virtual базы.** Ошибка компиляции или не то поведение.
- **const correctness дырки.** mutable/data race через неконстантные указатели внутри const метода.
- **static и виртуальность.** static virtual в C++ нет; polymorphism только через instance methods.
- **noexcept ложный.** Если метод бросает, контракт нарушен; std::terminate в некоторых путях.
- **constexpr с side effects.** Не всё тело можно вычислить на этапе компиляции; overload может быть runtime-only.
