# Чем static_cast отличается от dynamic_cast?

## Суть

`static_cast` выполняет проверяемые компилятором преобразования без runtime type check: числа, явные иерархии указателей, `void*`. `dynamic_cast` работает с полиморфными типами (есть virtual), проверяет фактический тип во время выполнения; для указателей даёт `nullptr` при неудаче, для ссылок бросает `std::bad_cast`.

## Что ответить на собесе

- `static_cast`: compile-time, быстрее, без RTTI; downcast безопасен только если вы уверены в типе.
- `dynamic_cast`: runtime RTTI, нужен virtual в базе, безопасный downcast/cross-cast.
- Показываю идиому: `dynamic_cast<Derived*>(base)` + проверка на `nullptr`.
- Упоминаю, что `dynamic_cast` на объект без полиморфизма не скомпилируется.
- Честно добавляю: частый downcast намекает на слабый дизайн; лучше virtual dispatch или `std::variant`.

## Как это работает

Компилятор вшивает RTTI для полиморфных классов. `dynamic_cast` сравнивает указатель с таблицей типов. `static_cast` лишь пересматривает тип на этапе компиляции.

```cpp
#include <iostream>

struct Base {
    virtual ~Base() = default;
    virtual void ping() const { std::cout << "base\n"; }
};

struct Worker : Base {
    void ping() const override { std::cout << "worker\n"; }
    void task_id() const { std::cout << "job\n"; }
};

int main() {
    Worker concrete;
    Base* base = &concrete;

    auto* fast = static_cast<Worker*>(base);
    fast->task_id();

    Base* errored = new Base();
    auto* safe = dynamic_cast<Worker*>(errored);
    if (!safe) std::cout << "not a worker\n";
    delete errored;
}
```

`static_cast` с `errored` дал бы указатель без runtime проверки и UB при вызове `task_id`.

## Подводные камни

- **Downcast без проверки.** `static_cast` на неверном объекте даёт UB.
- **Ссылки.** `dynamic_cast<Worker&>(base)` бросает, а не возвращает optional.
- **Множественное наследование.** `dynamic_cast` может менять адрес указателя.
- **RTTI off.** `-fno-rtti` ломает `dynamic_cast`.
- **Злоупотребление cast.** Предпочитаю полиморфный интерфейс вместо цепочек cast.
