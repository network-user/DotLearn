# Когда использовать static_cast?

## Суть

`static_cast` для преобразований, которые compiler может проверить statically: numeric conversions, void*↔T*, upcast/downcast без runtime check, explicit enum↔int, user-defined conversion chains без опасного reinterpret.

## Что ответить на собесе

- Числа: `double`→`int`, enum class↔underlying.
- Иерархия: upcast implicit often, downcast static когда тип guaranteed by logic.
- void* из C API обратно в `T*` после known allocation.
- Не для polymorphic downcast без guarantee; там `dynamic_cast`.
- Честно: `static_cast` downcast быстрее RTTI, но ошибка типа = UB.

## Как это работает

Compile-time проверяет допустимость; runtime не валидирует downcast.

```cpp
#include <iostream>

struct Base { virtual ~Base() = default; };
struct Derived : Base { int tag = 7; };

int main() {
    const double ratio = 2.9;
    int whole = static_cast<int>(ratio);

    Derived concrete;
    Base* poly = &concrete;
    Derived* back = static_cast<Derived*>(poly);

    std::cout << whole << ' ' << back->tag << '\n';
    return 0;
}
```

Upcast/downcast в примере ok, потому что `poly` реально указывает на `Derived`. Без этой гарантии нужен `dynamic_cast`.

## Подводные камни

- **Narrowing surprises.** Fraction loss, overflow modular для unsigned.
- **Downcast без проверки.** Тихий UB при wrong dynamic type.
- **void* round trip.** Только если lifetime и alignment сохранены.
- **const removal.** Для const использую `const_cast`, не static.
- **Overloading ambiguity.** Cast может выбрать неожиданный overload set.
