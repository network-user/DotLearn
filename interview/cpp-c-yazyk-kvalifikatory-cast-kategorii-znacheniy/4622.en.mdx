# What is a lambda expression in C++?

## In short

A lambda is syntactic sugar for an unnamed function object: capture list, parameters, optional trailing return type, and body. The compiler builds a closure type with `operator()`, usable in STL algorithms and `std::function`.

## How to answer in an interview

- Syntax: `[capture](params) -> Ret { body }`.
- Capture: `[]`, `[=]`, `[&]`, `[x,&y]`, C++14 init-capture.
- Generic lambda: `[](auto a, auto b){}`.
- Store in `auto`, `std::function`, or template callables.
- Honestly: each lambda has a unique type; `std::function` adds type erasure and allocation.

## How it works

A captureless lambda converts to a function pointer. With capture the compiler emits a struct.

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> data{5, 2, 8, 1, 3};

    std::sort(data.begin(), data.end(), [](int lhs, int rhs) {
        return lhs > rhs;
    });

    int scale = 2;
    auto scaled = [scale](int v) { return v * scale; };

    for (int n : data) {
        std::cout << scaled(n) << ' ';
    }
    return 0;
}
```

The first lambda is a stateless comparator; the second captures `scale` by value for a map step in the same scope.

## Pitfalls

- **Default capture `[&]`.** Hides lifetime dependencies.
- **mutable lambda.** Mutates copies of captures, not originals with `[=]`.
- **IIFE.** Immediately invoked lambdas for const init inside a function.
- **Template instantiation bloat.** Many distinct lambdas in headers inflate code size.
- **Exception spec.** Lambdas follow the same noexcept rules as ordinary `operator()`.
