# Какие операторы приведения типов есть в C++?

## Суть

В C++ четыре именованных cast: `static_cast`, `dynamic_cast`, `const_cast`, `reinterpret_cast`. Каждый покрывает узкий класс преобразований и заменяет небезопасные C-style `(T)expr`, делая намерение видимым в коде и review.

## Что ответить на собесе

- `static_cast`: числовые и иерархические up/down cast, известные на compile time.
- `dynamic_cast`: безопасный downcast при полиморфной базе; для указателя даёт `nullptr`, для ссылки бросает.
- `const_cast`: только добавление/снятие cv-квалификаторов, не меняет тип значения.
- `reinterpret_cast`: побитовая переинтерпретация; почти всегда UB без строгих условий.
- Честно: если cast нужен постоянно, часто проблема в дизайне API, а не в выборе оператора.

## Как это работает

`static_cast` режет дробную часть при `double`→`int` и проверяет базовые правила наследования. `dynamic_cast` требует хотя бы одной virtual function в базе и использует RTTI.

```cpp
#include <iostream>

struct PolyBase {
    virtual ~PolyBase() = default;
};

struct PolyDerived : PolyBase {};

int main() {
    const double pi = 3.14;
    const int whole = static_cast<int>(pi);

    PolyBase* base_ptr = new PolyDerived();
    auto* derived_ptr = dynamic_cast<PolyDerived*>(base_ptr);

    if (derived_ptr) {
        std::cout << "downcast ok, int part=" << whole << '\n';
    }

    delete base_ptr;
    return 0;
}
```

`const_cast` снимает const только если исходный объект не был truly const; иначе модификация UB. `reinterpret_cast` читает байты `int` как `char` без гарантий endianness и strict aliasing.

## Подводные камни

- **C-style cast.** Маскирует любой из четырёх cast и скрывает опасность.
- **dynamic_cast без virtual.** Не компилируется или бессмысленен для non-polymorphic баз.
- **const_cast для truly const.** Запись через снятый const на literal или const object это UB.
- **reinterpret_cast и alignment.** Чтение misaligned указателя ломает программу на некоторых архитектурах.
- **static_cast downcast без проверки.** Быстрее RTTI, но ошибка типа проявится позже как UB.
