# Что такое битовые операции в C++?

## Суть

Битовые операторы (`&`, `|`, `^`, `~`, `<<`, `>>`) работают над представлением целых типов по битам: маски флагов, упаковка полей, быстрые множители/делители степени двойки. Применяют к unsigned или осознанно к signed с учётом sign extension.

## Что ответить на собесе

- AND маскирует биты, OR включает, XOR переключает, NOT инвертирует все биты.
- `<< n` умножает на 2^n; `>> n` делит с логическим или арифметическим сдвигом.
- Флаги: `flags |= MASK; if (flags & MASK)`.
- Prefer `std::uint32_t`/`unsigned` для предсказуемой ширины.
- Честно: shift на signed negative implementation-defined/UB при переполнении shift count.

## Как это работает

Операнды приводятся к common integral type; каждый бит комбинируется независимо.

```cpp
#include <cstdint>
#include <iostream>

enum : std::uint8_t { Read = 1 << 0, Write = 1 << 1, Exec = 1 << 2 };

int main() {
    const std::uint8_t a = 0b0101;
    const std::uint8_t b = 0b0011;

    std::uint8_t perms = Read | Write;
    perms |= Exec;

    std::cout << (a & b) << ' '
              << (a | b) << ' '
              << (a ^ b) << ' '
              << static_cast<unsigned>(~a & 0xF) << ' '
              << (a << 1) << ' '
              << (a >> 1) << ' '
              << static_cast<unsigned>(perms) << '\n';
    return 0;
}
```

`Read | Write | Exec` собирает права; `&` проверяет бит, `^` toggles без OR side effects.

## Подводные камни

- **Shift >= width.** UB в C++ если shift amount ≥ bit width типа.
- **Signed right shift.** Arithmetic vs logical platform nuances.
- **Operator precedence.** `a & b == c` парсится как `a & (b == c)`; нужны скобки.
- **Promotion to int.** `uint8_t` ops promote; cast back если нужен exact width store.
- **«Оптимизация» без профиля.** Современные compilers часто так же оптимизируют `*2`.
