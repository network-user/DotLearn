# Can a pointer be reassigned to another address?

## In short

An ordinary `T*` stores an address and can be reassigned: `ptr = &other`. That distinguishes pointers from references. Exception: `T* const` fixes the pointer itself; only `*ptr` changes the pointee.

## How to answer in an interview

- `int* p` is reseat-able; `int* const p` has a fixed address.
- `const int* p` may change target address but not value through the pointer.
- Reassignment does not free old heap memory; ownership is separate via delete/smart pointers.
- Raw pointer reseating is common in algorithms and intrusive lists.
- Honestly: for optional rebind with alias syntax there is no `optional<T&>`; use a pointer or wrapper.

## How it works

Pointer assignment copies the address value; object lifetime does not depend on pointer reassignment.

```cpp
#include <iostream>

int main() {
    int left = 10;
    int right = 20;
    int* cursor = &left;

    std::cout << *cursor << '\n';
    cursor = &right;
    std::cout << *cursor << '\n';
    return 0;
}
```

After `cursor = &right`, dereferencing reads 20; `left` is unchanged.

## Pitfalls

- **Dangling after reseat.** The old object may leave scope; stored aliases are unsafe.
- **Leak with owning pointers.** Overwriting `p = q` without delete loses the only heap handle.
- **const pointer confusion.** Read `const T*` vs `T* const` right-to-left.
- **Pointer arithmetic.** reseat + offset needs a valid contiguous range.
- **Smart pointer reset.** `unique_ptr::reset` is controlled reseat plus delete of the old resource.
