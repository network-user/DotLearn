# Какие способы приведения типов есть в C++?

## Суть

Помимо четырёх именованных cast, в C++ остаются implicit conversions и C-style `(T)expr`. Явные cast нужны, когда преобразование узкое или рискованное; именованные операторы ограничивают класс допустимых преобразований и документируют намерение.

## Что ответить на собесе

- Перечисляю `static_cast`, `dynamic_cast`, `const_cast`, `reinterpret_cast` и когда каждый уместен.
- Добавляю: неявные promotion/conversion компилятор делает сам; explicit cast отключает часть проверок осознанно.
- C-style cast объединяет все виды и его избегаю в modern C++.
- Для polymorphic downcast предпочитаю `dynamic_cast` или virtual dispatch вместо blind `static_cast`.
- Честно: `std::bit_cast` (C++20) для type-punning без `reinterpret_cast` там, где типы trivially copyable.

## Как это работает

`static_cast` покрывает numeric narrowing и upcast в иерархии. `dynamic_cast` проверяет RTTI. `const_cast` меняет только cv. `reinterpret_cast` трактует битовое представление и почти всегда требует expert review.

```cpp
#include <bit>
#include <cstdint>
#include <iostream>

float to_float(std::uint32_t bits) {
    return std::bit_cast<float>(bits);
}

int main() {
    const double ratio = 3.99;
    int rounded = static_cast<int>(ratio);

    std::uint32_t raw = 0x40490fdb;
    float pi_approx = to_float(raw);

    std::cout << rounded << ' ' << pi_approx << '\n';
    return 0;
}
```

Неявное `double`→`int` в присваивании иногда разрешено, но явный `static_cast` показывает намерение отбросить дробь. `bit_cast` копирует биты без нарушения strict aliasing.

## Подводные камни

- **Смешение способов.** C-style cast скрывает, какой именно cast применился.
- **Implicit narrowing.** Без cast компилятор может предупредить или запретить с `-Werror`.
- **reinterpret vs bit_cast.** Второй безопаснее для trivially copyable, первый для произвольных указателей.
- **dynamic_cast на ссылке.** Неудача бросает `std::bad_cast`, не `nullptr`.
- **const_cast ради API legacy.** Снятие const и запись в read-only память это UB.
