# Можно ли переназначить указатель на другой адрес?

## Суть

Обычный указатель `T*` хранит адрес и может быть переназначен: `ptr = &other`. Это отличает pointer от reference. Исключение: `T* const` фиксирует сам указатель, меняется только pointee через `*ptr`.

## Что ответить на собесе

- `int* p` reseat-able; `int* const p` address fixed.
- `const int* p` меняет target address, но не value через pointer.
- Переназначение не освобождает старый heap; ownership отдельно через delete/smart ptr.
- Raw pointer reseat часто в алгоритмах и intrusive lists.
- Честно: если нужен optional rebind + alias syntax, иногда `optional<T&>` нет; pointer или wrapper.

## Как это работает

Присваивание pointer копирует address value; object lifetime не зависит от pointer reassignment.

```cpp
#include <iostream>

int main() {
    int left = 10;
    int right = 20;
    int* cursor = &left;

    std::cout << *cursor << '\n';
    cursor = &right;
    std::cout << *cursor << '\n';
    return 0;
}
```

После `cursor = &right` разыменование читает 20; `left` не изменился.

## Подводные камни

- **Dangling после reseat.** Старый object мог выйти из scope; сохранённые alias опасны.
- **Leak при ownership pointer.** Перезапись `p = q` без delete теряет единственный handle на heap.
- **const pointer confusion.** `const T*` vs `T* const` читаются справа налево.
- **Pointer arithmetic.** reseat + offset требует valid contiguous range.
- **Smart pointer reset.** `unique_ptr::reset` эквивалент controlled reseat + delete old.
