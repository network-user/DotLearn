# What is dynamic_cast?

## In short

`dynamic_cast` is a runtime-checked cast for polymorphic classes: downcasts and crosscasts validated against dynamic type via RTTI. Pointers get `nullptr` on failure; references throw `std::bad_cast`.

## How to answer in an interview

- Works only when the hierarchy has a virtual function (usually virtual `~Base`).
- Safer than `static_cast` when the static base type does not guarantee the derived type.
- Pointer form: `if (auto* d = dynamic_cast<Derived*>(b))`.
- Reference form is rare: exceptions or try/catch; redesign is often better.
- Honestly: unavailable with `-fno-rtti`; embedded code uses tagged unions or vtable hooks.

## How it works

Compiler/runtime compares the object's dynamic type to the target. With multiple inheritance it may adjust the pointer offset.

```cpp
#include <iostream>

struct Animal {
    virtual ~Animal() = default;
    virtual void speak() const { std::cout << "...\n"; }
};

struct Dog : Animal {
    void speak() const override { std::cout << "woof\n"; }
    void fetch() const { std::cout << "fetch\n"; }
};

void interact(Animal* pet) {
    if (auto* dog = dynamic_cast<Dog*>(pet)) {
        dog->fetch();
    } else {
        pet->speak();
    }
}

int main() {
    Dog d;
    interact(&d);
    return 0;
}
```

A successful cast calls derived-only API; otherwise fallback base behavior without UB.

## Pitfalls

- **No virtual in the base.** Cast is ill-formed or meaningless.
- **Performance in hot loops.** RTTI costs more than a virtual call; cache type once.
- **Reference casts.** Exception flow is heavier than nullptr checks.
- **Mixing with const.** `dynamic_cast` respects const; other paths remove const differently.
- **Design smell.** Frequent downcasts suggest visitor or variant refactors.
