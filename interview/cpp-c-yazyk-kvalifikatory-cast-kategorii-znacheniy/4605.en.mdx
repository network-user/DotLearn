# What is the inline keyword for?

## In short

`inline` serves two jobs: (1) hint the optimizer to embed the function body; (2) allow a function definition in a header without breaking ODR when included in multiple translation units. The second role matters more today.

## How to answer in an interview

- I separate the compiler hint from linkage/ODR.
- I explain why templates and in-class methods are effectively inline.
- I say when it fits: tiny accessors, header-only utilities.
- I warn about code bloat with large inline bodies.
- I add honestly: in a .cpp file `inline` rarely changes perf; LTO works differently.

## How it works

A non-template function in a header without `inline` triggers multiply defined symbols at link time. `inline` gives weak linkage so the linker merges copies. The optimizer separately chooses call-site expansion.

```cpp
// metrics.hpp
#pragma once
#include <iostream>

inline int square(int value) {
    return value * value;
}

inline void report(int value) {
    std::cout << "sq=" << square(value) << '\n';
}
```

Several .cpp files can include `metrics.hpp` without link errors.

## Pitfalls

- **Expecting speedup.** The keyword alone does not accelerate code.
- **Huge header functions.** They raise compile time and TU size.
- **Not the same as static.** `static` in .cpp is internal linkage, different semantics.
- **Virtual calls.** Virtual dispatch blocks ordinary inline through a base pointer.
- **ODR violations.** Inline definitions in headers must be identical in every TU.
