# Как static влияет на локальные переменные?

## Суть

`static` local живёт до конца программы (static storage duration), инициализируется один раз при первом проходе, сохраняет value между вызовами функции. Scope остаётся локальным: имя видно только внутри функции.

## Что ответить на собесе

- Function-local static: persistent state без global name pollution.
- C++11 thread-safe one-time init (magic statics) для function-local static.
- Использую для counters, memoization, lazy singleton function-local.
- Не thread-safe для read-modify-write без mutex если shared mutable state.
- Честно: hidden global state усложняет тесты; inject dependency если можно.

## Как это работает

Storage в static segment; visibility block-scoped.

```cpp
#include <iostream>

int visits() {
    static int tally = 0;
    ++tally;
    return tally;
}

int main() {
    std::cout << visits() << '\n';
    std::cout << visits() << '\n';
    std::cout << visits() << '\n';
    return 0;
}
```

Вывод 1, 2, 3: `tally` не уничтожается между вызовами. Инициализация `= 0` выполняется один раз.

## Подводные камни

- **Order of initialization.** Static locals init on first use; cross-function order может surprise в сложных графах.
- **Data races.** Concurrent increment без sync UB.
- **Testing reset.** Static state persists между tests unless reload process.
- **Destructor.** Static locals destroyed at exit in reverse order; IO may be shut down.
- **Reentrancy.** Recursive first call during init rare but possible with care.
