# Что такое явное приведение типов?

## Суть

Explicit cast это преобразование, записанное программистом явно, в отличие от implicit promotion/conversion компилятора. В modern C++ предпочитают именованные cast вместо C-style `(T)e`, чтобы ограничить класс допустимых преобразований.

## Что ответить на собесе

- C-style `(int)x` и functional `int(x)` объединяют несколько cast; избегаю.
- `static_cast` для numeric и compile-time known hierarchy.
- Остальные три cast для RTTI, const и reinterpret.
- Explicit cast документирует потерю точности или change of representation.
- Честно: если cast нужен часто, часто проблема в API types, не в синтаксисе.

## Как это работает

Компилятор проверяет только правила конкретного cast operator; runtime checks только у `dynamic_cast`.

```cpp
#include <iostream>

int main() {
    const double meters = 3.75;
    const int whole_meters = static_cast<int>(meters);

    const int ascii = 65;
    const char letter = static_cast<char>(ascii);

    std::cout << whole_meters << ' ' << letter << '\n';
    return 0;
}
```

`static_cast<int>` отбрасывает дробь осознанно. `static_cast<char>` сужает int с implementation-defined behavior если вне range (для ASCII ok).

## Подводные камни

- **Narrowing без cast.** List init `{}` запрещает narrowing silently.
- **Signed/unsigned mix.** Explicit не делает операцию безопасной автоматически.
- **Functional cast на тип с explicit ctor.** Может обойти explicit constructor (до C++17 nuances).
- **reinterpret для «удобства».** Почти всегда code smell.
- **Cast const away и write.** UB если объект изначально const.
