# What is explicit type casting?

## In short

An explicit cast is a conversion written by the programmer, unlike implicit compiler promotion/conversion. Modern C++ prefers named casts over C-style `(T)e to limit the allowed conversion class.

## How to answer in an interview

- C-style `(int)x` and functional `int(x)` merge several casts; I avoid them.
- `static_cast` for numeric and compile-time-known hierarchy cases.
- The other three casts cover RTTI, const, and reinterpret.
- Explicit casts document precision loss or representation changes.
- Honestly: frequent casts often mean API types are wrong, not the syntax.

## How it works

The compiler checks only the chosen cast operator's rules; runtime checks apply only to `dynamic_cast`.

```cpp
#include <iostream>

int main() {
    const double meters = 3.75;
    const int whole_meters = static_cast<int>(meters);

    const int ascii = 65;
    const char letter = static_cast<char>(ascii);

    std::cout << whole_meters << ' ' << letter << '\n';
    return 0;
}
```

`static_cast<int>` deliberately drops the fraction. `static_cast<char>` narrows the int with implementation-defined behavior if out of range (fine for ASCII).

## Pitfalls

- **Narrowing without a cast.** Brace init `{}` forbids silent narrowing.
- **Signed/unsigned mix.** Explicit does not make the operation safe by itself.
- **Functional cast on explicit ctor types.** May bypass explicit constructors (pre-C++17 nuances).
- **reinterpret for convenience.** Almost always a code smell.
- **Cast const away and write.** UB if the object was truly const.
