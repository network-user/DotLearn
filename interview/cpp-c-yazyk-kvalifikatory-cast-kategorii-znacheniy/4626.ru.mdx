# Что означает mutable в C++?

## Суть

`mutable` на data member разрешает менять его даже в `const` member function. Это для физического состояния, не входящего в logical const API: cache, mutex, statistics, lazy init.

## Что ответить на собесе

- Logical const: метод не меняет observable behavior для клиента.
- `mutable std::mutex` в const method для thread-safe lazy init.
- `mutable` cache после expensive compute в const getter.
- Не злоупотребляю: business fields не делаю mutable ради const-correctness hack.
- Честно: если mutable много, возможно const interface неверен.

## Как это работает

constness метода не распространяется на mutable members.

```cpp
#include <iostream>
#include <string>

class Greeting {
    std::string text;
    mutable int query_count = 0;

public:
    explicit Greeting(std::string msg) : text(std::move(msg)) {}

    const std::string& message() const {
        ++query_count;
        return text;
    }

    int queries() const { return query_count; }
};

int main() {
    const Greeting hi("hello");
    std::cout << hi.message() << ' ' << hi.queries() << '\n';
    std::cout << hi.message() << ' ' << hi.queries() << '\n';
    return 0;
}
```

`message()` const, но счётчик растёт; клиент видит stable string, внутренний metric меняется.

## Подводные камни

- **Data race на mutable без mutex.** const method не implies thread-safe.
- **Cache invalidation.** Забытый flush при mutate non-const path.
- **mutable reference members.** Редко; lifetime still critical.
- **Сериализация.** mutable stats не всегда должны попадать в snapshot.
- **Замена const object design.** Иногда separate stats object чище.
