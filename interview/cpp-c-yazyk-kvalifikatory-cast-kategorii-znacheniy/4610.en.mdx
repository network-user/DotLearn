# What type cast operators exist in C++?

## In short

C++ has four named casts: `static_cast`, `dynamic_cast`, `const_cast`, and `reinterpret_cast`. Each covers a narrow conversion class and replaces unsafe C-style `(T)expr`, making intent visible in code and review.

## How to answer in an interview

- `static_cast`: numeric and hierarchy up/down casts known at compile time.
- `dynamic_cast`: safe downcast with a polymorphic base; pointers get `nullptr`, references throw.
- `const_cast`: only add/remove cv-qualifiers, does not change the value type.
- `reinterpret_cast`: bitwise re-interpretation; almost always UB without strict conditions.
- Honestly: if casts are constant, the problem is often API design, not operator choice.

## How it works

`static_cast` truncates fractions on `double`→`int` and checks basic inheritance rules. `dynamic_cast` needs at least one virtual function in the base and uses RTTI.

```cpp
#include <iostream>

struct PolyBase {
    virtual ~PolyBase() = default;
};

struct PolyDerived : PolyBase {};

int main() {
    const double pi = 3.14;
    const int whole = static_cast<int>(pi);

    PolyBase* base_ptr = new PolyDerived();
    auto* derived_ptr = dynamic_cast<PolyDerived*>(base_ptr);

    if (derived_ptr) {
        std::cout << "downcast ok, int part=" << whole << '\n';
    }

    delete base_ptr;
    return 0;
}
```

`const_cast` removes const only when the original object was not truly const; otherwise writes are UB. `reinterpret_cast` reads `int` bytes as `char` with no endianness or strict aliasing guarantees.

## Pitfalls

- **C-style cast.** Masks any of the four casts and hides danger.
- **dynamic_cast without virtual.** Won't compile or is meaningless for non-polymorphic bases.
- **const_cast on truly const.** Writing through stripped const on a literal or const object is UB.
- **reinterpret_cast and alignment.** Misaligned reads break on some architectures.
- **static_cast downcast without checks.** Faster than RTTI, but wrong types surface later as UB.
