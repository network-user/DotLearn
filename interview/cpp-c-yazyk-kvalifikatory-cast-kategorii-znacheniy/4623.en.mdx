# What is a smart pointer?

## In short

A smart pointer is an RAII wrapper around a raw pointer: destructor or reset runs delete/a custom deleter. `unique_ptr`, `shared_ptr`, and `weak_ptr` model exclusive vs shared ownership.

## How to answer in an interview

- `unique_ptr`: single owner, move-only, prefer `make_unique`.
- `shared_ptr`: refcount plus control block; `make_shared` avoids two allocations.
- `weak_ptr`: non-owning observer; call `lock()` before use.
- They replace manual `new/delete` in most owning scenarios.
- Honestly: not for every pointer; stack/array/subobject ownership needs no heap wrapper.

## How it works

The smart pointer destructor frees the resource when ownership is active.

```cpp
#include <iostream>
#include <memory>

struct Beacon {
    Beacon() { std::cout << "on\n"; }
    ~Beacon() { std::cout << "off\n"; }
    void ping() const { std::cout << "ping\n"; }
};

int main() {
    auto sole = std::make_unique<Beacon>();
    sole->ping();

    std::shared_ptr<Beacon> shared = std::make_shared<Beacon>();
    std::weak_ptr<Beacon> observer = shared;

    if (auto locked = observer.lock()) {
        locked->ping();
    }
    return 0;
}
```

`sole` is destroyed at scope end. `shared` lives while strong count > 0; `weak` does not extend lifetime.

## Pitfalls

- **shared_ptr cycles.** Parent/child both shared → leak; break with weak.
- **get() escape.** Stored raw outlives the smart pointer → UAF.
- **Array delete mismatch.** `unique_ptr<T>` vs `unique_ptr<T[]>`.
- **Custom deleter size.** `unique_ptr<T, Deleter>` affects layout and conversions.
- **make_shared exception safety.** Better than `shared_ptr(new T)` in complex expressions.
