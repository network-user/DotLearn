# What does mutable mean in C++?

## In short

`mutable` on a data member allows modifying it even inside a `const` member function. Use it for physical state outside logical const API: caches, mutexes, statistics, lazy init.

## How to answer in an interview

- Logical const: the method does not change observable client behavior.
- `mutable std::mutex` in a const method for thread-safe lazy init.
- `mutable` cache after expensive work in a const getter.
- Do not abuse it: business fields should not become mutable to hack const-correctness.
- Honestly: many mutable members may mean the const interface is wrong.

## How it works

Method constness does not apply to mutable members.

```cpp
#include <iostream>
#include <string>

class Greeting {
    std::string text;
    mutable int query_count = 0;

public:
    explicit Greeting(std::string msg) : text(std::move(msg)) {}

    const std::string& message() const {
        ++query_count;
        return text;
    }

    int queries() const { return query_count; }
};

int main() {
    const Greeting hi("hello");
    std::cout << hi.message() << ' ' << hi.queries() << '\n';
    std::cout << hi.message() << ' ' << hi.queries() << '\n';
    return 0;
}
```

`message()` is const yet the counter grows; the client sees a stable string while internal metrics change.

## Pitfalls

- **Data races on mutable without a mutex.** const methods are not thread-safe by default.
- **Cache invalidation.** Forgetting flush on non-const mutation paths.
- **mutable reference members.** Rare; lifetime still matters.
- **Serialization.** mutable stats may not belong in snapshots.
- **Replacing design.** A separate stats object is sometimes cleaner.
