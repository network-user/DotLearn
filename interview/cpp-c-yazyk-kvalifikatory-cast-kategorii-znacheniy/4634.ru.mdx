# Чем отличается простое присваивание от составного?

## Суть

Оператор `=` копирует или перемещает правую часть в левый операнд. Составные формы (`+=`, `-=`, `*=`, `/=`, `%=`, побитовые и сдвиговые) сначала выполняют операцию с текущим значением левой части, затем записывают результат обратно. Для пользовательских типов обе формы могут вызывать разные перегрузки.

## Что ответить на собесе

- `a = b` всегда заменяет значение `a`; `a += b` семантически «изменить `a` на месте».
- Для встроенных типов `a += b` эквивалентно `a = a + b`, но может избежать лишней копии у тяжёлых объектов.
- У классов `operator+=` часто дешевле, чем `operator=` после временного `a + b`.
- Составные операторы работают и с перегрузками для своих типов (`std::string`, контейнеры).
- Честно: для целых чисел компилятор обычно генерирует тот же код; выигрыш заметен на строках и контейнерах.

## Как это работает

```cpp
int score = 10;
score = 25;
score += 5;

std::string label = "dot";
label += ".learn";

class Counter {
public:
    Counter& operator+=(int delta) {
        value_ += delta;
        return *this;
    }
    Counter& operator=(int v) {
        value_ = v;
        return *this;
    }
    int value() const { return value_; }
private:
    int value_ = 0;
};

Counter tally;
tally = 3;
tally += 2;
```

`score = 25` полностью перезаписывает. `score += 5` прибавляет к текущему. У `std::string` `+=` дописывает без создания промежуточной строки для `label + ".learn"`. В `Counter` `operator+=` мутирует объект и возвращает ссылку для цепочек; `operator=` задаёт новое состояние с нуля.

## Подводные камни

- **Возврат ссылки.** Составные операторы обычно возвращают `T&`; простое `operator=` в C++11+ тоже, иначе ломаются цепочки.
- **Самоприсваивание.** `a = a` безопасно при корректном `operator=`; `a += a` тоже, но кастомная логика должна это учитывать.
- **Порядок вычисления.** В `a[i++] = ...` и сложных выражениях порядок операндов не гарантирован до C++17 для некоторых случаев.
- **Побитовые формы.** `&=`, `|=`, `^=` применимы к целым; путать с логическими `&&`/`||`.
- **const левая часть.** Ни одна форма не присвоит в `const` объект; ошибка на этапе компиляции.
