# Чем отличаются указатель и ссылка?

## Суть

Указатель хранит адрес и сам является отдельной переменной: может быть `nullptr`, переназначаться и участвовать в арифметике. Ссылка — псевдоним существующего объекта: обязана инициализироваться, не бывает «пустой» и после привязки не переключается на другой объект.

## Что ответить на собесе

- Ссылку беру для обязательных аргументов и возвратов без копии; указатель — когда объект может отсутствовать.
- Указатель явно показывает опциональность (`nullptr`) и владение (raw vs smart).
- Ссылка не требует разыменования в синтаксисе; меньше шума в API функций.
- Для массивов и итерации по буферу нужен указатель или итератор, не ссылка на элемент с арифметикой.
- Честно: «ссылка всегда валидна» верно только если исходный объект жив; висячая ссылка так же опасна, как висячий указатель.

## Как это работает

```cpp
int payload = 42;
int* addr = &payload;
int& alias = payload;

addr = nullptr;
// int& orphan; // ошибка: ссылка без инициализации

void bump(int& x) { ++x; }
void maybe(int* x) {
    if (x) { ++(*x); }
}

int data[] = {10, 20, 30};
int* walk = data;
++walk;
```

`addr` можно обнулить и направить на другой `int`. `alias` всегда означает тот же `payload`, пока он существует. `bump` меняет оригинал без копии. `maybe` принимает отсутствие значения. `walk` сдвигается по массиву: у ссылки такой арифметики нет.

## Подводные камни

- **Висячие alias.** Ссылка на локальную переменную или временный после выхода из scope недействительна.
- **nullptr vs ссылка.** Ссылку нельзя передать как «нет значения» без `std::optional<std::reference_wrapper>` или указателя.
- **Перегрузка ref/ptr.** Две перегрузки `void f(int&)` и `void f(int*)` путают вызовы с литералами.
- **T&& vs T&.** Rvalue-ссылка — отдельный механизм; не смешивать с lvalue-ссылкой в ответе.
- **Размер и представление.** Ссылка часто реализуется как указатель под капотом, но семантика языка другая; `sizeof` ссылки — размер объекта.
