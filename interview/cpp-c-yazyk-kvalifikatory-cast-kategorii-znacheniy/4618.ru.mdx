# Пример применения std::move

## Суть

Типичный паттерн: передать тяжёлый объект в функцию/контейнер как rvalue через `std::move`, чтобы сработал move constructor/assignment и не копировался буфер. После move исходник остаётся valid-but-empty.

## Что ответить на собесе

- `process(std::move(vec))` или `container.push_back(std::move(item))`.
- Parameter `T&&` + move inside принимает rvalue; lvalue требует explicit `std::move`.
- Return local big object often без move: RVO elides copy/move.
- `std::move` не делает объект «волшебно быстрым» без move ops у типа.
- Честно: лишний `std::move` на return local иногда блокирует NRVO.

## Как это работает

`std::vector` move constructor переносит pointer/size/capacity; donor обнуляется.

```cpp
#include <iostream>
#include <utility>
#include <vector>

void ingest(std::vector<int>&& payload) {
    std::cout << "received size=" << payload.size() << '\n';
}

int main() {
    std::vector<int> buffer{1, 2, 3, 4, 5};
    ingest(std::move(buffer));
    std::cout << "donor size=" << buffer.size() << '\n';
    return 0;
}
```

После вызова `buffer.size()` часто 0: heap массива переехал в параметр `payload`. Чтение элементов `buffer` запрещено, destroy allowed.

## Подводные камни

- **Move from const.** Copy вместо move, сюрприз в perf.
- **std::move на return.** Может помешать copy elision в C++17.
- **Moved-from std::string.** Не гарантирован empty(); только valid.
- **Concurrent move.** Два потока move одного объекта = data race.
- **Small buffer optimization.** `std::string` move может быть cheap copy всё равно для SSO.
