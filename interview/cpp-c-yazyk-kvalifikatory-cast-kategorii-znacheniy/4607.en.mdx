# Why does a lambda need variable capture?

## In short

A C++ lambda compiles to an object with `operator()`, and "fields" come from the capture list: outer variables become members of that object. Without capture the lambda only sees its parameters; with capture it reads or mutates the call site context and can keep state across invocations.

## How to answer in an interview

- I explain that capture turns the lambda into a closure with its own state.
- I distinguish `[=]`, `[&]`, `[x]`, `[&x]`, and `mutable` for mutating copies.
- I give a `std::count_if` example with a threshold from an outer variable.
- I note that capture by reference is unsafe if the lambda outlives the object; for async I prefer by value or `std::move`.
- Honestly: C++14 generic lambdas and init-capture `[v = std::move(x)]` are often cleaner than bare `[&]`.

## How it works

The compiler generates a unique functor. Capture list entries become fields: a copy for `[var]`, a reference for `[&var]`. A generic lambda (auto parameters) still requires explicit capture of outer names.

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values{1, 2, 3, 4, 5};
    int cutoff = 3;

    auto above_cutoff = [cutoff](int item) {
        return item > cutoff;
    };

    const int matched = std::count_if(values.begin(), values.end(), above_cutoff);
    std::cout << "Above " << cutoff << ": " << matched << '\n';
    return 0;
}
```

`cutoff` is copied into the closure once; the algorithm receives a callable with the threshold baked in. With `[&cutoff]` read behavior matches, but lifetime must cover all deferred calls. `mutable` allows mutating by-value captures inside the call operator.

## Pitfalls

- **Dangling with `[&]`.** A lambda posted to a queue or thread holds a reference to a stack local.
- **Implicit capture of everything.** `[=]` and `[&]` hide dependencies; reviews often require explicit names.
- **Capturing `this`.** `[=]` copies the pointer, not the object; for data members you often want `[*this]` (C++17).
- **Init-capture and move.** Without `[buf = std::move(buf)]` expensive objects get copied twice.
- **constexpr lambdas.** Capture is restricted: not everything can be evaluated at compile time.
