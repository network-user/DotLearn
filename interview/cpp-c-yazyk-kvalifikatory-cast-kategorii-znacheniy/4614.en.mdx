# Can a reference be rebound to another object?

## In short

After initialization a reference is bound for life: a later "assignment" changes the value of the same object, not the binding address. References are neither nullable nor reseat-able, unlike pointers.

## How to answer in an interview

- Answer: no, C++ has no reference reseating; only initial binding.
- `ref = other` assigns a value, it does not rebind.
- For optional rebind I use a pointer or `std::reference_wrapper` in containers with care.
- Reference parameters give value syntax without copying.
- Honestly: rvalue references and forwarding refs are different types; I do not confuse them with lvalue references.

## How it works

The compiler implements an lvalue reference as an alias; the reference has no separate storage. Assignment goes to the referenced object.

```cpp
#include <iostream>

void bump(int& slot) {
    ++slot;
}

int main() {
    int alpha = 5;
    int beta = 10;
    int& alias = alpha;

    alias = beta;
    bump(alias);

    std::cout << alpha << ' ' << beta << '\n';
    return 0;
}
```

After `alias = beta`, `alpha` becomes 10; `alias` still refers to `alpha`, not `beta`. `bump` increments `alpha` to 11.

## Pitfalls

- **Confusion with pointers.** `T*&` can be reassigned; `T&` cannot.
- **Reference to a temporary.** `int& r = 42;` only with const or extended lifetime; otherwise UB.
- **Return reference to local.** Dangling, same as with pointers.
- **reference_wrapper.** Stores a pointer internally and can reseat via wrapper assignment, not via the reference itself.
- **Range-for by reference.** `for (auto& x : container)` avoids copies but iterator invalidation rules remain.
