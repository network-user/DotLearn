# struct and const: different roles in C++

## In short

`struct` declares a user-defined type and groups fields (and methods) in one object. `const` is an immutability qualifier for variables, members, pointers, and methods. They operate at different levels: type shape versus mutation constraints.

## How to answer in an interview

- `struct` answers "how is the object laid out"; `const` answers "may it change."
- In C++ `struct` is nearly the same as `class`; the default difference is public vs private access.
- I initialize `const` members in the constructor initializer list and never mutate them later.
- A `const` method promises not to change the object's visible state (except `mutable` fields).
- Honestly: interviewers sometimes mean "struct vs class"; I clarify we are comparing a type keyword with a qualifier, not two class declaration styles.

## How it works

```cpp
struct Vec2 {
    float x = 0.f;
    float y = 0.f;

    float length() const {
        return x * x + y * y;
    }
};

struct Config {
    const int maxRetries;
    int timeoutMs;

    Config(int retries, int timeout)
        : maxRetries(retries), timeoutMs(timeout) {}
};

const Config defaults(3, 5000);
```

`Vec2` models a point; `length() const` reads fields without mutation. In `Config`, `maxRetries` is fixed at construction. The whole `defaults` object is `const`, so `timeoutMs` cannot be changed through it. `struct` alone does not make data immutable; `const` applies the restriction where needed.

## Pitfalls

- **struct vs class.** Interviews often expect that comparison; if the question mentions `const`, separate the topics explicitly.
- **const member without ctor.** Requires an initializer list; assignment in the constructor body fails.
- **mutable in const methods.** Caches and counters may change but weaken immutability claims.
- **const object calls only const methods.** Non-const overloads are unavailable without `const_cast`.
- **POD and aggregates.** A `struct` may be an aggregate with designated initializers (C++20); `const` fields break some aggregate rules.
