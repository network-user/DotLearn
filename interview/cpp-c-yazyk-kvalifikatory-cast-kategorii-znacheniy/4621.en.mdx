# What are bitwise operations in C++?

## In short

Bitwise operators (`&`, `|`, `^`, `~`, `<<`, `>>`) act on integer bit patterns: flag masks, packed fields, fast multiply/divide by powers of two. Use unsigned or signed deliberately with sign extension in mind.

## How to answer in an interview

- AND masks bits, OR sets them, XOR toggles, NOT inverts all bits.
- `<< n` multiplies by 2^n; `>> n` divides with logical or arithmetic shift.
- Flags: `flags |= MASK; if (flags & MASK)`.
- Prefer `std::uint32_t`/`unsigned` for predictable width.
- Honestly: shifting signed negatives is implementation-defined; oversized shift count is UB.

## How it works

Operands convert to a common integral type; each bit combines independently.

```cpp
#include <cstdint>
#include <iostream>

enum : std::uint8_t { Read = 1 << 0, Write = 1 << 1, Exec = 1 << 2 };

int main() {
    const std::uint8_t a = 0b0101;
    const std::uint8_t b = 0b0011;

    std::uint8_t perms = Read | Write;
    perms |= Exec;

    std::cout << (a & b) << ' '
              << (a | b) << ' '
              << (a ^ b) << ' '
              << static_cast<unsigned>(~a & 0xF) << ' '
              << (a << 1) << ' '
              << (a >> 1) << ' '
              << static_cast<unsigned>(perms) << '\n';
    return 0;
}
```

`Read | Write | Exec` builds permissions; `&` tests a bit, `^` toggles without OR side effects.

## Pitfalls

- **Shift >= width.** UB in C++ when shift amount ≥ type bit width.
- **Signed right shift.** Arithmetic vs logical platform nuances.
- **Operator precedence.** `a & b == c` parses as `a & (b == c)`; add parentheses.
- **Promotion to int.** `uint8_t` ops promote; cast back for exact-width stores.
- **Micro-optimization myths.** Modern compilers often optimize `*2` equally well.
