# Что такое умный указатель?

## Суть

Smart pointer это RAII-обёртка над raw pointer: в деструкторе (или при reset) вызывается delete/custom deleter. `unique_ptr`, `shared_ptr`, `weak_ptr` задают модель exclusive vs shared ownership.

## Что ответить на собесе

- `unique_ptr`: один владелец, move-only, `make_unique`.
- `shared_ptr`: refcount + control block, `make_shared` предпочтительнее двух аллокаций.
- `weak_ptr`: non-owning наблюдатель, `lock()` перед use.
- Заменяют ручной `new/delete` в большинстве owning сценариев.
- Честно: не для каждого указателя; stack/array/subobject ownership без heap.

## Как это работает

Деструктор smart pointer освобождает ресурс, если ownership активен.

```cpp
#include <iostream>
#include <memory>

struct Beacon {
    Beacon() { std::cout << "on\n"; }
    ~Beacon() { std::cout << "off\n"; }
    void ping() const { std::cout << "ping\n"; }
};

int main() {
    auto sole = std::make_unique<Beacon>();
    sole->ping();

    std::shared_ptr<Beacon> shared = std::make_shared<Beacon>();
    std::weak_ptr<Beacon> observer = shared;

    if (auto locked = observer.lock()) {
        locked->ping();
    }
    return 0;
}
```

`sole` уничтожается при выходе из scope. `shared` живёт пока strong count > 0; `weak` не продлевает lifetime.

## Подводные камни

- **shared_ptr cycles.** Parent/child оба shared → leak; один weak.
- **get() escape.** Сохранённый raw переживает smart pointer → UAF.
- **array delete mismatch.** `unique_ptr<T>` vs `unique_ptr<T[]>`.
- **Custom deleter size.** `unique_ptr<T, Deleter>` влияет на layout и convertibility.
- **make_shared exception safety.** Лучше чем `shared_ptr(new T)` в сложных выражениях.
