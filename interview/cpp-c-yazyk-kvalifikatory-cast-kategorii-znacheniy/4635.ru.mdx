# Struct и const: разные роли в C++

## Суть

`struct` объявляет пользовательский тип и группирует поля (и методы) в одном объекте. `const` — квалификатор неизменяемости для переменных, полей, указателей и методов. Это разные уровни языка: тип против ограничения на мутацию.

## Что ответить на собесе

- `struct` отвечает на вопрос «как устроен объект»; `const` на «можно ли его менять».
- В C++ `struct` почти равен `class`, отличие по умолчанию — public vs private.
- `const` член инициализирую в списке инициализации конструктора, позже не меняю.
- `const` метод обещает не менять видимое состояние объекта (кроме `mutable`).
- Честно: на собесе иногда путают с «struct vs class»; уточняю, что сравнивают тип и квалификатор, не два способа объявления класса.

## Как это работает

```cpp
struct Vec2 {
    float x = 0.f;
    float y = 0.f;

    float length() const {
        return x * x + y * y;
    }
};

struct Config {
    const int maxRetries;
    int timeoutMs;

    Config(int retries, int timeout)
        : maxRetries(retries), timeoutMs(timeout) {}
};

const Config defaults(3, 5000);
```

`Vec2` описывает геометрию точки; `length() const` читает поля без изменений. В `Config` поле `maxRetries` фиксируется при создании. Объект `defaults` целиком `const`: менять `timeoutMs` через него нельзя. `struct` не делает данные неизменяемыми сам по себе; `const` накладывает запрет там, где нужно.

## Подводные камни

- **struct vs class.** На собесе часто ждут сравнение с `class`; если вопрос про `const`, явно развести темы.
- **const член без ctor.** Требует initializer list; присваивание в теле конструктора не сработает.
- **mutable в const методе.** Кэш и счётчики можно менять, но это ослабляет гарантию неизменяемости.
- **const объект вызывает только const методы.** Не-const перегрузки недоступны без `const_cast`.
- **POD и агрегаты.** `struct` может быть агрегатом с designated initializers (C++20); `const` поля ломают часть агрегатных правил.
