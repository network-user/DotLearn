# Можно ли переназначить ссылку на другой объект?

## Суть

После инициализации ссылка навсегда привязана к объекту: повторное «присваивание» меняет значение того же объекта, а не адрес привязки. Ссылка не nullable и не reseat-able, в отличие от указателя.

## Что ответить на собесе

- Ответ: нет, reseat ссылки в C++ нет; только первичная инициализация.
- `ref = other` это assignment значения, не rebinding.
- Для optional rebind использую указатель или `std::reference_wrapper` в контейнере с осторожностью.
- Ссылки в параметрах функций дают синтаксис значений без копирования.
- Честно: rvalue reference и forwarding refs это другой тип; их не путаю с lvalue reference.

## Как это работает

Компилятор реализует lvalue reference как alias; storage у ссылки как отдельного объекта нет. Присваивание проходит в referenced object.

```cpp
#include <iostream>

void bump(int& slot) {
    ++slot;
}

int main() {
    int alpha = 5;
    int beta = 10;
    int& alias = alpha;

    alias = beta;
    bump(alias);

    std::cout << alpha << ' ' << beta << '\n';
    return 0;
}
```

После `alias = beta` значение `alpha` становится 10; `alias` всё ещё ссылается на `alpha`, не на `beta`. `bump` увеличивает `alpha` до 11.

## Подводные камни

- **Путаница с указателем.** `T*&` можно переназначить; `T&` нельзя.
- **Ссылка на temporary.** `int& r = 42;` только с const или продлённым lifetime; иначе UB.
- **Return reference to local.** Dangling так же, как у указателя.
- **reference_wrapper.** Хранит pointer под капотом и может reseat через assign wrapper, не через саму ссылку.
- **Range-for by reference.** `for (auto& x : container)` безопаснее, чем копия, но iterator invalidation те же.
