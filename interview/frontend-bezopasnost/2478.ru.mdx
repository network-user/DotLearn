# Шифруются ли cookies при HTTPS

## Суть

TLS шифрует весь HTTP-трафик, включая заголовки Cookie и Set-Cookie, на wire. Это confidentiality in transit. Cookie value itself not encrypted at rest in browser store; flags HttpOnly/Secure/SameSite задают правила использования, не шифрование содержимого.

## Что ответить на собесе

- HTTPS + Secure flag: cookie не уйдёт по plain HTTP.
- TLS защищает от passive sniffing; MITM блокирую HSTS + valid cert chain.
- Cookie payload readable in DevTools Application tab; не кладу secrets без need.
- Честно: TLS не защищает от XSS reading non-HttpOnly cookies via `document.cookie`.
- __Host- prefix cookies enforce Secure, Path=/, no Domain (stricter).

```http
Set-Cookie: sid=opaque-id; Secure; HttpOnly; SameSite=Lax; Path=/
```

Browser stores cookie jar per domain. On request matching domain/path/https rules, adds `Cookie: sid=opaque-id` header inside TLS record.

Encryption != integrity alone: TLS also authenticates server cert. HSTS (`Strict-Transport-Security`) prevents sslstrip downgrade on repeat visits.

## Подводные камни

- **Mixed content.** HTTP subresource on HTTPS page does not encrypt cookies automatically for that subresource.
- **Secure false on staging.** Cookie set without Secure leaks on accidental http:// link.
- **Subdomain leakage.** Domain=.corp.com sends cookie to all subdomains.
- **JS-readable session.** Without HttpOnly XSS exfiltrates despite TLS.
- **Local network proxies.** Corporate SSL inspection breaks pin expectations; document for users.
