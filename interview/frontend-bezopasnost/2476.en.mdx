# Authentication in web applications

## In short

Authentication answers «who are you?»: verifying identity via password, OTP, WebAuthn, or SSO. It precedes authorization (what you may do). On the frontend I collect credentials, send them over TLS, receive session/token; access decisions stay off the client.

## How to answer in an interview

- I separate authentication vs authorization vs identification.
- Passwords only via POST over HTTPS; no plaintext storage on client; optional client-side hash only if spec requires.
- MFA/WebAuthn reduces phishing; push OTP is weaker than FIDO2.
- Honestly: «remember me» widens compromise window; refresh rotation is mandatory.
- Generic login errors («invalid credentials»), not «user not found».

## How it works

```ts
async function submitLogin(form: { email: string; password: string }) {
  const res = await fetch('/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(form),
  });
  if (!res.ok) throw new Error('invalid credentials');
  return res.json() as Promise<{ mfaRequired?: boolean }>;
}

async function verifyTotp(code: string) {
  await fetch('/auth/mfa', {
    method: 'POST',
    credentials: 'include',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ code }),
  });
}
```

Factors: knowledge (password), possession (OTP key), inherence (biometric via WebAuthn). Passwordless passkeys use public-key crypto and resist phishing.

After success the server issues session cookie or tokens. Frontend route guards call `/me`, not a local «isLoggedIn=true» flag without verification.

## Pitfalls

- **Client-only auth gate.** `if (localStorage.token)` without server verify is fake security.
- **Credential stuffing.** Rate limits and CAPTCHA on login UI coordinated with API.
- **OAuth login button.** Social login still needs account linking policy.
- **Autofill leaks.** Login forms in hidden iframes for scraping.
- **Logout everywhere.** Forgotten tabs with valid refresh after password reset.
