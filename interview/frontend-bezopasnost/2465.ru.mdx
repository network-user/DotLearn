# Какие ресурсы ограничивает Content Security Policy

## Суть

CSP это whitelist источников для загрузки и исполнения контента на странице. Директивы вроде `script-src`, `style-src`, `img-src`, `connect-src`, `frame-ancestors` задают откуда можно тянуть JS, CSS, картинки, XHR/fetch, iframe. `default-src` задаёт fallback для не указанных типов.

## Что ответить на собесе

- Начинаю с report-only, смотрю violations в Reporting API, потом enforce.
- `script-src 'self' nonce-...` вместо blanket `unsafe-inline`.
- `connect-src` ограничивает fetch/WebSocket; часто забывают analytics endpoint.
- Честно: `'unsafe-inline'` для style часто нужен legacy UI; для script стараюсь без него.
- `frame-ancestors` только в HTTP header, не в meta.

## Как это работает

```http
Content-Security-Policy:
  default-src 'self';
  script-src 'self' https://cdn.example;
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  font-src 'self' https://fonts.example;
  connect-src 'self' https://api.example wss://live.example;
  media-src 'self';
  object-src 'none';
  worker-src 'self';
  frame-src https://embed.example;
  frame-ancestors 'self';
  base-uri 'self';
  form-action 'self';
```

Ключевые директивы:

| Директива | Что контролирует |
|-----------|------------------|
| `script-src` | external + inline scripts |
| `style-src` | CSS files и inline style |
| `img-src` | `<img>`, favicon, некоторые CSS url |
| `connect-src` | fetch, XHR, EventSource, WebSocket |
| `font-src` | `@font-face` |
| `frame-src` | `<iframe>` который **вы** встраиваете |
| `frame-ancestors` | кто встраивает **вас** |

`'none'`, `'self'`, host sources, nonce/hash для inline script. Violation → browser блокирует load/eval и шлёт report если настроен `report-uri`/`report-to`.

## Подводные камни

- **Meta tag limits.** `frame-ancestors`, `report-uri` в meta игнорируются.
- **Third-party GTM.** Один контейнер тянет десятки доменов; CSP становится дырявым.
- **Webpack inline.** Без nonce/hash dev HMR ломается; разные политики dev/prod.
- **blob:/data:.** Нужны явно если используете workers или canvas export.
- **Upgrade-insecure-requests.** Ломает mixed content legacy, но скрывает проблемы вместо фикса.
