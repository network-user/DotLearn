# Which resources Content Security Policy can restrict

## In short

CSP is a whitelist of sources for loading and executing content on a page. Directives like `script-src`, `style-src`, `img-src`, `connect-src`, and `frame-ancestors` define where JS, CSS, images, XHR/fetch, and iframes may come from. `default-src` is the fallback for unspecified types.

## How to answer in an interview

- I start report-only, review violations via Reporting API, then enforce.
- I use `script-src 'self' nonce-...` instead of blanket `unsafe-inline`.
- `connect-src` limits fetch/WebSocket; teams often forget analytics endpoints.
- Honestly: `'unsafe-inline'` for styles is common in legacy UI; I avoid it for scripts.
- `frame-ancestors` works only in HTTP headers, not meta tags.

## How it works

```http
Content-Security-Policy:
  default-src 'self';
  script-src 'self' https://cdn.example;
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  font-src 'self' https://fonts.example;
  connect-src 'self' https://api.example wss://live.example;
  media-src 'self';
  object-src 'none';
  worker-src 'self';
  frame-src https://embed.example;
  frame-ancestors 'self';
  base-uri 'self';
  form-action 'self';
```

Key directives:

| Directive | Controls |
|-----------|----------|
| `script-src` | external and inline scripts |
| `style-src` | CSS files and inline style |
| `img-src` | `<img>`, favicon, some CSS urls |
| `connect-src` | fetch, XHR, EventSource, WebSocket |
| `font-src` | `@font-face` |
| `frame-src` | `<iframe>` you embed |
| `frame-ancestors` | who embeds you |

Sources include `'none'`, `'self'`, hosts, nonce/hash for inline script. Violations block load/eval and send reports when `report-uri`/`report-to` is configured.

## Pitfalls

- **Meta tag limits.** `frame-ancestors` and `report-uri` are ignored in meta.
- **Third-party GTM.** One container pulls many domains; CSP gets porous.
- **Webpack inline.** Without nonce/hash dev HMR breaks; split dev/prod policies.
- **blob:/data:.** Must be explicit for workers or canvas export.
- **Upgrade-insecure-requests.** Fixes mixed content silently instead of surfacing bugs.
