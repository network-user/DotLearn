# XSS on the frontend: types and defenses

## In short

Cross-Site Scripting runs attacker JavaScript in your origin context. Stored XSS lives in server data, Reflected bounces from a request parameter, DOM-based XSS comes from unsafe URL/DOM handling on the client. Defenses: encoding, CSP, safe render APIs, sanitization.

## How to answer in an interview

- I render user text via `textContent` or framework escaping, not `innerHTML`.
- I enable CSP: `script-src` without `unsafe-inline`, nonce/hash for inline when unavoidable.
- For CMS HTML I run allowlist sanitizers (DOMPurify), not regex.
- Honestly: CSP does not replace encoding; stored XSS hits everyone, DOM XSS bypasses server WAF.
- I keep dependencies updated and enable Trusted Types where the browser supports it.

## How it works

Three attack classes:

1. **Stored:** a comment with `<script>` is saved and served to every viewer.
2. **Reflected:** the server embeds a search query in HTML without escaping.
3. **DOM-based:** client code reads `location.hash` and writes to the DOM.

```ts
function escapeHtml(raw: string): string {
  return raw
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&#39;');
}

function renderGreeting(nameFromQuery: string): void {
  const slot = document.querySelector('#greet');
  if (!slot) return;
  slot.textContent = nameFromQuery;
}

const meta = document.createElement('meta');
meta.httpEquiv = 'Content-Security-Policy';
meta.content = "default-src 'self'; script-src 'self'; object-src 'none'; base-uri 'self'";
document.head.appendChild(meta);
```

React/Vue/Svelte escape `{expr}` by default; risk sits in `dangerouslySetInnerHTML` and markdown libs without sanitize. HttpOnly cookies do not stop XSS once script runs in the DOM: same-origin fetch still acts as the user.

## Pitfalls

- **JSON in script tags.** `</script>` inside JSON breaks parsing; use `type="application/json"` plus separate parse.
- **href/src injection.** `javascript:` and `data:` in attributes enable XSS without `<script>`.
- **postMessage.** Listeners without `origin` checks accept iframe payloads.
- **Third-party scripts.** GTM/widgets widen attack surface; start with CSP report-only.
- **Markdown/HTML editors.** Preview panes often render raw HTML; sanitize on save and render.
