# Roadmap

## Phase 0 — Foundation (this commit)

- [x] Monorepo skeleton (pnpm workspaces + Turborepo)
- [x] `packages/contracts` with Zod topic schema
- [x] `apps/web` Vite + React scaffold
- [x] `apps/api` NestJS scaffold with submissions module
- [x] `.cursor/skills/lesson-forge` with full contract docs and templates
- [x] One demo topic (`sql-fundamentals`) to validate the contract
- [x] Topic submission UI placeholder + admin route placeholder

## Phase 1 — Minimum Viable Player (1–2 weeks)

- [ ] Real topic loader (`packages/lesson-engine`) using `import.meta.glob`
- [ ] Theory rendering with MDX and Shiki syntax highlighting
- [ ] First sandbox: `sql.js` in a Web Worker; exercise runner that diffs result-sets
- [ ] Per-exercise progress saved in IndexedDB (Dexie)
- [ ] FSRS spaced repetition for flashcards
- [ ] Validator CLI: `pnpm validate` walks `topics/**`, runs each gold solution, fails on mismatch

## Phase 2 — Codegen loop (1–2 weeks)

- [ ] `lesson-forge` integrated end-to-end: agent generates → validator passes → PR opened
- [ ] Second topic generated by the skill alone (e.g. Python OOP), proving the contract works
- [ ] Pyodide runtime adapter
- [ ] AI providers abstraction (`packages/ai-providers`) with OpenAI, Anthropic, Ollama, OpenRouter; settings UI for BYOK
- [ ] Editor (Monaco) inside the exercise pane

## Phase 3 — Submissions + admin (1 week)

- [ ] `apps/api` submissions endpoint: create + list + review (approve/reject)
- [ ] In-app "+ Add topic" dialog wired to the API
- [ ] `/admin` page in `apps/web` listing pending submissions
- [ ] Email notifications when a submission is approved (later)
- [ ] GitHub Action: on PR touching `topics/**`, run `pnpm validate` and label the PR

## Phase 4 — Polish + community (open-ended)

- [ ] PWA / offline support
- [ ] Topic catalog page with search + tags
- [ ] LOOM-style knowledge graph linking concepts across topics
- [ ] Export/import progress as JSON
- [ ] Public catalog of community-approved topics (read from main branch)

## Explicitly NOT in scope for MVP

- Authentication / user accounts (BYOK keeps things stateless)
- Monetization (see README; BYOK eliminates AI cost passthrough)
- Marketplace for paid topics
- Multi-language UI (English-only first; the contract supports `language` field already)
- Mobile apps
- Real Postgres / Judge0 server-side sandboxes (Pyodide + sql.js cover 95% of needs)
