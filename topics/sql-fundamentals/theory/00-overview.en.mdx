Application data almost always lives in tables, and almost any question you ask of it - "how many orders were placed yesterday", "which customers haven't come back in a while", "which products bring in revenue" - comes down in practice to a single query. SQL is not a separate language you master from scratch: it is a way to describe *which slice of the data* you want and hand the work of assembling that slice to the database itself. You say what you want back; how to pull it off the disk is the engine's problem.

**What you'll be able to do.** Pull exactly the rows and columns you need out of a table; sort a result and take only its top; collapse thousands of rows into a single number (sum, average, count); join data from several tables into one result; compute figures per group; and assemble multi-step answers from nested queries.

**The route through this topic.** Six sections, each with a live sandbox where you can change the query and run it right in the browser:

1. **`SELECT` and filtering** - read the slice of a table you need and keep rows by condition.
2. **`ORDER BY` and `LIMIT`** - order the result and take only the first rows.
3. **Aggregation** - fold many rows into a total: `COUNT`, `SUM`, `AVG`.
4. **`JOIN`** - connect rows from different tables by a shared key.
5. **`GROUP BY` and `HAVING`** - compute totals per group rather than across the whole table.
6. **Subqueries and CTEs** - nest one query inside another and break a hard question into steps.

**What you need first.** Nothing to install: the database (SQLite) runs right on the page. It's enough to understand that data can be laid out in rows and columns - we'll pick up everything else along the way, moving from a simple query to compound ones.
