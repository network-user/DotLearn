# Агрегация с COUNT, SUM, AVG

Агрегатные функции сворачивают много строк в одно число. Они отвечают на вопросы вроде «сколько пользователей», «какой средний чек заказа», «каков максимальный возраст».

## Четыре основных инструмента

```sql
SELECT COUNT(*) FROM users;
SELECT SUM(amount) FROM orders;
SELECT AVG(amount) FROM orders;
SELECT MIN(age), MAX(age) FROM users;
```

`COUNT(*)` считает строки, включая те, где есть `NULL`. `COUNT(column)` считает только не-NULL значения этой колонки.

## GROUP BY

Агрегация становится интересной, когда строки разбиваются на группы по колонке.

```sql
SELECT country, COUNT(*) AS users_count
FROM users
GROUP BY country;
```

Каждая колонка в `SELECT`, которая **не** обёрнута в агрегат, обязана присутствовать в `GROUP BY`.

## HAVING

`WHERE` фильтрует строки до агрегации. `HAVING` фильтрует группы после.

```sql
SELECT country, COUNT(*) AS users_count
FROM users
GROUP BY country
HAVING COUNT(*) >= 10;
```

## Когда это применять

Везде, где нужна сводка: дашборды, отчёты, проверки качества данных, алерты.

## Типичные ошибки

- Добавить неагрегированную колонку в `SELECT`, не включив её в `GROUP BY`. Движки могут вернуть произвольное значение для такой колонки.
- Написать `WHERE COUNT(*) > 10`. `WHERE` выполняется до агрегации; правильное место - `HAVING`.
- Забыть, что `AVG` возвращает среднее с плавающей точкой; результат не совпадает с `SUM / COUNT` при разных типах.
