# ORDER BY и LIMIT

Без `ORDER BY` движок SQL вправе вернуть строки в любом порядке. Если вывод выглядит отсортированным - это совпадение, на которое нельзя полагаться. Сортировка задаётся явно.

## ORDER BY

```sql
SELECT * FROM users ORDER BY age;
```

По умолчанию сортировка по возрастанию. Для убывания используйте `DESC`.

```sql
SELECT * FROM users ORDER BY age DESC;
```

Сортировать можно по нескольким колонкам. Второй ключ работает только при равенстве первого.

```sql
SELECT * FROM users ORDER BY country ASC, age DESC;
```

## LIMIT

`LIMIT` ограничивает размер результата. Полезен для топ-N запросов и пагинации.

```sql
SELECT * FROM users ORDER BY age DESC LIMIT 3;
```

Без `ORDER BY` `LIMIT` возвращает произвольное подмножество. Почти всегда это ошибка.

## Когда это применять

Каждый раз, когда Вы показываете пользователю список. Топ-N отчёты, ленты «последние 10», постраничные таблицы - всё строится на `ORDER BY` + `LIMIT`.

## Типичные ошибки

- Использовать `LIMIT` без `ORDER BY` и ожидать детерминированный результат.
- Забыть, что порядок `NULL` зависит от движка: SQLite по умолчанию ставит `NULL` первым, PostgreSQL - последним.
- Пагинация через `OFFSET` на больших таблицах (она замедляется). С ростом данных предпочтительна keyset-пагинация, но это тема для позже.
