# SELECT and filtering

By the end of this section you can read specific columns from a table and keep only the rows you care about. This is the read half of CRUD — the operation you'll perform a thousand times a week.

## The shape of SELECT

A `SELECT` always names what to return and where to read from.

```sql
SELECT name, age FROM users;
```

This returns two columns for every row in `users`. To return all columns, use `*`.

```sql
SELECT * FROM users;
```

## Filtering with WHERE

`WHERE` runs before the projection; it decides which rows survive.

```sql
SELECT * FROM users WHERE age >= 18;
```

You can combine conditions with `AND` and `OR`. Group with parentheses when in doubt.

```sql
SELECT * FROM users
WHERE (age >= 18 AND country = 'PL') OR is_admin = 1;
```

## When to use this

`SELECT ... WHERE` is the building block of every read query. The remaining tools — ordering, aggregation, joins — wrap around this base.

## Common mistakes

- Using `=` to compare against `NULL`. `WHERE column = NULL` matches nothing; use `IS NULL` instead.
- Forgetting that string comparisons are case-sensitive in most engines. `WHERE name = 'anna'` will not match `'Anna'`.
- Putting `WHERE` after `ORDER BY`. The order is `SELECT ... FROM ... WHERE ... ORDER BY ...`.
