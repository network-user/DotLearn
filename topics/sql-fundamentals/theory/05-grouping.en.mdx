# GROUP BY and HAVING

`GROUP BY` folds many rows into one row per group. `HAVING` filters those
groups after aggregation — it's `WHERE` for aggregates.

## One row per bucket

<SideSql
  title="Books per author"
  fixture={`
    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author TEXT NOT NULL,
      pages INTEGER NOT NULL
    );
    INSERT INTO book VALUES
      (1, 'Compilers',           'Ada',   620),
      (2, 'Algorithms',          'Ada',   480),
      (3, 'Notes on math',       'Ada',    90),
      (4, 'Operating systems',   'Linus', 700),
      (5, 'Kernel quickstart',   'Linus', 120),
      (6, 'COBOL essentials',    'Grace', 220);
  `}
  query={`
    SELECT author, COUNT(*) AS books, SUM(pages) AS total_pages
    FROM book
    GROUP BY author
    ORDER BY author;
  `}
/>

Every column in `SELECT` must either appear in `GROUP BY` or be wrapped in
an aggregate (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`).

## HAVING — filter after the fold

`WHERE` filters rows **before** grouping. `HAVING` filters **after**.

<SideSql
  title="Authors with more than one long book"
  fixture={`
    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author TEXT NOT NULL,
      pages INTEGER NOT NULL
    );
    INSERT INTO book VALUES
      (1, 'Compilers',           'Ada',   620),
      (2, 'Algorithms',          'Ada',   480),
      (3, 'Notes on math',       'Ada',    90),
      (4, 'Operating systems',   'Linus', 700),
      (5, 'Kernel quickstart',   'Linus', 120),
      (6, 'COBOL essentials',    'Grace', 220);
  `}
  query={`
    SELECT author, COUNT(*) AS long_books
    FROM book
    WHERE pages >= 400
    GROUP BY author
    HAVING COUNT(*) >= 2
    ORDER BY author;
  `}
/>

Trace the order: filter long books with `WHERE`, fold by author, then keep
only authors with at least two long ones.

<Callout type="tip" title="WHERE vs HAVING">
If the filter can be expressed without an aggregate, prefer `WHERE` — it
runs earlier and on fewer rows. Reserve `HAVING` for predicates on
aggregates (`HAVING SUM(x) > 100`).
</Callout>

## Common COUNT pitfalls

- `COUNT(*)` counts rows including NULLs.
- `COUNT(col)` counts non-NULL values of `col` — they differ when `col` has NULLs.
- `COUNT(DISTINCT col)` counts unique non-NULL values.
