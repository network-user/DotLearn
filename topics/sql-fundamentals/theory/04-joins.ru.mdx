# JOIN-ы

`JOIN` склеивает две таблицы строка к строке по условию, которое Вы
задаёте. Форма результата зависит от того, какой JOIN Вы выберете.

Переключайте тип JOIN и следите, какие строки попадают в результат, а какие
остаются «без пары». Цветная точка - значение ключа соединения.

## Исходные данные

<SideSql

    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

## INNER JOIN - только совпадения

`INNER JOIN` оставляет только строки, для которых условие истинно с обеих сторон.
Книга без автора и автор без книг исчезают из результата.

<SideSql
  title="INNER JOIN"
  fixture={`
    CREATE TABLE author (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL
    );
    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author_id INTEGER REFERENCES author(id)
    );
    INSERT INTO book VALUES
      (1, 'Analytical engine', 1),
      (2, 'Kernel notes', 2),
      (3, 'Loose pages', NULL);
  `}
  query={`
    SELECT a.name, b.title
    FROM author a
    INNER JOIN book b ON b.author_id = a.id
    ORDER BY a.name;
  `}
/>

Grace и «Loose pages» отсутствуют: у Grace нет книги, а у «Loose pages»
нет автора.

## LEFT JOIN - сохранить левую сторону, NULL справа

`LEFT JOIN` сохраняет каждую строку левой таблицы, даже если совпадения нет,
заполняя колонки правой стороны значением `NULL`.

<SideSql
  title="LEFT JOIN"
  fixture={`
    CREATE TABLE author (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL
    );
    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author_id INTEGER REFERENCES author(id)
    );
    INSERT INTO book VALUES
      (1, 'Analytical engine', 1),
      (2, 'Kernel notes', 2),
      (3, 'Loose pages', NULL);
  `}
  query={`
    SELECT a.name, b.title
    FROM author a
    LEFT JOIN book b ON b.author_id = a.id
    ORDER BY a.name;
  `}
/>

Grace вернулась - с `NULL` вместо названия книги.

## Анти-JOIN: «левое без правого»

Сочетание `LEFT JOIN` с `WHERE right.col IS NULL` даёт строки левой
стороны, у которых **нет совпадения** справа. Авторы без книг:

```sql
SELECT a.name
FROM author a
LEFT JOIN book b ON b.author_id = a.id
WHERE b.id IS NULL;
```

<Callout type="tip" title="Ставьте интересующую сторону слева">
Анти-JOIN читается ясно, когда сторона, которая Вас интересует («у каких
авторов нет книг?»), стоит слева. Тогда форма результата отражает сам
вопрос.
</Callout>

## Подводные камни

- `JOIN` без `ON` - это перекрёстное соединение, дающее декартово произведение. Большое.
- С `LEFT JOIN` важно, где стоит фильтр - в `ON` или в `WHERE`: `WHERE`
  срабатывает после соединения и может случайно превратить его обратно в INNER JOIN.
- `SELECT *` после JOIN сталкивается на одинаковых именах колонок. Уточняйте
  через алиасы.
