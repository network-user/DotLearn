# Aggregation with COUNT, SUM, AVG

Aggregate functions collapse many rows into one number. They answer questions like "how many users", "what's the average order value", "what's the maximum age".

## The four staples

```sql
SELECT COUNT(*) FROM users;
SELECT SUM(amount) FROM orders;
SELECT AVG(amount) FROM orders;
SELECT MIN(age), MAX(age) FROM users;
```

`COUNT(*)` counts rows including those with `NULL`. `COUNT(column)` counts only non-null values of that column.

## GROUP BY

Aggregation gets interesting when you partition rows by a column.

```sql
SELECT country, COUNT(*) AS users_count
FROM users
GROUP BY country;
```

Every column in `SELECT` that is **not** inside an aggregate must appear in `GROUP BY`.

## HAVING

`WHERE` filters rows before aggregation. `HAVING` filters groups after.

```sql
SELECT country, COUNT(*) AS users_count
FROM users
GROUP BY country
HAVING COUNT(*) >= 10;
```

## When to use this

Anywhere you need to summarize: dashboards, reports, sanity checks on data quality, alerts.

## Common mistakes

- Adding a non-aggregated column to `SELECT` without putting it in `GROUP BY`. Engines may return arbitrary values for that column.
- Using `WHERE COUNT(*) > 10`. `WHERE` runs before aggregation; the right place is `HAVING`.
- Forgetting that `AVG` returns the floating-point average; the result is not the same as `SUM / COUNT` when types differ.
