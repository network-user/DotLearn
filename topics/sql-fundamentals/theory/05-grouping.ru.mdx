# GROUP BY и HAVING

`GROUP BY` сворачивает много строк в одну строку на группу. `HAVING`
фильтрует эти группы после агрегации - это `WHERE` для агрегатов.

Нажмите «Сгруппировать» и посмотрите, как строки разлетаются по корзинам
своего автора, а у каждой корзины появляется агрегат:

## Одна строка на группу

<SideSql
  title="Книги по авторам"
  fixture={`
    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author TEXT NOT NULL,
      pages INTEGER NOT NULL
    );
    INSERT INTO book VALUES
      (1, 'Compilers',           'Ada',   620),
      (2, 'Algorithms',          'Ada',   480),
      (3, 'Notes on math',       'Ada',    90),
      (4, 'Operating systems',   'Linus', 700),
      (5, 'Kernel quickstart',   'Linus', 120),
      (6, 'COBOL essentials',    'Grace', 220);
  `}
  query={`
    SELECT author, COUNT(*) AS books, SUM(pages) AS total_pages
    FROM book
    GROUP BY author
    ORDER BY author;
  `}
/>

Каждая колонка в `SELECT` обязана либо присутствовать в `GROUP BY`, либо
быть обёрнутой в агрегат (`COUNT`, `SUM`, `AVG`, `MIN`, `MAX`).

## HAVING - фильтр после свёртки

`WHERE` фильтрует строки **до** группировки. `HAVING` - **после**.

<SideSql
  title="Авторы с более чем одной длинной книгой"
  fixture={`
    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author TEXT NOT NULL,
      pages INTEGER NOT NULL
    );
    INSERT INTO book VALUES
      (1, 'Compilers',           'Ada',   620),
      (2, 'Algorithms',          'Ada',   480),
      (3, 'Notes on math',       'Ada',    90),
      (4, 'Operating systems',   'Linus', 700),
      (5, 'Kernel quickstart',   'Linus', 120),
      (6, 'COBOL essentials',    'Grace', 220);
  `}
  query={`
    SELECT author, COUNT(*) AS long_books
    FROM book
    WHERE pages >= 400
    GROUP BY author
    HAVING COUNT(*) >= 2
    ORDER BY author;
  `}
/>

Проследите порядок: сначала `WHERE` отбирает длинные книги, потом свёртка
по автору, и только затем остаются авторы хотя бы с двумя длинными книгами.

<Callout type="tip" title="WHERE против HAVING">
Если фильтр можно выразить без агрегата, предпочитайте `WHERE` - он
выполняется раньше и на меньшем числе строк. Оставьте `HAVING` для
предикатов над агрегатами (`HAVING SUM(x) > 100`).
</Callout>

## Типичные ловушки COUNT

- `COUNT(*)` считает строки, включая NULL.
- `COUNT(col)` считает не-NULL значения `col` - результаты различаются, когда в `col` есть NULL.
- `COUNT(DISTINCT col)` считает уникальные не-NULL значения.
