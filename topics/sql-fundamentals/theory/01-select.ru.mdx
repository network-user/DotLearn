# SELECT и фильтрация

К концу этого раздела Вы сможете читать нужные колонки из таблицы и оставлять только те строки, которые Вас интересуют. Это «читающая» половина CRUD - операция, которую Вы будете выполнять тысячу раз в неделю.

## Форма SELECT

`SELECT` всегда указывает, что вернуть и откуда читать.

```sql
SELECT name, age FROM users;
```

Этот запрос возвращает две колонки для каждой строки в `users`. Чтобы вернуть все колонки, используйте `*`.

```sql
SELECT * FROM users;
```

## Фильтрация с WHERE

`WHERE` выполняется до проекции: он решает, какие строки выживут.

```sql
SELECT * FROM users WHERE age >= 18;
```

Условия можно комбинировать через `AND` и `OR`. Если сомневаетесь - группируйте скобками.

```sql
SELECT * FROM users
WHERE (age >= 18 AND country = 'PL') OR is_admin = 1;
```

## Когда это применять

`SELECT ... WHERE` - строительный блок любого читающего запроса. Остальные инструменты - сортировка, агрегация, JOIN - надстраиваются над этой основой.

## Типичные ошибки

- Сравнение с `NULL` через `=`. Условие `WHERE column = NULL` не находит ничего; используйте `IS NULL`.
- Забыть, что сравнение строк в большинстве движков чувствительно к регистру. `WHERE name = 'anna'` не найдёт `'Anna'`.
- Поставить `WHERE` после `ORDER BY`. Правильный порядок: `SELECT ... FROM ... WHERE ... ORDER BY ...`.
