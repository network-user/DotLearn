# JOINs

A `JOIN` glues two tables together row by row, using a condition you
specify. The shape of the result depends on which join you pick.

Switch the join type and watch which rows make it into the result and which
end up without a match. The colored dot is the join-key value.

## The setup

<SideSql
  title="Two related tables"
  fixture={`
    CREATE TABLE author (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL
    );
    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author_id INTEGER REFERENCES author(id)
    );
    INSERT INTO book VALUES
      (1, 'Analytical engine', 1),
      (2, 'Kernel notes', 2),
      (3, 'Loose pages', NULL);
  `}
/>

## INNER JOIN — only matches

`INNER JOIN` keeps only the rows where the condition is true on both sides.
A book with no author and an author with no books both disappear.

<SideSql
  title="INNER JOIN"
  fixture={`
    CREATE TABLE author (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL
    );
    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author_id INTEGER REFERENCES author(id)
    );
    INSERT INTO book VALUES
      (1, 'Analytical engine', 1),
      (2, 'Kernel notes', 2),
      (3, 'Loose pages', NULL);
  `}
  query={`
    SELECT a.name, b.title
    FROM author a
    INNER JOIN book b ON b.author_id = a.id
    ORDER BY a.name;
  `}
/>

Grace and "Loose pages" are absent — Grace has no book, and "Loose pages"
has no author.

## LEFT JOIN — keep the left side, NULL on the right

`LEFT JOIN` keeps every row from the left table even when no match exists,
filling the right-side columns with `NULL`.

<SideSql
  title="LEFT JOIN"
  fixture={`
    CREATE TABLE author (
      id INTEGER PRIMARY KEY,
      name TEXT NOT NULL
    );
    INSERT INTO author VALUES (1, 'Ada'), (2, 'Linus'), (3, 'Grace');

    CREATE TABLE book (
      id INTEGER PRIMARY KEY,
      title TEXT NOT NULL,
      author_id INTEGER REFERENCES author(id)
    );
    INSERT INTO book VALUES
      (1, 'Analytical engine', 1),
      (2, 'Kernel notes', 2),
      (3, 'Loose pages', NULL);
  `}
  query={`
    SELECT a.name, b.title
    FROM author a
    LEFT JOIN book b ON b.author_id = a.id
    ORDER BY a.name;
  `}
/>

Grace is back — with `NULL` for the title.

## Anti-join: "left without right"

Combine `LEFT JOIN` with `WHERE right.col IS NULL` to get rows from the
left side that have **no match** on the right. Authors without books:

```sql
SELECT a.name
FROM author a
LEFT JOIN book b ON b.author_id = a.id
WHERE b.id IS NULL;
```

<Callout type="tip" title="Pick the smaller side as left">
Anti-joins read clearly when the side you care about ("which authors are
missing books?") is on the left. The shape of the result then mirrors the
question.
</Callout>

## Pitfalls

- `JOIN` without `ON` is a cross join and produces a Cartesian product. Big.
- Putting filters in `ON` vs `WHERE` matters with `LEFT JOIN` — `WHERE`
  fires after the join and can turn it back into an inner join by accident.
- `SELECT *` after a join collides on duplicate column names. Qualify with
  aliases.
