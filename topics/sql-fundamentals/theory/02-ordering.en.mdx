# ORDER BY and LIMIT

Without `ORDER BY`, a SQL engine is free to return rows in any order. If your output looks sorted, that's a coincidence you cannot rely on. Sorting is explicit.

## ORDER BY

```sql
SELECT * FROM users ORDER BY age;
```

The default is ascending. Use `DESC` for descending.

```sql
SELECT * FROM users ORDER BY age DESC;
```

You can sort by multiple columns. The second key only matters when the first is equal.

```sql
SELECT * FROM users ORDER BY country ASC, age DESC;
```

## LIMIT

`LIMIT` caps the result size. Useful for top-N queries and pagination.

```sql
SELECT * FROM users ORDER BY age DESC LIMIT 3;
```

Without `ORDER BY`, `LIMIT` returns an arbitrary subset. Almost always wrong.

## When to use this

Every time you display a list to a user. Top-N reports, "latest 10" feeds, paginated tables — all built on `ORDER BY` + `LIMIT`.

## Common mistakes

- Using `LIMIT` without `ORDER BY` and expecting deterministic results.
- Forgetting that `NULL` ordering is engine-specific. SQLite puts `NULL` first by default; PostgreSQL puts it last.
- Paginating with `OFFSET` on large tables (it gets slow). Prefer keyset pagination once datasets grow — but that's a later topic.
