# Функция это имя для блока команд, который вы зовёте много раз

Борис пишет скрипт ночного бэкапа. Один и тот же блок «проверить, что каталог существует, заархивировать его, дописать строку в лог» встречается у него трижды: для базы, для загрузок пользователей и для конфигов. Пока блоков три, их ещё можно удержать в голове. Но когда в блоке находится ошибка, править её нужно сразу в трёх местах, и одно из них Борис каждый раз забывает.

Функция убирает это повторение. Вы один раз описываете блок команд под именем, а дальше вызываете это имя там, где раньше стоял copy-paste. Правка в одном месте меняет поведение всех вызовов сразу.

## Функция принимает аргументы, выполняет тело и оставляет после себя код возврата

За словом «функция» прячется простая механика. При вызове вы передаёте аргументы, тело функции выполняется с этими аргументами, а когда оно заканчивается, наружу остаётся одно число: код возврата. Ноль означает «успех», любое ненулевое значение означает «что-то пошло не так». Именно на это число смотрят `if`, `&&` и `||`, когда решают, что делать дальше.

## Как объявить и вызвать функцию

Объявление задаёт имя и тело в фигурных скобках. Вызов это просто имя функции, написанное как обычная команда.

```bash
greet() {
  echo "Привет, $1"
}

greet Анна
```

```
Привет, Анна
```

Скобки `()` после имени при объявлении пустые: аргументы туда не пишут. При вызове скобки тоже не нужны, аргументы идут через пробел после имени, ровно как у команд `ls` или `cp`.

## Аргументы внутри функции это её собственные $1, $2, $@

Внутри тела `$1`, `$2`, `$@` и `$#` относятся к аргументам самой функции, а не к аргументам скрипта. Это частый источник путаницы: снаружи `$1` это первый аргумент скрипта, а стоит зайти внутрь функции, и `$1` уже означает её первый аргумент.

```bash
show() {
  echo "первый: $1, всего: $#"
}

show Москва Казань
```

```
первый: Москва, всего: 2
```

## Код возврата: return и $?

`return N` завершает функцию с кодом возврата `N`, числом от 0 до 255. Если `return` не написан, код возврата функции равен коду её последней команды. Прочитать код только что завершившейся команды или функции можно из переменной `$?`.

```bash
is_positive() {
  [ "$1" -gt 0 ]
}

is_positive 5
echo $?
is_positive -3
echo $?
```

```
0
1
```

Здесь `return` нет, поэтому код возврата функции берётся от проверки `[ "$1" -gt 0 ]`: она даёт 0, когда условие истинно, и 1, когда ложно.

## Функция отдаёт наружу статус, а значение через echo

Важно не спутать две разные вещи. `return` передаёт только маленькое число статуса, а не строку и не результат вычисления. Если функции нужно «вернуть» имя файла или сумму, она печатает результат в стандартный вывод, а вызывающий код захватывает этот вывод подстановкой команды `$( ... )`.

```bash
full_name() {
  echo "$1 $2"
}

name=$(full_name Анна Иванова)
echo "Здравствуйте, $name"
```

```
Здравствуйте, Анна Иванова
```

## local удерживает переменную внутри функции

По умолчанию переменная, которой присвоили значение внутри функции, становится глобальной и переживает выход из функции. Слово `local` ограничивает её областью функции, чтобы она не затирала одноимённую переменную снаружи.

```bash
counter=10

bump() {
  local counter=1
  counter=$((counter + 1))
}

bump
echo "$counter"
```

```
10
```

Без `local` вызов `bump` испортил бы внешний `counter`. С `local` внутренняя переменная живёт только на время работы функции.

## Что вас удивит

Первый сюрприз в том, что `return` умеет передавать только число от 0 до 255. Попытка вернуть через `return` строку или большое число молча ломается: `return "готово"` вызовет ошибку, а `return 300` завернётся по модулю 256 и даст неожиданный код 44. Строки и суммы отдают через `echo`, а не через `return`.

Второй сюрприз про то, что функцию нужно объявить до первого вызова. bash читает скрипт сверху вниз, и если вызов стоит раньше объявления, интерпретатор ещё не знает такой команды и падает с ошибкой «command not found». Поэтому функции обычно собирают в начале скрипта.

## Типичные ошибки

**Спутать return и вывод значения.** `return $(($1 + $2))` не «возвращает сумму»: он кладёт её в код возврата, который ограничен диапазоном 0 до 255 и предназначен для статуса, а не для данных. Сумму печатают через `echo` и захватывают через `$( ... )`.

**Забыть local.** Присваивание внутри функции без `local` меняет глобальную переменную с тем же именем. В длинном скрипте это приводит к трудноуловимым ошибкам, когда одна функция незаметно портит состояние другой.

**Ждать внутри функции аргументы скрипта.** Внутри тела `$1` это первый аргумент функции, а не скрипта. Чтобы передать функции аргумент скрипта, его передают явно: `myfunc "$1"`.

**Вызвать функцию до её объявления.** bash не знает функцию, пока не дочитает её объявление, поэтому вызов выше по тексту завершается ошибкой «command not found».

<Detail summary="Почему if можно писать прямо перед вызовом функции">
`if myfunc; then ...; fi` работает потому, что `if` проверяет код возврата любой команды, а функция это команда. Ветка `then` срабатывает, когда функция вернула 0, и ветка `else`, когда ненулевое значение. Никакого сравнения со словами `true` или `false` при этом не происходит: решает именно число кода возврата.
</Detail>
