# Массивы и кортежи

Список цен, история заказов, набор тегов - данные постоянно приходят пачками. В JavaScript пачку хранит массив, и он всеяден: рядом могут лежать число, строка и объект, а длина меняется как угодно. Обычно это лишняя свобода. Список цен должен содержать только числа; координата на карте - ровно два числа, широту и долготу, и ни одним больше. TypeScript даёт два инструмента под эти два случая: типизированный массив для однородной пачки любой длины и кортеж для фиксированного набора значений с ролью у каждой позиции.

Разница между ними - это разница между «сколько угодно значений одного вида» и «ровно столько значений, и на каждом месте свой смысл». Научимся аннотировать оба и увидим, где путаница между ними приводит к ошибкам.

## Массив: сколько угодно значений одного типа

Тип массива - это тип элемента с квадратными скобками:

```ts
const prices: number[] = [2999, 899, 1499];
const cities: string[] = ['Москва', 'Казань'];
```

`number[]` читается «массив чисел». В такой массив нельзя положить строку, но можно сколько угодно чисел: длина не ограничена. Все элементы обязаны быть одного типа, и TypeScript следит за этим - `prices.push('дорого')` подсветится ошибкой.

Есть и вторая, равнозначная запись через `Array`:

```ts
const prices: Array<number> = [2999, 899, 1499];
```

`Array<number>` и `number[]` - один и тот же тип, записанный двумя способами. Угловые скобки здесь - это передача типа-параметра, о чём подробно в главе про generics. На практике чаще пишут короткую форму `number[]`, а `Array<number>` встречается, когда тип элемента сам длинный.

Если массив меняться не должен, его помечают `readonly`:

```ts
const prices: readonly number[] = [2999, 899];
```

Теперь `push`, `pop` и присваивание по индексу запрещены на этапе типов - массив только для чтения.

## Кортеж: фиксированная длина с ролями

Кортеж (tuple) описывает массив известной длины, где у каждой позиции свой тип и свой смысл. Записывают его перечислением типов в квадратных скобках:

```ts
let point: [number, number];
point = [55.7, 49.1];
```

Тип `[number, number]` требует ровно два числа. `point = [55.7]` - ошибка, не хватает элемента; `point = [55.7, 49.1, 0]` - ошибка, элемент лишний. Позиции могут быть разных типов, и это главная сила кортежа:

```ts
let entry: [string, number];
entry = ['Казань', 2999];
```

Здесь первый элемент - всегда строка, второй - всегда число. Кортеж - естественный тип для пары «ключ-значение», для координаты, для результата, где важен порядок. Именно кортеж возвращает, например, `useState` в React: `[значение, функция-сеттер]`.

Позициям можно дать имена - они не влияют на проверку, но делают тип читаемым:

```ts
type Coord = [lat: number, lon: number];
```

Имена `lat` и `lon` подскажут в редакторе, что означает каждая позиция.

## Крайние случаи

Главная ловушка - кортеж это тоже массив, и в рантайме он обычный массив. Значит методы вроде `push` физически работают, и TypeScript их не всегда останавливает: `point.push(0)` тип не нарушает, хотя логически ломает «ровно два элемента». Кортеж защищает длину при присваивании целиком, но не гарантирует её после мутирующих методов. Хотите настоящей неизменности - помечайте кортеж `readonly`.

Второй момент - вывод типа по умолчанию считает литерал массива массивом, а не кортежем. Для `const p = [55.7, 49.1]` тип вывелся как `number[]`, а не `[number, number]`: TypeScript выбирает более общий вариант. Нужен кортеж - либо аннотируйте явно, либо добавьте `as const`, который зафиксирует и длину, и значения.

<Callout type="warning" title="Пустой массив без аннотации становится any[]">
Запись const items = [] даёт тип any[]: выводить тип элемента не из чего, ведь массив пуст. Дальше в него можно класть что угодно, и проверка молчит. Если массив рождается пустым и наполняется позже, укажите тип элемента сразу: const items: string[] = [].
</Callout>

## Частые ошибки

**Брать массив там, где нужен кортеж.** `number[]` разрешает и один элемент, и пять. Если значений всегда ровно два и у них разные роли - это кортеж `[number, number]`, а не массив.

**Ждать от кортежа защиты длины после push.** Мутирующие методы обходят проверку длины. Для неизменяемого кортежа используйте `readonly`.

**Забыть аннотировать пустой массив.** `[]` без типа - это `any[]`, дыра в проверке. Пишите тип элемента при объявлении.

**Смешивать типы в массиве по недосмотру.** `[1, 'два', 3]` без аннотации выведется как `(string | number)[]`, и потом каждый элемент придётся сужать. Если это не задумано, храните однородные данные.

<Detail summary="as const: превратить литерал в набор литеральных типов">
Суффикс `as const` после литерала говорит TypeScript: считай это значение максимально узким и неизменяемым. Массив `[55.7, 49.1] as const` получает тип `readonly [55.7, 49.1]` - не просто числа, а ровно эти два, да ещё и только для чтения. Для объектов `as const` делает все поля `readonly` и сужает строки до литералов. Это незаменимо, когда из данных нужно вывести точный union: массив тегов `['new', 'paid'] as const` позволяет получить тип `'new' | 'paid'` прямо из значения, не дублируя список в отдельном типе. Обычному изменяемому коду `as const` не нужен, но для констант-справочников он экономит ручную типизацию.
</Detail>
