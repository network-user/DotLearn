# Раскладка, которую рисуют, а не собирают

Flexbox выстраивает элементы вдоль одной оси - ряд или колонку. Но макет страницы двумерен: шапка сверху, боковое меню слева, контент справа, подвал снизу - и всё это должно выравниваться и по горизонтали, и по вертикали одновременно. Собирать такое из вложенных flex-контейнеров можно, но это как чертить таблицу линейкой по одной линии. CSS Grid даёт сразу двумерную сетку: вы объявляете колонки и строки, а затем расставляете элементы по клеткам - иногда буквально рисуя макет ASCII-словами.

## Треки, линии и единица fr

`display: grid` и `grid-template-columns` задают колонки-треки. Число значений - число колонок, каждое значение - ширина трека.

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;
}
```

Здесь три колонки: первая ровно 200 пикселей, две другие - по одной доле `fr`. Единица `fr` (fraction) делит оставшееся после фиксированных треков место. Считается так же, как flex-grow: сложить все `fr`, вычесть фиксированные колонки из ширины контейнера, разделить остаток на сумму долей.

Если контейнер шириной 800 пикселей, первая колонка съедает 200, остаток 600 делится между двумя `1fr` - по 300 пикселей каждая. `grid-template-rows` задаёт строки той же логикой, а `gap` - промежутки между всеми треками сразу.

## repeat, minmax и адаптивная сетка без медиазапросов

Писать `1fr 1fr 1fr 1fr` руками утомительно; `repeat(4, 1fr)` короче и означает то же. По-настоящему интересно, когда `repeat` соединяют с `minmax` и ключевым словом `auto-fit`.

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 16px;
}
```

`minmax(200px, 1fr)` означает «трек не уже 200 пикселей, но готов растянуться на долю `fr`». `auto-fit` говорит браузеру: помести столько колонок такого размера, сколько влезет в текущую ширину, а лишнее место раздай существующим. Результат - сетка карточек, которая сама меняет число колонок при изменении ширины окна, без единого медиазапроса. Это визитная карточка Grid.

Сколько колонок получится - считается честно. При ширине 850 пикселей и `gap: 10px`: четыре колонки просят 4 × 200 + 3 × 10 = 830 пикселей (три промежутка между четырьмя треками) - помещается. Пять колонок просили бы 5 × 200 + 4 × 10 = 1040 - не помещается. Значит, колонок будет четыре, и каждая растянется, поделив свободное место.

<Callout type="info">
Разница auto-fit и auto-fill проявляется, когда карточек мало. auto-fill сохраняет пустые треки (карточки не растягиваются на всю ширину), auto-fit схлопывает пустые треки в ноль, и существующие карточки заполняют строку целиком. Для галерей обычно хотят auto-fit.
</Callout>

## Размещение элементов по линиям

Треки разделены линиями, пронумерованными с единицы. Элемент можно растянуть на несколько колонок или строк, указав линии начала и конца - или проще, ключевым словом `span`.

```css
.hero {
  grid-column: 1 / 3;
}
.wide {
  grid-column: span 2;
}
```

`grid-column: 1 / 3` ставит элемент от линии 1 до линии 3, то есть на две колонки. `span 2` - то же самое относительно текущего места: занять два трека. Так баннер растягивают во всю ширину сетки, а обычные карточки оставляют по одной клетке. Аналогично `grid-row` управляет вертикальным размахом. Удобный приём - отрицательные номера линий: `-1` означает самую последнюю линию сетки, поэтому `grid-column: 1 / -1` растягивает элемент от первого до последнего трека независимо от их числа, что идеально для заголовка или разделителя во всю ширину.

Всё, что вы не разместили явно, Grid раскладывает сам, слева направо и сверху вниз, заполняя свободные клетки. Это автоматическое размещение (auto-placement) и делает Grid удобным для галерей: вы описываете сетку один раз, а десятки карточек сами занимают места по порядку, без единого правила на каждую.

## Именованные области: макет словами

Самая наглядная сила Grid - `grid-template-areas`. Вы даёте клеткам имена прямо строками, буквально рисуя макет, а затем привязываете элементы к именам.

```css
.page {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
}
.page__header { grid-area: header; }
.page__sidebar { grid-area: sidebar; }
.page__main { grid-area: main; }
.page__footer { grid-area: footer; }
```

## Grid или Flexbox

Правило большого пальца: Flexbox - для одномерных раскладок (ряд кнопок, панель навигации, список тегов), Grid - для двумерных (макет страницы, галерея, сложные карточки). Разница ещё и в отправной точке: Flexbox отталкивается от содержимого - элементы сами делят место по своим размерам и `flex`-долям; Grid отталкивается от контейнера - вы сначала чертите сетку, а содержимое расставляете по готовым клеткам. Поэтому Flexbox гибче там, где число и размер элементов заранее неизвестны, а Grid строже там, где важно жёсткое выравнивание по обеим осям.

Они не конкуренты, а партнёры: внешний макет часто на Grid, а содержимое каждой ячейки выстроено Flexbox. Признак, что пора на Grid, - вы начали вкладывать flex-контейнеры друг в друга только затем, чтобы совместить горизонтальное и вертикальное выравнивание. Это верный сигнал, что двумерную задачу пытаются решить одномерным инструментом.

## Частые ошибки

- **Считать `fr` до вычета фиксированных треков.** `fr` делит только остаток после пикселей, процентов и `minmax`-минимумов, а не всю ширину контейнера.
- **Путать `auto-fit` и `auto-fill`.** При малом числе карточек `auto-fill` оставит пустые треки, а `auto-fit` растянет имеющиеся. Симптом - «карточки не на всю ширину».
- **Забывать про `gap` при расчёте числа колонок.** Промежутков на один меньше числа треков, но они реально съедают место и влияют на то, сколько колонок влезет.
- **Строить макет страницы на вложенных flex вместо Grid.** Работает, но хрупко; двумерное выравнивание - это про Grid и `grid-template-areas`.

<Detail summary="Неявная сетка и auto-flow">
Вы задаёте явную сетку через template-свойства, но если элементов больше, чем клеток, Grid создаёт треки автоматически - это неявная (implicit) сетка. Её размер задаёт `grid-auto-rows` (или `grid-auto-columns`), а направление добавления новых клеток - `grid-auto-flow`: по умолчанию `row` (заполнять построчно), но `column` заставит расти вбок, а приписка `dense` разрешит Grid возвращаться назад и затыкать оставшиеся дыры более поздними мелкими элементами. Понимание явной и неявной сетки объясняет, почему элементы иногда «появляются» в неожиданных рядах: они не поместились в явную сетку и ушли в авто-треки.
</Detail>
