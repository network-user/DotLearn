# Strings: how Python stores text and builds phrases from it

Almost everything your screen shows you is text. A bank SMS, the recipient's name in the Ozon app, a pickup point address, the caption "Your order has been delivered". None of these phrases sat somewhere ready-made: a program assembled each one from pieces, plugged in a name, added a city, trimmed extra spaces. In the previous lesson you learned to put text into a variable and met the `str` type. Time to learn how to actually work with that text.

Here's the lesson's task. A user filled out a delivery form and typed their name as "  anna  ", with stray spaces on both sides and a lowercase letter. By the end of the lesson you'll turn that input into a tidy "Anna" and assemble the phrase "Anna, your order has arrived in Kazan" with a single line of code.

## A chain of characters with numbers

First, how to write a string. Single or double quotes both work: `'Anna'` and `"Anna"` define the exact same string, and Python sees no difference between them. A difference appears only when a quote character is needed inside the text itself: a string with an apostrophe like `"it's"` is easier to wrap in double quotes, and a string with double quotes inside is easier to wrap in single ones. This course defaults to double quotes.

Now the key idea. Inside, a string is a chain of characters, and each character sits at its own position. That position number is called an index, and counting starts not from one but from zero. Look at the figure: the word "Kazan" has six letters in Russian (Казань), the first sits at position 0, the last at position 5.

The `len()` function reports how many characters are in a string:

```python
city = "Казань"
print(len(city))
```

The code prints 6. The function counts every character in a row: letters, digits, punctuation, spaces. The string `"Нижний Новгород"` has fifteen characters, because the space between the words is a character in its own right too.

Once you know the index, you can grab a character: after the variable name, add square brackets with the number inside.

```python
city = "Казань"
print(city[0])
print(city[5])
```

The first call prints "К", the second "ь". Notice the shift you'll need to get used to: the string's length is 6, but the last character's index is 5. Zero-based counting is to blame. Ask for character number 6, and the program crashes with `IndexError`: position 6 simply doesn't exist in this string.

For the last character there's a handier trick than computing the index: negative numbers. They count positions from the end: `city[-1]` returns "ь", `city[-2]` returns "н", and so on down to `city[-6]`. The figure above labels the negative indexes under the cells. Remember `s[-1]` as the standard way to grab the last character.

What if you need not one character but a chunk, say the first three letters? That's what slices are for, covered in detail in the lesson on slices and sequences. For now, single characters are enough.

## Concatenate and repeat

Strings add up with a plus sign, an operation called concatenation:

```python
name = "Анна"
greeting = "Привет, " + name + "!"
print(greeting)
```

Plus docks strings end to end, with no spaces or extras added automatically: the space after the comma in `"Привет, "` is one you type yourself. Multiplication works too: a string multiplied by a whole number repeats that many times.

```python
print("=" * 20)
print("Чек № 10482")
```

The first line prints twenty equals signs in a row. The trick comes in handy when you need to visually separate one part of the output from another.

Here's the lesson's first trap: plus only glues a string to another string. The expression `"Цена: " + 100` crashes with `TypeError`, Python refuses to add text and a number together. You already know one cure from the previous lesson, `str(100)`. A second, far more convenient one, is coming up right in the next section.

## f-strings: the go-to way to assemble text

Gluing text with pluses quickly turns into a thicket of quotes and plus signs: a space goes missing here, a quote fails to close there. That's why real code assembles text with f-strings. Put the letter `f` right before the opening quote, and curly braces inside the string come alive: whatever's written in them gets evaluated by Python and dropped into the text.

```python
name = "Анна"
city = "Казань"
print(f"{name}, ваш заказ прибыл в город {city}")
```

The code prints "Анна, ваш заказ прибыл в город Казань". Spaces show up right where you'd expect, and the phrase reads like a phrase. Curly braces accept not just a variable name but a whole expression:

```python
price = 1290
count = 3
print(f"Итого: {price * count} рублей")
```

This f-string computes the product itself, turns it into text, and drops in 3870. Treat f-strings as your default tool: whenever you need to assemble text from data, reach for the letter `f` first.

## Methods: a string's built-in abilities

Beyond operators, a string has its own actions, called through a dot and known as methods. The pattern looks like this: the value, a dot, the action's name, and parentheses, just like a function call.

```python
name = "анна"
print(name.upper())
print(name.capitalize())
```

`upper()` returns the same string in uppercase, "АННА". `capitalize()` makes the first letter uppercase and the rest lowercase, giving "Анна". There's also `lower()`, which pushes every letter to lowercase and is handy when text needs comparing without caring about case.

The `strip()` method peels spaces off both edges of a string, leaving inner spaces alone. It's exactly what fixes messy form input:

```python
raw = "  анна  "
print(raw.strip())
```

`replace()` swaps every occurrence of one substring for another, and `count()` counts how many times a substring shows up:

```python
address = "Тверь, улица Советская, 14"
print(address.replace("Тверь", "Казань"))
print(address.count("а"))
```

The first call prints the address with Kazan instead of Tver, the second prints 2: the lowercase "а" appears in "улица" and "Советская". A capital "А" doesn't count, `count()` treats it as a different character.

Methods return a new string, so you can chain them: each next call applies to the result of the one before it.

```python
raw = "  анна  "
print(raw.strip().capitalize())
```

First `strip()` removes the spaces, giving "анна", then `capitalize()` lifts the first letter. "Анна" appears on screen: the lesson's opening task solved in a single line.

The lesson's last tool isn't even a method, it's an operator. `in` checks whether a substring appears inside a string:

```python
address = "Казань, улица Баумана, 5"
print("Казань" in address)
print("Москва" in address)
```

The first check prints `True`, the second `False`. Those are the same boolean values from the previous lesson; in the lesson on conditions you'll start plugging checks like this into `if` and branching your program. Keep in mind the check is case-sensitive: in practice, text is often lowercased first with `lower()` so that "казань" and "Казань" count as the same thing.

## Strings can't be changed

Now for the lesson's big surprise. Since you can read a character by index, it seems natural that you could fix a string in place too: "cat" becomes "hat" by swapping the first letter.

```python
word = "кот"
word[0] = "р"
```

This code doesn't work. Python stops it with a `TypeError` whose message says a string doesn't support item assignment. Reading a character by number is fine, writing one is not. A string in Python is immutable: from the moment it's created, its contents never change, ever.

But wait, what about `replace()` and `upper()`, don't they change the string? They don't. Every method builds a new string and returns it, leaving the original untouched. That gives rise to a second, quieter version of the same surprise:

```python
name = "анна"
name.upper()
print(name)
```

The code prints "анна" in lowercase. The method honestly built "АННА", but nobody saved the result, and it's gone. The fix: assign the result back to the variable.

```python
name = "анна"
name = name.upper()
print(name)
```

The model from the previous lesson comes in handy here: a variable is a label on a value. The string "анна" didn't change and couldn't have changed, the label `name` simply got re-stuck onto the new string "АННА", while the old one is left nameless and will soon be discarded.

<Callout type="warning">
String methods never change the original string, they always return a new one. Call a method and don't assign the result, and the work vanishes for nothing.
</Callout>

## Frequent mistakes

- **Forgetting the letter f before the quotes.** The code `print("Привет, {name}!")` prints the curly braces literally, name and all. Only the `f` prefix turns on substitution.
- **Calling a method and losing the result.** `name.strip()` on its own changes nothing. Write `name = name.strip()` if you need the result later.
- **Adding a string and a number.** `"Цена: " + 100` gives `TypeError`. Wrap the number in `str()`, or assemble the text with an f-string, which turns the number into text for you.
- **Missing the last character.** For a string of length 6, the last index is 5, and `s[6]` gives `IndexError`. `s[-1]` is the safer bet.

<Detail summary="Why Russian letters are no worse than English ones">
Python strings store text in Unicode, a worldwide table of characters where the Latin alphabet, Cyrillic, ideographs, and symbols like № all have their own number. That's why `len("Казань")` returns 6 exactly the same way `len("Berlin")` does, and `"ё".upper()` honestly returns "Ё". Older languages, and Python 2, had a harder time with this: Russian text meant wrestling with encodings, and a string's length in bytes didn't match its number of letters. Modern Python has spared you almost all of that pain; encodings only resurface when reading files and sending text over a network.
</Detail>
