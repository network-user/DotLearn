# Первые команды: переменные и типы данных

Анна покупает книги на Ozon: одна книга стоит 450 рублей, нужно три штуки, доставка ещё 199. Калькулятор выдаст итог и тут же его забудет. Завтра цена поменяется, и весь расчёт придётся набирать заново, кнопку за кнопкой. Программа устроена иначе: она запоминает каждое значение под собственным именем, а сам расчёт хранит как рецепт. Поменяли цену, запустили заново, получили свежий итог. Умение дать значению имя и дальше работать с именем это первый навык программиста, и именно он осваивается в этом уроке.

Здесь вы напишете свои первые строки на Python: научитесь печатать на экран, заведёте переменные, разберётесь, чем число 5 отличается от текста «5», и увидите, почему деление в Python отвечает не совсем так, как калькулятор. Никакой подготовки не требуется, а каждый пример можно запустить и поменять прямо на этой странице.

## Программа это список команд

Программа на Python это обычный текст, составленный из команд. Python читает его сверху вниз, строку за строкой, и выполняет каждую команду по очереди: ничего не пропускает, не забегает вперёд, не меняет порядок. Первая команда, без которой не обходится ни один урок, это `print()`. Она печатает на экран то, что вы положили внутрь круглых скобок.

```python
print("Привет!")
print(2 + 2)
```

Здесь две команды. Первая печатает текст: кавычки говорят Python «выведи это буквально». Вторая сначала вычисляет выражение `2 + 2`, а уже потом печатает результат, число 4. Разница принципиальная: всё, что в кавычках, воспринимается как есть, всё, что без кавычек, Python пытается вычислить. Порядок строк тоже имеет значение: если попросить напечатать переменную раньше, чем она появилась, Python честно ответит, что такого имени не знает.

## Переменная это наклейка с именем

Представьте кухонную полку с банками. В банках крупы, а чтобы не путаться, на банки клеят наклейки: «рис», «гречка». Наклейка не хранит крупу, она лишь указывает на банку. Переменная в Python устроена так же: значение живёт в памяти само по себе, а имя это наклейка, приклеенная к значению.

```python
price = 450
qty = 3
print(price * qty)
```

Строка `price = 450` читается справа налево: создать значение 450 и приклеить к нему наклейку с именем `price`. Знак `=` здесь не «равно» из школьной математики, а команда присваивания. Дальше именем можно пользоваться где угодно: в третьей строке Python видит `price * qty`, идёт по наклейкам, находит 450 и 3, перемножает их и печатает 1350.

Во многих учебниках переменную рисуют коробкой, в которую кладут значение. Для Python точнее именно наклейка: к одному значению могут вести сразу несколько имён. Взгляните на рисунок: после присваивания `cost = price` оба имени ведут к одному и тому же объекту, ничего никуда не копируется.

Пока вы работаете с числами, разница между наклейкой и коробкой незаметна. Но модель имён-наклеек это настоящее устройство Python, и в уроке про списки и словари она станет решающей: там две наклейки на одном списке регулярно удивляют новичков. Полезно привыкнуть к честной картинке с самого начала.

Наклейку можно переклеить. Повторное присваивание снимает имя со старого значения и приклеивает к новому:

```python
balance = 1000
balance = 750
print(balance)
```

Программа напечатает 750. Имя всегда ведёт к последнему присвоенному значению, а старая связь исчезает без следа: Python не хранит историю переклеек.

## Как называть переменные

Имя переменной собирается из латинских букв, цифр и подчёркивания, но начинаться с цифры не может: `price2` годится, а `2price` уронит программу с ошибкой `SyntaxError` ещё до запуска. Большие и маленькие буквы различаются: `Price` и `price` это два независимых имени, и опечатка в регистре создаст вторую переменную вместо того, чтобы изменить первую.

В Python принят стиль snake_case: слова пишутся маленькими буквами, а между словами ставится подчёркивание, например `delivery_price` или `user_age`. И главное правило хорошего имени: оно должно отвечать на вопрос «что здесь лежит». Через неделю `price_rub` расскажет о содержимом сам, а над `p` придётся вспоминать, что вы имели в виду.

## Четыре типа данных

У каждого значения в Python есть тип. Для старта достаточно четырёх:

- `int`, целые числа: 450, 3, 0.
- `float`, дробные числа: 3.5, 199.99. Дробная часть отделяется точкой, а не запятой.
- `str`, строки, то есть любой текст в кавычках: "Анна", "Казань", "450".
- `bool`, логический тип: всего два значения, True и False.

Узнать тип любого значения помогает функция `type()`:

```python
print(type(450))
print(type(3.5))
print(type("450"))
print(type(True))
```

Присмотритесь к третьей строке. Значение "450" взято в кавычки, поэтому его тип str: это текст, а не число. Для Python надпись «мёд» на банке и сам мёд это принципиально разные вещи. Чуть ниже вы увидите, как их смешение ломает программу. Названия типов стоит запомнить ещё и потому, что они постоянно всплывают в сообщениях об ошибках, и умение их узнавать экономит массу времени.

Про `bool` пока достаточно знать, что он существует: этот тип отвечает на вопросы «да или нет» и станет главным героем урока про условия. Строкам тоже посвящён отдельный урок, следующий по порядку.

## Арифметика и первый сюрприз

С числами Python обращается как калькулятор: `+` складывает, `-` вычитает, `*` умножает, `/` делит. Есть и возведение в степень: `2 ** 10` даст 1024. Пока никаких неожиданностей. А теперь запустите вот это:

```python
print(6 / 2)
```

На экране появится 3.0, а не 3. Деление `/` всегда возвращает float, даже когда всё делится нацело. Это сделано намеренно: результат деления заранее неизвестен, поэтому Python выбирает один предсказуемый тип ответа вместо того, чтобы угадывать по ситуации.

Когда нужен именно целый ответ, работает другая пара операторов. `//` делит и отбрасывает дробную часть, `%` возвращает остаток от деления. Бытовой пример: семь конфет на двоих. Каждому достанется `7 // 2`, то есть по три конфеты, а `7 % 2`, то есть одна конфета, останется в вазе.

```python
print(7 // 2)
print(7 % 2)
print(7 / 2)
```

Теперь соберите всё вместе: ниже живой пример с заказом Анны из начала урока. Поменяйте `qty` на 2 и запустите снова, итог пересчитается сам. Затем замените `/` на `//` в последней строке и сравните ответы. Наконец, допишите строку `print(type(total))` и проверьте свою интуицию насчёт типов.

## Из строки в число и обратно

Данные приходят в программу текстом гораздо чаще, чем хочется. Форма на сайте вроде Госуслуг отдаёт введённый возраст строкой, даже если в поле одни цифры; файл с прайсом хранит цену как текст "450". Попробуйте сложить такую «цену» с числом:

```python
print("5" + 5)
```

Программа падает с ошибкой `TypeError`. И это забота о вас: Python не знает, чего вы хотели, склеить текст в «55» или сложить числа и получить 10, поэтому отказывается гадать. Выход это явное преобразование типов:

```python
age = int("34")
price = float("199.99")
label = str(450)
print(age + 1)
```

`int()` превращает строку из цифр в целое число, `float()` в дробное, а `str()` работает в обратную сторону и делает из числа текст. После `int("34")` с возрастом можно считать как с обычным числом: программа напечатает 35. Склейку строк через `+` и другие фокусы с текстом вы подробно разберёте в уроке про строки.

## Подводные камни

Два случая регулярно ловят новичков. Во-первых, `int()` не округляет, а отбрасывает дробную часть: `int(3.9)` вернёт 3, а не 4. Во-вторых, дробные числа хранятся приближённо:

```python
print(int(3.9))
print(0.1 + 0.2)
```

Вторая строка напечатает 0.30000000000000004. Это не поломка вашего компьютера: так float работает во всех языках программирования, от Python до JavaScript. Практический вывод простой: деньги считают в целых копейках, типом int, а float оставляют для измерений, где крошечная погрешность в пятнадцатом знаке никому не мешает.

<Detail summary="Почему 0.1 + 0.2 не равно 0.3">
Компьютер хранит числа в двоичной системе. Дробь 0.1 в двоичной записи бесконечна, как 1/3 в привычной десятичной: 0.3333 и так далее без конца. Тип float отводит числу 64 бита, поэтому бесконечный «хвост» обрезается, и в памяти лежит ближайшее представимое число, чуть-чуть отличающееся от честной одной десятой. При сложении двух таких приближений погрешность выползает в последние знаки результата. Для денег и других точных величин используют целые числа или специальный тип Decimal из стандартной библиотеки.
</Detail>

<Callout type="info" title="Как устроены упражнения курса">
В упражнениях вы часто увидите заготовку вида `def order_total(price, qty):` и под ней тело из `...`. Дописывать нужно только тело: замените `...` на свои строки и верните результат командой `return`. Слово `def` объявляет функцию, его подробный разбор ждёт вас в последнем уроке курса. Пока относитесь к заготовке как к бланку: шапка уже заполнена, значения параметров придут сами, ваша задача записать, что с ними сделать.
</Callout>

## Частые ошибки

- **Текст без кавычек.** `name = Анна` падает с `NameError`: без кавычек Python ищет переменную с именем Анна, а не текст. Всё, что задумано как текст, берите в кавычки.
- **Строка плюс число.** `"5" + 5` вызывает `TypeError`. Сначала приведите оба значения к одному типу: `int("5") + 5` даст 10, а `"5" + str(5)` даст "55".
- **Имя с цифры.** `2fast = 10` это `SyntaxError`. Цифры в имени разрешены, но не первым символом.
- **Ожидать от `=` сравнения.** Знак `=` присваивает. Проверка «равны ли значения» пишется двойным знаком `==` и подробно разбирается в уроке про условия.
