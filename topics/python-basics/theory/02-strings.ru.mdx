# Строки: как Python хранит текст и собирает из него фразы

Почти всё, что показывает вам экран, это текст. СМС от банка, имя получателя в приложении Ozon, адрес пункта выдачи, подпись «Ваш заказ доставлен». Ни одна из этих фраз не лежала где-то готовой: программа собрала каждую из кусочков, подставила имя, добавила город, убрала лишние пробелы. В прошлом уроке вы научились класть текст в переменную и познакомились с типом `str`. Пришло время научиться с этим текстом работать.

Вот задача на урок. Пользователь заполнил форму доставки и ввёл имя как «  анна  », с лишними пробелами по краям и со строчной буквы. К концу урока вы сможете одной строкой кода превратить этот ввод в аккуратное «Анна» и собрать фразу «Анна, ваш заказ прибыл в Казань».

## Цепочка символов с номерами

Сначала о том, как строку записать. Подойдут одинарные или двойные кавычки: `'Анна'` и `"Анна"` задают одну и ту же строку, Python не видит между ними разницы. Разница появляется, только когда кавычка нужна внутри самого текста: строку с апострофом вроде `"it's"` удобнее заключить в двойные кавычки, а строку с двойными кавычками внутри, наоборот, в одинарные. В этом курсе по умолчанию используются двойные.

Теперь главное. Внутри строка устроена как цепочка символов, где каждый символ стоит на своей позиции. Номер позиции называют индексом, и нумерация начинается не с единицы, а с нуля. Посмотрите на рисунок: слово «Казань» состоит из шести символов, первый стоит на позиции 0, последний на позиции 5.

Сколько символов в строке, сообщает функция `len()`:

```python
city = "Казань"
print(len(city))
```

Код напечатает 6. Функция считает все символы подряд: буквы, цифры, знаки препинания и пробелы. В строке `"Нижний Новгород"` пятнадцать символов, потому что пробел между словами тоже полноценный символ.

Зная индекс, символ можно достать: после имени переменной поставьте квадратные скобки с номером внутри.

```python
city = "Казань"
print(city[0])
print(city[5])
```

Первый вызов напечатает «К», второй «ь». Обратите внимание на сдвиг, к которому придётся привыкнуть: длина строки 6, а индекс последнего символа 5. Виновата нумерация с нуля. Попросите символ с номером 6, и программа упадёт с ошибкой `IndexError`: позиции 6 в этой строке просто нет.

Для последнего символа есть приём удобнее, чем высчитывать индекс: отрицательные номера. Они отсчитывают позиции с конца: `city[-1]` вернёт «ь», `city[-2]` вернёт «н» и так далее до `city[-6]`. На рисунке выше отрицательные индексы подписаны под ячейками. Запомните `s[-1]` как стандартный способ взять последний символ.

А если нужен не один символ, а кусок, скажем первые три буквы? Для этого существуют срезы, их подробный разбор ждёт вас в уроке про срезы и последовательности. Пока хватит одиночных символов.

## Склеить и повторить

Строки складываются плюсом, эта операция называется конкатенацией:

```python
name = "Анна"
greeting = "Привет, " + name + "!"
print(greeting)
```

Плюс пристыковывает строки друг к другу вплотную, без пробелов и прочих добавок: пробел после запятой в `"Привет, "` вы ставите сами. Работает и умножение: строка, умноженная на целое число, повторяется столько раз.

```python
print("=" * 20)
print("Чек № 10482")
```

Первая строка напечатает двадцать знаков равенства подряд. Приём выручает, когда нужно визуально отделить одну часть вывода от другой.

Здесь же поджидает первая ловушка: плюс склеивает строку только со строкой. Выражение `"Цена: " + 100` уронит программу с ошибкой `TypeError`, складывать текст с числом Python отказывается. Одно лекарство вы знаете из прошлого урока, `str(100)`. Второе, куда более удобное, прямо в следующем разделе.

## f-строки: главный сборщик текста

Склейка плюсами быстро превращается в частокол кавычек и плюсов: то пробел потерялся, то кавычка не закрылась. Поэтому в живом коде текст собирают f-строками. Поставьте букву `f` вплотную перед открывающей кавычкой, и внутри строки заработают фигурные скобки: всё, что написано в них, Python вычислит и подставит в текст.

```python
name = "Анна"
city = "Казань"
print(f"{name}, ваш заказ прибыл в город {city}")
```

Код напечатает «Анна, ваш заказ прибыл в город Казань». Пробелы видны сразу, фраза читается как фраза. Внутри фигурных скобок разрешено не только имя переменной, но и целое выражение:

```python
price = 1290
count = 3
print(f"Итого: {price * count} рублей")
```

Такая f-строка сама вычислит произведение, превратит его в текст и подставит 3870. Считайте f-строки инструментом по умолчанию: когда надо собрать текст из данных, первым делом тянитесь к букве `f`.

## Методы: встроенные умения строки

Кроме операторов у строки есть собственные действия, они вызываются через точку и называются методами. Запись выглядит так: значение, точка, имя действия и круглые скобки, как у функции.

```python
name = "анна"
print(name.upper())
print(name.capitalize())
```

`upper()` возвращает ту же строку в верхнем регистре, «АННА». `capitalize()` делает первую букву заглавной, а остальные строчными: получится «Анна». Существует и `lower()`, он опускает все буквы в нижний регистр и выручает, когда текст нужно сравнивать без оглядки на регистр.

Метод `strip()` снимает пробелы с обоих краёв строки, не трогая пробелы внутри. Ровно он чинит кривой ввод из формы:

```python
raw = "  анна  "
print(raw.strip())
```

`replace()` заменяет все вхождения одной подстроки на другую, а `count()` считает, сколько раз подстрока встречается:

```python
address = "Тверь, улица Советская, 14"
print(address.replace("Тверь", "Казань"))
print(address.count("а"))
```

Первый вызов напечатает адрес с Казанью вместо Твери, второй напечатает 2: строчная «а» встречается в словах «улица» и «Советская». Заглавная «А» не в счёт, для `count()` это другой символ.

Методы возвращают новую строку, поэтому их можно выстраивать цепочкой: каждый следующий применяется к результату предыдущего.

```python
raw = "  анна  "
print(raw.strip().capitalize())
```

Сначала `strip()` уберёт пробелы, получится «анна», затем `capitalize()` поднимет первую букву. На экране появится «Анна»: задача из начала урока решена одной строкой.

Последний инструмент урока даже не метод, а оператор. `in` проверяет, встречается ли в строке подстрока:

```python
address = "Казань, улица Баумана, 5"
print("Казань" in address)
print("Москва" in address)
```

Первая проверка напечатает `True`, вторая `False`. Это те самые булевы значения из прошлого урока; в уроке про условия вы начнёте подставлять такие проверки в `if` и разветвлять программу. Учтите, что проверка чувствительна к регистру: на практике строку часто заранее опускают в нижний регистр через `lower()`, чтобы «казань» и «Казань» считались одним и тем же.

## Строку нельзя изменить

Теперь главный сюрприз урока. Раз символ можно прочитать по индексу, кажется естественным и исправить строку на месте: было «кот», поменяли первую букву, стало «рот».

```python
word = "кот"
word[0] = "р"
```

Этот код не сработает. Python остановит его с ошибкой `TypeError`, в тексте которой сказано, что строка не поддерживает присваивание по индексу. Читать символ по номеру можно, записывать нельзя. Строка в Python неизменяема: с момента создания её содержимое не меняется никогда.

Но постойте, а как же `replace()` и `upper()`, разве они не меняют строку? Не меняют. Каждый метод строит новую строку и возвращает её, исходная остаётся нетронутой. Отсюда вторая, тихая форма того же сюрприза:

```python
name = "анна"
name.upper()
print(name)
```

Код напечатает «анна» со строчной буквы. Метод честно построил «АННА», но результат никто не сохранил, и он пропал. Исправление: присвойте результат обратно переменной.

```python
name = "анна"
name = name.upper()
print(name)
```

Здесь пригодится модель из прошлого урока: переменная это ярлык на значение. Строка «анна» не изменилась и не могла измениться, просто ярлык `name` переклеили на новую строку «АННА», а старая осталась без имени и скоро будет выброшена.

<Callout type="warning">
Методы строк никогда не меняют исходную строку, они всегда возвращают новую. Если вызвать метод и не присвоить результат, работа пропадёт впустую.
</Callout>

## Частые ошибки

- **Забыть букву f перед кавычками.** Код `print("Привет, {name}!")` напечатает фигурные скобки буквально, вместе с именем внутри. Подстановку включает только префикс `f`.
- **Вызвать метод и потерять результат.** Строка `name.strip()` сама по себе ничего не меняет. Пишите `name = name.strip()`, если результат нужен дальше.
- **Сложить строку с числом.** `"Цена: " + 100` даёт `TypeError`. Оберните число в `str()` или соберите текст f-строкой, она превратит число в текст сама.
- **Промахнуться мимо последнего символа.** У строки длиной 6 последний индекс 5, а обращение `s[6]` даёт `IndexError`. Надёжнее просить `s[-1]`.

<Detail summary="Почему русские буквы ничем не хуже английских">
Строки Python хранят текст в Юникоде, всемирной таблице символов, где свой номер есть у латиницы, кириллицы, иероглифов и знаков вроде №. Поэтому `len("Казань")` возвращает 6 ровно так же, как `len("Berlin")`, а `"ё".upper()` честно возвращает «Ё». В старых языках и в Python 2 с этим было тяжелее: русский текст требовал возни с кодировками, а длина строки в байтах не совпадала с числом букв. Современный Python избавил вас от этой боли почти целиком; кодировки всплывут снова только при чтении файлов и передаче текста по сети.
</Detail>
