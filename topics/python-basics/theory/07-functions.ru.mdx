# Вы писали функции с первого урока

Пора раскрыть секрет, который тянется через весь курс. В каждом упражнении вы получали заготовку вроде `def taxi_price(km):` и дописывали строки внутрь, под первую строку с непонятным словом `def`. Трогать эту строку было нельзя, объяснять её никто не объяснял, и она послушно работала. Признание: всё это время вы писали функции. Каждое решённое упражнение уже было функцией, вы просто заполняли готовое тело, пока заголовок терпеливо ждал своего часа.

Сегодня заготовки заканчиваются. Вы разберёте заголовок по косточкам, научитесь объявлять функции целиком, от `def` до `return`, и распутаете главную путаницу новичков: чем возврат значения отличается от печати на экран. Это финальный урок, поэтому последняя задача соберёт вместе условия, списки, циклы и арифметику из всех шести предыдущих уроков.

## Зачем давать логике имя

Представьте расчёт поездки на такси: посадка 100 рублей плюс 25 рублей за километр. Посчитать один раз легко. Но в программе такой расчёт нужен снова и снова: для поездки Анны, для поездки Бориса, для отчёта за месяц. Копировать одни и те же строки по всей программе плохо: когда в расчёте найдётся ошибка, править придётся каждую копию, и одну вы обязательно пропустите.

Функция решает проблему иначе. Вы описываете шаги один раз и даёте им имя, а дальше вызываете расчёт по имени сколько угодно раз, с разными числами. Подойдёт бытовая аналогия: кофейный автомат. Вы бросаете монеты и нажимаете кнопку, это вход. Внутри автомат шумит по одному и тому же сценарию, это тело. Наружу выезжает стакан, это результат. Ради второго кофе никто не разбирает автомат заново: достаточно ещё раз нажать кнопку.

Прочитайте схему слева направо: аргументы входят в функцию, тело шаг за шагом превращает их в результат, `return` выносит готовое значение наружу, и оно оказывается ровно там, откуда функцию позвали. Держите эту картинку в голове весь урок: каждый новый термин займёт на ней своё место.

## Анатомия объявления: от def до return

Вот функция целиком, вместе с вызовом:

```python
def greet(name):
    text = "Привет, " + name + "!"
    return text

message = greet("Анна")
print(message)
```

Первая строка называется заголовком. `def` сообщает Python: «дальше объявление функции». Следом идёт имя `greet`: его вы придумываете сами, по тем же правилам, что и имена переменных. В круглых скобках перечислены параметры, здесь он один, `name`. Двоеточие закрывает заголовок, а всё, что написано ниже с отступом, это тело функции: шаги, которые выполнятся при вызове.

<MarginNote label="Имена">
Имена функций пишут маленькими буквами через подчёркивание: `taxi_price`, `send_sms`. Хорошее имя отвечает на вопрос «что сделать».
</MarginNote>

Важная деталь: строка с `def` сама по себе ничего не вычисляет. Python читает объявление, запоминает функцию под её именем и идёт дальше. Тело просыпается только при вызове: имя плюс круглые скобки с аргументами, `greet("Анна")`.

Два похожих слова здесь легко перепутать, поэтому зафиксируем. Параметр это имя в скобках объявления: `name`. Аргумент это конкретное значение, переданное при вызове: `"Анна"`. Параметры объявляются один раз, аргументы меняются от вызова к вызову:

```python
print(greet("Борис"))
print(greet("Вера"))
```

Одна функция, два вызова, два разных приветствия. Именно ради этого функции и существуют.

## return отдаёт значение и сразу выходит

У `return` две работы. Первая: отдать значение наружу, в место вызова. Вторая, менее очевидная: немедленно завершить функцию. Как только `return` сработал, функция закончена, и оставшиеся строки её тела не выполняются.

Вторая работа раскрывается в ветвлениях, знакомых вам по третьему уроку:

```python
def transport(rubles):
    if rubles < 300:
        return "автобус"
    return "такси"
```

Когда денег меньше 300 рублей, срабатывает первый `return`, и до последней строки дело не доходит. Когда условие ложно, ветка пропускается, и функция упирается во второй `return`. Такой приём называют ранним выходом, а несколько `return` в разных ветках это обычный читаемый стиль: каждая ветка сама отвечает за свой результат.

## print показывает человеку, return отдаёт программе

Теперь главная ловушка урока. Об неё спотыкается почти каждый новичок, поэтому пройдём медленно. Перед вами две функции, обе «складывают» числа:

```python
def add_print(a, b):
    print(a + b)

def add_return(a, b):
    return a + b
```

Вызовите каждую с аргументами 2 и 3, и на первый взгляд разницы не будет: на экране появится пятёрка. Но разница принципиальная. `print` показывает значение человеку, который смотрит в консоль, и на этом всё: самой программе число не достаётся. `return` отдаёт значение программе: его можно сохранить в переменную, сравнить с другим, передать в следующую функцию.

Проверим на практике. Попробуем сохранить «результат» первой функции:

```python
result = add_print(2, 3)
print(result)
```

На экране появятся две строки: сначала 5, потом None. Пятёрку напечатал `print` внутри функции. А в `result` легло None, специальное значение Python со смыслом «здесь пусто». Правило без исключений: функция, дошедшая до конца тела без `return`, возвращает None. Печать внутри ничего не меняет, `print` это не возврат.

Коварство None в том, что всплывает он не сразу. Взгляните:

```python
def double_price(x):
    print(x * 2)

total = double_price(450) + 100
```

Код напечатает 900 и тут же упадёт с ошибкой `TypeError`: к None нельзя прибавить 100. Симптом «функция вроде печатает всё правильно, но использовать результат не получается» почти всегда означает одно: внутри стоит `print` там, где нужен `return`.

## Необязательные параметры и аргументы по имени

В заголовке `delivery_price` мелькнула новинка: `express=False`. Это параметр со значением по умолчанию. Если при вызове аргумент не передан, Python молча подставляет умолчание. Поэтому `delivery_price(2)` работает: вес обязателен, а срочность нет.

```python
def greet(name, polite=True):
    if polite:
        return "Здравствуйте, " + name
    return "Привет, " + name

print(greet("Анна"))
print(greet("Борис", polite=False))
```

Второй вызов демонстрирует ещё один приём: аргумент передан по имени, `polite=False`. Такие аргументы называют именованными. Имя избавляет читателя от угадывания, что означало бы голое `False` в скобках, а когда умолчаний несколько, позволяет задать только нужное и пропустить остальные. Правило одно: в заголовке параметры с умолчаниями идут после обязательных, вариант `def greet(polite=True, name)` Python отвергнет ещё до запуска.

## Переменные живут внутри функции

Остался последний кирпичик: судьба переменных, созданных в теле. Ответ короткий: они живут только внутри функции и только пока длится один вызов.

```python
def delivery_price(weight_kg):
    price = 199 + 40 * weight_kg
    return price

total = delivery_price(2)
print(price)
```

Последняя строка упадёт с ошибкой `NameError: name 'price' is not defined`. Переменная `price` родилась внутри вызова и исчезла вместе с ним. Наружу вышло только значение, которое вынес `return`, и оно лежит в `total`. Зону, где имя существует, называют областью видимости. Для вас это защита, а не ограничение: внутри функции переменные можно называть свободно, не опасаясь столкновений с именами в остальной программе, и каждый вызов начинает работу со своим чистым набором.

## Частые ошибки

- **`print` вместо `return`.** На экране всё правильно, а проверка не проходит, или попытка посчитать что-то с результатом падает с `TypeError`. Случается это потому, что печать выглядит как результат. Лекарство: функция возвращает значение, а печатает пусть вызывающий код.
- **Вызов без скобок.** `greet` без скобок это сама функция как значение, а не её результат. После `message = greet` в переменной лежит функция; тело выполняется только со скобками, `greet("Анна")`.
- **Ожидание локальной переменной снаружи.** Посчитали `price` внутри функции, а печатаете после вызова: получите `NameError`. Наружу значение выносит только `return`.
- **Код после `return`.** Строки ниже сработавшего `return` не выполнятся никогда. Если добавленный в функцию код «почему-то не работает», проверьте, не стоит ли он за ранним выходом.

<Detail summary="Имя функции это обычная переменная">
Строка `def greet(...):` делает две вещи: создаёт объект-функцию и привязывает его к имени `greet`, точно так же, как `x = 5` привязывает число к имени `x`. Поэтому функцию можно положить в другую переменную: после `say = greet` вызов `say("Анна")` работает как родной. А `print(greet)` без скобок честно покажет, что лежит в имени: нечто вроде `<function greet at 0x7f9c...>`. Мысль «функции это такие же значения, как числа и строки» ещё не раз выстрелит в продвинутых темах Python, от сортировки с ключом до декораторов.
</Detail>
