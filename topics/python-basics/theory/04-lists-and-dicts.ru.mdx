# Списки и словари: много значений под одним именем

Представьте программу, которая считает корзину в интернет-магазине. Анна выбрала на Ozon двадцать товаров, и каждый нужно где-то держать. Из прошлых уроков вы знаете один способ хранения: переменные. Завести `item1`, `item2`, `item3` и дальше до `item20`? Уже к пятой строке видно, что затея провальная: код раздувается, а обойти все покупки разом невозможно. Хуже того, вы заранее не знаете, сколько будет товаров: корзина наполняется, пока программа работает, а переменные так появляться не умеют.

Для таких задач в Python есть коллекции, то есть значения, внутри которых лежат другие значения. В этом уроке вы освоите две главные. Список хранит значения по порядку, как очередь или полку. Словарь хранит пары «ключ и значение», как записная книжка. К концу урока вы соберёте корзину покупок, посчитаете её стоимость одной строкой, построите телефонную книжку и разберёте ловушку, на которую наступает почти каждый новичок.

## Список: полка с пронумерованными местами

Список создаётся квадратными скобками, значения внутри перечисляются через запятую. Полезная картинка: полка с пронумерованными местами. На каждом месте лежит одно значение, номер места называется индексом, и отсчёт, как и у строк из второго урока, начинается с нуля.

```python
cart = ["хлеб", "молоко", "сыр"]
print(cart)
print(len(cart))
print(cart[0])
```

Знакомая функция `len()` для списка возвращает количество элементов: здесь их три, сколько бы букв ни было в самих словах. Обращение по индексу повторяет строки: `cart[0]` даёт первое значение, `cart[2]` последнее из трёх. Схема выше показывает, почему «третий элемент» и «индекс 3» не одно и то же: места с номером 3 на этой полке нет, и запрос `cart[3]` остановит программу с ошибкой `IndexError`.

Хранить в списке можно значения любого типа: строки, числа, булевы значения, даже другие списки. Чаще всего элементы однотипны, так с ними проще работать, но Python этого не требует. Списки вокруг вас в каждом приложении: история операций в банковском приложении, плейлист, очередь заказов в доставке. Везде, где значения выстроены по порядку и их количество меняется, программист достаёт список.

## Список живёт и меняется

Со строками во втором уроке вы выяснили неприятную на первый взгляд вещь: изменить строку нельзя, любой метод возвращает новую. Список устроен противоположным образом: он изменяемый. Три операции покрывают большинство задач. Метод `.append()` дописывает значение в конец. Присваивание по индексу кладёт на существующее место новое значение. Метод `.remove()` находит первое вхождение значения и убирает его.

```python
cart = ["хлеб", "молоко", "сыр"]
cart.append("яблоки")
cart[0] = "батон"
cart.remove("сыр")
print(cart)
```

Пройдите код глазами сверху вниз. После `append` полка удлинилась на одно место справа. Присваивание `cart[0] = "батон"` заменило хлеб, длина не изменилась. Вызов `remove` убрал сыр, а соседи сдвинулись влево, не оставив дырки. Печать покажет `['батон', 'молоко', 'яблоки']`. Заметьте главное: все три операции меняют сам список. Новый объект не создаётся, меняется тот же.

Проверить, лежит ли значение на полке, помогает оператор `in`, знакомый по строкам. Он возвращает булево значение, то самое из урока про условия, поэтому его удобно ставить прямо в `if`.

```python
cart = ["батон", "молоко", "яблоки"]
print("молоко" in cart)
if "сыр" not in cart:
    print("надо купить сыр")
```

Со списком чисел Python посчитает сводку без единой лишней строки: `sum()` складывает все элементы, `min()` и `max()` находят наименьший и наибольший. Ниже живой пример, где корзина и цены в рублях работают парой. Допишите свой товар ещё одним вызовом `append` в каждый из списков и запустите снова: количество, итог и максимум пересчитаются сами.

## Словарь: поиск по имени, а не по номеру

Полка с номерами хороша, пока важен порядок. Теперь другая задача: хранить телефоны знакомых. В списке номер Бориса лежал бы, допустим, третьим, и чтобы найти его, пришлось бы помнить позицию. Записная книжка устроена удобнее: вы ищете по имени. Для этого в Python есть словарь. Он хранит записи, где у каждого значения есть собственный ключ, и находит значение по ключу.

```python
phones = {"Анна": "+7 901 123-45-67", "Борис": "+7 902 555-66-77"}
print(phones["Анна"])
```

Словарь создаётся фигурными скобками. Записи перечисляются через запятую, внутри каждой сначала идёт ключ, затем двоеточие, затем значение. Чтение выглядит как у списка, только в квадратных скобках стоит не номер места, а ключ. Словари повсюду, где у данных есть естественное имя: цена по названию товара, население по городу, поля профиля на Госуслугах.

Добавление и перезапись делаются одной и той же конструкцией:

```python
phones = {"Анна": "+7 901 123-45-67"}
phones["Вера"] = "+7 903 777-88-99"
phones["Анна"] = "+7 901 000-00-00"
print(len(phones))
```

Правило простое. Если ключа не было, появляется новая запись, так в книжку попала Вера. Если ключ был, старое значение затирается новым, так у Анны сменился номер. Двух записей с одним ключом в словаре не бывает, поэтому `len` напечатает 2, а не 3.

А что при чтении ключа, которого нет? Квадратные скобки остановят программу с ошибкой `KeyError`, ровно как список останавливал её с `IndexError`. Когда отсутствие ключа это штатная ситуация, а не поломка, берите метод `.get()`: вторым аргументом он принимает запасное значение и возвращает его вместо ошибки.

```python
phones = {"Анна": "+7 901 123-45-67"}
print(phones.get("Борис", "номера нет"))
```

Оператор `in` со словарём проверяет только ключи. Проверка `"Анна" in phones` даст `True`, а проверка по самому номеру телефона даст `False`, даже если номер в словаре лежит: в значения оператор не заглядывает.

Значением может быть что угодно: строка, число, другой словарь и даже целый список. Так под одним ключом живёт сразу несколько значений:

```python
orders = {"Анна": ["хлеб", "молоко"], "Борис": ["сыр"]}
orders["Борис"].append("чай")
print(orders["Борис"])
```

## Сюрприз: копия, которой не было

Теперь обещанная ловушка. Интуиция подсказывает, что запись `b = a` изготавливает второй такой же список. Проверьте:

```python
a = ["хлеб", "молоко"]
b = a
b.append("сыр")
print(a)
```

Печать выведет `['хлеб', 'молоко', 'сыр']`. Сыр добавлялся через `b`, а изменился будто бы и `a`. Разгадка в том, что второго списка никогда не существовало. Имя переменной это бирка, привязанная к объекту, и строка `b = a` просто вешает вторую бирку на тот же самый список. Список один, бирок две, поэтому изменение видно через любую из них.

Когда копия действительно нужна, попросите её явно: `b = a.copy()`. С этого момента списков два и живут они независимо. Заметьте, что со строками и числами такой сюрприз невозможен, и вот почему: их нельзя изменить на месте. Две бирки на одной строке безобидны, ведь никакая операция эту строку не поменяет. Запомните механику бирок: она же сработает в седьмом уроке, когда вы начнёте передавать списки в функции.

## Частые ошибки

- **Отсчитывать индексы с единицы.** `cart[1]` это второй элемент, а не первый. Привычка из жизни сильнее логики, поэтому ошибка всплывает даже у опытных. Сверяйтесь с первым элементом: он всегда `cart[0]`.
- **Присваивать результат append.** Запись `cart = cart.append("сыр")` уничтожает список: метод меняет его на месте, а возвращает `None`, и это `None` попадает в `cart`. Вызывайте `cart.append("сыр")` отдельной строкой, без присваивания.
- **Читать сомнительный ключ квадратными скобками.** Если ключа может не оказаться, `phones["Вера"]` уронит программу с `KeyError`. Для штатного «может и не быть» есть `.get()` с запасным значением.
- **Ждать от `b = a` копию.** Оба имени указывают на один объект, изменения видны через оба. Нужна независимая копия, пишите `a.copy()`.

<Detail summary="Почему словарь находит ключ мгновенно">
Список ищет значение перебором: чтобы понять, есть ли сыр в корзине из тысячи позиций, Python в худшем случае просмотрит всю тысячу. Словарь устроен хитрее. По ключу он вычисляет число, называемое hash, и это число сразу указывает, где внутри словаря лежит значение. Поиск не зависит от размера: что десять записей, что миллион, ответ приходит за одинаковое время. Отсюда же ограничение: ключом может быть только неизменяемое значение, например строка или число. Список ключом не станет: он меняется, а вместе с ним менялся бы и hash, и словарь потерял бы свою запись.
</Detail>
