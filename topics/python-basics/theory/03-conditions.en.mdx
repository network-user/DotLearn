# How a program makes decisions

Every program you've written so far has run like a recipe: line by line, top to bottom, no detours. Now picture a museum ticket counter. Anna is 4, she gets in free. Boris is 15, his ticket costs 150 rubles. Vera is 34, hers costs 400. It's one program at the counter, but it has to behave differently depending on age. A recipe made of straight-line instructions can't do that.

In this lesson you'll teach a program to pick a road. By the end you'll be writing code that decides on its own what to print and what to compute: let a visitor through or ask for a ticket, grant a discount or refuse one. It's the first step from calculator to a program that reacts to data.

## Mental model: a hallway of checkpoints

Before writing code, let's fix a picture in your head. A program with conditions is like a hallway lined with checkpoints, one after another. Each checkpoint asks exactly one question, answerable only "yes" or "no". Answer "yes", and the checkpoint leads you through its own side door: its branch of code runs, and you leave the hallway. Answer "no", and you move on to the next checkpoint. If every answer turns out "no", a door for everyone else is waiting at the end of the hallway.

Remember the hallway's key property from the figure: checks run strictly in order, and the first "yes" stops the whole chain. Building that hallway in Python takes two building blocks: yes-or-no questions and the branch itself. Let's start with questions.

## Questions answered True or False

A question in Python is written as a comparison. Any comparison has exactly two possible answers: `True` and `False`. These are values of a separate type, `bool`, the fourth in your collection after `int`, `float`, and `str` from the first lesson.

There are six comparison operators:

- `==` equal to (exactly two signs)
- `!=` not equal to
- `>` greater than, `<` less than
- `>=` greater than or equal to, `<=` less than or equal to

```python
age = 15
print(age >= 18)
print(age < 18)
print(age == 15)
print(type(age == 15))
```

The program prints `False`, `True`, `True`, and `<class 'bool'>`. A comparison's answer is an ordinary value: you can print it, store it in a variable, and, most importantly, plug it straight into a branch.

Look closely at `==`. You already use a single equals sign: `age = 15` puts a number into a variable. Two signs ask a question: `age == 15` asks whether the current value equals fifteen, and returns `True` or `False`. Assignment and comparison look alike but do opposite things, which makes confusing the two the single most common beginner mistake. We'll come back to it at the end of the lesson.

## if: run code only on "yes"

The word `if` creates a branch. After it comes a condition, and the line must end with a colon. Below it sits the branch's body: the lines that run only when the condition is true.

```python
total = 3500

if total >= 3000:
    print("Доставка бесплатно")
print("Спасибо за заказ")
```

The order came to 3500 rubles, the condition `total >= 3000` came out `True`, and the program printed both lines. Change the amount to 2000, and the shipping line disappears while "thanks for your order" stays.

How did Python know only the shipping line belonged to `if`? By indentation. The body is shifted right by exactly 4 spaces, while the "thanks for your order" line sits back at the left edge, so it's outside the branch and always prints. In most languages, block boundaries are marked with special brackets or keywords, and indentation is purely cosmetic. Python flips that: indentation is syntax, not decoration. Code with different indentation is a different program, code with uneven indentation is an error. The standard is simple: 4 spaces per level, and every line of one body shifted the same amount.

## else: the fallback plan

Often you need a second scenario: if the condition fails, do something else. For that, after the `if` body comes `else` with its own colon and its own body. `else` has no condition of its own: it fires whenever `if` got a "no".

```python
total = 1800

if total >= 3000:
    print("Доставка бесплатно")
else:
    print("Доставка 300 рублей")
```

Now exactly one of the two branches runs, never both and never neither: either the total reached free shipping, or the program honestly asks for 300 rubles.

## elif: a chain of checks

Back to the museum counter, where there are three scenarios. The word `elif`, short for "else if", adds extra questions.

```python
age = 15

if age < 7:
    print("Бесплатно")
elif age < 18:
    print("Билет 150 рублей")
else:
    print("Билет 400 рублей")
```

This is the hallway from the figure, written as code. Python checks conditions top to bottom and runs the first branch whose condition turns out true, skipping the rest without even checking them. For Boris, 15 years old, the first question gets a "no", the second a "yes": "ticket costs 150 rubles" prints, and the chain is done.

A useful consequence follows: the second branch doesn't need to check "between 7 and 18". Once the program reaches `elif`, the first question has already gotten a "no", so the age is definitely not under seven. Every following branch silently knows all the previous ones failed. There can be as many `elif` branches as you like, and `else` is optional, but if it's there, it comes last.

## and, or, not: several conditions at once

Sometimes one question isn't enough. Shipping is free if the order is at least 3000 rubles or the customer has a subscription. A discount applies if it's a weekday and before noon. Three words combine conditions like that:

- `and` gives `True` only when both conditions are true
- `or` gives `True` when at least one is true
- `not` flips the answer: turns `True` into `False` and back

```python
total = 2400
is_premium = True

if total >= 3000 or is_premium:
    print("Доставка бесплатно")
else:
    print("Доставка 300 рублей")
```

The total didn't reach the threshold, but `is_premium` holds `True`, so `or` came out true and shipping is free. Notice that `is_premium` is already a `bool` value, so you plug it into the condition as-is. No need to write `is_premium == True`.

Python lets you write an "in between" check quite compactly. The condition "age from 18 to 59 inclusive" can be built from two comparisons: `age >= 18 and age < 60`. Or with one double comparison: `18 <= age < 60`. Python reads it the way a mathematician would, and both forms mean the same thing. Many languages don't allow this trick, so it's a small gift from Python.

## Surprise: every value has its own truthiness

So far a comparison has sat inside the condition. But Python accepts any value there, and here's the lesson's surprise: an empty string `""`, zero `0`, and an empty list `[]` count as false in a condition, while anything non-empty and non-zero counts as true.

```python
name = ""

if name:
    print("Привет, " + name)
else:
    print("Вы не представились")
```

The string is empty, so `if name:` sent the program to `else`. Experienced Python developers use this as a shortcut: `if name:` reads as "if the name is filled in", without an explicit comparison against an empty string. You'll meet lists in the next lesson, but the rule is worth remembering now: an empty collection acts false, a non-empty one acts true.

<Callout type="warning">
Truthiness and equality aren't the same thing. Zero behaves as false in a condition, but it's still a number, not the word `False`. Use truthiness to ask "is there something there", and when the exact value matters, write an explicit comparison.
</Callout>

## Frequent mistakes

- **A single `=` instead of two.** The line `if age = 18:` ends in a `SyntaxError`. Blame the habit of assignment. Inside a condition it's always a comparison: `if age == 18:`.
- **A missing colon.** Lines with `if`, `elif`, and `else` must end with a colon. Recent Python versions hint at it directly in the error text: `expected ':'`.
- **Uneven indentation.** When lines in the same body shift by a different number of spaces, the program crashes with `IndentationError`. Keep exactly 4 spaces and don't mix spaces with tabs; a good editor handles this for you.
- **Comparing a number to a string.** The expression `"5" == 5` calmly returns `False`, no error at all: a string and a number are different types, and they aren't equal. Data coming from outside, a file, a web form, often arrives as strings. Convert it first using the function you know from the first lesson: `int("5") == 5` gives `True`.

<Detail summary="Why Python chose indentation over brackets">
In most languages that existed by the late 1980s, a condition's body was wrapped in brackets or special keywords, and indentation was purely a matter of taste. As a result, code could look one way and run another: the eye reads indentation, the compiler reads brackets, and the compiler always wins the argument. Guido van Rossum, Python's author, removed the source of the mismatch itself: since people navigate by indentation anyway, let indentation define the program's structure. Code's appearance and its meaning always match. A 4-space indent width is fixed in PEP 8, Python's official style guide.
</Detail>
