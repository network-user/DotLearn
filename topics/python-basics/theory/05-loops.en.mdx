# Loops: one command instead of thirty lines

Anna exported a month of purchases from her banking app: thirty numbers, from 90 rubles for coffee to 15000 for headphones from Ozon. She needs the total spent. Reaching for a calculator feels wasteful when there's a whole programming language at hand. But writing thirty lines like `total = total + 90` isn't the answer either: next month there will be thirty-two purchases, and you'd have to rewrite everything. What you really want is to hand the computer one command: "go through every number and add them up."

That's exactly what loops were invented for. In this lesson you'll learn to iterate over lists and strings with `for`, generate number sequences with `range`, build up a total in an accumulator variable, and spin a `while` loop while a condition holds true. Along the way you'll also see how to accidentally write a program that never stops, and what to do so that doesn't happen.

## The for loop: the body runs once per element

Let's start with Python's most common loop. Anna has a list of purchases, like in the previous lesson, and first she just wants to print every price:

```python
purchases = [90, 250, 15000]
for price in purchases:
    print(price)
```

The `for` line reads almost like a plain sentence: "for each price in purchases, do the following." After it comes the loop body: lines indented by four spaces, exactly like `if` from the lesson on conditions. Python runs the body once for every element of the list. One such repetition is called a pass, or an iteration.

The variable `price` is the loop variable, and you pick its name yourself. On the first pass Python drops the list's first element, the number 90, into it and runs the body. On the second pass `price` holds 250, on the third 15000. Once the elements run out, the loop is done, and the program continues with the line after the body.

## The main pattern: an accumulator

Printing prices is nice, but Anna needs a total. Here's the trick: before the loop, create a variable with a starting value of 0, and inside the loop add each element to it. That variable is called an accumulator:

```python
purchases = [90, 250, 15000]
total = 0
for price in purchases:
    total += price
print(total)
```

The line `total += price` is familiar from the first lesson: shorthand for `total = total + price`. Each pass takes the accumulator's current value, adds the next price, and drops the result back in. After the third pass `total` holds 15340, and the same program, unchanged, will total three purchases or three thousand just as well.

Need a count instead of a sum, add one. Need a product, start from one and multiply. Need a long string, start empty and glue on pieces.

## for iterates over strings too

You remember from the strings lesson: a string is a sequence of characters. So `for` handles it the same way it handles a list, except the loop variable gets one character at a time:

```python
for letter in "Анна":
    print(letter)
```

The loop runs four passes and prints "А", "н", "н", "а", each letter on its own line. The trick comes in handy when checking every character of a password or counting vowels in a word.

## range: numbers on demand

What if there's nothing to iterate over, but you still need to repeat an action, say, five times? Building a list of 0, 1, 2, 3, 4 by hand feels unnecessary. That's what `range` is for:

```python
for n in range(5):
    print(n)
```

`range(5)` produces five numbers in a row, starting from zero: 0, 1, 2, 3, 4. Here's the lesson's first honest surprise: there's no 5 in the output. Counting starts at zero, and the right boundary never makes it into the range. Read `range(5)` as "five numbers starting from zero," not "numbers up to and including five."

`range` also has a two-argument form: `range(a, b)` starts at `a`, but the rule about the right boundary still applies, `b` never makes it into the range.

```python
for year in range(2024, 2027):
    print(year)
```

The program prints 2024, 2025, and 2026. 2027 never shows up in the output. That's why, when you need numbers 1 through 10 inclusive, you write `range(1, 11)`: the right boundary is always one more than the last number you actually want. This isn't a language quirk: `range(5)` produces exactly five items, so the number of passes is obvious from the notation, and you'll meet the same half-open-boundary rule again in the next lesson on slices.

## A counter with a condition

The second most common task after a sum: count how many elements match a condition. For that, drop an `if` from the branching lesson inside the loop. Anna wants to know how many of her purchases cost more than 1000 rubles:

```python
prices = [350, 1200, 90, 2500, 640]
expensive = 0
for price in prices:
    if price > 1000:
        expensive += 1
print(expensive)
```

It's the same accumulator, except it adds one, not the price, and only when the condition is true, not every pass. Notice the double indent: the line `expensive += 1` sits eight spaces in, because it's inside `if`, which is itself inside `for`. The program prints 2: 1200 and 2500 matched the condition.

Now combine both ideas and experiment. Run the code below, then add a 5000 purchase to the list and rerun: both numbers change. After that, change the threshold from 1000 to 500 and confirm against the list that the count of expensive purchases climbs to four.

## while: repeat while a condition holds

`for` shines when you already know what to iterate over: a list, a string, a range. But Boris is saving up for a phone that costs 30000 rubles, putting aside 7000 a month, and wants to know how many months that will take. How many passes it takes is unknown in advance, that's the whole point of the question. Python has a second loop for cases like this, `while`:

```python
savings = 0
months = 0
while savings < 30000:
    savings += 7000
    months += 1
print(months)
```

`while` checks its condition before every pass. As long as `savings` holds less than 30000, the body runs again; the moment the condition turns false, the loop ends. Here the body fires five times: savings grow as 7000, 14000, 21000, 28000, and 35000, and at 35000 the check no longer passes. The program prints 5.

The rule for choosing between the two loops is simple. Know what to iterate over, or how many times to repeat, reach for `for`. Know only the condition that should keep things going, reach for `while`.

`while` comes with the most famous loop trap of all. Look at this code, but don't run it:

```python
savings = 0
while savings < 30000:
    print("копим")
```

The body never changes `savings`, so the condition stays true forever, and the loop never ends. A loop like this is called infinite: the program hangs and keeps printing "saving up" until something outside stops it.

<Callout type="warning">
The body of a `while` loop must contain a line that pushes the condition toward false: bumping a counter, shrinking a remainder, flipping a flag. If a loop won't stop, the first thing to check is whether the variable from the condition is actually changing. The sandbox on this page cuts off a hung loop with a timeout, but a real program will hang until someone kills it by force.
</Callout>

## break and continue: leave early, skip a pass

Two keywords let you steer a loop from the inside. `break` immediately ends the whole loop:

```python
for price in [90, 250, 9000, 60]:
    if price > 1000:
        break
    print(price)
```

The program prints 90 and 250. At 9000 `break` fired, the loop ended entirely, and 60 never got a look. That's how you search for the first matching element: find it and leave right away, without wasting time on the rest of the list.

`continue` acts more gently: it skips the rest of the current pass and jumps straight to the next element:

```python
for n in range(5):
    if n == 2:
        continue
    print(n)
```

The output holds 0, 1, 3, and 4: at 2 the `print` didn't run, but the loop kept going. Both keywords work in `for` and in `while` alike. Use them sparingly: with three different `break` statements in one loop, a reader struggles to tell at which point the loop actually ends.

## Frequent mistakes

- **Losing the colon or the indent.** A `for` or `while` line ends with a colon, and the body shifts by four spaces. Forget the colon, get `SyntaxError`; forget the indent, get `IndentationError`. That happens because Python determines a body's boundaries purely from indentation.
- **Zeroing the accumulator inside the loop.** If the line `total = 0` ends up inside the body, every pass wipes out everything collected so far, and after the loop `total` holds only the last addition. Zero the accumulator exactly once, strictly before the loop.
- **Expecting range's right boundary to be included.** `range(1, 10)` ends at 9. It feels like you "asked for up to ten," but the right boundary is never included. Need a ten, write `range(1, 11)`.
- **Forgetting to change the condition variable in while.** The classic source of infinite loops: the condition checks a variable the body never touches. Before writing the body, decide which line will push the loop toward the finish.

<Detail summary="The loop variable survives after the loop">
After a `for` loop finishes, the loop variable doesn't disappear: it keeps the last element. If you write `print(price)` after the loop over purchases, the program prints 15000. Sometimes that's handy while debugging, but more often it's a surprise: the name looks free, yet it's holding the tail end of an old loop. In many other languages the loop variable is visible only inside the loop, Python isn't one of them. Simple rule: don't reuse a loop variable's name for new tasks further down in the code.
</Detail>
