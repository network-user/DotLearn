# Циклы: одна команда вместо тридцати строк

Анна выгрузила из банковского приложения список покупок за месяц: тридцать чисел, от 90 рублей за кофе до 15000 за наушники с Ozon. Ей нужна общая сумма расходов. Считать на калькуляторе обидно: под рукой целый язык программирования. Но и писать тридцать строк вида `total = total + 90` не выход: в следующем месяце покупок будет тридцать две, и всё придётся переписывать. Хочется отдать компьютеру одну команду: «пройди по всем числам и сложи их».

Ровно для этого придуманы циклы. В этом уроке вы научитесь перебирать списки и строки с помощью `for`, получать ряды чисел через `range`, собирать итог в переменную-накопитель и крутить `while`, пока условие истинно. Заодно узнаете, как случайно написать программу, которая никогда не остановится, и что делать, чтобы этого не случилось.

## Цикл for: тело выполняется для каждого элемента

Начнём с самого частого цикла в Python. У Анны есть список покупок, как в прошлом уроке, и для начала она хочет просто напечатать каждую цену:

```python
purchases = [90, 250, 15000]
for price in purchases:
    print(price)
```

Строка с `for` читается почти как русское предложение: «для каждой цены в покупках сделай следующее». После неё идёт тело цикла: строки с отступом в четыре пробела, точно так же, как у `if` из урока про условия. Python выполняет тело один раз для каждого элемента списка. Одно такое повторение называют витком или итерацией.

Переменная `price` называется переменной цикла, её имя вы придумываете сами. На первом витке Python кладёт в неё первый элемент списка, число 90, и выполняет тело. На втором витке в `price` лежит 250, на третьем 15000. Элементы закончились, значит цикл завершён, и программа продолжается со строки после тела.

## Главный паттерн: накопитель

Печатать цены приятно, но Анне нужна сумма. Приём выглядит так: до цикла заводят переменную со стартовым значением 0, а внутри цикла прибавляют к ней каждый элемент. Такую переменную называют накопителем:

```python
purchases = [90, 250, 15000]
total = 0
for price in purchases:
    total += price
print(total)
```

Запись `total += price` знакома вам по первому уроку: это сокращение для `total = total + price`. Каждый виток берёт текущее значение накопителя, прибавляет очередную цену и кладёт результат обратно. После третьего витка в `total` лежит 15340, и та же программа без единой правки посчитает и три покупки, и три тысячи.

## for перебирает и строки

Из урока про строки вы помните: строка это последовательность символов. Поэтому `for` работает с ней так же, как со списком, только переменная цикла получает по одному символу:

```python
for letter in "Анна":
    print(letter)
```

Цикл сделает четыре витка и напечатает «А», «н», «н», «а», каждую букву на отдельной строке. Приём пригодится, когда нужно проверить каждый символ пароля или посчитать гласные в слове.

## range: числа по заказу

А если перебирать нечего, но действие нужно повторить, скажем, пять раз? Собирать руками список из чисел 0, 1, 2, 3, 4 не хочется. Для таких случаев есть `range`:

```python
for n in range(5):
    print(n)
```

`range(5)` выдаёт пять чисел подряд, начиная с нуля: 0, 1, 2, 3, 4. Здесь спрятан первый честный сюрприз урока: пятёрки в выводе нет. Счёт начинается с нуля, а правая граница в диапазон не входит. Читайте `range(5)` как «пять чисел от нуля», а не как «числа до пяти включительно».

У `range` есть форма с двумя аргументами: `range(a, b)` начинает с `a`, но правило про правую границу никуда не девается, число `b` в диапазон не входит.

```python
for year in range(2024, 2027):
    print(year)
```

Программа напечатает 2024, 2025 и 2026. Года 2027 в выводе не будет. Поэтому, когда нужны числа от 1 до 10 включительно, пишут `range(1, 11)`: правая граница на единицу больше последнего нужного числа. Это не каприз языка: в `range(5)` ровно пять элементов, поэтому по записи сразу видно число витков, и то же правило полуоткрытых границ вы встретите в следующем уроке про срезы.

## Счётчик с условием

Вторая по популярности задача после суммы: посчитать, сколько элементов подходит под условие. Для этого внутрь цикла ставят `if` из урока про ветвление. Анна хочет узнать, сколько её покупок оказались дороже 1000 рублей:

```python
prices = [350, 1200, 90, 2500, 640]
expensive = 0
for price in prices:
    if price > 1000:
        expensive += 1
print(expensive)
```

Это тот же накопитель, только прибавляется не цена, а единица, и не на каждом витке, а лишь когда условие истинно. Обратите внимание на двойной отступ: строка `expensive += 1` сдвинута на восемь пробелов, потому что она внутри `if`, который сам находится внутри `for`. Программа напечатает 2: под условие подошли 1200 и 2500.

## while: повторять, пока условие истинно

`for` хорош, когда заранее есть что перебирать: список, строку, диапазон. Но Борис копит на телефон за 30000 рублей, откладывая по 7000 в месяц, и хочет узнать, сколько месяцев уйдёт. Сколько витков понадобится, заранее неизвестно: это и есть ответ задачи. Для таких случаев в Python существует второй цикл, `while`:

```python
savings = 0
months = 0
while savings < 30000:
    savings += 7000
    months += 1
print(months)
```

`while` проверяет условие перед каждым витком. Пока в `savings` лежит меньше 30000, тело выполняется ещё раз; как только условие стало ложным, цикл заканчивается. Здесь тело сработает пять раз: накопления растут как 7000, 14000, 21000, 28000 и 35000, а на 35000 проверка уже не проходит. Программа напечатает 5.

Правило выбора между двумя циклами простое. Знаете, что перебирать или сколько раз повторять, берите `for`. Знаете только условие, при котором надо продолжать, берите `while`.

С `while` связана самая известная ловушка циклов. Посмотрите на код, но не запускайте его:

```python
savings = 0
while savings < 30000:
    print("копим")
```

Тело не меняет `savings`, значит условие навсегда останется истинным, и цикл не закончится никогда. Такой цикл называют бесконечным: программа зависает и печатает «копим», пока её не остановят снаружи.

<Callout type="warning">
В теле `while` обязана быть строка, которая приближает условие к ложному: увеличивает счётчик, уменьшает остаток, меняет флаг. Если цикл не останавливается, первым делом проверьте, меняется ли переменная из условия. Песочница на этой странице прерывает зависший код по таймауту, но настоящая программа будет висеть, пока её не завершат принудительно.
</Callout>

## break и continue: выйти раньше и пропустить виток

Двумя служебными словами циклом можно управлять изнутри. `break` немедленно завершает весь цикл:

```python
for price in [90, 250, 9000, 60]:
    if price > 1000:
        break
    print(price)
```

Программа напечатает 90 и 250. На числе 9000 сработал `break`, цикл закончился целиком, и до 60 дело не дошло. Так ищут первый подходящий элемент: нашли и сразу вышли, не тратя время на остаток списка.

`continue` действует мягче: он пропускает остаток текущего витка и сразу переходит к следующему элементу:

```python
for n in range(5):
    if n == 2:
        continue
    print(n)
```

В выводе будут 0, 1, 3 и 4: на двойке `print` не выполнился, но цикл продолжил работу. Оба слова действуют и в `for`, и в `while`. Пользуйтесь ими умеренно: когда в одном цикле три разных `break`, читателю трудно понять, в какой момент цикл закончится.

## Частые ошибки

- **Потерять двоеточие или отступ.** Строка с `for` и `while` заканчивается двоеточием, а тело сдвигается на четыре пробела. Забудете двоеточие, получите `SyntaxError`; забудете отступ, получите `IndentationError`. Так происходит потому, что границы тела Python определяет только по отступам.
- **Обнулить накопитель внутри цикла.** Если строка `total = 0` попала в тело, каждый виток стирает всё собранное, и после цикла в `total` останется только последняя прибавка. Накопитель обнуляют ровно один раз, строго до цикла.
- **Ждать, что правая граница range входит в диапазон.** `range(1, 10)` заканчивается на 9. Кажется, что вы «попросили до десяти», но правая граница не входит никогда. Нужна десятка, пишите `range(1, 11)`.
- **Забыть изменить переменную условия в while.** Классический источник бесконечных циклов: условие проверяет переменную, которую тело не трогает. Прежде чем писать тело, решите, какая строка будет приближать цикл к финишу.

<Detail summary="Переменная цикла живёт и после цикла">
После завершения `for` переменная цикла не исчезает: в ней остаётся последний элемент. Если после цикла по покупкам написать `print(price)`, программа напечатает 15000. Иногда это удобно при отладке, но чаще становится сюрпризом: имя выглядит свободным, а в нём лежит хвост старого цикла. Во многих других языках переменная цикла видна только внутри цикла, в Python это не так. Правило простое: не переиспользуйте имя переменной цикла для новых задач ниже по коду.
</Detail>
