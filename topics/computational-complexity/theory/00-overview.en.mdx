title: "Computational complexity and asymptotics: a review of the topic"
estimatedMinutes: 3

You have two functions that do the same thing, and you need to choose the one that can handle the data growth. On a test set of hundreds of lines, both respond instantly, and there seems to be no difference. And then a million lines arrive in production - and one function still fits in a split second, and the second freezes for minutes. This topic is about how to understand in advance, without running code on big data, which functions will break under load and why. The stopwatch is deceiving here: it depends on the hardware and the mood of the processor. The complexity looks deeper - at how the algorithm's performance grows as the input gets larger.

**What you will learn to do.** Count the work of the algorithm in steps, not in seconds, so that comparisons do not depend on your laptop; read labels like `O(n)` and `O(n log n)` and understand what they mean; look at someone else's function and name its complexity without running it; see the hidden cost of operations with lists and estimate not only time, but also memory.

**Topic route.** Four sections, each with a sandbox, where the code can be immediately run directly in the browser:

1. **Steps instead of seconds** - why the time on the clock lies about the speed of the algorithm and how to calculate the work in elementary operations from the size of the input.
2. **Asymptotics** - what `O`, `Theta` and `Omega` mean, how “no worse than `n²`” differs from “exactly `n²`”, and how to simplify `3n² + 5n + 17` to one term.
3. **Code analysis** - rules for reading complexity directly from the code: adding consecutive blocks, multiplying nested loops, finding out the logarithm by halving.
4. **Memory and Amortization** - how to value additional memory along with time and why `list.append` is called `O(1)` when sometimes it copies the entire list.

**What you need in advance.** There is no need to install anything: Python runs directly in the browser, the sandboxed code runs on the page. It’s enough to be able to read simple functions with loops in Python - we’ll cover everything about complexity, asymptotics and analysis along the way, starting from the very basics.
