`@retry` повторяет вызов при ошибке - но сколько раз? Два? Пять? Хочется писать `@retry(3)` и настраивать декоратор под каждое место. Но тут возникает развилка: `@retry` без скобок и `@retry(3)` со скобками - синтаксически разные вещи, и обычный декоратор так не умеет. Чтобы декоратор принимал собственные аргументы, нужен ещё один уровень вложенности. В этом разделе мы разберём три этажа функций, на которых стоит любой настраиваемый декоратор.

К концу раздела Вы поймёте, чем `@retry` отличается от `@retry(3)`, напишете фабрику декораторов с тремя уровнями `def` и научитесь читать эту матрёшку, не путаясь, какой уровень что принимает.

## Аргумент декоратора требует лишнего этажа

Вспомните: `@log` означает `func = log(func)`. А что означает `@retry(3)`? Сначала вычисляется `retry(3)` - это вызов, он должен вернуть декоратор. И уже этот декоратор применяется к функции: `func = retry(3)(func)`. Значит, `retry(3)` обязан вернуть функцию-декоратор. Отсюда три уровня.

Каждый этаж захватывает что-то своё. Внешняя функция `retry(times)` принимает аргумент декоратора и захватывает `times`. Средняя `decorator(func)` принимает оборачиваемую функцию и захватывает `func`. Внутренняя `wrapper(*args, **kwargs)` принимает аргументы вызова и видит обе свободные переменные - `times` и `func` - по правилам замыкания.

## Соберём фабрику по этажам

Начнём с простого примера: декоратор `repeat(n)`, который вызывает функцию `n` раз подряд и возвращает результат последнего вызова.

```python
def repeat(n):
    def decorator(func):
        def wrapper(*args, **kwargs):
            result = None
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator
```

Прочитаем матрёшку изнутри. `wrapper` вызывает `func` ровно `n` раз; `n` оно берёт из внешней `repeat`, `func` - из `decorator`, обе захвачены замыканием. `decorator` возвращает `wrapper`. `repeat` возвращает `decorator`. Теперь применим:

```python
@repeat(3)
def announce():
    print("привет")

announce()
```

Печатается `привет` три раза. Разворачивается это так: `repeat(3)` вернул `decorator`, затем `decorator(announce)` вернул `wrapper`, и имя `announce` стало указывать на `wrapper`. Вызов `announce()` крутит цикл на три итерации.

## @retry(3) и @retry - это разные вещи

Здесь кроется ловушка, на которую попадаются все. Фабрику `repeat` нельзя применять без скобок. `@repeat` (без `(3)`) передаст в `repeat` саму функцию вместо числа: `n` станет функцией, а `range(n)` упадёт. Скобки обязательны, потому что декоратором служит не `repeat`, а то, что возвращает `repeat(3)`.

```python
@repeat(2)
def beep():
    print("бип")

beep()
```

Печатается `бип` дважды. А вот `@repeat` без скобок сломался бы: `repeat` получил бы `beep` в параметр `n` и вернул `decorator`, ожидающий функцию, но `decorator` тут же применился бы... не к чему - имя `beep` уже стало бы `decorator`. Признак: «фабрика декораторов» всегда вызывается со скобками.

<Callout type="warning">Декоратор-фабрику (`@repeat(2)`) всегда пишут со скобками, даже если аргументов нет по смыслу. Обычный декоратор (`@log`) - всегда без скобок. Скобки - это вызов фабрики, который и порождает настоящий декоратор.</Callout>

## Потрогайте руками

Запустите код: `@tag("INFO")` оборачивает результат функции в префикс. Сейчас тег фиксирован при декорировании. Попробуйте создать вторую функцию с другим тегом, например `@tag("WARN")`, и убедитесь, что каждая обёртка помнит свой `prefix` независимо - это та же изоляция замыканий, что у фабрик из раздела про closures.

## Один декоратор, работающий и со скобками, и без - сложнее

Иногда хочется, чтобы декоратор работал в обеих формах: `@cache` и `@cache(maxsize=100)`. Это требует развилки внутри: проверить, передали ли первым аргументом функцию (форма без скобок) или настройку (форма со скобками). Приём рабочий, но запутанный, и почти всегда лишний - стандартная библиотека предпочитает явность.

```python
import functools

def trace(func=None, *, label="TRACE"):
    if func is None:
        return functools.partial(trace, label=label)
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return f"[{label}] " + str(func(*args, **kwargs))
    return wrapper
```

Если `func` передали (форма `@trace`), декорируем сразу. Если нет (форма `@trace(label="X")`), возвращаем `partial`, который при следующем вызове получит функцию. Пока достаточно знать, что такая универсальность возможна; в большинстве кода выбирают одну форму и не усложняют.

## Когда поведение удивит

Аргумент фабрики захватывается один раз - в момент `@repeat(3)`, то есть при определении функции. Если позже изменить переменную, из которой брали аргумент, декоратор этого не заметит: он уже зафиксировал значение в своей ячейке.

```python
factor = 2

def multiply_by(k):
    def decorator(func):
        def wrapper(*args, **kwargs):
            return func(*args, **kwargs) * k
        return wrapper
    return decorator

@multiply_by(factor)
def value():
    return 10

factor = 100
print(value())
```

Печатается `20`, а не `1000`. В момент `@multiply_by(factor)` значение `factor` было `2`, и именно оно ушло в `k`. Дальнейшее изменение `factor` на `multiply_by` уже не влияет: декоратор применился один раз и захватил `k = 2`.

## Типичные ошибки

- **Применить фабрику без скобок.** `@repeat` вместо `@repeat(3)` передаст в фабрику функцию вместо аргумента; обычно это падает с `TypeError` глубже по коду. Фабрику зовут со скобками всегда.
- **Перепутать уровни.** Внешняя функция принимает аргумент декоратора, средняя - функцию, внутренняя - аргументы вызова. Возврат не с того уровня (`return wrapper` вместо `return decorator` снаружи) ломает разворачивание.
- **Забыть `functools.wraps` на самой внутренней обёртке.** Уровней стало больше, но обёртка по-прежнему одна - и метаданные теряет тоже она. `@functools.wraps(func)` ставят над `wrapper`.

<Detail summary="Почему именно три уровня, а не два или четыре">

Число уровней равно числу «порций» аргументов, которые декоратор должен принять по отдельности, плюс один для самого вызова. Обычный декоратор: одна порция (функция) плюс вызов - два уровня (`decorator` и `wrapper`). Декоратор с аргументами: две порции (аргументы декоратора, потом функция) плюс вызов - три уровня. Если бы понадобилось два независимых этапа настройки, появился бы четвёртый уровень, но на практике такое почти не встречается: лишнюю настройку передают одним вызовом фабрики через несколько параметров.

</Detail>
