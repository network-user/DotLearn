The `@` sign above a function looks like an incantation, but a clear mechanic sits behind it: a decorator is a function that takes another function and returns a new one with added behavior. Logging, caching, permission checks, timing - all of it wraps around your code without touching it. To make decorators stop being magic, you have to assemble them from building blocks: functions as objects, closures, wrappers. This topic takes exactly that path - from the basics to confident use.

**What you'll be able to do.** Treat functions as objects; understand closures and variable capture; write your first decorator and grasp what it actually replaces; preserve metadata with `functools.wraps`; build decorators that take arguments; stack several decorators; decorate classes and use classes as decorators; and apply decorators to real tasks.

**The route through this topic.** Eight sections with runnable Python examples right in the browser:

1. **Functions as objects** - a function can be passed and returned.
2. **Closures** - how a function remembers its environment.
3. **Your first decorator** - wrapping a function by hand and with `@`.
4. **functools.wraps** - keeping the name and docstring.
5. **Decorators with arguments** - one more layer of wrapping.
6. **Stacking decorators** - the order of application matters.
7. **Class decorators** - and classes acting as decorators.
8. **Decorators in practice** - cache, timing, checks.

**What you need first.** Basic comfort with Python: functions, arguments, return values, and the essentials of dictionaries. Everything decorator-specific we cover from scratch; the code runs on the page.
