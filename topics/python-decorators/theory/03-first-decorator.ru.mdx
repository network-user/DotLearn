Представьте, что у Вас десяток функций, и к каждой надо добавить одно и то же: записать в лог, что её вызвали, замерить время, проверить права. Можно вписать эти строчки в начало каждой функции - и обречь себя на копипаст, который рассыплется при первой правке. Декоратор решает задачу иначе: Вы пишете обёртку один раз и навешиваете её на любую функцию одной строкой `@`. Сейчас все три кирпича из прошлых разделов - функция-объект, передача функции, замыкание - складываются в этот приём.

К концу раздела Вы напишете свой первый декоратор вручную, поймёте, что `@log` - это всего лишь сахар для `func = log(func)`, и научитесь прокидывать через обёртку любые аргументы и возвращаемое значение с помощью `*args` и `**kwargs`.

## Декоратор - это функция, которая заворачивает функцию

Декоратор принимает функцию и возвращает новую функцию, которая обычно вызывает исходную, добавив что-то до или после. «Что-то до» и «что-то после» живут в обёртке (wrapper), а сама исходная функция захватывается замыканием. Посмотрите, как обёртка оборачивает вызов: до тела отрабатывает один код, потом управление уходит внутрь, и на обратном пути отрабатывает второй.

<DecoratorWrap decoratorName="log_call" funcName="add" before="print('вызов')" after="print('готово')" />

Запишем это кодом. Декоратор `log_call` берёт функцию `func`, строит вокруг неё `wrapper` и возвращает именно `wrapper`, а не результат вызова.

```python
def log_call(func):
    def wrapper():
        print("вызов")
        result = func()
        print("готово")
        return result
    return wrapper
```

Здесь `func` - свободная переменная, захваченная замыканием: `wrapper` будет помнить, какую функцию ему завернули, даже после того, как `log_call` завершится. Внутри `wrapper` мы сами решаем, когда позвать `func()` и что сделать вокруг.

## Навешиваем обёртку вручную

Декоратор - обычная функция, поэтому применить его можно без всякого синтаксиса `@`: вызвать и переприсвоить имя.

```python
def hello():
    return "привет"

hello = log_call(hello)
print(hello())
```

Печатается:

```
вызов
готово
привет
```

Разберём по шагам. `log_call(hello)` вернул `wrapper`, и мы переприсвоили `hello` на этот `wrapper`. Теперь `hello` указывает на обёртку. Вызов `hello()` запускает `wrapper`: тот печатает `вызов`, зовёт исходную функцию (она захвачена в `func`), печатает `готово` и возвращает её результат. Снаружи кажется, что `hello` просто «оброс» логированием.

## Синтаксис @ - это тот же вызов, только короче

Строчка `hello = log_call(hello)` встречается так часто, что для неё придумали сахар. Записать `@log_call` над определением функции - ровно то же самое.

```python
@log_call
def hello():
    return "привет"
```

Эти две формы эквивалентны до последнего байта: Python видит `@log_call`, создаёт функцию `hello`, тут же передаёт её в `log_call` и переприсваивает имя `hello` результату. Декоратор применяется один раз - в момент определения функции, а не при каждом вызове.

## Потрогайте руками

Запустите код: декоратор `announce` объявляет вызов и возвращает результат. Сейчас обёртка работает только с функцией без аргументов. Попробуйте добавить второй декорированный пример - например, функцию, которая что-то возвращает, - и проследите, что `return func()` действительно прокидывает результат наружу.

Вывод показывает порядок: сначала `-> до вызова`, потом тело `работаю`, потом `<- после вызова`, и только затем напечатанный результат `42`. Обёртка управляет тем, что происходит вокруг настоящего вызова.

## Универсальная обёртка: *args и **kwargs

Обёртка из примеров принимает ноль аргументов, поэтому декорировать ею функцию с параметрами нельзя - будет `TypeError`. Чтобы декоратор подходил к любой функции, обёртка должна принимать любые аргументы и прокидывать их в `func` нетронутыми. Для этого служат `*args` (любые позиционные) и `**kwargs` (любые именованные).

```python
def log_call(func):
    def wrapper(*args, **kwargs):
        print("вызов", func.__name__)
        result = func(*args, **kwargs)
        return result
    return wrapper

@log_call
def add(a, b):
    return a + b

print(add(2, 3))
```

Печатается `вызов add`, затем `5`. В сигнатуре `wrapper(*args, **kwargs)` звёздочки собирают аргументы в кортеж и словарь; в вызове `func(*args, **kwargs)` те же звёздочки распаковывают их обратно. Так одна обёртка обслуживает и `add(2, 3)`, и функцию с именованными параметрами, и функцию без аргументов.

<Callout type="tip">Сигнатура `wrapper(*args, **kwargs)` и вызов `func(*args, **kwargs)` - стандартная пара для декоратора, который должен подходить к любой функции. Не возвращать `func(...)` из обёртки - значит молча терять результат: `add(2, 3)` вернёт `None`.</Callout>

## Когда поведение удивит

Самая обидная ошибка - забыть `return` в обёртке. Декоратор продолжит «работать», но проглотит результат.

```python
def silent(func):
    def wrapper(*args, **kwargs):
        func(*args, **kwargs)
    return wrapper

@silent
def square(x):
    return x * x

print(square(5))
```

Печатается `None`, а не `25`. Обёртка вызвала `square`, та честно вернула `25`, но `wrapper` не написал `return`, поэтому наружу ушло `None` - значение по умолчанию для функции без `return`. Декоратор обязан вернуть то, что вернула обёрнутая функция, если только он намеренно не меняет результат.

## Типичные ошибки

- **Обёртка без `*args, **kwargs`.** `def wrapper():` подходит только к функциям без параметров. Декорировав `add(a, b)`, получите `TypeError: wrapper() takes 0 positional arguments`. Используйте `wrapper(*args, **kwargs)`.
- **Забыт `return` в обёртке.** Без `return func(*args, **kwargs)` декоратор проглатывает результат, и вызов возвращает `None`.
- **Возврат `wrapper()` вместо `wrapper`.** Декоратор должен вернуть саму обёртку (`return wrapper`), а не вызвать её. `return wrapper()` выполнит обёртку прямо при декорировании и подставит вместо имени её результат.

<Detail summary="Что значит @, если присмотреться к байт-коду">

Синтаксис `@decorator` появился в Python 2.4 именно как сокращение для повторного присваивания. Интерпретатор при определении функции выполняет: построить объект-функцию, вызвать на нём декоратор, привязать имя к результату. Поэтому декоратор отрабатывает один раз - на этапе импорта модуля, когда выполняется `def`, а не при каждом вызове. Если декоратор печатает что-то на верхнем уровне (вне `wrapper`), это сообщение появится при загрузке модуля, ещё до первого вызова функции. Несколько строк `@` над одним `def` применяются снизу вверх - об этом будет отдельный раздел про стек декораторов.

</Detail>
