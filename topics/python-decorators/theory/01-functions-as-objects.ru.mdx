# Функция в Python - это значение, которое можно положить в переменную

Вы привыкли, что функцию определяют и потом вызывают. Но в Python `def greet():` делает кое-что большее: создаёт объект и привязывает к нему имя. Этот объект ничем принципиально не отличается от числа или строки - его можно присвоить другому имени, положить в список, передать в другую функцию аргументом и вернуть из неё как результат. Понять декораторы без этого факта невозможно, потому что декоратор - это просто функция, которая принимает функцию и возвращает функцию.

К концу раздела Вы будете видеть за именем функции живой объект: понимать, чем `greet` отличается от `greet()`, передавать функции в `map` и `sorted`, собирать их в словарь-диспетчер вместо лестницы `if/elif` и возвращать одну функцию из другой.

## Имя и объект - это две разные вещи

Когда интерпретатор доходит до `def`, он строит объект-функцию и кладёт его в память, а имя после `def` делает указателем на этот объект. Точно так же `x = 5` кладёт в память число и направляет на него имя `x`. Разница только в типе объекта.

Отсюда первый практический вывод: `greet` и `greet()` - это не одно и то же. `greet` - сам объект-функция, его можно передать дальше. `greet()` - команда «выполни этот объект и дай результат». Перепутать их - источник доброй половины ошибок с декораторами.

```python
def greet():
    return "привет"

f = greet
print(f())
print(greet is f)
```

Печатается `привет` и `True`. Присваивание `f = greet` не вызвало функцию и не скопировало её - оно повесило второй ярлык на тот же объект, поэтому `f()` делает ровно то же, что `greet()`, а `greet is f` подтверждает: объект один.

## У функции есть атрибуты

Раз функция - объект, у неё есть собственные данные. Самые полезные - `__name__` (имя, под которым её определили) и `__doc__` (строка документации). Эти два атрибута ещё аукнутся в разделе про `functools.wraps`: неаккуратный декоратор их затирает.

```python
def area(width, height):
    "Площадь прямоугольника."
    return width * height

print(area.__name__)
print(area.__doc__)
```

Печатается `area` и `Площадь прямоугольника.`. Имя хранится на объекте, а не «снаружи» - поэтому даже после `f = area` атрибут `f.__name__` остаётся `area`: ярлык сменился, объект - нет.

## Функцию можно передать аргументом

Если функция - значение, её можно отдать другой функции на вход. Встроенные `map`, `filter`, `sorted` именно этим и живут: им передают функцию-правило, а они применяют его к данным.

```python
def shout(text):
    return text.upper() + "!"

words = ["раз", "два", "три"]
print([shout(w) for w in words])
print(list(map(shout, words)))
```

Обе строки печатают `['РАЗ!', 'ДВА!', 'ТРИ!']`. В `map(shout, words)` мы передаём именно `shout` без скобок - объект-функцию, а не результат её вызова. `map` потом сам вызовет её для каждого слова.

Особенно наглядно это с `sorted` и параметром `key`: туда уходит функция, по которой считается «вес» каждого элемента.

```python
people = [("Аня", 30), ("Боб", 17), ("Вера", 25)]
by_age = sorted(people, key=lambda p: p[1])
print(by_age)
```

Печатается `[('Боб', 17), ('Вера', 25), ('Аня', 30)]`. `lambda p: p[1]` - это безымянная функция-объект, переданная в `sorted` как правило сортировки.

## Потрогайте руками

Ниже функции `add` и `mul` лежат в словаре под строковыми ключами - получился диспетчер, который заменяет цепочку `if op == "+"`. Запустите код: выведется `7`. Теперь попробуйте добавить третью операцию `"-"` со своей функцией и вызвать `dispatch("-", 10, 4)` - убедитесь, что выбор операции свёлся к обычному поиску по словарю, без единого `if`.

## Функцию можно вернуть из функции

Последний кирпич перед декораторами: функция может не только принять другую функцию, но и создать новую и вернуть её. Внутреннюю функцию определяют через `def` прямо в теле внешней и отдают её именем, без скобок.

```python
def make_greeter():
    def greeter():
        return "привет"
    return greeter

g = make_greeter()
print(g())
```

Печатается `привет`. `make_greeter()` вернул объект `greeter`, мы положили его в `g` и затем вызвали через `g()`. Обратите внимание на `return greeter` без скобок: возвращается сама функция, а не результат её вызова. Как такая возвращённая функция запоминает переменные из внешней - тема следующего раздела про замыкания.

## Когда поведение удивит

Самая частая неожиданность - забытые или лишние скобки. Они не вызывают ошибку сразу, а тихо подменяют смысл.

```python
def greet():
    return "привет"

handlers = [greet()]
print(handlers)
```

Печатается `['привет']`, хотя автор, скорее всего, хотел список функций `[greet]`. Скобки выполнили `greet` прямо на месте, и в список попала строка-результат, а не функция. Если потом написать `handlers[0]()`, будет `TypeError: 'str' object is not callable` - строку нельзя вызвать.

<Callout type="warning">`func` - это объект-функция, который можно передать дальше. `func()` - команда выполнить его прямо сейчас. В коде с функциями высшего порядка всегда спрашивайте себя: я передаю функцию или её результат?</Callout>

## Типичные ошибки

- **Лишние скобки при передаче.** `map(shout(), words)` сначала попытается вызвать `shout` без аргументов и упадёт. Передавать нужно `shout` - объект, а не `shout()`.
- **Проверка через `callable` забыта.** Если в словарь-диспетчер по ошибке положить не функцию, а её результат, `operations[op](a, b)` упадёт с `not callable`. Класть в такие структуры нужно сами функции.
- **Возврат вызова вместо функции.** `return greeter()` во внешней функции вернёт строку, а не функцию; имя без скобок (`return greeter`) возвращает объект, готовый к вызову позже.

<Detail summary="Что значит «функции - объекты первого класса»">

Говорят, что в Python функции - это «граждане первого класса» (first-class citizens). Это означает ровно тот набор прав, который мы перебрали: объект можно присвоить имени, передать аргументом, вернуть из функции и хранить в структурах данных. Не во всех языках так: например, в старом C функцию нельзя создать во время выполнения и вернуть как обычное значение. В Python функция, лямбда, метод и даже класс - всё это вызываемые объекты, и `callable(x)` проверяет именно это право быть вызванным.

```python
print(callable(print), callable(42))
```

Печатается `True False`: `print` вызвать можно, число - нет.

</Detail>
