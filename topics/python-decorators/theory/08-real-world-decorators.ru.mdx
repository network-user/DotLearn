Все механизмы собраны: функция-объект, замыкание, синтаксис `@`, `wraps`, аргументы, стек, классы. Пора применить их к задачам, которые встречаются в каждом втором проекте. Четыре рецепта - кэширование результатов, повтор при сбое, замер времени и автоматическая регистрация - покрывают добрую часть того, ради чего декораторы вообще придумали. В этом разделе мы соберём их по кирпичикам, которые Вы уже знаете.

К концу раздела у Вас будет рабочий набор: `memoize`, `retry`, `timed` и `register`, - и понимание, какие из них уже есть в стандартной библиотеке, чтобы не изобретать велосипед.

## memoize: не считать дважды одно и то же

Если функция чистая (один и тот же вход даёт один и тот же выход) и дорогая, повторно её считать незачем. `memoize` запоминает результат по аргументам в словаре и при повторе отдаёт готовое.

```python
import functools

def memoize(func):
    cache = {}
    @functools.wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    return wrapper
```

Словарь `cache` живёт в замыкании - один на функцию, переживает вызовы. Ключ - кортеж `args` (кортежи хешируемы, поэтому годятся в ключи). Первый вызов с данными аргументами считает и кладёт в кэш, последующие берут из кэша. Проверим на наивном Фибоначчи, который без кэша вызывает себя экспоненциально много раз:

```python
@memoize
def fib(n):
    return n if n < 2 else fib(n - 1) + fib(n - 2)

print(fib(30))
```

Печатается `832040` - и почти мгновенно, потому что каждый `fib(k)` считается один раз.

<Callout type="tip">В реальном коде вместо самописного `memoize` берут `functools.lru_cache` (или `functools.cache` в Python 3.9+): тот же приём, но с ограничением размера, потокобезопасностью и статистикой. Самописный `memoize` полезен, чтобы понять, что внутри.</Callout>

## retry: повторить при сбое

Сетевые вызовы и обращения к диску иногда падают по случайной причине, и второй попытки достаточно. `retry(times)` повторяет вызов до `times` раз, пробрасывая исключение только если все попытки исчерпаны.

```python
import functools

def retry(times):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            last_error = None
            for _ in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as error:
                    last_error = error
            raise last_error
        return wrapper
    return decorator
```

Это декоратор с аргументом, поэтому три уровня. В цикле на `times` попыток `wrapper` пытается вернуть результат; при исключении запоминает его и идёт на следующий круг. Если успеха не случилось, после цикла он перевыбрасывает последнюю ошибку - так вызывающий код узнает о провале, а не получит молчаливый `None`.

## Потрогайте руками

Запустите код: `flaky` падает на первых двух вызовах и срабатывает на третьем (счётчик в замыкании имитирует нестабильность). Декоратор `retry(3)` даёт ему ровно три попытки. Поменяйте `retry(3)` на `retry(2)` и убедитесь, что двух попыток не хватает - наружу прорвётся исключение.

## timed: сколько это заняло

Декоратор-профайлер замеряет время вызова. Он ничего не меняет в результате - только печатает (или собирает) длительность. Здесь важно использовать `time.perf_counter` - монотонные часы для измерения интервалов, а не `time.time`, который может прыгнуть при синхронизации системного времени.

```python
import functools, time

def timed(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"{func.__name__}: {elapsed:.6f} с")
        return result
    return wrapper
```

`start` берётся до вызова, разница считается после - классическая обёртка «до/после» из первого раздела про декораторы, только между ними замер. `func.__name__` здесь читаемо именно потому, что стоит `functools.wraps`: без него во всех замерах печаталось бы `wrapper`.

## register: собрать обработчики в реестр

Последний рецепт - регистрация. Вместо ручного списка обработчиков декоратор сам добавляет функцию в реестр в момент определения. Это тот же приём, что с классами в прошлом разделе, но для функций - так строят маршрутизаторы команд и веб-роуты.

```python
COMMANDS = {}

def command(name):
    def decorator(func):
        COMMANDS[name] = func
        return func
    return decorator

@command("hello")
def say_hello():
    return "привет"

@command("bye")
def say_bye():
    return "пока"

print(COMMANDS["hello"]())
```

Печатается `привет`. `command("hello")` - фабрика: она захватывает имя и возвращает декоратор, который кладёт функцию в словарь под этим именем и возвращает функцию нетронутой. Теперь добавить новую команду - значит просто написать `@command("...")`, и она сама окажется в реестре.

## Когда поведение удивит

Самописный `memoize` с ключом `args` не умеет кэшировать вызовы с именованными аргументами и ломается на нехешируемых аргументах (список, словарь). Это не недосмотр примера, а настоящая граница приёма.

```python
import functools

def memoize(func):
    cache = {}
    @functools.wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    return wrapper

@memoize
def total(items):
    return sum(items)

print(total([1, 2, 3]))
```

Падает с `TypeError: unhashable type: 'list'`: ключом стал кортеж `([1, 2, 3],)`, внутри которого список, а список нельзя хешировать, поэтому `args not in cache` не выполнить. Поэтому `lru_cache` тоже требует хешируемых аргументов - кэширование по своей природе нуждается в сравнимом, неизменяемом ключе.

## Типичные ошибки

- **`retry`, проглатывающий ошибку.** Если после исчерпания попыток не перевыбросить исключение, вызов вернёт `None`, и сбой останется незамеченным. Сохраняйте последнюю ошибку и поднимайте её.
- **`timed` на `time.time`.** Для измерения интервалов берут `time.perf_counter`; `time.time` привязан к системным часам и может скакнуть. Разница в точности и в надёжности.
- **`memoize` без учёта хешируемости.** Кэш по `args` падает на списках и словарях и не различает позиционные и именованные аргументы. Для продакшена берите `functools.lru_cache` и помните о его ограничениях.

<Detail summary="Что уже готово в стандартной библиотеке">

Перед тем как писать декоратор, загляните в `functools`. Там уже есть `lru_cache`/`cache` (мемоизация с лимитом и статистикой через `cache_info()`), `cached_property` (ленивое кэшируемое свойство), `total_ordering` (достроить операторы сравнения), `singledispatch` (диспетчеризация по типу первого аргумента) и `partial` (частичное применение). Для повторов с экспоненциальной задержкой и джиттером есть зрелые сторонние библиотеки вроде `tenacity`. Самописные версии хороши для обучения и для случаев, когда нужен ровно один маленький штрих; для остального переиспользуйте проверенное.
