Сеть выдала тензор чисел. Среди них где-то спрятаны рамки, классы и уверенности - но в «сыром» виде это не координаты, а параметры поправок: `tx`, `ty`, `tw`, `th`, логит objectness и логиты классов. Чтобы превратить их в рамку на изображении, нужно декодирование - небольшой, но строгий набор формул. Перепутать здесь сложение с умножением или забыть сигмоиду - значит получить рамки, уехавшие за край кадра.

К концу раздела Вы будете понимать, почему центр декодируется через сигмоиду и сложение с индексом ячейки, а размер - через экспоненту и умножение на якорь, как objectness и вероятности классов собираются в одну уверенность, и почему именно такие функции выбраны.

## Сырой выход - это поправки, а не координаты

На каждую пару «ячейка плюс якорь» сеть выдаёт вектор: четыре числа геометрии `(tx, ty, tw, th)`, одно число objectness и по числу на каждый класс. Это не готовая рамка. `tx, ty` - поправка центра внутри ячейки, `tw, th` - поправка размера относительно якоря. Декодирование переводит поправки в координаты по фиксированным формулам.

Центр рамки в координатах сетки:

```python
import math

def sigmoid(t):
    return 1 / (1 + math.exp(-t))

def decode_center(t, cell_index):
    return cell_index + sigmoid(t)

print(decode_center(0, 2), decode_center(2, 0))
```

Печатается `2.5 0.8807970779778823`: при `tx = 0` сигмоида даёт ровно `0.5`, и центр садится в середину ячейки `2`, то есть в `2.5`. При `tx = 2` сигмоида - около `0.88`, центр сдвигается к правому краю ячейки `0`.

## Почему сигмоида для центра

Сигмоида сжимает любое число в интервал `(0, 1)`. Это ровно то, что нужно центру: его смещение `bx = cell_x + sigmoid(tx)` гарантированно остаётся **внутри своей ячейки**. Без сигмоиды сеть могла бы предсказать центр в соседней ячейке, и тогда ответственность за объект «перетекла» бы туда, где её не ждали. Сигмоида запирает центр в границах ответственной ячейки - предсказание остаётся стабильным.

Подвигайте веса на схеме: что бы ни было на входе, выход сигмоиды не выходит за `(0, 1)`. Именно это свойство переносится на декодирование центра - сумма «индекс ячейки плюс сигмоида» не может выскочить за пределы одной клетки.

## Почему экспонента для размера

Размер устроен иначе: `bw = anchor_w * exp(tw)`. Здесь поправка не сдвигает, а **масштабирует** якорь. Экспонента выбрана по двум причинам. Во-первых, она всегда положительна, поэтому ширина и высота не могут стать отрицательными при любом `tw`. Во-вторых, она работает мультипликативно: `tw = 0` даёт множитель `1` (размер якоря без изменений), `tw > 0` увеличивает, `tw < 0` уменьшает, причём симметрично в логарифмическом масштабе.

```python
import math

def decode_size(t, anchor):
    return anchor * math.exp(t)

print(decode_size(0, 4), decode_size(1, 2))
```

Печатается `4.0 5.43656...`: при `tw = 0` размер равен якорю `4`; при `tw = 1` якорь `2` растягивается в `e` раз, примерно до `5.44`. Симметрия логарифма означает, что `tw = 1` и `tw = -1` масштабируют в `e` и в `1/e` - удвоение и деление, а не плюс-минус константа.

## Objectness и вероятности классов

Геометрия - половина дела. Вторая половина - уверенность. Сеть выдаёт логит objectness (есть ли в этом слоте объект вообще) и логиты классов. Логит objectness прогоняют через сигмоиду и получают вероятность объекта `p_obj` из `(0, 1)`. Вероятности классов получают softmax (или независимыми сигмоидами в новых версиях). Итоговая уверенность по конкретному классу - **произведение**: `p_obj * p_class`.

```python
objectness = 0.8
class_probs = [0.1, 0.7, 0.2]
best_class = class_probs.index(max(class_probs))
confidence = objectness * max(class_probs)
print(best_class, round(confidence, 2))
```

Печатается `1 0.56`: вероятнее всего класс с индексом `1` (его `p_class = 0.7`), и итоговая уверенность `0.8 * 0.7 = 0.56`. Произведение, а не один из множителей: слот с уверенным классом, но низким objectness, всё равно получает низкую итоговую уверенность - и наоборот.

<Callout type="info">Уверенность - это произведение двух вероятностей, поэтому она не больше любой из них. Высокий `p_class` сам по себе ничего не значит, если сеть не уверена, что в слоте вообще есть объект.</Callout>

## Потрогайте руками

Соберите полную рамку из поправок. Запустите код: при всех `t = 0` центр садится в середину ячейки, а размер совпадает с якорем. Поменяйте `tx` на `2` и посмотрите, как центр сдвигается вправо, но не выходит за ячейку. Затем поставьте `tw = 1` - ширина вырастет примерно в `e` раз. Убедитесь, что отрицательный `tw` уменьшает рамку, а не уводит её в минус.

    return 1 / (1 + math.exp(-t))

def decode_box(tx, ty, tw, th, gx, gy, aw, ah):
    bx = gx + sigmoid(tx)
    by = gy + sigmoid(ty)
    bw = aw * math.exp(tw)
    bh = ah * math.exp(th)
    return [bx, by, bw, bh]

print(decode_box(0, 0, 0, 0, 2, 3, 4, 2))`}

## Где это удивит

Координаты после декодирования - в единицах сетки, а не в пикселях. Центр `2.5` означает «два с половиной шага сетки», а не «два с половиной пикселя». Чтобы получить пиксели, результат умножают на размер шага сетки (`размер изображения / S`). Пропустить этот пересчёт - значит получить рамку в углу кадра: все координаты окажутся в диапазоне от нуля до `S` вместо нуля до ширины изображения.

Второй сюрприз - забытая сигмоида у objectness. Сырой логит objectness может быть и отрицательным, и большим положительным; без сигмоиды он не вероятность. Умножить сырой логит на `p_class` - значит получить «уверенность» больше единицы или отрицательную.

## Типичные ошибки

- **Сложить размер с якорем вместо умножения.** Размер мультипликативен: `bw = anchor_w * exp(tw)`, а не `anchor_w + tw`. Сложение ломает масштабную природу поправки и легко даёт отрицательную ширину.
- **Забыть сигмоиду для центра.** Без сжатия в `(0, 1)` центр может выскочить за ячейку, и за объект «случайно» ответит соседняя клетка.
- **Перепутать единицы.** Декодированные координаты - в шагах сетки; в пиксели их переводит умножение на размер шага. Не смешивайте две системы.
- **Умножить логиты, а не вероятности.** Итоговая уверенность - произведение вероятностей `p_obj * p_class`, и обе должны быть пропущены через сигмоиду/softmax до перемножения.

<Detail summary="Откуда взялись именно эти формулы">

В YOLOv2 авторы заменили прямое предсказание координат на предсказание поправок к якорям с сигмоидой для центра - и это заметно стабилизировало обучение на ранних шагах. Причина в том, что сигмоида ограничивает центр своей ячейкой, поэтому в начале обучения, когда веса случайны, предсказания не «разлетаются» по всему кадру. Экспонента для размера обеспечивает положительность и мультипликативную симметрию вокруг якоря. Эти два решения - сигмоида для смещения и экспонента для масштаба - стали стандартом и сохранились в большинстве последующих версий, даже когда менялось почти всё остальное.

</Detail>
