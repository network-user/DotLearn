После декодирования у Вас не аккуратный список объектов, а ворох рамок: один автомобиль породил шесть почти одинаковых прямоугольников от соседних ячеек, каждый со своей уверенностью. Показать их все - значит обвести одну машину шесть раз. Нужен фильтр, который из плотного облака дубликатов оставит по одной рамке на объект, причём самую уверенную. Этот фильтр - Non-Maximum Suppression, и без него выход любого детектора нечитаем.

К концу раздела Вы будете понимать алгоритм NMS по шагам, реализуете его через IoU и порог, разберётесь, что именно подавляется и относительно чего, и увидите, как выбор порога меняет результат - от слипшихся объектов до лишних дубликатов.

## Идея: жадный отбор по уверенности

NMS - жадный алгоритм. Он работает с одним классом за раз и повторяет три действия, пока есть кандидаты:

<Steps>
  <li>Взять рамку с наибольшей уверенностью среди оставшихся и оставить её (она «победитель»).</li>
  <li>Удалить из кандидатов все рамки, чей IoU с победителем больше порога, - это дубликаты того же объекта.</li>
  <li>Повторить с тем, что осталось.</li>
</Steps>

Название буквально описывает суть: «подавление не-максимумов». Из группы пересекающихся рамок максимум по уверенности выживает, остальные подавляются. Порог IoU решает, что считать «той же рамкой»: высокий порог подавляет только сильно совпадающие, низкий - даже слабо задевающие.

Прошагайте визуализацию. На первом шаге побеждает рамка с уверенностью `0.92`; вторая, сильно с ней пересекающаяся, подавляется. Рамка `0.88` справа не пересекается с победителем, поэтому переживает первый шаг и сама становится победителем на следующем. Итог - две рамки на два объекта вместо трёх кандидатов.

## Подавление - всегда относительно текущего победителя

Тонкость, на которой спотыкаются: рамки подавляются по IoU **с текущим победителем**, а не друг с другом по цепочке. Если рамка `B` пережила победителя `A` (низкий IoU с `A`), она не «наследует» подавление: на следующем шаге она сама сравнивается со своими кандидатами. Поэтому две слабо пересекающиеся рамки могут обе выжить, даже если каждая по отдельности пересекается с третьей.

```python
def iou(a, b):
    ax1, ay1, ax2, ay2 = a
    bx1, by1, bx2, by2 = b
    iw = max(0, min(ax2, bx2) - max(ax1, bx1))
    ih = max(0, min(ay2, by2) - max(ay1, by1))
    inter = iw * ih
    union = (ax2 - ax1) * (ay2 - ay1) + (bx2 - bx1) * (by2 - by1) - inter
    return inter / union if union > 0 else 0.0

print(round(iou((0, 0, 4, 4), (1, 1, 5, 5)), 3))
```

Печатается `0.391`: две рамки `4x4`, сдвинутые по диагонали на единицу, перекрываются на `3x3 = 9`, объединение `16 + 16 - 9 = 23`, и `9/23 ≈ 0.391`. При пороге `0.3` вторая рамка подавится, при пороге `0.5` - переживёт. Один и тот же набор рамок при разном пороге даёт разный ответ.

## Реализация

Соберём алгоритм. Удобно работать с индексами: отсортировать их по убыванию уверенности, забирать первый, отбрасывать пересекающиеся, повторять.

```python
def nms(boxes, scores, threshold):
    order = sorted(range(len(boxes)), key=lambda i: scores[i], reverse=True)
    keep = []
    while order:
        best = order.pop(0)
        keep.append(best)
        order = [i for i in order if iou(boxes[best], boxes[i]) <= threshold]
    return keep
```

Сортировка ставит самые уверенные вперёд. На каждом обороте `pop(0)` забирает текущего победителя в `keep`, а список `order` пересобирается, оставляя только тех, кто **не** слишком пересёкся с победителем (`IoU <= threshold`). Когда кандидаты кончились, в `keep` лежат индексы выживших в порядке убывания уверенности.

Поднимите порог до `0.5` и запустите снова: теперь IoU внутри пар (около `0.39`) ниже порога, подавления не происходит, и выживают все четыре - `[0, 1, 2, 3]`. Так видно, что порог - это рычаг «насколько агрессивно склеивать дубликаты».

## Где это удивит

Слишком низкий порог склеивает разные объекты. Два человека, стоящие вплотную, дают рамки с заметным IoU; при пороге `0.2` NMS решит, что это дубликаты, и подавит одного из них. Объект просто исчезнет из выдачи - не потому что детектор его не нашёл, а потому что постобработка его «съела». Это типичная причина пропавших объектов в плотных сценах.

Слишком высокий порог оставляет дубликаты: при пороге `0.9` почти ничего не подавляется, и одна машина снова обведена несколько раз. Стандартный компромисс - порог в районе `0.4..0.5`, но он зависит от того, насколько тесно стоят объекты в Вашей задаче.

## Типичные ошибки

- **Подавлять по IoU между всеми парами сразу.** NMS подавляет только относительно текущего победителя на каждом шаге, а не все пересекающиеся пары глобально. Глобальное подавление выкинуло бы рамки, которые должны выжить.
- **Смешивать классы.** NMS обычно применяют отдельно для каждого класса: рамка «кошка» не должна подавлять рамку «собака», даже если они сильно пересеклись. Иначе перекрытие объектов разных классов теряет один из них.
- **Сравнивать score с порогом IoU.** Порог в NMS - это порог по IoU между рамками, а не по уверенности. Отсев по уверенности (`score < conf_threshold`) - отдельный шаг, который обычно делают до NMS.
- **Перепутать знак сравнения.** Подавляются рамки с `IoU > threshold`; выживают с `IoU <= threshold`. Поставить строгое и нестрогое неравенство наоборот - сдвинуть поведение ровно на границе.

<Detail summary="Soft-NMS - не выбрасывать, а понижать">

Жёсткий NMS удаляет дубликат целиком, и это вредит в плотных сценах: законная рамка соседнего объекта, сильно пересёкшаяся с победителем, исчезает. Soft-NMS вместо удаления **понижает** уверенность пересекающейся рамки пропорционально IoU - чем сильнее перекрытие, тем сильнее штраф. Рамка остаётся кандидатом со сниженным score и может выжить, если её уверенность всё ещё высока. На датасетах с тесными объектами это заметно поднимает recall, не меняя самой идеи отбора по убыванию уверенности.
