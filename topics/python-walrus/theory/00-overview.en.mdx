You read a line from a file, immediately check whether it is empty, and if not, do something with it. In plain Python this splits into two lines: first assign the result to a variable, then check that variable in `if` or `while`. That leads to an extra "first read" before the loop, duplicated function calls, and an exit condition hidden in the middle of the body instead of the loop header. The walrus operator `:=` was designed for exactly these spots: it lets you compute a value, store it in a variable, and use it right away, all in one expression.

**What you will learn to do.** Combine computation, assignment, and checking in one expression; rewrite an "infinite" read loop so the exit condition sits in the `while` header; remove repeated calls inside a `comprehension`; and, equally important, spot places where `:=` only obscures code and avoid putting it there.

**Topic route.** Three sections, each with a live sandbox where you can change and run code right in the browser:

1. **Basics of `:=`**: how assignment-as-expression differs from ordinary `=`, what value it returns, and where the syntax requires parentheses.
2. **Loops and comprehensions**: how `while (chunk := read())` pulls reading and checking into one place, and where `:=` inside a comprehension is safe versus where it causes subtle bugs.
3. **Readability**: when `:=` makes code clearer, which anti-patterns to remember, and a simple rule for the disputed "walrus or two lines" case.

**Prerequisites.** Nothing to install: Python runs right in the browser. You only need to be comfortable with ordinary assignment, `if`, and `while`. We will cover everything else about `:=` along the way, from the first expression to a mature "when not to use it" rule.
