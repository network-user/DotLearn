# Присваивание, у которого есть значение

Обычное `=` в Python ничего не возвращает. Вы пишете `n = len(data)`, и эта строка просто связывает имя `n` с числом - сама по себе она не является значением, её нельзя поставить внутрь `if` или передать в функцию. Поэтому, когда нужно одновременно вычислить что-то, запомнить результат в переменную и тут же его проверить, приходится писать две строки: сначала присвоить, потом сравнить. Оператор `:=`, который в сообществе зовут «walrus» из-за сходства с глазами и клыками моржа, объединяет эти два шага в одно выражение.

К концу раздела Вы будете понимать, чем `:=` отличается от `=`, какое значение он возвращает, где синтаксис требует обернуть его в скобки, и почему его придумали именно как выражение, а не как ещё одну форму обычного присваивания.

## Ментальная модель: присваивание, которое возвращает то, что присвоило

Главная идея в одном предложении: `name := expr` вычисляет `expr`, связывает результат с именем `name` и **возвращает этот же результат** как значение всего выражения. Обычное `name = expr` делает первые два шага, но третьего у него нет - результата у строки присваивания не существует, её нельзя использовать как часть большего выражения.

Из-за этого различия `=` и `:=` живут в разных местах программы. `=` - это *оператор-инструкция* (statement), он занимает целую строку. `:=` - это *оператор-выражение* (expression), он встраивается внутрь другого выражения: в условие `if`, в аргумент функции, в `while`.

Посмотрите на нижнюю строку рисунка: `(n := len(data))` не просто записывает длину в `n`, оно ещё и «прокидывает» это число наружу. Поэтому такую запись можно поставить прямо в `if`, и проверка увидит то самое значение, которое только что попало в переменную.

## Первый разбор: одно вычисление вместо двух строк

Классический мотив walrus - «вычислить, запомнить, проверить». Без него получается две строки, где имя `value` повторяется.

```python
value = compute()
if value > 10:
    print(value)
```

С `:=` присваивание переезжает внутрь условия. Переменная `value` после этого остаётся доступной в теле `if` и ниже - walrus не создаёт временную область видимости, он связывает имя в обычной окружающей области.

```python
if (value := compute()) > 10:
    print(value)
```

Обе версии делают ровно одно и то же. Разница в том, что во второй `compute()` и проверка результата стоят рядом, а имя `value` фигурирует один раз слева от `:=`. Это и есть типичная польза: меньше повторов, вычисление и его проверка не разнесены по разным строкам.

<MarginNote label="Откуда «walrus»">Официальное название из PEP 572 - «assignment expression». Прозвище «walrus operator» прижилось потому, что `:=`, если присмотреться, похоже на глаза и клыки моржа.</MarginNote>

## Скобки: где они обязательны

Чаще всего `:=` приходится оборачивать в скобки. Это не стилистика, а требование грамматики: язык специально сделал так, чтобы walrus был визуально заметен и не путался с обычным `=`. В условии выше скобки вокруг `(value := compute())` обязательны - без них Python не разберёт строку.

```python
total = (subtotal := 100) + (tax := 20)
print(subtotal, tax, total)
```

Печатается `100 20 120`. Здесь две переменные `subtotal` и `tax` получают значения прямо в арифметическом выражении, и каждая обёрнута в свои скобки. Без скобок такое выражение не скомпилируется. Запомните грубое правило: если сомневаетесь, нужны ли скобки вокруг `:=`, - ставьте их, лишними они почти никогда не будут.

<Callout type="warning">
`:=` нельзя использовать как самостоятельную инструкцию: строка `x := 5` - синтаксическая ошибка. Для обычного присваивания на отдельной строке есть `=`. Walrus имеет смысл только там, где его возвращаемое значение кому-то нужно.
</Callout>

## Потрогайте оператор руками

Запустите код как есть: walrus присвоит `n` длину строки и тут же вернёт это число в условие. Затем поменяйте текст в `text` на короткий (например, `"hi"`) и запустите снова - условие станет ложным, ветка `else` напечатает другое. Обратите внимание, что `n` доступна и после `if`: walrus связывает имя в обычной области видимости, а не прячет его внутри условия.

## Краевой случай, который вас удивит: := внутри f-string

Walrus можно поставить в самые неожиданные места - например, прямо внутрь f-string. Выражение вычислится, имя свяжется, а в строку подставится результат.

```python
import math

print(f"корень: {(r := math.sqrt(2)):.3f}")
print(r)
```

Печатается `корень: 1.414` и затем `1.4142135623730951`. Переменная `r` пережила f-string и доступна в следующей строке. Технически это работает, но читается тяжело - запоминать переменную внутри форматной строки почти всегда плохая идея. Полезно знать, что так *можно*, чтобы понимать чужой код, и почти никогда не стоит так писать самому.

## Ещё один краевой случай: walrus и приоритет операций

`:=` связывает слабее почти всех операторов. Из-за этого `x := a + b` присвоит `x` результат `a + b`, а не только `a`. Но без скобок такое выражение вне разрешённых позиций всё равно не примут, поэтому на практике вы пишете `(x := a + b)`, и приоритет внутри скобок вопросов не вызывает.

```python
data = [3, 1, 4, 1, 5]
if (biggest := max(data)) > 4:
    print("максимум", biggest, "больше четырёх")
```

Печатается `максимум 5 больше четырёх`. Сначала полностью вычисляется `max(data)`, потом результат попадает в `biggest`, и только затем сравнивается с `4`. Скобки делают порядок однозначным и заодно удовлетворяют грамматику.

## Типичные ошибки

- **Путать `:=` и `=` по смыслу.** `=` - инструкция без значения, целая строка. `:=` - выражение со значением, которое встраивается внутрь другого выражения. Если вам не нужно возвращаемое значение, берите обычное `=`.
- **Забыть скобки.** `if value := compute() > 10:` разберётся не так, как вы думаете, или не разберётся вовсе. Оборачивайте walrus в скобки: `if (value := compute()) > 10:`.
- **Писать `:=` отдельной строкой.** `x := 5` на своей строке - синтаксическая ошибка. Walrus оправдан только когда его значение кому-то передаётся: в условие, в аргумент, в `while`.
- **Ждать новую область видимости.** Walrus не прячет переменную локально - имя остаётся в окружающей области и доступно после блока. Это не баг, но об этом легко забыть.

<Detail summary="Почему walrus вообще добавили: история PEP 572">

Предложение PEP 572 обсуждалось долго и горячо - спор о нём стал одной из причин, по которой Гвидо ван Россум в 2018 году отошёл от роли BDFL. Аргумент «за»: в Python давно не хватало способа присвоить и использовать значение в одном выражении, из-за чего код вокруг `while`, чтения потоков и фильтрующих `comprehensions` получался многословным. Аргумент «против»: новый оператор слишком похож на `=` и провоцирует трудночитаемый код.

Компромисс - именно тот строгий синтаксис, который вы видели выше: walrus нельзя ставить голой инструкцией, во многих позициях он требует скобок, и язык намеренно не даёт писать им то, что естественнее выразить через обычное `=`. Оператор появился в Python 3.8 (2019). Подробности и мотивацию читайте в самом [PEP 572](https://peps.python.org/pep-0572/).

</Detail>
