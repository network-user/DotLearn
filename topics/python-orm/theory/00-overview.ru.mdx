title: "ORM и raw SQL в Python (SQLAlchemy): обзор темы"
estimatedMinutes: 3
---

Вы написали SQL-запрос и получили в Python кортеж вроде `(1, 'Анна', 30)`. Дальше начинается рутина: разобрать его по полям, собрать обратно строку `INSERT`, не забыть про экранирование, закрыть соединение. На паре запросов терпимо, на реальном приложении - сотни строк ручного перевода между объектами Python и строками таблиц. ORM берёт этот перевод на себя: Вы описываете, какой класс соответствует какой таблице, и работаете с обычными объектами, а SQL библиотека генерирует сама. Но магия дорого обходится тому, кто не понимает, что под ней происходит, - поэтому здесь Вы сначала поработаете с базой руками, а только потом доверите перевод ORM.

**Что вы научитесь делать.** Выполнять запросы из Python напрямую и безопасно подставлять данные через параметры; описывать таблицы как классы в стиле SQLAlchemy 2.x; собирать запросы через `select()` и ходить по связям как по атрибутам объекта; видеть, какой именно SQL уходит в базу, и ловить дорогие ошибки вроде N+1.

**Маршрут темы.** Четыре раздела, каждый - с песочницей, где код запускается прямо в браузере:

1. **Raw SQL** - открываете соединение и курсор, выполняете запрос руками и подставляете данные параметрами, а не f-строкой.
2. **Ментальная модель ORM** - разбираете, что и на что отображается, и объявляете первую модель SQLAlchemy 2.x.
3. **Запросы и связи** - собираете выборку через `select()` и превращаете внешние ключи в атрибуты `.user` и `.orders`.
4. **Проблема N+1** - распознаёте, как невинный цикл рождает тысячу запросов, и чините это жадной загрузкой.

**Что нужно заранее.** Устанавливать ничего не надо: Python выполняется прямо в браузере. Пригодится уверенность в базовом SQL из темы «Основы SQL» - `SELECT`, `JOIN`, фильтрация; всё остальное, включая работу с самой SQLAlchemy, разберём по ходу.
