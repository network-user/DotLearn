# Синтаксис встроенного языка: переменные, операторы, условия, циклы

Встроенный язык 1С похож на десяток других процедурных языков, и именно это чаще всего подводит. Разработчик, пришедший из Python или JavaScript, по привычке пишет `==` для сравнения, ставит фигурные скобки вокруг блока и удивляется, почему конфигуратор ругается. Язык 1С русскоязычный, блоки закрываются словами `КонецЕсли` и `КонецЦикла`, а знак `=` работает сразу в двух ролях. Разберём базовый синтаксис так, чтобы эти мелочи перестали отнимать время.

## Оператор один, а смыслов два

В большинстве языков присваивание и сравнение записывают разными знаками: `=` и `==`. Во встроенном языке 1С знак `=` один, и его роль определяется позицией. Слева от него имя переменной, значит это присваивание. Внутри условия это сравнение. Отдельного `==` в языке просто нет, и попытка его написать вызывает синтаксическую ошибку.

Каждый оператор языка завершается точкой с запятой. Объявлять переменную заранее необязательно: первое присваивание её и создаёт. Но если хочется задекларировать переменную явно (например, в начале модуля), для этого есть слово `Перем`.

```bsl
Перем Приветствие;

Приветствие = "Здравствуйте";
Сумма = 100;
Ставка = 0.2;
```

## Условия: лесенка Если / ИначеЕсли / Иначе

Ветвление строится словами `Если`, `Тогда`, `ИначеЕсли`, `Иначе` и обязательным `КонецЕсли`. Никаких фигурных скобок: границы блока задают ключевые слова. Логические операторы тоже русские: `И`, `ИЛИ`, `НЕ`. Сравнение «не равно» записывается как `<>`.

```bsl
Если Сумма >= 100000 Тогда
    Скидка = 0.1;
ИначеЕсли Сумма >= 30000 Тогда
    Скидка = 0.05;
Иначе
    Скидка = 0;
КонецЕсли;
```

Когда нужно просто выбрать одно из двух значений, необязательно разворачивать всю лесенку. Для этого есть тернарный оператор `?(Условие, ЗначениеЕслиИстина, ЗначениеЕслиЛожь)`.

```bsl
Статус = ?(Сумма > 0, "Оплачен", "Ожидает оплаты");
```

Тернарный оператор возвращает значение, поэтому его удобно подставлять прямо в присваивание или в аргумент функции. Злоупотреблять вложенностью не стоит: три вложенных `?(...)` читаются хуже честной лесенки `ИначеЕсли`.

## Циклы: со счётчиком, по коллекции и по условию

Циклов во встроенном языке три вида, и все закрываются словом `КонецЦикла`. Цикл со счётчиком идёт от начального значения `По` конечное включительно.

```bsl
Итог = 0;
Для Счётчик = 1 По 5 Цикл
    Итог = Итог + Счётчик;
КонецЦикла;
```

Обратите внимание: граница `По 5` входит в диапазон, поэтому тело выполнится ровно пять раз, а `Счётчик` пройдёт значения от 1 до 5. Это отличается от `range(5)` в Python, который останавливается на 4. Перенос этой привычки прямо в 1С даёт классическую ошибку на единицу.

Второй вид перебирает элементы коллекции целиком, не заботясь об индексах.

```bsl
Для Каждого Товар Из СписокТоваров Цикл
    Сообщить(Товар);
КонецЦикла;
```

Третий вид работает, пока истинно условие. Его берут, когда число повторений заранее неизвестно.

```bsl
Остаток = 10;
Пока Остаток > 0 Цикл
    Остаток = Остаток - 3;
КонецЦикла;
```

Внутри любого цикла работают `Прервать` (выйти из цикла немедленно) и `Продолжить` (перейти к следующему витку, пропустив остаток тела).

## Что удивляет на границах

Индексация коллекций во встроенном языке начинается с нуля, а цикл со счётчиком по привычке пишут с единицы. Поэтому обход массива по индексу выглядит как `Для Сч = 0 По Массив.Количество() - 1`, и минус единица здесь не опечатка, а необходимость: у массива из пяти элементов индексы от 0 до 4.

<Callout type="warning" title="Регистр в именах не важен, а вот пробел в конце строки важен">
Ключевые слова и имена переменных нечувствительны к регистру: `Если`, `ЕСЛИ` и `если` для платформы одно и то же. А вот значения строк регистр и пробелы сохраняют дословно: `"Оплачен"` и `"оплачен "` это разные строки, и сравнение их через `=` даст ложь.
</Callout>

## Типичные ошибки

**Написать `==` вместо `=`.** Привычка из Си-подобных языков. В 1С сравнение и присваивание это один знак `=`, а `==` вызывает ошибку синтаксиса.

**Забыть закрывающее слово блока.** Каждый `Если` требует `КонецЕсли`, каждый `Цикл` требует `КонецЦикла`, и оба заканчиваются точкой с запятой. Пропуск обнаружится только при синтаксической проверке модуля.

**Перенести `range(N)` буквально.** В 1С граница `По N` включается в диапазон. Тело `Для Сч = 1 По 5` выполняется пять раз, а не четыре.

**Спутать `<>` и `!=`.** Оператора `!=` в языке нет. «Не равно» записывается только как `<>`.

<Detail summary="Почему в 1С нет отдельного оператора присваивания вроде :=">

Некоторые языки (Pascal, Go в короткой форме) разделяют присваивание и сравнение разными знаками именно для того, чтобы случайное `=` в условии не превратилось в тихую запись значения. Во встроенном языке 1С эта неоднозначность снимается грамматикой: конструкция `Если ... Тогда` синтаксически ждёт логическое выражение, а не оператор присваивания, поэтому `=` внутри неё всегда трактуется как сравнение. Платформа не даст случайно «присвоить» значение в условии, потому что присваивание там просто не разрешено грамматикой.

</Detail>
