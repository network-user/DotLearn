# Таблица значений: данные в виде таблицы прямо в памяти

Массив хранит список, соответствие работает как словарь, но данные в 1С часто табличные: строки продаж с товаром, количеством и суммой; список сотрудников с должностью и окладом. Держать это в параллельных массивах неудобно и опасно. Для табличных данных во встроенном языке есть отдельная коллекция: таблица значений. Это полноценная таблица в оперативной памяти: именованные колонки, произвольное число строк, поиск, итоги и группировка. Табличные части документов, кстати, устроены так же, поэтому навык переносится один в один.

## Колонки сначала, строки потом

Таблица создаётся пустой, затем ей описывают колонки, а уже потом добавляют строки. Колонки живут в коллекции `Колонки`, строки добавляет метод `Добавить`, который возвращает новую пустую строку.

```bsl
Товары = Новый ТаблицаЗначений;
Товары.Колонки.Добавить("Наименование");
Товары.Колонки.Добавить("Сумма");

НоваяСтрока = Товары.Добавить();
НоваяСтрока.Наименование = "Хлеб";
НоваяСтрока.Сумма = 50;
```

Метод `Добавить` возвращает ссылку на созданную строку, и её поля заполняют через точку по имени колонки. К значению в строке можно обратиться и по имени `Строка.Сумма`, и по индексу колонки `Строка[1]`.

## Обход и подсчёт

Перебор строк это обычный `Для Каждого`. Число строк даёт `Количество()`.

```bsl
Для Каждого Строка Из Товары Цикл
    Сообщить(Строка.Наименование + ": " + Строка(Строка.Сумма));
КонецЦикла;

Сообщить(Товары.Количество());
```

## Поиск: Найти и НайтиСтроки

Пробегать таблицу циклом ради поиска нужной строки необязательно. Метод `Найти(Значение, "Колонка")` возвращает первую подходящую строку или `Неопределено`. Метод `НайтиСтроки(Отбор)` возвращает массив всех строк, удовлетворяющих отбору, где отбор задаётся структурой «колонка и значение».

## Итоги и свёртка

Две операции превращают таблицу значений в мини-инструмент отчётности. `Итог("Колонка")` складывает все значения числовой колонки. `Свернуть` группирует строки: первым аргументом идут колонки группировки, вторым идут числовые колонки, которые нужно просуммировать внутри групп.

```bsl
Всего = Товары.Итог("Сумма");

Товары.Свернуть("Наименование", "Сумма");
```

После `Свернуть` строки с одинаковым наименованием схлопнутся в одну, а их суммы сложатся. Это тот же принцип, что `GROUP BY` в запросе, только над таблицей в памяти.

<Callout type="tip" title="Табличная часть документа это таблица значений">
Реквизит вроде `Документ.Товары` (табличная часть) ведёт себя как таблица значений: те же `Добавить`, обход `Для Каждого`, `Итог` и `НайтиСтроки`. Освоив таблицу значений, вы уже умеете работать с табличными частями документов.
</Callout>

## Типичные ошибки

**Добавлять строки до колонок.** Пока колонки не описаны, заполнять поля строки нечем. Сначала `Колонки.Добавить`, потом `Добавить` строк.

**Обращаться к несуществующей колонке.** `Строка.Сумаа` с опечаткой не создаст колонку, а вызовет ошибку. Имя поля должно точно совпадать с именем колонки.

**Искать циклом то, что находит НайтиСтроки.** Ручной перебор ради поиска нужной строки многословен и медленнее. `Найти` и `НайтиСтроки` для этого и существуют.

**Забыть, что Свернуть меняет исходную таблицу.** Метод не возвращает новую таблицу, а перестраивает текущую. Если исходные строки ещё нужны, сначала сделайте копию через `Скопировать()`.

<Detail summary="Типизация колонок и сортировка">

Колонке можно задать тип через `Новый ОписаниеТипов("Число")` вторым аргументом `Колонки.Добавить`. Тогда платформа будет контролировать, что в колонку кладут число, а пустое значение колонки будет соответствовать её типу (для числа это ноль). Сортируют таблицу методом `Сортировать("Сумма Убыв")`: имя колонки и направление, несколько колонок перечисляют через запятую. Сортировка, как и свёртка, меняет саму таблицу, а не возвращает копию.

</Detail>
