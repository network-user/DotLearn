# Коллекции: Массив, Структура, Соответствие

Одно значение в переменной это хорошо, но реальные данные ходят пачками: список товаров, набор реквизитов, справочник цен по наименованию. Для этого во встроенном языке есть коллекции, и выбор правильной экономит и код, и нервы. Массив хранит упорядоченный список, структура держит набор именованных полей, соответствие работает как словарь с ключами любого типа. Разберём, чем они отличаются и когда какую брать.

## Массив: упорядоченный список

Массив создаётся через `Новый Массив`, элементы добавляются методом `Добавить`, а обращение идёт по индексу, который начинается с нуля.

```bsl
Список = Новый Массив;
Список.Добавить("Хлеб");
Список.Добавить("Молоко");

Сообщить(Список[0]);
Сообщить(Список.Количество());
```

Здесь `Список[0]` это `"Хлеб"`, а `Количество()` равно `2`. Перебрать массив можно циклом `Для Каждого`, не заботясь об индексах.

```bsl
Для Каждого Товар Из Список Цикл
    Сообщить(Товар);
КонецЦикла;
```

Именно с массивом связана самая частая ошибка на единицу: если обходить по индексу, верхняя граница это `Количество() - 1`, потому что у списка из двух элементов индексы `0` и `1`.

## Структура: набор именованных полей

Когда нужно собрать вместе несколько разнородных значений и обращаться к ним по имени, берут структуру. Её ключи это строковые идентификаторы: буквы, цифры и подчёркивание, начинающиеся с буквы, без пробелов.

```bsl
Контрагент = Новый Структура;
Контрагент.Вставить("Наименование", "ООО Ромашка");
Контрагент.Вставить("ИНН", "7701234567");

Сообщить(Контрагент.Наименование);
```

К полю обращаются двумя способами: через точку `Контрагент.Наименование` или через квадратные скобки `Контрагент["Наименование"]`. Заполнить структуру можно и одной строкой прямо в конструкторе: первым аргументом идёт список ключей через запятую, дальше значения по порядку.

```bsl
Контрагент = Новый Структура("Наименование, ИНН", "ООО Ромашка", "7701234567");
```

## Соответствие: словарь с любыми ключами

Структура удобна, пока набор полей известен заранее. Но если ключи приходят из данных (цена по наименованию товара, остаток по складу, счётчик по ссылке), нужен словарь с произвольными ключами. Это соответствие.

```bsl
Цены = Новый Соответствие;
Цены.Вставить("Хлеб", 50);
Цены.Вставить("Молоко", 80);

Сообщить(Цены["Хлеб"]);
```

Ключом соответствия может быть не только строка, но и число, дата или ссылка на элемент справочника. Метод `Получить(Ключ)` безопаснее прямого доступа по скобкам: если ключа нет, он вернёт `Неопределено`, а не прервёт выполнение. Перебор идёт парами «ключ и значение».

```bsl
Для Каждого КлючЗначение Из Цены Цикл
    Сообщить(КлючЗначение.Ключ + ": " + Строка(КлючЗначение.Значение));
КонецЦикла;
```

## Список значений на закуску

Есть ещё `СписокЗначений`, коллекция, заточенная под взаимодействие с пользователем. Каждый её элемент несёт не только значение, но и представление, пометку и картинку, а у списка есть методы показа диалога выбора. Для внутренних расчётов обычно хватает массива или соответствия; список значений берут, когда данные показывают человеку и дают выбрать.

## Типичные ошибки

**Выйти за границу массива.** Обход по индексу до `Количество()` вместо `Количество() - 1` даёт обращение к несуществующему элементу и ошибку. Проще перебирать `Для Каждого`, если индекс не нужен.

**Дать структуре ключ с пробелом.** `Контрагент.Вставить("Полное имя", ...)` недопустимо: ключ структуры это идентификатор. Для произвольных ключей нужно соответствие.

**Ждать ошибку вместо Неопределено.** `Соответствие.Получить(Ключ)` для отсутствующего ключа вернёт `Неопределено`, а не прервёт код. Проверяйте результат, если ключ может отсутствовать.

**Путать Вставить у структуры и Добавить у массива.** Массив пополняют методом `Добавить(Значение)`, а структуру и соответствие методом `Вставить(Ключ, Значение)`.

<Detail summary="Массив фиксированного размера и копирование коллекций">

`Новый Массив(3)` создаёт массив сразу из трёх элементов, заполненных `Неопределено`, к которым можно обращаться по индексу без предварительного `Добавить`. Ещё одна тонкость: присваивание коллекции копирует ссылку, а не сами данные. `Б = А`, где `А` это массив, даёт две переменные, указывающие на один и тот же массив, и добавление через `Б` видно в `А`. Чтобы получить независимую копию, берут функцию `Скопировать(А)`.

</Detail>
