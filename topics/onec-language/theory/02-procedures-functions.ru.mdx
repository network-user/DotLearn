# Процедуры и функции: как разбить код на кирпичи

Как только модуль перерастает пару десятков строк, повторяющиеся куски логики хочется вынести и назвать. Встроенный язык предлагает для этого два инструмента, и разница между ними принципиальна: процедура делает работу, а функция возвращает результат. Спутать их легко, а расплата наступает в момент вызова, когда платформа отказывается присвоить переменной «ничто». Разберёмся, когда что писать, как передаются параметры и почему `Знач` перед параметром иногда спасает от неуловимой ошибки.

## Процедура делает, функция возвращает

Процедура выполняет действие и не отдаёт наружу значения. Её объявление открывается словом `Процедура` и закрывается словом `КонецПроцедуры`.

```bsl
Процедура ЗаписатьВЛог(Текст)
    Сообщить("Лог: " + Текст);
КонецПроцедуры
```

Функция вычисляет и возвращает значение оператором `Возврат`. Её объявление закрывается словом `КонецФункции`.

```bsl
Функция ВычислитьСкидку(Сумма, Процент)
    Возврат Сумма * Процент / 100;
КонецФункции
```

Вызывают их по-разному, и это прямое следствие смысла. Процедуру вызывают отдельным оператором, потому что присваивать нечего. Результат функции подставляют в выражение.

```bsl
ЗаписатьВЛог("Заказ проведён");
Скидка = ВычислитьСкидку(100000, 10);
```

Попытка написать `Х = ЗаписатьВЛог(...)` вызовет ошибку: процедура не возвращает значения, и присвоить его переменной нельзя.

## Параметры по умолчанию

Если параметр нужен не всегда, ему задают значение по умолчанию прямо в объявлении через знак `=`. Тогда при вызове его можно опустить.

```bsl
Функция ВычислитьСкидку(Сумма, Процент = 5)
    Возврат Сумма * Процент / 100;
КонецФункции
```

Теперь `ВычислитьСкидку(100000)` посчитает по ставке 5 процентов, а `ВычислитьСкидку(100000, 10)` перекроет умолчание десяткой. Параметры со значением по умолчанию принято ставить в конце списка: так вызов остаётся однозначным.

## По ссылке или по значению

Вот деталь, которая ловит даже опытных разработчиков. По умолчанию параметры во встроенном языке передаются **по ссылке**. Это значит, что присваивание параметру внутри метода меняет ту переменную, которую передали при вызове.

```bsl
Процедура Удвоить(Число)
    Число = Число * 2;
КонецПроцедуры

Значение = 5;
Удвоить(Значение);
Сообщить(Значение);
```

Этот код выведет `10`, а не `5`: процедура изменила саму переменную `Значение`. Если такое поведение нежелательно, параметр помечают словом `Знач`. Тогда метод получает копию, и оригинал остаётся прежним.

```bsl
Процедура Удвоить(Знач Число)
    Число = Число * 2;
КонецПроцедуры
```

Теперь тот же вызов оставит `Значение` равным `5`. Хорошая привычка: помечать `Знач` все входные параметры, которые метод не должен менять. Это делает контракт метода честным и предотвращает случайные побочные эффекты.

## Экспорт открывает метод наружу

Метод, объявленный без `Экспорт`, виден только внутри своего модуля. Чтобы его можно было вызвать из другого модуля (например, из формы обратиться к общему модулю), в конце заголовка добавляют слово `Экспорт`.

```bsl
Функция ВычислитьСкидку(Знач Сумма, Знач Процент = 5) Экспорт
    Возврат Сумма * Процент / 100;
КонецФункции
```

Именно набор экспортных методов образует программный интерфейс модуля. Всё, что без `Экспорт`, остаётся внутренней кухней и снаружи недоступно.

## Типичные ошибки

**Присвоить результат процедуры.** Процедура не возвращает значения. `Итог = МояПроцедура()` вызовет ошибку. Нужна функция с `Возврат`.

**Забыть Возврат в функции.** Если выполнение дойдёт до `КонецФункции`, не встретив `Возврат` со значением, функция вернёт `Неопределено`. Часто это тихая ошибка: вызывающий код получает не то, что ждал.

**Случайно изменить входной параметр.** Поскольку передача идёт по ссылке, присваивание параметру меняет переменную вызывающего. Пометьте параметр словом `Знач`, если не хотите такого эффекта.

**Поставить обязательный параметр после параметра по умолчанию.** Параметры со значением по умолчанию должны идти в конце, иначе вызов становится неоднозначным.

<Detail summary="Возврат без значения: досрочный выход из процедуры">

В процедуре `Возврат` можно использовать без значения, просто чтобы выйти пораньше. Это удобно для проверки предусловий в начале метода: если данные не подходят, выходим сразу, не заворачивая остальной код в лишний `Если`. В функции `Возврат` без значения тоже завершает работу, но тогда функция вернёт `Неопределено`, что почти всегда признак ошибки в логике.

</Detail>
