# Как свернуть цикл с append в одно читаемое выражение

Почти любой код, который строит новый список, выглядит одинаково: завести пустой список, пройтись циклом по источнику, что-то посчитать, вызвать `append`. Четыре строки ради одной мысли «преобразуй каждый элемент». Comprehension сворачивает эти четыре строки в одно выражение, которое читается как фраза: «возьми `x*x` для каждого `x` из `nums`». Сводить его к синтаксическому сахару всё же рано. У comprehension своя область видимости, свой ускоренный байткод и несколько ловушек, на которых спотыкаются и опытные разработчики.

К концу раздела вы научитесь писать comprehension для списков, словарей и множеств, ставить условие-фильтр и не путать его с условным выражением, разворачивать вложенные comprehension в правильном порядке и объяснять, почему переменная цикла не протекает наружу.

## Порядок чтения: источник, потом фильтр, потом выражение

Хотя выражение-преобразование стоит в начале строки, применяется оно последним. Comprehension удобно читать справа налево по смыслу: сперва `for` перебирает источник, затем `if` отсеивает лишнее, и только к выжившим элементам применяется выражение слева. На рисунке этот конвейер показан по шагам.

Базовая форма без условия просто преобразует каждый элемент. Возьмём цены и поднимем их на 20 процентов:

```python
prices = [100, 250, 90]
with_markup = [p * 1.2 for p in prices]
print(with_markup)
```

Печатается `[120.0, 300.0, 108.0]`. Тот же результат дал бы цикл с `append`, но comprehension короче и, что важнее, сообщает намерение одним взглядом: «новый список из тех же элементов, каждый умножен на 1.2».

Условие после `for` превращает comprehension в фильтр. Оно решает, дойдёт ли элемент до выражения слева:

```python
users = ["Анна", "Борис", "Вера", "Артём"]
on_a = [name for name in users if name.startswith("А")]
print(on_a)
```

Останется `['Анна', 'Артём']`. Элементы, не прошедшие `if`, до выражения `name` не доходят и в результат не попадают.

## Не только списки: словари и множества

Та же скобочная форма строит словари и множества, меняется лишь тип внешних скобок. Фигурные скобки с двоеточием `{ключ: значение for ...}` дают словарь, фигурные без двоеточия `{элемент for ...}` дают множество. Соберём для каждого имени его длину, а затем оставим только уникальные длины:

```python
names = ["Анна", "Борис", "Вера"]
lengths = {name: len(name) for name in names}
unique = {len(name) for name in names}
print(lengths)
print(unique)
```

Печатается `{'Анна': 4, 'Борис': 5, 'Вера': 4}` и `{4, 5}`. Словарь сохранил порядок вставки, множество схлопнуло две четвёрки в одну и порядок не гарантирует.

Запустите пример и поменяйте условие: добавьте `if len(word) > 3` в оба comprehension и посмотрите, как из словаря и множества исчезают короткие слова. Затем попробуйте построить обратный словарь `{длина: слово}` и убедитесь, что одинаковые длины затирают друг друга, оставляя последнее слово.

## Вложенность: два for читаются как вложенные циклы

Когда источников два, comprehension разворачивается в вложенные циклы в том же порядке, в каком записаны `for`. Левый `for` внешний, правый внутренний. Развернуть матрицу в плоский список помогает пара `for`:

```python
matrix = [[1, 2], [3, 4], [5, 6]]
flat = [x for row in matrix for x in row]
print(flat)
```

Получится `[1, 2, 3, 4, 5, 6]`. Читайте это как «для каждой `row` из `matrix`, для каждого `x` из `row` возьми `x`». Порядок `for row ... for x ...` совпадает с порядком вложенных `for` в обычном цикле, а не наоборот, и это первое, что стоит проверить, когда результат перевёрнут.

<Callout type="warning">
Не путайте плоский `[x for row in matrix for x in row]` с вложенным `[[x for x in row] for row in matrix]`. Первый даёт один плоский список, второй сохраняет структуру и возвращает список списков. Скобки решают всё.
</Callout>

## Где comprehension удивляет

Переменная цикла живёт только внутри comprehension и наружу не протекает. Этим comprehension в Python 3 отличается от обычного `for`, где переменная остаётся видимой после цикла.

```python
squares = [i * i for i in range(3)]
print(squares)
print(i)
```

Первая строка печатает `[0, 1, 4]`, вторая падает с `NameError: name 'i' is not defined`. Причина в том, что comprehension исполняется в собственной скрытой функции: интерпретатор оборачивает его в отдельную область видимости, и `i` остаётся её локальной переменной. Побочный эффект приятный: comprehension не затирает переменную `i`, которая, возможно, уже жила в вашем коде.

У скрытой функции есть и обратная сторона. Внутри comprehension в теле класса не видны имена, объявленные на уровне класса, потому что скрытая функция их не захватывает. А если внутри comprehension нужно и посчитать значение, и переиспользовать его, на помощь приходит walrus-оператор `:=`, который присваивает прямо в выражении.

<MarginNote label="Смежная тема">
Walrus `:=` даёт захватить промежуточное значение прямо внутри comprehension, например `[y for x in data if (y := f(x)) > 0]`. Ему посвящён отдельный курс «Walrus-оператор».
</MarginNote>

Ещё одна тонкость касается места условия. Условие после `for` фильтрует, а условное выражение `A if cond else B` перед `for` преобразует и ничего не выбрасывает:

```python
data = [4, -2, 7, -5]
kept = [x for x in data if x > 0]
clamped = [x if x > 0 else 0 for x in data]
print(kept)
print(clamped)
```

`kept` равен `[4, 7]`, потому что отрицательные отсеяны фильтром. `clamped` равен `[4, 0, 7, 0]`, потому что тернарное выражение оставило длину списка прежней, заменив отрицательные на ноль. Один и тот же `if`, но смысл зависит от того, стоит он до выражения или после источника.

## Частые ошибки

Ниже собраны промахи, которые встречаются чаще всего.

- **Перепутанный порядок вложенных `for`.** Если плоский список выходит перевёрнутым, почти всегда виноват порядок `for`: он читается слева направо как внешний и внутренний цикл, а не справа налево.
- **Comprehension ради побочного эффекта.** Запись `[print(x) for x in xs]` печатает элементы, но заодно строит и выбрасывает список из `None`. Для чистого побочного эффекта берите обычный `for`, а не comprehension.
- **Фильтр вместо тернарного выражения.** Хотите заменять элементы, а не выкидывать, значит нужно условное выражение `x if cond else y` перед `for`, а не `if` после источника. Перепутав их, вы либо укоротите список, либо получите синтаксическую ошибку.
- **Слишком глубокая вложенность.** Comprehension в три `for` и два `if` читать тяжелее, чем обычный цикл. Когда выражение перестаёт помещаться в голове, разверните его обратно в цикл или генератор.

<Detail summary="Почему comprehension быстрее ручного цикла с append">

Дело не в магии, а в байткоде. Обычный цикл на каждой итерации ищет атрибут `append` у списка и вызывает его как обычный метод: поиск имени плюс вызов функции повторяются на каждом шаге. Comprehension компилируется в специализированную инструкцию `LIST_APPEND`, которая добавляет элемент напрямую, без поиска атрибута и без накладных расходов на вызов метода.

```python
import dis
dis.dis("[x for x in nums]")
```

В разборке видно инструкцию `LIST_APPEND` вместо пары «загрузить метод append, вызвать его». На больших списках эта разница накапливается, поэтому comprehension обычно ощутимо быстрее эквивалентного цикла с ручным `append`.

</Detail>
