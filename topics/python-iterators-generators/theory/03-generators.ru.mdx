# Функция, которая отдаёт значения по одному и засыпает между ними

Обычная функция живёт одну вспышку: её вызвали, она посчитала, вернула результат и забыла всё, что знала. Локальные переменные исчезли, позиция в коде потеряна. Но что если функции нужно отдавать миллион строк лога по одной, не держа их все в памяти, или порождать бесконечную последовательность чисел? Для этого нужна функция, которая умеет поставить себя на паузу, вернуть одно значение, а на следующий запрос проснуться ровно там, где заснула, со всеми своими переменными. Такую функцию даёт ключевое слово `yield`, а сама она называется генератором.

К концу раздела вы поймёте, чем `yield` отличается от `return`, почему вызов генераторной функции не исполняет её тело, как генератор экономит память на больших данных и почему по одному и тому же генератору нельзя пройтись дважды.

## Генератор работает как машина состояний

Стоит функции содержать хотя бы один `yield`, и Python превращает её в генераторную: теперь её вызов не запускает код, а возвращает объект-генератор, который умеет останавливаться и продолжаться. Каждый `next()` доводит выполнение до очередного `yield`, отдаёт значение и замораживает кадр функции вместе со всеми локальными переменными. Следующий `next()` размораживает кадр и продолжает с той же строки.

Вот простейший генератор, обратный отсчёт. Обратите внимание, что переменная `n` живёт между вызовами `next` и хранит состояние отсчёта:

```python
def countdown(n):
    while n > 0:
        yield n
        n -= 1

gen = countdown(3)
print(next(gen))
print(next(gen))
print(next(gen))
```

Печатается `3`, `2`, `1`. Между двумя `next` кадр `countdown` заморожен на строке с `yield`: `n` помнит своё значение, цикл помнит, что не завершён. Четвёртый `next(gen)` возбудил бы `StopIteration`, потому что цикл дошёл до конца и функция вернулась.

## Ленивость: код бежит только когда просят

Ключевое свойство генератора состоит в лени. Тело не исполняется ни на грамм, пока кто-то не запросит первый элемент. Запустите пример: сообщение «генератор создан» печатается сразу, а «считаю 0», «считаю 1» появляются только по мере обхода в цикле. Поменяйте `range(3)` на `range(6)` и убедитесь, что вычисления растягиваются на каждый шаг, а не происходят разом.

<PyDemo
  title="Когда именно бежит код генератора"
  code={`def squares(limit):
    print("генератор создан")
    for i in range(limit):
        print("считаю", i)
        yield i * i

gen = squares(3)
print("до цикла")
for value in gen:
    print("получил", value)`}
/>

Порядок вывода показывает суть: «до цикла» печатается раньше любого «считаю», потому что `squares(3)` только создал генератор и тело не тронул. Работа началась на первом шаге цикла и шла порциями, чередуя вычисление и потребление. Именно поэтому генератор способен породить бесконечную последовательность: он никогда не пытается вычислить всё сразу.

## Память: один кадр вместо всех элементов

Список честно вычисляет и хранит все свои элементы. Генератор держит в памяти только текущий кадр и рождает элементы по одному, когда их просят. На миллионе строк это разница между гигабайтом и парой килобайт.

## return в генераторе не отдаёт значение, а завершает

Внутри генератора `return` не возвращает результат наружу, как в обычной функции. Он просто завершает генератор, а значение, если оно есть, прячется в атрибут `value` исключения `StopIteration`. Достать его руками можно так:

```python
def take_until_zero(numbers):
    total = 0
    for n in numbers:
        if n == 0:
            return total
        total += n
        yield total

gen = take_until_zero([2, 3, 0, 9])
print(list(gen))
```

Печатается `[2, 5]`. Дойдя до нуля, генератор выполнил `return total` и остановился, а `9` в результат уже не попал. Само значение `total` при этом ушло в `StopIteration.value`, и увидеть его напрямую можно, только перехватив исключение или через `yield from`, о котором речь в разделе про продвинутые генераторы.

## Где генераторы удивляют

Генератор это итератор, а значит одноразовый. Пройти по одному и тому же объекту-генератору дважды не выйдет: второй проход увидит пустоту.

```python
gen = (n for n in range(3))
print(sum(gen))
print(sum(gen))
```

Первый `sum` печатает `3`, второй печатает `0`. К моменту второго вызова генератор исчерпан и назад не отматывается. Чтобы обойти данные дважды, либо создайте генератор заново, либо зафиксируйте их списком.

Второй сюрприз касается отложенного старта. Раз тело не исполняется до первого `next`, любая проверка аргументов в начале генераторной функции сработает не при вызове, а лишь при первом запросе элемента.

```python
def positives(numbers):
    if not numbers:
        raise ValueError("пустой вход")
    for n in numbers:
        if n > 0:
            yield n

gen = positives([])
print("генератор создан, исключения ещё нет")
```

Сообщение печатается спокойно, хотя вход пустой. `ValueError` возникнет только на первом `next(gen)`, а не на строке `positives([])`. Если проверку нужно выполнять сразу, оборачивают генератор в обычную функцию, которая валидирует вход и лишь затем возвращает внутренний генератор.

## Частые ошибки

Ниже промахи, на которых генераторы подводят чаще всего.

- **Повторный проход по исчерпанному генератору.** Сохранили генератор, посчитали по нему сумму, потом хотите распечатать, а там пусто. Нужны данные дважды, значит пересоздайте генератор или зафиксируйте список.
- **Путаница функции и объекта.** `countdown` это генераторная функция, а `countdown(3)` это генератор. В `for` и `next` ставят именно вызов `countdown(3)`, а не имя функции.
- **Ожидание, что `return` что-то отдаст.** В генераторе `return` завершает работу, а не выдаёт значение через `yield`. Возвращаемое значение прячется в `StopIteration.value`.
- **Проверка аргументов в теле генератора.** Она сработает при первом `next`, а не при вызове. Ранняя валидация требует функции-обёртки поверх генератора.

<Detail summary="Что значит заморозить кадр, и какие бывают состояния генератора">

За паузу отвечает кадр выполнения (frame). Обычная функция после `return` уничтожает свой кадр, а генератор его сохраняет: у объекта-генератора есть ссылка на кадр в атрибуте `gi_frame`, где лежат все локальные переменные и указатель на текущую строку. `yield` сохраняет этот указатель и отдаёт управление наружу, `next` восстанавливает и продолжает.

Сам генератор проходит через состояния: только что созданный (`GEN_CREATED`, ни одного `next` ещё не было), выполняющийся прямо сейчас (`GEN_RUNNING`), приостановленный на `yield` (`GEN_SUSPENDED`) и завершённый (`GEN_CLOSED`). Прочитать текущее состояние помогает `inspect.getgeneratorstate(gen)`.

```python
import inspect
def g():
    yield 1
gen = g()
print(inspect.getgeneratorstate(gen))
next(gen)
print(inspect.getgeneratorstate(gen))
```

Печатается `GEN_CREATED`, затем `GEN_SUSPENDED`.

</Detail>
