# Не просто найти, а разобрать на части

Паттерн `\d\d\.\d\d\.\d{4}` находит дату «09.05.1945» целиком, и для подсветки в тексте этого достаточно. Но реальная задача обычно глубже: вытащить день, месяц и год по отдельности, переставить их в формат ISO, сравнить с другой датой. Совпадение нужно не просто найти, его нужно разобрать на части. Для этого регулярные выражения дают группы, а заодно решают и вторую задачу: как сказать «здесь допустим один из нескольких вариантов текста».

К концу раздела вы будете захватывать части совпадения круглыми скобками, обращаться к ним по номеру и имени, строить альтернативы через `|` и ссылаться на уже захваченный текст прямо внутри паттерна.

## Скобки захватывают, номера считаются слева

Круглые скобки в паттерне делают две вещи сразу: объединяют кусок паттерна в один элемент и запоминают текст, который под этот кусок совпал. Запомненный текст называется захватом, а сами скобки группой. Группы нумеруются по открывающим скобкам слева направо, начиная с единицы; номер ноль зарезервирован за всем совпадением. m = re.search(r"(\\d\\d)\\.(\\d\\d)\\.(\\d\\d\\d\\d)", "Победа: 09.05.1945, парад")

Номера работают, пока групп две-три. В длинном паттерне считать скобки мучительно, поэтому группам дают имена: `(?P<day>\d\d)`. К именованной группе обращаются через `m.group("day")`, а метод `m.groupdict()` собирает все именованные захваты в словарь:

```python
import re

m = re.search(r"(?P<day>\d\d)\.(?P<month>\d\d)\.(?P<year>\d{4})", "09.05.1945")
print(m.group("day"))
print(m.groupdict())
```

Печатается `09` и `{'day': '09', 'month': '05', 'year': '1945'}`. Именованные группы сохраняют и обычные номера, так что `m.group(1)` продолжает работать.

## Альтернатива: один из вариантов

Вертикальная черта `|` предлагает движку варианты: `кот|собака` совпадает либо с «кот», либо с «собака». У альтернативы самый низкий приоритет из всех операций, она делит на варианты всё, что находится слева и справа, до границ паттерна или объемлющей группы. Паттерн `до свидания|встречи` читается не как «до свидания» или «до встречи», а как «до свидания» или «встречи». Границы вариантов задают группой: `до (?:свидания|встречи)`.

Здесь появилась новая запись: `(?:...)`. Это группа без захвата. Она группирует, но не запоминает текст и не получает номера. Берите её всегда, когда скобки нужны только для приоритета: с квантификатором `(?:кот)+`, с альтернативой `(?:да|нет)`. Паттерн останется читаемым, а нумерация захватов не поплывёт.

Важная деталь: альтернатива выбирает первый подходящий вариант, а не самый длинный. Паттерн `кош|кошка` на строке «кошка» совпадёт с «кош», потому что этот вариант стоит раньше и уже успешен. Если варианты перекрываются, ставьте более длинный первым: `кошка|кош`.

## Обратные ссылки: совпасть с уже захваченным

Захват доступен не только после поиска, но и прямо внутри паттерна. Запись `\1` означает «здесь должен стоять точно тот же текст, который захватила группа 1». Классическое применение: поиск случайно задвоенных слов.

```python
import re

print(re.findall(r"(\w+) \1", "это было было давно"))
```

Печатается `['было']`. Обратите внимание: `\1` это не «повтори паттерн группы», а «повтори её захват». Паттерн `(\w+) \1` не совпадёт с «было есть», хотя оба слова подходят под `\w+`. Для именованных групп та же ссылка пишется как `(?P=name)`.

## Где группы удивляют

Главный сюрприз связан с `findall`. Пока в паттерне нет групп, он возвращает список полных совпадений. Появилась одна группа, и он возвращает список её захватов. Появилось две и больше, и он возвращает список кортежей. Форма результата зависит от количества скобок:

```python
import re

text = "Иванов Иван, Петрова Анна"
print(re.findall(r"(\w+) (\w+)", text))
```

Печатается `[('Иванов', 'Иван'), ('Петрова', 'Анна')]`, а не список полных совпадений. Это поведение ломает код, когда кто-то добавляет «безобидные» скобки в работающий паттерн. Нужны скобки без смены формы результата, ставьте `(?:...)`.

Второй сюрприз: группа под квантификатором хранит только последний повтор. В `re.findall(r"(ха)+", "хахаха ха")` каждый матч отдаёт захват группы, а не всё совпадение, и это `['ха', 'ха']`: от «хахаха» осталась последняя «ха». Нужно всё совпадение целиком, оборачивайте повтор во внешнюю группу: `((?:ха)+)`.

## Частые ошибки

- **`|` без группы съедает полпаттерна.** `^москва|питер$` означает «москва в начале» или «питер в конце», а не «одно из двух слов целиком». Почему так выходит: у альтернативы самый низкий приоритет. Лечится группой: `^(?:москва|питер)$`.
- **Захватывающие скобки «для порядка».** Каждая лишняя пара скобок меняет результат `findall` и сдвигает номера групп. Скобки только для группировки пишите как `(?:...)`.
- **Короткий вариант раньше длинного.** `кош|кошка` никогда не найдёт «кошка» целиком. Перекрывающиеся альтернативы упорядочивают от длинной к короткой.
- **`\1` в надежде на «такой же по форме».** Обратная ссылка требует тот же текст, а не то же правило. «12-13» не совпадёт с `(\d\d)-\1`.

<Detail summary="Нумерация вложенных групп и ссылки по имени">

Когда группы вложены, номер каждой определяется позицией её открывающей скобки. В паттерне `((\d+)-(\d+))` группа 1 это всё выражение `(\d+)-(\d+)`, группа 2 первое число, группа 3 второе. Правило одно: идите по паттерну слева направо и считайте только открывающие скобки, игнорируя `(?:` и другие конструкции с вопросительным знаком, они номеров не получают.

```python
import re

m = re.search(r"((\d+)-(\d+))", "матч 3-2 закончился")
print(m.group(1), m.group(2), m.group(3))
```

Печатается `3-2 3 2`. Ссылаться на именованную группу внутри паттерна лучше тоже по имени, `(?P=name)`: такой паттерн переживёт вставку новых групп без правки номеров.

</Detail>
