# Условный рендеринг

Экран редко выглядит одинаково при любых данных. Пустая корзина показывает «Здесь пока пусто», полная - список товаров и кнопку оформления. Гость видит «Войти», авторизованный - своё имя. Всё это - условный рендеринг: разметка ветвится в зависимости от состояния. Раз JSX - это выражение, а компонент - обычная функция, никакого особого синтаксиса не нужно: ветвление делают теми же средствами JavaScript, только аккуратно вписанными в разметку.

В этом уроке соберём четыре способа показать одно или другое и разберём ловушку с числом, на которой обжигаются все.

## Четыре способа ветвить разметку

Первый и самый частый - тернарный оператор прямо в фигурных скобках. Он выбирает одно из двух выражений:

```jsx
function Greeting({ user }) {
  return <p>{user ? `Привет, ${user.name}` : 'Войдите в аккаунт'}</p>;
}
```

Второй способ - когда нужно показать что-то или совсем ничего. Здесь удобен оператор `&&`: если условие истинно, выводится правая часть, если ложно - не выводится ничего.

```jsx
function Cart({ items }) {
  return (
    <div>
      <h2>Корзина</h2>
      {items.length > 0 && <button>Оформить заказ</button>}
    </div>
  );
}
```

Кнопка появится, только когда в корзине что-то есть. Работает это благодаря природе `&&` в JavaScript: выражение `истина && X` возвращает `X`, а `ложь && X` возвращает ложное значение, которое React не рисует.

Третий способ - ранний `return`, когда целый компонент в каком-то состоянии не должен рисовать основную разметку. Мы уже видели его в уроке про компоненты:

```jsx
function Warning({ show }) {
  if (!show) {
    return null;
  }
  return <p>Мало на складе</p>;
}
```

Четвёртый - вынести разметку в переменную до `return`. Когда веток много и тернарники начинают вкладываться друг в друга, читаемее собрать нужный кусок заранее, а в JSX подставить готовую переменную. Логику можно писать любую - `if`, `switch`, что угодно, - лишь бы она стояла до `return`, а не внутри фигурных скобок JSX.

## Ловушка нуля

Вот код, который выглядит правильным и предаёт новичка:

```jsx
function Badge({ count }) {
  return <div>{count && <span>Новых: {count}</span>}</div>;
}
```

Пока `count` больше нуля, всё хорошо. Но когда `count` равен `0`, на экране появляется одинокий «0». Причина в том, что `0 && X` в JavaScript возвращает `0` - а число `0`, в отличие от `false` и `null`, React честно рисует. Условие хотело сказать «нечего показывать», а вывело ноль.

<Callout type="warning" title="Слева от логического И должно стоять булево значение">
Ловушка срабатывает, когда слева от `&&` стоит число, которое может быть нулём (`length`, счётчик, цена). Лечится приведением к булеву: `count > 0 && ...` или `Boolean(count) && ...`. Тернарник этой болезнью не страдает, потому что всегда выбирает одну из двух явных веток.
</Callout>

## Краевые случаи

Значения `null`, `undefined`, `true` и `false` в разметке не рисуют ничего - на этом и держится приём с `&&`. Опасны только числа: `0` виден, а `NaN` тоже выводится как «NaN». Ещё одна мелочь: тернарник хорош для «одно или другое», но три-четыре вложенных тернарника превращают JSX в нечитаемую лапшу - это сигнал вынести выбор в переменную или в отдельный компонент.

## Типичные ошибки

**Число слева от `&&`.** `{items.length && ...}` при пустом списке выведет «0». Приводите к булеву: `{items.length > 0 && ...}`.

**`if` внутри фигурных скобок JSX.** В `{}` идёт выражение, а инструкция `if` - нет. Ветвьте тернарником или уводите `if` до `return`.

**Вложенные тернарники.** Два и больше уровней `? :` внутри JSX нечитаемы. Соберите разметку в переменную или разбейте на компоненты.

**Забыть ветку «иначе» там, где она нужна.** `&&` показывает или ничего; если нужно именно «одно или другое», берите тернарный оператор с обеими ветками.

<Detail summary="Почему логическое И и тернарник ведут себя именно так">

Оба оператора - обычные средства JavaScript, а не изобретение React. Тернарный `a ? b : c` всегда вычисляется в `b` или `c` - в одно из двух значений, поэтому он безопасен: что бы ни стояло в условии, наружу выйдет явно указанная ветка. Оператор `&&` - это «логическое И» с коротким замыканием: он возвращает левое значение, если оно ложно, и правое, если левое истинно. React же рисует всё, кроме `null`, `undefined`, булевых значений и пустой строки. Пересечение этих двух правил и даёт ловушку: ложный, но «рисуемый» ноль проскакивает наружу. Понимая механику, вы будете предвидеть такие случаи, а не ловить их постфактум.

</Detail>
