# Пропсы: данные текут сверху вниз

Компонент `ProductCard` из прошлого урока умеет рисовать карточку товара - но пока одну и ту же. Чтобы из одного компонента получилась сотня разных карточек каталога, ему нужно передавать данные: этому - «Молоко» за 89 рублей, тому - «Хлеб» за 45. Способ передать данные в компонент называется пропсами (props, от properties), и работает он ровно как аргументы обычной функции. Родитель задаёт значения, компонент их читает и подставляет в разметку.

В этом уроке вы научитесь передавать пропсы, удобно их принимать, отдавать компоненту вложенное содержимое через `children` и поймёте главное ограничение: пропсы доступны только для чтения.

## Атрибуты тега становятся аргументами функции

Пропсы вы уже писали - это атрибуты в JSX. Когда родитель ставит атрибуты на теге компонента, React собирает их в один объект и передаёт первым аргументом в функцию компонента.

```jsx
function UserCard(props) {
  return <p>{props.name}, {props.age} лет</p>;
}

function App() {
  return <UserCard name="Анна" age={30} />;
}
```

React соберёт из атрибутов объект `{ name: 'Анна', age: 30 }` и вызовет `UserCard(props)`. Внутри компонента к значениям обращаются через `props.name` и `props.age`. Обратите внимание на разницу: строку задают в кавычках, а число - в фигурных скобках. `age="30"` передало бы строку «30», а не число 30, и арифметика с ней позже удивила бы вас.

## Деструктуризация: берём сразу нужные поля

Писать `props.` перед каждым полем утомительно. Поэтому пропсы почти всегда разбирают прямо в списке параметров - через деструктуризацию объекта, знакомую вам по JavaScript:

```jsx
function UserCard({ name, age }) {
  return <p>{name}, {age} лет</p>;
}
```

Смысл тот же, что и раньше, но читается чище: сразу видно, какие поля компонент ожидает. Тут же удобно задать значения по умолчанию на случай, если родитель что-то не передал:

```jsx
function Badge({ text = 'новинка', color = 'gray' }) {
  return <span className={color}>{text}</span>;
}
```

Если родитель напишет просто `<Badge />`, компонент подставит «новинка» и «gray». Значения по умолчанию избавляют от проверок «а вдруг проп не передали».

## children: содержимое между тегами

Иногда компоненту передают не отдельные значения, а целый кусок разметки - то, что стоит между открывающим и закрывающим тегами. React кладёт это содержимое в специальный проп `children`.

```jsx
function Panel({ title, children }) {
  return (
    <section>
      <h3>{title}</h3>
      <div>{children}</div>
    </section>
  );
}

function App() {
  return (
    <Panel title="Доставка">
      <p>Завтра до 12:00 по Москве</p>
    </Panel>
  );
}
```

Всё, что вложено внутрь `<Panel>...</Panel>`, придёт в `children` и встанет туда, где компонент его выводит. Так делают обёртки: карточки, модальные окна, панели - каркас задаёт компонент, а начинку передаёт родитель.

## Пропсы доступны только для чтения

Вот правило, которое отличает пропсы от обычных переменных: **компонент не имеет права менять свои пропсы**. Он получает их, читает и рисует - но не присваивает им новые значения. Пропсы принадлежат родителю; ребёнок лишь смотрит на них.

```jsx
function UserCard({ name }) {
  name = 'Другой';
  return <p>{name}</p>;
}
```

Так делать нельзя. React строит поток данных в одну сторону - сверху вниз, от родителя к детям, - и это делает приложение предсказуемым: чтобы понять, почему у карточки такое имя, вы поднимаетесь к родителю, который его задал, а не гадаете, кто из детей его втихаря переписал. Если ребёнку нужно, чтобы данные изменились, он не правит проп, а просит об этом родителя - через функцию-коллбэк. Как именно, разберём в уроке про подъём состояния.

<Callout type="tip" title="Пропсы - фотография, а не пульт">
Проп - это снимок данных на момент рендера, а не рычаг управления родителем. Ребёнок читает снимок и рисует по нему. Захотеть новый снимок он может, но сменить его вправе только тот, кто снимок сделал, - родитель.
</Callout>

## Краевые случаи

Строки передают в кавычках, всё остальное - в фигурных скобках: числа `count={3}`, массивы `items={list}`, объекты `style={obj}`, функции `onSave={handleSave}`. У булевых пропсов есть сокращение: `<input disabled />` означает то же, что `disabled={true}`. А пропущенный проп приходит как `undefined` - отсюда и польза значений по умолчанию.

## Типичные ошибки

**Менять пропсы внутри компонента.** Присваивание `props.name = ...` ломает однонаправленный поток и часто просто не сработает. Данные меняет тот, кто ими владеет; ребёнок сигналит наверх коллбэком.

**Строка вместо числа.** `age="30"` кладёт в проп строку. Для чисел, булевых и любых нестроковых значений нужны фигурные скобки: `age={30}`.

**Путать проп и внутреннюю переменную.** Проп приходит извне и меняется только у родителя. Значение, которое компонент считает сам, - это не проп; для собственной изменяемой памяти есть состояние, о нём следующий урок.

<Detail summary="Проп-функция: как ребёнок разговаривает с родителем">

Раз пропсы едут только вниз, встаёт вопрос: как ребёнок сообщает родителю, что пользователь нажал кнопку? Ответ - передать вниз не только данные, но и функцию. Родитель кладёт свою функцию в проп (`onLike={handleLike}`), ребёнок вызывает её в нужный момент, и управление возвращается наверх, к тому, кто владеет данными. Данные текут вниз, события - вверх через коллбэки. Это основа урока про подъём состояния; пока достаточно знать, что функция - такой же законный проп, как число или строка.

</Detail>
