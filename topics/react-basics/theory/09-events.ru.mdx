# Обработка событий

До сих пор наши компоненты только показывали данные. Пора дать пользователю нажимать кнопки, вводить текст и отправлять формы - иначе интерфейс мёртв. В React события подключают прямо в разметке: вы вешаете на элемент проп-обработчик вроде `onClick`, кладёте в него функцию, и React вызовет её, когда событие случится. А внутри обработчика вы делаете единственное, что оживляет экран, - меняете состояние.

В этом уроке разберём, как вешать обработчики, как передавать в них аргументы, что за объект события прилетает первым аргументом и почему одна лишняя пара скобок превращает обработчик клика в код, срабатывающий сам собой при рендере.

## Обработчик - это функция в пропе

События подключают камелкейс-пропами: `onClick`, `onChange`, `onSubmit`, `onMouseEnter`. В такой проп кладут функцию, и React сам вызовет её в нужный момент:

```jsx
function BuyButton() {
  function handleClick() {
    alert('Товар добавлен в корзину');
  }
  return <button onClick={handleClick}>Купить</button>;
}
```

Обратите внимание: в `onClick` передаётся `handleClick` - имя функции, без скобок. Вы отдаёте React саму функцию, чтобы он вызвал её позже, по клику. Часто обработчик пишут прямо на месте стрелочной функцией, особенно когда он короткий:

```jsx
function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(count + 1)}>
      Кликов: {count}
    </button>
  );
}
```

Вот и весь секрет интерактивности: событие вызывает обработчик, обработчик меняет состояние сеттером, смена состояния запускает рендер, и экран показывает новое число. Круг замкнулся.

## Передать аргумент: оберните в стрелку

Частая ситуация: в списке товаров каждая кнопка «Удалить» должна знать свой `id`. Но если написать `onClick={remove(id)}`, функция вызовется сразу при рендере, а не по клику (об этой ловушке - ниже). Правильно обернуть вызов в стрелочную функцию: тогда в проп попадёт функция, а вызов с аргументом произойдёт уже по клику:

```jsx
function Row({ id, remove }) {
  return <button onClick={() => remove(id)}>Удалить</button>;
}
```

Стрелка `() => remove(id)` - это функция, которую React придержит до клика. Внутри неё вы вольны вызвать что угодно и с какими угодно аргументами.

## Объект события

Первым аргументом в обработчик React передаёт объект события - обёртку над настоящим браузерным событием, одинаковую во всех браузерах. Из него читают данные о произошедшем: у поля ввода это `e.target.value`, текущий текст.

```jsx
function NameField() {
  const [name, setName] = useState('');
  return (
    <input value={name} onChange={(e) => setName(e.target.value)} />
  );
}
```

На каждое нажатие клавиши `onChange` вызывает обработчик, тот берёт свежий текст из `e.target.value` и кладёт его в состояние. Поле, у которого `value` привязано к состоянию, а `onChange` его обновляет, называют управляемым: источник истины - состояние, а не сам DOM-элемент.

У объекта события есть и методы. Главный из них - `preventDefault`, отменяющий поведение браузера по умолчанию. Классический случай - форма: по умолчанию её отправка перезагружает страницу, чего в приложении на React почти никогда не нужно.

```jsx
function SearchForm({ onSearch }) {
  function handleSubmit(e) {
    e.preventDefault();
    onSearch();
  }
  return (
    <form onSubmit={handleSubmit}>
      <button>Искать</button>
    </form>
  );
}
```

<Callout type="warning" title="Скобки решают всё">
`onClick={handleClick}` передаёт функцию - React вызовет её по клику. `onClick={handleClick()}` вызывает функцию прямо во время рендера и кладёт в `onClick` её результат (обычно `undefined`). Разница в одной паре скобок, а поведение противоположное: во втором случае по клику не произойдёт ничего, зато обработчик выстрелит сам собой при отрисовке.
</Callout>

## Краевые случаи

Имена событий строго камелкейс: `onClick`, а не `onclick`; последнее React не распознает как обработчик. Объект события - синтетический: это обёртка React, а не «сырое» событие браузера, хотя ведёт себя почти так же. И помните, что обработчик - обычная функция: в ней доступны пропсы и состояние компонента через замыкание, поэтому передавать их отдельно не нужно.

## Типичные ошибки

**Вызвать функцию вместо передачи.** `onClick={handleClick()}` срабатывает при рендере, а не по клику. Передавайте ссылку `onClick={handleClick}` или оборачивайте в стрелку.

**Забыть стрелку при аргументе.** `onClick={remove(id)}` вызовется сразу. Нужен аргумент - пишите `onClick={() => remove(id)}`.

**Забыть preventDefault в форме.** Без `e.preventDefault()` отправка формы перезагрузит страницу, и состояние потеряется.

**Строчные имена событий.** `onclick` React не подхватит. Все обработчики - камелкейс.

<Detail summary="Почему событие синтетическое">

React не вешает слушатели на каждый ваш элемент по отдельности. Вместо этого он ставит их на корень приложения и сам разводит события к нужным обработчикам - это экономит память и упрощает работу. Чтобы поведение не зависело от причуд конкретного браузера, React заворачивает нативное событие в собственный объект с одинаковым во всех браузерах интерфейсом - его и называют синтетическим. В 99% случаев вы работаете с ним как с обычным событием: читаете `e.target`, зовёте `e.preventDefault`. А если понадобится именно нативное событие, оно доступно через `e.nativeEvent`.

</Detail>
