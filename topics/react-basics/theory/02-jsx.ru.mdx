# JSX: разметка прямо в JavaScript

Первое, что удивляет в коде на React, - разметка, написанная прямо посреди JavaScript, без кавычек и без шаблонных строк:

```jsx
const greeting = <h1>Привет, Москва!</h1>;
```

Строка выглядит как HTML, но лежит в переменной, как обычное значение. Это JSX - расширение синтаксиса, которое позволяет описывать разметку выражением языка. Браузер JSX не понимает, поэтому перед запуском его переводят в обычные вызовы функций. В этом уроке вы научитесь читать JSX, вставлять в него данные и обходить три-четыре правила, на которых спотыкаются все новички.

## JSX - это не строка, а вызов функции

Ключ к пониманию: JSX не текст и не HTML. Каждый тег сборщик превращает в вызов `React.createElement`, а тот возвращает обычный объект - лёгкое описание узла с полями «тип» и «свойства». Эти две записи означают ровно одно и то же:

```jsx
const a = <h1 className="title">Привет</h1>;
const b = React.createElement('h1', { className: 'title' }, 'Привет');
```

## Фигурные скобки впускают JavaScript

Внутри JSX любое место, где нужны данные, открывают фигурными скобками. В них можно поставить любое выражение: переменную, арифметику, вызов функции, обращение к полю объекта.

```jsx
const name = 'Анна';
const price = 250;
const el = (
  <p>
    {name}, к оплате {price * 2} ₽
  </p>
);
```

React вычислит выражения и подставит результат: получится «Анна, к оплате 500 ₽». Важное слово - «выражение». В скобки идёт то, что возвращает значение: `price * 2`, `user.name`, тернарный оператор. Инструкции туда не помещаются: `if`, `for`, объявление переменной внутри `{}` работать не будут, потому что они ничего не возвращают. Ветвление внутри разметки делают тернарным оператором, а перебор - методом `map`, и этому посвящены отдельные уроки.

## Атрибуты: похоже на HTML, но не HTML

Атрибуты пишут почти как в HTML, с двумя оговорками. Значение-строку задают в кавычках, а значение-выражение - в фигурных скобках:

```jsx
const avatar = <img src={user.photo} alt="Аватар" width={48} />;
```

Первое отличие от HTML прячется в именах. JSX - это JavaScript, а там `class` и `for` уже заняты под ключевые слова языка. Поэтому атрибут класса пишут как `className`, а атрибут `for` у подписи - как `htmlFor`. Остальные составные имена идут в camelCase: `onclick` становится `onClick`, `tabindex` - `tabIndex`.

<Callout type="warning" title="class молча не сработает">
Если по привычке написать `class="title"`, React в режиме разработки предупредит в консоли, а стиль, скорее всего, не применится. Первое, что стоит проверить, когда CSS-класс «не цепляется», - не забыли ли вы букву `Name` в слове `className`.
</Callout>

## Один корень на возврат

JSX-выражение обязано иметь один корневой элемент. Вернуть два соседних тега подряд нельзя - это как попытаться вернуть из функции два значения через запятую. Есть два выхода: обернуть в общий тег или в пустой фрагмент `<>...</>`, который группирует детей, не добавляя лишнего узла в DOM.

```jsx
function Card() {
  return (
    <>
      <h2>Заголовок</h2>
      <p>Текст карточки</p>
    </>
  );
}
```

Скобки вокруг возвращаемого JSX - не синтаксис React, а защита от коварства JavaScript: если написать `return` и перенести тег на следующую строку без скобок, автоматическая расстановка точек с запятой превратит возврат в `return;` и функция отдаст `undefined`. Круглые скобки прижимают разметку к `return` и убирают эту ловушку.

## Что рендерится, а что нет

Не всякое значение в фигурных скобках видно на экране, и это регулярно удивляет. Строки и числа выводятся как есть. А вот `null`, `undefined`, `true` и `false` не рисуют ничего - это удобно для условного показа. Ловушка одна: число `0` - обычное значение и честно выводится как «0».

```jsx
const items = [];
const el = <div>{items.length && <p>Есть товары</p>}</div>;
```

Когда массив пуст, `items.length` равно `0`, и вместо «ничего» на экране появится «0». Полный разбор этой ловушки ждёт в уроке про условный рендеринг; пока просто запомните, что `0` виден, а `false` - нет.

## Типичные ошибки

**`class` вместо `className`.** Самая частая опечатка первых дней. Стиль не применяется, в консоли висит предупреждение.

**Забытый корневой элемент.** Два тега подряд в `return` без обёртки - ошибка сборки. Оберните во фрагмент `<>...</>` или в общий тег.

**`if` внутри фигурных скобок.** В `{}` идут выражения, а не инструкции. Ветвление в разметке пишут тернарным оператором; развесистую логику выносят в переменную до `return`.

**Переменная как текст.** Запись `<p>name</p>` выведет буквально слово «name». Чтобы подставить значение переменной, её обрамляют фигурными скобками: `<p>{name}</p>`.

<Detail summary="Комментарии и пробелы внутри JSX">

Комментарий внутри разметки пишут выражением в скобках: `{/* это заметка */}`. Обычный `//` там не сработает, потому что это по-прежнему JSX, а не строка кода. Переносы строк и лишние пробелы между тегами JSX схлопывает так же, как это делает HTML, поэтому за красивым форматированием можно не следить: на экран попадёт один пробел там, где в коде их несколько.

</Detail>
