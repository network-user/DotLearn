# Аутентификация: OAuth2, OIDC и Keycloak

Практический разбор того, как современные приложения делегируют вход и выдают доступ: протокол OAuth2 и его роли, надстройка OpenID Connect с id_token, модель Keycloak (realm, client, roles), ключевые потоки (authorization code + PKCE, client credentials) и устройство JWT - его подпись, валидация и обновление через refresh token. Тема для backend- и frontend-разработчиков, которым нужно встроить вход через identity provider и понимать, что именно происходит за редиректами и заголовком `Authorization`.

## Концепты

1. OAuth2 и OIDC: основы
2. Модель Keycloak: realm, client, roles
3. Потоки: authorization code + PKCE, client credentials
4. JWT, валидация и refresh
