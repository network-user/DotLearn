# Вы всегда где-то стоите: первые шаги в терминале

Коллега присылает инструкцию: «откройте терминал, перейдите в каталог проекта и запустите скрипт». Вы открываете чёрное окно, видите мигающий курсор и понимаете, что щёлкать здесь не по чему. Ни папок с иконками, ни адресной строки, ни кнопки «назад». Любая опечатка выглядит угрожающе, и главное, совершенно непонятно, куда вы вообще попали.

Хорошая новость: для уверенной навигации хватает трёх команд. `pwd` отвечает на вопрос «где я», `ls` показывает, что лежит вокруг, `cd` перемещает в другое место. К концу урока вы будете ходить по файловой системе быстрее, чем в графическом проводнике, и перестанете бояться потеряться. Потеряться здесь попросту невозможно.

## Терминал и шелл: с кем вы разговариваете

Сначала два слова о собеседнике. Терминал это окно, которое умеет показывать текст и принимать ввод с клавиатуры. Внутри него работает шелл, программа-переводчик, обычно bash или zsh: вы набираете строку, нажимаете Enter, шелл разбирает команду и выполняет её. Больше о внутренностях пока знать не нужно. Все команды урока вы вводите после приглашения шелла; в примерах оно обозначено символом `$`, набирать его не надо.

## Одно дерево на всю систему

В Windows файлы живут на дисках `C:` и `D:`, и у каждого диска своё дерево. В Linux дерево одно на всю систему. Оно растёт из корня, каталога с именем `/`, и у каждого файла есть адрес, который читается от корня. Запись `/home/anna/docs/notes.txt` означает: в корне лежит каталог `home`, в нём `anna`, в нём `docs`, а в нём файл `notes.txt`. Такой адрес называется путём.

Несколько каталогов первого уровня стоит узнавать в лицо. В `/home` живут домашние каталоги пользователей: у пользователя anna это `/home/anna`, и почти вся ваша работа будет происходить внутри него. В `/etc` лежат настройки системы, в `/var` изменяемые данные вроде логов, `/tmp` служит складом временных файлов, а в `/usr/bin` находятся сами программы, включая `ls`, которую вы вот-вот запустите.

## Точка «вы здесь»

Главная идея, без которой терминал кажется хаосом: шелл всегда стоит в каком-то каталоге дерева. Этот каталог называется текущим, и все команды по умолчанию смотрят именно на него. На рис. 1 текущий каталог подсвечен: это `/home/anna/docs`. Если вы не знаете, где стоите, спросите:

```bash
$ pwd
/home/anna
```

`pwd` печатает абсолютный путь текущего каталога. Запомните её первой: она ничего не меняет, поэтому безопасна в любой ситуации и мгновенно снимает вопрос «где я».

<Callout type="tip" title="Приглашение подсказывает">
  Во многих системах приглашение шелла уже содержит текущий каталог, например `anna@laptop:~/docs$`. Значок `~` в нём означает домашний каталог.
</Callout>

## Осмотреться: ls

`ls` перечисляет содержимое текущего каталога:

```bash
$ ls
docs  music  otchet.txt
```

По умолчанию имена укладываются в несколько колонок. Ключ `-1` печатает по одному имени в строке. Ключ `-l` добавляет подробности:

```bash
$ ls -l
drwxr-xr-x 2 anna anna 4096 июл  8 10:20 docs
-rw-r--r-- 1 anna anna  532 июл  9 18:03 otchet.txt
```

Пока читайте этот вывод поверхностно. Первая буква строки различает каталог (`d`) и обычный файл (`-`), дальше идут права доступа, владелец, размер в байтах, дата изменения и имя. Права подробно разберёт отдельный урок; сейчас достаточно уметь отличать файлы от каталогов.

## Перемещаться: cd

`cd` меняет текущий каталог. Путь к цели записывается двумя способами, и разница между ними принципиальная. Абсолютный путь начинается с `/`, читается от корня и работает из любого места:

```bash
$ cd /home/anna/docs
$ pwd
/home/anna/docs
```

Рис. 2 разбирает такой путь по сегментам: каждый следующий каталог уточняет предыдущий, и цепочка всегда начинается с корня.

Относительный путь начинается не с `/` и отсчитывается от текущего каталога. Стоя в `/home/anna`, в тот же `docs` вы попадёте гораздо короче:

```bash
$ cd docs
```

У относительных путей есть два служебных имени: `..` означает родительский каталог, `.` текущий. Команда `cd ..` поднимает на один уровень вверх, из `/home/anna/docs` обратно в `/home/anna`. Работают точки в любом месте дерева, потому что записи `.` и `..` есть в каждом каталоге.

Ещё три сокращения экономят время каждый день. Символ `~` разворачивается в ваш домашний каталог, поэтому `cd ~/docs` приводит в `/home/anna/docs` откуда угодно. `cd` без аргументов тоже ведёт домой. А `cd -` возвращает туда, где вы были до последнего перехода, и заодно печатает этот путь:

```bash
$ pwd
/home/anna
$ cd /var/log
$ cd -
/home/anna
```

Последний инструмент навигации не команда, а клавиша. Начните набирать имя каталога и нажмите Tab: шелл допишет его сам. Если подходящих имён несколько, второе нажатие Tab покажет список. Автодополнение не только ускоряет набор, оно страхует от опечаток: раз Tab ничего не дописывает, такого имени в текущем каталоге, скорее всего, нет.

## Что вас удивит

Первый сюрприз: `ls` показывает не всё. Файлы, чьи имена начинаются с точки, считаются скрытыми. Так прячутся служебные настройки вроде `.bashrc` в домашнем каталоге: файл существует, но обычный `ls` молчит о нём. Ключ `-a` раскрывает картину целиком:

```bash
$ ls -a
.  ..  .bashrc  docs  music  otchet.txt
```

Заодно стали видны те самые `.` и `..`, которые живут в каждом каталоге.

Второй сюрприз: пробел разделяет аргументы. Каталог с именем `my docs` командой `cd my docs` открыть не получится: шелл решит, что вы передали два отдельных имени. Возьмите имя в кавычки, `cd "my docs"`, или доверьте набор клавише Tab: автодополнение само экранирует пробел.

Третий сюрприз ждёт пришедших из Windows: имена регистрозависимы. `Report.txt` и `report.txt` это два разных файла, и они спокойно живут в одном каталоге. Если `cd Docs` жалуется на отсутствие каталога, а `ls` показывает `docs`, дело именно в регистре.

## Типичные ошибки

Путают относительный путь с абсолютным. `cd docs` сработает, только если `docs` лежит в текущем каталоге; из любого другого места придёт ответ `No such file or directory`. Так выходит потому, что относительный путь молча приклеивается к текущему каталогу. Лекарство: сначала `pwd`, потом либо перейдите в нужное место, либо укажите абсолютный путь.

Пишут абсолютный путь там, где хватает короткого относительного: стоя в `/home/anna`, набирают `cd /home/anna/docs` вместо `cd docs`. Формально не ошибка, но лишний набор и лишние шансы на опечатку. Причина обычно в недоверии к текущему каталогу; взгляд на `pwd` возвращает уверенность.

Забывают кавычки на именах с пробелами и получают загадочные жалобы на несуществующие файлы. Правило простое: имя с пробелом всегда берите в кавычки.

Паникуют, «потерявшись». Состояния «неизвестно где» в терминале не существует: `pwd` мгновенно называет ваше место, а `cd` без аргументов гарантированно возвращает домой. Обе команды безопасны, начинайте с них любое разбирательство.

<Detail summary="Почему cd встроена в шелл, а не лежит в /usr/bin">
  `ls` и `pwd` это обычные программы из `/usr/bin`. С `cd` такой фокус не прошёл бы: текущий каталог принадлежит конкретному процессу. Отдельная программа сменила бы каталог самой себе, завершилась, и шелл остался бы где стоял. Поэтому смену каталога шелл выполняет сам, и справка по `cd` живёт в документации шелла, а не в отдельной man-странице программы.
</Detail>
