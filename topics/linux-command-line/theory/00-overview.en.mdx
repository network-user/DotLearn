A black window with a blinking cursor looks hostile right up until you learn a dozen commands. After that it turns into the fastest tool you own: a server over ssh, a container with no GUI, a CI script, your home router. The same terminal greets you everywhere, and the same tricks work everywhere. This topic doesn't try to cover "all of Linux" - just the minimum that admins and developers reach for every single day.

**What you will learn to do.** Know where you are in the filesystem and move around it; create, copy, move, and delete files without fear of wiping the wrong thing; read a permission string like `rwxr-xr-x` and change permissions with `chmod`; chain small utilities into pipelines with pipes and redirection; find what you need with `grep` and `find`; see which processes are running and stop them cleanly; and understand where the shell's environment variables come from and how it locates commands through `PATH`.

**Topic route.** Seven sections, from "where am I" to how the environment works:

1. **Filesystem and navigation**: the directory tree, `pwd`, `ls`, `cd`.
2. **Working with files**: `mkdir`, `touch`, `cp`, `mv`, `rm`, `cat`.
3. **Permissions**: rwx triads and `chmod`.
4. **Pipes and redirection**: streams and the `>`, `>>`, `|` operators.
5. **Search**: `grep` inside files, `find` by attributes.
6. **Processes**: `ps`, `top`, signals, and `kill`.
7. **Environment variables and PATH**: `export` and how commands get found.

**What you need going in.** Nothing but a willingness to open a terminal. Every session in these lessons is deterministic: the file tree is fixed, so you learn to predict what a command will print and then check yourself. The best way to make it stick is to run the commands yourself in any terminal you have handy - Linux, macOS, or WSL on Windows all work.
