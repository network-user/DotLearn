# Without nonlinearity, a deep network collapses to one layer

In the previous section the neuron ended with a strange number `z` that could be anything: -100 or +1000. An activation function turns it into a meaningful answer. But it has a bigger job than cosmetics: activation is what lets a network learn anything more complex than a straight line. Remove it and a stack of a hundred layers computes exactly what one layer would. In this section you meet four workhorse activations (sigmoid, tanh, ReLU, softmax), understand why nonlinearity matters, and learn to pick a function for the task.

## Why nonlinearity at all

A network layer is input times a weight matrix plus bias: a linear transform. Put two linear layers back to back without activation between them and you can multiply them ahead of time and replace them with one: the composition of linear functions is linear again. No matter how many layers you stack, without activation the network stays linear and can only do what one neuron can: draw a straight boundary.

Activation breaks that linearity: a bend is inserted between layers, and only then does adding depth start to buy new capability. The plot shows how each function bends the line in its own way, and that bend accumulates layer by layer. Switch functions in the widget and compare the shapes.

## Four workhorse functions

**Sigmoid** squeezes any number into 0..1, so its output reads as a probability. The formula `1 / (1 + exp(-z))`: for large negative `z` the output nears 0, for large positive `z` it nears 1, at zero exactly 0.5. Its main problem is saturation: at the edges the curve is nearly flat, the derivative is near zero, and the training signal fades.

**Tanh** has the same S-shape but range -1..1 and center at zero. Centering around zero usually makes it more convenient than sigmoid for hidden layers, though saturation at the edges remains.

**ReLU** (rectified linear unit) is the most used activation in modern hidden layers. The rule is simple: `max(0, z)`. Negative values become zero; positive values pass through unchanged. ReLU does not saturate on the positive side, is cheap to compute, and passes gradient well, which is why deep networks became trainable in practice once ReLU spread.

```python
def relu(z):
    return max(0, z)

print(relu(3.0), relu(-2.0), relu(0.0))
```

This prints `3.0 0 0.0`: positive passed, negative zeroed. ReLU has one downside, **dead neurons**: if the input stays negative, output is always zero, gradient is zero too, and the neuron stops learning. Variants like Leaky ReLU address this with a non-zero negative side.

## Softmax: activation for choosing one of many

The first three functions work on one number. **Softmax** works on a whole vector and belongs on the output of a classifier when you must pick one class from several. It turns a vector of raw numbers (called logits) into probabilities: each value is positive and the whole sum is exactly 1.

```python
from math import exp

def softmax(zs):
    exps = [exp(z) for z in zs]
    total = sum(exps)
    return [e / total for e in exps]

print(softmax([2.0, 1.0, 0.1]))
```

This prints roughly `[0.659, 0.242, 0.099]`: sum equals 1, and the largest logit gets the largest probability. Softmax amplifies gaps: even a small lead in logits becomes a noticeable lead in probability because the exponential grows fast.

## Try it yourself

Run the code: it computes softmax for three logits and checks the sum equals 1. Now make the first logit much larger (for example, `5.0`) and watch its probability jump toward one while the rest nearly vanish. Then make all three logits equal and confirm softmax returns equal shares.

## When behavior surprises you

The naive softmax above overflows on large logits: `exp(1000)` is infinity and division breaks. In practice you subtract the maximum logit from all logits before exponentiating; the result is the same (proportions do not change) but numbers stay finite. This is the standard "stable softmax" trick, already built into libraries.

A second surprise is confusing sigmoid and softmax on the output. For choosing **one** class from several use softmax (probabilities compete, sum equals 1). For several independent labels ("the image has a cat", "the image has a dog", both can be true) use a separate sigmoid on each output.

## Common mistakes

- **Activation on the output layer of regression.** If the network predicts price or temperature (any real number), the last layer needs no activation: sigmoid there would squeeze the answer into 0..1 and break prediction. Activations sit between layers and on classifier outputs, not on regression outputs.
- **Sigmoid in every hidden layer of a deep network.** Saturation makes gradient fade as you go deeper, and lower layers barely learn. Hidden layers default to ReLU.
- **Forgetting normalization in softmax.** If you divide exponentials by something other than their sum, the vector stops being a probability distribution. Softmax output must sum to 1.

<Detail summary="Why ReLU helped deep learning so much">

Sigmoid and tanh derivatives at the edges approach zero. During training gradient flows from output to input, multiplying at each layer (covered in the backpropagation section). Multiplying many numbers less than one quickly gives nearly zero: the vanishing gradient problem that kept deep networks from training until the mid-2000s. ReLU's derivative on the positive side is exactly 1; it does not shrink gradient, and the signal reaches lower layers. ReLU plus sensible weight initialization in the early 2010s made deep networks practical.

</Detail>
