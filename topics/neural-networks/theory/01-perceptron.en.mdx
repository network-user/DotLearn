# One neuron: weighted sum plus activation

Imagine you need to decide whether to approve a loan. Three numbers come in: income, age, and number of late payments. Some features matter more than others, and each pulls the decision in its own direction. A perceptron is the smallest device that can make such decisions: it takes input numbers, assigns each a weight, sums them into one number, and passes it through a simple function that produces the answer. Networks built from these units, stacked in layers, power everything from digit recognition to large language models.

By the end of this section you will compute a neuron's output by hand, explain what each weight does and why bias matters, see how the answer changes when weights shift, and write a `forward` function that repeats this arithmetic in Python.

## A neuron is a dot product, then a threshold

Inside one neuron exactly two steps happen. First, inputs are multiplied by weights and summed — the weighted sum, or dot product of the input and weight vectors. A separate number called **bias** is added: it shifts the sum up or down regardless of the inputs. That result is denoted `z`:

```
z = w1*x1 + w2*x2 + ... + wn*xn + b
```

The second step is **activation**: `z` passes through a function that maps any value to an answer in the right range. The figure shows both steps: inputs on the left converge at the sum node, each with its weight, bias is added to the sum, and only then does the result go into activation.

A weight says how strongly and in which direction an input affects the sum. A positive weight pulls `z` up as the input grows; a negative one pulls it down. A weight near zero makes the input almost irrelevant. Bias is the "default mood": even when all inputs are zero, `z` equals `b`, and the neuron can lean toward "yes" or "no" without any signal on the input.

## Computing the output by hand

Take concrete numbers. Let inputs `x = [1, 0.5, -1]`, weights `w = [0.8, -0.4, 0.3]`, and bias `b = 0.1`. Compute `z`:

```
z = 0.8*1 + (-0.4)*0.5 + 0.3*(-1) + 0.1
z = 0.8 - 0.2 - 0.3 + 0.1 = 0.4
```

You get `z = 0.4`. That is not yet the neuron's answer — it is the raw weighted sum. To turn it into an output you need activation. The best-known is **sigmoid**, which squeezes any number into the range 0 to 1:

```
sigmoid(z) = 1 / (1 + exp(-z))
```

For `z = 0.4` you get `sigmoid(0.4) ≈ 0.599`. If you read the output as a probability of "yes", the neuron leans slightly toward a positive answer. Activations are covered in detail in [[activations]]; for now what matters is that without one you would be left with `z`, which can fly off in any direction.

## Bias shifts the decision boundary

To feel the role of bias, keep inputs and weights the same but change `b`. With `b = 0.1` the sum is `0.4`; with `b = -0.5` the same inputs give `z = -0.2`, and `sigmoid(-0.2) ≈ 0.450` — the neuron tips toward "no". Weights set the slope of the boundary between "yes" and "no"; bias moves the boundary itself. Without bias the boundary always passes through zero, and the neuron could not, for example, say "yes" only above a certain income threshold.

## When behavior surprises you

A single-layer perceptron can only draw a **straight** boundary between classes. That is its hard limit: XOR (answer "yes" when exactly one of two inputs equals 1) cannot be solved by one neuron no matter how you tune the weights. You need at least two layers — the inability of a lone perceptron to learn XOR stalled the field for years in 1969. A multi-layer network gets around this, and we reach that in [[forward-pass]].

A second surprise is input scale. If one feature is measured in thousands (income) and another in ones (late payments), equal weights let income dominate simply because of magnitude. Inputs are almost always normalized before feeding a network.

## Common mistakes

- **Forgetting bias.** The sum `w·x` without `b` always goes to zero when inputs are zero, and the decision boundary is pinned to the origin. Bias is a separate learnable parameter — do not skip it.
- **Swapping weights and inputs.** In the dot product `w1` multiplies `x1` specifically. Shifting by one position (`w1*x2`) breaks the meaning silently — no error, just a wrong result.
- **Confusing `z` with the output.** `z` is the raw sum, unbounded. The output is `activation(z)`. Printing `z` where a probability is expected is a common reason for "probabilities" above one.

<Detail summary="Where the word perceptron came from">

Frank Rosenblatt proposed the model in 1958, inspired by the biological neuron: dendrites collect signals (inputs and weights), the cell body sums them, and when a threshold is exceeded the axon fires (activation). The original perceptron used a step activation — strictly 0 or 1 — so it could not be trained with gradient descent: a step has no useful derivative. The move to smooth activations like sigmoid in the 1980s made deep network training possible.

</Detail>
