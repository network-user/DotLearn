# Как сеть узнаёт, в какую сторону подвинуть каждый вес

У нас есть сеть, которая считает ответ (прямой проход), и число, говорящее, насколько она ошиблась (потеря). Осталось главное: как подправить миллионы весов, чтобы потеря упала? Перебор бессмыслен - вариантов бесконечно много. Ответ дают две идеи, работающие в паре. **Градиентный спуск** говорит, в какую сторону двигать вес: туда, где потеря убывает. **Обратное распространение** (backpropagation) эффективно считает эту сторону сразу для всех весов за один проход назад по сети. В этом разделе Вы поймёте обе и руками сделаете один шаг обновления веса.

## Спуск по склону потери

Представьте потерю как функцию от весов: меняете вес - меняется потеря. Получается рельеф, «ландшафт потерь», и наша цель - спуститься в его низину. В каждой точке у склона есть наклон - **градиент**. Он показывает направление наискорейшего **роста** потери; значит, чтобы её уменьшить, надо шагать в **противоположную** сторону. Это и есть градиентный спуск: на каждом шаге сдвигаем веса против градиента.

Размер шага задаёт **learning rate** (скорость обучения) - множитель перед градиентом. Слишком маленький - спуск ползёт целую вечность. Слишком большой - шаги перелетают через низину, и шарик скачет туда-сюда, не сходясь. Подбор learning rate - одно из главных практических решений при обучении.

Формула одного шага для одного веса:

```
w = w - learning_rate * градиент
```

Минус здесь принципиален: градиент смотрит вверх по склону, а мы хотим вниз. На двумерном рельефе путь спуска виден как линия, сбегающая к центру низины.

## Один шаг обновления руками

Возьмём один вес `w = 0.5`, learning rate `0.1` и градиент потери по этому весу `2.0` (его-то и считает обратное распространение). Делаем шаг:

```python
w = 0.5
learning_rate = 0.1
gradient = 2.0

w = w - learning_rate * gradient
print(round(w, 4))
```

Печатается `0.3`: положительный градиент означает «увеличение w поднимает потерю», поэтому шаг тянет вес вниз, к `0.3`. Будь градиент отрицательным, вес, наоборот, вырос бы. Обновление всегда движет вес против знака градиента.

## Цепное правило - сердце обратного распространения

Откуда берётся градиент потери по конкретному весу, спрятанному глубоко в сети? Потеря зависит от весов не напрямую, а через цепочку: вес -> сумма нейрона -> активация -> следующий слой -> ... -> потеря. **Цепное правило** из анализа говорит, как перемножить локальные производные вдоль этой цепочки, чтобы получить общую. Обратное распространение применяет его эффективно: оно идёт от потери назад к входу, на каждом слое умножая приходящий сверху градиент на локальную производную слоя и передавая результат дальше вниз. За один проход назад так считаются градиенты сразу всех весов.

Ключевая мысль: backprop не выдумывает новую математику - это аккуратное переиспользование уже посчитанных промежуточных значений прямого прохода и перемножение производных по цепочке. Без него пришлось бы считать производную по каждому весу отдельно, что для миллионов весов нереально.

## Считаем градиент численно

Точную производную даёт цепное правило, но её можно и приблизить численно, чтобы прочувствовать смысл: градиент - это «насколько изменится потеря, если чуть-чуть тронуть вес». Берём малый сдвиг `h`, смотрим, как меняется функция, и делим:

```python
def numeric_gradient(f, x, h=1e-5):
    return (f(x + h) - f(x - h)) / (2 * h)

print(round(numeric_gradient(lambda x: x ** 2, 3.0), 4))
```

Для `f(x) = x^2` в точке 3 производная равна `2*3 = 6`, и численная оценка даёт `6.0`. Этот приём (двусторонняя разность) - не то, чем считают градиенты на практике (это делает backprop, он быстрее и точнее), но он отлично показывает, что градиент - это чувствительность выхода к входу.

## Когда поведение удивит

Большой learning rate не просто замедляет - он способен **расходить** обучение: потеря не падает, а растёт от шага к шагу, потому что каждый шаг перелетает минимум всё дальше. Если во время обучения потеря уходит в бесконечность или становится `NaN`, первое подозрение - слишком большой learning rate.

Второй сюрприз - градиентный спуск находит **локальный** минимум, ближайший к старту, а не обязательно глобальный. У сложного рельефа потерь низин много. На практике это редко проблема: у больших сетей хорошие локальные минимумы дают достойное качество, а случайность в данных и инициализации помогает не застрять в плохом.

## Типичные ошибки

- **Перепутать знак в шаге.** `w = w + lr * gradient` гонит вес вверх по склону, и потеря растёт. Шаг всегда против градиента: `w = w - lr * gradient`.
- **Один learning rate на все случаи жизни.** Удачный шаг для одной задачи разносит другую. Learning rate подбирают, а часто и уменьшают по ходу обучения (расписание).
- **Считать, что backprop - это отдельный алгоритм обучения.** Backprop только считает градиенты; шаг по ним делает градиентный спуск (или его варианты вроде Adam). Это две разные роли, работающие в паре.

<Detail summary="Почему на практике берут не весь датасет, а батчи (SGD)">

Честный градиентный спуск считал бы градиент по всему датасету на каждый шаг - дорого и медленно. **Стохастический** градиентный спуск (SGD) оценивает градиент по небольшому случайному батчу примеров. Оценка шумная, но в среднем верная, а шагов в секунду получается в тысячи раз больше. Этот шум даже полезен: он помогает выскакивать из плохих узких минимумов. Современные оптимизаторы (Adam, AdamW) добавляют к SGD адаптивный шаг для каждого веса и инерцию (momentum), отчего обучение идёт устойчивее и быстрее.

</Detail>
