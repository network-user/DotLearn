# A network that memorized the answers is useless on new data

The goal of training is not to memorize the training set but to generalize: answer well on data the network has never seen. Those two are easy to confuse. A powerful enough network can drill the training examples to zero loss and still fail on new ones. That is **overfitting**, the main practical enemy when training neural networks. In this section you split data into train and validation, read both curves to spot overfitting, and fight it with dropout, early stopping, and related techniques.

## Train, validation, test: three non-overlapping sets

To measure generalization honestly, data is split. **Train**: what the network uses to fit weights. **Validation**: what you check during training and tune hyperparameters on (layer count, learning rate), without fitting weights to it. **Test**: the final set touched once at the very end for an unbiased score. The main rule: examples from validation and test must not appear in train, or "evaluation" becomes a memory test, not a generalization test.

Loss is computed separately on train and validation. While the network learns something useful, both fall. The moment train loss keeps falling but validation loss turns upward marks the start of overfitting: the network started fitting specific training examples at the expense of the general rule.

## What overfitting looks like in numbers

The gap between train and validation is a direct indicator. A small gap with both losses low means the network generalizes. A large gap (train near zero, validation high) means the network memorized train. A simple calculation on this gap helps diagnose.

```python
def overfit_gap(train_loss, val_loss):
    return val_loss - train_loss

print(round(overfit_gap(0.04, 0.78), 4))
print(round(overfit_gap(0.30, 0.32), 4))
```

This prints `0.74` and `0.02`: the first pair is clear overfitting (huge gap), the second is healthy training (losses close). Low train loss alone says nothing about quality: look at validation and the gap.

## Fighting overfitting

**More data** is the most reliable fix: the more diverse the train set, the harder to memorize and the more the network must generalize. When you cannot add data, use augmentation (rotations, noise for images) to artificially enlarge the set.

**Dropout** during training randomly "turns off" some neurons each step (zeros their outputs). The network cannot rely on any single neuron and must learn robust, distributed features. At inference (prediction) dropout is off: all neurons work.

**Early stopping** simply ends training where validation loss stopped improving. That is the inflection point on the plot above: training further means overfitting.

**L2 regularization** adds a penalty for large weights to the loss, pushing them toward zero. Smaller weights give a smoother, simpler function that is harder to overfit.

## When behavior surprises you

The mirror problem is **underfitting**: the network is too weak or trained too little, and **both** train and validation loss stay high. Regularization only hurts here: you need the opposite, namely more parameters, longer training, less dropout. Diagnose from the pair of curves: high train loss means underfitting; low train with high validation means overfitting.

A second surprise: dropout is on only during training. If you leave it active at inference, predictions become random from run to run because different neurons are disabled each time. Frameworks switch modes automatically (`model.train()` versus `model.eval()` in PyTorch), but forgetting to switch is a classic mistake.

## Common mistakes

- **Validation leaking into train.** If the same (or nearly the same) examples appear in both train and validation, quality looks inflated and real generalization is worse than expected. The split must be strict.
- **Tuning hyperparameters on test.** If you adjust layer count and learning rate while watching test, test stops being an honest final measure. Use validation for tuning; touch test once.
- **Treating underfitting with regularization.** High train loss means insufficient capacity or training, not overfitting. Dropout and L2 only make it worse.

<Detail summary="Why dropout works at all">

Randomly disabling neurons can be understood as training a huge ensemble of "thinned" subnetworks with shared weights: each step activates a different random subnetwork. At inference all neurons are on, which approximately averages predictions across those subnetworks. Ensembles historically generalize better than single models, and dropout delivers a similar effect almost for free. Typical dropout rate is 0.2–0.5; too high blocks learning entirely, too low barely regularizes.

</Detail>
