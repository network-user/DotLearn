# Слой - это пачка нейронов, считаемая одним умножением матриц

Один нейрон умеет мало. Сила сети - в том, что нейроны выстроены в слои, и выход одного слоя становится входом следующего. Этот путь данных от входа к ответу называется **прямым проходом** (forward pass). Хорошая новость: ничего нового по сравнению с одним нейроном здесь нет - это та же взвешенная сумма плюс активация, только посчитанная для всех нейронов слоя сразу одним умножением матриц. В этом разделе Вы разберёте, как слой превращает вектор входа в вектор выхода, соберёте многослойный персептрон (MLP) и руками прогоните через него число.

## От нейрона к слою: одна матрица вместо стопки векторов

В прошлых разделах нейрон считал `z = w·x + b` - одно число из вектора входа и вектора весов. В слое таких нейронов несколько, и у каждого свой вектор весов и свой bias. Если сложить векторы весов всех нейронов в строки матрицы `W`, то выходы всего слоя получаются одним действием:

```
z = W @ x + b
a = activation(z)
```

Здесь `x` - вектор входа, `W` - матрица весов (строка на каждый нейрон), `b` - вектор смещений, `@` - матричное умножение, `a` - вектор выходов слоя после активации. Размеры обязаны сходиться: если на входе 3 числа, а в слое 4 нейрона, то `W` имеет форму `4 x 3`, и на выходе получаются 4 числа. На рисунке видно, как сигнал зажигается слой за слоем - каждый следующий слой питается выходом предыдущего.

## Многослойный персептрон (MLP)

MLP - это несколько полносвязных слоёв подряд: каждый нейрон слоя соединён со всеми выходами предыдущего. Между слоями стоит нелинейная активация (иначе, как мы помним, всё схлопнется в один слой). Считаем по цепочке: выход первого слоя становится входом второго, и так до конца.

```python
def layer(x, weights, biases):
    z = []
    for neuron_weights, b in zip(weights, biases):
        s = sum(xi * wi for xi, wi in zip(x, neuron_weights))
        z.append(s + b)
    return z
```

Здесь `weights` - список векторов весов (по одному на нейрон), `biases` - список смещений. Функция возвращает вектор сырых сумм слоя `z`. Дальше к нему применяют активацию и подают результат в следующий слой.

```python
def relu(vec):
    return [max(0.0, v) for v in vec]

x = [1.0, 2.0]
W1 = [[0.5, -0.5], [1.0, 0.0]]
b1 = [0.0, 1.0]

hidden = relu(layer(x, W1, b1))
print(hidden)
```

Первый нейрон даёт `0.5*1 + (-0.5)*2 + 0 = -0.5`, после ReLU - `0.0`. Второй даёт `1.0*1 + 0.0*2 + 1 = 2.0`, ReLU оставляет `2.0`. Печатается `[0.0, 2.0]` - это вход для следующего слоя.

## Поток размерностей

Полезно держать в голове, как меняется длина вектора по мере прохода. Вход длины 2 пришёл в слой из 2 нейронов - вышел вектор длины 2. Подайте его в слой из 3 нейронов - выйдет вектор длины 3. Число нейронов в слое задаёт длину его выхода, а длина входа должна совпадать с числом весов у каждого нейрона. Несостыковка размерностей - самая частая причина падения forward-прохода.

Отсюда же видно, зачем скрытым слоям делают «широкое горло»: сеть может принять вектор из 2 чисел, раздуть его до 64 во внутренних слоях и снова сжать до 2 на выходе. Промежуточная размерность - это запас места, в котором сеть раскладывает признаки; больше нейронов в скрытом слое означает больше выразительности, но и больше параметров для обучения. Выбор этих размеров (ширина слоёв и их число) - часть проектирования архитектуры, а не данных.

## Когда поведение удивит

Прямой проход - это только вычисление ответа, в нём нет обучения. Веса берутся как есть и не меняются: чтобы они стали осмысленными, нужен обратный проход и градиентный спуск (раздел [[backprop-gradient-descent]]). Свежая, случайно инициализированная сеть честно посчитает forward-проход и выдаст полную чепуху - это нормально.

Второй сюрприз - порядок «сумма, потом активация» нельзя нарушать внутри слоя, но активацию **последнего** слоя выбирают отдельно от скрытых. Скрытые слои обычно используют ReLU, а выходной - softmax (классификация) или вообще без активации (регрессия). Поставить ReLU на выход классификатора - распространённая ошибка.

## Типичные ошибки

- **Перепутать форму матрицы весов.** Для входа длины `n` и слоя из `m` нейронов матрица должна быть `m x n` (строка на нейрон). Транспонированная форма `n x m` даст ошибку размерности или, хуже, тихо неверный результат.
- **Забыть активацию между слоями.** Два слоя `layer(layer(x))` без активации посередине эквивалентны одному. Нелинейность ставится после каждого скрытого слоя, иначе глубина бесполезна.
- **Применить активацию выходного слоя к скрытым по инерции.** softmax уместен только на выходе классификатора; внутри сети между слоями он не нужен и мешает.

<Detail summary="Почему на практике считают батчами">

В примере мы прогоняли один вектор `x`. На практике сеть обрабатывает сразу пачку примеров - **батч**. Тогда вход становится матрицей `X` (строка на пример), и весь слой считается как `X @ W.T + b` одним умножением матриц для всего батча. Это и быстрее (одно крупное матричное умножение вместо цикла по примерам), и нужно для обучения: градиент усредняют по батчу. Библиотеки вроде PyTorch и сделаны вокруг такого матричного представления, поэтому ускорители (GPU) загружены полезной работой.
