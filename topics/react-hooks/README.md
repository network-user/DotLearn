# React-хуки

Глубокий разбор хуков React для тех, кто уже пишет компоненты, но путается в перерисовках, зависимостях и замыканиях. Тема идёт от правил хуков к деталям каждого встроенного хука и заканчивается самой частой болью новичка: устаревшими замыканиями и неправильным массивом зависимостей. Разбираем `useState` с функциональными апдейтами и батчингом, `useEffect` с очисткой и порядком срабатывания, `useRef`, мемоизацию через `useMemo` и `useCallback`, `useContext`, `useReducer` и написание собственных хуков.

Тема концептуальная (`runtime: none`): код React нельзя выполнить прямо в песочнице, поэтому упор сделан на понимание модели рендеринга. Упражнения проверяют предсказание вывода при перерисовке, сигнатуры хуков, массивы зависимостей и правила хуков.

## Концепты

1. **Правила хуков** - почему хуки вызывают только на верхнем уровне и только из компонентов и других хуков.
2. **useState** - функциональные апдейты, батчинг, ленивая инициализация, снимок состояния на рендер.
3. **useEffect** - когда срабатывает эффект, массив зависимостей, функция очистки и порядок вызовов.
4. **useRef** - мутабельная ячейка, которая переживает рендеры и не вызывает перерисовку, и ссылки на DOM.
5. **useMemo и useCallback** - мемоизация значений и функций, стабильные ссылки, когда она реально нужна.
6. **useContext** - общее состояние без проброса пропсов через каждый уровень.
7. **useReducer** - состояние как набор переходов, когда он лучше useState.
8. **Кастомные хуки** - вынос логики с состоянием в переиспользуемую функцию.
9. **Массив зависимостей и замыкания** - устаревшие замыкания, пропущенные зависимости и как их чинить.
