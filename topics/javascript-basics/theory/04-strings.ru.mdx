# Строки: текст, с которым работает программа

Анна собирает для интернет-магазина текст подтверждения заказа: «Спасибо за заказ, Анна! Итого: 1500 ₽». Первая версия склеена плюсом: `'Спасибо за заказ, ' + name + '! Итого: ' + price + ' ₽'`. Код работает, пока однажды `price` не приходит числом 1500, а к нему прибавляется скидка 200 тем же плюсом: `'Итого: ' + price + discount`. Вместо вычитания получается склейка `'Итого: 1500200'`. Плюс в JavaScript ведёт себя по-разному со строками и числами, и это единственная причина, по которой безобидная на вид строка ломает сумму заказа.

В этой главе вы разберётесь, чем строка отличается от числа при использовании `+`, освоите шаблонные литералы, которые решают задачу Анны без единого плюса, и запомните горстку методов, которыми строки читают и режут. Заодно узнаете, почему `str[0] = 'Ы'` в JavaScript ничего не меняет.

## Строка как пронумерованная лента символов

Строка выглядит одним куском текста, но внутри устроена как лента символов, и у каждого символа есть номер. Нумерация начинается с нуля, как и у массива из предыдущей главы. У строки `'Привет'` буква «П» лежит под номером 0, а последняя буква «т» под номером 5, потому что символов шесть, а счёт начинается с нуля.

Символ по номеру достают квадратными скобками, точно как элемент массива: `word[0]` вернёт «П», `word[word.length - 1]` вернёт последний символ независимо от длины строки. Свойство `length` считает количество символов, включая пробелы: у `'привет мир'` оно равно 10.

<PullQuote>Строка ведёт себя как ряд пронумерованных ячеек: тот же индекс от нуля, то же свойство length, что и у массива.</PullQuote>

## Строки неизменяемы

Раз строка похожа на массив, кажется естественным попробовать заменить один символ через индекс:

```js
const word = 'кот';
word[0] = 'Р';
console.log(word);
```

В консоли всё равно печатается `'кот'`. Строки в JavaScript неизменяемы: ни один существующий символ поменять нельзя. В обычном скрипте и в консоли присваивание по индексу молча игнорируется, а в строгом режиме (`'use strict'` или ES-модуль, то есть дефолт современного кода) та же строка бросает TypeError. Саму строку не меняет ни один из режимов. Единственный способ получить другой текст, это собрать новую строку и положить её в переменную заново:

```js
let word = 'кот';
word = 'Р' + word.slice(1);
console.log(word);
```

Здесь `word` не изменился, а был полностью заменён новым значением. Эта деталь объясняет, почему все методы строк ниже возвращают новую строку вместо того, чтобы менять исходную: изменять им попросту нечего.

## Плюс со строками: конкатенация против сложения

Оператор `+` перегружен: между двумя числами он складывает, а если хотя бы один операнд строка, он склеивает текст, предварительно превратив число в строку. Из-за этого правила порядок операндов решает исход:

```js
console.log('Итого: ' + 100 + 50);
console.log('Итого: ' + (100 + 50));
console.log(100 + 50 + ' ₽');
```

Первая строка печатает `'Итого: 10050'`: `'Итого: ' + 100` уже даёт строку, и следующий `+ 50` дальше только приклеивает текст «50». Вторая строка со скобками печатает `'Итого: 150'`, потому что сложение внутри скобок выполняется первым, пока оба операнда ещё числа. Третья строка печатает `'150 ₽'` по той же причине: числа складываются друг с другом раньше, чем в дело вступает строка. Именно на этой асимметрии и споткнулась Анна из вступления.

<Callout type="warning" title="+ не различает намерение">
JavaScript не спрашивает, что вы хотели: сложить числа или склеить текст. Он смотрит на типы операндов слева направо. Если ожидаете сумму, а рядом есть строка, оборачивайте числовую часть в скобки или считайте её в отдельной переменной до склейки.
</Callout>

## Шаблонные литералы: минус все плюсы

ES6 добавил запись через обратные кавычки, которая решает задачу Анны без единого `+`. Внутри `${...}` можно подставить любое выражение, а движок сам превратит его в строку в правильном месте:

```js
const name = 'Анна';
const price = 1500;
const discount = 200;

const message = `Спасибо за заказ, ${name}! Итого: ${price - discount} ₽`;
console.log(message);
```

Печатается `'Спасибо за заказ, Анна! Итого: 1300 ₽'`. Вычитание `price - discount` находится внутри `${}`, а значит оба операнда остаются числами до самого конца, и никакая соседняя строка не превращает их в текст раньше времени. Подставлять можно не только переменные, но и вызовы функций, и целые выражения: `${price > 1000 ? 'дорого' : 'недорого'}`.

У шаблонных литералов есть и вторая полезная особенность: внутри них перенос строки работает буквально, без специального символа `\n`:

```js
const receipt = `Заказ №42
Итого: 1300 ₽`;
console.log(receipt);
```

Обычная строка в одинарных или двойных кавычках такой перенос не разрешает: если внутри неё нажать Enter, JavaScript выдаст синтаксическую ошибку, и перенос придётся вставлять символом `\n`.

## Методы, которые часто нужны

Все методы ниже строку не меняют, а возвращают новую. Значения удобно проверить по цепочке `console.log`.

```js
const raw = '  Привет, Мир  ';

console.log(raw.trim());
console.log(raw.trim().toUpperCase());
console.log(raw.trim().toLowerCase());
console.log(raw.includes('Мир'));
console.log(raw.trim().slice(0, 7));
console.log(raw.trim().split(', '));
console.log(raw.trim().replace('Мир', 'JavaScript'));
```

`trim()` убирает пробелы по краям, но не внутри строки, и печатает `'Привет, Мир'`. `toUpperCase()` и `toLowerCase()` меняют регистр всех символов. `includes('Мир')` возвращает `true`, если подстрока встречается хоть раз, и не подсказывает, где именно. `slice(0, 7)` вырезает символы с номера 0 по номер 6 включительно: конец диапазона в `slice` не входит в результат, поэтому длина вырезанного куска равна разности границ, семь минус ноль. `split(', ')` режет строку по разделителю и возвращает массив: `['Привет', 'Мир']`. `replace('Мир', 'JavaScript')` меняет первое совпадение подстроки и отдаёт новую строку целиком.

<MarginNote label="Соседние методы">`slice` умеет и отрицательные индексы: `text.slice(-3)` вернёт последние три символа. Метод `indexOf('Мир')` вместо true/false возвращает номер первого вхождения подстроки или -1, если её нет.</MarginNote>

## Ловушка: конец slice не входит в результат

Границы `slice` путают почти всех новичков, потому что интуиция подсказывает включать оба конца:

```js
const word = 'JavaScript';
console.log(word.slice(0, 4));
console.log(word.length);
```

Печатается `'Java'`, а не `'JavaS'`. Символ с номером 4, буква «S», в вырезанный кусок не попадает: второй аргумент `slice` это граница, до которой режут, а не номер последнего включённого символа. Проверочный приём тот же, что и с длиной массива: если нужны первые `n` символов, второй аргумент `slice` равен ровно `n`, потому что счёт символов внутри диапазона начинается с нуля.

## Частые ошибки

Первая ошибка: попытка изменить строку через индекс, как в примере с `word[0] = 'Р'` выше. Вне строгого режима присваивание молча не срабатывает: JavaScript не бросает ошибку, а просто игнорирует строку (в строгом режиме и в ES-модулях честнее: там будет TypeError). Если нужно поменять один символ, соберите новую строку через `slice` и конкатенацию либо шаблонный литерал.

Вторая ошибка: сложение числа со строкой там, где ожидалось арифметическое сложение, разобранное в разделе про `+` выше. Она особенно коварна в подсчёте сумм и счётчиков: `total = total + input`, если `input` пришёл строкой из поля ввода, склеит текст вместо того чтобы прибавить число.

Третья ошибка: `includes` вместо `indexOf` там, где важна позиция подстроки, а не сам факт совпадения:

```js
const url = 'https://example.com/page';
console.log(url.includes('page'));
console.log(url.indexOf('page'));
```

`includes` отвечает только true или false, а `indexOf` дополнительно называет номер символа, с которого начинается совпадение, здесь 20. Если дальше по коду нужно вырезать текст после найденного слова через `slice`, без номера позиции не обойтись, а `includes` его не даёт.

<Detail summary="Сравнение строк через == и ===">
Строки сравниваются посимвольно: `'кот' === 'кот'` даёт `true`, а `'кот' === 'Кот'` даёт `false`, потому что регистр учитывается. Используйте `===`, а не `==`: они совпадают для двух строк, но `==` дополнительно приводит типы, и `'' == false` неожиданно окажется `true`. Привычка всегда писать `===` избавляет от необходимости держать эти правила приведения в голове.
</Detail>
