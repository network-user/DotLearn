# Код, который принимает решения: условия и циклы

Представьте страницу оплаты в интернет-магазине, который обещает скидку 10 процентов на заказы от 1000 рублей. Анна собрала корзину на 1450 рублей, Борис на 990. Странице придётся посмотреть на сумму и повести себя по-разному: Анне показать цену со скидкой, Борису обычную. Заранее, в момент написания кода, вы не знаете, какой вариант понадобится: решение принимается во время выполнения, по живым данным.

А вечером тот же магазин рассылает письма всем, кто оформил заказ за день. Заказов может быть три, а может тридцать тысяч, и писать тридцать тысяч одинаковых строк никто не станет. Две способности, выбирать ветку и повторять действие, превращают программу из калькулятора в систему с поведением. К концу этой главы вы научите JavaScript и тому и другому.

## Развилка: один вопрос, два пути

В основе каждого решения лежит вопрос, на который можно ответить только «да» или «нет». Сумма не меньше 1000? Поле с именем заполнено? JavaScript вычисляет такой вопрос до одного из двух значений, `true` или `false`, а конструкция `if` выбирает по нему дорогу. Присмотритесь к нижнему блоку схемы: что бы ни выбрала развилка, к оплате заказ придёт в любом случае. Условие меняет маршрут, а не пункт назначения. Эта картинка переживёт любой синтаксис: развилки устроены так в каждом языке программирования, меняются только скобки вокруг них.

В коде та же развилка выглядит так:

```js
const total = 1450;
if (total >= 1000) {
  console.log('Скидка 10%');
} else {
  console.log('Без скидки');
}
```

`if` принимает условие в круглых скобках и блок кода в фигурных. Если условие истинно, выполняется первый блок; иначе блок после `else`. Сам `else` необязателен: без него при ложном условии не произойдёт ничего, и программа пойдёт дальше.

Когда вариантов больше двух, ветки выстраивают в цепочку через `else if`:

```js
if (total >= 5000) {
  console.log('Скидка 15%');
} else if (total >= 1000) {
  console.log('Скидка 10%');
} else {
  console.log('Без скидки');
}
```

Проверки идут сверху вниз, и выигрывает первая истинная: для суммы 6200 сработает только ветка с 15 процентами, хотя условие `total >= 1000` тоже истинно. Поэтому порядок веток имеет значение: начинайте с самого строгого условия, иначе до щедрой скидки выполнение никогда не доберётся.

## Сравнения: почему всегда `===`

Внутри условий живут операторы сравнения. Для чисел они выглядят знакомо по школьной математике: `>` и `<` строгие, `>=` и `<=` нестрогие, «больше или равно» и «меньше или равно». А вот с равенством у JavaScript особые отношения: операторов два, `==` и `===`, и разница между ними стоила разработчикам многих ночных отладок.

Полезная деталь: результат сравнения ведёт себя как обычное значение. Выражение `total >= 1000` вычисляется в `true` или `false` ещё до того, как `if` его увидит, поэтому результат можно сохранить в переменную с говорящим именем и подставить в условие позже. Код от этого начинает читаться как связный текст.

Нестрогое `==` перед сравнением пытается привести значения к одному типу. Звучит услужливо, работает коварно:

```js
console.log(5 == '5');
console.log('' == 0);
console.log(5 === '5');
console.log('' === 0);
```

Первые две строки печатают `true`: число 5 и строка `'5'` «как бы равны», а пустая строка «как бы равна» нулю. Вторая пара печатает `false`, потому что строгое `===` первым делом смотрит на типы: число и строка различаются уже на этом шаге, и никакого приведения не происходит. Теперь представьте проверку промокода, где пустая строка из незаполненного поля внезапно совпала с числом 0 из базы. Такие ошибки не падают с грохотом, они тихо пропускают не те данные, и искать их приходится днями.

<MarginNote label="История">Первую версию JavaScript Брендан Айк собрал за десять дней в 1995 году. Нестрогое `==` задумывалось как удобство для веб-форм, откуда все данные приходят строками.</MarginNote>

Отсюда правило, которое не обсуждается: сравнивайте только через `===` и `!==`. Нестрогие `==` и `!=` оставьте старому коду, который писали до вас.

## truthy и falsy: что `if` делает с нелогическими значениями

В условие можно положить не только сравнение, но и любое значение: запись `if (name)` синтаксически корректна. Перед проверкой JavaScript приведёт значение к логическому типу и как бы спросит: считать ли это «чем-то»? Значения, которые превращаются в `false`, называются falsy, и их ровно шесть:

- `false`
- `0`
- `''` (пустая строка)
- `null`
- `undefined`
- `NaN` (специальное «не число», которое получается из неудавшейся арифметики вроде `Number('привет')`)

Всё остальное truthy, то есть превращается в `true`. Этот список стоит выучить наизусть, как таблицу умножения: в повседневном коде другого falsy вы не встретите (за его пределами разве что экзотика вроде `0n` у BigInt). Практическая польза немедленная: `if (name)` читается как «если имя заполнено», потому что пустая строка falsy.

```js
const name = '';
if (name) {
  console.log('Привет, ' + name);
} else {
  console.log('Представьтесь, пожалуйста');
}
```

Для значения `''` выполнится ветка `else`, и программа вежливо попросит имя. Проверить любую догадку про truthy и falsy можно за секунду: откройте консоль браузера и вызовите `Boolean(значение)`, он возвращает ровно то, что увидит `if`.

## Цикл for: счётчик, условие, шаг

Теперь вторая способность: повторение. Классический цикл `for` держит всё управление повторами в одной строке-заголовке:

```js
for (let i = 1; i <= 3; i += 1) {
  console.log('Отправляю письмо номер ' + i);
}
```

В заголовке три части, разделённые точками с запятой. Первая выполняется один раз перед стартом и заводит счётчик: `let i = 1`. Вторая проверяется перед каждым витком: пока `i <= 3` истинно, тело выполнится ещё раз. Третья срабатывает после каждого витка и двигает счётчик: `i += 1` означает «увеличить i на единицу». Как только условие впервые оказывается ложным, цикл заканчивается, и выполнение продолжается за его закрывающей скобкой.

Пройдите трассу глазами: i равно 1, условие истинно, печать; i равно 2, печать; i равно 3, печать; i равно 4, условие ложно, стоп. Ровно три письма. Привычка проговаривать витки на маленьком примере спасает от целого класса ошибок, о котором чуть ниже.

С тремя частями заголовка можно играть независимо. Шаг `i += 2` пройдёт только по нечётным номерам, старт с `let i = 10` вместе с шагом `i -= 1` поведёт отсчёт в обратную сторону, как перед запуском ракеты. Конструкция одна, поведение любое, лишь бы условие продолжения однажды стало ложным.

## for...of: перебор списка значений

Чаще, чем «повтори N раз», встречается задача «пройди по всем элементам». Пусть у вас есть список сумм заказов за день. Об устройстве таких списков мы подробно поговорим в пятой главе; пока достаточно того, что значения перечисляются в квадратных скобках через запятую. Цикл `for...of` выдаёт элементы по одному, счётчик ему не нужен:

```js
const totals = [1450, 990, 2100, 540];
for (const total of totals) {
  if (total >= 1000) {
    console.log(total);
  }
}
```

На каждом витке в `total` попадает очередное значение, а условие внутри решает его судьбу. Рисунок ниже показывает этот конвейер: четыре заказа входят, условие пропускает дальше два.

## while: пока условие истинно

Иногда число повторов заранее неизвестно: повторять запрос, пока сервер не ответил; уменьшать долг, пока он больше нуля. Для таких случаев есть `while`, у которого из всего управления только условие:

```js
let attempts = 0;
while (attempts < 3) {
  attempts += 1;
}
console.log(attempts);
```

Условие проверяется перед каждым витком, а тело обязано рано или поздно сделать его ложным. Уберите строку `attempts += 1`, и цикл станет бесконечным: вкладка браузера замрёт, вентилятор ноутбука взвоет. Это не теоретическая страшилка, а самая частая авария новичка при работе с `while`.

Формально через `while` можно записать и рассылку писем из примера выше, эти циклы взаимозаменяемы. Но читатель вашего кода благодарен подсказкам: `for` сообщает «число повторов известно», `while` предупреждает «крутимся до события». Выбирайте цикл по смыслу, а не по привычке.

## break и continue: управление изнутри

Внутри тела цикла доступны два слова, меняющие его ход. `break` немедленно завершает весь цикл. Он полезен, когда искомое уже найдено и дальше перебирать незачем:

```js
const totals = [1450, 990, 2100, 540];
for (const total of totals) {
  if (total >= 2000) {
    console.log('Нашли крупный заказ: ' + total);
    break;
  }
}
```

`continue` мягче: он обрывает только текущий виток и передаёт управление следующему. Код ниже пропускает мелкие заказы, но из цикла не выходит:

```js
for (const total of totals) {
  if (total < 1000) {
    continue;
  }
  console.log(total);
}
```

Запомнить легко: `break` ломает цикл целиком, `continue` продолжает его со следующего элемента.

## Краевой случай: строка '0' против числа 0

Вернёмся к falsy и посмотрим на пример, который ловит даже опытных разработчиков:

```js
if ('0') {
  console.log('строка прошла');
}
if (0) {
  console.log('число прошло');
}
```

Напечатается только «строка прошла». Число 0 входит в список шести falsy значений, а правило для строк смотрит исключительно на длину: в `'0'` есть один символ, значит она truthy, что бы её текст ни означал. Подвох в том, что данные из форм и адресной строки приходят строками: пользователь ввёл ноль, вы проверили `if (value)`, и «пустая» на первый взгляд проверка пропустила его дальше. Мораль: когда речь о числах, сравнивайте явно. Запись `if (count > 0)` честнее и читается лучше, чем `if (count)`.

## Частые ошибки

**Одно `=` вместо `===` в условии.** Запись `if (total = 1000)` не сравнивает, а присваивает, и результат присваивания (1000, truthy) отправляет выполнение в ветку `if` всегда. Заказ на 990 рублей получит скидку, а заодно и испорченную сумму. Редактор подчёркивает такое предупреждением; не игнорируйте его.

**Ошибка на единицу.** Выбор между `<` и `<=` решает, случится последний виток или нет. Цикл «от 1 до n включительно» требует `<=`; машинально написанное `<` молча отрежет последний элемент. Лекарство то же, что и выше: трасса на маленьком примере.

**Бесконечный `while`.** Если тело не приближает условие к ложному, цикл не закончится никогда. Чаще всего переменную забывают менять внутри вложенной ветки `if`, которая срабатывает не на каждом витке. Перед запуском задайте себе один вопрос: какая строка тела меняет переменную из условия?

**`for...in` вместо `for...of`.** Похожий по написанию `for...in` перебирает не значения, а строковые ключи, поэтому для списка значений нужен именно `for...of`.

<Detail summary="Откуда взялось нестрогое ==">
  В ранних версиях языка `==` было единственным равенством. Оно приводило типы, потому что почти все данные на страницах девяностых приходили из форм в виде строк, и сравнение строки `'5'` с числом 5 без приведения раздражало бы авторов первых скриптов. Строгое `===` добавили позже, в третьей редакции стандарта ECMAScript в 1999 году, а `==` оставили ради совместимости со старым кодом. Так язык и живёт с двумя равенствами: удобным для 1997 года и правильным для сегодняшнего.
</Detail>

<Detail summary="Тернарный оператор: развилка в одну строку">
  Когда обе ветки лишь выбирают значение, вместо пяти строк `if/else` пишут одну: `const price = total >= 1000 ? total * 0.9 : total;`. Читается как вопрос: условие, затем значение для «да» после знака вопроса и значение для «нет» после двоеточия. Приём хорош ровно до тех пор, пока выражение помещается в голове; вложенные тернарники превращаются в ребус, и обычный `if/else` снова выигрывает.
</Detail>
