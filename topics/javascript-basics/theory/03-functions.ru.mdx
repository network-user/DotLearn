# Функции: расчёт, который живёт в одном месте

В интернет-магазине, который собирает Анна, скидка постоянного покупателя считается в трёх местах: в карточке товара, в корзине и в письме с подтверждением заказа. Формула везде одна, Анна честно скопировала её два раза. Потом правило меняется: вместо десяти процентов маркетинг просит пятнадцать. Анна правит карточку и корзину, а про письмо забывает. Неделю покупатели видят на сайте одну цену, а в письме другую, и поддержка Ozon-магазина разбирает жалобы. Ошибка случилась не потому, что Анна плохо считает проценты. Она случилась потому, что один и тот же расчёт существовал в трёх экземплярах.

Лекарство от таких историй придумано давно: записать расчёт один раз, дать ему имя и дальше обращаться к нему только по имени. В JavaScript такое именованное место называется функцией. К концу главы вы будете объявлять функции тремя способами, понимать, что именно делает `return`, и уверенно читать стрелочную запись, которая в современном коде встречается на каждом шагу.

## Коробка: аргументы входят, результат выходит

Ментальная модель функции умещается в одну картинку. Есть коробка с именем. У коробки есть входы: значения, которые вы передаёте при вызове. Внутри спрятана последовательность шагов. И есть один выход: значение, которое коробка отдаёт наружу через `return`. На рисунке ниже этот маршрут показан для расчёта скидки: числа 1000 и 10 входят внутрь, наружу выходит 900.

Важная деталь модели: снаружи не видно, что происходит внутри коробки. Вызывающий код знает только имя, входы и выход. Именно поэтому функция лечит копирование: когда правило скидки меняется, вы меняете содержимое коробки в одном месте, и карточка, корзина и письмо тут же начинают считать по-новому.

У этой закрытости есть и второе следствие: функцию можно проверять отдельно от остальной программы. Подали на вход 1000 и 10, ожидаете 900. Не нужно поднимать весь магазин, чтобы убедиться, что скидка считается верно. Чем больше логики упаковано в такие коробки, тем проще искать ошибки: вы проверяете коробки по одной, а не распутываете программу целиком.

<PullQuote>Снаружи у функции видны только имя, входы и выход; всё остальное её личное дело.</PullQuote>

## Function declaration: классическая запись

Самый старый и самый узнаваемый способ объявить функцию выглядит так:

```js
function priceWithDiscount(price, percent) {
  return price - (price * percent) / 100;
}

const total = priceWithDiscount(1000, 10);
console.log(total);
```

Теперь по частям. Слово `function` начинает объявление. Дальше идёт имя `priceWithDiscount`: по нему функцию будут вызывать. В круглых скобках перечислены параметры `price` и `percent`: локальные имена, которые существуют только внутри тела. Тело заключено в фигурные скобки, а `return` отдаёт результат наружу. Вызов `priceWithDiscount(1000, 10)` подставляет 1000 на место `price`, 10 на место `percent`, и в консоль печатается 900.

Имя функции выбирайте как короткий ответ на вопрос «что она делает»: `priceWithDiscount`, `deliveryLabel`, `sendEmail`. Хорошее имя избавляет читателя от необходимости заглядывать в тело. Если имя никак не придумывается, это ранний сигнал, что коробка пытается делать два дела сразу и её стоит разделить на две.

Здесь же стоит развести два термина, которые постоянно путают. Параметр живёт в объявлении функции: это имя, заглушка под будущее значение. Аргумент живёт в вызове: конкретное значение, которое встанет на место параметра. В примере выше `price` и `percent` параметры, а 1000 и 10 аргументы.

## return останавливает функцию немедленно

У `return` два свойства, которые стоит выучить сразу. Первое: он не просто отдаёт значение, он мгновенно завершает функцию. Код после сработавшего `return` не выполняется никогда. На этом строится удобный приём с ранним выходом:

```js
function deliveryLabel(total) {
  if (total >= 1000) {
    return 'бесплатная доставка';
  }
  return 'доставка 300 ₽';
}

console.log(deliveryLabel(1500));
console.log(deliveryLabel(400));
```

Если заказ дорогой, срабатывает первый `return`, и до второго дело не доходит. Если условие ложно, функция спокойно идёт дальше и возвращает ответ по умолчанию. Такой код читается как список правил: сначала особый случай, потом общий.

Второе свойство: функция без `return` всё равно что-то возвращает. Этим значением будет `undefined`, то самое «значения нет» из первой главы. JavaScript не считает отсутствие `return` ошибкой, поэтому забытый `return` не роняет программу, а тихо протаскивает `undefined` дальше, и ломается уже какой-нибудь код тремя строками ниже. К этой ловушке вернёмся в конце главы.

## Функция как значение: function expression

В JavaScript функция ведёт себя как полноценное значение, наравне с числом или строкой. Её можно создать без имени и положить в переменную:

```js
const sayHello = function (name) {
  return 'Привет, ' + name;
};

console.log(sayHello('Анна'));
```

Такая запись называется function expression: безымянная функция создаётся как значение и присваивается переменной `sayHello`, а вызывается через имя переменной. Практическая разница с declaration проявляется в одном месте: declaration можно вызвать даже выше той строки, где он записан, а expression доступен только после присваивания. Почему движок так делает, разобрано во врезке в конце главы.

## Стрелочные функции: та же коробка, короче запись

В 2015 году стандарт ES6 добавил ещё один способ записи, и сегодня в кодовых базах он самый частый. Стрелочная функция избавляется от слова `function`:

```js
const priceWithDiscount = (price, percent) => {
  return price - (price * percent) / 100;
};
```

Читается запись так: «параметры `price` и `percent` переходят в тело». Пока тело стоит в фигурных скобках, стрелка ведёт себя как обычный function expression: нужен явный `return`, иначе наружу уйдёт `undefined`.

<MarginNote label="История">До ES6 каждая функция в JavaScript писалась через слово function. Стрелочная запись пришла из математической нотации лямбда-функций.</MarginNote>

Настоящая экономия начинается, когда всё тело состоит из одного выражения. Тогда фигурные скобки и `return` убираются разом:

```js
const double = (x) => x * 2;

console.log(double(21));
```

Краткая форма возвращает результат выражения сама, без слова `return`. Это называют неявным return. Правило стоит проговорить дважды, потому что именно из него растёт самая частая ошибка со стрелками: нет фигурных скобок, значит результат выражения возвращается автоматически; есть фигурные скобки, значит вы обязаны написать `return` руками.

Ещё две мелочи, которые встретятся вам в чужом коде. Когда параметр ровно один, круглые скобки вокруг него разрешено опустить: запись `x => x * 2` работает так же, как `(x) => x * 2`, а какой стиль выбрать, команды решают сами. Когда параметров нет совсем, пустые скобки обязательны: `() => 42`.

## Параметры по умолчанию

Иногда аргумент хочется сделать необязательным. Для этого параметру задают значение по умолчанию прямо в скобках объявления:

```js
function greet(name = 'гость') {
  return 'Привет, ' + name;
}

console.log(greet('Анна'));
console.log(greet());
```

Первый вызов печатает «Привет, Анна». Во втором аргумента нет, поэтому `name` получает значение `'гость'`, и печатается «Привет, гость». Значение по умолчанию подставляется и тогда, когда вызов явно передал `undefined`. А вот `null` и пустая строка считаются нормальными переданными значениями: для них умолчание не сработает, и это регулярно удивляет.

Умолчания сильнее всего выручают в функциях с редко меняющимися настройками. Стоимость доставки почти всегда стандартная, но для отдалённых регионов другая: сделайте её параметром со значением по умолчанию, и девять вызовов из десяти останутся короткими, а десятый явно передаст свою цену. Обычный порядок такой: сначала обязательные параметры, после них необязательные с умолчаниями.

## Функцию можно передать в другую функцию

Из того, что функция ведёт себя как значение, следует приём, на котором держится половина JavaScript: функцию можно передать в другую функцию как аргумент. Принимающая сторона решает, когда и сколько раз её вызвать:

```js
function applyTwice(fn, value) {
  return fn(fn(value));
}

const increment = (x) => x + 1;

console.log(applyTwice(increment, 40));
```

`applyTwice` ничего не знает про сложение: она получает функцию через параметр `fn` и дважды её применяет, поэтому в консоль уходит 42. Передаваемую функцию часто называют callback. Передавать можно любую функцию, не только стрелку: declaration тоже уходит в аргументы по своему имени, просто стрелки короче, и в роли колбэков они прижились лучше всего. На этом же приёме построен метод массивов `map`, который ждёт вас в главе «map, filter и reduce».

## Ловушка: фигурные скобки съели результат

Одна стрелочная запись выглядит почти как другая, а ведёт себя иначе. Проверьте себя: что напечатает этот код?

```js
const thrice = (x) => {
  x * 3;
};

console.log(thrice(5));
```

Печатается `undefined`. Выражение `x * 3` честно вычислилось и тут же пропало: фигурные скобки включили режим «обычное тело», а из него результат наружу выносит только `return`. Исправления два: либо дописать `return x * 3;`, либо убрать фигурные скобки вовсе и вернуться к краткой форме.

Чаще всего эта ловушка срабатывает при переделке кода. Была краткая стрелка, понадобилось добавить в неё второй шаг, вы обернули тело в фигурные скобки, дописали новую строку, а про `return` для старой забыли. Функция продолжает выглядеть почти как раньше, но возвращать перестала.

<Callout type="warning" title="Забытый return не падает">
Самое неприятное свойство этой ошибки в том, что программа продолжает работать. Значение `undefined` уползает дальше и ломает вычисления совсем в другом месте. Если видите неожиданный `undefined`, первым делом проверяйте `return` в своих функциях.
</Callout>

## Частые ошибки

Первая ошибка: вызов без круглых скобок.

```js
const result = priceWithDiscount;
console.log(typeof result);
```

Печатается `function`, а не число. Без скобок вы не вызвали функцию, а взяли её как значение и положили в переменную целиком. В консоли такая пропажа выглядит как строка вида `[Function: priceWithDiscount]` там, где ждали число. Лечится добавлением скобок с аргументами: `priceWithDiscount(1000, 10)`.

Вторая ошибка живёт в голове, а не в синтаксисе: путаница параметра и аргумента. Внутри тела начинающие пишут расчёт через конкретное число 1000 вместо параметра `price`, и функция навсегда считает скидку от одной цены. Путаница возникает потому, что в момент написания функции конкретных значений ещё нет и мыслить заглушками непривычно. Помогает правило: внутри тела существуют только параметры, конкретные числа приходят снаружи при вызове.

Третья ошибка: код после `return`.

```js
function orderSummary(total) {
  return 'итого: ' + total;
  console.log('заказ собран');
}
```

Строка с `console.log` не выполнится никогда: `return` уже остановил функцию. Линтеры помечают такой код как недостижимый (unreachable). Всё, что должно случиться, делайте до `return`, а сам `return` ставьте последним осмысленным шагом ветки.

Эти три ошибки объединяет одно неприятное свойство: JavaScript почти никогда не сообщает о них сразу. Программа запускается, работает и выдаёт странный результат где-то дальше по коду. Поэтому при отладке функций начинайте с трёх вопросов: та ли функция вызвана, вызвана ли она со скобками и каждая ли ветка возвращает значение через `return`.

<Detail summary="Почему declaration можно вызвать до объявления">
Перед выполнением скрипта движок JavaScript просматривает весь код и регистрирует function declaration заранее. Механизм называется hoisting, по-русски «подъём». Function expression и стрелки в переменных целиком не поднимаются: имя переменной движок знает, но значение появится в ней только на строке присваивания, поэтому более ранний вызов упадёт с ошибкой.
</Detail>

<Detail summary="Одна строка про this">
Стрелочные функции не имеют собственного `this` и берут его из окружающего кода. Пока вы пишете отдельные функции, разница никак не проявляется; она станет важной позже, когда дойдёте до объектов с методами и обработчиков событий.
</Detail>
