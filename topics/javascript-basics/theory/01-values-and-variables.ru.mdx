# Значения, типы и let/const

Анна пишет свой первый скрипт для интернет-магазина: нужно сложить цену чайника и стоимость доставки, чтобы показать покупателю итог. Цена 2999 рублей, доставка 299, на экране должно появиться 3298. Скрипт отрабатывает без единой ошибки и уверенно печатает 2999299. Ни красного текста, ни предупреждения: программа убеждена, что всё сделала правильно.

Разгадка в том, что цена пришла из поля ввода, а всё, что приходит из поля ввода, JavaScript считает текстом. Плюс получил два текста и склеил их вместо того, чтобы сложить два числа. Чтобы такие сюрпризы не ломали ваши программы, нужно разобраться в трёх вещах: какие значения бывают в JavaScript, как узнать тип значения и как правильно дать значению имя. К концу главы вы будете читать историю Анны как открытую книгу и точно знать, чем её лечить.

## Имя указывает на значение

Всё, чем оперирует программа, состоит из значений: число 2999, текст «Казань», ответ «да» или «нет». У каждого значения есть тип, и тип определяет, что со значением можно делать: числа подчиняются арифметике, тексты склеиваются и режутся, логические значения отвечают только «истина» или «ложь».

Переменная при этом вовсе не «коробка с данными», как рисуют в старых учебниках. Точнее представлять её как бирку с именем, от которой тянется стрелка к значению. На рисунке ниже имена стоят слева, значения лежат в коробках справа, и каждая стрелка связывает имя с его текущим значением. Обратите внимание, где написан тип: он живёт в коробке рядом со значением, а не на бирке с именем.

Чтобы создать такую стрелку, объявите переменную одним из двух ключевых слов: `let` или `const`.

```js
let price = 2999;
const city = 'Казань';
let isPaid = false;
```

`let` создаёт имя, стрелку которого позже можно перенаправить на другое значение. `const` создаёт имя со стрелкой, зафиксированной навсегда: одно присваивание при объявлении, и на этом всё.

```js
let total = 2999;
total = total + 299;
console.log(total);
```

Первая строка создаёт имя `total` и направляет стрелку на число 2999. Вторая вычисляет новое число 3298 и перенаправляет стрелку на него. Старое значение при этом не «изменилось изнутри»: числа в JavaScript неизменяемы, имя просто стало указывать на другую коробку.

```js
const rate = 0.2;
rate = 0.25;
```

Вторая строка падает с ошибкой `TypeError: Assignment to constant variable`. И это хорошая новость: `const` защищает вас от случайной перезаписи того, что меняться не должно. Отсюда рабочее правило, принятое в большинстве команд: объявляйте всё через `const`, а `let` берите только там, где переприсвоение действительно нужно. Тогда каждый `let` в коде читается как честное предупреждение: это значение будет меняться.

Пара слов о самих именах. JavaScript разрешает почти любые названия, но коллеги скажут вам спасибо за осмысленные: `totalPrice` вместо `t`, `isPaid` вместо `flag`. Слова внутри имени соединяют в стиле camelCase: первое слово со строчной буквы, каждое следующее с заглавной. И помните про регистр: `totalPrice` и `totalprice` для языка два разных имени, опечатка в регистре создаст новую переменную вместо обращения к старой.

Заметьте важное следствие модели со стрелками: тип принадлежит значению, а не имени. В JavaScript нельзя объявить «переменную типа число», как в строго типизированных языках. Имя, созданное через `let`, сегодня может указывать на число, а завтра на строку. Такую свободу называют динамической типизацией, и она обоюдоострая: писать быстрее, но следить за типами приходится вам самим.

## Пять примитивов на каждый день

Значения в JavaScript делятся на две большие категории: примитивы и объекты. Примитивных типов, с которыми вы будете сталкиваться постоянно, пять:

- `number`: все числа сразу, и целые 2999, и дробные 3.14, и отрицательные; отдельного целого типа нет.
- `string`: текст в одинарных или двойных кавычках, например `'Казань'` или `"чайник"`.
- `boolean`: ровно два значения, `true` и `false`.
- `undefined`: «значения ещё нет»; его получает переменная, объявленная без присваивания.
- `null`: «значения нет, и это сделано намеренно»; пустота, которую поставил программист.

Ещё два примитива, `bigint` для сверхбольших целых чисел и `symbol` для уникальных ключей, встречаются заметно реже: на старте достаточно знать, что они существуют.

Вторая категория объединяет всё остальное: объекты. Объект собирает несколько значений под одной крышей: у заказа в магазине есть название товара, цена и флаг оплаты. Массивы и функции формально тоже объекты, о них пойдёт разговор в следующих главах.

Узнать тип значения помогает оператор `typeof`: он принимает значение и возвращает строку с названием типа.

```js
console.log(typeof 2999);
console.log(typeof 'Казань');
console.log(typeof true);
console.log(typeof undefined);
```

Код печатает `number`, `string`, `boolean` и `undefined`. Откройте консоль браузера клавишей F12, вкладка Console, и проверьте сами: введите `typeof 'Ozon'`, затем `typeof (2 + 2)`. Консоль выполняет JavaScript мгновенно и прощает любые ошибки, это лучший тренажёр для экспериментов из этой главы.

## const фиксирует стрелку, а не содержимое

Самое частое недопонимание вокруг `const` звучит так: «это же константа, значит внутри ничего нельзя менять». Для примитивов так и есть: число 2999 невозможно «отредактировать», его можно только заменить целиком, а замену `const` как раз запрещает. Но с объектами картина другая, и её стоит разглядеть на рисунке.

Проверим прямо в коде:

```js
const order = { item: 'чайник', price: 2999 };
order.price = 2599;
console.log(order.price);
```

Код печатает 2599 и не выбрасывает ошибок. `const` зафиксировал стрелку от имени `order` к объекту, поэтому переприсвоить `order` целиком не выйдет. А сам объект остался обычным изменяемым значением: полю `price` можно записать новую цену, к объекту можно добавить поле `delivery`. Стрелка неподвижна, коробка живёт своей жизнью. Запомните формулу: `const` защищает ссылку, а не содержимое.

## Три сюрприза, которые ловят каждого новичка

<MarginNote label="История">
  Брендан Айк собрал первую версию JavaScript за десять дней в мае 1995 года. Несколько
  странностей языка, включая знаменитое поведение typeof null, растут именно из той спешки.
</MarginNote>

Теперь вернёмся к скрипту Анны и разберём его сюрприз вместе с двумя родственниками.

```js
console.log('2' + 2);
console.log(2 + 2);
```

Первая строка печатает `22`, вторая печатает `4`. Когда хотя бы один операнд у `+` строка, JavaScript приводит второй операнд к строке и склеивает. Автоматическое приведение типов называют коэрцией. Забавная деталь: умножение ведёт себя ровно наоборот. Выражение `'5' * 2` даёт число `10`, потому что у `*` нет строкового режима и он приводит строку к числу. Один оператор тянет всё к строкам, другой к числам: полагаться на угадывание здесь нельзя, полагаться стоит на `typeof` и явное преобразование через `Number()`.

Кстати, именно так лечится скрипт Анны. Значение из поля ввода достаточно один раз явно превратить в число, и плюс снова заработает как сложение:

```js
const rawPrice = '2999';
const price = Number(rawPrice) + 299;
console.log(price);
```

Код печатает 3298, как и хотела Анна. Правило на будущее: всё, что пришло снаружи, из поля ввода, адресной строки или файла, сначала приводите к нужному типу и только потом пускайте в вычисления.

Второй сюрприз прячется в дробях:

```js
console.log(0.1 + 0.2);
console.log(0.1 + 0.2 === 0.3);
```

Печатается `0.30000000000000004`, затем `false`. Это не поломка JavaScript: почти все языки, включая Python и Java, хранят дроби в двоичном формате, где 0.1 представимо только приближённо. Для денег отсюда следует практичный вывод: храните суммы в копейках целыми числами, а дроби сравнивайте с допуском, а не на строгое равенство.

Третий сюрприз занял почётное место в истории языка:

```js
console.log(typeof null);
```

Печатается `object`. Не `null`, как было бы логично, а именно `object`. Это признанная ошибка первой реализации 1995 года: тип значения кодировался внутренним тегом, у объектов тег был нулевым, и `null`, представленный нулевым указателем, попал в ту же категорию. Комитет по стандартизации не раз обсуждал исправление и каждый раз отказывался: за тридцать лет накопились миллионы страниц, чей код полагается на текущее поведение. Отсюда правило: на `null` проверяйте строгим сравнением `value === null`, а `typeof` для этой задачи не годится.

## Частые ошибки

Три ошибки встречаются у новичков чаще остальных, и все три теперь легко объяснить через модель со стрелками.

**Переприсвоение `const`.** Код падает с `TypeError: Assignment to constant variable`. Обычно так выходит, когда имя объявили константой, а потом решили накапливать в нём результат. Лечится двумя способами: либо честно поменяйте объявление на `let`, либо, что чаще правильнее, заведите новое имя для нового значения.

**Путаница `undefined` и `null`.** Оба значения сообщают «здесь пусто», поэтому новички используют их вперемешку. Разница в авторстве. `undefined` ставит сам язык: переменная без значения, отсутствующее поле объекта, функция без return. `null` ставите вы, когда хотите явно сказать «тут ничего нет, и я это проверил». Держитесь договора: `null` пишет человек, `undefined` оставляют языку. Тогда проверка `value === null` читается как осмысленный сигнал, а не как случайность.

**Объявление без `let` и `const`.** Строка `total = 2999` без ключевого слова в старом «небрежном» режиме не вызывает ошибку: она молча создаёт глобальную переменную, видимую из любого места программы. Две функции, случайно выбравшие одно имя, начнут перетирать данные друг друга, и такой баг можно искать часами. В строгом режиме и внутри модулей тот же код честно падает с `ReferenceError`. Правило без исключений: каждое имя рождается со словом `const` или `let`.

<Detail summary="Куда делся var">
  До стандарта ES2015 переменные объявляли словом `var`, и в старом коде оно встречается на
  каждом шагу. У `var` два неприятных свойства. Во-первых, он игнорирует границы блоков:
  переменная, объявленная внутри if, видна и снаружи него. Во-вторых, он разрешает объявить одно
  имя дважды, и второе объявление молча перетирает первое. `let` и `const` исправили обе
  проблемы, поэтому в новом коде `var` не используют. Увидели `var` в чужом проекте: перед вами
  либо код до 2015 года, либо старая привычка автора.
</Detail>

<Detail summary="Почему 0.1 + 0.2 не даёт ровно 0.3">
  Числа в JavaScript хранятся в формате IEEE 754 с двойной точностью: 64 бита, двоичная система
  счисления. Дробь 0.1 в двоичной записи бесконечна, как 1/3 в десятичной, поэтому в память
  попадает ближайшее представимое число, чуть отличающееся от настоящей одной десятой. Сумма двух
  таких приближений даёт третье приближение, и оно оказывается чуть больше 0.3. Для вывода на
  экран выручает метод `toFixed`, для денег надёжнее целые копейки, а для сравнения дробей
  используйте допуск: `Math.abs(a - b) < 1e-9`.
</Detail>

Прежде чем переходить к упражнениям, устройте себе минутную проверку в консоли. Предскажите результат каждого выражения, а потом введите его и сверьтесь: `typeof ''`, `typeof 0`, `'10' * '2'`, `'10' + '2'`. Если все четыре предсказания сошлись, ментальная модель у вас уже рабочая.
