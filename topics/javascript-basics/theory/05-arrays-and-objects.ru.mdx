# Сто товаров, одна переменная

Представьте корзину покупателя в интернет-магазине. Пока в ней три товара, всё держится на честном слове: вы заводите переменные `price1`, `price2`, `price3`, рядом с ними `name1`, `name2`, `name3`, а сумму заказа считаете, складывая три числа вручную. Покупатель добавляет четвёртый товар, и вы дописываете четвёртую пару переменных и правите формулу. На Ozon в корзину можно положить сотню позиций. Сотню пар переменных завести теоретически можно, но обойти их циклом уже не выйдет: у имён переменных нет номеров, к `price37` нельзя обратиться «по счётчику».

В этой главе вы познакомитесь с двумя структурами, на которых держится почти весь код на JavaScript. Первая хранит много значений по порядку и выдаёт их по номеру. Вторая описывает одну вещь целиком: у товара есть название, цена, наличие на складе. В конце вы соедините их в главный рабочий паттерн, список товаров, и узнаете, чем переменная с массивом отличается от переменной с числом. На этом отличии спотыкается почти каждый новичок, поэтому мы разберём его с картинкой.

## Массив: ячейки с номерами

Массив хранит значения по порядку, в пронумерованных ячейках, и целиком живёт под одним именем. Создают его квадратными скобками, элементы разделяют запятыми:

```js
const list = ['хлеб', 'молоко', 'сыр'];
```

Нумерация начинается с нуля: «хлеб» лежит в ячейке 0, «молоко» в ячейке 1, «сыр» в ячейке 2. Чтобы достать элемент, после имени массива пишут номер в квадратных скобках: `list[0]` вернёт «хлеб», `list[2]` вернёт «сыр». Сколько элементов лежит в массиве, знает свойство `length`: у `list` оно равно 3. Отсюда приём, который вы будете применять постоянно. Раз нумерация идёт с нуля, номер последнего элемента на единицу меньше длины:

```js
const last = list[list.length - 1];
```

Длина равна 3, значит последняя занятая ячейка носит номер 2, и в `last` попадёт «сыр». Выражение выглядит громоздко, но пишется настолько часто, что рука быстро запоминает его сама. Отрицательные номера, кстати, не работают: `list[-1]` не достанет последний элемент, как это бывает в других языках, а честно поищет ключ «минус один» и ничего не найдёт.

<MarginNote label="Почему с нуля">Индекс исторически означает смещение от начала: первый элемент лежит в самом начале, его смещение равно нулю. Эту традицию JavaScript унаследовал от языка C вместе с большей частью синтаксиса.</MarginNote>

Массив умеет расти и сжиматься. Метод `push` добавляет элемент в конец, метод `pop` снимает последний элемент и возвращает его:

```js
list.push('кефир');
const taken = list.pop();
```

После `push` длина стала 4, после `pop` снова 3, а в `taken` оказался «кефир». Пара `push` и `pop` закрывает удивительно много задач: история действий для отмены, стопка открытых экранов, накопление результатов в цикле.

Искать значение помогают два метода. `includes` отвечает, есть ли значение в массиве, а `indexOf` возвращает номер ячейки, в которой значение нашлось:

```js
list.includes('молоко');
list.indexOf('сыр');
```

Первый вызов вернёт `true`, второй вернёт 2. Если значения нет, `includes` честно скажет `false`, а вот `indexOf` вернёт число -1, и его легко принять за настоящий индекс, если забыть о проверке.

## Объект: имя вместо номера

Теперь посмотрим на один товар. У него есть название, цена, признак наличия, и обращаться к ним хочется по смыслу, а не по номеру ячейки. Для этого в JavaScript есть объект: набор пар «ключ и значение» в фигурных скобках.

```js
const item = { name: 'чай', price: 250 };
```

Читают свойства двумя способами. Точка подходит, когда ключ известен заранее и написан в коде буквально. Квадратные скобки нужны, когда ключ лежит в переменной:

```js
item.name;
item['price'];
const key = 'price';
item[key];
```

Первые два обращения вернут «чай» и 250 и полностью взаимозаменяемы. Третье интереснее: в скобках стоит не строка, а переменная `key`, внутри которой строка лежит. Точка так не умеет. Запись `item.key` искала бы свойство с буквальным именем key, а не заглядывала бы в переменную.

Новые свойства добавляют обычным присваиванием, пересоздавать объект для этого не нужно:

```js
item.inStock = true;
item.seller = { name: 'Чайный дом', city: 'Казань' };
```

Второе присваивание кладёт внутрь объекта другой объект. Такая вложенность встречается повсюду: ответ сервера, настройки приложения, профиль пользователя. К глубине добираются цепочкой: `item.seller.city` вернёт «Казань». Полный список ключей объекта возвращает `Object.keys(item)`: это обычный массив строк `['name', 'price', 'inStock', 'seller']`, к которому применимы все приёмы из первой половины главы.

## Массив объектов: паттерн на каждый день

Соединим обе структуры и вернёмся к корзине из начала главы. Каждый товар станет объектом, а корзина массивом таких объектов:

```js
const products = [
  { name: 'хлеб', price: 45 },
  { name: 'молоко', price: 89 },
  { name: 'сыр', price: 320 },
];
```

Три товара или сто, код обработки не меняется. Перебирает массив цикл `for...of`: он выдаёт элементы по одному, и возиться с индексами не приходится:

```js
let total = 0;
for (const product of products) {
  total = total + product.price;
}
```

На каждом витке цикла в `product` попадает очередной объект, а `product.price` достаёт его цену. После цикла в `total` лежит 454, сумма корзины в рублях. Замените сложение на условие, и тот же каркас посчитает товары дешевле ста рублей или найдёт самый дорогой. Именно в таком виде данные приходят из реального мира: список заказов, сообщения чата, строки таблицы почти всегда оказываются массивом объектов, поэтому связку с `for...of` вы будете писать буквально каждый день.

## Переменная хранит ссылку, а не сам объект

Пора выполнить обещание из вступления. Когда вы пишете `const n = 5`, число можно считать лежащим прямо в переменной. С массивами и объектами картина другая: сама «коробка» с данными живёт в памяти отдельно, а в переменной хранится только ссылка на неё, что-то вроде адреса склада на стикере.

Из этого следует неожиданный факт: две переменные могут указывать на один и тот же объект.

```js
const item = { name: 'чай', price: 250 };
const copy = item;
```

Имя `copy` обманывает: никакой копии не появилось. Присваивание перенесло стикер с адресом, и теперь обе переменные ведут к одной коробке. Значит, изменение через одну переменную видно через другую:

```js
copy.price = 199;
console.log(item.price);
```

Напечатается 199, хотя `item` никто «не трогал». Через какую переменную смотреть на объект, значения не имеет, ведь объект один.

Та же модель объясняет поведение `const`, которое поначалу выглядит противоречиво. `const` запрещает перепривязать переменную к другому значению, то есть заменить ссылку на стикере. А содержимое коробки, куда ссылка ведёт, менять никто не запрещал:

```js
const cart = ['хлеб'];
cart.push('молоко');
```

Этот код работает: `push` меняет массив изнутри, ссылка в `cart` осталась прежней. А вот присваивание `cart = ['сыр']` упадёт с ошибкой, потому что оно пытается положить в `const` новую ссылку.

## Краевые случаи, о которых лучше узнать сейчас

Чтение несуществующей ячейки или несуществующего ключа не роняет программу. JavaScript молча возвращает `undefined`:

```js
const list = ['хлеб', 'молоко', 'сыр'];
const item = { name: 'чай' };
console.log(list[10]);
console.log(item.price);
```

Обе команды напечатают `undefined`. Это удобно и опасно одновременно: опечатка в имени свойства не вызовет ошибку в месте чтения, а «выстрелит» позже, когда `undefined` попадёт в арифметику и превратит результат в `NaN` где-нибудь в другом конце программы. Если видите `NaN` в сумме заказа, первым делом проверяйте имена ключей и границы индексов.

Второй сюрприз касается сравнения. Оператор `===` сравнивает массивы и объекты по ссылке, а не по содержимому:

```js
console.log([1] === [1]);
```

Напечатается `false`. Слева и справа созданы два разных массива, две отдельные коробки, и совпадение содержимого оператор не волнует. `true` получится только тогда, когда обе стороны указывают на один и тот же объект, как `item` и `copy` из примера выше. Содержимое сравнивают вручную: сначала длины, затем элементы один за другим.

## Частые ошибки

**Точка вместо скобок при ключе в переменной.** Ошибка коварна тем, что ничего не падает:

```js
const field = 'price';
const wrong = item.field;
const right = item[field];
```

`item.field` ищет свойство с буквальным именем field, его в объекте нет, и в `wrong` попадёт `undefined`. Правило простое: ключ написан в коде буквально, ставьте точку; ключ приезжает из переменной, ставьте скобки.

**Промах на единицу у границы.** Классика жанра: обратиться к `list[list.length]` в надежде получить последний элемент. Ячейки с таким номером не существует, придёт `undefined`. Последний элемент живёт в `list[list.length - 1]`, и это стоит проговорить про себя ещё раз.

**Мутация «копии», которая оказалась оригиналом.** Вы «скопировали» массив присваиванием, отсортировали копию или дописали в неё элементы, а потом обнаружили, что исходные данные тоже изменились. Присваивание копирует ссылку, а не содержимое. Настоящую копию создают явно, об этом ниже.

<Detail summary="Как скопировать массив или объект по-настоящему">

Поверхностную копию создаёт спред-синтаксис: три точки перед именем раскладывают старое содержимое в новую коробку.

```js
const orig = ['хлеб', 'молоко'];
const copied = [...orig];
copied.push('сыр');
```

`orig` остался из двух элементов, потому что `copied` указывает на новый массив. Для объектов работает та же запись с фигурными скобками: `{ ...item }`. Подвох спрятан в слове «поверхностную»: копируется только верхний уровень. Если внутри массива лежат объекты, в копию попадут ссылки на те же самые объекты, и вложенный товар окажется общим у копии и оригинала. Глубокую копию делает встроенная функция `structuredClone`, но нужна она реже, чем кажется.
