# Strings: The Text a Program Works With

Anna is assembling the order confirmation text for an online store: "Thanks for your order, Anna! Total: $1500." The first version is glued together with plus signs: `'Thanks for your order, ' + name + '! Total: $' + price`. The code works, until one day `price` arrives as the number 1500, and a discount of 200 gets added to it with the same plus sign: `'Total: $' + price + discount`. Instead of subtraction, you get the concatenation `'Total: $1500200'`. Plus in JavaScript behaves differently with strings and numbers, and that's the sole reason an innocent-looking string breaks the order total.

In this chapter you'll figure out how a string differs from a number when it meets `+`, learn template literals, which solve Anna's problem without a single plus sign, and pick up a handful of methods for reading and slicing strings. Along the way you'll learn why `str[0] = 'X'` does absolutely nothing in JavaScript.

## A string as a numbered tape of characters

A string looks like one solid piece of text, but under the hood it's built like a tape of characters, and every character has a number. Numbering starts at zero, same as the array from the previous chapter. In the string `'Привет'`, the letter "П" sits at position 0, and the last letter "т" sits at position 5, because there are six characters and counting starts from zero.

You grab a character by its number with square brackets, exactly like an array element: `word[0]` returns "П," `word[word.length - 1]` returns the last character regardless of the string's length. The `length` property counts characters, spaces included: `'привет мир'` has a length of 10.

<PullQuote>A string behaves like a row of numbered cells: the same zero-based indexing, the same length property as an array.</PullQuote>

## Strings are immutable

Since a string resembles an array, it seems natural to try replacing a character by index:

```js
const word = 'кот';
word[0] = 'Р';
console.log(word);
```

The console still prints `'кот'`. Strings in JavaScript are immutable: no existing character can be changed. In an ordinary script and in the console, an assignment by index is silently ignored, while in strict mode (`'use strict'`, or an ES module, which is the default for modern code) that same line throws a TypeError. Either way, the string itself never changes. The only way to get different text is to build a new string and store it in the variable fresh:

```js
let word = 'кот';
word = 'Р' + word.slice(1);
console.log(word);
```

Here `word` wasn't modified, it was completely replaced with a new value. This detail explains why every string method below returns a new string instead of altering the original: there's nothing for them to alter.

## Plus with strings: concatenation versus addition

The `+` operator is overloaded: between two numbers it adds, and if either operand is a string, it glues text together, converting a number to a string first. Because of this rule, operand order decides the outcome:

```js
console.log('Итого: ' + 100 + 50);
console.log('Итого: ' + (100 + 50));
console.log(100 + 50 + ' ₽');
```

The first line prints `'Итого: 10050'`: `'Итого: ' + 100` is already a string, and the following `+ 50` just appends the text "50." The second line, with parentheses, prints `'Итого: 150'`, because the addition inside the parentheses runs first, while both operands are still numbers. The third line prints `'150 ₽'` for the same reason: the numbers add to each other before any string joins in. This is exactly the asymmetry that tripped up Anna in the intro.

<Callout type="warning" title="+ doesn't read your intent">
JavaScript never asks whether you meant to add numbers or glue text. It looks at operand types, left to right. If you're expecting a sum and there's a string nearby, wrap the numeric part in parentheses or compute it in a separate variable before concatenating.
</Callout>

## Template literals: minus every plus sign

ES6 added a syntax with backticks that solves Anna's problem without a single `+`. Inside `${...}` you can drop in any expression, and the engine converts it to a string in exactly the right spot:

```js
const name = 'Анна';
const price = 1500;
const discount = 200;

const message = `Спасибо за заказ, ${name}! Итого: ${price - discount} ₽`;
console.log(message);
```

It prints `'Спасибо за заказ, Анна! Итого: 1300 ₽'`. The subtraction `price - discount` sits inside `${}`, so both operands stay numbers right up to the end, and no neighboring string turns them into text too early. You can drop in more than variables: function calls and whole expressions work too, like `${price > 1000 ? 'дорого' : 'недорого'}`.

Template literals have a second useful trait: line breaks work literally inside them, no `\n` needed:

```js
const receipt = `Заказ №42
Итого: 1300 ₽`;
console.log(receipt);
```

An ordinary string in single or double quotes doesn't allow that: press Enter inside one, and JavaScript throws a syntax error, forcing you to insert the break as `\n`.

## Methods you'll reach for often

Every method below leaves the string untouched and returns a new one. It's handy to check them one by one through a chain of `console.log` calls.

```js
const raw = '  Привет, Мир  ';

console.log(raw.trim());
console.log(raw.trim().toUpperCase());
console.log(raw.trim().toLowerCase());
console.log(raw.includes('Мир'));
console.log(raw.trim().slice(0, 7));
console.log(raw.trim().split(', '));
console.log(raw.trim().replace('Мир', 'JavaScript'));
```

`trim()` strips whitespace from the edges, not the middle, printing `'Привет, Мир'`. `toUpperCase()` and `toLowerCase()` change the case of every character. `includes('Мир')` returns `true` if the substring shows up at all, without telling you where. `slice(0, 7)` cuts characters 0 through 6 inclusive: the end of a `slice` range isn't included in the result, so the length of the cut piece equals the difference between the bounds, seven minus zero. `split(', ')` cuts the string at every separator and returns an array: `['Привет', 'Мир']`. `replace('Мир', 'JavaScript')` swaps out the first match of the substring and hands back the whole new string.

<MarginNote label="Related methods">`slice` also accepts negative indexes: `text.slice(-3)` returns the last three characters. The method `indexOf('Мир')` returns the index of the first match instead of true/false, or -1 if there isn't one.</MarginNote>

## Trap: slice's end isn't included in the result

`slice` boundaries confuse nearly every newcomer, because intuition says to include both ends:

```js
const word = 'JavaScript';
console.log(word.slice(0, 4));
console.log(word.length);
```

It prints `'Java'`, not `'JavaS'`. The character at index 4, the letter "S," never makes it into the cut piece: `slice`'s second argument is the boundary you cut up to, not the index of the last included character. The same trick you used with array length works here: if you need the first `n` characters, `slice`'s second argument is exactly `n`, because counting inside the range starts at zero.

## Common mistakes

First mistake: trying to change a string by index, like the `word[0] = 'Р'` example above. Outside strict mode the assignment silently fails to do anything: JavaScript doesn't throw an error, it just ignores it (strict mode and ES modules are more upfront: there you get a TypeError). If you need to change one character, build a new string using `slice` and concatenation, or a template literal.

Second mistake: adding a number and a string where you meant arithmetic addition, covered in the section on `+` above. It's especially sneaky in totals and counters: `total = total + input`, if `input` came in as a string from an input field, glues text instead of adding a number.

Third mistake: reaching for `includes` instead of `indexOf` when the substring's position matters, not just whether it's there:

```js
const url = 'https://example.com/page';
console.log(url.includes('page'));
console.log(url.indexOf('page'));
```

`includes` only answers true or false, while `indexOf` also names the character position where the match starts, 20 here. If you need to slice out the text after that word further down the code, you can't do it without the position, and `includes` doesn't give you one.

<Detail summary="Comparing strings with == and ===">
Strings compare character by character: `'кот' === 'кот'` gives `true`, while `'кот' === 'Кот'` gives `false`, because case matters. Use `===`, not `==`: they agree for two strings, but `==` additionally converts types, and `'' == false` unexpectedly turns out `true`. Getting into the habit of always writing `===` saves you from having to keep those conversion rules in your head.
</Detail>
