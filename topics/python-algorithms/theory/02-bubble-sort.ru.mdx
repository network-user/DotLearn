# Пузырьковая сортировка: большие числа всплывают наверх

У вас есть список чисел в случайном порядке, и его нужно упорядочить. Самый простой способ, который придумывает почти каждый человек самостоятельно: пройтись по списку и менять местами соседей, стоящих неправильно. Большие числа при этом постепенно «всплывают» к концу, как пузырьки воздуха в воде - отсюда и название. Алгоритм неэффективен на больших данных, но он идеален, чтобы понять, что такое сортировка вообще, и научиться рассуждать о шагах алгоритма.

К концу раздела вы сможете написать пузырьковую сортировку с нуля, объяснить, почему её сложность O(n²), ускорить её ранним выходом по флагу и понять, что значит «устойчивая» сортировка.

## Идея: чините пары соседей

Список отсортирован тогда и только тогда, когда каждая пара соседей стоит в правильном порядке: левый не больше правого. Пузырьковая сортировка превращает это наблюдение в действие. Она идёт по списку слева направо и сравнивает каждую пару соседних элементов. Если левый больше правого, они стоят неверно - меняем их местами. Один такой проход не упорядочивает список целиком, но гарантированно «выталкивает» самый большой элемент в конец: он будет побеждать в каждом сравнении и доедет до правого края.

После первого прохода наибольший элемент гарантированно стоит на своём финальном месте. Значит, следующий проход может не доходить до него - и так с каждым разом «отсортированный хвост» растёт, а зона работы сокращается.

## От одного прохода к полной сортировке

Один проход выталкивает один элемент. Чтобы упорядочить весь список из `n` элементов, проходов нужно до `n - 1`: после стольких проходов на местах окажутся все элементы, кроме первого, а он тогда автоматически тоже на месте.

```python
def bubble_sort(xs):
    a = list(xs)
    n = len(a)
    for i in range(n - 1):
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
    return a
```

Внешний цикл считает проходы. Внутренний идёт по парам соседей, и его верхняя граница - `n - 1 - i`: с каждым проходом `i` растёт, а зона сравнений сокращается, потому что хвост уже отсортирован. Строка `a[j], a[j + 1] = a[j + 1], a[j]` - это обмен соседей через распаковку кортежа, без временной переменной.

## Ранний выход: не работайте, когда уже отсортировано

Базовая версия делает все `n - 1` проходов всегда, даже если список упорядочился раньше. Это видно на входе `[1, 2, 3, 4]`: после нулевого прохода обменов не было, а алгоритм упрямо продолжает. Лекарство - флаг `swapped`. Если за целый проход не случилось ни одного обмена, значит все соседи уже на местах и список отсортирован - можно выходить.

```python
def bubble_sort(xs):
    a = list(xs)
    n = len(a)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
                swapped = True
        if not swapped:
            break
    return a
```

На уже отсортированном списке эта версия делает один проход, ничего не меняет, видит `swapped == False` и завершается. Это превращает лучший случай из O(n²) в O(n): один проход без обменов.

<Callout type="tip">Флаг сбрасывается в начале каждого прохода и поднимается при любом обмене. Проверять его нужно после внутреннего цикла, а не внутри: решение «всё отсортировано» принимается по итогам целого прохода.</Callout>

## Сложность: почему O(n²)

Считаем сравнения. В худшем случае (список отсортирован наоборот) ни один проход не сократить ранним выходом. Внутренний цикл делает `n-1`, потом `n-2`, и так до 1 сравнения - сумма примерно `n²/2`. Отбрасывая константу `1/2`, получаем O(n²) по времени. По памяти алгоритм скромен: он переставляет элементы на месте и заводит лишь пару переменных, то есть O(1) дополнительной памяти (наша версия копирует вход в `a`, чтобы не портить аргумент, - это O(n), но сама сортировка добавочной памяти не требует).

<Callout type="warning">Ранний выход улучшает лучший и средний случай, но не худший. На обратно отсортированных данных обмен происходит в каждой паре каждого прохода, и флаг всегда поднят - O(n²) остаётся.</Callout>

## Устойчивость: равные элементы не переставляются зря

Сортировка **устойчива** (stable), если она сохраняет относительный порядок элементов с равными ключами. Пузырьковая сортировка устойчива, потому что меняет соседей только при строгом `>`: равные элементы (`a[j] == a[j+1]`) условие не проходят и остаются в исходном порядке. Это важно, когда вы сортируете записи по одному полю, не желая ломать порядок по другому.

```python
people = [("Анна", 30), ("Борис", 25), ("Вера", 30)]
```

Сортируя такой список по возрасту устойчивым алгоритмом, вы получите Анну раньше Веры (как во входе), потому что их возраст равен и менять их местами нет причины. Если бы вы случайно написали `>=` вместо `>`, равные элементы стали бы переставляться - и устойчивость пропала бы.

## Типичные ошибки

- **Выход за границу списка.** Внутренний цикл сравнивает `a[j]` и `a[j + 1]`, поэтому `j` должен доходить только до `n - 2`. Диапазон `range(n - 1 - i)` это обеспечивает; если написать `range(n)`, обращение к `a[j + 1]` выйдет за конец списка и даст `IndexError`.
- **Сравнение `>=` вместо `>`.** С `>=` равные соседи переставляются без нужды: устойчивость теряется, а алгоритм делает лишнюю работу. Меняйте местами только при строгом `>`.
- **Проверять флаг внутри внутреннего цикла.** Решение о выходе принимается по итогам *целого* прохода. Проверка `if not swapped: break` стоит после внутреннего цикла, а сам `swapped` сбрасывается перед ним.

<Detail summary="Почему пузырьковую почти не используют на практике">

Для реальной сортировки берут `list.sort()` или `sorted()` - в CPython это Timsort, гибрид сортировки слиянием и вставками со сложностью O(n log n) в худшем случае. На миллионе элементов O(n log n) - это около двадцати миллионов операций, а O(n²) - триллион. Пузырьковая ценна как учебный пример: на ней удобно понять обмены, проходы, инварианты и устойчивость, прежде чем переходить к быстрым алгоритмам.

```python
print(sorted([5, 1, 4, 2, 8]))
```
