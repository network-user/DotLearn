# Bubble sort: large numbers float to the top

You have a list of numbers in random order and you need to organize it. The simplest way, which almost every person comes up with on their own: go through the list and swap the places of neighbors who are in the wrong order. Large numbers gradually “float” towards the end, like air bubbles in water - hence the name. The algorithm is not efficient on large data, but it is ideal for understanding what sorting is in general and learning to reason about the steps of the algorithm.

By the end of this section, you'll be able to write bubble sort from scratch, explain why it's complexity is O(n²), speed it up with early flag exit, and understand what "robust" sort means.

## Idea: fix pairs of neighbors

The list is sorted if and only if each pair of neighbors is in the correct order: the left one is not greater than the right one. Bubble sort turns this observation into action. It goes through the list from left to right and compares each pair of adjacent elements. If the left one is larger than the right one, they are positioned incorrectly - we swap them. One such pass does not order the entire list, but it is guaranteed to “push” the largest element to the end: it will win in every comparison and reach the right edge.

After the first pass, the largest element is guaranteed to be in its final place. This means that the next pass may not reach it - and so each time the “sorted tail” grows, and the work area shrinks.

## From one pass to full sorting

One pass ejects one element. To order the entire list of `n` elements, you need up to `n - 1` passes: after so many passes, all the elements will be in place except the first one, and then it will automatically be in place too.

```python
def bubble_sort(xs):
    a = list(xs)
    n = len(a)
    for i in range(n - 1):
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
    return a
```

The outer loop counts the passes. The internal one goes through pairs of neighbors, and its upper bound is `n - 1 - i`: with each pass `i` grows, and the comparison area shrinks, because the tail is already sorted. The line `a[j], a[j + 1] = a[j + 1], a[j]` is an exchange of neighbors via tuple unpacking, without a temporary variable.

## Early exit: don't work when already sorted

The basic version does all `n - 1` passes always, even if the list has been ordered before. This can be seen at the input `[1, 2, 3, 4]`: after the zero pass there were no exchanges, but the algorithm stubbornly continues. Medicine - `swapped` flag. If not a single exchange has occurred during the entire passage, then all the neighbors are already in place and the list has been sorted - you can leave.

```python
def bubble_sort(xs):
    a = list(xs)
    n = len(a)
    for i in range(n - 1):
        swapped = False
        for j in range(n - 1 - i):
            if a[j] > a[j + 1]:
                a[j], a[j + 1] = a[j + 1], a[j]
                swapped = True
        if not swapped:
            break
    return a
```

On an already sorted list, this version makes one pass, changes nothing, sees `swapped == False` and exits. This turns the best case from O(n²) to O(n): one pass without exchanges.

<Callout type="tip">Флаг сбрасывается в начале каждого прохода и поднимается при любом обмене. Проверять его нужно после внутреннего цикла, а не внутри: решение «всё отсортировано» принимается по итогам целого прохода.</Callout>

## Complexity: why O(n²)

We count comparisons. In the worst case (the list is sorted in reverse), no pass can be shortened by an early exit. The inner loop does `n-1`, then `n-2`, and so on until 1 comparison - the sum is approximately `n²/2`. Discarding the constant `1/2`, we get O(n²) in time. The algorithm is modest in terms of memory: it rearranges the elements in place and introduces only a couple of variables, that is, O(1) additional memory (our version copies the input to `a` so as not to spoil the argument - this is O(n), but the sorting itself does not require additional memory).

<Callout type="warning">Ранний выход улучшает лучший и средний случай, но не худший. На обратно отсортированных данных обмен происходит в каждой паре каждого прохода, и флаг всегда поднят - O(n²) остаётся.</Callout>

## Stability: equal elements are not rearranged in vain

A sort is **stable** if it preserves the relative order of elements with equal keys. Bubble sort is stable because it changes neighbors only under strict `>`: equal elements (`a[j] == a[j+1]`) do not pass the condition and remain in the original order. This is important when you are sorting records by one field but don't want to break the order by another.

```python
people = [("Анна", 30), ("Борис", 25), ("Вера", 30)]
```

By sorting such a list by age using a stable algorithm, you will get Anna before Vera (as in the input), because their ages are equal and there is no reason to swap them. If you accidentally wrote `>=` instead of `>`, equal elements would be rearranged - and stability would be lost.

## Common mistakes

- **Out of bounds list.** The inner loop compares `a[j]` and `a[j + 1]`, so `j` should only go as far as `n - 2`. The range `range(n - 1 - i)` provides this; if you write `range(n)`, accessing `a[j + 1]` will go beyond the end of the list and give `IndexError`.
- **Compare `>=` instead of `>`.** With `>=`, equal neighbors are rearranged unnecessarily: stability is lost, and the algorithm does unnecessary work. Swap only with strict `>`.
- **Check the flag inside the inner loop.** The exit decision is made based on the results of the *entire* pass. The `if not swapped: break` check comes after the inner loop, and `swapped` itself is reset before it.

<Detail summary="Why bubble is almost never used in practice">

For real sorting, take `list.sort()` or `sorted()` - in CPython this is Timsort, a hybrid of merge and insertion sort with O(n log n) complexity in the worst case. On a million elements, O(n log n) is about twenty million operations, and O(n²) is a trillion. Bubble is valuable as a teaching example: it is convenient to understand exchanges, passages, invariants and stability before moving on to fast algorithms.

```python
print(sorted([5, 1, 4, 2, 8]))
```

</Detail>
