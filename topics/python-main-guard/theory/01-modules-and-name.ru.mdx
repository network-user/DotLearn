# Откуда Python знает, запустили его или импортировали

Вы написали файл `greet.py`, дважды кликнули по нему и увидели приветствие в консоли. Потом тот же файл понадобился в другом скрипте, Вы написали `import greet` - и приветствие снова напечаталось, хотя Вы ничего не запускали, а только хотели взять оттуда одну функцию. Один и тот же файл повёл себя по-разному, и разницу нужно как-то уметь различать изнутри кода. Python даёт для этого ровно один инструмент: в каждом модуле автоматически появляется переменная `__name__`, и её значение зависит от того, как именно файл попал в работу.

К концу раздела Вы будете понимать, что любой `.py`-файл - это модуль, что при импорте Python выполняет его сверху донизу, и что переменная `__name__` хранит либо строку `"__main__"`, либо имя модуля. Это фундамент, на котором в следующих разделах вырастет идиома `if __name__ == "__main__"`.

## Ментальная модель: файл - это модуль, импорт - это запуск файла

Главное, что мешает новичкам: импорт кажется чем-то лёгким, вроде «дай мне функцию из соседнего файла». На деле `import greet` означает «найди файл `greet.py`, выполни его весь, сверху вниз, и дай мне доступ к тому, что осталось в его пространстве имён». Определения функций и классов при этом исполнении создаются, но не вызываются - а вот любая «голая» строка вроде `print(...)` на верхнем уровне исполнится немедленно, потому что для Python нет разницы между «запуском» и «импортом» на уровне исполнения тела файла. Разница только в одной переменной.

## Что Python кладёт в `__name__`

Переменная `__name__` появляется в каждом модуле автоматически, объявлять её не нужно. Правило одно: если файл запущен как главная программа, в `__name__` лежит строка `"__main__"`; если файл импортирован, там лежит имя модуля, под которым его импортировали.

```python
print("Меня зовут:", __name__)
```

Положите эту строку в файл `who.py`. Запустите его командой `python who.py` - напечатается `Меня зовут: __main__`. Теперь из другого файла или из интерактивной консоли выполните `import who` - и напечатается `Меня зовут: who`. Тот же `print`, то же тело файла, но `__name__` уже другой.

<MarginNote label="Почему «__main__»">Двойные подчёркивания вокруг имени - это соглашение Python для «магических» имён, которые задаёт сам интерпретатор. `__main__` читается как «главный модуль», точка входа в программу.</MarginNote>

## Имя при импорте - это имя модуля, а не имя файла на диске

Когда модуль импортирован, в `__name__` попадает не путь к файлу и не «что угодно», а ровно то имя, по которому Вы к нему обратились в `import`. Для простого файла рядом это его имя без расширения.

```python
import math
print(math.__name__)
```

Печатается `math`: у любого импортированного модуля атрибут `__name__` доступен снаружи как `module.__name__`. Это та же самая переменная, что внутри модуля видна просто как `__name__`. Иначе говоря, `__name__` - это не секретная внутренняя деталь, а обычный атрибут модуля, к которому можно обратиться и снаружи.

## Импорт выполняется один раз

Ещё одна деталь, без которой картина неполная: Python выполняет тело модуля при импорте только при **первом** импорте. Результат кэшируется, и повторные `import` той же строки берут уже готовый модуль из кэша, не исполняя его снова.

```python
import who
import who
import who
```

Если в `who.py` есть `print` на верхнем уровне, при трёх таких строках он напечатается ровно один раз. Это объясняет, почему импорты в большой программе не приводят к лавине повторных исполнений: модуль исполняется единожды, дальше он живёт в `sys.modules`.

<Callout type="info">
Кэш импортированных модулей лежит в `sys.modules` - это обычный словарь. Если из него удалить запись, следующий `import` снова выполнит тело модуля. В обычном коде так делать не нужно, но это объясняет, что «один раз» - не магия, а просто проверка наличия в словаре.
</Callout>

## Сюрприз: имя модуля меняется при импорте из пакета

Вот что удивляет даже тех, кто давно пишет на Python. Значение `__name__` импортированного модуля зависит от того, **откуда** его импортировали. Если `helpers.py` лежит внутри пакета `tools`, то при `import tools.helpers` его `__name__` станет не `"helpers"`, а `"tools.helpers"` - полное имя с точкой.

```python
import tools.helpers
print(tools.helpers.__name__)
```

Печатается `tools.helpers`. То есть `__name__` отражает положение модуля в дереве импортов, а не имя файла на диске. Один и тот же файл `helpers.py`, импортированный напрямую и через пакет, получит разные значения `__name__`. Запомните это: позже это объяснит, почему сравнивать `__name__` с захардкоженным именем модуля - плохая идея, а сравнивать с `"__main__"` - надёжно.

## Типичные ошибки

- **Думать, что импорт «просто берёт функцию».** Импорт исполняет весь файл. Любой `print`, открытие файла или сетевой запрос на верхнем уровне модуля выполнится при импорте - даже если Вы хотели взять оттуда только одну функцию.
- **Путать имя файла и `__name__`.** Внутри запущенного файла `__name__` равно `"__main__"`, а не имени файла. Имя файла без расширения попадает в `__name__` только при импорте.
- **Ожидать, что тело модуля исполнится при каждом импорте.** Оно исполняется один раз; повторные импорты берут модуль из кэша `sys.modules`.

<Detail summary="А что лежит в __name__ интерактивной сессии и в нотбуке">

Когда Вы запускаете Python без файла - просто `python` в терминале или ячейку в Jupyter, - код тоже исполняется в модуле с именем `"__main__"`. Поэтому в интерактивной сессии `__name__` равно `"__main__"`, как если бы Вы запустили скрипт. Это удобно: код, написанный с расчётом на `if __name__ == "__main__"`, в нотбуке ведёт себя как «главная программа», а импортированный из соседнего файла - как модуль. Деталь про пакеты из официальной документации: при запуске через `python -m package.module` имя главного модуля всё равно становится `"__main__"`, а не `"package.module"` - флаг `-m` запускает модуль именно как точку входа.

</Detail>
