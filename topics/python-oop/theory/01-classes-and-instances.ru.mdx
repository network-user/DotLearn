# Класс - это чертёж, экземпляр - вещь по чертежу

Вы пишете программу про счётчики посещений. Их три: на главной странице, в корзине, на странице оплаты. Можно завести три переменные и три набора функций, которые их увеличивают, но они разъедутся при первой же правке. Класс решает эту задачу иначе: Вы один раз описываете, *что такое счётчик* - какие у него данные и что он умеет, - а потом штампуете сколько угодно отдельных счётчиков, каждый со своим состоянием.

К концу раздела Вы сможете определить класс, понять, кто такой `self` и почему он передаётся явно, отличить атрибут экземпляра от атрибута класса, объяснить, когда два имени указывают на один объект, а когда на два разных, и обходить три ловушки, на которых спотыкается почти каждый новичок.

## Один чертёж, много экземпляров

Класс - это описание, а не вещь. Сам по себе он ничего не считает, как чертёж дома сам по себе никого не селит. Вещь появляется, когда Вы *создаёте экземпляр* (instance) - конкретный объект, построенный по чертежу и хранящий собственные значения.

```python
class Counter:
    def __init__(self, start=0):
        self.value = start

    def tick(self):
        self.value += 1
```

Ключевое слово `class` вводит новый тип; по соглашению имена классов пишут в `PascalCase`. Внутри - два метода. `__init__` - конструктор: он выполняется один раз в момент создания экземпляра и раздаёт ему стартовые атрибуты. `tick` - обычный метод, меняющий состояние уже существующего объекта.

Создаётся экземпляр вызовом класса, как будто это функция. Аргументы уходят в `__init__`.

```python
c = Counter(10)
c.tick()
c.tick()
print(c.value)
```

Печатается `12`. А главное свойство видно, когда экземпляров несколько: у каждого свои атрибуты, и состояние между ними не перетекает.

```python
a, b = Counter(), Counter()
a.tick()
print(a.value, b.value)
```

Здесь выводится `1 0`: `a.tick()` тронул только `a`. Один класс - сколько угодно независимых объектов. Понаблюдайте за этим вживую: нажимайте «создать экземпляр» и смотрите, как из одного чертежа выходят отдельные счётчики, каждый со своим адресом и своим значением `value`.

## self - это сам экземпляр, переданный явно

Самый частый вопрос новичка: откуда в каждом методе берётся `self` и почему его не передают при вызове. Разгадка в том, что `c.tick()` - это сокращение. Python разворачивает его в `Counter.tick(c)`: метод живёт на классе, а экземпляр перед точкой подставляется первым аргументом. Поэтому `self` - не магическое слово, а обычный параметр, в который Python кладёт «тот объект, на котором вызвали метод».

```python
c = Counter(10)
c.tick()
Counter.tick(c)
print(c.value)
```

Обе формы делают одно и то же; печатается `12`. Имя `self` - всего лишь соглашение, технически на его месте могло бы стоять любое имя, но менять его не стоит: весь Python-код читается в расчёте на `self`. Из этого же следует, почему `self.value` внутри `tick` и `c.value` снаружи - один и тот же атрибут: оба обращаются к одному объекту, просто под разными именами.

<Callout type="info">Запись `c.value` - не «переменная value внутри c». Это «найди на объекте, на который смотрит `c`, атрибут `value`». Если завтра на тот же объект будет смотреть и имя `d`, то `d.value` прочитает ровно то же самое.</Callout>

## Два имени - один объект или два объекта?

Раз `self` и `c` - это просто имена, направленные на объект, возникает вопрос: когда у нас правда два разных счётчика, а когда два имени одного. Присваивание `d = c` не создаёт копию - оно вешает на тот же объект второй ярлык.

```python
c = Counter(10)
d = c
d.tick()
print(c.value, c is d)
```

Печатается `11 True`. `d.tick()` увеличил единственный объект, а `c` это увидел, потому что `c` и `d` - два имени одной вещи. Оператор `is` спрашивает именно про это: «это буквально один и тот же объект?». Сравните с настоящими двумя объектами:

```python
e = Counter(10)
f = Counter(10)
print(e.value == f.value, e is f)
```

Здесь `True False`: значения совпадают, но это разные объекты с разными адресами в памяти. На рисунке видно различие - имена `c` и `d` сходятся в одну ячейку, а `e` и `f` указывают на разные.

## Атрибуты экземпляра и атрибуты класса

Атрибуты, заданные через `self.x = ...` внутри `__init__`, принадлежат экземпляру: у каждого объекта свой `value`. Но атрибут можно объявить и на уровне класса - тогда он один на всех.

```python
class Pet:
    species = "неизвестно"

    def __init__(self, name):
        self.name = name
```

`species` записан в теле класса, поэтому `Pet.species` - одно значение, видимое из любого экземпляра. `name` задаётся в `__init__`, поэтому у каждого питомца он свой. Атрибуты класса удобны для общих констант и значений по умолчанию: их не нужно дублировать в каждом объекте.

```python
rex = Pet("Рекс")
murka = Pet("Мурка")
print(rex.name, murka.name, rex.species)
```

Печатается `Рекс Мурка неизвестно`: имена разные, вид общий. Когда Вы читаете `rex.species`, Python не находит `species` на самом объекте и поднимается искать его на классе - так общий атрибут оказывается доступен через любой экземпляр.

## Методы - это просто функции, лежащие на классе

Стоит закрепить: метод не «встроен» в каждый объект. Он один, лежит на классе, а через точку у экземпляра вызывается с подстановкой `self`. Атрибуты можно создавать и вне `__init__` - присваивание `self.x = ...` в любом методе добавляет атрибут на объект в момент выполнения.

```python
class Profile:
    def __init__(self, name):
        self.name = name

    def activate(self):
        self.active = True
```

После `__init__` у профиля есть только `name`. Атрибут `active` появится лишь после вызова `activate()` - до этого обращение к нему даст `AttributeError`. Поэтому хороший тон - заводить все атрибуты в `__init__`, пусть и со значением `None`: тогда у объекта предсказуемая форма с самого рождения.

## Когда поведение удивит

Атрибут класса, разделяемый всеми, - отличный инструмент, пока значение неизменяемое. С изменяемым (список, словарь) он превращается в ловушку: общий объект делят все экземпляры, и правка через один из них видна всем остальным.

```python
class Cart:
    items = []

    def add(self, product):
        self.items.append(product)

a = Cart()
b = Cart()
a.add("яблоко")
print(b.items)
```

Печатается `['яблоко']`, хотя в `b` ничего не клали. Список `items` создан один раз на уровне класса; `self.items.append(...)` не создаёт новый атрибут на экземпляре, а *меняет* тот самый общий список. Лекарство - заводить изменяемое состояние в `__init__`: перенесите `self.items = []` в конструктор, и у каждой корзины список станет собственным.

<Callout type="warning">Изменяемый атрибут класса (`items = []`, `cache = {}`) почти всегда баг. Неизменяемые значения по умолчанию на уровне класса безопасны; изменяемые - заводите в `__init__`.</Callout>

Второй сюрприз - сравнение. Без собственного `__eq__` оператор `==` для ваших объектов сводится к `is`, то есть к тождеству.

```python
print(Counter(5) == Counter(5))
```

Печатается `False`: значения одинаковые, но объекты разные, а равенство «по содержимому» Python из коробки не предполагает. Как научить класс осмысленному `==`, разбирается в разделе [[dunder-methods]].

## Типичные ошибки

- **Забыть `self` в сигнатуре метода.** Если написать `def tick():`, то при вызове `c.tick()` Python всё равно подставит экземпляр первым аргументом - и получите `TypeError: tick() takes 0 positional arguments but 1 was given`. Первый параметр любого метода экземпляра - `self`.
- **Изменяемый аргумент по умолчанию.** `def __init__(self, items=[])` создаёт список один раз при определении функции, и он становится общим для всех вызовов, где аргумент опущен, - тот же эффект, что у изменяемого атрибута класса. Пишите `items=None`, а внутри `self.items = items or []`.
- **Перепутать класс и экземпляр.** `c = Counter` (без скобок) присваивает `c` сам класс, а не объект по нему; `c.tick()` тогда падает, ведь некому быть `self`. Вызов класса - всегда со скобками: `Counter()`.

<Detail summary="Где физически лежат атрибуты">

У большинства объектов атрибуты хранятся в словаре `__dict__`. У экземпляра он содержит только то, что присвоено через `self.x = ...`; у класса - методы и атрибуты класса.

```python
c = Counter(10)
print(c.__dict__)
print('tick' in Counter.__dict__)
```

Печатается `{'value': 10}` и `True`: значение лежит на экземпляре, а метод `tick` - на классе. Когда Вы пишете `c.tick`, Python не находит `tick` в `c.__dict__`, поднимается к классу, находит там функцию и связывает её с `c`. Этот же подъём «экземпляр → класс» объясняет, почему атрибут класса виден через любой экземпляр.

</Detail>
