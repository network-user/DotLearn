# Классы и экземпляры

Class - это чертёж. Instance - конкретный объект, созданный по этому чертежу. К концу раздела Вы сможете определить класс, дать каждому экземпляру собственное состояние и вызывать методы, которые с этим состоянием работают.

## Определение класса

Ключевое слово `class` вводит новый тип. По соглашению имена классов пишут в `PascalCase`.

```python
class Counter:
    def __init__(self, start=0):
        self.value = start

    def tick(self):
        self.value += 1
```

`__init__` - конструктор: он выполняется один раз при создании экземпляра. `self` - неявный первый аргумент каждого метода; он указывает на экземпляр, с которым идёт работа.

## Создание экземпляров

Вызовите класс как функцию. Python передаст аргументы в `__init__`.

```python
c = Counter(10)
c.tick()
c.tick()
print(c.value)   # 12
```

Каждый экземпляр хранит собственные атрибуты. Два счётчика не делят состояние между собой.

```python
a, b = Counter(), Counter()
a.tick()
print(a.value, b.value)   # 1 0
```

Понаблюдайте за этим вживую: один class порождает сколько угодно экземпляров, и у каждого - своя память и свои значения атрибутов.

## Атрибуты экземпляра и атрибуты класса

Атрибуты, определённые внутри `__init__` через `self.x = ...`, принадлежат **экземпляру**. Атрибуты, определённые на уровне класса, принадлежат **классу** и разделяются всеми экземплярами.

```python
class Pet:
    species = "unknown"   # class attribute, shared

    def __init__(self, name):
        self.name = name  # instance attribute, per-object
```

`Pet.species` - одно значение, видимое из каждого экземпляра `Pet`. `Pet("Rex").name` задаётся для каждого экземпляра отдельно.

## Типичные ошибки

- Забыть `self` в списке параметров метода. Без `self` место вызова не сможет передать экземпляр.
- Изменить изменяемое значение по умолчанию на уровне класса (`items = []`) и удивиться, что все экземпляры его разделяют. Инициализируйте изменяемое состояние внутри `__init__`.
- Сравнить экземпляры через `==` и удивиться результату `False`. Без собственного `__eq__` Python сравнивает по идентичности - см. раздел [[dunder-methods]].
