# Композиция против наследования

Наследование моделирует отношение **is-a**: `Dog` - это `Animal`. Композиция
моделирует **has-a**: у `Car` есть `Engine`.

Ловушка наследования в том, что оно жёстко связывает два класса -
навсегда. Подклассы видят каждый метод, добавленный в базовый класс, а изменения
базы расходятся по всем потомкам. Композиция даёт свободу: мелкие части, которые можно заменять.

## Изношенная иерархия

<Callout type="warning" title="Усталость от наследования">
Когда у родительского класса есть метод, нужный *большинству*, но не всем
потомкам, это признак того, что Вы растягиваете is-a слишком сильно.
</Callout>

```python
class Vehicle:
    def drive(self): ...
    def refuel(self): ...

class ElectricVehicle(Vehicle):
    def refuel(self):
        raise NotImplementedError("electric vehicles don't refuel")
```

`NotImplementedError` - тот самый дурной запах. Базовый класс пообещал
возможность, которую подклассы не могут предоставить.

## Переформулируем через композицию

<PyDemo
  title="Has-a, а не is-a"
  code={`class CombustionEngine:
    def start(self):
        return "vroom"

class ElectricMotor:
    def start(self):
        return "whirr"

class Car:
    def __init__(self, propulsion):
        self.propulsion = propulsion

    def drive(self):
        return f"car says {self.propulsion.start()}"

print(Car(CombustionEngine()).drive())
print(Car(ElectricMotor()).drive())`}
/>

`Car` всё равно, *что* его приводит в движение, лишь бы у детали был метод
`start()`. Это **утиная типизация** плюс композиция.

Сравните оба подхода вживую: измените базовый класс слева и посмотрите, как
правка задевает всех наследников; затем замените деталь справа - остальная
система этого даже не заметит.

## Смешивайте, когда это оправдано

Композиция и наследование не враги. Небольшой базовый класс для общего
поведения плюс составные части для всего специфичного - часто правильное
решение. Следите за запахами:

- Наследование ради пары общих методов → лучше mixin или вспомогательный класс.
- `raise NotImplementedError` в подклассе → иерархия неверна.
- Глубина больше двух уровней → проведите ревизию. Каждый уровень умножает связанность.
