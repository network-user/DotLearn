# Composition vs inheritance

Inheritance models **is-a**: a `Dog` is an `Animal`. Composition models
**has-a**: a `Car` has an `Engine`.

The trap with inheritance is that it ties two classes together tightly —
forever. Subclasses see every method the base adds, and base changes ripple
into every descendant. Composition gives you a knob: small parts, swappable.

## A worn-out hierarchy

<Callout type="warning" title="Inheritance fatigue">
When the parent class has a method that *most* but not all children need,
that's a sign you're stretching is-a too far.
</Callout>

```python
class Vehicle:
    def drive(self): ...
    def refuel(self): ...

class ElectricVehicle(Vehicle):
    def refuel(self):
        raise NotImplementedError("electric vehicles don't refuel")
```

The `NotImplementedError` is the bad smell. The base class promised a
capability that subclasses cannot deliver.

## Reframe as composition

<PyDemo
  title="Has-a, not is-a"
  code={`class CombustionEngine:
    def start(self):
        return "vroom"

class ElectricMotor:
    def start(self):
        return "whirr"

class Car:
    def __init__(self, propulsion):
        self.propulsion = propulsion

    def drive(self):
        return f"car says {self.propulsion.start()}"

print(Car(CombustionEngine()).drive())
print(Car(ElectricMotor()).drive())`}
/>

`Car` doesn't care *what* powers it as long as the part has a `start()`
method. This is **duck typing** plus composition.

## Mix when it makes sense

Composition and inheritance aren't enemies. A small base class for shared
behaviour plus composed parts for everything specific is often the right
call. Watch for the smells:

- Forcing inheritance to share a couple of methods → prefer a mixin or a helper.
- `raise NotImplementedError` in a subclass → the hierarchy is wrong.
- More than two levels deep → audit. Each level multiplies coupling.

<KeyTakeaways
  items={[
    "Default to composition; reach for inheritance when the is-a really holds.",
    "Replace deep hierarchies with small, swappable parts.",
    "If a subclass has to disable a parent method, the model is wrong.",
  ]}
/>
