# Classes and instances

A class is a blueprint. An instance is one concrete object created from that blueprint. By the end of this section you can define a class, give each instance its own state, and call methods that operate on that state.

## Defining a class

The `class` keyword introduces a new type. Convention: use `PascalCase` for class names.

```python
class Counter:
    def __init__(self, start=0):
        self.value = start

    def tick(self):
        self.value += 1
```

`__init__` is the constructor — it runs once when an instance is created. `self` is the implicit first argument of every method; it refers to the instance being acted on.

## Creating instances

Call the class like a function. Python forwards the arguments to `__init__`.

```python
c = Counter(10)
c.tick()
c.tick()
print(c.value)   # 12
```

Each instance carries its own attributes. Two counters do not share state.

```python
a, b = Counter(), Counter()
a.tick()
print(a.value, b.value)   # 1 0
```

## Instance vs class attributes

Attributes defined inside `__init__` and assigned through `self.x = ...` belong to the **instance**. Attributes defined at class scope belong to the **class** and are shared across instances.

```python
class Pet:
    species = "unknown"   # class attribute, shared

    def __init__(self, name):
        self.name = name  # instance attribute, per-object
```

`Pet.species` is one value visible from every `Pet` instance. `Pet("Rex").name` is set per instance.

## Common mistakes

- Forgetting `self` in a method's parameter list. Without `self`, the call site cannot pass the instance.
- Mutating a class-level mutable default (`items = []`) and being surprised that all instances share it. Initialize mutable state inside `__init__` instead.
- Comparing instances with `==` and being surprised that it returns `False`. Without a custom `__eq__`, Python compares by identity — see the [[dunder-methods]] section.
