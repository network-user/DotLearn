# Dataclasses

A class that mostly carries data — coordinates, configurations, records —
usually means writing the same `__init__`, `__repr__`, and `__eq__` over
and over. The `@dataclass` decorator generates them.

## The pattern

<PyDemo
  title="Hand-rolled vs @dataclass"
  code={`from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

p = Point(3, 4)
q = Point(3, 4)
print(p)
print(p == q)`}
/>

You get `Point(x=3, y=4)` from `__repr__` and `True` from `__eq__`, all
without writing them.

## Defaults and `field`

Inline defaults work like regular function arguments. For **mutable
defaults** (lists, dicts), use `field(default_factory=...)` — otherwise
all instances will share one object.

<Callout type="warning" title="Mutable defaults">
This is a Python footgun, not a dataclass-specific one. `field(default_factory=list)`
gives every instance its own fresh list.
</Callout>

```python
from dataclasses import dataclass, field

@dataclass
class Cart:
    user_id: int
    items: list = field(default_factory=list)
```

Step through it and confirm every cart gets its own list:

<PyStepper
  title="Each instance gets its own list"
  code={`from dataclasses import dataclass, field

@dataclass
class Cart:
    user_id: int
    items: list = field(default_factory=list)

a = Cart(1)
b = Cart(2)
a.items.append("book")
print(a.items, b.items)`}
/>

## Frozen — make it hashable

`@dataclass(frozen=True)` makes instances immutable. This also makes them
**hashable**, so you can put them in `set`s and use them as `dict` keys.

<PyDemo
  title="Frozen dataclass"
  code={`from dataclasses import dataclass

@dataclass(frozen=True)
class Color:
    r: int
    g: int
    b: int

palette = {Color(255, 0, 0), Color(0, 255, 0), Color(255, 0, 0)}
print(len(palette))`}
/>

Two `Color(255, 0, 0)` collapse to one entry — equality + hashability
working together.

## When *not* to use a dataclass

- Heavy invariants in `__init__` that don't map to simple defaults.
- Lots of derived methods that overshadow the data shape.
- Performance-critical inner loops where attribute access overhead matters
  (rare, but real).
