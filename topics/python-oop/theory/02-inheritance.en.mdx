# Inheritance and super()

Inheritance lets one class reuse the behavior of another. A subclass starts with every attribute and method of its base class, then adds or overrides on top.

## Defining a subclass

Put the base class in parentheses after the subclass name.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

class Dog(Animal):
    def speak(self):
        return f"{self.name} barks"
```

`Dog` inherits `__init__` and `name` for free. It overrides `speak`.

```python
d = Dog("Rex")
print(d.speak())   # Rex barks
```

## Calling the parent with super()

`super()` returns a proxy to the parent class. Use it when you want to extend, not replace, parent behavior.

```python
class LoudDog(Dog):
    def speak(self):
        return super().speak().upper()

print(LoudDog("Rex").speak())   # REX BARKS
```

`super()` also matters in `__init__` — without it, the parent's setup is skipped and instance attributes may be missing.

```python
class TimedCounter(Counter):
    def __init__(self, start=0):
        super().__init__(start)
        self.created_at = "today"
```

## Method resolution order

When you call `obj.method()`, Python walks the class chain looking for the first match. For single inheritance the chain is obvious: subclass → base → `object`. For multiple bases Python uses the **MRO** (method resolution order), which you can inspect with `Class.__mro__`.

## Common mistakes

- Forgetting `super().__init__(...)` in the subclass constructor. The parent's attributes never get set; later code crashes with `AttributeError`.
- Calling `super(self).method()` instead of `super().method()` — the empty form is the modern, correct one.
- Overriding a method and breaking its contract. Subclasses should accept the same arguments and return a compatible value.
