# Наследование и super()

Наследование позволяет одному классу переиспользовать поведение другого. Подкласс получает все атрибуты и методы базового класса, а затем добавляет своё или переопределяет существующее.

## Определение подкласса

Укажите базовый класс в скобках после имени подкласса.

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        return f"{self.name} makes a sound"

class Dog(Animal):
    def speak(self):
        return f"{self.name} barks"
```

`Dog` бесплатно наследует `__init__` и `name`. Метод `speak` он переопределяет.

```python
d = Dog("Rex")
print(d.speak())   # Rex barks
```

## Вызов родителя через super()

`super()` возвращает прокси к родительскому классу. Используйте его, когда хотите расширить поведение родителя, а не заменить его.

```python
class LoudDog(Dog):
    def speak(self):
        return super().speak().upper()

print(LoudDog("Rex").speak())   # REX BARKS
```

`super()` важен и в `__init__` - без него инициализация родителя пропускается, и атрибуты экземпляра могут оказаться не созданы.

```python
class TimedCounter(Counter):
    def __init__(self, start=0):
        super().__init__(start)
        self.created_at = "today"
```

## Порядок разрешения методов

Когда Вы вызываете `obj.method()`, Python идёт по цепочке классов и берёт первое совпадение. Для одиночного наследования цепочка очевидна: подкласс → базовый класс → `object`. Для нескольких базовых классов Python использует **MRO** (method resolution order), который можно посмотреть через `Class.__mro__`.

## Типичные ошибки

- Забыть `super().__init__(...)` в конструкторе подкласса. Атрибуты родителя так и не создаются; дальнейший код падает с `AttributeError`.
- Написать `super(self).method()` вместо `super().method()` - современная и корректная форма именно пустая.
- Переопределить метод и нарушить его контракт. Подклассы должны принимать те же аргументы и возвращать совместимое значение.
