# Dunder methods

Dunder methods (double-underscore methods) are how a class hooks into built-in syntax and functions: `len(x)`, `x == y`, `print(x)`, `x[i]`. They are the integration points between your types and the rest of Python.

## Display: `__str__` and `__repr__`

`__str__` powers `str(x)` and `print(x)`. `__repr__` powers the REPL display and `repr(x)`, and should be unambiguous for debugging.

```python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

    def __str__(self):
        return f"({self.x}, {self.y})"
```

If you define only `__repr__`, `str(x)` falls back to it. Defining only `__str__` leaves the REPL showing `<Point object at 0x...>`.

## Equality: `__eq__`

By default `a == b` is identity comparison. Override `__eq__` to make value comparison work.

```python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y
```

Returning `NotImplemented` lets Python try `other.__eq__(self)` instead of declaring inequality outright.

## Length and iteration

Implement `__len__` to make `len(x)` work, and `__iter__` (or `__getitem__`) to make `for item in x` work.

```python
class Bag:
    def __init__(self, items):
        self._items = list(items)

    def __len__(self):
        return len(self._items)

    def __iter__(self):
        return iter(self._items)
```

## Common mistakes

- Defining `__eq__` without `__hash__`. Python silently sets `__hash__ = None`, breaking use in sets and dict keys. Either also define `__hash__` or accept the type is unhashable.
- Returning `False` from `__eq__` for an unrelated type. Return `NotImplemented` so Python can try the other side's `__eq__`.
- Confusing `__str__` with `__repr__`. `__str__` is for end users; `__repr__` is for debugging and should round-trip back to a constructor call when possible.
