# Dunder-методы

Dunder-методы (методы с двойным подчёркиванием) - это способ, которым класс подключается к встроенному синтаксису и функциям: `len(x)`, `x == y`, `print(x)`, `x[i]`. Это точки интеграции между Вашими типами и остальным Python.

## Отображение: `__str__` и `__repr__`

`__str__` отвечает за `str(x)` и `print(x)`. `__repr__` отвечает за вывод в REPL и `repr(x)` и должен быть однозначным для отладки.

```python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

    def __str__(self):
        return f"({self.x}, {self.y})"
```

Если определить только `__repr__`, `str(x)` будет использовать его как запасной вариант. Если определить только `__str__`, REPL продолжит показывать `<Point object at 0x...>`.

## Равенство: `__eq__`

По умолчанию `a == b` - сравнение по идентичности. Переопределите `__eq__`, чтобы заработало сравнение по значению.

```python
class Point:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y
```

Возврат `NotImplemented` даёт Python шанс попробовать `other.__eq__(self)` вместо того, чтобы сразу объявить объекты неравными.

## Длина и итерация

Реализуйте `__len__`, чтобы заработал `len(x)`, и `__iter__` (или `__getitem__`), чтобы заработал `for item in x`.

```python
class Bag:
    def __init__(self, items):
        self._items = list(items)

    def __len__(self):
        return len(self._items)

    def __iter__(self):
        return iter(self._items)
```

## Типичные ошибки

- Определить `__eq__` без `__hash__`. Python молча выставляет `__hash__ = None`, и тип перестаёт работать в множествах и как ключ словаря. Либо определите ещё и `__hash__`, либо примите, что тип нехэшируемый.
- Вернуть `False` из `__eq__` для постороннего типа. Возвращайте `NotImplemented`, чтобы Python мог попробовать `__eq__` другой стороны.
- Перепутать `__str__` и `__repr__`. `__str__` - для конечных пользователей; `__repr__` - для отладки и по возможности должен восстанавливаться обратно в вызов конструктора.
