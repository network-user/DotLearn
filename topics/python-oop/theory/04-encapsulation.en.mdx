# Encapsulation

Python has no `private` or `protected` keyword. Visibility is a **social
contract** expressed through naming. Read this concept as: *what is the
convention, and how do I add a real boundary when I need one?*

## The two leading-underscore conventions

There are two conventions, with different meanings:

- `_attr` — **single underscore**: "internal, please don't touch." Linters
  and IDEs respect it; the language does not.
- `__attr` — **double underscore**: triggers *name mangling*. Python
  rewrites `self.__attr` to `self._ClassName__attr` so subclasses cannot
  accidentally clobber it.

<Callout type="tip" title="Use name mangling sparingly">
Name mangling is for collision avoidance, not for "real privacy." If you
just want to discourage external use, a single underscore is enough.
</Callout>

## Property — the gate

When you want to expose a value but **control access**, replace the bare
attribute with a `property`. The instance still looks like it has an
attribute, but you get a function call on every read or write.

<PyDemo
  title="Property gate"
  code={`class Account:
    def __init__(self, balance):
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("balance cannot be negative")
        self._balance = value

a = Account(100)
print(a.balance)
a.balance = 50
print(a.balance)`}
/>

The getter looks like attribute access, the setter validates. If you ever
add this gate later, callers do not change.

## Name mangling in action

<PyStepper
  title="What __ does at runtime"
  code={`class Box:
    def __init__(self):
        self.__secret = 42

b = Box()
print(b._Box__secret)`}
/>

Step through and watch how the local `self.__secret` becomes the
mangled `_Box__secret` on the instance. The original attribute name no
longer exists.

## Rule of thumb

1. Default to **public**. Most state is public.
2. Add a leading `_` when the attribute is an implementation detail.
3. Use `@property` to add validation, lazy computation, or read-only fields.
4. Reach for `__name` only when subclasses might collide.
