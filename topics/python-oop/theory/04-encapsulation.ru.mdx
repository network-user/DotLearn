# Инкапсуляция

В Python нет ключевых слов `private` и `protected`. Видимость - это **социальный
договор**, выраженный через имена. Читайте этот раздел как ответ на вопрос:
*какова конвенция и как добавить настоящую границу, когда она нужна?*

## Две конвенции с ведущим подчёркиванием

Конвенций две, и значения у них разные:

- `_attr` - **одно подчёркивание**: «внутреннее, пожалуйста, не трогайте».
  Линтеры и IDE его уважают; язык - нет.
- `__attr` - **двойное подчёркивание**: включает *name mangling*. Python
  переписывает `self.__attr` в `self._ClassName__attr`, чтобы подклассы
  не могли случайно его затереть.

<Callout type="tip" title="Используйте name mangling умеренно">
Name mangling нужен для защиты от коллизий, а не для «настоящей приватности».
Если Вы просто хотите отговорить от внешнего использования, одного
подчёркивания достаточно.
</Callout>

Проверьте все три уровня сами - что произойдёт при обращении снаружи класса:

## Property - шлюз доступа

Когда нужно открыть значение наружу, но **контролировать доступ**, замените
обычный атрибут на `property`. Снаружи экземпляр по-прежнему выглядит так,
будто у него есть атрибут, но каждое чтение и запись проходят через вызов функции.

<PyDemo
  title="Шлюз через property"
  code={`class Account:
    def __init__(self, balance):
        self._balance = balance

    @property
    def balance(self):
        return self._balance

    @balance.setter
    def balance(self, value):
        if value < 0:
            raise ValueError("balance cannot be negative")
        self._balance = value

a = Account(100)
print(a.balance)
a.balance = 50
print(a.balance)`}
/>

Геттер выглядит как обращение к атрибуту, сеттер выполняет проверку. Если
добавить такой шлюз позже, вызывающий код менять не придётся.

## Name mangling в действии

<PyStepper
  title="Что __ делает во время выполнения"
  code={`class Box:
    def __init__(self):
        self.__secret = 42

b = Box()
print(b._Box__secret)`}
/>

Пройдите по шагам и посмотрите, как локальный `self.__secret` превращается
в искажённое имя `_Box__secret` на экземпляре. Атрибута с исходным именем
больше не существует.

## Практическое правило

1. По умолчанию делайте атрибуты **публичными**. Большая часть состояния публична.
2. Добавляйте ведущий `_`, когда атрибут - деталь реализации.
3. Используйте `@property` для валидации, ленивых вычислений и полей только для чтения.
4. Берите `__name` только тогда, когда возможны коллизии с подклассами.
