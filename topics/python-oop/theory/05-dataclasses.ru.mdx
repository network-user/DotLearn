# Dataclasses

Класс, который в основном носит данные - координаты, конфигурации, записи, -
обычно означает один и тот же `__init__`, `__repr__` и `__eq__`, написанные
снова и снова. Декоратор `@dataclass` генерирует их за Вас.

## Паттерн

<PyDemo
  title="Вручную против @dataclass"
  code={`from dataclasses import dataclass

@dataclass
class Point:
    x: float
    y: float

p = Point(3, 4)
q = Point(3, 4)
print(p)
print(p == q)`}
/>

Вы получаете `Point(x=3, y=4)` из `__repr__` и `True` из `__eq__`, не написав
ни одного из них.

## Значения по умолчанию и `field`

Обычные значения по умолчанию работают как аргументы функции. Для **изменяемых
значений по умолчанию** (списки, словари) используйте `field(default_factory=...)` -
иначе все экземпляры будут делить один объект.

<Callout type="warning" title="Изменяемые значения по умолчанию">
Это ловушка самого Python, а не только dataclass. `field(default_factory=list)`
даёт каждому экземпляру собственный свежий список.
</Callout>

```python
from dataclasses import dataclass, field

@dataclass
class Cart:
    user_id: int
    items: list = field(default_factory=list)
```

Пройдите по шагам и убедитесь, что у каждой корзины свой собственный список:

<PyStepper
  title="Каждому экземпляру - свой список"
  code={`from dataclasses import dataclass, field

@dataclass
class Cart:
    user_id: int
    items: list = field(default_factory=list)

a = Cart(1)
b = Cart(2)
a.items.append("book")
print(a.items, b.items)`}
/>

## Frozen - делаем тип хэшируемым

`@dataclass(frozen=True)` делает экземпляры неизменяемыми. Заодно они становятся
**хэшируемыми**: их можно класть в `set` и использовать как ключи `dict`.

<PyDemo
  title="Frozen dataclass"
  code={`from dataclasses import dataclass

@dataclass(frozen=True)
class Color:
    r: int
    g: int
    b: int

palette = {Color(255, 0, 0), Color(0, 255, 0), Color(255, 0, 0)}
print(len(palette))`}
/>

Два `Color(255, 0, 0)` схлопываются в одну запись - равенство и хэшируемость
работают вместе.

## Когда dataclass *не* подходит

- Тяжёлые инварианты в `__init__`, которые не сводятся к простым значениям по умолчанию.
- Много производных методов, за которыми не видно формы данных.
- Критичные к производительности внутренние циклы, где накладные расходы
  на доступ к атрибутам имеют значение (редко, но бывает).
