# Управляющие конструкции: if, for, switch

Если вы пришли из C, Java или JavaScript, первое, что удивит в управляющих конструкциях Go, - чего в них нет. Нет цикла `while`. Нет `do-while`. Нет скобок вокруг условий. Нет провала между ветками `switch`. Go свёл ветвление и повторение к трём словам - `if`, `for`, `switch` - и убрал у них всё, что порождало ошибки в других языках. Меньше форм означает меньше мест, где можно споткнуться. Разберём каждую.

## if: условие без скобок, но с обязательными фигурными

Условие пишут без круглых скобок, а фигурные скобки обязательны даже для одной строки - это отсекает целый класс ошибок, когда забытая скобка молча меняет смысл кода.

```go
if age >= 18 {
    fmt.Println("совершеннолетний")
} else {
    fmt.Println("нет")
}
```

Особая примета Go - `if` с инициализатором. Перед условием можно объявить переменную, и она будет видна только внутри `if` и его `else`:

```go
if n := len(name); n > 10 {
    fmt.Println("длинное имя,", n, "символов")
}
```

Переменная `n` живёт лишь внутри этого ветвления и не засоряет остальную функцию. Такой приём особенно хорош с функциями, которые возвращают значение и признак ошибки, - но об этом в уроке про ошибки.

## for: единственный цикл на все случаи

В Go ровно один цикл - `for`. Он же играет роль `while` и `do-while` из других языков, просто меняя форму.

Классическая форма со счётчиком знакома всем:

```go
sum := 0
for i := 1; i <= 5; i++ {
    sum += i
}
```

Уберите инициализатор и пост-выражение, оставьте только условие - получится аналог `while`:

```go
n := 8
for n > 1 {
    n = n / 2
}
```

Уберите и условие - получится бесконечный цикл, из которого выходят по `break`:

```go
for {
    line := readLine()
    if line == "" {
        break
    }
    process(line)
}
```

Ключевые слова `break` и `continue` работают как везде: первое прерывает цикл, второе переходит к следующей итерации.

## switch: без break и без случайного провала

В C и Java забытый `break` внутри `switch` - классический баг: выполнение «проваливается» в следующую ветку. Go разорвал эту традицию. Каждая ветка `case` завершается сама собой, `break` писать не нужно.

```go
switch day {
case "сб", "вс":
    fmt.Println("выходной")
case "пт":
    fmt.Println("почти выходной")
default:
    fmt.Println("рабочий день")
}
```

Одна ветка может перечислять несколько значений через запятую, как `case "сб", "вс"` выше. А `switch` без выражения превращается в аккуратную замену длинной лестницы `if-else if`:

```go
switch {
case score >= 90:
    grade = "A"
case score >= 75:
    grade = "B"
default:
    grade = "C"
}
```

Ветки проверяются сверху вниз, срабатывает первая подходящая. Поэтому более узкие условия ставят выше более широких.

<Callout type="tip" title="Провалиться всё-таки можно - но только по просьбе">
Если вам действительно нужно поведение «провалиться в следующую ветку», в Go есть ключевое слово fallthrough. Но оно требует явного написания, поэтому провал случается только когда вы этого хотите, а не когда забыли break. Умолчание безопасно, редкий случай - возможен.
</Callout>

## Типичные ошибки

**Искать `while`.** Его нет. Условный цикл - это `for` с одним лишь условием: `for x > 0 { ... }`.

**Ставить круглые скобки вокруг условия.** `if (x > 0) {` соберётся, но `go fmt` их не любит, а сообщество считает лишними. Пишите `if x > 0 {`.

**Ожидать провала в `switch`.** Из мира C приходит привычка, что без `break` выполнение течёт дальше. В Go каждая ветка замкнута; специально добавленное поведение - только через `fallthrough`.

**Пытаться использовать переменную из инициализатора `if` снаружи.** `if v := f(); v > 0 {}` объявляет `v` только внутри ветвления. За его пределами `v` уже не существует.

<Detail summary="Помеченные циклы: break сразу из двух вложенных for">

Иногда нужно выйти не из внутреннего цикла, а сразу из внешнего. Go позволяет пометить цикл меткой и прервать именно его:

```go
outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i*j > 2 {
            break outer
        }
    }
}
```

`break outer` выходит из обоих циклов разом. Без метки `break` прервал бы только внутренний `for`, и внешний продолжил бы крутиться. Тот же приём работает с `continue`.

</Detail>
