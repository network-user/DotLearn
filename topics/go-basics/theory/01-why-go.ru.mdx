# Зачем нужен Go и как он устроен

Представьте, что вы написали сетевой сервис и хотите отдать его коллеге. На Python вы просите поставить нужную версию интерпретатора и десяток пакетов; на Java - держать в системе JVM. В Go вы отдаёте один файл. Внутри него уже лежит всё: ваш код, стандартная библиотека, среда исполнения. Скопировали на сервер, запустили - работает. Go придумали в Google в 2009 году именно как ответ на боль больших команд: медленная компиляция, запутанные зависимости, код, который через год не может прочитать даже автор. Язык нарочно сделали маленьким и скучным - и в этой скуке его сила.

## Философия: меньше способов сделать одно и то же

Главная ставка Go - на простоту чтения, а не письма. В языке нет наследования классов, нет исключений, нет перегрузки операторов, нет дженериков в том виде, к которому вы привыкли в C++ (базовые появились лишь в версии 1.18). Спецификация языка помещается в один вечер чтения. Это осознанный размен: вы теряете часть выразительных трюков, но любой человек в команде читает чужой код без словаря идиом.

Go компилируется в машинный код заранее (*ahead-of-time*), а не исполняется интерпретатором построчно. Отсюда две вещи, которые вы почувствуете сразу: программа стартует мгновенно, и на сервере не нужно ничего, кроме самого бинаря. При этом компиляция быстрая - большой проект собирается секунды, а не минуты, и это тоже было целью с самого начала.

## Пакет, функция main и точка входа

Любой файл Go начинается с объявления пакета. Пакет - это единица организации кода: набор файлов в одной папке, которые видят внутренние имена друг друга. Программа, которую можно запустить, живёт в специальном пакете `main` и обязана иметь функцию `main` - именно с неё начинается выполнение.

```go
package main

import "fmt"

func main() {
    fmt.Println("Привет, Go")
}
```

Разберём по строкам. `package main` объявляет, что это исполняемая программа. `import "fmt"` подключает пакет стандартной библиотеки для форматированного ввода-вывода. `func main()` - точка входа. `fmt.Println` печатает строку и переводит строку. Обратите внимание: имя `Println` начинается с заглавной буквы. В Go это не стиль, а правило видимости - об этом чуть ниже.

## Заглавная буква решает, кто увидит имя

В Go нет ключевых слов `public` и `private`. Видимость имени определяется регистром первой буквы. Имя с заглавной (`Println`, `User`, `Sum`) экспортируется - его видно из других пакетов. Имя со строчной (`count`, `parse`) доступно только внутри своего пакета.

```go
package geometry

func Area(r float64) float64 {
    return pi * r * r
}

const pi = 3.14159
```

Здесь `Area` видна снаружи, а константа `pi` спрятана внутри пакета. Правило простое настолько, что о нём легко забыть: если из соседнего пакета «не виден» ваш тип или функция, первым делом проверьте регистр первой буквы.

## Инструменты, которые идут в комплекте

Go - это не только язык, но и набор команд, встроенных в одну утилиту `go`. Их немного, и вы будете пользоваться ими каждый день.

<Steps>
  <li><code>go run main.go</code> - скомпилировать и сразу запустить, ничего не сохраняя. Удобно во время разработки.</li>
  <li><code>go build</code> - собрать бинарь и положить его рядом. Это то, что вы отдаёте в продакшн.</li>
  <li><code>go fmt</code> - отформатировать код по единому стилю. Не по вашему, а по общему для всех.</li>
  <li><code>go test</code> - запустить тесты из файлов с суффиксом <code>_test.go</code>.</li>
</Steps>

Отдельного упоминания заслуживает `go fmt`. В большинстве языков команда годами спорит о табах, пробелах и месте фигурной скобки. В Go спор закрыт: есть один канонический формат, и `go fmt` приводит к нему любой код. Ревью перестают вязнуть в стиле и говорят по существу.

<Callout type="tip" title="go fmt не спрашивает вашего мнения - и это благо">
Единый формат означает, что весь код Go в мире выглядит одинаково. Открывая незнакомый проект, вы не тратите ни секунды на привыкание к чужим отступам. Инструмент отнимает свободу расставлять пробелы по-своему и взамен возвращает время.
</Callout>

## Что стоит знать заранее

Go строг там, где другие языки снисходительны. Неиспользованная переменная - это ошибка компиляции, а не предупреждение. Неиспользованный импорт - тоже ошибка. Поначалу это раздражает: закомментировали строку для отладки, и проект перестал собираться. Но у строгости есть цель - в рабочем коде не накапливается мёртвый груз из брошенных переменных и лишних зависимостей.

## Типичные ошибки

**Ожидать привычных возможностей.** Новичок из мира ООП ищет `class` и наследование и не находит. Go предлагает другой набор кубиков - структуры, методы и интерфейсы - и удобнее сразу учить его модель, чем переносить чужую.

**Считать, что раз язык простой, то и системы на нём простые.** Простой синтаксис не отменяет сложности конкурентности, сетевых протоколов и распределённых систем - Go как раз для них и создавали.

**Забывать про регистр имён.** «Почему функцию не видно из другого пакета» - в девяти случаях из десяти имя начинается со строчной буквы.

<Detail summary="Почему бинарь такой большой">

Простейшая программа на Go весит несколько мегабайт, хотя печатает одну строку. Причина в том, что бинарь статический: внутрь вшита вся стандартная библиотека, которую вы затронули, и рантайм со сборщиком мусора и планировщиком горутин. Вы платите размером файла за то, что на целевой машине не нужно вообще ничего - ни интерпретатора, ни разделяемых библиотек нужных версий. Для сервера, который годами крутится в одном контейнере, это выгодный размен.

</Detail>
