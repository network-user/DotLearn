# Функции, множественный возврат и defer

В большинстве языков функция возвращает одно значение, и если нужно вернуть ещё признак ошибки, приходится изворачиваться: заводить кортеж, кидать исключение или писать значение в поле объекта. В Go функция спокойно возвращает два значения, три, сколько нужно, и это не костыль, а основа стиля. Пара «результат и ошибка» пронизывает всю стандартную библиотеку. А чтобы гарантированно закрыть файл или освободить ресурс, есть `defer` - отложенный вызов, который сработает, что бы ни случилось внутри функции. Разберём и то, и другое.

## Объявление функции

Функция начинается с `func`, дальше имя, параметры в скобках и тип результата.

```go
func square(n int) int {
    return n * n
}
```

Если несколько параметров подряд одного типа, тип пишут один раз, у последнего:

```go
func add(a, b int) int {
    return a + b
}
```

Здесь `a` и `b` оба `int`. Мелочь, но в больших сигнатурах она заметно чище.

## Множественный возврат

Функция возвращает несколько значений - типы результата перечисляют в скобках.

```go
func divmod(a, b int) (int, int) {
    return a / b, a % b
}

quotient, remainder := divmod(17, 5)
```

Здесь `quotient` станет `3`, а `remainder` - `2`. Самое частое применение - вернуть результат вместе с признаком успеха: `func find(key string) (string, bool)` отдаёт найденное значение и `true`, либо пустую строку и `false`. Идиома «значение и ошибка» из этого же ряда, и ей посвящён отдельный урок.

Если какой-то из возвращённых результатов вам не нужен, отправьте его в пустой идентификатор:

```go
_, remainder := divmod(17, 5)
```

## Именованные результаты

Результатам можно дать имена прямо в сигнатуре. Тогда это готовые переменные, а `return` без аргументов вернёт их текущие значения:

```go
func split(sum int) (a, b int) {
    a = sum * 4 / 9
    b = sum - a
    return
}
```

Голый `return` в конце возвращает `a` и `b`. Приём хорош для коротких функций; в длинных именованный `return` легко теряется из виду, и явные значения читаются лучше.

## defer: уборка, которая точно случится

`defer` откладывает вызов функции до момента, когда окружающая функция завершится, - неважно, обычным `return` или из-за паники. Классика жанра - открыли ресурс, тут же запланировали его закрытие:

```go
func readConfig(path string) {
    f := open(path)
    defer f.Close()
    parse(f)
}
```

`f.Close()` выполнится при выходе из `readConfig`, даже если `parse` завершится ошибкой. Уборка стоит рядом с открытием ресурса, и её невозможно забыть в одной из веток выхода.

Когда отложенных вызовов несколько, они выполняются в обратном порядке - последний запланированный первым (LIFO), как снятие тарелок со стопки.

## Одна ловушка defer, которую стоит увидеть заранее

Аргументы отложенного вызова вычисляются в момент, когда встречается `defer`, а не когда он срабатывает. Это удивляет почти всех:

```go
func demo() {
    i := 0
    defer fmt.Println(i)
    i = 10
    fmt.Println(i)
}
```

Программа напечатает `10`, а потом `0`. Строка `fmt.Println(i)` под `defer` зафиксировала значение `i` равным `0` сразу же, хотя сам вызов состоится в конце. К моменту выхода `i` уже `10`, но отложенный вызов держит старую копию аргумента.

<Callout type="warning" title="defer не бесплатен по времени жизни">
Отложенный вызов срабатывает при выходе из функции, а не из блока. Если вы в цикле на тысячу итераций открываете ресурс и пишете defer, все тысяча закрытий накопятся до конца функции, а не после каждой итерации. В таком случае оберните тело цикла в отдельную функцию, чтобы defer срабатывал на каждом шаге.
</Callout>

## Типичные ошибки

**Ждать, что `defer` выполнится в конце блока.** Он выполняется в конце *функции*. Внутри цикла отложенные вызовы копятся до самого выхода.

**Забыть, что аргументы `defer` вычисляются сразу.** Если нужно захватить актуальное значение переменной на момент выхода, оберните вызов в анонимную функцию: `defer func() { fmt.Println(i) }()`.

**Игнорировать часть возвращённых значений молча.** Нельзя просто «не взять» второй результат - Go потребует переменную или `_`. Это заставляет осознанно решить, что делать с каждым результатом.

<Detail summary="Функции - это значения">

Функцию в Go можно присвоить переменной, передать в другую функцию и вернуть из неё. Тип такой переменной - сигнатура: `var op func(int, int) int`. На этом строятся колбэки и стратегии:

```go
func apply(nums []int, f func(int) int) {
    for i := range nums {
        nums[i] = f(nums[i])
    }
}
```

`apply` принимает срез и функцию преобразования, применяя её к каждому элементу. Передать туда можно любую функцию с подходящей сигнатурой - хоть именованную, хоть анонимную прямо на месте вызова.

</Detail>
