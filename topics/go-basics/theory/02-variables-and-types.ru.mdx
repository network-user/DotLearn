# Переменные, типы и оператор :=

В динамических языках переменная - это просто ярлык, который сегодня держит число, а завтра строку. Go так не умеет и не хочет. Здесь у каждой переменной есть тип, зафиксированный навсегда, и компилятор проверяет его ещё до запуска. Попытка сложить строку с числом не доживёт до продакшна - она не пройдёт сборку. Взамен вы получаете уверенность: если проект собрался, то целые перемешанные с текстом типы вас уже не подведут в три часа ночи. Разберёмся, как объявлять переменные тремя способами, что такое нулевое значение и почему `int` не складывается с `float64`.

## Три способа объявить переменную

Полная форма выглядит так: ключевое слово `var`, имя, тип, значение.

```go
var age int = 30
var city string = "Казань"
```

Если значение задано сразу, тип обычно можно опустить - компилятор выведет его сам:

```go
var age = 30
var city = "Казань"
```

Внутри функции есть более короткая запись - оператор `:=`. Он объявляет переменную и сразу присваивает значение, выводя тип из правой части:

```go
age := 30
city := "Казань"
ratio := 1.5
```

Здесь `age` получает тип `int`, `city` - `string`, `ratio` - `float64`. Оператор `:=` работает только внутри функций; на уровне пакета доступна лишь форма с `var`. На практике `:=` - самая частая запись в теле функции, а `var` берут там, где переменную объявляют без начального значения.

## Нулевое значение: неинициализированного мусора не бывает

Объявите переменную без значения - она не станет «мусором» из памяти и не будет `undefined`. Go присваивает ей *нулевое значение* её типа.

У чисел нулевое значение - `0`, у строк - пустая строка `""`, у булевых - `false`, у указателей, слайсов и мап - `nil`. Это правило избавляет от целого класса ошибок «переменная объявлена, но не задана»: она всегда задана чем-то осмысленным. Счётчик, объявленный как `var count int`, честно начинается с нуля, и его не нужно инициализировать вручную.

## Основные типы

Целые числа бывают со знаком (`int`, `int8`, `int32`, `int64`) и без (`uint`, `uint8` и так далее). В обычной жизни берут `int` - его ширина совпадает с разрядностью машины. Дробные - `float64` (по умолчанию) и `float32`. Текст - `string`, он неизменяем и хранит байты в UTF-8. Логический тип - `bool`. Есть два псевдонима, которые встретятся часто: `byte` - это `uint8`, а `rune` - это `int32`, символ Unicode.

```go
var count int = 42
var price float64 = 199.90
var name string = "Ozon"
var active bool = true
```

## Константы

Значение, которое не должно меняться, объявляют через `const`. Константы вычисляются на этапе компиляции и не могут зависеть от того, что известно только во время работы программы.

```go
const maxRetries = 3
const pi = 3.14159
```

<Callout type="info" title="Нетипизированные константы гибче, чем кажется">
Константа `const big = 1 << 30`, объявленная без типа, подстраивается под контекст: её можно присвоить и в `int`, и в `float64`, и в `int64`. Тип у неё появляется только в момент использования. Именно поэтому числовые литералы в Go так удобно смешиваются, пока остаются константами.
</Callout>

## Строгость типов: чему вас научит компилятор

Go не делает неявных числовых преобразований. Сложить `int` и `float64` напрямую нельзя - это ошибка компиляции, а не тихое приведение, как в JavaScript. Нужное преобразование пишут явно:

```go
count := 3
ratio := 2.5
total := float64(count) * ratio
```

Без `float64(count)` проект не соберётся. Поначалу это кажется придиркой, но именно так Go не даёт незаметно потерять точность или получить неожиданный результат из-за скрытого приведения.

## Типичные ошибки

**Использовать `:=` вместо `=` при повторном присваивании.** Оператор `:=` объявляет *новую* переменную. Если `age` уже существует, второе `age := 40` в той же области видимости - ошибка; нужно простое `age = 40`.

**Смешивать числовые типы без преобразования.** `var x int = 5; y := x * 2.0` не соберётся: `2.0` - это `float64`. Приведите явно.

**Забывать, что `:=` не работает на уровне пакета.** Глобальную переменную объявляют только через `var`; короткая форма доступна лишь в теле функции.

**Оставлять объявленную переменную неиспользованной.** В Go это ошибка компиляции. Если переменная нужна «на будущее», её либо удаляют, либо присваивают в пустой идентификатор `_`.

<Detail summary="Почему пустой идентификатор _ так важен">

Знак подчёркивания `_` - это «мусорка» Go: место, куда сбрасывают ненужное значение. Функция вернула два результата, а вам нужен только первый? Второй присваивают в `_`, и компилятор перестаёт требовать, чтобы значение использовали. Это единственный законный способ сказать языку «да, я знаю про это значение и намеренно его игнорирую». Пустой идентификатор встретится вам постоянно - от обработки ошибок до перебора коллекций.

</Detail>
