# Заимствование, ссылки и срезы

Из прошлого урока остался неудобный осадок: если передать `String` в функцию, она заберёт владение, и снаружи значением больше не попользоваться. Гонять его туда-обратно через возврат - лапша. Нужен способ дать функции *посмотреть* на данные, а иногда и *поправить* их, но оставить владельца на месте. Такой способ называется заимствованием (borrowing), а инструмент - ссылка. И на тех же правилах вырастают срезы: способ сослаться не на всё значение, а на его кусок, не копируя ни байта.

## Ссылка одалживает доступ, но не владение

Ссылка - это `&` перед значением. Она даёт функции доступ к данным, но владельцем остаётся исходная переменная. Когда ссылка перестаёт использоваться, ничего не освобождается: одолженное просто возвращают хозяину.

```rust
fn length(s: &String) -> usize {
    s.len()
}

fn main() {
    let s = String::from("Rust");
    let n = length(&s);
    println!("{} {}", s, n);
}
```

Функция `length` принимает `&String` - ссылку, а не саму строку. Внутри она читает длину, но не владеет буфером, поэтому после вызова `s` в `main` жива и печатается. Программа выведет `Rust 4`. Сравните с прошлым уроком, где передача по значению «съедала» переменную: здесь `&` меняет всё.

## Изменяемая ссылка правит оригинал

Обычная ссылка `&` даёт только чтение. Чтобы через ссылку что-то изменить, нужны две вещи: сама переменная должна быть `mut`, и ссылку берут как `&mut`.

```rust
fn push_bang(s: &mut String) {
    s.push('!');
}

fn main() {
    let mut s = String::from("Rust");
    push_bang(&mut s);
    println!("{}", s);
}
```

`push_bang` получает `&mut String` и дописывает символ прямо в буфер владельца. После вызова `s` в `main` уже содержит `Rust!` - функция изменила оригинал, ничего не перемещая и не возвращая.

## Правило заёма: либо много читателей, либо один писатель

Здесь Rust ставит ограничение, которое поначалу удивляет, а потом оказывается сердцем его безопасности. В любой момент для одних данных можно иметь либо сколько угодно неизменяемых ссылок `&`, либо ровно одну изменяемую `&mut`, но не то и другое одновременно.

Зачем такая строгость? Потому что одновременный читатель и писатель - это гонка данных: один смотрит на значение, а другой в тот же миг его меняет, и читатель видит мусор. Сборщик мусора от этого не спасает, ручная дисциплина в C - забывается. Rust просто не даёт написать такой код: правило проверяется на этапе компиляции, поэтому целый класс ошибок конкурентности отсекается ещё до запуска.

<Callout type="warning" title="Ссылка не может пережить своего владельца">
Rust не позволит вернуть ссылку на локальную переменную функции: та умрёт в конце блока, а ссылка осталась бы «висячей». В C это классический источник багов (dangling pointer), в Rust - ошибка компиляции. Отсюда простое следствие: ссылка всегда указывает на живые данные.
</Callout>

## Срез: окно в кусок данных

Срез (slice) - это ссылка не на всё значение, а на его непрерывный участок. Он не владеет данными и ничего не копирует: внутри лежат указатель на начало куска и его длина. Срез строки записывают как `&s[start..end]`, где `end` не входит в диапазон.

```rust
fn main() {
    let s = String::from("hello world");
    let hello = &s[0..5];
    let world = &s[6..11];
    println!("{} {}", hello, world);
}
```

`hello` смотрит на первые пять байт, `world` - на байты с 6 по 10. Оба это `&str` - строковые срезы. Программа напечатает `hello world`, но важно, что новых строк не создавалось: оба среза указывают внутрь того же буфера, которым владеет `s`.

Срезы работают не только со строками. Кусок массива или вектора - это `&[T]`:

```rust
fn main() {
    let nums = [10, 20, 30, 40, 50];
    let mid = &nums[1..4];
    println!("{:?}", mid);
}
```

`{:?}` - это отладочный формат, он печатает срез как `[20, 30, 40]`. Диапазон `1..4` берёт элементы с индексами 1, 2 и 3; правая граница `4` не включается. Именно поэтому функциям обычно передают `&[T]`, а не `&Vec<T>`: срез принимает и вектор, и обычный массив, и другой срез.

## Типичные ошибки

**Забыть `&` и случайно переместить значение.** `length(s)` вместо `length(&s)` заберёт владение строкой. Компилятор подскажет, но привычка ставить `&` для «просто посмотреть» экономит время.

**Взять `&mut` без `mut` у переменной.** Изменяемую ссылку можно брать только с изменяемой переменной. `let s = ...; &mut s` не соберётся - нужно `let mut s`.

**Держать `&` и `&mut` одновременно.** Частый случай: читаете элемент вектора по ссылке и тут же пытаетесь его изменить. Правило заёма это запрещает; обычно помогает сузить область жизни первой ссылки или переставить операции.

**Резать строку по середине символа.** Индексы среза строки считаются в байтах. Для латиницы байт равен символу, а для кириллицы нет: `&s[0..1]` на строке «Привет» вызовет панику, потому что первая буква занимает два байта. Об этом подробно в уроке про строки.

<Detail summary="Почему &str удобнее, чем &String">

Строковый литерал `"привет"` в коде имеет тип `&str` - это срез, указывающий прямо в бинарь программы. `String` - владелец буфера на куче. Если функция принимает `&str`, ей можно передать и литерал, и срез `&s[..]` от `String` (последний приводится к `&str` автоматически). Функция, принимающая `&String`, теряет эту гибкость: литерал в неё не передать без обёртки. Поэтому идиома Rust - принимать `&str` там, где нужно только прочитать текст.

</Detail>
