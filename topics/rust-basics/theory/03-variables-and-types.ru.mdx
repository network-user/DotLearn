# Переменные, изменяемость и базовые типы

Новичок в Rust почти всегда пишет что-то вроде `let x = 5;`, а следующей строкой - `x = 6;`. Компилятор отвечает отказом: `cannot assign twice to immutable variable`. Первая реакция - раздражение: как это переменная не может измениться? Вторая, через пару уроков, - облегчение. В большинстве языков изменяемость - это то, что можно, если не отменить явно. В Rust наоборот: неизменяемость - это то, что дано по умолчанию, а изменяемость приходится включать осознанно, одним словом `mut`. Разница кажется мелкой, но именно она не даёт половине багов из категории «кто-то тихо поменял значение, пока я не смотрел» появиться на свет.

## Заперто по умолчанию, открыто по запросу

Проще всего думать об этом как о двух состояниях одной переменной. `let` даёт связывание, которое запечатано: значение записали один раз, и больше в эту ячейку ничего не положить. `let mut` снимает печать - теперь в переменную можно писать повторно, но это видно в объявлении, а не угадывается по контексту.

## Объявление и вывод типа

Тип переменной почти всегда можно не писать - компилятор выводит его из значения справа:

```rust
let x = 5;
let name = "Rust";
```

Если вывод неоднозначен или хочется зафиксировать тип явно, аннотация ставится после имени через двоеточие:

```rust
let x: i32 = 5;
let price: f64 = 199.9;
```

Изменяемая переменная объявляется так же, только с `mut` перед именем:

```rust
let mut counter = 0;
counter = counter + 1;
```

Без `mut` вторая строка не скомпилируется. Константы - отдельная сущность: они объявляются через `const`, требуют явного типа и обязаны вычисляться на этапе компиляции, а не во время работы программы.

```rust
const MAX_CONNECTIONS: u32 = 100;
```

Имя константы по соглашению пишут заглавными буквами с подчёркиванием. В отличие от `let`, `const` нельзя сделать `mut` - константа неизменяема по определению, без вариантов.

## Базовые типы

Целые числа делятся на знаковые (`i8`, `i16`, `i32`, `i64`, `i128`) и беззнаковые (`u8`, `u16`, `u32`, `u64`, `u128`). Если тип не указан явно и компилятор не может вывести иное, по умолчанию берётся `i32`. Отдельно стоит `usize` - целое без знака шириной с указатель на текущей платформе, о нём чуть ниже. Для дробных чисел два типа - `f32` и `f64`, и по умолчанию используется `f64`. Логический тип `bool` хранит `true` или `false`. Символьный тип `char` - это не байт, а скалярное значение Unicode шириной 4 байта, поэтому в `char` спокойно помещается `'я'` или `'字'`, а не только латиница.

```rust
let age: u8 = 30;
let ratio: f32 = 1.5;
let is_ready: bool = true;
let letter: char = 'я';
```

Кортеж собирает значения разных типов в одну переменную фиксированной длины. Доступ к элементам идёт через точку и индекс:

```rust
let pair = (1, 6.4);
let first = pair.0;
let second = pair.1;
```

Массив хранит элементы одного типа и фиксированной длины, известной на этапе компиляции. Тип массива записывается как `[тип; длина]`:

```rust
let numbers: [i32; 3] = [10, 20, 30];
let first = numbers[0];
```

## Затенение: новая переменная под старым именем

Затенение (shadowing) выглядит как повторное присваивание, но им не является:

```rust
let x = 5;
let x = x + 1;
let x = x * 2;
```

Каждая строка с `let x` создаёт новое связывание, которое временно перекрывает предыдущее с тем же именем. Итоговое значение `x` - `12`, и старые связывания просто больше не видны, хотя формально всё ещё существуют в памяти до конца области видимости. Отличие от `mut` принципиальное: `mut` меняет значение внутри одной и той же ячейки, не меняя её тип, а затенение заводит новую ячейку и может даже сменить тип:

```rust
let spaces = "   ";
let spaces = spaces.len();
```

Здесь `spaces` сначала строка, а после затенения - число. Так сделать с `mut` нельзя: `let mut spaces = "   "; spaces = spaces.len();` не скомпилируется, потому что тип переменной с `mut` зафиксирован при объявлении и меняться не может.

## Крайние случаи

Переполнение целых ведёт себя по-разному в зависимости от сборки. В debug-сборке (та, что используется при `cargo run` по умолчанию) переполнение `u8` за пределы `255` вызывает панику во время выполнения. В release-сборке (`cargo build --release`) паники не будет - значение обернётся по модулю разрядности типа, и это легко принять за корректный результат, если не проверять границы явно.

`char` - это кодовая точка Unicode, а не байт и не «буква» в бытовом смысле: некоторые видимые символы на экране на самом деле состоят из нескольких `char`, соединённых модификаторами. Для большинства учебных задач это не важно, но объясняет, почему `char` занимает 4 байта, а не 1.

Rust принципиально не делает неявных числовых приведений. `i32` и `i64` нельзя сложить напрямую, даже если один из них помещается в другой без потери точности:

```rust
let small: i32 = 10;
let big: i64 = 20;
let total = big + small as i64;
```

Без `as i64` компилятор откажется собирать код - несовпадение типов считается ошибкой, а не поводом для тихого приведения.

## Типичные ошибки

**Забыть `mut` и попытаться присвоить значение повторно.** Компилятор укажет точную строку и предложит добавить `mut` в объявление - это одна из самых дружелюбных ошибок в языке, но появляется она постоянно, пока привычка не выработалась.

**Путать затенение с изменением значения.** `let x = x + 1;` без `mut` работает, потому что это не изменение старой переменной, а создание новой с тем же именем. Начинающие иногда решают, что это доказывает необязательность `mut`, и удивляются, когда `x = x + 1;` без `let` в той же ситуации не компилируется.

**Ждать неявного приведения типов.** Код вида `let result = i32_value + i64_value;` не соберётся ни при каких обстоятельствах. Приведение всегда явное, через `as`, и это осознанное решение языка, а не недоработка.

<Detail summary="Зачем нужен usize и когда его брать">

`usize` - это целое без знака, ширина которого равна разрядности указателя на целевой платформе: 32 бита на 32-битных системах, 64 бита на большинстве современных. Rust использует его для всего, что связано с размером в памяти: индексы массивов и срезов, длины коллекций, результат `vec.len()`. Причина в том, что индекс не может быть отрицательным и не может превышать адресуемое пространство памяти - `usize` гарантирует ровно это, ничего не занимая сверх нужного и не давая слишком мало. Если переменная используется как индекс или хранит длину чего-либо, `usize` - правильный выбор по умолчанию; для остальной арифметики обычно берут `i32` или `i64`, в зависимости от диапазона значений.

</Detail>
