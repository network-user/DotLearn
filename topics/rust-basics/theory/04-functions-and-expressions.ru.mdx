# Функции и выражения

В большинстве языков `if` - это оператор: он принимает решение, но сам по себе ничего не «стоит». В Rust почти всё - выражение, то есть вычисляется в значение, которым можно тут же воспользоваться. `if` вычисляется в значение. Блок кода в фигурных скобках вычисляется в значение. Даже тело функции - это, по сути, один большой блок-выражение. Как только эта идея укладывается в голове, код на Rust начинает читаться иначе: функции пишутся короче, `return` встречается реже, а последняя строка блока начинает нести смысл, которого в других языках у неё никогда не было.

## Выражение против оператора

Оператор (statement) выполняет действие, но не возвращает значения - к нему нельзя присвоить `let y = ...`. Выражение (expression) вычисляется во что-то и это что-то можно передать дальше. Ключевое правило Rust: последняя строка блока без точки с запятой в конце - это значение всего блока. Если точку с запятой поставить, строка превращается в оператор, а блок начинает возвращать пустой тип `unit`, обозначаемый как `()`.

## Объявление функции и возврат без return

Функция начинается с `fn`, дальше имя, параметры с типами в скобках и, если нужно, тип результата после стрелки:

```rust
fn square(x: i32) -> i32 {
    x * x
}
```

Тело функции - это блок-выражение. Последняя строка `x * x` без точки с запятой становится значением, которое функция возвращает - никакого `return` не требуется. Если написать `x * x;` с точкой с запятой, функция перестанет компилироваться: сигнатура обещает `i32`, а блок из-за точки с запятой вернёт `()`.

Ранний выход всё же нужен, когда результат готов до конца функции - тогда используется явный `return`:

```rust
fn abs(x: i32) -> i32 {
    if x < 0 {
        return -x;
    }
    x
}
```

Внутри `if` стоит `return -x;` с точкой с запятой - это обычный оператор возврата, который завершает функцию досрочно. Если условие ложно, выполнение доходит до `x` в конце - это финальное выражение блока функции, и оно становится результатом.

## Блок и if как выражения

Раз блок в фигурных скобках - выражение, его можно присвоить переменной прямо на месте, ограничив область видимости промежуточных вычислений:

```rust
let y = {
    let a = 3;
    let b = 4;
    a + b
};
```

Здесь `a` и `b` существуют только внутри блока, а `y` получает `7` - значение последней строки. `if` работает точно так же: обе ветки должны возвращать значения одного типа, и весь `if` вычисляется в это значение.

```rust
let n = -5;
let sign = if n > 0 { 1 } else { -1 };
```

`sign` получает `-1`. Веток может быть больше через `else if`, но во всех ветках, включая `else`, тип результата обязан совпадать - иначе компилятор не сможет вывести единый тип для `sign`.

Функция, которая ничего не возвращает по смыслу, на самом деле возвращает `()` - единичный тип, у которого одно-единственное значение. Так ведут себя, например, функции, заканчивающиеся вызовом `println!` без последующего выражения:

```rust
fn greet(name: &str) {
    println!("Привет, {}!", name);
}
```

Стрелку `-> ()` в таких сигнатурах не пишут - отсутствие стрелки и означает `()`.

## Крайние случаи

Самая частая неожиданность - лишняя точка с запятой после последнего выражения функции или блока. Она превращает выражение в оператор, и вместо ожидаемого `i32` функция начинает возвращать `()`. Компилятор в таком случае обычно указывает на конкретную строку и подсказывает убрать точку с запятой, но пока это правило не станет рефлексом, ошибка будет появляться регулярно, особенно у тех, кто привык ставить `;` в конце каждой строки по инерции из других языков.

## Типичные ошибки

**Лишняя точка с запятой в конце возвращаемого выражения.** `fn square(x: i32) -> i32 { x * x; }` не соберётся: сигнатура обещает `i32`, а блок с точкой с запятой отдаёт `()`. Решение - убрать точку с запятой у последней строки.

**Ждать, что у оператора есть значение.** Присвоить результат `let`-объявления или другого оператора нельзя: `let y = (let x = 5);` не существует как конструкция. Значение может быть только у выражения.

**Забыть указать тип возврата, когда функция что-то возвращает.** Без `-> Тип` в сигнатуре компилятор считает, что функция возвращает `()`, и любое непустое финальное выражение в теле вызовет ошибку несовпадения типов.

<Detail summary="Зачем Rust вообще различает выражения и операторы">

Разделение на выражения и операторы - не формальность, а то, что позволяет `if`, `match` и блокам работать как полноценные строительные блоки программы, а не только как управление потоком. В языках, где `if` - оператор, для «выбрать одно из двух значений» приходится заводить отдельный тернарный оператор вроде `condition ? a : b`. В Rust отдельный тернарный оператор не нужен - `if condition { a } else { b }` делает то же самое, оставаясь тем же самым `if`, который используется и для ветвления без результата. Та же логика распространяется на `match`: он вычисляется в значение и часто заменяет собой одновременно и переключатель, и функцию сопоставления. Цена этой гибкости - необходимость аккуратно следить за точками с запятой, потому что именно они решают, участвует строка в вычислении значения или нет.

</Detail>
