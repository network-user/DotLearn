# Структуры и методы

Собрать что-то похожее на объект - с полями и поведением - хочется в любом языке, а классов в Rust нет вовсе. Ответ языка обходится двумя простыми кубиками, каждый из которых отвечает ровно за одну вещь. `struct` описывает форму данных: какие поля, какого типа, в каком порядке. Блок `impl` рядом привязывает к этому типу поведение - методы, которые знают, как эти поля читать и менять. Вместе они дают почти всё, ради чего в других языках заводят классы, но без наследования, виртуальных таблиц и конструктора по умолчанию, который вызывается сам по себе.

## Данные отдельно, поведение отдельно

`struct` сама по себе ничего не умеет. Объявление структуры - это просто список именованных полей, а обращение к полю через точку - обычное чтение памяти, никакой логики за ним нет. Поведение живёт отдельно, в блоке `impl <Тип>`, и именно там объявляют функции, которые понимают, что делать с этими полями. Такое разделение непривычно для тех, кто раньше писал классы, где поля и методы объявлены в одном месте, но оно даёт свободу: `impl`-блоков для одного типа может быть несколько, а сама структура остаётся простым описанием памяти, без единой строчки логики внутри.

На рисунке `p` - это не указатель на объект где-то в куче, а сама структура целиком, с обоими полями внутри. Rust не заводит для простых структур отдельного заголовка «объекта»: `p.x` - это чтение поля `x` прямо по месту, без косвенных обращений и без рантайма, который решал бы, куда идти за данными.

## Объявление и создание

```rust
struct Point {
    x: i32,
    y: i32,
}
```

Поля перечисляют по имени и типу через запятую. Создать значение можно, указав все поля явно, по имени:

```rust
let p = Point { x: 3, y: 4 };
```

Обратиться к полю - через точку: `p.x` вернёт `3`, `p.y` - `4`. Инициализация по именам, а не по порядку, устойчива к рефакторингу: добавить поле в середину структуры можно, не переписывая существующие литералы, если у нового поля есть разумное значение по умолчанию через отдельный конструктор.

## Метод с `&self`

Функция становится методом, если объявить её внутри `impl` для типа и первым параметром указать приёмник. Самый частый вариант - `&self`, ссылка на текущий экземпляр, которая позволяет читать поля, не забирая владение структурой.

```rust
impl Point {
    fn sum(&self) -> i32 {
        self.x + self.y
    }
}
```

Вызов выглядит как в любом объектном языке: `p.sum()`. Rust сам разворачивает это в `Point::sum(&p)` - `self` внутри тела метода и есть та самая ссылка на `p`. После вызова `p` остаётся годной: метод только прочитал поля и ничего не забрал.

## `&mut self` меняет поля

Метод, который должен поменять структуру, берёт приёмник как `&mut self` - изменяемую ссылку. Вызвать такой метод можно только на переменной, объявленной с `mut`, - то же правило, что и для обычных изменяемых ссылок.

```rust
impl Point {
    fn shift(&mut self, dx: i32, dy: i32) {
        self.x += dx;
        self.y += dy;
    }
}
```

```rust
let mut p = Point { x: 3, y: 4 };
p.shift(1, 1);
```

После вызова `p.x` равен `4`, `p.y` равен `5`. Без `mut` у переменной `p` компилятор откажется собирать эту программу ещё до того, как дело дойдёт до самого метода.

<Callout type="info" title="Точка сама решает, какая нужна ссылка">
Вызывая `p.shift(1, 1)`, вы не пишете `(&mut p).shift(1, 1)` - Rust сам подставляет нужный вид ссылки по сигнатуре метода. Из-за этого удобства вызовы `p.sum()` и `p.shift(...)` на месте выглядят одинаково; понять, читает метод структуру или меняет её, можно только заглянув в объявление внутри `impl`.
</Callout>

## Ассоциированная функция-конструктор

Конструктора по умолчанию в Rust нет, и синтаксиса вроде `new Point()` тоже. Вместо этого пишут обычную функцию внутри `impl`, но без `self` в параметрах. Такую функцию называют ассоциированной - она привязана к типу целиком, а не к конкретному уже существующему значению.

```rust
impl Point {
    fn new() -> Point {
        Point { x: 0, y: 0 }
    }
}

let origin = Point::new();
```

Вызывают ассоциированную функцию не через точку на значении, а через `::` - так же обращаются к содержимому модуля. По соглашению такую функцию почти всегда называют `new`, но это именно соглашение сообщества, а не зарезервированное слово языка.

## Кортежные структуры

Иногда полям не нужны имена - тогда структуру объявляют как кортеж с именем типа:

```rust
struct Pair(i32, i32);

let pair = Pair(1, 2);
println!("{}", pair.0);
```

Поля кортежной структуры адресуют по номеру: `pair.0` и `pair.1`. Это удобно для лёгкой обёртки над одним или двумя значениями, где имена полей ничего не добавили бы к смыслу - например, для пары координат или обёртки над одиночным числом с собственным типом.

## Крайние случаи

Три формы приёмника значат три разных отношения к структуре. `&self` даёт только чтение и не мешает остальному коду читать её же в это же время. `&mut self` даёт право менять поля, но требует, чтобы переменная была объявлена как `mut`. Третья форма, просто `self` без `&`, забирает структуру целиком: метод потребляет значение, и после вызова исходная переменная больше не годна - ровно то же перемещение, что и при обычном присваивании. Такую форму берут редко, обычно для методов вида `into_...`, которые превращают одно значение в другое и не должны оставлять после себя недоделанный оригинал.

Ещё одна деталь, которая ловит новичков: поля структуры по умолчанию приватны за пределами модуля, где она объявлена. Снаружи модуля к `p.x` не обратиться напрямую, пока поле не помечено `pub`. В однофайловых примерах этого не видно, но в реальном коде так прячут внутреннее устройство типа за методами, оставляя снаружи только то, что явно решили показать.

## Типичные ошибки

**Забыть `&self` в сигнатуре метода.** Без приёмника компилятор решит, что это ассоциированная функция, и вызов `p.sum()` не найдёт метод точкой - придётся звать `Point::sum(&p)` явно, а это не то, чего обычно хотят.

**Ждать конструктор по умолчанию.** В Rust нет ни `Point()`, ни `new Point`. Пишут обычную ассоциированную функцию, по соглашению `new`, и вызывают её как `Point::new()`.

**Менять поле в методе с `&self`.** `&self` даёт только чтение; строка `self.x += 1` внутри такого метода не соберётся. Нужен `&mut self` в сигнатуре, а у вызывающей переменной - `mut`.

<Detail summary="new - это просто имя, а не ключевое слово">

Rust не резервирует `new` под конструкторы: это обычная ассоциированная функция, и назвать её можно как угодно - `Point::origin()`, `Point::from_tuple(t)`, что угодно осмысленное. `new` прижилось как соглашение, потому что читатель кода из почти любого другого языка сразу понимает назначение функции по имени. Отличие ассоциированной функции от метода не в имени, а в приёмнике: у ассоциированной функции нет `self`, она не привязана к уже существующему значению, и вызывают её через `Тип::имя`, а не через точку на переменной. Поэтому в одном `impl`-блоке спокойно уживаются оба вида сразу: `Point::new()` создаёт значение с нуля, а `p.sum()` работает с уже существующим `p`.

</Detail>
