# Векторы и строки

Массив в Rust имеет длину, зафиксированную в его типе: `[i32; 3]` и `[i32; 4]` даже не один и тот же тип. Это тесно ровно в тот момент, когда число элементов заранее неизвестно, а именно так почти всегда и бывает в реальном коде. Нужен контейнер, который умеет расти, и Rust даёт для этого `Vec`. А со строками ситуация ещё интереснее: они устроены хитрее, чем кажется на первый взгляд, и попытка взять из строки «третью букву» простым индексом обычно даже не компилируется.

## Указатель, длина, ёмкость

`Vec<T>` устроен как растущий массив на куче, которым владеет переменная: внутри лежат указатель на буфер, текущая длина (`len`) и выделенная ёмкость (`cap`). Пока элементов помещается в выделенный буфер, `push` просто дописывает данные и увеличивает `len`; когда `len` догоняет `cap`, Rust выделяет новый буфер побольше, переносит туда старые данные и освобождает прежний. `String` устроена точно так же изнутри, только хранит не произвольные `T`, а текст в кодировке UTF-8. А `&str` ничего не хранит и ничем не владеет: это срез, окно на уже существующий текст, будь то часть `String` или строковый литерал в бинарнике программы.

На рисунке у `v` есть свободное место: `len` равен трём, `cap` равен четырём, значит один `push` ещё поместится в текущий буфер без перевыделения памяти; следующий за ним потребует уже новый, больший буфер.

## Рабочие примеры

Вектор создают макросом `vec!`, растят через `push`, читают по индексу:

```rust
let mut v = vec![1, 2, 3];
v.push(4);
println!("{}", v[0]);
```

Перебор по ссылке не заставляет вектор отдавать владение элементами:

```rust
let mut total = 0;
for x in &v {
    total += x;
}
```

То же самое компактнее через итератор и `sum`:

```rust
let total: i32 = v.iter().sum();
```

`String` растят похожими методами: `push_str` дописывает срез, а `+` или `format!` склеивают несколько кусков:

```rust
let mut s = String::from("Rust");
s.push_str(" на практике");
println!("{}", s);
```

Метод `len()` у строки считает не буквы, а байты, и на кириллице это заметно:

```rust
let word = String::from("Привет");
println!("{}", word.len());
```

Программа напечатает `12`, хотя в слове «Привет» шесть букв: каждая кириллическая буква в UTF-8 занимает два байта. Перебор по символам, а не по байтам, делает метод `chars`:

```rust
for c in word.chars() {
    print!("{} ", c);
}
```

Этот цикл напечатает ровно шесть символов, каждый на своём месте.

## Крайние случаи

Индекс за границей вектора паникует в рантайме, а не возвращает `None` или пустое значение: `v[10]` на трёхэлементном векторе аварийно завершит программу с сообщением о выходе за границы.

Строку нельзя проиндексировать по номеру символа: `word[0]` даже не пройдёт проверку типов, потому что у `String` оператор `[]` не реализован для одиночного индекса. Причина в переменной ширине UTF-8: символ может занимать от одного до четырёх байт, и «символ номер 0» не всегда означает «байт номер 0», а быстрого способа посчитать байтовое смещение произвольного символа без прохода по строке не существует.

Метод `len()` возвращает число байт, а не число символов, и для текста с кириллицей, эмодзи или иероглифами эти числа расходятся. Срез строки по байтовым границам, которые приходятся на середину многобайтового символа, тоже паникует: `&word[0..1]` на слове «Привет» упадёт, потому что первая буква занимает целых два байта, а срез просит только один.

## Типичные ошибки

**Пытаться индексировать `String` по номеру буквы.** Такой код просто не соберётся; чтобы получить n-ю букву, вызывают `word.chars().nth(n)`, который вернёт `Option<char>` и учтёт многобайтовость.

**Считать `len()` числом символов.** Для ASCII-текста байты и символы совпадают, и ошибка долго остаётся незаметной, пока в строку не попадёт кириллица или эмодзи.

**Менять вектор, пока держишь ссылку на его элемент.** `let first = &v[0]; v.push(5);` не соберётся: `push` может перевыделить буфер и сделать старый указатель `first` висячим, а правило заёма из прошлого урока не позволяет держать `&v[0]` и изменяемый доступ к `v` одновременно.

<Detail summary="Байты, скалярные значения и графемные кластеры">

UTF-8 выбрали для строк Rust не случайно: это самая распространённая кодировка в вебе и файловых форматах, и она совместима с ASCII на уровне байтов. Но у «одного символа» в Юникоде на самом деле три разных уровня. Байт служит единицей хранения; у кириллицы и большинства иероглифов на один видимый знак уходит по два-четыре байта. Скалярное значение Юникода (`char` в Rust) представляет собой уже полноценную кодовую точку, независимо от того, сколько байт она заняла в UTF-8. А графемный кластер обозначает то, что человек интуитивно назовёт «одной буквой на экране»: например, «й» иногда хранится как одна кодовая точка, а иногда как «и» плюс отдельный диакритический знак, и тогда для глаза это один символ, а для `chars()` уже два. Стандартная библиотека Rust даёт байты (`len`, срезы) и скалярные значения (`chars`) из коробки, а графемные кластеры для сложных языков (тайский, некоторые эмодзи-последовательности) требуют отдельного крейта вроде `unicode-segmentation`.
