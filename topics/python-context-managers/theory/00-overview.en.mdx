You opened a file, acquired a lock, or started a transaction - and halfway through work an exception fired. Control unwound up the stack but the resource stayed held: file not closed, lock still taken, database frozen mid-transaction. Once it is invisible; in a long-lived service such leaks accumulate until they take the process down. This topic is about tying resource release to leaving a block so you cannot forget - however the work ends.

**What you will learn to do.** Reliably release any resource with `with` without relying on manual `try/finally`; write your own context managers via the `__enter__`/`__exit__` protocol; build them as short functions with `@contextmanager`; suppress chosen exceptions and manage a variable number of resources with `suppress` and `ExitStack`; apply all of this to real locks, transactions, and temporary state changes.

**Topic route.** Four sections, each with a sandbox where code runs in the browser:

1. **The `with` statement** - what guarantee it gives and why it beats closing resources by hand.
2. **The `__enter__`/`__exit__` protocol** - the contract that makes an object a context manager and how to write one from scratch.
3. **`contextlib`** - building a context manager from an ordinary function via `yield` without multiplying classes.
4. **Real patterns** - locks, all-or-nothing transactions, and reversible state substitution in production.

**What you need beforehand.** Nothing to install: Python runs in the browser. Be comfortable with classes and methods from the OOP topic - the protocol builds on them. We cover the rest step by step.
