# What this course is about

Every time you open a website, submit a form, or hit an API from your code, the same thing happens under the hood: the client sends a text message following a strict set of rules, and the server replies with another text message. Those rules are called HTTP, and the entire web runs on them: browsers, mobile apps, microservices, payment-provider webhooks.

This course dissects the protocol itself, not the frameworks built on top of it. Once you understand what method idempotency means, why the `Accept` header exists, and what separates a 401 from a 403, every web framework stops being a black box: FastAPI, Django, and Express are just convenient handles on the same underlying HTTP fields.

Seven concepts, ordered from foundations to applied detail:

1. **Request and response** - what an HTTP message is made of.
2. **Methods and idempotency** - the protocol's verbs and the promises they make.
3. **Status codes** - how the server reports what happened.
4. **Headers and content negotiation** - metadata and format agreements.
5. **Cookies and sessions** - memory layered on top of a memoryless protocol.
6. **HTTPS and TLS** - what that padlock in the address bar actually buys you.
7. **Caching** - how the web avoids asking twice.

This topic is theory-first: there's no code to run, but you will read raw HTTP messages. That's a useful skill on its own - it's exactly the form traffic takes in browser DevTools, in curl, and in proxy logs.
