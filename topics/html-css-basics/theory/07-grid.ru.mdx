# Когда flexbox начинает не хватать

Flexbox прекрасно справляется с рядом или колонкой — одним направлением за раз. Но макет страницы целиком — шапка сверху, боковое меню слева, контент справа, подвал снизу — это уже сетка из строк и колонок одновременно, а не одна линия элементов. Пытаться собрать такую сетку вложенными друг в друга flex-контейнерами можно, но каждый уровень вложенности добавляет свой слой хрупких договорённостей о ширине. Grid устроен именно для этого случая: он раскладывает элементы по строкам и колонкам сразу, одним объявлением на родителе.

## Сетка определяется на родителе, а не на детях

```css
.page {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  gap: 16px;
}
```

`display: grid` включает сетку на контейнере. `grid-template-columns` и `grid-template-rows` описывают её строение: сколько колонок и строк, и какого они размера. В примере выше две колонки (боковая панель ровно 200px, всё остальное — вторая колонка) и три строки (высота по контенту, всё оставшееся пространство, снова высота по контенту — типичная заготовка для шапки, основной области и подвала). Дети контейнера — прямые потомки `.page` — автоматически занимают ячейки этой сетки по порядку, без единой строчки CSS на них самих.

## Единица fr: доля свободного пространства

`fr` (fraction) — единица, придуманная специально для grid: она означает не пиксели и не проценты, а долю оставшегося после фиксированных колонок свободного пространства.

```css
.layout { grid-template-columns: 1fr 2fr 1fr; }
```

Три колонки поделят пространство в пропорции 1:2:1 — средняя займёт ровно вдвое больше места, чем каждая из крайних, при этом расчёт не требует знать точную ширину контейнера в пикселях, в отличие от процентов, которые придётся пересчитывать при добавлении отступов. `fr` можно свободно смешивать с фиксированными значениями, как в примере с `200px 1fr` выше: сначала браузер выделяет место фиксированным колонкам, а оставшееся делит между `fr`-колонками пропорционально их числу.

## gap вместо margin-хаков

Как и во flexbox, `gap` в grid задаёт расстояние между ячейками сразу в двух направлениях, без ручных отступов на каждом элементе и без компенсации у крайних элементов сетки.

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
}
```

`repeat(3, 1fr)` — сокращённая запись для «три одинаковые колонки по 1fr», эквивалент `1fr 1fr 1fr`; для сетки из десяти одинаковых колонок разница в удобстве становится куда заметнее.

## Размещение элементов вручную: grid-column и grid-row

Дети сетки не обязаны занимать ровно одну ячейку по порядку — их можно явно растянуть на несколько строк или колонок через номера линий сетки.

```css
.hero {
  grid-column: 1 / 3;
  grid-row: 1 / 2;
}
```

Линии сетки нумеруются с единицы, начиная от левого (или верхнего) края; запись `1 / 3` означает «от первой линии до третьей», то есть элемент растягивается на две колонки. Такая точечная расстановка удобна для одного-двух особых элементов макета (широкий баннер сверху сетки карточек), но для всего макета целиком читать номера линий неудобно — для этого у grid есть более наглядный инструмент.

## Именованные области: grid-template-areas

```css
.page {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-areas:
    "sidebar header"
    "sidebar main"
    "sidebar footer";
}
.sidebar { grid-area: sidebar; }
.header  { grid-area: header; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }
```

`grid-template-areas` рисует макет буквально текстом: каждая строка в кавычках — это строка сетки, каждое слово в ней — имя области, которое должно занять эту ячейку. Повторяющееся имя `sidebar` в трёх строках подряд говорит браузеру растянуть боковую панель на все три строки сразу. Дальше каждый элемент получает своё имя через `grid-area`, и сопоставление с CSS-макетом становится настолько наглядным, что читается почти как ASCII-скетч самой страницы — большая редкость для CSS, где раскладку обычно приходится представлять в уме.

<Callout type="warning">
Каждая строка `grid-template-areas` обязана содержать одинаковое число слов, а прямоугольник, который занимает одно имя области, должен быть цельным — без дырок и без буквы «Г». Иначе браузер сочтёт объявление невалидным и проигнорирует его целиком.
</Callout>

## Резиновые сетки без медиазапросов: auto-fit и minmax

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 16px;
}
```

Эта строка — один из самых полезных «рецептов» grid: сетка карточек, которая сама решает, сколько колонок поместится, без единого `@media`. `minmax(220px, 1fr)` задаёт границы каждой колонки: не уже 220px, не шире её равной доли. `auto-fit` заставляет браузер вписать столько колонок такой ширины, сколько влезает в контейнер, растягивая последний ряд, если колонок получилось меньше, чем есть места. Сузьте окно браузера — колонок автоматически станет меньше, карточки останутся не уже 220px; расширьте — колонок прибавится сама собой.

<MarginNote label="auto-fit против auto-fill">
Похожее значение `auto-fill` отличается тонко: оно резервирует место под пустые колонки, если реальных элементов не хватает на всю ширину, из-за чего существующие карточки не растягиваются на освободившееся место. Для адаптивной сетки карточек почти всегда нужен именно `auto-fit`.
</MarginNote>

## Частые ошибки

- **Писать grid-template-areas с разным числом колонок в разных строках.** Правило целиком становится невалидным, и браузер использует запасное поведение сетки, будто области не задавали вовсе.
- **Путать fr с процентами.** `fr` делит именно свободное пространство после вычета фиксированных колонок и gap, а не всю ширину контейнера — в сетке `200px 1fr` вторая колонка получает не 100%, а «всё, что осталось после 200px».
- **Использовать grid там, где хватило бы flexbox, и наоборот.** Одна ось (ряд кнопок, меню) — задача flexbox; сетка из строк и колонок одновременно (макет страницы, галерея) — задача grid. Смешение подходов не ошибка сама по себе, но выбор не того инструмента усложняет код без необходимости.
- **Забыть, что элементы без явного grid-area просто занимают ячейки по порядку.** Если разметка в HTML идёт не в том порядке, что ожидается визуально, без явного размещения сетка расставит их не так, как задумано.

<Detail summary="Неявная сетка: grid-auto-rows и grid-auto-flow">
Если элементов больше, чем ячеек в явно заданной сетке, grid не «сломается» — он молча достроит дополнительные строки по правилам неявной сетки. `grid-auto-rows` задаёт высоту этих достроенных строк (по умолчанию она подстраивается под содержимое), а `grid-auto-flow: dense` заставляет алгоритм размещения заполнять более ранние пустые ячейки, если поздний элемент туда помещается, а не оставлять дыры в сетке ради строгого порядка.
</Detail>
