# Три карточки в ряд без единого float

Ещё десять лет назад выстроить три блока в ряд, растянуть их на равную ширину и красиво отцентровать по вертикали внутри общей полосы требовало `float`, «магических» отступов и хаков вроде `clearfix`. Flexbox превратил эту рутинную задачу в несколько строк CSS на родителе, ни одна из которых не трогает сами карточки напрямую. К концу раздела вы будете раскладывать ряды и колонки, не подбирая проценты методом тыка.

## Одна строка включает целую систему координат

Flexbox активируется одним свойством на родительском элементе — контейнере, и сразу меняет поведение всех его прямых детей — элементов.

```css
.row {
  display: flex;
}
```

```html
<div class="row">
  <div class="card">Раз</div>
  <div class="card">Два</div>
  <div class="card">Три</div>
</div>
```

Три `div.card`, которые без flex встали бы друг под другом (они блочные), выстраиваются в ряд автоматически — уже одна эта строка `display: flex` заменяет собой большую часть былых хаков. Дальше в игру вступают две оси: главная (main axis) — направление, вдоль которого выстраиваются элементы, и поперечная (cross axis) — перпендикулярная ей. По умолчанию главная ось горизонтальна, поперечная вертикальна, но `flex-direction: column` меняет их местами целиком: все свойства, которые вы выучите для главной оси, начинают работать по вертикали, а свойства поперечной — по горизонтали.

## Распределение вдоль главной оси: justify-content

`justify-content` управляет тем, как свободное место вдоль главной оси распределяется между элементами.

```css
.row { display: flex; justify-content: space-between; }
```

Ключевые значения: `flex-start` (по умолчанию, элементы жмутся к началу оси), `center` (группа по центру), `flex-end` (к концу), `space-between` (первый и последний элемент прижаты к краям, остальное место поровну между элементами), `space-around` (равные отступы вокруг каждого элемента, из-за чего крайние промежутки визуально вдвое меньше средних) и `space-evenly` (буквально одинаковые промежутки везде, включая края). Панель навигации с логотипом слева и ссылками справа — классический `justify-content: space-between` на одну строку, без единого `margin-left: auto` вручную.

## Выравнивание по поперечной оси: align-items

`align-items` делает для поперечной оси то же, что `justify-content` для главной — но с другим набором значений, потому что здесь важна ещё и высота самих элементов.

```css
.row { display: flex; align-items: center; height: 120px; }
```

Три карточки разной высоты, вставленные в контейнер с `align-items: center`, окажутся выровнены по вертикальному центру относительно друг друга, независимо от собственной высоты каждой — задача, которая до flexbox решалась через таблицы или пиксельные расчёты вручную. Значения те же по смыслу: `stretch` (по умолчанию — все элементы растягиваются на всю высоту контейнера, если явно не задана своя), `flex-start`, `center`, `flex-end`. Отдельный элемент можно выключить из общего правила через `align-self`, которое переопределяет `align-items` контейнера только для одного ребёнка.

## Гибкость элементов: grow, shrink, basis

Название «flexbox» — про гибкость самих элементов, а не только про их расстановку. Три свойства управляют тем, как элемент делит свободное или недостающее место с соседями, и почти всегда пишутся вместе через сокращение `flex`.

```css
.sidebar { flex: 0 0 240px; }  /* не расти, не сжиматься, базовая ширина 240px */
.content { flex: 1 1 0; }      /* расти и сжиматься, стартуя с нуля */
```

`flex-basis` — стартовый размер элемента до распределения свободного места, по смыслу похож на `width` вдоль главной оси. `flex-grow` — число, показывающее, во сколько раз охотнее элемент заберёт себе лишнее свободное место по сравнению с соседями: `flex-grow: 2` заберёт вдвое больше свободного пространства, чем сосед с `flex-grow: 1`. `flex-shrink` — то же самое, но при нехватке места: чем больше число, тем сильнее элемент сожмётся первым, освобождая место остальным. Сокращение `flex: 1` (эквивалент `flex: 1 1 0`) — самый частый рецепт «займи всё оставшееся место» для резиновой колонки рядом с колонкой фиксированной ширины.

## Перенос строк и gap

По умолчанию flex-контейнер пытается втиснуть все элементы в одну строку, сжимая их до предела. `flex-wrap: wrap` разрешает перенос на новую строку, когда места перестаёт хватать — необходимое свойство для сеток карточек, которые должны адаптироваться под ширину экрана.

```css
.grid-row {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}
```

`gap` задаёт расстояние между элементами напрямую, без `margin` на каждой карточке и без сопутствующей возни с лишним отступом у самого последнего элемента в ряду — типичная головная боль margin-based вёрстки, которую `gap` устраняет полностью.

## Частые ошибки

- **Перепутать justify-content и align-items.** Симптом: «выравниваю по центру, а элементы съезжают не туда» — на самом деле работает не та ось. Проверьте, что означает выбранный `flex-direction`.
- **Ставить flex: 1 на все элементы сразу и удивляться, что они одинаковой ширины.** Это ожидаемое поведение: при равном flex-grow свободное место делится поровну.
- **Забыть flex-wrap на сетке карточек.** Без него элементы будут сжиматься до микроскопической ширины на узком экране вместо переноса на новую строку.
- **Использовать margin вместо gap для расстояния между элементами.** Работает, но требует ручной компенсации у крайних элементов; gap делает то же самое одной строкой без побочных эффектов.

Если flex-wrap создаёт несколько строк, у контейнера появляется дополнительная свобода — как распределить сами эти строки по высоте контейнера, если он выше, чем нужно для их содержимого. За это отвечает `align-content`, с теми же значениями, что у `justify-content`. Свойство не действует, пока строка ровно одна: `align-items` уже полностью описывает такой случай.
