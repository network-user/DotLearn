# Кто перебил ваш стиль: селекторы и три счётчика

Вы дописали правило для кнопки, сохранили файл и обновили страницу. Кнопка не изменилась. Открываете DevTools, находите элемент, и вот оно: ваше свойство в списке есть, но перечёркнуто, а строчкой выше стоит чужое правило из другого конца проекта. Стиль написан, применён и тут же проигран кому-то, кого вы даже не звали.

Побеждает не то правило, что написано позже, и не то, что «точнее по смыслу». Браузер решает такие конфликты арифметикой: у каждого селектора есть вычислимый вес, специфичность, и свойство достаётся правилу с наибольшим весом. Хорошая новость: этот вес считается в уме за пару секунд. К концу статьи вы будете называть победителя до того, как заглянете в DevTools, и поймёте, почему один-единственный класс перевешивает хоть одиннадцать тегов.

## Селектор отвечает на вопрос «кому»

Правило CSS состоит из двух частей. Внутри фигурных скобок объявления: что именно поменять. Слева от них селектор: каким элементам это применить. Базовых видов селекторов немного, и большинство вы наверняка встречали.

```css
h1 { font-size: 32px; }
.card { border-radius: 12px; }
#menu { position: sticky; }
[disabled] { opacity: 0.5; }
a:hover { text-decoration: underline; }
p::first-line { font-variant: small-caps; }
```

Селектор по тегу выбирает все элементы одного типа: правило для `h1` покрасит каждый заголовок первого уровня на странице. Селектор класса начинается с точки и ловит элементы, у которых это слово присутствует в атрибуте `class`; один и тот же класс может висеть на сотне элементов, а у одного элемента может быть несколько классов, поэтому класс остаётся главным рабочим инструментом стилизации. Селектор `#menu` находит элемент с `id="menu"`; id обязан быть уникальным в пределах страницы, так что такой селектор всегда целится ровно в один элемент. Квадратные скобки проверяют атрибут: `[disabled]` поймает каждый выключенный элемент формы независимо от тега, а `[href^="/docs"]` только ссылки, адрес которых начинается с `/docs`.

Отдельно живут два вида «псевдо». Псевдокласс пишется с одним двоеточием и описывает состояние настоящего элемента: `:hover` под курсором, `:focus` в фокусе клавиатуры, `:first-child` первый среди детей своего родителя. Псевдоэлемент пишется с двумя двоеточиями и дорисовывает то, чего в разметке нет: `::before` вставляет виртуальный узел перед содержимым, а `::first-line` оборачивает первую строку абзаца, где бы она ни заканчивалась при текущей ширине окна.

## Комбинаторы собирают цепочки

Точность прицела появляется, когда простые селекторы соединяются в цепочки. Соединителей четыре, и два из них новички путают чаще всего.

```css
nav a { color: inherit; }
nav > a { padding: 8px 12px; }
h2 + p { margin-top: 4px; }
h2 ~ p { color: gray; }
```

Пробел это комбинатор потомка: `nav a` найдёт ссылку внутри `nav` на любой глубине, хоть через пять уровней вложенных списков. Стрелка `>` строже, это комбинатор ребёнка: `nav > a` возьмёт только те ссылки, что лежат в `nav` напрямую, без промежуточных обёрток. Плюс выбирает ближайшего следующего соседа: `h2 + p` это ровно один абзац, стоящий сразу после заголовка. Тильда шире: `h2 ~ p` возьмёт все абзацы после заголовка в пределах общего родителя.

<MarginNote label="Как читает браузер">Браузер сопоставляет селектор справа налево: для `nav a` он сначала берёт каждую ссылку и только потом проверяет, есть ли над ней `nav`. Ключевая часть селектора всегда правая.</MarginNote>

Запятая к комбинаторам не относится, это группировка: `h1, h2, h3` применяет один набор объявлений к трём независимым селекторам. Каждый участник группы живёт своей жизнью, и вес у каждого считается отдельно.

## Три счётчика, а не сумма баллов

Теперь главное. Когда два правила претендуют на одно свойство одного элемента, браузер сравнивает специфичности их селекторов. Считается она так: пройдите по селектору и заполните три счётчика. Первый: сколько в нём id-селекторов. Второй: сколько классов, атрибутов и псевдоклассов. Третий: сколько тегов и псевдоэлементов. Получается тройка вида a·b·c, и это не одно число, а кортеж из трёх независимых разрядов.

Сравнение идёт поразрядно, слева направо, как у номеров версий: 2.0.1 новее, чем 1.9.9, сколько бы девяток ни стояло в хвосте. В столбце id единица против нуля, и на этом всё закончилось. Побеждает `#menu a`, хотя простых селекторов в проигравшей цепочке больше.

## Четыре примера для тренировки

Прикиньте вес каждого селектора самостоятельно, а потом сверьтесь с разбором ниже.

```css
ul li a.external { color: seagreen; }
#header .logo { height: 40px; }
a[target="_blank"]:hover { outline: 1px dotted; }
main article > p::first-line { font-weight: 600; }
```

Первый: три тега и один класс, итого 0·1·3. Второй: один id и один класс, 1·1·0. Третий: тег `a`, атрибут `[target="_blank"]` и псевдокласс `:hover`, вместе 0·2·1. Четвёртый: три тега и псевдоэлемент `::first-line`, 0·0·4; комбинатор `>` в счёт не идёт. Теперь расставим по убыванию силы: впереди 1·1·0, за ним 0·2·1, следом 0·1·3, в хвосте 0·0·4. Заметьте: селектор с самой длинной записью оказался слабейшим, потому что все его части лежат в младшем разряде.

## Что весит ноль и кого не победить

Универсальный селектор `*` и все комбинаторы весят ровно ноль: цепочка `* > *` это по-прежнему 0·0·0, а `nav > a` весит столько же, сколько `nav a`. Комбинатор меняет то, какие элементы будут найдены, но не вес селектора.

Псевдокласс `:not()` устроен хитрее. Сам он не добавляет ничего, но его аргумент считается как обычно: `p:not(.intro)` весит 0·1·1, тег плюс класс внутри скобок. Забыв про это, легко удивиться, почему «отрицающий» селектор вдруг оказался тяжелее простого соседа.

Самый полезный сюрприз раздела: разряды не переполняются. Селектор из одиннадцати тегов весит 0·0·11 и проигрывает одному классу с его 0·1·0. Сколько бы вы ни накапливали в младшем счётчике, в старший это не перетечёт. Поэтому «пересилить» класс, дописывая теги в цепочку, невозможно в принципе: селектор растёт, а старшие разряды стоят на месте.

<Callout type="warning">
Стиль в атрибуте `style` вообще не участвует в соревновании селекторов: инлайн-объявление сильнее любой специфичности. Перебить его из файла может только `!important`. Это одна из причин, по которым инлайн-стили считаются плохим тоном: они выключают обычные механизмы CSS.
</Callout>

## Частые ошибки

- **Чинить проигравший селектор через `!important`.** Работает мгновенно, потому и опасно: следующему исключению понадобится собственный `!important`, и правила начинают перекрикивать друг друга. Правильный ход: посмотреть в DevTools, кто победил, и либо ослабить победителя, либо поднять свой вес на минимальный шаг, добавив один класс.
- **id-селекторы в стилях.** Вес 1·0·0 не перебить никаким количеством классов, и каждое исключение из «стиля по id» требует селектора ещё тяжелее. Так начинается война специфичности, в которой код только тяжелеет. Оставьте id якорям и JavaScript, а стилизуйте классами.
- **Путать `nav a` и `nav > a`.** Пробел не похож на оператор, поэтому разница ускользает. Если ссылки в меню обёрнуты в `li`, селектор `nav > a` не найдёт ничего, и правило молча не сработает. Читайте пробел как «где-то внутри», а стрелку как «сразу внутри».

<Detail summary=":is(), :where() и вес современных селекторов">
Функции-селекторы `:is()` и `:where()` группируют альтернативы: `:is(article, aside) p` короче, чем два селектора через запятую. Вес они считают по-разному. `:is()` берёт специфичность самого тяжёлого из своих аргументов, даже если реально совпал лёгкий. `:where()` всегда даёт ноль, чем удобен для базовых стилей библиотек: такие правила легко перекрыть одним классом, не вступая в войну специфичности. Тот же нулевой трюк используют современные CSS-сбросы.
</Detail>
