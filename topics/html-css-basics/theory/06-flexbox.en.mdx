# Three Cards in a Row Without a Single float

Ten years ago, lining up three blocks in a row, stretching them to equal widths, and centering them neatly along a shared strip meant reaching for `float`, mystery-meat margins, and hacks like `clearfix`. Flexbox turned that routine chore into a few lines of CSS on the parent, none of which touch the cards themselves directly. By the end of this section you'll be laying out rows and columns without guessing at percentages by trial and error.

## One Line Switches On an Entire Coordinate System

Flexbox turns on with a single property on the parent element, the container, and immediately changes how all of its direct children, the items, behave.

```css
.row {
  display: flex;
}
```

```html
<div class="row">
  <div class="card">Раз</div>
  <div class="card">Два</div>
  <div class="card">Три</div>
</div>
```

The three `div.card` elements, which without flex would stack on top of each other (they're block-level), line up in a row automatically, that one line of `display: flex` alone replaces most of the old hacks. From there, two axes come into play: the main axis, the direction items line up along, and the cross axis, perpendicular to it. By default the main axis runs horizontally and the cross axis vertically, but `flex-direction: column` swaps them entirely: every property you learn for the main axis now works vertically, and cross-axis properties switch to horizontal.

## Distributing Space Along the Main Axis: justify-content

`justify-content` controls how leftover space along the main axis gets distributed between items.

```css
.row { display: flex; justify-content: space-between; }
```

Key values: `flex-start` (the default, items huddle at the start of the axis), `center` (the group sits centered), `flex-end` (huddled at the end), `space-between` (first and last items pinned to the edges, remaining space split evenly between items), `space-around` (equal space around each item, which makes the outer gaps look half the size of the inner ones), and `space-evenly` (literally identical gaps everywhere, edges included). A nav bar with a logo on the left and links on the right is the classic `justify-content: space-between` in a single line, no manual `margin-left: auto` required.

## Aligning Along the Cross Axis: align-items

`align-items` does for the cross axis what `justify-content` does for the main one, but with a different set of values, because here the items' own height matters too.

```css
.row { display: flex; align-items: center; height: 120px; }
```

Three cards of different heights, dropped into a container with `align-items: center`, end up vertically centered relative to each other, regardless of each card's own height, a problem that used to require tables or manual pixel math before flexbox. The values carry the same meaning: `stretch` (the default, every item stretches to fill the container's full height unless it has a height of its own), `flex-start`, `center`, `flex-end`. A single item can opt out of the shared rule with `align-self`, which overrides the container's `align-items` for just that one child.

<PullQuote>justify-content works along the main axis, align-items works across it. Mixing the two up is the single most common flexbox mistake beginners make.</PullQuote>

## Item Flexibility: grow, shrink, basis

The name "flexbox" is about the flexibility of the items themselves, not just how they're arranged. Three properties control how an item shares leftover or missing space with its neighbors, and they're almost always written together through the `flex` shorthand.

```css
.sidebar { flex: 0 0 240px; }  /* don't grow, don't shrink, base width 240px */
.content { flex: 1 1 0; }      /* grow and shrink, starting from zero */
```

`flex-basis` is an item's starting size before leftover space gets distributed, conceptually similar to `width` along the main axis. `flex-grow` is a number showing how eagerly an item claims leftover free space compared to its neighbors: `flex-grow: 2` claims twice as much free space as a neighbor with `flex-grow: 1`. `flex-shrink` works the same way but for a shortage of space: the bigger the number, the more aggressively that item shrinks first, freeing up room for the rest. The shorthand `flex: 1` (equivalent to `flex: 1 1 0`) is the go-to recipe for "take all remaining space," perfect for a flexible column sitting next to a fixed-width one.

## Wrapping and gap

By default a flex container tries to cram every item onto one line, squeezing them as far as it can. `flex-wrap: wrap` allows items to wrap onto a new line once space runs out, a must for card grids that need to adapt to screen width.

```css
.grid-row {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}
```

`gap` sets the distance between items directly, without a `margin` on every card and without the usual fuss over the trailing gap on the last item in a row, a classic headache of margin-based layouts that `gap` eliminates outright.

## Common Mistakes

- **Mixing up justify-content and align-items.** The symptom: "I centered it, but the items are drifting the wrong way," when really the wrong axis is at work. Check what your `flex-direction` actually means.
- **Setting flex: 1 on every item and being surprised they're all the same width.** That's expected: with equal flex-grow, leftover space splits evenly.
- **Forgetting flex-wrap on a card grid.** Without it, items squeeze down to a microscopic width on a narrow screen instead of wrapping to a new line.
- **Using margin instead of gap for spacing between items.** It works, but it needs manual compensation at the edges; gap does the same job in one line with no side effects.

<Detail summary="align-content: aligning multiple lines as a group">
If flex-wrap creates several lines, the container gains an extra degree of freedom: how to distribute those lines themselves across the container's height, if it's taller than their combined content needs. `align-content` handles that, with the same values as `justify-content`. The property does nothing while there's only a single line, `align-items` already fully covers that case.
</Detail>
