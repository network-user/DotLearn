# Переменная в Python - это ярлык, а не коробка

Во многих языках переменная похожа на коробку: Вы кладёте в неё число, и число физически лежит «внутри» переменной. В Python всё устроено иначе, и пока эта разница не уложилась в голове, такие вещи как «почему изменился список, который я не трогал» или «почему два списка оказались одним» будут казаться магией. На самом деле магии нет: в Python переменная - это имя, наклеенное на объект, который живёт сам по себе где-то в памяти.

К концу раздела Вы будете уверенно отличать имя от объекта, понимать, что такое идентичность объекта и чем `is` отличается от `==`, видеть разницу между изменяемыми и неизменяемыми типами и не попадаться на двух классических ловушках: алиасинге и изменяемом аргументе по умолчанию.

## Имена, ссылки и объекты

В Python всё - объект: число `42`, строка `"hi"`, список, функция и даже сам класс. Каждый объект живёт в памяти и имеет три вещи: тип, значение и идентичность - уникальный «адрес», который не меняется за всю жизнь объекта. Функция `id()` возвращает этот идентификатор (в CPython - адрес объекта в памяти).

Переменная же ничего не хранит. Присваивание `x = 42` не «кладёт 42 в x», а создаёт объект `42` и привязывает к нему имя `x`. Имя - это ссылка, стрелка, направленная на объект. Несколько имён могут указывать на один объект.

```python
x = [1, 2, 3]
y = x
print(x is y)
```

Печатается `True`: `y = x` не копирует список, а вешает второе имя на тот же объект. Оба имени, `x` и `y`, - стрелки в одну и ту же ячейку памяти. На рисунке это два ярлыка на одном объекте, а не два объекта.

Раз `x` и `y` - один объект, изменение через одно имя видно через другое. Это и называется алиасингом.

```python
x = [1, 2, 3]
y = x
y.append(4)
print(x)
```

Печатается `[1, 2, 3, 4]`, хотя `x` Вы не трогали. Объект один, имён два.

## is против ==: идентичность против равенства

Эти два оператора отвечают на разные вопросы. `a is b` спрашивает: «это буквально один и тот же объект?» - то есть совпадают ли их `id()`. `a == b` спрашивает: «равны ли их значения?» - и для этого вызывает метод `__eq__`.

```python
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b, a is b)
```

Печатается `True False`: списки равны по содержимому, но это два разных объекта в памяти. Сравнивать через `is` нужно только тогда, когда Вас интересует именно тождество объекта - классический случай - проверка `x is None`.

<Callout type="warning">Никогда не пишите `x == None` или сравнение с синглтоном через `==`. Используйте `is None`: `None` существует в единственном экземпляре, и `is` проверяет идентичность напрямую, без вызова `__eq__`, который у чужого объекта мог бы солгать.</Callout>

## Изменяемые и неизменяемые объекты

Объекты делятся на два лагеря. Неизменяемые (immutable) - `int`, `float`, `str`, `bool`, `tuple`, `frozenset` - после создания нельзя изменить; любая «модификация» порождает новый объект. Изменяемые (mutable) - `list`, `dict`, `set`, экземпляры обычных классов - можно менять на месте, сохраняя идентичность.

```python
n = 10
print(id(n))
n += 1
print(id(n))
```

Два `id()` будут разными: `n += 1` не меняет объект `10`, а создаёт объект `11` и перевешивает имя `n` на него. Объект `10` остался прежним. Сравните со списком:

```python
nums = [1, 2]
before = id(nums)
nums += [3]
print(id(nums) == before)
```

Печатается `True`: `nums += [3]` для списка - это `extend` на месте, идентичность сохраняется. Один и тот же оператор `+=` ведёт себя по-разному для изменяемого и неизменяемого объекта - в этом корень многих сюрпризов.

## Копии: поверхностная против глубокой

Срез `a[:]`, `list(a)` и `a.copy()` создают поверхностную копию (shallow copy): новый список, но элементы - те же объекты, что и в оригинале. Пока элементы неизменяемы, разница незаметна. Но если внутри лежат изменяемые объекты, поверхностная копия делит их с оригиналом.

```python
import copy
original = [[1, 2], [3, 4]]
shallow = original[:]
shallow[0].append(99)
print(original[0])
```

Печатается `[1, 2, 99]`: внешний список скопирован, а вложенные - нет, они общие. Чтобы скопировать структуру целиком, нужна глубокая копия (deep copy), рекурсивно дублирующая всё содержимое.

```python
import copy
original = [[1, 2], [3, 4]]
deep = copy.deepcopy(original)
deep[0].append(99)
print(original[0])
```

Печатается `[1, 2]`: глубокая копия независима. Правило простое: поверхностной копии достаточно, пока внутри только неизменяемые объекты; для вложенных изменяемых структур берите `deepcopy`.

## Когда поведение удивит

Самая известная ловушка Python - изменяемый аргумент по умолчанию. Значение по умолчанию вычисляется один раз, в момент определения функции, а не при каждом вызове.

```python
def add_item(item, bucket=[]):
    bucket.append(item)
    return bucket

print(add_item("a"))
print(add_item("b"))
```

Печатается `['a']`, затем `['a', 'b']`. Список `bucket` создан единожды и живёт между вызовами, накапливая всё, что в него клали. Лекарство - значение по умолчанию `None`, а реальный список создавать внутри функции.

<Callout type="tip">Шаблон против этой ловушки: `def add_item(item, bucket=None):` и в теле `if bucket is None: bucket = []`. Так новый список рождается на каждом вызове.</Callout>

## Типичные ошибки

- **Считать, что присваивание копирует.** `b = a` для списка не делает копию - оба имени смотрят на один объект. Нужна копия - берите `a[:]`, `list(a)`, `a.copy()` или `copy.deepcopy(a)`.
- **Использовать `==` вместо `is` для `None`.** Технически `x == None` обычно работает, но это полагается на корректный `__eq__`; идиома - `x is None`.
- **Полагаться на `is` для чисел и строк.** `a is b` для двух `int` или `str` иногда `True`, иногда `False` - это зависит от интернирования (см. раздел про оптимизации). Для сравнения значений всегда `==`.

<Detail summary="Что показывает id() на самом деле">

`id()` возвращает целое число, гарантированно уникальное и постоянное для объекта на время его жизни. В CPython это адрес объекта в памяти, но язык этого не обещает: другая реализация (PyPy, Jython) вправе возвращать что угодно, лишь бы выполнялась гарантия уникальности и постоянства. Поэтому полагаться на конкретные значения `id()` нельзя - только на сравнение `id(a) == id(b)`, что и делает `is`. И учтите: после смерти объекта его `id` может быть переиспользован новым объектом, так что сравнивать сохранённые «старые» id бессмысленно.

</Detail>
