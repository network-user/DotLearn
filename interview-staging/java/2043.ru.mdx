# Что такое Scope в Maven

## 1️⃣ Как кратко ответить

Scope в Maven определяет видимость и доступность зависимости в различных фазах сборки проекта. Основные типы scope: `compile`, `provided`, `runtime`, `test`, `system` и `import`. Они управляют тем, как и когда зависимости доступны в проекте.

## 2️⃣ Подробное объяснение темы

В Maven, системе управления проектами и сборки, scope (область видимости) определяет, как и когда зависимости доступны в проекте. Это важно для управления зависимостями, чтобы избежать конфликтов и избыточности.

### Основные типы scope:

1. **compile**:

**Описание**: Это значение по умолчанию. Зависимости с этим scope доступны на всех этапах: компиляция, тестирование и выполнение.
2. **Применение**: Используется для библиотек, которые необходимы для компиляции и выполнения кода.
3. **Пример**: Библиотеки, которые являются частью основного функционала приложения.
4. **provided**:

**Описание**: Зависимости с этим scope необходимы для компиляции, но не включаются в финальный артефакт. Предполагается, что они будут предоставлены средой выполнения (например, сервером приложений).
5. **Применение**: Используется для библиотек, которые предоставляются контейнером или средой выполнения.
6. **Пример**: `javax.servlet` API для веб-приложений, которые будут развернуты на серверах приложений.
7. **runtime**:

**Описание**: Зависимости с этим scope не нужны для компиляции, но необходимы для выполнения и тестирования.
8. **Применение**: Используется для библиотек, которые требуются только во время выполнения.
9. **Пример**: JDBC драйверы, которые необходимы для подключения к базе данных во время выполнения.
10. **test**:

**Описание**: Зависимости с этим scope используются только для компиляции и выполнения тестов.
11. **Применение**: Используется для библиотек, которые необходимы только для тестирования.
12. **Пример**: JUnit или TestNG для написания и выполнения тестов.
13. **system**:

**Описание**: Похож на `provided`, но требует явного указания пути к JAR файлу. Не рекомендуется использовать, так как нарушает переносимость проекта.
14. **Применение**: Используется в редких случаях, когда зависимость не может быть загружена из репозитория.
15. **Пример**: Локальные библиотеки, которые не доступны в репозиториях.
16. **import**:

**Описание**: Используется только в управлении зависимостями (dependency management) для импорта зависимостей из другого POM файла.
17. **Применение**: Используется для импорта BOM (Bill of Materials) файлов, которые управляют версиями зависимостей.
18. **Пример**: Импорт BOM файла для управления версиями зависимостей в многомодульных проектах.

### Пример использования scope в POM файле:

```xml
<dependencies>
    <!-- Зависимость с областью видимости compile -->
    <dependency>
        <groupId>org.apache.commons</groupId>
        <artifactId>commons-lang3</artifactId>
        <version>3.12.0</version>
        <scope>compile</scope>
    </dependency>

    <!-- Зависимость с областью видимости provided -->
    <dependency>
        <groupId>javax.servlet</groupId>
        <artifactId>javax.servlet-api</artifactId>
        <version>4.0.1</version>
        <scope>provided</scope>
    </dependency>

    <!-- Зависимость с областью видимости runtime -->
    <dependency>
        <groupId>mysql</groupId>
        <artifactId>mysql-connector-java</artifactId>
        <version>8.0.26</version>
        <scope>runtime</scope>
    </dependency>

    <!-- Зависимость с областью видимости test -->
    <dependency>
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.13.2</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

- **`compile`**: Зависимость `commons-lang3` будет доступна на всех этапах сборки.
- **`provided`**: Зависимость `javax.servlet-api` будет использоваться только для компиляции, но не будет включена в финальный артефакт.
- **`runtime`**: Зависимость `mysql-connector-java` будет использоваться только во время выполнения.
- **`test`**: Зависимость `junit` будет использоваться только для тестирования.

Понимание и правильное использование scope в Maven помогает эффективно управлять зависимостями, минимизировать размер артефактов и избегать конфликтов версий.
