# Какие знаешь ключи в реляционных БД

## 1️⃣ Как кратко ответить

В реляционных базах данных ключи используются для обеспечения целостности данных и установления связей между таблицами. Основные типы ключей: первичный ключ (Primary Key), внешний ключ (Foreign Key), уникальный ключ (Unique Key) и составной ключ (Composite Key).

## 2️⃣ Подробное объяснение темы

Ключи в реляционных базах данных играют важную роль в обеспечении целостности данных и установлении связей между таблицами. Рассмотрим основные типы ключей:

1. **Первичный ключ (Primary Key):**

Это уникальный идентификатор для каждой записи в таблице.
2. Не может содержать NULL значений.
3. В таблице может быть только один первичный ключ.
4. Пример: В таблице `Users` столбец `user_id` может быть первичным ключом, так как он уникально идентифицирует каждого пользователя.
5. **Внешний ключ (Foreign Key):**

Используется для установления связи между двумя таблицами.
6. Ссылается на первичный ключ в другой таблице.
7. Обеспечивает целостность данных, гарантируя, что значения в одной таблице соответствуют значениям в другой.
8. Пример: В таблице `Orders` столбец `user_id` может быть внешним ключом, ссылающимся на `user_id` в таблице `Users`.
9. **Уникальный ключ (Unique Key):**

Гарантирует уникальность значений в одном или нескольких столбцах.
10. В отличие от первичного ключа, может содержать NULL значения.
11. В таблице может быть несколько уникальных ключей.
12. Пример: В таблице `Users` столбец `email` может быть уникальным ключом, чтобы гарантировать, что каждый адрес электронной почты используется только один раз.
13. **Составной ключ (Composite Key):**

Состоит из двух или более столбцов, которые вместе образуют уникальный идентификатор для записи.
14. Используется, когда один столбец не может уникально идентифицировать запись.
15. Пример: В таблице `CourseRegistrations` составной ключ может состоять из `student_id` и `course_id`, чтобы уникально идентифицировать каждую регистрацию.

Ключи в реляционных базах данных обеспечивают структурированность и целостность данных, что позволяет эффективно управлять и манипулировать данными в сложных системах.
