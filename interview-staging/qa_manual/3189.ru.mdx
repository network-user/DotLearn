# Для какого функционала делать UI тест-кейс

## 1️⃣ Как кратко ответить

UI тест-кейсы создаются для функционала, который напрямую взаимодействует с пользователем через интерфейс. Это включает в себя проверку корректности отображения элементов, функциональности кнопок, форм, навигации, а также проверку пользовательского опыта и доступности.

## 2️⃣ Подробное объяснение темы

UI тест-кейсы предназначены для проверки пользовательского интерфейса приложения. Они помогают убедиться, что интерфейс работает так, как ожидается, и предоставляет пользователю положительный опыт взаимодействия. Рассмотрим, для какого функционала стоит разрабатывать UI тест-кейсы:

1. **Отображение элементов интерфейса**:

Проверка правильности отображения всех элементов интерфейса, таких как кнопки, поля ввода, изображения и текст. Например, убедиться, что кнопка "Отправить" видна и расположена в правильном месте.
2. **Функциональность элементов**:

Проверка, что все интерактивные элементы, такие как кнопки и ссылки, работают корректно. Например, при нажатии на кнопку "Отправить" должна происходить отправка формы.
3. **Навигация**:

Убедиться, что навигация по приложению интуитивно понятна и работает без сбоев. Например, проверка, что при нажатии на логотип происходит переход на главную страницу.
4. **Валидация форм**:

Проверка, что формы корректно обрабатывают вводимые данные и отображают соответствующие сообщения об ошибках. Например, если поле "Электронная почта" оставлено пустым, должно появляться сообщение "Это поле обязательно для заполнения".
5. **Адаптивность и кросс-браузерность**:

Убедиться, что интерфейс корректно отображается на различных устройствах и в разных браузерах. Например, проверка, что сайт одинаково хорошо выглядит на мобильных устройствах и настольных компьютерах.
6. **Доступность**:

Проверка, что интерфейс доступен для пользователей с ограниченными возможностями. Например, убедиться, что все элементы интерфейса могут быть доступны с клавиатуры и имеют соответствующие атрибуты для экранных читалок.
7. **Пользовательский опыт (UX)**:

Оценка удобства использования интерфейса. Например, проверка, что процесс регистрации нового пользователя прост и понятен.

Пример простого UI тест-кейса для проверки кнопки "Отправить" на форме:

```plaintext
Название: Проверка кнопки "Отправить" на форме обратной связи

Предусловия:
- Пользователь находится на странице с формой обратной связи.

Шаги:
1. Ввести корректные данные в поля формы.
2. Нажать на кнопку "Отправить".

Ожидаемый результат:
- Данные успешно отправляются.
- Пользователь видит сообщение об успешной отправке.

Комментарии:
- Убедиться, что кнопка "Отправить" активна только при заполнении всех обязательных полей.
- Проверить, что после отправки формы пользователь перенаправляется на страницу подтверждения.
```

UI тест-кейсы важны для обеспечения качества пользовательского интерфейса, так как они помогают выявить и устранить проблемы, которые могут негативно повлиять на пользовательский опыт.
