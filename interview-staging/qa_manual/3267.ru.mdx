# Какие объекты баз данных представляют собой сохраненный код на SQL

## 1️⃣ Как кратко ответить

Сохраненный код на SQL в базах данных представлен объектами, такими как хранимые процедуры, функции, триггеры и представления. Эти объекты позволяют выполнять повторяющиеся задачи, автоматизировать операции и обеспечивать целостность данных.

## 2️⃣ Подробное объяснение темы

В реляционных базах данных SQL (Structured Query Language) используется для управления и манипуляции данными. Некоторые объекты баз данных позволяют сохранять и выполнять код на SQL, что упрощает автоматизацию и управление данными. Рассмотрим основные из них:

1. **Хранимые процедуры (Stored Procedures):**

**Что это:** Это набор SQL-инструкций, которые сохраняются в базе данных и могут быть выполнены по запросу. Они могут принимать входные параметры и возвращать выходные значения.
2. **Зачем нужны:** Хранимые процедуры используются для выполнения сложных операций, которые могут включать несколько SQL-запросов. Они помогают уменьшить количество сетевых взаимодействий между приложением и базой данных, так как вся логика выполняется на стороне сервера.
3. **Пример:**```sql
CREATE PROCEDURE GetEmployeeDetails
@EmployeeID INT
AS
BEGIN
    SELECT * FROM Employees WHERE EmployeeID = @EmployeeID;
END;
```

`CREATE PROCEDURE GetEmployeeDetails`: Создает новую хранимую процедуру с именем `GetEmployeeDetails`.
4. `@EmployeeID INT`: Определяет входной параметр `EmployeeID` типа `INT`.
5. `SELECT * FROM Employees WHERE EmployeeID = @EmployeeID;`: Выполняет выборку данных из таблицы `Employees`, где `EmployeeID` соответствует переданному параметру.
6. **Функции (Functions):**

**Что это:** Это объекты, которые выполняют вычисления и возвращают одно значение. В отличие от процедур, функции всегда возвращают значение и могут быть использованы в SQL-запросах.
7. **Зачем нужны:** Функции полезны для выполнения повторяющихся вычислений или преобразований данных, которые можно использовать в различных частях SQL-запросов.
8. **Пример:**```sql
CREATE FUNCTION GetFullName
(@FirstName NVARCHAR(50), @LastName NVARCHAR(50))
RETURNS NVARCHAR(101)
AS
BEGIN
    RETURN @FirstName + ' ' + @LastName;
END;
```

`CREATE FUNCTION GetFullName`: Создает новую функцию с именем `GetFullName`.
9. `@FirstName NVARCHAR(50), @LastName NVARCHAR(50)`: Определяет входные параметры `FirstName` и `LastName`.
10. `RETURNS NVARCHAR(101)`: Указывает, что функция возвращает строку типа `NVARCHAR` длиной до 101 символа.
11. `RETURN @FirstName + ' ' + @LastName;`: Возвращает полное имя, объединяя `FirstName` и `LastName`.
12. **Триггеры (Triggers):**

**Что это:** Это специальные виды хранимых процедур, которые автоматически выполняются в ответ на определенные события в таблице, такие как `INSERT`, `UPDATE` или `DELETE`.
13. **Зачем нужны:** Триггеры используются для автоматизации задач, таких как ведение журнала изменений, проверка целостности данных или выполнение каскадных операций.
14. **Пример:**```sql
CREATE TRIGGER trgAfterInsert
ON Employees
AFTER INSERT
AS
BEGIN
    INSERT INTO EmployeeLog (EmployeeID, Action)
    SELECT EmployeeID, 'INSERT' FROM inserted;
END;
```

`CREATE TRIGGER trgAfterInsert`: Создает новый триггер с именем `trgAfterInsert`.
15. `ON Employees AFTER INSERT`: Указывает, что триггер срабатывает после вставки (`INSERT`) в таблицу `Employees`.
16. `INSERT INTO EmployeeLog (EmployeeID, Action)`: Вставляет данные в таблицу `EmployeeLog`.
17. `SELECT EmployeeID, 'INSERT' FROM inserted;`: Выбирает `EmployeeID` из специальной таблицы `inserted` и добавляет запись о действии `INSERT`.
18. **Представления (Views):**

**Что это:** Это виртуальные таблицы, которые представляют собой результат выполнения SQL-запроса. Представления не хранят данные, а отображают их из одной или нескольких таблиц.
19. **Зачем нужны:** Представления упрощают сложные запросы, обеспечивают безопасность данных, скрывая определенные столбцы, и предоставляют пользователям упрощенный интерфейс для работы с данными.
20. **Пример:**```sql
CREATE VIEW EmployeeView AS
SELECT EmployeeID, FirstName, LastName
FROM Employees
WHERE IsActive = 1;
```

`CREATE VIEW EmployeeView AS`: Создает новое представление с именем `EmployeeView`.
21. `SELECT EmployeeID, FirstName, LastName FROM Employees`: Определяет, какие столбцы и из какой таблицы будут отображаться в представлении.
22. `WHERE IsActive = 1;`: Фильтрует данные, показывая только активных сотрудников.

Эти объекты баз данных позволяют разработчикам и администраторам эффективно управлять данными, автоматизировать процессы и обеспечивать целостность и безопасность данных в системе.
