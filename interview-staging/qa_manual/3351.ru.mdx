# Как добавить токен авторизации в Postman

## 1️⃣ Как кратко ответить

В Postman добавление токена авторизации осуществляется через вкладку "Authorization" в настройках запроса. Выберите тип авторизации "Bearer Token" и введите ваш токен в соответствующее поле. Это обеспечит автоматическую отправку токена в заголовке каждого запроса.

## 2️⃣ Подробное объяснение темы

Postman — это инструмент для тестирования API, который позволяет отправлять HTTP-запросы и получать ответы. Для доступа к защищенным ресурсам API часто требуется авторизация, которая может быть реализована с помощью токенов. Один из распространенных методов — использование Bearer Token.

**Зачем это нужно?**

Токены авторизации используются для подтверждения прав доступа пользователя к ресурсам API. Это позволяет API различать пользователей и предоставлять доступ только тем, кто имеет соответствующие права.

**Как это работает в Postman?**

1. **Открытие Postman и выбор запроса:**

Запустите Postman и выберите коллекцию или создайте новый запрос, к которому вы хотите добавить токен авторизации.
2. **Переход на вкладку "Authorization":**

В интерфейсе Postman выберите вкладку "Authorization". Это место, где вы можете настроить параметры авторизации для вашего запроса.
3. **Выбор типа авторизации:**

В выпадающем списке "Type" выберите "Bearer Token". Этот тип авторизации используется для передачи токена в заголовке HTTP-запроса.
4. **Ввод токена:**

В поле "Token" введите ваш токен. Это может быть строка, полученная от сервера после успешной аутентификации.
5. **Отправка запроса:**

После настройки авторизации отправьте запрос. Postman автоматически добавит заголовок `Authorization` с вашим токеном в каждый запрос, отправляемый к API.

**Пример:**

Предположим, у вас есть API, который требует Bearer Token для доступа к защищенным данным. Ваш токен выглядит как `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`.

- Откройте Postman и создайте новый GET-запрос к вашему API, например, `https://api.example.com/data`.
- Перейдите на вкладку "Authorization".
- Выберите "Bearer Token" в качестве типа авторизации.
- Введите `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...` в поле "Token".
- Нажмите "Send", чтобы отправить запрос.

**Что происходит под капотом:**

Когда вы отправляете запрос, Postman добавляет заголовок `Authorization` с вашим токеном:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

Этот заголовок передается серверу, который проверяет токен и, если он действителен, предоставляет доступ к запрашиваемым ресурсам.

Использование токенов авторизации в Postman упрощает процесс тестирования API, обеспечивая безопасность и контроль доступа к данным.
