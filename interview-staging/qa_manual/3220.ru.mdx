# Какие знаешь критерии завершения тест кейса

## 1️⃣ Как кратко ответить

Критерии завершения тест кейса включают: успешное выполнение всех шагов теста, достижение ожидаемого результата, отсутствие критических дефектов, подтверждение выполнения всех пред- и постусловий, а также документирование результатов тестирования.

## 2️⃣ Подробное объяснение темы

Критерии завершения тест кейса — это условия, которые должны быть выполнены, чтобы тест кейс считался завершенным. Они помогают определить, когда тестирование конкретного сценария можно считать полным и успешным. Рассмотрим основные критерии:

1. **Успешное выполнение всех шагов теста**: Каждый тест кейс состоит из последовательности шагов, которые необходимо выполнить. Завершение тест кейса подразумевает, что все шаги были выполнены в соответствии с описанием.
2. **Достижение ожидаемого результата**: Для каждого тест кейса определен ожидаемый результат. Тест считается завершенным, если фактический результат совпадает с ожидаемым. Это подтверждает, что тестируемая функциональность работает корректно.
3. **Отсутствие критических дефектов**: Если в процессе выполнения тест кейса обнаружены дефекты, они должны быть оценены. Тест кейс может считаться завершенным, если дефекты не являются критическими и не препятствуют дальнейшему тестированию или использованию системы.
4. **Подтверждение выполнения всех пред- и постусловий**: Перед началом тестирования должны быть выполнены все предусловия, а после завершения — все постусловия. Это гарантирует, что тестирование проводилось в корректных условиях и результаты можно считать достоверными.
5. **Документирование результатов тестирования**: Все результаты тестирования должны быть задокументированы. Это включает в себя запись фактических результатов, любых отклонений от ожидаемого поведения и выявленных дефектов. Документация позволяет анализировать результаты и принимать обоснованные решения о дальнейшем тестировании.

Пример:

```plaintext
Тест кейс: Проверка авторизации пользователя

Предусловия:
- Пользователь зарегистрирован в системе.
- Система доступна для тестирования.

Шаги:
1. Открыть страницу авторизации.
2. Ввести корректный логин и пароль.
3. Нажать кнопку "Войти".

Ожидаемый результат:
- Пользователь успешно авторизован и перенаправлен на главную страницу.

Постусловия:
- Пользователь остается авторизованным в системе.

Критерии завершения:
- Все шаги выполнены.
- Ожидаемый результат достигнут.
- Критических дефектов не обнаружено.
- Все предусловия и постусловия выполнены.
- Результаты тестирования задокументированы.
```

Этот пример иллюстрирует, как критерии завершения применяются на практике. Они обеспечивают уверенность в том, что тестирование проведено полноценно и результаты можно использовать для принятия решений о качестве продукта.
