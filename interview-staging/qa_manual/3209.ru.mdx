# Как фиксируется выполнение тест-кейса: статус, комментарий, фактический результат, вложения

## 1️⃣ Как кратко ответить

Выполнение тест-кейса фиксируется через статус (например, Passed, Failed, Blocked), комментарий для пояснений, фактический результат, который сравнивается с ожидаемым, и вложения (скриншоты, логи) для документирования и анализа.

## 2️⃣ Подробное объяснение темы

Фиксация выполнения тест-кейса — это важный процесс в тестировании, который позволяет отслеживать и документировать результаты тестирования. Это необходимо для понимания текущего состояния тестируемого продукта и для принятия решений о его качестве. Рассмотрим основные элементы фиксации выполнения тест-кейса:

1. **Статус тест-кейса**:

**Passed**: Тест-кейс выполнен успешно, фактический результат соответствует ожидаемому.
2. **Failed**: Тест-кейс не выполнен, фактический результат не соответствует ожидаемому.
3. **Blocked**: Тест-кейс не может быть выполнен из-за внешних причин, например, недоступности тестируемой функции.
4. **Not Executed**: Тест-кейс не был выполнен по каким-либо причинам.
5. **Комментарий**:

Комментарии используются для предоставления дополнительной информации о выполнении тест-кейса. Например, если тест-кейс провален, в комментарии можно указать причину, предположения о возможных дефектах или условия, при которых тест-кейс не прошел.
6. Пример: "Тест-кейс провален из-за ошибки валидации на странице входа."
7. **Фактический результат**:

Это описание того, что произошло в ходе выполнения тест-кейса. Фактический результат сравнивается с ожидаемым, чтобы определить, прошел ли тест-кейс.
8. Пример: Ожидаемый результат — "Пользователь успешно авторизован", фактический результат — "Ошибка 500 на сервере".
9. **Вложения**:

Вложения включают скриншоты, логи, видео или другие файлы, которые помогают документировать выполнение тест-кейса. Они особенно полезны для анализа причин провала тест-кейса.
10. Пример: Скриншот экрана с ошибкой или лог-файл, содержащий информацию о сбое.

Пример фиксации выполнения тест-кейса в тестовой системе:

```plaintext
Тест-кейс: TC-001
Статус: Failed
Комментарий: Ошибка валидации на странице входа
Фактический результат: Ошибка 500 на сервере
Вложения: 
  - Screenshot_2023-10-01.png
  - server_log_2023-10-01.txt
```

Каждый из этих элементов играет важную роль в процессе тестирования, обеспечивая прозрачность и возможность анализа результатов тестирования. Это помогает команде быстро реагировать на проблемы и улучшать качество продукта.
