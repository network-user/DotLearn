# Как установить тестовую сборку на iOS/Android (TestFlight, .apk/.aab, enterprise, ad-hoc)

## 1️⃣ Как кратко ответить

Для установки тестовой сборки на iOS через TestFlight необходимо получить приглашение на тестирование и установить приложение через TestFlight. Для Android используйте .apk или .aab файл, который можно установить напрямую на устройство, разрешив установку из неизвестных источников. Для iOS enterprise и ad-hoc сборок требуется установка профиля конфиденциальности и доверие сертификату разработчика.

## 2️⃣ Подробное объяснение темы

Установка тестовых сборок на мобильные устройства позволяет QA-инженерам и тестировщикам проверять приложения до их официального релиза. Существуют различные методы установки тестовых сборок в зависимости от платформы и типа сборки.

### iOS

### TestFlight

TestFlight — это сервис от Apple для распространения бета-версий приложений.

1. **Приглашение на тестирование**: Разработчик отправляет приглашение на тестирование через TestFlight. Вы получите электронное письмо с приглашением или код для доступа.
2. **Установка TestFlight**: Установите приложение TestFlight из App Store.
3. **Установка приложения**: Откройте TestFlight, примите приглашение и установите приложение.

### Enterprise и Ad-hoc

Эти методы позволяют устанавливать приложения без публикации в App Store.

- **Enterprise**: Используется для распространения приложений внутри компании. Требуется корпоративный сертификат.
- **Ad-hoc**: Позволяет устанавливать приложение на ограниченное количество устройств, указанных в профиле.

1. **Получение .ipa файла**: Разработчик предоставляет .ipa файл и профиль конфиденциальности.
2. **Установка профиля**: Установите профиль конфиденциальности на устройстве.
3. **Доверие сертификату**: Перейдите в "Настройки" > "Основные" > "Управление устройством" и доверьте сертификату разработчика.
4. **Установка приложения**: Установите .ipa файл через iTunes или сторонние инструменты.

### Android

### .apk/.aab файлы

.apk и .aab — это форматы файлов для установки приложений на Android.

1. **Получение файла**: Разработчик предоставляет .apk или .aab файл.
2. **Разрешение установки из неизвестных источников**: Перейдите в "Настройки" > "Безопасность" и разрешите установку из неизвестных источников.
3. **Установка файла**: Откройте файл на устройстве и следуйте инструкциям для установки.

### Пример установки .apk файла

```java
// Пример кода для установки .apk файла через ADB (Android Debug Bridge)

// Подключите устройство к компьютеру и убедитесь, что ADB установлен и настроен

// Убедитесь, что устройство подключено и ADB распознает его
adb devices

// Установите .apk файл на устройство
adb install path/to/your/app.apk
```

- `adb devices`: Проверяет, подключено ли устройство и распознается ли оно ADB.
- `adb install path/to/your/app.apk`: Устанавливает .apk файл на подключенное устройство.

Эти методы позволяют QA-инженерам и тестировщикам устанавливать и тестировать приложения на различных стадиях разработки, обеспечивая качество и функциональность перед официальным релизом.
