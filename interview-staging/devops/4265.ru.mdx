# Какие типы адресов используются на сетевом уровне модели OSI

## 1️⃣ Как кратко ответить

На сетевом уровне модели OSI используются логические адреса, такие как IP-адреса. Эти адреса позволяют идентифицировать устройства в сети и обеспечивают маршрутизацию данных между различными сетями.

## 2️⃣ Подробное объяснение темы

Сетевой уровень модели OSI отвечает за маршрутизацию данных между различными сетями. Для этого он использует логические адреса, которые позволяют уникально идентифицировать каждое устройство в сети. Основной тип логического адреса, используемого на этом уровне, — это IP-адрес.

### Логические адреса

Логические адреса, такие как IP-адреса, отличаются от физических адресов (например, MAC-адресов), которые используются на канальном уровне. Логические адреса назначаются программно и могут изменяться в зависимости от сетевой конфигурации.

### IP-адреса

IP-адреса — это основной тип адресов, используемых на сетевом уровне. Они бывают двух версий: IPv4 и IPv6.

- **IPv4**: Состоит из 32 бит и обычно записывается в виде четырех десятичных чисел, разделенных точками (например, 192.168.1.1). IPv4 предоставляет около 4,3 миллиарда уникальных адресов.
- **IPv6**: Состоит из 128 бит и записывается в виде восьми групп шестнадцатеричных чисел, разделенных двоеточиями (например, 2001:0db8:85a3:0000:0000:8a2e:0370:7334). IPv6 решает проблему нехватки адресов в IPv4, предоставляя огромное количество уникальных адресов.

### Зачем нужны логические адреса

Логические адреса необходимы для маршрутизации данных между различными сетями. Они позволяют маршрутизаторам определять, куда отправить пакеты данных, чтобы они достигли нужного устройства. Без логических адресов маршрутизация была бы невозможна, и устройства не могли бы обмениваться данными за пределами локальной сети.

### Пример использования IP-адресов

Рассмотрим пример, где компьютер в одной сети отправляет данные на сервер в другой сети.

1. Компьютер отправляет данные с указанием IP-адреса сервера.
2. Маршрутизатор в локальной сети компьютера получает данные и использует IP-адрес для определения следующего маршрута.
3. Данные проходят через несколько маршрутизаторов, каждый из которых использует IP-адрес для определения следующего шага.
4. В конечном итоге данные достигают маршрутизатора в сети сервера, который передает их на сервер.

Этот процесс показывает, как IP-адреса позволяют устройствам в разных сетях обмениваться данными, обеспечивая глобальную связность в Интернете.
