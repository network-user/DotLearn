# Какими инструментами пользуешься для настройки CI/CD

## 1️⃣ Как кратко ответить

Для настройки CI/CD я использую Jenkins, GitLab CI/CD, и GitHub Actions. Эти инструменты позволяют автоматизировать сборку, тестирование и развертывание приложений, обеспечивая быструю и надежную доставку программного обеспечения.

## 2️⃣ Подробное объяснение темы

CI/CD (Continuous Integration/Continuous Deployment) — это практика, которая позволяет автоматизировать процесс интеграции и развертывания программного обеспечения. Она помогает командам разработчиков быстрее и надежнее доставлять изменения в коде, минимизируя риск ошибок и упрощая процесс выпуска новых версий.

### Инструменты для настройки CI/CD

1. **Jenkins**

**Что это:** Jenkins — это open-source сервер автоматизации, который поддерживает множество плагинов для построения, развертывания и автоматизации любого проекта.
2. **Как работает:** Jenkins позволяет создавать конвейеры (pipelines), которые автоматизируют процесс сборки, тестирования и развертывания. Конвейеры могут быть настроены с помощью Jenkinsfile, который описывает шаги процесса.
3. **Пример использования:**```groovy
pipeline {
    agent any
    stages {
        stage('Build') {
            steps {
                echo 'Building...'
                // Команды для сборки проекта
            }
        }
        stage('Test') {
            steps {
                echo 'Testing...'
                // Команды для запуска тестов
            }
        }
        stage('Deploy') {
            steps {
                echo 'Deploying...'
                // Команды для развертывания
            }
        }
    }
}
```

`pipeline {}`: Определяет начало конвейера.
4. `agent any`: Указывает, что конвейер может выполняться на любом доступном агенте.
5. `stages {}`: Содержит набор этапов, таких как сборка, тестирование и развертывание.
6. `stage('...') {}`: Определяет отдельный этап в конвейере.
7. `steps {}`: Содержит команды, которые будут выполнены на данном этапе.
8. **GitLab CI/CD**

**Что это:** Интегрированное решение для CI/CD, встроенное в GitLab. Оно позволяет автоматизировать процесс сборки, тестирования и развертывания прямо из репозитория.
9. **Как работает:** GitLab CI/CD использует файл `.gitlab-ci.yml`, который хранится в корне репозитория и описывает все этапы конвейера.
10. **Пример использования:**```yaml
stages:
  - build
  - test
  - deploy

build_job:
  stage: build
  script:
    - echo "Building the project"
    # Команды для сборки проекта

test_job:
  stage: test
  script:
    - echo "Running tests"
    # Команды для запуска тестов

deploy_job:
  stage: deploy
  script:
    - echo "Deploying the project"
    # Команды для развертывания
```

`stages:`: Определяет последовательность этапов.
11. `build_job:`, `test_job:`, `deploy_job:`: Определяют отдельные задания для каждого этапа.
12. `stage:`: Указывает, к какому этапу относится задание.
13. `script:`: Содержит команды, которые будут выполнены в рамках задания.
14. **GitHub Actions**

**Что это:** Платформа автоматизации от GitHub, которая позволяет создавать рабочие процессы для автоматизации задач, связанных с разработкой.
15. **Как работает:** Рабочие процессы описываются в файлах YAML, которые хранятся в директории `.github/workflows` вашего репозитория.
16. **Пример использования:**```yaml
name: CI

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Set up Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '14'
      - name: Install dependencies
        run: npm install
      - name: Run tests
        run: npm test
```

`name:`: Имя рабочего процесса.
17. `on:`: Определяет события, которые запускают рабочий процесс (например, push или pull_request).
18. `jobs:`: Содержит набор заданий, которые будут выполнены.
19. `build:`: Имя задания.
20. `runs-on:`: Указывает, на какой платформе будет выполняться задание.
21. `steps:`: Содержит шаги, которые будут выполнены в рамках задания.
22. `uses:`: Указывает на использование готового действия из GitHub Marketplace.
23. `run:`: Определяет команды, которые будут выполнены.

Эти инструменты помогают автоматизировать и ускорить процесс разработки, обеспечивая более частые и надежные релизы.
